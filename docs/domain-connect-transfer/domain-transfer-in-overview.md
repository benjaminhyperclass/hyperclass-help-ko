## 도메인 이전이란?

도메인 이전(Domain Transfer In)은 다른 등록업체(예: GoDaddy, Namecheap, IONOS, Cloudflare Registrar)에서 등록한 도메인을 Hyperclass 플랫폼으로 옮기는 기능입니다. 이전 완료 후에는 DNS, 결제, 갱신, 보안을 한 곳에서 관리할 수 있어서 더 이상 여러 업체 간을 오갈 필요가 없습니다.

**도메인을 이전하는 이유는?**
중앙 집중식 관리, 갱신 깜빡함 방지, 더 원활한 문제 해결, 그리고 비즈니스의 안정성과 연속성을 위해서입니다.

---

## 이전 과정 한눈에 보기

- **도메인 입력** → 이전할 도메인 알려주기
- **이전 가능 여부 확인** → 이전 가능한지 확인
- **DNS 레코드 검토** → 모든 DNS 설정을 업로드/스캔하여 검증
- **이전 완료** → 잠금 해제, 인증(EPP) 코드 입력, 결제
- **현재 등록업체에서 승인** → 이전 완료를 위한 승인 확인

**소요 시간:** 대부분의 이전은 현재 등록업체에서 승인한 후 **1-5일** 내에 완료됩니다.

---

## 시작하기 전 체크리스트

- 해당 도메인의 **현재 등록업체** 계정에 로그인할 수 있음
- 도메인이 **60일 이상** 등록되어 있고 최근에 이전되지 않음
- **DNSSEC**와 **privacy/whois 보호** 기능을 일시적으로 비활성화할 수 있음
- 도메인을 **잠금 해제**하고 **인증(EPP) 코드**를 받을 수 있음
- **DNS 제공업체**에 접근할 수 있음 (등록업체와 다를 수 있음)

**중요:** **등록업체**는 도메인 등록 정보를 관리하고, **DNS 제공업체**는 DNS 레코드를 호스팅합니다. 과거에 네임서버를 변경한 경우(예: Cloudflare로), DNS 제공업체는 해당 네임서버가 가리키는 곳이며 **반드시 등록업체와 같지는 않습니다**.

## 2단계: 이전 가능 여부 확인

시스템이 자동으로 등록업체 규칙과 기술적 준비 상태를 확인합니다.

**이전 가능한 경우 다음을 확인할 수 있습니다:**

- **도메인 이름**과 **현재 등록업체**
- **현재 만료일**과 **새로운 만료일** (이전 시 1년 연장됨)
- **이전 수수료** (만료일 연장에 적용됨)

**다음의 경우 이전할 수 없습니다:**

- **현재 등록업체** 또는 **TLD**가 지원되지 않음
- **이곳에서 원래 구매**한 도메인
- **지난 60일 내에 등록/이전**된 도메인
- 도메인의 **EPP 상태**가 이전을 차단함 (예: pendingDelete, redemptionPeriod, serverHold)
- 이전 시 **최대 허용 기간**을 초과하게 됨
- 확인 중 발견된 기타 정책 충돌

이전 불가능한 도메인은 **Continue(계속)** 버튼이 비활성화되고 이유를 설명하는 메시지가 표시됩니다.

## 3단계: DNS 레코드 검토 (다운타임 방지를 위한 중요 단계)

![DNS 레코드 검토 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057614556/original/wc9wNujdc4hjo11vylZqNHABX0boP5ReqQ.png?1762338830)

도메인 이전은 DNS 레코드를 **자동으로 마이그레이션하지 않습니다**. 웹사이트, 이메일, 연동이 계속 작동하도록 필요한 모든 레코드가 있는지 확인해야 합니다.

레코드를 가져오는 두 가지 방법이 있습니다:

### 옵션 A: DNS 존 파일 업로드 (권장)

완전하고 손실 없는 가져오기에 가장 좋습니다.

- **DNS 제공업체에 로그인** (등록업체이거나 네임서버를 변경한 경우 Cloudflare 같은 다른 호스트일 수 있음)
- 도메인의 **존 파일**을 내보내기/다운로드
- DNS 검토 단계에서 파일을 **업로드**

**팁:** 내보내기 옵션을 찾을 수 없다면 제공업체 지원팀에 연락해서 도메인의 **존 파일 내보내기**를 요청하세요.

### 옵션 B: DNS 레코드 자동 스캔

실제 DNS를 스캔합니다. 빠르지만 **일부 레코드가 누락될 수 있습니다** (예: 내부/숨겨진 레코드나 특수 제공업체 타입).

- 스캔 결과를 DNS 제공업체의 레코드와 **항상** 교차 확인하세요
- 누락된 **A, AAAA, CNAME, MX, TXT, SRV, CAA** 레코드를 수동으로 추가하세요

### 검토 및 편집

DNS 테이블에서 다음을 할 수 있습니다:

- 타입별로 그룹화된 모든 항목 보기
- 레코드 **추가**, **편집**, **삭제**
- 이름/타입/값으로 검색

![DNS 레코드 편집 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057614630/original/UO2rkeEr-gFB6eqXkf9bccJHgYGkmwJ1cQ.png?1762338882)

**경고:** 누락되거나 잘못된 DNS 레코드는 **웹사이트 다운타임**과 **이메일 전송 문제**를 일으킬 수 있습니다. 진행하기 전에 수동으로 확인하세요.

## 4단계: 이전 완료

![이전 완료 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057614824/original/euTbdy_quv33c7UJz9gvIObPTiSFA6H4dg.png?1762338982)

이 단계에서 인증과 결제를 완료합니다.

### 4.1 DNSSEC 및 Privacy (Whois) 비활성화

- **현재 등록업체**에 로그인
- **Domain(도메인)** 또는 **DNS** 설정으로 이동
- **DNSSEC 비활성화**
- **Whois/Privacy 보호** 기능 끄기
- 변경사항 저장

**참고:** DNSSEC 제거는 전파되는 데 몇 분이 걸릴 수 있습니다.

### 4.2 도메인 잠금 해제 및 인증 코드(EPP) 받기

- 등록업체 계정에서 **Domain Management(도메인 관리)** 열기
- 도메인 잠금(**Registrar Lock**) **비활성화**
- **Authorization/EPP/Transfer** 코드를 찾아서 복사

일부 등록업체는 화면에 표시하고, 다른 곳은 등록자 이메일로 보내줍니다.

### 4.3 검증 및 제출

이전 화면에서 다음을 진행합니다:

- **잠금 해제 상태**, **인증 코드**, 레지스트리 수준 상태 검증
- 이전 수수료로 **지갑에서 결제** (1년 연장에 적용됨)
- 이전 요청 **제출**

## 5단계: 현재 등록업체에서 승인 (필수)

이전은 현재 등록업체에서 **승인**할 때까지 **대기** 상태로 남아있습니다.

**승인 방법:**

- **이메일 승인:** 등록업체에서 보낸 링크 클릭
- **대시보드 승인:** 등록업체의 이전 센터에서 승인

**아무 조치를 취하지 않으면:** 많은 등록업체가 **5-7일** 후 자동 승인합니다. 즉시 승인하면 보통 더 빨라집니다.

---

- DNS, SSL, 갱신 등을 위한 **Manage(관리)** 기능 이용 가능

## 문제 해결 및 일반적인 오류

- **도메인 잠금을 해제할 수 없어요**
Security/Transfer(보안/이전) 설정에서 찾아보거나 등록업체 지원팀에 문의하세요.

- **EPP 코드가 틀렸거나 유효하지 않아요**
등록업체에서 새로운 코드를 요청하고 도메인이 **잠금 해제** 상태인지 확인하세요.

- **이전이 거부됐어요**
가능한 원인: 직접 취소, 60일 잠금 시행, 또는 시간 내 승인 안 함.
결과: 상태가 **Rejected(거부됨)**로 표시되고, **Transfer to Us(저희에게 이전)** 버튼으로 재시도 가능.

- **이전 후 다운타임이 발생해요**
대부분 누락되거나 일치하지 않는 DNS 레코드 때문입니다. 저희 DNS 테이블과 DNS 제공업체의 레코드를 비교해서 차이점을 수정하세요.

## 자주 묻는 질문

1. **이전 시 다운타임이 발생하나요?**
완료 전에 DNS 레코드를 검증하고 완전히 준비했다면 발생하지 않습니다. 항상 교차 확인하세요.

2. **등록업체와 DNS 제공업체가 다른데 어떻게 해야 하나요?**
**DNS 제공업체**(네임서버가 가리키는 곳)에 로그인해서 DNS 레코드를 내보내기/비교하세요. 등록업체에는 전체 DNS 목록이 없습니다.

3. **이전은 얼마나 걸리나요?**
현재 등록업체에서 *승인한 후* 보통 **1-5일**입니다.

4. **나중에 다시 이전할 수 있나요?**
가능하지만 ICANN은 보통 이전 후 **60일** 대기를 요구합니다.

5. **1년 연장되는 건 무엇인가요?**
이전 완료 시 **도메인의 만료일**이 연장됩니다 (최대 기간 규칙 적용).
