## 개요: 도메인이란 무엇인가요?

도메인은 사람들이 브라우저에 입력해서 당신의 온라인 서비스에 접속하는 웹 주소입니다(예: yourbrand.com). 저희 플랫폼에서 도메인을 사용하면 퍼널, 웹사이트, 블로그, 웨비나, 클라이언트 포털, 이메일 커뮤니케이션 등 다양한 도구에 브랜딩을 적용할 수 있습니다.

### DNS 레코드란 무엇인가요?

DNS 레코드(Domain Name System records)는 인터넷의 필수 구성 요소입니다. 마치 디지털 전화번호부처럼, 사람이 읽기 쉬운 도메인 이름(예: yourbrand.com)을 컴퓨터가 이해할 수 있는 IP 주소로 변환하여 브라우저와 서비스가 도메인이나 서브도메인에 접근하는 방법을 알려줍니다.

## 저희 플랫폼에서 도메인으로 할 수 있는 일

모든 도메인 연결을 한 곳에서 관리할 수 있어 명확한 가시성과 간편한 설정을 보장합니다.

![도메인 관리 대시보드 이미지](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047258561/original/tvuBWFXDe2WbCM1yUj4uSJxV4undgcJvSg.jpeg?1748329124)

### 퍼널 / 웹사이트 / 스토어 / 웨비나 / 블로그

도메인을 사용하여 퍼널, 이커머스 스토어, 웨비나, 블로그 등 고객 대상 페이지를 호스팅할 수 있습니다.

### 클라이언트 포털

커스텀 도메인을 사용하여 고객 전용 대시보드를 호스팅할 수 있습니다. 고객이 에셋, 캠페인, 리포트에 접근할 때 브랜딩된 경험을 제공합니다.

### 브랜딩 도메인 (화이트라벨용)

도메인을 연결하여 하위 계정이나 SaaS 기능을 화이트라벨로 만들 수 있습니다. 이렇게 하면 기본 URL이 당신의 브랜딩으로 대체됩니다.

### 워드프레스

저희 워드프레스 호스팅을 사용하는 경우, 제공된 DNS 레코드를 사용하여 도메인을 워드프레스 사이트로 연결할 수 있습니다. DNS 설정이 완료되면 시스템이 자동으로 감지하고 검증합니다.

### 이메일

이메일 전송 인증(예: SPF, DKIM, DMARC)을 위해 도메인이 필요합니다. 도메인을 연결하면 이메일 전달률이 향상되고 메시지가 스팸으로 분류되지 않도록 보장합니다.

## 시작하기 전에

### 필요한 것들:

- 활성 계정
- 도메인 (다음 중 하나):
  - 플랫폼 내에서 구매
  - 제3자 등록업체를 통해 구매 (예: Namecheap, GoDaddy)
- 도메인의 DNS 설정에 대한 접근 권한 (수동 설정 시 필요)

## 도메인 구매: 어디서 시작할까요?

도메인을 연결하기 전에 먼저 도메인을 소유해야 합니다. 도메인을 구매하는 방법은 두 가지입니다:

### 1. 플랫폼에서 직접 구매

- 플랫폼 내에서 완전히 처리되는 가장 쉬운 옵션입니다.
- Settings(설정) > Domains(도메인) > Purchase Domain(도메인 구매)로 이동합니다.
- 원하는 도메인 이름을 검색하고 구매를 완료합니다.
- 내부에서 구매한 도메인은 자동으로 Internal(내부)로 라벨링되며 내장된 DNS 관리 및 SSL 기능이 제공됩니다.

### 2. 제3자 등록업체에서 구매

- GoDaddy, Namecheap, Google Domains 등의 제공업체를 사용합니다.
- 구매 후, DNS 레코드를 업데이트하여 도메인을 수동으로 연결해야 합니다.
- 시스템에서 External Domains(외부 도메인)으로 분류됩니다.

**팁**: 외부에서 구매하는 경우 등록업체의 DNS 설정에 접근할 수 있는지 확인하세요. 설정 과정에서 필요합니다.

## 중앙 집중식 도메인 관리

내부 도메인과 외부 도메인(제3자 제공업체를 통해 연결) 모두 각 하위 계정의 중앙 집중식 도메인 설정 페이지에서 관리할 수 있습니다.

### 할 수 있는 작업

이 페이지에서 다음과 같은 작업을 할 수 있습니다:

- 도메인 또는 서브도메인 추가
- 연결된 상품(퍼널, 웹사이트, 이메일 등) 조회
- 도메인 및 SSL 상태 확인
- DNS 설정 접근 (내부 도메인의 경우)
- 필요한 조치 식별 및 해결 (예: 검증, 연결 문제)

도메인은 명확하게 라벨링됩니다:

- **Internal Domains(내부 도메인)** – 저희 플랫폼을 통해 구매
- **External Domains(외부 도메인)** – 외부 제공업체에서 연결

## 도메인 관리

- Connected Products(연결된 상품)와 DNS Records(DNS 레코드) 탭 포함 (DNS는 내부 도메인에서 사용 가능)
- 연결된 상품 유형(퍼널, 이메일, 웹사이트) 목록
- 상태 표시기로 빠른 문제 해결 지원
- 필터로 상품 유형별 정렬 가능; 메뉴(점 세 개)에서 편집/제거 옵션 제공

추가 기능:

- 빠른 도메인 검색을 위한 검색 바
- 내부 도메인의 자동 갱신 및 만료 추적
- 주의가 필요한 도메인을 시각적 표시기로 하이라이트

## 루트 도메인 vs 서브도메인 vs WWW

### 루트 도메인

- 메인 도메인 이름입니다.
- 예시: example.com
- 기본 사이트나 홈페이지에 사용됩니다.

### 서브도메인

- 루트 도메인 앞에 추가하는 접두사입니다.
- 예시: blog.example.com, store.example.com
- 새 도메인 없이도 다른 섹션(블로그, 퍼널, 포털 등)을 분리하는 데 사용됩니다.

### WWW (www.yourbrand.com)

- 기술적으로는 yourbrand.com의 서브도메인입니다.
- 예시: www.example.com
- 기본 웹 주소로 자주 사용됩니다.
- **권장사항**: 한 가지 버전(www 포함 또는 제외)을 선택하고 다른 버전을 리다이렉트하여 일관성과 SEO를 유지하세요.
