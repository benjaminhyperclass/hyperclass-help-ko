# 처음 사용하는 사용자가 멤버십 포털에서 로그인 비밀번호를 설정하고 잊어버린 비밀번호를 요청하는 방법

이 문서는 사용자가 매직 링크 시스템을 사용하여 처음으로 비밀번호를 설정하거나 잊어버린 비밀번호를 재설정하는 방법을 설명합니다. 결제나 자동화를 통해 액세스가 부여되든 상관없이, 이 가이드는 회원들이 안전하게 로그인하고 인증 정보를 관리하는 방법을 정확히 알 수 있도록 도와줍니다. 강의와 클라이언트 포털 접근에 모두 적용되어 플랫폼 전반에 걸쳐 일관된 경험을 제공합니다.

# **비밀번호 설정 과정이란?**

사용자가 강의나 멤버십 영역에 액세스하게 되면, 더 이상 시스템에서 생성된 비밀번호를 받지 않습니다. 대신, 자신만의 비밀번호를 생성하도록 안내하는 보안 매직 링크가 포함된 이메일을 받게 됩니다. 이 현대적인 방식은 보안을 향상시키고, 사용자 경험을 개선하며, 관리자가 인증 정보를 수동으로 관리할 필요를 없애줍니다. 처음 로그인하든 비밀번호 재설정을 요청하든 동일한 흐름이 적용됩니다.

## 설정 방법: 사용자가 비밀번호를 설정하거나 재설정하는 방법

사용자가 강의에 액세스하는 다양한 방법과 비밀번호를 생성하거나 재설정하는 방법을 알아보세요.

### **결제 완료 후 (오퍼 구매 후) :**

스크린샷에서 보는 바와 같이, 새로운 사용자는 결제 과정을 완료한 후 강의나 멤버십에 액세스 권한을 얻게 됩니다. 이메일과 전체 이름을 입력한 후 **"결제 진행(Proceed to checkout)"** 버튼을 클릭하면 해당 오퍼에 등록됩니다. 이 작업은 또한 매직 로그인 링크가 포함된 자동 이메일을 발송하여, 사용자가 자신만의 비밀번호를 설정하고 강의에 액세스할 수 있도록 안내합니다.

![결제 과정 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044610969/original/LS1lS3ZpmrhnyklrnjDPQyU2w76p64Ugog.png?1743991978)

### **관리자의 수동 등록**

관리자는 연락처 프로필로 이동하여 드롭다운 메뉴에서 오퍼를 선택함으로써 사용자를 수동으로 강의에 등록시킬 수 있습니다. 이 작업은 동일한 매직 링크 이메일을 발송하여, 사용자가 비밀번호를 설정하고 강의에 액세스할 수 있도록 합니다.

### **자동화된 워크플로우 또는 스마트 리스트 액션**

사용자는 "오퍼 액세스 부여(Offer Access Granted)"나 "멤버십 신규 가입(Membership New Signup)"과 같은 트리거를 사용하는 워크플로우나 스마트 리스트 액션을 통해 자동으로 강의에 등록될 수 있습니다. 이러한 자동화는 매직 링크가 포함된 시스템 이메일을 발송합니다. 사용자가 링크를 클릭하면 비밀번호를 설정하라는 안내를 받고 즉시 액세스 권한을 얻게 됩니다.

![워크플로우 자동화 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044612928/original/4eZko-NSaJjo3mwadVL8xHDY9L0e9TIosw.png?1743998486)

### **감사 페이지**

가입을 완료한 후, 사용자는 **"대시보드 보기(View Dashboard)"** 버튼이 있는 확인 메시지를 보게 됩니다. 이 버튼을 클릭하면 강의에 액세스하기 전에 새 비밀번호를 설정하라는 안내가 표시되는 페이지로 이동합니다.

![감사 페이지 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044611001/original/jFqID9jOzIA6xusuFatf-pRXPGcd6cCbCw.png?1743992158)

### **비밀번호 설정**

버튼을 클릭하면 새 비밀번호를 설정하라는 안내가 표시됩니다. 비밀번호를 설정한 후, 강의나 멤버십 대시보드로 리다이렉트됩니다.

![비밀번호 설정 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044612758/original/oBR8RMEtAb1yJfxinKsaPPMKAW5izf2pjA.png?1743998098)

### **비밀번호 재설정**

사용자가 비밀번호를 잊어버린 경우, 로그인 페이지로 가서 **"비밀번호를 잊으셨나요(Forgot Password)"**를 클릭할 수 있습니다. 이메일을 입력하면 비밀번호 재설정 링크를 받게 됩니다. 이 링크는 새 비밀번호를 설정하는 페이지로 리다이렉트하고, 재설정이 완료되면 자동으로 강의나 포털에 로그인됩니다. 아래는 사용자가 비밀번호를 잊어버렸을 때 옵션을 통해 비밀번호 재설정 링크를 요청했을 때 받는 이메일의 예시입니다.

비밀번호 재설정 링크는 2시간 후에 만료됩니다. 주어진 시간 내에 비밀번호를 재설정해야 합니다.

![비밀번호 재설정 이메일 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044613562/original/74YDK23_UOpCXiG4ntAYbfK7pABJzrOsEA.png?1744000459)

선택적 보안 단계: 비밀번호 재설정이나 비밀번호 변경 시, 사용자는 모든 기기에서 활성화된 세션을 무효화하도록 선택할 수 있습니다. 계정이 다른 기기에서 로그인되어 있다고 의심되는 경우 이 옵션을 사용하세요.

## 자주 묻는 질문

**Q:** 새 사용자가 비밀번호 설정을 위한 이메일을 받도록 하려면 어떻게 해야 하나요?

사용자와 연결된 이메일이 정확한지 확인하고, 시스템에서 생성된 "강의 로그인 URL(Courses Login URL)" 이메일이 활성화되어 있는지 확인하세요. 이메일 전송 로그를 확인하여 이메일이 성공적으로 발송되었는지 확인할 수 있습니다.

**Q: 사용자가 비밀번호 재설정 이메일을 받지 못했다고 신고하면 어떻게 해야 하나요?**

시스템에서 사용자의 이메일 주소를 확인하고 이메일 전송 로그를 점검하세요. 이메일이 스팸으로 표시되지 않도록 이메일 도메인이 제대로 인증되었는지(예: SPF, DKIM을 통해) 확인하세요. 필요하다면 시스템에서 이메일을 수동으로 다시 보낼 수 있습니다.

**Q: 사용자를 위해 비밀번호를 재설정해줄 수 있나요?**

보안상의 이유로 관리자는 사용자의 비밀번호를 직접 재설정할 수 없습니다. 하지만 사용자에게 "비밀번호를 잊으셨나요(Forgot Password)" 옵션을 사용하도록 안내하거나 시스템에서 비밀번호 설정 이메일을 다시 보낼 수 있습니다.

**Q: 사용자의 비밀번호 설정 문제를 모니터링하거나 해결하려면 어떻게 해야 하나요?**

CRM에서 사용자의 계정 상태를 확인하여 멤버십 포털에 제대로 추가되었는지 확인하세요. 이메일 배송 실패나 스팸 필터링 등의 문제에 대해 이메일 전송 가능성 로그를 검토하세요. 문제가 지속되면 지원팀에 도움을 요청하세요.
