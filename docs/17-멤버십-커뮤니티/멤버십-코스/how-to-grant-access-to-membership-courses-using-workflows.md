# **워크플로우로 멤버십 접근 권한 부여란?**

워크플로우로 멤버십 접근 권한 부여하기는 Hyperclass 멤버십 강의에 사용자를 수동으로 등록할 필요 없이 자동으로 등록하는 방법입니다. 한 명씩 수동으로 추가하는 대신, 트리거(예: 폼 제출, 결제 완료, 태그 추가)와 액션인 'Course Grant offer(강의 권한 부여 오퍼)'를 설정하면 자동으로 즉시 접근 권한이 부여돼요.

---

## 워크플로우로 멤버십 강의 접근 권한을 부여하는 주요 이점

- **대규모 자동화**: 사용자 가입이나 결제 시 자동으로 접근 권한을 부여하여 수동 등록의 필요성을 제거합니다.

- **향상된 사용자 경험**: 회원들이 지연 없이 즉시 강의에 접근할 수 있어 만족도를 높이고 문의를 줄입니다.

- **오류 감소**: 자동화는 사용자 등록을 잊거나 잘못된 접근 수준을 부여하는 등의 인적 오류 위험을 최소화합니다.

- **시간 절약**: 워크플로우가 관리 업무를 처리하는 동안 가치 있는 콘텐츠 제작에 집중할 수 있습니다.

- **맞춤형 트리거**: 특정 폼 제출이나 구매 후 접근 권한을 부여하는 등 비즈니스 모델에 맞게 워크플로우를 조정할 수 있습니다.

---

## 워크플로우로 멤버십 강의 접근 권한 부여하는 방법

### 1단계: 워크플로우로 이동하기

- 하위 계정에 로그인하세요.
- 대시보드에서 **Automation(자동화)** > **Workflows(워크플로우)**로 이동하세요.

![워크플로우 메뉴로 이동](https://jumpshare.com/share/PG5a8nkQMSpaHAk6u8QK+/Screen+Shot+2025-11-24+at+6.02.42+PM.png)

### 2단계: 새 워크플로우 만들기

- **+ Create Workflow(워크플로우 만들기)** 버튼을 클릭하세요.
- 처음부터 만들거나, 템플릿을 사용하거나, AI를 사용해서 워크플로우를 만드세요.

![새 워크플로우 만들기](https://jumpshare.com/share/FjI0DUh9HRkJIIDHClUQ+/Screen+Shot+2025-11-24+at+6.07.57+PM.png)

### 3단계: 트리거 설정하기

- 워크플로우를 시작할 **트리거**를 선택하세요. 예를 들면:
  - 폼 제출됨(Form submitted)
  - 결제 완료됨(Payment completed)
- 필터를 사용해서 특정 폼이나 결제 소스를 선택하세요.

![트리거 설정하기](https://jumpshare.com/share/WqF375o6FyRIxKIEXikT+/GIF+Recording+2025-11-24+at+6.12.43+PM.gif)

### 4단계: 액션 추가하기

- 액션으로 **Course Grant Offer(강의 권한 부여 오퍼)**를 선택하세요.
- 접근 권한을 부여할 **강의 또는 멤버십 상품**을 선택하세요.

![액션 추가하기](https://jumpshare.com/share/liNEHINTkegpJTxR9IhQ+/GIF+Recording+2025-11-24+at+6.16.57+PM.gif)

### 5단계: 테스트 및 워크플로우 발행하기

- **테스트**를 실행해서 워크플로우가 예상대로 작동하는지 확인하세요.
- 멤버십 접근 권한이 올바르게 부여되고 이메일 등의 추가 액션이 트리거되는지 확인하세요.

![워크플로우 테스트하기](https://jumpshare.com/share/TLfUb9zhdSvuoADyq8Ur+/Screen+Shot+2025-11-24+at+6.19.35+PM.png)

- 테스트가 완료되면 워크플로우를 발행(Publish)으로 토글하고 워크플로우를 저장하세요.

![워크플로우 발행하기](https://jumpshare.com/share/Zw7lJxsMhqdtryUxd7n7+/Screen+Shot+2025-11-24+at+6.21.06+PM.png)

## 자주 묻는 질문

**Q: 멤버십 설정의 환영 이메일이 전송되지 않는 이유는 뭔가요?**

이는 대개 **워크플로우에도 환영 이메일을 만들어서** 발생합니다. 워크플로우에서 환영 이메일을 프로그래밍하면 시스템이 멤버십 설정의 기본 환영 이메일 전송 옵션을 자동으로 비활성화합니다. 그 결과 워크플로우에 명시적으로 이메일 스텝이 포함되지 않는 한 **환영 이메일이 전혀 전송되지 않습니다**.

환영 이메일을 보내는 방법은 **하나만** 사용하세요:

- **Membership(멤버십)** 설정에서 활성화하거나
- 이메일 스텝이 있는 **Workflow(워크플로우)**에서 프로그래밍하거나

**Q: 서로 다른 멤버십 강의에 대해 여러 워크플로우를 설정할 수 있나요?**

네, 각 강의에 대해 별도 워크플로우를 만들어서 특정 요구사항에 따라 트리거와 액션을 맞춤 설정할 수 있어요.

**Q: 결제가 실패하면 어떻게 되나요?**

결제 실패 시 사용자에게 알리고 결제가 해결될 때까지 접근 권한을 일시정지하도록 워크플로우를 구성할 수 있어요.

**Q: 멤버십을 위해 워크플로우와 이메일 캠페인을 결합할 수 있나요?**

물론이죠! 워크플로우를 사용해서 환영 이메일이나 강의 완료 리마인더 등의 개인화된 이메일 캠페인을 트리거할 수 있어요.

**Q: 워크플로우를 통해 접근 권한을 얻은 사용자를 어떻게 추적하나요?**

Hyperclass는 사용자 액션과 워크플로우 성능을 모니터링할 수 있는 추적 및 리포팅 도구를 제공해요.
