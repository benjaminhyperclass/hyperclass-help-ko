# 처음 사용하는 회원이 멤버십 포털 로그인 비밀번호를 설정하거나 잊어버린 비밀번호를 재설정하는 방법

이 문서는 매직 링크 시스템을 사용하여 사용자가 처음 비밀번호를 설정하거나 잊어버린 비밀번호를 재설정하는 방법을 설명합니다. 결제를 통해 접근권한을 받든 자동화를 통해 받든, 이 가이드는 회원이 안전하게 로그인하고 계정 정보를 관리하는 방법을 정확히 알 수 있도록 도움을 줍니다. 강의(Course)와 클라이언트 포털(Client Portal) 접근에 모두 적용되어 플랫폼 전체에서 일관된 경험을 제공합니다.

# **비밀번호 설정 프로세스란?**

사용자가 강의나 멤버십 영역에 접근권한을 얻으면, 더 이상 시스템에서 자동 생성된 비밀번호를 받지 않습니다. 대신 안전한 매직 링크가 포함된 이메일을 받아 직접 비밀번호를 생성하도록 안내받습니다. 이 현대적인 방식은 보안을 향상시키고 사용자 경험을 개선하며, 관리자가 계정 정보를 수동으로 관리할 필요를 없애줍니다. 첫 로그인이든 비밀번호 재설정 요청이든 동일한 흐름이 적용됩니다.

## 설정 방법: 사용자가 비밀번호를 설정하거나 재설정하는 방법

사용자가 강의에 접근권한을 받는 다양한 방법과 비밀번호를 생성하거나 재설정하는 방법을 알아보세요.

### **결제 완료 후 (오퍼 구매 후):**

새로운 사용자는 스크린샷에 표시된 것처럼 결제 프로세스를 완료한 후 강의나 멤버십에 대한 접근권한을 받습니다. 이메일과 성명을 입력하고 **"Proceed to checkout(결제 진행)"**을 클릭하면 오퍼(Offer)에 등록됩니다. 이 과정에서 매직 로그인 링크가 포함된 자동 이메일이 발송되어, 사용자가 직접 비밀번호를 설정하고 강의에 접근할 수 있도록 안내합니다.

![결제 완료 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044610969/original/LS1lS3ZpmrhnyklrnjDPQyU2w76p64Ugog.png?1743991978)

### **관리자의 수동 등록**

관리자는 연락처(Contacts) 프로필로 이동하여 드롭다운 메뉴에서 오퍼를 선택해 수동으로 사용자를 강의에 등록할 수 있습니다. 이 작업 역시 동일한 매직 링크 이메일을 발송하여 사용자가 비밀번호를 설정하고 강의에 접근할 수 있게 해줍니다.

### **자동화 워크플로우 또는 스마트 리스트 액션**

사용자는 "Offer Access Granted(오퍼 접근 권한 부여)" 또는 "Membership New Signup(멤버십 신규 가입)"과 같은 트리거를 사용하는 워크플로우나 스마트 리스트 액션을 통해 자동으로 강의에 등록될 수 있습니다. 이러한 자동화는 매직 링크가 포함된 시스템 이메일을 발송합니다. 사용자가 이 링크를 클릭하면 비밀번호를 설정하라는 안내를 받고 즉시 접근할 수 있습니다.

![워크플로우 자동화 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044612928/original/4eZko-NSaJjo3mwadVL8xHDY9L0e9TIosw.png?1743998486)

### **감사 페이지**

가입을 완료한 후, 사용자는 **"View Dashboard(대시보드 보기)"** 버튼이 있는 확인 메시지를 보게 됩니다. 이 버튼을 클릭하면 강의에 접근하기 전에 새 비밀번호를 설정하라는 안내를 받는 페이지로 이동합니다.

![감사 페이지 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044611001/original/jFqID9jOzIA6xusuFatf-pRXPGcd6cCbCw.png?1743992158)

### **비밀번호 설정**

해당 버튼을 클릭하면 새 비밀번호를 설정하라는 안내를 받습니다. 비밀번호를 설정한 후에는 강의 또는 멤버십 대시보드로 리다이렉트됩니다.

![비밀번호 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044612758/original/oBR8RMEtAb1yJfxinKsaPPMKAW5izf2pjA.png?1743998098)

### **비밀번호 재설정**

사용자가 비밀번호를 잊어버린 경우, 로그인 페이지로 가서 **"Forgot Password(비밀번호 찾기)"**를 클릭할 수 있습니다. 이메일을 입력하면 비밀번호 재설정 링크를 받게 됩니다. 이 링크는 새 비밀번호를 설정하는 페이지로 리다이렉트하고, 재설정이 완료되면 자동으로 강의나 포털에 로그인됩니다. 아래는 사용자가 비밀번호 찾기 옵션을 통해 비밀번호 재설정 링크를 요청했을 때 받는 이메일의 예시입니다.

비밀번호 재설정 링크는 2시간 후 만료됩니다. 주어진 시간 내에 비밀번호를 재설정해야 합니다.

![비밀번호 재설정 이메일](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044613562/original/74YDK23_UOpCXiG4ntAYbfK7pABJzrOsEA.png?1744000459)

선택적 보안 단계: 비밀번호 재설정이나 비밀번호 변경 중에 사용자는 모든 기기에서의 활성 세션을 무효화하도록 선택할 수 있습니다. 계정이 다른 기기에 로그인되어 있다고 의심되는 경우 이 옵션을 사용하세요.

## 자주 묻는 질문

**Q:** 새 사용자가 비밀번호 설정 이메일을 확실히 받도록 하려면 어떻게 해야 하나요?

사용자와 연결된 이메일이 정확한지 확인하고, 시스템에서 생성되는 "Courses Login URL" 이메일이 활성화되어 있는지 확인하세요. 이메일이 성공적으로 발송되었는지 이메일 전송 로그를 확인할 수 있습니다.

**Q:** 사용자가 비밀번호 재설정 이메일을 받지 못했다고 신고하면 어떻게 해야 하나요?

시스템에서 사용자의 이메일 주소를 확인하고 이메일 전송 로그를 검토하세요. 이메일이 스팸으로 표시되지 않도록 이메일 도메인이 적절히 인증되었는지 확인하세요(예: SPF, DKIM을 통해). 필요한 경우 시스템에서 이메일을 수동으로 다시 발송할 수 있습니다.

**Q:** 사용자를 대신해서 비밀번호를 재설정할 수 있나요?

보안상의 이유로 관리자는 사용자의 비밀번호를 직접 재설정할 수 없습니다. 하지만 사용자에게 "비밀번호 찾기" 옵션을 사용하도록 안내하거나 시스템에서 비밀번호 설정 이메일을 다시 발송할 수 있습니다.

**Q:** 사용자의 비밀번호 설정 문제를 모니터링하거나 해결하려면 어떻게 해야 하나요?

CRM에서 사용자의 계정 상태를 확인하여 멤버십 포털에 적절히 추가되었는지 확인하세요. 반송이나 스팸 필터링과 같은 문제가 있는지 이메일 전송 가능성 로그를 검토하세요. 문제가 지속되면 고객지원팀에 문의하세요.
