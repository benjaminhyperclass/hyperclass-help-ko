# 기존 하위 계정을 SaaS 모드로 전환하는 방법

- 에이전시 계정에서 메인 네비게이션 바의 **Sub-Accounts(하위 계정)** 탭으로 이동합니다.

- 전환하려는 하위 계정을 찾아 목록 우측 하단의 **세 점** 아이콘을 선택합니다.

- **Switch To SaaS(SaaS로 전환)** 를 선택합니다.

![하위 계정을 SaaS로 전환하는 화면](https://jumpshare.com/share/hGnHgYlDeA5VA5hHT16H+/Screen+Shot+2026-02-03+at+12.41.30+AM.png)

- **고객이 아직 Stripe에 존재하지 않는 경우**, **Create New Customer in Stripe(Stripe에서 새 고객 생성)** 옵션을 선택합니다. **SMS**, **이메일**로 고객에게 링크를 보내거나 링크를 직접 **복사**해서 **전달**하세요. 그런 다음 고객에게 해당 링크에서 카드 정보를 등록하도록 요청하세요.

- **Proceed(계속)** 을 클릭합니다.

![Stripe에서 새 고객을 생성하는 화면](https://jumpshare.com/share/iRDjEPxOgkxdr21r77MA+/Screen+Shot+2026-02-03+at+12.51.01+AM.png)

- **고객이 이미 Stripe에 존재하는 경우**, Link Existing Customer in Stripe(Stripe의 기존 고객 연결) 옵션을 선택하고, 고객을 **검색**하여 **선택**한 후 **Proceed(계속)** 을 누릅니다.

![Stripe의 기존 고객을 연결하는 화면](https://jumpshare.com/share/yvJrCkwMQQoAhX7UCxej+/Screen+Shot+2026-02-03+at+12.59.12+AM.png)

- 위의 두 옵션 모두에서 **Proceed(계속)** 을 클릭하면, **SaaS Configurator(SaaS 설정)에서 해당 고객에게 적용할 플랜을 선택하라는 안내**가 표시됩니다.

중요: SaaS 모드를 활성화하려면 반드시 플랜을 선택해야 합니다. 플랜이 완전히 효력을 발휘하려면 고객이 결제 방법을 추가해야 합니다.

계속하기 전에 플랜 세부사항을 검토하세요. 구독 플랜 선택(Select Subscription Plan) 모달에서 하이레벨은 이제 각 플랜에 대한 더 많은 정보를 표시하므로 활성화하려는 내용을 확인할 수 있습니다.

플랜 선택 버튼 위의 **Show Full Details(전체 세부사항 보기)** 옵션을 클릭하세요.

![플랜 선택 화면](https://jumpshare.com/share/7v1siL0HSgt6gnu9u2W9+/Screenshot+2025-11-21+at+1.54.11%E2%80%AFPM.png)

- 다음 사항을 **검토**할 **수 있습니다:** 리빌링 요금, 플랜에 포함된 애드온, 플랜에 포함된 마켓플레이스 앱, 플랜에 포함된 커스텀 메뉴 링크.

![플랜 세부사항 화면](https://jumpshare.com/share/rbz3LnVy8nRbmHFpEwF1+/Screenshot+2025-11-21+at+1.55.15%E2%80%AFPM.png)

- **플랜을 선택**하고 **Continue(계속)** 을 클릭합니다.

![플랜 선택 완료 화면](https://jumpshare.com/share/baD1dldAsCjUeBgfdr0C+/Screen+Shot+2026-02-03+at+1.06.00+AM.png)

- 선택한 플랜은 고객이 결제 방법(카드)을 추가하면 활성화됩니다. 그때까지는 **Hold(보류)** 상태가 됩니다.

![플랜 보류 상태 화면](https://jumpshare.com/share/8ksVfUlrQWbFtqnc8ZYu+/Screen+Shot+2026-02-03+at+1.11.13+AM.png)

## **자주 묻는 질문**

**Q: "공유 링크" 옵션이 작동하지 않거나 사용할 수 없을 때는 어떻게 해야 하나요?**

"공유 링크" 옵션이 작동하지 않는다면, 먼저 브라우저나 네트워크 설정이 팝업이나 링크 자체를 차단하고 있지 않은지 확인하세요. 다른 브라우저를 사용하거나 브라우저 캐시를 지워보세요. 문제가 계속되면 하이레벨 계정 내에서 이 기능에 영향을 줄 수 있는 제한이나 권한 설정이 있는지 확인하세요. 추가 지원이 필요하면 하이레벨 고객지원에 문의할 수도 있습니다.

**Q: 미납 인보이스나 미결제 잔액이 있는 기존 하위 계정을 SaaS 모드로 전환할 수 있나요?**

네, 미납 인보이스나 미결제 잔액이 있어도 기존 하위 계정을 SaaS 모드로 전환할 수 있습니다. 하지만 청구 및 계정 관리의 복잡함을 피하기 위해 미결제 잔액을 해결하는 것이 일반적으로 좋은 방법입니다. 원활한 전환을 위해 모든 금전적 문제를 최신 상태로 유지하세요.

**Q: 하위 계정의 SaaS 모드 전환 과정 상태를 어떻게 추적할 수 있나요?**

SaaS 모드 전환 상태를 추적하려면 하이레벨 대시보드에서 하위 계정의 상태를 모니터링할 수 있습니다. 전환 과정과 관련된 업데이트나 알림을 확인하세요. 변경사항이나 업데이트가 보이지 않으면 과정이 성공적으로 완료되었는지 확인하세요. 구독이 생성되어 활성 상태인지 확인하기 위해 Stripe 계정도 확인해야 할 수 있습니다. 자세한 추적을 위해서는 문제가 발생하면 하이레벨 고객지원에 문의하는 것을 고려하세요.

**Q: SaaS 모드를 활성화하기 전에 생성된 하위 계정에 플랜을 연결할 수 있나요?**

네, 하지만 이 문서의 단계를 사용해서 수동으로 SaaS 플랜을 할당해야 합니다. 레거시 계정은 플랜을 할당하지 않는 한 자동으로 전환되지 않습니다.

**Q: 하위 계정이 SaaS 모드로 전환될 때 SaaS 플랜 티어에 포함되지 않은 기능에 대한 접근 권한을 잃게 되나요?**

네. 하위 계정이 SaaS 모드로 이동하고 특정 플랜 티어(예: 스타터, 프로페셔널, 또는 프리미엄)가 할당되면, 해당 티어에서 활성화된 기능에만 접근할 수 있습니다. 이전에 사용 가능했지만 선택한 플랜에 포함되지 않은 기능은 자동으로 비활성화됩니다. 이러한 기능을 유지하려면 SaaS Configurator(SaaS 설정)를 통해 할당된 플랜에 해당 기능이 포함되어 있는지 확인하세요.
