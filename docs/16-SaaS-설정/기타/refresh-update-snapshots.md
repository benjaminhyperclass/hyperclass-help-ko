# 스냅샷 새로고침 및 업데이트

고객 하위 계정(서브어카운트)에서 새로운 워크플로우, 폼, 퍼널 등을 변경해도 기존 스냅샷은 자동으로 업데이트되지 않습니다. 이러한 새로운 변경사항을 적용하고 향후 가져오기나 업데이트에 사용할 수 있도록 하려면 스냅샷을 수동으로 새로고침해야 합니다.

# **스냅샷 새로고침 방법**

- 에이전시 뷰로 이동하여 왼쪽 메뉴에서 **Account Snapshots(계정 스냅샷)** 탭을 클릭하세요

- 업데이트하려는 스냅샷을 찾아 해당 스냅샷 옆의 새로고침 아이콘을 클릭하세요

- 포함하려는 새로운 자산이나 수정된 자산을 선택하세요

- **Refresh(새로고침)**를 클릭하여 업데이트를 완료하세요

![스냅샷 새로고침 과정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059137335/original/4ImXJgLPogiQje4C__bwGTGY17ytoezW_g.gif?1764053794)

**참고:** **새로고침**을 클릭하면 Hyperclass가 소스 하위 계정에서 선택한 자산을 불러옵니다. 얼마나 많은 자산이 로드되었는지 보여주는 진행 표시를 볼 수 있습니다. 일부 자산 로드가 실패해도 전체 프로세스를 중단하지 않고 나머지 자산에 대해 새로고침이 계속됩니다.

![새로고침 진행 상황](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059751597/original/KaN1CEBWntC56zsL_DSczkAWr18LzgSRwA.png?1764725377)

## **새로고침 중 실패한 자산 처리**

새로고침 중에 하나 이상의 자산이 로드에 실패하면 해당 자산은 실패로 표시되고 나머지 선택된 자산은 계속 처리됩니다.

전체 새로고침을 다시 시작하지 않고 실패한 자산만 다시 로드할 수 있습니다. 이를 통해 일시적인 문제(예: 간헐적인 타임아웃)를 해결하면서 전체 스냅샷 업데이트를 차단하지 않을 수 있습니다.

![실패한 자산 재시도](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059751665/original/6rEEssPXG7lNT-jUk3kLLk6ZwDavXwMUIA.png?1764725603)

## **스냅샷 새로고침 후 워크플로우 연락처 정리**

스냅샷 새로고침으로 인해 스냅샷에서 생성된 워크플로우 내부의 단계가 제거되면(예: 대기 단계 삭제), Hyperclass는 삭제된 단계에서 대기 중이던 연락처를 자동으로 제거하여 연락처가 멈춰있지 않도록 합니다.

확인할 수 있는 내용:

- 다음에 영향을 받은 워크플로우를 열 때 일회성 알림이 나타납니다(해당되는 경우에만 표시됩니다).

![워크플로우 정리 알림](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064759193/original/CMYs9reyZPN0_CYDFRSRENDtRlHYVACoTg.png?1770890041)

- **Execution Logs(실행 로그)**에서 워크플로우는 "Removed by - Snapshot Refresh"라고 표시된 항목을 기록하며, 사이드 패널에서 더 자세한 내용을 확인할 수 있습니다.

![실행 로그의 스냅샷 정리 기록](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064759370/original/gjFq-I3Wq8aVLo4u1Pc4XOyD0kax7TOgcw.png?1770890085)

참고: 이는 새로고침으로 단계가 삭제될 때 스냅샷에서 생성된 워크플로우에만 적용됩니다. 워크플로우에서 단계를 직접 삭제하면 이미 대기 중인 연락처가 제거됩니다.

## **"업데이트 푸시" 클릭 시 일어나는 일**

스냅샷을 새로고침한 후에는 해당 스냅샷에 대해 **"Push Update(업데이트 푸시)"** 버튼이 활성화됩니다. 이를 클릭하면 다음을 할 수 있습니다:

- **업데이트를 받을 하위 계정을 선택**(이전에 이 스냅샷을 가져온 계정들)

- **푸시할 자산을 선택**(예: 워크플로우만, 또는 특정 폼만)

- **전체 스냅샷을 다시 로드하지 않고** 해당 하위 계정들에 **수동으로 업데이트 동기화**

[자세한 비디오 가이드는 여기서 확인하세요](../../19-에이전시-뷰/Snapshots/pushing-loading-snapshot-updates-to-client-accounts.md)

**참고:** 업데이트는 자신의 에이전시 내 하위 계정에만 푸시됩니다. 외부 에이전시는 새 링크를 사용해서 다시 가져와야 합니다.

## **빠른 팁**

- 공유하거나 업데이트를 푸시하기 전에 항상 스냅샷을 새로고침하세요

- Push Update 기능을 사용하여 선택한 변경사항을 고객 계정에 동기화하세요

- 푸시된 업데이트는 선택적이며 파괴적이지 않습니다 - 기존 데이터는 안전하게 유지됩니다
