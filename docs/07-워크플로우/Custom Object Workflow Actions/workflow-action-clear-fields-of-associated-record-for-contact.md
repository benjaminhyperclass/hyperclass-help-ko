# 워크플로우 액션 - 연락처의 연관 레코드 필드 초기화

## 개요

**연관 레코드의 필드 초기화(Clear Fields of an Associated Record)** 액션을 사용하면 연락처와 연관된 커스텀 객체 내의 특정 필드를 초기화할 수 있습니다. 이는 오래된 정보를 정리하거나, 폼 입력값을 재설정하거나, 워크플로우에서 특별한 조건을 처리할 때 유용합니다.


## 액션명

Clear Fields of Associated Record for Contact(연락처의 연관 레코드 필드 초기화)


## 액션 설명
![커스텀 객체의 연관 레코드 필드를 선택해서 값을 초기화하는 워크플로우 액션 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041054690/original/EJDSgBhbWPet8kBhbbJeTCWWriARecUORQ.jpeg?1738756190)


이 액션은 커스텀 객체 내의 특정 필드를 선택해서 레코드는 그대로 유지하면서 해당 필드의 값만 지울 수 있습니다. 연관 라벨에 따라 어떤 레코드의 필드를 초기화할지가 결정됩니다.


## 액션 세부사항

- 이 액션은 하위 계정에서 커스텀 객체를 생성하고, 해당 객체 중 최소 하나와 연락처 사이에 연관 라벨을 생성한 후에만 사용 가능합니다.
- 액션이 작동하려면 커스텀 객체와 연락처 사이에 하나 이상의 연관이 필요합니다.
- 여러 개의 필드를 선택해서 한 번에 초기화할 수 있습니다.
- 이 액션은 레코드 자체를 삭제하지 **않고**, 선택한 필드의 값만 지웁니다.


## 예시


소매 업체에서 **고객 적립 포인트**를 **적립 프로그램** 커스텀 객체로 관리하고 있습니다. 고객이 포인트로 리워드를 교환할 때, **적립 포인트** 필드를 초기화해서 교환 내역을 반영해야 합니다.

워크플로우는 특정 날짜에 트리거될 수 없으므로, **교환 요청 처리** 같은 관련 **이벤트가 발생**했을 때 이 액션이 실행됩니다.

#### **예시 설정**

- **업데이트할 객체** → 적립 프로그램
- **연관 라벨** → 고객 리워드
- **초기화할 필드:**
  - **적립 포인트**
  - **마지막 교환 날짜**

#### **작동 방식**

- **고객이 포인트를 교환**하면 워크플로우가 트리거되어 **적립 포인트** 필드를 초기화합니다.
- **마지막 교환 날짜**도 함께 재설정되어 향후 교환 내역을 정확하게 추적할 수 있습니다.
- 이렇게 하면 교환된 포인트가 실수로 재사용되는 것을 방지하면서도 고객의 적립 레코드는 그대로 유지됩니다.

이 기능은 비즈니스에서 **적립 포인트 교환을 정확하게 추적**하고 고객이 교환 후 포인트를 재사용할 수 없도록 보장하는 데 도움이 됩니다.
