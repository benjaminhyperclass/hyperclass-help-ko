## 개요

"**대화 편집(Edit Conversation)**" 액션을 사용하면 대화의 상태를 변경할 수 있습니다. 대화를 읽음 또는 읽지 않음으로 표시하거나, 보관하거나 보관 해제할 수 있습니다. 이 기능은 대화를 효과적으로 관리하고 어떤 메시지가 처리되었는지 추적하는 데 유용합니다.

## 액션 이름

**대화 편집(Edit Conversation)**

## 액션 설명

"대화 편집" 액션으로 다음 작업을 할 수 있습니다:

- 대화를 읽음 또는 읽지 않음으로 표시
- 대화를 보관하여 활성 대화 목록에서 제거
- 대화 보관을 해제하여 최근 대화 탭으로 다시 이동

이 액션은 빠른 상태 업데이트와 정리를 통해 대화 관리를 효율화합니다.

| 읽음 또는 읽지 않음으로 표시 | 대화를 읽음 또는 읽지 않음으로 표시할지 선택합니다. 옵션:<ul><li>없음</li><li>읽음으로 표시</li><li>읽지 않음으로 표시</li></ul> | 아니요 |

#### **설정 방법**

- 액션 이름: "읽음 표시 및 보관"과 같은 설명적인 이름을 입력하세요.
- 읽음 또는 읽지 않음으로 표시: 대화의 읽음 상태를 변경할 옵션을 선택하세요.
- 보관 또는 보관 해제: 대화를 보관할지 또는 최근 대화 탭으로 되돌릴지 선택하세요.

#### 추천 트리거

- **연락처가 답장함:** 연락처가 답장을 보낼 때 대화를 자동으로 읽음으로 표시하려면 이 액션을 사용하세요.
- **예약 상태 변경:** 예약이 완료로 표시된 후 대화를 보관하려면 이 액션을 사용하세요.

## 예시

**시나리오:** 고객 서비스 이슈가 해결되면 대화를 자동으로 보관하고 읽음으로 표시하고 싶습니다.

- 연락처가 답장을 보낼 때 실행되는 워크플로우 트리거를 설정합니다.
- "대화 편집" 액션을 추가합니다.
- 대화를 읽음으로 표시하고 보관하도록 설정합니다.
- 이렇게 하면 해결된 모든 대화가 자동으로 활성 대화 목록에서 제거되어, 팀이 정리된 상태를 유지하고 미해결 이슈에만 집중할 수 있습니다.

이러한 설정은 깨끗하고 정리된 커뮤니케이션 채널을 유지하는 데 도움이 되며, 지원 팀이 활성 상태이고 미해결된 대화만 즉시 볼 수 있도록 보장합니다.
