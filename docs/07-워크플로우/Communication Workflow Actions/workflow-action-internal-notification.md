# 워크플로우 액션 - 내부 알림

워크플로우의 내부 알림(Internal Notifications)을 통해 팀은 중요한 이벤트에 대해 즉시 업데이트를 받을 수 있습니다. 신규 리드, 고액 구매, 내부 업데이트 등 어떤 상황이든, 이 액션은 이메일, 인앱 알림, SMS 또는 왓츠앱을 통해 적절한 사람에게 알림을 보내줍니다. 알림 유형과 설정에 따라, 내부 알림은 연락처 기록이 필요 없는 무연락처 시나리오에서도 실행될 수 있습니다. 이 가이드에서는 워크플로우 내에서 내부 알림을 효과적으로 설정하고 사용하는 방법을 설명합니다.

# **내부 알림이란 무엇인가요?**

내부 알림은 Hyperclass 계정 내의 선택된 사용자, 역할 또는 팀에게 전송되는 자동 알림입니다. 이러한 알림은 특정 워크플로우 조건에 의해 트리거되며, 팀원들이 선호하는 커뮤니케이션 채널을 통해 적시에 업데이트를 받을 수 있도록 보장합니다.

내부 알림은 신규 리드, 폼 완료, 파이프라인 업데이트와 같은 중요한 이벤트에 대한 즉각적인 가시성이 필요한 영업, 지원, 운영 팀에게 유용합니다.

## 내부 알림의 주요 장점

내부 알림은 팀 커뮤니케이션을 간소화하고 중요한 업데이트를 놓치지 않도록 도와줍니다.

- **실시간 업데이트**: 중요한 워크플로우 트리거에 대해 팀원들에게 즉시 알림을 보냅니다.

- **다채널 지원**: 이메일, 인앱 알림, SMS 또는 왓츠앱으로 전송 가능합니다.

- **역할 기반 타겟팅**: 개별 사용자, 사용자 역할(예: 모든 관리자) 또는 특정 팀에게 알림을 보낼 수 있습니다.

- **책임감 향상**: 리드나 작업이 배정될 때 적절한 사람들이 정보를 받도록 보장합니다.

- **맞춤형 콘텐츠**: 템플릿, 트리거 링크, 동적 필드로 알림을 개인화할 수 있습니다.

---

## **내부 알림 설정 방법**

워크플로우에서 내부 알림을 설정하는 것은 간단합니다. 다음 단계를 따르세요:

### 워크플로우 빌더 접근

워크플로우를 생성하거나 편집하려면 좌측 메뉴의 **Automation(자동화)** 탭으로 이동하세요. 여기서 내부 알림을 포함한 모든 워크플로우 액션을 관리할 수 있습니다.

우측 상단의 **+ Create Workflow(+ 워크플로우 생성)** 버튼을 클릭하고 처음부터 시작하거나 템플릿을 사용하거나 레시피를 선택할지 결정하세요. 이는 내부 알림 설정의 기반을 마련합니다.

![워크플로우 생성 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052237482/original/fPZgN9LK1Yzck1-b_WGA5Eqqv9XaQhKYGg.png?1756041643)

### **내부 알림 액션 선택**

워크플로우 빌더에서 "+" 버튼을 클릭하여 이 워크플로우 단계에 추가할 수 있는 사용 가능한 액션 목록을 엽니다. 액션 목록에서 아래로 스크롤하여 **Send Internal Notification(내부 알림 보내기)**을 선택하세요. 이를 통해 이메일, 인앱, SMS 또는 왓츠앱을 통해 팀원들에게 알림을 구성할 수 있습니다.

![내부 알림 액션 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239636/original/QELi4ehOwplU_E6tcLJ36RLcsJI7pTXdZg.png?1756048657)

### **액션 이름 지정**

내부 알림을 위한 명확하고 설명적인 **액션 이름**을 입력하세요. 구체적인 이름을 사용하면(예: "영업팀에게 알림 – 이메일") 워크플로우를 체계적으로 유지하고 쉽게 관리할 수 있습니다.

![액션 이름 입력](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239638/original/2S_Dguqq1hvKNxEMtUBWP843lJq9uaWxRA.png?1756048681)

### **알림 유형 선택**

드롭다운 메뉴에서 보내고자 하는 **알림 유형**을 선택하세요. 옵션에는 이메일, 인앱 알림, SMS 또는 왓츠앱이 있으며, 팀이 알림을 받는 방식에 따라 달라집니다. 워크플로우 시나리오에 가장 적합한 채널을 선택하세요. 예를 들어, 세부적인 업데이트는 이메일, 빠른 알림은 인앱, 긴급 알림은 SMS, 채팅 스타일 커뮤니케이션은 왓츠앱을 사용하세요.

![알림 유형 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239646/original/xLZijg_xs6D6typPqeEuzKZHHcvFi6RcdQ.png?1756048698)

### 1. 알림 유형 - 이메일

**이메일 알림**을 설정할 때는 메시지가 어떻게 전달되고 수신자에게 표시되는지를 정의하는 핵심 필드를 구성하게 됩니다. 이러한 옵션은 이메일이 명확하고 전문적이며 적절한 팀원에게 도달하도록 보장합니다.

- From Name(발신자 이름): 수신자의 받은편지함에 발신자로 표시될 이름을 입력하세요. **태그 아이콘**을 사용하여 커스텀 값을 삽입하여 발신자 세부정보를 동적으로 만들 수도 있습니다.

- From Email(발신자 이메일): 알림을 보낼 이메일 주소를 지정하세요. 비워두면 시스템 기본값이 적용됩니다. 더 큰 유연성을 위해 태그 아이콘으로 **커스텀 값**을 삽입할 수 있습니다.

- To User Type(수신자 유형): 이메일을 받아야 하는 내부 사용자 유형을 선택하세요(예: 관리자, 배정된 사용자 또는 특정 역할). 이를 통해 메시지가 적절한 팀원에게 전달됩니다.

- CC/BCC: CC(모든 사람에게 표시) 또는 BCC(다른 사람들에게 숨김)로 추가 수신자를 추가하여 더 넓은 커뮤니케이션을 할 수 있습니다. 여기의 각 항목은 별도의 이메일로 계산되어 워크플로우를 방해하지 않고 이해관계자를 복사할 수 있습니다.

- Subject(제목): 이메일의 목적을 명확히 나타내는 제목 줄을 입력하세요. 이미 제목 줄이 정의된 템플릿을 사용하는 경우 이 필드는 비워둘 수 있습니다. **커스텀 값**을 사용하여 제목을 동적으로 개인화할 수도 있습니다.

![이메일 알림 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239655/original/xynD9Dxq2JUqaUyfWN-oKhtfx6D9_-aIKg.png?1756048712)

### **이메일 콘텐츠 설정**

1. Templates(템플릿): 시간을 절약하고 일관성을 유지하기 위해 드롭다운에서 미리 작성된 **이메일 템플릿**을 선택하세요. 필요한 경우 추가 콘텐츠로 템플릿을 사용자 정의할 수 있습니다.

2. Message(메시지): 여기서 **커스텀 메시지**를 작성하거나 편집하세요. 서식 지정을 위한 도구 모음과 개인화된 알림을 위해 동적 값(예: 연락처 이름, 사용자 정보)을 삽입하는 **태그 아이콘**을 사용하세요.

Write with AI(AI로 작성하기): **Write with AI**를 클릭하여 이메일 메시지에 대한 제안 콘텐츠를 생성하세요. 이를 통해 필요에 맞게 편집할 수 있는 전문적인 알림을 빠르게 작성할 수 있습니다.

3. Add Attachment(첨부파일 추가): **Add Attachment**를 클릭하여 알림에 파일을 포함하세요. 이는 문서, 계약서 또는 추가 리소스를 팀과 공유하는 데 유용합니다.

![이메일 콘텐츠 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239659/original/lM76enR7jRBBiyRRa7QP6K8avA-hnvbwWA.png?1756048728)

### 2. 알림 유형 – 알림

유형으로 **Notification(알림)**을 선택하면 Hyperclass 내부의 **종 아이콘** 아래에 직접 알림이 전달됩니다. 이는 이메일이나 SMS가 필요하지 않은 빠른 업데이트에 이상적입니다.

1. Type of Notification(알림 유형): 인앱 알림을 보내려면 **Notification**을 선택하세요. 이러한 알림은 플랫폼 내에서 볼 수 있으며 팀원들이 실시간으로 업데이트를 받는 데 도움이 됩니다.

2. Title(제목): 알림 상단에 나타날 간결한 제목을 입력하세요. **태그 아이콘**을 사용하여 더 개인화된 알림을 위한 동적 값을 삽입할 수도 있습니다.

3. Message(메시지): 알림의 주요 콘텐츠를 작성하세요. 간결하고 실행 가능하게 유지하고, 서식 도구나 **커스텀 값**을 사용하여 관련 세부정보를 포함하세요.

![인앱 알림 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239661/original/cLIiLFgRZnLNeeoZHN6Ixo6ED4LQmGnNgg.png?1756048745)

### **알림 리다이렉트 및 수신자**

1. Redirect Page(리다이렉트 페이지): 사용자가 알림을 클릭했을 때 이동할 **리다이렉트 페이지**를 선택하세요. 이를 통해 연락처 기록이나 기회와 같은 가장 관련성 높은 화면으로 직접 안내할 수 있습니다.

2. To User Type(수신자 유형): 이 알림을 받아야 하는 **사용자 유형**을 선택하세요. 옵션에는 특정 역할, 배정된 사용자 또는 팀이 포함되며, 항상 적절한 사람들이 정보를 받도록 보장합니다.

![알림 리다이렉트 및 수신자 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239672/original/NuKFhobJ5ZoybcjN5DsseRF2OIuU1SRDBA.png?1756048759)

### 3. 알림 유형 – SMS

알림 유형으로 **SMS**를 선택하면 사용자의 등록된 전화번호로 직접 텍스트 알림을 보낼 수 있습니다. 이는 긴급하거나 시간에 민감한 업데이트에 가장 적합합니다.

1. Type of Notification(알림 유형): 내부 사용자에게 문자 메시지로 알림을 전달하려면 **SMS**를 선택하세요.

2. To User Type(수신자 유형): SMS를 받을 사용자 유형을 선택하세요(예: 관리자, 배정된 사용자 또는 팀).

3. Templates(템플릿): 빠른 설정을 위해 미리 작성된 SMS 템플릿을 선택하거나, 커스텀 메시지를 작성하려면 비워두세요.

4. Message(메시지): 여기에 SMS 콘텐츠를 작성하세요. 짧고 명확하게 유지하고, 커스텀 값을 위한 **태그 아이콘**과 AI 지원 개인화를 위한 **Write with AI** 옵션을 사용하세요.

![SMS 알림 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239682/original/C8JdrCrAiydqRAp6q-1e8bibRRsVCaq2kg.png?1756048780)

### **SMS 첨부파일 및 테스트**

1. Add Attachment(첨부파일 추가): **Add Attachment**를 클릭하여 SMS 알림에 파일을 포함하세요. 이는 지원 문서나 리소스를 팀과 공유하는 데 유용합니다.

2. Add Files via URL(URL을 통한 파일 추가): 이 필드에 파일 URL을 붙여넣고 **+ Add**를 클릭하여 SMS에 직접 파일을 첨부하세요. 이를 통해 팀원들이 연결된 리소스에 즉시 액세스할 수 있습니다.

3. Test Phone Number(테스트 전화번호): 유효한 전화번호(국가 코드 포함)를 입력하고 **Send Test SMS(테스트 SMS 보내기)**를 클릭하여 알림을 미리 봅니다. 이를 통해 SMS 콘텐츠와 첨부파일이 전송 전에 올바르게 표시되는지 확인할 수 있습니다.

![SMS 첨부파일 및 테스트](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239687/original/FFweZxUkmZpNJvnp9fBbbbod_isS0DUiHA.png?1756048794)

### 4. 알림 유형 – 왓츠앱

**WhatsApp**을 선택하면 등록된 왓츠앱 번호를 통해 팀원들에게 인스턴트 메시지를 직접 보낼 수 있습니다. 이는 실시간, 대화형 업데이트에 가장 적합합니다.

1. Type of Notification(알림 유형): 왓츠앱 메시징 채널을 통해 알림을 보내려면 **WhatsApp**을 선택하세요.

2. To User Type(수신자 유형): 왓츠앱 메시지를 받을 **사용자 유형**을 선택하세요(예: 관리자, 특정 역할 또는 배정된 사용자).

3. Templates(템플릿): 일관성을 유지하고 왓츠앱의 메시징 요구사항을 충족하기 위해 승인된 왓츠앱 템플릿을 선택하세요.

4. Message(메시지): 여기에 커스텀 왓츠앱 메시지를 입력하세요. 사용자나 연락처 세부정보와 같은 동적 값을 위해 **태그 아이콘**을 사용하여 개인화할 수 있습니다.

![왓츠앱 알림 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052239688/original/-3wHO1al6hBPBOI-bWpaDQ1ebJwfF4I6sg.png?1756048807)

## 기술적 참고사항 및 제한사항

- **속도 제한:** 기본적으로 각 사용자는 5분마다 최대 500개의 알림(이메일, SMS 또는 인앱)을 받을 수 있습니다.

- **전제조건:** 전화번호, 이메일 주소 및 왓츠앱 연동이 적절히 설정되어 있는지 확인하세요.

- **외부 사용자:** 내부 알림은 Hyperclass 내부의 팀원만을 위한 것이며, 연락처나 클라이언트를 위한 것이 아닙니다.

- **CC/BCC 제한사항:** 이메일 알림에서만 사용 가능합니다. 각 CC와 BCC 수신자는 이메일 사용량/청구를 증가시킵니다.

## **연락처 없이 내부 알림 실행하기**

내부 알림은 알림 유형과 수신자 선택에 따라 무연락처 시나리오에서 실행될 수 있습니다. 이는 액션이 성공적으로 실행되기 위해 항상 연락처 기록이나 연락처 데이터를 필요로 하지 않는다는 의미입니다.

- 인앱 내부 알림: 연락처가 있거나 **없이도** 실행될 수 있습니다.

- 단순한 무연락처 메시지: 알림이 단순한 내부 메시지만 보내고 연락처 관련 필드나 커스텀 필드에 의존하지 않는 경우, **무연락처** 플로우에서도 예상대로 실행될 수 있습니다.

- 동적/연락처 관련 데이터: 알림 콘텐츠가 배정된 사용자 세부정보, 연락처 필드 또는 커스텀 필드와 같은 연락처별 정보에 의존하는 경우, 이러한 값들은 **무연락처** 플로우에서 올바르게 채워지지 않을 수 있습니다.

### **무연락처 진입 동작**

연락처 없이 내부 알림이 트리거될 때, 사용자는 해당 진입과 연결된 일반적인 연락처 세부정보를 볼 수 없을 수 있습니다. 이러한 경우 알림은 표준 연락처 프로필 뷰 대신 영숫자 식별자와 함께 나타날 수 있습니다. 이는 무연락처 실행에 대한 예상되는 동작입니다.

### 이메일/SMS 수신자 처리

| 수신자 옵션 | 동작 |
|------------|------|
| **All Users(모든 사용자)** | 사용자의 구성된 이메일 주소나 전화번호로 전송합니다. |
| **Particular Users(특정 사용자)** | 선택된 사용자들에게 전송합니다. |
| **Assigned User(배정된 사용자)** | 연락처가 존재하고 배정된 사용자가 있으며, SMS의 경우 유효한 전화번호가 있을 때만 전송합니다. 연락처나 배정된 사용자가 없으면 해당 단계는 건너뛰고 워크플로우는 계속 진행됩니다. |

## 자주 묻는 질문

**Q: 워크플로우에서 내부 알림이 실행된 것으로 표시되는데, 사용자가 알림을 받지 못했거나 대화/프로필에서 볼 수 없는 이유는 무엇인가요?**

일반적인 원인은 해당 사용자 프로필이 이전에 DND(Do Not Disturb, 방해금지)로 표시되었기 때문입니다. 경우에 따라 워크플로우는 알림이 실행된 것으로 표시할 수 있지만, 사용자는 여전히 관련 이메일이나 알림 기록을 예상대로 볼 수 없을 수 있습니다. 해결 방법:

- 연락처(Contacts)로 이동하여 해당 연락처/프로필을 엽니다.
- 좌측 패널에서 DND를 찾아 끄고 다시 켜서 프로필 상태를 새로고침합니다.
- 워크플로우를 다시 테스트합니다.

**Q: 사용자가 여러 수신자 카테고리에 나열된 경우(예: 배정된 사용자, CC, 팀) Hyperclass는 전달을 어떻게 처리하나요?**

Hyperclass는 카테고리 간 수신자를 자동으로 중복 제거합니다. 이는 같은 사용자가 여러 수신자 필드(예: CC와 사용자 역할)에 추가된 경우, 중복을 방지하여 하나의 알림만 받는다는 의미입니다.

**Q: 선택한 알림 유형(SMS, 이메일, 왓츠앱)이 수신자에 대해 적절히 구성되지 않은 경우 어떻게 되나요?**

선택한 채널이 구성되지 않은 경우(예: 사용자가 SMS를 위한 유효한 전화번호가 없거나 왓츠앱 연동이 연결되지 않은 경우), 해당 특정 채널의 알림은 실패합니다. 그러나 이것이 워크플로우가 다음 액션으로 계속 진행하는 것을 막지는 않습니다.

**Q: 내부 알림의 첨부파일이 저장 공간이나 메시지 제한에 포함되나요?**

네. 이메일, SMS 또는 왓츠앱 알림에 포함된 첨부파일은 계정의 파일 저장 공간에 포함되며, 채널에 따라 메시지 크기 제한에도 기여할 수 있습니다(예: SMS 통신사는 파일 크기 제한을 적용). 너무 큰 파일은 전달을 방해할 수 있으므로 첨부파일을 항상 작고 최적화된 상태로 유지하세요.

**Q: 워크플로우가 대량으로 실행될 때(예: 수천 개의 연락처를 한 번에 등록할 때) 내부 알림은 어떻게 영향을 받나요?**

알림은 사용자당 5분마다 500개의 Hyperclass **속도 제한**을 받습니다. 워크플로우가 대량 알림을 트리거하는 경우, 이 임계값을 초과하는 추가 메시지는 대기열에 들어갑니다. 시스템 부하에 따라 제한이 재설정되면 전달되거나 워크플로우 조건이 만료되면 삭제될 수 있습니다.
