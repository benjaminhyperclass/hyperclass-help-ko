# 워크플로우 액션 - 수학 연산 (Math Operation)

이 문서에서는 워크플로우에서 수학 연산(Math Operation) 액션을 사용하여 숫자나 커스텀 필드 간의 덧셈, 뺄셈, 곱셈, 나눗셈 등 간단한 계산을 수행하는 방법을 설명합니다. 또한 복잡한 계산을 여러 단계로 나누어 처리하는 방법도 다룹니다.

**중요**: 현재 연산 순서(PEMDAS 또는 BODMAS)가 필요한 복잡한 수식은 하나의 수학 연산 액션에서 지원되지 않습니다. 사용자는 여러 단계를 거쳐 수동으로 계산을 나누어야 합니다.

## 수학 연산 워크플로우 액션이란?

수학 연산 워크플로우 액션을 사용하면 워크플로우 내에서 직접 다양한 계산을 수행할 수 있습니다. 숫자 필드를 더하고, 빼고, 곱하고, 나누어서 특정 액션이나 트리거에 따라 자동으로 업데이트할 수 있습니다. 날짜 필드의 경우 일, 월, 년을 더하거나 빼서 리마인더나 일정 관리와 같은 시간에 민감한 프로세스를 쉽게 자동화할 수 있습니다.

예를 들어, 구독 서비스에서는 수학 연산 액션을 사용하여 시작 날짜에 특정 기간을 더해 갱신 날짜를 계산하고, 고객이 적시에 알림을 받을 수 있도록 할 수 있습니다.

## 수학 연산 워크플로우 액션의 주요 장점

워크플로우에서 이 기능을 사용할 때의 주요 장점은 다음과 같습니다. 계산을 자동화하고 정확성을 향상시키며 프로세스를 원활하게 만드는 데 도움이 됩니다.

- **자동화 & 효율성**: 수동 계산을 없애고 오류를 줄이며 시간을 절약합니다. 리드 스코어링, 수수료 계산, 마감일 관리와 같은 반복적인 작업을 간소화합니다.

- **커스터마이징 & 연동**: 커스텀 필드와 값을 지원하여 유연한 설정이 가능하며, 다른 워크플로우 액션, 트리거, 조건 분기(If/Else)와 원활하게 연동되어 고급 자동화를 구현할 수 있습니다.

- **정확성 & 확장성**: 숫자, 통화, 날짜 필드의 정확한 업데이트를 보장하며, 추가적인 수동 작업 없이도 대용량 데이터를 처리하여 일관된 성능을 유지합니다.

## 수학 연산 워크플로우 액션 설정하기

이제 이 액션 사용의 장점을 이해했으니, 올바른 필드 선택, 계산 적용, 값 업데이트를 통해 수학 연산 액션을 설정하는 방법을 알아보겠습니다. 또한 동적 필드, 여러 연산, 커스텀 값을 활용하여 계산을 정확하고 자동화된 상태로 유지하는 방법도 배워보겠습니다.

### 자동화 메뉴로 이동

Automation(자동화) 섹션으로 이동하여 워크플로우를 선택하거나 생성합니다. 수학 연산 액션을 추가할 새 워크플로우를 만들거나 기존 워크플로우를 선택합니다.

![자동화 메뉴 이동](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040699092/original/NToaC5zQG_CWDZ9waZNDP9ev6dPN9EZQ0A.png?1738225302)

### 수학 연산 액션 추가하기

수학 연산 액션을 추가하기 전에 워크플로우 트리거를 설정하는 것이 중요합니다. 트리거는 워크플로우가 언제 활성화되어 수학 연산을 실행할지 결정합니다. 트리거가 설정되면 "Add Action(액션 추가)"를 클릭하고 목록에서 "Math Operation(수학 연산)"을 선택하여 워크플로우 내에서 계산을 설정합니다.

![수학 연산 액션 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040699125/original/3tpRQ11Rb27zCnrIPhlbXG_CKPygq6Riug.png?1738225328)

### 액션 이름 지정

기본적으로 액션 이름은 "Math Operation"으로 되어 있지만, 명확성을 위해 이름을 바꿀 수 있습니다. 워크플로우를 더 쉽게 따라갈 수 있도록 설명적인 이름을 사용하세요. 명확한 이름은 특히 복잡한 워크플로우에서 액션을 빠르게 식별하는 데 도움이 됩니다.

![액션 이름 지정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040699202/original/FCfvGnFRxkKEw-Lqwu48qGT-fowWcRBwkA.png?1738225368)

### 필드 선택

액션 이름을 지정한 후, 다음 단계는 계산이 적용될 필드를 선택하는 것입니다. "Select Field(필드 선택)" 드롭다운을 클릭하면 세 가지 옵션이 나타납니다:

- **날짜 필드**: 이 옵션을 사용하면 현재 날짜(Current Date)와 생년월일(Date of Birth)과 같은 시스템 생성 날짜 필드를 사용하여 일정 관리, 리마인더, 나이 확인을 위한 날짜 기반 계산을 수행할 수 있습니다. 각 옵션을 이해해 보겠습니다:
  - 현재 날짜(Current Date): 워크플로우가 실행될 때의 현재 날짜를 나타냅니다. 마감일 계산, 후속 리마인더, 또는 오늘 날짜를 기준으로 한 미래 이벤트 일정 등의 작업에 유용합니다.
  - 생년월일(Date of Birth): 연락처에 기록된 생년월일을 가리킵니다. 이 옵션은 나이 확인, 생일 리마인더, 기념일 계산 등에 자주 사용됩니다.

- **커스텀 필드**: CRM 설정에 특정한 필드들로, 보험 만료일이나 계약 시작일과 같은 필드입니다. 커스텀 필드를 선택하면 시스템에 저장된 데이터를 기반으로 계산을 수행할 수 있어, 자동 갱신 리마인더, 계약 연장, 마일스톤 추적 설정에 유용합니다.

- **커스텀 값**: 이 옵션을 사용하면 저장된 필드를 선택하는 대신 고정 값을 수동으로 입력할 수 있습니다. 날짜 필드에 특정 일수를 더하거나 리드 스코어나 가격 조정에 대한 퍼센트 증가를 계산하는 등의 고정 계산을 적용할 때 유용합니다.

![필드 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040699271/original/w7SprVSTGgJKJt5Viv52QjPdR8j-q7PPCA.png?1738225424)

### 연산 유형

날짜 필드, 커스텀 필드, 또는 커스텀 값 중 하나의 필드를 선택한 후, 다음 단계는 적용하고자 하는 계산 유형을 정의하는 것입니다.

**날짜 필드**: 날짜 필드에서는 일, 월, 년을 더하거나 빼는 유연성을 가지며, 자동 리마인더, 마감일, 일정 조정을 쉽게 설정할 수 있습니다.

![날짜 필드 연산](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040699781/original/kyZjAWaK4Mf5TolqKX4E_1XHg1oVTCbFig.png?1738225730)

**숫자 및 통화 필드**: 숫자 필드에서는 덧셈, 뺄셈, 곱셈, 나눗셈과 같은 연산을 수행하여 값을 자동으로 업데이트할 수 있습니다.

![숫자 필드 연산](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040699770/original/TaSK-vRWZJ1vL3tijr5EwKBDAteo2RWldw.png?1738225716)

### 연산 값 설정

연산 유형을 선택한 후, 계산을 위한 숫자 값을 입력해야 합니다. 고정 숫자를 수동으로 입력하거나 "+" 및 "-" 아이콘을 사용하여 필요에 따라 값을 빠르게 조정할 수 있습니다.

### 태그 아이콘

태그 아이콘을 사용하면 고정 숫자를 수동으로 입력하는 대신 커스텀 필드에서 동적 값을 선택할 수 있습니다. 이를 통해 계산이 실시간 데이터 변경에 자동으로 적응하여 워크플로우를 더욱 유연하고 효율적으로 만들 수 있습니다.

![태그 아이콘](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040700033/original/MvgquKYjg7H11hMO6Ze4Xn8W3qWY4ziyhA.png?1738225929)

### 삭제 아이콘

삭제 아이콘(휴지통)을 사용하면 더 이상 필요하지 않은 연산을 제거할 수 있습니다. 이 기능을 통해 전체 수학 연산 액션을 다시 구축하지 않고도 빠른 조정이 가능하여 계산을 깔끔하고 정확하게 유지할 수 있습니다. 실수가 있거나 연산이 더 이상 필요하지 않으면 휴지통을 클릭하여 삭제하세요.

### 여러 연산 추가하기

하나의 수학 연산 액션 내에서 여러 수학 연산을 추가하려면 Add Operator(연산자 추가) 옵션을 사용합니다. 이 기능은 먼저 값을 더한 다음 결과에 곱하는 것과 같은 일련의 계산을 수행할 때 유용합니다. Add Operator를 클릭하면 별도의 워크플로우 액션이 필요 없이 계속해서 계산을 세밀하게 조정할 수 있어 복잡한 자동화 설정을 더욱 효율적으로 만들 수 있습니다.

![여러 연산 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040700144/original/aj2JjzQ031NeRIPy6e1get9w2Nad09suyQ.png?1738226001)

### 업데이트 필드

Update Field(업데이트 필드) 옵션은 계산 결과가 저장될 위치를 결정합니다. 선택한 필드가 처리되는 데이터 유형과 일치하는지 확인해야 합니다. 숫자 필드는 숫자 값으로 업데이트하고, 날짜 필드는 날짜 기반 계산을 저장해야 합니다.

![업데이트 필드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040700168/original/I86tyUHLU3FGPqLSGNTw-o617v7HV9a7aw.png?1738226023)

### 저장 및 발행

**"Save(저장)"**을 클릭하여 수학 연산 액션을 워크플로우에 적용합니다. 워크플로우 설정이 완료되면 **"Publish(발행)"**를 클릭하여 워크플로우를 활성화합니다.

## 자주 묻는 질문

**Q. 커스텀 날짜 필드에서 수학 연산을 수행할 수 있나요?**

네, 커스텀 날짜 필드에서 덧셈이나 뺄셈을 수행하여 리마인더나 일정 관리와 같은 시간에 민감한 작업을 자동화할 수 있습니다.

**Q. 수학 연산에서 값의 범위에 제한이 있나요?**

숫자 필드는 양수와 음수 값을 모두 저장할 수 있지만, 통화 필드는 0 이상의 값만 허용합니다. 날짜 필드에서는 음수 값이 지원되지 않습니다.

**Q. 수학 연산 액션을 조건 분기(If/Else)와 함께 사용할 수 있나요?**

물론입니다! 수학 연산 액션의 결과를 조건 분기에서 사용하여 동적 워크플로우를 만들 수 있습니다. 예를 들어, 계산된 리드 스코어를 사용하여 참여 수준에 따라 리드를 다른 영업 경로로 안내할 수 있습니다.

**Q. 여러 수학 연산을 한 번에 수행할 수 있나요? (예: 1 + 2 × 3 - 4 ÷ 5)**

현재 하이퍼클래스의 수학 연산 워크플로우 액션은 한 번에 하나의 간단한 연산만 지원합니다(두 값 간의 덧셈, 뺄셈, 곱셈, 또는 나눗셈).

중요사항:
- 이 액션은 자동으로 연산 순서(PEMDAS 또는 BODMAS)를 적용하지 않습니다.
- 복잡한 계산을 수행해야 하는 경우, 올바른 수학적 순서를 따라 여러 수학 연산 단계로 나누고 중간 결과를 저장할 커스텀 필드를 사용해야 합니다.
