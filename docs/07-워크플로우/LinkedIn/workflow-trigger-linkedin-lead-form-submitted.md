# 워크플로우 트리거 - LinkedIn 리드 폼 제출됨

LinkedIn 리드 생성 폼에서 새 리드가 제출되는 즉시 캡처하여 Hyperclass 워크플로우로 연결하세요. 이 가이드에서는 **LinkedIn 리드 폼 제출됨** 트리거가 무엇인지, 언제 실행되는지, 그리고 **페이지**나 커스텀 필드 등의 필터를 설정하여 정확한 라우팅을 하는 방법을 설명합니다. 또한 설정 단계, 테스트 팁, 그리고 문제 해결 방법도 포함되어 있어서 팀이 모든 LinkedIn 리드에 빠르게 대응할 수 있습니다.

---

# **LinkedIn 리드 폼 제출됨 트리거란?**

**LinkedIn 리드 폼 제출됨** 트리거는 Hyperclass 계정에 연결된 LinkedIn 리드 생성 폼을 제출한 직후 연락처를 워크플로우에 등록시킵니다. 이를 사용하여 즉시 후속 조치를 시작하고, 리드를 담당자에게 배정하고, 기회를 생성하고, 리드가 유입된 경로와 제공한 데이터를 기반으로 양육 자동화를 시작할 수 있습니다.

트리거 세부사항

| 필드 이름 | 설명 | 필수 여부 |
|---|---|---|
| Page | 리드 생성 폼을 소유한 LinkedIn 페이지입니다. 특정 페이지를 선택하여 어떤 제출이 워크플로우에 들어올지 범위를 정할 수 있습니다. | 아니오 |

### 필터에서 사용 가능한 연산자

| 연산자 | **설명** |
|---|---|
| **정확히 일치(Is)** | 필드가 선택한 값과 정확히 일치할 때 트리거됩니다. |
| **문구 포함(Contains Phrase)** | 필드가 입력한 문구를 포함할 때 트리거됩니다. |
| **문구 포함하지 않음(Does Not Contain Phrase)** | 필드가 입력한 문구를 포함하지 않을 때 트리거됩니다. |
| **비어있음(Is Empty)** | 필드에 값이 없을 때 트리거됩니다. |
| **비어있지 않음(Is Not Empty)** | 필드에 값이 있을 때 트리거됩니다. |

## **LinkedIn 리드 폼 제출됨의 주요 이점**

이점을 이해하면 더 빠르고 깔끔한 워크플로우를 설계하고 전환율을 개선할 수 있습니다. 이러한 장점을 활용하여 LinkedIn 리드를 필터링하고, 라우팅하고, 후속 조치하는 방법을 계획하세요.

- **속도-투-리드:** 제출 즉시 트리거되어 바로 문자, 이메일, 또는 전화를 걸 수 있습니다.

- **정확한 라우팅:** **페이지**나 매핑된 커스텀 필드로 필터링하여 올바른 담당자나 파이프라인에 배정할 수 있습니다.

- **깔끔한 데이터:** 필드 매핑 및 중복 제거와 결합하여 단일하고 정확한 연락처 기록을 유지할 수 있습니다.

- **자동화된 후속 조치:** 수동 작업 없이 양육 시퀀스, 할 일, 기회 생성을 시작할 수 있습니다.

- **보고서 명확성:** 일관된 소스 추적으로 캠페인 성과를 측정하는 데 도움이 됩니다.

---

## **사전 요구 사항**

이러한 사전 요구 사항을 완료하면 "리드가 들어오지 않는" 문제를 방지하고 테스트를 간단하게 만들 수 있습니다.

- **설정(Settings) → 연동(Integrations)**에서 올바른 **LinkedIn 페이지**와 **광고 계정**에 액세스할 수 있는 LinkedIn 리드 광고 연동이 연결되어 있어야 합니다.

- 해당 페이지에 최소 하나 이상의 **LinkedIn 리드 생성 폼**이 게시되어 있어야 합니다.

- 필드 매핑 계획이 수립되어 있어야 합니다(예: 이메일, 전화번호, 회사명, 직책을 매핑하고, 틈새/자격 요건을 Hyperclass의 해당 커스텀 필드에 매핑).

## 필터 논리 및 모범 사례

필터를 올바르게 결합하면 자격을 갖춘 제출만 자동화에 들어오게 하고 실수로 등록되는 것을 방지할 수 있습니다.

- 트리거의 여러 필터는 함께 평가됩니다(AND 논리).

- 넓게 시작하고(예: **페이지 → 정확히 일치 → 귀하의 페이지**) 그 다음 자격 요건을 추가하세요.

- 초기 런칭 시 과도한 필터링을 피하세요. 리드가 흘러들어오는 것을 확인한 후 자격 요건을 추가하세요.

- 페이지별 워크플로우를 단순하게 유지하세요. 페이지당 하나의 워크플로우가 많은 복잡한 조건보다 관리하기 쉽습니다.

---

## 권장 워크플로우 설정

몇 가지 워크플로우 설정으로 테스트를 쉽게 만들고 중복 아웃리치를 방지할 수 있습니다.

- **재진입 허용(Allow Re-entry):** 테스트 중이거나 리드가 여러 번 제출할 수 있는 경우 활성화하세요. 첫 번째 제출만 트리거해야 하는 경우 비활성화하세요.

- **응답 시 중지(Stop on Response, 선택사항):** 리드가 답장할 때 양육을 일시정지하는 데 사용하세요.

- **등록 로그(Enrollment Logs):** 실행 로그를 사용하여 제출 후 각 리드의 경로를 확인하세요.

## **LinkedIn 리드 폼 제출됨 설정 방법**

다음 단계에 따라 트리거를 구성하고, 필터를 추가하고, 저장하세요. 정확한 설정으로 원하는 리드만 워크플로우에 들어와 올바른 액션을 받게 됩니다.

### **워크플로우 열기**

왼쪽 네비게이션에서 **자동화(Automation) → 워크플로우(Workflows)**로 이동하여 우상단의 **워크플로우 생성(Create Workflow)**을 클릭하세요. **처음부터 시작(Start from Scratch)**(또는 **레시피 선택(Select a Recipe)**)을 선택하여 새 빌더 캔버스를 여세요.

![워크플로우 생성 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057970620/original/bjb7_ZM6As5IQnHDubVlwOgeHmbZ6pRfkQ.png?1762770898)

### **트리거 추가**

워크플로우 빌더에서 **+ 추가(Add)**에 마우스를 올리고 **트리거 추가(Add Trigger)**를 선택하세요. 화면 오른쪽에 트리거 구성 패널이 열립니다.

![트리거 추가 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057970757/original/ZO4Sp2ScsO5zON3zyKR7MraIqsAcKr8MJg.png?1762770937)

### **트리거 선택**

검색 바를 사용하거나 스크롤하여 **LinkedIn 리드 폼 제출됨(LinkedIn Lead Form Submitted)**을 찾아 선택하세요. 이렇게 하면 LinkedIn 리드 생성 이벤트를 워크플로우의 등록 소스로 선택합니다.

### **트리거 이름 지정**

팀원들이 한눈에 소스를 알아볼 수 있도록 명확한 **워크플로우 트리거 이름(Workflow Trigger Name)**을 입력하세요. "LinkedIn 리드 폼 제출됨" 또는 "LinkedIn 리드 — 페이지 범위"와 같은 이름이 좋습니다.

**필터 추가(Add filters)**를 클릭하여 **표준 필드(Standard Fields)**(예: **페이지**) 및 **커스텀 필드(Custom Fields)**가 나열된 필드 선택기를 여세요. 검색 박스를 사용하여 비즈니스 틈새 또는 업종과 같은 매핑된 필드를 빠르게 찾을 수 있습니다.

**페이지 → 정확히 일치(Is)**를 설정한 다음, 드롭다운에서 특정 LinkedIn 페이지를 선택하세요. 이렇게 하면 워크플로우 등록을 해당 페이지에서만 오는 제출로 제한합니다.

## 트리거 테스트

테스트를 통해 연동, 매핑, 필터가 라이브 전에 작동하는지 확인할 수 있습니다.

- LinkedIn 캠페인 관리자에서 연결된 페이지와 폼에 대해 테스트 리드를 제출하세요.

- Hyperclass에서 **연락처(Contacts)**를 열고 매핑된 필드와 함께 연락처가 생성/업데이트되었는지 확인하세요.

- 워크플로우의 **등록 내역 / 실행 로그(Enrollment History / Execution Logs)**를 열어 트리거가 실행되고 액션이 실행되었는지 확인하세요.

- 등록을 방해한 필터가 있는지 검토하고 필요에 따라 조정하세요.

## 자주 묻는 질문

**Q: 페이지 필터를 반드시 설정해야 하나요?**

아니오. 필터는 선택사항이지만, **페이지** 필터를 사용하면 등록을 특정 LinkedIn 페이지로 범위를 제한할 수 있습니다.

**Q: 정확한 LinkedIn 리드 생성 폼으로 필터링할 수 있나요?**

매핑이 폼 식별자를 노출하거나 폼 이름을 나타내는 커스텀 필드를 사용하는 경우, 해당 값으로 필터링할 수 있습니다. 그렇지 않으면 페이지 범위 지정과 트리거 후 조건을 사용하세요.

**Q: 중복은 어떻게 처리되나요?**

Hyperclass는 이메일/전화번호와 같은 식별자로 매칭을 시도합니다. 일치하는 항목이 있으면 연락처가 업데이트되고, 없으면 새 연락처가 생성됩니다. 중복 제거 환경설정을 조정하고 필요시 병합하세요.

**Q: 재제출 시 워크플로우가 다시 트리거되나요?**

**재진입 허용(Allow Re-entry)** 설정에 따라 다릅니다. 테스트나 리드가 여러 번 제출할 수 있는 캠페인의 경우 활성화하세요.

**Q: 매핑된 필드가 비어있으면 어떻게 되나요?**

**비어있음 / 비어있지 않음(Is Empty / Is Not Empty)** 연산자를 사용하여 해당 리드를 포함하거나 제외하거나, LinkedIn 폼을 조정하여 해당 답변을 필수로 만드세요.

**Q: 리드가 Hyperclass에 얼마나 빨리 나타나나요?**

리드는 일반적으로 제출 직후 동기화됩니다. 테스트 중에는 약간의 지연을 허용하고 실행 로그를 확인하세요.

**Q: 다른 페이지에 대해 별도의 워크플로우를 실행할 수 있나요?**

예. 페이지별로 하나의 워크플로우를 생성하거나 분기 액션과 함께 **페이지 → 정확히 일치(Is)** 필터를 사용하세요.

**Q: 트리거가 실행되었는지 어디서 확인할 수 있나요?**

워크플로우의 **등록 내역(Enrollment History)** 및 **실행 로그(Execution Logs)**를 열어 항목과 액션 경로를 확인하세요.
