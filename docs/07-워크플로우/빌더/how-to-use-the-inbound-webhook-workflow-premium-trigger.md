# 인바운드 웹훅 워크플로우 프리미엄 트리거 사용법

인바운드 웹훅(Inbound Webhook)은 외부 시스템이 GET, POST, PUT과 같은 다양한 HTTP 요청 방법을 사용하여 자동으로 데이터를 전송할 수 있게 해주어 원활한 CRM 연동을 가능하게 합니다. 이러한 실시간 데이터 전송 기능은 CRM 기능을 향상시키고 워크플로우를 간소화하여, 다양한 플랫폼과 도구 간의 효율적인 커뮤니케이션과 협업을 촉진합니다.

- 모든 요금제($97, $970, $297, $2970, $497, $4970)의 에이전시는 LC 프리미엄 트리거 & 액션에 액세스할 수 있습니다.
- 에이전시 설정을 통해 프리미엄 액션 & 트리거가 활성화되면, 기존 및 새 하위 계정은 100회의 무료 실행을 받게 됩니다.
- 에이전시가 기존 하위 계정에 대한 실행 비용 발생을 방지하려면, 에이전시 뷰 내에서 각 하위 계정에 대해 수동으로 재청구를 활성화해야 합니다(자세히 보기).
- SaaS 설정기에서 프리미엄 액션이 활성화된 경우, 생성된 새 하위 계정은 자동으로 LC 프리미엄 액션 & 트리거에 등록되며, 에이전시의 추가 조치가 필요하지 않습니다.

## 인바운드 웹훅이란?

인바운드 웹훅은 외부 시스템에서 CRM으로 자동 데이터 전송을 지원하여 연동과 상호 운용성을 향상시키는 강력한 기능입니다. 외부 시스템에서 이벤트가 발생하면, HTTP 요청(POST, GET, 또는 PUT)이 CRM 내의 트리거와 연결된 특정 URL로 전송되어 워크플로우가 시작됩니다. 이러한 실시간 데이터 전송 기능은 CRM의 기능성을 향상시키고 다양한 플랫폼과 도구 간의 워크플로우를 간소화합니다.

인바운드 웹훅을 활용하면 비즈니스는 프로세스를 자동화하고, 수동 작업을 줄이며, 인적 오류를 최소화하여 효율성과 생산성을 높일 수 있습니다. 또한 중요한 데이터가 다양한 시스템 간에 쉽게 이용 가능하고 동기화되므로 팀 간의 협업을 개선합니다. 이러한 원활한 연동은 궁극적으로 의사결정, 고객 만족도, 비즈니스 성과를 향상시킵니다.

### 어떤 활용 사례가 있나요?

인바운드 웹훅은 CRM 기능을 향상시키고 워크플로우를 자동화하는 다양한 시나리오에서 활용할 수 있습니다. 다음은 실용적인 활용 사례와 예시입니다:

전자상거래 주문 연동:

온라인 쇼핑몰을 운영하고 고객이 주문할 때 새로운 주문 정보로 CRM을 업데이트하고 싶다면, CRM에 인바운드 웹훅을 설정하여 새 주문이 생성될 때마다 전자상거래 플랫폼에서 데이터를 받을 수 있습니다.

예시: 전자상거래 플랫폼이 고객 이름, 이메일, 전화번호, 주문 금액과 같은 주문 세부사항과 함께 웹훅 URL로 HTTP POST 요청을 보냅니다. 그러면 CRM은 고객의 연락처 정보를 생성하거나 업데이트하고 주문 세부사항을 해당 기록에 추가합니다.

고객 지원 티켓 시스템 연동:

별도의 지원 티켓 시스템을 사용하여 고객 문제를 처리하는 경우, 고객이 지원 티켓을 생성할 때 인바운드 웹훅을 사용하여 티켓 세부사항을 CRM으로 전송할 수 있어 지원팀이 필요한 모든 정보를 확인할 수 있습니다.

예시: 지원 티켓 시스템이 고객 이메일, 티켓 제목, 설명을 포함한 티켓 세부사항과 함께 웹훅 URL로 HTTP POST 요청을 보냅니다. 그러면 CRM은 티켓을 해당 고객 기록과 연결하여 지원팀이 CRM에서 직접 티켓 정보에 액세스할 수 있게 합니다.

이벤트 등록 연동:

이벤트를 주최하고 이벤트 등록 플랫폼을 사용하는 경우, 인바운드 웹훅을 사용하여 참석자 등록 정보를 CRM으로 전송할 수 있습니다.

예시: 이벤트 등록 플랫폼이 이름, 이메일, 전화번호, 이벤트명과 같은 참석자 정보와 함께 웹훅 URL로 HTTP POST 요청을 보냅니다. 그러면 CRM은 참석자의 연락처 정보를 생성하거나 업데이트하고 이벤트 등록 세부사항을 해당 기록에 추가하여 이벤트 팀이 참석자를 효과적으로 관리하고 소통할 수 있게 합니다.

웹 폼에서 리드 수집:

인바운드 웹훅은 웹사이트나 랜딩 페이지의 웹 폼에서 리드를 수집하여 데이터를 CRM으로 전송할 수 있습니다.

예시: 방문자가 웹사이트에서 폼을 제출하면, 웹 폼이 이름, 이메일, 전화번호와 같은 리드의 정보와 함께 웹훅 URL로 HTTP POST 요청을 보냅니다. 그러면 CRM은 새 리드를 생성하거나 기존 연락처를 업데이트하여 영업팀이 리드와 즉시 후속 조치를 취할 수 있게 합니다.

이는 인바운드 웹훅이 다양한 비즈니스 시나리오에서 워크플로우를 자동화하고, 프로세스를 간소화하며, CRM 기능을 향상시킬 수 있는 방법의 몇 가지 예시에 불과합니다.

## 인바운드 웹훅 워크플로우 트리거 사용법

간단히 말해, 인바운드 웹훅 트리거를 사용하여 3가지 간단한 단계로 시스템에 데이터를 가져올 수 있습니다.

 
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48296396414/original/5xlRF80hbq19hXS2IeQjhZ5GlPK6xGxPLQ.png?1683557063)

#### 1단계

웹훅 URL을 가져와서 애플리케이션이나 시스템에서 사용합니다. 다음으로, 적절한 HTTP 요청 방법(POST, GET, 또는 PUT)을 사용하여 웹훅 URL로 데이터를 전송하고 "Test Trigger(트리거 테스트)" 버튼을 클릭합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48296396436/original/IlVVYc7eyGT9KdCkdq9wlbT68qjQH71BzA.jpeg?1683557066)

#### 2단계

지원되는 HTTP 요청 방법(POST, GET, 또는 PUT)을 사용하여 애플리케이션이나 시스템에서 받은 데이터를 선택하고 향후 참조를 위해 샘플 데이터로 저장합니다. 추가 최근 데이터를 보려면 "Load More(더 보기)"를 클릭합니다. 원하는 데이터를 선택했으면 트리거를 저장합니다.

이 단계에서는 커스텀 값(custom values)에 대한 데이터 참조를 저장할 수 있으며, 이는 다른 연락처 커스텀 필드나 로케이션 커스텀 값과 유사하게 처리할 수 있습니다.

다음 단계와 아래 예시에서 이 참조를 활용하는 방법에 대한 자세한 정보를 제공합니다.

트리거를 저장하면 "Create Update Contact(연락처 생성/업데이트)" 액션으로 이동합니다. 여기서 필요한 필드를 선택하고 트리거에서 받은 수신 데이터를 매핑할 수 있습니다. 새 연락처를 생성하거나 기존 연락처에 데이터를 매핑하려면 이메일 또는 전화 필드를 제공해야 합니다. 트리거 데이터에 이메일이나 전화가 없는 경우 워크플로우에서 이 액션을 제거할 수 있으며, 워크플로우는 연락처 없이도 계속 실행됩니다.

## 워크플로우가 연락처 없이 실행될 수 있나요?

- 워크플로우는 연락처 데이터 종속성 없이 연락처 없는 상태로 실행될 수 있으므로 인바운드 웹훅 트리거를 통해 모든 페이로드 데이터를 전송하고 워크플로우에서 사용할 수 있습니다.
- 연락처 없이 진행하여 연락처 정보에 종속되지 않는 액션을 사용할 수 있습니다. 커스텀 웹훅, Google 시트, Slack, ChatGPT 및 모든 내부 도구는 연락처 없이 실행될 수 있습니다.
- 필요한 경우 연락처 생성/업데이트 또는 연락처 찾기 액션을 사용하여 워크플로우에 연락처 데이터를 가져올 수 있습니다.

**예시:**
- 인바운드 웹훅을 통해 주문 데이터를 전송하고 주문 정보를 Google 시트에 추가하며, if/else를 사용하여 주문 금액에 따라 분류하고 Slack 알림을 보냅니다.
- 연락처 찾기 액션을 사용하여 연락처 ID로 연락처를 검색합니다.

#### 고려해야 할 중요 사항:

- 웹훅과 상호 작용할 때 항상 지원되는 방법(POST, GET, 또는 PUT)을 사용하여 요청을 보냅니다.
- 데이터가 유일하게 지원되는 데이터 형식인 JSON 객체로 전송되는지 확인합니다.
- 올바르게 컴파일하려면 키는 공백으로 구분되지 않는 단일 문자열이어야 합니다. 키 이름을 공백으로 구분하는 대신 CamelCase 또는 snake_case 사용을 고려하세요.
- 워크플로우에는 연락처 정보가 필요하므로 페이로드에 이메일 또는 전화번호를 제공하는 것이 필수입니다. 연락처를 찾거나 생성하려면 이메일 또는 전화가 필요합니다.
- 배열(Array)은 커스텀 값에서 지원되지 않습니다. 요청에서 보낼 수는 있지만 액션 내에서 사용할 수는 없습니다.
- 데이터 구조가 변경되는 경우, 다른 액션에서 해당 필드를 올바르게 처리하기 위해 인바운드 웹훅 트리거 설정 내에서 매핑 참조를 다시 선택합니다.
- 인바운드 웹훅 트리거 URL이 노출되거나 유출되어 원치 않는 요청을 방지하고 싶다면, 기존 인바운드 웹훅 트리거를 삭제하고 새 인바운드 웹훅 트리거를 추가합니다. 다른 종료 ID를 가진 새 URL이 생성되므로 새 URL에 맞게 연동을 업데이트합니다. 저장 후, 이전 URL의 수신 요청은 워크플로우에 들어가지 않으며 새 URL의 요청만 들어갑니다.
