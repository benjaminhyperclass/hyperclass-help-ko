# 워크플로우 트리거 - 주문 제출

주문 제출(Order Submitted) 트리거는 고객이 주문을 제출한 후 업셀을 제안할 수 있는 강력한 새로운 워크플로우 트리거입니다. 이 트리거는 1단계와 2단계 주문 폼 모두에서 작동하여, 사용하는 주문 폼 유형이나 구매한 상품 종류에 따라 업셀을 유연하게 맞춤 설정할 수 있습니다.

## 주문 제출 트리거는 언제 실행되나요?

**참고 사항**

메인 상품 구매 후 바로 업셀에 대해서도 같은 연락처에 대해 트리거가 실행되도록 하려면, 워크플로우 설정(Workflow Settings)에서 중복 허용(Allow Multiple Setting)을 켜두어야 합니다.

만약 **대기 단계**를 넣어서 첫 번째 인스턴스(메인 상품 구매로 인한)를 워크플로우에서 보류시키고, 고객이 업셀을 구매한다면, 고객이 여전히 워크플로우 안에 있기 때문에 업셀 구매로는 두 번째로 워크플로우에 들어갈 수 없습니다. 이 워크플로우에서는 대기 단계 사용을 피하세요. **같은 상품에 대해 업셀이나 범프를 판매할 계획이 없다면** 자유롭게 사용하셔도 됩니다.

---

이 트리거는 1단계 또는 2단계 주문 폼 제출과 업셀 시에 실행됩니다. 주문 폼이 제출될 때와 업셀 구매가 발생할 때 모두 실행되며, 각각의 경우에 상품 정보를 보관합니다. 즉, 최종 고객이 주문 폼에서 상품 A와 B를 구매하고 업셀에서 상품 C를 구매한다면, 두 번 실행됩니다(첫 번째는 A와 B 구매 시, 두 번째는 업셀 구매 시).

참고사항:

이것은 **버전 2** 퍼널에서만 작동합니다. (V2 퍼널에 대한 자세한 내용은 [이 글](../../06-사이트/기타/how-to-upgrade-a-version-1-funnel-to-version-2-.md)을 참조하세요) 버전 1 퍼널을 사용 중이라면 아래 표시된 버튼을 사용하여 버전 2로 업그레이드하세요:

## **주문 폼 제출 워크플로우 트리거와 어떻게 다른가요?**

---

주문 제출(Order Submitted)과 주문 폼 제출(Order Form Submitted) 워크플로우 트리거의 가장 중요한 차이점들은 다음과 같습니다:

- 여러 상품 구매 시 여러 번 실행되지 않습니다. 트리거는 단일 체크아웃 객체를 포함하여 주문 폼 제출 시 단 한 번만 정보를 전송합니다. 하나의 주문 세션에서 여러 상품을 구매하더라도 마찬가지입니다.
- 글로벌 상품/가격을 기준으로 트리거를 직접 필터링할 수 있습니다. 기존 트리거는 이 기능을 제공하지 않습니다.

![주문 제출 트리거 필터 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48271902305/original/HkW1R_dgtia5VEH8lw5XtAJoMKdamm2xog.png?1672224456)

- 워크플로우 내에서 주문 관련 커스텀 값을 사용할 수 있습니다. 여기에는 고객, 주문 세부사항, 쿠폰 정보, 결제 게이트웨이와 관련된 커스텀 값이 포함됩니다.

참고사항:

주문 제출 트리거가 워크플로우 트리거 단계 중 하나가 아닌 경우, 이러한 커스텀 값들은 커스텀 값 드롭다운에 표시되지 않습니다.

![주문 관련 커스텀 값](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48271903161/original/GfNiNTpIzmqHoleW7RVRvGJKHUBrcM8vow.png?1672224689)

- If 분기에서 주문 세부사항이나 상품 세부사항과 관련된 조건을 사용하여 장바구니 가격/구매한 상품/퍼널을 기준으로 워크플로우를 안내할 수 있습니다.

![조건 분기 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48271903365/original/Fy-l-g5na5xvLpx26kk97KGSVej6hjghwA.png?1672224749)

## 이 워크플로우를 통해 주문 확인을 위한 이메일 템플릿 사용하기

---

고객이 구매한 상품들은 이메일 빌더 템플릿의 쇼핑 카트 요소(Shopping Cart element)를 사용하여 채워지고, 해당 특정 템플릿을 사용하여 최종 고객에게 확인 이메일을 보낼 수 있습니다. 쇼핑 카트 요소는 구매 수량, 상품 이미지, 라인별 가격과 함께 상품 라인 항목을 자동으로 채웁니다.

참고사항:

쇼핑 카트 요소는 결제(Payments) ➝ 상품(Products) 페이지에 추가된 상품 이미지도 채웁니다. 상품에 이미지가 추가되지 않은 경우, 요소는 기본 이미지를 자동으로 채웁니다. 기본 시스템 생성 이미지를 피하기 위해 쇼핑 카트 요소를 사용할 때는 커스텀 상품 이미지를 사용하는 것을 강력히 권장합니다.

![쇼핑 카트 요소](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48271904923/original/uMFRmaT5jF11XlibbTZhrwlAA9Slk44k0w.png?1672225164)

값이 제대로 채워지려면 **쇼핑 카트** 요소가 템플릿에 추가되고, 같은 템플릿이 워크플로우의 이메일 보내기(Send Email) 액션에 추가되어야 합니다.

## **제출 유형 필터는 어떻게 작동하나요?**

**메인(Primary)** - 주문 폼의 체크아웃에서 메인 상품이 구매된 경우를 필터링합니다.

**범프(Bump)** - 주문 폼에서 범프 상품이 구매된 경우만 필터링합니다. 범프 상품이 구매되지 않았다면 트리거가 작동하지 않습니다.

**업셀(Upsell)** - 업셀 구매의 경우만 필터링합니다. 필터가 업셀로 설정된 경우, 주문 폼 제출(메인/범프 상품 구매) 시에는 트리거가 실행되지 않습니다.

![제출 유형 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48271933980/original/qbQT79u8jkDAXIjo053ldP95IVhs9kG4kw.png?1672231859)

## 이 트리거가 기존 주문 폼 제출 트리거에 영향을 주나요?

아니요, 기존 주문 폼 제출 트리거에는 어떤 변경도 가하지 않습니다. 하지만 주문 관련 커스텀 값, 쿠폰 정보, 상품 라인 항목을 활용하기 위해서는 주문 제출 트리거를 사용하는 것을 권장합니다.
