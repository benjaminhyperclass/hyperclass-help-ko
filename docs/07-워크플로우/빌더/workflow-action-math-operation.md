# 워크플로우 액션 - 수학 연산(Math Operation)

워크플로우의 수학 연산(Math Operation) 액션은 숫자와 날짜에 대한 계산을 수행할 수 있게 해주어, 자동화된 리드 스코어링과 일정 관리를 가능하게 합니다. 이 기능은 커스텀 변수와 필드를 지원하여 다양한 비즈니스 요구사항에 유연성을 제공하고, If/Else 조건과의 원활한 연동을 통해 리드 관리를 최적화합니다.

## 워크플로우의 수학 연산 액션이란?

워크플로우의 수학 연산 액션은 숫자 및 날짜 필드에 대한 계산과 조작을 자동화하여 비즈니스 프로세스와 의사결정을 간소화하는 강력한 도구입니다. 예를 들어, 헬스장이 회원의 갱신 날짜를 현재 멤버십 만료일에 1년을 더해 자동으로 업데이트하고 싶다고 가정해보겠습니다. 수학 연산 액션을 사용하면 헬스장은 이러한 계산을 수행하는 워크플로우를 손쉽게 설정하여 시간을 절약하고 정확성을 보장하면서 회원들에게 우수한 서비스를 제공하는 데 집중할 수 있습니다.

### 수학 연산의 우수한 사용 사례와 실제 비즈니스 문제 해결

**헬스장 멤버십 갱신:** 현재 만료일에 1년을 더해 멤버십 갱신 날짜를 자동 업데이트하여 고객에게 적시에 알림을 보내고 유지율을 개선합니다.

**대출 상환:** 각 상환 후 대출의 잔액을 계산하여 비즈니스의 재정 건전성을 추적합니다.

**영업 수수료:** 맞춤 수수료율과 달성된 영업 목표를 바탕으로 영업 대표의 수수료 금액을 계산하여 공정하고 정확한 보상을 보장합니다.

**리드 스코어링:** 전화 통화나 웹사이트 방문 등 리드가 비즈니스와 상호작용할 때마다 리드 점수를 증가시켜 관심도를 측정하고 후속 조치 우선순위를 정합니다.

**프로젝트 마감일:** 프로젝트 시작일에서 날짜를 더하거나 빼서 마감일을 계산하고 팀원들에게 자동 알림을 보냅니다.

**구독 만료:** 활성화 날짜에 구독 기간을 더해 소프트웨어 서비스의 구독 만료일을 계산하여 적시에 갱신 알림을 보냅니다.

**고객 생애 가치:** 평균 구매 금액에 구매 빈도를 곱해 고객의 생애 가치를 추정하여 고가치 고객을 식별하고 마케팅 노력을 최적화합니다.

**재고 관리:** 총 재고에서 판매된 항목을 빼서 재고 수준을 추적하고 재주문 프로세스를 자동화합니다.

**보증 추적:** 제품 구매일에 보증 기간을 더해 보증 적용 종료일을 결정하고 고객 지원을 용이하게 합니다.

**나이 확인:** 현재 날짜에서 고객의 생년월일을 빼서 연령 제한 제품이나 서비스에 대한 나이를 확인합니다.

### 수학 기능의 장점

워크플로우의 수학 연산 기능의 장점은 다음과 같습니다:

**자동화:** 반복적이고 시간이 많이 소요되는 계산을 간소화하여 더 가치 있는 업무를 위한 시간을 확보하고 전반적인 생산성을 향상시킵니다.

**정확성:** 계산에서 인적 오류를 최소화하여 정확한 결과와 의사결정을 위한 신뢰할 수 있는 데이터를 보장합니다.

**맞춤화:** 커스텀 필드와 변수를 사용하여 수학적 연산을 수행하고, 특정 비즈니스 요구사항과 시나리오에 맞게 계산을 조정합니다.

**향상된 의사결정:** 데이터 분석을 통해 귀중한 통찰력을 얻어 더 나은 정보에 기반한 전략적 결정을 내리고 비즈니스 성과를 최적화합니다.

**강화된 리드 관리:** 계산된 리드 점수와 기타 지표를 If/Else 조건에서 사용하여 영업 퍼널을 통해 리드를 더 효과적으로 라우팅하고 전환율과 영업 결과를 개선합니다.

2. **수학 연산 액션 추가:** 워크플로우 빌더에서 "액션 추가(Add Action)"를 클릭하고 사용 가능한 액션 목록에서 "수학 연산(Math Operation)" 액션을 선택합니다.

- 출력 값이 음수인 경우 0으로 대체됩니다.

## 숫자 / 금액 필드 유형

숫자 및 금액 필드는 숫자 값을 저장하도록 설계된 두 가지 유형의 커스텀 필드입니다. 리드 점수, 영업 수치 또는 재정 데이터와 같은 정량적 데이터를 워크플로우 내에서 보관하고 조작할 수 있습니다.

### 숫자 필드:

**목적:** 숫자 필드는 개수, 수량, 백분율 또는 비율과 같은 정수 또는 소수 값을 저장하도록 설계되었습니다.

**데이터 형식:** 숫자 필드는 시스템 구성에 따라 양수와 음수 모두, 그리고 다양한 정밀도의 소수 값을 허용할 수 있습니다.

**사용 사례:** 숫자 필드는 리드 점수, 제품 수량, 고객 만족도 평가 또는 전환율과 같은 데이터를 저장할 수 있습니다.

### 금액 필드:

**목적:** 금액 필드는 가격, 비용, 수익 또는 수수료와 같은 재정 데이터를 나타내는 통화 값을 저장하도록 설계되었습니다.

**데이터 형식:** 금액 필드는 지정된 통화 기호(예: USD, EUR, GBP)와 함께 숫자 값을 저장하며 일반적으로 센트나 통화의 더 작은 단위를 나타내기 위해 고정된 정밀도를 가집니다. 양수와 음수 값을 모두 처리할 수 있습니다.

**사용 사례:** 금액 필드는 제품 가격, 거래 금액, 영업 수익 또는 직원 급여와 같은 데이터를 저장할 수 있습니다.

숫자 및 금액 필드 모두 워크플로우의 수학 연산 액션에서 덧셈, 뺄셈, 곱셈 또는 나눗셈 계산을 수행하는 데 사용할 수 있습니다.

## 날짜 필드 유형

워크플로우에서 수학 연산 액션은 날짜 필드에도 적용할 수 있어, 특정 시간 단위를 더하거나 빼서 날짜를 조작하고 계산할 수 있습니다. 이 기능은 시간에 민감한 작업과 동적 날짜 기반 일정을 자동화하는 데 도움이 됩니다.

### 날짜 필드:

**목적:** 날짜 필드는 워크플로우의 이벤트, 마감일 또는 마일스톤을 나타내는 날짜와 타임스탬프를 저장합니다.

**데이터 형식:** 날짜 필드는 일반적으로 YYYY-MM-DD와 같은 표준화된 형식으로 정보를 저장하거나 시간 정보(예: YYYY-MM-DD HH:MM:SS)를 포함합니다.

**날짜 필드에 지원되는 수학 연산:**

**일 추가/빼기:**
- **설명:** 날짜 필드에서 특정 일수를 더하거나 뺍니다.
- **사용 사례:** 작업이나 프로젝트의 마감일 계산, 고객 상호작용의 후속 날짜 결정, 시작일을 기준으로 한 시험 기간 종료일 계산

**월 추가/빼기:**
- **설명:** 날짜 필드에서 특정 월수를 더하거나 뺍니다.
- **사용 사례:** 구독 서비스의 다음 청구일 계산, 월간 정기 이벤트 날짜 계산, 임대 또는 계약의 종료일 결정

**연 추가/빼기:**
- **설명:** 날짜 필드에서 특정 연수를 더하거나 뺍니다.
- **사용 사례:** 연간 멤버십이나 라이선스의 갱신일 계산, 기념일이나 프로젝트의 미래 재정 데이터 결정

### 현재 날짜 필드

워크플로우에서 수학 연산 액션은 현재 날짜에도 적용할 수 있어, 현재 날짜와 시간을 기반으로 한 계산을 수행할 수 있습니다. 이 기능은 현재 날짜나 시간과 저장된 날짜 필드의 관계에 반응하는 동적이고 시간에 민감한 워크플로우를 만드는 데 유용합니다.

**목적:** 현재 날짜 필드는 하위 계정의 시간대에 따른 현재 날짜와 시간을 나타냅니다. 현재 순간과 관련하여 날짜를 비교하고 계산하는 데 사용할 수 있습니다.

## 수학 연산에 음수 값 적용

금액 및 숫자 필드의 특정 특성을 바탕으로 워크플로우에서 수학 연산을 수행할 때 이러한 제약사항을 적용하는 것이 중요할 수 있습니다.

**참고사항:** 날짜 또는 현재 날짜 필드에서는 음수 값이 지원되지 않습니다.

### 금액 필드 제약사항:

**사용 사례:** 제품이나 서비스의 할인 계산. 적용된 할인이 음수 값을 초래하는 경우, 시스템은 최종 가격을 음수가 아닌 0으로 업데이트하여 고객이 음수 금액으로 청구되지 않도록 보장해야 합니다.

**해결책:** 금액 필드에서 수학 연산을 수행할 때는 음수 값을 초래할 수 있는 뺄셈 연산을 If/Else 조건으로 확인하는 것이 필수입니다. 결과가 음수인 경우 금액 필드를 0으로 업데이트합니다.

### 숫자 필드 제약사항:

**사용 사례:** 비즈니스의 순이익 또는 손실 계산. 이 경우 숫자 필드는 이익 또는 손실을 나타내는 양수 및 음수 값을 저장할 수 있습니다.

**해결책:** 숫자 필드에서 수학 연산을 수행할 때는 음수 값이 허용되므로 추가 제약사항 없이 직접 값을 더하거나 뺄 수 있습니다.
