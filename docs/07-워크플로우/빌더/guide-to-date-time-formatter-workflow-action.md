# 날짜/시간 포맷터 워크플로우 액션 가이드

날짜/시간 포맷터(Date/Time Formatter) 액션은 워크플로우 안에서 날짜나 날짜-시간을 다시 포맷하거나 비교할 수 있는 다양한 기능을 제공하는 도구입니다. 이 기능을 사용하면 다양한 애플리케이션, 데이터베이스 저장, 이메일, SMS, Slack 등의 커뮤니케이션 채널과의 호환성을 위해 날짜 형식을 쉽게 변환할 수 있습니다. 또한 날짜를 비교하여 일수 차이를 계산할 수 있어 날짜 관련 계산과 의사결정을 단순화합니다.

# 날짜/시간 포맷터 워크플로우 액션이란?

날짜/시간 포맷터 액션은 자동화 작업에서 날짜와 시간을 쉽게 다룰 수 있도록 도와주는 간단한 도구입니다. 3가지 주요 기능과 단계는 다음과 같습니다:

- 날짜 형식 변경 (예: 2024-12-09)
- 날짜-시간 형식 변경 (예: 2024-12-09 15:48)
- 두 날짜를 비교하여 차이 확인 (예: 2024-12-09 vs 2024-12-08)

이러한 기능을 통해 날짜와 시간의 표시 방식을 쉽게 변경하거나 두 날짜를 비교할 수 있어, 정보를 올바른 형식으로 공유하거나 저장하기가 간편해집니다.

## 날짜/시간 포맷터 액션의 주요 장점

날짜/시간 포맷터 액션은 워크플로우에서 날짜와 시간 데이터를 다뤄야 하는 사용자들에게 다음과 같은 여러 장점을 제공합니다:

- **일관성**: 다양한 플랫폼과 애플리케이션에서 날짜 및 시간 데이터를 일관된 형식으로 표시하여 가독성을 높이고 혼란을 줄입니다.
- **호환성**: 필요에 따라 날짜 및 시간 형식을 변환하여 다양한 도구와 시스템 간의 원활한 데이터 교환과 연동을 가능하게 합니다.
- **의사결정 지원**: 날짜 비교와 계산을 간소화하여 시간에 민감한 작업에서 정보에 기반한 의사결정을 지원합니다.
- **유연성**: 다양한 형식 옵션을 제공하여 특정 요구사항이나 선호도에 맞춰 조정할 수 있으며, 다양한 사용 사례에 적용할 수 있습니다.

## 날짜/시간 포맷터 액션 설정 방법

### **워크플로우 빌더 액세스**

Automation(자동화) > Create Workflow(워크플로우 만들기) > + Start from Scratch(처음부터 시작)로 이동합니다. 또는 기존 워크플로우 중 하나를 선택하여 날짜/시간 포맷터 워크플로우 액션을 사용할 수 있습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043086464/original/AOYu8H9yQ2VumEc63huR39VmAM2B67tTkg.png?1741750190)

### **날짜/시간 포맷터 액션 선택**

"+" 아이콘을 클릭하여 워크플로우 액션 목록을 확인한 다음, 목록에서 "Date/Time Formatter"를 선택합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043064433/original/J_svPbNFzFiE7Yn5d75y8ylD2RYTPJiiJA.png?1741706969)

### **다양한 액션 유형 이해**

날짜/시간 포맷터 액션은 워크플로우가 자동화를 위해 날짜 값을 올바르게 포맷하고 저장하며 비교할 수 있도록 보장합니다. 설정이 완료되면 다시 포맷되거나 계산된 값을 동적 변수로 저장하며, 이는 나중에 이메일 발송, CRM 필드 업데이트, 조건부 워크플로우 트리거 등의 워크플로우 액션에서 사용할 수 있습니다. 날짜/시간 포맷터 액션을 사용하여 날짜를 비교할 수도 있습니다.

이 액션을 사용하는 3가지 주요 방법이 있습니다:

- **Format Date(날짜 포맷)**: 날짜를 다른 형식으로 변환합니다.
- **Format Date-Time(날짜-시간 포맷)**: 날짜와 시간 모두의 형식을 변경합니다.
- **Compare Dates(날짜 비교)**: 두 날짜 간의 차이를 계산합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043064532/original/kzSdz_vTyGk9OgGFsjIW428Egk6ctG5MDQ.png?1741707004)

### 날짜 포맷 (Format DATE)

날짜 포맷 기능은 날짜/시간 포맷터 액션의 핵심 기능으로, 날짜 구조를 한 형식에서 다른 형식으로 변환하는 데 특화되어 있습니다. 날짜 포맷 기능에 대한 자세한 설명은 다음과 같습니다:

#### 필드 선택

다시 포맷하려는 날짜 필드를 선택합니다. 특정 날짜, 현재 날짜, 연락처의 날짜 필드, 날짜 유형 커스텀 필드, 예약 시작/종료 날짜, 커스텀 값, 또는 인바운드 웹훅 트리거 등 다양한 소스를 선택할 수 있습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043064633/original/c5y6pWoRV_BXXuqz3_pmcPpw1HnIn7VR1w.png?1741707067)

현재 연락처 커스텀 필드는 날짜-시간(Date Time) 구조를 지원하지 않습니다.

#### From Format (원본 형식)

특정 날짜, 현재 날짜, 연락처의 날짜 유형 표준 또는 커스텀 필드, 예약 시작/종료 날짜 등의 시스템 필드를 선택하면 도구가 형식을 자동 감지하여 미리 선택합니다. 커스텀 값이나 인바운드 웹훅 트리거를 선택하는 경우 일치하는 형식을 수동으로 선택해야 합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043064837/original/N-gVpbSNXEkJ76O_cwyNdPCFjSXEJoHDzA.png?1741707112)

#### To Format (변환할 형식)

사용 가능한 형식 목록에서 출력 날짜에 대한 원하는 형식을 선택합니다. 이것이 원래 날짜가 변환될 형식입니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043064972/original/gFsbZ2hBbNaXC7fvOwh7VVl2eQ9g1s8fRA.png?1741707182)

### 날짜-시간 포맷 (Format DATE-TIME)

날짜-시간 포맷 기능은 날짜/시간 포맷터 액션의 필수 기능으로, 날짜와 시간 정보를 모두 포함하는 구조를 다른 형식으로 변환하도록 설계되었습니다. 날짜 및 시간 포맷 기능에 대한 자세한 설명은 다음과 같습니다:

#### 필드 선택

다시 포맷하려는 날짜 및 시간 필드를 선택합니다. 특정 날짜 및 시간, 현재 날짜 및 시간, 예약 시작/종료 날짜 및 시간, 커스텀 값, 또는 인바운드 웹훅 트리거 등 다양한 소스를 선택할 수 있습니다. 참고로 연락처 커스텀 필드는 현재 날짜-시간 구조를 지원하지 않습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043085210/original/QfBq7h_U-0-ox4hOdBfaF4iUoi9xZwkalA.png?1741746520)

현재 연락처 커스텀 필드는 날짜-시간(Date Time) 구조를 지원하지 않습니다.

#### From Format (원본 형식)

특정 날짜 및 시간, 현재 날짜 및 시간, 예약 시작/종료 날짜 및 시간 등의 시스템 필드를 선택하면 도구가 형식을 자동 감지하여 미리 선택합니다. 커스텀 값이나 인바운드 웹훅 트리거를 선택하는 경우 일치하는 형식을 수동으로 선택해야 합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043085235/original/dDk15VGid2gSm1zfTK51IeJuowR3s8b5PQ.png?1741746552)

#### To Format (변환할 형식)

사용 가능한 형식 목록에서 원하는 출력 날짜 및 시간 형식을 선택합니다. 이것이 원래 날짜 및 시간이 변환될 형식입니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043085239/original/iViu4OjOa6H_kHgDvM65p2ljHgoJg0-Y2g.png?1741746584)

### 날짜 비교 (Compare Dates)

날짜 비교 기능은 날짜/시간 포맷터 액션의 유용한 기능으로, 사용자가 두 날짜를 비교하고 일수 차이를 확인할 수 있게 해줍니다. 날짜 비교 기능에 대한 자세한 개요는 다음과 같습니다:

#### 시작 날짜 선택

특정 날짜, 현재 날짜, 연락처의 날짜 필드, 날짜 유형 커스텀 필드, 예약 시작/종료 날짜, 커스텀 값, 또는 인바운드 웹훅 트리거 등 다양한 소스에서 시작 날짜를 선택합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043086393/original/BMpIxErNiu7p2RxN-ivzSHrYTopTp4Vxbg.png?1741749979)

#### 시작 날짜 형식

특정 날짜, 현재 날짜, 연락처의 날짜 유형 표준 또는 커스텀 필드, 예약 시작/종료 날짜 등의 시스템 필드를 선택하면 도구가 형식을 자동 감지하여 미리 선택합니다. 커스텀 값이나 인바운드 웹훅 트리거를 선택하는 경우 일치하는 형식을 수동으로 선택해야 합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043086414/original/AJn---_UmwDK3ATEaKwKTjjqcAYrBn1sLw.png?1741750010)

#### 종료 날짜 선택

시작 날짜와 같은 소스에서 종료 날짜를 선택합니다(특정 날짜, 현재 날짜, 연락처의 날짜 필드, 날짜 유형 커스텀 필드, 예약 시작/종료 날짜, 커스텀 값, 또는 인바운드 웹훅 트리거).

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043086424/original/oiwhL5F9dkcNN19ynLAhYESOb1LkLlL3tw.png?1741750049)

#### 종료 날짜 형식

시작 날짜 형식과 마찬가지로, 시스템 필드를 선택하면 도구가 형식을 자동 감지하여 미리 선택합니다. 커스텀 값이나 인바운드 웹훅 트리거의 경우 일치하는 형식을 수동으로 선택해야 합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043086433/original/GQhQqrwdeT1M47-jKsb41yZRm9ErtQ9xsg.png?1741750074)

차이는 종료 날짜 - 시작 날짜로 계산됩니다. 시작 날짜가 종료 날짜보다 늦으면 음수 값을 얻게 됩니다. 날짜 비교 기능은 만료일 계산, 이벤트 간 기간 확인, 두 날짜 간의 시간 차이에 기반한 의사결정 등 다양한 목적으로 날짜 비교가 필요한 사용자들에게 매우 유용합니다.

## 사용 사례

날짜/시간 포맷터 액션은 다양한 시나리오와 날짜 및 시간 데이터를 관리하고 조작해야 하는 사용자들에게 유용합니다. 다음은 몇 가지 사용 사례입니다:

- **이벤트 주최자**: 다양한 소스에서 받은 이벤트 시작 및 종료 날짜와 시간을 포맷하여 일관성을 유지하고 데이터를 더 읽기 쉽게 만듭니다.
- **마케팅 팀**: 개인화되고 시간에 민감한 커뮤니케이션을 위해 이메일 캠페인, SMS, Slack 메시지에서 포맷된 날짜 및 시간 정보를 활용합니다.
- **영업 팀**: 현재 날짜와 인보이스 만료일을 비교하여 결제 예정이거나 연체된 고객에게 자동 알림을 발송합니다.
- **고객 지원 팀**: 인바운드 웹훅 트리거를 통해 받은 날짜 및 시간 데이터를 다시 포맷하여 지원 티켓을 생성하거나 CRM 시스템을 업데이트합니다.
- **프로젝트 관리자**: 프로젝트 시작 날짜와 마감일을 비교하여 남은 시간을 계산하고 팀원들에게 알림을 보냅니다.
- **데이터 분석가**: 스프레드시트나 데이터베이스에 일관되게 포맷된 날짜 및 시간 데이터를 저장하여 분석과 리포트 생성을 더 쉽게 합니다.
- **소프트웨어 개발자**: 다양한 소스의 포맷된 날짜 및 시간 데이터를 애플리케이션에 통합하여 호환성과 원활한 데이터 처리를 보장합니다.
- **전자상거래 비즈니스**: 주문 및 배송 날짜 정보를 다시 포맷하여 더 나은 추적과 고객 커뮤니케이션을 제공합니다.

## 자주 묻는 질문

**Q. 포맷된 날짜나 시간을 다른 워크플로우 액션에서 어떻게 사용하나요?**

날짜/시간 포맷터 액션이 날짜나 시간을 처리하면, 다시 포맷된 값을 동적 변수(예: {{datetime_formatter.1.date}}, {{datetime_formatter.1.datetime}})에 저장합니다. 이메일 발송, CRM 필드 업데이트, 날짜 값에 기반한 조건부 로직 트리거 등의 후속 워크플로우 액션에서 이 변수를 참조할 수 있습니다.

**Q. "From Format" 필드에서 잘못된 형식을 선택하면 어떻게 되나요?**

선택한 원본 형식이 입력 날짜의 실제 형식과 일치하지 않으면, 워크플로우가 날짜를 올바르게 처리하지 못할 수 있습니다. 이로 인해 날짜 값이 누락되거나 부정확할 수 있습니다. 변환하기 전에 원본 형식이 실제 날짜 구조를 정확히 나타내는지 항상 확인하세요.

**Q. 연락처 레코드의 날짜 필드를 현재 날짜와 비교할 수 있나요?**

네, 날짜 비교 기능을 사용하면 저장된 연락처 날짜(예: "가입 날짜" 또는 "마지막 구매 날짜")를 현재 날짜와 비교할 수 있습니다. 이는 특정 일수가 지났을 때 알림, 후속 조치, 또는 자동화 액션을 트리거하는 데 유용할 수 있습니다. 일수 차이는 워크플로우에서 추가 사용을 위해 {{datetime_formatter.1.days}}에 저장됩니다.

**Q. 날짜/시간 포맷터 액션이 시간대 차이를 처리할 수 있나요?**

날짜/시간 포맷터 액션은 시스템의 기본 시간대 설정에 따라 날짜와 시간을 처리합니다. 워크플로우에 다른 시간대의 사용자들이 포함된 경우, 포맷을 적용하기 전에 모든 날짜/시간 값을 표준화하거나 적절히 변환했는지 확인하세요.
