# 워크플로우 액션 - 연락처 필드 업데이트

**연락처 필드 업데이트(Update Contact Field)** 워크플로우 액션을 사용하면 워크플로우에서 직접 표준 또는 커스텀 연락처 필드에 새 값을 입력하거나 커스텀 필드의 값을 지울 수 있습니다. 최신 개선사항으로 추가 필드 유형에 **동적 커스텀 값(Dynamic Custom Values)**을 사용할 수 있게 되어, 워크플로우가 텍스트나 숫자를 직접 입력하지 않고도 이전 단계나 저장된 필드의 실시간 값을 설정할 수 있습니다.

## 개요

Hyperclass의 연락처 필드 업데이트 액션은 워크플로우가 실행되는 정확한 시점에 연락처 기록의 데이터를 업데이트합니다. 필드에 값을 업데이트하거나 기존 값을 지울 수 있으며, 모든 변경사항은 **실행 로그(Execution Logs)**에서 확인할 수 있습니다.

이 액션은 개인화, 세분화, 리포팅, 후속 워크플로우 로직을 위해 연락처 데이터를 정확하게 유지하는 데 주로 사용됩니다.

---

## 연락처 필드 업데이트 액션 추가하기

- **Automation(자동화) → Workflows(워크플로우)**로 이동합니다.
- 기존 워크플로우를 열거나 새로 만듭니다.
- 액션을 추가하려는 위치에서 **+** 아이콘을 클릭합니다.
- 액션 패널에서 **Update Contact Field(연락처 필드 업데이트)**를 선택합니다.

## 액션 유형 선택

액션이 어떻게 동작할지 선택하세요. 선택에 따라 다음에 나타나는 설정 옵션이 달라집니다.

**액션 유형 옵션:**

- **Update field data(필드 데이터 업데이트)** – 필드에 새 값을 입력
- **Clear field data(필드 데이터 지우기)** – 저장된 값을 제거 (커스텀 필드만 가능)

## 필드 데이터 업데이트

이 모드를 사용하여 연락처 필드의 값을 덮어씁니다.

- 업데이트할 연락처 필드를 선택합니다.
- 다음 중 하나를 사용하여 값을 입력합니다:
  - **정적 값**, 또는
  - **동적 값(Dynamic Value)** (워크플로우나 연락처의 실시간 데이터)

워크플로우가 이 단계에 도달하면 새 값이 기존 값을 대체합니다.

![필드 데이터 업데이트 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064139707/original/YT7RvqDRGXyhtk1TLKL4XKICCyvbbpE0Hg.png?1770188240)

## 필드 데이터 지우기

이 모드를 사용하여 **커스텀 연락처 필드**에서 저장된 값을 제거합니다.

- 액션 유형으로 **Clear field data(필드 데이터 지우기)**를 선택합니다.
- 지울 커스텀 필드를 하나 이상 선택합니다.
- 액션이 실행되면 필드 값이 비어있는 상태로 설정됩니다.

**주의:** 값을 지우면 스마트 리스트, 필터, 워크플로우 조건에 영향을 줄 수 있습니다. 발행하기 전에 항상 종속성을 검토하세요.

---

## 연락처 필드 업데이트에서 동적 값 사용하기

이제 **동적 값(Dynamic Values)**을 사용하여 이전 워크플로우 단계의 실시간 출력이나 저장된 연락처 필드를 사용해 연락처 필드를 설정할 수 있습니다. 이를 통해 값을 직접 입력할 필요가 없어지고 중복 로직을 방지할 수 있습니다.

동적 값은 각 개별 연락처에 대해 런타임에 해결됩니다.

---

## 지원되는 필드 유형

**연락처 필드 업데이트**에서 동적 값을 지원하는 필드 유형은 다음과 같습니다:

- **숫자형(Numeric)** – 점수, 개수
- **선택/드롭다운(Select/Dropdown)** *(여기서는 옵션 ID가 필요하지 않음)*
- **금액(Monetary)** – 총액, 금액

**중요:** 선택/드롭다운 필드는 연락처 필드를 업데이트할 때 **옵션 ID가 필요하지 않습니다**. (이는 옵션 ID가 필요한 If/Else 조건과 다릅니다.)

---

## 동적 값 사용 방법

- 워크플로우에 **연락처 필드 업데이트** 액션을 추가합니다.
- 액션 유형으로 **Update field data(필드 데이터 업데이트)**를 선택합니다.
- 대상 연락처 필드를 선택합니다.
- 값 입력란에서 **Dynamic Value(동적 값)**로 전환합니다.
- 다음 중에서 값을 선택합니다:
  - 이전 워크플로우 단계의 출력, 또는
  - 저장된 연락처 필드
- 워크플로우를 저장하고 발행합니다.

---

## 사용 예시

환자가 예약을 확인한 후, 워크플로우가 확인 유형(예: *후속 진료* 또는 *신규 방문*)을 캡처합니다. 그런 다음 해당 값이 커스텀 연락처 필드에 동적으로 입력되어 기록을 정확하게 유지하고 올바른 후속 커뮤니케이션을 보장합니다.

---

## 여러 필드 업데이트

하나의 액션에서 둘 이상의 필드를 업데이트할 수 있습니다:

- **Add field(필드 추가)**를 클릭하여 추가 필드 행을 포함합니다.
- 명확성과 감사를 위해 일부 팀은 업데이트와 지우기를 별도의 액션으로 분리하는 것을 선호합니다.

---

## 필드 유형 고려사항 및 검증

오류를 방지하려면 값이 필드 유형과 일치하는지 확인하세요:

- **텍스트/숫자:** 적절한 형식을 사용하세요.
- **날짜:** 유효한 날짜 값이나 날짜 기반 동적 값을 사용하세요.
- **드롭다운/체크박스/라디오:** 값이 기존 옵션과 일치해야 합니다.
- **이메일/전화:** 올바르게 형식화된 값을 제공하세요.

필드는 이미 존재해야 합니다. 이 액션은 새 필드를 생성하지 않습니다.

## 자주 묻는 질문

**Q: 이메일이나 전화번호 같은 표준 연락처 필드를 업데이트할 수 있나요?**

네. 필드 데이터 업데이트는 표준 및 커스텀 연락처 필드를 모두 지원합니다.

**Q: 표준 연락처 필드를 지울 수 있나요?**

아니요. 필드 데이터 지우기는 커스텀 연락처 필드만 지원합니다.

**Q: 하나의 액션에서 여러 필드를 업데이트할 수 있나요?**

네. **Add field(필드 추가)**를 사용하여 같은 액션에서 여러 필드를 업데이트할 수 있습니다.

**Q: 동적 값과 토큰은 어떻게 다른가요?**

동적 값을 사용하면 이전 워크플로우 단계나 필드의 실시간 출력을 선택할 수 있습니다. 토큰도 여전히 작동하지만, 동적 값을 사용하면 값을 수동으로 입력하거나 매핑할 필요가 없습니다.

**Q: 잘못된 형식을 사용하면 어떻게 되나요?**

업데이트가 검증에 실패할 수 있습니다. 필드 유형을 검토하고 실행 로그를 확인하세요.

**Q: 값을 지우면 스마트 리스트나 워크플로우에 영향을 주나요?**

네. 값을 지우면 목록 멤버십이나 분기 로직이 변경될 수 있습니다.
