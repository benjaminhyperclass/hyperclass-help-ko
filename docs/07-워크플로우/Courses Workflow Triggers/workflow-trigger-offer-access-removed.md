# 오퍼 접근 권한 제거됨 워크플로우 트리거란

오퍼 접근 권한 제거됨 워크플로우 트리거는 연락처의 특정 오퍼에 대한 접근 권한이 취소될 때 활성화됩니다. 이를 통해 비즈니스는 연락처 알림, CRM 기록 업데이트, 관련 팀 통지 등의 후속 작업을 자동화할 수 있습니다. 이 트리거를 워크플로우에 통합함으로써 조직은 프로세스를 간소화하고 운영 효율성을 유지하며 커뮤니케이션을 개선할 수 있습니다.

## 오퍼 접근 권한 제거됨 트리거의 주요 장점

- **자동 알림:** 접근 권한이 취소될 때 연락처와 내부 팀에 실시간으로 알림을 보내어 모든 사람이 정보를 공유할 수 있습니다.

- **워크플로우 효율성 향상:** CRM 기록 업데이트나 이메일 발송 같은 반복적인 작업을 자동화하여 시간을 절약하고 오류를 줄입니다.

- **고객 경험 개선:** 접근 권한 취소 후 연락처에게 명확한 커뮤니케이션과 다음 단계를 제공하여 투명성과 신뢰를 유지합니다.

- **운영 간소화:** 접근 제어를 효율적으로 관리하여 자격을 갖춘 연락처만이 특정 오퍼에 접근할 수 있도록 보장합니다.

- **맞춤형 필터:** 필터를 사용하여 워크플로우를 세분화하고 특정 오퍼를 대상으로 하여 정확한 자동화를 보장합니다.

## 트리거 설정: 단계별 과정

### 워크플로우 설정 접근

CRM 플랫폼의 자동화 또는 워크플로우 설정으로 이동합니다. 새 워크플로우를 처음부터 만들거나 "오퍼 접근 권한 제거됨" 트리거를 구현하려는 기존 워크플로우를 선택합니다.

### 새 워크플로우 트리거 추가

**"+ Add New Trigger(+ 새 트리거 추가)"**를 클릭하고 드롭다운 메뉴에서 **"Offer Access Removed(오퍼 접근 권한 제거됨)"**를 선택합니다.

![오퍼 접근 권한 제거됨 트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038864532/original/YwXGvYx3qUI3FqQFADjipWPH2Xn2ypXvlw.png?1735053117)

### 트리거 이름 설정

"15% 할인 접근 권한 취소"와 같이 설명적인 이름을 제공합니다.

![트리거 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038864757/original/qt6xI_t_fnA3vwwI6rblM3JR_f6Pi0PKlA.png?1735053406)

### 필터 설정

필터를 설정하여 워크플로우 활성화 조건을 세분화합니다:

**오퍼(Offer)**

접근 권한이 제거될 때 워크플로우를 트리거할 특정 오퍼(예: "얼리버드 할인")를 선택합니다.

![오퍼 필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038864790/original/pnS-1ClRSqZeDWkKQEFhJ7rsDyIs0wD6-A.png?1735053438)

---

### 트리거 저장

적절한 버튼을 클릭하여 트리거를 저장합니다.

### 테스트 및 발행

샘플 데이터를 사용하여 워크플로우를 테스트하고 기능을 확인합니다. 검증이 완료되면 발행(Publish) 토글을 활성화하여 워크플로우를 활성화합니다.

## 사용 사례

### 할인 접근 권한 취소 이메일

**시나리오:** 고객의 프로모션 할인에 대한 접근 권한이 취소되어 알림이 필요한 경우입니다.

**트리거 설정:**
- **트리거:** 오퍼 접근 권한 제거됨

**필터:**
- 오퍼: "15% 얼리버드 할인"

**결과:** 워크플로우가 고객에게 접근 권한 취소를 알리고 대안 오퍼나 다음 단계를 제공하는 이메일을 발송합니다.

---

### 멤버십에 대한 내부 알림

**시나리오:** 연락처의 프리미엄 멤버십 접근 권한이 제거되어 고객 지원 팀이 후속 조치를 취해야 하는 경우입니다.

**트리거 설정:**
- **트리거:** 오퍼 접근 권한 제거됨

**필터:**
- **오퍼:** "프리미엄 멤버십"

**결과:** 워크플로우가 지원 팀에 알림을 보내어 연락처에게 갱신 옵션을 제안할 수 있게 합니다.

---

### 접근 추적을 위한 CRM 업데이트

**시나리오:** 기간 한정 오퍼에 대한 접근 권한이 취소되어 정확한 추적을 위해 연락처의 CRM 프로필을 업데이트해야 하는 경우입니다.

**트리거 설정:**
- **트리거:** 오퍼 접근 권한 제거됨

**필터:**
- **오퍼:** "독점 여름 딜"

**결과:** 워크플로우가 제거 내용을 반영하도록 CRM 기록을 업데이트하고 보고용으로 연락처에 적절한 태그를 지정합니다.

## 자주 묻는 질문

**이 트리거가 같은 워크플로우에서 여러 오퍼를 처리할 수 있나요?**

네, 더 많은 필터를 추가하거나 더 넓은 조건을 설정하여 여러 오퍼에 대해 활성화되도록 워크플로우를 구성할 수 있습니다.

**한 연락처에 대해 같은 오퍼의 접근 권한이 여러 번 취소되면 어떻게 되나요?**

접근 권한이 제거될 때마다 트리거가 활성화되므로 중복 작업을 피하기 위해 조건을 포함한 워크플로우를 설계하는 것이 중요합니다.

**워크플로우가 올바른 오퍼를 대상으로 하는지 어떻게 확인할 수 있나요?**

"오퍼" 필터를 사용하여 정확한 오퍼를 지정하세요. 샘플 데이터로 워크플로우를 테스트하여 예상대로 트리거되는지 확인합니다.

**이 트리거를 워크플로우의 다른 트리거와 결합할 수 있나요?**

네, 오퍼 접근 권한 제거됨 트리거는 "태그 추가됨"이나 "기회 상태 변경됨" 같은 다른 트리거와 결합하여 고급 워크플로우를 만들 수 있습니다.
