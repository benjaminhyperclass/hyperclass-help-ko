쿠폰 코드가 만료일에 도달해서 비활성화될 때 자동으로 작업을 실행할 수 있습니다. 이 가이드에서는 설정부터 주요 이점까지 모든 내용을 안내해 드려서, 쿠폰이 만료되는 순간 워크플로우가 원활하게 실행되도록 도와드립니다. 프로모션 마무리, 고객 후속 관리, 내부 알림에 완벽한 기능입니다.

# **쿠폰 코드 만료 워크플로우 트리거란?**

쿠폰 코드 만료 트리거는 쿠폰 코드가 예정된 만료일에 도달해서 비활성화될 때 실행됩니다. 마케팅팀과 영업팀이 기간 한정 프로모션을 관리하고, 놓친 고객에게 후속 관리를 하거나, 만료된 쿠폰을 교체하거나, 오퍼가 더 이상 유효하지 않을 때 내부 팀에 알릴 수 있는 훌륭한 도구입니다.

## **쿠폰 코드 만료 워크플로우 트리거의 주요 이점**

- **즉시 알림**: 주요 쿠폰이 만료될 때 영업, 고객지원, 마케팅 팀에게 알려서 팀이 바로 대응할 수 있습니다
- **고객 재참여**: 놓친 고객에게 후속 오퍼나 연장을 보내서 전환 기회를 높입니다
- **캠페인 정리**: 쿠폰이 만료될 때 태그 보관, 기회 닫기, 광고 중단을 자동화해서 데이터를 깔끔하게 관리합니다
- **일관된 고객 경험**: 고객이 다시 사용하려 하기 전에 오퍼가 더 이상 유효하지 않다는 것을 확인해서 혼란을 방지합니다

## 쿠폰 코드 만료 워크플로우 트리거 설정하기

워크플로우 빌더에서 이 트리거를 설정하려면 다음 단계를 따르세요:

#### **1단계: 워크플로우에 트리거 추가하기**

- **Automation(자동화) > Workflows(워크플로우)**로 이동하세요
- 새 워크플로우를 만들거나 기존 워크플로우를 편집하세요
- 워크플로우 빌더에서 **+ Add New Trigger(새 트리거 추가)**를 클릭해서 시작하세요

![워크플로우 트리거 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050551956/original/KS83c0dS2PvtlP4YCrV05IBN_GsjzQallg.png?1753727277)

#### **2단계: 트리거 유형 선택하기**

사용 가능한 트리거 목록에서 **Payments(결제)** 섹션으로 스크롤해서 **Coupon Code Expired(쿠폰 코드 만료)**를 선택하세요

![쿠폰 코드 만료 트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050551986/original/iS9zpAvHNi_rsInKm1vVz0rroVNwkUHKVA.png?1753727377)

#### **3단계: 트리거 이름 지정하기**

- **Workflow Trigger Name(워크플로우 트리거 이름)** 필드에 설명적인 이름을 입력해서 워크플로우를 체계적으로 관리하세요
- 예시: 여름세일쿠폰만료

![트리거 이름 지정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050552764/original/FcSxv_3jNMqPs69syQRA2cMIuKgOQyp5SA.png?1753729190)

#### **4단계: 특정 시나리오를 대상으로 하는 필터 적용하기**

- 트리거 이름을 지정한 후, 필터를 적용해서 워크플로우가 실행될 정확한 조건을 좁힐 수 있습니다
- 필터를 추가하려면 **+ Add Filters(필터 추가)**를 클릭하세요
- 각 필터에는 특정 연산자가 있어서 필요한 조건을 정확하게 설정할 수 있습니다. [사용 가능한 필터 목록 보기](#사용-가능한-필터)

![필터 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050552825/original/HbC14qdtt-BcO1q8Vt54um71dhz6jADN3A.png?1753729308)

#### **5단계: 저장하고 워크플로우 구성하기**

- 트리거와 필터가 설정되면 **Save Trigger(트리거 저장)**를 클릭하고 원하는 액션을 워크플로우에 추가하기 시작하세요
- 완료되면 **Test Workflow(워크플로우 테스트)**를 사용해서 모든 것이 예상대로 작동하는지 확인한 다음 **Publish(발행)**해서 워크플로우를 활성화하세요

![워크플로우 저장 및 발행](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050552860/original/x4q_5CI9nIOpvsgrNMWld-e8aXcCFPqaPA.png?1753729421)

## 사용 가능한 필터

| 필터 | 연산자 | 입력 유형 |
|------|--------|-----------|
| 쿠폰 코드 | Is Any Of / Is Empty / Is None Of / Is Not Empty | 활성 쿠폰 코드의 다중 선택 드롭다운 |
| 쿠폰 이름 | Is Any Of / Is Empty / Is None Of / Is Not Empty | 활성 쿠폰 이름의 다중 선택 드롭다운 |
| 쿠폰 유형 | Is / Is Not / Is Empty / Is Not Empty | 단일 선택 드롭다운 (금액 vs 비율) |

## **만료된 쿠폰에 권장하는 워크플로우 액션**

쿠폰 코드 만료 트리거가 활성화되면 다음과 같은 액션을 추가하는 것을 고려해보세요:

- **이메일 발송**: 고객에게 쿠폰이 만료되었다고 알리고 새로운 프로모션을 제공하세요
- **내부 알림**: 주요 쿠폰이 만료될 때 영업팀이나 마케팅팀에게 알리세요
- **태그 추가**: 후속 관리나 세분화를 위해 연락처에 태그를 적용하세요 (예: 놓친프로모션)
- **기회 생성/업데이트**: 리뎀션을 놓친 연락처를 육성 시퀀스로 라우팅하세요
- **SMS 발송**: 선별된 고객에게 빠른 후속 오퍼나 쿠폰 연장을 전달하세요

## **자주 묻는 질문**

**Q: 이 트리거는 정확히 언제 실행되나요?**
쿠폰의 예정된 만료일에 도달해서 쿠폰이 자동으로 비활성화될 때 실행됩니다.

**Q: 특정 유형의 쿠폰만 대상으로 할 수 있나요?**
네, 쿠폰 유형 필터를 사용해서 비율 또는 고정 금액 쿠폰을 대상으로 할 수 있습니다.

**Q: 필터를 적용하지 않으면 어떻게 되나요?**
만료되는 모든 쿠폰에 대해 워크플로우가 트리거됩니다. 범위를 좁히려면 쿠폰 코드 Is 또는 쿠폰 이름 Contains와 같은 필터를 적용하세요.

**Q: 다른 쿠폰에 대해 다른 액션을 보낼 수 있나요?**
네, 다른 쿠폰 코드 또는 쿠폰 유형 필터로 별도의 워크플로우를 설정하면 각각에 대해 고유한 액션을 실행할 수 있습니다.

**Q: 고객이 이미 만료된 쿠폰을 사용하려고 할 때도 이 트리거가 실행되나요?**
아니요. 트리거는 쿠폰이 활성에서 만료 상태로 바뀔 때만 실행되며, 실패한 리뎀션 시도에서는 실행되지 않습니다.
