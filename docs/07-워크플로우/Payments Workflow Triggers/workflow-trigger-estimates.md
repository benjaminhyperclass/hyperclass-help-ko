# 워크플로우 트리거 - 견적서

"견적서(Estimates)" 트리거는 견적서 발송, 승인, 거절 등 견적서와 관련된 특정 이벤트를 기반으로 작업을 자동화하는 강력한 도구입니다. 이 가이드에서는 견적서 트리거의 개념, 주요 이점, 설정 방법, 그리고 워크플로우를 효율화하는 다양한 활용 사례를 안내드립니다. 이 가이드를 통해 견적서 트리거를 활용한 자동화 워크플로우를 설정하고 고객 참여도를 높일 수 있습니다.

## 견적서 트리거란 무엇인가요?

견적서 트리거는 견적서에 대한 특정 작업이 수행될 때 워크플로우를 활성화하도록 설계되었습니다. 고객에게 견적서를 발송하거나, 프로젝트가 승인되거나, 검토 후 거절되는 등의 상황에서 자동으로 적절한 후속 조치와 작업이 실행됩니다. 기업은 이를 통해 견적서 상태를 추적하고, 기회를 관리하며, 고객과 명확한 소통을 유지할 수 있습니다.

## 견적서 트리거의 주요 이점

- **자동 후속 조치**: 수동 알림이 필요 없습니다. 후속 이메일, 감사 메시지, 다음 단계 안내를 자동으로 발송합니다.
- **향상된 고객 경험**: 견적서가 승인되었을 때 즉시 응답하여 고객에게 정보를 제공하고 참여도를 유지합니다.
- **기회 관리**: 승인된 견적서를 자동으로 새로운 기회로 전환하고 즉시 조치를 위해 적절한 팀원에게 배정합니다.
- **원활한 커뮤니케이션**: 견적서 상태 변경 사항을 내부 팀에 알려 모든 구성원이 동일한 정보를 공유하도록 합니다.

## 견적서 트리거의 필터와 연산자 이해하기

견적서 트리거를 설정할 때 필터를 적용하여 활성화를 세밀하게 조정할 수 있습니다. 필터를 통해 트리거가 실행되어야 하는 특정 조건을 정의하여 자동화를 더욱 정확하고 효과적으로 만들 수 있습니다. 필터를 세밀하게 조정하려면 견적서 관련 데이터를 어떻게 평가할지 결정하는 연산자(논리 조건)를 사용합니다.

견적서 워크플로우를 구축할 때 사용할 수 있는 3가지 필터 옵션은 다음과 같습니다:

- **견적서 상태**: 특정 상태의 견적서에만 트리거를 활성화합니다
- **견적서 금액**: 특정 금액 조건을 만족하는 견적서에만 트리거를 활성화합니다
- **견적서 템플릿**: 특정 템플릿으로 생성된 견적서에만 트리거를 활성화합니다

다음은 견적서 트리거 내에서 다양한 필터와 연산자가 어떻게 작동하는지에 대한 상세 설명입니다:

### 견적서 상태

| 상태 | 설명 |
|------|------|
| Accepted (승인됨) | 고객이 견적서를 승인하여 제안된 조건과 가격에 동의한 상태 |
| Declined (거절됨) | 고객이 견적서를 검토했지만 진행하지 않기로 선택한 상태 |
| Invoiced (인보이스 발행됨) | 견적서가 인보이스로 전환된 상태. 고객이 진행에 동의했으며 결제가 예상되거나 처리된 상태 |
| Sent (발송됨) | 견적서가 생성되어 고객에게 발송되었지만 아직 열람, 승인, 거절되지 않은 상태 |
| Viewed (열람됨) | 고객이 견적서를 열어서 확인했지만 아직 승인 또는 거절하지 않은 상태 |

### 상태 필터 연산자

| 연산자 | 설명 |
|--------|------|
| Is (일치함) | 견적서 상태가 특정 값과 일치할 때만 트리거됩니다 (예: "Accepted") |
| Is Empty (비어 있음) | 견적서에 상태가 할당되지 않았을 때 트리거됩니다 (처리되지 않은 견적서를 찾는 데 유용) |
| Is Not (일치하지 않음) | 견적서 상태가 지정된 값과 일치하지 않을 때 트리거됩니다 (예: "Is Not Declined"는 "Declined"를 제외한 모든 상태에서 트리거) |
| Is Not Empty (비어 있지 않음) | 견적서에 어떤 상태든 할당되어 있을 때만 트리거됩니다 (불완전하거나 처리되지 않은 견적서 제외에 도움) |

**예시**: 견적서가 거절되었을 때만 작업을 트리거하려면 견적서 상태 → Is → Declined를 사용합니다.

![견적서 상태 필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040857388/original/Elsb6sKbJ8-F4OGTYGQvGfsmMCD-jOh6xQ.png?1738490635)

### 견적서 금액 필터

이 필터는 견적서의 금전적 가치를 기준으로 트리거를 세밀하게 조정합니다. 예를 들어, 1,499달러를 초과하는 견적서에 대해 알림을 보내려면 다음 필터 값을 사용합니다: 견적서 금액 > Greater Than > 1499.

| 연산자 | 설명 |
|--------|------|
| Equals To (같음) | 견적서 금액이 지정된 금액과 정확히 일치할 때 활성화 |
| Greater Than (초과) | 견적서 금액이 특정 임계값을 초과할 때 트리거 |
| Greater Than or Equals To (이상) | 견적서 금액이 설정된 금액과 같거나 높을 때 트리거 |
| Is Empty (비어 있음) | 견적서에 금액이 할당되지 않았을 때 활성화 (불완전한 견적서 식별에 유용) |
| Is Not Empty (비어 있지 않음) | 견적서에 어떤 금액이든 할당되어 있을 때 활성화 |
| Is Not Equal To (같지 않음) | 견적서 금액이 특정 금액과 일치하지 않을 때 트리거 |
| Less Than (미만) | 견적서 금액이 정의된 금액보다 낮을 때 트리거 |
| Less Than or Equal To (이하) | 견적서 금액이 주어진 금액과 같거나 낮을 때 트리거 |

![견적서 금액 필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040857434/original/gcG-Bjk3naX5RuSifryTTLRZpZeboecmuA.png?1738490834)

### 견적서 템플릿 필터

이 필터는 특정 견적서 템플릿에 대해서만 트리거가 활성화되도록 보장합니다. 사용 가능한 연산자:

- **Is (일치함)**: 견적서가 특정 템플릿으로 생성된 경우에만 활성화
- **Is Empty (비어 있음)**: 견적서 생성 시 템플릿이 사용되지 않았을 때 트리거 (수동으로 생성된 견적서 식별에 유용)
- **Is Not (일치하지 않음)**: 견적서가 특정 템플릿으로 생성되지 않았을 때 트리거
- **Is Not Empty (비어 있지 않음)**: 견적서가 어떤 템플릿을 사용하여 생성된 경우에만 트리거

**예시**: "프리미엄 패키지" 템플릿을 기반으로 한 견적서에만 자동 이메일을 발송하려면 견적서 템플릿 → Is → Premium Package를 사용합니다.

## 견적서 트리거 설정하기

Hyperclass에서 견적서 트리거를 설정하는 과정을 안내해드리겠습니다:

### 자동화 메뉴로 이동

Automation(자동화) 섹션으로 이동하여 + Create Workflow(워크플로우 생성) 옵션을 선택합니다. 견적서 트리거를 사용할 기존 워크플로우를 선택하거나 새 워크플로우를 처음부터 생성하세요.

![자동화 메뉴에서 워크플로우 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040856446/original/TB_vMRVJQMxaQ1GUT3yDHIJJvkzxQGsIlA.jpeg?1738487449)

### 견적서 트리거 추가

새 트리거 추가 옵션을 선택한 다음, 사용 가능한 워크플로우 트리거 목록에서 Estimates(견적서)를 선택합니다.

![견적서 트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040856456/original/4jIEmwnL7N_m4OE6qZS6AJ8kz6ezcWp2qA.png?1738487495)

### 트리거 이름 지정

트리거의 목적을 명확히 반영하도록 트리거 이름을 "견적서 후속 조치 워크플로우"와 같이 설명적으로 업데이트하세요. 여기서는 트리거를 "Estimate Declined(견적서 거절됨)"로 명명했습니다.

![트리거 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040856471/original/BC8ynz3J7YqB1SZVqGjwtNhmUfK8Law92w.png?1738487534)

### 필터 설정

트리거를 추가한 후, 필터를 사용하여 특정 기준에서만 활성화되도록 세밀하게 조정할 수 있습니다.

+ Add Filters(필터 추가)를 클릭하고 표준 필드 중 하나를 선택합니다.

필드를 선택한 후 적절한 연산자를 선택합니다. 트리거와 필터 설정을 완료했으면 Save(저장)를 클릭합니다.

![필터 설정 과정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040810187/original/K1lyhNifbJOZaTRJ02EwsDf7FkH-jDWFLQ.gif?1738334712)

## 자주 묻는 질문

**Q. 고객이 견적서를 열람했을 때 워크플로우를 트리거하는 방법은 무엇인가요?**

"견적서 상태(Estimate Status)" 필터를 사용하고 "Viewed(열람됨)"로 설정하면 고객이 견적서를 열어보았을 때 워크플로우가 트리거됩니다.

**Q. 특정 금액 이상의 견적서에 대해서만 워크플로우를 트리거할 수 있나요?**

네, "견적서 금액(Estimate Value)" 필터를 사용하고 원하는 금액과 함께 "Greater Than(초과)"으로 설정하세요. 이렇게 하면 고액 견적서에 대해서만 워크플로우가 활성화됩니다.

**Q. 하나의 트리거에 여러 필터를 결합할 수 있나요?**

물론입니다! "견적서 상태"와 "견적서 금액"을 결합하는 등 여러 필터를 추가하여 워크플로우 요구사항에 맞는 더 구체적인 트리거를 생성할 수 있습니다.

**Q. 견적서 상태에 따라 다른 팀에 알림을 보내는 방법은 무엇인가요?**

각 견적서 상태(예: 발송됨, 승인됨, 거절됨)에 대해 별도의 워크플로우를 설정하고 관련 팀에 알리기 위한 내부 알림을 액션으로 추가할 수 있습니다.

**Q. 승인된 견적서에 대해 자동으로 기회를 생성할 수 있나요?**

네, 워크플로우에 "기회 생성(Create Opportunity)" 액션을 추가하면 승인된 견적서가 자동으로 CRM에서 새로운 기회를 생성하도록 할 수 있습니다.
