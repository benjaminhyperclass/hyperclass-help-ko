# 워크플로우 액션 - 견적서 발송 (Send Estimate)

# 견적서 발송 워크플로우 액션이란

견적서 발송 액션은 견적서 템플릿으로부터 자동으로 견적서를 생성하고 선택한 채널(이메일 또는 SMS 문자)을 통해 연락처에게 전달합니다. 일관된 브랜딩, 세금 계산, 총액이 포함된 제안서를 수동 작업 없이 파이프라인 기반의 영업 활동에서 지속적으로 발송하도록 설계되었습니다.

간단히 말해서, 워크플로우가 이 액션에 도달하면 하이퍼클래스는 선택한 템플릿으로부터 견적서를 작성하고, 사용 가능한 연락처/기회 관리 세부 정보를 채워 넣은 후, 선택한 발송자와 채널을 사용하여 연락처에게 전송합니다.

## **견적서 발송 액션의 주요 장점**

- **제안서 발송 속도**: 리드가 특정 단계에 도달하거나, 폼을 제출하거나, 가격을 문의할 때 즉시 견적서를 발송합니다.

- **일관성 및 브랜딩**: 모든 견적서가 올바른 템플릿, 로고, 세금, 통화를 사용하도록 보장하여 임시로 만든 PDF를 방지합니다.

- **채널 유연성**: 보유한 연락처 데이터와 고객의 선호하는 커뮤니케이션 방식에 따라 **이메일** 또는 **SMS 문자**로 전달합니다.

- **추적 가능성**: 각 견적서가 언제 발송되고, 조회되고, 승인되고, 결제되었는지 기록을 보관합니다.

## **사전 준비사항**

다음 항목들을 미리 준비하면 전달 실패와 템플릿 불일치 문제를 방지하여 영업 과정을 원활하게 할 수 있습니다.

- 로케이션에 결제 기능이 활성화되어 있고 결제 프로세서가 연결되어 있어야 합니다.

- 최소 하나의 **견적서 템플릿**이 생성되고 활성화되어 있어야 합니다(브랜딩, 세금, 통화 설정).

- 연락처가 선택한 채널에 따라 유효한 **이메일** 및/또는 **전화번호**를 보유하고 있어야 합니다.

- 이메일 발송 도메인이 인증되고 규정을 준수하며, SMS 문자 발송이 규정을 준수하고 연락처가 수신에 동의했어야 합니다.

## **워크플로우에서 견적서 발송 액션 설정하는 방법**

다음 단계를 따라 워크플로우에 액션을 추가하고 일관되고 예측 가능한 결과를 위해 설정해보세요.

- 하위 계정에 로그인합니다.

- Automations(자동화) > Workflows(워크플로우)로 이동합니다.
![워크플로우 메뉴 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064719189/original/4mVu7Ke8FwkzhXqYfkNdJ1AiVOUgwWHmZQ.jpeg?1770831880)

- 새 워크플로우를 생성하거나 기존 워크플로우를 엽니다.
![워크플로우 생성 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064719188/original/y2GeElPYuGlLguqDiv1j0_cZHtxo-i_BDQ.jpeg?1770831880)

- + Add New Trigger(+ 새 트리거 추가) 버튼을 클릭하여 견적서 발송 액션과 연관된 트리거를 추가합니다(예: 파이프라인 단계 변경, 기회 상태 변경, 폼 제출 등).

![트리거 추가 GIF](https://jumpshare.com/share/nOMmbw8Ihs2VaiOTezBW+/GIF+Recording+2026-02-11+at+11.17.29+PM.gif)

- +를 클릭하여 액션을 추가하고 Send Estimate(견적서 발송)을 검색합니다.

![견적서 발송 액션 추가 GIF](https://jumpshare.com/share/idL65o39Ln64vYbtGugQ+/GIF+Recording+2026-02-11+at+11.18.38+PM.gif)

- From User(발송자)와 Estimate Template(견적서 템플릿)을 선택합니다.

![발송자와 템플릿 선택 GIF](https://jumpshare.com/share/idL65o39Ln64vYbtGugQ+/GIF+Recording+2026-02-11+at+11.18.38+PM.gif)

- Payment Mode(결제 모드)를 선택합니다 - 실제 결제용은 Live 모드, 청구 불가능한 테스트 검증용은 Test 모드를 사용합니다.

- Sending Mode(발송 모드)를 선택합니다: Create as Draft(초안으로 생성) (견적서가 발송 전 검토를 위해 초안으로 저장됨) 또는 Send Directly(직접 발송) (워크플로우 실행 후 즉시 견적서 발송) 중 선택합니다.

- Channel(채널)을 선택합니다: 인보이스 전달 방식을 Email(이메일), Text(문자), 또는 Email & Text(이메일 및 문자) 중에서 선택합니다. 기본값: Email & Text.

- Save Action(액션 저장)을 클릭합니다.

![액션 설정 스크린샷](https://jumpshare.com/share/n3Jh3US5rMFWGnBmwhFT+/Screenshot+2026-02-11+at+11.30.10%E2%80%AFPM.png)

- 모든 설정이 올바르게 되었는지 워크플로우를 테스트합니다. 워크플로우를 발행하고 저장합니다.

![워크플로우 테스트 스크린샷](https://jumpshare.com/share/1cQVrs8r6S14HnF0R1kh+/Screen+Shot+2026-02-11+at+11.33.32+PM.png)

## **예시: 주택 리모델링 견적서**

시나리오: 주택 리모델링 회사에서 잠재 고객이 관심을 보일 때 신속하게 견적서를 발송하고자 합니다.

설정:

- 트리거: Opportunity Created(기회 생성) (또는 기타 관련 트리거)
- 액션 이름: 주택 리모델링 견적서 발송
- 발송자: Sarah Johnson
- 견적서 템플릿: 주택 리모델링 제안서
- 견적서 모드: Live

결과: 지정된 트리거가 발생하면, 선택된 견적서가 자동으로 고객에게 발송되어 전환율 향상 가능성이 높아집니다.

## **자주 묻는 질문**

**Q: 연락처가 이메일과 전화번호를 모두 가지고 있다면 어떻게 되나요?**

Channel(채널) 필드에서 Email and Text(이메일 및 문자) 옵션을 선택하세요.

**Q: Test 모드가 실제 고객에게 알림을 보내나요?**

아니요, Live 모드로 전환하기 전에 복사본, 변수, 전달을 검증하기 위해 본인이 제어하는 테스트 연락처와 함께 Test 모드를 사용하세요.

**Q: 견적서를 발송할 때 어느 사용자 신원이 사용되나요?**

선택한 From User(발송자)가 사용됩니다. 해당 사용자가 나중에 비활성화되면 액션을 활성 발송자로 업데이트하세요.

**Q: 견적서가 조회되거나 승인되었는지 어디서 확인할 수 있나요?**

연락처 또는 견적서 영역을 열어 발송됨, 조회됨, 승인됨, 거절됨, 결제됨과 같은 전달 기록과 상태 이벤트를 확인하세요.
