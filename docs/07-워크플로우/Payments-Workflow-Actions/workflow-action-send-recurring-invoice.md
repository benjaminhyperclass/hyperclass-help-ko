# 워크플로우의 정기 인보이스 발송 액션이란?

워크플로우의 정기 인보이스는 워크플로우 내에서 예정된 청구를 자동화하여, 당신이 고객에게 집중하는 동안 인보이스가 자동으로 발송되도록 합니다. 시각적 워크플로우 빌더에서 시작 날짜, 종료 규칙, 빈도, 자동 결제를 설정할 수 있어 수동 작업 없이도 현금 흐름을 예측 가능하게 유지할 수 있습니다.

## 정기 인보이스 발송 액션의 주요 장점

- **시간 절약**: 수동 인보이스 생성 및 후속 조치를 제거하여 시간을 절약합니다

- **유연한 주기**: 일일, 주간, 월간, 연간 일정으로 모든 플랜에 맞춤 설정 가능

- **시작 날짜 제어**: 연락처의 트리거 날짜 또는 고정된 캘린더 날짜에서 시작

- **종료 규칙**: 무한 반복 또는 X회 후 종료 선택 가능

- **자동 결제**: 첫 번째 성공적인 결제 후 저장된 카드로 자동 청구

- **오류 감소**: 누락되거나 중복된 인보이스를 방지

- **워크플로우 통합**: 청구 이벤트가 태그, 이메일, 후속 조치와 동기화됩니다

## **사전 요구사항**

필수 사항을 갖추면 정기 인보이스가 문제없이 실행됩니다.

- **인보이스 템플릿**: `Payments(결제) → Invoices(인보이스)`에서 최소 하나의 템플릿 생성

- **결제 게이트웨이**: `Payments(결제) → Integrations(연동)`에서 Stripe(또는 지원되는 다른 게이트웨이) 연결

## 정기 인보이스 액션 설정 옵션

정기 인보이스 발송 액션은 매우 세밀하게 설정할 수 있습니다. 이 섹션에서는 Publish를 클릭하기 전에 모든 플랜에 맞게 청구를 조정할 수 있도록 각 옵션을 설명합니다.

#### **시작 날짜 옵션**

**시작 날짜** 드롭다운에서 일정이 언제 시작되는지 선택하여 인보이스가 적절한 시점에 고객에게 도달하도록 합니다.

- **액션 날짜**: 연락처가 워크플로우 트리거에 도달하자마자 일정을 시작합니다(예: 태그 추가 또는 연락처 변경)

- **고정 날짜**: 특정 캘린더 날짜에 청구를 맞춥니다(예: 2025년 7월 1일)

#### **종료 기준 및 정지 규칙**

**정지** 드롭다운은 일정이 언제 끝나는지 제어하여 과다 청구나 의도하지 않은 청구를 방지합니다.

- **절대 안 함**: 수동으로 정지할 때까지 계속됩니다

- **X회 후**: 할부 플랜에 완벽합니다(예: 12개월 결제)

#### **반복 빈도**

**빈도** 드롭다운은 인보이스가 얼마나 자주 반복되는지 결정하고, 선택에 따라 추가 선택기(예: 요일, 월)를 표시합니다.

- **일간**: N일마다 - 고빈도 결제에 이상적

- **주간**: 특정 요일이나 액션 날짜에 발송 - 주간 코칭에 적합

- **월간**: 날짜별 또는 주별(예: 둘째 주 수요일) - 멤버십과 리테이너에 일반적

- **연간**: 월과 일을 선택하여 N년마다 - 연간 갱신에 이상적

#### **자동 결제**

**자동 결제 활성화**를 토글하면 첫 번째 성공적인 결제 후 자동으로 청구하여 수금을 간소화합니다.

- **게이트웨이 요구사항**: 현재 Stripe가 연결된 하위 계정에서 지원됩니다

- **첫 인보이스는 성공해야 함**: 성공적인 수동 결제 후에만 카드 정보가 저장됩니다

## 워크플로우에서 정기 인보이스 발송 액션 설정하는 방법

정기 인보이스 발송 액션을 올바르게 설정하는 데는 몇 번의 클릭만 필요합니다. 오류 없이 청구를 자동화하려면 아래의 번호가 매겨진 단계를 따르세요.

### **1단계:** 워크플로우 빌더로 이동

- Hyperclass 대시보드에서 `Automation(자동화) → Workflows(워크플로우)`로 이동합니다.

- 우측 상단 모서리의 파란색 **+ Create Workflow(워크플로우 생성)** 버튼을 클릭하여 새 워크플로우를 만듭니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049011373/original/yFXCqMLZ1da5zLCe_gmAJVo9tSo_FkKRlA.png?1751055820)

### **2단계:** 워크플로우 트리거 추가

- 워크플로우 빌더에서 **+ Add Trigger(트리거 추가)**를 클릭합니다.

- **트리거를 선택**합니다. 예: 계약 서명/폼 제출/기회 상태 변경.

- **Save Trigger(트리거 저장)**를 클릭합니다.

![](https://jumpshare.com/share/IiK5F1gz15xPHD5ZpVP4+/GIF+Recording+2026-02-12+at+12.22.44+AM.gif)

### 3단계: **정기 인보이스 발송** 액션 추가

- 트리거 바로 아래의 **➕**를 클릭하여 액션을 추가합니다.

- 액션 **검색창**에 Invoice를 입력하고 Payments 아래의 **Send Recurring Invoice(정기 인보이스 발송)**를 선택합니다.

![](https://jumpshare.com/share/rJQ9lshtXeJqB7LC4FVX+/GIF+Recording+2026-02-12+at+12.24.33+AM.gif)

### 4단계: 액션 설정 구성

- 쉽게 식별할 수 있도록 액션 이름을 지정합니다(예: "월 리테이너 – 실버").

- **From User(발송자)**를 선택하여 발송자를 선택합니다.

- 인보이스 템플릿을 선택합니다.

- **모드**를 **Test(테스트)**(시험 실행용) 또는 **Live(라이브)**(실제 청구용)로 토글합니다.

- **시작 날짜**에서 **액션 날짜**(연락처가 워크플로우 트리거에 도달하자마자 일정 시작) 또는 **고정 날짜**(특정 캘린더 날짜에 청구 맞춤)를 선택합니다.

- **정지**에서 **절대 안 함**(수동으로 정지할 때까지 계속) 또는 **X회 후**(할부 플랜에 완벽, 예: 12개월 결제)를 선택합니다.

- **빈도** 드롭다운을 열어 일간, 주간, 월간, 연간을 선택하고 나타나는 간격 옵션을 조정합니다.

- **사전 발송 일수**를 입력하여 마감일 N일 전에 인보이스를 발송합니다.

- **자동 결제 활성화**를 켜서 첫 결제 후 고객의 카드를 저장하고 후속 인보이스를 자동으로 청구합니다.

- 채널 선택: 인보이스가 전달되는 방법을 선택합니다 **이메일**, **문자**, 또는 이메일 & 문자. **기본값:** 이메일 & 문자.

- **Save Action(액션 저장)**을 클릭합니다.

![](https://jumpshare.com/share/TkbmYE9Irf9hrvZisxIw+/GIF+Recording+2026-02-12+at+12.19.36+AM.gif)

### 5단계: 테스트 및 발행

**팁:** 워크플로우 탐색에 대한 자세한 내용은 [워크플로우 빌더 안내서](workflow-builder-walkthrough.md)를 참조하세요

- 워크플로우 설정이 완료되면 우측 상단의 파란색 **Save(저장)** 버튼을 클릭합니다.

- 워크플로우 상단에서 **Test Workflow(워크플로우 테스트)**를 클릭하여 워크플로우를 테스트합니다.

- 모든 것이 올바르게 보이면 **Publish(발행)**로 토글하여 워크플로우를 활성화합니다.

![](https://jumpshare.com/share/BjgHfxHLA7ZNYwlhXFqO+/Screen+Shot+2026-02-12+at+12.25.53+AM.png)

## **자주 묻는 질문**

**Q: 정기 인보이스를 발송하려면 결제 게이트웨이가 필요한가요?**
네. 자동 결제를 활성화하거나 인보이스를 발송하기 전에 Stripe(또는 지원되는 다른 게이트웨이)를 연결해야 합니다.

**Q: 자동 결제 중 고객의 카드가 실패하면 어떻게 되나요?**
인보이스는 미결제 상태로 남습니다. Payment Failed 트리거를 사용한 워크플로우를 설정하여 재시도하고 고객에게 알림을 보내세요.

**Q: 정기 인보이스가 고객의 인보이스 기록에 나타나나요?**
네—생성된 모든 인보이스는 연락처의 레코드에 기록됩니다.

**Q: Live/Test 토글이 기존 일정에 영향을 주나요?**
아니요. 토글 변경 후 생성되는 인보이스에만 적용됩니다.

**Q: 연락처 속성에 따라 다른 템플릿을 보낼 수 있나요?**
조건부 분기를 사용하여 서로 다른 템플릿을 가진 여러 정기 인보이스 발송 액션을 추가하세요.
