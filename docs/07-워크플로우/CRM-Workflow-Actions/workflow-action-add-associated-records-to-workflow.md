# 워크플로우 액션 – 연관 레코드를 워크플로우에 추가

## 개요

연관 레코드를 워크플로우에 추가(Add Associated Records to Workflow) 액션은 트리거 객체와 연관된 레코드들을 다른 워크플로우에 자동으로 추가할 수 있게 해주는 기능입니다.

이 액션은 모든 워크플로우 유형에서 사용할 수 있습니다 — 연락처(Contact) 기반, 회사(Company) 기반, 커스텀 객체(Custom Object) 기반 워크플로우 모두에서 말이죠. 트리거 레코드와 연관된 레코드들을 다른 워크플로우에 자동으로 추가할 수 있습니다.

이 액션은 기존의 [워크플로우에 추가(Add to Workflow) 액션](../Internal-Tools-Workflow-Actions/action-add-to-workflow-workflow-action-add-to-workflow.md) 기능을 확장한 것입니다. 워크플로우에 추가 액션은 현재 레코드(예: 연락처)만을 다른 워크플로우에 추가하는 반면, 연관 레코드를 워크플로우에 추가 액션은 연관 라벨을 기준으로 연관된 레코드들(연락처, 회사, 커스텀 객체)을 등록할 수 있습니다.

## 액션 이름

연관 레코드를 워크플로우에 추가(Add Associated Records to Workflow)

## 액션 설명

이 액션을 통해 워크플로우 사용자는 다음을 할 수 있습니다:

- **객체 유형을 선택**합니다.

- 어떤 연관 레코드를 선택할지 결정하는 **연관 라벨을 선택**합니다. (회사-연락처 연관에는 적용되지 않음)

- 해당 연관 레코드들을 다른 워크플로우에 직접 등록합니다.

이 액션은 하위 계정에서 커스텀 객체를 생성(또는 연락처 사용)하고 트리거 레코드와 연관 레코드 간에 연관 라벨이 존재하는 경우에만 가능합니다.

## 액션 상세사항

![연관 레코드를 워크플로우에 추가 액션 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054809211/original/AsJo3DMQEwM_1N24PA3G-AYtaD1Dyxju5g.png?1759143523)

- 객체 선택(Select an Object): 추가할 레코드 유형을 선택합니다.

- 연관 라벨(Association Label): 필수 드롭다운입니다. 어떤 연관 레코드가 포함될지 정의합니다.

- 워크플로우(Workflow): 연관 레코드들이 추가될 워크플로우를 선택합니다.

## 예시

#### 예시 1: 자동차 ↔ 딜러 (커스텀 객체 워크플로우)

자동차 CRM에서 자동차가 딜러와 연관되어 있습니다. 자동차 레코드의 상태가 '판매 준비 완료'로 업데이트되면, 워크플로우가 자동으로 연관된 모든 딜러(배정된 딜러 라벨 하에)를 딜러 알림 워크플로우에 추가합니다.

설정

- 객체 → 딜러
- 연관 라벨 → 배정된 딜러
- 워크플로우 → 딜러 알림 워크플로우

작동 방식

- 자동차 레코드가 '판매 준비 완료'로 업데이트됩니다.
- 액션이 '배정된 딜러' 라벨 하에 있는 연관된 모든 딜러를 찾습니다.
- 각 딜러가 딜러 알림 워크플로우에 등록되어 자동 알림을 받게 됩니다.

#### 예시 2: 연락처 ↔ 연락처 (연락처 워크플로우)

대학교에서 학생과 보호자의 연락처를 모두 관리합니다. 새 학기가 시작되면, 워크플로우가 자동으로 모든 보호자 연락처(부모-자녀 라벨로 연관됨)를 학기 시작 알림 워크플로우에 추가합니다.

설정

- 객체 → 연락처(보호자)
- 연관 라벨 → 부모-자녀
- 워크플로우 → 학기 시작 알림

작동 방식

- 학기 시작 시 학생 연락처 레코드가 워크플로우를 트리거합니다.
- 액션이 부모-자녀 라벨 하에 있는 연관된 모든 보호자 연락처를 찾습니다.
- 해당 보호자들이 학기 시작 알림 워크플로우에 등록되어 적시에 업데이트를 받게 됩니다.

#### 예시 3: 회사 ↔ 연락처 (회사 워크플로우)

마케팅 에이전시에서 회사와 연관된 연락처를 관리합니다. 회사 레코드가 VIP 고객 프로그램에 추가되면, 연관된 모든 연락처가 자동으로 VIP 커뮤니케이션 워크플로우에 등록됩니다.

설정

- 객체 → 연락처
- 워크플로우 → VIP 커뮤니케이션 워크플로우

작동 방식

- 회사 레코드가 새로운 상태(VIP 고객)로 업데이트됩니다.
- 해당 연락처들이 VIP 커뮤니케이션 워크플로우에 등록되어 VIP 전용 업데이트를 받게 됩니다.
