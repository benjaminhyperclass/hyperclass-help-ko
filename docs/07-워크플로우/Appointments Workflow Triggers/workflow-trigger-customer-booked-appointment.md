# 워크플로우 트리거 - 고객 예약 완료

이 문서는 **"고객 예약 완료(Customer Booked Appointment)"** 워크플로우 트리거를 자세히 안내합니다. 고객이 직접 예약을 등록할 때 자동으로 작업을 실행하는 강력한 도구입니다. 이 문서를 통해 트리거의 작동 방식, 주요 장점, 설정 과정, 실제 활용 사례를 이해할 수 있습니다.

## 고객 예약 완료 워크플로우 트리거란?

고객 예약 완료 워크플로우 트리거는 고객이 예약 링크나 온라인 예약 시스템을 통해 팀의 개입 없이 직접 예약을 등록할 때마다 활성화됩니다. 예약을 기반으로 후속 조치, 알림, 안내를 자동화하고 싶은 비즈니스에 필수적인 트리거입니다. 일반 예약에만 작동하며 반복 예약은 지원하지 않습니다.

---

## 고객 예약 완료 트리거의 주요 장점

예약 기반 워크플로우를 자동화하면 운영 효율성을 크게 향상시키고 고객 경험을 개선할 수 있습니다. 주요 장점은 다음과 같습니다:

- **자동 알림:** 이메일이나 SMS를 통한 시기적절한 예약 알림으로 노쇼율을 줄입니다.

- **향상된 고객 경험:** 자동 확인 메시지와 후속 조치로 고객과의 소통을 유지하고 참여도를 높입니다.

- **동행자 등록 지원:** 주 연락처뿐만 아니라 예약의 동행자에게도 워크플로우를 자동화합니다. 다중 참가자 예약의 모든 참가자에게 알림, 확인, 후속 조치를 보냅니다.

- **효율적인 팀 알림:** 예약이 등록될 때마다 관련 팀원에게 자동으로 알림을 전송합니다.

- **유연한 커스터마이징:** 필터를 사용해 특정 캘린더, 캘린더 그룹, 특정 태그가 있는 고객에게만 워크플로우를 실행합니다.

- **효과적인 리소스 관리:** 알림과 후속 조치를 자동화하여 팀이 일정을 수동 관리하는 대신 서비스 제공에 집중할 수 있습니다.

## 고객 예약 완료 트리거 설정하기

워크플로우에서 고객 예약 완료 트리거를 설정하려면 다음 단계를 따르세요:

### 워크플로우 빌더 접근

**Automation(자동화)** 섹션으로 이동합니다. 새 워크플로우를 처음부터 만들거나 "고객 예약 완료" 트리거를 사용할 기존 워크플로우를 선택하세요.

### 새 트리거 추가

"Add New Trigger(새 트리거 추가)"를 클릭하고 "Customer Booked Appointment"를 입력합니다. 목록에서 선택하세요.

![고객 예약 완료 트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039511639/original/QEwrxU5nNDDAWa5ooiHjxx9uGsDQpXDLhg.png?1736393761)

### 트리거 이름 설정

"신규 예약 등록 트리거"와 같이 트리거를 쉽게 식별할 수 있는 설명적인 이름을 입력하세요.

![트리거 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068508556/original/mu54oUjD-RgoXPvg080TspkhC-hlttuL-Q.png?1775457382)

### 워크플로우 등록 대상 선택

트리거 이름을 설정한 후, 트리거가 실행될 때 누가 워크플로우에 등록될지 선택하세요. 이 설정은 **"워크플로우에 등록될 대상(Who should be enrolled into the workflow?)"** 아래에 표시되며 세 가지 옵션을 제공합니다:

- **연락처만(Contact only):** 예약의 주 연락처에게만 워크플로우가 실행됩니다. 이것이 기본 설정입니다.

- **연락처와 동행자(Contact and Guests):** 주 연락처와 예약에 추가된 각 동행자 연락처에게 각각 워크플로우가 실행됩니다. 모든 참석자가 알림이나 후속 조치를 받아야 하는 다중 참가자 예약에 유용합니다.

- **동행자만(Guests only):** 동행자 연락처에게만 워크플로우가 실행되고 주 연락처는 제외됩니다. 주 연락처가 이미 별도의 워크플로우를 가지고 있고 동행자만 대상으로 하고 싶을 때 사용합니다.

**중요:** 고객 예약 완료 트리거의 경우, 예약이 등록된 **후에** 추가된 동행자는 이 트리거 이벤트에 등록되지 않습니다. 예약 시점에 있던 동행자만 포함됩니다.

![워크플로우 등록 대상 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068508470/original/hbKRE-AxbAal8IO-kCh6DQG2wq8R5R5ljg.png?1775457221)

### 필터 설정

다음 필터를 사용해 트리거를 세밀하게 조정할 수 있습니다:

- **Has Tag(태그 보유):** 특정 태그를 가진 고객이 예약을 등록할 때만 트리거가 활성화되도록 하려면 이 필터를 선택하세요.

- **In Calendar(캘린더 내):** 특정 캘린더를 대상으로 하려면 이 필터를 사용하세요. 선택된 캘린더에 예약이 등록될 때만 워크플로우가 실행됩니다.

- **In Calendar Group(캘린더 그룹 내):** 캘린더 그룹을 대상으로 하려면 이 필터를 선택하세요. 선택된 그룹 내의 모든 캘린더에 예약이 등록될 때 워크플로우가 실행됩니다.

![필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039511687/original/n4_hvmC9gRzuiDr8BmhogC3uwBodFFTN8A.png?1736393909)

### 트리거 저장

필터 설정이 완료되면 **Save(저장)** 버튼을 클릭하세요. 우측 상단의 파란색 저장 버튼을 클릭하여 전체 워크플로우도 저장하는 것을 잊지 마세요.

## 활용 사례

이 트리거를 사용할 수 있는 몇 가지 예시를 살펴보겠습니다:

### 활용 사례 #1 예약 알림

**시나리오:** 치과 병원이 노쇼를 줄이기 위해 예약 24시간 전에 환자에게 자동 알림을 보내고 싶어합니다.

**결과:** 워크플로우가 예약 24시간 전에 환자에게 이메일과 SMS 알림을 보내어 참석률을 향상시킵니다.

### 활용 사례 #2 예약 후 후속 조치

**시나리오:** 부동산 중개업체가 매물 보기를 예약한 고객에게 후속 이메일을 보내 세부사항을 확인하고 필요한 안내사항을 제공하고 싶어합니다.

**결과:** 워크플로우가 예약 세부사항, 매물까지의 길 안내, 필요 서류 체크리스트가 포함된 확인 이메일을 보냅니다.

### 활용 사례 #3 VIP 고객 내부 알림

**시나리오:** 금융 자문 회사가 VIP 고객이 예약을 등록할 때마다 시니어 어드바이저에게 알림을 보내고 싶어합니다.

**결과:** 워크플로우가 시니어 어드바이저에게 내부 이메일 알림을 보내어 예정된 예약에 대비할 수 있도록 합니다.

---

### 활용 사례 #4 그룹 예약 관리

**시나리오:** 피트니스 센터가 고객이 그룹 캘린더에서 세션을 예약할 때마다 알림을 자동화하여 그룹 트레이닝 세션을 관리하고 싶어합니다.

**결과:** 워크플로우가 고객에게 이메일 확인을 보내고 배정된 트레이너에게 내부 알림을 전송합니다.

### 활용 사례 #5 다중 참가자 예약 알림

**시나리오:** 컨설팅 회사가 주요 고객과 추가 이해관계자들이 동행자로 참석하는 전략 세션을 예약합니다. 모든 참가자가 확인 이메일과 준비 체크리스트를 받기를 원합니다.

**결과:** 등록 설정에서 "연락처와 동행자"를 선택함으로써, 워크플로우가 주 연락처와 각 동행자에게 개인화된 확인 이메일과 준비 체크리스트를 보내어 모든 참가자가 준비된 상태로 참석할 수 있도록 합니다.

## 자주 묻는 질문

**Q. 이 트리거를 반복 예약에 사용할 수 있나요?**

아니요, 고객 예약 완료 트리거는 일반(일회성) 예약에만 작동합니다. 반복 예약은 지원되지 않습니다.

**Q. 다양한 유형의 예약에 대해 서로 다른 워크플로우를 실행하려면 어떻게 하나요?**

"캘린더 내(In Calendar)" 또는 **"캘린더 그룹 내(In Calendar Group)"** 같은 필터를 사용해 특정 예약 유형을 대상으로 하고 각각에 대해 별도의 워크플로우를 설정할 수 있습니다.

**Q. 팀원이 고객을 대신해 수동으로 예약을 등록하면 어떻게 되나요?**

이 트리거는 고객이 예약 링크를 통해 직접 예약을 등록할 때만 활성화됩니다. 수동으로 등록된 예약은 워크플로우를 실행하지 않습니다.

**Q. 예약 48시간 전과 24시간 전처럼 여러 간격으로 알림을 보낼 수 있나요?**

네, 동일한 워크플로우 내에서 여러 알림 액션을 추가하고 각각에 다른 대기 시간을 설정할 수 있습니다(예: 48시간 대기 후 첫 번째 알림 전송, 추가로 24시간 대기 후 두 번째 알림 전송).

**Q. VIP 고객이 예약한 예약을 어떻게 추적할 수 있나요?**

**"태그 보유(Has Tag)"** 필터를 사용해 특정 태그(예: "VIP 고객")를 가진 고객이 예약을 등록할 때만 실행되는 워크플로우를 만들 수 있습니다. 이렇게 하면 중요한 고객에 대한 시기적절한 후속 조치와 특별한 처리를 보장할 수 있습니다.

**Q. 예약 동행자에게도 워크플로우 알림을 보낼 수 있나요?**

네. 트리거를 설정할 때 "워크플로우에 등록될 대상(Who should be enrolled into the workflow?)" 설정에서 "연락처와 동행자" 또는 "동행자만"을 선택하세요. 이를 통해 예약에 추가된 동행자 연락처에게도 워크플로우가 실행됩니다.

**Q. 예약이 등록된 후에 동행자가 추가되면 어떻게 되나요?**

예약 등록 후에 추가된 동행자는 원래 트리거 이벤트에 등록되지 않습니다. 예약 시점에 있던 동행자만 포함됩니다.

**Q. 등록 설정을 업데이트하면 기존 워크플로우에 영향을 주나요?**

아니요. 기존의 모든 트리거 설정은 "연락처만"의 기본 동작으로 변경되지 않고 유지됩니다. 동행자를 포함하려면 등록 설정을 수동으로 업데이트해야 합니다.
