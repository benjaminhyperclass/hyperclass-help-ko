# **연락처 태그 추가 워크플로우 액션이란?**

이 섹션에서는 연락처 태그 추가 액션이 워크플로우 내에서 연락처에게 하나 이상의 태그를 자동으로 적용하는 방법을 설명합니다. 태그는 연락처를 정리하고, 분류를 만들며, 자동화를 트리거하고, 리포팅을 지원하는 데 도움이 됩니다. 이 액션의 작동 방식을 이해하면 정확한 분류와 시스템 전반의 원활한 자동화 로직을 보장할 수 있습니다.

**연락처 태그 추가** 워크플로우 액션을 사용하면 행동, 유입 경로, 또는 참여 단계에 따라 연락처를 쉽게 분류할 수 있어, 고객을 세분화하고 더 개인화된 자동화 여정을 구축할 수 있습니다.

---

## 연락처 태그 추가 액션의 주요 장점

연락처에 자동으로 태그를 지정하면 데이터베이스에 구조와 명확성을 제공하여, 워크플로우가 깔끔하고 예측 가능한 세분화로 작동할 수 있습니다. 이러한 장점은 에이전시가 시스템 일관성을 유지하고, 타겟팅을 개선하며, 수동 작업을 제거하는 데 도움이 됩니다.

- **세분화:** 타겟 커뮤니케이션을 위해 연락처를 의미 있는 그룹으로 정리합니다.

- **자동화:** 태그 로직을 기반으로 워크플로우 라우팅을 트리거하거나 제어하는 데 도움이 됩니다.

- **개인화:** 연락처의 행동이나 속성을 기반으로 맞춤형 메시징을 가능하게 합니다.

- **데이터베이스 정리:** 여러 워크플로우와 캠페인에서 기록을 구조화된 상태로 유지합니다.

- **확장성:** 명확한 분류 기준으로 성장하는 계정을 지원합니다.

---

## 연락처 태그 추가 옵션

이 액션 내의 각 설정은 태그가 선택, 생성, 또는 액션 설정에서 제거되는 방식에 역할을 합니다. 이러한 옵션을 숙지하면 중복이나 잘못된 분류를 일으키지 않고 자동화 중에 올바른 태그가 적용됩니다.

### **태그 선택**

- 드롭다운 목록에서 기존 태그 중 하나 또는 여러 개를 선택합니다.

- 드롭다운은 검색 기능을 지원하여 태그를 빠르게 찾을 수 있습니다.

- 선택된 태그는 드롭다운 위에 라벨 아이템으로 표시됩니다.

### 태그 삭제

- **"x"** 아이콘을 클릭하면 액션 설정에서 태그가 제거됩니다.

- 이는 연락처 레코드에서 태그를 제거하는 것이 **아니라** 액션 구성에서만 제거합니다.

### **새 태그 추가**

- 필요한 태그가 존재하지 않는 경우, 드롭다운에 새 태그 이름을 입력합니다.

- **새 태그 추가**를 클릭하면 즉시 생성되고 선택됩니다.

- 새로 생성된 태그는 향후 워크플로우와 수동 태깅에서 사용할 수 있게 됩니다.

## 연락처 태그 추가 액션 설정 방법

이 액션을 올바르게 설정하면 워크플로우가 일관되고 예측 가능한 방식으로 태그를 할당할 수 있습니다. 다음 단계를 따르면 오류를 방지하고 태그가 정확한 세분화와 자동화 동작을 유도할 수 있습니다.

### 워크플로우 만들기

왼쪽 메뉴의 **Automation(자동화)** 탭으로 이동하여 워크플로우 라이브러리에 접근합니다. 여기서 기존 워크플로우를 보거나 새로운 워크플로우를 만들 수 있습니다.

워크플로우 목록 페이지의 오른쪽 상단에 있는 **Create Workflow(워크플로우 만들기)**를 클릭하여 새 자동화를 구축합니다. 드롭다운에서 처음부터 시작하거나 레시피를 선택하는 등 필요에 가장 적합한 워크플로우 생성 방법을 선택합니다.

![워크플로우 만들기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155060006609/original/1PIb22hBLgOor97iOO02UN6i-l8Z7Kgf1A.png?1764952023)

### 연락처 태그 추가 선택

워크플로우 빌더에서 **Actions(액션)** 패널을 열고 Contact 섹션으로 스크롤하여 **Add Contact Tag(연락처 태그 추가)** 옵션을 찾습니다. 이 액션을 선택하면 연락처가 워크플로우의 이 단계에 도달했을 때 하나 이상의 태그를 적용할 수 있습니다.

![연락처 태그 추가 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155060006991/original/FqCJFFonvE4doYCTxvO_q0ZVE8kbl6TWaQ.png?1764952256)

### **액션 이름 지정**

**Action Name(액션 이름)** 필드에 명확하고 설명적인 이름을 입력하여 워크플로우 내에서 이 단계가 무엇을 하는지 식별할 수 있도록 합니다. 의미 있는 이름을 사용하면 나중에 워크플로우를 검토하거나 편집할 때 로직을 이해하기 쉽습니다.

![액션 이름 지정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155060007311/original/pSG7P7eLCDwVO2BluBVcZs98m_0aRulXdA.png?1764952409)

### **태그 선택**

**Tags(태그)** 드롭다운을 사용하여 목록에서 기존 태그를 선택하거나, 드롭다운 화살표를 클릭하여 사용 가능한 모든 옵션을 봅니다. 선택한 태그 옆의 **X**를 클릭하면 언제든지 제거할 수 있어, 저장하기 전에 선택을 수정할 수 있습니다.

![태그 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155060007521/original/WVxOuvZnEmK1WZGe03oNQaOJtQoY_K_TVg.png?1764952603)

### 새 태그 만들기

필요한 태그가 존재하지 않는 경우, 필드에 새 태그 이름을 입력하여 생성 옵션을 생성합니다. **Add New Tag(새 태그 추가)**를 클릭하면 즉시 시스템에 추가되고 액션에 포함됩니다.

![새 태그 만들기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155060007719/original/LX_hTVPkzcpZ1U8Matwc1YNsYqmv6l53fw.png?1764952718)

### **태그 모드 선택**

Tags 필드 옆의 점 세 개 메뉴를 열어 **Standard(표준)**와 **Dynamic(동적)** 태깅 모드 사이를 전환합니다. 이를 통해 태그를 선택한 대로 정확히 적용할지, 워크플로우 데이터를 기반으로 동적으로 생성할지 결정할 수 있습니다.

![태그 모드 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155060007838/original/bYYYzQkw7723U2XaCrivTGhBtNSYKt9hcw.png?1764952831)

### 액션 저장

**Save Action(액션 저장)**을 클릭하여 태그 설정을 완료하고 단계를 워크플로우에 추가합니다. 이렇게 하면 연락처가 자동화의 이 지점에 도달할 때마다 선택한 태그가 자동으로 적용됩니다.

![액션 저장](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155060007938/original/4o6s1bCKLqaVnsSooM0aasFe6fkSpJyY6g.png?1764952954)

# 자주 묻는 질문

**Q: 여러 태그를 한 번에 추가할 수 있나요?**

예. 연락처 태그 추가 액션은 필요한 만큼 많은 태그를 선택할 수 있습니다.

**Q: 기회의 커스텀 필드 값을 기반으로 자동으로 태그를 추가할 수 있나요?**

예. **Opportunity Custom Field Changed(기회 커스텀 필드 변경)** 트리거나 워크플로우 내의 조건부 로직을 사용하여 필드 값을 확인한 다음, 연락처 태그 추가 액션을 사용하여 적절한 태그를 자동으로 적용할 수 있습니다.

**Q: 이 액션을 사용하여 태그를 제거할 수 있나요?**

아니요. 태그 제거는 **Remove Contact Tag(연락처 태그 제거)** 액션을 사용해야 합니다.

**Q: 누구나 새 태그를 만들 수 있나요?**

태그 생성은 HighLevel 계정 내의 사용자 권한에 따라 달라집니다.

**Q: 태그는 대소문자를 구분하나요?**

예. "Facebook"과 "facebook"은 별개의 태그로 취급됩니다.

**Q: 이 태그가 다른 워크플로우를 트리거하나요?**

예. **Contact Tag(연락처 태그)** 트리거를 사용하는 워크플로우가 있다면 트리거됩니다.

**Q: 태그를 사용하여 캠페인을 세분화할 수 있나요?**

물론입니다. 태그는 스마트 리스트, 캠페인, 워크플로우의 주요 필터링 도구 중 하나입니다.
