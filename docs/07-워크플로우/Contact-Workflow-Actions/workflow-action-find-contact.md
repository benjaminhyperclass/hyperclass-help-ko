# 연락처 찾기 워크플로우 액션이란

연락처 찾기(Find Contact) 워크플로우 액션을 사용하면 표준 필드나 커스텀 필드 같은 지정된 조건을 사용하여 데이터베이스에서 **연락처를 식별**할 수 있습니다. 이 액션은 **연락처가 미리 지정되지 않은 상태로 트리거되지만** 이메일 같은 **식별 정보**가 있는 워크플로우에 완벽합니다.

일반적으로 이 액션은 워크플로우가 연락처가 연결되지 않은 트리거에 의해 실행될 때 사용됩니다. 예를 들어, 트리거가 인바운드 웹훅(Inbound Webhook)인 경우 하나 이상의 쿼리 매개변수 페이로드를 가집니다. 쿼리 매개변수에 사용자 ID는 포함되지 않지만 이메일이나 기타 식별 정보가 포함된 경우, 연락처 찾기 액션을 사용하여 이메일을 사용자 ID로 변환함으로써 해당 연락처에 대한 액션을 실행할 수 있습니다.

## 연락처 찾기 워크플로우 액션의 주요 장점

- 향상된 워크플로우 효율성: 연락처 검색 프로세스를 자동화하여 시간을 절약하고 수동 오류를 줄입니다.

- 유연한 검색 기능: 표준 필드와 커스텀 필드를 지원하여 정확하고 맞춤형 연락처 검색이 가능합니다.

- 연락처 없는 통합: 인바운드 웹훅 같은 연락처가 없는 트리거로 실행되는 워크플로우를 위해 특별히 설계되었습니다.

- 고급 매칭: 외부 참조 매칭을 포함하여 "Stripe 고객 ID" 같은 커스텀 식별자 기반 검색이 가능합니다.

- 향상된 고객 참여: 커뮤니케이션과 후속 조치를 위한 정확한 타겟팅을 보장하여 고객 상호작용을 개선합니다.

- 분기 지원: 연락처를 찾았는지 여부에 따라 워크플로우에서 자동으로 별도의 경로를 생성하여 각 시나리오에 맞는 맞춤형 액션을 가능하게 합니다.

## **지원되는 매칭 필드**

신뢰할 수 있는 필드를 선택하면 매칭 정확도가 향상됩니다. 가능한 고유 식별자를 사용하고 매칭 전에 수신 데이터가 일관되게 형식화되었는지 확인하세요.

- 표준 필드: 이메일, 전화, 이름/성 등 (선택기에서 선택).

- 커스텀 필드: 자체 필드 사용 (예: 고객 ID, 구독 등급). 커스텀 필드가 존재하고 연락처에 채워져 있는지 확인하세요.

- 외부 참조: Stripe 고객 ID 같은 웹훅/앱 데이터를 커스텀 필드에 매핑하고 이를 매칭에 사용합니다.

---

## **매칭 로직 및 분기 동작**

예측 가능한 매칭 규칙은 더 안전한 자동화를 설계하고 "연락처 찾음 vs. 연락처 찾지 못함" 결과를 계획하는 데 도움이 됩니다.

- 필드 간 AND 로직: 여러 필드를 지정하면 연락처를 반환하기 위해 모든 필드가 일치해야 합니다 (예: 이메일 AND 전화 AND 커스텀 필드).

- 분기: 이 액션은 두 개의 경로(연락처 찾음, 연락처 찾지 못함)를 제공하므로 그에 따라 후속 조치를 취할 수 있습니다 (예: 필드 업데이트 vs. 연락처 생성).

- 여러 연락처가 매칭될 수 있는 경우: 더 구체적인 필드로 조건을 강화하세요 (예: 이메일이나 외부 ID 추가). 필요시 연락처 중복 제거 기본 설정 및 중복 합치기 가이드와 함께 사용하세요.

## "연락처 찾기" 워크플로우 액션 설정하기

워크플로우에서 "연락처 찾기" 액션을 설정하는 단계를 따라하세요.

### 워크플로우 설정 액세스

CRM의 "자동화" 섹션 내 워크플로우 빌더로 이동하세요. 처음부터 새 워크플로우를 생성하거나 "연락처 찾기" 워크플로우 액션을 구현하려는 기존 워크플로우를 선택하세요.

![워크플로우 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054568172/original/frx13mRCX3L_q-7wBWOZyZLO6ig9t7zUgw.png?1758795446)

### 새 액션 추가

"+" 아이콘을 클릭하고 드롭다운 메뉴에서 "연락처 찾기(Find Contact)" 액션을 선택하세요.

![액션 추가 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039249538/original/QeM8v2EHcsAv9Pjb6KYgMm5jB1HlsXax2Q.png?1735920178)

### 액션 이름 지정

"이메일로 연락처 찾기"와 같이 설명적인 이름을 제공하세요.

![액션 이름 지정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039249578/original/F7sI1DcqMK1NL5qk-dvd3J0ozCIbDQBbvw.png?1735920212)

### 표준 필드 선택

이름, 성, 이메일, 전화 등의 옵션이 포함됩니다.

![표준 필드 선택 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039249673/original/gK9muzOm9tJ3EK88Yjkl3w_y2OPGHe9Qxg.png?1735920304)

### 커스텀 필드 선택(선택사항)

커스텀 필드를 아직 생성하지 않았다면 "고객 ID"나 "구독 등급" 같은 고유 필드를 추가하세요. 이미 있다면 시스템 내에서 생성한 커스텀 필드를 찾기 위해 아래로 스크롤하세요.

![커스텀 필드 선택 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039249651/original/6ihylo4OdVHg-gBuJXxczL-wmQr2jSGGeg.png?1735920282)

### 트리거 설정

"인바운드 웹훅" 같은 트리거와 액션을 연결하여 동적으로 활성화하세요.

### 분기 계획

연락처 찾음 하위의 액션들(예: 연락처 필드 업데이트, 이메일 발송)과 연락처 찾지 못함 하위의 액션들(예: 연락처 생성)을 구성하세요.

![분기 계획 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054565262/original/oxBKfAodT0DEOIu6uGMOBbElnooOdL7WEw.png?1758794233)

### 테스트 및 저장

테스트 데이터를 사용하여 설정의 정확성을 검증하세요. 설정이 완료되면 액션을 저장하세요.

### 워크플로우 발행

연락처 검색 프로세스를 효과적으로 자동화하기 위해 워크플로우를 활성화하세요.

## **추가 이메일 또는 전화번호 포함(선택사항)**

기본적으로 연락처 찾기는 연락처의 기본 **이메일 또는 전화** 필드만 매칭합니다.

연락처에 추가 이메일 주소나 전화번호가 저장되어 있다면 이러한 값을 매칭에 포함할 수 있습니다:

1. 표준 필드 선택에서 이메일 또는 전화를 선택하세요.

2. 선택한 필드 아래의 체크박스를 활성화하여 검색에 **추가 이메일/전화번호를 포함**하세요.

## **사용 사례**

이러한 실제 사례들은 언제 연락처 찾기를 사용할지, 어떤 필드를 우선적으로 사용할지, 매칭이 되지 않을 때 무엇을 할지 보여줍니다.

1. 다른 도구에서 새 리드가 들어올 때(이메일로 매칭)

폼, 광고 또는 연동에서 누군가의 이메일이 전송되면 이메일로 연락처 찾기를 사용하세요.

연락처를 찾은 경우: 워크플로우를 계속 진행—태그 추가, 세부정보 업데이트, 메시지 발송.
연락처를 찾지 못한 경우: 연락처를 생성한 후 계속 진행.

2. 구매 또는 구독 업데이트(고객 ID로 매칭)

결제 시스템은 종종 안정적인 고객 번호(고객 ID 같은)를 보냅니다. 이를 커스텀 필드에 저장하고 해당 값으로 찾으세요.

연락처를 찾은 경우: 플랜, 상태 또는 평생 가치를 업데이트.
연락처를 찾지 못한 경우: 연락처를 생성하고 고객 ID를 저장한 후 진행.

3. 캘린더 예약(이메일로 매칭)

일정 앱은 보통 이메일을 필요로 합니다. 이를 사용하여 예약을 올바른 사람과 연결하세요.

연락처를 찾은 경우: 예약을 연결하고 확인 메시지를 발송.
연락처를 찾지 못한 경우: 예약 세부정보로 연락처를 생성하고 팀에 알림.

4. 파트너 추천(추천 코드로 매칭)

파트너는 고유 코드를 전달하여 크레딧을 받을 수 있습니다. 해당 코드를 커스텀 필드에 저장하고 이로 찾으세요.

연락처를 찾은 경우: 추천 정보를 추가/업데이트하고 연락처에 태그 부여.
연락처를 찾지 못한 경우: 추천 코드로 연락처를 생성하고 적절한 너처링 시작.

5. 고객 ID 없는 온라인 주문(이메일로, 선택적으로 성으로 매칭)

상점에서 때때로 이메일과 이름만 보냅니다. 이메일로 시작하되, 많은 연락처가 비슷한 이메일을 공유하면 성을 추가하세요.

연락처를 찾은 경우: 주문 내역과 총액을 업데이트.
연락처를 찾지 못한 경우: 연락처를 생성하고 "고객"으로 태그 부여.

## 자주 묻는 질문

Q. 연락처를 찾지 못하면 어떻게 되나요?

연락처 찾기 액션을 설정하면 "연락처 찾음"과 "연락처 찾지 못함" 두 개의 분기가 생성됩니다. "연락처 찾지 못함" 분기에 적절한 액션들을 생성할 수 있습니다.

Q. 외부 참조 매칭은 어떻게 작동하나요?

"Stripe 고객 ID" 같은 외부 식별자를 커스텀 필드에 저장할 수 있습니다. 연락처 찾기 액션은 이 필드를 사용하여 CRM에서 해당 연락처를 찾습니다.

Q. 이 액션을 다른 워크플로우 액션과 결합할 수 있나요?

물론입니다. 연락처 찾기 액션은 "이메일 발송"이나 "연락처 업데이트" 같은 다른 액션과 연결하여 복잡한 워크플로우를 만들 수 있습니다.

Q. 검색에 여러 필드를 사용할 수 있나요?

네, 검색 조건을 세분화하기 위해 여러 필드를 설정할 수 있습니다. 연락처를 찾기 위해서는 지정된 모든 필드 값이 일치해야 합니다 (예: 이것 AND 이것 AND 이것).

**Q. 다른 트리거를 사용할 수 있나요?**

인바운드 웹훅 트리거 외에도 커스텀 앱 마켓플레이스 트리거(앱 마켓플레이스 앱을 지원하기 위해 서드파티 개발자가 만든 프리미엄 트리거)를 통해 필요한 병합 필드를 전달할 수 있습니다.
