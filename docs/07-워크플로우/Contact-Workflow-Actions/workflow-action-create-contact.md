# 연락처 생성 워크플로우 액션이란

연락처 생성(Create Contact) 워크플로우 액션은 미리 정의된 조건에 따라 새로운 연락처 레코드를 자동으로 생성하거나 기존 레코드를 업데이트하는 기능입니다. 정확한 연락처 정보를 수집하고 유지함으로써 원활한 커뮤니케이션과 데이터 관리를 보장합니다. 이는 고객 온보딩, 리드 관리 및 기타 연락처 기반 활동을 자동화하는 워크플로우의 핵심 구성 요소입니다.

## 연락처 생성 워크플로우 액션의 주요 장점

- **데이터 정확도 향상:** 연락처 레코드가 항상 최신 상태로 유지되어 오류와 중복을 줄입니다.

- **프로세스 간소화:** 연락처 정보를 추가하거나 업데이트하는 수동 작업을 자동화하여 시간과 노력을 절약합니다.

- **커뮤니케이션 강화:** 정확한 연락처 정보를 유지하여 적시 후속 조치와 개인화된 커뮤니케이션을 가능하게 합니다.

- **커스터마이징 가능한 필드:** 기본 필드와 커스텀 필드를 모두 지원하여 비즈니스의 특정 요구사항에 맞게 연락처 레코드를 맞춤화할 수 있습니다.

- **원활한 연동:** 다른 워크플로우 액션 및 트리거와 쉽게 연동하여 견고한 자동화 프로세스를 구축합니다.

## 연락처 생성 워크플로우 액션 설정하기

### **워크플로우 설정에 접근**

CRM의 자동화(Automation) 섹션 내 워크플로우 빌더로 이동합니다. 새 워크플로우를 처음부터 만들거나 "연락처 생성(Create Contact)" 워크플로우 액션을 구현하려는 기존 워크플로우를 선택합니다.

### **새 액션 추가**

"+" 아이콘을 클릭하고 드롭다운 메뉴에서 Create Contact를 선택합니다.

![워크플로우에서 연락처 생성 액션 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039310054/original/WuFv53oy7nz43NidBrihXSA-VHDWn73uZw.png?1736153467)

### **액션 이름 지정**

"웹 폼으로부터 연락처 생성"과 같은 설명적인 이름을 입력합니다.

![액션 이름 지정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039310124/original/yIUzSZ0til10P3zvfOfYySrAuYAzA94NFg.png?1736153517)

### **필드 선택**

연락처 생성 또는 업데이트 프로세스에 포함할 필드를 설정합니다:

### **기본 필드**

이름(First Name), 성(Last Name), 이메일(Email), 전화번호(Phone), 주소(Address) 등이 포함됩니다.

![기본 필드 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039310202/original/nrd2uo6Bpw7q9JOC2PzN9yzKPz1tHP98XA.png?1736153562)

### **커스텀 필드**

"멤버십 ID"나 "추천 소스"와 같이 조직에 특화된 고유 필드를 추가합니다.

![커스텀 필드 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039310264/original/ZgS3qv1q1xL2KKE9rKybCq_mOEmKpUYYog.png?1736153616)

### **설정 저장**

**"저장(Save)"** 버튼을 클릭하여 설정을 확인합니다.

### **테스트 및 발행**

테스트 데이터로 워크플로우가 예상대로 작동하는지 검증합니다. 확인 후 발행(Publish) 토글을 활성화하여 워크플로우를 활성화합니다.

## **활용 사례**

### **웹 폼에서 연락처 자동 생성**

**시나리오:** 비즈니스에서 웹 폼 제출로부터 고객 정보를 수집하고 연락처 레코드를 자동으로 생성하거나 업데이트하려고 합니다.

**액션 설정:**

- **액션:** 연락처 생성(Create Contact)
- **필드:** 이름(First Name), 성(Last Name), 이메일(Email), 전화번호(Phone), 도시(City), 국가(Country)

**결과:** 워크플로우가 CRM에서 연락처 레코드를 생성하거나 업데이트하고 후속 조치를 위해 영업팀에 알립니다.

![웹 폼 연락처 생성 예시](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039343561/original/TIFnSKhBhNfXN0OC5QIiycpiPtHLKFWzNQ.png?1736175733)

### **구매 후 연락처 레코드 업데이트**

**시나리오:** 고객이 구매할 때 CRM 프로필에 거래 세부 정보로 업데이트해야 합니다.

**액션 설정:**

- **액션:** 연락처 생성(Create Contact)
- **필드:** 전체 이름(Full Name), 이메일(Email), 구매 내역(Purchase History, 커스텀 필드)

**결과:** 워크플로우가 최신 구매 정보로 고객 프로필을 업데이트하고 "활성 고객(Active Customer)" 태그를 부여합니다.

---

### **외부 데이터 소스에서 리드 관리**

**시나리오:** 서드파티 플랫폼의 리드를 CRM으로 가져와 레코드를 자동으로 생성하거나 업데이트해야 합니다.

**액션 설정:**

- **액션:** 연락처 생성(Create Contact)
- **필드:** 회사명(Business Name), 연락처 소스(Contact Source), 전화번호(Phone), 이메일(Email)

**결과:** 워크플로우가 모든 리드를 관련 세부 정보와 함께 CRM에 정확히 기록하여 후속 조치를 준비합니다.

---

### **신규 고객 온보딩**

**시나리오:** 구독 기반 비즈니스에서 각 신규 고객을 위한 연락처 레코드를 생성하고 온보딩 전문가에게 배정하려고 합니다.

**액션 설정:**

- **액션:** 연락처 생성(Create Contact)
- **필드:** 이름(First Name), 성(Last Name), 구독 등급(Subscription Tier, 커스텀 필드), 이메일(Email)

**결과:** 워크플로우가 연락처 레코드를 생성하고, 온보딩 팀에 배정하며, 환영 이메일을 발송합니다.

---

### **이벤트 등록 정보 수집**

**시나리오:** 이벤트 등록 폼의 참석자 정보를 추적 및 커뮤니케이션을 위해 CRM에 기록해야 합니다.

**액션 설정:**

- **액션:** 연락처 생성(Create Contact)
- **필드:** 전체 이름(Full Name), 이메일(Email), 이벤트 이름(Event Name, 커스텀 필드)

**결과:** 워크플로우가 모든 참석자의 연락처 레코드를 생성하고 향후 참여를 위해 이벤트 이름으로 태그를 부여합니다.

## **자주 묻는 질문**

**Q. 연락처가 이미 존재한다면 어떻게 되나요?**

연락처 생성 액션은 기존 레코드를 새로운 정보로 업데이트하여 데이터 일관성을 보장하고 중복을 방지합니다.

**Q. 이 액션을 커스텀 필드와 함께 사용할 수 있나요?**

네, 이 액션은 기본 필드와 커스텀 필드를 모두 지원하여 특정 요구사항에 맞게 연락처 레코드를 맞춤화할 수 있습니다.

**Q. 중복 레코드를 방지하려면 어떻게 해야 하나요?**

이 액션은 이메일이나 전화번호와 같은 일치 기준을 사용하여 기존 연락처를 식별하고 새 레코드를 생성하는 대신 기존 레코드를 업데이트합니다.

**Q. 이 액션을 다른 워크플로우 액션과 결합할 수 있나요?**

물론입니다. 연락처 생성 액션은 **"이메일 발송(Send Email)"**이나 **"할 일 배정(Assign Task)"**과 같은 액션과 원활하게 연동하여 포괄적인 워크플로우를 구축할 수 있습니다.

**Q. 연락처가 생성되거나 업데이트될 때 팀에 알림을 보낼 수 있나요?**

네, 같은 워크플로우 내에서 알림 액션을 추가하여 새로운 연락처나 업데이트된 연락처에 대해 관련 팀에 알릴 수 있습니다.
