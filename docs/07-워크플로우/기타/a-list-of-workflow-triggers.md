# 워크플로우 트리거 전체 목록

이 문서에서는 Hyperclass 워크플로우 빌더(Workflow Builder)에서 사용할 수 있는 모든 트리거(Trigger)를 살펴보겠습니다. 트리거는 특정 이벤트를 기반으로 워크플로우에서 액션을 시작하는 역할을 합니다. 트리거는 카테고리별로 정리되어 있어 필요한 것을 빠르게 찾을 수 있습니다. 이 가이드는 각 트리거를 요약하고 효과적으로 이해하고 사용할 수 있는 핵심 정보를 제공합니다.

# 트리거와 액션 이해하기

트리거는 워크플로우를 시작하는 이벤트로, 다양한 액션이 수행되도록 이끕니다. 워크플로우에서 지정한 순서대로 실행됩니다.

- 워크플로우 트리거: 특정 이벤트를 기반으로 워크플로우를 시작합니다.
- 워크플로우 액션: 트리거 다음에 실행되며 지시된 작업을 수행합니다.

# 워크플로우 트리거 전체 목록

이 마스터 리스트를 사용하여 현재 Hyperclass에서 사용 가능한 모든 트리거가 자동화 라이브러리에 반영되어 있는지 확인하세요. 각 항목은 워크플로우를 시작하는 요소와 유용한 이유를 독자가 빠르게 스캔할 수 있도록 간단하고 복사하기 쉬운 설명입니다.

### **연락처**

- **생일 알림:** 설정한 오프셋을 사용하여 연락처의 생일 또는 그 전후에 실행됩니다.

- **연락처 변경:** 지정한 연락처 필드가 정의한 값으로 변경될 때 실행됩니다.

- **연락처 생성:** 새 연락처 레코드가 CRM에 추가될 때 실행됩니다.

- **연락처 DND:** 연락처의 방해 금지 설정이 켜지거나 꺼질 때 실행됩니다.

- **연락처 태그:** 선택한 태그가 연락처에 추가되거나 제거될 때 실행됩니다.

- **커스텀 날짜 알림:** 연락처의 선택한 커스텀 날짜 필드 전/당일/후에 실행됩니다.

- **노트 추가:** 연락처에 새 노트가 추가될 때 실행됩니다.

- **노트 변경:** 기존 연락처 노트가 편집될 때 실행됩니다.

- **할 일 추가:** 연락처에 대한 할 일이 생성될 때 실행됩니다.

- **할 일 알림:** 할 일의 알림 시간이 되었을 때 실행됩니다.

- **할 일 완료:** 연락처의 할 일이 완료로 표시될 때 실행됩니다.

- **연락처 참여 점수:** 참여 점수가 설정한 규칙을 충족할 때 실행됩니다.

### **이벤트**

- **인바운드 웹훅:** 워크플로우의 웹훅 URL로 데이터가 수신될 때 실행됩니다.

- **스케줄러:** 연락처 없이 시간 기반 일정에 따라 실행됩니다.

- **통화 세부사항:** 통화 로그가 선택한 세부사항이나 결과와 일치할 때 실행됩니다.

- **이메일 이벤트:** 이메일 전달, 열람, 클릭, 반송, 스팸, 구독 취소 시 실행됩니다.

- **고객 응답:** 연락처가 연결된 채널에서 응답할 때 실행됩니다.

- **대화 AI 트리거:** 구성된 대화 AI 이벤트가 발생할 때 실행됩니다.

- **커스텀 트리거:** 비표준 사용 사례를 위해 정의한 커스텀 이벤트에서 실행됩니다.

- **폼 제출:** 선택한 Hyperclass 폼이 제출될 때 실행됩니다.

- **설문 제출:** 선택한 설문이 제출될 때 실행됩니다.

- **트리거 링크 클릭:** 연락처가 정의된 트리거 링크를 클릭할 때 실행됩니다.

- **비디오 추적:** 시청자가 비디오의 선택한 비율에 도달할 때 실행됩니다.

- **번호 검증:** 전화번호 검증 결과에 따라 실행됩니다.

- **메시징 오류 – SMS:** 발신 SMS가 특정 오류를 반환할 때 실행됩니다.

- **LinkedIn 리드 폼 제출:** LinkedIn 리드 생성 폼 제출이 수신될 때 실행됩니다.

- **퍼널/웹사이트 페이지뷰:** 연락처가 지정된 페이지/URL 또는 UTM을 조회할 때 실행됩니다.

- **새 리뷰 수신:** 평판 관리에 새 리뷰가 도착할 때 실행됩니다.

- **WhatsApp 클릭 광고:** 클릭-투-WhatsApp 광고에서 인바운드 WhatsApp 스레드가 시작될 때 실행됩니다.
