# 워크플로우 액션이란? (전체 목록)

Hyperclass의 워크플로우 액션(Workflow Actions)은 워크플로우 빌더 내에서 다양한 작업의 자동화 및 관리를 지원하는 핵심 구성 요소입니다. 액션은 트리거 이벤트 발생 후 순차적으로 실행되어 작업 순서를 정밀하게 제어할 수 있습니다.

# **트리거와 액션 이해하기**

트리거는 워크플로우를 시작하는 이벤트로, 다양한 액션이 실행되도록 합니다. 워크플로우에서 지정한 순서대로 시간순으로 진행됩니다.

- 워크플로우 트리거: 특정 이벤트를 기반으로 워크플로우를 시작합니다.
- 워크플로우 액션: 트리거 후에 이어지며 지시된 작업을 수행합니다.

## **연락처 액션**

**연락처 생성(Create Contact):**

- 시스템에 새 연락처를 추가합니다. 새로운 리드 관리에 유용합니다.
- 활용법: 리드 수집과 정리를 자동화할 수 있습니다.

**연락처 찾기(Find Contact):**

- 제공된 데이터를 기반으로 시스템에서 연락처를 찾습니다.
- 활용법: 기존 연락처를 업데이트하거나 참조하는 데 도움이 됩니다.

**연락처 필드 업데이트(Update Contact Field):**

- 연락처의 특정 필드를 수정합니다.
- 활용법: 연락처 정보를 동적으로 업데이트할 수 있습니다.

**연락처 태그 추가(Add Contact Tag):**

- 연락처에 태그를 추가하여 정리와 분류를 돕습니다.
- 활용법: 더 나은 연락처 관리와 타겟팅을 가능하게 합니다.

**연락처 태그 제거(Remove Contact Tag):**

- 연락처에서 태그를 제거합니다.
- 활용법: 연락처 목록 정리와 분류 유지에 도움이 됩니다.

**사용자에게 배정(Assign to User):**

- 시스템의 사용자에게 연락처를 배정합니다.
- 활용법: 개인화된 커뮤니케이션과 작업 할당이 가능합니다.

**배정된 사용자 제거(Remove Assigned User):**

- 연락처에서 배정된 사용자를 제거합니다.
- 활용법: 연락처 재배정이나 관리에 도움이 됩니다.

**대화 편집(Edit Conversation):**

- 대화를 표시, 보관 또는 보관 해제로 편집합니다.
- 활용법: Hyperclass 내에서 대화 관리를 지원합니다.

**방해 금지 비활성화/활성화(Disable/Enable DND):**

- 연락처의 "방해 금지" 설정을 비활성화하거나 활성화합니다.
- 활용법: 연락처 선호도에 따라 아웃바운드 커뮤니케이션을 제어합니다.

**메모 추가(Add Note):**

- 연락처에 커스텀 메모를 추가합니다.
- 활용법: 연락처 상호작용을 수동으로 추적할 수 있습니다.

**할 일 추가(Add Task):**

- 연락처와 관련된 할 일을 생성합니다. 연락처가 있으면 해당 연락처에 연결되고, 연락처 없이 워크플로우가 실행되면 연락처 없는 할 일을 생성합니다. 인바운드 웹훅과 함께 사용하면 연락처가 없어도 내부 할 일을 생성할 수 있습니다.
- 활용법: 후속 조치와 작업 관리에 도움이 됩니다.

**연락처 복사(Copy Contact):**

- 다른 하위 계정으로 연락처를 복제합니다.
- 활용법: 계정 간 데이터 공유와 관리를 지원합니다.

**연락처 삭제(Delete Contact):**

- 시스템에서 연락처를 제거합니다.
- 활용법: 연락처 목록 유지 관리에 도움이 됩니다.

**연락처 참여도 점수 수정(Modify Contact Engagement Score):**

- 연락처의 참여도 점수를 조정합니다.
- 활용법: 연락처 상호작용과 반응성을 평가하는 데 도움이 됩니다.

**연락처 팔로워 추가/제거(Add/Remove Contact Followers):**

- 연락처에 팔로워를 추가하거나 제거합니다.
- 활용법: 팀이나 부서 내에서 연락처 공유 가시성을 지원합니다.

## **커뮤니케이션 액션**

**이메일 발송(Send Email)**

- 연락처에게 이메일을 보냅니다.
- 활용법: 이메일을 통한 개인화된 커뮤니케이션으로 후속 조치, 업데이트, 알림을 제공합니다.

**SMS 발송(Send SMS)**

- 연락처에게 SMS를 보냅니다.
- 활용법: 문자를 통한 직접적이고 즉시 커뮤니케이션으로 알림, 리마인더, 빠른 상호작용에 적합합니다.

**Slack 메시지 발송(Send Slack Message)**

- Slack이 연동된 경우 Slack을 통해 메시지를 보냅니다.
- 활용법: Slack 내에서 팀 커뮤니케이션과 내부 알림을 지원하여 협업과 작업 관리를 효율화합니다.

**전화(Call)**

- 연락처에게 전화를 겁니다. 고객이 받으면 사용자에게 연결을 시도합니다.
- 활용법: 리드 아웃리치와 관리를 자동화하여 자동 다이얼링과 배정된 사용자와의 효율적 연결에 유용합니다.

**메신저(Messenger)**

- 연락처에게 페이스북 메시지를 보냅니다.
- 활용법: 페이스북 메신저를 통한 인바운드 메시지 관리로 개인화된 커뮤니케이션과 고객 참여를 가능하게 합니다.

**인스타그램 DM(Instagram DM)**

- 인스타그램 다이렉트 메시지를 보냅니다.
- 활용법: 인스타그램 다이렉트 메시지를 통한 고객 참여로 소셜 미디어 상호작용과 고객 지원을 향상시킵니다.

**수동 작업(Manual Action)**

- 사용자가 연락처에 대해 수동 작업을 수행하도록 안내합니다.
- 활용법: 수동 개입의 유연성을 제공하여 리드 관리와 개인화된 상호작용에 유용합니다.

**구글 마이 비즈니스 메시징(GMB Messaging)**

- 구글 마이 비즈니스 메시지에 응답합니다.
- 활용법: 구글 마이 비즈니스를 통한 고객 커뮤니케이션으로 지역 비즈니스 참여와 고객 서비스를 향상시킵니다.

**내부 알림 발송(Send Internal Notification)**

- 배정된 사용자나 연락처에게 알림을 보냅니다.
- 활용법: 시스템의 업데이트나 트리거에 대해 팀원에게 알려 팀 협업과 작업 관리를 돕습니다.

**리뷰 요청 발송(Send Review Request)**

- 리뷰 요청을 보냅니다.
- 활용법: 리뷰 요청을 자동화하여 평판 관리와 고객 피드백 수집을 향상시킵니다.

**대화 AI(Conversation AI)**

- 고객과의 인바운드 대화를 관리합니다.
- 활용법: AI를 활용하여 여러 채널에서 고객 문의를 처리하고 응답 시간과 고객 만족도를 향상시킵니다.

**페이스북 인터랙티브 메신저(Facebook Interactive Messenger)**

- 페이스북 게시물의 댓글에 응답합니다.
- 활용법: 페이스북에서 고객과 소통하여 소셜 미디어 존재감과 상호작용을 향상시킵니다.

**인스타그램 인터랙티브 메신저(Instagram Interactive Messenger)**

- 인스타그램 게시물의 댓글에 응답합니다.
- 활용법: 인스타그램에서 참여를 높이고 커뮤니티 상호작용과 브랜드 인지도를 구축합니다.

**댓글에 답장(Reply in Comments)**

- 페이스북이나 인스타그램 게시물의 댓글에 답장합니다.
- 활용법: 댓글에 응답하여 참여를 지원하고 팔로워와의 관계를 구축합니다.

**왓츠앱(WhatsApp)**

- 왓츠앱 메시지를 보냅니다.
- 활용법: 고객 지원과 참여를 위한 왓츠앱 가이드라인을 준수하면서 왓츠앱을 통한 커뮤니케이션을 가능하게 합니다.

**실시간 채팅 메시지 발송(Send Live Chat Message)**

- 실시간 채팅 메시지에 응답합니다.
- 활용법: 실시간 채팅을 통한 문의에 응답하여 실시간 고객 지원을 향상시키고 고객 만족도를 높입니다.

## **데이터 전송**

**웹훅/커스텀 웹훅(Webhook/Custom Webhook):**

- Hyperclass에서 외부 애플리케이션이나 서비스로 데이터를 전송합니다.
- 활용법: 다른 도구 및 서비스와의 연동을 지원합니다.

**구글 시트(Google Sheets):**

- 구글 시트의 데이터를 관리하여 업데이트나 조회를 가능하게 합니다.
- 활용법: 데이터 관리와 보고서 작성을 자동화합니다.

## **내부 도구 액션**

**조건 분기(If Else)**

- 조건에 따라 분기를 생성합니다.
- 활용법: 조건에 따라 연락처를 특정 경로로 안내하여 개인화된 고객 경험을 제공하고 세분화와 맞춤화에 유용합니다.

**대기 단계(Wait Step)**

- 특정 시간 동안 워크플로우를 지연시킵니다.
- 활용법: 나중에 액션이나 커뮤니케이션을 예약하는 데 유용하여 워크플로우 효율성과 타이밍을 향상시킵니다.

**목표 이벤트(Goal Event)**

- 연락처를 특정 이벤트 목표로 안내합니다.
- 활용법: 연락처가 워크플로우의 불필요한 단계를 건너뛸 수 있도록 도와 고객 여정과 자동화를 최적화하는 데 유용합니다.

**분할(Split)**

- 워크플로우 내에서 분할 테스트를 수행합니다.
- 활용법: 다양한 경로나 액션을 비교할 수 있어 마케팅 메시지와 전략 최적화에 유용합니다.

**커스텀 값 업데이트(Update Custom Value)**

- 커스텀 값을 업데이트합니다.
- 활용법: 이벤트나 트리거를 기반으로 커스텀 필드를 동적으로 업데이트하여 데이터 관리와 개인화를 향상시킵니다.

**이동(Go To)**

- 연락처를 다른 워크플로우로 안내합니다.
- 활용법: 복잡한 고객 여정을 중복 없이 관리하여 워크플로우 간 연락처 이동을 지원합니다.

**워크플로우에서 제거(Remove from Workflow)**

- 워크플로우에서 연락처를 제거합니다.
- 활용법: 특정 워크플로우에서 연락처를 제거하여 커뮤니케이션과 액션을 관리하고 타겟팅된 상호작용을 보장합니다.

**배열(Arrays)**

- 여러 값을 하나의 단위로 처리합니다.
- 활용법: 데이터 관리와 조작을 단순화하여 데이터 컬렉션의 정렬, 검색, 반복을 가능하게 합니다.

**드립 모드(Drip Mode)**

- 배치 크기로 워크플로우를 통해 연락처를 드립 방식으로 진행시킵니다.
- 활용법: 연락처가 워크플로우를 통과하는 속도를 제어하여 커뮤니케이션 채널의 과부하를 방지하고 평판과 전달률을 유지합니다.
