# 인바운드 웹훅 워크플로우 프리미엄 트리거 사용 방법

인바운드 웹훅(Inbound Webhook)은 외부 시스템이 GET, POST, PUT 등 다양한 HTTP 요청 방식을 사용해 자동으로 데이터를 전송할 수 있도록 하여 원활한 CRM 연동을 가능하게 합니다. 이러한 실시간 데이터 전송 기능은 CRM 기능을 향상시키고 워크플로우를 간소화하여 다양한 플랫폼과 도구 간의 효율적인 커뮤니케이션과 협업을 촉진합니다.

- 모든 플랜의 에이전시($97, $970, $297, $2970, $497, $4970)는 LC 프리미엄 트리거 및 액션에 접근할 수 있습니다.
- 에이전시 설정(Agency settings)에서 프리미엄 액션 및 트리거를 활성화하면, 기존 및 새로운 하위 계정(Sub-Account)은 100회의 무료 실행 권한을 받습니다.
- 에이전시가 기존 하위 계정에 대한 실행 비용 발생을 방지하려면, 에이전시 뷰에서 각 하위 계정에 대해 재청구(rebilling)를 수동으로 활성화해야 합니다(자세한 정보).
- SaaS 설정(SaaS Configurator)에서 프리미엄 액션이 활성화된 경우, 새로 생성되는 하위 계정은 자동으로 LC 프리미엄 액션 및 트리거에 등록되며 에이전시의 추가 조치가 필요하지 않습니다.

## 인바운드 웹훅이란 무엇인가요?

인바운드 웹훅은 외부 시스템에서 CRM으로 자동 데이터 전송을 촉진하여 통합과 상호 운용성을 향상시키는 강력한 기능입니다. 외부 시스템에서 이벤트가 발생하면 HTTP 요청(POST, GET, 또는 PUT)이 CRM 내 트리거와 연결된 특정 URL로 전송되어 워크플로우가 시작됩니다. 이러한 실시간 데이터 전송 기능은 CRM 기능을 향상시키고 다양한 플랫폼과 도구 간의 워크플로우를 간소화합니다.

인바운드 웹훅을 활용함으로써 비즈니스는 프로세스를 자동화하고 수동 작업을 줄이며 사람의 실수를 최소화하여 효율성과 생산성을 높일 수 있습니다. 또한 중요한 데이터가 서로 다른 시스템 간에 즉시 사용 가능하고 동기화되므로 팀 간의 협업이 향상됩니다. 이러한 원활한 통합은 궁극적으로 의사 결정, 고객 만족도, 비즈니스 성과를 개선합니다.

### 어떤 경우에 유용하게 사용할 수 있나요?

인바운드 웹훅은 CRM 기능을 향상시키고 워크플로우를 자동화하는 다양한 시나리오에서 사용할 수 있습니다. 다음은 몇 가지 실용적인 사용 사례와 예시입니다:

**이커머스 주문 연동:**
온라인 스토어가 있고 고객이 주문할 때 새로운 주문 정보로 CRM을 업데이트하고 싶다면, 새 주문이 생성될 때마다 이커머스 플랫폼에서 데이터를 받을 수 있도록 CRM에 인바운드 웹훅을 설정할 수 있습니다.

예시: 이커머스 플랫폼이 고객명, 이메일, 전화번호, 주문 금액 등의 주문 세부 정보와 함께 웹훅 URL로 HTTP POST 요청을 보냅니다. 그러면 CRM이 고객의 연락처 정보를 생성하거나 업데이트하고 주문 세부 정보를 고객 레코드에 추가합니다.

**고객 지원 티켓팅 시스템 연동:**
고객 문제를 처리하기 위해 별도의 지원 티켓팅 시스템을 사용하는 경우가 있습니다. 고객이 지원 티켓을 생성할 때 인바운드 웹훅을 사용하여 티켓 세부 정보를 CRM으로 전송하면 지원팀이 필요한 모든 정보를 확보할 수 있습니다.

예시: 지원 티켓팅 시스템이 고객 이메일, 티켓 제목, 설명 등의 티켓 세부 정보와 함께 웹훅 URL로 HTTP POST 요청을 보냅니다. 그러면 CRM이 티켓을 해당 고객 레코드와 연결하여 지원팀이 CRM에서 직접 티켓 정보에 접근할 수 있습니다.

**이벤트 등록 연동:**
이벤트를 주최하고 이벤트 등록 플랫폼을 사용한다면, 인바운드 웹훅을 사용하여 참석자 등록 정보를 CRM으로 전송할 수 있습니다.

예시: 이벤트 등록 플랫폼이 이름, 이메일, 전화번호, 이벤트명 등의 참석자 정보와 함께 웹훅 URL로 HTTP POST 요청을 보냅니다. 그러면 CRM이 참석자의 연락처 정보를 생성하거나 업데이트하고 이벤트 등록 세부 정보를 레코드에 추가하여 이벤트팀이 효과적으로 참석자를 관리하고 소통할 수 있습니다.

**웹 폼에서 리드 수집:**
인바운드 웹훅은 웹사이트나 랜딩 페이지의 웹 폼에서 리드를 수집하여 CRM으로 데이터를 전송할 수 있습니다.

예시: 방문자가 웹사이트의 폼을 제출하면, 웹 폼이 이름, 이메일, 전화번호 등의 리드 정보와 함께 웹훅 URL로 HTTP POST 요청을 보냅니다. 그러면 CRM이 새로운 리드를 생성하거나 기존 연락처를 업데이트하여 영업팀이 즉시 리드를 후속 조치할 수 있습니다.

이것들은 인바운드 웹훅이 워크플로우를 자동화하고 프로세스를 간소화하며 다양한 비즈니스 시나리오에서 CRM 기능을 향상시킬 수 있는 몇 가지 예시일 뿐입니다.

## 인바운드 웹훅 워크플로우 트리거는 어떻게 사용하나요?

간단히 말해서, 인바운드 웹훅 트리거를 사용하여 3단계로 데이터를 시스템에 가져올 수 있습니다.

![인바운드 웹훅 설정 단계](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48296396414/original/5xlRF80hbq19hXS2IeQjhZ5GlPK6xGxPLQ.png?1683557063)

#### 1단계

웹훅 URL을 가져와서 애플리케이션이나 시스템에서 사용하세요. 다음으로, 적절한 HTTP 요청 방법(POST, GET, 또는 PUT)을 사용하여 웹훅 URL로 데이터를 전송하고 "Test Trigger" 버튼을 클릭하세요.

![웹훅 URL 복사](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48296396436/original/IlVVYc7eyGT9KdCkdq9wlbT68qjQH71BzA.jpeg?1683557066)

#### 2단계

지원되는 HTTP 요청 방법(POST, GET, 또는 PUT)을 사용하여 애플리케이션이나 시스템에서 받은 데이터를 선택하고, 향후 참조를 위해 샘플 데이터로 저장하세요. "Load More"를 클릭하여 추가 최근 데이터를 확인하세요. 원하는 데이터를 선택한 후 트리거를 저장하세요.

이 단계에서는 커스텀 변수의 데이터 참조를 저장할 수 있으며, 이는 다른 연락처 커스텀 필드나 로케이션 커스텀 변수와 유사하게 처리됩니다.

다음 단계와 아래 예시들에서 이 참조를 활용하는 방법에 대한 자세한 정보를 제공합니다.

#### 3단계

트리거를 저장한 후 "Create Update Contact" 액션으로 이동합니다. 거기서 필요한 필드를 선택하고 트리거에서 받은 수신 데이터를 매핑할 수 있습니다. 새 연락처를 생성하거나 기존 연락처에 데이터를 매핑하려면 이메일 또는 전화 필드를 제공해야 한다는 점에 유의하세요. 트리거 데이터에 이메일이나 전화가 포함되지 않은 경우 워크플로우에서 이 액션을 제거할 수 있으며 워크플로우는 연락처 없이도 계속 실행됩니다.

## 워크플로우가 연락처 없이 실행될 수 있나요?

- 워크플로우는 연락처 데이터 종속성 없이 연락처 없는 상태로 실행될 수 있으므로 인바운드 웹훅 트리거를 통해 페이로드 데이터를 전송하고 워크플로우에서 사용할 수 있습니다.
- 연락처 없이도 진행할 수 있으며 연락처 정보에 종속되지 않는 액션들을 사용할 수 있습니다. 커스텀 웹훅, Google 시트, Slack, ChatGPT 및 모든 내부 도구들은 연락처 없이도 실행될 수 있습니다.
- 필요한 경우 Create/Update 또는 Find Contact 액션을 사용하여 연락처 데이터를 워크플로우로 가져올 수 있습니다.

**예시:**
- 인바운드 웹훅을 통해 주문 데이터를 전송하고 주문 정보를 구글 시트에 추가하며, 조건 분기를 사용해 주문 금액에 따라 분류하고 Slack 알림을 보냅니다.
- Find contact 액션을 사용하여 Contact ID로 연락처를 검색합니다.

#### 고려해야 할 중요 사항들:

- 웹훅과 상호 작용할 때는 항상 지원되는 방법(POST, GET, 또는 PUT)을 사용하여 요청을 보내세요.
- 데이터가 JSON 객체로 전송되는지 확인하세요. 이는 지원되는 유일한 데이터 형식입니다.
- 올바르게 컴파일하려면 키가 공백 구분 없이 단일 문자열이어야 합니다. 키 이름을 공백으로 구분하는 대신 CamelCase나 snake_case 사용을 고려하세요.
- 페이로드에 이메일이나 전화번호 제공이 필수입니다. 워크플로우에는 연락처 정보가 필요하기 때문입니다. 연락처를 찾거나 생성하려면 이메일이나 전화가 필요합니다.
- 커스텀 값에서는 배열이 지원되지 않습니다. 요청에서 전송할 수는 있지만 액션 내에서 사용할 수는 없습니다.
- 데이터 구조가 변경되면 인바운드 웹훅 트리거 설정 내에서 매핑 참조를 다시 선택하여 다른 액션에서 해당 필드들을 올바르게 처리하세요.
- 인바운드 웹훅 트리거 URL이 노출되거나 유출되어 원하지 않는 요청을 방지하고 싶다면, 기존 인바운드 웹훅 트리거를 삭제하고 새로운 인바운드 웹훅 트리거를 추가하세요. 다른 종료 ID가 있는 새 URL이 생성되므로 새 URL에 맞게 연동을 업데이트하세요. 저장하면 이전 URL의 수신 요청은 워크플로우에 진입하지 않으며, 새 URL의 요청만 들어옵니다.
