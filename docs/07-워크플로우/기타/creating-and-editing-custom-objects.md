# 커스텀 오브젝트란?

커스텀 오브젝트를 사용하면 각각 고유한 필드, 연결, 자동화를 가진 새로운 레코드 타입을 정의할 수 있어서, CRM이 실제 비즈니스 프로세스를 반영하도록 만들 수 있습니다. 표준 오브젝트로는 부족할 때 이상적입니다(예: MLS, 방 개수, 상태를 추적하는 "부동산" 또는 품종, 예방접종을 추적하는 "반려동물"). 오브젝트를 만든 후에는 뷰, 필터, 자동화 전반에서 사용할 수 있습니다.

## 커스텀 오브젝트의 주요 이점

실용적인 장점을 이해하면 유연하고 미래에도 사용할 수 있는 데이터 모델을 설계하는 데 도움이 됩니다.

- **유연성**: 커스텀 필드, 라벨, 연결을 가진 모든 엔티티를 정의할 수 있습니다.
- **자동화**: 생성/업데이트 시 워크플로우를 트리거하고 오브젝트 전용 액션을 실행할 수 있습니다.
- **인사이트**: 스마트 리스트와 목록 보기를 통해 오브젝트 데이터를 분석하고, 고급 필터로 정확하게 필터링할 수 있습니다.
- **거버넌스**: 관리자가 오브젝트 정의를 관리하여 명확한 소유권으로 무결성을 유지합니다.
- **플랜 접근 & 제한**: 모든 플랜에서 사용 가능; 로케이션(Location)당 최대 10개 커스텀 오브젝트.

## 필드 타입 & 고유 필드

올바른 필드 타입과 고유성 규칙을 선택하면 데이터를 깨끗하고 병합하기 쉽게 유지할 수 있습니다.

- **현재 지원되는 고유 필드 타입**: 단일 라인 텍스트, 다중 라인 텍스트, 숫자, 전화번호
- **제한**: 오브젝트당 최대 10개 고유 필드
- **범위**: 고유성은 하위 계정(Sub-account) 전체와 모든 진입점(UI, 워크플로우, 폼, API)에서 적용됩니다.
- **되돌릴 수 없는 변경**: 고유 필드를 비고유로 다운그레이드하면, 다시 고유로 만들 수 없습니다.

## 연결 & 라벨

연결은 레코드들을 연결하여(예: 기회 관리 ↔ 부동산) 실제 관계를 반영하고 오브젝트 간 뷰를 지원합니다.

- 오브젝트 간 다대다 또는 일대다 링크를 정의합니다.
- 스마트 리스트, 필터, 자동화에서 연결을 사용합니다.
- **라벨 제한**: 두 오브젝트 간 최대 10개 고유 라벨 (명확성을 위해 "구매자", "등록됨" 등)
- 기회 관리는 이제 통합된 파이프라인 맥락을 위해 커스텀 오브젝트와의 유연한 연결을 지원합니다.

## 커스텀 오브젝트 사용 가능한 위치

커스텀 오브젝트를 위한 올바른 영역을 선택하면 강력하고 호환 가능한 구축을 보장합니다. 이 표를 사용해 현재 가용성을 확인하고 구현을 자신 있게 계획하세요.

| 가용성 | 기능 |
|--------|------|
| **지원됨** | 연락처 & 기회 관리(연결), 워크플로우(트리거 & 액션), 폼, 설문 & 퀴즈, 스마트 리스트, 목록 보기, 고급 필터, 리포팅/대시보드, API, 웹훅, 할 일 |
| **지원 안 됨** | 이메일 캠페인, 대량 이메일, 대량 SMS, 대화 UI, 퍼널 & 웹사이트(동적 CO 데이터), 캘린더 & 스케줄링, 평판/리뷰, 결제 & 인보이스, 회사 오브젝트 내부 |
| **출시 예정** | 노트, 퍼널 & 웹사이트(동적 바인딩), AI 지식 베이스, 연결된 연락처에 워크플로우를 통한 이메일 |

## 커스텀 오브젝트 설정 방법

깔끔한 초기 설정은 모델이 성장함에 따라 안정적인 자동화와 리포팅을 보장합니다. 아래 단계는 가독성을 위해 약간 명확하게 정리되었습니다.

### 1단계 — 오브젝트 만들기

- 하위 계정 → 설정(Settings) → 오브젝트(Objects)로 이동하세요.
- 커스텀 오브젝트 추가(Add Custom Object)를 클릭하세요.

![커스텀 오브젝트 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058405899/original/19muD-f-IgODYh2yNQ--bLnKCPn89sP4Yg.png?1763126731)

### 2단계 — 오브젝트 세부사항 추가

- 단수 이름(Singular name)을 입력하세요 (예: Pet).
- 복수 이름(Plural name)을 입력하세요 (예: Pets).
- 내부 이름(Internal Name)과 기본 필드(Primary field)(기본적으로 생성됨)를 검토하세요. </>을 클릭하여 생성 중에 내부 이름을 조정할 수 있습니다.
- 기본 표시 필드(Primary display field)에 라벨을 제공하세요 (예: Pet Name, Pet Identification Number).
- 아이콘(Icon)을 선택하고 팀이 오브젝트의 목적을 이해할 수 있도록 설명(Description)을 추가하세요.
- 커스텀 오브젝트 만들기(Create Custom Object)를 클릭하세요.

![커스텀 오브젝트 세부사항](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058406204/original/fgF0GrDpzjlp6BKbCr0R0F-t6lSOdYrBxA.png?1763126831)

### 3단계 — 오브젝트 세부사항 편집

- 생성 후, 오브젝트가 오브젝트 페이지에 표시됩니다.
- 오브젝트를 클릭하여 열고 세부사항을 조정하세요.

![오브젝트 편집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058406333/original/_BTexHTI5lwA1w4QfD8N8itV1YC4bALzAw.jpeg?1763126905)

### 선택사항 — 커스텀 오브젝트 삭제 (되돌릴 수 없음)

- 오브젝트 이름 옆의 ⋯ (점 세 개)을 클릭하세요.
- 삭제(Delete)를 선택하세요.
- 확인하기 위해 DELETE를 입력하세요.

![오브젝트 삭제](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058406564/original/DIem8kq3voYpJAFkfcf-oqn43QIQXi24Cg.gif?1763127036)

중요: 삭제하면 오브젝트와 모든 레코드, 연결, 워크플로우, 연결된 커스텀 필드가 제거되며, 이는 복구할 수 없습니다.

## 워크플로우에서 커스텀 오브젝트 사용하기 (자동화)

커스텀 오브젝트(및 필드/연결)를 만든 후에는 오브젝트 기반 워크플로우를 사용하여 해당 오브젝트 주변의 프로세스를 자동화할 수 있습니다. 즉, 워크플로우가 커스텀 오브젝트 레코드 자체에서 "실행"됩니다(연락처만이 아닌). 이를 통해 커스텀 오브젝트 레코드가 생성/업데이트될 때 자동화를 트리거하고 관련 레코드 생성/업데이트, 필드 지우기, 다른 도구로 오브젝트 데이터 전송 등의 액션을 취할 수 있습니다.

### 커스텀 오브젝트 워크플로우의 사용 가능한 트리거

오브젝트 기반 워크플로우에서는 다음 트리거 타입을 사용하여 자동화를 시작할 수 있습니다:

| 트리거 이름 | 동작 |
|-------------|------|
| [커스텀 오브젝트] 생성됨 (예: 주택 생성됨) | 커스텀 오브젝트 레코드가 생성될 때 워크플로우를 시작합니다. |
| [커스텀 오브젝트] 변경됨 (예: 주택 변경됨) | 커스텀 오브젝트 레코드가 업데이트/변경될 때 워크플로우를 시작합니다. |
| 인바운드 웹훅 | 인바운드 웹훅이 수신될 때 워크플로우를 시작합니다 (외부 시스템 업데이트에 유용). |

### 커스텀 오브젝트 워크플로우의 사용 가능한 액션

트리거가 실행된 후 다음과 같은 액션을 사용하여 다음 단계를 자동화할 수 있습니다:

| 액션 이름 | 동작 |
|-----------|------|
| [커스텀 오브젝트] 또는 연결된 레코드 생성 | 새 커스텀 오브젝트 레코드(또는 연결된 레코드)를 생성합니다. |
| [커스텀 오브젝트] 또는 연결된 레코드 업데이트 | 커스텀 오브젝트 레코드(또는 연결된 레코드)의 필드를 업데이트합니다. |
| [커스텀 오브젝트] 또는 연결된 레코드의 필드 지우기 | 커스텀 오브젝트 레코드(또는 연결된 레코드)의 선택된 필드를 지웁니다. |
| 커스텀 웹훅 | 커스텀 오브젝트 데이터를 외부 엔드포인트로 전송합니다. |
| Google 시트 | 커스텀 오브젝트 데이터를 Google 시트로 전송/로그합니다. |

또한 If/Else, 대기(Wait), 이동(Go To), 포매터(Formatters), 커스텀 값 업데이트, 워크플로우 추가/제거 등과 같은 일반적인 워크플로우 유틸리티와 커스텀 오브젝트 자동화를 결합하여 더 고급 로직을 구축할 수 있습니다.

### 커스텀 오브젝트 워크플로우 만드는 방법

- 자동화(Automation)로 이동하여 **+ 워크플로우 만들기** → 워크플로우를 구축하려는 [커스텀 오브젝트] 기반 워크플로우(예: 주택, 차량, 정책, 반려동물 등)를 클릭하세요.

## 자주 묻는 질문

**Q. 커스텀 오브젝트를 만들거나 편집할 수 있는 사람은 누구인가요?**
로케이션 레벨의 관리자만 가능합니다. 일반 사용자는 읽기 전용 액세스 권한을 갖습니다.

**Q. 커스텀 오브젝트를 몇 개까지 만들 수 있나요?**
모든 플랜에서 로케이션당 최대 10개입니다.

**Q. 나중에 내부 이름이나 기본 표시 필드를 변경할 수 있나요?**
아니요—오브젝트가 생성된 후에는 이러한 필드를 편집할 수 없습니다.

**Q. 현재 커스텀 오브젝트가 어디에서 지원되나요?**
현재 가용성과 계획된 영역에 대해서는 위의 지원 매트릭스 표를 참조하세요.

**Q. 커스텀 오브젝트를 삭제하면 어떻게 되나요?**
삭제는 영구적이며 오브젝트와 모든 관련 레코드, 연결, 워크플로우, 커스텀 필드가 제거됩니다.

**Q. 커스텀 오브젝트가 생성되거나 업데이트될 때 액션을 자동화하려면 어떻게 하나요?**
워크플로우에서 커스텀 오브젝트 트리거와 액션을 사용하세요(예: 생성/업데이트/지우기, 레코드 찾기, 웹훅).
