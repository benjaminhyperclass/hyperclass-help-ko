# 커스텀 웹훅 워크플로우 액션 사용 가이드

커스텀 웹훅(Custom Webhook) 액션은 CRM과 외부 서비스 간의 실시간 데이터 교환을 가능하게 해줍니다. 다양한 HTTP 메소드, 인증 방법, 헤더, 쿼리 파라미터를 사용하여 맞춤형 요청을 보낼 수 있습니다. 이 사용자 친화적인 도구는 API 응답과 서버 제한사항을 고려하면서 효율적이고 정확한 워크플로우를 보장합니다.

모든 요금제($97, $970, $297, $2970, $497, $4970)의 에이전시는 LC 프리미엄 트리거 & 액션에 액세스할 수 있습니다.
- 에이전시 설정에서 프리미엄 액션 & 트리거가 활성화되면, 기존 및 신규 하위 계정 모두 100회의 무료 실행이 제공됩니다.
- 기존 하위 계정에 대한 실행 비용 발생을 피하려면, 에이전시 보기에서 각 하위 계정에 대해 수동으로 재청구를 활성화해야 합니다(자세한 정보).
- SaaS 구성기에서 프리미엄 액션이 활성화되어 있다면, 새로 생성되는 하위 계정은 자동으로 LC 프리미엄 액션 & 트리거에 등록되며, 에이전시의 추가 조치가 필요하지 않습니다.

# 커스텀 웹훅 액션이란?

커스텀 웹훅 - LC 프리미엄 액션은 CRM 시스템과 외부 서비스 간의 실시간 통신을 가능하게 하는 강력하고 유연한 기능입니다. 다양한 HTTP 및 인증 방법을 사용하여 지정된 URL로 커스텀 데이터 요청을 구성하고 전송할 수 있습니다.

이 기능은 또한 헤더, 쿼리 파라미터 추가와 커스텀 값 매핑을 지원하여 필요에 맞는 맞춤형 요청 구조를 만들 수 있습니다. 자동완성 제안과 사용자 친화적인 인터페이스를 통해 커스텀 웹훅은 데이터 교환을 간소화하고 워크플로우의 정확성을 보장합니다. 하지만 오류와 워크플로우 실행 중단을 방지하기 위해 이 기능을 사용할 때 API 응답 성공과 서버 제한사항을 고려하는 것이 중요합니다.

## 이 기능은 누구에게 유용한가요?

이 기능은 원활한 데이터 교환, 자동화, 향상된 기능을 위해 CRM 시스템을 외부 서비스와 연동해야 하는 비즈니스와 개인에게 유용합니다. 특히 다양한 플랫폼에서 워크플로우를 간소화하고, 데이터 일관성을 개선하며, 프로세스를 더 효율적으로 관리하고자 하는 마케터, 개발자, IT 전문가에게 특히 유용합니다.

## 주요 장점

- 정확한 연동: CRM 시스템과 외부 서비스 간의 통신을 생성하여 오류가 발생하기 쉬운 수동 데이터 입력을 대체합니다. 자동완성이 데이터 매핑을 제안합니다.
- 유연성: 다양한 HTTP 메소드, 인증 방법, 헤더, 쿼리 파라미터를 지원하여 필요에 맞는 커스텀 요청을 만들 수 있습니다.
- 실시간 업데이트: 실시간 알림과 업데이트를 촉진하여 응답성과 의사결정을 향상시킵니다.
- 확장성: 다양한 업계에 적응 가능하며 운영을 확장할 때 변경이 필요하지 않습니다.

## 커스텀 웹훅 설정 방법

### 1단계: URL 입력

코딩이나 서버 운영 없이도 원하는 모든 URL로 데이터 요청을 전송할 수 있습니다.

### 2단계: 메소드 선택

POST, GET, DELETE, PUT과 같이 필요에 맞는 HTTP 메소드를 선택할 수 있습니다.

![메소드 선택 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294091211/original/fn4VdG-17mzKyBYaCKVsdkdCo_VMZC7tmg.gif?1682327875)

### 3단계: 인증 방법 선택

다양한 유형의 인증 방법을 지원하므로 웹훅에 액세스할 때 필요에 맞는 방법을 선택할 수 있습니다.

![인증 방법 선택 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294091321/original/KDoCu7K61OSjQcvg9r4zWGOgjFAD1idHLg.gif?1682327914)

### 4단계: 헤더 및 쿼리 파라미터

요청 본문 전송 외에도 웹훅으로 요청을 보낼 때 헤더와 쿼리 파라미터를 포함할 수 있습니다.

헤더는 인증 자격 증명, 콘텐츠 타입, 캐시 제어와 같은 요청에 대한 추가 정보를 제공하는 데 사용할 수 있습니다.

반면 쿼리 파라미터는 웹훅 엔드포인트로 매개변수를 전달하는 데 사용할 수 있으며, 받는 응답 데이터를 필터링하거나 수정할 수 있습니다.

![헤더 및 쿼리 파라미터 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294091453/original/NTQRsd_lJ9jCRHBNB3U3Q5s4jhVcucDt6A.gif?1682327951)

### 5단계: 커스텀 웹훅에서 전송할 데이터 구성

커스텀 웹훅 프리미엄 기능을 사용하면 요구사항에 맞는 모든 구조로 커스텀 값을 매핑할 수 있습니다. 사용자 친화적인 텍스트박스는 인바운드 웹훅 변수를 포함한 모든 커스텀 값에 대한 자동완성 제안을 제공합니다.

이 기능은 매핑 프로세스를 간소화하고 입력과 일치하는 잠재적 값을 제안하여 시간을 절약합니다. 또한 데이터 매핑의 정확성과 일관성을 보장하는 데 도움이 됩니다.

![데이터 구성 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294091557/original/Sw2g0QozPRAO007VglNJeG3jDYJRHUwiaw.gif?1682327991)

## 문제 해결

커스텀 웹훅 액션의 성공적인 실행은 참조된 API의 성공적인 응답에 달려 있습니다. 따라서 오류가 반환되면 해당 연락처의 워크플로우 실행에 반영되어 액션이 실패(그 후 건너뛰기)되거나 지수 백오프로 재시도하게 됩니다.

대상으로 하는 실행 수를 수신 서버가 처리할 수 있는지 확인하세요. 그렇지 않으면 한계에 도달하여 오류가 발생할 수 있습니다. 커스텀 웹훅 액션은 연락처가 해당 단계에 도달하는 즉시 실행되므로, 수신 서버의 제한사항을 염두에 두고 워크플로우에 연락처를 추가해야 합니다.
