# WhatsApp 템플릿의 트리거 링크란?

WhatsApp 템플릿 트리거 링크 지원 기능을 사용하면 WhatsApp V2 템플릿에 추적 가능하고 자동화가 준비된 CTA를 배치할 수 있습니다. 연락처가 트리거 링크(버튼 또는 인라인 링크)를 탭하면 Hyperclass가 클릭을 기록하므로 워크플로우가 즉시 분기할 수 있습니다. 이메일과 SMS의 트리거 링크와 유사하게 작동하여 타겟 오퍼, 시기적절한 리마인더, 측정 가능한 고객 참여를 가능하게 합니다.

- 호환성: 워크플로우 내 WhatsApp V2 발송 액션과 함께 작동
- 추적 대상: 연락처가 탭한 CTA/링크
- 자동화: 트리거 링크 클릭에 대한 분기, 후속 조치 발송, 태그 설정, 필드 업데이트 등
- 디자인 옵션: 트리거 링크 버튼(권장) 또는 본문 텍스트의 인라인 트리거 링크

## WhatsApp 템플릿 트리거 링크의 주요 장점

이러한 장점을 통해 트리거 링크를 언제 사용하고 어떻게 템플릿을 구성해야 최고의 전환율과 깔끔한 분석을 얻을 수 있는지 결정할 수 있습니다.

- 클릭 레벨 어트리뷰션: 연락처가 탭한 CTA를 정확히 파악하여 정밀한 라우팅 가능
- 더 스마트한 자동화: 클릭한 링크에 따라 워크플로우를 분기하고 맞춤형 다음 단계 발송
- 빠른 테스트: 버튼 레이블이나 링크 목적지에 대한 A/B 테스트로 전환율 개선
- 깔끔한 핸드오프: 클릭에서 얻은 태그/필드를 사용해 영업이나 서비스 후속 조치 개인화
- 추가 도구 불필요: Hyperclass 내에서 기본 추적 - 외부 추적 도구 불필요

## 사전 요구사항

템플릿이 올바르게 발송되고 추적되도록 먼저 액세스 권한과 승인을 확인하세요.

- 활성 WhatsApp 연결(Hyperclass WhatsApp 구독) 및 완료된 Meta 온보딩
- 템플릿 생성 및 워크플로우 편집 권한이 있는 하위 계정 액세스
- 하나 이상의 트리거 링크 생성 완료(Marketing(마케팅) → Trigger Links(트리거 링크) → Links(링크))
- 승인되었거나 승인 가능한 WhatsApp V2 템플릿(카테고리, 샘플 콘텐츠, 변수가 올바르게 채워짐)

## WhatsApp 트리거 링크 옵션 (하나 선택 또는 둘 다 사용)

추적되는 CTA를 버튼 또는 인라인 링크로 배치할 수 있습니다. 버튼은 더 깔끔하고 탭하기 쉽고, 인라인 링크는 카피에 문장 레벨의 링크가 필요할 때 유용합니다.

- 트리거 링크 버튼(권장): 유형이 트리거 링크인 버튼을 추가하고 링크에 매핑
- 인라인 트리거 링크(본문 텍스트): 변수를 삽입하고 트리거 링크에 매핑하면 링크가 메시지 본문 내에 표시됨

## 버튼 제한 및 디자인 가이드라인

클릭을 최대화하고 혼란을 방지하기 위해 CTA를 명확하고 규정에 맞게 유지하세요.

- 제한: 메시지당 총 버튼 수 ≤ 2개 (트리거 링크 버튼 + 일반 버튼)
- 선택의 과부하를 줄이기 위해 집중된 1~2개의 CTA 사용 권장
- 워크플로우 분기와 깔끔하게 매핑되는 설명적인 레이블 사용(예: "가격 보기", "픽업 예약")
- 템플릿 생성 중 변수에 샘플 콘텐츠 제공하여 Meta 승인 간소화
- 나중에 후속 템플릿을 보내는 경우(아직 답장 없음), 해당 템플릿을 미리 준비/승인해 둘 것

## WhatsApp 템플릿에서 트리거 링크 설정 및 사용 방법

아래 순서를 따르세요. 트리거 링크 생성부터 클릭 기반 분기 구축까지, 데모 플로우를 따라하여 빠르게 테스트하고 안전하게 배포할 수 있습니다.

### 1단계: 트리거 링크 생성

- Marketing(마케팅) → Trigger Links(트리거 링크) → Links(링크)로 이동합니다.
![트리거 링크 생성 과정](https://jumpshare.com/share/1hddN9YA9hV2zGO1r5Ur+/GIF+Recording+2025-09-09+at+10.47.53+PM.gif)

- Add Link(링크 추가)를 클릭하고 이름을 입력하세요(예: 20% 할인 입문 오퍼). 그리고 목적지 URL을 붙여넣으세요(예: 폼이 있는 랜딩 페이지).
- Save(저장)를 클릭합니다.
![트리거 링크 설정 화면](https://jumpshare.com/share/YI6cvgrsItux5YQdBO5V+/Screen+Shot+2025-09-09+at+10.52.46+PM.png)

### 2단계: WhatsApp 템플릿 생성

- Settings(설정) → WhatsApp → Templates(템플릿)으로 이동하고 Create Template(템플릿 생성)을 클릭합니다.
![WhatsApp 템플릿 생성 과정](https://jumpshare.com/share/KhynY0il5K98xUlKoRID+/GIF+Recording+2025-09-09+at+10.57.41+PM.gif)

- Template name(템플릿 이름)을 입력하고 Category(카테고리)(예: Marketing), Language(언어)를 설정합니다.
- (선택사항) 시각적 임팩트를 위해 Header(헤더)(예: 이미지 로고)를 추가합니다.
- Body(본문) 카피를 작성합니다. 필요한 곳에 변수를 추가하세요(예: 연락처 First Name). 승인을 위한 샘플 콘텐츠를 포함하세요.
- (선택사항) Footer(푸터)를 추가합니다(예: "수신 거부하려면 STOP으로 답장").
![템플릿 편집 과정](https://jumpshare.com/share/5g5li2TrM5VfjL6wWcrE+/GIF+Recording+2025-09-09+at+11.02.14+PM.gif)

### 3단계: 트리거 CTA 추가

#### A) 트리거 링크 버튼(권장)

- 템플릿 편집기에서 Button(버튼) → Type(유형): Trigger Link(트리거 링크)를 추가합니다.
- 1단계에서 생성한 트리거 링크를 선택합니다.
- 총 버튼 수를 2개 이하로 유지합니다.
- Create(생성) 버튼을 클릭합니다.
![트리거 링크 버튼 추가 과정](https://jumpshare.com/share/jpzQXWHgdKAOgfpRixks+/GIF+Recording+2025-09-09+at+11.08.59+PM.gif)

#### B) 인라인 트리거 링크(본문 텍스트)

- URL이 나타날 Body(본문)에 커서를 위치시킵니다.
- Add variable(변수 추가)를 클릭한 다음 해당 변수를 트리거 링크에 매핑합니다(예: 20% 할인 입문 오퍼).
- Meta 승인을 위한 샘플 콘텐츠(목적지 URL 붙여넣기)를 추가합니다.
![인라인 트리거 링크 추가 과정](https://jumpshare.com/share/npixuhtkSElsxK1hXAz4+/GIF+Recording+2025-09-09+at+11.18.27+PM.gif)

### 4단계: 제출 및 승인

- Create/Save(생성/저장)를 클릭하여 Meta 승인을 위해 템플릿을 제출합니다.
- 승인 후 템플릿이 Active(활성)로 표시됩니다.
![템플릿 승인 상태](https://jumpshare.com/share/JJ5VmeiZ0xfp5KlUj0tQ+/Screen+Shot+2025-09-10+at+12.48.13+AM.png)

### 5단계: 워크플로우로 추적

- Automation(자동화) → Workflows(워크플로우) → Create from scratch(처음부터 생성)으로 이동하여 이름을 지정합니다(예: Track 20% Offer – WhatsApp).
- Add New Trigger(새 트리거 추가)를 클릭합니다.
- 워크플로우 트리거로 Trigger Link Clicked(트리거 링크 클릭됨)를 선택합니다.
- Add Filters(필터 추가) 버튼을 클릭하고 드롭다운에서 Trigger Link(트리거 링크)를 선택합니다.
- 이전에 생성하여 WhatsApp 템플릿에서 사용한 링크를 선택합니다.
- Save Trigger(트리거 저장) 버튼을 클릭합니다.
- 워크플로우를 발행하고 Workflow History(워크플로우 이력)와 Template Analytics(템플릿 분석)(전달, 읽음, 버튼 클릭)에서 결과를 모니터링합니다.
![워크플로우 설정 과정](https://jumpshare.com/share/f51lXJvSrIOgHrV8c7Qu+/GIF+Recording+2025-09-10+at+12.54.36+AM.gif)

## 자주 묻는 질문

**Q: 버튼 클릭을 추적하기 위해 연동이 필요한가요?**
아니요. 트리거 링크는 Hyperclass에서 기본적으로 추적되며 워크플로우 분기를 작동시킬 수 있습니다.

**Q: 트리거 링크 버튼과 인라인 트리거 링크의 차이점은 무엇인가요?**
버튼은 더 깔끔하고 탭하기 쉽고, 인라인 링크는 본문 텍스트에 위치합니다. 둘 다 클릭을 추적하고 워크플로우 분기를 트리거할 수 있습니다.

**Q: 한 메시지에 몇 개의 버튼을 사용할 수 있나요?**
트리거 링크 버튼과 일반 버튼을 합쳐서 총 2개입니다.

**Q: 연락처가 아직 답장하지 않은 경우 후속 조치할 수 있나요?**
네, 하지만 답장 창 외부에 해당하는 아웃바운드 메시지에는 승인된 템플릿을 사용해야 합니다.

**Q: 어떤 CTA가 가장 좋은 성과를 냈는지 어떻게 알 수 있나요?**
Template Analytics(템플릿 분석)과 Workflow History(워크플로우 이력)를 사용하여 버튼/링크 클릭과 다운스트림 전환을 비교하세요.

**Q: 템플릿을 개인화(예: 이름)할 수 있나요?**
네. 변수를 삽입하고 Meta 승인을 위한 샘플 콘텐츠를 포함하세요. 발송 시 연락처별로 변수가 렌더링됩니다.
