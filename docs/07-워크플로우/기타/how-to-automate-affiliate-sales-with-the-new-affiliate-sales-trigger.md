# "신규 제휴 판매" 트리거로 제휴 판매 후속 조치 자동화하기

판매 후 제휴 마케터와의 소통을 자동화해서 시간을 절약하고 싶으신가요? 워크플로우의 신규 제휴 판매 트리거가 바로 그 일을 해드립니다. 축하 메시지 발송부터 지급 정보 등록 알림까지—모든 것을 자동으로 처리합니다.

## 신규 제휴 판매 트리거란?

신규 제휴 판매 트리거는 제휴 마케터가 성공적으로 판매를 완료할 때마다 실행됩니다—일회성, 정기 구독, 수동 추적 거래 모두 포함됩니다.

활용 방법:

- 제휴 마케터에게 축하 메시지 발송
- 고객에게 알림 발송
- 할 일 배정
- 연락처 분류
- 그 외 다양한 자동화

**중요: 워크플로우 실행 대상 (구매자 vs 제휴 마케터)**

워크플로우의 자동화는 항상 "특정 레코드를 기준으로" 실행됩니다. 신규 제휴 판매 트리거의 경우, 워크플로우는 일반적으로 고객/구매자 측에서 시작되므로, 이메일/SMS 발송 같은 표준 액션은 제휴 마케터가 아닌 구매자 연락처에게 메시지를 보냅니다.

제휴 마케터와 소통하려면 (축하, 알림, 상태 업데이트) 제휴 전용 워크플로우 액션을 사용하세요 (예: 제휴 마케터에게 이메일 발송). [자세히 보기](../../10-마케팅/Affiliate-Manager-Automation/automate-affiliate-sales-follow-ups-with-the-new-affiliate-sale-trigger.md).

## 사용 가능한 필터

다음 필터로 더욱 세밀하게 제어할 수 있습니다:

- 제휴 마케터 필터 – 특정 제휴 마케터만 타겟팅
- 캠페인 필터 – 선택한 캠페인의 판매에만 액션 실행
- 지급 방법 확인 – 제휴 마케터가 지급 방법을 등록했거나 등록하지 않은 경우에만 실행
- 세금 양식 확인 – 제휴 마케터의 세금 양식 제출 여부에 따라 실행

## 설정 방법

워크플로우에서 제휴 판매 트리거를 사용하려면:

- Automations(자동화) → Workflows(워크플로우)로 이동
- Create New Workflow(새 워크플로우 만들기) 클릭하거나 기존 워크플로우 열기
- 트리거 추가: New Affiliate Sale(신규 제휴 판매) 검색
- 선택 사항인 필터 적용 (제휴 마케터, 캠페인, 지급 방법, 세금 양식)
- 액션 추가 (이메일, SMS, 할 일, 태그 등)
- Save(저장) 클릭

## 인기 활용 사례

제휴 판매 트리거의 실제 활용 사례들을 소개합니다:

### 1. 제휴 마케터에게 축하 메시지 발송

- 이유: 제휴 마케터의 동기를 유지하고 재프로모션을 촉진합니다.
- 방법: 트리거 실행 후 제휴 커뮤니케이션 액션(예: Internal Notification(내부 알림))을 추가해서 구매자가 아닌 제휴 마케터에게 메시지가 전송되도록 합니다.

참고: 이 액션은 제휴 판매에 대한 내부 알림(비즈니스/관리자에게)을 보냅니다. 제휴 마케터에게 직접 이메일을 보내지는 않습니다.

- 예시 메시지: "새로운 판매를 축하드립니다! 🎉 계속 화이팅—커미션이 늘어나고 있어요."

내부 알림(Internal Notification) 액션을 추가해서 새로운 제휴 판매가 발생할 때마다 팀에 알림을 보내세요.

"제휴 마케터에게 이메일 발송" 액션은 제휴 마케터와의 커뮤니케이션이 아닌 내부 알림용입니다.

### 2. 제휴 마케터에게 지급 정보 완료 알림

- 이유: 지급 지연을 줄입니다
- 예시: 알림 발송: "커미션을 받으시려면 지급 방법을 업데이트해주세요."

### 3. 제휴 판매 고객에게 태그 추가

- 이유: 제휴 유입 리드를 향후 캠페인을 위해 분류합니다
- 예시: 태그 추가: "제휴 고객"

**알려진 제한사항**

일부 사용자는 신규 제휴 판매 트리거가 자동으로 워크플로우를 제휴 마케터의 커뮤니티 사용자 프로필로 "전환"하기를 기대합니다. 워크플로우는 중간에 기본 레코드를 자동으로 바꾸지 않으므로—제휴 마케터 커뮤니티 포인트 지급은 제휴 마케터의 레코드에서 실행되도록 자동화를 라우팅해야 할 수 있습니다.

**해결 방법**

주로 제휴 마케터와 커뮤니케이션해야 한다면, 트리거 직후에 제휴 커뮤니케이션 액션(예: 제휴 마케터에게 이메일 발송)을 사용하세요.

커뮤니티 포인트를 안정적으로 지급해야 한다면, 대상 사용자가 그룹 멤버일 때만 포인트를 지급하는 워크플로우 설계를 사용하고, Grant community group leaderboard points(커뮤니티 그룹 리더보드 포인트 지급)를 적용하세요.

이러한 자동화를 구현하면 제휴 마케터와 고객 모두에게 원활한 경험을 제공하여 만족도, 참여도, 성과를 향상시킬 수 있습니다.
