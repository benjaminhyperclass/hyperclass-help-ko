# 워크플로우 이메일 액션의 동기화 편집 체크박스 사용하기(지속 동기화)

새로운 동기화 변경 기능으로 워크플로우의 이메일 액션 변경 사항이 템플릿을 업데이트할지 여부를 제어할 수 있습니다. 이메일 액션용 템플릿을 선택할 때 체크박스로 동기화를 쉽게 활성화하거나 비활성화할 수 있습니다.

## 새로운 기능

### 템플릿 선택 시 동기화 편집 체크박스란?

이메일 액션을 생성하고 기존 템플릿을 선택할 때, "템플릿에 편집 동기화(Sync Edits to Template)" 체크박스가 나타납니다.

"템플릿에 편집 동기화" 체크박스를 사용하면 워크플로우 이메일과 연결된 템플릿의 변경 사항이 항상 서로 자동으로 업데이트되는지 제어할 수 있습니다. 켜두면 동기화 상태를 유지하고, 끄면 독립적으로 작동하여 각각 별도로 편집할 수 있습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052494303/original/uSBciiXrkLhMU66n1pNYcaeEFFfqgD-dRw.png?1756300264)

### 동기화 가능성: 동기화가 ON 또는 OFF일 때 일어나는 일

**동기화가 ON일 때:**
- 워크플로우 이메일에서 변경한 모든 사항이 원본 템플릿에 자동으로 업데이트되고, 템플릿 변경 사항도 워크플로우 이메일에 업데이트됩니다.
- 이는 양방향 동기화입니다—워크플로우 액션과 템플릿이 항상 일치합니다.

**동기화가 OFF일 때:**
- 워크플로우 이메일에서 변경한 사항이 원본 템플릿에 영향을 주지 않으며, 향후 템플릿 변경 사항도 워크플로우에 영향을 주지 않습니다.
- 워크플로우 이메일과 템플릿이 이제 독립적인 콘텐츠와 업데이트를 갖게 됩니다.

## 장점

**동기화가 ON일 때:**
- 워크플로우의 이메일이나 원본 템플릿의 변경 사항이 항상 서로 업데이트됩니다.
- 템플릿과 워크플로우 전반에서 콘텐츠를 완벽하게 일치시킵니다.
- 한 곳에서 업데이트를 관리하여 시간을 절약합니다.

**동기화가 OFF일 때:**
- 워크플로우 이메일 액션과 원본 템플릿이 완전히 독립적입니다.
- 다른 템플릿이나 액션에 영향을 주지 않고 특정 워크플로우용으로 이메일을 사용자 지정할 수 있습니다.
- 특정 사용 사례에서 템플릿에서 '벗어나고' 싶을 때 이상적입니다.

## 사용 방법

1. 워크플로우에서 이메일 액션을 추가하거나 편집할 때 템플릿을 선택합니다.
2. "템플릿에 편집 동기화" 체크박스가 나타납니다.

**체크(ON으로 설정)하는 경우:**
- 워크플로우나 템플릿의 변경 사항이 서로 업데이트된다는 모달이 확인을 요청합니다.
- "동기화 켜기"를 클릭하여 확인하면, 이제 모든 편집 내용이 동기화 상태를 유지합니다.
- 예: 워크플로우 이메일을 편집하면 템플릿이 업데이트되고, 템플릿을 편집하면 워크플로우 이메일도 업데이트됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052494386/original/7oTiQYlZJ3DSAYu1VZI4EMpZsvtTTOZKBA.png?1756300305)

**체크하지 않거나 OFF로 설정하는 경우:**
- 워크플로우 이메일과 템플릿이 서로 영향을 주지 않는다는 모달이 설명합니다.
- "동기화 끄기"를 클릭하여 확인하면, 이제 편집 내용이 독립적입니다.
- 예: 워크플로우 이메일의 변경 사항이 템플릿을 업데이트하지 않고, 템플릿 변경 사항이 이 워크플로우 이메일을 업데이트하지 않습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052494668/original/gwi9Dl-AyUTgWS0foDd-eGa1LubTcAlU0g.png?1756300437)

### 동기화 변경 모달:

이 모달은 템플릿 섹션에서 템플릿에 연결된 모든 워크플로우 이메일 액션과 변경 사항을 동기화하고 있는지 확인할 수 있습니다.

템플릿에서 이메일 빌더의 케밥 메뉴에서 "동기화 변경"을 통해 접근할 수 있습니다.

모달 내에서 다음을 볼 수 있습니다:
- 템플릿에 연결된 모든 이메일 액션 목록
- 각 액션의 지속 동기화 상태(활성화/비활성화)
- 활성화된 이메일 액션은 체크박스가 체크되고 비활성화된 이메일 액션은 체크해제됩니다
- 사용자는 이메일 액션을 선택할 수 있고, 지속 동기화가 비활성화된 이메일 액션에 대해 한 번만 템플릿 콘텐츠를 이메일 액션에 동기화할 수 있습니다
- 사용자가 '**한 번 동기화**' 버튼을 클릭하면 화면에 성공 알림이 나타납니다.
- 사용자는 이 모달에서 지속 동기화가 활성화된 이메일 액션의 체크박스를 해제할 수 없습니다.

## 자주 묻는 질문

**Q1:** "템플릿에 편집 동기화" 기능이 무엇인가요?
**A:** 워크플로우 이메일 액션에서 변경한 사항이 연결된 템플릿에 자동으로 업데이트되고, 그 반대도 가능하게 하는 기능입니다. 활성화하면 두 가지가 항상 동기화 상태를 유지합니다.

**Q2:** 동기화 변경 옵션이 언제 나타나나요?
**A:** 이메일 액션을 추가하거나 편집하고 기존 템플릿을 선택할 때 "템플릿에 편집 동기화" 체크박스가 나타납니다.

**Q3:** 동기화를 ON으로 하면 어떻게 되나요?
**A:** 워크플로우 이메일이나 원본 템플릿의 모든 편집 내용이 양쪽 모두에 반영됩니다—모든 것이 완벽하게 일치 상태를 유지합니다.

**Q4:** 동기화를 OFF로 하면 어떻게 되나요?
**A:** 워크플로우 이메일과 템플릿이 독립적이 됩니다. 한쪽에서 변경한 사항이 다른 쪽에 영향을 주지 않습니다.

**Q5:** 어떤 워크플로우가 템플릿에 동기화되어 있는지 어떻게 확인하나요?
**A:** 이메일 빌더의 케밥 메뉴로 이동하여 "동기화 변경"을 선택하세요.

**Q6:** 설정 후에 동기화 설정을 변경할 수 있나요?
**A:** 네, 나중에 어떤 이메일 액션에 대해서도 동기화를 ON 또는 OFF로 설정할 수 있습니다. 변경 사항은 모달에서 확인됩니다.

**Q7:** "템플릿에 편집 동기화" 체크박스의 기본 동작은 무엇인가요?
**A:** 동기화 변경 체크박스의 기본 동작:
- 템플릿에 변경사항이 있는 기존 이메일 액션: 기본값 OFF
- 템플릿에 변경사항이 없는 기존 이메일 액션: 기본값 ON
- 템플릿을 사용하는 새 이메일 액션: 기본값 OFF

**Q8:** 스냅샷을 생성하거나 워크플로우를 복제할 때 동기화 체크박스는 어떻게 되나요?
**A:** 동기화 체크박스 상태가 유지됩니다. 스냅샷 생성 후 대상 계정으로 이어지며, 복제된 워크플로우에서도 동일하게 유지됩니다.
