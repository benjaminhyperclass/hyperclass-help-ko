# 워크플로우 액션 - 목표 이벤트

**목표 이벤트 워크플로우 액션(Goal Event Workflow Action)**을 사용하면 연락처가 수행하는 특정 마일스톤이나 행동을 추적할 수 있습니다. 링크 클릭, 폼 제출, 예약 확인, 결제 완료 등의 행동을 말이죠. 연락처가 설정한 목표 조건을 충족하면, 시스템이 자동으로 워크플로우 내에서 해당 연락처의 경로를 조정하여 자동화를 더욱 지능적이고 반응성 있게 만들 수 있습니다.

목표 이벤트를 활용하면 복잡한 워크플로우를 간소화하고, 중복된 트리거를 제거하며, Hyperclass 내에서 고객 여정 개인화를 향상시킬 수 있습니다.

---

## **목표 이벤트 워크플로우 액션이란?**

목표 이벤트는 워크플로우 내에서 특정 목표를 설정하여 연락처를 원하는 결과로 자동 안내할 수 있게 해줍니다. 연락처가 목표 조건을 충족하면(예: 링크 클릭이나 폼 제출), 시스템이 워크플로우의 어느 단계에 있든 상관없이 즉시 정의된 목표 단계로 이동시킵니다.

목표 이벤트는 자동화의 동적 체크포인트 역할을 합니다. 연락처가 모든 단계를 거치기를 기다리는 대신, 워크플로우가 주요 참여 행동이 발생했을 때를 지능적으로 감지하여 그에 따라 반응합니다. 이를 통해 시간을 절약하고 리드 처리 및 참여도 측면에서 정확성을 향상시킵니다.

---

## 목표 이벤트의 주요 이점

목표 이벤트는 자동화가 실시간 연락처 행동에 동적으로 반응할 수 있도록 하여 워크플로우 효율성을 향상시킵니다. 이를 통해 각 연락처가 맥락에 맞는 관련 커뮤니케이션을 받고 부드럽고 개인화된 여정을 경험할 수 있습니다.

- **자동화 효율성:** 목표가 달성되면 단계를 자동으로 건너뛰거나 워크플로우를 종료합니다.

- **참여도 향상:** 연락처가 클릭, 열람 또는 원하는 행동을 수행할 때 즉시 반응합니다.

- **로직 단순화:** 여러 트리거와 조건 분기(If/Else)를 하나의 스마트 이벤트로 대체합니다.

- **추적 강화:** 이메일 이벤트, 링크 클릭, 태그, 예약, 결제 등을 모니터링합니다.

- **커스터마이징 가능한 액션:** 목표 달성 시 또는 미달성 시 정확히 무엇이 일어날지 정의합니다.

---

## **지원되는 목표 이벤트 유형**

각 목표 유형은 특정 연락처 행동이나 마일스톤을 반영합니다. 올바른 목표를 선택하면 워크플로우가 의미 있는 행동에 반응하고 자동화를 비즈니스 프로세스에 맞출 수 있습니다.

### 폼 제출

하나 이상의 폼 제출에 워크플로우가 반응하도록 합니다.

**주요 특징:**
- 여러 폼을 선택할 수 있는 다중 선택 드롭다운
- 연락처가 선택된 폼 중 하나라도 제출하면 목표 달성

**사용 사례 예시:**
에이전시에서 사용자가 어떤 리드 폼이라도 작성했을 때 후속 조치를 보내고 싶어 합니다. 워크플로우에서 선택된 폼들과 함께 "폼 제출까지 대기"를 사용하여 폼 응답을 중앙화합니다.

### **결제 수신**

결제 이벤트를 기반으로 워크플로우 액션을 트리거합니다.

**주요 특징:**
- "목표 유형 선택 → 결제 수신"에서 사용 가능
- 성공 또는 실패 상태와 상품별로 필터링 가능

**사용 사례 예시:**
- 첫 번째 결제 후 온보딩 시작
- 성공적인 거래에 대한 "감사합니다" 이메일 발송
- 실패하거나 연체된 결제에 대한 알림 트리거

![결제 수신 목표 이벤트 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058400782/original/XdS_bSkvmdwKND6TF1LcxHEUbZNTLOIpMA.jpeg?1763124403)

### **문서 상태**

제안서와 계약서의 생명주기 이벤트인 *조회함*, *서명함*, *거절함*, *완료함* 등을 추적합니다.

**사용 사례 예시:**
- 문서를 조회했지만 24시간 내에 서명하지 않았을 때 알림 발송
- 계약서가 "완료됨"으로 표시되면 온보딩 시작
- 문서가 거절되었을 때 후속 조치 트리거

![문서 상태 목표 이벤트 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058401519/original/9gOkwfUrZ5cHKNvcskj7qCZRteSrmMU1Uw.png?1763124532)

### **이메일 이벤트**

연락처가 이메일과 상호작용할 때를 감지합니다. 열기, 클릭, 구독 해지 등의 행동을 말이죠.

**지원되는 이벤트 예시:**
- 열람함
- 클릭함
- 구독 해지함 (Mailgun만 해당)
- 신고함
- 스팸으로 표시함

**사용 사례 예시:**
- 연락처가 캠페인 이메일을 열거나 클릭했을 때 후속 워크플로우 트리거
- 구독 해지 후 "재참여" 시퀀스로 이동
- 참여도에 따라 타겟 드립 이메일 발송

---

### **트리거 링크 클릭**

메시지 내의 특정 추적 링크를 연락처가 클릭했을 때를 추적합니다.

**사용 사례 예시:**
- 연락처가 상품이나 데모 링크를 클릭했을 때 육성 시퀀스 시작
- 클릭한 콘텐츠 링크에 따라 관심사별로 사용자 세분화
- 리드가 이벤트 등록 링크를 클릭했을 때 후속 이메일 시작

---

### **연락처 태그 추가 또는 제거**

연락처 레코드에 특정 태그가 추가되거나 제거될 때 반응합니다.

**사용 사례 예시:**
- "상품 구매함" 태그가 추가되면 연락처를 새 단계로 이동
- "고객" 태그가 추가되면 육성 워크플로우 중단
- "구독 해지함" 태그가 제거되면 유지 캠페인 트리거

---

### **예약 상태**

연락처의 예약 상태가 *신규*, *확인됨*, *참석함* 등의 특정 상태로 변경될 때를 모니터링합니다.

**사용 사례 예시:**
- 상담 예약이 확인된 후 온보딩 계속 진행
- 상태가 "참석함"으로 업데이트될 때 사후 예약 설문 발송
- "노쇼" 예약에 대한 알림이나 재예약 플로우 트리거

## **목표 이벤트 액션 설정 방법**

목표 이벤트를 설정하면 워크플로우가 중요한 고객 행동에 자동으로 반응하도록 할 수 있습니다. 올바른 설정은 수동 개입 없이 자동화가 원활하게 흘러가도록 보장합니다.

### **1단계: 워크플로우 빌더로 이동**

- **Automation(자동화) → Workflows(워크플로우)**로 이동합니다.
- **+ Create Workflow(워크플로우 생성)**를 클릭합니다.
- **Start from Scratch(처음부터 시작)**를 선택하거나 기존 워크플로우를 엽니다.

![워크플로우 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058401984/original/0ArelTzGBkMyvrcDiosRNvzqb8RQai0ZXQ.jpeg?1763124690)

### **2단계: 목표 이벤트 액션 추가**

- **+** 버튼을 클릭하여 새 액션을 추가합니다.
- **Internal(내부)** 아래에서 **Goal Event(목표 이벤트)**를 선택합니다.

![목표 이벤트 액션 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058402051/original/U90HAJj6vZhgrYnpnuXJqz4R368FfeWJ3g.png?1763124723)

### **3단계: 목표 이벤트 구성**

- 설명적인 **Action Name(액션 이름)**을 입력합니다(예: "리드 전환 목표").
- **Select Type of Goal(목표 유형 선택)**에서 지원되는 목표 유형 중 하나를 선택합니다(이메일, 트리거 링크, 태그, 예약 등).
- 특정 이벤트 기준을 정의합니다(예: 특정 트리거 링크 클릭, 폼 제출, 예약 확인 등).

![목표 이벤트 구성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058402123/original/LM3m1zxhsa7lgOcnpcs4mdHtaJpDSEfzpQ.png?1763124765)

![목표 이벤트 세부 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058402145/original/AhDmwOAn0_-onV-tKK0xoKnIrO2wd7HGrA.png?1763124788)

### **4단계: 워크플로우 동작 선택**

- **If Contact Reaches This Goal Without Meeting Conditions(연락처가 조건을 충족하지 않고 이 목표에 도달할 경우)** 아래에서 다음 중 하나를 선택합니다:
  - **End this workflow(이 워크플로우 종료)**
  - **Continue anyway(어쨌든 계속 진행)**
  - **Wait until the goal is met(목표가 충족될 때까지 대기)**
- **Save Action(액션 저장)**을 클릭하여 적용합니다.

![워크플로우 동작 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058402175/original/stJvS-x5WOQAfDab7oRrpjK-uyrTdCHY2A.png?1763124807)

### **5단계: 워크플로우 발행**

- 오른쪽 상단의 **Publish(발행)**를 클릭하여 워크플로우를 활성화합니다.
- 이제 Hyperclass가 목표 이벤트 기준을 충족하는 연락처를 자동으로 감지합니다.

## **자주 묻는 질문**

**Q: 하나의 이메일 이벤트가 여러 목표 이벤트를 충족할 수 있나요?**

네, 하나의 이벤트(이메일 열기 등)가 워크플로우가 그렇게 구성되어 있다면 여러 목표를 트리거할 수 있습니다.

**Q: 목표 이벤트가 영업 시간 외에도 트리거되나요?**

네. 시스템이 이벤트를 지속적으로 모니터링하므로, 목표를 충족하는 연락처는 즉시 이벤트를 트리거합니다.

**Q: 연락처가 같은 목표 이벤트를 여러 번 충족할 수 있나요?**

아니요, 연락처가 특정 목표 이벤트를 한 번 충족하면 해당 워크플로우 내에서 다시 평가되지 않습니다.

**Q: 목표 이벤트를 충족한 후 예약이 재조정되면 어떻게 되나요?**

워크플로우는 초기 이벤트에만 반응합니다. 이후의 변경사항은 진행 상황을 되돌리지 않습니다.

**Q: 하나의 워크플로우에서 여러 목표 이벤트를 사용할 수 있나요?**

워크플로우당 하나의 목표 이벤트 액션만 추가할 수 있습니다. 하나의 목표 이벤트 내에서 여러 기준을 구성할 수는 있지만, 같은 워크플로우에 둘 이상의 목표 이벤트 액션을 추가할 수는 없습니다.
