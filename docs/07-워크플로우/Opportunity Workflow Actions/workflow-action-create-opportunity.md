## 개요

기회 생성(Create Opportunity) 액션은 지정된 파이프라인과 단계에 새로운 기회를 생성하는 데 사용됩니다. 기회명, 소스, 상태 등의 세부 정보를 자동으로 채울 수 있습니다. 또한 같은 연락처에 대해 이미 기회가 존재하는지에 따라 중복 기회 생성을 활성화하거나 비활성화할 수 있습니다. 이 액션은 특정 워크플로우 조건이 충족될 때 자동으로 기회를 생성하여 리드-기회 전환 과정을 간소화하려는 비즈니스에 특히 유용합니다.

## 액션 이름

기회 생성(Create Opportunity)

## 액션 설명

트리거가 작동하면 기회 생성 액션이 CRM 또는 영업 파이프라인에 새로운 기회 레코드를 추가합니다. 파이프라인, 파이프라인 단계, 기회명, 소스 등 다양한 필드를 구성하여 새로 생성된 기회가 기존 영업 프로세스에 자연스럽게 통합되도록 할 수 있습니다. 연락처 ID를 기준으로 중복 허용 여부를 제어할 수 있는 유연성도 있습니다. 연락처에 이미 기존 기회가 있고 중복이 비활성화된 경우, 해당 연락처에 대해서는 새로운 기회가 생성되지 않습니다.

## 액션 세부사항

![기회 생성 액션 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155043048688/original/BYV_IHghvGkzOAkHMcQvDRQvcEQ7ydamDA.png?1741698740)

| 값 이름 | 설명 | 필수 |
|---------|------|------|
| 액션 이름(Action Name) | 워크플로우에서 이 단계를 식별하기 위한 고유한 라벨입니다. | 예 |
| 파이프라인(Pipeline) | 새 기회가 생성될 파이프라인입니다(예: "프로젝트", "영업" 등). | 예 |
| 파이프라인 단계(Pipeline Stage) | 선택한 파이프라인 내에서 새로 생성된 기회가 배치될 단계입니다(예: "신규 리드", "자격 검증"). 비어있으면 선택한 파이프라인의 첫 번째 단계가 기본값입니다. | 아니요 |
| 기회명(Opportunity Name) | 새 기회의 이름입니다. 비어있으면 연락처 이름이 기본값입니다(사용 가능한 경우). 커스텀 값과 필드를 사용해 동적으로 생성할 수도 있습니다(예: {{contact.first_name}}). | 아니요 |
| 기회 소스(Opportunity Source) | 기회가 어디서 발생했는지 나타냅니다(예: "웹사이트", "추천", "이벤트"). 비어있으면 기본 소스가 없고 그냥 빈 상태로 남습니다. | 아니요 |
| 상태(Status) | 새로 생성된 기회의 상태입니다(예: "열림", "신규", "대기 중"). 비어있으면 "열림"이 기본값입니다. | 아니요 |
| 중복 기회(Duplicate Opportunity) | 같은 연락처 ID에 대해 이미 기회가 존재할 때 새 기회 생성을 활성화하거나 비활성화합니다. 비활성화하면 해당 연락처에 이미 연결된 기회가 있을 때 새 기회가 생성되지 않습니다. | 해당없음 |
| 기회 금액(Opportunity Value) | 기회의 잠재 가치를 나타내는 숫자 필드입니다. 비어있으면 0이 기본값입니다. | 아니요 |
| 소유권 설정(Ownership Settings) | 기회 및 파이프라인(Opportunities & Pipelines) 설정에 따라 기회 소유자가 자동으로 연락처의 소유자가 되거나 할당되지 않은 상태로 남을 수 있습니다. 이 동작은 시스템의 "기회 설정 사용자 정의" 에서 전환할 수 있습니다. | 아니요 |
| 기타 커스텀 필드(Other Custom Fields) | 새 기회를 생성할 때 채우고 싶은 추가 커스텀 필드입니다. 대부분의 필드에서 커스텀 값을 선택하거나 입력할 수 있습니다. | 아니요 |

### 추가 참고사항

날짜 필드의 경우 "지금(Right Now)" 같은 커스텀 값을 사용하여 새 기회에 현재 날짜와 시간을 추가할 수도 있습니다.

### 특수 상황

- **연락처 필드 누락**: 연락처에 이름이나 성이 없는 경우, 시스템은 기회명으로 사용할 수 있는 연락처 식별자(예: 이메일)를 사용하려고 시도합니다.
- **연락처 없음**: 워크플로우와 연결된 연락처가 없는 경우(예: 연락처 데이터 없는 인바운드 웹훅), 기회를 생성할 수 없습니다.
- **중복 확인**: 중복 로직은 기회명이 아닌 연락처 ID를 기반으로 합니다. 중복이 비활성화되어 있고 같은 연락처가 이 액션을 다시 트리거하면 새 기회가 생성되지 않습니다.

## 예시

**시나리오**: 웹사이트에서 "프로젝트 문의" 폼을 작성한 모든 신규 리드에 대해 새 기회를 생성하려고 합니다.

- **트리거**: 연락처가 "프로젝트 문의" 폼을 제출
- **액션**: 기회 생성
  - **액션 이름**: "기회 생성"
  - **파이프라인**: "프로젝트"
  - **파이프라인 단계**: "신규 리드"
  - **기회명**: {{contact.first_name}} - 프로젝트 문의
  - **기회 소스**: "웹사이트"
  - **상태**: "열림"
  - **기회 금액**: 0 (지정되지 않은 경우)
  - **중복 기회**: 활성화 (같은 연락처가 여러 프로젝트 문의를 할 수 있도록)

이 액션이 실행되면 "프로젝트" 파이프라인의 "신규 리드" 단계에 새 기회가 생성됩니다. 같은 연락처가 또 다른 문의를 제출하고 중복이 활성화되어 있으면 두 번째 기회가 생성됩니다. 중복이 비활성화되어 있으면 첫 번째 기회만 생성됩니다.
