# 워크플로우 액션 - 회사 또는 연결된 연락처 업데이트

## 개요

회사 또는 연결된 연락처 업데이트 액션을 사용하면 회사 기반 워크플로우에서 회사 레코드나 회사에 연결된 연락처의 정보를 자동으로 업데이트할 수 있습니다.

이 액션은 관련 이벤트나 트리거(웹훅 업데이트, 폼 제출, 레코드 변경 등)가 발생할 때마다 회사 또는 연락처 정보를 동적으로 업데이트하여 정확한 CRM 데이터를 유지하는 데 도움이 되도록 설계되었습니다.

## 액션 이름

회사 또는 연결된 연락처 업데이트

## 액션 설명

이 액션을 사용하면 다음 작업을 할 수 있습니다:

- 기존 회사 레코드나 회사에 연결된 연락처를 자동으로 업데이트
- 워크플로우 필드, 웹훅 매개변수 또는 고정 텍스트 값을 기존 CRM 필드에 매핑하여 업데이트
- 들어오는 자동화 트리거를 기반으로 회사 및 연락처 데이터를 수정하거나 풍부하게 만들기
- 회사 기반 워크플로우 전반의 데이터 관리 간소화

이 액션은 기존 레코드를 수정하는 것으로, 새 레코드를 생성하지는 않습니다. 새 항목을 생성하려면 대신 회사 또는 연결된 연락처 생성 액션을 사용하세요.

## 액션 상세 내용

![회사 또는 연결된 연락처 업데이트 액션 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055521719/original/rq6GvM4emLWOzDVwZ6ENN5cM4OU2mS9QZQ.png?1759928110)

I. **업데이트할 개체 레코드**

- 업데이트할 대상을 선택합니다:
  - **회사**: 워크플로우에서 현재 회사 레코드를 수정
  - **연결된 연락처**: 해당 회사에 연결된 하나 이상의 연락처 정보를 업데이트

II. **필드 매핑**

- 워크플로우 소스의 데이터를 업데이트할 필드에 매핑합니다
- 매핑 예시:
  - 회사 업종: {{inboundWebhookRequest.body.industry}}
  - 연락처 전화번호: {{inboundWebhookRequest.body.phone}}
  - 연락처 역할: {{inboundWebhookRequest.body.role}}

III. **동적 값 소스**

다음에서 동적 값을 사용할 수 있습니다:

- 인바운드 웹훅 페이로드 (예: {{inboundWebhookRequest.body.companyName}})
- 회사 필드 (워크플로우를 트리거한 레코드의 필드)
- 고정 텍스트 값 (예: "활성", "워크플로우에 의해 업데이트됨")

IV. **필드 추가**

- **+ 필드 추가**를 사용하여 여러 필드를 한 번에 업데이트합니다
- 각 행은 업데이트할 필드-값 쌍을 나타냅니다

V. **업데이트 동작**

- 연결된 연락처를 업데이트할 때, 이 액션은 워크플로우의 회사 레코드에 현재 연결된 연락처만 업데이트합니다
- 일치하는 연결된 연락처가 없으면 액션이 자동으로 건너뜁니다

VI. **액션 저장**

- **액션 저장**을 클릭하여 확인합니다
- 워크플로우가 실행되면 워크플로우 컨텍스트의 최신 값을 기반으로 선택한 필드가 업데이트됩니다

VII. **엣지 케이스 및 참고사항**

- 이 액션은 회사 기반 워크플로우에서만 사용할 수 있습니다
- 새 레코드를 생성할 수 없으며, 이를 위해서는 회사 또는 연결된 연락처 생성을 사용해야 합니다
- 중요한 정보를 덮어쓰는 것을 방지하기 위해 올바른 필드 매핑을 확인하세요

## 예시

#### **예시 1: 웹훅 데이터에서 회사 정보 업데이트**

**목표**: 외부 연동에서 새로운 리드 데이터를 받을 때마다 회사 세부 정보를 자동으로 업데이트합니다.

**설정**

- 워크플로우 유형: 회사 기반
- 트리거: 인바운드 웹훅 - CRM 연동에서 업데이트된 회사 데이터
- 업데이트할 개체 레코드: 회사
- 필드 매핑:
  - 회사명 → {{inboundWebhookRequest.body.company}}
  - 업종 → {{inboundWebhookRequest.body.industry}}
  - 도메인 → {{inboundWebhookRequest.body.domain}}

**흐름**

- 업데이트된 회사 데이터가 웹훅을 통해 전송됩니다
- 워크플로우가 업데이트 액션을 트리거합니다
- 기존 회사 레코드가 새 값으로 자동 업데이트됩니다

#### **예시 2: 회사 하위 연결된 연락처 정보 업데이트**

**목표**: 회사 레코드에 연결된 연락처의 직책과 전화번호를 자동으로 업데이트합니다.

**설정**

- 워크플로우 유형: 회사 기반
- 트리거: 회사 업데이트 또는 인바운드 웹훅 → 직원 데이터 업데이트
- 업데이트할 개체 레코드: 연결된 연락처
- 필드 매핑:
  - 직책 → {{inboundWebhookRequest.body.title}}
  - 전화번호 → {{inboundWebhookRequest.body.phone}}

**흐름**

- 직원 정보가 외부에서 업데이트됩니다
- 워크플로우가 회사에 연결된 연락처를 식별합니다
- 연락처의 직책과 전화번호 필드가 자동으로 업데이트됩니다
