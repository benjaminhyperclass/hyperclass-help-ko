# 워크플로우 트리거 - 그룹 액세스 승인

## 개요

그룹 액세스 승인 트리거 워크플로우는 그룹 접근 권한을 자동으로 관리하여 커뮤니티 상호작용과 사용자 경험을 향상시킵니다.

## 트리거 이름

Group Access Granted (그룹 액세스 승인)

## 트리거 설명

"Group Access Granted" 액션은 사용자가 커뮤니티 내 특정 그룹에 대한 접근 권한을 받을 때 실행됩니다. 이 자동화는 사용자가 시기적절한 업데이트를 받고 관련 콘텐츠 및 멤버와 상호작용할 수 있도록 보장합니다.

## 설정 방법

- **워크플로우 빌더로 이동**: Hyperclass 계정에서 Workflow Builder(워크플로우 빌더)에 접속하세요.
- **새 워크플로우 생성**: "Create a Workflow(워크플로우 생성)"을 클릭하세요.
- 관련 트리거 유형을 선택: "Group Access Granted(그룹 액세스 승인)"
- **트리거 세부사항 설정**: 어떤 그룹 접근 권한이 승인되는지 지정하세요.
- 자동화하고 싶은 후속 액션을 추가하세요 (예: 환영 메시지나 알림 발송).
- **워크플로우 테스트**: 통제된 환경에서 테스트하여 워크플로우가 예상대로 작동하는지 확인하세요.
- **저장 및 활성화**: 확인 후 워크플로우를 저장하고 사용을 위해 활성화하세요.

## 예시

예를 들어, 새로운 멤버가 북클럽 커뮤니티에 가입하면 "도서 토론" 그룹에 자동으로 접근 권한을 부여하는 워크플로우를 설정할 수 있습니다. 이를 통해 멤버는 예정된 토론에 대한 즉각적인 업데이트를 받고 다른 멤버들과 바로 소통할 수 있습니다.
