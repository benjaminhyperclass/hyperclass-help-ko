## **개요**

하이퍼클래스의 인바운드 웹훅 트리거(Inbound Webhook Trigger)를 사용하면 **외부 애플리케이션**에서 오는 데이터를 받아서 **워크플로우를 자동 시작**할 수 있습니다. 이 트리거로 프로세스를 자동화하고 하이퍼클래스를 다양한 서드파티 시스템과 원활하게 연동할 수 있어요.

## **트리거 이름**

Inbound Webhook Trigger (인바운드 웹훅 트리거)

---

## 트리거 설명

인바운드 웹훅 트리거는 하이퍼클래스 웹훅 URL(각 트리거마다 고유)로 들어오는 POST/GET/PUT 요청을 자동으로 감지합니다. 이를 통해 외부 애플리케이션의 이벤트(폼 제출, 결제 알림, 데이터 변경 등)에 실시간으로 반응하여 하이퍼클래스 워크플로우 내에서 자동화를 실행할 수 있어요.

### **인바운드 웹훅 트리거 설정 방법**

1. 워크플로우 설정 접근:
   - 하이퍼클래스에서 새 워크플로우를 만들거나 기존 워크플로우를 여세요.
   - 워크플로우 트리거로 "Inbound Webhook"을 선택하세요.

2. 웹훅 URL 생성: 워크플로우 전용 고유 웹훅 URL이 자동 생성됩니다. 이 URL을 복사해서 외부 애플리케이션에서 사용하세요.

3. 외부 애플리케이션 설정: 외부 앱(예: Zapier, Integromat)에서 하이퍼클래스 웹훅 URL로 데이터를 보내는 트리거를 만드세요. 복사한 URL로 POST 요청을 보내도록 설정하고, 필요한 데이터를 JSON 형식으로 포함시키세요.

4. 연동 테스트: 외부 애플리케이션에서 하이퍼클래스 웹훅 URL로 테스트 요청을 보내 연동이 정상 작동하는지 확인하세요.

5. 수신 데이터 매핑: 하이퍼클래스에서 외부 애플리케이션으로부터 받은 데이터를 워크플로우 내의 관련 필드나 변수에 매핑할 수 있습니다.

6. 트리거 저장: 설정이 완료되면 "Save Trigger(트리거 저장)"를 클릭해서 설정을 완료하세요.

## 예시

**시나리오: 결제 프로세서 연동**

1. **웹훅 URL**: 하이퍼클래스에서 워크플로우를 만들고 웹훅 URL을 생성합니다.

2. **외부 애플리케이션 설정**: Stripe 같은 결제 처리 애플리케이션에서 결제 성공 시 트리거되는 웹훅을 설정합니다. 하이퍼클래스 웹훅 URL로 결제 정보를 JSON 형식으로 POST 요청을 보내도록 설정해요.

3. **전송 데이터**: JSON 페이로드는 다음과 같은 형태입니다:

```json
{
  "customer_id": "CUST_001",
  "amount": 100,
  "currency": "USD",
  "status": "successful"
}
```

4. **워크플로우 액션**: 이 데이터를 받으면 하이퍼클래스 워크플로우가 다음과 같이 설정될 수 있습니다:
   - 리드의 상태를 "결제 완료"로 업데이트
   - 고객에게 확인 이메일 발송
   - 결제 상태에 따른 추가 후속 액션 트리거

이런 연동을 통해 실시간 업데이트와 자동화가 가능해져서 하이퍼클래스 CRM 프로세스의 효율성을 크게 향상시킬 수 있어요.
