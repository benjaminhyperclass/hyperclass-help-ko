# 워크플로우 트리거 - 할 일 완료(Task Completed)

할 일을 효율적으로 관리하는 것은 비즈니스 운영을 원활하게 유지하는 핵심입니다. Hyperclass의 할 일 완료 워크플로우 트리거는 할 일이 완료로 표시되자마자 자동으로 후속 조치와 다음 단계를 실행해줍니다. 이 가이드에서는 기능, 장점, 단계별 설정 방법을 자세히 안내해드립니다.

## 할 일 완료 워크플로우 트리거란?

할 일 완료 워크플로우 트리거는 CRM에서 할 일이 완료로 표시될 때마다 워크플로우의 액션을 자동으로 실행하도록 설계되었습니다. 이를 통해 후속 조치, 알림, 프로세스 진행을 자동화하여 수동 작업을 줄이고 팀 생산성을 향상시킬 수 있습니다.

## 할 일 완료 워크플로우 트리거의 주요 장점

- **자동 후속 조치**: 할 일 완료 후 다음 단계가 자동으로 실행됩니다.

- **팀 협업 향상**: 담당 팀원이나 관련 이해관계자에게 완료된 할 일을 알려줍니다.

- **프로세스 효율성**: 수동 개입 없이 워크플로우에서 할 일을 자동으로 진행시킵니다.

- **사용자별 자동화**: 담당 사용자 필터를 기반으로 워크플로우를 트리거할 수 있습니다.

- **정확한 할 일 추적**: 완료된 할 일로 CRM 데이터를 최신 상태로 유지합니다.

## 할 일 완료 워크플로우 트리거 설정하기

### **워크플로우 빌더 접근하기**

먼저 Hyperclass 대시보드의 **Automation(자동화)** 섹션으로 이동하세요. 새 워크플로우를 설정하는 경우 **+ Create Workflow(워크플로우 생성)**을 클릭한 다음 **+ Start from Scratch(처음부터 시작)**를 선택하세요. 또는 기존 워크플로우를 수정하여 할 일 완료 트리거를 추가할 수도 있습니다.

![워크플로우 생성 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042034342/original/g5qw4injNkpmSYRFxrfnNtwlm9iHugRvMg.png?1740148687)

### **할 일 완료 트리거 추가하기**

워크플로우에 할 일 완료 트리거를 추가하려면 **+ Add New Trigger(새 트리거 추가)**를 클릭하고 사용 가능한 트리거 목록에서 **Task Completed(할 일 완료)**를 검색하세요. 찾으면 선택하여 할 일이 완료로 표시될 때마다 워크플로우가 자동으로 시작되도록 하세요.

![할 일 완료 트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042034384/original/2vOEXiiM3_red78xpkaDpM34_hOSqB1HYA.png?1740148710)

### **담당 사용자 필터 적용하기**

자동화를 세밀하게 조정하고 특정 팀원이 완료한 할 일에만 반응하도록 하려면 담당 사용자 필터를 적용할 수 있습니다. 이 필터를 통해 누가 할 일을 완료했는지에 따라 워크플로우를 트리거할 수 있습니다.

할 일 완료 트리거를 선택한 후 담당 사용자 필터를 선택하고 적절한 연산자를 설정하세요:

- **Is(~임)**: 선택한 사용자가 할 일을 완료했을 때만 워크플로우가 트리거됩니다.
- **Is Empty(비어 있음)**: 할 일에 담당자가 지정되지 않은 경우 워크플로우가 트리거됩니다.
- **Is Not(~이 아님)**: 선택한 사용자를 제외한 모든 사용자에 대해 워크플로우가 트리거됩니다.
- **Is Not Empty(비어 있지 않음)**: 할 일에 담당자가 지정된 경우 워크플로우가 트리거됩니다.

연산자를 선택하면 다음 드롭다운에 사용 가능한 모든 CRM 사용자가 표시되어 특정 팀원을 선택할 수 있습니다. 필터를 적용하지 않으면 어떤 사용자가 할 일을 완료하든 워크플로우가 트리거되어 비즈니스 요구에 따라 유연하게 사용할 수 있습니다.

![담당 사용자 필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042034421/original/RgmJIhW_ih4ZdJ9coSzQA4KPjKhiTu2o3Q.png?1740148745)

### **워크플로우 액션 정의하기**

트리거 설정 후 워크플로우 액션을 추가하여 다음 단계를 자동화하세요.

예시:

- **Send Notification(알림 전송)**: 할 일이 완료되면 팀원들에게 알림을 보냅니다.
- **Update CRM Record(CRM 레코드 업데이트)**: 할 일 완료 후 연락처나 기회 세부정보를 수정합니다.
- **Move Opportunity to the Next Stage(기회를 다음 단계로 이동)**: 할 일이 영업 기회와 연결된 경우 다음 단계로 진행시킵니다.
- **Trigger Another Workflow(다른 워크플로우 트리거)**: 할 일 완료를 기반으로 다른 워크플로우를 활성화합니다.

특정 워크플로우 액션 사용에 대한 자세한 가이드가 필요하시면 Knowledge Base 라이브러리의 해당 문서를 참고하세요.

## 자주 묻는 질문

**Q: 특정 사용자가 완료한 할 일에 대해서만 워크플로우를 트리거할 수 있나요?**

네, 담당 사용자 필터를 사용하고 "Is(~임)" 연산자로 팀원을 선택하면 됩니다.

**Q: 할 일에 담당자가 지정되지 않은 경우는 어떻게 되나요?**

"Is Empty(비어 있음)" 필터 연산자를 사용하여 담당자가 지정되지 않은 할 일에만 워크플로우를 트리거할 수 있습니다.

**Q: 이 트리거로 자동 알림을 보낼 수 있나요?**

트리거 자체는 알림을 보내지 않지만, Send Notification 액션을 추가하여 팀원들에게 알릴 수 있습니다.

**Q: 이 워크플로우로 할 일 완료 후 기회를 업데이트할 수 있나요?**

네, 할 일이 완료로 표시되면 기회 파이프라인이나 상태를 업데이트하는 액션을 설정할 수 있습니다.

**Q: 워크플로우가 제대로 작동하는지 어떻게 확인하나요?**

워크플로우 실행 로그에서 할 일이 완료될 때 트리거가 올바르게 실행되는지 추적할 수 있습니다.
