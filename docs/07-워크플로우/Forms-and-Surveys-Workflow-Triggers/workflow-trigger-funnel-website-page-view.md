## 개요

**퍼널/웹사이트 페이지 조회(Funnel/Website PageView)** 트리거를 사용하면 연락처가 퍼널이나 웹사이트의 특정 페이지를 조회할 때 자동 작업을 실행할 수 있습니다. 이러한 페이지 조회를 추적하여 페이지별 행동에 따라 다양한 자동화를 트리거할 수 있습니다. 예를 들어 후속 이메일을 보내거나 콘텐츠 상호작용에 따라 연락처에 태그를 추가할 수 있어요.

## 트리거 이름

퍼널/웹사이트 페이지 조회(Funnel/Website PageView)

## 트리거 설명

이 트리거는 연락처가 지정된 퍼널 또는 웹사이트 페이지를 방문할 때 활성화됩니다. 리드나 고객이 어떤 페이지를 방문하는지에 따라 작업을 자동화하여 개인화된 후속 조치나 특정 액션을 실행할 수 있어요.

## 트리거 세부 사항

| 필드명 | 설명 | 필수 |
|--------|------|------|
| 퍼널/웹사이트(Funnel/Website) | 모니터링할 퍼널 또는 웹사이트를 지정 | 예 |
| 페이지(Page) | 퍼널/웹사이트 내에서 모니터링할 특정 페이지를 정의 | 아니오 |
| UTM 캠페인(UTM Campaign) | 특정 UTM 캠페인과 연결된 방문을 추적 | 아니오 |
| UTM 소스(UTM Source) | 특정 UTM 소스와 연결된 방문을 추적 | 아니오 |
| UTM 매체(UTM Medium) | 특정 UTM 매체와 연결된 방문을 추적 | 아니오 |
| UTM 콘텐츠(UTM Content) | 특정 UTM 콘텐츠와 연결된 방문을 추적 | 아니오 |
| 페이지 경로(Page Path) | 보다 세밀한 추적을 위해 특정 페이지 경로를 지정 | 아니오 |

## 조건 연산자

| 연산자 | 설명 |
|--------|------|
| 같음(Is) | 값이 지정된 입력과 정확히 일치해야 함 |
| 다음 중 하나(Is any of) | 값이 나열된 옵션 중 하나일 수 있음 |
| 다음이 아닌(Is none of) | 값이 나열된 옵션 중 어느 것도 아니어야 함 |
| 비어있음(Is empty) | 필드가 비어있어야 함 |
| 비어있지 않음(Is not empty) | 필드가 비어있지 않아야 함 |

## 설정 방법

1. **트리거 선택**: 워크플로우 빌더의 사용 가능한 트리거 목록에서 *퍼널/웹사이트 페이지 조회(Funnel/Website PageView)* 트리거를 선택하세요.

2. **이름 지정**: "퍼널 페이지 방문 - 결제"와 같이 의미 있는 트리거 이름을 설정하세요.

3. **필터 설정**:
   - **퍼널/웹사이트**: 모니터링하려는 특정 퍼널 또는 웹사이트를 선택하세요.
   - **페이지**: 추적하려는 정확한 페이지를 선택적으로 지정하세요.
   - **UTM 매개변수**: UTM 캠페인, 소스, 매체 등의 필터를 설정하여 페이지 조회 조건을 세분화하세요.

4. **추가 필터**: **필터 추가(Add filters)**를 클릭하여 필요에 따라 특정 페이지, 경로 또는 UTM 캠페인을 모니터링하는 추가 필터를 추가할 수 있어요.

## 예시

**시나리오**: 방문자가 퍼널의 "결제" 페이지에 도달했을 때를 추적하고, 장바구니를 버리면 후속 이메일을 보내려고 합니다.

- **트리거**: 퍼널/웹사이트 페이지 조회(Funnel/Website PageView)
- **퍼널/웹사이트**: 퍼널 이름 (예: 부기 서비스 퍼널)
- **페이지**: "결제"와 같음
- **액션**: 구매가 완료되지 않으면 일정 시간 후 도움을 제안하는 후속 이메일 발송

이 설정을 통해 구매의 마지막 단계에서 이탈한 고객을 다시 유도할 수 있어요.
