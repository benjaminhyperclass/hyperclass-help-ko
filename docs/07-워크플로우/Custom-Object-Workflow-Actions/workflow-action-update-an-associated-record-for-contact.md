### 개요

**연관 레코드 업데이트(Update an Associated Record)** 액션을 사용하면 연락처와 연결된 커스텀 객체의 특정 필드를 수정할 수 있습니다. 이를 통해 수동 개입 없이도 핵심 정보가 정확하고 최신 상태로 유지됩니다.

---

### **액션 이름**

Update Associated Record for Contact

---

### **액션 설명**

![연관 레코드 업데이트 액션 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041053104/original/i1jCeUn69LXyTgcgnRKn9rEdCkggGSm_tw.png?1738755088)

이 액션을 통해 연락처와 연결된 기존 커스텀 객체 레코드를 업데이트할 수 있습니다. 선택한 연관 라벨에 따라 하나 또는 여러 레코드에 업데이트가 적용됩니다.

---

### **액션 세부사항**

- 이 액션은 하위 계정에서 커스텀 객체를 생성하고, 해당 객체 중 최소 하나와 연락처 간에 연관 라벨을 만든 후에만 사용할 수 있습니다.
- 선택된 레코드 내의 하나 또는 여러 필드를 업데이트합니다.
- **연관 라벨을 사용하여 업데이트할 레코드를 결정** - "모든 연관 레코드"를 선택하여 연결된 모든 레코드를 업데이트할 수도 있습니다.
- 커스텀 값과 커스텀 필드 - 머지 필드를 사용하여 객체 레코드에 정보를 추가할 수 있습니다.

---

### **예시: 여행 잔여 슬롯 추적**

한 가전제품 회사가 보증이 포함된 제품을 판매하며, 각 제품에는 다음을 추적하는 **보증(Warranty)** 커스텀 객체가 있습니다:

- **보증 시작 날짜**
- **보증 만료 날짜**  
- **서비스 이력**

제품을 구매하면 **보증 시작 날짜**가 자동으로 설정되어야 하고, 보증 청구가 들어오면 **서비스 이력** 필드가 업데이트되어야 합니다.

#### **예시 설정** (서비스 요청 후 보증 정보 업데이트)

- **업데이트할 객체** → 보증(Warranty)
- **연관 라벨** → 구매 제품(Purchased Product)
- **업데이트할 필드:**
  - **서비스 이력** → 서비스 세부사항 추가
  - **최종 서비스 날짜** → 오늘 날짜로 설정

#### **작동 방식**

- 제품을 구매하면 **연관 레코드 생성(Create an Associated Record)** 액션으로 **보증 시작 날짜**가 설정됩니다.
- 고객이 서비스 요청을 제출하면 워크플로우가 **연관 레코드 업데이트** 액션을 트리거합니다:
  - **서비스 이력** 필드에 수행된 서비스 세부사항이 업데이트됩니다.
  - **최종 서비스 날짜**가 오늘 날짜로 설정됩니다.

이를 통해 회사는 각 제품의 서비스 기록을 최신 상태로 유지하여 원활한 보증 추적과 청구 처리를 보장할 수 있습니다.
