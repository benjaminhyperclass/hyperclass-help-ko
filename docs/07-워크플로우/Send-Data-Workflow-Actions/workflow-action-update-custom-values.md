# 워크플로우 액션 - 커스텀 값 업데이트

Hyperclass의 **커스텀 값 업데이트(Update Custom Value)** 워크플로우 액션을 사용하면 워크플로우 내에서 기존 [커스텀 값](../../23-레거시-자동화/Logic-&amp;-Fulfillment/how-to-use-custom-values.md)을 동적으로 업데이트하거나 교체할 수 있습니다. 이 액션은 프로세스 자동화, 캠페인 개인화, CRM 데이터 정확성 유지를 위한 강력한 도구입니다. 고객 등급 업데이트, 외부 API 데이터 동기화, 자동화 트리거 관리 등 다양한 상황에서 워크플로우의 유연성과 효율성을 크게 향상시킵니다.

## **커스텀 값 업데이트 액션이란?**

**커스텀 값 업데이트** 액션을 사용하면 워크플로우에서 커스텀 값을 동적으로 수정할 수 있습니다. 현재 상황이나 특정 조건에 따라 커스텀 값의 값을 설정, 업데이트 또는 교체할 수 있습니다.

- 실시간 트리거에 따라 CRM 데이터를 동적으로 업데이트
- 변수를 사용하여 동적 값 설정
- CRM 전반에서 정확하고 일관된 데이터 유지

---

## **주요 활용 사례**

- **캠페인 동적 개인화**: 구매 마일스톤 달성 후 고객 등급을 자동 업데이트

- **데이터 동기화**: 외부 API 데이터로 CRM 필드를 동적으로 업데이트

- **실시간 자동화**: 고객 행동에 대응하여 할인 코드나 적립금을 동적으로 조정

- **트리거 기반 업데이트**: 사용자 상호작용에 따라 상태 필드를 자동 변경

- **예시 시나리오**: "사용자가 50만원 이상 구매 시, 커스텀 값 업데이트 액션을 사용해 고객 등급을 '브론즈'에서 '골드'로 업데이트"

다음 예시에서는 연락처에 "VIP" 태그가 추가될 때, 할인율 커스텀 값이 10%에서 15%로 업데이트되어 향후 상호작용 시 더 높은 할인을 받을 수 있게 됩니다.

- **트리거**: 연락처 태그 추가, 조건은 "VIP" 태그가 연락처에 추가되는 것
- **액션**: 커스텀 값 업데이트
- **커스텀 값**: 할인율 (이전에 생성됨)
- **현재 값**: 10%
- **새 값**: 15%

---

## **커스텀 값 업데이트 액션 설정 방법**

워크플로우에서 **커스텀 값 업데이트** 액션을 설정하는 단계를 안내해드립니다.

**1단계**: **자동화(Automation) > 워크플로우(Workflows)**로 이동합니다. 새 워크플로우를 추가하거나 기존 워크플로우를 편집합니다.

![워크플로우 메뉴 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039509092/original/xIr8vhfB8RNFCvCUZQo5FoAKjjDTrfzNvw.png?1736383274)

**2단계**: **"+" 버튼**을 클릭하여 액션을 추가합니다. "update custom value"의 일부나 전체 이름으로 검색하거나 찾아보세요.

![액션 추가 및 검색](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039509111/original/S7pIzLiepub8l1e-HLgEbi-j90CkZEXSdw.png?1736383408)

**3단계**: 변경할 커스텀 값을 선택하고 새 값을 설정하여 필드를 구성합니다. 예를 들어, 커스텀 값이 특정 캘린더 링크라면, 이를 다른 링크로 변경할 수 있습니다.

![필드 구성 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039509144/original/ZcXp5SXMnQPSDfCv7p2tqCrmk-w-RTqIRw.png?1736383538)

**4단계**: 액션과 워크플로우를 저장하고 테스트합니다. 워크플로우에서 새로운 커스텀 값을 참조하려면, 해당 액션이 실행된 직후부터 사용할 수 있습니다. 동일한 커스텀 값을 불러오면 새로운 데이터가 포함되어 있을 것입니다.

---

### **필드 설명**

| **필드** | **설명** | **예시** |
|----------|----------|----------|
| **액션 이름** | 워크플로우에 표시되는 액션의 설명적 이름 | 고객 등급 업데이트 |
| **커스텀 값** | 업데이트할 특정 커스텀 값을 선택 | 고객 등급 |
| **현재 값** | 현재 값을 표시 (사용 가능한 경우) | 브론즈 |
| **새 값** | 고정 값 또는 동적 변수 설정 | 골드 |

## **고급 활용 사례**

- **조건부 로직(If/Else 액션) 사용**: 특정 조건이 충족된 경우에만 필드를 조정합니다. 예: 현재 값이 "브론즈"인 경우에만 고객 등급을 업데이트

- **API와의 연동**: 외부 API의 수신 데이터를 사용하여 커스텀 값을 자동 업데이트합니다. 예: "마지막 API 동기화 날짜" 커스텀 필드를 동적으로 업데이트

- **수학 연산 통합**: 수학 연산을 사용하여 숫자 커스텀 값을 동적으로 계산하고 업데이트합니다. 예: 사용자의 적립금 잔액에 50포인트를 동적으로 추가

- **동적 할인 코드**: 실시간 워크플로우에서 할인 코드를 동적으로 변경합니다. 예: *"사용자가 한 달 내 3번 구매를 완료하면, 할인 등급을 자동으로 '다음 구매 10% 할인'으로 업데이트"*

## 추천 트리거

- **연락처 태그 추가**: 연락처에 새 태그가 추가될 때 (예: "VIP", "고우선순위"), 해당 연락처의 중요도를 반영하는 관련 커스텀 값을 업데이트할 수 있습니다.
  예시: "VIP" 태그가 추가되면, **할인율** 커스텀 값을 업데이트하여 더 높은 할인율을 제공

- **연락처 업데이트**: 특정 연락처 필드가 업데이트될 때 액션을 트리거합니다. 예를 들어, 고객의 선호도나 연락처 정보가 변경되면 해당하는 커스텀 값을 업데이트할 수 있습니다.
  예시: **이메일 구독** 필드가 "구독함"으로 업데이트되면, **이메일 발송 빈도** 커스텀 값을 "주간"으로 업데이트

- **폼 제출**: 폼 제출 후, 폼에 제공된 데이터를 기반으로 커스텀 값을 업데이트할 수 있습니다.
  **예시**: 폼 제출이 워크플로우를 트리거하여 **설문 점수** 또는 **선호도** 커스텀 값을 업데이트

- **예약 완료**: 예약이 성공적으로 완료된 후, 예약 유형이나 기타 관련 기준에 따라 커스텀 값을 업데이트할 수 있습니다.
  **예시**: 연락처가 고액 상담을 예약하면, **예약 중요도** 커스텀 값을 "높음"으로 업데이트

- **기회 상태 변경**: 파이프라인에서 기회의 상태가 변경될 때 (예: "검증됨"에서 "성사"로), 고객 기대치 관리에 도움이 되는 커스텀 값을 업데이트할 수 있습니다.
  **예시**: 기회 상태가 "성사"로 변경되면, **고객 등급** 커스텀 값을 "플래티넘"으로 업데이트

- **설문 제출**: 연락처가 설문을 제출할 때, 응답을 사용하여 향후 상호작용에 활용할 커스텀 값을 동적으로 업데이트할 수 있습니다.
  **예시**: 고객 설문 응답에서 높은 만족도 점수를 표시하면, **고객 만족도** 커스텀 값을 해당 점수로 업데이트

- **주문 처리 완료**: 주문이 처리 완료된 후, 연락처의 커스텀 값을 업데이트하여 주문 기록이나 고객 상태를 추적할 수 있습니다.
  예시: 주문이 처리 완료되면, **마지막 주문 날짜** 커스텀 값을 현재 날짜로 업데이트

- **링크 클릭**: 연락처가 이메일이나 SMS의 링크를 클릭할 때, 워크플로우가 행동에 따라 커스텀 값을 동적으로 업데이트할 수 있습니다.
  **예시**: 새 제품 링크를 클릭하면, **제품 관심도** 커스텀 값을 클릭한 제품에 대한 관심을 반영하도록 업데이트

## 자주 묻는 질문

**Q: 커스텀 값 업데이트 액션에서 변수를 사용할 수 있나요?**
**A:** 네, {{contact.custom_field}}와 같은 변수를 동적 업데이트에 사용할 수 있습니다.

**Q: 이 액션이 다른 워크플로우를 트리거할 수 있나요?**
**A:** 네, 업데이트가 추가 워크플로우의 트리거 역할을 할 수 있습니다.

**Q: 워크플로우에서 커스텀 값 업데이트 액션의 개수에 제한이 있나요?**
**A:** 아니요, 필요에 따라 여러 액션을 사용할 수 있습니다.

**Q: 현재 값이 조건과 일치하지 않으면 어떻게 되나요?**
**A:** 현재 값이 지정된 조건과 일치하지 않으면 (조건이 설정된 경우) 업데이트가 진행되지 않습니다.

**Q: If/Else 액션에서 커스텀 값 업데이트를 사용할 수 있나요?**
**A:** 네, 더 고급 워크플로우를 위해 조건부 로직과 결합할 수 있습니다.

**Q: 여러 커스텀 값을 업데이트하거나 일괄 업데이트할 수 있나요?**
**A:** 하나의 커스텀 값 업데이트 액션은 한 번에 하나의 커스텀 값을 업데이트할 수 있습니다. 더 많은 커스텀 값을 업데이트하거나 같은 커스텀 값을 다시 업데이트하려면 더 많은 액션을 사용하세요.

**Q: 커스텀 값 업데이트를 사용해 다른 하위 계정의 커스텀 값을 업데이트할 수 있나요?**
**A:** 업데이트되는 커스텀 값은 워크플로우가 실행되는 하위 계정과 동일한 하위 계정에 있어야 합니다.

**Q: 업데이트 후 되돌릴 수 있나요?**
**A:** 아니요, 커스텀 값 업데이트 액션에는 "되돌리기" 기능이 없습니다.
