## 개요

회사 변경됨 트리거는 회사 레코드의 선택된 필드가 업데이트될 때마다 워크플로우를 시작합니다. 이 트리거를 사용하면 회사 정보 변경에 즉시 반응할 수 있어, 자동화가 항상 최신 데이터와 동기화를 유지할 수 있습니다.

## 트리거 이름

회사 변경됨 (Company Changed)

## 트리거 설명

이 트리거는 다음 상황에서 실행됩니다:

- 회사 레코드가 업데이트됩니다.
- 업데이트가 트리거에서 설정한 하나 이상의 필드 조건과 일치합니다.

모니터링할 특정 필드를 선택할 수 있습니다(예: 업종, 상태, 도메인). 해당 필드가 변경되면 워크플로우가 자동으로 시작됩니다.

## 설정 방법

![회사 변경됨 트리거 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055005957/original/2jOQH3Y4U0-yFKxL8sniERm7idQUxcGW4A.png?1759320818)

- **트리거 추가**
  워크플로우 트리거로 회사 변경됨(Company Changed)을 선택합니다.

- **모니터링할 필드 선택**
  변경 사항을 추적할 회사 필드(기본 또는 커스텀)를 하나 이상 선택합니다.
  예시: 업종, 도메인, 상태

- **연산자 적용**
  각 필드에 대해 연산자를 선택합니다:
  - **비어있지 않음** – 필드에 값이 있을 때 실행됩니다.
  - **구문 포함** – 필드 값에 지정된 키워드/구문이 포함될 때 실행됩니다.

- **다중 필터 추가(선택사항)**
  AND 논리를 사용하여 필터를 결합합니다(트리거가 실행되려면 모든 조건이 충족되어야 함).

- **트리거 저장**
  저장하면 설정된 조건을 만족하는 회사가 자동으로 워크플로우에 등록됩니다.

## 예시

#### 예시 1: 업종 변경 추적

**목표:** 회사의 업종에 "의료"가 포함될 때 온보딩을 트리거합니다.

**설정**
- 필터: 업종(Industry) → 구문 포함(Contains Phrase) → 의료
- 액션: 의료 업종 온보딩 워크플로우에 등록

**작동 방식**
- 회사 업종 필드가 "의료 서비스 업체"로 업데이트됩니다.
- 워크플로우가 트리거됩니다.
- 해당 회사가 의료 업종 전용 워크플로우에 등록됩니다.

#### 예시 2: 도메인 확보 보장

**목표:** 회사 레코드가 생성되거나 업데이트될 때 도메인이 비어있지 않으면 팀원에게 알립니다.

**설정**
- 필터: 도메인(Domain) → 비어있지 않음(Is not empty)
- 액션: 내부 알림 발송

**작동 방식**
- 회사 도메인 필드에 값이 입력됩니다.
- 워크플로우가 트리거됩니다.
- 영업팀에 내부 알림이 발송됩니다.
