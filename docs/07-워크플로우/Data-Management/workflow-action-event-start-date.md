## 개요

워크플로우의 **이벤트 시작 날짜(Event Start Date)** 액션은 특정 날짜와 시간을 설정하여 대기(Wait) 이벤트나 다른 미래 액션을 트리거할 수 있게 해줍니다. 이를 통해 후속 이벤트가 특정 시간이나 날짜를 기준으로 실행되어, 후속 작업의 타이밍을 더욱 정교하게 제어할 수 있습니다.

## 액션 이름

**Event Start Date(이벤트 시작 날짜)**

## 액션 설명

**이벤트 시작 날짜** 액션은 특정 날짜, 시간, 또는 커스텀 필드 값을 기준점으로 설정하여 후속 워크플로우 액션, 특히 **대기(Wait)** 이벤트의 참조점으로 사용합니다. 이 액션은 설정된 이벤트 시간을 기준으로 액션이 일시정지되거나 트리거되도록 하여, 자동 후속 연락, 리마인더, 또는 기타 예약된 작업을 자동화하는 데 유용합니다.

### **설정 방법**:

- **액션 이름(Action Name)**: 이 액션을 설명하는 이름을 입력하세요. 예: "웨비나 이벤트 시작 시간"
- **타입(Type)**: 다음 옵션 중에서 선택하세요:
  - **특정 요일(Specific Day)**: 반복되는 특정 요일을 설정하고 싶을 때 선택합니다 (예: 매월 특정 일 또는 매주 특정 요일)
  - **특정 날짜/시간(Specific Date/Time)**: 정확한 날짜와 시간을 설정할 때 사용합니다 (예: 2024년 9월 14일 오전 10:30)
  - **커스텀 필드(Custom Field)**: 커스텀 필드에서 날짜를 가져올 때 사용합니다 (예: "이벤트 등록일" 필드)
- **커스텀 필드 선택(Select Custom Field)** (**커스텀 필드** 타입에서만 표시): 날짜가 저장된 관련 커스텀 필드를 선택합니다
- **날짜 선택(Select Date)** (**특정 날짜/시간** 타입에서만 표시): 캘린더 선택기에서 날짜와 시간을 선택합니다
- **현재 월/주의 요일(Current Day of Month/Week)** (**특정 요일** 타입에서만 표시): 이벤트가 현재 월의 특정 일 또는 현재 주의 특정 요일을 기준으로 시작될지 선택합니다

### **필드 상세 정보:**

| 필드 이름 | 설명 | 필수 |
|----------|------|------|

| 타입(Type) | 특정 요일, 특정 날짜/시간, 커스텀 필드 중 선택 | 예 |
| 커스텀 필드 선택(Select Custom Field) | 커스텀 필드 타입을 선택했을 때 나타나는 필드. 커스텀 날짜 필드를 선택합니다 | 아니오 |
| 날짜 선택(Select Date) | 특정 날짜/시간 타입을 선택했을 때 나타나는 필드. 캘린더 선택기에서 날짜를 선택합니다 | 아니오 |
| 현재 월/주의 요일(Current Day of Month/Week) | 특정 요일 타입을 선택했을 때 나타나며, 일 또는 주 기반 트리거를 선택할 수 있습니다 | 예 |

## 예시

**타입: 특정 날짜/시간**
- 사용 사례: 특정 날짜에 예정된 이벤트 1주일 후에 후속 이메일을 트리거하고 싶은 경우
- 설정 예시: 2024년 9월 14일을 이벤트 시작일로 선택합니다. 워크플로우는 이를 모든 후속 시간 기반 액션의 기준점으로 사용합니다

**타입: 커스텀 필드**
- 사용 사례: "이벤트 등록일"과 같은 커스텀 날짜 필드를 기준으로 후속 액션을 설정하고 싶은 경우
- 설정 예시: 워크플로우가 "이벤트 등록일" 커스텀 필드에서 날짜를 가져와서 해당 날짜를 기준으로 리마인더 이메일이나 할 일 배정 등의 후속 액션을 트리거합니다

**타입: 특정 요일**
- 사용 사례: 반복되는 이벤트나 작업의 경우, 현재 주의 특정 요일이나 월의 특정 일을 기준으로 액션을 자동화하고 싶은 경우
- 설정 예시: "현재 월의 특정 일"을 선택하여 매월 1일마다 반복되는 리마인더를 설정합니다

## 추가 참고사항

- **날짜 형식**: 날짜와 시간이 올바른 형식(예: MM-DD-YYYY HH:MM)으로 되어 있는지 확인하세요. 이는 워크플로우 타이밍 오류를 방지합니다
- **사용 사례**: 리마인더, 후속 연락, 또는 특정 날짜에 따라 달라지는 모든 액션을 자동화하는 데 완벽합니다
- **트리거**: 이 액션을 **대기(Wait)** 이벤트와 결합하여 특정 날짜나 시간까지 연락처를 대기시킨 후 다음 액션으로 진행할 수 있습니다
