# 워크플로우 트리거 - 파이프라인 단계 변경

영업 기회를 효율적으로 추적하고 관리하는 것은 체계적인 영업 프로세스를 유지하는 핵심입니다. 파이프라인 단계 변경 워크플로우 트리거(Pipeline Stage Changed Workflow Trigger)는 기회가 한 단계에서 다른 단계로 이동할 때 자동으로 작업을 실행하여 적시 후속 조치, 알림, 업데이트를 보장합니다. 이 가이드에서는 이 트리거의 기능, 이점, 그리고 최적화된 파이프라인 관리를 위한 설정 방법을 안내해드립니다.

# 파이프라인 단계 변경 워크플로우 트리거란?

파이프라인 단계 변경 워크플로우 트리거는 영업 파이프라인 내에서 기회가 한 단계에서 다른 단계로 이동할 때 활성화됩니다. 단계 전환을 기반으로 워크플로우를 자동화하여 리드가 빠지는 일 없이 관리할 수 있도록 도와줍니다. 이 트리거를 사용하면 기회가 진행, 퇴행하거나 새 파이프라인 단계로 재배정될 때 실행되어야 하는 작업을 정의할 수 있습니다.

## 파이프라인 단계 변경 워크플로우 트리거의 주요 이점

- **후속 조치 자동화**: 각 단계 변경 시 적시 알림을 보내 영업 담당자가 잠재 고객과 지속적으로 소통할 수 있도록 합니다.

- **수동 작업 감소**: 파이프라인 이동을 수동으로 추적할 필요를 없애 전략적 영업 활동에 더 많은 시간을 할애할 수 있습니다.

- **파이프라인 가시성 향상**: 기회 진행 상황에 대한 실시간 인사이트를 제공하여 더 나은 예측과 의사결정을 가능하게 합니다.

- **사용자 맞춤 필터링**: 파이프라인, 단계, 배정된 사용자, 리드 가치 등 특정 조건에 따라 워크플로우를 설정할 수 있습니다.

- **응답 시간 개선**: 아웃리치와 내부 커뮤니케이션을 자동화하여 중요한 영업 단계에서 신속한 조치를 보장합니다.

## 파이프라인 단계 변경 워크플로우 트리거 설정하기

### 워크플로우 빌더 접근

CRM 대시보드에서 Automation(자동화) > Workflows(워크플로우)로 이동합니다. Create New Workflow(새 워크플로우 만들기)를 클릭하거나 기존 워크플로우를 편집하세요.

### 워크플로우 트리거 선택

`+ Add New Trigger(새 트리거 추가)`를 클릭하고 목록에서 Pipeline Stage Changed(파이프라인 단계 변경)를 검색합니다. Pipeline Stage Changed를 트리거로 선택하세요.

워크플로우 트리거에 용도를 반영하는 명확하고 설명적인 이름을 지어주세요. 이를 통해 여러 워크플로우를 관리할 때 트리거의 기능을 빠르게 식별할 수 있습니다.

### 필터 설정

필터는 특정 조건이 충족되어야만 워크플로우가 트리거되도록 설정하여 언제, 어떻게 워크플로우가 실행될지 제어할 수 있게 도와줍니다. 모든 기회 변경에 대해 워크플로우를 트리거하는 대신, 필터를 사용하면 비즈니스에 가장 중요한 것들만 대상으로 할 수 있습니다. 특정 파이프라인의 기회, 특정 팀원에게 배정된 기회, 또는 특정 이유로 실패 처리된 거래에만 작업을 자동화하고 싶든 상관없이, 필터는 자동화를 세밀하고 맞춤화할 수 있는 유연성을 제공합니다. 이를 통해 워크플로우가 더 정확하고 관련성 있으며 영업 프로세스에 맞춰지도록 보장할 수 있습니다.

![필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155065763970/original/xa_HvOOR8vPFm_8fQBtF4lMuOsVb_tAy5g.png?1772101325)

#### 배정 대상 필터

Assigned To(배정 대상) 필터는 기회에 배정된 사용자를 기준으로 워크플로우가 트리거되도록 보장합니다. 이 필터는 후속 조치를 지시하거나, 할 일을 배정하거나, 기회가 파이프라인을 통해 이동할 때 특정 팀원에게 알림을 보내는 데 특히 유용합니다.

![배정 대상 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042363650/original/j4BOkGGKQ7xDL_xBkrZXtqgtrryqMg8biQ.png?1740664257)

#### 태그 필터 (이전 "Has Tag")

태그 필터를 사용하여 특정 태그 조건과 일치하는 기회에만 워크플로우를 트리거하세요. 태그 필터는 기회에 특정 태그가 포함된 경우에만 워크플로우를 트리거할 수 있게 해주므로, 기회를 세분화하고 리드의 우선순위를 정하며 타겟 작업을 자동화하는 데 특히 유용합니다. 태그는 리드 유형, 우선순위, 소스 또는 기타 관련 분류와 같은 속성을 기반으로 기회를 분류하는 라벨 역할을 합니다. 태그 필터를 설정할 때, 워크플로우가 활성화되기 위해 어떤 태그가 있어야 하는지 지정할 수 있어 적절한 기회에만 자동화가 적용되도록 보장합니다.

예시: 기회가 "높은 우선순위"로 태그가 지정되면 다음 단계로 이동할 때 워크플로우가 활성화됩니다.

![태그 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155065763802/original/vDBNog5lhciogk9I3LPoOK0x7CzN_XOshg.png?1772101275)

#### 파이프라인 내 필터

In Pipeline(파이프라인 내) 필터는 워크플로우 트리거를 특정 파이프라인으로 제한하여 선택된 파이프라인 내의 기회에 대해서만 자동화가 활성화되도록 합니다. 이 필터는 서로 다른 영업 프로세스, 고객 유형 또는 서비스 제공을 위해 여러 파이프라인을 관리하는 비즈니스에 특히 유용합니다. 파이프라인을 지정함으로써 사용자는 관련 기회에만 워크플로우가 적용되도록 하고 관련 없는 파이프라인에서 불필요한 자동화를 방지할 수 있습니다.

예시: "기업 영업" 파이프라인 내에서 기회가 이동할 때만 워크플로우가 활성화됩니다.

![파이프라인 내 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042363876/original/oAKEstTXufxn_LLmvtnYhsr8_NdH6YWYfA.png?1740664390)

#### 리드 가치 필터

Lead Value(리드 가치) 필터는 기회에 배정된 금전적 가치를 기준으로 워크플로우를 트리거합니다. 이를 통해 비즈니스는 리드의 재정적 중요도에 따라 작업을 자동화할 수 있어, 고가치 거래는 적절한 후속 조치를 받고 저가치 기회는 다르게 처리되도록 보장합니다.

예시: 기회의 가치가 $10,000를 초과하면 영업 관리자에게 알림을 보냅니다.

![리드 가치 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042363981/original/ih30kq-4ZMDpZHus1Oh3AXUEyk7BrYyQtw.png?1740664432)

#### 실패 사유 필터

Lost Reason(실패 사유) 필터는 기회가 실패(Lost)로 표시되고 특정 사유가 배정될 때 워크플로우를 트리거하여, 비즈니스가 거래가 왜 실패했는지 추적하고 후속 조치를 자동화하며 잠재 고객을 회복하거나 향후 유사한 손실을 방지하는 전략을 구현할 수 있게 합니다. 기회가 파이프라인에서 Lost 단계로 이동하면 사용자는 사유(예: "가격이 너무 높음", "경쟁사 선택", "응답 없음")를 배정할 수 있어, 특정 실패 사유가 선택될 때만 워크플로우가 활성화되도록 보장합니다.

예시: 실패 사유가 "예산 제약"인 경우, 유연한 가격 계획을 제안하는 자동 후속 이메일을 보냅니다.

![실패 사유 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042364228/original/kki_iWGEv2YLSlewo_PlI4yixV68EY5vVQ.png?1740664590)

#### 파이프라인 단계 필터

Pipeline Stage(파이프라인 단계) 필터는 기회가 파이프라인 내의 특정 단계로 이동할 때 워크플로우를 트리거합니다. 이를 통해 비즈니스는 단계 전환을 기반으로 작업을 자동화하여 기회가 진행됨에 따라 적시 후속 조치, 할 일 배정 또는 알림을 보장할 수 있습니다. 사용자는 어떤 단계 전환이 워크플로우를 활성화해야 하는지 정의할 수 있어 리드를 관리하고 영업 파이프라인 이동을 효율적으로 추적하기 쉽게 만듭니다.

예시: 기회가 "협상"에서 "성사"로 이동하면 재무팀에 알림을 보냅니다.

![파이프라인 단계 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042364328/original/JHzyQQFJyOxBYMLPCT2PKB8tlFggH5aVCQ.png?1740664628)

#### 상태 필터

Status(상태) 필터는 기회의 현재 상태를 기준으로 워크플로우를 트리거합니다. 이를 통해 기회가 영업 프로세스의 특정 단계에 도달할 때 자동화가 실행되어 팀이 파이프라인을 효과적으로 추적하고 관리할 수 있도록 도와줍니다. 사용자는 다음 상태 옵션 중에서 선택할 수 있습니다:

- **Open(진행 중)**: 기회가 여전히 활성 상태이고 진행 중입니다.
- **Won(성사)**: 거래가 성공적으로 완료되었습니다.
- **Lost(실패)**: 기회가 판매로 전환되지 않았습니다.
- **Abandoned(포기)**: 기회를 더 이상 추진하지 않습니다.

예시: 기회가 "성사"로 표시되면 환영 이메일 시퀀스를 트리거합니다.

![상태 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042364875/original/g0iRsRHN_JARCYsJozticR4tVJUgZ-7hIQ.png?1740664959)

### 워크플로우 액션 정의

트리거가 설정되면 기회가 단계를 변경할 때 수행되어야 하는 액션을 정의하세요.

- **알림 보내기**: 배정된 영업 담당자에게 단계 변경에 대해 알림을 보냅니다.
- **기회 세부 정보 업데이트**: 상태, 우선순위, 태그 등의 필드를 자동으로 수정합니다.
- **후속 조치 할 일 생성**: 잠재 고객에게 연락하도록 영업팀에 할 일을 배정합니다.
- **다른 워크플로우 트리거**: 기회 진행에 따라 추가 워크플로우를 활성화합니다.
- **연락처에 이메일 발송**: 파이프라인 이동을 기반으로 고객 커뮤니케이션을 자동화합니다.

### 워크플로우 저장 및 발행

- 필터와 액션 설정을 완료한 후 Save Trigger(트리거 저장)를 클릭하여 변경사항을 적용하세요.
- 모든 필요한 워크플로우 단계가 제대로 설정되었는지 확인한 후 Publish(발행)를 클릭하여 자동화를 활성화하세요.
- 기대한 대로 작동하는지 확인하기 위해 기회를 단계별로 수동으로 이동시켜 워크플로우를 테스트하세요.

## 자주 묻는 질문

**Q1: 기회를 수동으로 이동시키면 어떻게 되나요?**
기회를 수동으로 다른 단계로 이동시켜도 설정된 필터 조건을 만족한다면 워크플로우가 트리거됩니다.

**Q2: 같은 워크플로우에서 여러 필터를 사용할 수 있나요?**
네, 여러 필터를 조합하여 매우 구체적인 자동화 규칙을 만들 수 있습니다. 예를 들어 특정 파이프라인과 단계의 기회가 특정 사용자에게 배정되었을 때만 액션을 트리거하도록 설정할 수 있습니다.

**Q3: 이 트리거는 다른 기회 기반 트리거와 어떻게 상호 작용하나요?**
