# 워크플로우 트리거 - 기회 생성됨

**기회 생성됨 트리거(Opportunity Created Trigger)**는 시스템에서 새 기회가 생성될 때 워크플로우를 자동화할 수 있는 기능입니다. 이 트리거는 파이프라인에 새로운 영업 기회가 만들어질 때 알림, 후속 작업, 업무 배정 등의 액션을 자동화하는 데 유용합니다.

## 트리거명: 기회 생성됨 트리거

### 트리거 설명

기회 생성됨 트리거는 시스템에서 새 기회가 생성될 때 활성화됩니다. 이를 통해 파이프라인 단계, 리드 가치, 배정된 사용자 등 기회 특성에 따라 업무 배정, 팀원 알림, 상태 업데이트 등의 프로세스를 자동화할 수 있습니다.

### **트리거 세부사항**

| 필드명 | 설명 | 필수 여부 |
|--------|------|-----------|
| 배정 대상(Assigned To) | 기회가 배정된 사용자 | 아니요 |
| 태그 보유(Has Tag) | 특정 태그를 기준으로 기회 필터링 | 아니요 |
| 파이프라인(In Pipeline) | 기회를 추적할 파이프라인 선택 | 아니요 |
| 리드 가치(Lead Value) | 리드와 연결된 금액 | 아니요 |
| 실패 사유(Lost Reason) | 기회를 실패로 표시할 때 제공한 사유 | 아니요 |
| 상태(Status) | 기회의 현재 상태 (신규, 진행 중 등) | 아니요 |

| 연산자 | 설명 |

| 보다 작은(Less than) | 입력값보다 작은 값 |

| 비어있지 않음(Is Not Empty) | 필드가 비어있지 않은 경우 |

## 설정 방법

- **트리거 선택**: 워크플로우 빌더에서 *기회 생성됨(Opportunity Created)*을 트리거로 선택하세요.

- **트리거 이름 지정**: "고액 리드 기회 생성됨"과 같이 의미있는 트리거명을 지정하세요.

- **필터 설정**:
  - 배정 대상: 어느 사용자에게 배정된 기회인지를 기준으로 필터링합니다.
  - 파이프라인: 특정 파이프라인의 기회만 필터링합니다.
  - 리드 가치: 기회의 금액을 기준으로 조건을 설정합니다.

- **태그**: 태그 필터를 사용하여 태그로 기회를 포함하거나 제외합니다. 태그는 다중 선택 연산자(같음/같지 않음/포함/제외)를 지원하며, 고급 규칙을 위해 여러 태그 필터를 추가할 수 있습니다.

- **추가 필터**: 필요시 **필터 추가(Add filters)** 버튼을 클릭하고 원하는 필드와 조건을 선택하여 추가 필터를 적용하세요.

## **예시**

시나리오: 파이프라인에서 리드 가치가 $10,000를 초과하는 새 기회가 생성될 때 영업 관리자에게 알림을 보내고 싶습니다.

- 트리거: 기회 생성됨(Opportunity Created)
- 필터: 리드 가치가 $10,000보다 큰 경우
- 액션: 영업 관리자에게 알림

이 설정을 통해 중요한 기회들이 표시되고 주요 이해관계자에게 알림을 보내 신속하게 조치할 수 있습니다.
