# 워크플로우 트리거 - 기회 변경 시

기회 관리를 효율적으로 수행하는 것은 체계적이고 효과적인 영업 파이프라인을 유지하는 데 필수적입니다. Hyperclass의 '기회 변경 시(Opportunity Changed)' 워크플로우 트리거는 기회 레코드가 업데이트될 때마다 자동으로 액션을 실행하여 변경 사항에 실시간으로 대응합니다. 상태 변경만 추적하는 '기회 상태 변경 시(Opportunity Status Changed)' 트리거와 달리, 이 트리거는 리드 가치 조정, 단계 이동, 재배정, 커스텀 필드 업데이트 등의 수정 사항을 감지하는 더 광범위한 기능을 제공합니다. 이 가이드에서는 CRM 자동화를 간소화하는 데 도움이 되는 이점, 설정 방법, 모범 사례를 안내해드립니다.

## **기회 변경 시 워크플로우 트리거란?**

Hyperclass의 '기회 변경 시(Opportunity Changed)' 워크플로우 트리거는 기회 레코드에서 변경이 발생할 때마다 자동으로 액션을 실행합니다. 상태 업데이트만 추적하는 '기회 상태 변경 시(Opportunity Status Changed)'와 달리, 이 트리거는 리드 가치, 파이프라인 단계, 배정, 커스텀 필드 업데이트 등 더 광범위한 수정 사항에 대응할 수 있습니다. 이러한 유연성을 통해 비즈니스는 워크플로우를 간소화하고 CRM에서 실시간 정확성을 유지할 수 있습니다.

## **기회 변경 시 워크플로우 트리거의 주요 이점**

- **포괄적 추적:** 상태 업데이트뿐만 아니라 기회의 모든 변경 사항을 기반으로 액션을 자동화합니다.

- **실시간 알림:** 기회가 업데이트될 때 팀원에게 즉시 알림을 보냅니다.

- **향상된 CRM 정확성:** 자동화된 기록 관리와 데이터 일관성을 보장합니다.

- **개선된 영업 파이프라인 관리:** 기회가 수정될 때 후속 조치나 할 일 배정을 트리거합니다.

- **맞춤형 자동화:** 워크플로우 필터를 적용하여 특정 기회 변경에 대한 자동화를 맞춤화합니다.

- **유연한 필터 연산자:** "변경됨(Has Changed)", "다음으로 변경됨(Has Changed To)", "같음(Equals)" 연산자를 사용하여 연락처 변경 시(Contact Changed) 트리거와 유사하게 워크플로우가 언제 트리거될지 정확히 제어합니다.

## 기회 변경 시 트리거 설정

이 트리거를 설정하면 비즈니스가 CRM의 기회 업데이트를 기반으로 워크플로우 액션을 자동화할 수 있습니다. 다음 단계를 따라주세요:

### **1단계: 워크플로우 빌더 접속**

Hyperclass 계정에 로그인하고, **자동화(Automations) > 워크플로우(Workflows)**로 이동한 다음, 새 워크플로우를 만들거나 기존 워크플로우를 엽니다.

![워크플로우 빌더 접속](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041720419/original/m6PRzF3NQEdGf2bwDa0gTGcZOx2E7ZohQw.jpeg?1739792393)

### **2단계: "기회 변경 시" 트리거 추가**

**+ 새 트리거 추가(+ Add New Trigger)**를 클릭한 다음, 사용 가능한 트리거 목록에서 **기회 변경 시(Opportunity Changed)**를 검색합니다. 찾으면 선택하여 설정을 진행합니다.

![기회 변경 시 트리거 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041720461/original/tmfJtToO53129fB6M02t87Jr_3ILyrltQg.png?1739792417)

### 3단계: 트리거 이름 지정

트리거의 목적을 반영하는 명확하고 설명적인 이름을 지정합니다. 잘 명명된 트리거는 시스템 내에서 여러 자동화를 관리할 때 워크플로우를 식별하기 쉽게 만듭니다.

![트리거 이름 지정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041720473/original/T_nj3fnpSNPbt3ZRN2xo2WAk64F-oTpyFA.png?1739792436)

### 4단계: 필터 적용

필터는 기회에 특정 변경이 발생했을 때만 워크플로우가 트리거되도록 세밀하게 조정하는 데 도움이 됩니다.

#### 담당자 필터(Assigned To Filter)

담당자 필터는 기회가 다른 팀원에게 재배정될 때 워크플로우를 트리거합니다. 이는 리드를 공동으로 관리하는 영업 및 지원 팀에 특히 유용합니다. 이 필터를 사용하면 비즈니스가 기회의 소유권이 변경될 때 후속 조치를 자동화하여 팀원 간의 원활한 인수인계를 보장할 수 있습니다.

사용 가능한 연산자:

• **변경됨(Has Changed)**: 담당 사용자가 다른 사용자로 변경될 때 트리거됩니다.

• **다음으로 변경됨(Has Changed To)**: 기회가 선택한 특정 사용자에게 배정될 때만 트리거됩니다.

• **같음(Equals)**: 특정 사용자에게 현재 배정된 기회에서 변경이 발생할 때 트리거됩니다.

예를 들어, 고가치 리드가 수석 영업 담당자에게 재배정되면 워크플로우가 자동으로 새 담당자에게 알림을 보내고, 내부 기록을 업데이트하거나, 후속 시퀀스를 트리거할 수 있습니다. 이는 책임감을 유지하고 전환 과정에서 기회를 놓치지 않도록 보장합니다.

![담당자 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062695015/original/oX8LId93YSrM_6h6ypg_ETg4eWeCctBMlg.png?1768460530)

#### **태그 필터(Tag filter)**

태그 필터를 사용하여 기회 태그를 기반으로 기회 변경 시 트리거가 언제 실행될지 제어합니다.

태그 필터를 사용하면 기회에 할당된 태그를 기반으로 워크플로우를 트리거할 수 있습니다. 이 필터는 기회에 특정 태그가 포함되어 있을 때만 자동화가 활성화되도록 하여 워크플로우 실행에 대한 더 많은 제어를 제공합니다.

예를 들어, "핫 리드(Hot Lead)"나 "후속 조치 필요(Follow-Up Needed)" 같은 태그가 기회에 적용되면 워크플로우를 설정하여 자동화된 후속 이메일을 보내고, 영업 담당자에게 알림을 보내거나, 리드를 다른 파이프라인 단계로 이동시킬 수 있습니다. 이는 특히 리드를 세분화하고, 고가치 기회의 우선순위를 정하며, 리드 분류에 기반한 특정 액션을 트리거하는 데 유용합니다.

![태그 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155065759707/original/PBNHetgQ1i_K8kTPAQ6RXumtFW0ICqat5Q.png?1772099697)

#### **파이프라인 내 필터(In Pipeline Filter)**

파이프라인 내 필터는 기회가 특정 파이프라인 내에서 변경이 발생할 때 워크플로우를 트리거합니다. 이는 선택된 파이프라인에 속한 기회에 대해서만 자동화가 활성화되도록 하여 더 타겟팅된 워크플로우 실행을 가능하게 합니다.

사용 가능한 연산자:

• **변경됨(Has Changed)**: 기회가 다른 파이프라인으로 이동할 때 트리거됩니다.

• **다음으로 변경됨(Has Changed To)**: 기회가 선택한 특정 파이프라인으로 이동할 때만 트리거됩니다.

• **같음(Equals)**: 특정 파이프라인 내의 기회에서 변경이 발생할 때 트리거됩니다.

예를 들어, 리드가 영업 파이프라인 내에서는 이동하지만 고객 지원 파이프라인에서는 이동하지 않는 경우, 워크플로우를 영업 관련 변경에 대해서만 트리거되도록 설정할 수 있습니다. 이는 영업, 온보딩, 고객 유지 등 다양한 프로세스를 처리하는 여러 파이프라인을 가진 비즈니스에 특히 유용합니다. 이 필터를 적용하면 팀이 적절한 컨텍스트 내에서 워크플로우가 실행되도록 보장하여 불필요한 자동화 트리거를 줄이고 워크플로우 효율성을 향상시킬 수 있습니다.

![파이프라인 내 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062695255/original/6UdEe0QrbHEr2aOpUhHDq-BaGS0-BC_2AA.png?1768460744)

#### 리드 가치 필터(Lead Value Filter)

리드 가치 변경 필터는 기회의 금전적 가치가 증가하거나 감소할 때 워크플로우를 트리거합니다. 이를 통해 비즈니스는 잠재적 매출 변경을 기반으로 액션을 자동화하여 중요한 업데이트를 놓치지 않도록 할 수 있습니다.

예를 들어, 리드의 가치가 증가하는 경우(아마도 업셀이나 수정된 견적으로 인해) 워크플로우가 영업 팀에게 알림을 보내고, 파이프라인 단계를 조정하거나, 맞춤형 후속 이메일을 보낼 수 있습니다. 반대로 리드 가치가 감소하는 경우, 시스템이 계정 관리자에게 검토를 위해 알림을 보내거나 손실 방지를 위한 유지 전략을 트리거할 수 있습니다.

![리드 가치 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041721293/original/i6B2z0QUdLHa5dw3e0vNit3Ny8IeavXOQw.png?1739792952)

#### **실패 사유 필터(Lost Reason Filter)**

실패 사유 필터를 사용하면 비즈니스가 기회가 실패로 표시되고 사유가 할당될 때 워크플로우를 자동화할 수 있습니다. 이 필터는 실패한 거래를 전략적으로 추적하고 대응하는 데 도움이 되어, 비즈니스가 영업 접근 방식과 고객 유지 노력을 개선할 수 있도록 합니다.

이 필터의 작동 방식을 세밀하게 조정하기 위해 사용자는 다음 필터 연산자 중에서 선택할 수 있습니다:

**변경됨(Has Changed):** 실패 사유가 이전 값에 관계없이 업데이트될 때마다 워크플로우를 트리거합니다. 이는 실패한 거래에 할당된 사유의 모든 수정 사항을 추적하는 데 유용합니다.

**다음으로 변경됨(Has Changed To):** 실패 사유가 특정 값으로 업데이트될 때만 워크플로우를 트리거합니다. 예를 들어, 실패 사유가 "가격이 너무 높음"으로 변경되면 시스템이 할인 제안과 함께 자동 후속 이메일을 보낼 수 있습니다.

**같음(Equals):** 특정 실패 사유가 할당된 기회에서 변경이 발생할 때 트리거됩니다.

![실패 사유 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062695321/original/WJ4n2v2MIVa3EbDV3y1zwASjldc5TKlbDA.png?1768460799)

#### **상태 필터(Status Filter)**

상태 필터를 사용하면 기회의 현재 상태를 기반으로 워크플로우를 트리거할 수 있습니다. 이는 기회가 파이프라인의 특정 단계에 도달할 때 액션이 실행되도록 하여 팀이 영업 진행 상황을 더 효율적으로 추적하고 관리하는 데 도움이 됩니다.

이 필터를 세밀하게 조정하기 위한 사용 가능한 필터 연산자는 다음과 같습니다:

**같음(Is):** 기회가 선택된 상태와 일치할 때 워크플로우를 트리거합니다. 예를 들어, "성사(Won)"가 선택되면 기회가 성사로 표시될 때만 워크플로우가 활성화됩니다.

**같지 않음(Is Not):** 기회가 선택된 상태와 일치하지 않을 때 워크플로우를 트리거합니다. 이는 "포기됨(Abandoned)" 기회를 무시하는 등 특정 상태가 워크플로우를 트리거하지 않도록 필터링할 때 유용할 수 있습니다.

**변경됨(Has Changed):** 상태가 다른 상태로 변경될 때 트리거됩니다.

**다음으로 변경됨(Has Changed To):** 상태가 선택한 특정 값으로 변경될 때만 트리거됩니다.

연산자를 선택한 후, 사용자는 드롭다운에서 사용 가능한 상태 옵션 중에서 선택할 수 있습니다.

![상태 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062695356/original/QSRtNo9GVMrX3r5t6KkZj-noHgIHJ3Y7uA.png?1768460850)

**체크박스 필드(기회 커스텀 필드 필터)**

체크박스 커스텀 필드 변경을 기반으로 기회를 필터링합니다.

• **추가됨(Added):** 특정 체크박스 옵션이 추가/체크될 때 트리거됩니다.

• **제거됨(Removed):** 특정 체크박스 옵션이 제거/체크 해제될 때 트리거됩니다.

• **같음(Equals):** 체크박스가 특정 값과 같은 기회에서 변경이 발생할 때 트리거됩니다.

**라디오 선택 필드(기회 커스텀 필드 필터)**

라디오 선택 커스텀 필드 변경을 기반으로 기회를 필터링합니다.

• **변경됨(Has Changed):** 라디오 선택 값이 다른 옵션으로 변경될 때 트리거됩니다.

• **다음으로 변경됨(Has Changed To):** 라디오 선택이 선택한 특정 값으로 변경될 때만 트리거됩니다.

• **같음(Equals):** 라디오 선택이 특정 값과 같은 기회에서 변경이 발생할 때 트리거됩니다.

**드롭다운 필드(기회 커스텀 필드 필터의 단일 및 다중 선택)**

드롭다운 커스텀 필드 변경을 기반으로 기회를 필터링합니다. 단일 선택과 다중 선택 드롭다운 모두에서 동일하게 작동합니다.

• **변경됨(Has Changed):** 드롭다운 값이 변경될 때 트리거됩니다.

• **다음으로 변경됨(Has Changed To):** 드롭다운이 선택한 특정 값으로 변경될 때만 트리거됩니다.

• **같음(Equals):** 드롭다운이 특정 값과 같은 기회에서 변경이 발생할 때 트리거됩니다.

**날짜 필드(기회 커스텀 필드 필터)**

날짜 커스텀 필드가 이제 기회 변경 시 트리거에서 필터로 사용 가능합니다.

• **변경됨(Has Changed):** 날짜 필드 값이 변경될 때 트리거됩니다.

• **다음으로 변경됨(Has Changed To):** 날짜 필드가 선택한 특정 날짜로 변경될 때만 트리거됩니다.

• **같음(Equals):** 날짜 필드가 특정 날짜와 같은 기회에서 변경이 발생할 때 트리거됩니다.

### 4단계: 워크플로우 액션 설정

트리거가 설정되면 다음과 같은 액션을 정의합니다:

- **이메일 또는 SMS 알림 보내기:** 기회 업데이트에 대해 영업 담당자에게 알림을 보냅니다.

- **할 일 배정:** CRM에서 후속 조치 할 일을 자동으로 생성합니다.

- **연락처 레코드 업데이트:** 기회 변경을 기반으로 고객 세부 정보를 수정합니다.

- **다른 워크플로우 트리거:** 고급 자동화를 위해 이 트리거를 다른 워크플로우와 연결합니다.

### 5단계: 저장, 테스트 및 발행

- **트리거 저장(Save Trigger)**을 클릭하여 설정을 완료합니다.

- 워크플로우가 올바르게 작동하는지 확인하기 위해 테스트를 실행합니다.

- 확인되면 **발행(Publish)**을 클릭하여 자동화를 활성화합니다.

## **자주 묻는 질문**

**Q: "기회 상태 변경 시(Opportunity Status Changed)" 트리거와 어떻게 다른가요?**

"기회 변경 시(Opportunity Changed)" 트리거는 배정, 리드 가치, 파이프라인, 커스텀 필드를 포함한 기회 내의 모든 변경을 추적합니다. "기회 상태 변경 시(Opportunity Status Changed)"는 기회의 상태(예: 진행 중, 성사, 실패)가 수정될 때만 활성화됩니다.

**Q: 하나의 워크플로우에 여러 필터를 설정할 수 있나요?**

네! 수석 영업 담당자에게 재배정된 고가치 리드만 추적하는 등 여러 필터를 결합할 수 있습니다.

**Q: 워크플로우가 올바르게 작동하는지 어떻게 테스트하나요?**

트리거를 설정한 후 기회를 수동으로 업데이트하고 워크플로우가 예상된 액션을 실행하는지 확인합니다. 문제를 해결하려면 워크플로우 로그를 검토하세요.

**Q: 이 트리거가 기회 커스텀 필드와 작동하나요?**

네! 기회 레코드 내의 특정 커스텀 필드가 업데이트될 때 활성화되도록 트리거를 설정할 수 있습니다.

**Q: "변경됨(Has Changed)", "다음으로 변경됨(Has Changed To)", "같음(Equals)" 연산자의 차이점은 무엇인가요?**

"변경됨(Has Changed)"은 해당 특정 필드 값이 무엇이든 변경될 때 트리거됩니다. "다음으로 변경됨(Has Changed To)"은 필드가 선택한 특정 값으로 변경될 때만 트리거됩니다. "같음(Equals)"은 해당 특정 필드 값을 현재 가지고 있는 기회에서 변경이 발생할 때 트리거됩니다.

**Q: 이제 날짜 커스텀 필드를 필터로 사용할 수 있나요?**

네! 날짜 커스텀 필드가 이제 기회 변경 시 트리거에서 필터로 사용 가능합니다. "변경됨(Has Changed)", "다음으로 변경됨(Has Changed To)", 또는 "같음(Equals)" 연산자를 사용하여 날짜 필드 변경을 기반으로 워크플로우를 트리거할 수 있습니다.
