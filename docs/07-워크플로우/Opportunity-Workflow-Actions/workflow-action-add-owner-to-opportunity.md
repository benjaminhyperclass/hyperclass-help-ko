# 워크플로우 액션 - 기회에 담당자 추가

## **개요**

워크플로우의 기회에 담당자 추가(Add Owner to Opportunity) 액션을 사용하면 기존 기회(Opportunity)의 담당자를 자동으로 지정할 수 있습니다. 이를 통해 연락처가 파이프라인을 진행함에 따라 기회 담당자가 연락처 담당자와 일치하도록 유지할 수 있습니다. 이 액션은 각 기회마다 영업 파이프라인을 통한 진행을 관리할 수 있는 지정된 담당자가 있도록 보장하는 데 유용합니다.

## 액션 이름

"Add Owner to Opportunity"

워크플로우 빌더에서 이 액션을 추가할 때 표시되는 라벨입니다.

---

## 액션 설명

트리거가 실행되면, 기회에 담당자 추가 액션은 선택한 사용자를 해당 기회의 새 담당자로 설정합니다. 플랫폼 설정에서 연락처 담당자와 기회 담당자를 동기화하도록 구성되어 있다면, 연락처의 담당자도 자동으로 업데이트됩니다. 현재 미할당된 기회에만 이 지정을 적용하도록 선택할 수도 있습니다. 그렇지 않으면 기존 담당자가 재정의됩니다.

중요: 이 액션은 기회가 컨텍스트에 있어야 작동합니다. 즉, 워크플로우가 기회에 의해 트리거되었거나 이전 기회 찾기(Find Opportunity) 단계에서 기회를 찾았어야 합니다. 컨텍스트에 기회가 없으면 이 액션은 아무것도 하지 않습니다.

## 액션 세부사항

| 값 이름 | 설명 | 필수 여부 |
|---------|------|----------|
| Action Name | 워크플로우에서 이 단계를 식별하는 고유한 라벨 | 예 |
| User | 기회의 담당자로 지정될 사용자. 일반적으로 계정의 사용자 목록이 드롭다운으로 표시됨 | 예 |
| Only Apply to Unassigned Opportunities | 활성화하면 현재 담당자가 없는 기회에만 담당자 지정을 제한하는 토글. 기회에 이미 담당자가 있으면 변경되지 않음. 비활성화하면 기존 담당자가 재정의됨 | 아니오 |

### **동기화 설정**

- 연락처와 기회 담당자가 동기화되도록 설정된 경우(기회 관리 설정에서 구성), 여기서 기회 담당자를 변경하면 연락처의 담당자도 함께 변경됩니다.

- 연락처와 기회 담당자가 동기화되지 않는 경우, 기회 담당자만 변경됩니다.

### **특수 상황**

- 여러 기회: 기회 찾기 단계에서 여러 잠재적 일치 항목을 찾더라도, 하나만(기회 찾기에서 선택한 가장 빠른 것 또는 가장 늦은 것) 업데이트됩니다.

- 컨텍스트에 기회가 없는 경우: 기회를 찾지 못했거나 트리거되지 않았으면 액션이 건너뛰어집니다.

## **예시**

시나리오: 워크플로우가 고가치 리드를 식별하고(기회 찾기 액션을 통해) 우선 후속 조치를 위해 특정 사용자에게 할당합니다.

- 트리거: 폼 제출 데이터를 기반으로 리드가 고가치로 분류됨

- 액션: 기회 찾기
  - 필터: 기회 금액이 5,000 초과

- 액션: 기회에 담당자 추가
  - Action Name: "고가치 리드를 시니어 세일즈 담당자에게 할당"
  - User: "Senior Sales Rep"
  - Only Apply to Unassigned Opportunities: 비활성화 (기존 담당자를 재정의하기 위해)

이 구성으로 기회 찾기 단계에서 찾은 "고가치" 기준에 맞는 모든 기회가 "Senior Sales Rep"에게 할당되어 즉시 관심과 후속 조치를 보장합니다. 시스템이 연락처와 기회 담당자를 동기화하도록 설정된 경우, 연락처 담당자도 그에 따라 업데이트됩니다.

## **자주 묻는 질문**

**Q: 이 액션이 현재 기회 담당자를 덮어쓰나요?**

네. 연락처에 할당된 사용자가 있으면, 이 액션은 현재 기회 담당자를 그 사용자로 교체합니다. 이미 다른 사람이 할당되어 있어도 마찬가지입니다.

**Q: 기회에 다른 사용자를 수동으로 할당하고 싶다면 어떻게 해야 하나요?**

대신 기회 업데이트(Update Opportunity) 워크플로우 액션을 사용하세요. 이 액션을 사용하면 기회에 할당할 특정 사용자를 수동으로 선택할 수 있습니다.

**Q: 사용자가 할당된 후 기회를 볼 수 있나요?**

권한이 허용하는 경우에만 가능합니다. 계정에서 "본인 데이터만 보기(Only Assigned Data)" 규칙을 적용하는 경우, 사용자는 기록을 보거나 관리하려면 기회의 파이프라인에 접근할 수 있는 권한이 필요합니다.
