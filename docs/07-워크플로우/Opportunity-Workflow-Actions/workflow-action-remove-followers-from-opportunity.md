# 워크플로우 액션 - 기회에서 팔로워 제거

## 개요

기회에서 팔로워 제거(Remove Follower(s) From Opportunity) 액션을 사용하면 기회의 팔로워 목록에서 특정 사용자를 제거하거나 모든 팔로워를 한 번에 제거할 수 있습니다. 이 액션은 일반적으로 기회 찾기(Find Opportunity) 단계 다음에 오거나 기회가 트리거된 워크플로우에서 사용되어 특정 기회가 컨텍스트에 있어야 합니다. 컨텍스트에 기회가 없으면 액션이 건너뛰어집니다.

## 액션 이름

Remove Follower(s) From Opportunity

## 액션 설명

기회에서 팔로워 제거 액션이 트리거되면, 지정한 사용자를 기회의 팔로워 목록에서 제거합니다. 모든 팔로워 제거(Remove All Followers)가 활성화되면 사용자 목록에 관계없이 기회의 모든 팔로워가 제거됩니다. 이는 팔로워 목록을 정리하거나 관련 팀원만 기회에 할당된 상태로 유지하는 데 유용합니다.

| Action Name | 워크플로우에서 이 단계를 식별하는 데 도움이 되는 고유 라벨 | 예 |
| Users | 기회의 팔로워 목록에서 제거할 사용자 (예: "Account User"). 해당 사용자가 현재 팔로워가 아니라면 변경되지 않습니다. | 아니요 ("Remove All Followers"가 비활성화된 경우 제외) |
| Remove All Followers | 활성화하면 Users 필드에 지정된 사용자와 관계없이 기회의 모든 팔로워를 제거하는 토글 | 아니요 |

### 예외 상황

- **컨텍스트에 기회가 없는 경우**: 워크플로우 컨텍스트에 기회가 없으면(트리거되지도 않고 기회 찾기를 통해서도 발견되지 않음) 이 액션이 건너뛰어집니다.
- **여러 기회**: 기회 찾기 단계가 여러 레코드와 일치할 수 있더라도 하나의 기회(가장 이른 것 또는 가장 늦은 것)만 영향을 받습니다.
- **팔로워가 아닌 사용자**: 지정된 사용자가 현재 기회를 팔로우하고 있지 않으면 해당 사용자에 대해서는 변경이 이루어지지 않습니다.
- **모든 팔로워 제거 vs 특정 사용자**: 모든 팔로워 제거가 활성화되면 시스템은 특정 사용자 목록을 무시하고 모든 팔로워를 제거합니다.

## 예시

**시나리오**: 완료된 기회에서 사용자 그룹의 할당을 해제하여 레코드 소유자만 팔로워로 남기는 워크플로우

- **트리거**: 기회가 "완료(Closed)" 단계로 이동할 때
- **액션**: 기회 찾기(Find Opportunity)
  - 필터: 상태가 "완료(Closed)"인 기회
- **액션**: 기회에서 팔로워 제거(Remove Follower(s) From Opportunity)
  - Action Name: "완료된 거래에서 팔로워 정리"
  - Users: "영업팀 A", "영업팀 B"
  - Remove All Followers: 비활성화 (지정된 사용자만 제거)

이 예시에서 기회 찾기 단계가 완료된 기회를 찾으면, 기회에서 팔로워 제거 액션이 지정된 사용자들을 팔로워 목록에서 제거합니다. 모든 팔로워를 제거해야 한다면 모든 팔로워 제거를 활성화하면 됩니다.
