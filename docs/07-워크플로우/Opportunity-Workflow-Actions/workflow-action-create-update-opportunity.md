## 개요

자동화 워크플로우의 "기회 생성 또는 업데이트(Create or Update Opportunity)" 액션은 영업 파이프라인 내에서 기회를 관리하도록 설계되었습니다. 이 기능을 통해 미리 정의된 조건에 따라 기회를 자동으로 생성하거나 업데이트할 수 있어, 영업 프로세스를 체계적이고 효율적으로 유지할 수 있습니다.

## 액션명

Create Opportunity(기회 생성)

## 액션 설명

"기회 생성 또는 업데이트(Create or Update Opportunity)" 액션은 선택한 파이프라인과 파이프라인 단계 내에서 새로운 기회를 생성하거나 기존 기회를 업데이트합니다. 이 액션은 리드를 추적하고 영업 프로세스의 여러 단계를 통해 잠재적인 매출을 관리하는 데 도움이 됩니다.

하지만 이 액션은 다음을 지원하지 않습니다:

- 워크플로우를 트리거한 기회 (즉, 워크플로우를 시작하게 한 기회를 자동으로 업데이트하지 않습니다)
- "기회 찾기(Find Opportunity)" 단계 (즉, 이전의 "기회 찾기" 액션으로 찾은 기회를 사용하지 않습니다)

## 액션 세부 사항

"기회 생성 또는 업데이트(Create or Update Opportunity)" 액션을 구성하는 세부 사항은 다음과 같습니다:

![기회 생성/업데이트 액션 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038428191/original/FL0knisuSnMYjK8PkXZ3pLdWt054jH15jQ.png?1734370334)

| 항목명 | 설명 | 필수 여부 |
|--------|------|-----------|
| Action Name(액션명) | 워크플로우에서 쉽게 식별할 수 있도록 이 액션에 지정하는 이름 | 예 |
| In Pipeline(파이프라인) | 기회가 생성되거나 업데이트될 특정 영업 파이프라인 | 예 |
| In Pipeline Stage(파이프라인 단계) | 선택한 파이프라인 내에서 기회가 배치될 단계 | 예 |
| Opportunity Name(기회명) | 쉬운 식별을 위해 기회에 지정되는 이름 | 예 |
| Opportunity Source(기회 출처) | 리드가 어디서 왔는지를 나타내는 기회의 출처 | 예 |
| Lead Value(리드 가치) | 리드의 예상 가치 (선택사항) | 아니오 |
| Status(상태) | 기회의 현재 상태 (예: 진행 중, 완료) | 예 |
| Allow Opportunity to Move to Any Previous Stage in Pipeline(파이프라인의 이전 단계로 기회 이동 허용) | 필요시 기회를 파이프라인의 이전 단계로 이동할 수 있게 허용 | 아니오 |
| Allow Duplicate Opportunities(중복 기회 허용) | 유사한 기회가 이미 존재하는 경우 중복 기회 생성을 허용 | 아니오 |

### 중요 참고사항

- **사용 중단**: "기회 생성 또는 업데이트(Create or Update Opportunity)" 액션은 단계적으로 폐지되고 있습니다. 대신 더 세밀한 제어와 추가 기능(예: "기회 찾기" 단계 활용 또는 트리거된 기회 업데이트)을 지원하는 Create Opportunity(기회 생성) 또는 Update Opportunity(기회 업데이트) 액션 사용을 권장합니다. 
통합된 액션을 점진적으로 폐지하고 있지만, 새로운 워크플로우에서만 적용됩니다. 기존 워크플로우에서 통합된 액션을 사용하는 경우 계속해서 그대로 작동합니다.

- **"기회 찾기" 또는 트리거된 기회 지원 불가**: 이 액션은 워크플로우를 트리거한 기회를 업데이트하지 않으며, 기회 찾기(Find Opportunity) 액션을 통해 찾은 기회도 사용하지 않습니다. 오직 자체 로직으로 기존 기회를 찾거나 새로운 기회를 생성하는 방식으로만 작동합니다.

## 예시: 캘린더 예약 퍼널의 신규 리드 관리

**시나리오:** 캘린더 예약 퍼널에서 새로운 리드가 들어올 때마다 영업 파이프라인에 기회를 생성하고자 합니다. 이를 통해 모든 리드가 영업 프로세스를 통해 적절히 추적되고 관리되도록 합니다.

**"기회 생성 또는 업데이트(Create or Update Opportunity)" 액션을 사용한 해결책:**

**트리거:** 캘린더 예약 퍼널에서 새로운 리드가 생성될 때 워크플로우가 시작됩니다.

**액션 1:** Create Opportunity(기회 생성) 액션으로 파이프라인에 새로운 기회를 생성합니다.

**구현 단계:**

- 새로운 리드가 생성될 때 트리거되도록 워크플로우에 Create Opportunity(기회 생성) 액션을 추가합니다.
- Action Name(액션명)을 "Create Opportunity"로 설정합니다.
- In Pipeline(파이프라인) 드롭다운에서 적절한 영업 파이프라인을 선택합니다 (예: "YOUR PROMOTION Pipeline").
- In Pipeline Stage(파이프라인 단계) 드롭다운에서 기회의 초기 단계를 선택합니다 (예: "New Leads").
- Opportunity Name(기회명)을 "Deal 251 Opportunity"와 같이 설명적으로 설정합니다.
- Opportunity Source(기회 출처)를 리드의 출처를 나타내도록 정의합니다 (예: "Lead from Calendar Appointment Funnel").
- 리드의 예상 가치가 있다면 Lead Value(리드 가치)를 선택적으로 입력할 수 있습니다.
- Status(상태)를 "open"으로 설정합니다.
- 필요에 따라 추가 설정을 구성합니다:
  - 파이프라인에서 기회를 뒤쪽 단계로 이동할 수 있는 유연성을 원한다면 "Allow Opportunity to Move to Any Previous Stage in Pipeline(파이프라인의 이전 단계로 기회 이동 허용)"을 활성화합니다.
  - 유사한 리드에 대해 중복 기회 생성을 허용하는 프로세스라면 "Allow Duplicate Opportunities(중복 기회 허용)"를 활성화합니다.

**결과:** 이 자동화는 캘린더 예약 퍼널의 모든 새로운 리드가 자동으로 영업 파이프라인의 기회로 추적되도록 보장합니다. 이를 통해 영업 프로세스를 효과적으로 체계화하고 관리할 수 있으며, 어떤 리드도 놓치지 않게 됩니다.
