## 개요

Hyperclass의 번호 검증 트리거(Number Validation Trigger)는 전화번호 검증 상태에 따라 워크플로우를 자동으로 실행할 수 있게 해주는 기능입니다. 이 기능은 유효한 번호에만 SMS를 발송하여 전송 불가 메시지로 인한 비용을 절약하고 전체적인 커뮤니케이션 효율성을 향상시키는 데 중요한 역할을 합니다.

## 트리거 이름

Number Validation(번호 검증)

## 트리거 설명

번호 검증 기능이 활성화된 경우 작동하며, 첫 번째 메시지를 보내기 전에 연락처 번호가 일반 전화(유선전화)이거나 유효하지 않은지 확인합니다.

## 설정 방법

번호 검증 트리거를 활성화하려면 다음 단계를 따르세요:

- 에이전시 설정(Agency Settings)으로 이동합니다.
- Phone integration settings(전화 연동 설정)으로 이동하여 각 로케이션에서 번호 검증을 활성화합니다.
- 검증 트리거를 활용하여 다양한 검증 결과에 적절히 대응하는 워크플로우를 구성합니다.

![전화 연동 설정에서 번호 검증 활성화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155031483049/original/X3Fs4ydaAoSWBqKxOqqHaitOlfeuFay_0g.png?1724319617)

### **참고사항:**

번호 검증 트리거는 검증 설정을 기반으로만 작동합니다. 연락처가 생성되고 CRM에서 연락처 번호에 대한 검증이 실행될 때 작동합니다. Twillio 오류 코드를 기반으로는 실행되지 않습니다.

## 사용 예시

- **SMS 캠페인 관리**: 전화번호가 검증되어 SMS 수신이 가능한 것으로 확인되면, 트리거가 자동으로 해당 연락처를 활성 SMS 캠페인에 추가할 수 있습니다. 반대로 번호가 유효하지 않거나 SMS 수신이 불가능한 경우, 트리거가 해당 연락처를 캠페인에서 제거하여 전송 불가 메시지로 인한 자원 낭비를 방지할 수 있습니다.

- **리드 자격 검증**: 리드가 폼을 통해 전화번호를 제출하면, 검증 트리거가 번호의 상태를 확인할 수 있습니다. 번호가 유효하지 않으면 리드에 "유효하지 않음" 태그를 추가하거나 데이터베이스에서 제거할 수 있습니다. 이는 깔끔하고 효과적인 리드 목록 유지에 도움이 됩니다.

- **오류 알림**: 검증 오류가 발생할 때마다 마케팅 팀에 알림을 보내도록 트리거를 설정할 수 있습니다. 예를 들어, 유선전화로 인해 번호 검증이 실패하면 팀에 자동 메시지를 보내 리드 정보를 검토하고 적절한 조치를 취하도록 할 수 있습니다.

- **컴플라이언스 관리**: TCPA와 같은 규정 준수가 중요한 상황에서, 검증 트리거는 유효하지 않은 것으로 표시된 번호로 메시지 발송을 방지할 수 있습니다. 이는 원하지 않는 메시지 발송과 관련된 위험을 완화하고 커뮤니케이션 전략이 법적 기준을 준수하도록 보장합니다.

- **자동 태그 지정**: 검증 시 트리거가 검증 결과에 따라 연락처에 자동으로 태그를 추가할 수 있습니다. 예를 들어, 유효하지 않은 것으로 확인된 번호는 "연락 금지" 태그를 추가하여 아웃리치 활동을 보다 쉽게 관리할 수 있습니다.

**자주 묻는 질문**

### **1. 번호 검증 트리거는 CRM의 리드 관리를 어떻게 개선하나요?**

번호 검증 트리거는 유효한 전화번호만 커뮤니케이션에 사용되도록 보장하여 리드 관리를 향상시킵니다. 유효하지 않은 번호를 캠페인에서 자동으로 제거하고 검토용으로 태그를 지정함으로써, 사용자는 깔끔하고 효과적인 연락처 목록을 유지할 수 있어 더 나은 참여도와 전환율을 달성할 수 있습니다.

### **2. CRM에서 번호 검증 트리거를 사용하는 이점은 무엇인가요?**

CRM에서 번호 검증 트리거를 사용하면 다음과 같은 여러 이점이 있습니다:
- 유효한 번호에만 메시지를 보내 커뮤니케이션 효율성 향상
- 전송 불가 메시지로 인한 비용 절약
- 검증 결과에 따른 자동화된 작업을 통한 리드 관리 간소화
- 유효하지 않은 번호로의 메시지 발송 방지를 통한 규정 준수 강화

### **3. 번호 검증 트리거는 CRM에서 컴플라이언스에 어떻게 도움이 되나요?**

번호 검증 트리거는 유효하지 않거나 SMS 수신이 불가능한 번호로 메시지가 발송되는 것을 방지하여 컴플라이언스에 도움을 줍니다. 이는 원하지 않는 커뮤니케이션의 위험을 줄이고 TCPA(전화 소비자 보호법)와 같은 규정 준수를 보장합니다. 깔끔한 연락처 목록을 유지함으로써 잠재적인 법적 문제를 피하고 브랜드 평판을 보호할 수 있습니다.

### **4. CRM에서 번호 검증 트리거를 커스터마이징할 수 있나요?**

네, CRM의 번호 검증 트리거는 검증 결과에 따른 커스텀 워크플로우 생성을 허용합니다. 사용자는 특정 조건과 액션을 설정하여 고유한 비즈니스 요구사항에 맞게 트리거의 동작을 조정할 수 있습니다. 이러한 유연성은 트리거가 리드 관리에서 최대 효율성과 효과성을 위해 최적화될 수 있도록 보장합니다.

### **5. CRM에서 번호 검증 트리거는 수동 번호 검증과 어떻게 비교되나요?**

수동 번호 검증과 비교했을 때, CRM의 번호 검증 트리거는 다음과 같은 여러 장점을 제공합니다:
- 자동화된 검증 과정으로 시간 절약 및 사람의 실수 위험 감소
- 모든 리드에 대한 검증 규칙의 일관적인 적용
- 검증 결과에 따른 액션 트리거 가능, 리드 관리 워크플로우 간소화
- 실시간 검증으로 유효한 번호만 커뮤니케이션에 사용 보장

### **6. CRM의 번호 검증 트리거로 어떤 유형의 전화번호를 검증할 수 있나요?**

CRM의 번호 검증 트리거는 휴대폰, 일반 전화(유선전화), VoIP 번호를 포함한 다양한 유형의 전화번호를 검증할 수 있습니다. 검증 과정은 번호가 SMS 메시지를 수신할 수 있는지 확인하여, 커뮤니케이션 노력이 유효하고 연결 가능한 연락처에 집중되도록 보장합니다.
