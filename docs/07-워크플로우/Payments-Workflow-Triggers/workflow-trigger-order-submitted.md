# 워크플로우 트리거 - 주문 제출

구매 후 액션을 효율적으로 관리하는 것은 모든 비즈니스에게 중요합니다. Hyperclass의 주문 제출 트리거(Order Submitted Trigger)는 1단계 또는 2단계 주문 폼을 통해 완료된 주문을 기반으로 워크플로우를 자동화합니다. 이 가이드는 이 트리거가 무엇을 하는지, 다른 주문 관련 트리거와 어떻게 다른지, 그리고 최적화된 워크플로우 자동화를 위해 어떻게 설정하는지 이해하는 데 도움이 될 것입니다.

# 주문 제출 워크플로우 트리거란?

주문 제출 트리거는 주문 폼을 통해 주문이 완료되었을 때 워크플로우를 활성화합니다. 기본 주문 폼 트리거와 달리, 이 트리거는 기본 구매와 업셀 거래를 모두 캡처하여 다음과 같은 용도에 이상적입니다:

- 주문 확인 이메일 전송
- 업셀 후속 조치 자동화
- 주문 세부 정보로 고객 기록 업데이트
- 구매 행동에 따른 고객 세분화

이 트리거는 커스텀 값을 지원하여 비즈니스가 워크플로우 내에서 상품명, 수량, 가격, 결제 방법을 동적으로 참조할 수 있게 합니다.

## 주문 제출 워크플로우 트리거의 주요 장점

- **자동화된 참여**: 주문 확인과 업셀 제안을 즉시 전송합니다.
- **매출 증대**: 업셀 워크플로우를 통해 매출을 늘릴 수 있습니다.
- **시간 절약 자동화**: 수동 주문 관리 업무를 줄입니다.
- **자세한 주문 인사이트**: 더 나은 자동화를 위해 상품과 결제 데이터를 캡처합니다.

---

## 주문 제출 vs. 주문 폼 제출

자동화에 적합한 트리거를 선택하기 위해서는 이 트리거들 간의 차이점을 이해하는 것이 중요합니다.

| 특징 | 주문 제출 | 주문 폼 제출 |
|------|-----------|-------------|
| 트리거 활성화 | 각 상품 구매마다 실행 (기본, 범프, 업셀) | 구매한 상품에 관계없이 주문 제출당 한 번만 실행 |
| 필터 기능 | 상품, 가격, 소스, 제출 유형으로 필터링 가능 | 상품/가격으로 필터링 불가 |
| 커스텀 값 | 상품명, 가격, 수량 등 주문 세부정보 제공 | 제한적인 주문 세부정보만 사용 가능 |

## 주문 제출 워크플로우 트리거 설정하기

### 워크플로우 빌더 접근

자동화(Automation) 탭으로 이동하여 워크플로우(Workflows)를 선택합니다. 처음부터 시작하는 경우 `+ 워크플로우 생성(Create Workflow)`을 클릭하여 새 자동화를 구축하세요. 기존 워크플로우에 이 트리거를 추가하려면 목록에서 워크플로우를 찾아 편집을 위해 열기만 하면 됩니다.

![워크플로우 빌더 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041447244/original/KLPycPHwFyJvar-6Uifu3gF0yGEd0wRCng.png?1739354237)

### 주문 제출 트리거 추가

워크플로우 빌더 내에서 `새 트리거 추가(Add New Trigger)`를 클릭하여 자동화 설정을 시작합니다. 검색창에 "Order Submitted"를 입력하고 사용 가능한 트리거 목록에서 선택합니다.

트리거를 선택한 후 상품 유형, 주문 소스 또는 제출 세부정보에 따른 필터를 적용하여 추가로 커스터마이즈할 수 있습니다.

![주문 제출 트리거 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041448111/original/C9kYGDOj50r0ywoURh10ZgKotD6Y141hiQ.png?1739354689)

### 트리거 이름 지정

트리거의 목적을 반영하는 명확하고 구체적인 이름을 지정하세요. 잘 명명된 트리거는 워크플로우 내에서 관리하고 식별하기 쉽게 하며, 자동화 설정을 검토할 때 명확성을 보장합니다.

![트리거 이름 지정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041449901/original/Jr9Vkn5E8TTTXimIQ_7xH8yh0AquP-jb8A.png?1739355684)

### 필터 적용하기

필터를 통해 주문 세부정보를 기반으로 조건을 지정하여 워크플로우 트리거를 세밀하게 조정할 수 있습니다. 이렇게 하면 특정 기준이 충족될 때만 자동화가 실행되도록 할 수 있습니다.

아래는 사용 가능한 필터와 해당 연산자로, 비즈니스 요구에 맞는 정확한 커스터마이징이 가능합니다.

#### 글로벌 상품

이 필터를 통해 특정 상품이나 가격을 기반으로 워크플로우를 트리거할 수 있습니다. 드롭다운 메뉴에서 "Is" 또는 "Is Not"을 선택하여 상품을 포함하거나 제외할 수 있으며, 관련 구매에만 자동화가 실행되도록 할 수 있습니다. 이는 특정 상품에 대한 타겟 후속 조치나 독점적인 구매 후 액션을 설정하는 데 유용합니다.

- **Is**: 지정된 상품이 구매되었을 때만 워크플로우를 트리거합니다.
- **Is Not**: 선택된 상품을 제외하여 지정된 상품을 제외한 모든 다른 상품에 대해 워크플로우가 실행되도록 합니다.

![글로벌 상품 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041451561/original/VB2bGF6uQ2iu5TCoVnSXZfn9WnLiiFrHYg.png?1739356561)

#### 주문 소스

이 필터를 통해 주문이 어디서 이루어졌는지 결정할 수 있으며, 특정 소스에서의 구매에 대해서만 워크플로우가 트리거되도록 할 수 있습니다. 선택 가능한 옵션은 다음과 같습니다:

| 필터 옵션 | 설명 |
|-----------|------|
| External | 외부 연동이나 서드파티 시스템을 통해 이루어진 주문 |
| Form | 일반 폼(주문 폼이 아닌)을 통해 제출된 주문 |
| Membership | 멤버십 기반 상품에 대한 구매 |
| Order Form | 주문 폼을 통해 처리된 거래 |
| Payment Link | 직접 결제 링크를 통해 완료된 주문 |
| Store | 온라인 스토어에서 이루어진 구매 |
| Survey | 설문 폼을 통해 제출된 주문 |

연산자:
- **Is**: 선택된 주문 소스에 대해서만 워크플로우를 트리거합니다.
- **Is Not**: 선택된 주문 소스를 워크플로우 트리거에서 제외합니다.

![주문 소스 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041451651/original/W6xZaFYIhJbqLJjgA7n2mVsyX0laW4SHEQ.png?1739356598)

#### 하위 소스

이 필터를 통해 주문 소스를 더욱 세밀하게 세분화할 수 있으며, 비즈니스가 선택된 소스 내의 특정 퍼널, 페이지 또는 기타 하위 카테고리를 타겟팅할 수 있도록 도와줍니다. 주문이 메인 주문 소스의 특정 하위 환경 내에서 이루어졌을 때만 워크플로우가 트리거되도록 합니다.

예를 들어:
- 주문 폼(Order Form)이 주문 소스로 선택된 경우, 하위 소스 필터를 통해 주문 폼이 제출된 특정 퍼널이나 웹사이트를 지정할 수 있습니다.
- 스토어(Store)가 주문 소스인 경우, 하위 소스 필터는 구매가 발생한 특정 상품 카테고리나 스토어 섹션을 식별하는 데 도움이 됩니다.

연산자:
- **Is**: 주문이 선택된 하위 소스와 일치할 때만 워크플로우를 트리거합니다.
- **Is Not**: 선택된 하위 소스의 주문을 제외합니다.

![하위 소스 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041453920/original/XSKgeqL9Ut_CvNhh52QzaUF8dCMQ-6S4og.png?1739357968)

#### 퍼널/웹사이트 내

이 필터는 특정 퍼널이나 웹사이트를 통해 주문 폼이 제출되었을 때만 워크플로우가 트리거되도록 하여, 비즈니스가 사용된 정확한 판매 채널을 기반으로 자동화를 세분화할 수 있게 해줍니다. 이는 여러 판매 퍼널을 운영하거나 다양한 상품이나 서비스에 대해 서로 다른 웹사이트를 가진 비즈니스에게 특히 유용합니다.

예를 들어:
- 여러 퍼널에서 상품을 판매하는 경우, 특정 퍼널 내에서 주문이 제출되었을 때만 워크플로우를 트리거할 수 있습니다.
- 다양한 프로모션에 대해 서로 다른 랜딩 페이지를 가지고 있는 경우, 이 필터를 사용하여 퍼널이나 웹사이트 내의 특정 페이지에서 주문이 들어왔을 때만 워크플로우를 트리거할 수 있습니다.

사용 가능한 연산자:
- **Is**: 선택된 퍼널이나 웹사이트를 통해 주문이 제출된 경우에만 워크플로우가 트리거됩니다.
- **Is Not**: 선택된 퍼널이나 웹사이트의 주문을 제외합니다.

![퍼널/웹사이트 내 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041455510/original/TcbjTdl0sBMEDuLY37dmIIXHxIff2vGtSg.png?1739359008)

#### 페이지 조건

이 필터는 퍼널이나 웹사이트 내의 특정 페이지에서 주문 폼이 제출되었을 때만 워크플로우가 트리거되도록 합니다. 비즈니스가 고객이 구매를 완료한 정확한 페이지를 기반으로 고도로 타겟팅된 자동화를 만들 수 있게 해줍니다.

예를 들어:
- 비즈니스가 퍼널 내에 여러 페이지(예: 결제 페이지, 감사 페이지, 업셀 페이지)를 가지고 있는 경우, 이 필터는 등록 폼이나 결제 페이지 같은 선택된 페이지에서 주문이 제출되었을 때만 워크플로우가 트리거되도록 합니다.
- 웹사이트의 다양한 페이지에 서로 다른 주문 폼이 존재하는 경우, 이 필터를 사용하여 퍼널이나 웹사이트의 모든 페이지가 아닌 특정 주문 페이지로만 워크플로우 실행을 제한할 수 있습니다.

사용 가능한 연산자:
- **Is**: 선택된 페이지에서 주문이 제출된 경우에만 워크플로우가 트리거됩니다.
- **Is Not**: 선택된 페이지의 주문을 제외합니다.

![페이지 조건 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041456803/original/pPELnia5Lg3ESK6wczTCfBKZn1uWCSOAkA.png?1739359842)

#### 제출 유형

이 필터를 통해 이루어진 구매 유형에 따라 워크플로우가 트리거되도록 하여, 비즈니스가 그에 따른 후속 조치와 구매 후 액션을 자동화할 수 있습니다. 특정 유형의 거래가 발생할 때만 워크플로우가 활성화되도록 합니다.

| 연산자 | 설명 |
|--------|------|
| Primary | 고객이 결제 시 주 상품을 구매할 때 트리거됩니다. 이는 주문 폼을 통해 이루어진 초기 상품 구매입니다. |
| Bump | 결제 중 범프 오퍼가 수락될 때 트리거됩니다. 범프 오퍼는 고객이 주 상품과 함께 구매할 수 있는 선택적 추가 상품입니다. |
| Upsell | 결제 후 업셀이 수락될 때 트리거됩니다. 업셀은 초기 구매 후에 제시되는 추가 제안으로, 종종 주문 금액을 늘리기 위한 것입니다. |

![제출 유형 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041457552/original/FYpZwBGu8g7zoNf_bNGKc_3bPsa0vB6VTw.png?1739360381)

#### 상품 가격

이 필터는 상품의 특정 가격을 기반으로 워크플로우를 트리거하여, 비즈니스가 정가, 할인가 또는 프로모션가를 기반으로 액션을 구분할 수 있게 해줍니다. 상품이 지정된 가격대에서 구매되었을 때만 워크플로우가 활성화되도록 합니다.

사용 가능한 연산자:
- **Is**: 상품이 주문 폼에 설정된 특정 가격으로 구매되었을 때 워크플로우를 트리거합니다.
- **Is not**: 상품이 지정된 가격 이외의 다른 가격으로 구매되었을 때 워크플로우를 트리거합니다.

![상품 가격 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041457186/original/FYkeDjYIGGnznC1EeVoKA8LszdSfuPL4ng.png?1739360116)

### Allow Multiple 활성화 (업셀 & 범프 모범 사례)

기본 구매와 업셀 모두에 대한 원활한 자동화를 보장하려면, 워크플로우 설정에서 다중 허용(Allow Multiple) 설정을 활성화하는 것이 좋습니다. 이를 통해 범프 오퍼와 업셀을 포함하여 각 구매 이벤트에 대해 워크플로우가 별도로 트리거될 수 있습니다.

**다중 허용이 켜져 있을 때**: 워크플로우가 같은 연락처에 대해 여러 번 트리거될 수 있어, 기본 구매와 그 이후의 업셀 구매가 독립적으로 처리되도록 합니다.

**다중 허용이 꺼져 있을 때**: 기본 구매로부터 워크플로우가 여전히 활성 상태인 경우, 고객이 한 업셀 구매는 워크플로우의 새 인스턴스를 트리거하지 않습니다.

팁: 다중 허용을 켜두면 지연을 방지하고 업셀과 범프 오퍼가 워크플로우를 올바르게 트리거하도록 하여 매출 기회와 고객 참여를 최적화할 수 있습니다.

![다중 허용 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155041458623/original/X3hNZ7vn-fIj-15ytERk82g925XxdyR9_Q.png?1739361076)

### 워크플로우 저장 및 발행

주문 제출 트리거를 설정하고 필요한 필터를 적용했다면, 이제 워크플로우를 마무리할 차례입니다.

- **저장(Save)**을 클릭하여 모든 트리거 설정이 올바르게 적용되었는지 확인합니다.
- **발행(Publish)**을 클릭하여 워크플로우를 활성화하고 주문을 자동으로 처리하기 시작하도록 합니다.
- 테스트 주문을 제출하고 워크플로우 히스토리(Workflow History)를 확인하여 트리거가 예상대로 실행되는지 **워크플로우를 테스트**하세요.

## 자주 묻는 질문

**Q: 이 트리거가 업셀 구매에도 작동하나요?**

네, 주문 제출 트리거는 워크플로우 설정에서 다중 허용(Allow Multiple)이 활성화되어 있으면 기본 구매와 업셀 모두에 대해 실행됩니다.

**Q: 이 트리거를 사용하여 주문 확인을 보낼 수 있나요?**

네, 주문 확인 이메일을 자동화할 수 있고 장바구니 요소(Shopping Cart element)를 사용하여 상품 세부정보를 동적으로 포함할 수 있습니다.

**Q: 특정 상품에 대해서만 워크플로우가 트리거되도록 하려면 어떻게 하나요?**

글로벌 상품(Global Product) 필터를 사용하고 "Is" 또는 "Is Not"을 선택하여 특정 상품을 포함/제외하세요.

**Q: 이 트리거가 기존 주문 폼 워크플로우에 영향을 미치나요?**

아니요, 주문 제출 트리거는 주문 폼 제출 트리거(Order Form Submitted Trigger)와 독립적으로 작동하여 워크플로우 실행에 대한 더 세밀한 제어를 가능하게 합니다.

**Q: 사용된 결제 방법에 따라 워크플로우를 트리거할 수 있나요?**

네, 결제 게이트웨이(Stripe, PayPal 등)를 기반으로 필터를 적용하여 다른 결제 방법에 대해 다른 워크플로우를 트리거할 수 있습니다.
