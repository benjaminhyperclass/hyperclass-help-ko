# 워크플로우 트리거 - 쿠폰 사용 한도 도달

쿠폰 코드가 최대 사용 한도에 도달했을 때 자동으로 작업을 실행할 수 있습니다. 이 가이드에서는 설정부터 핵심 이점까지 모든 것을 안내하여, 쿠폰의 사용 한도에 도달하는 순간 워크플로우를 원활하게 트리거할 수 있도록 도와드립니다. 캠페인 관리, 내부 알림, 프로모션 제어에 완벽한 기능입니다.

## **쿠폰 사용 한도 도달 워크플로우 트리거란?**

쿠폰 사용 한도 도달 트리거는 쿠폰 코드가 최대 허용 사용 횟수에 도달했을 때 실행됩니다. 이 트리거는 해당 쿠폰을 사용한 마지막 거래가 성공적으로 완료된 후에 활성화됩니다.

한정 기간 또는 한정 수량 프로모션을 관리하고, 과도하게 사용된 쿠폰을 자동으로 일시 정지시키거나, 특정 오퍼를 교체하거나 종료해야 한다고 팀에 알려야 하는 마케팅 및 운영 팀에게 유용한 도구입니다.

## **쿠폰 사용 한도 도달 워크플로우 트리거의 주요 이점**

- **한정 수량 쿠폰의 과사용 방지**: 사용 한도에 도달하면 즉시 알림을 받습니다

- **라이브 프로모션의 운영 제어 유지**: 적절한 시점에 쿠폰을 자동으로 비활성화하거나 교체합니다

- **수동 추적 없이 알림이나 후속 조치 트리거**: 수동으로 사용량을 모니터링할 필요 없이 고수요 캠페인을 놓치지 않습니다

- **고객 경험 개선**: 자동화된 액션을 통해 만료된 쿠폰을 대안 오퍼로 교체합니다

- **프로모션 루프 효과적 마감**: 캠페인의 사용 목표에 도달했을 때 영업 및 마케팅 팀이 정보를 얻을 수 있도록 합니다

## **쿠폰 사용 한도 도달 워크플로우 트리거 설정하기**

워크플로우 빌더에서 이 트리거를 설정하려면 다음 단계를 따르세요:

#### **1단계:** 워크플로우에 트리거 추가

- Automation(자동화) > Workflows(워크플로우)로 이동합니다
- 새 워크플로우를 생성하거나 기존 워크플로우를 편집합니다
- 워크플로우 빌더에서 + Add New Trigger(+ 새 트리거 추가)를 클릭하여 시작합니다

![새 트리거 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050401471/original/2ENHmA9VN1KwSs6oNQkcFlNrULOlDUsKJA.png?1753388951)

#### **2단계:** 트리거 유형 선택

사용 가능한 트리거 목록에서 Payments(결제) 섹션으로 스크롤하여 Coupon Redemption Limit Reached(쿠폰 사용 한도 도달)를 선택합니다

![트리거 유형 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050401529/original/tu7X7T-ItOhrgrqmHv6I_2m5W8JxY0K37w.png?1753389059)

#### **3단계:** 트리거 이름 지정

- Workflow Trigger Name(워크플로우 트리거 이름) 필드에 워크플로우를 체계적으로 관리할 수 있는 설명적인 이름을 입력합니다
- 예시: "플래시세일쿠폰한도도달"

![트리거 이름 지정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050401090/original/3tY5oaN8xXbcEI5fzzosu7Rn6Idd08XIhw.png?1753388014)

#### **4단계:** 특정 시나리오를 타겟팅하기 위한 필터 적용

- 트리거 이름을 지정한 후, 이 워크플로우가 실행되어야 하는 정확한 조건을 좁히기 위해 필터를 적용할 수 있습니다
- 필터를 추가하려면 **+ Add Filters(+ 필터 추가)**를 클릭합니다
- 각 필터는 관심 있는 조건을 정확히 타겟팅하기 위한 특정 연산자(Is, Contains, Greater Than, Before/After 등)를 제공합니다. [사용 가능한 필터 목록은 여기를 클릭하세요](#사용-가능한-필터)

![필터 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050401224/original/zuviGWg5T8OsLb4htEOJP-G84HYbv1eBDg.png?1753388345)

#### **5단계:** 워크플로우 저장 및 구축

- 트리거와 필터가 설정되면 **Save Trigger(트리거 저장)**를 클릭하고 워크플로우에 원하는 액션을 추가하기 시작합니다
- 완료되면 **Test Workflow(워크플로우 테스트)**를 사용하여 모든 것이 예상대로 작동하는지 확인한 다음, **Publish(발행)**를 클릭하여 워크플로우를 활성화합니다

![워크플로우 저장 및 발행](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049300913/original/xIU1T3gXpIxVA2z8z3tjWi6HdCKKyQMCJg.png?1751551021)

## 사용 가능한 필터

| 필터 | 연산자 | 입력 유형 |
|------|--------|-----------|
| Coupon Code(쿠폰 코드) | Is Any Of / Is Empty / Is None Of / Is Not Empty | 활성 쿠폰 코드의 다중 선택 드롭다운 |
| Coupon Name(쿠폰 이름) | Is Any Of / Is Empty / Is None Of / Is Not Empty | 활성 쿠폰 이름의 다중 선택 드롭다운 |
| Coupon Type(쿠폰 유형) | Is / Is Empty / Is Not / Is Not Empty | 단일 선택 드롭다운 (금액 vs 퍼센트) |
| Redemption Limit(사용 한도) | Equals To / Greater Than / Greater Than or Equal To / Is Empty / Is Not Empty / Is Not Equal To / Less Than / Less Than or Equal To | 숫자 입력 상자 |

## **쿠폰 한도 알림을 위한 권장 워크플로우 액션**

쿠폰 사용 한도 도달 트리거가 활성화되면 다음과 같은 액션을 자동화할 수 있습니다:

- **내부 알림**: 쿠폰이 사용 한도에 도달했을 때 마케팅 또는 영업 팀에 즉시 알립니다

- **이메일 발송**: 관리자나 캠페인 매니저에게 알려서 적절한 조치를 취할 수 있도록 합니다

- **태그 추가**: 보고 및 세분화를 위해 쿠폰이나 관련 캠페인에 태그를 적용합니다

- **쿠폰 일시정지 또는 비활성화**: 쿠폰 관리 로직에 연결된 경우 API를 통해 쿠폰을 자동으로 비활성화합니다

- **SMS 발송**: 고수요 쿠폰이 만료될 때 문자를 통해 내부 직원에게 알립니다

## **자주 묻는 질문**

**Q: 이 트리거는 사용 한도가 임박했을 때 실행되나요, 아니면 도달했을 때만 실행되나요?**
이 트리거는 완료된 거래 후 쿠폰의 최대 허용 사용 횟수에 도달했을 때만 실행됩니다.

**Q: 어떤 쿠폰 유형이 한도에 더 빨리 도달하는지 추적할 수 있나요?**
네, Coupon Type(쿠폰 유형) 필터를 사용하여 퍼센트 vs 고정 금액 쿠폰에 대한 활동을 좁힐 수 있습니다.

**Q: 필터를 적용하지 않으면 어떻게 되나요?**
워크플로우는 사용 한도에 도달하는 모든 쿠폰에 대해 트리거됩니다. 특정 캠페인을 모니터링하려면 Coupon Code Is 또는 Coupon Name Contains 같은 필터를 사용하세요.

**Q: 다양한 사용 한도를 기준으로 트리거할 수 있나요?**
네, Redemption Limit(사용 한도) 필터를 Greater Than 또는 Is 같은 연산자와 함께 사용하여 다양한 한도 수준을 기반으로 조건부 워크플로우를 만들 수 있습니다.
