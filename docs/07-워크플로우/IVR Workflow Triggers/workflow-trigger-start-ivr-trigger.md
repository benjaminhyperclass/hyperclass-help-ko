# 워크플로우 트리거 - Start IVR Trigger

## 개요

Hyperclass의 "Start IVR Trigger"를 사용하면 워크플로우 내에서 자동 음성 응답(IVR) 시스템을 만들고 관리할 수 있습니다. 이 트리거(Trigger)를 통해 발신자를 다양한 옵션으로 안내하는 다단계 음성 메뉴를 시작할 수 있어, 고객이 올바른 부서에 연결되거나 원하는 정보를 효율적으로 받을 수 있습니다. 음성사서함은 IVR 트리거로 추적할 수 없습니다.

## 트리거 이름

Start IVR Trigger

## 트리거 설명

Start IVR Trigger는 IVR 전화번호로 수신 전화가 걸려오는 등 특정 이벤트가 발생할 때 IVR 시퀀스를 실행하도록 설계되었습니다. 이 트리거는 통화 라우팅을 자동화하여 발신자가 사전 녹음된 음성 안내와 상호작용하고 전화기 키패드를 사용해 옵션을 선택할 수 있게 합니다.

## 설정 방법

**사전 요구사항**

**Start IVR Trigger**를 사용하기 전에 하위 계정이 다음 요구사항을 충족하는지 확인하세요:

- 하위 계정이 **LC Phone** 요금제에 구독되어 있어야 합니다.
- IVR에 사용할 전화번호가 **LC Phone** 설정에서 구성되어 있어야 합니다.
- 워크플로우 내에서 트리거하기 전에 IVR 설정이 생성되어 있어야 합니다.

**중요:** IVR 기능은 **LC Phone** 요금제가 활성화된 계정에서만 사용할 수 있습니다. 하위 계정이 LC Phone 요금제를 사용하지 않는 경우, **Start IVR Trigger**는 수신 전화 라우팅에서 작동하지 않습니다.

**단계별 가이드**

- 액션 유형 선택: 사용 가능한 트리거 목록에서 "Start IVR Trigger"를 선택합니다.
- 액션 이름 지정: "Trigger IVR call"과 같이 워크플로우 트리거 이름에 설명적인 이름을 입력합니다.
- **트리거를 설정**하여 수신 전화로 활성화되도록 하고 전화번호를 선택합니다.
- 필요에 따라 필터에 **커스텀 필드를 추가**합니다.
- 저장 및 활성화: 트리거를 저장하고 활성화하여 수신 전화에 대한 IVR 시스템 사용을 시작합니다.

**참고**: 전화번호가 IVR 워크플로우에 매핑되면 다른 IVR 워크플로우에 매핑할 수 없습니다.

## 예시

**시나리오:** 발신자를 회사 내 올바른 부서로 안내하는 IVR 시스템을 설정하고 싶습니다.

- **워크플로우 트리거 생성:**
  주요 비즈니스 라인으로 수신 전화가 걸려올 때 "Start IVR" 트리거가 활성화되도록 설정합니다.

- 트리거 이름과 필터를 선택합니다.

- 필터 내에서 IVR이 활성화될 수신 전화번호를 추가합니다.

- **저장 및 활성화:**
  IVR 설정을 완료하고 워크플로우를 활성화합니다. 이제 모든 수신 전화가 IVR 시스템을 트리거합니다.
