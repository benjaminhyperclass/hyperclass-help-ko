# 폼 제출 워크플로우 트리거란?

폼 제출 워크플로우 트리거는 지정된 폼이 제출될 때 활성화되어 일련의 자동화된 작업을 실행합니다. 이 트리거는 데이터 수집, 알림 전송, 레코드 업데이트, 후속 작업 생성 등의 업무를 간소화하는 데 도움이 됩니다. 폼 제출에 대한 신속한 응답과 효율적인 처리를 보장하므로, 고객과의 상호작용 및 내부 프로세스 관리에 필수적인 도구입니다.

## 폼 제출 워크플로우 트리거의 주요 장점

- 효율성 향상: 폼 제출로 실행되는 반복 작업을 자동화하여 시간을 절약하고 수동 작업을 줄입니다.
- 소통 개선: 고객에게 자동 응답을 보내고 팀원들에게 후속 조치를 위한 내부 알림을 전송합니다.
- 데이터 정확성: 제출된 폼 데이터를 CRM에 직접 통합하여 정확하고 최신 상태의 기록을 유지합니다.
- 맞춤형 워크플로우: 특정 폼이나 제출 조건을 대상으로 하는 필터를 설정하여 맞춤형 작업을 수행할 수 있습니다.
- 확장성: 여러 워크플로우를 지원하여 기업이 대량의 폼 제출을 수월하게 처리할 수 있습니다.

## 폼 제출 워크플로우 트리거 설정하기

다음 단계에 따라 워크플로우에 폼 제출 트리거를 구현해보세요:

### 워크플로우 설정에 접근하기

워크플로우 자동화 섹션으로 이동합니다. 새 워크플로우를 처음부터 만들거나 "폼 제출" 트리거를 구현하려는 기존 워크플로우를 선택하세요.

### 새 트리거 추가

"Add New Trigger(새 트리거 추가)"를 클릭하고 드롭다운 메뉴에서 "Form Submitted(폼 제출)"을 선택하세요.

![폼 제출 트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039463070/original/9PRqjo-q1aQp-Hf1yPsWxa_kivz5xiU4JQ.png?1736332197)

### 트리거 이름 설정

"고객 피드백 폼 제출"과 같이 설명적인 이름을 입력하세요.

![트리거 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039463134/original/Qe9odJyYHDOjHhnmsv93-PJjT0loq3HJMg.png?1736332229)

### 필터 설정

필터를 구성하여 워크플로우 활성화 조건을 세부적으로 설정하세요:

폼 선택:

워크플로우를 활성화할 폼을 지정하세요. 예를 들어, 폼 목록에서 "고객 피드백 폼"을 선택하세요.

![폼 필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039463171/original/ApmXW_y7okXwmzDDI-6eFI23BASDiiU5DQ.png?1736332250)

### 트리거 저장

저장 버튼을 클릭하여 설정을 확인하세요.

### 테스트 및 발행

샘플 데이터를 사용하여 워크플로우를 테스트하고 기능을 확인하세요. 검증이 완료되면 Publish(발행) 토글을 활성화하여 워크플로우를 가동하세요.

## 활용 사례

### 고객 정보 수집

시나리오: 기업에서 온라인 폼을 통해 고객 정보를 수집하고 관리하는 과정을 자동화하고 싶어합니다.

트리거 설정:
- 트리거: 폼 제출
- 필터: 폼이 "고객 정보 폼"인 경우

결과: 제출된 데이터가 CRM에 입력되고, 영업 팀에게 알림이 전송되며, 후속 작업이 생성됩니다.

![고객 정보 수집 워크플로우](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155039463388/original/51-7JIAcJBNjvbAtjO7MENaaSjvietsngw.png?1736332327)

### 리드 자격 심사 워크플로우

시나리오: 마케팅 팀에서 폼 제출을 통해 리드를 수집하고 자동으로 자격을 심사하고 싶어합니다.

트리거 설정:
- 트리거: 폼 제출
- 필터: 폼이 "리드 수집 폼"인 경우

결과: 워크플로우가 리드 점수를 부여하고, CRM에 리드 세부사항을 업데이트하며, 추가 조치를 위해 영업 팀에게 알림을 보냅니다.

### 이벤트 등록 관리

시나리오: 이벤트 주최자가 참석자 등록에 폼을 사용하고 소통 과정을 자동화해야 합니다.

트리거 설정:
- 트리거: 폼 제출
- 필터: 폼이 "이벤트 등록 폼"인 경우

결과: 등록자에게 이메일 확인서가 발송되고, 참석자 세부사항이 CRM의 참석자 목록에 추가됩니다.

### 고객 피드백 수집

시나리오: 기업에서 온라인 폼을 통해 고객 피드백을 수집하고 응답을 분석하고 싶어합니다.

트리거 설정:
- 트리거: 폼 제출
- 필터: 폼이 "고객 피드백 폼"인 경우

결과: 워크플로우가 고객에게 감사 이메일을 보내고, 응답을 분석용 피드백 데이터베이스에 추가합니다.

### 예약 알림

시나리오: 의료 기관에서 폼을 통해 예약을 받고 관련 팀에게 알림을 보내야 합니다.

트리거 설정:
- 트리거: 폼 제출
- 필터: 폼이 "예약 신청 폼"인 경우

결과: 워크플로우가 스케줄링 팀에게 알림을 보내고, 예약 캘린더를 업데이트하며, 환자에게 확인 이메일을 발송합니다.

## 자주 묻는 질문

**Q. 이 트리거가 하나의 워크플로우에서 여러 폼을 처리할 수 있나요?**

네, 더 많은 필터를 추가하거나 더 넓은 조건을 설정하여 여러 폼에 대해 활성화되도록 워크플로우를 구성할 수 있습니다.

**Q. 같은 사용자가 폼을 여러 번 제출하면 어떻게 되나요?**

트리거는 폼이 제출될 때마다 활성화됩니다. 중복 제출을 효과적으로 관리하려면 워크플로우에서 조건이나 대기 시간을 사용하세요.

**Q. 폼 제출 데이터를 응답 이메일에 포함시키려면 어떻게 하나요?**

이메일 단계에서 연락처 및 커스텀 필드 병합 태그를 사용하여 제출된 폼 데이터를 가져올 수 있습니다. 예를 들어, 폼에서 이름, 이메일, 피드백을 수집한다면 {{contact.name}}, {{contact.email}} 또는 폼과 연결된 커스텀 필드와 같은 병합 필드를 사용하여 이메일 본문에 이 필드들을 삽입할 수 있습니다. 이메일이 발송될 때 이러한 자리표시자는 폼에 제출된 실제 데이터로 자동 대체됩니다.

**Q. 불완전한 데이터가 포함된 폼 제출은 어떻게 처리하나요?**

워크플로우 내에 조건을 추가하여 필수 필드를 검증하고, 알림 발송이나 불완전한 제출 건너뛰기와 같은 특정 작업을 수행할 수 있습니다.

**Q. 이 트리거가 외부 폼 도구와 호환되나요?**

폼 제출 트리거는 CRM 내에서 생성된 폼과 작동합니다. 외부 도구의 경우 제출을 캡처하기 위해 연동 또는 웹훅을 사용해야 할 수 있습니다.
