# 기회 상태 변경 워크플로우 트리거란?

기회 상태 변경 워크플로우 트리거는 CRM에서 기회의 상태가 변경될 때 활성화됩니다. 예를 들어 "진행 중"에서 "성사"나 "실패"로 이동할 때 작동합니다. 이 트리거를 통해 팀원에게 알림 보내기, CRM 기록 업데이트, 후속 액션 시작 등의 프로세스를 자동화할 수 있습니다. 이 기능을 활용하면 워크플로우를 간소화하고, 수동 작업을 줄이며, 팀 간 시기적절한 커뮤니케이션을 보장할 수 있어요.

## **기회 상태 변경 트리거의 주요 장점**

- **자동 알림:** 기회 상태 업데이트에 대해 관련 팀원에게 즉시 알림을 보내 내부 커뮤니케이션을 개선합니다.

- **워크플로우 효율성 향상:** 상태 변경으로 트리거되는 일상적인 액션을 자동화하여 수동 개입을 줄입니다.

- **CRM 데이터 정확성 향상:** CRM 기록을 자동으로 업데이트하여 기회 추적의 일관성과 최신성을 보장합니다.

- **커스터마이징 가능한 필터:** **"이전 상태(Moved From Status)"**나 **"파이프라인(In Pipeline)"** 같은 필터를 사용하여 비즈니스 요구에 맞는 맞춤형 워크플로우를 만들 수 있습니다.

- **영업 프로세스 간소화:** 기회 단계 간 더 빠른 전환을 가능하게 하여 시기적절한 인보이스 발행, 후속 조치, 인수인계를 보장합니다.

## **트리거 설정: 단계별 가이드**

### 워크플로우 설정 접근

CRM의 자동화 또는 워크플로우 설정으로 이동합니다. 새 워크플로우를 처음부터 만들거나 "노트 추가" 트리거를 구현하려는 기존 워크플로우를 선택하세요.

### 새 트리거 추가

**"+ Add New Trigger"** 버튼을 클릭하고 드롭다운 메뉴에서 Opportunity Status Changed를 선택합니다.

### 트리거 이름 지정

"기회 성사 알림"과 같이 설명적인 이름을 제공합니다.

![트리거 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038815346/original/QeUKYjchgeZ9z1hgH1k_VVxSnJk0qUmnTQ.png?1734975136)

### 필터 설정

필터를 설정하여 트리거가 언제 활성화될지 세밀하게 조정하세요:

- **담당자(Assigned To):** 기회를 담당하는 특정 사용자나 팀을 지정합니다.

- **태그(Tag)** (이전 Has Tag): 고급 연산자와 다중 선택을 사용하여 태그별로 기회를 필터링합니다(등록됨 / 등록되지 않음 / 다음 중 하나 / 해당 없음). 고급 포함/제외를 위해 여러 태그 필터를 추가할 수도 있습니다.

- **파이프라인(In Pipeline):** 특정 파이프라인의 기회로 트리거를 제한합니다.

- **리드 가치(Lead Value):** 워크플로우를 활성화할 최소 또는 최대 리드 가치를 설정합니다.

- **이전 상태(Moved From Status):** 변경 전 원래 상태를 정의합니다.

- **변경 후 상태(Moved To Status):** 변경 후 새로운 상태를 지정합니다.

- **파이프라인 단계(Pipeline Stage):** 파이프라인 내 특정 단계로 트리거를 좁힙니다.

![필터 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155065764060/original/iNktmpy_LDTWGShWExjlrjPe5Ntp6-p8Qg.png?1772101354)

### 저장 및 테스트

설정을 저장하고 샘플 데이터를 사용하여 워크플로우가 예상대로 작동하는지 테스트합니다.

### 워크플로우 발행

Publish 토글을 활성화하여 워크플로우를 활성화하고 자동화를 시작합니다.

## **사용 사례**

### **영업 마일스톤 알림**

**시나리오:** 영업 기회가 "협상 중"에서 "성사"로 이동합니다.

**트리거 설정:**
- **트리거:** 기회 상태 변경
- **필터:**
  - **담당자:** 김철수
  - **이전 상태:** 협상 중
  - **변경 후 상태:** 성사

**결과:** 워크플로우가 영업사원에게 축하 이메일을 보내고, 재무팀에 인보이스 발행을 알리며, CRM의 마감 상태를 업데이트합니다.

### **실패한 기회 후속 조치**

**시나리오:** 기회가 "진행 중"에서 "실패"로 이동합니다.

**트리거 설정:**
- **트리거:** 기회 상태 변경
- **필터:**
  - **태그:** "우선순위 높음"
  - **변경 후 상태:** 실패

**결과:** 워크플로우가 잠재고객에게 피드백을 요청하는 후속 이메일을 보내고, 추가 평가를 위해 계정 관리자에게 알립니다.

### **파이프라인 전환 알림**

**시나리오:** 기회가 "Enterprise Deals" 파이프라인의 "검토 중" 단계로 이동합니다.

**트리거 설정:**
- **트리거:** 기회 상태 변경
- **필터:**
  - **파이프라인:** Enterprise Deals
  - **파이프라인 단계:** 검토 중

**결과:** 워크플로우가 영업 관리자에게 알리고 검토팀에 기회 평가 할 일을 배정합니다.

### **리드 재검증 워크플로우**

**시나리오:** 휴면 리드가 다시 참여하여 "실패"에서 "재개"로 이동합니다.

**트리거 설정:**
- **트리거:** 기회 상태 변경
- **필터:**
  - **이전 상태:** 실패
  - **변경 후 상태:** 재개

**결과:** 워크플로우가 리드를 영업사원에게 배정하고 우선순위 상태를 업데이트합니다.

### **마케팅 캠페인 성과 추적**

**시나리오:** 특정 캠페인의 기회가 "진행 중"에서 "성사"로 이동합니다.

**트리거 설정:**
- **트리거:** 기회 상태 변경
- **필터:**
  - **태그:** 캠페인-X
  - **변경 후 상태:** 성사

**결과:** 워크플로우가 캠페인 성과 리포트를 생성하고 마케팅팀에 알립니다.

## **자주 묻는 질문**

**같은 트리거에 여러 필터를 사용할 수 있나요?**

네, "태그"와 **"파이프라인"** 같은 필터를 조합하여 특정 조건에서만 활성화되는 정확한 워크플로우를 만들 수 있습니다.

**기회 상태가 연속으로 여러 번 변경되면 어떻게 되나요?**

트리거는 상태가 변경될 때마다 활성화되므로, 중복 액션을 피하기 위해 워크플로우에 조건이나 쿨다운 기간을 사용하는 것이 중요합니다.

**여러 파이프라인에서 액션을 트리거할 수 있나요?**

네, 서로 다른 파이프라인에 대해 별도의 워크플로우를 설정하거나 필터를 사용하여 필요에 따라 특정 파이프라인에 트리거를 적용할 수 있습니다.

**태그가 없는 기회는 어떻게 처리하나요?**

**"태그 없음(Doesn't Have Tag)"** 필터를 사용하여 특정 태그가 없는 기회를 대상으로 하면, 태그가 없는 기회에만 워크플로우가 활성화됩니다.

**기회가 실패한 이유를 추적할 수 있나요?**

네, **"실패 사유(Lost Reason)"** 필터를 사용하여 기회가 실패한 이유를 식별하고, 피드백 설문 발송이나 리드 프로필 업데이트 등 해당 사유에 대응하는 워크플로우를 트리거할 수 있습니다.
