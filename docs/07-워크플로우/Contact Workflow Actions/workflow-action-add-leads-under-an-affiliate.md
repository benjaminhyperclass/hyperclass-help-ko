# 워크플로우 액션 - 제휴사 하위 리드 추가

**제휴사 하위 리드 추가(Add Leads Under an Affiliate)** 워크플로우 액션을 사용하면 선택한 캠페인 내에서 리드를 특정 제휴사에 자동으로 배정할 수 있습니다. 이 기능은 리드 관리를 간소화하고, 정확한 제휴사 크레딧을 보장하며, 리드를 적절한 제휴 파트너와 연결하여 참여도를 높이는 데 도움이 됩니다. 시간을 절약하고 수동 작업을 줄일 수 있어요.

# **제휴사 하위 리드 추가 액션이란?**

이 워크플로우 액션은 리드 생성 프로세스를 제휴 관리 시스템과 직접 연결합니다. 트리거가 발동되면 새 리드를 선택된 제휴사와 캠페인 하위에 자동으로 배정하여, 모든 연락처가 성과 보고서, 수수료 또는 후속 조치를 위해 적절히 추적되고 귀속되도록 보장합니다.

이 액션은 제휴사 배정 관리 방식에 따라 수동 또는 자동으로 사용할 수 있어요.

---

## **제휴사 하위 리드 추가 액션의 주요 장점**

이 기능이 제휴사 워크플로우를 어떻게 개선하는지 이해하면 자동화 가능성을 극대화하고 인적 오류를 최소화할 수 있습니다. 이 액션을 구현할 때의 주요 장점은 다음과 같아요.

- **자동화:** 수동 입력 없이 리드를 올바른 제휴사에 자동 배정합니다.

- **정확성:** 캠페인 데이터와 직접 동기화하여 제휴사 추적 오류를 방지합니다.

- **유연성:** 수동(Manual), 어트리뷰션 자동 추적(Auto-Track via Attribution), 커스텀 매핑(Custom Mapping) 등 다양한 배정 방법을 선택할 수 있습니다.

- **연동:** 워크플로우 자동화, 폼, 제휴사 추적 링크를 원활하게 연결합니다.

- **확장성:** 리드 라우팅을 자동화하여 대규모 제휴사 네트워크를 효율적으로 처리합니다.

# 워크플로우 액션 설정 방법

**제휴사 하위 리드 추가** 워크플로우 액션을 설정하면 리드가 수집되는 방식에 따라 캠페인의 제휴사에게 리드를 자동 또는 수동으로 배정할 수 있습니다.

다음 단계에서는 워크플로우 트리거 설정, 액션 추가, 설정 방법 선택, 워크플로우 발행까지의 과정을 안내합니다.

### **워크플로우 빌더 접근**

왼쪽 메뉴의 **Automation(자동화)** 섹션으로 이동한 다음 **Workflows(워크플로우)**를 선택하세요. **워크플로우 목록**에서 오른쪽 상단의 **Create Workflow(워크플로우 생성)**을 클릭합니다.

워크플로우 필요에 따라 레시피, 템플릿 또는 처음부터 시작하도록 선택할 수 있어요. 이 단계는 워크플로우 빌더를 열어서 자동화 구조 설정을 시작할 수 있게 합니다.

![워크플로우 빌더 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057427792/original/tv22d0uAxG_kXkldg7Sw-mNcsb_iwTK37w.png?1762167673)

### 워크플로우 트리거 추가

제휴사 액션을 추가하기 전에 워크플로우에 **트리거**가 있는지 확인하세요. 트리거는 리드가 언제 제휴사 하위에 추가될지를 결정합니다.

트리거가 없으면 리드가 워크플로우에 진입하지 않고 액션이 실행되지 않습니다.

일반적인 트리거는 다음과 같아요:

- **Form Submitted(폼 제출):** 연락처가 특정 폼을 완성할 때
- **Tag Added(태그 추가):** 연락처에 태그가 적용될 때

### **워크플로우 액션 추가**

트리거를 설정한 후 워크플로우에 **제휴사 하위 리드 추가** 액션을 추가하세요.

**+ Add(추가)** 버튼을 클릭한 다음 드롭다운 메뉴에서 **Add Action(액션 추가)**를 선택합니다.

이렇게 하면 자동화 설정에 적합한 액션을 선택할 수 있는 사용 가능한 워크플로우 액션 목록이 열립니다. 이 단계에서 제휴사 리드 배정을 처리할 새 액션 블록을 추가합니다.

![워크플로우 액션 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057426145/original/OIZKKiCio23PynVlAPspLfNfHkzl6a87Og.jpeg?1762166888)

### 액션 유형 선택

액션 메뉴가 열리면 **Actions(액션)** 하위의 **Affiliate(제휴사)** 섹션으로 스크롤하여 **Add Leads Under an Affiliate(제휴사 하위 리드 추가)**를 선택하세요.

이렇게 하면 워크플로우가 제휴 관리(Affiliate Manager) 시스템에 연결되어 특정 제휴사 캠페인과 제휴사 하위에 리드를 추가할 수 있습니다.

이 액션을 선택하면 워크플로우 빌더에 나타나며 설정할 준비가 완료됩니다.

![액션 유형 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057426311/original/Jw2a3lv7PbtA1rd8EBTsEUo8SlX60gle3w.jpeg?1762166953)

### 수동 배정 선택

리드를 특정 제휴사와 캠페인에 직접 배정하려면 **수동 배정(Manual Assignment)**을 선택하세요.

이는 고정 리드 라우팅, 내부 워크플로우 또는 제휴사 배정이 미리 정해진 경우에 이상적입니다.

이 설정에서:

- Choose Action Mapping(액션 매핑 선택) 하위에서 **Manual(수동)**을 선택합니다.
- 드롭다운 목록에서 제휴사 **Campaign(캠페인)**을 선택합니다.
- 캠페인이 선택되면 시스템에서 해당 캠페인에 연결된 제휴사가 표시됩니다.
- 목록에서 원하는 **Affiliate(제휴사)**를 선택합니다.
- 설정을 저장합니다.

이렇게 하면 이 워크플로우에 진입하는 모든 새 리드가 자동으로 선택한 제휴사에 배정됩니다.

![수동 배정 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057426391/original/LZ5YcEpMsdXR_uen_vf_qQ3GPNWvF3Vb5Q.jpeg?1762166995)

### 어트리뷰션을 통한 자동 추적 설정

**어트리뷰션을 통한 제휴사 자동 추적(Auto-Track Affiliate via Attribution)** 방법은 제휴사 ID(am_id)가 포함된 추적 링크를 기반으로 리드를 제휴사에 자동 배정합니다.

이는 제휴사 링크를 사용하여 트래픽을 유도하는 자동화된 설정에 가장 적합해요.

작동 방식:

- *Choose Action Mapping(액션 매핑 선택)* 하위에서 **Auto-Track Affiliate via Attribution(어트리뷰션을 통한 제휴사 자동 추적)**을 선택합니다.

- **Attribution Source(어트리뷰션 소스)**를 선택합니다:
  - **First Attribution(첫 번째 어트리뷰션):** 연락처를 추천한 첫 번째 제휴사에게 크레딧을 부여합니다.
  - **Latest Attribution(최근 어트리뷰션):** 가장 최근의 제휴사 상호작용에 크레딧을 부여합니다.

- 설정을 저장합니다.

제휴사 URL(예: https://yourdomain.com/?am_id=1234)을 통해 리드가 유입되면, HighLevel이 해당 ID를 캡처하여 자동으로 올바른 제휴사에게 크레딧을 부여합니다.

![어트리뷰션을 통한 자동 추적](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057426453/original/zpQh8LCaLbSn9UNAqQZFcTUhODFd671Dlw.jpeg?1762167017)

### 커스텀 매핑 설정

**커스텀 매핑(Custom Mapping)**을 사용하여 커스텀 필드나 워크플로우 변수에 저장된 데이터를 통해 제휴사를 동적으로 배정하세요.

이 방법은 추적 링크가 아닌 제휴사 데이터가 동적으로 전달되는 API 연동이나 외부 시스템에 이상적입니다.

설정 방법:

- *Choose Action Mapping(액션 매핑 선택)* 하위에서 **Custom Mapping(커스텀 매핑)**을 선택합니다.

- 매핑 소스를 입력합니다:
  - **커스텀 필드** 사용 (예: {{contact.custom_field.am_id}})
  - 또는 **워크플로우 변수** 사용 (예: {{workflow.am_id}})

- 시스템이 제휴사 ID를 올바른 제휴사 레코드와 매칭하여 자동으로 리드를 배정합니다.

이 설정은 고급 라우팅이나 통합된 제휴사 데이터 흐름에 최대한의 유연성을 제공합니다.

![커스텀 매핑 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057426494/original/hc3hXwhgX4T05iQnW_NmKWNptjBvxE8tKQ.jpeg?1762167037)

### 워크플로우 테스트 및 발행

활성화하기 전에 설정을 테스트하여 제휴사가 올바르게 배정되는지 확인하세요. 테스트를 통해 트리거, 캠페인 선택, 매핑 로직, 어트리뷰션이 모두 의도한 대로 작동하는지 확인할 수 있습니다.

완료 단계:

- 오른쪽 상단의 **Test Workflow(워크플로우 테스트)**를 클릭하여 리드 진입을 시뮬레이션합니다.

- **Execution Logs(실행 로그)**를 검토하여 제휴사 배정이 성공적으로 이루어졌는지 확인합니다.

- 확인되면 **Draft(초안)**에서 **Publish(발행)**으로 토글하여 워크플로우를 활성화합니다.

이제 워크플로우가 실행되며 선택한 설정 방법을 사용하여 제휴사에게 리드를 자동으로 배정할 준비가 완료되었습니다.

## **자주 묻는 질문**

**Q: am_id에 일치하는 제휴사가 없으면 어떻게 되나요?**

워크플로우는 리드를 배정하지 않고 완료됩니다. am_id가 캠페인의 기존 제휴사와 일치하는지 확인해보세요.

**Q: 자동으로 배정된 제휴사를 수동으로 변경할 수 있나요?**

네. 수정이 필요한 경우 제휴 관리(Affiliate Manager) 내에서 제휴사를 수동으로 재배정할 수 있습니다.

**Q: 이 액션이 작동하려면 판매나 보상이 필요한가요?**

아니요. 보상이나 금액이 관련되지 않아도 제휴사 하위에 리드를 배정할 수 있습니다. 보상이나 수수료는 제휴사 링크를 통해 적격한 구매나 전환이 이루어질 때만 적용됩니다.

**Q: "첫 번째 어트리뷰션"과 "최근 어트리뷰션"의 차이점은 무엇인가요?**

"첫 번째 어트리뷰션"은 초기 추천자에게 크레딧을 부여하고, "최근 어트리뷰션"은 전환 전 가장 최근의 추천자에게 크레딧을 부여합니다.

**Q: 한 리드에 여러 제휴사를 배정할 수 있나요?**

아니요. 정확한 추적을 위해 리드는 캠페인당 하나의 제휴사에만 배정될 수 있습니다.

**Q: 활성화하기 전에 이 워크플로우를 테스트할 수 있나요?**

네. 발행하기 전에 샘플 데이터로 **Test Workflow(워크플로우 테스트)** 기능을 사용하여 올바른 배정을 확인할 수 있습니다.
