## 연락처 필드 업데이트 액션이란?

'연락처 필드 업데이트' 액션을 사용하면 연락처 레코드의 특정 필드를 자동으로 수정할 수 있습니다. 주소 변경, 이메일 수정, 수신거부(DND) 상태 변경 등 연락처 정보의 변화를 시스템에 즉시 반영합니다. 이 액션을 통해 연락처 데이터를 최신 상태로 유지하고, 정확하며 고객 상호작용과 일치하는 정보를 관리할 수 있습니다.

## 이 액션 사용의 주요 장점

- **정확한 연락처 데이터**: 연락처 데이터베이스를 항상 최신 상태로 유지하여 오래되거나 잘못된 정보의 위험을 줄입니다.
- **향상된 개인화**: 업데이트된 연락처 정보를 활용해 더욱 개인화되고 타겟팅된 커뮤니케이션을 제공합니다.
- **워크플로우 효율성 개선**: 수동 개입 없이 자동으로 연락처 레코드를 업데이트하여 시간과 노력을 절약합니다.
- **고객 경험 간소화**: 정확한 연락처 정보를 유지하여 관련성 높고 적시의 커뮤니케이션을 제공합니다.

## 연락처 필드 업데이트 액션 설정: 단계별 과정

CRM의 "Automation(자동화)" 섹션으로 이동합니다. 상단에 있는 "Workflows(워크플로우)" 메뉴를 클릭하세요.
![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038202189/original/yBmOc0UE7LlMJuFPwspRPSRxnznpty2Erg.png?1733994774)

워크플로우 편집기를 열고 "연락처 필드 업데이트" 액션을 추가하려는 워크플로우를 선택합니다. 또는 처음부터 새 워크플로우를 만들 수도 있습니다. "+ Create Workflow(워크플로우 생성)" 버튼을 클릭하고 드롭다운 메뉴에서 "+ Start from scratch(처음부터 시작)" 옵션을 선택하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038202228/original/Ynsn0vA1vlsO2t5dPicDrR7WQwHTPr2CiQ.png?1733994813)

**"+"** 아이콘을 클릭하고 사용 가능한 액션 목록에서 **"Update Contact Field(연락처 필드 업데이트)"**를 선택합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038222515/original/OC089a5ZRBwnfvMciICewJWGcYylqc9x5g.png?1734007769)

액션에 명확하고 설명적인 이름을 지정하세요(예: "연락처 주소 업데이트"). 이렇게 하면 워크플로우에서 식별하기 쉬워집니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038224174/original/BbSBZ3BEifxBP3I1f6lAv-IKwU_ns-fwXw.png?1734008745)

액션 유형은 1. 기존 정보 지우기와 2. 정보 업데이트 두 가지 기능을 제공합니다. 실제 상황에서는 워크플로우를 설정하여 먼저 연락처 레코드의 기존 정보를 지운 다음 새 정보로 교체할 수 있습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038224330/original/et1pi77XvigbbIxBJubSMJwe8PhTzdpLiQ.png?1734008828)

"Add field(필드 추가)" 버튼을 클릭하여 추가 필터를 추가합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038224220/original/KgJcvLBHb5uIsHv9mk0C3e08IY2MS8mL9g.png?1734008758)

시/도, 국가, 전화번호, 커스텀 필드(Custom Fields) 등 업데이트하려는 필드를 선택하세요.

필터 추가를 완료한 후 액션을 저장하고 샘플 연락처를 사용해 테스트하여 필드가 예상대로 업데이트되는지 확인하세요.

## 활용 사례

1. **고객 이사 후 연락처 위치 업데이트**

- 시나리오: 고객이 새 도시로 이사하여 폼이나 고객지원을 통해 주소를 업데이트했습니다.
- 해결책: 연락처 레코드의 시/도 및 국가 필드를 자동으로 업데이트합니다.
- 단계:
  - 트리거: 고객이 폼을 통해 주소를 업데이트
  - 액션: "연락처 필드 업데이트" 액션을 사용하여 시/도를 "서울"로, 국가를 "한국"으로 변경
  - 결과: CRM이 업데이트된 위치를 반영하고, 고객 커뮤니케이션이 새 위치에 맞춰 조정됩니다.

2. **수신거부(DND) 상태 업데이트**

- 시나리오: 고객이 마케팅 커뮤니케이션을 수신거부했습니다.
- 해결책: 연락처 레코드의 DND 필드를 "예"로 업데이트하여 향후 마케팅 이메일에서 제외합니다.
- 단계:
  - 트리거: 고객이 이메일의 "수신거부" 링크를 클릭
  - 액션: "연락처 필드 업데이트" 액션을 사용하여 DND = 예로 설정
  - 결과: 연락처가 더 이상 마케팅 메시지를 받지 않아 이메일 수신 설정을 준수합니다.

3. **신규 리드의 연락처 출처 업데이트**

- 시나리오: 새 마케팅 캠페인에서 리드를 획득했고, 그 출처를 추적하고 싶습니다.
- 해결책: 연락처 출처 필드를 "봄 세일 캠페인"으로 업데이트하여 리드 출처를 추적합니다.
- 단계:
  - 트리거: 봄 세일을 위한 리드가 폼을 제출
  - 액션: "연락처 필드 업데이트" 액션을 사용하여 연락처 출처 = 봄 세일 캠페인으로 설정
  - 결과: 시스템이 신규 리드의 출처를 추적하여 더 나은 리포팅과 캠페인 분석이 가능합니다.

## FAQ

1. **하나의 액션으로 여러 필드를 동시에 업데이트할 수 있나요?**

네, 동일한 액션 내에서 여러 필드를 선택하고 동시에 업데이트할 수 있어 시간을 절약하고 복잡성을 줄일 수 있습니다.

2. **연락처에 존재하지 않는 필드를 업데이트하려고 하면 어떻게 되나요?**

필드가 존재하지 않으면 업데이트가 실패하거나 무시될 수 있습니다. 이를 방지하려면 연락처 레코드나 CRM 시스템에서 필드가 제대로 생성되었는지 확인하세요.

3. **"연락처 필드 업데이트" 액션이 작동했는지 어떻게 알 수 있나요?**

샘플 연락처를 사용해 워크플로우를 테스트하고 연락처 레코드에 변경사항이 반영되었는지 확인하세요. 일부 플랫폼은 업데이트 추적을 위한 활동 로그도 제공합니다.

4. **잘못 업데이트된 경우 되돌릴 수 있나요?**

아니오, 필드가 업데이트되면 자동으로 되돌릴 수 없습니다. 수정하려면 올바른 정보로 필드를 다시 업데이트해야 합니다.
