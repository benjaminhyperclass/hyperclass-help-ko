# 담당자 제거란 무엇인가요?

담당자 제거 액션은 연락처에서 CRM 사용자 할당을 해제하여, 연락처를 담당자 없는 상태로 만드는 기능이에요. 이 액션은 연락처의 상태나 워크플로우 조건이 변경되어 더 이상 직접적인 사용자 할당이 필요하지 않을 때 특히 유용해요. 담당자를 제거함으로써 연락처가 다른 워크플로우에 다시 들어가거나 향후 처리를 위해 미할당 상태로 유지될 수 있어요.

---

## 이 워크플로우 액션의 주요 이점

- **연락처 할당 초기화:** 사용자 할당을 지워서 연락처를 재참여나 재할당을 위해 초기화해요.

- **자동화된 워크플로우 관리:** 특정 트리거(할 일 완료, 상태 변경 등)가 발생했을 때 사용자 할당을 자동으로 제거하여 수동 개입을 없애요.

- **사용자 가용성 향상:** 더 이상 직접적인 관심이 필요하지 않은 연락처를 제거하여 사용자를 자유롭게 해요.

- **워크플로우 유연성 강화:** 충돌하거나 오래된 할당 없이 연락처가 워크플로우 간을 이동할 수 있게 해요.

## 액션 설정: 단계별 과정

### 액션 추가

워크플로우 빌더에서 **"+"** 아이콘을 클릭하여 새 액션을 추가하세요.

### 액션 선택

사용 가능한 옵션에서 **"Remove Assigned User(담당자 제거)"** 액션을 선택하세요.

![액션 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038521354/original/nPGAcmAlHVvuDCLwpW49olfUUYCX2AjH-w.jpeg?1734503829)

### 액션 이름 설정

워크플로우를 체계적으로 관리하기 위해 "사용자 할당 초기화" 또는 "담당자 지우기"와 같은 설명적인 이름을 제공하세요.

![액션 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038521793/original/la0nMoZj9cn8Bc-oDlyggFkZIs951sm1og.png?1734504467)

### 액션 저장

추가로 설정할 필드가 없으므로 이름을 설정한 후 액션을 **"저장"**하세요.

### 테스트 및 발행

워크플로우가 의도한 대로 할당을 제거하는지 테스트하고, **"발행"** 토글을 활성화하여 워크플로우를 활성화하세요.

## 사용 사례

### 영업 파이프라인 정리

**시나리오:** 연락처의 기회 상태가 "성사(Closed Won)" 또는 "실패(Closed Lost)"로 변경되었어요. 영업 팀은 더 이상 후속 조치가 필요하지 않고, 연락처는 다른 워크플로우를 위해 초기화되어야 해요.

**해결책:**

- **트리거:** 기회 상태 변경
- **액션:** 담당자 제거
- **결과:** 연락처가 영업사원에게서 해제되어, 영업사원이 활성 기회에 집중할 수 있게 돼요.

### 고객 지원 할 일 완료

**시나리오:** 지원 담당자가 티켓 해결과 같은 연락처 관련 할 일을 완료했어요. 할 일이 완료되면 연락처는 더 이상 직접 할당이 필요하지 않아요.

**해결책:**

- **트리거:** 할 일 완료
- **액션:** 담당자 제거
- **결과:** 담당자가 해제되어 새로운 지원 티켓을 효율적으로 처리할 수 있게 돼요.

## 자주 묻는 질문

**담당자가 제거된 후 연락처는 어떻게 되나요?**

담당자가 제거되면 연락처는 미할당 상태가 되고 활성 사용자 할당 없이 시스템에 남아있어요. 그 후 다른 워크플로우나 트리거를 통해 다시 할당될 수 있어요.

**이 액션을 하나의 워크플로우에서 여러 트리거와 결합할 수 있나요?**

네, 담당자 제거 액션을 "할 일 완료" 및 "기회 상태 변경"과 같은 여러 트리거와 결합하여 연락처 할당을 동적으로 관리하는 복잡한 워크플로우를 만들 수 있어요.

**이 액션이 할당 히스토리를 제거하나요?**

아니요, 담당자를 제거해도 연락처의 할당 히스토리가 지워지지 않아요. 활동 로그나 보고 도구를 통해 이전 사용자 할당을 여전히 추적할 수 있어요.

**이 액션이 다른 워크플로우를 방해하지 않도록 하려면 어떻게 해야 하나요?**

충돌을 방지하려면 발행 전에 워크플로우를 테스트하고 모든 트리거의 조건을 주의 깊게 검토하세요. 담당자 제거 액션이 필요한 곳에만 적용되도록 하세요.

**담당자 제거 액션을 사용한 후 즉시 연락처를 다시 할당할 수 있나요?**

네, 현재 할당을 제거한 후 워크플로우에 후속 액션을 추가하여 "사용자에게 할당(Assign to User)" 액션을 사용해 연락처를 새로운 사용자에게 재할당할 수 있어요.
