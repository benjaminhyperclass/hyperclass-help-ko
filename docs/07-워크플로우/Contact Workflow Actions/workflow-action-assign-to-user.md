# 사용자에게 할당 워크플로우 액션이란

"사용자에게 할당(Assign To User)" 워크플로우 액션을 사용하면 미리 정의한 조건에 따라 연락처를 한 명 또는 여러 팀원(사용자)에게 자동으로 할당할 수 있습니다. 단일 사용자 할당과 라운드 로빈이나 커스텀 트래픽 분배를 사용한 다중 사용자 분배를 모두 지원합니다. 이 액션은 균등한 업무 부하 분산, 추적 개선, 원활한 팀 협업을 가능하게 합니다. 리드, 티켓, 할 일 등 어떤 것을 할당하든, 이 기능은 여러분의 특정 워크플로우 요구사항에 맞게 조정될 수 있도록 설계되었습니다.

---

## 사용자에게 할당 액션의 주요 장점

- **자동 할당:** 수동으로 연락처를 할당할 필요가 없어 시간과 노력을 절약합니다.

- **공정한 분배:** 라운드 로빈이나 균등 분배 방식을 사용하여 팀원들 간의 업무 부하를 균형있게 배분합니다.

- **커스텀 배분:** 팀 역량이나 경력에 따라 연락처를 불균등하게 할당하여 효율성을 극대화할 수 있습니다.

- **추적 개선:** 각 연락처에 대한 책임자를 명확히 정의하여 책임감을 보장합니다.

- **유연한 워크플로우:** 영업 리드 관리부터 고객 지원까지 다양한 용도로 이 기능을 활용할 수 있습니다.

---

워크플로우 빌더를 열고, "+" 아이콘을 클릭하여 새 액션을 추가한 다음, "Assign To User(사용자에게 할당)"를 선택합니다.

## 사용자 할당 워크플로우 액션 설정

### 액션 이름

워크플로우에서 쉽게 식별할 수 있도록 액션에 설명적인 이름을 지정합니다(예: "캘리포니아 에이전트에게 할당").

![액션 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038398659/original/xQ_gqdcSxRKlGxIt7ft6EL4Ycf6Cc-6AEg.png?1734351363)

### 사용자 선택

드롭다운 메뉴를 사용하여 작업을 할당받을 한 명 또는 여러 명의 사용자를 선택합니다.

- **단일 사용자:** 모든 연락처가 한 사용자에게 할당됩니다.

- **다중 사용자:** 두 명 이상의 사용자를 선택하면, 트리거가 각 할당마다 지정된 사용자들 사이에서 자동으로 "라운드 로빈" 방식으로 순환합니다. 예를 들어 Tom과 Jane을 선택했다면, 첫 번째 트리거는 연락처를 Tom에게 할당하고, 두 번째 트리거는 Jane에게, 세 번째는 다시 Tom에게 할당하며, 이런 식으로 모든 연락처에 사용자가 할당될 때까지 계속됩니다.

![사용자 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038398702/original/fZUNbW2D6puroXT_BlT32tUUXppzwRWMFw.png?1734351410)

## 사용자별 할당 분배 (가중 라운드 로빈)

사용자별 연락처 할당 분배는 미리 정의된 규칙에 따라 선택된 사용자들 사이에 작업, 연락처, 또는 리드를 균등하게 또는 불균등하게 분배합니다.

### 균등 분배

트래픽 분배 옵션에서 **"Equally(균등)"**를 선택하면, 라운드 로빈 할당 방식과 유사하게 모든 사용자에게 트래픽이 균등하게 분배됩니다(예: 1:1, 2:2 등). 이는 불균등 옵션으로 전환하기 전까지 기본으로 활성화되어 있습니다.

![균등 분배](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038398768/original/wB6JjvQcN9Fqdrfc0K-JF1F1hLloQeZjgg.png?1734351458)

### 불균등 분배

**"Unevenly(불균등)"** 옵션을 선택하면 선택된 각 사용자에 대해 트래픽 분배 비율을 커스터마이징할 수 있습니다. 예를 들어 5:2 비율을 지정하면, 사용자 A가 5개의 트래픽을 받고 사용자 B가 2개의 트래픽을 받습니다. 따라서 사용자 B는 사용자 A가 5번째를 완료한 후에만 트래픽을 받게 됩니다.

트래픽 분배가 불균등이고 세 명의 사용자에게 다음과 같은 가중치가 있다면:

- Alan: 3
- Bobby: 3
- Charlie: 1

워크플로우 트리거 1, 2, 3은 Alan에게 할당되고, 트리거 4, 5, 6은 Bobby에게 할당되며, 트리거 7은 Charlie에게 할당된 후 사이클이 반복됩니다.

![불균등 분배](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038398834/original/Q8Oo5U7LuQNaR6py2qQRg7kUc0Ld9twuyQ.png?1734351507)

## 미할당 연락처에만 적용

"Only Apply To Unassigned Contacts(미할당 연락처에만 적용)" 토글이 꺼져 있고, 이미 할당된 연락처에 대해 트리거가 실행되면, 새 작업이 이전 작업을 덮어쓰고 해당 연락처가 새 사용자에게 할당됩니다.

- 활성화: 아직 할당되지 않은 연락처만 할당합니다.
- 비활성화: 이미 할당된 연락처도 재할당합니다.

![미할당 연락처 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155038398928/original/0BznyGYpQH0LmWFOcLecTlnAff6YSmzfIA.png?1734351559)

## 표준 vs 동적 사용자 ID

할당에 사용할 한 명 이상의 사용자를 선택하는 사용자 드롭다운에서는 표준(Standard)이 기본으로 선택되어 있으며, 하위 계정의 모든 사용자가 포함된 드롭다운 메뉴를 보여줍니다. 동적(Dynamic)을 선택하면 필드가 일반 텍스트 필드로 변경되어 사용자 ID를 직접 입력하거나 병합 태그를 통해 입력할 수 있습니다.

동적 보기에서는 하나 이상의 사용자 ID를 수동으로 입력할 수 있습니다. 하지만 일반적인 사용 방법은 워크플로우 앞 단계에서 사용자 ID가 포함된 변수를 선택하는 것입니다.

![표준 vs 동적 사용자 ID](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049113485/original/EoPnQm4e7pZUEblOdG_NAoJWRhqDmTmrOA.png?1751322196)

## 동적으로 사용자 할당하는 방법

대부분의 경우 사용자 할당 로직이 간단하거나 Hyperclass 워크플로우 내부에서 정의되기 때문에 표준 선택기를 사용해야 합니다. 하지만 일부 고급 사용자들은 Hyperclass 외부에서 실행되는 더 **복잡한** 사용자 할당 로직을 가지고 있습니다.

고급 사용자 할당 사례 예시:

여러 계정 관리자가 다양한 고객 등급을 담당하고, 교육을 받으며, 때로는 서로를 대신하는 디지털 마케팅 에이전시.

새 리드 도착 → 외부 시스템에서 다음을 평가:
- 리드 소스 (Google 광고, 추천, 콜드 아웃리치)
- 회사 규모 (직원 수, 매출)
- 서비스 관심사 (SEO, PPC, 소셜 미디어)
- 지역

외부 시스템의 스마트 할당:
- 기업 리드 → 시니어 계정 관리자 (사용자 ID #1 또는 #2)
- 교육 중인 기업 리드 → 일반 계정 관리자 (사용자 ID #3)
- 중간 시장 리드 → 일반 계정 관리자 (사용자 ID #3, #4, 또는 #5)
- 소규모 사업체 리드 → 주니어 계정 관리자 (사용자 ID #6, #7, #8, 또는 #9)
- 소규모 사업체 리드 오버플로 → 일반 계정 관리자 (사용자 ID #4 또는 #5)

워크플로우가 인바운드 웹훅을 받아 연락처에 사용자 ID를 할당합니다.

**1단계**: 워크플로우 생성

새 워크플로우를 생성하고 인바운드 웹훅 트리거(Inbound Webhook Trigger), 연락처 찾기 액션(Find Contact Action), 사용자 할당 액션(Assign User Action)을 추가합니다.

**2단계**: 설정 웹훅 보내기

제공된 URL에 예시를 보내도록 선택한 시스템을 사용하여 인바운드 웹훅을 설정합니다. 이 경우 빠른 무료 테스트를 위해 [https://reqbin.com/](https://reqbin.com/)을 사용했습니다. user_id 매개변수와 contact_email 매개변수를 설정합니다. "status: success: test request received"가 표시되면 인바운드 웹훅 트리거로 돌아갈 수 있습니다.

![웹훅 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050404924/original/CxMP-vX-_jp_YcLOXeZ69gOIsuuAgcRZFw.png?1753402439)

**3단계**: 인바운드 웹훅 트리거 구성

인바운드 웹훅 트리거로 돌아가서 성공한 요청을 찾고, 매핑을 확인한 후 트리거를 저장합니다. 이제 인바운드 웹훅 트리거와 그 쿼리 매개변수들이 나중에 워크플로우에서 사용할 수 있게 됩니다.

![웹훅 트리거 구성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050404909/original/ycVgw3eqg96m7UHhv4aBW2D09n7nWGmo2g.png?1753402333)

**4단계**: 연락처 찾기 액션 구성

사용자 > 이메일 필드를 선택하고 인바운드 웹훅 > QueryParams > contact_email 필드로 매핑하여 연락처 찾기 액션을 편집합니다.

![연락처 찾기 액션 구성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050404981/original/4pyUW9uHJITGo40ep0Ls_8zwP6Q0CpJFkQ.png?1753402743)

**5단계**: 사용자 할당 액션 구성

표준에서 동적으로 전환하여 사용자 할당 액션을 편집합니다. 그 다음 병합 필드 선택기에서 인바운드 웹훅 트리거 > QueryParams > user_id 필드를 선택합니다.

![사용자 할당 액션 구성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050404998/original/5xdpsodLHZeXlytfNQF5K1EUs9AQrQCnqA.png?1753402894)

**6단계**: 테스트 및 저장

다양한 조건에서 워크플로우를 테스트해야 합니다. 만족스러우면 반드시 저장하고, 실제로 운영할 준비가 되었을 때 발행합니다.

## 자주 묻는 질문

**워크플로우에서 "사용자에게 할당" 액션의 목적은 무엇인가요?**

"사용자에게 할당" 액션을 사용하면 연락처를 특정 팀원에게 자동으로 할당하여 효율적인 업무 부하 관리와 추적을 보장할 수 있습니다.

**"사용자에게 할당" 액션에서 한 명의 사용자만 선택하면 어떻게 되나요?**

한 명의 사용자를 선택하면 워크플로우에서 트리거되는 모든 연락처가 그 사용자에게 할당되며, 다른 사용자에게는 할당이 순환되지 않습니다.

**"미할당 연락처에만 적용" 토글을 켜지 않고도 "사용자에게 할당" 액션을 사용할 수 있나요?**

네, 가능하지만 워크플로우 트리거가 실행될 때 이전에 할당된 연락처가 새 사용자에게 재할당될 수 있습니다.

**라운드 로빈 할당과 균등 트래픽 분배의 차이점은 무엇인가요?**

라운드 로빈은 사용자들 사이에서 한 번에 하나씩 할당을 순환시킵니다. 균등 트래픽 분배는 시간이 지남에 따라 사용자들이 동일한 수의 할당을 받도록 보장합니다.
