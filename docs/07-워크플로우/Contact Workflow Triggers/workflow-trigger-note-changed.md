# 워크플로우 트리거 - 노트 변경

## 개요

**노트 변경(Note Changed)** 트리거는 연락처 레코드의 기존 노트가 수정될 때마다 워크플로우 내에서 특정 액션을 시작하도록 설계되었습니다. 중요한 노트의 업데이트를 추적하고, 팀원들에게 변경 사항을 알리며, 최신 커뮤니케이션 기록을 유지하는 데 유용합니다.

## 트리거 이름

**노트 변경(Note Changed)**

## 트리거 설명

**노트 변경(Note Changed)** 트리거는 연락처 프로필의 기존 노트가 수정될 때 워크플로우를 활성화합니다. 노트 내용 변경에 대한 응답으로 관련 팀원에게 알림을 보내거나, 연락처 세부 정보를 업데이트하거나, 업데이트된 정보를 기반으로 연락처에 태그를 추가하는 등의 액션을 실행할 수 있습니다.

## 설정 방법

**노트 변경(Note Changed)** 트리거를 설정하려면, 워크플로우 트리거 이름을 설정하고 사용 가능한 필터 중에서 선택하여 이 워크플로우가 활성화되는 조건을 세밀하게 조정하세요.

| 필드 | 설명 | 필수 여부 |
|------|------|-----------|
| Workflow Trigger Name | 쉬운 식별을 위해 트리거에 부여하는 이름 | 예 |
| Doesn't Have Tag | 연락처가 지정된 태그를 갖지 않은 경우에만 워크플로우가 트리거됨 | 아니오 |
| Has Tag | 연락처가 지정된 태그를 가진 경우에만 워크플로우가 트리거됨 | 아니오 |

## 예시

이 예시에서는 특정 태그를 가진 연락처의 기존 노트가 변경될 때 트리거되도록 워크플로우를 설정했습니다:

- **Workflow Trigger Name:** Note Changed
- **필터:**
  - **Has Tag:** Important Client

### 예시 워크플로우

연락처 레코드의 노트가 변경되면 다음과 같은 액션이 트리거될 수 있습니다:

- **계정 관리자에게 알림:** 배정된 계정 관리자에게 중요 고객 프로필의 노트에 대한 변경 사항을 검토하라는 내부 알림을 발송합니다.
- **할 일 업데이트:** 수정된 노트 내용을 바탕으로 팀원이 적절한 후속 조치를 취할 수 있도록 할 일을 자동으로 업데이트하거나 생성합니다.
- **변경 사항 기록:** 노트 변경을 특정 로그에 기록하거나 CRM을 업데이트하여 수정 내역을 확실히 남깁니다.

이 설정을 통해 노트가 변경될 때마다 워크플로우가 활성화되어, 노트 업데이트에 기반한 즉각적인 액션과 커뮤니케이션이 가능합니다.
