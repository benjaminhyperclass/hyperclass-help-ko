# **Contact Changed 워크플로우 트리거란?**

"Contact Changed(연락처 변경됨)" 워크플로우 트리거는 연락처의 세부 정보가 업데이트될 때마다 활성화됩니다. 예를 들어:

- **태그 변경**: "관심 고객" 같은 태그가 추가되면, 워크플로우가 후속 이메일을 전송할 수 있습니다.
- **DND 상태 변경**: 연락처가 "수신 거부"를 설정하면, 워크플로우가 메시지 전송을 중단할 수 있습니다.
- **배정 사용자 변경**: 연락처가 새 영업 담당자에게 배정되면, 워크플로우가 이를 알릴 수 있습니다.
- **커스텀 필드 업데이트**: "선호 서비스" 같은 필드가 업데이트되면, 워크플로우가 맞춤형 제안을 보낼 수 있습니다.

이 트리거는 비즈니스가 빠르게 대응하고 반복 작업을 자동화하여 모든 것을 체계적으로 관리하는 데 도움이 됩니다.

## Contact Changed 워크플로우 트리거를 사용하는 이유

"Contact Changed(연락처 변경됨)" 트리거는 연락처 세부 정보의 업데이트를 추적하고 즉시 반응하여 변경 사항을 놓치지 않게 해주는 똑똑한 도우미와 같습니다. 이메일 전송이나 후속 작업 배정 등의 작업을 자동으로 처리하여 시간을 절약합니다. 연락처가 더 빠르고 관련성 높은 응답을 받을 수 있도록 하여 고객 서비스를 개선합니다. 마지막으로, 업데이트된 비즈니스 정보에 대한 후속 조치 등 새 리드나 변경 사항에 대해 행동을 취할 수 있도록 하여 기회를 만들어줍니다.

## **Contact Changed 워크플로우 트리거 설정 방법**

워크플로우 트리거를 설정하는 간단한 단계를 따라해보세요:

### **새 워크플로우 시작**

CRM의 Automations(자동화) 섹션으로 이동합니다. Workflows(워크플로우)를 클릭합니다.

![자동화 섹션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042857643/original/aRL66-fTn8CSScdpzCGdMFVjJ_uoaRHBYg.jpeg?1741354411)

### **워크플로우 생성**

우상단의 "+ Create Workflow(+ 워크플로우 생성)" 버튼을 클릭한 다음 "+ Start from scratch(+ 처음부터 시작)"를 선택합니다.

![워크플로우 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042857750/original/g2e_R3_MlcvEjT8ChgPyZuFiEIiNckEaOw.png?1741354485)

### 트리거 추가

Add New Trigger(새 트리거 추가)를 클릭하고 "Contact Changed"를 검색합니다.

![트리거 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042857798/original/JpmfNH6kkyomwS3TN5QT2EEbgtxBVfXkew.jpeg?1741354510)

### 트리거 이름 지정

"웹사이트 주소 업데이트됨"과 같이 설명적인 이름을 지정합니다.

![트리거 이름](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042857847/original/odFFgG3SRJRDtrEiXYFTCjj1HmrI1IJ1MA.jpeg?1741354529)

### **필터 설정**

트리거를 활성화할 연락처 변경 유형을 결정합니다.

- "웹사이트"와 같은 특정 필드의 변경을 추적하려면 **Custom Field(커스텀 필드)**를 선택합니다.
- 추가되거나 제거된 태그에 워크플로우가 반응하도록 하려면 **Tags(태그)**를 선택합니다.
- "수신 거부" 설정 변경에 워크플로우가 응답하도록 하려면 **DND**를 사용합니다.

![필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042857884/original/L5u60_mR5N1XoLwaT7YnnGeNujUD82UYNQ.jpeg?1741354554)

## **자주 묻는 질문**

**Q. Contact Changed 워크플로우를 트리거할 수 있는 연락처 변경 유형은 무엇인가요?**

"Contact Changed(연락처 변경됨)" 트리거는 다음과 같은 다양한 연락처 업데이트를 기반으로 활성화될 수 있습니다:

- 태그 추가 또는 제거 – 예: 연락처에 "핫 리드" 태그가 지정되면 영업 후속 조치 트리거
- 연락처 유형 변경 – 예: 연락처가 "리드"에서 "고객"으로 변경되는 경우
- 커스텀 필드 업데이트 – 예: 연락처의 "선호 서비스"가 업데이트되면 맞춤형 프로모션 전송
- 배정 사용자 변경 – 예: 연락처가 새 영업 담당자에게 재배정되면 즉시 담당자에게 알림
- DND 상태 변경 – 예: 연락처가 "수신 거부"를 활성화하면 모든 발신 메시지 중단

**Q. 단일 워크플로우 내에서 여러 연락처 변경을 추적할 수 있나요?**

네! "Contact Changed(연락처 변경됨)" 트리거 아래에 여러 필터를 추가하여 다양한 유형의 변경을 동시에 추적할 수 있습니다. 예를 들어, 태그가 추가되거나 커스텀 필드가 업데이트될 때 트리거되도록 워크플로우를 구성하여 자동화가 여러 이벤트에 응답하도록 할 수 있습니다.

**Q. 연락처 정보가 여러 번 변경되면 어떻게 되나요? 매번 워크플로우가 트리거되나요?**

네, 추적된 변경이 발생할 때마다 워크플로우가 트리거됩니다. 하지만 불필요한 작업(예: 여러 이메일 전송)을 방지하기 위해 다음을 할 수 있습니다:

- 워크플로우 조건을 사용하여 작업이 이미 수행되었는지 확인
- 특정 변경에만 트리거되도록 필터 적용
- 과도한 자동화를 피하기 위해 시간 지연 추가

**Q. 삭제된 태그나 제거된 이전 값을 기반으로 액션을 트리거할 수 있나요?**

아니요, "Contact Changed(연락처 변경됨)" 트리거는 새 업데이트를 감지하지만 제거된 값은 추적하지 않습니다. 태그가 제거되는 것을 기반으로 액션을 자동화해야 하는 경우 다음을 고려해보세요:

- "If/Else(조건 분기)" 조건을 사용하여 연락처에 특정 태그가 여전히 있는지 확인
- 태그 제거를 수동으로 모니터링하고 그에 따라 다른 워크플로우 트리거
