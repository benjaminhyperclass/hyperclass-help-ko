# 워크플로우 목표 이벤트 설정 방법

워크플로우 내에서 특정 조건이나 목표 이벤트를 충족한 연락처가 이전 단계를 거치지 않고 워크플로우 내의 특정 단계로 자동으로 이동할 수 있다면 얼마나 좋을까요?

이제 워크플로우 목표 이벤트 액션(Workflow Goal Events Action)으로 이것이 가능합니다. 이 가이드에서는 목표 이벤트가 무엇인지, 그리고 언제 어떻게 사용하는지 알아보겠습니다.

참고사항:

버전 1에서는 목표 이벤트 필터를 이메일 이벤트, 태그 추가/제거, 트리거 링크 클릭, 예약 상태로만 제한했습니다. 향후 폼 제출 등 더 많은 목표 이벤트를 추가할 계획입니다. 또한 현재 워크플로우당 하나의 목표 이벤트만 제한하고 있습니다.

## 워크플로우에서 목표 이벤트 설정하는 방법

목표 이벤트(Goal Event) 워크플로우 액션을 선택합니다 > "이메일 이벤트 수신", "트리거 링크 클릭", "연락처 태그 추가", "연락처 태그 제거", "예약 상태" 중에서 선택합니다.

![목표 이벤트 설정하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000617849/original/Bk6vPFLzLVPdSIO57APfW-ekrtxgHpR-Pw.gif?1686138124)

## 목표 이벤트란 무엇인가요?

목표 이벤트는 워크플로우에 대한 연락처 목표를 설정할 수 있게 해줍니다. 연락처가 워크플로우에 진입하는 순간부터 시스템은 연락처가 어떤 단계에 있든 상관없이 지정된 목표 이벤트가 발생하는지 "감지"합니다.

## 목표 이벤트는 어떻게 작동하나요?

연락처가 액션에서 지정한 목표 조건을 만족하면, 워크플로우 내에서 현재 어떤 위치에 있든 상관없이 목표 단계로 이동됩니다.

현재 워크플로우 목표 액션에서는 이메일 이벤트, 태그, 트리거 링크 클릭이 지원됩니다. 더 많은 액션이 추가되는지 다시 확인해 주세요.

## 연락처가 조건을 만족하지 않고 목표 이벤트에 도달하면 어떻게 되나요?

자동화에서 연락처가 목표 액션에 도달했을 때 목표 이벤트가 달성되지 않았다면, 다음 3가지 옵션 중에서 선택할 수 있습니다:

1. 이 워크플로우 종료
2. 그대로 계속 진행
3. 목표가 달성될 때까지 대기

## 현재 지원되는 목표 이벤트
![지원되는 목표 이벤트 유형](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000617892/original/KHLM_BOkYEMtuk1PaNhw23uuZJZEBDmvKw.png?1686138149)

### 1. 이메일 이벤트 수신:

"이메일 이벤트 수신" 필터의 경우, 시스템이 인식하기를 원하는 이벤트를 선택해야 합니다. 여러 옵션이 있습니다:

- 클릭됨
- 열림
- 구독 취소 (Mailgun만 해당)
- 신고됨
- 스팸

![이메일 이벤트 설정하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000617981/original/kpPIV3ryxf4-F3YBRJonnKJX9Oa0-55qqQ.gif?1686138197)

#### 활용 사례:

- 참여 기반 마케팅: 이메일을 열거나 클릭할 때 참여도가 높은 사용자를 위해 설계된 워크플로우 단계로 자동 이동
- 드립 이메일 캠페인: 이전 이메일을 열어본 후에만 드립 시리즈의 다음 이메일로 연락처 이동
- 구독 취소 처리: 이메일 구독을 취소한 연락처에게 재참여 워크플로우 트리거
- 행동 기반 개인화: 연락처가 이메일에서 특정 유형의 콘텐츠(예: "IT 기기")를 더 자주 클릭하면, 관심사에 기반한 개인화된 콘텐츠를 보내는 워크플로우 트리거
- 이벤트 등록: 이메일로 이벤트를 홍보하는 경우, 연락처가 이벤트 초대장을 열었을 때 등록을 유도하는 후속 워크플로우 트리거
- 웨비나 후속 조치: 웨비나 재생 링크를 이메일로 보낸 후, 재생 링크가 포함된 이메일을 연락처가 열었을 때 후속 워크플로우 트리거
- 설문 참여: 이메일로 설문을 보내고 연락처가 설문 이메일을 열었을 때 참여 감사 메시지나 리워드를 보내는 워크플로우 트리거
- 이메일 강의 완료: 이메일 강의를 제공하는 경우, 연락처가 마지막 강의 이메일을 열었을 때 수료증이나 배지를 보내는 워크플로우 트리거
- 할인 쿠폰 사용: 할인 코드가 포함된 이메일을 보낸 경우, 연락처가 할인 이메일을 열었을 때 만료 전에 할인을 사용하라는 리마인드 워크플로우 트리거
- 영업 후속 조치: 영업팀의 경우, 제안서나 견적서를 이메일로 보낸 후 연락처가 이메일을 열었을 때 거래 진행을 위한 후속 워크플로우 트리거

#### 자주 묻는 질문:

**단일 이메일 이벤트가 여러 목표 이벤트를 만족할 수 있나요?**

네, 이메일 열기와 같은 단일 이메일 이벤트는 해당 특정 이벤트에 대해 목표가 설정되어 있다면 여러 목표 이벤트를 만족할 수 있습니다.

**근무 시간 외에 이메일 이벤트가 발생해도 목표 이벤트가 트리거되나요?**

네, 시스템은 24시간 내내 지정된 이메일 이벤트를 "감지"합니다. 근무 시간 외에도 목표 이벤트가 만족되면 연락처가 목표 단계로 이동됩니다.

**연락처가 이메일을 여러 번 열어도 매번 목표 이벤트가 트리거되나요?**

아니요, 연락처가 특정 이메일 이벤트 목표를 만족하면, 해당 워크플로우에서 동일한 목표 이벤트에 대해 다시 평가되지 않습니다.

**워크플로우의 일부가 아닌 이메일에 대해 이메일 이벤트 목표가 설정되면 어떻게 되나요?**

목표 이벤트는 연락처가 워크플로우에 있는 동안 목표와 연결된 특정 이메일을 열거나 클릭할 때만 트리거됩니다. 이메일이 워크플로우의 일부가 아니라면 목표 이벤트가 만족될 수 없습니다.

**연락처가 이메일을 열지 않거나 링크를 클릭하지 않을 때 목표 이벤트를 설정할 수 있나요?**

목표 이벤트는 현재 연락처가 특정 행동을 취했을 때의 긍정적 조건만 지원합니다. 이메일을 열지 않는 것과 같은 부정적 조건은 현재 목표 이벤트에서 지원되지 않습니다.

### 2. 트리거 링크 클릭:

"트리거 링크 클릭"의 경우, 드롭다운 메뉴에 나열된 트리거 링크를 선택해 주세요. 트리거 링크를 생성하지 않았다면, 먼저 생성한 후 진행해 주세요.

![트리거 링크 설정하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000618040/original/nM9BaE78J9oHahNEhB82wj3v10bKARZv2g.gif?1686138227)

#### 활용 사례:

- 웹사이트 참여: 웹사이트의 특정 링크를 클릭할 때 참여도가 높은 사용자를 위해 설계된 워크플로우 단계로 연락처 이동
- 설문 응답: 다른 설문 응답에 대해 다른 트리거 링크를 사용하여, 응답에 따라 맞춤형 워크플로우 단계로 연락처 이동
- 상품 관심: 특정 상품과 관련된 링크를 클릭할 때 해당 상품에 대한 관심을 기르도록 설계된 워크플로우로 연락처 이동
- 콘텐츠 마케팅: 블로그 게시글이나 가이드를 공유한 경우, 연락처가 콘텐츠 링크를 클릭했을 때 관심도를 측정하거나 피드백을 수집하는 후속 워크플로우 트리거
- 웨비나 가입: 웨비나를 홍보하는 경우, 연락처가 등록 링크를 클릭했을 때 후속 워크플로우 트리거
- 전자상거래 구매: 전자상거래 사업체의 경우, 연락처가 "지금 구매" 또는 "장바구니에 추가" 링크를 클릭했을 때 구매 후 워크플로우 트리거
- 멤버십 또는 구독 가입: 멤버십이나 구독을 홍보하는 경우, 연락처가 가입 링크를 클릭했을 때 환영 워크플로우 트리거
- 체험판 활성화: 무료 체험을 제공하는 소프트웨어 회사의 경우, 연락처가 "체험판 활성화" 링크를 클릭했을 때 온보딩 워크플로우 트리거
- 제휴 마케팅: 제휴 마케터인 경우, 연락처가 제휴 링크를 클릭했을 때 후속 워크플로우 트리거
- 다운로드 확인: 무료 자료 다운로드를 제공하는 경우, 연락처가 다운로드 링크를 클릭했을 때 추가 참여를 유도하는 후속 워크플로우 트리거

#### 자주 묻는 질문:

**연락처가 워크플로우에 진입하기 전에 트리거 링크를 클릭하면 목표 이벤트가 트리거되나요?**

아니요, 연락처가 워크플로우에 있는 동안 목표 조건을 만족해야 트리거 링크 클릭 목표 이벤트가 활성화됩니다.

**연락처가 트리거 링크를 여러 번 클릭하면 매번 목표 이벤트가 트리거되나요?**

아니요, 연락처가 특정 트리거 링크 클릭 목표 이벤트를 만족하면, 해당 워크플로우에서 동일한 목표 이벤트에 대해 다시 평가되지 않습니다.

**연락처가 트리거 링크를 클릭했지만 목표 이벤트의 다른 조건을 만족하지 않으면 어떻게 되나요?**

해당 목표 이벤트에 설정된 모든 조건을 만족하지 않는 한 연락처는 목표 이벤트 단계로 이동되지 않습니다.

**어떤 웹사이트의 트리거 링크든 트리거 링크 클릭 목표 이벤트에 사용할 수 있나요?**

시스템이 클릭을 감지하고 목표 이벤트를 트리거하려면, 트리거 링크는 이메일 마케팅이나 자동화 플랫폼에서 추적되는 링크여야 합니다.

**같은 워크플로우에서 동일한 트리거 링크를 다른 목표 이벤트에 사용할 수 있나요?**

네, 각 목표 이벤트가 고유한 조건을 가지고 있다면 동일한 트리거 링크를 다른 목표 이벤트에 사용할 수 있습니다.

### 3. 연락처 태그 (추가 & 제거):

- 연락처 태그 추가 — 지정된 태그가 추가되면 연락처가 목표 단계로 이동됩니다.

![태그 추가 설정하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000618117/original/OsdRwJ0gLdMJWnBSOHg1i7UlsnmtTJEX1g.gif?1686138277)

- 연락처 태그 제거 — 지정된 태그가 제거되면 연락처가 목표 단계로 이동됩니다.

![태그 제거 설정하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000618142/original/RiOgby_RfH9T4-uEqZiIPB7hgr_9PNU48A.gif?1686138299)

워크플로우에서 태그 목표 액션을 사용하는 예시:

상품을 판매하려고 하는데, 상품 구매가 완료되자마자 연락처에 "상품 구매됨" 태그를 추가하고 싶다고 가정해 보겠습니다.

한편, 연락처를 육성 워크플로우에 계속 두고 싶습니다. "상품 구매됨" 태그가 연락처에 추가되자마자 워크플로우를 중단하고 연락처를 제거하고 싶습니다.

시스템은 이미 다른 트리거와 워크플로우를 사용하여 이를 수행할 수 있는 기능을 갖추고 있었지만, 목표 이벤트 액션 > 태그 기능의 추가로 추가 워크플로우의 필요성이 제거됩니다.

워크플로우 끝에 목표 단계를 추가하고 목표 이벤트를 연락처 태그 추가 - "상품 구매됨"으로 설정하기만 하면 됩니다.

태그가 추가되면 연락처가 목표 단계로 이동되어 계속 진행됩니다. 이 경우 마지막 단계이므로 연락처를 제거하게 됩니다.

#### 활용 사례:

- 영업 퍼널 관리: 연락처가 "리드"에서 "예상 고객"에서 "협상"에서 "성사"로 진행할 때 영업 워크플로우의 다른 단계를 통해 자동으로 이동
- 상품 구매: "상품 구매됨" 태그가 연락처에 추가될 때 구매 후 후속 조치나 업셀 워크플로우를 트리거하기 위해 목표 이벤트 사용
- 참여도 추적: "높은 참여도" 또는 "낮은 참여도"와 같은 태그를 사용하여 다른 참여 수준을 위해 설계된 워크플로우 간에 연락처 이동
- 구독 상태: "구독됨" 또는 "구독 취소됨" 태그로 표시되는 연락처의 구독 상태 변경에 기반하여 유지 또는 재참여 전략을 위한 다른 워크플로우 트리거
- 행동 기반 세분화: "전자책 다운로드" 또는 "웨비나 참석"과 같은 행동에 기반하여 연락처를 세분화하고 이러한 행동에 맞춘 워크플로우 트리거
- 고객 만족도: 설문 조사나 피드백 양식에서 수집한 고객 만족도 수준을 나타내는 태그를 사용하여 고객 유지 또는 손상 제어를 위한 워크플로우 트리거
- 이벤트 참석: "이벤트 X 참석"과 같은 태그로 추적되는 특정 이벤트에 참석한 연락처에게 이벤트 후 후속 조치나 특별 할인 제안 트리거
- 추천 프로그램: 추천 프로그램을 운영하는 경우, 다른 사람을 추천한 연락처에 "추천인" 태그를 추가하고 특별 감사 메시지나 리워드를 위한 워크플로우 트리거
- 업셀/교차 판매: 고객이 특정 상품이나 서비스를 구매하면, 관련 상품이나 서비스를 업셀하거나 교차 판매하도록 설계된 워크플로우를 트리거하는 특정 태그 추가
- 체험 사용자: 소프트웨어나 구독 기반 서비스의 경우, 무료 체험 중인 연락처에 "체험 사용자" 태그를 추가하고 이러한 체험을 유료 구독으로 전환하도록 설계된 워크플로우 트리거

#### 자주 묻는 질문:

**워크플로우 외부에서 태그가 추가되거나 제거된 것을 추적할 수 있나요?**

네, 시스템은 변경 사항이 어디서 오든 상관없이 연락처의 태그에 대한 모든 변경 사항을 "감지"하고 이에 따라 목표 이벤트를 트리거합니다.

**일부 연락처가 이미 워크플로우에 진입한 후 목표 태그를 변경하고 싶으면 어떻게 하나요?**

목표 태그를 변경해도 이미 워크플로우에 진입한 연락처에는 영향을 주지 않습니다. 변경 후 워크플로우에 진입하는 연락처만 새로운 목표 태그에 대해 평가됩니다.

**같은 워크플로우에서 다른 목표 이벤트에 동일한 태그를 사용할 수 있나요?**

동일한 태그에 대해 추가와 제거에 대한 별도의 목표 이벤트를 가질 수 있습니다. 연락처는 목표 조건을 만족할 때 목표 단계로 이동된다는 점을 기억하세요.

**연락처가 같은 워크플로우에서 동일한 태그 목표 이벤트를 두 번 이상 만족할 수 있나요?**

아니요, 태그 목표 이벤트를 만족한 후 연락처가 목표 단계로 이동되면, 연락처가 동일한 워크플로우에 있는 동안 태그가 제거되고 다시 추가되지 않는 한 해당 워크플로우에서 동일한 목표 이벤트에 대해 다시 평가되지 않습니다.

**연락처의 라이프사이클 단계 변경을 추적하고 해당 워크플로우를 자동화하기 위해 태그를 어떻게 사용할 수 있나요?**

"리드", "예상 고객", "고객"과 같은 다른 라이프사이클 단계에 해당하는 태그를 할당하고, 워크플로우에서 이러한 태그에 대한 목표 이벤트를 설정할 수 있습니다. 시스템은 단계가 변경됨에 따라 연락처를 적절한 단계로 자동 이동시킵니다.

### 4. 예약 상태:

예약 상태의 경우, 드롭다운 메뉴에서 캘린더를 선택한 다음 신규, 확인됨, 참석함 예약 상태 중 하나 이상을 선택해 주세요.

- 신규 — 선택한 캘린더의 예약 상태가 "신규"로 변경되면 연락처가 목표 단계로 이동됩니다.
- 확인됨 — 선택한 캘린더의 예약 상태가 "확인됨"으로 변경되면 연락처가 목표 단계로 이동됩니다.
- 참석함 — 선택한 캘린더의 예약 상태가 "참석함"으로 변경되면 연락처가 목표 단계로 이동됩니다.
