# 워크플로우 액션 일시 비활성화

워크플로우에서 삭제하거나 연결을 끊지 않고도 액션을 일시적으로 비활성화할 수 있습니다. 이를 통해 더 안전하고 빠르게 테스트, 디버깅, 반복 작업을 할 수 있고, 준비가 되면 다시 활성화하면 됩니다.

---

## 워크플로우 액션 일시 정지란?

워크플로우 액션 일시 정지(Pause Workflow Actions) 기능을 사용하면 고급 빌더(Advanced Builder)와 표준 빌더(Standard Builder) 모두에서 워크플로우 노드를 일시적으로 켜거나 끌 수 있습니다. 비활성화된 노드는 실행 중에 건너뛰어지므로, 기존 설정을 잃지 않으면서도 디버깅, A/B 테스트, 반복 작업을 할 수 있습니다.

- **중단 없는 일시 정지**: 개별 노드를 끄면서 변형을 테스트하거나 문제를 격리할 수 있습니다.
- **비파괴적**: 연결이 그대로 유지되므로 연결을 제거하거나 다시 구축할 필요가 없습니다.
- **명확한 시각적 표시**: 비활성화된 단계는 캔버스에서 흐리게 표시되며 런타임에 건너뛰어집니다.

## 워크플로우 액션 일시 정지의 주요 장점

워크플로우 액션 일시 정지는 강력한 편의성 개선을 제공합니다:

- **두 빌더 모두 지원**: 고급 빌더와 표준 빌더 모두에서 사용 가능하므로, 선호하는 빌더에 관계없이 동일한 테스트 및 문제 해결 워크플로우를 사용할 수 있습니다.
- **더 빠른 테스트**: 단계를 비활성화하고 워크플로우를 실행한 후 즉시 다시 활성화할 수 있습니다.
- **재구축 불필요**: 설정이 그대로 유지되므로 경로를 다시 매핑할 필요가 없습니다.
- **깔끔한 실행**: 시스템이 자동으로 비활성화된 노드를 건너뛰므로 연락처가 멈추지 않습니다.
- **안전한 반복 작업**: 프로덕션 로직에 영향을 주지 않고 디버깅이나 A/B 테스트를 할 수 있습니다.

## 활성화/비활성화 토글

고급 또는 표준 빌더에서 임의의 노드에 마우스를 올리면 활성화/비활성화 스위치가 나타납니다. 토글을 끄면 노드가 회색으로 변하고 "Disabled(비활성화됨)" 라벨이 추가되며, 다시 켜면 일반 색상과 실행이 복원됩니다. 비활성화된 노드는 실시간 실행 중에 완전히 무시됩니다.

## 시각적 표시 및 실행 동작

- **비활성화된 노드**: Disabled(비활성화됨) 태그와 함께 회색으로 표시됩니다.
- **비활성화된 노드**: 유일한 진행 경로일 때도 자동으로 건너뛰어지므로 다운스트림 액션은 여전히 실행됩니다.
- **활성화된 노드**: 재활성화 후 설정대로 정확히 실행됩니다.

## 워크플로우 액션 일시 정지 작동 방식

- **활성화/비활성화 토글**: 노드에 마우스를 올리고 일시정지 아이콘을 클릭합니다.
- **캔버스 신호**: 비활성화된 노드는 실행되지 않음을 나타내기 위해 흐리게 표시됩니다.
- **실행**: 워크플로우 실행 중에 비활성화된 노드는 건너뛰어집니다.
- **다운스트림 로직**: 비활성화된 노드가 나중 액션으로 가는 유일한 경로의 일부라면, 워크플로우는 비활성화된 노드를 건너뛰고 현재 경로로 접근 가능한 활성화된 다운스트림 노드를 계속 실행합니다.

## 자주 묻는 질문

**Q: 연락처가 비활성화된 노드에서 멈추게 되나요?**
A: 아니요. 시스템이 자동으로 비활성화된 노드를 건너뛰고 활성화된 경로로 계속 진행합니다.

**Q: 전체 브랜치를 비활성화할 수 있나요?**
A: 네. 해당 브랜치의 각 노드를 토글로 끄세요. 다시 활성화할 때까지 모든 비활성화된 노드가 건너뛰어집니다.

**Q: "특정 날짜에 워크플로우 일시정지" 기능과 같은 건가요?**
A: 아니요. 워크플로우 액션 일시 정지는 단일 워크플로우 내의 개별 단계를 일시정지하는 것이고, 특정 날짜에 워크플로우 일시정지는 날짜 범위 동안 전체 발행된 워크플로우를 일시정지하는 것입니다.

**Q: 비활성화된 노드가 있는 상태에서 표준 빌더로 전환하면 어떻게 되나요?**
A: 빌더가 로드되기 전에 해당 노드를 삭제하거나 다시 활성화하라는 안내가 표시됩니다.

**Q: 비활성화된 노드도 실행 제한 횟수에 포함되나요?**
A: 아니요. 건너뛰어지므로 실행 횟수를 소모하지 않습니다.

**Q: 노드를 일괄적으로 활성화하거나 비활성화할 수 있나요?**
A: 출시 시점에는 노드별로 하나씩 토글해야 합니다. 일괄 작업은 아직 지원되지 않습니다.

**Q: 로그에 건너뛴 노드가 표시되나요?**
A: 네. 실행 로그는 비활성화된 노드를 "Skipped(건너뜀)"로 표시하여 테스트 중에 명확한 추적성을 제공합니다.
