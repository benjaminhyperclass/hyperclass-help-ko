# 날짜/시간 포매터 프리미엄 워크플로우 액션 사용 방법

날짜/시간 포매터(Date/Time Formatter) 액션은 워크플로우 내에서 날짜 또는 날짜와 시간 구조를 다시 포맷하고 날짜를 비교할 수 있는 다양한 도구입니다. 이 기능을 사용하면 다른 애플리케이션, 데이터베이스 저장소, 또는 이메일, SMS, Slack과 같은 커뮤니케이션 채널과의 호환성을 위해 날짜 포맷을 쉽게 변환할 수 있습니다. 또한 날짜를 비교하여 일수 차이를 계산함으로써 날짜 관련 계산과 의사 결정을 간단하게 만들어줍니다.

## 날짜/시간 포매터 프리미엄 워크플로우 액션이란?

날짜/시간 포매터 액션은 자동화된 작업에서 날짜와 시간을 다루는 데 도움이 되는 간단한 도구입니다. 다음은 세 가지 주요 기능과 단계입니다:

**날짜 형식 변경:**
a. 변경하려는 날짜를 선택합니다.
b. 도구가 자동으로 날짜 형식을 인식합니다.
c. 목록에서 새로운 형식을 선택합니다.
d. 도구가 날짜를 새로운 형식으로 변경합니다.

**날짜와 시간 형식 변경:**
a. 변경하려는 날짜와 시간을 선택합니다.
b. 도구가 자동으로 날짜와 시간 형식을 인식합니다.
c. 목록에서 새로운 형식을 선택합니다.
d. 도구가 날짜와 시간을 새로운 형식으로 변경합니다.

**두 날짜 비교하여 차이 확인:**
a. 시작 날짜와 종료 날짜를 선택합니다.
b. 도구가 자동으로 날짜 형식을 인식합니다.
c. 도구가 두 날짜 간의 일수 차이를 계산합니다.

이러한 기능을 통해 날짜와 시간의 모양을 쉽게 변경하거나 두 날짜를 비교할 수 있어, 올바른 형식으로 정보를 공유하거나 저장하기가 간단해집니다.

### 주요 활용 사례와 유용한 대상

날짜/시간 포매터 액션은 날짜와 시간 데이터를 관리하고 조작해야 하는 사용자에게 다양한 시나리오에서 유용합니다:

**이벤트 주최자:** 다양한 소스에서 받은 이벤트 시작 및 종료 날짜와 시간을 포맷하여 일관성을 유지하고 데이터 가독성을 높입니다.

**마케팅 팀:** 개인화되고 시간에 민감한 커뮤니케이션을 위해 이메일 캠페인, SMS 또는 Slack 메시지에서 포맷된 날짜와 시간 정보를 사용합니다.

**영업 팀:** 현재 날짜와 인보이스 만료일을 비교하여 결제 예정이거나 연체된 고객에게 자동 알림을 보냅니다.

**고객 지원 팀:** 인바운드 웹훅 트리거를 통해 받은 날짜와 시간 데이터를 다시 포맷하여 지원 티켓을 생성하거나 CRM 시스템을 업데이트합니다.

**프로젝트 매니저:** 프로젝트 시작 날짜와 마감일을 비교하여 남은 시간을 계산하고 팀원에게 알림을 보냅니다.

**데이터 분석가:** 일관되게 포맷된 날짜와 시간 데이터를 스프레드시트나 데이터베이스에 저장하여 분석과 보고서 생성을 더 쉽게 만듭니다.

**소프트웨어 개발자:** 다양한 소스의 포맷된 날짜와 시간 데이터를 애플리케이션에 통합하여 호환성과 원활한 데이터 처리를 보장합니다.

**전자상거래 비즈니스:** 주문 및 배송 날짜 정보를 다시 포맷하여 더 나은 추적과 고객 커뮤니케이션을 제공합니다.

### 이 기능의 장점

날짜/시간 포매터 액션은 워크플로우에서 날짜와 시간 데이터를 다루어야 하는 사용자에게 여러 가지 장점을 제공합니다:

**일관성:** 다양한 플랫폼과 애플리케이션에서 날짜와 시간 데이터가 균일하게 포맷되도록 보장하여 가독성을 향상시키고 혼란을 줄입니다.

**호환성:** 필요에 따라 날짜와 시간 형식을 변환하여 다양한 도구와 시스템 간의 원활한 데이터 교환과 통합을 촉진합니다.

**시간 절약:** 날짜와 시간 포맷팅 작업을 자동화하여 수동 작업과 오류를 줄이고, 사용자가 더 중요한 작업에 집중할 수 있게 합니다.

**개인화:** 포맷된 날짜와 시간 데이터를 개인화된 메시지, 이메일 또는 알림에 사용할 수 있게 하여 커뮤니케이션 효과를 향상시킵니다.

**의사 결정:** 날짜 비교와 계산을 단순화하여 시간에 민감한 운영에서 정보에 기반한 의사 결정을 지원합니다.

**유연성:** 특정 요구사항이나 선호도에 맞는 다양한 형식 옵션을 제공하여 다양한 사용 사례에 적응할 수 있게 합니다.

**향상된 데이터 관리:** 데이터베이스나 스프레드시트의 일관성을 보장하여 날짜와 시간 데이터의 저장과 분석을 간소화합니다.

## 날짜/시간 포매터 프리미엄 워크플로우 액션 사용 방법

날짜/시간 포매터 액션을 사용하면 필요에 따라 날짜 또는 날짜와 시간을 다시 포맷할 수 있습니다. 또한 이 액션을 사용하여 날짜를 비교할 수도 있습니다.

- 날짜 포맷
- 날짜와 시간 포맷  
- 날짜 비교

### 날짜 포맷

날짜 포맷 기능은 날짜/시간 포매터 액션의 핵심 기능으로, 날짜 구조를 한 형식에서 다른 형식으로 변환하도록 특별히 설계되었습니다. 다음은 날짜 포맷 기능에 대한 자세한 설명입니다:

#### 필드 선택:

다시 포맷하려는 날짜 필드를 선택합니다. 특정 날짜, 현재 날짜, 연락처의 날짜 필드, 날짜 유형 커스텀 필드, 예약 시작/종료 날짜, 커스텀 값, 또는 인바운드 웹훅 트리거 등 다양한 소스를 선택할 수 있습니다.

![날짜 필드 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294319014/original/tm7FJaNoXpSQajtoZTpkwsg4LagmNPb7rw.png?1682421407)

**참고사항:**
현재 연락처 커스텀 필드는 날짜 시간 구조를 지원하지 않습니다.

#### 원본 형식:

특정 날짜, 현재 날짜, 연락처의 날짜 유형 표준 또는 커스텀 필드, 예약 시작/종료 날짜와 같은 시스템 필드를 선택하면 도구가 자동으로 형식을 감지하고 미리 선택합니다. 커스텀 값이나 인바운드 웹훅 트리거를 선택하는 경우 일치하는 형식을 수동으로 선택해야 합니다.

![원본 형식 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294319162/original/mw5kczQTW7cmuWmuHWf8RF2xy-5-uDvUQA.png?1682421438)

#### 대상 형식:

사용 가능한 형식 목록에서 출력 날짜에 대한 원하는 형식을 선택합니다. 이것이 원본 날짜가 변환될 형식입니다.

![대상 형식 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294319380/original/JDbxWXpuMLUudXFpvZXJRnvFj6hctWQwQw.png?1682421475)

#### 출력:

날짜 포맷 기능을 설정한 후에는 워크플로우 내의 후속 액션에서 결과를 사용할 수 있습니다. 출력은 동적 변수인 {{datetime_formatter.1.date}}를 사용하여 접근할 수 있습니다.

![날짜 포맷 출력](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294319488/original/uH_zpk6TLrKSRh9_PH5bJ7MFckDrSI7jKA.png?1682421500)

### 날짜 비교

날짜 비교 기능은 날짜/시간 포매터 액션의 유용한 기능으로, 사용자가 두 날짜를 비교하고 일수 차이를 얻을 수 있게 해줍니다. 다음은 날짜 비교 기능에 대한 자세한 개요입니다:

#### 시작 날짜 선택:

특정 날짜, 현재 날짜, 연락처의 날짜 필드, 날짜 유형 커스텀 필드, 예약 시작/종료 날짜, 커스텀 값, 또는 인바운드 웹훅 트리거 등 다양한 소스에서 시작 날짜를 선택합니다.

![시작 날짜 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294321323/original/WIjwrYhbXTQ3u5fizM5oET0mAmkJ3HAJnQ.png?1682421937)

#### 시작 날짜 형식:

특정 날짜, 현재 날짜, 연락처의 날짜 유형 표준 또는 커스텀 필드, 또는 예약 시작/종료 날짜와 같은 시스템 필드를 선택하면 도구가 자동으로 형식을 감지하고 미리 선택합니다. 커스텀 값이나 인바운드 웹훅 트리거를 선택하는 경우 일치하는 형식을 수동으로 선택해야 합니다.

![시작 날짜 형식](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294321759/original/GHgSfbLrrMcUzrz0Fhfz2jmHNrcXlzMX4A.png?1682422042)

#### 종료 날짜 선택:

시작 날짜와 동일한 소스(특정 날짜, 현재 날짜, 연락처의 날짜 필드, 날짜 유형 커스텀 필드, 예약 시작/종료 날짜, 커스텀 값, 또는 인바운드 웹훅 트리거)에서 종료 날짜를 선택합니다.

![종료 날짜 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294321793/original/cd1OHHTxcB3WwtPNFic3vcJpA8lA23FYPQ.png?1682422053)

#### 종료 날짜 형식:

시작 날짜 형식과 마찬가지로, 시스템 필드를 선택하면 도구가 자동으로 형식을 감지하고 미리 선택합니다. 커스텀 값이나 인바운드 웹훅 트리거의 경우 일치하는 형식을 수동으로 선택해야 합니다.

![종료 날짜 형식](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294321830/original/JfHGMRvsHnH15caaHONgtXEkdXBO27JfUQ.png?1682422060)

#### 출력:

날짜 비교 기능을 설정한 후에는 워크플로우 내의 후속 액션에서 결과를 사용할 수 있습니다. 일수 차이를 나타내는 출력은 동적 변수인 {{datetime_formatter.1.days}}를 사용하여 접근할 수 있습니다.

![날짜 비교 출력](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48294321867/original/BPxgH0dYBvU7tv0GgXdpp_wvi_MmctMYNQ.png?1682422070)

**참고사항:**
차이는 종료 날짜 - 시작 날짜로 계산됩니다. 시작 날짜가 종료 날짜를 초과하면 음수 값을 얻게 됩니다.

날짜 비교 기능은 만료일 계산, 이벤트 간 기간 결정, 또는 두 날짜 간의 시간 차이를 기반으로 한 의사 결정과 같은 다양한 목적으로 날짜 비교를 수행해야 하는 사용자에게 매우 유용합니다.

### 주의사항:

- 이벤트 시작 날짜 설정, 연락처 커스텀 필드 업데이트, If/Else 조건에서 비교하기 위해 인바운드 웹훅 트리거 데이터를 포맷합니다.
- 예를 들어 날짜 형식이 MM/DD/YYYY인데 날짜를 DD/MM/YYYY로 포맷하는 앱으로 보내야 하는 경우, 날짜/시간 포매터를 사용하여 날짜 형식을 변경하고 커스텀 웹훅을 통해 보낼 수 있습니다.
- 커스텀 포맷된 날짜 시간 정보를 Google Sheets에 저장
- 이메일/SMS/Slack에서 커스텀 포맷된 날짜 시간 정보 사용
- 현재 날짜(시작 날짜)와 인보이스 만료일(종료 날짜)을 비교하고 이메일/SMS/Slack에서 동적 값 사용
- 예: "귀하의 인보이스가 {{datetime_formatter.1.days}}일 후에 만료됩니다"
- 참고: 연체 시, 즉 현재 날짜(시작 날짜)가 인보이스 만료일(종료 날짜)보다 큰 경우 음수 값으로 결과를 얻게 됩니다.
