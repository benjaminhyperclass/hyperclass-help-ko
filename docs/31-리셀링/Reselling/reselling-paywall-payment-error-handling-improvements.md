# 리셀링 - 페이월 결제 오류 처리 개선

Hyperclass의 최신 페이월 개선 사항은 더 스마트한 Stripe 오류 처리, 명확한 화면 메시지, 그리고 구독 체크아웃이 원활하게 진행되도록 하는 내장 보호 기능을 도입합니다. WhatsApp, WordPress, Yext, 광고 또는 기타 마켓플레이스 상품을 리셀링하는 경우에도 안정적으로 작동합니다.

---

## 리셀링 페이월 결제 오류 처리란?

리셀링 페이월 오류 처리는 고객 정보를 검증하고, Stripe와 통신하며, 체크아웃 중 문제가 발생했을 때 로케이션이나 에이전시 사용자에게 무엇을 보여줄지 결정하는 백그라운드 로직입니다. 이 로직을 개선함으로써 Hyperclass는 이제 누락된 주소, 비활성화된 상품, 네트워크 문제 등 더 많은 예외 상황을 포착하여 귀하나 고객이 좌절하기 전에 처리합니다.

---

## 새로운 오류 처리의 주요 장점

- 실시간으로 이해하기 쉬운 오류 메시지가 페이월 화면에 직접 표시되어, 사용자가 결제 실패 이유를 정확히 알 수 있습니다.

- 고객이 여러 탭을 열었을 때 중복 구독을 방지하는 추가 Stripe 보호 기능이 있습니다.

- 고객 주소, 기본 결제 방법, 결제 수단에 대한 검증 확인으로 "알 수 없는 오류" 메시지가 줄어듭니다.

- 네트워크 중단과 Stripe 타임아웃에 대한 복원력으로 장바구니 이탈을 줄입니다.

- WhatsApp, WordPress, Yext, 광고 및 향후 마켓플레이스 앱에서 통일된 로직으로 혼란이 적습니다.

---

### Stripe 오류 메시지

더 스마트한 Stripe 응답은 암호 같은 API 코드를 일반인이 이해할 수 있는 안내(예: "카드 결제 거절됨—계속하려면 새 카드를 추가하세요")로 변환합니다. 시스템은 이제 Stripe의 "기본 결제 방법 없음" 및 "결제 수단 누락" 오류도 표시하여, 사용자가 무작정 재시도하는 대신 결제 정보를 업데이트하도록 안내합니다.

### 고객 주소 검증

모든 페이월 프로세스는 이제 Stripe가 구독을 생성하기 전에 고객의 결제 주소가 존재하고 올바르게 형식화되었는지 확인합니다. 주소가 누락되거나 유효하지 않으면, 페이월이 거래를 차단하고 사용자에게 프로필을 완성하도록 지시합니다. 이 사전 확인으로 가장 일반적인 Stripe 실패 원인 중 하나가 제거됩니다.

### 체크아웃 중 상품 비활성화

에이전시가 고객이 구매 중인 동안 상품을 비활성화(또는 삭제)하면, 페이월은 이제 거래를 우아하게 취소하고 설명이 포함된 알림을 표시합니다. 이렇게 하면 고아 구독과 결제 분쟁을 방지할 수 있습니다.

### 네트워크 중단 처리

일시적인 네트워크 끊김으로 인해 고객이 로딩 스피너를 계속 바라보게 되는 일이 없습니다. 페이월은 안전한 작업을 자동으로 재시도하며, 재시도가 여전히 실패하면 "연결 끊어짐—다시 시도해 주세요" 배너를 표시하여 중복 결제를 방지합니다.

### 중복 구독 보호 장치

여러 탭을 열거나 결제 및 구독 버튼을 두 번 클릭하면 이전에는 두 개의 활성 Stripe 구독이 생성될 위험이 있었습니다. 페이월은 이제 체크아웃 중에 고객 레코드를 잠궈서 하나의 구독만 생성되도록 보장합니다.

## 안정적인 페이월 체크아웃 설정 방법

깔끔한 Stripe 연결과 정확한 고객 프로필이 오류 없는 결제를 위한 가장 빠른 방법입니다.

- Stripe를 연결하고(`Settings(설정) ➜ Integrations(연동) ➜ Stripe`) 계정이 최소 하나의 활성 결제 방법이 있는 라이브 모드인지 확인하세요.

- `Payments(결제) ➜ Integrations(연동) ➜ Manage Payment Methods(결제 방법 관리)`에서 하위 계정별로 허용된 결제 방법을 구성하고, 필요에 따라 카드, ACH, 월렛 또는 Klarna를 활성화하세요.

- 모든 로케이션에 대한 결제 주소를 확인하세요(`Payments(결제) ➜ Customers(고객) ➜ Edit Customer(고객 편집)`).

- 페이월 외부에서 구독을 판매하는 경우, 결제 재시도 규칙을 활성화하여(`Payments(결제) ➜ Settings(설정) ➜ Subscription(구독)`) 최대 3회까지 자동 재시도하도록 설정하세요.

- 고객에게 중복 구독 잠금 기능의 혜택을 받으려면 단일 브라우저 탭에서 결제를 완료하도록 권장하세요.

- (선택사항) 페이월 근처에 FAQ나 지원 링크를 삽입하여 사용자가 카드 거절 문제를 즉시 해결할 수 있도록 하세요.

## 자주 묻는 질문

**Q: 고객이 "기본 결제 방법을 찾을 수 없습니다"라고 표시되면 어떻게 해야 하나요?**

A: 고객에게 페이월에서 "결제 방법 추가"를 클릭하고 유효한 카드를 입력하도록 요청하세요. Stripe는 결제 방법 없이는 구독을 생성할 수 없습니다.

**Q: 여러 번 재시도 후 구독이 "미결제" 상태로 변경된 이유는 무엇인가요?**

A: 구성된 재시도 횟수가 실패한 후, Hyperclass는 구독을 미결제로 표시합니다(자동 취소를 활성화한 경우 취소). 고객은 여전히 생성된 인보이스를 결제하여 구독을 다시 활성화할 수 있습니다.

**Q: 새로운 오류 메시지를 커스터마이징할 수 있나요?**

A: 문구는 규정 준수를 위한 Stripe의 권장 표현을 따르므로 현재 편집할 수 없습니다.

**Q: 에이전시가 제가 이미 구매한 상품을 비활성화하면 어떻게 되나요?**

A: 향후 재청구가 자동으로 중단되고 고객이 알림을 받습니다. 기존 서비스는 상품 약관에 따라 현재 청구 기간 종료까지 계속될 수 있습니다.

**Q: 이러한 개선 사항이 일회성 요금에도 영향을 미치나요, 아니면 구독에만 해당하나요?**

A: 대부분의 변경 사항은 구독 플로우를 대상으로 하지만, 더 명확한 Stripe 오류 메시지는 일회성 페이월 구매에도 적용됩니다.

**Q: 샌드박스에서 이러한 시나리오를 어떻게 테스트할 수 있나요?**

A: Stripe 테스트 카드(예: 거절된 결제의 경우 4000 0000 0000 0002)를 사용하고 네트워크 중단 처리를 테스트하려면 잠시 인터넷 연결을 끊어보세요.
