# 왓츠앱: 메시지 템플릿(스니펫) 발송하기

스니펫은 24시간 왓츠앱 고객 서비스 창 안에서 사용할 수 있는 재사용 가능한 메시지 블록입니다. 이 글에서는 대화(Conversations)와 워크플로우(Workflows)에서 스니펫을 발송하는 방법을 설명하고, 업데이트된 왓츠앱 템플릿 처리 기능을 소개합니다. 이제 발송 시 템플릿을 미리보기(카테고리 및 언어 메타데이터 포함)하고, 로컬 파일에서 한 번의 클릭으로 템플릿 미디어를 교체할 수 있습니다. 이러한 개선 사항은 대화와 일괄 작업에서 모두 적용됩니다.

---

## **개요**

스니펫은 더 빠르고 일관된 답변을 위해 활성 왓츠앱 대화에 삽입할 수 있는 저장된 텍스트 또는 멀티미디어 블록입니다. 왓츠앱 템플릿(사전 승인된 메시지)은 별개의 기능으로, 24시간 창 밖의 연락처에게 메시지를 보낼 때 사용합니다.

최근 업데이트:

- 템플릿을 대화 작성기와 일괄 작업 내에서 **실시간으로 미리보기**할 수 있습니다
- 템플릿 선택 시 **카테고리**와 **언어**가 표시됩니다
- 템플릿 **미디어**를 로컬 파일에서 **한 번의 클릭으로 교체**할 수 있습니다
- 새로운 UI 경험이 Labs를 통해 제공될 수 있습니다

**참고:** 업데이트된 파이프라인/UI 기능을 활성화하려면(Labs에서 제공되는 경우) **하위 계정(Sub-account) → Labs**로 이동하세요. (이 T3 템플릿 업데이트는 기본적으로 제공될 예정이지만, UI가 보이지 않으면 Labs를 확인해보세요.)

## **단계별 안내 — 스니펫 발송, 템플릿 미리보기, 미디어 교체**

### **1단계 — 스니펫 또는 템플릿 준비하기**

- **마케팅(Marketing) → 템플릿(Templates) (스니펫)**으로 이동합니다
- **템플릿 추가(Add Template) (스니펫) → 텍스트 템플릿 추가(Add Text Template) (스니펫)**를 클릭하여 스니펫을 만들거나 기존 스니펫을 편집합니다
- 빠른 선택을 위해 명확한 이름으로 스니펫을 저장합니다

![스니펫 생성 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064131720/original/ceZDjknikYyX69AvwtF8oT8kh_GvgRKZLQ.png?1770168661)

### **2단계 — 대화에서 스니펫 발송하기**

- **대화(Conversations)**로 이동합니다
- 연락처와의 활성 왓츠앱 대화를 엽니다. (지난 24시간 내에 연락처로부터 메시지가 있어야 합니다)
- 채널로 **왓츠앱(WhatsApp)**이 선택되었는지 확인합니다
- 작성기에서 **스니펫 삽입(Insert Snippets)** (또는 스니펫 아이콘)을 클릭합니다
- 원하는 스니펫을 선택하면 작성기에 자동으로 입력됩니다
- 필요시 스니펫 텍스트를 편집합니다(커스텀 값(Custom Values)이나 트리거 링크(Trigger Links) 사용 가능)
- **발송(Send)**을 클릭합니다

![대화에서 스니펫 사용하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064131725/original/TGGfxxBq0qAMAsz_Q3DGMPnhGgzElrs-mw.png?1770168682)

### **3단계 — 발송 시 왓츠앱 템플릿 미리보기 (대화 및 일괄 작업)**

- 메시지 작성기(대화) 또는 일괄 작업 템플릿 선택에서:
- 자유 형식이나 스니펫 대신 **왓츠앱 템플릿(WhatsApp Template)**을 선택합니다
- UI에서 템플릿 내용의 **즉시 미리보기**가 표시됩니다
- 미리보기에서 템플릿 이름 옆에 **템플릿 카테고리**(예: 마케팅, 유틸리티)와 **언어**가 표시됩니다
- 템플릿을 확인하고 발송하거나 예약합니다

참고사항:
- 실시간 미리보기로 왓츠앱 설정을 열어서 템플릿 내용을 확인할 필요가 없습니다
- 발송 전에 미리보기를 사용하여 카테고리/언어를 확인하세요

![템플릿 미리보기 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064131758/original/hQHl5nPJRG8jv5tBJJKLT1OTxwe60nWe6w.png?1770168782)

### **4단계 — 왓츠앱 템플릿에서 미디어 교체하기 (한 번의 클릭)**

- 선택한 템플릿에 미디어(이미지, 문서 또는 동영상)가 포함된 경우, 작성기에 미디어 자리 표시자와 **미디어 교체(Replace media)** 옵션이 표시됩니다
- **미디어 교체(Replace media)**를 클릭합니다
- 로컬 기기에서 파일을 선택합니다(파일 브라우저가 열립니다)
- 선택한 미디어가 작성기 미리보기에서 템플릿 미디어를 즉시 교체합니다
- 메시지를 검토하고 발송합니다

참고사항:
- 이 기능은 템플릿 선택이 이루어지는 **대화**와 **일괄 작업**에서 사용할 수 있습니다
- 미디어 교체는 Meta/왓츠앱의 템플릿 자체를 변경하지 않으며, 해당 메시지 인스턴스의 발송 시에만 미디어를 교체합니다

![미디어 교체 기능](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064131831/original/okDZf39N8wkfQajbs28YJrIj0ME3JzoAdA.png?1770169177)

### **5단계 — 워크플로우를 통해 스니펫 또는 템플릿 발송하기**

- **자동화(Automation) → 워크플로우 생성(Create Workflow)**(또는 기존 워크플로우 편집)으로 이동합니다
- 초기 트리거를 추가한 다음 **왓츠앱: 고객 서비스 창 확인(WhatsApp: Customer Service Window Check)** 액션을 추가하여 연락처가 24시간 창 내에 있는지 확인합니다
- 확인의 **열림(Open)** 브랜치 아래에 **왓츠앱(WhatsApp)** 액션을 추가합니다
- 템플릿을 반드시 사용해야 한다면 **왓츠앱 템플릿(WhatsApp Template)**을 선택하고, 그렇지 않으면 **없음 - 자유 형식 메시지(None – Free form message)**를 선택하여 스니펫 내용을 프로그래밍 방식으로 삽입합니다
- 템플릿을 사용하는 경우, 워크플로우 작성기에서 대화와 유사하게 템플릿 미리보기가 표시되고 미디어 교체가 가능합니다
- 워크플로우를 저장하고 발행합니다

![워크플로우에서 왓츠앱 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064131873/original/PiyeF_OcVZKBV5AGBdT-UoddCqLnv2h4lw.png?1770169310)

### **6단계 — 일괄 작업: 템플릿 발송 및 미디어 교체**

- **연락처(Contacts)**(또는 일괄 작업이 가능한 곳)로 이동하여 메시지를 보낼 연락처를 선택합니다
- **일괄 작업(Bulk Action) → 왓츠앱 발송(Send WhatsApp)**(또는 플랫폼의 일괄 왓츠앱 플로우)을 선택합니다
- **왓츠앱 템플릿(WhatsApp Template)**을 선택하고 실시간 미리보기를 사용하여 카테고리와 언어를 확인합니다
- 템플릿에 미디어가 포함된 경우 **미디어 교체(Replace media)** 컨트롤을 사용하여 로컬 파일로 교체합니다
- 확인하고 일괄 작업을 발송합니다

참고사항:
- 일괄 작업은 대화와 동일한 템플릿 미리보기 및 미디어 교체 UX를 따릅니다
- 수신자가 메시징 규칙(예: 옵트인, 허용된 콘텐츠)을 충족하는지 확인하세요

![일괄 작업에서 왓츠앱 템플릿 사용](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064131897/original/G6-Vtif6kGmeBjDIx5pViSPrpxF7Wu-ogQ.png?1770169397)

## 문제 해결 및 팁

- 템플릿 미리보기나 미디어 교체 옵션이 보이지 않으면, 대화 UI를 새로고침하고 계정이 업데이트된 경험을 지원하는지 확인해보세요
- 미디어 교체가 실패하면, 파일 유형과 크기가 왓츠앱 및 플랫폼 제한을 충족하는지 확인하세요
- 엄격한 자리 표시자가 있는 템플릿은 여전히 Meta의 승인된 구조를 준수해야 할 수 있습니다. 미디어 교체는 템플릿 형식 규칙을 우회하지 않습니다

## 자주 묻는 질문

**스니펫이 무엇인가요?**

스니펫은 Hyperclass에 저장된 사전 정의된 텍스트 또는 멀티미디어 블록으로, 빠르고 일관된 응답을 위해 활성 왓츠앱 대화에 삽입할 수 있습니다.

**스니펫과 왓츠앱 템플릿의 차이점은 무엇인가요?**

왓츠앱 템플릿은 24시간 창 밖의 사용자에게 연락하기 위해 필요한 사전 승인된 메시지로, Meta/왓츠앱의 승인을 받아야 합니다. 스니펫은 24시간 창 내에서 사용되는 내부적이고 편집 가능한 정형 응답으로 승인이 필요하지 않습니다.

**왓츠앱 템플릿을 발송 전에 미리볼 수 있나요?**

네. 이제 템플릿을 대화 작성기와 일괄 작업에서 직접 미리볼 수 있습니다. 미리보기에서는 템플릿의 카테고리와 언어와 함께 메시지 내용이 표시됩니다.

**발송 시 템플릿 안의 미디어를 교체할 수 있나요?**

네. 템플릿에 미디어가 포함된 경우 미디어 교체를 클릭하고 로컬 기기에서 파일을 업로드할 수 있습니다. 이 기능은 대화와 일괄 작업에서 사용할 수 있습니다.

**스니펫이 24시간 창 밖에서도 작동하나요?**

아니요. 스니펫은 연락처에게 열린 24시간 대화 창이 있을 때(즉, 지난 24시간 내 인바운드 메시지)만 발송할 수 있습니다. 그 창 밖에서 메시지를 보내려면 사전 승인된 왓츠앱 템플릿을 사용해야 합니다.

**미디어를 교체하면 저장된 템플릿이 변경되나요?**

아니요. 미디어 교체는 해당 발송 인스턴스에만 미디어를 교체합니다. 저장된 템플릿은 변경되지 않으며 여전히 원래 승인된 콘텐츠를 반영합니다.

**새로운 UI 기능이 보이지 않으면 어디서 활성화하나요?**

선택적 UI 업데이트를 위해 **하위 계정(Sub-account) → Labs**를 확인하세요. 계정에서 옵션이 노출되지 않고 여전히 기능이 보이지 않으면, 계정 수준 확인을 위해 지원팀에 문의하세요.
