# Uberall이란?

Uberall은 다중 지점 비즈니스가 검색엔진, 디렉토리, 지도, 앱에서 정확하고 실시간으로 온라인 존재감을 관리할 수 있도록 돕는 글로벌 리스팅 관리 엔진입니다. 모든 고객의 로컬 SEO를 향상시키는 원스톱 솔루션이라고 생각하시면 됩니다.

---

## **Uberall의 주요 장점**

- **퍼블리셔 커버리지:** Google, Bing, Facebook, Apple Maps, TomTom 등을 포함한 풍부한 [퍼블리셔 네트워크](https://uberall.com/en-us/directories/products)

- **지리적 범위:** 20개 이상 국가에서 리스팅 관리 지원

- **대규모 중복 제거:** 중복 콘텐츠를 자동으로 감지하고 제거

- **낮은 엔티티 충돌:** 게시 충돌 가능성을 크게 줄임 (기존 시스템 대비)

- **GDPR, ISO & PCI 준수:** 글로벌 보안 및 개인정보 보호 표준을 충족. [자세히 알아보기](https://uberall.helpjuice.com/en_US/userrights/security-and-compliance-at-uberall)

- **작업 관리자 & 데이터 정리 도구:** 플랫폼 간 위치 데이터 업데이트를 표준화하고 간소화

## **Yext와 Uberall 중 어떤 것을 선택해야 할까요?**

| 기능 | Uberall | Yext |
|------|---------|------|
| 엔티티 충돌 위험 | 낮음 | 보통 |
| 중복 제거 | 자동 | 수동 |
| 작업 관리 & 정리 도구 | 예 | 부분적 |
| 지리적 커버리지 (20개 이상 국가) | 예 | 예 |
| 50개 미만 지점 SMB 지원 | 예 | 예 |
| 보안 & 컴플라이언스 | GDPR, ISO, PCI | HIPAA, ISO |
| 고객 지원 | 고품질 | 고품질 |

팁: 수동 작업을 최소화하고, 강력한 글로벌 디렉토리 커버리지를 원하며, 더 나은 자동화로 빠르게 확장하고 싶다면 Uberall을 선택하세요.

## **가격**

에이전시에서 **Uberall**이나 **Yext** 중 어느 것을 선호하는 리스팅 관리 엔진으로 선택하든 가격은 동일합니다. 다음 요금으로 리스팅 상품을 구독할 수 있습니다:

- **1개월: $30**
- **6개월: $150**
- **12개월(1년): $300**

이는 비용에 제약받지 않고 고객의 필요에 가장 적합한 엔진을 선택할 수 있는 완전한 유연성을 제공합니다.

---

## **기존 엔티티 구독**

이미 리스팅에 구독된 비즈니스 엔티티가 있나요?

- 현재 엔진(Yext 또는 Uberall)으로 계속 작동합니다
- 엔진 선택은 **선택 후 새로 생성되는 엔티티에만** 영향을 미칩니다
- 기존 엔티티를 다른 엔진으로 전환하고 싶다면:
  - 현재 엔진에서 **구독 해지**
  - 에이전시 레벨 엔진이 전환된 후 **다시 구독**

## **시작하기**

#### **엔진 선택 (에이전시 레벨 설정)**

리스팅 상품에 처음 접근하면 선호하는 리스팅 관리 엔진으로 **Uberall**과 **Yext** 중에서 선택할 수 있는 모달이 나타납니다.

중요: 선택한 엔진은 에이전시의 모든 하위 계정과 그들이 관리하는 비즈니스 엔티티에 적용됩니다. 동일한 에이전시에서 두 엔진을 병렬로 실행할 수는 없습니다.

#### **체크박스 확인**

엔진을 선택한 후 체크박스로 확인해야 합니다:

*"이 엔진이 내 하위 계정들이 생성하는 모든 새 엔티티의 기본값이 될 것임을 이해하며, 이 선택으로 진행하는 것에 동의합니다"*

---

## **리스팅 엔진 선택 방법 (에이전시 레벨 설정)**

엔진 선택은 에이전시 레벨에서만 가능하며, 에이전시 하위의 모든 새 하위 계정과 비즈니스 엔티티는 자동으로 이 엔진을 상속받습니다.

선호하는 리스팅 엔진을 설정하는 방법은 다음과 같습니다:

#### **1단계:** 리셀링으로 이동

- Hyperclass 앱에서 에이전시 대시보드에 로그인
- 좌측 네비게이션 메뉴에서 리셀링(Reselling) 탭 클릭

#### 2단계: 선택 모달 열기

- 아래로 스크롤하여 리스팅(Listings) 카드 찾기
- **엔진 변경(Change Engine)**을 클릭하여 선택 모달 열기

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050910010/original/RqG6H1Jc5eovWK4mYdx4bzhNVD04yq2d1g.png?1754331131)

#### **3단계:** 선호하는 엔진 선택

- **Uberall**과 **Yext**의 나란한 비교와 각각의 주요 장점 및 차별화 요소를 확인
- 선호하는 엔진 선택
- 엔진을 선택한 후, 경고 문구를 읽고 체크박스로 확인

중요: 하위 계정은 항상 리스팅 상품의 **화이트라벨** 버전을 경험하므로, 기본 엔진 이름을 보지 못하고 에이전시가 선택한 엔진으로 구동되는 매끄러운 리스팅 인터페이스만 보게 됩니다.

#### 4단계: 리스팅 엔진 확인

- 확인되면 선택한 엔진이 리스팅 카드에 **Powered by Uberall** 또는 **Powered by Yext**로 표시됩니다

## 나중에 엔진 변경하기

엔진은 선택 후 처음에는 고정되지만, **새 엔티티에 대해서는 동일한 엔진 선택 모달을 통해 변경을 요청할 수 있습니다.** 유의사항:

- 모든 기존 엔티티는 현재 엔진에서 계속 활성 상태로 유지됩니다. 새 엔진으로 전환한다는 것은 본질적으로 한 엔진에서 구독을 취소하고 다른 엔진에서 활성화한다는 의미이므로 퍼블리셔 데이터 동기화 지연이 발생할 수 있기 때문입니다.

- 새 엔진은 이후 모든 구독에 대해 활성 엔진이 됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050910096/original/cs3NekcNZ-TIvLwVkh4kYldi2-5gmQbatg.jpeg?1754331324)

## **자주 묻는 질문**

**Q: 일부 하위 계정에는 Uberall을, 다른 계정에는 Yext를 사용할 수 있나요?**
아니요, 동일한 에이전시에서 두 엔진을 병렬로 실행할 수는 없습니다. 나중에 전환할 수 있지만 변경은 하위 계정 레벨이 아닌 에이전시 레벨에서 이루어집니다.

**Q: 엔진을 전환할 때 현재 리스팅은 어떻게 되나요?**
현재 엔진(Yext 또는 Uberall)에서 그대로 계속 작동합니다. 엔진 선택은 선택 후 새로 생성되는 엔티티에만 영향을 미칩니다.

**Q: 내 고객이 Uberall이나 Yext 브랜딩을 보게 되나요?**
하위 계정은 항상 리스팅 상품의 화이트라벨 버전을 경험하므로 기본 엔진 이름을 보지 못합니다.
