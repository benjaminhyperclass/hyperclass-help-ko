# 온라인 리스팅에서 Entity History를 Audit Logs와 함께 사용하는 방법

이 문서에서는 리스팅 플랫폼의 새로운 Entity History 기능을 사용하는 방법을 알려드려요. Hyperclass의 Audit Logs와 통합되어 리스팅에 대한 모든 업데이트를 완전히 볼 수 있습니다 — 무엇이 변경되었는지, 누가 변경했는지, 언제 변경되었는지까지요. 컴플라이언스와 책임 추적에 완벽합니다!

### **리스팅의 Entity History란?**

Entity History는 각 리스팅 레코드에 대한 상세한 감사 추적을 제공합니다. 사용자가 시작한 변경이든 시스템에서 생성한 변경이든 모든 변경 사항이 기록되고 Hyperclass의 Audit Logs에서 확인할 수 있어요. 이를 통해 투명성이 향상되고 컴플라이언스 추적이 간소화됩니다.

### **Entity History의 주요 장점**

Entity History가 운영 감독과 데이터 투명성 개선에 어떻게 도움이 되는지 알아보세요.

- 각 리스팅 엔티티의 업데이트에 대한 완전한 기록 로그 조회
- 완전한 책임 추적을 위한 사용자 또는 시스템 프로세스별 변경 추적
- 실시간 기록으로 내부 감사 및 컴플라이언스 지원
- Audit Logs를 통해 리스팅 인터페이스에서 직접 로그 접근
- 데이터 편집의 명확한 문서화로 협업 향상
- 모듈식 로깅 덕분에 향후 시스템 개선 사항 대비

### **Audit Logs에서 Entity History 접근 및 사용 방법**

리스팅 업데이트를 추적하기 위해 Entity History 패널을 찾고 탐색하는 방법을 알아보세요.

- Hyperclass 계정에서 **리스팅** 탭으로 이동하세요
- 사이드바 메뉴에서 **Audit Logs** 옵션을 클릭하세요
- Audit Logs 화면에서 **Entity History** 탭을 선택하세요
- 엔티티 ID, 타임스탬프 또는 사용자별로 정렬하려면 필터를 사용하세요
- 엔티티를 클릭하면 전체 업데이트 타임라인과 데이터 변경 사항을 볼 수 있어요

### **Entity History 기능 상세 설명**

Entity History 기능의 기술적 기능과 구조를 이해해보세요.

**향상된 감사 상세 정보**

사용자 ID, 타임스탬프, 엔티티 ID 및 변경 전후의 상세한 데이터를 캡처합니다.

**사용자 귀속**

누가 변경했는지 쉽게 식별 — 사용자는 고유 ID를 통해 연결됩니다.

**모듈식 아키텍처**

향후 리스팅 기능과의 통합을 지원하는 미래 지향적 설계입니다.

**커스텀 감사 추적**

표준 감사 이벤트를 넘어선 리스팅 액션의 광범위한 추적입니다.

### **자주 묻는 질문**

**Q: Entity History 로그를 내보낼 수 있나요?**

현재 직접 내보내기는 지원되지 않지만, 감사 로깅과의 통합을 통해 API를 통한 내보내기가 가능합니다.

**Q: 누가 이 로그를 볼 수 있나요?**

Audit Log 접근 권한이 있는 사용자만 Entity History를 볼 수 있어요.

**Q: 삭제된 리스팅도 포함되나요?**

네, 삭제도 변경 전후 상태 스냅샷과 함께 추적됩니다.

**Q: 시스템에서 시작된 변경도 볼 수 있나요?**

네. 자동화나 워크플로우로 인한 변경은 그에 맞게 라벨이 표시됩니다.
