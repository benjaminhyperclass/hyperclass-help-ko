# 구글 비즈니스 프로필을 통한 통화 추적과 부재중 문자 자동 발송

구글 비즈니스 프로필(GBP)에서는 두 개의 전화번호를 등록할 수 있습니다. 이는 에이전시가 구글 비즈니스 프로필 리스팅에서 발생하는 통화를 추적하면서도 SEO 모범 사례를 유지할 수 있는 훌륭한 기회입니다(실제 사업장 전화번호를 보조 자리로 이동하는 것이 검색 순위에 부정적인 영향을 미치지 않는다는 것이 확인되었습니다).

Hyperclass의 구글 비즈니스 프로필 연동 기능을 사용하면 고객의 기존 사업장 전화번호를 보조 자리로 자동 이동하고, LeadConnector 번호를 첫 번째 자리에 추가할 수 있습니다.

![구글 비즈니스 프로필 통화 추적 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48096339534/original/ipAbRDhAbv48wA5WpohY-8c_g29I1hLwEg.png?1617420303)

GBP 통화 추적을 활성화하면 다음과 같이 작동합니다:

- GBP의 전화번호가 이미 해당 Hyperclass 하위 계정의 LeadConnector 번호인 경우: 이미 Hyperclass에서 사업장 통화가 추적되고 있으므로 아무것도 변경되지 않습니다.

- GBP의 전화번호가 해당 Hyperclass 하위 계정의 LeadConnector 번호 중 하나의 착신전환 번호로 설정되어 있는 경우: Hyperclass가 현재 번호를 보조 자리로 이동하고 LeadConnector 번호를 기본 자리에 추가합니다.

- GBP의 전화번호가 해당 Hyperclass 하위 계정의 Twilio 번호 중 어느 것의 착신전환 번호도 아닌 경우: Hyperclass가 현재 번호를 보조 자리로 이동하고, LeadConnector 번호를 기본 자리에 추가하며, 현재 번호를 해당 Hyperclass 하위 계정의 LeadConnector 번호의 착신전환 번호로 설정합니다.

## GBP에 추가할 LeadConnector 번호 선택

구글 비즈니스 프로필(GBP)을 Hyperclass의 로케이션에 연결하면, 플랫폼이 자동으로 LeadConnector 번호를 선택하여 GBP의 기본 전화번호로 설정합니다. 번호 선택 방식은 다음과 같습니다:

- **기본 로케이션 번호 (전화번호 탭에서 지정)**:
  로케이션에 `Settings(설정) → Phone Numbers(전화번호)`에서 지정된 전용 전화번호가 있는 경우, 해당 번호가 GBP로 전송될 기본 추적 번호로 사용됩니다.
  - 이는 온보딩 과정이나 수동 설정 중에 생성된 번호인 경우가 많습니다.

- **착신전환이 활성화된 하위 계정의 첫 번째 번호**:
  명시적으로 기본 번호로 설정된 번호가 없는 경우, Hyperclass는 착신전환이 활성화된 첫 번째 사용 가능한 번호를 선택합니다.

- **하위 계정의 사용 가능한 번호로 대체**:
  착신전환이 구성되지 않은 경우, Hyperclass는 해당 로케이션에서 사용 가능한 LeadConnector 번호를 선택합니다.

- **번호가 없는 경우? 먼저 번호를 지정해야 합니다**:
  하위 계정에서 번호를 찾을 수 없는 경우 연동이 완료되지 않습니다. GBP 연동이 작동하려면 먼저 번호를 지정하거나 생성해야 합니다.

## 실제 사용 예시

다음 예시들은 구글 비즈니스 프로필(GBP)에서 정확히 무엇이 변경되는지, 통화가 어디로 가는지, Hyperclass가 어떻게 추적하는지, 언제 부재중 문자 자동 발송이 실행되는지를 보여줍니다.

### 예시 1: GBP에서 이미 Hyperclass(LeadConnector) 번호를 사용하는 경우

상황: 사업장의 GBP 기본 번호가 이미 연결된 Hyperclass 하위 계정의 LeadConnector 번호입니다.

**GBP 통화 추적을 켰을 때 일어나는 일**
- GBP에서는 아무것도 변경되지 않습니다(이미 추적되는 번호로 통화가 이동하고 있기 때문입니다).

**실제 상황에서는**
- 고객이 Google에서 "ABC 치과"를 검색하고 통화 버튼을 누릅니다.
- 통화가 LeadConnector 번호로 연결됩니다(Hyperclass에서 추적됨).
- 팀이 응답하면 → Hyperclass에 통화가 기록됩니다.
- 아무도 응답하지 않으면 → 부재중 문자 자동 발송이 통화자에게 자동으로 문자를 발송할 수 있습니다(로케이션 설정에서 활성화된 경우).

### 예시 2: GBP 기본 번호가 Hyperclass 번호에서 이미 사용 중인 착신전환 번호와 같은 경우

상황: GBP에 사업장의 실제 번호(예: 유선전화)가 표시되고, 동일한 번호가 이미 Hyperclass 하위 계정의 LeadConnector 번호 중 하나의 착신전환 번호로 설정되어 있습니다.

**GBP 통화 추적을 켰을 때 일어나는 일**
- Hyperclass가 현재 GBP 번호를 보조 자리로 이동합니다
- Hyperclass가 LeadConnector 번호를 기본 자리에 설정합니다

**실제 상황에서는**
- 토글 전: GBP 기본 = (555) 111-2222 (실제 번호).
- 토글 후: GBP 기본 = LeadConnector 추적 번호; GBP 보조 = (555) 111-2222.
- 고객이 Google에서 통화 버튼을 누르면 → LeadConnector 번호로 통화가 연결됩니다(추적됨).
- Hyperclass가 통화를 (555) 111-2222로 착신전환합니다(이미 그렇게 구성되어 있었기 때문입니다).
- 부재중인 경우, 부재중 문자 자동 발송(활성화된 경우)이 LeadConnector 번호에서 통화자에게 문자를 발송합니다.
