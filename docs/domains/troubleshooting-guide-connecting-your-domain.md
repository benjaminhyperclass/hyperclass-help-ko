## 1. 레코드 충돌: 여러 A 레코드

![레코드 충돌 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045733019/original/DIvH4iHNY9DqPOPkav9Mc_1s6mjbbl8msw.png?1745817030)

### A 레코드란?

A 레코드는 도메인 이름을 IP 주소에 연결합니다. 방문자를 웹사이트가 호스팅된 올바른 서버로 안내하는 역할을 합니다.

### 언제 발생하나요?

도메인에 여러 A 레코드가 있으면 시스템이 어떤 IP 주소를 사용해야 할지 혼란스러워합니다. 이로 인해 웹사이트가 올바르게 로드되지 않는 문제가 발생할 수 있습니다.

### 해결 방법

1. 도메인 등록업체의 DNS 설정(GoDaddy, Namecheap, Cloudflare 등)에 로그인하세요.
2. 해당 도메인의 DNS 설정을 찾으세요.
3. IP 주소를 가리키는 @ 또는 yourdomain.com과 같은 A 레코드를 찾으세요.
   - @는 루트 도메인(예: yourdomain.com)을 의미합니다.
4. 도메인이나 하위 도메인(예: www.yourdomain.com)을 IP 주소에 매핑하는 A 레코드를 모두 삭제하세요.
5. 변경사항을 저장하고 1-2분간 적용될 때까지 기다리세요.
6. 도메인을 플랫폼에 다시 연결하세요.

## 2. AAAA 레코드 충돌

![AAAA 레코드 충돌 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045733029/original/-RNzp0jA3JWzd4QZVqOi5rai-iejIYL9Zw.png?1745817064)

### AAAA 레코드란?

AAAA 레코드는 A 레코드와 유사하지만 IPv6 주소에 사용되며, 이는 IP 주소의 최신 버전입니다.

### 언제 발생하나요?

- 플랫폼이 도메인에서 AAAA 레코드를 감지했을 때
- 시스템은 IPv4 레코드만 허용합니다. AAAA 레코드(IPv6)가 있으면 시스템 요구사항과 충돌하여 올바른 도메인 연결을 방해합니다.

### 해결 방법

1. 도메인 등록업체의 DNS 관리 패널에 로그인하세요.
2. AAAA 레코드(보통 AAAA 또는 IPv6로 표시)를 찾으세요.
3. AAAA 레코드를 삭제하세요.
4. 변경사항을 저장하고 1-2분간 기다리세요.
5. 도메인을 플랫폼에 다시 연결하세요.

---

## 중요 참고사항 (GoDaddy 사용자용)

GoDaddy에서 도메인을 등록한 경우, DNS Proxy(DNSSEC 또는 DNS protection이라고도 함)가 꺼져 있는지 확인하세요. GoDaddy의 프록시는 자동으로 AAAA 레코드를 재생성하거나 변경사항을 방해할 수 있습니다. 프록시를 비활성화하면 DNS 업데이트가 제대로 적용됩니다. 자세한 방법은 [여기](https://www.godaddy.com/en-in/help/turn-dnssec-on-or-off-6420)를 참조하세요.

## 3. CAA 레코드 충돌

![CAA 레코드 충돌 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045733052/original/RtgRUz9io4MZ7vJqjTtFBqQB-nIQxnVIIQ.png?1745817122)

### CAA 레코드란?

CAA(Certificate Authority Authorization) 레코드는 어떤 인증 기관(CA)이 도메인용 SSL 인증서를 발급할 수 있는지 지정합니다. 인증서 발급을 승인된 공급자로 제한하여 보안 계층을 추가합니다.

### 언제 발생하나요?

CAA 레코드가 존재하지만 필요한 값(pki.goog)이 포함되어 있지 않으면, 플랫폼이 도메인용 SSL 인증서를 발급할 수 없습니다.

### 해결 방법

#### 권장 방법 (대부분 사용자용)

1. 도메인 등록업체의 DNS 관리 패널에 로그인하세요.
2. CAA 레코드(보통 CAA로 표시)를 찾으세요.
3. CAA 레코드를 삭제하세요.
4. 변경사항을 저장하고 1-2분간 기다리세요.
5. 도메인을 플랫폼에 다시 연결하세요.

#### 고급 사용자용 (삭제 대신 수정)

CAA 레코드를 유지하되 플랫폼을 승인하고 싶다면, 기존 CAA 레코드에 다음 값을 추가하세요:
- `pki.goog`

이렇게 하면 다른 보안 설정을 제거하지 않고도 플랫폼이 도메인용 SSL 인증서를 발급할 수 있습니다.

## 4. DNS 레코드가 일치하지 않음

![DNS 레코드 불일치 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045733059/original/F56NxP7-Oqt6VC1T0xNvVEB7O70WboJcUA.png?1745817158)

### 무슨 의미인가요?

이 오류는 도메인의 DNS 설정이 플랫폼 요구사항과 일치하지 않을 때 발생합니다. 다음과 같은 원인이 있을 수 있습니다:

- 충돌하는 레코드
- 잘못된 구성
- 도메인이 다른 서비스에 연결됨

### 언제 발생하나요?

- DNS 설정이 올바르게 구성되지 않았을 때
- DNS 설정이 올바르더라도 전파 지연이 발생할 수 있습니다. 변경사항이 반영되도록 몇 분간 기다리세요.

### 해결 방법

#### 1. DNS 레코드 확인

1. DNS 조회 도구(예: DNS Checker)를 사용해 DNS 전파를 확인하세요.
2. 레코드를 플랫폼의 필요한 DNS 구성과 비교하세요. 대부분의 플랫폼은 특정 값(예: A 레코드용 IP 주소 또는 CNAME 레코드)을 제공합니다.

#### 2. 충돌 확인

- 레코드가 일치하는 경우 → 전파를 위해 몇 분간 기다리세요 (최대 48시간 소요 가능)
- 레코드가 충돌하는 경우 → 잘못된 항목을 제거하거나 업데이트하세요:
  - A 레코드가 필요한 경우 추가 A 레코드가 없는지 확인하세요.
  - CNAME 레코드가 필요한 경우 충돌하는 A 또는 AAAA 레코드를 제거하세요.

#### 3. 도메인 재연결

1. 변경사항을 저장하고 1-2분간 기다리세요.
2. 도메인을 재연결하고 확인하세요.

---

## 중요 참고사항 (GoDaddy 사용자용)

도메인 공급자가 GoDaddy인 경우, 도메인이 파킹되어 있지 않은지 확인하세요.

1. GoDaddy 계정에 로그인하세요.
2. My Products > Domains로 이동하고 해당 도메인 옆의 Manage DNS를 클릭하세요.
3. 도메인이 파킹되어 있지 않은지 확인하세요:
   - 도메인이 파킹되어 있으면 GoDaddy가 보유하고 있어 어떤 서비스에도 연결할 수 없습니다.
   - GoDaddy의 지침에 따라 도메인 파킹을 해제하세요.

## 5. 제품 충돌: 도메인이 다른 제품에 이미 연결됨

![제품 충돌 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046744361/original/qCyHgP6j6vMpWj-CPVI18stp1mKpr1fyPA.png?1747388058)

### 제품 충돌이란?

제품 충돌은 **도메인이나 하위 도메인이 이미 플랫폼 내의 다른 제품 유형(WordPress, Funnel, Client Portal 등)에 구성되어 있을 때** 발생합니다. 각 제품 유형은 올바르게 작동하려면 도메인에 대한 독점적 제어가 필요합니다.

### 언제 발생하나요?

이 문제는 도메인(예: my.abcabc.com)을 웨비나나 클라이언트 포털 같은 제품에 연결하려고 하지만, 해당 도메인이 이미 다른 제품(예: WordPress)에 연결되어 있을 때 발생합니다.

### 해결 방법

- 연결하려는 제품에 다른 도메인을 사용하거나,
- 현재 연결된 제품에서 도메인을 제거하세요:
  1. 계정의 에이전시 또는 로케이션 설정으로 이동하세요.
  2. Domains(도메인) > Manage Domain(도메인 관리) > Connected Products(연결된 제품)으로 이동하세요.
  3. 기존 제품(예: WordPress, Funnel 등)에서 도메인 연결을 삭제하세요.

**참고**: 충돌하는 제품이 같은 로케이션에 있는 경우, 시스템이 오류 메시지에 "here" 링크를 표시하여 도메인 설정으로 이동해 충돌을 관리할 수 있도록 안내합니다.

## 6. 도메인이 다른 곳에 연결됨

### 언제 발생하나요?

도메인이 이미 다른 에이전시나 하위 계정에 연결되어 있을 때입니다.

### 해결 방법

- 필요한 경우 지원팀에 연락하여 도메인 해제를 요청하세요.

## 추가 참고사항

✅ **전파 시간**: DNS 변경사항은 전 세계적으로 전파되는 데 24-48시간이 걸릴 수 있지만, 보통 몇 분 내에 적용됩니다. 문제가 지속되면 인내심을 갖고 DNS 설정을 다시 확인하세요.

✅ **레코드 재확인**: 도메인을 재연결하기 전에 DNS 조회 도구를 사용해 변경사항이 올바르게 전파되었는지 확인하세요.

✅ **캐시 지우기**: 브라우저 캐시를 지우거나 시크릿 모드를 사용해 오래된 DNS 정보가 표시되지 않도록 하세요.
