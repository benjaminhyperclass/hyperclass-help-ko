# Facebook 연동의 문제 해결 CTA란?

문제 해결 CTA는 권한 누락, 매핑되지 않은 폼 등 Facebook 연동의 일반적인 문제를 빠르게 식별하고 해결하는 새로운 가이드 도구입니다. 또한 개선된 리드 동기화(Sync Leads) 기능을 포함하여 수동 문제 해결 단계에 의존하지 않고 몇 번의 클릭만으로 처리되지 않은 리드를 처리할 수 있습니다.

## 문제 해결 CTA의 주요 장점

새로운 개선사항은 지원팀에 의존하지 않고 Facebook 연동을 자신 있게 관리할 수 있도록 도와줍니다. 중요한 이유는 다음과 같습니다:

- 권한 및 폼 매핑 문제의 자체 해결을 간소화합니다.
- 폼이 올바르게 매핑되고 권한이 최신 상태인지 확인하여 놓친 리드를 줄입니다.
- 수동 동기화보다는 자동 리드 동기화의 적절한 사용을 권장합니다.
- 더 나은 집중도와 사용성을 위한 깔끔한 UI를 제공합니다.

## Facebook 연동에서 문제 해결 CTA 사용법

새로운 문제 해결 CTA를 사용하여 Facebook 연동 문제를 해결하는 단계별 가이드입니다:

- 계정의 Facebook Integration 카드를 엽니다.
- 업데이트된 레이아웃을 확인하세요. Settings(설정)와 Disconnect(연결 해제) 버튼이 깔끔한 보기를 위해 점 3개 메뉴로 이동되었습니다.
- 문제가 있는 경우:
  - Facebook 폼이 매핑되지 않았으면 경고 아이콘이 표시됩니다.
  - 클릭하면 문제를 해결하기 위해 Facebook Form Field Mapping(Facebook 폼 필드 매핑)으로 이동합니다.
  - 권한이 누락된 경우 다른 경고 아이콘이 표시됩니다.
  - 클릭하면 연결 문제를 해결하라는 메시지가 나타납니다.
- Troubleshoot(문제 해결)를 클릭하여 가이드 문제 해결 과정으로 진입합니다.
- 문제 해결 과정에서:
  - 강조 표시된 권한 문제나 매핑 경고를 검토합니다.
  - 필요한 경우 Sync Leads(리드 동기화) 옵션을 사용하여 처리되지 않은 리드를 처리합니다.
- 리드 동기화 사용 시:
  - 드롭다운에서 시간 범위를 선택합니다(예: 지난 7일, 30일).
  - 검토를 위해 동기화될 내용의 요약을 보여주는 모달이 나타납니다.
  - 확인하여 하위 계정에 연결된 모든 Facebook 페이지의 모든 매핑된 폼에 대해 리드 동기화를 시작합니다.
- 동기화 후 Facebook 리드가 예상대로 CRM에 나타나는지 확인합니다.

## 중요 참고사항

- 리드 동기화가 의도적으로 문제 해결 메뉴 하위로 이동된 이유는 수동 워크플로우로 오용되는 것을 방지하기 위해서입니다. 리드가 자동으로 동기화되지 않을 때만 사용해야 합니다.
- 이제 단일 페이지 동기화가 아닌 하위 계정의 모든 페이지와 매핑된 폼을 한 번에 동기화하는 기능을 지원합니다.
- 경고 아이콘은 해결되지 않은 문제를 직접 안내하여 혼란과 지원 요청을 줄입니다.
- Facebook 연동은 적절한 폼 매핑과 권한에 의존하므로, 지속적인 리드 동기화를 위해 이러한 경고를 해결하는 것이 중요합니다.

## 자주 묻는 질문

**Q: 리드 동기화가 왜 문제 해결 메뉴로 이동되었나요?**
많은 사용자가 근본 문제를 해결하기보다는 수동 동기화 단계로 의존했기 때문입니다. 문제 해결 메뉴로 이동함으로써 이것이 문제 해결 작업임을 강조합니다.

**Q: 리드 동기화를 사용하지 않고도 리드가 자동으로 동기화되나요?**
네! 폼 매핑과 권한이 올바른 한 리드는 자동으로 동기화됩니다.

**Q: 단일 페이지에 대해서만 리드를 동기화할 수 있나요?**
아니요. 새로운 디자인은 리드 누락을 방지하기 위해 의도적으로 모든 페이지의 모든 매핑된 폼을 동기화합니다.

**Q: 폼이 매핑되지 않으면 어떻게 되나요?**
경고 아이콘이 표시됩니다. 클릭하면 Facebook 폼 필드를 매핑하는 페이지로 바로 이동합니다.
