# 워크플로우에서 Facebook 컨버전 API 트리거

워크플로우를 사용하여 자동화에 Facebook 컨버전 이벤트 액션을 추가할 수 있습니다. 이를 통해 Facebook 픽셀 쿠키 없이도 컨버전 데이터를 Facebook으로 다시 전송할 수 있어요!

## 자주 묻는 질문

**Q: FB 컨버전 이벤트와 함께 사용할 수 있는 초기 워크플로우 트리거는 무엇인가요?**

- Facebook 컨버전 API 액션의 퍼널 이벤트의 경우, 다음 트리거를 사용할 수 있습니다: 폼 제출, 설문 제출, 고객 예약 완료, 주문 폼 제출. (예약의 경우 "고객 예약 완료"에서만 작동하며 일반적인 "예약" 트리거로는 작동하지 않습니다. "고객 예약 완료"는 위젯용 트리거입니다. 자세한 내용은 여기를 참조하세요.)
- Facebook 컨버전 API 액션의 리드 이벤트의 경우, "Facebook 리드 폼 제출"과 "파이프라인 단계 변경" 트리거만 사용할 수 있습니다. (연락처가 Facebook 리드 폼에서 유입된 경우에만 작동합니다.)

**Q: 사용해야 하는 이벤트 세부정보 파라미터는 무엇인가요?**

이벤트 소스 URL

**Q: 사용할 수 있는 고객 정보 파라미터는 무엇인가요?**

- 클라이언트 IP 주소 - 해시화하지 않음
- 클라이언트 사용자 에이전트 - 해시화하지 않음
- 이메일 주소
- 이름
- 성
- 브라우저 ID (fbp) 쿠키 – 해시화하지 않음
- 클릭 ID (fbc) 쿠키 – 해시화하지 않음

**Q: 액세스 토큰과 픽셀 ID에 커스텀 값을 사용할 수 있나요?**

네, 해당 필드에서 커스텀 값이 작동합니다.

**Q: "주문 폼 제출" 트리거에서 {{Order.Amount}} 커스텀 값이 작동하지 않는 이유는 무엇인가요?**

{{Order.Amount}} 커스텀 값은 완료된 결제에 의존하므로 주문 제출 트리거에서만 작동합니다. 주문 폼 제출에서는 작동하지 않는데, 이 트리거는 결제 처리 전에 발생하므로 주문 금액이 아직 확정되지 않았기 때문입니다.

**Q: Facebook 비즈니스 매니저에서 테스트 이벤트가 보이지 않는 이유는 무엇인가요?**

테스트 컨버전이 보이지 않으면 진단(Diagnostics) 탭에서 오류를 확인해보세요. 일반적인 문제는 Facebook이 도메인을 차단한 경우입니다. 설정(Settings) → 하단으로 스크롤 → "허용 목록의 도메인(Domains In Your Allow List)"에서 도메인을 승인할 수 있습니다.

**Q: '워크플로우 테스트' 버튼이 FB 컨버전 테스트에서 작동하나요?**

네

**Q: "리드"를 선택했는데 Facebook에서 컨버전이 "커스텀 이벤트"로 보고되는 이유는 무엇인가요?**

이는 테스트 데이터를 보낼 때 발생합니다(Facebook이 이렇게 하는 이유는 확실하지 않습니다). 하지만 실제 컨버전을 실행할 때는 "리드"로 표시됩니다.

**Q: 'Call', 'opportunity change status', 'tag' 등과 같은 트리거 이름으로 오프라인 이벤트를 Facebook 컨버전 API 액션과 함께 사용할 수 있나요?**

네, 가능합니다. 이 이벤트를 전송하기 위해 마지막으로 가능한 픽셀 데이터를 사용합니다. 기본적으로 fbclid ID가 발견되면 데이터가 컨버전 API로 전송됩니다. 다음 예시로 설명할 수 있습니다:

**예시 1:**

Facebook 폼 제출로 연락처가 생성되어 첫 번째 어트리뷰션 소스가 유료 소셜(Facebook)이 됩니다. Facebook 폼 제출 트리거로 워크플로우를 사용하면 연락처에 fbclid가 있게 되고 워크플로우는 컨버전 API(CAPI)로 데이터를 전송합니다.

추가 동작 - 나중에 해당 연락처가 기회 추가/업데이트 트리거를 통해 기회로 전환된 경우, 워크플로우에서 기회 상태 트리거를 사용하여 컨버전 API로 데이터를 전송할 수 있습니다. Facebook 폼 제출에서 유입된 연락처는 fbclid를 전달하기 때문입니다.

**예시 2:**

Google 광고, 자연 검색 또는 직접 트래픽으로 연락처가 생성되어 첫 번째 어트리뷰션 소스가 유료 검색(Google) 또는 직접 트래픽이 됩니다. 나중에 해당 연락처가 Facebook 폼과 상호작용하여 폼을 작성하면, 최신 어트리뷰션은 유료 소셜(Facebook)이 되고 연락처에 fbclid가 생깁니다. 이제 에이전시가 Facebook 폼 제출 → Facebook 컨버전 API로 워크플로우를 실행하면 fbclid와 함께 컨버전 API로 데이터를 전송합니다.

추가 동작 - 나중에 해당 연락처가 기회 추가/업데이트 트리거를 통해 기회로 전환된 경우, 워크플로우에서 기회 상태 트리거를 사용하여 컨버전 API로 데이터를 전송할 수 있습니다. Facebook 폼 제출에서 유입된 연락처는 fbclid를 전달하기 때문입니다.
