## Zoom 연동이란?

CRM과 Zoom 연동을 통해 사용자는 플랫폼 내에서 직접 Zoom 미팅을 예약하고, 시작하고, 관리할 수 있습니다. 이를 통해 사용자에게 원활한 경험을 제공하며, 서로 다른 애플리케이션과 플랫폼 간을 전환할 필요가 없어집니다.

CRM 내에서 Zoom 연동을 설정하면, 새로 예약되는 각 미팅마다 고유한 Zoom 미팅 링크를 동적으로 생성할 수 있습니다. 사용자 예약 가능 시간 및 캘린더 설정에서 미팅 위치를 Zoom으로 설정하면 이 과정이 자동으로 진행됩니다.

이러한 고유 미팅 링크 생성은 보안을 강화하고 미팅 중 원하지 않는 방해를 줄여주는 중요한 기능입니다. 각 참가자가 서로 다른 링크를 받기 때문에 무단 접근을 어렵게 만듭니다.

예를 들어, 고객과 미팅을 예약할 때 Zoom을 미팅 위치로 선택하면, CRM이 자동으로 새 Zoom 미팅을 생성하고 고유 미팅 링크를 캘린더 이벤트에 포함시킵니다. 고객은 고유한 Zoom 링크가 포함된 미팅 초대를 받게 됩니다. 이 링크는 해당 단일 미팅을 위한 것으로, 모든 관련자에게 편의성과 보안을 제공합니다.

또한 Zoom 연동을 통해 CRM 내에서 이러한 모든 미팅을 추적할 수 있습니다. 예약된 모든 Zoom 미팅이 플랫폼 내에 자동으로 기록되어 약속 관리와 참여 추적이 쉬워집니다.

### 사용 사례

**팀 미팅**: Zoom 연동으로 팀 협업을 쉽게 만드세요. 미팅이 설정될 때마다 고유한 Zoom 링크가 자동으로 생성되어 팀 내 커뮤니케이션을 간소화합니다.

**고객 상담**: 이 연동으로 고객 상담을 간소화하세요. 고객이 미팅을 예약할 때마다 고유한 Zoom 링크가 자동으로 생성되어 상호작용에 전문성과 편의성을 더합니다.

**웨비나**: Zoom 연동으로 웨비나 경험을 향상시키세요. 참가자가 등록하면 개인화된 Zoom 링크가 자동으로 생성되어 웨비나 관리를 원활하고 효율적으로 만듭니다.

**가상 오피스 아워**: 교사와 교수는 Zoom 연동을 사용하여 오피스 아워나 추가 도움 세션을 효율적으로 관리할 수 있습니다. 학생이 시간대를 예약하면 고유한 Zoom 미팅이 생성되어 일정 조정의 번거로움을 없앱니다.

**온라인 피트니스 클래스**: 피트니스 강사는 이 연동을 활용하여 손쉽게 수업을 관리할 수 있습니다. 고객이 세션을 예약할 때마다 고유한 Zoom 링크가 전송되어 원활한 가상 피트니스 경험을 제공합니다.

**가상 부동산 투어**: Zoom 연동으로 비즈니스를 변화시키세요. 잠재 구매자가 투어를 예약하면 시스템이 고유한 Zoom 미팅을 생성하여 부동산 관람을 그들의 편안한 공간으로 가져다줍니다.

**원격 의료 예약**: Zoom 연동으로 의료 서비스를 더욱 접근 가능하게 만드세요. 환자가 예약을 하면 고유한 Zoom 미팅이 생성되어 의료 서비스를 집으로 더 가까이 가져다줍니다.

**온라인 요리 수업**: Zoom 연동으로 요리 수업 경험을 시작하세요. 참가자가 등록하면 개인화된 Zoom 링크가 생성되어 요리 전문 지식을 그들의 주방으로 가져다줍니다.

### 이 기능의 장점:

다음은 Zoom 연동 기능의 기술적 장점들입니다:

**자동 미팅 예약**: 연동이 설정되면 각 미팅에 대해 고유한 Zoom 링크가 자동으로 생성됩니다. 이는 수동 설정에 상당한 시간을 절약하고 오류를 방지합니다.

**중앙화된 관리**: 연동을 통해 사용자는 예약, 일정 변경, 취소 등 미팅의 모든 측면을 CRM에서 직접 관리할 수 있습니다. 이러한 중앙화는 효율성을 향상시키고 잘못된 의사소통 가능성을 줄입니다.

**다중 사용자 접근성**: 참가자 수에 관계없이 각 참가자에 대해 고유한 Zoom 링크가 자동으로 생성됩니다. 이는 보안을 보장하고 전체적인 미팅 경험을 향상시킵니다.

**캘린더 동기화**: 연동을 통해 Zoom 미팅 예약이 CRM 캘린더와 직접 동기화됩니다. 이 기능은 중복 예약을 방지하고 시간 관리를 향상시킵니다.

**실시간 업데이트**: 일정 변경이나 취소와 같이 CRM에서 이루어진 변경 사항이 Zoom 플랫폼에 실시간으로 반영되어 불일치를 줄이고 전체적인 조정을 향상시킵니다.

**보안 강화**: 각 미팅은 고유한 링크를 가지므로 초대받지 않은 손님이 미팅에 참여할 위험을 줄입니다.

**자동 알림**: 연동은 참가자에게 미팅 세부사항과 변경사항에 대한 알림을 자동으로 전송합니다. 이는 메시지 전송의 수동 작업을 줄이고 미팅 참석률을 향상시킵니다.

**데이터 추적**: 연동을 통해 참석률, 미팅 시간 등을 포함한 미팅 데이터의 종합적인 추적이 가능합니다. 이는 패턴 분석과 비즈니스 프로세스 개선에 도움이 될 수 있습니다.

## 연동 사전 요구사항

• 모든 사용자가 캘린더용 Zoom 연동 사용을 시작할 수 있습니다

• 각 사용자는 자신의 Zoom 계정을 자신의 프로필에 연결하며, 사용자가 배정된 각 그룹 캘린더의 모든 예약에서 고유 링크가 생성됩니다

• 동적 Zoom 링크는 라운드 로빈 및 공동 예약 캘린더에서만 사용할 수 있습니다.

### 여러 사용자가 같은 Zoom 계정 연동하기

Zoom 연동은 이메일 주소가 아닌 백엔드의 고유 사용자 ID와 연결됩니다. Zoom 연결은 이제 여러 사용자와 여러 로케이션에서 같은 Zoom 계정 사용을 지원합니다. 공유 연결을 관리하려면 `Calendars(캘린더) → Settings(설정) → Connections(연결) → Video Conferencing(화상회의)`를 사용하세요.

이는 같은 에이전시 내에서 서로 다른 이메일 주소를 가진 사용자나 여러 에이전시에서 Zoom 계정을 연결해야 하는 사용자에게 문제를 일으킬 수 있습니다.

특정 시나리오에서, 사용자 계정이 삭제되고 고객지원의 도움으로 복원되면, 같은 이메일 주소에 두 개의 사용자 ID가 연결되는 결과를 초래할 수 있습니다. 새로운 사용자 ID로 인해 복원된 계정이 원래 Zoom 계정에 재연결할 수 없어 추가 문제로 이어질 수 있습니다.

시스템 내에서 사용자 ID와 이메일 주소가 서로 다른 식별자로 취급되며, 이것이 Zoom과의 연동의 기반이 된다는 것을 기억하는 것이 중요합니다. 연동에서 고유 식별자는 이메일 주소가 아닌 사용자 ID라고 생각하는 것이 필수입니다.

이것이 복잡해 보일 수 있지만, 이 구분을 이해하면 잠재적인 연동 문제를 방지하고 좌절감을 완화하는 데 도움이 됩니다.

더 나은 이해를 위해 몇 가지 예를 들어보겠습니다.

"ABC 마케팅"이라는 에이전시를 생각해보세요. 세 명의 사용자가 있습니다:

사용자 A: 사용자 ID "A001", 이메일 "a@abcmarketing.com"

사용자 B: 사용자 ID "B002", 이메일 "b@abcmarketing.com"

사용자 C: 사용자 ID "C003", 이메일 "c@abcmarketing.com"

사용자들은 에이전시 플랫폼의 각자 프로필에 Zoom 계정을 연동하려고 합니다.

사용자 A가 사용자 ID "A001"로 Zoom 계정을 성공적으로 연결했습니다. 어떤 이유로 사용자 A의 프로필이 삭제되고 고객이 고객지원에 연락하여 복원됩니다. 복원 시, 사용자 A는 새로운 사용자 ID "A004"를 받지만 이메일은 "a@abcmarketing.com"으로 그대로 남아 있습니다.

여기서 혼란이 시작될 수 있습니다. 사용자 A가 Zoom 계정을 재연결하려고 시도합니다. 하지만 그들의 Zoom 계정은 여전히 이전 사용자 ID "A001"에 연결되어 있습니다. 따라서 새로운 사용자 ID "A004"로 연결을 시도하면 시스템이 이를 차단합니다. 시스템 관점에서 보면 Zoom 계정이 이미 다른 사용자 ID에 연결되어 있기 때문입니다.

이것이 Zoom 연동이 이메일 주소가 아닌 사용자 ID와 연결된다는 것을 기억하는 것이 중요한 이유입니다. 이메일 주소가 동일하더라도 시스템은 Zoom에 연결할 때 사용자 ID를 고유 식별자로 처리합니다.

또 다른 점으로, 사용자 A는 Zoom 계정이 여전히 사용자 ID "A001"에 연결되어 있기 때문에 사용자 B의 프로필에도 자신의 Zoom 계정을 연동할 수 없습니다. 따라서 같은 에이전시 내에 있더라도 이러한 고유 사용자 ID 때문에 Zoom 계정을 공유할 수 없습니다.

이 기본 메커니즘을 이해하면 혼란을 피하고 연동 과정을 쉽게 할 수 있을 것입니다.

참고: 여러 하위 계정에서 같은 사용자에게 서로 다른 Zoom 계정을 연결하거나 여러 사용자가 같은 Zoom 계정을 사용하는 것은 권장하지 않습니다.

## 여러 로케이션과 사용자에서 하나의 Zoom 계정 사용

여러 로케이션에서 같은 Zoom 계정을 연결하고 여러 사용자가 사용할 수 있습니다.

- **로케이션 간 같은 계정**: 여러 로케이션에서 같은 Zoom 계정을 연결할 수 있습니다.

- **사용자 간 같은 계정**: 여러 사용자가 같은 Zoom 계정을 연결하고 사용할 수 있습니다.

- **제한사항**: 사용자는 로케이션당 하나의 Zoom 계정을 연결할 수 있습니다.

### 동적 링크 요구사항

동적 Zoom 링크는 연결된 Zoom 계정이 캘린더에서 미팅 위치로 선택되었을 때만 생성됩니다.

### 연결된 Zoom 계정 제거 또는 변경

다음 경로에서 연결된 Zoom 계정을 제거하거나 변경할 수 있습니다:
`Connections(연결) → Video Conferencing(화상회의)`

- `Calendars(캘린더) → Settings(설정) → Connections(연결) → Video Conferencing(화상회의)`로 이동합니다.

- 기존 Zoom 연결을 제거하거나 다른 Zoom 계정을 연결합니다.

- 동적 Zoom 링크를 생성해야 하는 모든 캘린더의 미팅 위치 설정을 확인합니다.

## 1단계: Zoom 연동 설치

다음 방법으로 예약된 미팅에 대해 Zoom을 설정할 수 있습니다:

- **사용자 연동**
`User Profile(사용자 프로필) > Integrations(연동) > Zoom > Connect(연결)`

- 대화상자에서 Connect to Zoom(Zoom에 연결) 클릭 (아직 연결되지 않은 경우)

- Zoom 로그인 자격 증명을 입력하고 Sign in(로그인) 클릭

- Zoom이 Zoom 계정에 액세스할 수 있도록 권한을 부여하라는 메시지가 표시됩니다. Allow(허용) 클릭

- 플랫폼으로 리디렉션되고, User Profile Integrations(사용자 프로필 연동) 아래에 Zoom이 나타납니다.

- **기본 미팅 위치**
`User Profile(사용자 프로필) > Default Meeting Location(기본 미팅 위치) > Zoom > Connect(연결)`

- 대화상자에서 Connect to Zoom(Zoom에 연결) 클릭 (아직 연결되지 않은 경우)

- Zoom 로그인 자격 증명을 입력하고 Sign in(로그인) 클릭

- Zoom이 Zoom 계정에 액세스할 수 있도록 권한을 부여하라는 메시지가 표시됩니다. Allow(허용) 클릭

- 플랫폼으로 리디렉션되고, User Profile Integrations(사용자 프로필 연동) 아래에 Zoom이 나타납니다.

- 그런 다음 드롭다운에서 'Zoom'을 선택하고 'Save(저장)'을 누르면 다른 모든 캘린더의 기존 미팅 위치를 Zoom으로 기본 설정할지 확인하는 대화상자가 표시됩니다.

- **캘린더 설정**
`Settings(설정) > Calendar Settings(캘린더 설정) > User [x] > Meeting Location(미팅 위치)`

- 대화상자에서 Connect to Zoom(Zoom에 연결) 클릭 (아직 연결되지 않은 경우)

- Zoom 로그인 자격 증명을 입력하고 Sign in(로그인) 클릭

- Zoom이 Zoom 계정에 액세스할 수 있도록 권한을 부여하라는 메시지가 표시됩니다. Allow(허용) 클릭

- 플랫폼으로 리디렉션되고, User Profile Integrations(사용자 프로필 연동) 아래에 Zoom이 나타납니다.

- 그런 다음 드롭다운에서 'Zoom'을 선택하면 모든 예약에 Zoom 링크가 포함된 미팅을 만들 수 있습니다.

참고: 프로필 설정 연동 옵션을 사용하려면 해당 사용자로 로그인되어 있어야 합니다. 하위 계정 레벨 설정에서 Profile 설정 옵션을 찾을 수 없다면, 에이전시 레벨 팀 설정에서 사용자 역할을 편집하여 해당 사용자를 특정 로케이션에 배정해야 합니다.

## 관리자가 다른 사용자의 Zoom 설정하기

해당 사용자가 이미 계정에 Zoom 연동을 설치했다면, 팀 관리에서 해당 사용자의 기본 미팅 위치로 Zoom을 선택할 수 있습니다.

참고: 각 사용자는 현재 자신의 계정에 로그인되어 있어야 Zoom을 연동할 수 있습니다. 에이전시 관리자는 이제 ["사용자로 로그인" 기능](login-as-user-agency-admin-only-.md)을 사용하여 에이전시의 모든 사용자에 대해 이 과정을 완료할 수 있습니다.

## 2단계: Zoom 위치로 예약 알림 워크플로우 설정

워크플로우 빌더는 많은 용도가 있는 강력한 도구입니다. 이 사용 사례에서는 동적 Zoom 링크를 포함한 다가오는 예약의 내부 및 외부 알림을 자동화하는 쉬운 액세스를 제공합니다. 다음 트리거와 액션을 조합하여 워크플로우를 만들거나 사용 가능한 템플릿 레시피 중 하나를 사용할 수 있습니다.

- "Add New Workflow Trigger(새 워크플로우 트리거 추가)" 클릭

![트리거 추가 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000206660/original/S3Jj3DZ7tw5oVVrf675riHl1iYjyzSLZsA.png?1685555017)

- "Appointment Status(예약 상태)"를 트리거로 선택하고 다음 필터를 추가합니다:

Event Type: 일회성 예약인지, 반복 예약인지, 또는 둘 중 어느 것이든 상관없이 실행할 것인지

- Appointment Status: 예약 자동 확인을 위한 캘린더 설정은 모든 예약이 이 필터를 트리거하도록 보장합니다

- In Calendar: 특정 캘린더와 특정 사용자를 세분화할 때 선택사항입니다.
