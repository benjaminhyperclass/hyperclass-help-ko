## Zoom 연동이란?

이 CRM과의 Zoom 연동을 통해 사용자는 플랫폼 내에서 직접 Zoom 미팅을 예약하고, 시작하고, 관리할 수 있습니다. 이는 사용자에게 원활한 경험을 제공하여 다른 애플리케이션과 플랫폼 간 전환할 필요를 없앱니다.

CRM 내에서 Zoom 연동이 설정되면, 예약되는 각 새로운 미팅에 대해 고유한 Zoom 미팅 링크를 자동으로 생성할 수 있습니다. 이 과정은 사용자 예약 가능 시간(availability)과 캘린더 설정에서 미팅 위치가 Zoom으로 설정되어 있을 때 자동으로 이루어집니다.

이 고유한 미팅 링크 생성은 보안을 강화하고 미팅 중 원치 않는 방해를 줄여주는 중요한 기능입니다. 각 참가자가 다른 링크를 받게 되어 권한 없는 사람이 접근하기 어렵게 만듭니다.

예를 들어, 고객과 미팅을 예약할 때 미팅 위치로 Zoom을 선택하면, CRM이 자동으로 새로운 Zoom 미팅을 생성하고 고유한 미팅 링크를 캘린더 이벤트에 포함시킵니다. 고객은 고유한 Zoom 링크가 포함된 미팅 초대를 받게 됩니다. 이 링크는 해당 단일 미팅만을 위한 것으로, 모든 관련자에게 편의성과 보안을 제공합니다.

또한 Zoom 연동을 통해 CRM 내에서 이 모든 미팅을 추적할 수 있습니다. 예약된 모든 Zoom 미팅은 플랫폼 내에 자동으로 문서화되어 예약 관리와 참여 추적을 더욱 쉽게 만듭니다.

### 사용 사례

**팀 미팅**: Zoom 연동으로 팀 협업을 쉽게 만드세요. 미팅이 설정될 때마다 고유한 Zoom 링크가 자동으로 생성되어 팀 내 커뮤니케이션을 간소화합니다.

**고객 상담**: 이 연동으로 고객 상담을 간소화하세요. 고객이 미팅을 예약할 때마다 자동으로 고유한 Zoom 링크를 생성하여 상호작용에 전문성과 편의성을 더합니다.

**웨비나**: Zoom 연동으로 웨비나 경험을 향상시키세요. 참가자가 등록하면 개인화된 Zoom 링크가 자동으로 생성되어 웨비나 관리를 원활하고 효율적으로 만듭니다.

**가상 상담 시간**: 교사와 교수는 Zoom 연동을 사용하여 상담 시간이나 추가 도움 세션을 효율적으로 관리할 수 있습니다. 학생이 시간을 예약하면 고유한 Zoom 미팅이 생성되어 예약 번거로움을 없앱니다.

**온라인 피트니스 수업**: 피트니스 강사는 이 연동을 활용해 수업 관리를 쉽게 할 수 있습니다. 고객이 세션을 예약할 때마다 고유한 Zoom 링크가 전송되어 부드러운 가상 피트니스 경험을 제공합니다.

**가상 부동산 투어**: Zoom 연동으로 비즈니스를 변화시키세요. 잠재 구매자가 투어를 예약하면 시스템이 고유한 Zoom 미팅을 생성하여 편안한 공간에서 부동산을 볼 수 있게 합니다.

**원격 진료 예약**: Zoom 연동으로 의료 서비스를 더욱 접근 가능하게 만드세요. 환자가 예약을 하면 고유한 Zoom 미팅이 생성되어 의료 서비스를 집으로 가져다 줍니다.

**온라인 요리 수업**: Zoom 연동으로 요리 수업 경험을 시작하세요. 참가자가 등록하면 개인화된 Zoom 링크가 생성되어 요리 전문 지식을 그들의 주방으로 가져다 줍니다.

### 이 기능의 장점:

다음은 Zoom 연동 기능의 기술적 장점들입니다:

**자동 미팅 예약**: 연동이 설정되면 각 미팅에 대해 고유한 Zoom 링크가 자동으로 생성됩니다. 이는 수동 설정에 드는 상당한 시간을 절약하고 오류를 방지합니다.

**중앙화된 관리**: 연동을 통해 사용자는 예약, 일정 변경, 취소를 포함한 미팅의 모든 측면을 CRM에서 직접 관리할 수 있습니다. 이러한 중앙화는 효율성을 개선하고 의사소통 오류 가능성을 줄입니다.

**다중 사용자 접근성**: 참가자 수에 관계없이 각 참가자에 대해 고유한 Zoom 링크가 자동으로 생성됩니다. 이는 보안을 보장하고 전반적인 미팅 경험을 개선합니다.

**캘린더 동기화**: 연동을 통해 Zoom 미팅 예약을 CRM 캘린더와 직접 동기화할 수 있습니다. 이 기능은 이중 예약을 방지하고 시간 관리를 향상시킵니다.

**실시간 업데이트**: 일정 변경이나 취소와 같은 CRM의 변경사항이 Zoom 플랫폼에 실시간으로 반영되어 불일치를 줄이고 전반적인 조정을 개선합니다.

**보안 강화**: 각 미팅마다 고유한 링크가 있어 초대받지 않은 손님이 미팅에 참여할 위험을 줄입니다.

**자동 알림**: 연동이 미팅 세부사항과 변경사항을 포함한 알림을 참가자에게 자동으로 보냅니다. 이는 메시지 전송의 수동 작업을 줄이고 미팅 참석률을 향상시킵니다.

**데이터 추적**: 연동을 통해 참석률, 미팅 시간 등을 포함한 포괄적인 미팅 데이터 추적이 가능합니다. 이는 패턴 분석과 비즈니스 프로세스 개선에 도움이 될 수 있습니다.

## 연동 사전 요구사항

• 모든 사용자가 캘린더용 Zoom 연동을 사용할 수 있습니다

• 각 사용자는 자신의 Zoom 계정을 자신의 프로필에 연결하고, 사용자가 배정된 각 그룹 캘린더의 모든 예약에서 고유한 링크가 생성됩니다

• 동적 Zoom 링크는 라운드 로빈(Round Robin)과 공동 예약(Collective) 캘린더에서만 사용할 수 있습니다.

### 여러 사용자와 같은 Zoom 계정 연동하기

Zoom 연동은 이메일 주소가 아닌 백엔드의 고유 사용자 ID에 연결됩니다. Zoom 연결은 이제 같은 Zoom 계정을 여러 사용자와 여러 로케이션에서 사용할 수 있도록 지원합니다. 공유 연결을 관리하려면 캘린더(Calendars) → 설정(Settings) → 연결(Connections) → 화상 회의(Video Conferencing)를 사용하세요.

이는 같은 에이전시 내에서 다른 이메일 주소를 가진 사용자나 여러 에이전시에서 Zoom 계정을 연결해야 하는 사용자에게 문제를 일으킬 수 있습니다.

특정 시나리오에서, 사용자 계정이 삭제되고 지원팀의 도움으로 복원되면, 같은 이메일 주소와 연관된 두 개의 사용자 ID가 생길 수 있습니다. 복원된 계정이 새로운 사용자 ID로 인해 원래 Zoom 계정에 재연결할 수 없게 되어 추가적인 복잡성을 야기할 수 있습니다.

시스템 내에서 사용자 ID와 이메일 주소는 별개의 식별자로 취급되며, 이는 Zoom과의 연동의 기반이 됩니다. 연동에 있어서는 이메일 주소가 아닌 사용자 ID를 고유 식별자로 고려하는 것이 중요합니다.

이는 복잡해 보일 수 있지만, 이 구분을 이해하면 잠재적인 연동 문제를 피하고 좌절감을 덜 수 있습니다.

더 나은 이해를 위해 몇 가지 예를 들어보겠습니다.

"ABC 마케팅"이라는 에이전시를 고려해보세요. 그들은 세 명의 사용자가 있습니다:

사용자 A - 사용자 ID "A001", 이메일 "a@abcmarketing.com"

사용자 B - 사용자 ID "B002", 이메일 "b@abcmarketing.com"

사용자 C - 사용자 ID "C003", 이메일 "c@abcmarketing.com"

사용자들은 에이전시 플랫폼의 각자 프로필에 Zoom 계정을 연동하고 싶어합니다.

사용자 A가 사용자 ID "A001"로 Zoom 계정을 성공적으로 연결했다고 가정합니다. 어떤 이유로 사용자 A의 프로필이 삭제되고 고객이 지원팀에 문의하여 복원됩니다. 복원 시 사용자 A는 새로운 사용자 ID, 예를 들어 "A004"를 받지만, 이메일은 "a@abcmarketing.com"으로 그대로 남습니다.

여기서 혼란이 시작될 수 있습니다. 사용자 A가 Zoom 계정을 재연결하려고 시도합니다. 하지만 그들의 Zoom 계정은 여전히 이전 사용자 ID "A001"에 연결되어 있습니다. 따라서 새로운 사용자 ID "A004"로 연결하려고 시도하면, 시스템이 이를 차단합니다. 시스템 관점에서는 Zoom 계정이 이미 다른 사용자 ID에 연결되어 있기 때문입니다.

이것이 Zoom 연동이 이메일 주소가 아닌 사용자 ID에 연결된다는 것을 기억하는 것이 중요한 이유입니다. 시스템은 이메일 주소가 같더라도 Zoom에 연결할 때 사용자 ID를 고유 식별자로 취급합니다.

또 다른 점은, 사용자 A는 자신의 Zoom 계정을 사용자 B의 프로필에도 연동할 수 없습니다. Zoom 계정이 여전히 사용자 ID "A001"에 연결되어 있기 때문입니다. 따라서 같은 에이전시 내에 있더라도, 이러한 고유한 사용자 ID로 인해 Zoom 계정을 공유할 수 없습니다.

이러한 기본 메커니즘을 이해하면 혼란을 피하고 연동 프로세스를 수월하게 할 수 있습니다.

참고: 여러 하위 계정에서 같은 사용자에게 다른 Zoom 계정을 연결하거나 여러 사용자가 같은 Zoom 계정을 사용하는 것은 권장되지 않습니다.

## 로케이션과 사용자 전반에서 하나의 Zoom 계정 사용

여러 로케이션에서 같은 Zoom 계정을 연결하고 여러 사용자가 사용할 수 있습니다.

- **로케이션 전반의 같은 계정**: 여러 로케이션에서 같은 Zoom 계정을 연결할 수 있습니다.

- **사용자 전반의 같은 계정**: 여러 사용자가 같은 Zoom 계정을 연결하고 사용할 수 있습니다.

- **제한**: 사용자는 로케이션당 하나의 Zoom 계정을 연결할 수 있습니다.

### 동적 링크 요구사항

동적 Zoom 링크는 연결된 Zoom 계정이 캘린더에서 미팅 위치로 선택되었을 때만 생성됩니다.

### 연결된 Zoom 계정 제거 또는 전환

연결된 Zoom 계정을 다음 경로에서 제거하거나 전환할 수 있습니다:
연결(Connections) → 화상 회의(Video Conferencing)

- 캘린더(Calendars) → 설정(Settings) → 연결(Connections) → 화상 회의(Video Conferencing)로 이동합니다.

- 기존 Zoom 연결을 제거하거나 다른 Zoom 계정을 연결합니다.

- 동적 Zoom 링크를 생성해야 하는 모든 캘린더의 미팅 위치 설정을 확인합니다.

## 1단계: Zoom 연동 설치

예약된 미팅에 대해 Zoom을 다음과 같은 방법으로 설정할 수 있습니다:

**사용자 연동**
사용자 프로필(User Profile) > 연동(Integrations) > Zoom > 연결(Connect)

- 대화상자에서 Zoom에 연결(Connect to Zoom)을 클릭합니다 (이미 연결되지 않은 경우)

- Zoom 로그인 자격 증명을 입력한 후 로그인(Sign in)을 클릭합니다.

- Zoom이 귀하의 Zoom 계정에 접근할 수 있도록 권한을 부여하라는 메시지가 표시됩니다. 허용(Allow)을 클릭합니다.

- 플랫폼으로 리디렉션되며, 사용자 프로필 연동(User Profile Integrations)에 Zoom이 표시됩니다.

**기본 미팅 위치**
사용자 프로필(User Profile) > 기본 미팅 위치(Default Meeting Location) > Zoom > 연결(Connect)

- 대화상자에서 Zoom에 연결(Connect to Zoom)을 클릭합니다 (이미 연결되지 않은 경우)

- Zoom 로그인 자격 증명을 입력한 후 로그인(Sign in)을 클릭합니다.

- Zoom이 귀하의 Zoom 계정에 접근할 수 있도록 권한을 부여하라는 메시지가 표시됩니다. 허용(Allow)을 클릭합니다.

- 플랫폼으로 리디렉션되며, 사용자 프로필 연동(User Profile Integrations)에 Zoom이 표시됩니다.

- 그 다음 드롭다운에서 'Zoom'을 선택하고 '저장(Save)'을 눌러 다른 모든 캘린더의 기존 미팅 위치를 Zoom으로 기본 설정할지 확인하는 대화상자를 표시할 수 있습니다.

**캘린더 설정**
설정(Settings) > 캘린더 설정(Calendar Settings) > 사용자 [x] > 미팅 위치(Meeting Location)

- 대화상자에서 Zoom에 연결(Connect to Zoom)을 클릭합니다 (이미 연결되지 않은 경우)

- Zoom 로그인 자격 증명을 입력한 후 로그인(Sign in)을 클릭합니다.

- Zoom이 귀하의 Zoom 계정에 접근할 수 있도록 권한을 부여하라는 메시지가 표시됩니다. 허용(Allow)을 클릭합니다.

- 플랫폼으로 리디렉션되며, 사용자 프로필 연동(User Profile Integrations)에 Zoom이 표시됩니다.

- 그 다음 드롭다운에서 'Zoom'을 선택하여 모든 예약에 Zoom 링크가 포함되도록 할 수 있습니다.

**참고**: 프로필 설정 연동 옵션을 사용하려면 해당 사용자로 로그인해야 합니다. 하위 계정 레벨 설정에서 프로필(Profile) 설정 옵션을 찾을 수 없다면, 에이전시 레벨 팀 설정에서 사용자의 역할을 편집하여 특정 로케이션에 사용자를 배정해야 합니다.

## 관리자로서 다른 사용자의 Zoom 설정하기

해당 사용자가 이미 계정에 Zoom 연동을 설치했다면, 팀 관리(Team Management)에서 해당 사용자의 기본 미팅 위치로 선택할 수 있습니다.

**참고**: 각 사용자는 Zoom을 연동하려면 현재 자신의 계정에 로그인되어 있어야 합니다. 에이전시 관리자는 이제 "[사용자로 로그인(Login As User) 기능]([login-as-user-agency-admin-only-](login-as-user-agency-admin-only-.md))"을 사용하여 에이전시의 모든 사용자에 대해 이 프로세스를 완료할 수 있습니다.

## 2단계: Zoom 위치로 예약 알림 워크플로우 설정

워크플로우 빌더는 다양한 용도로 사용할 수 있는 강력한 도구입니다. 하지만 이 사용 사례에서는 동적 Zoom 링크를 포함한 다가오는 예약의 내부 및 외부 알림을 자동화할 수 있는 쉬운 접근을 제공합니다. 다음 트리거와 액션을 결합하여 워크플로우를 구축하거나 사용 가능한 템플릿 레시피 중 하나를 사용할 수 있습니다.

- "새 워크플로우 트리거 추가(Add New Workflow Trigger)"를 클릭합니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000206660/original/S3Jj3DZ7tw5oVVrf675riHl1iYjyzSLZsA.png?1685555017)

- 트리거로 "예약 상태(Appointment Status)"를 선택하고 다음 필터를 추가합니다:

이벤트 유형: 일회성 예약인지, 반복 예약인지, 아니면 둘 다에 대해 실행되길 원하는지

- 예약 상태: 자동 예약 확인을 위한 캘린더 설정으로 모든 예약이 이 필터를 트리거하도록 보장

- 캘린더에서: 특정 캘린더와 특정 사용자로 범위를 좁히는 경우 선택사항

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000206662/original/yPxrWPmscg8nb2ajniYeIIK41mZBuhgzNQ.png?1685555017)

- "첫 번째 액션 추가(Add your first Action)"를 클릭합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000206661/original/UKPzULnHxI0XGo-sHcaDI4wjAnTXDNAEFQ.png?1685555017)

- 첫 번째 필요한 액션은 미팅 위치 값이 업데이트되도록 보장하는 1분 시간 지연 "대기(Wait)" 단계입니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000206664/original/iGbrgntY2P6mG-d7jRX-OOtWDozGO88BpQ.png?1685555018)

- 다음 액션 단계는 예약 완료 확인 알림을 시작합니다: "이메일/SMS 보내기(Send Email/SMS)"는 연락처의 기본 이메일과 전화번호로 알림을 보냅니다. 동적 zoom 링크를 제공하기 위해 메시지에 커스텀 값 {{appointment.meeting_location}}을 포함해야 합니다.

커스텀 값/병합 필드 링크는 [여기를 클릭하세요](https://gohighlevelassist.freshdesk.com/support/solutions/articles/48001078171-custom-values-merge-fields).

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155000206666/original/n3MremtNoREea-aDu7-xIMMGxJS8Pxlo3g.png?1685555020)

- 선택적으로, "내부 알림(Internal Notification)" 액션을 사용하여 배정된 사용자나 외부 연락처에게 이메일, SMS, 인앱 예약 알림을 포함할 수 있습니다.
