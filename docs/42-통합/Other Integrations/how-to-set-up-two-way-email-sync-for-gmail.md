# Gmail 양방향 이메일 동기화 설정하는 방법

Gmail 양방향 동기화를 통해 Gmail 계정과 CRM을 연결하고 두 플랫폼 간 이메일을 동기화할 수 있습니다. 이 연동(Integration)은 플랫폼 간 연결을 생성하고 양방향으로 동기화합니다.

- 기존 연락처(Contacts)의 인바운드/아웃바운드 이메일을 Hyperclass로 동기화
- Gmail에서 연락처 생성과 대화(Conversations) 추가를 원활하게 처리
- 실시간 양방향 이메일 동기화로 효율성 향상

중요 참고사항: Gmail 양방향 동기화는 에이전시(Agency) 대시보드(Dashboard) 레벨이 아닌 하위 계정(Sub-account) 내에서 설정해야 합니다. 하위 계정에 직접 로그인하거나 에이전시 뷰에서 "Login As" 기능을 사용하여 하위 계정에 접근해야 합니다. 이렇게 해야 Gmail 연결이 올바른 CRM 로케이션(Location) 내의 대화에 적용됩니다.

## Gmail 양방향 동기화란?

Gmail 양방향 동기화는 Gmail과 CRM 간 이메일의 양방향 동기화를 가능하게 하는 기능입니다. 사용자가 Gmail 계정을 CRM과 연동하여 두 시스템 간 이메일을 동기화 상태로 유지하는 연결을 설정할 수 있습니다. Gmail에서 전송, 수신 또는 업데이트된 모든 이메일이 자동으로 CRM에 반영되며, 그 반대도 마찬가지입니다. 양방향 동기화를 통해 사용자는 플랫폼에 관계없이 통합되고 최신 상태의 이메일 뷰를 확보할 수 있습니다. 이 기능은 이메일 캠페인(Campaign)을 효율화하고 생산성을 향상시키며, 중요한 이메일 커뮤니케이션을 Gmail과 CRM 양쪽에서 모두 접근할 수 있도록 보장합니다.

## CRM과 이메일 계정 간 양방향 동기화 작동 방식

동기화가 설정되는 두 가지 경우가 있습니다:

1. CRM에서 연락처에게 이메일을 보내서 두 플랫폼 간 동기화를 시작할 수 있습니다.
2. Gmail 동기화는 이미 CRM에 저장된 연락처가 Gmail 양방향 동기화를 연동한 사용자의 Gmail 이메일 주소로 이메일을 보내는 경우에도 작동합니다. 해당 이메일은 대화 탭으로 동기화됩니다.

참고: 하위 계정 사용자이기도 한 연락처의 이메일은 동기화되지 않습니다. 종종 하위 계정 사용자들이 다양한 목적으로 연락처로 추가되는데, 그러한 이메일은 기밀 정보를 포함할 수 있어 동기화되지 않습니다. 이메일 주소가 사용자에게 속하지 않으면 위에 언급된 대로 이메일이 동기화됩니다.

참고사항

사용자가 여러 하위 계정에 추가되어 있고 모든 계정에서 동일한 Gmail 계정으로 Gmail 양방향 동기화를 연동한 경우, 연락처의 이메일이 모든 하위 계정의 대화 탭으로 전송되지만, 사용자가 답장하는 하위 계정의 연락처 인스턴스만 유지됩니다. 다른 모든 인스턴스의 동기화는 중단됩니다. 사용자가 기존 연락처에 배정되어야 한다는 추가 필터는 연락처가 속하지 않은 하위 계정으로 대화가 전송되지 않도록 보장합니다.

## 기타 기능

### 이메일 업데이트

이전 연결을 해제하지 않고 연결된 이메일 ID를 다른 것으로 변경할 수 있도록 도와줍니다.

![이메일 업데이트 기능](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48287100704/original/n1NZvUej6xWS0WzV2yaUOKHk26_hCz7Q_w.png?1678792018)

CRM에서 새로 보내는 이메일은 새로 추가된 이메일 주소로 동기화를 시작합니다. 이전에 연결된 이메일 ID의 향후 메시지(동일 스레드)는 CRM과 개인 이메일 간 동기화가 중단됩니다.

### 이메일 연결 해제

연결을 끊고 CRM과의 동기화를 중단할 수 있도록 도와줍니다. 연결 해제 후에는 이메일이나 메시지가 두 플랫폼 간에 동기화되지 않습니다.

![이메일 연결 해제](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48287100962/original/TLimKgIGciSFjVlNRy0jRHtoAQdktaxIpA.png?1678792076)

### BCC 주소

Gmail/Outlook에서 이메일을 보낼 때 BCC 주소를 참조(Cc) 또는 숨은참조(Bcc) 필드에 포함할 수 있습니다. 이렇게 하면 자동으로 연락처와 대화가 CRM에 추가되어 커뮤니케이션을 효율화하고 모든 관련 데이터를 중앙화할 수 있습니다. 향후 이 연락처로부터 Gmail/Outlook 인박스(Inbox) 레벨에서 받는 모든 이메일이 자동으로 CRM과 동기화됩니다.

![BCC 주소 기능](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155003796789/original/y8e4pNvsjXgTN9G3mDxS0Ro04NP8gkUtfA.png?1690445519)

참고사항:

BCC 주소 사용 시 Gmail에서 "전송 불완전" 오류가 발생할 수 있습니다. 이는 로깅 목적으로 BCC를 사용하는데 Gmail이 BCC 주소에 대한 전송 확인서를 받지 못하기 때문입니다. 오류가 발생해도 메시지가 CRM에 성공적으로 게시될 수 있습니다.

## 양방향 동기화는 개별 이메일에만 작동하나요? 대량 이메일이나 워크플로우도 포함되나요?

다양한 이메일 유형에 대한 발신자 도메인 매핑 작동 방식:

개별 이메일: 개인 이메일 계정(Gmail) 연결 시, Gmail 이메일 ID가 사용자가 개별 이메일로 보내는 이메일의 발신자 도메인으로 간주됩니다. 개별 이메일은 사용자에게 양방향 동기화가 활성(Active)화된 경우 Gmail에서 직접 전송됩니다. Google이 일일 500개 이메일 제한을 설정한다는 점을 유의하세요.

대량 이메일: 하위 계정 레벨의 이메일 제공자로부터 계속 전송됩니다. 양방향 동기화는 이에 영향을 주지 않습니다.

워크플로우 및 자동화: 자동화된 이메일의 경우, 양방향 동기화에서 전송되는 것이 아니라 정의된 하위 계정 레벨 제공자로부터 발송됩니다. 이러한 배치는 자동화된 CRM 워크플로우와 이메일 발송 시스템을 원활하게 통합(Integration)합니다.

참고사항:

이 설정으로 사용자는 1:1 이메일에 양방향 동기화를 활용하면서 대량 이메일도 성공적으로 전송할 수 있습니다. Gmail의 일일 500개 이메일 제한은 개별 이메일에만 적용되므로, 사용자는 이 제한에 대한 걱정 없이 LeadConnector/SMTP를 통해 대량 이메일을 보낼 수 있습니다. 이는 사용자가 Gmail의 일일 이메일 한도에 제약받지 않고 대량 이메일 캠페인을 수행할 수 있게 하는 사려 깊은 설계 기능입니다.

## 자주 묻는 질문

### 기존 연락처로부터 받은 이메일은 어떻게 되나요?
기존 연락처로부터의 모든 인바운드 이메일이 CRM과 Gmail/Outlook 인박스에 반영됩니다. 이 양방향 동기화를 통해 두 플랫폼에서 모든 커뮤니케이션의 완전한 기록을 보유할 수 있습니다.

### 동기화된 계정을 제거하면 기존 이메일이 삭제되나요?
아니요, 동기화된 계정을 제거해도 기존 이메일은 삭제되지 않습니다. 이전에 동기화된 이메일은 그대로 유지되며, 동기화는 계정 제거 후 새로 전송 또는 수신되는 이메일에만 영향을 줍니다.

### 계정 제거 후 새 이메일이 동기화되나요?
계정 제거 후 전송 또는 수신되는 새 이메일은 CRM과 Gmail/Outlook 간에 동기화되지 않습니다. 두 플랫폼 간 동기화는 기존 이메일에 대해서는 계속되지만, 새 이메일은 동기화에 포함되지 않습니다.

### Gmail의 모든 기존 연락처가 CRM과 동기화되나요?
예, 양방향 동기화는 Gmail의 기존 연락처를 식별하여 CRM과 동기화합니다. 이러한 연락처로부터의 수신 이메일은 해당 연락처 아래에 자동으로 채워지지만, 과거 이메일(연동 연결 이전)은 동기화될 수 없습니다.

### 연락처로부터 새 이메일을 받으면 어떻게 되나요?
기존 연락처로부터의 새 수신 이메일은 CRM과 Gmail에서 새 이메일로 반영됩니다. 이메일이 기존 대화 스레드에 속하는 경우, 같은 스레드 내의 새 이메일로 표시됩니다.

### CRM이나 Gmail에서 보낸 이메일은 어떻게 되나요?
CRM에서 보낸 발신 이메일은 동기화된 메일의 보낸 메일함에 자동 기록됩니다. 반대로 동기화된 메일 제공자의 이메일은 해당 CRM 연락처의 대화 아래에 자동으로 나타납니다.

### 참조/숨은참조 수신자는 동기화에서 어떻게 처리되나요?
사용자가 Gmail에서 참조나 숨은참조에 포함되거나 여러 수신자에 포함된 경우, 이 정보가 CRM의 해당 연락처 아래에 반영되어 모든 커뮤니케이션의 포괄적인 가시성을 보장합니다. 참조 및 숨은참조 이메일 주소로는 새 연락처가 생성되지 않습니다.

### 연락처가 다른 사람에게 이메일을 보내지만 CRM 사용자를 포함하는 경우는 어떻게 되나요?
이 경우 이메일이 CRM에 반영되어 해당 연락처와 연결되며, 완전한 대화 기록이 유지됩니다.

### 여러 수신자가 관련된 경우 대화 스레드는 어떻게 관리되나요?
이메일에 여러 수신자(받는 사람)가 있는 경우, 첫 번째 연락처가 CRM의 대화 탭이 됩니다. 모든 후속 이메일이 이 스레드에 나타납니다.

### 동일한 동기화된 이메일 주소에 대해 같은 연락처가 여러 로케이션에 존재하는 경우는 어떻게 되나요?
CRM에서 발송되는 이메일은 해당 로케이션에만 반영됩니다. 후속 답장과 커뮤니케이션도 해당 로케이션에만 특정되어야 합니다. 그러나 연락처에서 CRM 사용자로 보내는 이메일은 모든 로케이션에 반영되어야 합니다.

### BCC 주소란 무엇이고 어떻게 작동하나요?
Gmail/Outlook에서 이메일을 보낼 때 BCC 주소를 참조 또는 숨은참조 필드에 포함할 수 있습니다. 이것은 자동으로 대화와 연락처를 CRM에 추가하여 커뮤니케이션과 데이터 관리를 효율화합니다. 향후 이 연락처로부터 Gmail/Outlook 인박스 레벨에서 받는 모든 이메일이 자동으로 CRM과 동기화됩니다.

### Gmail 로그인 자격 증명이 변경되면 보안 연결은 어떻게 처리되나요?
로그인 자격 증명이 변경된 경우, CRM에서 재인증하고 이메일을 재동기화하여 보안 연결을 유지하도록 요구합니다.

### Gmail에서 이메일이 전달되면 어떻게 되나요?
기존 연락처에게 전달된 모든 이메일은 전송된 이메일로 처리되어 CRM의 해당 연락처 아래에 동기화됩니다.

### 여러 연락처에게 보낸 중복 이메일은 어떻게 되나요?
여러 연락처에게 전송된 중복 이메일은 CRM에서 처음 생성된 연락처와 연결됩니다.
