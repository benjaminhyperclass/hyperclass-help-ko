# Hyperclass에서 빠르게 로딩되는 페이지 만드는 방법

웹 페이지의 로딩 속도는 사용자 경험과 검색 엔진 최적화에 매우 중요합니다. Hyperclass의 페이지 빌더(Page Builder)를 사용할 때 다음 가이드라인을 따르면 더 빠른 페이지를 만들 수 있습니다.

## 이미지 최적화

### 적절한 파일 형식 선택
- **JPEG**: 사진이나 복잡한 이미지에 사용
- **PNG**: 투명 배경이 필요한 로고나 아이콘에 사용
- **WebP**: 가능한 경우 WebP 형식 사용 (용량이 작으면서 품질 유지)

### 이미지 크기 최적화
- 실제 표시 크기에 맞는 해상도로 이미지 업로드
- 불필요하게 큰 이미지(예: 2000px 이상)는 피하세요
- 모바일용 이미지는 별도로 최적화된 작은 버전 사용

### 압축 도구 활용
이미지 업로드 전에 다음 도구들로 압축하세요:
- TinyPNG
- JPEGmini
- ImageOptim

## 페이지 구조 최적화

### 섹션(Section) 수 최소화
- 꼭 필요한 섹션만 사용
- 유사한 내용은 하나의 섹션으로 통합

### 불필요한 요소(Element) 제거
- 사용하지 않는 텍스트 블록, 이미지, 버튼 삭제
- 숨겨진 요소도 로딩 시간에 영향을 주므로 완전히 제거

## 외부 리소스 관리

### 폰트 최적화
- Google Fonts는 필요한 스타일과 굵기만 선택
- 너무 많은 폰트 패밀리 사용 지양

### 외부 스크립트 최소화
- 꼭 필요한 추적 코드(Google Analytics, Facebook Pixel 등)만 사용
- 여러 개의 유사한 도구보다는 하나로 통합

## 콘텐츠 최적화

### 텍스트 압축
- 불필요한 공백이나 줄바꿈 제거
- 간결하고 핵심적인 내용으로 구성

### 비디오 최적화
- 직접 업로드보다는 YouTube나 Vimeo 임베드 사용
- 자동 재생 비디오는 페이지 로딩에 큰 영향을 주므로 신중하게 사용

## 성능 확인 도구

페이지 완성 후 다음 도구로 로딩 속도를 확인하세요:

- **Google PageSpeed Insights**: 모바일/데스크톱 성능 분석
- **GTmetrix**: 상세한 성능 리포트 제공
- **Pingdom**: 전 세계 다양한 위치에서 로딩 테스트

## 추가 팁

- 페이지를 단계적으로 구축하면서 중간중간 속도 테스트
- 모바일에서의 로딩 속도도 반드시 확인
- 중요한 콘텐츠는 페이지 상단(Above the fold)에 배치

이러한 최적화를 통해 방문자에게 더 나은 사용자 경험을 제공하고, 검색 엔진 순위도 향상시킬 수 있습니다.
