# 더블 옵트인 플로우를 구축하는 방법

더블 옵트인(Double Opt-In) 플로우는 이메일 마케팅에서 고객이 두 단계에 걸쳐 구독을 확인하도록 하는 중요한 프로세스입니다. 첫 번째 단계에서 고객이 이메일 주소를 입력하면, 두 번째 단계에서 확인 이메일을 통해 구독 의사를 재확인받습니다.

이 가이드에서는 Hyperclass 플랫폼에서 더블 옵트인 워크플로우(Workflow)를 만드는 방법을 단계별로 안내해드리겠습니다.

## 더블 옵트인이 필요한 이유

- **이메일 전달률 향상**: 실제 관심 있는 고객만 구독하므로 스팸 신고율이 낮아집니다
- **법적 컴플라이언스**: GDPR, CAN-SPAM 등 이메일 마케팅 법규를 준수할 수 있습니다
- **고품질 리스트**: 구독 의지가 확실한 고객들로만 이메일 리스트가 구성됩니다
- **발신자 평판 보호**: 이메일 서비스 제공업체로부터 좋은 평가를 받을 수 있습니다

## 1단계: 초기 설정

### 1. 폼(Form) 생성
1. **Sites(사이트) → Forms(폼)** 메뉴로 이동합니다
2. **Create Form(폼 생성)** 버튼을 클릭합니다
3. 이메일 주소 입력 필드를 추가합니다
4. 폼 제출 후 고객에게 "확인 이메일을 보냈습니다"라는 안내 메시지를 표시하도록 설정합니다

### 2. 태그(Tag) 준비
더블 옵트인 프로세스 관리를 위해 다음 태그들을 미리 생성해두세요:
- `pending-confirmation` (확인 대기 중)
- `double-optin-confirmed` (더블 옵트인 확인 완료)

## 2단계: 더블 옵트인 워크플로우 생성

### 1. 새 워크플로우 만들기
1. **Automation(자동화) → Workflows(워크플로우)** 메뉴로 이동합니다
2. **Create Workflow(워크플로우 생성)**을 클릭합니다
3. 워크플로우 이름을 "더블 옵트인 플로우"로 설정합니다

### 2. 트리거(Trigger) 설정
1. **Form Submitted(폼 제출)** 트리거를 선택합니다
2. 1단계에서 만든 폼을 선택합니다
3. 트리거 조건을 저장합니다

### 3. 액션(Action) 추가

#### 액션 1: 태그 추가
1. **Add/Remove Tags(태그 추가/제거)** 액션을 추가합니다
2. `pending-confirmation` 태그를 추가하도록 설정합니다

#### 액션 2: 확인 이메일 발송
1. **Send Email(이메일 발송)** 액션을 추가합니다
2. 이메일 템플릿(Email Template)을 새로 만들거나 기존 템플릿을 선택합니다
3. 이메일 내용에는 다음 요소들을 포함합니다:
   - 구독 확인 요청 메시지
   - **확인하기** 버튼 (트리거 링크 사용)
   - 구독을 원하지 않을 경우의 안내

#### 액션 3: 대기 시간 설정
1. **Wait(대기)** 액션을 추가합니다
2. 7일 대기로 설정합니다
3. 이는 확인하지 않은 연락처를 정리하기 위함입니다

#### 액션 4: 조건 분기 설정
1. **If/Else(조건 분기)** 액션을 추가합니다
2. 조건: 연락처에 `double-optin-confirmed` 태그가 **없는 경우**
3. YES 경로: 미확인 연락처 처리
4. NO 경로: 워크플로우 종료

#### 액션 5: 미확인 연락처 처리 (YES 경로)
1. **Add/Remove Tags(태그 추가/제거)** 액션을 추가합니다
2. `pending-confirmation` 태그를 제거합니다
3. 필요 시 `unconfirmed-lead` 태그를 추가할 수 있습니다

## 3단계: 확인 트리거 링크 설정

### 1. 트리거 링크(Trigger Link) 생성
1. **Marketing(마케팅) → Trigger Links(트리거 링크)** 메뉴로 이동합니다
2. **Create Trigger Link(트리거 링크 생성)**을 클릭합니다
3. 링크 이름을 "더블 옵트인 확인"으로 설정합니다

### 2. 확인 워크플로우 생성
1. 새 워크플로우를 생성합니다: "더블 옵트인 확인 완료"
2. **Link Click(링크 클릭)** 트리거를 선택합니다
3. 방금 만든 트리거 링크를 선택합니다

### 3. 확인 완료 액션 추가

#### 액션 1: 태그 업데이트
1. **Add/Remove Tags(태그 추가/제거)** 액션을 추가합니다
2. `pending-confirmation` 태그를 제거합니다
3. `double-optin-confirmed` 태그를 추가합니다

#### 액션 2: 환영 이메일 발송 (선택사항)
1. **Send Email(이메일 발송)** 액션을 추가합니다
2. 구독 확인 완료 및 환영 메시지를 포함한 이메일을 발송합니다

#### 액션 3: 메인 이메일 캠페인 추가 (선택사항)
1. **Add to Campaign(캠페인 추가)** 액션을 사용하여 기존 이메일 캠페인(Email Campaign)에 연락처를 추가할 수 있습니다

## 4단계: 확인 이메일 템플릿 작성

확인 이메일에는 다음 내용을 포함해야 합니다:

```
제목: [회사명] 구독 확인 요청

안녕하세요!

[회사명]의 뉴스레터 구독 신청을 해주셔서 감사합니다.

구독을 완료하시려면 아래 버튼을 클릭해주세요:

[구독 확인하기 버튼] ← 트리거 링크 연결

본 이메일은 귀하가 직접 신청하신 경우에만 발송됩니다.
구독을 원하지 않으시면 이 이메일을 무시하셔도 됩니다.

감사합니다.
[회사명] 팀
```

## 5단계: 테스트 및 최적화

### 1. 테스트 진행
1. 테스트 이메일 주소로 폼을 제출합니다
2. 확인 이메일이 올바르게 도착하는지 확인합니다
3. 확인 링크를 클릭하여 태그가 정확히 업데이트되는지 확인합니다

### 2. 모니터링 및 개선
- **Reporting(리포팅)** 섹션에서 더블 옵트인 완료율을 추적합니다
- 확인 이메일의 제목과 내용을 A/B 테스트하여 개선합니다
- 대기 시간(7일)을 조정하여 최적화합니다

## 고급 활용 팁

### 1. 세분화된 태그 관리
- 구독 소스별로 다른 태그 적용 (예: `blog-signup`, `webinar-signup`)
- 관심 분야별 세분화를 위한 추가 태그 사용

### 2. 리마인더 이메일
확인 대기 중인 연락처에게 2-3일 후 리마인더 이메일을 발송하는 별도 워크플로우를 만들 수 있습니다.

### 3. 스마트 리스트(Smart List) 활용
- `double-optin-confirmed` 태그가 있는 연락처만 포함하는 스마트 리스트를 생성
- 이메일 캠페인 발송 시 이 리스트만 사용

더블 옵트인 플로우를 올바르게 설정하면 이메일 마케팅의 효과를 크게 향상시킬 수 있습니다. 처음에는 구독자 수가 줄어들 수 있지만, 장기적으로는 더 높은 참여율과 전환율을 얻을 수 있습니다.
