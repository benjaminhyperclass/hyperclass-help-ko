# CSV 파일로 기회 가져오기

이 가이드에서는 CSV 파일을 사용해서 Hyperclass로 기회를 가져오는 방법을 안내합니다. 데이터 준비, 필드 매핑, 원활한 가져오기 과정을 통해 영업 파이프라인을 체계적으로 최신 상태로 유지하는 방법을 배워보세요.

**중요**: Hyperclass에서는 CSV(.csv) 파일 형식만 지원합니다. Excel이나 Google Sheets 같은 다른 파일 형식은 사용할 수 없어요. 자세한 내용은 [연락처 및 기회 가져오기용 CSV 파일 형식](https://hyperclass.gitbook.io/hyperclass-docs) 가이드를 참고하세요.

## 연락처 가져오기 사전 요구사항

가져오기를 시작하기 전에 다음 조건을 충족하는지 확인하세요:

- 사용자 권한: Admin(관리자) 권한이 있는 사용자만 연락처를 가져올 수 있습니다.

- 파일 형식: CSV(.csv) 형식이어야 합니다.

- 파일 크기 제한: CSV 파일은 30MB 미만이어야 합니다. 더 크면 작은 파일로 나누세요.

- 단일 시트 요구사항: CSV 파일은 하나의 시트/탭만 포함해야 합니다.

- CSV 요구사항: 첫 번째 행은 비어있으면 안 됩니다! 최소한 하나의 컬럼 헤더가 시스템 필드와 일치하는 헤더 행을 포함하세요.

## 피해야 할 일반적인 실수

❌ 헤더 불일치: CSV 헤더가 시스템 필드와 일치하는지 확인하세요.

❌ 필수 필드 공백: 모든 행에는 최소한 하나의 필수 필드(이름, 이메일, 전화번호)가 있어야 합니다.

❌ 전화번호에 특수문자 사용: 공백, 대시, 문자를 제거하세요.

❌ CSV 내 중복 레코드: 가져오기 전에 목록을 정리하세요.

## CSV로 기회 가져오기

기회 가져오기를 통해 Hyperclass CRM에서 거래를 대량으로 추가하거나 업데이트할 수 있습니다. 고객 온보딩이나 다른 CRM에서 전환할 때 데이터 입력을 간소화하여 파이프라인 단계가 정렬되고 영업 활동이 정확하게 추적되도록 합니다.

CSV 가져오기를 통한 효율적인 기회 관리는 파이프라인을 최신 상태로 유지하고 팀의 협업을 돕습니다.

- 시간 절약: 수십 또는 수백 개의 기회를 한 번에 업로드

- 데이터 정확성 유지: 구조화된 필드로 일반적인 오류 방지

- 일관성 향상: 파이프라인 단계와 값이 표준을 따르도록 보장

- 온보딩 가속화: 이전 시스템의 리드를 빠르게 통합

- 기존 거래 업데이트: 기회 단계나 값을 쉽게 수정

- 연락처와 기회 동기화: 기회 데이터가 연락처 레코드와 직접 연결되도록 보장

## **기회 가져오는 방법**

기회 가져오기는 영업 데이터를 CRM에 정확하고 효율적으로 입력하기 위한 일련의 단계를 포함합니다. CSV 파일 준비부터 필드 매핑, 가져오기 확인까지, 이 섹션에서는 성공적인 가져오기 과정을 완료하는 데 필요한 모든 것을 안내합니다. 다음 단계를 따라 CSV 파일을 사용하여 Hyperclass로 기회를 성공적으로 가져오세요.

### **1단계: **기회 관리 탭으로 이동

**Opportunities(기회 관리)** 탭에서 **Import(가져오기)** 버튼을 클릭하여 시작합니다.

![기회 관리 탭에서 가져오기 버튼 클릭](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045398745/original/dmNghj-ZxoR9JhTKtXJ_qinPuZaG_v_tTw.png?1745246916)

### **2단계: **가져올 항목 선택

기회만 가져올지, 아니면 연락처와 기회를 모두 가져올지 선택합니다.

![가져올 항목 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045398815/original/gB0nH9yhrsJcbDs1ccI61Se_uwwrNLUlkQ.png?1745247057)

### **3단계: **CSV 파일 업로드

**Import Opportunities(기회 가져오기)**를 선택한 후 Next(다음)를 클릭하여 CSV 파일을 업로드하고, 새 연락처를 추가하는지 기존 연락처를 업데이트하는지에 따라 올바른 가져오기 유형을 선택합니다.

![CSV 파일 업로드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045398892/original/qm1aXwlk4hODcM2__8teU7ZgUUYUANY32w.gif?1745247178)

### **4단계: **컬럼과 필드 매핑

파일의 각 컬럼이 시스템의 해당 필드에 올바르게 매핑되도록 합니다. 참고할 사항들:

- "Don't update Empty Values(빈 값 업데이트 안함)" 체크박스를 선택하면 가져오는 파일에 빈 값이 있고 해당 기회에 이미 값이 있는 필드는 업데이트하지 않습니다.

- 특정 필드를 가져오지 않기로 결정한 경우, 매핑하지 않은 상태로 두고 하단의 체크박스를 선택하여 진행할 수 있습니다. - "필수 필드가 모두 매핑되었는지 확인하세요. 계속하려면 모든 필드를 매핑하거나 매핑되지 않은 컬럼에 대해 데이터 가져오지 않기를 선택하세요."

- CSV에서 연락처와 기회가 같은 행에 있으면 자동으로 서로 매핑됩니다.

- CSV 컬럼을 필수 Hyperclass 필드에 맞춰 매핑하세요:
  - **Contact ID(연락처 ID)** (필수)
  - **Opportunity Name(기회명)** (필수)
  - **Pipeline(파이프라인)** (필수)
  - **기타 필드**: 기회 금액, 기회 ID (업데이트하는 경우)

올바른 값으로 매핑된 필드는 녹색 체크 표시가 나타납니다. 필드가 누락되거나 잘못된 경우 노란색 경고 삼각형이 나타납니다.

![컬럼과 필드 매핑](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045399088/original/i83vv5109onZK191E1UJ32Vt-Xa4dvoCew.gif?1745247525)

**팁**: 모든 필수 필드가 성공적으로 매핑되면 모든 표시기가 녹색으로 바뀝니다.

### **5단계: **확인, 검토, 최종 선택

설정을 검토하고, 필드 매핑을 확인하며, 가져오기를 시작하기 전에 동의를 확인하여 설정을 완료합니다. 이 마지막 단계에서는 연락처가 올바르게 태그되고, 정렬되며, 워크플로우에 추가되고, 데이터가 완전히 검증되도록 합니다. 마지막 단계에서는 다음을 할 수 있습니다:

- 가져온 연락처에 태그 추가 (선택사항)

- 문서 이름과 매핑된 필드 검토

- Start Bulk Import(일괄 가져오기 시작)를 클릭하여 과정 완료

필드 매핑을 검토하세요. 모든 필수 필드가 올바르게 매핑되었는지 확인하고, 커스텀 필드가 의도한 대로 할당되었는지 확인합니다. 선택사항: 매핑되지 않은 컬럼은 자동으로 무시됩니다.

![최종 확인 및 가져오기 시작](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045399230/original/1FXu2Z7mvmnfEZkhLSHNqygn_K8_M9aPmQ.gif?1745247756)

## **추가 옵션: 연락처와 기회 함께 가져오기**

CSV에 연락처와 거래(기회) 정보가 모두 포함되어 있다면, 같은 과정으로 둘 다 한 번에 가져올 수 있습니다. 새로운 리드를 영업 파이프라인에 직접 온보딩하면서 가져오기 과정에서 단계나 거래 금액을 연결하려는 경우 특히 유용합니다. 과정은 연락처 가져오기와 정확히 동일하며, 유일한 차이점은 초기 가져오기 설정에서 "Contacts(연락처)"와 "Opportunities(기회)"를 모두 선택하는 것입니다.

Contacts(연락처) 페이지에서 **Import(가져오기)** 버튼을 클릭한 후, 가져올 객체를 선택하라는 안내가 나타납니다. 둘 다 체크하고 위에서 설명한 동일한 단계로 진행하면 됩니다.

![연락처와 기회 함께 가져오기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045399289/original/ee7wjJSlhAu6H2VG-HVe8r8PAXJhpnsw2g.jpeg?1745247905)

**자주 묻는 질문**

**Q: 연락처 데이터 없이 기회만 가져올 수 있나요?**

아니요, 각 기회는 계정의 유효한 연락처 ID와 연결되어야 합니다.

**Q: 어떤 파일 형식이 필요한가요?**

CSV 형식(최대 30MB)이어야 합니다.

**Q: 여러 파이프라인에 기회를 가져올 수 있나요?**

네, 파이프라인 이름이 계정에 이미 존재하는 것과 일치하는 한 가능합니다.

**Q: 필드가 잘못 매핑되면 어떻게 되나요?**

가져오기가 실패하거나 해당 행을 건너뜁니다. 모든 필수 필드가 제대로 매핑되고 형식이 올바른지 확인하세요.

**Q: 가져오기를 미리보거나 취소할 수 있나요?**

가져오기는 취소할 수 없습니다. 가져오기 전에 항상 데이터를 다시 한 번 확인하세요. Bulk Actions(일괄 작업)에서 이전 가져오기 내역을 검토할 수 있습니다.
