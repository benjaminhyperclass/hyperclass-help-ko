# 단계별 가이드: 파이프라인 생성하기

Hyperclass의 파이프라인(Pipelines)은 영업 기회를 정의된 단계별로 시각적으로 추적하여 영업이나 서비스 워크플로우를 체계적으로 관리하고 자동화하는 데 도움을 줍니다. 이 문서에서는 업데이트된 파이프라인 환경을 사용하여 파이프라인에 접근하고, 새 파이프라인을 생성하며, 단계를 추가 및 관리하고, 대시보드 표시를 설정하는 방법을 설명합니다.

## **개요**

파이프라인을 사용하면 리드 획득부터 계약 성사 또는 온보딩까지 고객 여정의 각 단계를 매핑할 수 있습니다. 파이프라인은 플랫폼 전반의 자동화, 리포팅, 기회 관리 대시보드를 지원합니다.

**업데이트된 파이프라인 환경 (Labs)**
파이프라인 페이지가 이제 HighRise 디자인 시스템을 사용하여 더 빠르고 직관적인 경험을 제공합니다.
**하위 계정(Sub-account) → Labs**에서 이 업데이트를 활성화할 수 있습니다.

## **파이프라인 생성 및 관리 방법**

### **1단계:** 파이프라인 섹션 접근

하위 계정에서 파이프라인에 접근하려면:

- **하위 계정(Sub-account)**으로 이동
- 왼쪽 사이드바에서 **기회 관리(Opportunities)** 클릭
- 상단 네비게이션 메뉴에서 **파이프라인(Pipelines)** 선택

![파이프라인 메뉴 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064130713/original/B8seRPUvO12HmBstcfJYYLtRCYvSv5w2sg.png?1770164789)

### **2단계:** 새 파이프라인 생성

파이프라인 페이지에서:

- 우측 상단의 **새 파이프라인 생성(Create new pipeline)** 클릭
- 설명적인 **파이프라인 이름(Pipeline Name)** 입력
- **저장(Save)** 클릭

![새 파이프라인 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064130717/original/3BrXmQHJXVrtzOOSm8Ae9WfLVrdzP274qw.png?1770164811)

**참고:**
파이프라인 이름은 고유해야 합니다. 같은 하위 계정에서 이미 존재하는 이름으로는 파이프라인을 생성하거나 업데이트할 수 없습니다.

### **3단계:** 파이프라인 단계 추가 및 관리

파이프라인을 생성한 후 워크플로우를 구성하는 단계를 정의할 수 있습니다.

단계를 추가하려면:

- **단계 이름(Stage Name)** 입력
- **단계 추가(Add stage)**를 클릭하여 추가 단계 생성
- 전체 흐름을 정의할 때까지 반복

단계를 관리하려면:

- **위아래 화살표**를 사용하여 단계 순서 변경
- **퍼널과 파이 차트 아이콘**을 사용하여 리포팅 표시 제어
- 휴지통 아이콘을 사용하여 단계 삭제

![파이프라인 단계 관리](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064130722/original/lGp5tEhmVoybO8y8dy9ulhHUfmOHsVehGA.png?1770164833)

**중요사항:**
- 단계 이름은 파이프라인 내에서 고유해야 합니다
- **성사(Won)**와 **실패(Lost)** 단계는 자동으로 생성되므로 수동으로 추가할 필요가 없습니다

### **4단계:** 단계 색상 커스터마이징

각 단계에 색상을 지정하여 파이프라인 전체의 가시성을 향상시킬 수 있습니다.

**단계 색상을 설정하려면:**

- 파이프라인 열기
- **단계(Stages)** 탭으로 이동
- 각 단계의 색상 선택

![단계 색상 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067901862/original/rnzo2M3KVSmYhTnRwq3g_VcBNXDbnkTGzg.png?1774616013)

**색상 표시 방식 선택:**

- 기본값 (색상 없음)
- 색상 점
- 배경 색조

![색상 표시 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067901969/original/klOm-AP3gA1AoYtf2EHKNIB84D_SMNXHVQ.png?1774616071)

**단계 색상이 반영되는 곳:**

- 칸반 보드(Kanban)와 목록 보기(List views)

![칸반 보드 색상](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067901922/original/Q_SxAZrlOedv0Pfbc-03HL7h5Z7KjMXC_Q.png?1774616049)

- 기회 카드(Opportunity cards)
- 연락처 상세 보기(Contact details view)

![연락처 상세 보기 색상](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067901944/original/ZJLh93qgFwtFkjZU25Ai5NV_IiibRplvoA.png?1774616060)

### **5단계:** 파이프라인 단계 삭제

파이프라인 단계를 삭제할 때 이제 기존 기회를 잃지 않고 다른 단계로 이동시킬 수 있습니다.

단계를 삭제하려면:

- 편집할 파이프라인 열기
- 해당 단계 옆의 삭제 아이콘 클릭
- 기존 기회들의 이동 목적지 단계 선택
- 삭제 확인

삭제된 단계의 모든 기회는 선택한 단계로 자동 이동됩니다.

### **6단계:** 파이프라인 대시보드 표시 설정

파이프라인과 단계가 대시보드와 보고서에 표시되는 방식을 제어할 수 있습니다.

파이프라인 표시를 다음에서 활성화 또는 비활성화:

- 퍼널 차트(Funnel charts)
- 파이 차트(Pie charts)

각 단계 옆의 아이콘을 사용하여 단계별 표시 설정을 구성할 수 있습니다.

이러한 설정은 기회 관리 대시보드에서 파이프라인 데이터가 표시되는 방식을 결정합니다.

## **자주 묻는 질문**

### **성사와 실패 단계를 수동으로 생성해야 하나요?**

아니요. Hyperclass는 모든 파이프라인에 자동으로 **성사(Won)**와 **실패(Lost)** 단계를 추가합니다.

### **단계를 삭제하면 기회는 어떻게 되나요?**

삭제 시 모든 기존 기회를 다른 단계로 이동시킬 수 있어 데이터 손실을 방지할 수 있습니다.

### **파이프라인을 생성한 후 이름을 변경할 수 있나요?**

예, 업데이트된 이름이 하위 계정 내에서 고유하다면 변경 가능합니다.

### **업데이트된 파이프라인 환경이 기능을 변경하나요?**

아니요. 핵심 기능은 동일하게 유지됩니다. 업데이트는 UI 개선과 더 안전한 단계 관리에 초점을 맞췄습니다.
