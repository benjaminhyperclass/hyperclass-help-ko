# 기회 필터링이란?

필터링은 선택한 조건에 맞는 기회만 표시하도록 목록이나 보드를 좁혀주는 기능입니다. 필드(담당자, 상태 또는 날짜 등)와 연산자(*같음*, *같지 않음*, *보다 큰* 등)를 결합하여 팀이 가장 관련성 높은 거래에 집중하고, 검토 속도를 높이며, 파이프라인 관리 중 노이즈를 줄일 수 있습니다.

## **기회 필터링의 주요 이점**

- **빠른 검토:** 담당자, 단계 이동 또는 최근 활동별로 거래를 신속하게 분리합니다.
- **우선순위 개선:** 진행 중이거나 최근 업데이트된 기회에 집중하여 아웃리치를 계획합니다.
- **정확도 향상:** 연산자(같음, 같지 않음, 비어있음, 비어있지 않음)를 사용하여 놓칠 수 있는 예외 상황을 방지합니다.
- **깊은 인사이트:** 가치, 소스, 커스텀 필드별로 분류하여 파이프라인 품질을 분석합니다.
- **깔끔한 워크플로우:** 정확히 필요한 것만 보도록 설정하여 뷰 전환 시간을 절약합니다.

## **필터 논리 (AND/OR)**

논리 연산자는 여러 조건이 함께 작동하는 방식을 결정합니다. 두 가지 유형의 필터 조건 논리가 가능합니다: AND와 OR.

질문에 맞는 논리를 선택하세요(좁히기 vs 넓히기).

- **AND -** 각 기회에 대해 모든 조건이 충족되어야 함
- **OR -** 각 기회에 대해 최소 한 가지 조건이 충족되어야 함

이러한 필터 조건은 필터 오른쪽 사이드바 상단의 토글을 사용하여 변경할 수 있습니다.

- **담당자 = A 사람 AND 상태 = 진행 중**을 결합하여 담당자의 활성 거래에 집중합니다.
- **상태 = 성사 OR 상태 = 실패**를 결합하여 종료된 결과만 검토합니다.

## **표준 필터 개요**

표준 필터는 가장 일반적인 CRM 필드와 작업을 다룹니다. 책임, 진행 상황, 가치, 활동 최근도별로 정렬하는 데 사용하세요.

- 담당자: 하나 이상의 담당자를 선택합니다.
- 팔로워: 기회를 팔로우하는 사용자별로 필터링합니다.
- 상태: **진행 중, 성사, 실패, 포기**.
- 기회 소스: *같음*, *같지 않음*, *비어있음*, *비어있지 않음* (필요에 따라 값 입력).
- 캠페인 유형: 사용 시 캠페인 유형을 검색하고 선택합니다.
- 기회 금액: *같음*, *같지 않음*, *보다 작음*, *보다 큰*, *비어있음*, *비어있지 않음* (숫자 값 입력).

## **날짜 기반 필터**

날짜 필터는 최근 변경사항과 결과를 추적하는 데 도움이 됩니다. 사전 설정은 선택 속도를 높이고 보고서를 일관되게 유지합니다.

- 마지막 단계 변경일: 같음, 같지 않음, 어제, 오늘, 이번 주, 지난주, 최근 7일, 최근 30일, 이번 달, 지난달, 올해, 작년, 커스텀, 다음 기간 내, 지난 기간 내, 미만, 초과.
- 마지막 상태 변경일: 위와 동일한 사전 설정.
- 생성일 / 업데이트일: 위와 동일한 사전 설정.
- 기회 성사일 / 기회 실패일: 위와 동일한 사전 설정.

## 커스텀 필드 및 태그

조직 특화 필드는 고급 세분화를 지원합니다. 추가하는 모든 기회 커스텀 필드는 해당 데이터 유형과 일치하는 연산자와 함께 필터에서 사용할 수 있습니다.

**참고:** 기회의 **커스텀 필드**에 대해 자세히 알아보려면 이 [**가이드**](how-to-use-custom-fields-for-opportunities.md)를 확인하세요.

예시: 고객 유형(*같음 / 같지 않음* 구매자, 판매자 등), 효율성 등급(*=, ≠, <, >, 비어있음 / 비어있지 않음*), 선호하는 반려동물(텍스트/목록), 태그(난이도/유형), 시간 추적.

- 팁: 올바른 연산자가 나타나도록 필드 유형이 올바른지 확인하세요(숫자 vs 텍스트).

## **기회 필터링 방법**

**참고:** 기본적으로 파이프라인은 "진행 중" 기회만 표시하도록 필터링됩니다. 기회를 찾을 수 없다면 올바른 필터와 검색이 목록에 적용되었는지 확인하세요.

- 하위 계정에 로그인합니다.
- 왼쪽 사이드바에서 **기회 관리**로 이동합니다.
- 검토하려는 파이프라인을 선택합니다.

![파이프라인 선택](https://jumpshare.com/share/D8sJph7yTErAl2H52u5L+/Screen+Shot+2026-01-22+at+8.24.05+PM.png)

- 상태 영역 아래의 **고급 필터**(깔때기 아이콘)를 클릭합니다.

![고급 필터 버튼](https://jumpshare.com/share/wpWbLMZHBLBWjDd58VT9+/Screen+Shot+2026-01-22+at+8.25.57+PM.png)

- 필요에 따라 필터를 선택합니다:

담당자 및 **팔로워**.
- 상태: 진행 중, 성사, 실패, 포기.
- 연산자가 있는 기회 소스(*같음*, *같지 않음*, *비어있음*, *비어있지 않음*).
- 캠페인 유형.
- 날짜 필터: 마지막 단계 변경일; 마지막 상태 변경일; 생성일; 업데이트일; 기회 성사일; 기회 실패일 — 각각 사전 설정(어제, 오늘, 이번 주, 지난주, 최근 7일, 최근 30일, 이번 달, 지난달, 올해, 작년, 커스텀) 포함.
- 숫자 연산자가 있는 기회 금액(같음, 보다 작음, 보다 큰, 같지 않음, 비어있음, 비어있지 않음).
- 커스텀 필드(예: 고객 유형, 효율성 등급, 선호하는 반려동물), **태그**, **시간 추적**.

- 필터를 적용하여 결과를 확인합니다.

![필터 적용 결과](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067905683/original/fdZfbdSnMZNNQQDX83M1jpJW1MhlSMv8fA.png?1774618153)

## **자주 묻는 질문**

**Q: 여러 필터를 결합할 수 있나요?**
A: 네. 여러 조건을 추가하고 **AND** 또는 **OR** 논리를 선택하여 결과를 좁히거나 넓힐 수 있습니다.

**Q: 마지막 단계 변경과 마지막 상태 변경의 차이점은 무엇인가요?**
A: 단계는 파이프라인 단계 간 이동을 추적하고, 상태는 라이프사이클 결과(진행 중, 성사, 실패, 포기)를 추적합니다.

**Q: 가치나 숫자 필드에 어떤 연산자를 사용할 수 있나요?**
A: *같음, 같지 않음, 보다 작음, 보다 큰, 비어있음, 비어있지 않음*을 사용할 수 있습니다.

**Q: 최근에 성사했거나 실패한 거래를 필터링할 수 있나요?**
A: 네. **기회 성사일** 또는 **기회 실패일**을 사용하여 날짜 사전 설정이나 커스텀 범위를 선택하세요.
