# 같은 파이프라인에서 한 사람에 대한 여러 기회 관리하기

**연락처당 여러 기회 허용** 기능을 사용하면 같은 파이프라인에서 동일한 고객에 대해 두 개 이상의 활성 거래를 추적할 수 있습니다. 이를 통해 갱신, 업셀, 다중 제안 여정의 정확성을 높이면서 자동화와 보고서를 일관되게 유지할 수 있습니다. 설정 방법, 워크플로우 충돌 방지법, 모범 사례를 알아보세요.

# 연락처당 여러 기회 허용이란?

연락처당 여러 기회 허용(Allow Multiple Opportunities per Contact)은 하이퍼클래스의 하위 계정 설정으로, **같은 파이프라인에서 한 연락처에 대해 두 개 이상의 기회를 허용**합니다. 이를 통해 갱신, 추가 상품, 병렬 제품 라인, 다중 지역 서비스 등의 사용 사례를 지원하면서도 각 고객에 대한 깔끔한 CRM 기록을 유지할 수 있습니다.

---

## **연락처당 여러 기회 허용의 주요 장점**

이 기능을 활성화하면 연락처가 파이프라인과 자동화를 통해 이동하는 방식이 바뀝니다. 아래 목록을 통해 실질적인 이점을 확인하여 팀이 언제 적용할지, 중복 커뮤니케이션을 어떻게 최소화할지 결정할 수 있습니다.

- **갱신 및 업셀**: 기존 거래를 덮어쓰지 않고 동일 고객의 새로운 수익 이벤트를 추적할 수 있습니다.

- **병렬 제안**: 서로 다른 상품이나 서비스에 대해 동시에 별도의 기회를 진행할 수 있습니다.

- **더 깔끔한 자동화 로직**: 여러 기회가 존재할 때 워크플로우 액션을 통해 올바른 기회를 타겟팅할 수 있습니다.

- **더 정확한 예측**: 고객이 여러 거래를 진행 중일 때 전체 파이프라인 가치를 확인할 수 있습니다.

- **운영 유연성**: 동일한 연락처가 두 번 이상 참여하는 다중 지역이나 프랜차이즈 시나리오를 지원합니다.

---

## **연락처당 여러 기회 허용 설정 방법**

올바른 설정을 통해 CRM이 한 연락처에 대해 여러 카드를 허용하고 자동화가 예측 가능하게 작동하도록 할 수 있습니다. 아래 단계를 따라 진행한 후 샘플 레코드로 테스트해보세요.

- Settings(설정) → Objects(객체) → Opportunities(기회)로 이동합니다.
- Allow Multiple Opportunities per Contact(연락처당 여러 기회 허용)을 켜기(ON)로 토글합니다.
- **Save changes(변경사항 저장)**을 클릭합니다.

- 같은 파이프라인에서 동일한 연락처에 대해 두 개의 기회를 생성하여 테스트해보세요. 두 기회가 모두 나타나고 자동화가 예상대로 실행되는지 확인합니다.

## **자주 묻는 질문**

**Q: 이 설정을 켜면 기존 기회가 변경되나요?**
기존 기회 카드는 그대로 유지됩니다. 활성화는 단순히 동일한 연락처에 대해 추가 카드를 허용하는 것입니다(같은 파이프라인 내에서도).

**Q: 나중에 설정을 끄면 어떻게 되나요?**
기존 기회 카드는 삭제되지 않습니다. 시스템은 같은 파이프라인에서 동일한 연락처에 대한 새로운 중복 기회를 제한합니다. 생성/업데이트 오류를 방지하려면 자동화를 검토하세요.

**Q: 설정이 꺼져 있어도 연락처가 서로 다른 파이프라인에서 여러 기회를 가질 수 있나요?**
이 설정은 같은 파이프라인 내에서 동일 연락처의 중복을 관리합니다. 파이프라인 모델과 생성 규칙이 파이프라인 간 동작을 결정하므로, 워크플로우 로직을 테스트해서 확인하세요.

**Q: 자동화가 올바른 카드에서 작동하도록 하려면 어떻게 해야 하나요?**
Find Opportunity를 사용하여 가장 빠른/최신 또는 조건 기반 일치를 선택하고, 의도하지 않은 재진입을 방지하기 위해 입력 필터를 설정하세요.

**Q: 예측에서 수익이 중복 계산되나요?**
예측과 파이프라인 합계는 모든 열린 기회를 반영합니다. 대시보드와 보고서가 이러한 현실에 맞게 설계되었는지 확인하세요.

**Q: 이것이 워크플로우 단계 내의 "Duplicate Opportunity" 옵션과 같은 건가요?**
아니요. 워크플로우 복제는 해당 액션의 동작을 제어합니다. 글로벌 설정은 CRM이 연락처당 여러 카드를 허용하는지를 제어합니다.

**Q: 사용자 권한을 업데이트해야 하나요?**
사용자가 파이프라인에 액세스하고 기회를 생성/업데이트할 수 있는지 확인하고, 필요에 따라 역할을 조정하세요.

**Q: 여러 기회가 존재할 때 중복 메시지를 어떻게 방지하나요?**
보호장치를 추가하세요: Find Opportunity 사용, 트리거 강화, 재진입 규칙이나 억제 조건 적용.
