# 일반 질문

### Q: Hyperclass에서 기회를 내보내려면 어떻게 하나요?

![기회 내보내기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049332407/original/F-Gf__4alVkhG-RNMAuJVbasTgnRiMqaOw.png?1751612397)

- 왼쪽 사이드바에서 기회 관리(Opportunities) 메뉴로 이동하세요.
- 우측 상단의 케밥 메뉴(점 3개 메뉴)를 클릭하세요.
- "기회 내보내기(Export Opportunities)"를 선택하세요.
- 내보내기 파일이 처리되어 브라우저 기본 다운로드 폴더에 다운로드됩니다.

### Q: 특정 기회를 팀과 공유하려면 어떻게 하나요?

특정 기회를 공유하려면:

- 앱에서 원하는 기회를 여세요.
- 브라우저에서 URL을 복사하세요.
- 팀원이나 관계자와 링크를 공유하세요.

공유된 링크를 통해 해당 기회와 세부사항에 직접 접근할 수 있어 협업이 개선되고 시간을 절약할 수 있습니다.

### Q: 기회의 모든 활동을 추적하고 기회 ID를 찾으려면 어떻게 하나요?

- 기회 카드를 클릭하여 기회를 여세요.
- 왼쪽 하단의 기회 ID를 확인하고 클릭하세요.
- 해당 기회의 감사 로그로 이동됩니다.
- 감사 로그를 통해 최근 60일 동안 기회에서 발생한 모든 활동을 모니터링할 수 있습니다.

### Q: 한 기회에 여러 연락처 기능은 현재 어떻게 작동하나요?

- 기회 카드를 클릭하여 기회를 여세요.
- 추가 연락처 필드에서 기회에 연결하고 싶은 연락처를 추가할 수 있습니다.
- 기회의 주 연락처로 추가된 연락처는 동시에 추가 연락처 목록에 추가할 수 없습니다.
- "중복 기회 허용" 설정이 꺼져 있다면, 한 연락처를 서로 다른 기회의 주 연락처/추가 연락처 목록에 동시에 추가할 수 없습니다.
- 한 기회에 최대 10명의 추가 연락처를 추가할 수 있습니다.
- 목록과 워크플로우는 주 연락처에서만 작동하며 주 연락처의 세부사항을 가져옵니다. [향후 업데이트에서 추가 예정]
- 연락처의 기회 섹션에 해당 연락처와 연결된 기회도 표시할 예정입니다. [향후 업데이트에서 추가 예정]

### Q: 실패한 기회에서 연락처 이름과 세부사항을 찾으려면 어떻게 하나요?

- 왼쪽 메뉴에서 기회 관리(Opportunities)로 이동하세요.
- 고급 필터(Advanced Filters) 버튼을 클릭하세요.
- 상태에서 실패(Lost)를 선택하고 적용을 클릭하세요.
- 이제 목록에 실패한 기회만 표시됩니다.
- 연락처 세부사항을 보려면 기회 박스를 클릭하세요.

# 커스텀 필드 질문

### Q: 기회에서 내 커스텀 필드를 보려면 어떻게 하나요?

'기회 관리(Opportunities)'로 이동하여 기회를 선택하거나 생성하면 커스텀 필드 목록이 표시됩니다. 기회를 편집할 때 이 필드들을 입력할 수 있습니다. 필드를 폴더로 정리했다면 차례대로 나타납니다.

### Q: '빈 필드 숨기기' 기능은 무엇을 하나요?

'빈 필드 숨기기' 기능을 활성화하면 값이 없는 모든 필드가 숨겨집니다. 이를 통해 관련이 있거나 필수인 필드에만 집중할 수 있어 기회 데이터를 더 깔끔하고 간소화된 형태로 볼 수 있습니다.

### Q: 기회의 '실패 사유' 필드 값을 편집하려면 어떻게 하나요?

**단계 1:** Settings(설정) > Custom Fields(커스텀 필드)로 이동하세요.

**단계 2:** Lost Reason(실패 사유)을 선택하고 Bulk Actions(일괄 작업) > Edit(편집)을 클릭하세요.

![실패 사유 편집 단계 1](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155027588180/original/e7mXlPtOYxPbCAK_kfpkpg1dHGwky25FxQ.png?1718273194)

**단계 3:** 모달에서 실패 사유 값을 변경하고 저장하면 반영됩니다.

![실패 사유 편집 단계 2](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155027588275/original/oPwS_jwP0_nbvYCM2KInuuqJMxaPvTWCqA.png?1718273272)

실패 사유의 이름이 변경되면, 해당 실패 사유가 사용되는 모든 곳에서 새로운 이름으로 표시됩니다.
