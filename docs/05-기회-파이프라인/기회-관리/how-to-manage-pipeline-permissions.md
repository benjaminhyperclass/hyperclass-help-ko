## **파이프라인 권한이란?**

파이프라인 권한은 기회 관리(Opportunities) 내의 개별 파이프라인에 대한 열람 및 편집 권한을 정의합니다. 관리자는 모든 사용자에게 액세스를 부여하거나, 특정 역할/사용자만 대상으로 하거나, 특정 사용자를 명시적으로 제외할 수 있으며, 이때 더 넓은 하위 계정 역할을 변경할 필요가 없습니다. 이러한 세밀한 가시성 제어는 데이터를 안전하게 보호하고 영업 팀이 자신에게 중요한 거래에만 집중할 수 있게 해줍니다.

## **파이프라인 권한의 주요 장점**

- **일관성:** 영업 담당자가 거래를 처리할 수 있도록 허용하면서 단계(스테이지) 이름과 순서를 보존합니다.

- **데이터 프라이버시:** 계약직이나 신입 직원으로부터 매출/보드 데이터를 숨깁니다.

- **세밀한 제어:** 모든 사용자와 공유하거나, 특정 사용자와 공유하거나, 예외적으로 제외할 수 있습니다.

- **감사 가능성:** 규정 준수 요구사항에 따라 파이프라인 설정을 누가 변경했는지 추적합니다.

- **크로스 플랫폼:** 웹과 하이퍼클래스 모바일 앱에서 동일한 권한이 적용됩니다.

- **대시보드 동등성:** 위젯과 지표가 파이프라인 액세스 권한과 본인 데이터만 보기(Assigned Data) 설정을 준수합니다.

## 파이프라인 권한 관리하는 방법

단계 구조와 민감한 매출 데이터를 보호하면서 각 파이프라인을 누가 보고 편집할 수 있는지 설정하려면 다음 단계를 따르세요.

- **Opportunities(기회 관리) → Pipelines(파이프라인)**으로 이동합니다.

![파이프라인 메뉴](https://jumpshare.com/share/Ios9ymH4r1c5gM8yvDxx+/Screen+Shot+2026-02-09+at+7.31.12+PM.png)

- 파이프라인 이름 옆의 열쇠 아이콘을 클릭하여 **Manage permissions(권한 관리)**를 엽니다.

![권한 관리 아이콘](https://jumpshare.com/share/KIWixyBYGFKfP7dFSurB+/Screen+Shot+2026-02-09+at+7.34.08+PM.png)

- **공유 모드(Sharing mode)**를 선택합니다: 모든 사용자와 공유, 선택한 사용자/역할과 공유, 또는 특정 사용자 제외.

![공유 모드 선택](https://jumpshare.com/share/uW4VpLaqpDTccl4woHI0+/Screen+Shot+2026-02-09+at+7.24.56+PM.png)

- 포함된 각 사용자/역할에 대해 **액세스 수준(Access level)**을 설정합니다: 보기 전용(View Only), 편집(Edit), 또는 **액세스 없음(No Access)**.

![액세스 수준 설정](https://jumpshare.com/share/MefKbpikfyLgjuWSgt41+/Screenshot+2026-02-09+at+7.38.01%E2%80%AFPM.png)

- **Copy link(링크 복사)**를 클릭하여 이미 권한이 있는 팀원에게 직접 액세스를 공유합니다. (선택사항)

![링크 복사](https://jumpshare.com/share/PoctQ4SHFuobYR1mTpYa+/Screen+Shot+2026-02-09+at+7.40.13+PM.png)

- **Save(저장)**를 클릭하여 적용합니다.

![설정 저장](https://jumpshare.com/share/XGgwPAFOvDpeK3bm2UsL+/Screen+Shot+2026-02-09+at+7.41.54+PM.png)

## 파이프라인 권한의 공유 모드

각 파이프라인을 공유하는 세 가지 유연한 방법으로 모든 조직 구조를 반영할 수 있습니다.

- **모든 사용자와 공유:** 한 번의 클릭으로 모든 사용자에게 보기 전용, 편집 또는 액세스 없음을 부여합니다.

- **선택한 사용자와 공유:** 개별 사용자/역할을 선택하고 각각에 대해 보기 전용, 편집 또는 액세스 없음을 설정합니다.

- **특정 사용자 제외:** 파이프라인을 전체적으로 공개한 다음 몇 가지 예외에 대해서만 액세스를 제거합니다.

![공유 모드 옵션](https://jumpshare.com/share/uW4VpLaqpDTccl4woHI0+/Screen+Shot+2026-02-09+at+7.24.56+PM.png)

## 파이프라인 권한의 액세스 수준

명확한 라벨로 각 사용자가 무엇을 할 수 있는지 쉽게 알 수 있습니다.

**참고:** **관리자(Admin)** 사용자는 항상 파이프라인에 대한 **전체 액세스** 권한을 가집니다.

- **보기 전용(View Only):** 기회를 이동하고 업데이트할 수 있지만, 단계나 파이프라인을 추가/편집/삭제할 수 없습니다.

- **편집(Edit):** 파이프라인 단계(파이프라인 자체 포함)를 추가, 편집, 삭제할 수 있는 전체 제어 권한입니다.

- **액세스 없음(No Access):** 파이프라인이 웹, 모바일, API 응답에서 완전히 숨겨집니다.

![액세스 수준](https://jumpshare.com/share/MefKbpikfyLgjuWSgt41+/Screenshot+2026-02-09+at+7.38.01%E2%80%AFPM.png)

## 파이프라인 권한의 감사 로그 및 보호 장치

하이퍼클래스는 모든 생성, 업데이트 또는 권한 취소 작업을 사용자, 파이프라인, 작업, 타임스탬프와 함께 **60일간 자동으로 기록**합니다. 이 감사 추적 기록은 문제 해결과 규정 준수 검토를 간소화합니다.

- **Settings(설정) → Audit Logs(감사 로그)**로 이동합니다.

![감사 로그 접근](https://jumpshare.com/share/OZguNUBFCClOrApXHjfL+/GIF+Recording+2026-02-09+at+7.58.37+PM.gif)

- **사용자(User)**, **모듈(Module) - 파이프라인 권한**, **작업(Actions)**, **날짜 범위(Date Range)**로 필터링하여 누가 무엇을 언제 변경했는지 확인합니다.

![감사 로그 필터링](https://jumpshare.com/share/yBJXEproMLpU1nwCIQuF+/Screen+Shot+2026-02-09+at+8.03.57+PM.png)

- 시스템은 또한 해당 파이프라인에서 현재 기회를 소유하고 있는 사용자의 액세스 권한을 취소하는 것을 방지합니다. 먼저 소유권을 재배정하라는 메시지가 표시됩니다.

![소유권 재배정 경고](https://jumpshare.com/share/yEBWJSWsfvOYhJoFDBjC+/sHUEjYNe0ST15mQhinHzyua2gUV8i1XCAA.jpeg)

## 자주 묻는 질문

**Q: "보기 전용"과 "액세스 없음"의 차이점은 무엇인가요?**

보기 전용 사용자는 파이프라인 내에서 자신에게 할당된 기회를 여전히 이동하거나 업데이트할 수 있습니다. 액세스 없음은 파이프라인을 완전히 숨깁니다.

**Q: 사용자가 이미 기회를 소유하고 있는데 액세스 없음으로 설정하면 어떻게 되나요?**

하이퍼클래스가 해당 변경을 차단합니다. 먼저 해당 사용자의 기회를 재배정하거나 삭제한 후, 액세스를 제거하세요.

**Q: 권한이 모바일 앱과 동기화되나요?**

네. 웹, iOS, Android 및 모든 API 응답에서 동일한 규칙이 적용됩니다.

**Q: 각 사용자 대신 역할에 대량으로 권한을 할당할 수 있나요?**

네, 역할이 선택기에 나타납니다. 해당 역할의 모든 사용자가 선택한 수준을 상속받습니다.

**Q: 권한 변경이 활성 자동화나 웹훅을 중단시키나요?**

아니요. 자동화는 계속 실행됩니다. 그러나 액세스 권한이 없는 사용자 토큰으로 만든 API 호출은 빈 파이프라인 목록을 반환합니다.

**Q: "소유자만" 옵션이 있나요?**

"선택한 사용자와 공유" > 소유자 선택 > 다른 사용자를 액세스 없음으로 설정하여 진정한 소유자 전용 파이프라인을 만들 수 있습니다.

**Q: 감사 로그는 얼마나 오래 보관되나요?**

60일간의 기록을 다운로드하거나 검토할 수 있습니다.
