# 어떤 분에게 적합한가요?

아래 중 하나라도 해당되시면 잘 맞습니다.

- **컨설팅·코칭·강의 등 지식/전문 서비스를 파는 1인 사업자(또는 소규모 팀)** — 고객 한 분 한 분을 상담 → 결제 → 재구매까지 끌고 가는 사업이라면 딱 맞아요.
- **카카오톡·전화·인스타 DM·문자로 들어오는 문의 응대에 시간을 너무 많이 쓰는 분** — 채널마다 흩어진 대화를 하나의 통합 인박스로 모아, "누구한테 뭘 언제 보냈는지" 안 놓치게 됩니다.
- **상담/미팅 예약과 리마인드를 수기로 관리하다 노쇼·누락을 겪는 분** — 예약·확정·리마인드가 자동으로 돌고, 노쇼 시 자동 재예약까지 연결됩니다.
- **웨비나·무료특강·전략세션 같은 퍼널로 잠재고객을 꾸준히 모으고 싶은 분** — 랜딩 → 신청 → 예약 → 자동 알림 → 사후관리가 한 흐름으로 이어집니다.
- **사람을 더 뽑기는 부담스럽지만, 운영을 시스템화해서 혼자서도 돌아가게 만들고 싶은 분** — 도구 7~8개를 한 사람이 묶어 돌리던 구조를, 시스템이 대신 돌리게 바꿉니다.
- **플랫폼 운영사에 매출을 의존하지 않고, 자기만의 독립 결제 시스템을 갖고 싶은 분** — 하이퍼클래스는 사장님 개인 토스페이먼츠를 직접 연동합니다. 결제 대금이 중간 플랫폼을 거치지 않고 사장님 계좌로 바로 들어오고, 고객·결제 데이터도 사장님 소유예요. **내 사업의 결제·데이터 주권**을 갖는 구조입니다.
- **API·Webhook으로 전 세계 다양한 앱과 자유롭게 연동하고 싶은 분** — 페이스북, 구글, Stripe, Twilio, Zoom, Shopify, QuickBooks, WordPress 같은 주요 플랫폼과 직접 연동되고, 여기에 Zapier·Make·Pabbly와 1,500개 이상의 앱 마켓플레이스로 연동 범위를 더 넓힐 수 있어요. 노코드 연동으로 닿지 않는 특수한 도구라면 REST API와 Webhook으로 직접 만든 시스템과 데이터를 양방향으로 주고받을 수 있습니다. (한국 결제는 토스페이먼츠로 연동됩니다.)

**반대로, 이런 경우엔 다른 서비스가 더 맞을 수 있어요.**

- 단순히 **녹화 강의 영상만** 올려 파는 용도라면 → LMS 전용 서비스
- 수천 개 상품을 다루는 **대규모 이커머스 쇼핑몰** → 전문 커머스 플랫폼
- "도구만 있으면 고객이 저절로 생긴다"고 기대하는 경우 → 하이퍼클래스는 들어온 잠재고객을 새지 않게 잡아 전환·재구매로 끌고 가는 '운영 그릇'이지, 트래픽(광고·콘텐츠) 자체를 대신 만들어 주지는 않습니다.
