# 커스텀 객체 가이드: CRM 데이터를 더 체계적으로 관리하기

커스텀 객체(Custom Objects)를 사용하면 실제 비즈니스 운영 방식에 맞춰 데이터를 구조화할 수 있습니다. 이 가이드에서는 커스텀 객체가 무엇인지, 언제 사용해야 하는지, 그리고 내 비즈니스에 적합한지 판단하는 방법을 설명합니다.

커스텀 객체는 연락처, 회사, 영업 기회 같은 표준 레코드에 맞지 않는 데이터를 정리하고 관리하는 데 도움이 됩니다. 고유한 구조화된 레코드를 만들어 독특한 비즈니스 요소를 추적하고, 자동화를 개선하며, 깔끔하고 확장 가능한 시스템을 유지할 수 있습니다.

## **커스텀 객체란 무엇인가요?**

커스텀 객체는 표준 엔티티를 넘어서 비즈니스에서 추적하는 모든 것에 대한 레코드를 생성하고 관리할 수 있는 유연한 데이터 구조입니다.

다음과 같은 항목을 나타낼 수 있습니다:

- 부동산 매물
- 반려동물
- 프로젝트
- 구독
- 정책

각 커스텀 객체에는 다음이 포함됩니다:

- 고유한 레코드와 필드
- 다른 레코드와의 관계
- 자동화 및 리포팅 도구와의 호환성

---

## **커스텀 객체를 왜 사용해야 하나요?**

커스텀 객체는 기본 시스템을 넘어서 실제 비즈니스 운영을 반영하는 구조화된 환경을 구축하는 데 도움이 됩니다.

주요 장점은 다음과 같습니다:

- **더 나은 조직화:** 관련 없는 데이터를 핵심 레코드에서 분리
- **확장성:** 복잡함 없이 대용량 데이터셋 관리
- **정확한 자동화:** 구조화된 데이터로 더 깔끔한 워크플로우 구축
- **고급 관계 설정:** 여러 레코드를 쉽게 연결

---

## **언제 커스텀 객체를 사용해야 하나요?**

커스텀 객체는 개인이나 회사와 직접 연결되지 않는 구조화된 데이터를 추적해야 할 때 가장 유용합니다.

다음과 같은 경우에 사용하세요:

- 자산, 리스팅 또는 아이템을 추적할 때
- 반복 가능하고 구조화된 레코드가 필요할 때
- 여러 데이터 유형 간의 관계가 필요할 때

예시:

- 구매자와 중개인에게 연결된 부동산 매물
- 소유자에게 연결된 반려동물
- 고객에게 연결된 프로젝트

---

## **표준 레코드를 사용해야 하는 경우**

커스텀 객체는 강력하지만 항상 필요한 것은 아닙니다.

다음과 같은 경우에는 사용을 피하세요:

- 표준 레코드가 이미 필요를 충족하는 경우
- 이메일이나 캠페인 같은 내장 커뮤니케이션 도구가 필요한 경우
- 기존 데이터 구조를 중복으로 만드는 경우

이런 경우에는 세분화 도구나 필터를 사용하는 것이 좋습니다.

---

## **일반적인 예시**

### **좋은 사용 사례**

- **부동산:** 가격, 상태, 위치 정보가 있는 매물 추적
- **수의사 클리닉:** 의료 세부사항이 포함된 반려동물 기록 저장
- **프로젝트 관리:** 고객에게 연결된 프로젝트 추적

### **권장하지 않는 사례**

- **이벤트 참석자:** 커뮤니케이션을 위해서는 표준 레코드가 더 적합
- **내부 메모:** 객체를 만드는 대신 노트나 할 일을 사용

## **커스텀 객체 만드는 방법**

![객체 관리 페이지 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069652415/original/MeobEmCPosjc8slMyJxIFIEyeLECFzddIg.png?1776814010)

Settings(설정)에서 **Objects**를 클릭한 다음 **Add Custom Object**를 선택하세요.

## **다음 단계**

커스텀 객체를 생성한 후에는 다음과 같은 작업을 할 수 있습니다:

- 커스텀 필드 추가
- 레코드 간 관계 정의
- 워크플로우와 자동화 구축
- 데이터 수집을 위한 폼 생성
- 리포팅을 위한 대시보드 사용

---

## **자주 묻는 질문**

**Q: 커스텀 객체의 주요 목적이 무엇인가요?**

커스텀 객체는 표준 레코드에 속하지 않는 구조화된 데이터를 추적하는 데 도움이 됩니다.

**Q: 커스텀 객체를 다른 레코드와 연결할 수 있나요?**

네, 여러 유형의 레코드에 연결하여 관계를 구축할 수 있습니다.

**Q: 새로운 데이터에 항상 커스텀 객체를 사용해야 하나요?**

아니요. 표준 레코드가 사용 사례를 효과적으로 처리할 수 없을 때만 사용하세요.

**Q: 커스텀 객체가 자동화를 지원하나요?**

네, 워크플로우, 폼, 리포팅에서 사용할 수 있습니다.

---

## **도움이 필요하신가요?**

커스텀 객체 설정이나 데이터에 적합한 구조 결정에 도움이 필요하시면, 지원팀이나 시스템 관리자에게 문의하세요.
