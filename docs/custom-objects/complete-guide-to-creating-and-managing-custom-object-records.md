# 커스텀 객체 레코드 생성 및 관리 완전 가이드

커스텀 객체 레코드를 사용하면 비즈니스 요구에 맞춰 구조화된 데이터를 저장하고 관리할 수 있습니다. 이 가이드에서는 레코드를 효율적으로 생성, 업데이트, 연결, 삭제하는 방법을 설명하여 데이터를 체계적이고 접근하기 쉽게 유지할 수 있도록 도와드립니다. 이 단계를 따르면 워크플로우와 리포팅을 지원하는 깔끔하고 확장 가능한 레코드를 유지할 수 있습니다.

# **커스텀 객체 레코드란?**

커스텀 객체 레코드는 표준 데이터 필드를 넘어서는 정보를 저장하고 관리할 수 있는 유연한 방법을 제공합니다. 자산, 프로필, 항목 또는 비즈니스 전용 데이터와 같은 고유한 항목들을 구조화된 형태로 추적할 수 있습니다.

이러한 레코드는 정보를 중앙화하여 관계 관리, 정확도 유지, 데이터 전반의 가시성 향상을 더욱 쉽게 만들어줍니다.

---

## **커스텀 객체 레코드의 주요 장점**

커스텀 객체 레코드는 전문 데이터를 구성하고 관리하는 방식을 향상시킵니다.

- **유연한 데이터 구조:** 비즈니스 요구에 맞춘 정보 저장
- **향상된 구성:** 커스텀 레코드를 구조적이고 관리하기 쉽게 유지
- **더 나은 가시성:** 중앙화된 뷰에서 레코드에 접근하고 업데이트
- **더 강한 관계:** 레코드를 다른 데이터와 연결하여 더 나은 맥락 제공

---

## **새 커스텀 객체 레코드 생성하기**

새 레코드를 생성하면 커스텀 데이터 추적을 시작할 수 있습니다. 생성 시에는 기본 필드만 필요하며, 추가 세부 정보는 나중에 추가할 수 있습니다.

새 레코드를 생성하려면:

- 왼쪽 네비게이션 메뉴에서 커스텀 객체로 이동
- 우측 상단의 **Add** 버튼 클릭
- 기본 필드에 필수 값 입력
- **Save** 클릭하여 레코드 생성
- 여러 레코드를 계속 생성하려면 **Save and Add Another** 사용

이 단계에서는 기본 필드만 필요합니다. 추가 필드는 레코드 생성 후 업데이트할 수 있습니다.

![커스텀 객체 네비게이션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442806/original/SS2DWlwAIRDUchDZktR8A3SxSLY__SVy3Q.png?1774210287)

이 이미지는 새 레코드를 생성하기 전에 네비게이션 메뉴에서 커스텀 객체를 선택하는 위치를 보여줍니다.

**액션:** 원하는 커스텀 객체를 열어 레코드 추가를 시작하세요.

![레코드 생성 폼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442785/original/eya4JJh0THYTcapCMjFo9Fgyl7bRZdXaag.jpeg?1774210150)

이 이미지는 레코드 생성 시 필수 기본 필드를 입력하는 폼을 보여줍니다.

**액션:** 필수 값을 입력하고 **Save**를 클릭하세요.

---

## **기존 커스텀 객체 레코드 업데이트하기**

레코드를 업데이트하면 시간이 지남에 따라 정보가 변경될 때 데이터가 정확하고 최신 상태를 유지할 수 있습니다.

레코드를 업데이트하려면:

- 레코드가 포함된 커스텀 객체 열기
- 레코드 이름을 클릭하여 세부 정보 열기
- 필요한 필드(텍스트, 숫자, 드롭다운 등) 편집
- **Save** 클릭하여 변경사항 적용
- 업데이트를 저장하지 않으려면 **Cancel** 클릭

레코드를 업데이트하면 리포팅 정확도와 데이터 신뢰성이 향상됩니다.

![레코드 편집 패널](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442780/original/rVco9F1ntfxHoWX1s7pdLyY6j_lKxYTfUQ.png?1774210122)

이 이미지는 필드를 편집할 수 있는 레코드 상세 패널을 보여줍니다.

**액션:** 필요한 필드를 수정하고 **Save**를 클릭하세요.

---

## **관련 객체 관리하기**

관련 객체를 사용하면 레코드를 다른 레코드와 연결하여 데이터 전반에서 더 나은 맥락과 가시성을 제공할 수 있습니다.

관련 객체를 관리하려면:

- 커스텀 객체 레코드 열기
- **Related Objects** 섹션 찾기
- 연결하려는 객체 유형 옆의 **Add** 클릭
- 연결하려는 레코드 검색 및 선택
- 연결 저장

연결되면 연관된 레코드가 빠른 접근을 위해 관련 객체 패널에 나타납니다.

![관련 객체 패널](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442772/original/perJrUIYYIMGhTwevaimxBst4I0rdMS7NA.png?1774210080)

이 이미지는 레코드 내에서 연관된 레코드가 나타나는 위치를 보여줍니다.

**액션:** **Add**를 클릭하여 다른 레코드를 연결하세요.

---

## **커스텀 객체 레코드 삭제하기**

레코드를 삭제하면 데이터에서 영구적으로 제거됩니다. 이 작업은 레코드가 더 이상 필요하지 않을 때만 사용해야 합니다.

레코드를 삭제하려면:

- 제거하려는 레코드 열기
- 우측 상단의 점 3개 메뉴 클릭
- **Delete Record** 선택
- 삭제 확인

레코드 삭제는 되돌릴 수 없으므로 진행하기 전에 신중히 검토하세요.

![레코드 삭제 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442768/original/S7D72TnwaXfiAOebzOpp-c5ZPwg14z7LXg.png?1774210050)

이 이미지는 레코드 메뉴에서 삭제 액션에 접근하는 위치를 보여줍니다.

**액션:** **Delete Record**를 선택하고 확인하세요.

---

## **중요 사항**

이러한 동작을 이해하면 데이터 문제를 방지하고 적절한 사용을 보장할 수 있습니다.

- 레코드 생성 시에는 기본 필드만 필요
- 추가 필드는 나중에 업데이트 가능
- 관련 객체는 언제든지 추가 가능
- 삭제된 레코드는 복원할 수 없음

---

## **모범 사례**

이러한 방법을 따르면 깔끔하고 효과적인 데이터를 유지할 수 있습니다.

- 레코드에 명확하고 일관된 이름 사용
- 중복 항목 방지
- 정보를 정확하게 유지하기 위해 정기적으로 레코드 업데이트
- 의미 있는 데이터를 연결하기 위해 관련 객체 사용
- 사용하지 않는 레코드 정기적으로 검토 및 정리

---

## **자주 묻는 질문**

**Q: 레코드를 생성할 때 모든 필드를 완료해야 하나요?**

아니요. 기본 필드만 필요합니다. 추가 필드는 나중에 추가할 수 있습니다.

**Q: 저장한 후 레코드를 편집할 수 있나요?**

네. 언제든지 모든 레코드를 업데이트할 수 있습니다.

**Q: 레코드를 다른 레코드와 연결할 수 있나요?**

네. 관련 객체 섹션을 사용하여 레코드를 연결할 수 있습니다.

**Q: 삭제된 레코드를 복구할 수 있나요?**

아니요. 삭제된 레코드는 복원할 수 없습니다.

**Q: 연관된 레코드는 어디에서 볼 수 있나요?**

연관된 레코드는 해당 레코드의 관련 객체 섹션에서 볼 수 있습니다.

---

## **추가 도움이 필요하신가요?**

커스텀 객체 레코드 관리에 대한 도움이 필요하시면 Help Center에서 관련 주제를 검색하거나 추가 안내를 위해 지원팀에 문의하세요.
