# 커스텀 객체 연관성 - 레코드 생성, 관리 및 연결 방법

커스텀 객체 연관성(Custom Object Associations)을 사용하면 서로 다른 데이터 유형 간에 레코드를 연결하여 관계를 추적하고, 정보를 정리하고, 더 강력한 워크플로우를 구축할 수 있습니다. 이 기능을 통해 연락처를 커스텀 객체와 연결하거나 커스텀 객체끼리 연결하여 데이터 가시성과 관리 효율성을 높일 수 있습니다.

# 커스텀 객체 연관성이란?

커스텀 객체 연관성을 통해 라벨을 사용하여 레코드 간의 관계를 정의할 수 있습니다. 이러한 관계는 서로 다른 레코드들이 어떻게 연결되어 있는지 이해하는 데 도움이 되며, 상호작용을 보다 효과적으로 관리할 수 있게 해줍니다.

다음과 같은 작업이 가능합니다:

- 연락처(Contacts)를 커스텀 객체와 연결
- 커스텀 객체를 다른 커스텀 객체와 연결
- 라벨을 사용하여 관계 유형 정의
- 레코드 상세 페이지에서 직접 관계 확인 및 관리

모든 연관성은 다대다(many-to-many) 방식으로, 여러 레코드를 함께 연결할 수 있습니다.

---

## 연관성 생성하기

연관성을 사용하려면 먼저 객체 간의 관계 구조를 생성해야 합니다.

![커스텀 객체 연관성 탭](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442562/original/3Zp4gdGNCaS3f9jM5tIvapCmx3ix43Aw0w.png?1774208901)

이 화면은 선택한 객체의 연관성 탭을 여는 위치를 보여줍니다.

**방법:** `Settings(설정) → Custom Objects(커스텀 객체)`로 이동하여 객체를 선택하고 **Associations(연관성)** 탭을 클릭하세요.

![새 연관성 생성 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067338735/original/2C9sRGVY9mCSDG8X0dtg5L5gYzkt1X-CAQ.png?1773957622)

이 화면은 새 연관성을 생성하는 옵션을 보여줍니다.

**방법:** **Create Association(연관성 생성)**을 클릭하여 객체 간의 관계 설정을 시작하세요.

### 연관성 라벨 설정

라벨을 사용하여 레코드가 어떻게 관련되는지 정의할 수 있습니다:

- **단일 라벨:** 양쪽에 동일한 관계 (예: "소유자")
- **라벨 쌍:** 각 면에 다른 관계 이름 (예: "구매자"와 "판매자")

![라벨 이름 지정 및 관계 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442394/original/pQyXe3m5UuCOkex2gz6U8PjI7w2JjhZvNw.png?1774207929)

이 화면은 라벨 이름을 지정하고 관계를 설정하는 방법을 보여줍니다.

**방법:**

- 단일 또는 쌍 라벨을 선택
- 라벨 이름 입력
- 관계 미리보기 확인
- **Save(저장)** 클릭

### 주요 사항

- 모든 연관성은 **다대다** 방식입니다
- 객체 관계당 **최대 10개의 라벨**을 생성할 수 있습니다
- 라벨은 레코드가 연결되고 표시되는 방식을 정의합니다

## 연관성 관리하기

연관성을 생성한 후에는 이를 확인, 편집 또는 삭제할 수 있습니다.

![생성된 연관성 목록](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442381/original/fRcW4I8CHhYSqRwrIVYApHIxfzC-mP8xXQ.png?1774207863)

이 화면은 생성된 모든 연관성과 연결된 객체들을 보여줍니다.

**방법:** 이 목록에서 연관성 라벨, 연결된 객체, 관계 유형을 검토하세요.

### 연관성 편집하기

필요시 라벨 이름을 업데이트할 수 있습니다.

**방법:**

- 연관성 옆의 세 점 메뉴 클릭
- **Edit(편집)** 선택
- 라벨 이름 업데이트
- 변경사항 저장

참고: 내부 이름은 수정할 수 없습니다.

![기존 연관성 라벨 수정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442367/original/X_oInS55sF9b4nO2FVqKxppeO0IYsZhLiw.png?1774207777)

이 화면은 기존 연관성 라벨을 수정하는 방법을 보여줍니다.

**방법:** 라벨 이름을 업데이트하고 변경사항을 저장하세요.

### 연관성 삭제하기

현재 연결된 레코드가 없는 경우 연관성을 제거할 수 있습니다.

![삭제 확인 과정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067442420/original/VOYdwbxdzJtFZRBmlzZ46hsqDJAg9tACGQ.png?1774208063)

이 화면은 삭제 확인 과정을 보여줍니다.

**방법:**

- 메뉴에서 삭제 클릭
- **DELETE**를 입력하여 확인
- **Delete(삭제)** 클릭

중요사항:

- 이 작업은 되돌릴 수 없습니다
- 삭제하려면 연관성이 사용 중이지 않아야 합니다

## 레코드 연결하기

연관성 라벨을 생성한 후 레코드들을 서로 연결할 수 있습니다.

![레코드에 연관성 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067338773/original/q99GmslHrzGtUPFK4Jv1ODpJtzRbd0UIug.png?1773957737)

이 화면은 레코드에 연관성을 추가하는 방법을 보여줍니다.

**방법:**

- 레코드 열기
- **Related Objects(관련 객체)** 섹션 찾기
- 연관성 아이콘 또는 Add(추가) 버튼 클릭
- 연결할 객체 선택
- 하나 이상의 라벨 선택
- **Save(저장)** 클릭

### 다음 단계

- 연관성이 **Related Objects(관련 객체)** 패널에 나타남
- 연결된 레코드를 클릭하여 상세 정보 확인 가능
- 연관성을 통해 데이터 전반의 관계를 추적할 수 있음

---

## 여러 라벨 사용하기

동일한 레코드 간에 하나 이상의 라벨을 지정할 수 있습니다.

예시:

- 한 연락처가 레코드의 **소유자**이면서 동시에 **구매자**가 될 수 있음
- 여러 라벨을 통해 관계 정의에 더 많은 유연성 제공

---

## 중요한 제한사항과 규칙

- 각 라벨은 최대 **1,000개의 레코드**를 연결할 수 있습니다
- 연관성은 항상 **다대다** 방식입니다
- 레코드를 연결하기 전에 라벨을 먼저 생성해야 합니다
- 라벨을 삭제하려면 기존 관계를 모두 제거해야 합니다

---

## 모범 사례

다음 가이드라인을 따라 연관성을 깔끔하고 효과적으로 유지하세요:

- 명확하고 의미 있는 라벨 이름 사용
- 불필요한 중복 라벨 생성 피하기
- 사용하지 않는 연관성을 정기적으로 검토 및 정리
- 객체 간 관계가 다를 때 쌍 라벨 사용
- 구조를 단순하고 이해하기 쉽게 유지

---

## 자주 묻는 질문

**Q: 여러 레코드를 함께 연결할 수 있나요?**

네. 연관성은 다대다 관계를 지원하므로 여러 레코드를 연결할 수 있습니다.

**Q: 레코드 간에 하나 이상의 라벨을 사용할 수 있나요?**

네. 여러 라벨을 적용하여 다양한 유형의 관계를 정의할 수 있습니다.

**Q: 연관성 라벨을 언제든지 삭제할 수 있나요?**

아니요. 해당 라벨을 사용하는 기존 관계를 모두 제거한 후에야 삭제할 수 있습니다.

**Q: 연관성 라벨을 생성한 후 편집할 수 있나요?**

네. 라벨 이름은 업데이트할 수 있지만, 내부 설정은 변경할 수 없습니다.

**Q: 연결된 레코드를 어디서 볼 수 있나요?**

각 레코드 내의 **Related Objects(관련 객체)** 섹션에서 찾을 수 있습니다.

---

## 추가 도움이 필요하시면

연관성 생성이나 관리에 도움이 필요하시면 도움말 센터에서 "Custom Objects" 또는 "Associations"를 검색해보세요. 추가 안내가 필요하시면 고객지원팀에 문의하실 수도 있습니다.
