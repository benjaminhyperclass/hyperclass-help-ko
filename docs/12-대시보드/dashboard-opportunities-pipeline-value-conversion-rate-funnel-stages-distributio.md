# 대시보드: 기회 관리, 파이프라인 가치, 전환율, 퍼널, 단계별 분포

![대시보드 기회 관리 상세 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48050888625/original/WNBDN8uSwmNMSWj1JEY8iMm0irzmrag7ZQ.png?1595624291)

**날짜 범위 필터(Date Range filter)**는 리드의 **생성일(Date Created)**을 기준으로 합니다.

위의 기회 관리 화면에서는 1월 24일부터 7월 25일까지 등록된 4개의 리드가 OPEN 상태에 있음을 보여줍니다.

![날짜별 리드 확인](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48050889502/original/DxwSnHuXDWH3nRMU5RqMKEbn5ERJapyA7Q.png?1595624751)

특정 날짜에 등록된 리드의 수만 보고 싶다면, 원하는 날짜를 **더블클릭**하세요!

![특정 날짜 리드 확인](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48050890096/original/M-jMy0aW2XRmWzJYcKXgY3_4oGKyHuWjhA.png?1595625067)

**전환율(Conversion rate)** = WON 상태의 리드 수(1) ÷ 전체 등록 리드 수(5)

![WON 상태 설명](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48050890289/original/gAw4nTNHY2pHhAqGeq03APsW5Smq1NH9og.png?1595625168)

목록 끝에 있는 **WON**은 자동으로 추가되는 항목입니다. 따라서 WON 상태의 리드를 추적하기 위해 별도로 WON 단계를 만들 필요가 없습니다.

![퍼널 차트 설명](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48050891138/original/dK4UFuOfE6vqE37yqPZ9asdU-Is3eTWw1A.png?1595625737)

Prospects 단계 우측의 80%는 Facebook Leads 파이프라인에 등록된 리드 중 80%가 Prospects 단계로 진행될 것임을 의미합니다.

Facebook Leads 파이프라인에 등록된 리드 중 60%가 Booked 단계로 진행됩니다.

파이프라인은 영업 단계처럼 구성됩니다. 예를 들어, 초등학교, 고등학교, 대학교 같은 단계가 있다면, 한 사람이 대학교 단계에 있을 때 그 사람은 이미 초등학교와 고등학교를 거쳤다고 가정하므로 각 단계의 리드 수에 1씩 추가됩니다.

따라서 NO SHOW 단계의 리드를 추적하려는 경우 퍼널 차트를 참고하는 것은 적절하지 않습니다. 이 경우 Facebook Leads 파이프라인 리드의 40%가 NO SHOW 단계로 진행된다는 의미가 되기 때문입니다. 실제로는 1개 리드만 노쇼이고 다른 리드는 WON으로 표시되었으므로 40%가 아닌 20%여야 합니다. 리드가 WON으로 표시되면 제품 구매까지 모든 단계를 거쳤다고 가정하므로 모든 단계의 리드 수가 1씩 증가합니다.

퍼널 차트를 활용하려면 각 파이프라인 단계가 판매에 한 걸음씩 가까워지는 구조여야 합니다. 예: 잠재고객 → 설문 완료 → 상담 → 구매

파이프라인 생성에 대한 자세한 정보: [https://gohighlevelassist.freshdesk.com/support/solutions/articles/48000982197-create-a-pipeline](https://gohighlevelassist.freshdesk.com/support/solutions/articles/48000982197-create-a-pipeline)

노쇼, 참석했지만 구매하지 않음 같은 단계를 추적하려면 **단계별 분포 차트(Stages Distribution chart)**를 참고하세요.

![단계별 분포 차트](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48050893703/original/gq3NWTUSY7nybUQy-6v2JzlibpCVXfbabQ.png?1595627447)

위 차트는 단순히 현재 각 단계에 있는 리드의 수를 보여줍니다.

어떤 단계를 거쳐간 리드의 수를 추적하려면 **중복 기회 허용(Allow Duplicate Opportunity)**을 활성화하세요. 그러면 리드가 No show 단계를 거쳤을 때 해당 단계에 기회 카드가 남게 됩니다.

![중복 기회 허용 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48050893887/original/_Jpu2ayqNGHJvMoOPlttzHSgDt2rZ1TO_g.png?1595627534)

트리거 액션 **기회 추가/업데이트(Add/Update Opportunity)**에서도 **중복 기회 허용** 토글이 켜져 있는지 확인하세요.

![워크플로우 중복 기회 허용 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48050894478/original/oNZBb6deBDv6ZgqnurZSZcEOamtPPmDRKA.png?1595627906)
