# 대시보드 권한 관리하는 방법

대시보드는 생성자만 볼 수 있는 비공개로 설정하거나, 다른 역할의 사용자들과 다양한 접근 권한 수준으로 공유할 수 있어요.

---

## 대시보드 권한이란?

대시보드는 다음과 같은 권한 수준을 제공해요:

| 권한 수준 | 설명 |
|----------|------|
| FULL (전체) | 대시보드를 생성, 편집, 공유, 삭제할 수 있는 모든 권한을 부여합니다 |
| EDIT (편집) | 대시보드와 위젯을 편집할 수 있지만, 대시보드를 삭제할 수는 없습니다 |
| VIEW (보기) | 대시보드를 읽기 전용으로만 볼 수 있습니다 |
| NO ACCESS (접근 불가) | 대시보드에 완전히 접근할 수 없습니다 |

권한 수준별로 대시보드에서 수행할 수 있는 작업은 다음과 같아요:

| 작업 | FULL | EDIT | VIEW | NO ACCESS | 비공개 대시보드 소유자 |
|------|------|------|------|----------|-------------------|
| 권한 관리 | ✅ | ❌ | ❌ | ❌ | ✅ |
| 대시보드 삭제 | ✅ | ❌ | ❌ | ❌ | ✅ |
| 기본 대시보드로 설정 | ✅ | ❌ | ❌ | ❌ | ✅ |
| 대시보드 편집 | ✅ | ✅ | ❌ | ❌ | ✅ |
| 대시보드 복제 | ✅ | ✅ | ❌ | ❌ | ✅ |
| 대시보드 보기 | ✅ | ✅ | ✅ | ❌ | ✅ |
| 다른 하위 계정으로 복사 (에이전시 레벨만) | ✅ | ✅ | ✅ | ❌ | ✅ |

대시보드 소유자나 전체 권한을 가진 역할만 대시보드 권한을 편집할 수 있어요.

에이전시 관리자는 항상 비공개가 아닌 모든 대시보드에 전체 접근 권한을 갖습니다.

계정 관리자나 대시보드에 전체 접근 권한을 가진 에이전시 역할만 기본 대시보드로 설정할 수 있어요.
에이전시 역할만 대시보드를 다른 하위 계정으로 복사할 수 있습니다.

## 대시보드 권한 관리하기

### 새 대시보드에 권한 설정하기

**1단계: 대시보드 추가**

Dashboard(대시보드)로 이동해서 대시보드 선택 메뉴(파란색)를 클릭한 후, Add Dashboard(대시보드 추가)를 클릭하세요.

![대시보드 추가 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433881/original/rQJh1RSrr244FcvqH8VsYlSNaL1_5L9_6A.png?1743641657)

**2단계: 빈 대시보드 선택**

Create a Blank Dashboard(빈 대시보드 만들기) 아래에서 Select(선택)를 클릭하세요.

![빈 대시보드 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433890/original/lpbrI6m-ixLatlPsrrpGNDCwvkylimB_Fw.png?1743641724)

**3단계: 권한 설정 후 저장**

New Dashboard(새 대시보드) 모달에서 대시보드를 비공개로 두거나, 이 설정을 끄면 공유 권한에 접근할 수 있어요. 설정이 만족스러우면 Confirm(확인)을 클릭해서 해당 권한으로 새 대시보드를 생성하세요.

![새 대시보드 권한 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433908/original/9V9Sr9PLG29YqUu0VXW1O2t2yRRyqmHvdQ.png?1743641801)

---

### 기존 대시보드에 권한 설정하기

**1단계: 권한 관리 액션**

기존 대시보드에서 3점 "액션 메뉴"를 클릭한 후, Manage Permissions(권한 관리)를 클릭하세요.

![권한 관리 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433750/original/XBQjE4b9QWgmSCGpAV3ShVjvdPCuPizE5g.png?1743641029)

**2단계: 권한 설정 후 저장**

Private(비공개) 토글을 사용해서 비공개(본인만) 또는 공유 권한 사이를 전환하세요.

권한 변경에 만족하면 Save(저장)를 클릭해서 모달을 닫고 대시보드로 돌아가세요.

![기존 대시보드 권한 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433989/original/whua1cN1ou81Jrr3_UI7SGEnO-PPebUdng.png?1743642044)

## 에이전시와 계정 보기

로그인한 사용자의 역할에 따라 권한 화면과 옵션이 달라져요.

### 에이전시 권한 관리 보기

에이전시 관리자는 생성자가 비공개로 표시한 대시보드를 제외하고는 항상 모든 대시보드에 전체 접근 권한을 가져요. 에이전시 보기에서는 에이전시 사용자, 계정 관리자, 계정 사용자라는 세 가지 역할에 접근 설정이 있어요.

![에이전시 대시보드 권한 보기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044434191/original/ag_owqTRkCZiAF5mHoJ3MUHYMr0kLRkY-g.png?1743643164)

### 계정 권한 관리 보기

계정 보기에서는 계정 관리자와 계정 사용자라는 두 가지 역할에 접근 설정이 있어요.

![계정 대시보드 권한 보기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155023290478/original/LNohQGZJfVvUZVBvP4IWBFbbhDDct6fjHA.png?1711017852)

## 역할과 권한 계층 구조

대시보드 권한은 접근 수준의 원활한 관리를 보장하기 위해 명확한 계층 구조를 따라요:

**에이전시 관리자 > 에이전시 사용자 > 계정 관리자 > 계정 사용자**

상위 역할의 권한이 하위 역할의 권한을 재정의해요. 하위 역할은 상위 역할보다 높은 권한을 가질 수 없어요.

예를 들어, 대시보드가 계정 관리자:편집으로 설정되어 있고 계정 사용자:편집이 계정 관리자:보기로 변경되면, 계정 사용자도 자동으로 보기로 변경돼요.

## 데이터 내보내기

데이터 내보내기 권한은 사용자가 대시보드 위젯에서 데이터를 내보낼 수 있는지를 제어해요. 이는 팀이 보고서를 공유하면서도 관리자가 대시보드 데이터 다운로드를 언제 허용할지 제어할 수 있게 도와줘요.

---

**자주 묻는 질문**

**Q: 권한이 설정되지 않으면 어떻게 되나요?**
기본적으로 대시보드는 제한되지 않는 한 모든 사용자에게 표시돼요.

**Q: "본인 데이터만 보기"는 정확히 무엇을 제한하나요?**
대시보드 위젯을 필터링해서 사용자에게 배정된 레코드(연락처, 기회 등)와 관련된 데이터만 표시해요.

**Q: 대시보드에서 사용자나 역할을 제거하려면 어떻게 하나요?**
해당 대시보드의 권한 모달을 다시 열고 사용자나 역할의 체크를 해제하세요.
