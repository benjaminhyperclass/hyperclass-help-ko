# **자동 장바구니 이탈 이메일이란?**

자동 장바구니 이탈 이메일은 고객이 장바구니에 상품을 담고 결제 과정을 시작했으나, 이메일 주소를 입력한 후 구매를 완료하지 않았을 때 자동으로 발송됩니다. 시스템이 이러한 미완료 행동을 감지하고 고객이 다시 돌아와서 결제를 완료하도록 독려하는 후속 이메일을 자동으로 보냅니다. 이 이메일에는 이탈한 장바구니에 접근할 수 있는 링크가 포함됩니다.

이 기능은 **기본적으로 활성화**되어 있으며 백그라운드에서 조용히 작동하여, 스토어 운영자가 수동으로 작업할 필요 없이 **이탈 매출을 회수**하는 데 도움을 줍니다.

---

## 장바구니 이탈 이메일의 주요 장점

- 매출 회수: 이탈한 고객을 다시 참여시켜 놓친 기회를 매출로 전환합니다.

- 완전 자동화: 한번 활성화하면 시스템이 수동 설정 없이도 추적하고 이메일을 발송합니다.

- 메시지 커스터마이징: 브랜드 톤앤매너와 장바구니 가치에 맞게 이메일 내용을 맞춤 설정할 수 있습니다.

- 동적 상품 표시: 장바구니에 남겨진 정확한 상품을 보여줘서 감정적 재연결을 높입니다.

---

## 장바구니 이탈 링크 작동 방식

고객은 장바구니 이탈 이메일에 포함된 고유 링크를 통해 어떤 기기에서든 결제를 재개할 수 있습니다. 이 링크가 어떻게 작동하는지 이해하면 자동 장바구니 이탈 이메일 사용 시 올바른 기대치를 설정할 수 있습니다.

- **기기 간 접근:** 링크는 다양한 기기, 브라우저, 세션에서 작동합니다.

- **장바구니당 하나의 링크:** 각 URL은 단일 이탈 세션을 나타내며, 해당 상품들만 표시됩니다.

- **보기 전용 장바구니:** 쇼핑객은 이 링크에서 새로운 상품을 추가할 수 없습니다.

- **동적 업데이트:** 결제 페이지는 열 때의 최신 상품명과 가격을 보여줍니다.

- 재고 인식: 상품이 *품절*인 경우 *상품 삭제* 버튼이 나타납니다.

- 게스트 결제 지원: 게스트 쇼핑객도 임시 세션을 통해 구매를 완료할 수 있습니다.

- 사용 불가능한 장바구니 대체: 이탈 장바구니의 모든 상품이 삭제되거나 사용할 수 없는 경우, 결제 페이지에 쇼핑 계속하기 CTA가 표시됩니다.

## **장바구니 이탈 이메일 커스터마이징**

후속 이메일의 효과를 극대화하려면 **디자인**과 **내용** 모두를 커스터마이징하세요. 다음 두 가지 주요 방법이 있습니다:

### **기본 이메일 템플릿 재정의**

Hyperclass에는 기본 장바구니 이탈 이메일이 포함되어 있지만, 커스텀 템플릿으로 더 매력적으로 만들 수 있습니다.

### **이메일 템플릿 열기**

**Marketing(마케팅) → Emails(이메일) → Templates(템플릿)**로 이동하여 장바구니 이탈 리마인더에 사용될 이메일을 생성하거나 편집하세요. 여기서 Email Builder(이메일 빌더)를 사용하여 브랜드화된 개인화 메시지를 디자인할 수 있습니다.

### **Templates(템플릿) 탭 사용**

Email Marketing(이메일 마케팅) 내에서 **Templates(템플릿)** 탭을 클릭하여 저장된 모든 이메일 디자인을 확인하세요. 장바구니 이탈 알림에 사용할 템플릿을 선택하거나 생성하는 곳입니다.

## **자주 묻는 질문**

**Q. 장바구니 이탈 이메일을 비활성화할 수 있나요?**

네. 하위 계정에서 **Payments(결제) → Settings(설정) → Notifications(알림)**로 이동하여 이메일을 끄면 됩니다.

**Q. 퍼널이나 웹사이트 주문 폼에서도 작동하나요?**

아니요. 자동 장바구니 이탈 이메일은 **Payments(결제)** 하에서 관리되는 스토어 결제에만 적용되며, 퍼널이나 웹사이트 주문 폼에서는 트리거되지 않습니다. 이런 경우를 회수하려면 주문 폼 방문을 추적하고 지연 후 결제 기록이 없으면 후속 조치를 보내는 워크플로우를 만드세요.

**Q. 고객이 이메일을 제공하지 않으면 어떻게 되나요?**

알림이 발송되지 않습니다. 시스템은 장바구니 이탈 회수를 트리거하기 위해 이메일 주소가 필요합니다.

**Q. 이메일 발송 시간을 조정할 수 있나요?**

네. 기본 지연 시간은 10시간이지만, 동일한 알림 설정 패널에서 수정할 수 있습니다.

**Q. 여러 개의 장바구니 이탈 리마인더를 보낼 수 있나요?**

현재 Hyperclass는 장바구니 이탈당 하나의 자동 장바구니 이탈 알림만 지원합니다.

**Q. 장바구니 이탈에 이메일 대신 SMS를 사용할 수 있나요?**

기본적으로는 불가능합니다. 시스템은 이메일을 보내도록 설계되어 있습니다. 커스텀 워크플로우를 설정하여 SMS를 수동으로 보내거나 자동화의 일부로 포함시킬 수는 있습니다.

**Q: 장바구니 이탈 링크를 공유할 수 있나요?**

네. 다양한 기기, 브라우저, 세션에서 작동합니다.

**Q: 고객이 링크에서 새 상품을 추가할 수 있나요?**

아니요. 결제는 이탈 장바구니 내용에 대해서만 보기 전용입니다.

**Q: 이탈 장바구니의 모든 상품이 사용할 수 없는 경우 어떻게 되나요?**
