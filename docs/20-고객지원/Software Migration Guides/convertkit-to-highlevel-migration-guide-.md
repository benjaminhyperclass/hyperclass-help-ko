# 비주얼 자동화 마이그레이션

ConvertKit은 비주얼 자동화를 사용하여 이메일 퍼널과 자동화 워크플로우를 생성합니다. Hyperclass에서는 Workflows(워크플로우) 기능을 통해 이를 관리하며, 유사하거나 더 발전된 자동화 기능을 제공합니다.

### **1단계:** 비주얼 자동화 문서화

- **자동화 목록 작성:** ConvertKit에서 각 비주얼 자동화를 문서화하세요. 진입점(트리거), 액션, 이벤트, 조건을 포함합니다.

- **자동화 흐름 캡처:** Hyperclass에서 정확하게 재현할 수 있도록 각 자동화의 구체적인 흐름을 스크린샷으로 찍거나 메모해 두세요.

### **2단계:** Hyperclass에서 자동화 재생성

- **Workflows 접근:** Hyperclass에서 `Automation(자동화) → Workflows(워크플로우)`로 이동합니다.

- **새 워크플로우 생성:** `Create New Workflow(새 워크플로우 만들기)`를 클릭하고 ConvertKit 자동화를 복제할 워크플로우 설정을 시작합니다.

- **진입점(트리거) 설정:** Hyperclass의 트리거 옵션을 사용하여 ConvertKit의 진입점과 일치시킵니다(예: 폼 제출, 태그 추가, 구매 완료).

- **액션과 조건 추가:** 원본 자동화 흐름을 반영하여 액션과 조건(예: 이메일 발송, 대기 단계, if/else 분기)을 재생성합니다.

- **워크플로우 테스트:** 구축 완료 후 Hyperclass에서 워크플로우를 테스트하여 모든 트리거, 액션, 조건이 예상대로 작동하는지 확인합니다.

### **3단계:** 마이그레이션 후 점검

- **테스트 실행:** 테스트 연락처를 설정하여 워크플로우를 실행하고 이메일 발송, 태깅, 조건부 분기를 포함한 모든 단계가 올바르게 작동하는지 확인합니다.

# 구독자(연락처) 마이그레이션

ConvertKit의 구독자는 Hyperclass의 연락처와 동일합니다. 구독자 마이그레이션은 ConvertKit에서 내보내서 Hyperclass로 가져오는 과정이며, 태그와 세분화 정보도 포함됩니다.

### **1단계:** ConvertKit에서 구독자 내보내기

- **Subscribers 메뉴로 이동:** ConvertKit에서 `Subscribers → Grow`로 이동합니다.

- **모든 구독자 선택:** 상단의 체크박스를 클릭하여 모든 구독자를 선택합니다. 구독자가 30명을 초과하는 경우 나타나는 파란색 바의 링크를 클릭하여 확인하세요.

- **구독자 내보내기:** `Bulk Actions → Export`를 선택하여 내보내기를 시작합니다. 파일은 계정 소유자에게 이메일로 발송되거나 ConvertKit에서 직접 다운로드할 수 있습니다.

### **2단계:** Hyperclass로 구독자 가져오기

- **Contacts 메뉴로 이동:** Hyperclass에서 `Contacts(연락처) → Import Contacts(연락처 가져오기)`로 이동합니다.

- **CSV 파일 업로드:** ConvertKit에서 내보낸 CSV 파일을 업로드합니다.

- **필드 매핑:** CSV의 필드를 Hyperclass의 적절한 필드에 매핑합니다(예: 이름, 성, 이메일, 태그).

- **연락처 세분화 및 태그 지정:** ConvertKit의 태깅 및 세분화 정보를 사용하여 Hyperclass에서 유사한 세그먼트와 태그를 재생성합니다.

### **3단계:** 마이그레이션 후 점검

- **가져오기 확인:** Hyperclass에서 가져온 연락처를 검토하여 모든 데이터가 올바르게 매핑되고 세분화되었는지 확인합니다.

# 폼 마이그레이션

ConvertKit의 폼은 리드 수집과 구독자 관리에 사용됩니다. Hyperclass의 폼 빌더를 사용하면 CRM과 자동화 워크플로우에 추가적으로 통합된 폼을 재생성할 수 있습니다.

### **1단계:** 폼 세부사항 문서화

- **폼 목록 작성:** ConvertKit에서 생성한 폼을 문서화하세요. 필드, 디자인, 리다이렉트 URL, 인센티브 이메일, reCAPTCHA 등의 설정을 포함합니다.

- **커스텀 필드 캡처:** 데이터 수집에 사용된 모든 커스텀 필드를 기록해 두세요.

### **2단계:** Hyperclass에서 폼 재생성

- **폼 빌더 접근:** Hyperclass에서 `Sites(사이트) → Forms(폼)`으로 이동합니다.

- **새 폼 생성:** `Create New Form(새 폼 만들기)`를 클릭하고 ConvertKit 폼에서 사용된 필드를 추가하기 시작합니다.

- **디자인 커스터마이징:** 배경색, 폰트, 버튼 스타일 등의 디자인 요소를 조정하여 원본 폼과 일치시킵니다.

- **폼 액션 설정:** 제출 후 리다이렉트, 확인 이메일 발송, 태그 적용 등의 폼 액션을 설정합니다.

- **워크플로우와 통합:** 폼이 워크플로우를 트리거하는 경우 해당 Hyperclass 워크플로우와 통합되어 있는지 확인합니다.

### **3단계:** 마이그레이션 후 점검

- **폼 제출 테스트:** 각 폼을 통해 테스트 항목을 제출하여 데이터가 올바르게 수집되고 적절한 워크플로우를 트리거하는지 확인합니다.

# 랜딩 페이지 마이그레이션

ConvertKit의 랜딩 페이지는 리드 생성에 사용되는 독립적인 페이지입니다. Hyperclass를 사용하면 전체 마케팅 전략에 향상된 통합으로 이러한 랜딩 페이지를 재생성할 수 있습니다.

### **1단계:** 랜딩 페이지 세부사항 문서화

- **랜딩 페이지 목록 작성:** ConvertKit에서 각 랜딩 페이지를 문서화하세요. 사용된 템플릿, 콘텐츠, 커스텀 이미지, 폼이나 자동화와의 통합을 포함합니다.

- **커스텀 도메인 설정 캡처:** 랜딩 페이지에서 커스텀 도메인이나 서브도메인을 사용하는지 확인하세요.

### **2단계:** Hyperclass에서 랜딩 페이지 재생성

- **랜딩 페이지 빌더 접근:** Hyperclass에서 `Sites(사이트) → Funnels/Websites(퍼널/웹사이트)`로 이동하여 `Create New Landing Page(새 랜딩 페이지 만들기)`를 선택합니다.

- **템플릿 선택:** ConvertKit 랜딩 페이지의 레이아웃과 스타일에 맞는 템플릿을 선택합니다.

- **콘텐츠 재구축:** Hyperclass 랜딩 페이지에서 콘텐츠, 이미지, 디자인 요소를 재생성합니다.

- **폼 설정:** Hyperclass에서 생성한 폼을 랜딩 페이지에 삽입하고 워크플로우 및 연락처 목록에 올바르게 연결되어 있는지 확인합니다.

- **도메인 설정 구성:** ConvertKit에서 커스텀 도메인을 사용했다면 Hyperclass에서 동일한 도메인이나 서브도메인을 설정합니다.

### **3단계:** 마이그레이션 후 점검

- **미리보기 및 테스트:** 랜딩 페이지를 미리보고 테스트를 실행하여 폼 제출과 리다이렉트를 포함한 모든 요소가 예상대로 작동하는지 확인합니다.

# 이메일 브로드캐스트 마이그레이션

ConvertKit의 이메일 브로드캐스트는 Hyperclass의 이메일 캠페인과 유사합니다. Hyperclass에서 이러한 브로드캐스트를 재생성하여 이메일 마케팅 활동을 계속할 수 있습니다.

### **1단계:** 이메일 브로드캐스트 문서화

- **브로드캐스트 목록 작성:** ConvertKit에서 발송한 이메일 브로드캐스트를 문서화하세요. 제목, 이메일 내용, 수신자 세그먼트를 포함합니다.

- **이메일 템플릿 캡처:** ConvertKit 브로드캐스트에서 사용한 모든 이메일 템플릿의 사본을 저장합니다.

### **2단계:** Hyperclass에서 이메일 브로드캐스트 재생성

- **이메일 캠페인 접근:** Hyperclass에서 `Marketing(마케팅) → Email Campaigns(이메일 캠페인)`로 이동합니다.

- **새 캠페인 생성:** `Create New Campaign(새 캠페인 만들기)`를 클릭하고 ConvertKit의 내용과 디자인을 복제하여 이메일 브로드캐스트를 설정합니다.

- **수신자 세분화:** 가져온 태그와 세그먼트를 사용하여 브로드캐스트의 수신자 목록을 정의합니다.

- **예약 또는 발송:** 브로드캐스트를 즉시 발송할지 나중에 예약할지 선택합니다.

### **3단계:** 마이그레이션 후 점검

- **테스트 이메일 발송:** 테스트 이메일을 발송하여 디자인과 내용을 확인하고 다양한 이메일 클라이언트에서 올바르게 렌더링되는지 확인합니다.

# 커머스 기능 마이그레이션

ConvertKit은 디지털 제품과 구독 결제를 관리하는 도구를 제공합니다. Hyperclass의 결제 통합 기능은 광범위한 마케팅 및 CRM 기능과 통합된 유사한 기능을 지원합니다.

### **1단계:** 커머스 제품 문서화

- **제품 목록 작성:** ConvertKit에서 설정한 모든 디지털 제품이나 구독을 문서화하세요. 가격, 이행 방법, 관련 자동화를 포함합니다.

- **제품 세부사항 캡처:** 각 제품의 URL, 커스텀 도메인, 설정을 기록합니다.

### **2단계:** Hyperclass에서 커머스 제품 재생성

- **Payments & Order Forms 접근:** Hyperclass에서 `Sites(사이트) → Order Forms(주문 폼)`으로 이동하여 새 제품 페이지나 주문 폼을 설정합니다.

- **제품 가격 설정:** 일회성 결제, 구독, 결제 계획을 포함하여 ConvertKit에서 사용한 가격 모델을 재생성합니다.

- **이행 설정:** 디지털 제품 전달이나 구독 관리를 설정하여 고객이 구매 시 자동으로 제품을 받을 수 있도록 합니다.

- **워크플로우와 통합:** 제품 구매를 후속 이메일, 업셀, 고객 세분화를 위한 워크플로우와 연결합니다.

### **3단계:** 마이그레이션 후 점검

- **구매 흐름 테스트:** 테스트 구매를 수행하여 결제가 올바르게 처리되고 제품이 의도한 대로 전달되는지 확인합니다.

![커머스 기능 마이그레이션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155032543035/original/aBBQ6rlOmtjTCbzQ9zwhnUG0AU3vvdkPfg.png?1725903849)

# ConvertKit에서 데이터 내보내기

완전히 전환하기 전에 ConvertKit에서 구독자 목록과 참여 지표 같은 중요한 데이터를 내보내는 것이 필수입니다. 이는 참조용이나 Hyperclass로 가져오기 위함입니다.

### **1단계:** 구독자 목록 내보내기

- **Export로 이동:** ConvertKit에서 `Subscribers → Grow`로 이동하여 `Bulk Actions` 메뉴에서 `Export`를 선택합니다.

- **내보내기 파일 수신:** 내보내기 파일은 이메일로 발송되거나 ConvertKit 계정에서 직접 다운로드할 수 있습니다.

### **2단계:** 내보낸 데이터 검토

- **데이터 무결성 확인:** 내보낸 CSV 파일을 열어서 구독자 세부사항, 태그, 참여 지표, 커스텀 필드를 포함한 모든 필요한 정보가 정확하게 캡처되었는지 확인합니다. 이 데이터는 Hyperclass 내 전략을 수립하거나 백업 목적으로 사용할 수 있습니다.

### **3단계:** Hyperclass로 데이터 가져오기 (선택사항)

- **추가 데이터 가져오기:** 필요한 경우 ConvertKit에서 내보낸 태그, 참여 점수, 커스텀 필드 데이터 등 추가 데이터(구독자 외)를 Hyperclass로 가져올 수 있습니다.

- **추가 데이터 필드 설정:** ConvertKit의 커스텀 필드나 고유한 데이터 유형이 Hyperclass 내에서 올바르게 매핑되고 재생성되도록 합니다.

### **4단계:** 마이그레이션 후 점검

## 자주 묻는 질문

**ConvertKit의 비주얼 자동화를 Hyperclass로 어떻게 마이그레이션하나요?**

ConvertKit의 각 비주얼 자동화를 진입점(트리거), 액션, 이벤트, 조건을 포함하여 문서화하세요. 구체적인 흐름을 스크린샷이나 메모로 남겨 두세요. Hyperclass에서 `Automation → Workflows`로 이동하여 ConvertKit 자동화를 복제하는 새 워크플로우를 생성하고, 일치하는 트리거를 설정하고, 액션과 조건을 추가하여 모든 단계가 올바르게 작동하는지 테스트하세요.

**ConvertKit에서 구독자를 어떻게 내보내나요?**

ConvertKit에서 `Subscribers → Grow`로 이동하여 상단의 체크박스를 클릭해서 모든 구독자를 선택하세요(구독자가 30명을 초과하는 경우 파란색 바의 링크를 클릭하여 확인). 그다음 `Bulk Actions → Export`를 선택하세요. 파일은 계정 소유자에게 이메일로 발송되거나 ConvertKit에서 직접 다운로드할 수 있습니다.

**ConvertKit 구독자를 Hyperclass로 어떻게 가져오나요?**

Hyperclass에서 `Contacts → Import Contacts`로 이동하여 ConvertKit에서 내보낸 CSV 파일을 업로드하고, CSV의 필드를 Hyperclass의 적절한 필드(예: 이름, 성, 이메일, 태그)에 매핑하고, 태깅 정보를 사용하여 Hyperclass에서 유사한 세그먼트를 재생성하세요. 가져오기 후 모든 데이터가 올바르게 매핑되고 세분화되었는지 확인하세요.

**ConvertKit 폼을 Hyperclass에서 어떻게 재생성하나요?**

필드, 디자인, 설정, 커스텀 필드를 포함하여 ConvertKit 폼을 문서화하세요. Hyperclass에서 `Sites → Forms`로 이동하여 동일한 필드를 추가해서 새 폼을 만들고, 원본 폼과 일치하도록 디자인을 커스터마이징하고, 폼 액션(리다이렉트, 확인 이메일, 태그)을 설정하고, 해당 워크플로우와 통합하세요. 각 폼을 테스트하여 데이터 수집과 워크플로우 트리거를 확인하세요.

**ConvertKit에서 Hyperclass로 랜딩 페이지를 어떻게 마이그레이션하나요?**

템플릿, 콘텐츠, 이미지, 도메인 설정을 포함하여 ConvertKit의 각 랜딩 페이지를 문서화하세요. Hyperclass에서 `Sites → Funnels/Websites`로 이동하여 유사한 템플릿을 사용해서 새 랜딩 페이지를 만들고, 콘텐츠와 디자인 요소를 재구축하고, Hyperclass 폼을 삽입하고, 커스텀 도메인 설정을 구성하세요. 각 랜딩 페이지를 미리보고 테스트하여 모든 요소가 올바르게 작동하는지 확인하세요.

**Hyperclass에서 이메일 브로드캐스트를 어떻게 재생성하나요?**

제목, 내용, 수신자 세그먼트를 포함하여 ConvertKit 이메일 브로드캐스트를 문서화하세요. 이메일 템플릿의 사본을 저장하세요. Hyperclass에서 `Marketing → Email Campaigns`로 이동하여 내용과 디자인을 복제하는 새 캠페인을 만들고, 가져온 태그를 사용하여 수신자 목록을 정의하고, 예약하거나 발송하세요. 테스트 이메일을 발송하여 이메일 클라이언트에서 올바른 렌더링을 확인하세요.
