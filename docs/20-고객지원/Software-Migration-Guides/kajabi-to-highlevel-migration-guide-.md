# Kajabi에서 Hyperclass로 이전하기 (Migration Guide)

# 이전 준비

### 1단계: 현재 Kajabi 설정 검토

- 핵심 자산 파악: Kajabi 내 모든 강의, 오퍼, 자동화 및 기타 디지털 자산을 목록으로 정리하세요.

- 워크플로우 문서화: 업무에 중요한 Kajabi 자동화, 이메일 캠페인, 연동 설정을 상세히 기록하세요.

- 데이터 규모 평가: 이전해야 할 콘텐츠(동영상, 첨부파일 등)와 데이터(연락처, 주문 등)의 양을 파악하세요.

### 2단계: 이전 목표 정의

- 명확한 목표 설정: CRM 기능 강화, 통합 마케팅 자동화, 비용 절감 등 Hyperclass로 이전하려는 이유를 명확히 하세요.

- 우선순위 설정: 가장 중요한 콘텐츠와 워크플로우를 먼저 이전할 순서를 결정하세요.

### 3단계: 백업 준비

- 데이터 백업: 이전 과정을 시작하기 전에 모든 데이터를 안전하게 백업하세요.

# 강의 이전

Hyperclass의 Kajabi 가져오기 도구를 사용하면 동영상, 텍스트, 이미지를 포함한 강의를 쉽게 이전할 수 있습니다. 다만 과제와 퀴즈는 자동으로 이전되지 않으므로 Hyperclass에서 수동으로 다시 만들어야 합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042993816/original/qqQQl4sHcx6Vw4KR2Aq9tBdLgE8N-avogw.png?1741632257)

### 1단계: 성공적인 이전을 위한 준비

- 백업 만들기: 시작하기 전에 Kajabi 테마와 강의 콘텐츠를 백업해 두세요.

- 이전 계획 세우기: 수강생들의 불편을 최소화하고 원활한 과정을 위해 사용량이 적은 시간대에 이전을 시작하세요.

### 2단계: Hyperclass로 강의 이전하기

- Kajabi 강의 가져오기 도구 사용: Hyperclass의 사이트(Sites) → 멤버십(Memberships)으로 이동해 Kajabi에서 가져오기를 선택하세요.

- Kajabi 계정 인증: 안내에 따라 Kajabi 계정을 안전하게 연결하세요.

- 가져올 강의 선택: 이전하려는 강의를 선택하세요. 발행된 레슨만 이전되므로 모든 콘텐츠가 공개 상태인지 확인하세요.

- 가져오기 시작: 강의 가져오기를 클릭하고 완료될 때까지 기다리세요. 강의 규모에 따라 몇 분 정도 소요될 수 있습니다.

### 3단계: Hyperclass에서 과제와 퀴즈 다시 만들기

- 상품 선택
  Hyperclass의 사이트(Sites) → 멤버십(Memberships) → 상품(Products)으로 이동하세요.

- 평가를 추가하려는 상품을 선택하세요.

- 섹션 제목 옆의 "+" 아이콘을 클릭하고 평가 추가를 선택하세요.

- 평가 만들기
  평가 제목을 입력하고 저장을 클릭하세요.

- 합격 점수 설정 (선택사항):
  평가에 합격 점수가 필요한지 결정하세요.

- 필요하다면 합격에 필요한 최소 점수를 설정하세요.

- 커스텀 메시지:
  학습자 점수에 따른 합격/불합격 메시지를 작성하세요.

- 질문 추가:
  단일 선택(정답 1개) 또는 다중 선택(정답 여러 개) 중 선택하세요.

- 질문을 입력하고 "선택지 추가"를 사용해 가능한 답변을 입력하세요.

- "정답 옵션" 체크박스를 사용해 올바른 선택지를 표시하세요.

- 평가 저장 및 발행:
  저장 후 상품 편집 페이지로 돌아가세요.

- 평가가 준비되면 옆의 "발행" 옵션을 클릭하세요.

- 발행되면 모든 강의 회원이 평가를 보고 접근할 수 있습니다.

# 연락처 이전

### 1단계: Kajabi에서 연락처 목록 내보내기

- Kajabi의 연락처(Contacts) 탭으로 이동하세요.

- 모든 연락처를 선택하고 일괄 작업 드롭다운을 사용해 CSV 파일로 내보내세요.

- CSV 파일이 Kajabi 계정에 연결된 이메일 주소로 전송됩니다. 링크가 3일 후 만료되므로 즉시 다운로드하세요.

### 2단계: Hyperclass 가져오기를 위한 CSV 준비

- 날짜 형식 확인: CSV 내 모든 날짜 필드가 YYYY-MM-DD 형식인지 확인하세요. Hyperclass 호환성을 위해 필수입니다.

- 특수문자 인코딩: CSV 파일이 UTF-8 문자 인코딩을 사용하는지 확인해 특수문자와 악센트 문자를 올바르게 처리하세요.

- 데이터 정리: 파일에서 줄바꿈, 이모지, 불필요한 특수문자를 제거하세요. 이 단계가 가져오기 오류를 방지합니다.

- 커스텀 필드 재생성: Hyperclass에 로그인하여 연락처(Contacts) → 설정(Settings) → 커스텀 필드(Custom Fields)로 이동하세요. Kajabi와 유사하게 연락처 데이터를 저장할 커스텀 필드를 추가하세요.

### 3단계: Hyperclass로 연락처 가져오기

- Hyperclass 로그인: 좌측 메뉴를 사용해 연락처(Contacts) 섹션으로 이동하세요.

- 가져오기 과정 시작: 연락처 가져오기 버튼을 클릭하세요.

- CSV 파일 업로드: 준비된 CSV 파일을 선택하여 업로드하세요.

- 필드 매핑: Hyperclass가 CSV 파일의 필드를 Hyperclass의 해당 필드와 매핑하도록 안내합니다. 예를 들어 CSV의 "이름"을 Hyperclass의 "이름"과 매핑하세요.

필드 매핑 팁: 모든 필수 필드가 올바르게 매핑되었는지 확인하세요. CSV의 필드가 Hyperclass에 해당 필드가 없다면, 진행하기 전에 Hyperclass에서 커스텀 필드를 만들어야 할 수도 있습니다.

- 태그와 목록 할당: 가져오기 과정에서 연락처를 특정 목록에 할당하고 태그를 적용할 수 있습니다. "강의 A - 등록됨"이나 "Kajabi에서 가져옴" 같은 출처 표시 태그를 적용하는 것을 고려하세요.

- 가져오기 완료: 모든 필드 매핑과 태그/목록 설정 후 가져오기를 클릭하세요. Hyperclass가 가져오기를 처리하여 연락처가 계정에 추가됩니다.

[CSV를 통한 대량 가져오기 문제 해결에 대한 자세한 정보는 여기를 클릭하세요](../../02-연락처/스마트-리스트/troubleshooting-bulk-imports-via-csv.md)

# 캘린더 이전

Hyperclass와 마찬가지로 Kajabi의 예약은 일회성 및 반복 이벤트를 지원합니다. Google Calendar, Microsoft Outlook 같은 외부 캘린더를 이전하여 Hyperclass 캘린더에서 중복 예약을 방지하세요.

[Kajabi 캘린더와 예약에 Calendly를 사용 중이시라면 이 가이드를 확인하세요!](calendly-to-highlevel-migration-guide-.md)

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042993817/original/_7KO_3NlvTJnVZEQHfwHxKmwUgr4b6_OAg.png?1741632257)

### 1단계: Kajabi에 연결된 캘린더 검토

- 캘린더 연동: 상품(Products) → 코칭(Coaching) → 설정(Settings)으로 이동해 연결된 Google과 Outlook 캘린더를 확인하세요.

- 캘린더 세부사항 문서화: 각 캘린더의 용도와 사용 위치(웹사이트나 Zoom 미팅용 등)를 기록하세요. 유료 세션 포함 여부도 함께 기록하세요.

### 2단계: Hyperclass에서 Kajabi 캘린더 재생성

- Hyperclass 캘린더 만들기: 하위 계정 설정(Sub-account Settings) → 캘린더(Calendars)로 이동하세요. Kajabi의 캘린더와 일치하도록 각 캘린더를 설정하세요. 리마인더, 할당된 사용자, 미팅 위치 같은 설정을 조정하세요.

- 캘린더 연동 추가: 하위 계정 설정(Subaccount Settings) → 캘린더(Calendars) → 연결(Connections)로 이동해 Google이나 Outlook 캘린더를 Hyperclass와 동기화하세요.

- 반복 예약 설정: 반복 이벤트의 경우 Hyperclass 캘린더 설정에서 반복 규칙을 수동으로 구성하세요.

- 미리보기 및 테스트: 기존 Kajabi 설정과 비교하여 새 캘린더가 예상대로 작동하는지 확인하세요.

### 3단계: Hyperclass 캘린더 맞춤 설정

Hyperclass는 다양한 캘린더 스타일과 설정을 제공합니다. 일반적인 구성으로 시작하는 방법은 다음과 같습니다:

- 라운드 로빈 캘린더 만들기: 여러 팀원에게 예약을 배분하려면 이 설정을 사용하세요.

- 수업 일정 설정: 그룹 세션이나 수업을 위한 캘린더를 구성하세요.

- 반복 예약: 지속적인 미팅이나 수업을 위한 반복 세션을 설정하세요.

- 고급 맞춤 설정: 필요에 맞게 색상, 예약 가능 시간, 사용자 권한을 조정하세요.

이러한 캘린더 유형에 대한 자세한 안내는 Hyperclass 도움말 센터에서 캘린더 설정을 검색하세요.

# 오퍼 및 자동화 이전

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042993812/original/3ItRz8S-GVvhG8n6dC0VHa2L_084ug-5oQ.png?1741632256)

### 1단계: 기존 Kajabi 오퍼와 자동화 검토

- 활성 오퍼 목록: Kajabi 계정의 모든 무료 및 유료 오퍼를 식별하고, 관련 상품, 가격, 결제 설정을 포함하세요.

- 자동화 문서화: 강의 접근 권한이나 기타 작업을 제어하는데 사용되는 이메일 시퀀스, 태그, 트리거를 포함하여 Kajabi의 모든 자동화를 기록하세요.

### 2단계: Kajabi에서 이메일 템플릿 가져오기

- Hyperclass에서 새 이메일 템플릿 만들기:
  Hyperclass 계정의 이메일 마케팅(Email Marketing) → 템플릿(Templates)으로 이동하세요.

- 새로 만들기를 클릭하고 템플릿 가져오기를 선택하세요.

- 드롭다운 메뉴에서 Kajabi를 선택하세요.

- 제공된 이메일 ID 복사: 템플릿 전송에 사용됩니다.

- Kajabi로 이동: 가져오려는 템플릿을 복사한 이메일 ID로 테스트 이메일이나 캠페인으로 전송하세요.

- Hyperclass로 돌아가기: 계속을 클릭하고 템플릿 이름을 지정한 후 템플릿 만들기를 선택하세요.

자세한 내용은 [Hyperclass Kajabi 이메일 템플릿 가져오기 가이드](../../10-마케팅/how-to-use-the-kajabi-email-template-importer.md)를 확인하세요.

### 3단계: Hyperclass에서 오퍼 재생성

- Hyperclass의 멤버십으로 이동: "멤버십" 탭으로 이동하여 드롭다운 메뉴를 클릭한 후 "오퍼"를 선택하세요.

- 오퍼 만들기: 제목을 입력하고 상품(강의)을 선택하며 가격을 설정한 후 만들기를 클릭하세요.
오퍼는 발행하기 전까지 "초안" 탭에 표시됩니다.

- 멤버십 오퍼를 상품에 연결:
  "결제(Payments)" 탭을 클릭한 후 "상품(Products)" 탭으로 이동하세요.

- "+ 상품 만들기"를 클릭하고 상품 정보를 입력하세요.

- 가격 이름, 유형, 금액을 입력하세요.

- "추가 옵션"으로 이동하여 "멤버십" 옵션을 토글하고 적절한 멤버십 오퍼를 선택하세요.

참고: 오퍼가 드롭다운 메뉴에 나타나려면 발행되어야 합니다. 자세한 내용은 [Hyperclass 오퍼 설정 가이드](../../08-결제/기타/how-to-create-payment-products-of-membership-courses-.md)를 참조하세요.

### 4단계: Hyperclass에서 자동화 재생성

- 새 워크플로우 구축: Hyperclass의 워크플로우(Workflows)를 사용하여 Kajabi 자동화를 복제하세요. Kajabi 워크플로우와 일치하는 트리거, 액션, 조건을 설정하세요.

- 사용 가능한 액션: 다음과 같은 다양한 액션을 활용하세요:
  강의 접근 권한 부여: 학습자에게 특정 강의에 대한 접근 권한을 제공합니다.

- 이메일 발송: 이메일 시퀀스를 복제합니다.

- 조건 분기(If/Else): 태그와 사용자 여정을 관리합니다.

전체 개요는 [워크플로우와 자동화 소개](../../07-워크플로우/introduction-to-workflows-and-automations.md)와 [강의 접근 액션 가이드](../../07-워크플로우/기타/action-course-grant-offer.md)를 참조하세요.

### 5단계: 워크플로우 테스트

- 각 워크플로우를 테스트하여 예상대로 작동하는지 확인하세요. 트리거, 이메일 시퀀스, 강의 접근 설정을 검증하고 필요에 따라 조정하세요.

# 웹사이트와 퍼널 이전

Hyperclass의 웹사이트 빌더를 사용하여 홈페이지, 상품 페이지, 연락처 페이지를 포함한 Kajabi 사이트의 페이지를 재생성하세요. Hyperclass의 템플릿과 블록을 활용하여 브랜드에 맞게 디자인을 맞춤 설정하세요.

### 1단계: Hyperclass에서 웹사이트 페이지와 퍼널 재생성

- 사이트 빌더 접근: Hyperclass의 사이트(Sites) → 퍼널 & 웹사이트(Funnels & Websites)로 이동하여 시작하세요.

- 템플릿 라이브러리 사용: Kajabi 페이지와 가장 잘 맞는 템플릿을 찾아 선택하세요. 이렇게 하면 설정 속도를 높이고 일관된 디자인을 유지할 수 있습니다. [템플릿 라이브러리에 대한 자세한 내용은 여기에서 확인하세요](../../36-기타/리커버리/how-does-the-template-library-work-.md).

- 각 페이지 구축: 홈페이지, 상품 페이지, 랜딩 페이지, 연락처 페이지 같은 Kajabi의 주요 페이지를 재생성하세요. 히어로 배너, 후기, 연락처 폼 같은 섹션을 복제하기 위해 미리 만들어진 블록을 활용하세요.

### 2단계: 페이지 저장, 초안 작성, 발행

- 각 페이지나 퍼널을 구축하면서 작업을 저장하고, 변경사항을 미리보기하며, 준비가 되면 발행하세요. 이렇게 하면 모든 변경사항이 원할 때 바로 적용됩니다. 자세한 안내는 [페이지 저장 및 발행](../../06-사이트/기타/save-draft-and-publish-funnels-and-websites.md)을 방문하세요.

### 3단계: DNS 레코드 설정

- A 레코드 추가: 루트 도메인이나 서브도메인에 162.159.140.166을 사용하세요.
예: yoursitename.com

- 참고: Cloudflare를 사용하는 경우, Hyperclass가 Cloudflare 프록시를 지원하지 않으므로 프록시 상태를 DNS Only로 설정하세요.

- CNAME 레코드 추가: 도메인 등록업체(예: GoDaddy나 Cloudflare)에서 서브도메인에 sites.ludicrous.cloud를 사용하세요.
예: sometext.yoursitename.com

### 4단계: Hyperclass에 도메인 추가

- 설정(Settings) → 도메인(Domains)으로 이동한 후 + 도메인 추가를 클릭하세요.

- 도메인(예: mydomain.com)이나 서브도메인(예: offer.mydomain.com)을 입력하고 수동으로 레코드 추가를 선택하세요.

- 루트 도메인의 경우, www 리다이렉트를 활성화하여 트래픽을 메인 도메인으로 연결하세요.

### 5단계: 도메인을 퍼널이나 웹사이트에 연결

- DNS 확인 후 도메인과 연결할 퍼널이나 웹사이트를 선택하세요.

- 도메인의 기본 랜딩 페이지(예: 홈페이지)를 선택하세요.

- 도메인 연결을 클릭하세요. 시스템이 자동으로 SSL 인증서를 추가하며, 처리에 몇 분 정도 소요될 수 있습니다.

도메인 연결에 대한 자세한 내용은 Hyperclass 도메인 설정 가이드)를 방문하세요.

# 최종 점검 및 교육

### 1단계: 최종 데이터 검증 수행

- 데이터 교차 확인: 연락처, 강의 콘텐츠, 오퍼를 포함한 모든 데이터가 성공적으로 이전되었는지 확인하세요.

- 워크플로우 검증: 모든 워크플로우와 자동화가 Hyperclass에서 정상 작동하는지 확인하세요.

### 2단계: 팀원 교육

- Hyperclass 교육: Hyperclass의 CRM, 강의 관리, 자동화 도구 사용법에 대해 팀에게 교육을 제공하세요.

- 지원 리소스 활용: 팀원들이 지속적인 학습을 위해 Hyperclass의 지원 리소스를 활용하도록 격려하세요.

### 3단계: 모니터링 및 최적화

- 성과 모니터링: Hyperclass에서 새 설정의 성과를 추적하고 필요에 따라 조정하세요.

- 지속적인 개선: Hyperclass 설정을 정기적으로 검토하고 개선하여 비즈니스 목표에 부합하는지 확인하세요.
