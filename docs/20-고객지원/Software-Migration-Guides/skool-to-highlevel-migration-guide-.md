# Skool에서 Hyperclass로 마이그레이션 가이드

이 문서는 Skool에서 Hyperclass로 온라인 커뮤니티, 강의, 관련 콘텐츠를 이전하는 종합 가이드입니다. Hyperclass의 고급 CRM, 마케팅 자동화, 커뮤니티 구축 도구를 활용하여 회원 참여를 높이고 운영을 간소화하면서 원활하게 전환하는 것이 목표입니다.

# 마이그레이션 준비

### 1단계: 현재 Skool 설정 검토

- **핵심 자산 파악**: Skool에서 현재 사용 중인 커뮤니티 회원, 강의, 콘텐츠, 이벤트, 자동화 워크플로우 등 모든 필수 요소를 목록으로 정리하세요.

- **데이터 규모 평가**: 이전할 데이터의 크기와 복잡성을 파악하세요. 강의 수, 회원 수, 커뮤니티 게시물 등을 포함합니다.

### 2단계: 마이그레이션 목표 정의

- **명확한 목표 설정**: CRM 통합 개선, 고급 마케팅 자동화, 통합 플랫폼 등 Hyperclass로 이전하는 주요 이유를 정의하세요.

- **기능 우선순위 정하기**: 커뮤니티 참여, 강의 콘텐츠, 회원 관리에 중점을 두고 마이그레이션의 중요 구성 요소를 식별하세요.

### 3단계: 백업 준비

- **Skool 데이터 내보내기**: Skool에서 모든 관련 데이터(회원 목록, 강의, 커뮤니티 콘텐츠)를 CSV 파일이나 기타 적절한 형식으로 내보내세요.

- **문서 백업**: 마이그레이션 과정을 시작하기 전에 데이터 안전을 보장하기 위해 내보낸 모든 파일과 관련 문서를 안전하게 저장하세요.

# 데이터 이전

### 1단계: Skool에서 데이터 내보내기

- **회원 내보내기**: Skool에서 회원 섹션으로 이동해 내보내기 기능을 사용하여 이름, 이메일, 커스텀 필드를 포함한 회원 목록을 CSV 파일로 다운로드하세요.

- **강의 내보내기**: 비디오 링크, 설명, 리소스를 포함한 강의 콘텐츠를 다운로드하고, 모든 강의 자료가 Hyperclass에 재업로드할 수 있도록 저장되었는지 확인하세요.

- **커뮤니티 콘텐츠 내보내기**: Hyperclass에서 재생성해야 하는 중요한 커뮤니티 게시물, 토론, 고정된 콘텐츠를 수동으로 복사하거나 내보내세요.

### 2단계: Hyperclass로 데이터 가져오기

- **회원 가져오기**: Hyperclass에서 `연락처(Contacts) → 연락처 가져오기(Import Contacts)`로 이동해 Skool에서 내보낸 CSV 파일을 업로드하세요. 가져오기 과정에서 모든 필드가 올바르게 매핑되었는지 확인하세요.

- **강의 재생성**: Hyperclass의 강의(Courses) 섹션으로 이동해 이전에 내보낸 콘텐츠(비디오, 설명, 리소스)를 업로드하여 새 강의를 만드세요.

- **커뮤니티 콘텐츠 재구축**: Hyperclass의 커뮤니티 도구를 사용해 중요한 토론, 게시물, 기타 콘텐츠를 재생성하세요. 중요한 게시물이나 공지사항은 추천 게시물 기능을 사용해 고정하세요.

# 커뮤니티와 강의 재구축

### 1단계: Hyperclass에서 커뮤니티 채널 설정

- **채널 생성**: Hyperclass에서 Skool에 있던 각 커뮤니티나 토론 그룹에 대한 새 채널을 만드세요. 개인정보 옵션과 접근 레벨을 포함한 채널 설정을 맞춤화하세요.

- **회원 초대**: 모든 커뮤니티 회원에게 이메일이나 고유 초대 링크를 통해 초대를 보내세요. Hyperclass의 `연락처(Contacts) → 이메일 발송(Send Email)` 기능을 사용해 이 과정을 간소화하세요.

### 2단계: 강의 재생성 및 정리

- **강의 구조 설정**: Hyperclass의 강의 생성 도구를 사용해 Skool에서와 유사하게 강의를 구성하세요. 필요에 따라 섹션과 모듈을 만들고 모든 관련 콘텐츠를 업로드하세요.

- **강의 접근 권한 할당**: 각 강의에 대한 접근 권한을 설정하여 적절한 회원만 특정 콘텐츠에 접근할 수 있도록 하세요. 이는 Skool에서 이전에 설정했던 접근 제어를 모방하는 것입니다.

### 3단계: 회원 역할 및 권한 관리

- **역할 할당**: Hyperclass에서 회원에게 관리자(Admin), 중재자(Moderator), 기여자(Contributor) 등의 역할을 할당하여 커뮤니티 계층 구조와 권한을 보존하세요.

- **회원 권한 설정**: 할당된 역할에 따라 특정 채널, 강의, 커뮤니티 기능에 대한 접근을 제어하도록 회원 권한을 구성하세요.

# 이벤트와 자동화 재구축

### 1단계: Hyperclass에서 이벤트 재생성

- **이벤트 캘린더 설정**: Hyperclass의 캘린더 기능을 사용해 Skool에서 예약했던 이벤트를 재생성하세요. 반복 이벤트를 설정하고 모든 관련 회원이 초대되었는지 확인하세요.

- **이벤트 알림 구성**: Hyperclass의 커뮤니케이션 도구를 활용해 높은 회원 참여를 보장하기 위해 이벤트에 대한 자동 이메일 및 SMS 리마인더를 설정하세요.

### 2단계: 자동화 워크플로우 재구축

- **자동화 플로우 생성**: Hyperclass에서 `자동화(Automations) → 워크플로우(Workflows)`로 이동해 환영 메시지, 강의 완료 알림, 이벤트 리마인더 등 Skool의 자동화 과정을 재생성하세요.

- **자동화 테스트**: 새로 만든 자동화 워크플로우에서 테스트를 실행해 올바르게 작동하는지 확인하고, 필요에 따라 조정하세요.

# 테스트와 검증

### 1단계: 커뮤니티와 강의 기능 테스트

- **엔드 투 엔드 테스트 수행**: 채널 가입, 강의 접근, 토론 참여 등 회원 상호작용을 시뮬레이션하여 모든 것이 의도한 대로 작동하는지 확인하세요.

- **데이터 무결성 검증**: 회원 목록, 강의 콘텐츠, 커뮤니티 게시물 등 가져온 데이터를 교차 확인하여 모든 것이 정확하고 접근 가능한지 확인하세요.

### 2단계: 이벤트와 자동화 설정 검증

- **테스트 시나리오 실행**: 각 이벤트와 자동화 워크플로우를 테스트하여 트리거와 액션이 올바르게 작동하는지 확인하고, 모든 회원에게 원활한 경험을 보장하세요.

- **커뮤니케이션 채널 검증**: 이메일과 SMS를 포함한 모든 커뮤니케이션 액션이 워크플로우 내에서 올바르게 실행되는지 확인하세요.

# 팀 교육과 전환

### 1단계: 팀원 교육

- **Hyperclass 교육**: 팀원들에게 Hyperclass 인터페이스를 숙지시키는 교육 세션을 진행하세요. 특히 커뮤니티 관리, 강의 설정, 자동화 도구에 중점을 두세요.

- **문서 제공**: 전환 기간 동안 팀원들을 지원하기 위해 Hyperclass 기능에 맞춤화된 사용자 가이드와 SOP를 배포하세요.

### 2단계: 모니터링과 최적화

- **시스템 성능 모니터링**: 마이그레이션 후 커뮤니티 참여, 강의 완료율, 자동화 효과성의 성능을 정기적으로 검토하세요.

- **워크플로우 최적화**: 팀 피드백과 운영 요구사항에 따라 워크플로우와 커뮤니티 구조를 조정하세요.

# Skool 서비스 종료

### 1단계: 전환 기간

- **시스템 병렬 운영**: 회원과 관리자의 원활한 전환을 보장하기 위해 Skool과 Hyperclass를 잠시 병렬로 운영하는 것을 고려하세요.

- **Skool 단계적 제거**: Hyperclass에 대한 신뢰가 높아지고 모든 회원이 성공적으로 전환되면서 Skool에 대한 의존도를 점진적으로 줄이세요.

### 2단계: Skool 구독 해지

- **최종 백업**: Skool 구독을 해지하기 전에 필요한 모든 데이터가 백업되었는지 확인하세요.

- **공식 해지**: Skool의 프로세스를 따라 구독을 해지하고 관련 서비스를 종료하세요.

### 3단계: 마이그레이션 후 검토

- **성공 검토**: 마이그레이션의 성공을 평가하고, 향후 참조를 위해 발생한 문제와 해결책을 문서화하세요.

- **지속적인 모니터링**: Hyperclass의 성능을 계속 모니터링하며 커뮤니티와 강의 관리를 최적화하기 위해 필요한 조정을 하세요.
