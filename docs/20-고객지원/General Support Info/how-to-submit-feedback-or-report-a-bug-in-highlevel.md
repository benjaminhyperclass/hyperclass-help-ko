# 하이퍼클래스에서 피드백 제출 또는 버그 신고하는 방법

하이퍼클래스에서 피드백을 제출하거나 기술적 문제를 신고할 때, **"클릭하여 업로드"** 버튼을 클릭하면 이제 **전체 화면의 고해상도 스냅샷**이 자동으로 캡처됩니다. 이를 통해 정확한 시각적 컨텍스트를 제공하여 지원팀이 문제를 더 빠르게 진단하고 후속 질문을 줄일 수 있습니다.

## **개요**

하이퍼클래스에는 사용자가 플랫폼에서 직접 피드백을 제출하거나 버그를 신고할 수 있는 인앱 도구가 포함되어 있습니다. 문제 해결 정확도를 향상시키기 위해, 첨부파일을 추가하는 순간 전체 화면 이미지를 자동으로 캡처하도록 업로드 플로우가 개선되었습니다.

이를 통해 지원팀과 제품팀이 문제가 발생했을 때 화면에 정확히 무엇이 표시되었는지 확인할 수 있습니다.

---

## **피드백 및 버그 신고 액세스 방법**

피드백을 제출하거나 기술적 문제를 신고하려면:

- 하이퍼클래스의 아무 페이지나 엽니다.

- 우측 상단의 **도움말 및 지원(Help & Support)** 아이콘을 클릭합니다.

![피드백 제출 방법](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067027237/original/F7g41M81EDDNU_z_lQcJgUtqU9oHnJnGwA.png?1773667862)

- **티켓 생성(Raise Ticket)**을 선택하여 지원 플로우를 시작합니다.

![티켓 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067027269/original/dYGTTAu4eOoZUbC3YiWLOUkmoEUStRGFUA.png?1773667878)

---

## **기술 지원팀으로 라우팅**

티켓을 시작한 후, 하이퍼클래스는 요청을 올바르게 라우팅하기 위해 몇 가지 간단한 질문을 할 수 있습니다.

- **기술 지원(Technical Support)**을 선택합니다.

- 요청 시, 관련 범위를 선택합니다 (예: **하위 로케이션 내에서(Within a Sub-Location)**).

- 문제와 관련된 제품 영역을 선택합니다 (캘린더, 자동화, 대화 등).

라우팅이 완료되면 문제 제출 양식이 열립니다.

---

## **새로운 기능: 업로드 시 자동 화면 캡처**

문제 제출 양식에 도달하여 **"클릭하여 업로드"**를 클릭하면:

- **전체 화면의 고해상도 이미지**가 자동으로 캡처됩니다

- 캡처는 **업로드가 시작되는 순간 즉시** 발생합니다

- 추가 단계나 설정은 필요하지 않습니다

이 동작은 백그라운드에서 실행되며 스크린샷을 신고서에 직접 첨부합니다.

---

## **화면 캡처 작동 방식**

- 캡처는 **"클릭하여 업로드"**가 선택될 때**만** 실행됩니다

- 이미지에는 활성 창뿐만 아니라 **전체 화면**이 포함됩니다

- 처음 사용 시 화면 접근을 허용하는 브라우저 권한 프롬프트가 나타날 수 있습니다

![권한 요청](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064211407/original/jWw69h8oc7BI0x12C05zG3rxAX3aPKOVwQ.png?1770230544)

권한이 부여되면 캡처가 자동으로 완료됩니다.

---

## **스크린샷 첨부 완료**

화면이 캡처된 후:

- 제출 양식에 이미지가 첨부파일로 표시됩니다

- 파일이 성공적으로 추가되었음을 확인하는 표시가 나타납니다

- 스크린샷이 피드백이나 버그 신고와 함께 전송됩니다

![스크린샷 첨부 완료](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064211417/original/dfK4Sq_6vS8c3jFUYPyxo9fz7xwDho7tJQ.png?1770230560)

## **이 기능이 중요한 이유**

자동 전체 화면 캡처는 다음과 같은 도움을 줍니다:

- 지원팀과의 반복적인 커뮤니케이션 감소

- 조사 및 해결 속도 향상

- 부분 스크린샷으로는 놓칠 수 있는 UI 컨텍스트 캡처

- 복잡한 문제 진단 시 정확도 향상

---

## **변경되지 않은 사항**

이 개선사항은 다음을 **변경하지 않습니다**:

- 피드백이나 버그 신고를 시작하는 방법

- **클릭하여 업로드** 옵션이 표시되는 위치

- 사용자 역할, 권한 또는 액세스 수준

- 다른 파일을 수동으로 업로드하는 기능

---

## **자주 묻는 질문**

**Q: 화면 캡처가 자동으로 이루어지나요?**

예. **"클릭하여 업로드"**를 클릭하면 캡처가 자동으로 발생합니다.

**Q: 화면의 어느 부분이 캡처되나요?**

전체 화면이 고해상도로 캡처됩니다.

**Q: 특정 창을 선택할 수 있나요?**

아니요. 전체 화면 캡처는 완전한 진단 컨텍스트를 보장합니다.

**Q: 이것은 버그 신고에만 적용되나요?**

**클릭하여 업로드**를 사용하여 피드백을 제출하거나 기술적 문제를 신고할 때 적용됩니다.
