# 시작하기 - 파이프라인과 기회 관리 설정

영업 파이프라인을 정리하는 것이 어렵게 느껴질 수 있지만, 성공을 위한 업무 공간을 정리하는 것과 같다고 생각해보세요. 모든 것이 제자리에 있고 놓치는 것이 없어야 합니다. 체계적인 파이프라인이 있으면 리드를 쉽게 추적하고, 후속 조치를 자동화하며, 거래를 신속하게 성사시킬 수 있습니다. 영업을 체계적으로 정리할 준비가 되셨나요? 시작해보겠습니다!

## 영업 파이프라인 생성

새로운 영업 파이프라인을 설정하는 것은 매우 간단하며, 모든 리드가 영업 프로세스에서 어디에 위치하는지 정확히 시각화하는 데 도움이 됩니다. 시작하는 방법은 다음과 같습니다:

- **Opportunities(기회 관리)** 메뉴로 이동하세요.

- Pipelines(파이프라인)을 클릭하세요.

- **Create New Pipeline(새 파이프라인 만들기)**을 클릭하세요.

![새 파이프라인 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046582275/original/juyyLUC63RVJOoO-VFRQU5DmXJPyR1KiCw.png?1747185927)

- 세부 정보(이름, 단계 등)를 입력하고 **Save(저장)**을 클릭하세요.

![파이프라인 세부 정보 입력](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046582299/original/hosafNsbFiyMPIKTWvAwcYU_rpnev08erA.png?1747186085)

팀의 워크플로우에 맞게 파이프라인 단계를 커스터마이징하는 것을 고려해보세요:

- 신규 리드
- 핫 리드
- 예약 완료
- 노쇼
- 지연
- 성사
- 무응답

더 자세히 알고 싶으시다면:

- [단계별 가이드: 파이프라인 생성하기]([step-by-step-guide-creating-pipelines](step-by-step-guide-creating-pipelines.md))

- [파이프라인 이해하기]([understanding-pipelines](understanding-pipelines.md))

**다음으로**, 기존 연락처를 가져와서 파이프라인을 채울 수 있습니다.

## 연락처를 기회로 가져오기

이벤트나 다른 플랫폼에서 연락처 목록을 가지고 있으신가요? 좋은 소식입니다! 몇 번의 클릭만으로 연락처를 영업 파이프라인으로 바로 가져와서 그 리드들을 기회로 전환할 수 있습니다:

- Opportunities(기회 관리) 메뉴로 이동하세요.

- **Import(가져오기)**를 클릭하세요.

![가져오기 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046582317/original/Zn1BtwdZenzYQukjv876FbzvC4VGCHqg5w.png?1747186171)

- **Opportunities(기회)**를 선택하세요.

- **Next(다음)**를 클릭하세요.

![기회 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046582328/original/oC696YzN-yA3Ea80HnPh88ol08orcSJ36w.png?1747186223)

- 파일을 업로드하세요.

- **Next(다음)**를 클릭하세요.

![파일 업로드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046582334/original/XPHcPeBRLndpPjjzXVsjVhpYlJ-h34zovw.png?1747186283)

- 파일 컬럼을 기회 필드에 매핑하세요.

- **Next(다음)**를 클릭하세요.

![필드 매핑](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046582343/original/M1E8UukqUVBCGW6oEBKRK5Pdqo_2pQZLRQ.png?1747186348)

- 모든 정보를 확인하고 **Start Bulk Import(대량 가져오기 시작)**를 클릭하세요.

![가져오기 시작](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046582368/original/c6Ym2ups1E_UPGWwvQ54c8q_cCgepErDug.png?1747186449)

더 자세한 내용을 원하시면 [CSV를 통한 연락처 및 기회 가져오기 가이드]([importing-contacts-and-opportunities-via-csv](importing-contacts-and-opportunities-via-csv.md))를 확인해보세요.

**다음으로**, 파이프라인을 세밀하게 조정하기 위해 수동으로 기회를 추가하고 싶을 수 있습니다.

## 수동으로 기회 생성

때때로 기회는 하나씩 들어오며, 수동으로 추가하면 정확한 제어가 가능합니다. 게다가 빠르고 직관적입니다! 방법은 다음과 같습니다:

- **Opportunities(기회 관리)** 메뉴로 이동하세요.

- **Add opportunity(기회 추가)**를 클릭하세요.

- 기존 연락처를 선택하거나 새 연락처의 세부 정보를 입력하세요.

- 기회 세부 정보(파이프라인 이름, 단계, 금액 등)를 입력하세요.

- **Create(생성)**을 클릭하세요.

![기회 수동 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046582407/original/RcxPH_183Eo8JrU6o629m-oZD1Ug71KD8A.png?1747186618)

자세한 단계별 안내를 원하시면 [기회 생성 단계별 가이드]([step-by-step-guide-to-creating-opportunities](step-by-step-guide-to-creating-opportunities.md))를 참조하세요.

## 자동화로 기회 생성

기회 생성을 자동화하면 추가 클릭 없이도 파이프라인을 깔끔하게 유지할 수 있습니다. 아래는 누군가 폼을 제출할 때마다 실행되는 빠른 시작 예제이며, 다른 트리거로 교체할 수 있는 방법도 설명합니다.

- **워크플로우 열기**  
좌측 메뉴에서 Automation(자동화) → Workflows(워크플로우)로 이동하세요.

![워크플로우 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053481776/original/l1zU-0UMV4Uq7-LQOGr8pWzAQ3pNAdOWZw.png?1757510602)

- **새 워크플로우 시작**  
+ Create Workflow(워크플로우 생성) → Start from Scratch(처음부터 시작)을 클릭하세요(또는 사용하고 싶은 템플릿을 선택하세요).

- **트리거 추가 - 폼 제출**  
+ Add New Trigger(새 트리거 추가)를 클릭하고 Form Submitted(폼 제출됨)를 선택하세요.

- 리드 정보를 수집하는 특정 폼을 선택하세요.

- Save Trigger(트리거 저장)을 누르세요.

- **액션 추가 - 기회 생성/업데이트**  
트리거 아래의 +(액션 추가 버튼)를 클릭하고, Opportunity(기회) → Create/Update Opportunity(기회 생성/업데이트)를 선택하세요.

- 새 폼 리드가 나타날 파이프라인과 단계를 선택하세요.

- 상태를 Open(열림)(또는 다른 상태)으로 설정하고, 알려진 경우 금액을 입력하고, 나중에 재입력 시 레코드가 업데이트되도록 하려면 Allow Opportunity to Move(기회 이동 허용)를 토글하세요.

- Save Action(액션 저장)을 클릭하세요.

![워크플로우 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053482607/original/qohmxMv7OL6XTZvbXuPyKD1GpV389ALclA.gif?1757511004)

- **저장 및 발행**  
**Draft → Publish(초안 → 발행)** 버튼(우측 상단)을 토글하세요. 이제 모든 향후 폼 제출이 선택한 파이프라인 단계로 바로 들어갑니다. 완전 자동입니다.

**기타 일반적인 트리거 및 빠른 사용 사례**

| 트리거 | 한 줄 사용 사례 |
|--------|----------------|
| Customer Booked Appointment(고객 예약 완료) | 전화를 예약했을 때 잠재 고객을 "상담 예약됨"에 자동 추가 |
| Contact Tag(연락처 태그) | "핫 리드" 태그가 적용되는 순간 연락처를 성사 단계로 이동 |
| Facebook Lead Form Submitted(페이스북 리드 폼 제출) | FB 광고의 리드를 "새 페이스북 리드" 단계로 보내서 빠른 후속 조치 |
| Payment Received(결제 수신) | 구매가 완료되면 "성사 - 온보딩 대기" 단계에 기회 생성 |
| **Contact Created(연락처 생성)** | 모든 새 연락처를 "신규 리드"에서 시작하여 누락 방지 |
| Call Status(통화 상태) | 응답한 인바운드 통화를 담당자가 관리할 후속 조치 단계로 이동 |
| **Survey Submitted(설문 제출)** | 완료된 설문을 점수화를 위한 자격 심사 단계로 보내기 |

**참고**: Create/Update Opportunity(기회 생성/업데이트) 액션은 모든 워크플로우에서 동일하게 유지되며, 트리거만 변경됩니다. 리드가 파이프라인에 나타나야 하는 방식과 시기에 가장 적합한 트리거를 선택하세요.
