# **모바일 견적서 결제란?**

모바일 견적서 결제는 하위 계정 사용자가 견적서에 직접 결제 옵션을 삽입할 수 있는 기능입니다. 고객이 견적서를 승인하면 자동으로 인보이스로 리다이렉트되어 온라인에서 결제를 완료할 수 있습니다. 이 기능은 Hyperclass, LeadConnector, 화이트라벨 앱(버전 3.91 이상)에서 사용할 수 있습니다.

# **모바일 견적서 결제의 주요 장점**

견적서에 결제 기능을 추가하면 고객 경험이 단순해지고 매출 회수 속도가 빨라집니다.

- 견적서 승인 후 인보이스로 즉시 리다이렉트
- 모바일 결제와 원활한 연동
- 청구 관련 소통 횟수 감소
- 앱 내 결제 옵션으로 전문성 향상
- 오프라인/수동 결제 대안 지원
- 다양한 브랜드 모바일 앱에서 지원

# **견적서에서 직접 결제를 활성화하는 방법**

모바일 앱에서 직접 결제 가능한 견적서를 활성화하고 발송하는 방법을 알아보세요.

- **Hyperclass, LeadConnector, 또는 화이트라벨** 모바일 앱을 선택한 하위 계정으로 열어주세요.
- **네비게이션 드로어**를 탭하고 **Estimates(견적서)**를 선택합니다.
- 새 견적서를 만들거나 기존 초안을 열어주세요.
- 다음 필수 정보를 모두 입력합니다:
  - 견적서 이름과 번호
  - 발행일과 만료일
  - 연락처
  - 상품/서비스 항목별 세부사항
- **Review(검토)** 페이지로 이동합니다.
- 아래로 스크롤하여 **Payment Settings(결제 설정)** 섹션을 찾습니다.
- **"Enable Direct Payments(직접 결제 활성화)"**를 토글로 켭니다.
- **SMS, Email(이메일), 또는 둘 다**를 통해 견적서를 발송합니다.

# **견적서 승인 후 인보이스로 리다이렉트**

고객이 자신의 기기에서 견적서를 승인하면, 자동으로 해당 인보이스 페이지로 리다이렉트되어 원활하게 결제를 완료할 수 있습니다.

# **결제 실패 시 대처 방법**

고객이 결제 과정에서 문제를 겪는 경우, 인보이스 섹션에서 수동으로 조치를 취할 수 있습니다.

- 모바일 앱에서 **Invoices(인보이스)** 탭으로 이동합니다.
- 견적서와 연결된 인보이스를 찾습니다.
- 탭하거나 왼쪽으로 스와이프하여 **Record Payment(결제 기록)** 옵션을 표시합니다.
- 사용 가능한 결제 방법 중에서 선택합니다:
  - Tap to Pay(탭 결제)
  - Cash(현금)
  - Card(카드)
  - Manual entry(수동 입력)

# **자주 묻는 질문**

**Q: 이 기능을 지원하는 앱 버전은 무엇인가요?**

이 기능은 **버전 3.91 이상**의 Hyperclass, LeadConnector, 화이트라벨 모바일 앱에서 사용할 수 있습니다.

**Q: 견적서를 SMS와 이메일로 모두 보낼 수 있나요?**

네, 발송 과정에서 SMS, 이메일, 또는 둘 다를 선택해서 견적서를 보낼 수 있습니다.

**Q: 고객이 결제를 완료하지 않으면 어떻게 되나요?**

인보이스에 접근해서 다시 발송하거나 다양한 방법으로 수동으로 결제를 기록할 수 있습니다.

**Q: 이 기능이 데스크톱에서도 사용 가능한가요?**

현재 견적서 내 직접 결제 기능은 모바일 앱에서만 지원됩니다.

### **다음 단계**

- 이 기능을 사용하려면 모바일 앱을 **버전 3.91 이상**으로 업데이트하세요.
- 원활한 결제 관리를 위해 **Tap to Pay(탭 결제)**와 **수동 기록** 옵션을 살펴보세요.
- 더 빠른 처리를 위해 고객들이 자신의 기기에서 직접 견적서를 승인하고 결제하도록 안내해주세요.
