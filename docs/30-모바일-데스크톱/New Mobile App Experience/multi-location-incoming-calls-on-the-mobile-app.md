# 모바일 앱에서 멀티 로케이션 수신 전화

Hyperclass 모바일 앱의 **멀티 로케이션 수신 전화** 기능은 여러 하위 계정에서 작업하는 팀이 중요한 리드를 놓치지 않도록 도와줍니다. 기존에는 현재 로그인한 하위 계정에서만 전화를 받을 수 있어 기회 손실, 응답 지연, 불필요한 계정 전환이 발생했습니다.

이 새로운 기능을 통해 현재 활성화된 하위 계정과 상관없이 **최대 10개 로케이션에서 오는 전화를 모바일 앱에서 직접 받을 수 있습니다**. 각 수신 전화는 **발신 로케이션**을 명확히 표시하며, **통화 중에도 해당 로케이션으로 전환**하여 메모 작성, 연락처 접근, CRM 데이터 업데이트를 즉시 할 수 있습니다.

# 멀티 로케이션 수신 전화란?

> 💡 **중요 정보**
> - 현재 모바일 앱에서만 사용 가능합니다
> - 전화를 받으려면 해당 로케이션의 멤버여야 합니다
> - 최대 10개 로케이션까지 가능합니다

**멀티 로케이션 수신 전화**는 사용자가 여러 하위 계정(로케이션)에서 동시에 전화를 받도록 **선택 설정**할 수 있는 모바일 기능입니다. 활성화하면 현재 로그인한 계정과 상관없이 **선택한 모든 로케이션으로 들어오는 전화**에서 벨이 울립니다.

각 수신 전화 화면에는 어느 하위 계정에서 전화가 왔는지 알려주는 **로케이션 배지**가 표시됩니다. 통화 중에 **해당 로케이션으로 바로 전환**하여 메모 작성, 파이프라인 업데이트, 통화 로그 기록 등의 CRM 작업을 수행할 수 있습니다.

---

## 멀티 로케이션 수신 전화의 주요 장점

멀티 로케이션 수신 전화는 복잡한 조직에서 리드 응답을 간소화합니다.

- **리드 누락 제로** — 다른 하위 계정에 활성화되어 있어도 최대 10개 로케이션에서 전화를 받을 수 있습니다.

- **즉시 맥락 파악** — 통화 화면의 로케이션 배지를 통해 리드가 CRM 어디에 속하는지 정확히 알 수 있습니다.

- **원터치 작업 공간 전환** — 통화 중에 올바른 로케이션으로 바로 이동하여 연락처, 메모, 거래를 확인할 수 있습니다.

- **사용자별 제어** — 각 팀원이 자신의 디바이스에 울릴 로케이션을 선택하여 알림 과부하를 방지합니다.

- **생산성 향상** — 급하게 앱을 전환할 필요가 없고, 잘못된 로케이션 컨텍스트로 인한 후속 조치 오류를 줄입니다.

---

## 멀티 로케이션 링 설정 방법 (모바일)

멀티 로케이션 수신 전화를 활성화하려면 다음 단계를 따라하세요:

![멀티 로케이션 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155056084134/original/Ys6ATMFKg-8T-iFeyPXlkv39p5U3ZL8J7A.png?1760546949)

1. 모바일 앱을 열고 **Settings(설정) → Calls/Voice(통화/음성)**으로 이동합니다.

2. **Multi-Location Incoming Calls(멀티 로케이션 수신 전화)**를 토글로 켭니다.

3. 전화를 받고 싶은 **최대 10개 로케이션**을 선택합니다. 해당 로케이션의 사용자여야 합니다.

4. 활성화하면 다른 하위 계정에 활성화되어 있어도 선택한 모든 로케이션의 수신 전화에서 벨이 울립니다.

5. 통화 화면에서:
   - **로케이션 배지**를 확인하여 전화가 어디서 왔는지 파악합니다
   - 통화 중 **Switch Location(로케이션 전환)**을 눌러 해당 하위 계정으로 바로 이동합니다
   - 통화 후 올바른 CRM 작업 공간에서 후속 조치와 메모를 완료합니다

![로케이션 전환 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155056084204/original/kVMQVsKOqOjmHcFEIvx7P1mjwxcFVDZxNg.png?1760547028)

## 자주 묻는 질문

**Q: 10개보다 많은 로케이션에서 전화를 받을 수 있나요?**

A: 아니요. 성능 문제를 방지하고 수신 전화의 명확성을 유지하기 위해 최대 10개 로케이션까지 선택할 수 있습니다.

**Q: 이 기능을 활성화하려면 관리자 권한이 필요한가요?**

A: 아니요. 선택한 로케이션의 멤버라면 누구나 자신의 계정에 멀티 로케이션 수신 전화를 설정할 수 있습니다.

**Q: 전화가 어느 로케이션에 속하는지 볼 수 있나요?**

A: 네. 수신 전화 화면에 발신 로케이션 이름을 표시하는 **로케이션 배지**가 포함됩니다.

**Q: 통화 중에 통화 메모를 작성할 수 있나요?**

A: 네! 통화 중에 **Switch Location(로케이션 전환)**을 눌러 올바른 하위 계정으로 이동하여 CRM 도구에 접근할 수 있습니다.

**Q: 데스크톱에서도 사용할 수 있나요?**

A: 현재 멀티 로케이션 수신 전화는 **모바일 앱에서만** 사용 가능합니다.

**Q: 이미 다른 통화 중이면 어떻게 되나요?**

A: 수신 전화는 현재 통화 처리 규칙(예: 통화 대기 또는 음성사서함)을 따릅니다.

**Q: 데스크톱 버전도 있나요?**

A: 멀티 로케이션 링은 현재 모바일 전용입니다. 데스크톱은 활성 로케이션에서만 전화를 받습니다.

**Q: 워크플로우는 여전히 올바른 로케이션에서 실행되나요?**

A: 네. 통화 중에 올바른 하위 계정으로 전환하므로 모든 자동화와 통화 로그가 해당 로케이션에 연결됩니다.

**Q: 이 기능이 추가 LC Phone 분을 소모하나요?**

A: 아니요. 일반적인 수신 전화와 동일한 방식으로 과금됩니다.
