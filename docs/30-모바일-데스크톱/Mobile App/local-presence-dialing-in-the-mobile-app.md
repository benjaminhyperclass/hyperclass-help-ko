# 모바일 앱에서 현지 번호 발신 기능 사용하기

더 많은 리드에게 도달하고 전화 응답률을 높이세요. 모든 발신 전화가 현지 번호처럼 보이게 만들어 드립니다. Hyperclass 모바일 앱의 현지 번호 발신(Local Presence Dialing) 기능은 연락처의 지역번호와 일치하거나 가장 가까운 전화번호를 자동으로 선택합니다. 심지어 국제 전화도 지원하므로 잠재 고객이 전화를 받을 가능성이 훨씬 높아집니다.

---

## **모바일 현지 번호 발신 기능이란?**

현지 번호 발신(Local Presence Dialing)은 Hyperclass 모바일 앱에서 전화를 걸 때마다 최적의 발신자 번호를 자동으로 선택하는 기능입니다. 워크스페이스에서 구매하고 인증한 번호들 중에서 연락처의 지역번호(또는 가장 가까운 도시)와 일치하는 번호를 찾아서 상대방 휴대폰에 친숙한 현지 번호로 표시합니다. 이러한 현지 정체성은 즉각적인 신뢰를 구축하고 연결률을 눈에 띄게 향상시킵니다.

---

## **모바일 현지 번호 발신의 주요 장점**

현지 번호 발신 기능은 모든 통화를 소중하게 만드는 여러 사용자 중심 이점을 제공합니다.

- **높은 응답률** — 연락처가 친숙한 현지 번호에 응답할 가능성이 3-4배 더 높습니다.

- **도시 수준의 정확도** — 모바일은 가장 가까운 도시를 먼저 매칭하고, 그 다음 주 단위로 매칭한 후 대체 번호로 넘어갑니다.

- **국제 지원** — 미국이나 캐나다 외 지역으로 전화를 걸 때 자동으로 해당 국가의 번호를 사용합니다.

- **스마트 대체 로직** — 완벽한 매칭이 없어도 앱이 사용자 배정 번호, 하위 계정 기본 번호 순으로 대체하여 모든 전화가 발신되도록 보장합니다.

- **매끄러운 경험** — 웹앱의 기본 설정이 모바일로 자동 전환되므로 별도 설정이 불필요합니다.

---

## **국제 지원**

현지 번호 발신은 북미에만 제한되지 않습니다. 해외 연락처에 전화를 걸 때 모바일 앱이 먼저 해당 국가에서 보유한 번호를 찾습니다. 사용 가능한 번호가 있으면 자동으로 사용되어 국경을 넘어서도 신뢰를 구축하는 현지 발신자 번호를 제공합니다.

---

## **더 똑똑한 대체 로직**

이 향상된 기능은 완벽한 현지 번호를 사용할 수 없어도 전화가 항상 연결되도록 보장합니다.

- 정확한 지역번호 매칭을 먼저 검색
- 매칭이 없으면 사용자에게 배정된 번호 사용
- 여전히 사용 불가능하면 하위 계정의 기본 번호로 기본값 설정

이러한 자동화된 단계들이 추측을 제거하고 일관된 아웃리치를 유지합니다.

---

## **도시 수준 정확도 (모바일 전용)**

웹과 달리 모바일 앱은 주 단위를 넘어서 가장 가까운 도시 번호를 먼저 찾습니다. 예를 들어, 샌프란시스코의 415 지역번호 리드에게 전화를 걸 때 415 번호를 사용하려고 시도하고, 보유하지 않으면 다른 캘리포니아 번호를 찾은 후 대체 번호로 넘어갑니다. 결과적으로 지리적으로 가장 관련성 높은 발신자 번호를 제공합니다.

---

## **향상된 번호 선택기**

새롭게 디자인된 번호 선택기(Number Picker)는 일회성 재정의를 빠르고 명확하며 임시적으로 만들어줍니다.

전화 화면에서 접근할 수 있는 선택기는 발신 가능한 모든 번호를 나열합니다. 배정 태그(사용자, 워크스페이스 등)와 번호 유형(Local, Mobile, Toll-Free)까지 포함됩니다.

- 두 번의 탭으로 단일 통화에 대해 다른 번호를 선택합니다.
- 통화가 끝나면 시스템이 자동으로 기본 현지 번호 발신 설정으로 되돌아갑니다.

## **현지 번호 발신 설정 방법**

적절한 설정은 웹앱에서 한 번만 하면 모든 모바일 사용자에게 즉시 동기화됩니다.

**1단계:** 웹에서 원하는 로케이션을 열고, `Settings(설정) → Phone Numbers(전화번호) → Voice(음성) → Other Settings(기타 설정) → Outbound Call(발신 통화) → Default Phone Number for Outbound Calls(발신 통화 기본 전화번호)`로 이동합니다.

Local Presence Dialing(현지 번호 발신)을 선택합니다.

![현지 번호 발신 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155056182735/original/UrQvGUBQiQ-1fI3kYgcGX4riyI7di1zokQ.png?1760639052)

**2단계:** Hyperclass 모바일 앱을 v4.0.0 이상으로 업데이트합니다.

**3단계:** 앱을 열고 전화를 걸기 시작하세요. 현지 번호 발신이 자동으로 활성화됩니다.

## **자주 묻는 질문**

**Q: 현지 번호 발신에 추가 비용이 있나요?**

A: 아니요. 이 기능은 기존 전화 시스템 요금제에 포함되어 있습니다. 유일한 비용은 필요시 추가 번호를 구매하는 것뿐입니다.

**Q: 자체 Twilio 계정을 사용해도 작동하나요?**

A: 네. 현지 번호 발신은 연결된 Twilio 하위 계정의 번호에서 동일한 방식으로 선택합니다.

**Q: 연락처의 지역번호가 아직 지원되지 않으면 어떻게 되나요?**

A: 앱이 대체 로직(배정된 번호 → 하위 계정 기본값)을 따릅니다. 지원팀을 통해 새로운 지역번호 지원을 요청할 수 있습니다.

**Q: 단일 사용자에 대해 현지 번호 발신을 비활성화할 수 있나요?**

A: 토글은 하위 계정 수준에 있습니다. 하지만 모든 사용자가 번호 선택기를 통해 한 통화에 대해 일시적으로 발신자 번호를 재정의할 수 있습니다.

**Q: 현지 번호 발신이 SIM 기반 통화와 작동하나요?**

A: 아니요. SIM 기반 통화는 휴대폰의 통신사 번호를 사용합니다. 현지 번호 발신은 Hyperclass 다이얼러를 통해 걸리는 VoIP 통화에 적용됩니다.

**Q: 발신자 평판에 영향을 주나요?**

A: 여러 번호를 사용하면 통화량이 분산되어 더 건강한 평판을 유지하는 데 도움이 되지만, 스팸 태깅에 대해 각 번호를 모니터링해야 합니다.

**Q: 현지 번호 발신 선택에 무료 전화번호가 포함되나요?**

A: 무료 전화번호는 번호 선택기에 나타나지만 연락처의 국가 코드와 일치하지 않는 한 현지 번호 발신 로직에 의해 자동으로 선택되지는 않습니다.
