## **병합 필드와 커스텀 변수란?**

병합 필드와 커스텀 변수는 개인화된 정보를 동적으로 삽입할 수 있는 플레이스홀더입니다. 병합 필드는 {{something.here}}와 같은 형태로 표시됩니다. 이 필드들은 CRM이나 다른 모듈에서 데이터를 가져와 병합 필드를 대체함으로써 커뮤니케이션을 자동화하고 개인화하는 데 도움을 줍니다.

연락처 기반 워크플로우를 구축할 때는 연락처 필드와 함께 변수 선택기에서 회사 필드도 선택할 수 있습니다. 연락처가 회사 레코드와 연결되어 있다면, 워크플로우 단계와 조건에서 회사 정보(예: 회사명, 전화번호, 이메일, 웹사이트, 주소)를 가져올 수 있습니다. 연락처가 회사와 연결되어 있지 않다면 회사 값은 빈 값으로 표시됩니다.

병합 필드와 커스텀 변수의 유형은 다음과 같습니다:

- **표준 필드**: 연락처 이름, 예약 시간, 사용자 정보 등 기본 제공되는 수정 불가능한 필드
- **커스텀 필드**: 연락처와 기회 관리를 위해 사용자가 정의하는 필드
- **커스텀 값**: 템플릿과 워크플로우에서 재사용할 수 있는 값
- **커스텀 객체**: 고급 데이터 관리를 위한 사용자 생성 데이터 구조
- **트리거 링크**: 클릭 시 워크플로우와 캠페인에서 자동화된 작업을 실행하는 특수 링크
- **결제 링크**: 고객이 상품이나 서비스에 대해 직접 결제할 수 있는 링크

## **병합 필드와 커스텀 변수 사용의 핵심 이점**

- **커뮤니케이션 개인화**: 이메일과 SMS 메시지에 연락처 이름, 날짜 등의 세부 정보를 자동으로 삽입
- **프로세스 자동화**: 워크플로우에서 병합 필드를 사용하여 시간을 절약하고 수동 작업 줄이기
- **데이터 일관성 향상**: 모든 커뮤니케이션에서 정확하고 최신 정보 보장
- **반복 작업 간소화**: 효율성을 위해 커스텀 값과 트리거 링크 재사용

## 빈 필드에 대한 기본값 또는 대체값

현재는 변수가 비어있을 때 채워넣을 대체값이나 기본값을 지원하지 않습니다. 병합 필드는 그냥 공백으로 대체됩니다.

대안으로 콘텐츠 블록을 표시하거나 숨기는 방법을 사용할 수 있습니다.

예를 들어, 이메일 빌더에서는 조건부 전송(Conditional Send)을 사용하여 특정 조건이 충족될 때만 콘텐츠 블록을 표시할 수 있습니다. first_name 병합 필드가 포함된 블록(안녕하세요 {{contact.first_name}}님,)을 first_name이 비어있지 않을 때 보내도록 설정하고, 병합 필드가 없는 두 번째 블록(기본값인 "안녕하세요"와 같은)을 first_name 필드가 비어있을 때 보내도록 설정하면 비슷한 효과를 얻을 수 있습니다.

## 표준 필드

표준 필드는 모든 계정에서 기본적으로 사용할 수 있는 미리 정의된 병합 필드입니다. 이 필드들은 연락처 이름, 이메일 주소, 전화번호, 예약 정보(예: 시간과 날짜) 등의 필수 데이터 포인트를 다룹니다. 커스텀 필드와 달리 이 필드들은 내장되어 있으며 수정하거나 삭제할 수 없습니다. 표준 필드의 사용 사례는 다음과 같습니다:

- **연락처 이름**: 연락처의 이름
- **연락처 이메일**: 연락처의 이메일 주소
- **예약 날짜와 시간**: 예약의 예정 날짜와 시간
- **사용자 전체 이름**: 배정된 사용자의 전체 이름
- **회사 이름**: 귀하의 사업체나 조직의 이름

## 커스텀 필드

각 연락처나 기회에 고유한 사용자 정의 데이터 포인트로, 생일이나 선호도와 같은 개인화된 정보를 저장하여 세밀한 세분화와 커뮤니케이션에 사용됩니다. 커스텀 필드의 사용 사례는 다음과 같습니다:

- **고객 선호도**: 선호하는 연락 방법(이메일, SMS, 전화) 등 고객의 특정 관심사 추적
- **멤버십 세부사항**: 멤버십 레벨(예: 골드, 실버, 플래티넘) 저장하여 등급별 맞춤 프로모션 제공
- **상품 관심도**: 고객이 관심 있는 상품이나 서비스를 추적하는 필드 추가하여 타겟 후속 조치 진행
- **이벤트 참석**: 이벤트 등록이나 참석 세부사항을 기록하여 초대장이나 후속 메시지 개인화
- **담당자 배정**: 특정 연락처를 담당하는 계정 매니저 이름 저장
- **계약 갱신 날짜**: 계약 만료나 갱신 날짜를 추적하여 적시에 알림 발송
- **리드 소스**: 리드 유입 경로(예: Facebook, Google 광고, 추천) 기록하여 마케팅 성과 추적

## 커스텀 값

정적 정보(예: 회사 세부사항)를 위한 재사용 가능한 플레이스홀더로, 이메일 템플릿, 웹사이트, 퍼널, 워크플로우 등 CRM의 여러 영역에서 일관성을 보장하고 업데이트를 간소화합니다! 커스텀 값의 사용 사례는 다음과 같습니다:

- **회사 연락처 정보**: 이메일 하단에 회사 전화번호, 이메일, 주소를 삽입하는 커스텀 값 사용
- **표준 마감 인사**: "감사합니다, [회사명]"과 같은 표준 서명 문구를 위한 커스텀 값 추가
- **웹사이트 링크**: 웹사이트나 고객 지원 페이지 등 자주 사용하는 URL을 위한 커스텀 값 사용
- **규정 준수 명세**: 모든 템플릿에서 법적 면책 조항이나 규정 준수 명세를 일관되게 삽입
- **브랜드 태그라인**: 마케팅 자료와 캠페인에 회사 슬로건이나 태그라인 추가
- **휴일 인사**: 뉴스레터나 프로모션 메시지에 삽입할 재사용 가능한 계절별 인사말 생성

## 커스텀 객체

커스텀 객체를 사용하면 표준 필드를 넘어서는 고유한 데이터 구조를 생성할 수 있습니다. 커스텀 객체의 사용 사례는 다음과 같습니다:

- **구독 요금제**: 요금제 유형, 갱신 날짜, 청구 주기 등의 구독 세부사항 추적
- **재고 관리**: SKU 번호, 수량, 공급업체 세부사항을 포함한 상품 재고를 관리하는 객체 생성
- **부동산 매물**: 부동산 업체의 경우, 위치, 가격, 특징을 포함한 부동산 매물 세부사항 저장
- **공급업체 세부사항**: 계약, 연락 담당자, 결제 조건을 포함한 공급업체 정보 관리
- **예약 세부사항**: 서비스 유형, 위치, 배정된 직원과 같은 고급 예약 세부사항 기록

## 트리거 링크

트리거 링크는 클릭 시 워크플로우와 캠페인에서 특정 작업을 실행할 수 있는 특수 링크입니다. 트리거 링크의 사용 사례는 다음과 같습니다:

- **웨비나 등록**: 웨비나 등록 링크를 발송하고 클릭 시 확인 이메일을 자동으로 실행
- **설문 후속 조치**: 설문 링크를 클릭한 사람을 추적하고 자동으로 감사 메시지 발송
- **콘텐츠 다운로드**: eBook이나 백서와 같은 리소스 다운로드 링크를 제공하고 참여도 추적
- **업셀 기회**: 업그레이드 서비스 링크를 제공하고 클릭한 사람에게 후속 이메일 자동 실행
- **이벤트 초대**: 초대 링크를 발송하여 이벤트 RSVP를 추적하고 클릭을 기반으로 후속 조치 자동화
- **특별 제안**: 한정 시간 제안 링크를 발송하고 링크를 클릭하지 않은 경우 알림 이메일 실행

## 결제 링크

결제 링크는 고객이 결제할 수 있는 링크를 생성하여 상품이나 서비스를 판매할 수 있게 해줍니다. 결제 링크의 사용 사례는 다음과 같습니다:

- **이벤트 티켓**: 워크샵이나 컨퍼런스 등 이벤트 티켓 결제 수집
- **구독 요금제**: 반복 구독이나 멤버십을 위한 링크 생성
- **상품 판매**: 물리적 또는 디지털 상품(예: 책, 강의) 결제 링크 생성
- **보증금 수집**: 예약이나 예약금 보증금 수집
- **기부**: 자선 기부나 모금 캠페인을 위한 결제 링크 생성

## 자주 묻는 질문

**Q: 커스텀 필드와 커스텀 값의 차이점은 무엇인가요?**
커스텀 필드는 개별 연락처나 기회에만 적용되는 필드로, 생일이나 선호도와 같은 고유한 데이터를 저장합니다. 커스텀 값은 회사 세부사항이나 표준 메시지와 같이 커뮤니케이션 전반에서 일관된 정보를 위한 재사용 가능한 플레이스홀더입니다.

**Q: 병합 필드가 작동하지 않거나 올바르게 표시되지 않는 이유는 무엇인가요?**
병합 필드 형식이 누락되었거나 올바르지 않기 때문일 수 있습니다. 올바른 문법(예: {{contact.email}})을 사용하고 있는지 확인하세요. 또한 해당 필드의 데이터가 연락처나 사용자 레코드에 존재하는지 확인하세요.

**Q: 병합 필드는 대소문자를 구분하나요?**
네, 병합 필드는 대소문자를 구분합니다. 예를 들어 {{contact.name}}은 작동하지만 {{Contact.Name}}은 작동하지 않습니다.

**Q: 병합 필드가 올바르게 작동하는지 어떻게 테스트할 수 있나요?**
병합 필드를 테스트하려면 자신에게 테스트 이메일이나 SMS를 보내거나 템플릿 편집기에서 커뮤니케이션을 미리 봅니다. 이렇게 하면 병합 필드가 올바른 데이터를 가져오는지 확인할 수 있습니다.

**Q: 병합 필드에 사용할 수 있는 데이터가 없으면 어떻게 되나요?**
병합 필드에 해당하는 데이터가 없으면 공백으로 표시될 수 있습니다. 최상의 결과를 위해 연락처나 사용자 데이터가 완전한지 항상 확인하세요.

**Q: 커스텀 값을 업데이트하면 모든 템플릿에 반영되나요?**
네, 설정에서 커스텀 값을 업데이트하면 해당 값을 사용하는 모든 템플릿과 워크플로우가 자동으로 업데이트되어 일관성이 보장됩니다.

**Q: 표준 필드와 커스텀 필드는 어떻게 다른가요?**
표준 필드는 CRM에 미리 정의되고 내장된 필드이며, 커스텀 필드는 사용자가 정의하여 연락처와 기회에 대한 고유한 데이터를 저장하기 위해 생성할 수 있습니다.

**Q: 커스텀 객체는 커스텀 필드 및 커스텀 값과 어떻게 다른가요?**
커스텀 객체는 커스텀 필드와 값의 기능을 넘어서는 더 복잡한 관계와 데이터 관리를 가능하게 하는 고급 데이터 구조입니다.

**Q: 생성할 수 있는 커스텀 필드나 커스텀 값의 개수에 제한이 있나요?**
생성할 수 있는 양에 엄격한 제한은 없지만, 계정 내에서 명확성과 조직성을 유지하기 위해 효과적으로 관리하는 것이 좋습니다.

**Q: 웹사이트와 퍼널에서 커스텀 값을 사용할 수 있나요?**
네, 커스텀 값을 활용하여 웹사이트와 퍼널에서 일관된 정보를 동적으로 표시할 수 있어 개인화를 향상시킬 수 있습니다.

**Q: 더 나은 관리를 위해 커스텀 값을 어떻게 정리할 수 있나요?**
커스텀 값을 폴더로 정리할 수 있어, 목록이 늘어남에 따라 더 쉽게 관리하고 찾을 수 있습니다.

**Q: 커스텀 필드/값을 대량으로 생성하거나 업로드할 수 있나요?**
현재는 커스텀 필드/값의 대량 생성이나 업로드를 지원하지 않으므로 개별적으로 추가해야 합니다.

**Q: 표준 필드를 편집하거나 삭제할 수 있나요?**
아니요, 표준 필드는 내장된 필드로 수정하거나 삭제할 수 없지만, 템플릿과 워크플로우에서 사용할지 여부는 선택할 수 있습니다.
