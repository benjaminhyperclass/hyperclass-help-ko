# 🚀 시작하기 — 토스페이먼츠로 한국 원화 결제받기

> 하이퍼클래스에서 한국 고객으로부터 **원화(KRW) 결제**를 받기 위한 단계별 안내입니다.
> 이 가이드 한 페이지만 따라하시면 신용카드 · 정기결제 · PayPal · 가상계좌까지 모두 사용할 수 있습니다.

---

## 1. 이 가이드는 누구를 위한 것인가요?

다음 중 하나라도 해당되면 이 가이드를 따라 진행하실 수 있습니다.

- **하이퍼클래스 에이전시 또는 서브계정**을 운영 중이며, 한국 원화로 결제를 받고 싶은 경우
- 강의 · 멤버십 · 컨설팅 · SaaS 상품 등을 **정기결제(구독)** 로 판매하려는 경우
- 카카오페이 · 네이버페이 · 토스 · 페이코 등 한국 **간편결제 전체**를 지원하고 싶은 경우
- PayPal 등으로 **해외 고객**의 외화 결제도 받으려는 경우

---

## 2. 무엇이 가능해지나요?

설정이 완료되면 하이퍼클래스 체크아웃 · 퍼널 · 인보이스 · 결제 링크 어디서든 다음이 가능합니다.

| 결제 유형 | 지원 결제수단 |
|---|---|
| **일반 결제(일회성)** | 신용/체크카드, 계좌이체, 가상계좌, 카카오페이, 네이버페이, 토스, 페이코, 삼성페이, 애플페이 등 |
| **정기결제(구독)** | 빌링키 발급 후 자동 과금 — 월간 · 연간 · 분할 결제, 무료체험(Trial) 지원 |
| **해외 결제** | PayPal, AliPay 등 USD/EUR 등 외화 결제 |

하이퍼클래스의 모든 기존 기능 — 쿠폰, 세금, SaaS 모드, 워크플로우 자동화 — 와 그대로 연동됩니다.

## 3. 전체 흐름 한눈에 보기

이 가이드는 **"먼저 Test Key로 전체 기술 셋업을 검증한 뒤, 등기 서류를 발송하고 심사 대기에 들어가는"** 순서로 설계되어 있습니다. 1개월 이상의 심사 대기 시간이 시작되기 전에 모든 하이퍼클래스 연동이 정상 작동하는지 확인할 수 있어 안전합니다.

```
[Step 1]   사업 정보 정리                  ─ 1~2일
   ▼
[Step 2]   토스페이먼츠 일반결제 MID 신청    ─ 30분 (온라인 신청)
   ▼
[Step 3]   Test Key 발급 받기              ─ 신청 후 자동 발급 (수일 내)
   ▼
─── 여기서부터는 Test Key로 자유롭게 검증 가능 ───
   ▼
[Step 4]   HyperClass 앱 설치 + 키 연동    ─ 30분
   ▼
[Step 5]   상품 만들기 (Products)          ─ 1~2시간
   ▼
[Step 6]   결제 테스트 (Payment Links)     ─ 30분
   ▼
[Step 7]   결제경로 6단계 캡처             ─ 30분
   ▼
─── 모든 셋업 검증 완료 후 ───
   ▼
[Step 8]   등기 서류 발송                  ─ 1~2일
   ▼
[Step 9]   심사 대기 (★ 가장 긴 구간)      ─ 3~5주
   ▼
[Step 10]  라이브 키로 전환 + 카드사 본 심사 ─ 1~2주
   ▼
[Step 11]  (선택) 빌링/PayPal MID 추가     ─ 라이브 후
```

### ⏱ 예상 소요 기간

| 구간 | 기간 |
|---|---|
| 정보 입력 · MID 신청 · Test Key 발급 | 2~3일 |
| Test Key 연동 · 상품 만들기 · 결제 테스트 · 캡처 | 2~3일 |
| 등기 서류 발송 | 1~2일 |
| 🚨 **토스페이먼츠 심사 대기** | **3~5주** (2024~25년 현재 심사 적체) |
| 라이브 키 전환 + 카드사 본 심사 | 1~2주 |
| **총 소요 기간** | **약 1~2개월** |

> 💡 **지금 가장 큰 변수는 토스페이먼츠 심사 대기 시간입니다.** 2024~25년 현재 신청이 많이 밀려 있어 PG 연동까지 평균 한 달 이상 걸리고 있습니다. 결제 오픈 일정을 잡으실 때는 **최소 6주 이상**의 여유를 두고 시작하시는 것을 권장합니다.
>
> 단, 이 가이드 순서를 따르시면 심사 대기에 들어가기 전(Step 9 진입 전)에 하이퍼클래스 연동 · 상품 등록 · 결제 흐름까지 모두 검증해 두실 수 있어, **대기 시간을 비즈니스 준비에 활용**하실 수 있습니다.

## Step 1. 사업 정보 정리 및 신청 항목 결정

### 1-1. 사업자 기본 정보 정리

[4-2 체크리스트](#4-2-사업-기본-정보)의 모든 항목을 정확히 준비합니다.

> ⚠️ **상호명 · 주소는 사업자등록증과 완전히 동일**해야 합니다. 띄어쓰기 · 괄호까지 일치시켜 주세요. 불일치 시 심사가 반려됩니다.

### 1-2. 사업자 유형 선택

다음 중 본인의 유형을 결정합니다.

- 법인사업자
- 개인사업자
- 간이과세자
- 면세사업자
- 비영리법인

> 통신판매업 신고번호가 없는 경우 KB국민카드 결제수단이 자동 제외됩니다. 다른 카드사는 정상 심사되며, **간이과세자 · 공공기관**은 예외로 영향 받지 않습니다.

### 1-3. 카드사 심사 답변 5문항 작성

토스페이먼츠 신청 폼에서 다음 5문항에 답변해야 합니다. 미리 작성해 두면 신청이 빨라집니다.

| 항목 | 답변 양식 |
|---|---|
| **결제 가능한 URL** | 결제 가능한 상품 페이지 또는 메인 URL (`https://example.com/...`) |
| **환불정책 URL** | 환불정책이 명시된 페이지 URL |
| **안전한 결제 처리** | 토스페이먼츠 표준 결제창 사용 (1줄로 명시) |
| **최대 서비스 제공 기간** | 예: 1,000,000원 / 12개월 (월 환산: 83,333원 등) |
| **상품/서비스 상세** | 상품명 · 가격 · 결제수단 · 환불 기준을 상세히 |

> 💡 **유료 서비스의 14일 이내 결제 취소 절차 (3가지 조건)**: 별도 환불 페이지에 명시 — ① 사용 시작 전 14일 이내 ② 디지털콘텐츠는 이용 즉시 사용 간주 ③ 환불 처리 영업일 기준 N일 이내 등

### 1-4. 환불 정책 페이지 준비

홈페이지에 다음 항목이 모두 포함된 환불 정책 페이지를 준비합니다.

**A. 적합한 적용 범위**
- 1:1 코칭 · 1:2 코칭 · 1:3 그룹 코칭 (인원수 등 적용범위 명시)
- 그룹 컨설팅 / 강의 / 디지털 콘텐츠 / 환급 챌린지 등 항목별 환불 적용범위

**B. 디지털 콘텐츠 특례 명문화**
- "디지털 콘텐츠는 이용 즉시 사용 간주, 일부 환불 제한 가능" (예: 영상 1편당 5% 차감 등)

**C. 7일 단순 변심 가능 표기** (필수)

**D. 환불 신청 방법**
- 이메일 / support 시그널 / 카카오 1:1 채널 등 명확한 채널 명시
- 환불 처리 기간(영업일 기준)
- 결제수단별 환불 방식

### 1-5. 신청할 상점(MID) 결정

토스페이먼츠는 **결제 종류별로 별도의 상점아이디(MID)** 를 발급합니다.

> 🚨 **현재 신청 정책 (중요)**: 토스페이먼츠에서는 처음에 **일반 결제 MID 1개만** 신청할 수 있습니다. 일반 결제가 라이브로 검증된 이후, **빌링(정기결제) · PayPal MID를 추가 가맹 비용 없이** 추가로 신청할 수 있습니다.

따라서 진행 순서는 다음과 같습니다.

| 순서 | MID 종류 | 시기 |
|---|---|---|
| **1차 신청** | 🅰 **일반 결제 MID** (필수) | 지금 신청 |
| 2차 신청 | 🅱 빌링 MID (정기결제) | 일반 결제 라이브 검증 후 |
| 3차 신청 | 🅲 PayPal MID (해외) | 일반 결제 라이브 검증 후 |

이 가이드에서는 **🅰 일반 결제 MID**를 먼저 진행합니다. Step 11에서 추가 MID 신청 절차를 다룹니다.

### 1-6. 세부 결제수단 선택 (기본 권장)

일반 결제 MID 신청 시 다음 결제수단을 모두 활성화하는 것을 권장합니다.

- 신용 · 체크카드
- 계좌이체 / 가상계좌
- 카카오페이 · 네이버페이 · 토스 · 페이코
- 삼성페이 · 애플페이
- 해외카드 (원화 승인) — 별도 추가 심사 약 2~3주 소요

## Step 2. 토스페이먼츠 일반 결제 MID 신청

[Step 1](#step-1-사업-정보-정리-및-신청-항목-결정)에서 정리한 정보로 토스페이먼츠 가맹점 신청을 진행합니다.

### 2-1. 신청 사이트 접속

1. [https://tosspayments.com](https://tosspayments.com) 접속
2. 우측 상단 **[전자결제 신청하기]** 클릭
3. 로그인 또는 회원가입
4. "홈페이지 구축 상태" 단계에서 → **"직접 구축"** 선택

### 2-2. 일반 결제 패키지 선택

5. 패키지 선택 단계에서 → **"기본결제패키지"** 선택
   *(MID 1개 = 일반 결제 패키지)*

### 2-3. 신청서 작성

6. 신청인 정보 입력 → [Step 1](#step-1-사업-정보-정리-및-신청-항목-결정)에서 정리한 정보 그대로 기재
7. 카드사 심사 답변 5문항 입력 → [1-3](#1-3-카드사-심사-답변-5문항-작성)에서 작성한 답변 그대로

### 2-4. 결제 단계: "나중에 결제" 선택

8. 결제 단계 → **"나중에 결제"** 선택
   *(가입비 · 연회비는 면제 처리됩니다. 절대 선결제하지 마세요.)*

9. 신청 완료 → 영문 상점아이디(MID) 발급 (예: `tcommul9ok`)

> ✅ MID가 발급되면 이메일 알림이 옵니다. 발급된 MID는 다음 Step에서 사용됩니다.

## Step 3. Test Key 발급 받기

MID가 발급되면 토스페이먼츠에서 **Test Key 2개**를 함께 발급해 줍니다. 이 Test Key로 라이브 결제 전 모든 셋업을 안전하게 검증할 수 있습니다.

- **Client Key** (`test_ck_*` 로 시작) — 프론트엔드 식별용
- **Secret Key** (`test_sk_*` 로 시작) — 백엔드 인증용

### Test Key 확인 위치

1. [https://tosspayments.com](https://tosspayments.com) 로그인
2. **내 상점** 클릭
3. 좌측 메뉴 **개발자 센터** → **API 키**
4. 발급된 Test Key 2개 복사

> 💡 라이브 키(`live_ck_*` / `live_sk_*`)는 [Step 10 라이브 키 전환](#step-10-라이브-키-전환-및-카드사-본-심사) 후에 발급됩니다. 우선 Test Key로 모든 연동을 검증한 뒤 라이브 키로 전환하는 흐름입니다.

## Step 4. HyperClass 앱 설치 + Test Key 연동

### 4-1. 앱 마켓플레이스에서 HyperClass 앱 설치

1. **하이퍼클래스 관리자 페이지** 로그인
2. 좌측 메뉴 → **App Marketplace** 접속
3. 검색창에 **"HyperClass"** 또는 **"TossPayments"** 검색
4. **HyperClass - TossPayments** 앱 선택 → **[Install]**
5. **Install to more sub-accounts** 클릭 → 결제를 받을 서브계정 선택
6. OAuth 자동 인증 완료

> 💡 에이전시 · 서브계정 양쪽에 설치 가능합니다. 결제는 서브계정 단위로 운영되므로, **결제를 받을 서브계정에 반드시 설치**해 주세요.

### 4-2. Custom Page에서 Test Key 입력

7. 앱 설치 완료 후 → 좌측 메뉴 **HyperClass - TossPayments** 클릭
8. **"결제 설정"** 페이지로 이동
9. 다음 항목 입력:
   - **Client Key (Test)**: `test_ck_*` 입력
   - **Secret Key (Test)**: `test_sk_*` 입력
10. **[연결]** 또는 **[Save]** 버튼 클릭

### 4-3. 결제수단 활성화 확인

11. 좌측 메뉴 → **Payments** → **Integrations** 이동
12. **TossPayments** 가 결제수단으로 활성화되어 있는지 확인

> ✅ 이제 하이퍼클래스의 체크아웃 · 퍼널 · 인보이스 · Payment Links에서 TossPayments가 결제 옵션으로 자동 노출됩니다.

## Step 5. 상품 만들기 (Products)

하이퍼클래스의 **Payments → Products** 에서 판매할 상품을 등록합니다.

### 5-1. 일회성 상품 (One-time Product)

단발성 결제용 — 컨설팅, 1회 강의, 디지털 다운로드 등

1. **Payments → Products** → **[+ Create Product]**
2. Product Name + 설명 입력
3. **Pricing Type: One-time**
4. **Currency: KRW (한국 원)**
5. 금액 입력 (예: 49,000원)
6. **[Save]**

### 5-2. 월간 구독 상품 (Monthly Subscription)

매월 자동 과금 — 멤버십, SaaS, 정기 강의 등

1. **Payments → Products** → **[+ Create Product]**
2. Product Name 입력
3. **Pricing Type: Recurring**
4. **Billing Period: Monthly**
5. **Currency: KRW**
6. 금액 입력 (예: 49,000원/월)
7. **[Save]**

### 5-3. 연간 구독 상품 (Annual Subscription)

1년 일시납 또는 연간 자동 갱신

1. **[+ Create Product]**
2. **Pricing Type: Recurring**
3. **Billing Period: Yearly**
4. **Currency: KRW**
5. 금액 입력 (예: 490,000원/년)
6. **[Save]**

### 5-4. 무료체험(Trial) 상품 — 15일 등

무료체험 후 자동 월구독 전환 — 가장 일반적인 구독 모델

1. **[+ Create Product]**
2. **Pricing Type: Recurring**
3. **Billing Period: Monthly** (또는 Yearly)
4. **Setup payment: $0** *(플랫폼의 "Trial period in days" 필드 사용)*
5. **Trial Period: 15 days** *(Trial Period 단위는 항상 '일'입니다. 30일 = 30 입력)*
6. **Currency: KRW**
7. 금액 입력 (예: 49,000원/월 — 무료체험 종료 후 부과)
8. **[Save]**

### 5-5. 커뮤니티 구독 상품 (Communities + Group)

하이퍼클래스 **Communities** 모듈을 사용해 멤버십을 운영하는 경우:

1. **Memberships → Group** 생성
2. **Communities → Group** 생성 (Memberships와 연결)
3. **Workflow 트리거 설정**: "Payment Received → Group에 멤버 추가" / "Subscription Cancelled → Group에서 제거"
4. 위 5-2 또는 5-4 방식으로 Recurring 상품 생성
5. 테스트 결제 검증

### 5-6. 1:1 인보이스 견적서

특정 고객에게 맞춤 견적서를 보내고 결제받는 방식:

1. **Payments → Invoices** → **[+ New]**
2. Contact (수신자) 선택
3. Item 추가 — 항목 / 금액 입력 (예: 컨설팅 1회 — 500,000원)
4. **결제수단: TossPayments 선택**
5. **[Send]** → 이메일 / SMS 발송
6. 수신자가 링크 클릭 → 결제 → 완료 시 자동 "Paid" 표시

## Step 7. 결제경로 6단계 캡처 (카드사 본 심사 자료)

토스페이먼츠 카드사 본 심사를 위해 다음 6단계의 결제경로 화면 캡처가 필요합니다. **Step 10 라이브 전환 단계에서 토스페이먼츠에 제출**하게 되므로, 미리 PPT/PDF 양식으로 정리해 두세요.

> ⚠️ 다음 6장은 카드사 심사 핵심 자료입니다. 빠짐없이 캡처해 주세요.

### 7-1. ① 가맹점 정보 기재 화면
상호명 · 사업자번호 · 홈페이지 URL · 테스트 계정 ID/PW를 한 페이지에 정리

### 7-2. ② 하단 정보 (Footer) 캡처
홈페이지 하단에 다음 6가지가 모두 보이도록:
- 상호명 / 대표자명 / 사업자번호 / 통신판매업번호 / 사업장주소 / 유선전화

### 7-3. ③ 환불 규정 캡처
환불정책 페이지를 처음~끝까지 부분 캡처 (모든 항목이 보이게)

### 7-4. ④ 로그인 / 회원가입 캡처
로그인 페이지 + 회원가입 페이지 (비회원 구매 가능 시 생략 가능)

### 7-5. ⑤ 상품 선택 / 구매과정 캡처
상품 목록 → 상품 → 장바구니 → 주문서 작성 흐름을 슬라이드처럼 모두 캡처

### 7-6. ⑥ 카드 결제경로 캡처
신용/체크카드 선택 → 토스페이먼츠 결제창 → 카드 정보 입력 화면

### 캡처 시 유의사항

- **PPT/PDF 양식으로 제작** (테스트 계정으로 한 컷 캡처)
- **도메인이 보이도록** 브라우저 주소창까지 포함 캡처
- **PC 사이즈로 캡처** (반응형 모바일 사이즈 ✕)
- **상품 흐름이 모두 보이게** — 카드사가 결제 가능 여부를 확인합니다

> ✅ 여기까지 완료하면 하이퍼클래스 × 토스페이먼츠 연동의 **모든 기술적 검증이 끝난 상태**입니다. 이제 안전하게 등기 서류를 발송하고 정식 심사에 들어갈 수 있습니다.

## Step 8. 등기 서류 발송

Test Key 기반 검증이 모두 끝났으면, 토스페이먼츠 정식 가맹점 심사를 위해 등기 서류를 발송합니다.

### 8-1. 등기 발송 주소

```
서울시 강남구 테헤란로 131, 한국지식재산센터 15층 (06133)
토스페이먼츠 가맹점온보딩팀 담당자 앞
```

> 담당자 이름은 [Step 2](#step-2-토스페이먼츠-일반-결제-mid-신청) 신청 후 토스페이먼츠에서 보내는 회신 메일에 안내됩니다.

### 8-2. 등기 발송 시 포함할 서류

**법인사업자**
- 사업자등록증
- 법인등기부등본 (말소사항 포함)
- 주주명부 (앞 7자리 + 지분율 + 인감날인)
- 법인인감증명서

**개인사업자**
- 사업자등록증
- 대표자 신분증 사본 (마스킹 ✕)
- 통장사본 (정산 계좌)

**토스 첨부 양식**
- 외화결제서비스 추가신청서 (PDF 양식 1부) — PayPal MID 신청 시
- 온라인 강의 업종 확약서 (DOCX 양식 1부) — 강의 · 구독 사업자

**대리인 신청 시**
- 위임장 (토스 양식, 인감날인)
- 대리인 신분증 사본 (마스킹 ✕)

### 8-3. 보증보험 가입 (심사 후 안내)

심사 진행 중 또는 완료 후 별도로 처리해야 하는 항목입니다.

- 담보금액은 상품가격 · 결제기간에 따라 책정 → 토스 심사관이 구체적 금액 안내
- 첫 결제 발생일을 보증보험 시작일로 설정
- 보험료는 담보금액의 약 1%
- 인산아이비(insanib.co.kr) 또는 플러스기(plusgi.co.kr) 중 선택

### 8-4. 온라인 강의/구독 사업자 확약서

토스페이먼츠 첨부 양식(DOCX) 작성 후 등기로 회신합니다.

> 💡 등기 발송 시 다른 서류와 함께 동봉해서 한 번에 보내면 편합니다. [Step 7](#step-7-결제경로-6단계-캡처-카드사-본-심사-자료)의 6단계 캡처는 별도로 [Step 10](#step-10-라이브-키-전환-및-카드사-본-심사)에서 제출합니다.

## Step 9. 심사 대기

토스페이먼츠 가맹점 심사가 진행되는 동안 별도로 할 작업은 없습니다.

| 단계 | 평균 기간 |
|---|---|
| 서류 접수 확인 | 1~3일 |
| 가맹점 심사 | **3~5주** (현재 적체) |
| 카드사 1차 심사 (서류 기반) | 1~2주 (병행) |

> 💡 심사 중 추가 자료 요청이 오는 경우가 있습니다. 신청 시 등록한 담당자 이메일을 매일 확인해 주세요. 회신이 빠를수록 심사도 빨라집니다.

### 대기 시간 활용 팁

이 시간 동안 다음을 준비하시면 좋습니다.

- 마케팅 콘텐츠 · 랜딩페이지 작성
- 워크플로우(이메일/SMS 자동화) 셋업
- 멤버십 · 코스 콘텐츠 업로드
- 광고 캠페인 사전 기획

이 가이드 [Step 1~7](#step-1-사업-정보-정리-및-신청-항목-결정)을 먼저 마치셨기 때문에, 하이퍼클래스 결제 시스템 자체는 이미 검증된 상태입니다.

## Step 10. 라이브 키 전환 및 카드사 본 심사

### 10-1. 6단계 캡처 자료 제출

심사가 마무리 단계에 들어가면 [Step 7](#step-7-결제경로-6단계-캡처-카드사-본-심사-자료)에서 만든 6단계 캡처 자료를 토스페이먼츠 담당자에게 회신합니다. 카드사 본 심사가 시작됩니다.

### 10-2. 본 심사 통과 → 라이브 키 발급

- 평균 1~2주 소요
- 통과 시 라이브 키 발급: `live_ck_*` / `live_sk_*`

### 10-3. 라이브 키 입력

1. **HyperClass - TossPayments** → **결제 설정**
2. **Mode: Live** 로 전환
3. Live Client Key + Live Secret Key 입력
4. **[연결]** / **[Save]**

### 10-4. 실 결제 검증

- 본인 카드로 1,000원 ~ 10,000원 결제 1건 진행
- **Payments → Transactions** 에서 "Paid" 확인
- **부분/전체 환불** 1건 진행해서 환불 흐름까지 검증

> ✅ 라이브 결제 + 환불이 모두 정상 작동하면 **일반 결제 운영 준비 완료**입니다.

## Step 11. (선택) 빌링 / PayPal MID 추가 신청

일반 결제 MID가 **라이브로 안정 검증**된 이후, 다음 MID를 추가 신청할 수 있습니다.

### 11-1. 빌링 MID 추가 신청

정기결제 · 멤버십 · 분할결제가 필요한 경우:

1. 토스페이먼츠 로그인 → **전자결제 신청하기**
2. 패키지 선택: **"빌링(정기결제)"** 선택
3. [Step 2](#step-2-토스페이먼츠-일반-결제-mid-신청)와 동일한 절차 (가입비/연회비 면제 처리)
4. 별도 등기 발송 없음 (기존 서류 재활용)
5. 심사 기간: 일반 결제보다 짧음 (평균 1~2주)
6. 빌링 MID 발급 → HyperClass 앱 **결제 설정**에서 빌링 Client/Secret Key 추가 입력

> ✅ 빌링 MID가 활성화되면 하이퍼클래스의 모든 Recurring 상품이 자동으로 정기결제 처리됩니다.

### 11-2. PayPal MID 추가 신청

해외 고객 · 외화 결제가 필요한 경우:

1. 토스페이먼츠 로그인 → **전자결제 신청하기**
2. 패키지 선택: **"해외결제 / PayPal"** 선택
3. **외화결제서비스 추가신청서** (PDF) 작성 후 등기 발송
4. 별도 카드사 심사 — 1~2주 추가
5. PayPal MID 발급 → HyperClass 앱 **결제 설정**에서 PayPal Client/Secret Key 추가 입력

> 💡 **추가 가맹 비용 없음**: 일반 결제 MID가 활성화된 가맹점은 빌링/PayPal MID를 추가할 때 별도 가맹 가입비가 부과되지 않습니다.

## 자주 묻는 질문 (FAQ)

<details>
<summary><strong>Q1. 왜 등기 서류 발송을 Step 8로 미루나요?</strong></summary>

토스페이먼츠 심사는 1개월 이상 소요됩니다. 만약 처음부터 등기 서류를 보내고 심사 대기에 들어간 뒤 하이퍼클래스 셋업을 시작했는데 기술적인 문제가 발생하면, 그 1개월 시간이 모두 헛수고가 됩니다.

이 가이드는 **Test Key로 모든 셋업을 먼저 검증한 뒤 등기 서류를 보내는** 순서로 설계되어 있어, 심사 대기에 들어가는 순간 이미 하이퍼클래스 결제 시스템이 정상 작동함을 확인한 상태입니다. 라이브 키만 들어오면 곧바로 운영 시작할 수 있습니다.

</details>

<details>
<summary><strong>Q2. 토스페이먼츠 심사가 1개월 이상 걸리는 이유는?</strong></summary>

2024~25년 현재 토스페이먼츠 신청이 매우 많이 밀려 있어 심사 부서에 적체가 발생한 상태입니다. 평균 3~5주, 길게는 6주 이상도 소요됩니다. 결제 오픈 시점을 잡으실 때는 **최소 6주 이상의 여유**를 두고 시작하시는 것을 권장합니다.

</details>

<details>
<summary><strong>Q3. 처음부터 빌링 MID도 같이 신청하면 안 되나요?</strong></summary>

현재 토스페이먼츠 정책상 **처음에는 일반 결제 MID 1개만** 신청할 수 있습니다. 일반 결제가 라이브로 검증된 이후에만 빌링/PayPal MID를 추가할 수 있습니다. 정기결제가 비즈니스의 핵심이더라도 일반 결제부터 진행해야 합니다.

</details>

<details>
<summary><strong>Q4. 통신판매업 신고번호가 없어도 신청 가능한가요?</strong></summary>

가능합니다. 다만 **KB국민카드 결제수단이 자동 제외**됩니다. 다른 카드사(신한, 삼성, 현대, BC, 롯데, 하나, NH 등)는 정상 심사됩니다. 간이과세자 · 공공기관은 이 제한 예외입니다.

</details>

<details>
<summary><strong>Q5. Test Key로도 실제 결제가 발생하나요?</strong></summary>

토스페이먼츠 테스트 환경에서도 **실승인이 발생합니다**(실제 카드에서 결제됨). 단 당일 자정(24:00) 전에 **자동 취소**됩니다. 다만 가상계좌 입금건은 자동취소가 안 되므로 수동 환불이 필요합니다. 또한 국민카드 · 카카오뱅크 카드는 테스트 결제가 불가합니다.

</details>

<details>
<summary><strong>Q6. SaaS 모드에서도 사용 가능한가요?</strong></summary>

네, 가능합니다. 하이퍼클래스는 SaaS 모드에서 Stripe 외에 Custom Payment Providers를 지원하며, HyperClass - TossPayments도 이에 해당합니다. Agency Sub-Account에 앱을 설치한 후 SaaS Configurator에서 "Other" 결제 구조를 선택하면 됩니다.

</details>

<details>
<summary><strong>Q7. 한 번 설치한 앱을 여러 서브계정에 동시에 적용할 수 있나요?</strong></summary>

가능합니다. Agency View에서 설치할 때 **"Install to more sub-accounts"** 옵션으로 여러 서브계정에 한 번에 설치할 수 있습니다. 결제 키(Client/Secret Key)는 서브계정별로 별도 입력해야 합니다 (서브계정마다 독립된 토스 가맹점 사용 시).

</details>

<details>
<summary><strong>Q8. Test Key로 테스트 결제는 잘 되는데 라이브 결제는 어떻게 전환하나요?</strong></summary>

라이브 키 발급 후, HyperClass - TossPayments 앱의 **결제 설정** 페이지에서 **Mode: Live**로 전환하고 Live Client/Secret Key를 입력한 뒤 저장하면 됩니다. 기존에 등록된 상품 · 결제 링크는 그대로 사용됩니다.

</details>

<details>
<summary><strong>Q9. 환불은 어떻게 처리되나요?</strong></summary>

**Payments → Transactions** 에서 해당 결제를 선택하고 **[Refund]** 버튼을 클릭하면 토스페이먼츠로 환불 요청이 자동 전송됩니다. 전체 환불 · 부분 환불 모두 지원됩니다.

</details>

<details>
<summary><strong>Q10. 카드사 본 심사를 위한 결제경로 6단계 캡처가 꼭 필요한가요?</strong></summary>

네, 필수입니다. 라이브 키 발급 전 카드사 본 심사가 이루어지며, 6단계 캡처가 핵심 자료입니다. 누락 시 심사가 지연되거나 반려될 수 있으므로 [Step 7](#step-7-결제경로-6단계-캡처-카드사-본-심사-자료)의 모든 항목을 빠짐없이 진행해 주세요.

</details>

<details>
<summary><strong>Q11. 토스페이먼츠 외에 다른 한국 PG사도 연동할 수 있나요?</strong></summary>

기술적으로는 하이퍼클래스의 Custom Payment Provider 프레임워크로 어느 PG사든 연동 가능합니다. 다만 현재 하이퍼클래스에서 공식 지원하는 한국 PG는 토스페이먼츠입니다. 다른 PG사(KG이니시스, NHN KCP, 나이스페이먼츠 등) 연동이 필요한 경우 [benjamin@hyperclass.ai](mailto:benjamin@hyperclass.ai)로 문의해 주세요.

</details>
