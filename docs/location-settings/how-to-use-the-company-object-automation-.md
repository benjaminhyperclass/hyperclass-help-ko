# 회사 객체 자동화 사용법

회사 객체 자동화 설정을 사용하면 연락처의 비즈니스 세부 정보 필드를 회사 목적으로 재활용할 수 있습니다. 이 마이그레이션 기능을 통해 필드 데이터를 전송하고, 새 회사를 생성하며, 해당 연락처를 원활하게 매핑할 수 있습니다.

## 회사 객체 자동화 사용법

회사 객체 자동화에 접근하려면 하위 계정 Settings(설정) > Company(회사) 탭으로 이동하세요:

원하는 창에서 "Create & Associate companies with contacts(연락처와 회사 생성 및 연결)" 체크박스를 체크하면, 비즈니스 이름 필드에 비즈니스 이름이 작성된 모든 연락처가 해당 비즈니스 이름을 가진 회사에 추가되는 프로세스를 시작하라는 프롬프트가 표시됩니다.

![회사 객체 자동화 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48273571220/original/E1-3Y1Jf6IuMl-7JXLkAvy-H5T0pvl9J7g.png?1672982618)

**참고사항**

비즈니스 이름은 대소문자를 구분하지 않습니다. 예를 들어 한 연락처의 비즈니스 이름이 "Burger King"이고 다른 연락처가 "burger king"이라면, 두 개가 아닌 하나의 Burger King 회사가 생성됩니다. 비즈니스 이름에서 공백은 무시되지 않습니다. 예를 들어 한 연락처의 비즈니스 이름이 "BurgerKing"이고 다른 연락처가 "burger king"이라면, 두 비즈니스 이름 모두에 대해 별도의 회사가 생성됩니다. 시스템은 철자가 다른 두 비즈니스 이름을 하나의 회사에 속한다고 인식할 수 없습니다. 예를 들어 "Ice Cream Sunday"라는 비즈니스 이름을 가진 연락처는 "Ice Cream Sundae"라는 비즈니스 이름을 가진 다른 연락처와 같은 회사에 추가되지 않습니다. 알파벳 하나라도 다르면 해당 연락처에 대해 별도의 회사가 생성됩니다.

"Create & Associate companies with contacts(연락처와 회사 생성 및 연결)" 체크박스를 체크하면 이 변경사항을 저장할 수 있는 옵션이 표시됩니다. 프롬프트에서 계속 진행할지 묻습니다:

![확인 프롬프트](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48273571312/original/mPQ5gbB3QnVgq18_UJ07xHsY4iqVbufboA.png?1672982669)

"Yes(예)"를 클릭하면, 연락처 페이지의 일반 정보 섹션에서 비즈니스 이름 필드가 채워진 모든 연락처가 비즈니스 이름과 동일한 이름을 가진 회사에 추가됩니다.

해당 이름의 회사가 존재하지 않으면 새로 생성됩니다.

프로세스가 성공적으로 시작되면 확인 메시지가 표시됩니다:

![프로세스 시작 확인 메시지](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48273571382/original/8sH1uXNoYMV2yqAAN898dnQQIZ3ApRHLyA.png?1672982715)

비즈니스 이름 필드가 이 경우의 고유 식별자입니다. 대소문자 구분은 문제가 되지 않지만, 공백과 철자는 결정 요인입니다.

![비즈니스 이름 필드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48271707249/original/KG8veGT7R7SMcMz1Efp7fWbQyN1xiVn3wQ.png?1672131506)

![연락처 정보](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48271707245/original/aaxyEsY-Sr9jtrfOjneAVqTqvl3M2DrQ0A.png?1672131504)

연락처의 일반 정보 탭에 다음 필드가 채워져 있다면, 생성되는 회사에도 자동으로 매핑됩니다:

- 도로 주소
- 도시
- 국가
- 주/도
- 우편번호
- 웹사이트

## 고려해야 할 다양한 상황:

**비즈니스 이름이 고유하고 단일 연락처에만 있는 경우**

연락처에 있는 비즈니스 이름 값으로 새 회사가 생성됩니다.
그리고 연락처가 해당 회사와 연결됩니다.

**비즈니스 이름과 세부 정보가 여러 연락처에서 유사한 경우**

비즈니스 이름 값으로 새 회사가 생성됩니다. 모든 연락처가 새로 생성된 회사와 연결됩니다.

**비즈니스 이름은 동일하지만 연락처마다 세부 정보가 다른 경우**

비즈니스 이름 값으로 새 회사가 생성됩니다. 모든 연락처가 새로 생성된 회사와 연결됩니다. 아래 세부 정보는 가장 최근에 생성된 연락처의 정보가 고려됩니다:

- 도로 주소
- 도시
- 국가
- 주/도
- 우편번호
- 웹사이트

## 제한사항 및 예상 동작:

설정 기능은 관리자 사용자(에이전시 & 하위 계정)만 사용할 수 있습니다. 사용자 역할을 가진 팀원은 마이그레이션 프로세스를 시작할 수 없습니다.

**참고사항:**
사용자 역할을 가진 사용자도 설정을 볼 수는 있지만, 아무것도 수정할 수는 없습니다.

자동화 설정이 켜지면, 비즈니스 이름이 있는 모든 기존 연락처가 마이그레이션됩니다. 마이그레이션이 완료되면, 비즈니스 이름 필드 값이 있는 새 연락처와 자동으로 동기화됩니다.

연락처에 이미 회사가 연결되어 있다면, 프로세스 시작 시 비즈니스 이름이 다르더라도 다른 회사로 마이그레이션되지 않습니다. 기존 연락처의 마이그레이션은 연락처가 회사와 연결되어 있지 않은 경우에만 발생할 수 있습니다.

**참고사항:**
이미 회사에 추가된 연락처의 비즈니스 이름을 변경해도 자동화에는 영향을 주지 않습니다.

자동화 설정이 꺼지면 자동 동기화나 마이그레이션이 중단됩니다. 다시 켜면 마이그레이션이나 자동 동기화가 시작됩니다.

이 기능에는 모든 활동에 대한 다음과 같은 감사 로그가 있습니다:
- 설정 옵션을 시작한 사용자
- 회사 연결로 업데이트된 연락처
- 회사 생성
- 회사 업데이트

기존 연락처 마이그레이션 중에 설정이 꺼지고 다시 시작되면, 중단된 지점부터 마이그레이션 프로세스를 시작하여 나머지 연락처에 대해 완료합니다.

이 기능은 대량 가져오기, 공용 API, 폼 등을 통해 추가된 연락처에도 적용됩니다. 비즈니스 이름과 회사 간에 실시간 동기화됩니다.

**참고사항:**
가져오는 CSV 파일은 "Business Name"이라는 컬럼 헤더를 가진 컬럼이 있어야 하며, CRM으로 가져올 때 연락처가 회사에 추가되려면 연락처에 대한 각각의 비즈니스 이름이 채워져 있어야 합니다(회사 객체 자동화가 활성화된 경우). 가져오기 중 오류가 발생하여 철자가 틀린 비즈니스 이름이나 회사의 비즈니스 이름과 일치하지 않는 공백이 있는 비즈니스 이름으로 인해 잘못된 회사가 연락처에 연결된 경우, 문제가 있는 연락처를 삭제하고 올바른 비즈니스 이름으로 CSV를 다시 가져와야 합니다(연락처가 복원되는 것이 아니라 새 버전의 연락처가 생성되므로 삭제로 인해 손실된 데이터는 복구되지 않습니다).

이 프로세스는 하위 호환성이 없습니다. 비즈니스 이름을 기반으로 회사가 생성되면, 회사 이름이나 세부 정보의 변경사항이 연락처 일반 정보의 비즈니스 이름 필드 및 기타 동기화된 필드에 반영되지 않습니다.

비즈니스 이름 업데이트 기능을 지원하지 않습니다. 즉, 비즈니스 이름을 사용하여 회사가 생성되면, 비즈니스 이름의 업데이트가 회사 속성이나 연락처와의 관계에 영향을 주지 않습니다.

**참고사항:**
회사 객체 자동화는 이미 회사에 연결되어 있지 않은 기존 또는 새 연락처를 동일한 비즈니스 이름을 공유하는 회사와 연결하는 역할만 합니다. 그러한 회사가 존재하지 않으면 해당 비즈니스 이름으로 회사가 생성됩니다.
