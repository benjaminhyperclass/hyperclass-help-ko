# 반복 예약 설정 방법

반복 예약(Recurring Appointments)은 매일, 매주, 매월과 같이 일정한 간격으로 발생하는 약속이나 미팅입니다. 반복 예약은 정기 미팅이나 약속을 예약할 때 시간과 노력을 절약하는 데 도움이 됩니다. 매번 새로운 예약을 생성하는 대신, 하나의 반복 예약을 설정해서 여러 번의 예약을 처리할 수 있습니다. 이는 주간 미팅이나 월간 점검과 같이 정기적인 일정에 특히 유용합니다.

## 반복 예약 캘린더는 어떻게 설정하나요?

Settings(설정)을 클릭합니다.

![설정 메뉴 클릭](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283839127/original/t0i-OYnO597cpGNvWtrjoflwNDgjkyIMKg.png?1677420392)

Calendars(캘린더)를 클릭합니다.

![캘린더 메뉴 클릭](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283839879/original/e_vUEn7psmwTExeGk4qmlUzUNpke3Gbh6g.png?1677421316)

Create Calendar(캘린더 만들기)를 클릭합니다.

![캘린더 만들기 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283839913/original/0C63_wizc1h6sKV7vcNfpoI4X2oRN7bVkA.png?1677421372)

**Round Robin Booking(라운드 로빈 예약)** 아래의 **Select(선택)**를 클릭합니다.

![라운드 로빈 예약 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283839965/original/0KrW7PCc9NKKe0FEPyLkcKoavUMItTAPNQ.png?1677421443)

참고사항

모든 캘린더 유형에서 반복 예약을 지원합니다. 유일한 차이점은 라운드 로빈을 선택한 경우 캘린더에 최대 1명의 사용자 또는 팀원만 있어야 한다는 것입니다.

**Availability(예약 가능 시간)** 탭에서 **Recurring Appointments(반복 예약)**를 활성화합니다:

![반복 예약 토글 활성화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840055/original/nlSAxjv8OpAI2C8zrfoytnYIQappD4S7SQ.png?1677421542)

반복 예약을 활성화하면 반복 예약 설정을 볼 수 있으며, 사용 목적에 따라 설정을 선택할 수 있습니다.

![반복 예약 설정 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840128/original/88X8wjNcPWLFIiAF-8gZUpdcYlTmN-28Yg.png?1677421681)

"슬롯을 사용할 수 없는 경우" 드롭다운에는 세 가지 옵션이 있습니다:

![슬롯 사용 불가 시 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840460/original/AOfeyq6Z6N0P8PpxZtbKhK4zpZbZl_fm_Q.png?1677422054)

이는 의도한 시간대에 이미 다른 예약이나 이벤트가 예약되어 있는 경우 사용자가 다음 옵션 중 선택할 수 있다는 의미입니다:

**Continue Booking(예약 계속)**

"슬롯을 사용할 수 없는 경우" 드롭다운에서 예약 계속을 선택하면, 중복 예약이 발생할 때 어떻게 할지 묻는 추가 드롭다운이 나타납니다:

![중복 예약 처리 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840273/original/DHilYjLU6R1SX388AQrbu1ACjhRMO5u3PQ.png?1677421858)

예약 계속을 선택하면 캘린더에서 해당 슬롯을 중복 예약할 수 있도록 허용하는 것입니다.

**Skip booking Unavailable slot(사용 불가 슬롯 건너뛰기)**

이는 사용 가능한 슬롯만 예약하고 나머지는 무시한다는 의미입니다.

**Book Next Available Slot(다음 사용 가능 슬롯 예약)**

이는 캘린더가 사용 가능한 슬롯을 찾기 위해 미래의 얼마나 멀리까지 검색하든 상관없이 반복 예약을 진행한다는 의미입니다.

참고사항

사용 가능 여부는 다음 3개 슬롯까지만 확인하며, 사용할 수 없으면 예약을 취소합니다. 사용 가능하면 계속해서 모든 반복 예약을 완료합니다. 반복 예약 옵션(다음 예약 vs 예약 건너뛰기 vs 예약 계속)을 선택할 때 주의해야 합니다.

반복 예약은 표준 슬롯에만 예약할 수 있습니다(커스텀 슬롯의 경우 반복 예약 옵션이 표시되지 않음). 사용자는 기회 관리(Opportunities) 페이지에서 반복 예약을 할 수 없습니다. 반복 캘린더는 예약 드롭다운에서 사용할 수 있지만 단일 시간 슬롯에만 해당됩니다.

## **자주 묻는 질문**

### 기존 워크플로우 트리거가 반복 예약에도 작동하나요?

아니요, 작동하지 않습니다. 예약 트리거 단계에 새로운 필터를 추가했습니다. 따라서 모든 기존 트리거에 워크플로우 트리거 단계를 위한 새로운 필터를 추가해야 합니다.

![워크플로우 트리거 필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840827/original/5LJVNP4M4V1PcJdw4bhHXsd1U5LSVroS8Q.png?1677422413)

예약 트리거 단계에 새로운 이벤트 타입 필터를 추가했으므로, 이제 예약 타입에 따라 워크플로우 트리거를 제어할 수 있습니다. 워크플로우 트리거가 반복 예약에만 작동하게 하려면 이벤트 타입으로 Recurring(반복)을 선택하세요.

계정의 모든 기존 예약 상태 트리거에는 기본적으로 새로운 필터가 추가되었으며, 필터는 Event Type: Normal(이벤트 타입: 일반)입니다. 반복 예약에는 작동하지 않습니다.

**Event Type: Normal(이벤트 타입: 일반)**

이 필터를 선택하면 일반 예약(반복되지 않는)과 모든 기존 비반복 예약이 일반으로 간주됩니다. 이를 선택하면 현재 작동하는 방식과 동일하게 작동합니다.

**Event Type: Recurring(이벤트 타입: 반복)**

이 필터를 선택하면 반복 예약 설정이 켜진 캘린더에서 예약된 예약에만 트리거가 작동합니다.

**Event Type: Any(이벤트 타입: 모든 타입)**

이 필터를 선택하면 모든 타입의 예약에 대해 트리거가 작동합니다.

참고사항:

Customer Booked Appointment Trigger(고객 예약 트리거)는 더 이상 사용되지 않습니다. 이제 Modified By Filter(수정자 필터)를 선택하면:

![수정자 필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283841067/original/oc4eOeQBCx4TbF0am_aA-sClJBhUgf1_3g.png?1677422783)

**Customer(고객):** 고객 예약과 동일하게 작동하므로, 이 필터가 추가되면 위젯에서 예약된 예약에만 작동합니다.

**User(사용자):** 시스템 내부나 위젯에서 예약된 것 외의 다른 소스에서 예약된 모든 예약에 대해 작동합니다.

**API:** API를 통해 생성된 예약에만 작동합니다.

### 예약을 일괄 업데이트할 수 있나요?

사용자는 반복 예약을 일괄 업데이트(취소 또는 일정 변경)할 수 없습니다.

### 모든 반복 예약에 동일한 미팅 장소를 사용할 수 있나요?

미팅 장소가 Zoom이나 Google Meet인 경우, 모든 반복 예약은 시스템이 기본적으로 각 미팅마다 동적 미팅 URL을 생성하므로 서로 다른 미팅 URL을 갖게 됩니다.

### 반복 이벤트는 연동된 캘린더에 어떻게 동기화되나요?

동기화가 켜져 있는 경우, 이벤트는 서드파티 캘린더와 개별 이벤트로 동기화됩니다.

### 반복 예약에 대해 결제를 받을 수 있나요?

현재 반복 기능이 켜진 캘린더에서는 결제를 사용할 수 없습니다.
