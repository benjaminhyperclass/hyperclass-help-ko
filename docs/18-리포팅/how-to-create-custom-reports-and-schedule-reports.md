# 커스텀 리포트 생성 및 예약 방법

커스텀 리포트(Custom Reports)를 사용하면 맞춤형 리포트를 제작하고 예약할 수 있어, 팀이나 고객과 인사이트와 데이터를 공유하는 데 완벽합니다. 처음부터 새로 만들거나 기존 대시보드를 변환하든, 이 기능으로 리포트의 디자인, 내용, 예약을 완전히 제어할 수 있습니다. 활용 방법을 자세히 알아보세요.

---

**중요:** 모든 요금제($97, $297, $497) 하위 계정에서 이제 다중 대시보드와 완전 커스터마이즈 가능한 리포트를 이용할 수 있습니다.

**커스텀 리포트 빌더 사용 시 시간대가 데이터에 미치는 영향을 유의하세요:**

리포트 빌더 날짜 범위: 빌더에 표시되는 데이터는 브라우저에 설정된 시간대를 사용합니다.
이메일 미리보기: 예약된 이메일 미리보기는 **하위 계정의 시간대**를 사용합니다.
**구글 및 페이스북 광고 데이터:** 구글 애즈, 페이스북 애즈 같은 외부 연동의 경우, 데이터는 해당 광고 계정에 설정된 시간대를 기준으로 하며, 하위 계정 시간대와는 무관합니다.

---

### 1단계: 커스텀 리포트 접근

- "Reporting(리포팅)" 섹션으로 이동해서 Custom Reports(커스텀 리포트)를 클릭합니다.

- New Report(새 리포트)를 선택해서 리포트 작성을 시작합니다.

![커스텀 리포트 시작하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155035055490/original/XWop_yTg_BlD5QR9gtE0rOHuULkKgx62vQ.png?1729478541)

### 2단계: 새로 만들기 또는 기존 대시보드 활용

- 처음부터 새로 리포트를 만들거나 기존 대시보드를 리포트로 변환할 수 있습니다.

![리포트 생성 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155035055491/original/CN9NPXV5O74IZ5TAKE1b78VreNdJl28gtA.png?1729478560)

### 3단계: 리포트 빌더 사용하기

- 처음부터 시작하는 경우, 리포트 빌더(Report Builder)로 이동됩니다.

- 상단에서 리포트 이름을 목적에 맞게 변경할 수 있습니다.

![리포트 빌더 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155035055500/original/zmBtf-tN9LhiMM6vL9BqOZxeMPkAz9_AOQ.png?1729478592)

### 4단계: 표지 페이지 커스터마이즈

- 리포트의 첫 페이지인 표지 페이지는 우측의 Page Properties(페이지 속성) 메뉴를 통해 커스터마이즈하거나 제거할 수 있습니다.

![페이지 속성 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155035055523/original/mGNh0S6YZcrDx7yCBMKItqWbD2dT4I7dKw.png?1729478640)

- 페이지 속성 옵션:
  - 회사 로고 업로드
  - 각 페이지의 헤더 설정
  - 리포트 페이지의 배경 색상 조정

### 5단계: 위젯, 섹션, 테마 추가

- 표지 페이지 설정이 완료되면 좌측의 + 아이콘을 클릭해서 위젯 추가를 시작합니다.

- 위젯 사이드바에서 리포트에 포함할 위젯을 업로드하거나 커스터마이즈합니다.

- Elements Menu(요소 메뉴)를 사용해서 제목을 추가하고 리포트를 읽기 쉽게 섹션으로 구성합니다.

- Themes(테마) 섹션으로 전환해서 미리 정의된 테마 중 하나를 설정하거나 리포트에 커스텀 테마를 추가합니다.

### 6단계: 페이지 관리

- 새 페이지를 추가하려면 기존 페이지의 상단 또는 하단에 있는 + 아이콘을 클릭합니다.

![새 페이지 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155035055538/original/uaL0URUopHhC5wDig5wPB48i9pVXWzb73Q.png?1729478707)

- 모든 페이지 목록을 보려면 Page Icon(페이지 아이콘)을 클릭합니다.

- 페이지를 드래그 앤 드롭으로 순서를 변경하고 더 나은 구성을 위해 필요에 따라 이름을 바꿉니다.

![페이지 목록 및 순서 변경](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155035055540/original/zyglJDeH2oIk1fnjkCUcTp6ecOrC8uISQw.png?1729478725)

### 7단계: 리포트 다운로드

- 작업이 완료되면 상단의 Download(다운로드) 버튼을 클릭해서 리포트의 PDF 버전을 다운로드할 수 있습니다.

### 8단계: 리포트 저장, 발송 또는 예약

- 리포트 생성 후 Save(저장)를 클릭한 다음 Send(발송) 또는 Schedule(예약)을 선택합니다.

### 9단계: 리포트 예약 및 배송 설정

- Frequency(빈도): 리포트를 보낼 주기를 설정합니다(매일, 매주 등).

- Time(시간): 리포트가 전송될 정확한 시간을 선택합니다.

- Date Range(날짜 범위): 리포트에 포함될 데이터 기간을 선택합니다.

- "Widget-Level Date Settings(위젯 레벨 날짜 설정)" 토글: 활성화하면 커스텀 날짜 범위와 비교 설정이 있는 위젯들이 이메일로 발송되는 리포트에서 해당 설정을 유지합니다.

- Comparison Trends(비교 트렌드): "Compare With(비교 대상)" 옵션을 토글해서 트렌드 및 비교 데이터를 추가합니다.

- Sender Email(발신자 이메일): 리포트가 발송될 이메일 주소를 커스터마이즈합니다.

- Subject(제목): 리포트 이메일의 제목을 추가합니다.

- Recipients(수신자): 수신자의 이메일 주소를 입력합니다.

- Optional Email Body(선택사항 이메일 본문): 필요한 경우 커스텀 메시지를 추가합니다.

![리포트 예약 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155035055549/original/lBoGoTRg-viTXQNBGh5jjtfyg7ujRHoNHw.png?1729478745)

### 10단계: 테스트 이메일 발송

- 최종 확정 전에 테스트 이메일을 발송해서 수신자에게 리포트가 어떻게 표시될지 확인할 수 있습니다.

### 11단계: 최종 단계 - 저장 및 예약

- 모든 것이 완료되면 Save and Schedule(저장 및 예약)을 클릭해서 설정한 예약에 따라 리포트가 자동으로 발송되도록 합니다.

---

**중요 참고사항:**
이미지와 iframe은 수신자에게 발송되는 리포트 미리보기에는 표시되지만, 보안 제한으로 인해 PDF 내보내기에는 포함되지 않습니다.
