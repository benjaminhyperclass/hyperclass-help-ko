# **커스텀 리포트 템플릿 라이브러리란?**

커스텀 리포트 템플릿 라이브러리는 **리포팅(Reporting) › 커스텀 리포트(Custom Reports)**를 위한 전문적으로 디자인된 보고서 레이아웃의 앱 내 갤러리입니다. 처음부터 만들지 않고 템플릿으로 시작해서 브랜딩을 맞춤화하고 PDF나 이메일을 예약할 수 있어요. 템플릿은 에이전시가 디자인 표준을 적용하고 클라이언트 보고서 작성 속도를 높이는 데 도움을 줍니다.

라이브러리는 새 커스텀 리포트를 만들 때 나타납니다. 거기서 템플릿을 둘러보고, 미리보기하고, 적용할 수 있어서 페이지, 위젯, 테마가 미리 설정된 완전한 보고서가 열립니다.

## 커스텀 리포트 템플릿 라이브러리의 주요 장점

템플릿을 사용하면 설정 오류를 줄이면서 일관성과 속도를 보장할 수 있어요. 아래 장점들은 에이전시와 클라이언트 대면 팀에게 중요한 결과에 초점을 맞춘 내용입니다.

- **브랜드 일관성:** 클라이언트 전체에 걸쳐 폰트, 색상, 레이아웃을 일관되게 유지하세요.

- **빠른 리포트 생성:** 미리 만들어진 템플릿을 사용해 몇 분 만에 새 보고서를 만들고 배포하세요.

- **검증된 모범 사례:** 각 템플릿은 정확성과 효율성을 보장하기 위해 데이터 및 비즈니스 전문가가 디자인했습니다.

- **지속적인 업데이트:** 업계 트렌드와 사용자 피드백을 바탕으로 정기적으로 새 템플릿이 추가되는 확장되는 라이브러리에 액세스하세요.

- **커스터마이징 유연성:** 메트릭, 필터, 비주얼, 레이아웃 등 모든 구성 요소를 정확한 리포팅 요구사항에 맞게 편집할 수 있어요.

- **학습 곡선 감소:** 리포팅을 처음 접하는 사용자나 분석 구축 방법을 표준화하는 팀에게 이상적입니다.

- **시간 절약:** 반복적인 설정 작업을 없애고 데이터 포맷팅 대신 데이터 분석에 집중하세요.

## 템플릿 라이브러리에 커스텀 리포트 추가하기

완성된 재사용 가능한 보고서를 템플릿으로 변환하면 모든 팀이 동일한 레이아웃과 브랜딩을 재사용할 수 있어요. 먼저 올바른 권한을 설정한 다음 보고서를 템플릿으로 업로드하면 하위 계정에서 사용할 수 있게 됩니다.

**최소 접근 권한 설정**

- **리포팅(Reporting) › 커스텀 리포트(Custom Reports)**에서 **보고서 개요(Reports Overview)**에 있는 완성된 보고서를 찾으세요.

- 보고서 이름 옆의 **⋮ (점 3개)**를 클릭하고 **권한 관리(Manage permissions)**를 선택하세요.

다음 역할에 최소 **읽기(Read)** 권한을 부여하세요: **에이전시 사용자(Agency User)**, **계정 관리자(Account Admin)**, **계정 사용자(Account User)**.

![권한 설정하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055789165/original/oGQm8KKTXfA84EK8mf31qFD-A7E8r2n7nw.gif?1760253320)

**업로드하기 전에 - 템플릿용 보고서 준비하기**

- 레이아웃, 브랜딩, 페이지 순서를 최종 확정하세요.

- 클라이언트별 텍스트나 스크린샷을 일반적인 카피/플레이스홀더로 교체하세요.

- 재사용하려는 **날짜 범위**와 위젯 설정을 정규화하세요.

- ⋮ (점 3개)를 클릭하고 **보고서 템플릿 업로드(Upload Report Template)**를 선택하세요.

![템플릿 업로드하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055789208/original/iM1_LnXENcueFLixoWgjemiN2_tYNQDDIg.png?1760253434)

## 커스텀 리포트 템플릿 라이브러리 사용법

리포팅을 운영화하는 가장 빠른 방법은 미리 만들어진 템플릿으로 시작해서 실시간 메트릭으로 미리보고 예측 가능한 일정에 설정하는 것입니다. 아래 단계들은 선택부터 배송까지 팀이 리포팅 플로우를 유지할 수 있게 해줍니다.

- **리포팅으로 이동:** 대상 **하위 계정**을 열고 리포팅(Reporting) › 커스텀 리포트(Custom Reports)로 가서 **새 보고서(New Report)** 버튼을 클릭하세요.

![새 보고서 만들기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055788775/original/bOerS69c7IAqYF97wd3jjFp60ozsmKwnYQ.png?1760252504)

- **시작점 선택:** **대시보드에서(From Dashboard)**, **처음부터(From Scratch)**, 또는 **템플릿 라이브러리에서(From Template Library)** 중에서 선택하세요. **템플릿 라이브러리에서**를 선택해서 큐레이트된 디자인을 둘러보세요.

![시작점 선택하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055788781/original/zL1Oa1FSyA-bYzqt3vCb6miXabxjU-__og.png?1760252547)

- **템플릿 필터링 및 선택:** 라이브러리에서 **틈새**와 **태그**로 필터링해서 적합한 레이아웃을 빠르게 찾으세요. 템플릿을 클릭해서 미리보기한 다음 **템플릿 사용(Use Template)**을 클릭해서 적용하세요.

![템플릿 선택하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055788793/original/wQmdeHQ2nsazF2GNrkgktHVIrW1T7GQzQA.png?1760252595)

- **실제 데이터로 미리보기 및 예약:** 적용 후 **실시간 데이터 미리보기**가 나타납니다. **저장(Save) → 예약(Schedule)**을 클릭해서 **특정 날짜**나 **주간 일정**으로 배송을 설정하세요. (필요시 즉시 **전송(Send)**하거나 PDF를 **다운로드(Download)**할 수도 있어요.)

![미리보기 및 예약하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055788799/original/_2PE-ilAIivwY__lJ35_RuO4jvS0Gvk7fw.png?1760252621)

## **자주 묻는 질문**

**Q: 템플릿을 편집하면 이미 만든 보고서도 업데이트되나요?**
**아니요.** 템플릿은 시작점입니다. 편집은 해당 템플릿으로 **앞으로** 만들 보고서에만 영향을 미치며, 기존 보고서나 예약된 보고서는 그대로 유지됩니다.

**Q: 특정 하위 계정에서 특정 템플릿을 숨길 수 있나요?**
단일 템플릿을 직접 숨길 수는 없어요. **숨겨진 카테고리**를 사용하고 숨기려는 템플릿이 보이는 카테고리에 속하지 않도록 하세요.

**Q: 빌더, 이메일 미리보기, 광고 위젯 간에 숫자가 다른 이유는 뭔가요?**
**시간대** 때문입니다. 빌더는 브라우저 시간대를, 예약된 이메일 미리보기는 하위 계정의 시간대를, 연결된 광고 플랫폼(예: Google/Facebook)은 각각의 계정 시간대를 따릅니다.

**Q: 다운로드한 PDF에 이미지가 포함되나요?**
이미지와 아이프레임은 미리보기/이메일에 나타날 수 있지만 보안 제한으로 인해 다운로드한 PDF에는 포함되지 않습니다.
