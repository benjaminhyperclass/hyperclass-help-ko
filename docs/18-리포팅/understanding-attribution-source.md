# 어트리뷰션 소스 이해하기

# 어트리뷰션이란?

어트리뷰션(Attribution)은 연락처가 어떤 채널이나 소스를 통해 유입되었는지를 세분화하여 보여줍니다. 이 정보를 통해 성장 기회를 식별하고 어떤 채널이 가장 큰 잠재력을 제공하는지 판단할 수 있습니다. 예를 들어, 페이스북 광고와 구글 광고를 동시에 운영한다면, 어떤 것이 Hyperclass로 더 효과적으로 리드를 유입시키는지 알고 싶을 것입니다. 어트리뷰션이 이를 가능하게 합니다.

아래 가이드에서는 Hyperclass의 어트리뷰션과 광고 보고서 설정 방법까지 차근차근 안내해드리겠습니다. 시작해볼까요!

# "최초" 또는 "최근" 어트리뷰션이란?

연락처를 특정 소스에 어트리뷰션할 때, 최초와 최근 어트리뷰션을 고려하는 것이 일반적입니다. 두 가지 모두 모든 연락처에 항상 저장됩니다.

**최초 어트리뷰션** - 연락처가 우리 시스템과 처음 상호작용한 시점입니다. 이때 해당 세션의 어트리뷰션 데이터를 기록합니다. 예: 누군가가 문의 폼을 작성하는 경우

**최근 어트리뷰션** - 가장 최근에 기록된 상호작용입니다. 이때 가장 최근 세션의 어트리뷰션 데이터도 기록합니다. 예: 문의 폼을 작성한 후 Hyperclass의 2단계 주문 폼을 통해 제품을 구매하기로 결정하는 경우. 이때 최근 어트리뷰션 데이터가 기록됩니다.

**최근 어트리뷰션은 마지막으로 기록된 상호작용에 따라 항상 변경됩니다. 하지만 최초 어트리뷰션은 변경되지 않습니다.*

이제 최초와 최근 어트리뷰션이 무엇인지 알았으니, 이 정보가 어디에 저장되는지 알아보겠습니다.

# 연락처 어트리뷰션 데이터는 어디서 확인하나요?

최초든 최근 어트리뷰션 데이터든, 모든 연락처 레코드의 우측 하단에서 이 정보를 확인할 수 있습니다.

하위 계정에서 연락처(Contacts) > 연락처 레코드 선택 > 활동 탭 아래 우측 하단 컬럼에서 어트리뷰션 데이터를 확인하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48247964170/original/VSgXZHb-pBWW3cGC7emsU4xKKLFyaKQJ3g.gif?1661800754)

연락처 생성의 경우, 최초와 최근 어트리뷰션은 다음과 같이 정의됩니다:

- **최초 어트리뷰션**: 연락처의 첫 번째 상호작용이 시스템에 기록되는 것입니다. 보통 웹사이트에 처음 방문하거나 위에 나열된 소스와의 첫 상호작용입니다.
- **최근 어트리뷰션**: 시스템에서 연락처로 전환되기 전 마지막 상호작용입니다.

## 유료 검색 (구글 광고)

유료 검색으로 분류된 트래픽은 유료 검색 캠페인(예: Google AdWords)에서 유입된 것입니다. 이는 구글 광고 보고서 분석에 도움이 됩니다.

UTM 파라미터가 일치해야 합니다 (대소문자 구분):

{YourLandingPageUrl.com}?utm_source=adwords&utm_medium={adname}&utm_campaign={campaignname}&utm_content={adgroupname}&utm_keyword={keyword}&utm_matchtype={matchtype}&campaign_id={campaignid}&ad_group_id={adgroupid}&ad_id={creative}

**구글 광고 UTM 파라미터**

| 이름 | 키 | 값 | 유형 | 예시 |
|------|----|----|------|------|
| UTM 소스 | utm_source | adwords | - | - |
| UTM 매체 | utm_medium | {adname} | - | - |
| UTM 캠페인 | utm_campaign | {campaignname} | 수동, 변경 가능 | summer_sale |
| UTM 콘텐츠 | utm_content | {adgroupname} | - | - |
| 매치 타입 | utm_matchtype | {matchtype} | - | - |
| 캠페인 ID | campaign_id | {adgroupid} | 구글 광고가 설정, 고정 | 123456789 |
| 광고 그룹 ID | ad_group_id | {adgroupid} | - | - |
| 광고 ID | ad_id | {creative} | - | - |

자세한 정보는 [구글 광고 보고서 설정 방법](how-to-set-up-google-ad-reporting.md)을 참조하세요.

## 유료 소셜

유료 소셜로 분류된 트래픽은 유료 소셜 캠페인에서 유입된 것입니다. 페이스북에서 리드가 유입되는 경우 페이스북 광고 보고서 분석에 도움이 됩니다.

UTM 파라미터가 일치해야 합니다 (대소문자 구분):

{YourLandingPageUrl.com}?utm_source=fb_ad&utm_medium={{adset.name}}&utm_campaign={{campaign.name}}&utm_content={{ad.name}}&campaign_id={{campaign.id}}

**페이스북 및 인스타그램 광고 UTM 파라미터**

| 이름 | 키 | 값 | 유형 | 예시 |
|------|----|----|------|------|
| UTM 소스 | utm_source | fb_ad | - | - |
| UTM 매체 | utm_medium | {{adset.name}} | - | - |
| UTM 캠페인 | utm_campaign | {{campaign.name}} | 수동, 변경 가능 | summer_sale |
| UTM 콘텐츠 | utm_content | {{ad.name}} | - | - |
| 캠페인 ID | campaign_id | {{campaign.id}} | 페이스북 광고가 설정, 고정 | 123456789 |

자세한 정보는 [페이스북 광고 보고서 설정 방법](추적-어트리뷰션/how-to-set-up-facebook-ad-reporting.md)을 참조하세요.

## 직접 트래픽

직접 트래픽으로 분류된 트래픽은 소스에 대한 표시가 없는 것입니다. 일반적으로 이들은 브라우저에 URL을 직접 입력했거나 사이트에 접속하기 전에 모든 쿼리 파라미터를 제거한 경우입니다.

이러한 방문자의 소스 URL을 보려면 직접 트래픽 소스를 클릭하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48247964541/original/eIy9MeT34UCvxRzk02GYCS7qwWMvBizWbQ.gif?1661800923)

## 자연 검색

자연 검색으로 분류된 트래픽은 Google, Bing, Yahoo, Duckduckgo 같은 알려진 검색 엔진의 무료 검색 결과에서 유입된 것입니다.

검색 엔진에서 사용된 키워드를 보려면 소스 표에서 자연 검색 소스를 클릭하세요.

알 수 없는 키워드(SSL)가 있는 경우, 검색 엔진이 사용자 데이터를 암호화하기 때문일 가능성이 높습니다. 예를 들어, Google은 사용자가 입력한 모든 검색어를 암호화합니다.

## 소셜 미디어

자연 소셜로 분류된 트래픽은 소셜 미디어 웹사이트나 앱에서 유입된 것입니다. 예를 들어, 방문자가 자신의 소셜 미디어 계정에 콘텐츠나 웹사이트를 공유하고, 팔로워들이 콘텐츠를 방문하거나 소셜 미디어 메시지 안의 링크를 방문하는 경우입니다.

## 레퍼럴

레퍼럴로 분류된 트래픽은 웹사이트에 링크한 외부 사이트에서 유입된 것입니다. 검색 엔진이나 소셜 미디어 사이트가 아니어야 합니다. 참조 도메인은 사이트에 링크하는 여러 페이지를 가질 수 있습니다.

## 기타

수신 전화, SMS, 이메일, 왓츠앱 메시지 또는 페이스북 메시지와 같은 다양한 채널을 통해 리드가 생성되는 경우, 모든 정보를 한 곳에 표시하는 것이 중요합니다. 이는 모든 리드를 추적하고 누락되는 것이 없도록 보장하는 데 도움이 됩니다. 모든 리드를 관리하는 중앙 집중식 시스템을 갖춤으로써, 비즈니스는 잠재 고객을 놓치지 않을 수 있습니다.

## CRM UI

Hyperclass 앱 CRM을 통해 리드가 수동으로 생성된 경우 소스가 이것으로 정의됩니다. 이 분류는 Hyperclass 앱 CRM 내에서만 수동으로 생성된 리드를 식별하는 데 도움이 됩니다.

## 써드파티

Zap과 같은 써드파티 도구로 리드가 생성된 경우 소스가 이것으로 정의됩니다. 이 분류는 써드파티 연동을 통해 생성된 리드를 식별하는 데 도움이 됩니다.

# 어떤 이벤트가 어트리뷰션을 기록하나요?

같은 세션 내에서 연락처는 다음 작업 중 하나 이상을 해야 합니다:

- 폼/설문 제출
- 캘린더 예약 제출
- 채팅 위젯 (연락처 정보 제출 후)
- 주문 폼 제출 (1단계 또는 2단계)

모든 어트리뷰션 데이터가 캡처되려면 이 작업은 Hyperclass 폼, 설문, 캘린더, 채팅 위젯, 주문 폼이어야 합니다. Hyperclass가 아닌 이벤트는 UTM 파라미터 데이터를 포함하여 어트리뷰션 데이터를 캡처하지 않습니다.

# 어트리뷰션 소스는 어떻게 결정되나요?

연락처 어트리뷰션 소스가 왜 Hyperclass에 의해 연락처에 추가되었는지 이해하려면, 어트리뷰션 데이터를 결정할 때 무엇을 찾는지 알아야 합니다.

모든 어트리뷰션 이벤트는 트래픽을 특정 소스로 분류하기 위한 규칙 세트를 따르며, 전체 페이지 URL과 가능한 경우 참조 도메인을 이러한 규칙과 대조하여 확인합니다.

다음 순서로 적용됩니다:

| 순서 | 규칙 | 소스 |
|------|------|------|
| 1 | "utm_source" 파라미터에 구글 광고용 "adwords"라는 단어가 포함됨 | 유료 검색 |
| 2 | "gclid", "wbraid" 또는 "gbraid" 파라미터가 존재 (구글 클릭 ID)<br>"msclkid" 파라미터가 존재 (Bing/Yahoo용 마이크로소프트 클릭 ID) | 유료 검색 |
| 3 | "utm_source", "utm_medium", 또는 "utm_campaign" 파라미터가 존재하고 참조 도메인이 google.com | 유료 검색 |
| 4 | "utm_source" 파라미터에 페이스북 광고용 "fb_ad"라는 단어가 포함됨<br>utm_source=facebook은 더 이상 사용되지 않음, "fb_ad" 사용 권장<br>"utm_source" 파라미터에 링크드인 광고용 "linkedin_ad", 트위터 광고용 "twitter_ad", 레딧용 "reddit_ad"가 포함됨<br>왓츠앱 대화용 "ctwa_clid"가 존재 | 유료 소셜 |
| 5 | 참조 도메인이 소셜 미디어 사이트 | 소셜 미디어 |
| 6 | 참조 도메인이 검색 엔진 (Google, Yahoo, Bing, Duckduckgo) | 자연 검색 |
| 7 | 참조 도메인이 소셜 미디어 사이트나 검색 엔진이 아님 | 레퍼럴 |
| 8 | 참조 도메인이나 추적 URL이 없음 | 직접 트래픽 |
| 9 | 수신 전화/SMS/이메일/왓츠앱/페이스북 메시지 또는 위에서 다루지 않은 기타 소스에서 리드가 유입 | 기타 |
| 10 | CRM 시스템 내에서 수동으로 생성된 리드 | CRM UI |
| 11 | Zap 같은 써드파티 연동 도구에서 생성된 리드 | 써드파티 |

이제 Hyperclass가 특정 어트리뷰션 소스를 적용하는 방법을 알았으니, 각 어트리뷰션 소스를 함께 살펴보겠습니다.

# 광고 보고서 설정

유료 광고 어트리뷰션 추적을 설정해야 할 때는 페이스북, 인스타그램, 구글 광고 설정에 대한 다음 문서를 참조하세요.

## 페이스북 및 인스타그램 광고 보고서 설정

# **광고 보고서 문제 해결**

[페이스북 또는 인스타그램 광고 보고서](추적-어트리뷰션/how-to-set-up-facebook-ad-reporting.md)나 [구글 광고 보고서](how-to-set-up-google-ad-reporting.md)를 설정한 후 Hyperclass에서 어트리뷰션 데이터가 기록되지 않는다면, 다음 문제 해결 단계를 시도해볼 수 있습니다...

- **최종 URL의 공백이나 오타 확인**
UTM 파라미터를 생성할 때 오타, 복사 시 추가 공백, 대소문자 구분을 확인하세요. 제공된 예시를 사용하고, 복사해서 붙여넣기를 권장합니다. 올바르게 작성되었는지 잠시 시간을 내어 확인해보세요. 또한 예시는 대소문자를 구분하므로, 있어야 할 곳에 대문자나 소문자가 없으면 어트리뷰션 오류가 발생합니다.

최종 URL = 연락처가 광고에서 클릭하는 URL로, 사이트나 폼에서 다음 작업을 수행할 곳으로 이동합니다.

- **제출이 최종 URL에서 발생하는지 확인**
UTM 파라미터가 기록되려면, 사용자가 광고에서 랜딩한 후 이 페이지를 벗어나면 안 됩니다. 예를 들어, 누군가가 페이지에 랜딩한 후 버튼을 클릭하여 다른 페이지로 이동해서 폼을 제출하면, UTM의 어트리뷰션 데이터가 추가되지 않습니다. 따라서 폼을 작성할 때 이 페이지에 머물러야 합니다.

해결 방법으로는 팝업이나 제출 폼이 있는 섹션을 추가하여 데이터가 캡처되도록 하는 것입니다.

- **페이스북, 인스타그램, 구글 광고에 커스텀 UTM 파라미터를 추가하지 마세요. 설정 지침에 따라 Hyperclass UTM 템플릿을 사용하시기 바랍니다.**
설정 가이드에 제공된 템플릿 예시에 커스텀 UTM 파라미터를 추가하지 마세요. 이로 인해 문제가 발생할 수 있으며 데이터 누락이나 데이터 기록 실패가 발생할 수 있습니다.
