# 이메일 검증(Email Validation) 활성화하기

이메일 검증은 이메일 주소가 전달 가능한지 확인하는 과정입니다. 즉, 이메일을 보내기 전에 해당 이메일 주소로 메일 전달이 가능한지 미리 확인하는 도구예요.

**중요 안내:**
이메일 검증 기능은 LC Email 및 Mailgun 사용자만 이용할 수 있습니다.

## 이메일 검증 작동 방식

- 모든 이메일은 발송 전에 검증됩니다(이전에 검증되지 않은 경우). 이메일 발송 전에 검증 날짜를 확인하여 90일 이전에 검증되었다면 재검증이 실행됩니다.

- 연락처 상세 페이지에서 수동으로 확인할 수 있습니다. 연락처 레코드로 이동 → 오른쪽을 아래로 스크롤 → 봉투 아이콘 클릭

## FAQ

### 이메일 검증에서 무엇을 확인하나요?

- 이메일 주소 구문이 올바른지 확인
- 도메인이 이메일 수신 설정되어 있는지 확인
- 이메일 주소가 실제로 존재하는지 확인
- 이메일 주소가 고위험 주소가 아닌지 확인
- 이메일 주소가 역할 기반(info@domain.com, admin@domain.com 등)이 아닌지 확인

### 이메일 검증이 중요한 이유는 무엇인가요?

이메일 검증이 중요한 이유:

- 무효한 이메일 주소를 제거하여 전달률을 높입니다
- 이메일 목록을 최신 상태로 유지하여 메일링 리스트의 건전성을 관리합니다
- 클릭률(CTR), 열기율, 클릭-열기 비율(CTOR) 등 이메일 마케팅 지표를 개선합니다
- 발신자 평판을 보호합니다

### 이메일 검증이 전달률에 어떤 영향을 주나요?

이메일 전달률은 반송 이메일의 큰 영향을 받습니다. 이메일 검증을 미리 수행하여 이메일 반송을 줄일 수 있어요.

이메일 서비스 공급업체(ESP)가 구독자의 인박스에 이메일을 전달하지 못하면 IP 주소와 도메인 평판에 타격을 입습니다. 이는 향후 전달률에 영향을 미치는데, 이메일 클라이언트가 IP와 발신자 평판에 의존하여 메시지 전달 여부를 결정하기 때문입니다. 발신자나 IP 평판이 나쁘면 이메일 클라이언트는 귀하의 도메인에서 발송된 이메일을 구독자의 스팸 폴더에 넣거나 아예 전달하지 않을 수도 있어요.

### 언제 이메일 목록을 검증해야 하나요?

메일링 리스트를 구매했거나(추천하지 않음) 잘 모르는 메일링 리스트를 물려받은 경우, 이메일 검증기를 사용하는 것이 좋은 아이디어예요.

또한, 목록이 늘어났거나 마지막으로 이메일 주소를 검증한 지 시간이 많이 지났다면 오늘 메일링 리스트의 상태를 점검해보시기를 추천드려요.
