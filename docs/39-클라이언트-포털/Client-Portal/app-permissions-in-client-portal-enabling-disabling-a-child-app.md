# 클라이언트 포털의 앱 권한: 하위 앱 활성화/비활성화

클라이언트 포털에서 새로운 차원의 맞춤화와 제어를 제공하는 최신 기능인 앱 권한(App Permissions)을 소개합니다. 이 업데이트를 통해 사용자는 특정 요구 사항과 선호도에 따라 앱 권한을 조정할 수 있으며, 로그인 시 기본 페이지를 선택할 수도 있습니다.

## **앱 권한 맞춤 설정 방법**

### 앱 권한 액세스하기

- 클라이언트 포털 설정으로 이동: 클라이언트 포털에 로그인하고 설정 메뉴를 찾아보세요. 여기서 "앱 권한(App Permissions)"이라는 새로운 옵션을 찾을 수 있습니다.
- "앱 권한(App Permissions)" 선택: 이 옵션을 클릭하여 앱 권한을 확인하고 수정하세요.

![클라이언트 포털 앱 권한 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155037205550/original/TSoj3b6yGAgqNk1IhW8pbxpxOsZWiwOLwA.png?1732608523)

### 앱 활성화/비활성화

- 사용 가능한 앱 검토: 앱 권한 섹션에서 포털과 관련된 모든 사용 가능한 앱과 기능 목록을 확인할 수 있습니다.
- 권한 토글: 각 앱 옆의 토글 스위치를 사용하여 활성화하거나 비활성화하세요. 앱을 비활성화하면 클라이언트 포털 인터페이스에서 해당 기능과 표시가 제거됩니다.

![앱 권한 토글 스위치 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155037204944/original/Lr4k7zNg1ti8iS6JUEc1l7Ul3k4WcGj8_A.png?1732608165)

### 기본 페이지 설정

- 기본 페이지 옵션 찾기: 같은 앱 권한 영역에서 기본 페이지를 선택하는 옵션을 찾으세요.
- 선호하는 페이지 선택: 드롭다운 메뉴에서 사용자가 클라이언트 포털에 로그인할 때 즉시 보게 하려는 페이지를 선택하세요.

### **자동 권한 정의 (표시 규칙)**

클라이언트 포털 앱은 각 연락처에 대해 두 가지 표시 확인을 따릅니다:

- 클라이언트 포털 소유자가 앱 권한에서 앱을 활성화해야 합니다.
- 연락처는 해당 앱 내에 활성화되거나 관련된 내용이 있어야 합니다. 볼 내용이 없다면 포털을 깔끔하게 유지하기 위해 앱이 숨겨집니다.

표시는 자동으로 업데이트됩니다. 액세스를 허용하거나 새로운 콘텐츠를 사용할 수 있게 되면 연락처에게 앱이 표시되며, 앱 권한에서 앱을 비활성화하면 모든 사용자에게 숨겨집니다.

### **"적격 콘텐츠"의 의미 (예시)**

연락처는 다음과 같은 관련 콘텐츠가 있을 때만 앱을 볼 수 있습니다:

- 강의(Courses): 연락처는 액세스할 강의가 있은 후에 강의 앱을 볼 수 있습니다 (예: 첫 강의 구매/액세스 권한이 부여된 후).
- 인보이스 / 견적서 / 계약서: 연락처는 검토할 수 있는 항목이 있을 때 이러한 앱을 볼 수 있습니다 (예: 확인할 인보이스가 있을 때).
- 제휴(Affiliates): 연락처는 관련 제휴 액세스/활동이 있을 때 앱을 볼 수 있습니다.

## FAQ

Q: 나중에 기본 페이지를 변경할 수 있나요?
A: 물론입니다. 언제든지 앱 권한 설정을 다시 방문하여 기본 페이지를 조정하거나 앱 권한에 다른 수정 사항을 적용할 수 있습니다.

Q: 자주 사용하는 기능을 실수로 비활성화하면 어떻게 되나요?
A: 실수로 기능을 비활성화했다면 앱 권한 설정으로 돌아가서 해당 기능을 다시 토글하여 쉽게 재활성화할 수 있습니다.
