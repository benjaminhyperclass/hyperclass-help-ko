## **클라이언트 포털 URL**

클라이언트 포털 URL을 복사해서 고객들에게 전송할 수 있습니다. 이 URL은 설정한 커스텀 도메인에 따라 자동으로 업데이트됩니다.

![클라이언트 포털 URL](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155010292182/original/-uqXj0h84-UQHFaqLRuTU_2LqCsJmKNtUA.png?1697537422)

## **초대된 사용자와 가입한 사용자**

'초대된 사용자'는 작업 메뉴에서 초대 옵션을 선택해 클라이언트 포털에 초대된 사용자 수를 보여줍니다.

'가입한 사용자'는 실제로 클라이언트 포털에 가입한 사용자 수입니다.

![초대된 사용자와 가입한 사용자 통계](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155010292573/original/KHXwkkC02Mk2EehPylnHCT5tvgU7VTIirg.png?1697537631)

## **작업 1 - 매직 링크**

매직 링크를 사용하면 연락처가 별도의 로그인 없이 클라이언트 포털에 바로 접속할 수 있습니다. 이메일이나 메시지로 전송할 때 유용합니다.

![매직 링크 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155010295415/original/u_iu3EOTE-g0MEYFZAESg4GoNEWrqcaQRQ.png?1697538706)

참고: 매직 링크는 연락처별로 개별 생성됩니다. 워크플로우 트리거로 이메일에 매직 링크를 사용하고 싶다면, 자동으로 필요한 데이터를 가져와서 매직 링크를 생성하는 커스텀 값(Custom Value)을 사용하세요. 대시보드에서 생성한 매직 링크는 수명이 짧고 자동으로 만료됩니다. 고객이 링크가 더 이상 유효하지 않다고 신고하면, 새 매직 링크를 생성해서 다시 전송하세요.

## **작업 2 - 클라이언트 포털 초대**

초대 작업을 클릭해서 클라이언트 포털에 새 사용자를 초대할 수 있습니다. 이미 존재하는 연락처를 초대하거나 이메일 주소를 직접 입력할 수 있습니다. 초대장은 기본 도메인에서 발송됩니다.

![클라이언트 포털 초대](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155010295329/original/DUG2QqLn3r8PLV6IfoXzV5t0Z6Dd3CH2Xg.png?1697538679)

## **작업 3 - 로그인 이메일**

로그인 이메일은 빠른 로그인 버튼이 포함된 자동 이메일을 연락처에게 전송합니다. 고객들은 이메일의 로그인 버튼을 사용해 비밀번호 입력 없이 클라이언트 포털에 로그인할 수 있습니다.

![로그인 이메일 발송](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155010295799/original/MhNJZKg4lMyzxt_90RBibWwm3LeTgf8Enw.png?1697538833)

참고: 여러 연락처를 선택해서 로그인 이메일을 한 번에 보낼 수 있습니다.
