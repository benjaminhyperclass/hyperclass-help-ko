# **강제 앱 업데이트 기능이란?**

강제 앱 업데이트 기능은 사용자가 앱을 계속 사용하기 전에 모바일 애플리케이션을 최신 버전으로 업데이트하도록 요구하는 기능입니다. 새 버전이 출시되면 사용자는 해당 앱 스토어를 통해 앱을 업데이트하라는 안내를 받게 됩니다. 이를 통해 모든 사용자가 최신 기능과 보안 개선 사항에 액세스할 수 있습니다.

## **강제 앱 업데이트 기능의 주요 장점**

필수 앱 업데이트를 구현하면 사용자 경험과 애플리케이션 무결성을 향상시키는 여러 가지 이점이 있습니다.

- **보안 강화**: 모든 사용자가 최신 보안 패치가 적용된 버전을 사용하도록 보장합니다.

- **성능 향상**: 사용자는 최신 릴리스에 포함된 성능 최적화의 혜택을 받습니다.

- **기능 일관성**: 모든 사용자가 최신 기능과 개선 사항에 액세스할 수 있도록 보장합니다.

- **지원 이슈 감소**: 구버전 앱으로 인한 문제를 최소화하여 지원 요청을 줄입니다.

- **일관된 사용자 경험**: 모든 기기와 사용자에게 일관된 경험을 제공합니다.

## **강제 앱 업데이트 기능 활성화 방법**

강제 앱 업데이트 기능을 활성화하면 사용자가 중요한 업데이트를 건너뛸 수 없어 앱의 무결성과 사용자 신뢰를 유지할 수 있습니다.

- **클라이언트 포털 설정에 접근하기**:

Hyperclass 대시보드의 **Sites(사이트)** 섹션으로 이동합니다.

- **Client Portal(클라이언트 포털)**을 선택합니다.

- **Settings(설정)**을 클릭합니다.

- **강제 업데이트 활성화**:

설정 메뉴에서 **Force App Update(강제 앱 업데이트)** 옵션을 찾습니다.

- 토글을 **Enable(활성화)**로 전환합니다.

- **버전 세부사항 지정**:

앱의 최신 버전 번호를 입력합니다.

- Apple App Store와 Google Play Store 모두에 대한 직접 다운로드 링크를 제공합니다.

- **변경사항 저장**:

**Save(저장)**를 클릭하여 새 설정을 적용합니다.

## **강제 앱 업데이트 활성화 후 사용자 경험**

구현 후 사용자 여정을 이해하면 사용자 질문을 예상하고 원활한 전환을 보장하는 데 도움이 됩니다.

- **업데이트 안내**: 앱을 실행하면 사용자는 새 버전이 사용 가능하다는 안내를 보게 됩니다.

- **앱 스토어로 이동**: 업데이트 옵션을 선택하면 사용자는 최신 버전을 다운로드할 수 있는 해당 앱 스토어로 이동됩니다.

- **액세스 제한**: 업데이트가 완료될 때까지 사용자는 앱의 기능에 액세스할 수 없습니다.

- **원활한 전환**: 업데이트 후 사용자는 향상된 기능과 보안으로 정상적인 앱 사용을 재개할 수 있습니다.

## **자주 묻는 질문**

**Q: 사용자가 업데이트 안내를 건너뛰고 앱을 계속 사용할 수 있나요?**

A: 아니요, 강제 앱 업데이트 기능이 활성화되면 사용자는 앱을 최신 버전으로 업데이트해야 액세스를 다시 얻을 수 있습니다.

**Q: 강제 업데이트를 필요로 하는 앱 업데이트는 얼마나 자주 해야 하나요?**

A: 중요한 수정 사항이나 주요 기능 개선이 포함된 주요 릴리스에 대해 강제 업데이트를 활성화하는 것을 권장합니다.

**Q: 이 기능을 활성화하면 이미 최신 버전을 가진 사용자에게 영향을 주나요?**

A: 아니요, 최신 버전을 사용 중인 사용자는 업데이트 안내를 보지 않으며 중단 없이 앱을 계속 사용할 수 있습니다.

**Q: 예정된 필수 업데이트에 대해 사용자에게 알릴 방법이 있나요?**

A: 네, 앱의 알림 시스템을 사용하여 사용자에게 예정된 업데이트에 대해 알리고 사전에 업데이트하도록 권장할 수 있습니다.
