## 개요

- 음성 AI 채팅 위젯(Voice AI Chat Widget)을 통해 방문자는 채팅 위젯 내에서 직접 AI 에이전트와 실시간 음성 대화를 나눌 수 있습니다.
- 이 기능을 사용하면 전화번호, 다운로드, 전화 설정 없이도 브라우저의 마이크와 스피커를 통해 대화할 수 있습니다.
- 현재 Labs에서 제공되므로, 정식 출시 전에 기능을 활성화하고 테스트해볼 수 있습니다.

## 음성 AI 채팅 위젯의 기능

음성 AI 채팅 위젯은 기존 채팅 위젯 프레임워크 내에서 음성 전용 상호작용 모드로 작동합니다. 방문자는 마이크 아이콘을 클릭하여 라이브 음성 세션을 시작할 수 있으며, AI 에이전트가 실시간으로 대화식으로 응답합니다.

세션 중 방문자는 다음을 할 수 있습니다:

- 음성 대화 시작 또는 종료
- 필요에 따라 세션 음소거 또는 음소거 해제
- 연락처 정보와 세부사항을 음성으로 제공
- 연결을 끊고 다시 호출하여 언제든 대화 재시작

AI 에이전트는 설정된 음성 AI(Voice AI) 자동화를 사용하여 상호작용을 처리합니다. 현재 사용할 수 없는 통화 전환(Call Transfer)을 제외하고 기존의 모든 음성 AI 액션이 지원됩니다.

## 주요 기능

- 채팅 위젯에서 직접 브라우저 기반 음성 통화
- 발신 전화나 전화번호 불필요
- 브랜드나 비즈니스에 맞게 설정 가능한 AI 에이전트 이름
- 투명성을 위해 자동으로 표시되는 통화 녹음 안내
- 자동화된 또는 반복적인 오남용을 방지하는 내장 reCAPTCHA 보호
- 기존 음성 AI 액션 및 워크플로우와 완전 호환
- 최대 100개의 음성 AI 에이전트 생성 가능

## 음성 AI 채팅 위젯 활성화

### 1단계: Labs에서 활성화

- Settings(설정) > Labs > sub-account 탭으로 이동합니다.
- Voice AI Chat Widget 옵션을 찾습니다.
- 기능을 ON으로 토글하여 활성화합니다.

활성화하면 채팅 위젯(Chat Widget) 빌더에서 Voice AI 채팅 유형을 사용할 수 있습니다. 여러 계정을 관리하는 경우, 특정 하위 계정에 대해 선택적으로 활성화하거나 비활성화할 수 있습니다.

### 2단계: 위젯 설정

- Sites(사이트) > Chat Widgets(채팅 위젯)으로 이동합니다.

![채팅 위젯 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055903517/original/Q1-MJZf8ZZcm-4mIize8osYFg54_1pqfAQ.png?1760398254)

- 기존 위젯을 열거나 새로 만듭니다.

![위젯 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055903535/original/pfI4ppauYH6WWb4l7eF5XNzx3QPLIbpXzA.png?1760398318)

- Agent 탭에서 Voice AI Agent를 선택합니다.

![Voice AI Agent 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055903538/original/7oFcDlH7QnaLH-Afuhw-s6hgwPfUnLklZA.png?1760398339)

- Voice AI Agent Name 필드에 방문자에게 표시될 이름을 입력합니다(예: "영업 어시스턴트" 또는 "고객지원봇").

![에이전트 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055903574/original/1i2V1PS8vHbRAxeAfoyGMSV2_LCygq6BNg.png?1760398575)

- Save(저장)를 클릭하여 설정을 적용합니다.

**참고**: 음성 AI(Voice AI) 설정과 에이전트 생성이 완료되지 않으면 음성 AI 채팅 위젯을 생성할 수 없습니다.

[음성 AI 에이전트 생성 방법은 이 도움말 문서를 참조하세요.](../../13-AI-Employee/기타/creating-voice-ai-agents.md)

![음성 AI 에이전트 필요](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055903285/original/OUfa5GVYqS3pjXHa9HnrH1RZ4ScNJ90q0g.png?1760397130)

### 3단계: 웹사이트나 퍼널에 위젯 추가

- 채팅 위젯 빌더에서 Get Code(코드 받기)를 클릭합니다.

![코드 받기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055903541/original/R4YCb6laHSUzHEbuhWOCiu0tqMkn34o6DQ.png?1760398383)

- 생성된 `<script>` 태그를 복사합니다.
- 위젯을 표시하려는 웹사이트나 퍼널의 **Tracking Code(추적 코드)** 섹션, Header 또는 Footer Tracking에 이 코드를 붙여넣습니다.

![추적 코드 삽입](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066207795/original/aYSAbVD-GV-0KjSDpqxtXhVvuHmedVMhaQ.png?1772640349)

- 페이지를 발행하거나 업데이트합니다. 발행되면 사이트에 음성 AI 채팅 위젯이 나타납니다.

![위젯 작동 모습](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066208109/original/Lfz0rb_C2PHUnrgZ584jizp_iU8DYV3g6g.gif?1772640572)

### 4단계: 방문자 상호작용 방식

방문자가 채팅 위젯을 열면:

- 음성 채팅 기능을 나타내는 마이크 아이콘을 볼 수 있습니다.
- 아이콘을 클릭하면 라이브 음성 세션이 시작됩니다.
- 브라우저에서 마이크 액세스를 요청합니다(방문자가 허용해야 함).
- AI 에이전트가 실시간으로 대화식으로 응답합니다.
- 방문자는 원하는 대로 통화를 시작, 음소거 또는 종료할 수 있습니다.

![방문자 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055903550/original/mh4kek3BGwKqEbAsYHBnLVBzMolY9rxkZQ.png?1760398427)

- 같은 사용자가 여러 번 빠르게 연결/해제를 반복하면(60초 내 10회), 시스템이 정당한 사용인지 확인하기 위해 reCAPTCHA 인증을 트리거합니다.

![reCAPTCHA 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055903553/original/UKfdKiG-95x90FwGZZqvYEx2Gb1vGl6t8A.png?1760398471)

## 모범 사례

- 브랜드 톤에 맞는 명확하고 친근한 에이전트 이름을 선택하세요.
- "Contact", "Pricing", "Demo Request" 등 구매 의도가 높은 페이지에 음성 AI 위젯을 추가하세요.
- 원활한 기능을 위해 다양한 브라우저에서 마이크 권한을 테스트하세요.
- 에이전트를 훈련하거나 테스트할 때 배경 소음을 최소화하세요.
- 투명성을 유지하기 위해 통화가 녹음되지 않음을 사용자에게 알려주세요.
- Labs 단계에서 초기 사용량을 모니터링하여 사용자 행동을 파악하고 필요에 따라 에이전트와 위젯 설정을 조정하세요.

## 보안 및 컴플라이언스

음성 AI 채팅 위젯에는 내장된 보안 및 컴플라이언스 메커니즘이 포함되어 있습니다:

- reCAPTCHA 보호 기능으로 반복적인 연결/해제 활동을 통한 오남용을 방지합니다.
- 투명성을 위해 녹음하지 않음 안내가 자동으로 표시됩니다.
- 모든 대화는 브라우저 내에서 처리되어 사용자 프라이버시를 보장합니다.
- 마이크 및 스피커 액세스를 위한 표준 브라우저 권한 프로토콜을 따릅니다.

## 문제 해결

| 문제 | 가능한 원인 | 해결 방법 |
|------|------------|-----------|
| 음성 세션이 시작되지 않음 | 마이크 권한이 승인되지 않음 | 프롬프트가 나타날 때 사용자가 마이크 액세스를 허용하도록 요청하세요. |
| 오디오 출력 없음 | 스피커 음소거 또는 연결 해제 | 오디오 설정을 확인하고 장치 출력이 활성화되어 있는지 확인하세요. |
| 위젯이 나타나지 않음 | 스크립트가 올바르게 설치되지 않음 | 빌더에서 `<script>` 태그를 다시 복사하여 다시 삽입하세요. |
| 자주 나타나는 reCAPTCHA 프롬프트 | 빠른 연결 시도 감지 | 시스템 트리거를 피하기 위해 재연결 빈도를 늦추세요. |
| 세션 녹음 불가 | 지원하지 않는 기능 | 통화 녹음이 현재 불가능함을 사용자에게 알려주세요. |

## 예시: 리드 수집용 설정

이 예시는 음성 AI 채팅 위젯을 사용하여 음성을 통해 리드를 수집하는 방법을 설명합니다.

목표: 음성 상호작용을 통해 웹사이트 방문자를 자격을 갖춘 리드로 전환

설정 단계:

- Labs에서 음성 AI 채팅 위젯을 활성화합니다.
- 채팅 위젯을 생성하거나 열고 Voice AI Agent 유형을 선택합니다.
- 에이전트 이름을 "영업 어시스턴트"로 지정합니다.
- 다음과 같이 음성 AI 플로우를 설정합니다:
  - 방문자에게 인사
  - 이름과 비즈니스 세부사항 수집
  - 관심 유형 확인(예: 서비스 문의, 데모 요청)
  - CRM에 리드 저장
  - 정중하게 세션 종료
- Contact Us 또는 Pricing 페이지에 위젯을 삽입합니다.

예상되는 경험:

- 방문자가 마이크 아이콘을 클릭합니다.
- AI 에이전트가 인사하고 질문을 시작합니다.
- 방문자가 음성으로 답변을 제공합니다.
- 시스템이 자동으로 세부사항을 캡처하고 저장합니다.

결과: 양식 제출이나 수동 입력 없이도 리드 정보가 수집됩니다.

## 자주 묻는 질문

**Q: 방문자가 사용하려면 전화번호나 앱 다운로드가 필요한가요?**
아니요, 음성 대화는 방문자의 마이크와 스피커를 사용하여 브라우저에서 진행됩니다.

**Q: 음성 AI 에이전트를 먼저 설정하지 않고도 음성 AI 채팅 위젯을 만들 수 있나요?**
아니요, 음성 AI 채팅 위젯을 만들기 전에 음성 AI 설정과 에이전트 생성이 완료되어야 합니다.

**Q: 통화가 녹음되나요?**
위젯에 녹음하지 않음 안내가 표시되며, 이 기능에서는 통화 녹음이 지원되지 않습니다.

**Q: 음성 AI 채팅 위젯에서 사용되는 봇을 어떻게 설정하나요?**
"봇"은 당신의 음성 AI 에이전트입니다. 설정하려면 하위 계정에서 AI Agents(AI 에이전트) → Voice AI(음성 AI)로 이동하여 에이전트 설정(인사말, 프롬프트/행동, 설정한 액션/워크플로우 등)을 생성하거나 편집하세요. 위젯에서 사용하기 전에 음성 AI 에이전트 설정이 완료되어야 합니다.
