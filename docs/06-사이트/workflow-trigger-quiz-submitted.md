# 워크플로우 트리거 - 퀴즈 제출됨

퀴즈 제출 트리거(Quiz Submission Trigger)는 퀴즈가 완료될 때마다 자동 작업을 실행하는 기능입니다. 개인화된 피드백을 보내거나, 점수에 따라 사용자를 분류하거나, 팀에게 할 일을 배정하고 싶을 때, 이 트리거가 원활한 워크플로우 자동화를 보장합니다.

## 퀴즈 제출됨 트리거란?

퀴즈 제출됨 트리거는 연락처가 Hyperclass에서 발행된 퀴즈를 완료하고 제출할 때마다 활성화되는 워크플로우 진입 조건입니다. 이 기능은 비즈니스, 교육자, 마케터가 참여를 간소화하고, 응답을 자동화하며, 퀴즈 기반 상호작용을 최적화하는 데 도움을 줍니다. 이 트리거를 추가하면 퀴즈 데이터가 기록되는 순간 연락처를 자동으로 워크플로우에 등록할 수 있어, 실시간 참여와 맞춤형 여정을 가능하게 합니다.

---

## 퀴즈 제출됨 트리거의 주요 장점

퀴즈 제출됨 트리거는 누군가 퀴즈를 완료할 때마다 빠르고 확장 가능하게 반응할 수 있도록 도와줍니다.

- **실시간 참여**: 참가자가 퀴즈를 제출하는 순간 바로 응답하여 오픈률과 전환율을 높입니다

- **고급 필터**: 퀴즈 이름, 특정 점수, 커스텀 필드와 같은 조건을 설정하여 특정 대상을 타겟팅할 수 있습니다

- **커스텀 액션**: 퀴즈 결과에 따라 이메일 발송, SMS 알림, 할 일 배정 등 개인화된 후속 조치를 자동화합니다

- **운영 효율성**: 수동 내보내기와 스프레드시트 작업을 제거하여 점수가 자동화 시스템으로 직접 전달됩니다

- **확장 가능한 워크플로우**: 조건부 단계, 대기 시간, If/Else 논리와 결합하여 복잡한 육성 시퀀스를 손쉽게 구축할 수 있습니다

## 필터 옵션

필터는 Hyperclass에게 트리거가 언제 실행되어야 하는지 알려주며, 어떤 퀴즈 제출이 워크플로우를 활성화할지 세밀하게 제어할 수 있게 해줍니다. 고급 필터를 사용하면 퀴즈 이름, 전체 또는 카테고리 점수 임계값, 커스텀 필드와 같은 정확한 조건을 설정하여 특정 대상을 타겟팅하고 트리거가 정확히 언제 활성화될지 결정할 수 있습니다.

사용 가능한 각 필터 유형의 분석은 다음과 같습니다:

- **퀴즈 이름**: 특정 퀴즈를 선택하거나, Any를 사용하여 모든 퀴즈를 포함하거나, Is Not을 사용하여 특정 퀴즈를 제외합니다

- **전체 점수**: 80점보다 크거나 50-70점 사이와 같은 규칙을 만들어 총점으로 분류합니다

- **카테고리 점수**: 특정 섹션에서 우수한 성과를 보였거나 어려움을 겪은 학습자를 타겟팅합니다

- **커스텀 필드**: 퀴즈 설계 시 생성한 커스텀 점수 필드를 참조합니다

## 사용 사례 예시

- **학습자를 위한 개인화된 피드백**: 퀴즈 점수에 따라 학생들에게 자동으로 피드백 이메일을 발송하여 학습 경험 개선을 도와줍니다.

- **리드 스코어링 및 분류**: 점수에 따라 퀴즈 응시자를 다양한 카테고리로 분류하고, 타겟팅된 마케팅 캠페인을 위해 CRM 기록을 업데이트합니다.

- **강의 또는 상품 추천**: 퀴즈 성과에 따라 관련 강의나 상품을 추천하여 참여와 전환을 증가시킵니다.

- **영업팀을 위한 자동 후속 조치**: 높은 점수를 받은 퀴즈 응시자를 영업팀에 배정하여 개인화된 아웃리치와 후속 조치를 진행합니다.

- **이벤트 참여 추적**: 퀴즈 완료와 점수 임계값에 따라 이벤트 수료증이나 보상을 발송하는 액션을 트리거합니다.

## 퀴즈 제출됨 트리거 설정 방법

트리거는 퀴즈 완료 시 활성화되며 다음을 가능하게 합니다:

- **조건부 필터링**: 퀴즈 이름, 점수 임계값, 카테고리 기반 점수와 같은 조건을 적용합니다

- **자동 액션**: 이메일, SMS 메시지 발송, 할 일 배정, 기타 워크플로우 단계 시작

- **원활한 통합**: CRM, 마케팅 자동화, 리드 관리 도구와 연결합니다

아래 단계에 따라 퀴즈 제출됨 워크플로우 트리거를 설정하세요.

#### 1단계: 워크플로우 설정에 접근

- 좌측 네비게이션 바에서 **Automation(자동화)**으로 이동합니다

- **Create Workflow(워크플로우 생성)**를 클릭하거나, 워크플로우 이름 옆의 **점 3개** 아이콘을 클릭한 후 Edit Workflow(워크플로우 편집)를 선택하여 기존 워크플로우를 편집합니다

![워크플로우 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050970892/original/wy5sg_K13V7Kn9ewERkLV59Kcwd7cr5dzA.png?1754412739)

#### 2단계: 퀴즈 제출 트리거 추가

- 워크플로우 에디터에서 Add Trigger(트리거 추가)를 클릭합니다

- 사용 가능한 트리거 목록에서 **Quiz Submitted(퀴즈 제출됨)**을 선택합니다

![퀴즈 제출 트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050971056/original/ZfldMMxcFQgGvUTs6yaVvRRRbc1IgCpVVg.png?1754412809)

#### 3단계: 조건 정의

트리거가 언제 활성화되어야 하는지 결정하는 필터를 지정합니다:

- **퀴즈 이름**: 특정 퀴즈를 선택합니다

- **점수 임계값**: 최소 또는 최대 점수 조건을 설정합니다

- **카테고리 기반 점수**: 퀴즈 섹션 내 응답에 논리를 적용합니다

![조건 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050971209/original/cX-l0SXcHNiFpwionmwUMnzmLaSIA2tvpg.png?1754412900)

#### 4단계: 액션 설정

트리거 조건이 설정되면 자동 액션을 정의합니다:

- **이메일 발송**: 점수에 따른 개인화된 이메일 전달

- **SMS 발송**: 제출 후 즉시 사용자에게 알림

- **할 일 배정**: 팀을 위한 후속 할 일 생성

- **CRM 데이터 업데이트**: 퀴즈 결과를 CRM 시스템에 기록

#### 5단계: 저장 및 활성화

- **Save(저장)**를 클릭하여 워크플로우를 저장합니다

- 퀴즈 제출에 따른 자동 액션을 시작하려면 워크플로우를 발행합니다

## 자주 묻는 질문

**Q: 이 트리거는 모든 퀴즈에서 작동하나요?**

네, 트리거는 플랫폼 내에서 생성된 모든 퀴즈에 적용할 수 있습니다.

**Q: 외부 도구와 연동할 수 있나요?**

네, 퀴즈 제출 트리거를 CRM 시스템, 이메일 마케팅 플랫폼, 자동화 도구와 같은 외부 도구와 연동(Integration)을 통해 연결할 수 있습니다.

**Q: 하나의 워크플로우에서 여러 액션을 사용할 수 있나요?**

네, 퀴즈 결과에 따라 하나의 워크플로우 내에서 여러 액션(이메일 발송, CRM 데이터 업데이트, 팀 알림 등)을 설정할 수 있습니다.

**Q: 웹사이트 페이지나 퍼널 단계에 퀴즈를 추가할 수 있나요?**

네! 추가하고 싶은 퀴즈로 이동하여 Embed(임베드) 버튼을 클릭하고 임베드 코드를 복사하세요. 그다음 웹사이트 페이지나 퍼널 단계로 이동하여 커스텀 코드 요소(Element)를 추가합니다. 이 요소에 임베드 코드를 복사하고 페이지를 미리 보면 퀴즈가 추가된 것을 확인할 수 있습니다.
