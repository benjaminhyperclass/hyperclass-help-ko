## 커스텀 객체와 회사 객체란?

커스텀 객체(Custom Objects)는 일반적인 연락처 필드로는 담을 수 없는 비즈니스별 특수 데이터를 체계적으로 수집할 수 있게 해줍니다. 예를 들어 부동산 매물, 반려동물, 차량 정보 등을 말이죠. 이 데이터는 연락처에 연결되어 비즈니스에 가장 중요한 요소를 중심으로 CRM을 구성할 수 있습니다.

회사 객체(Company Object)는 비즈니스 레벨 데이터(고객사, 에이전시, 벤더 등)에 특화된 내장 객체입니다. 여러 연락처를 하나의 회사 아래에 연결하는 데 도움이 됩니다.

간단히 정리하면:

- 커스텀 객체: 프로세스나 업계에 특화된 고유한 데이터를 추적할 때 사용
- 회사 객체: 연락처가 어느 회사에 소속되거나 대표하는지 추적할 때 사용

## 할 수 있는 것들

- 커스텀 객체의 필드들(텍스트, 숫자, 날짜, 드롭다운 등)을 추가할 수 있습니다.
- 데이터를 연락처에 직접 연결할 수 있습니다(일대일, 일대다, 다대다).
- 연결된 레코드 간 데이터 업데이트를 자동으로 동기화할 수 있습니다.
- 폼, 설문, 퀴즈에서 커스텀 객체 필드를 사용할 수 있습니다.
- 각 빌더마다 한 가지 객체 타입만 사용하도록 제한하여 설정 문제를 방지할 수 있습니다.
- 연관 관계가 깨지지 않도록 중요한 "기본 필드(primary fields)"를 보호할 수 있습니다.

회사 객체의 특징:
- 이름, 전화번호, 이메일 등 몇 가지 기본 필드가 제공됩니다.
- 회사명이 자동으로 기본 키(primary key) 역할을 합니다.
- 연락처에서만 사용할 수 있으며, 각 연락처는 하나의 회사 연관만 가질 수 있습니다.
- 연락처 상세 화면에서 접근할 수 있어 연결된 회사 정보를 쉽게 볼 수 있습니다.

커스텀 객체의 유연성:
- 필드와 기본 필드를 직접 정의할 수 있습니다.
- 연락처당 여러 연관 관계를 지원할 수 있습니다.
- 회사, 연락처, 기회 관리에 속하지 않는 특수 데이터를 모델링할 수 있습니다.

## 사용 방법

- Settings(설정) > Custom Objects로 가서 커스텀 객체를 생성하거나 관리하세요.
![커스텀 객체 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055414880/original/FLsvSm2c1-_AHg5j2ye5_JJxSrcCzdsBuA.jpeg?1759839015)

- 회사 객체의 경우 Settings(설정) > Companies(회사)로 이동하세요(이미 내장되어 있음).
![회사 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055415900/original/OxijNS7NS988NmxHy2mP4-GyBXVN1wpDYQ.png?1759839343)

- 폼, 설문, 또는 퀴즈 빌더를 열어주세요.

- 빌더 안에서만 필드를 추가할 수 있습니다.
![폼 빌더에서 필드 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055414942/original/OPxkK7Vw7G9cZ9_d1dQlCXIujqevMSOJPQ.jpeg?1759839039)

- 객체가 연락처에 어떻게 연결될지 선택하세요(연관 관계 설정).

- 시스템이 자동으로 기본 필드를 추가합니다.

- 같은 객체 타입의 필드를 더 추가하세요.

- 저장하고 발행하면, 이제 폼 제출 시 연락처와 객체가 연결됩니다.

## 모범 사례

- 폼/설문/퀴즈마다 한 가지 객체 타입만 사용하세요.
- 커스텀 객체 필드를 추가하자마자 연관 관계를 설정하세요.
- 커스텀 객체의 기본 필드를 정의하세요. 없으면 시스템이 첫 번째 필드를 사용합니다.
- 회사 객체의 경우 기본 키(회사명)가 이미 설정되어 있다는 점을 기억하세요.
- 혼동을 피하기 위해 객체명을 간단하고 일관되게 유지하세요.
- 같은 빌더에서 회사 객체 필드와 커스텀 객체 필드를 함께 사용하지 마세요.

## 활용하기 좋은 경우

- 커스텀 객체: 업계별 특수 추적(부동산 매물, 반려동물, 차량, 케이스, 자산 등)
- 회사 객체: B2B 관계 - 여러 연락처를 하나의 비즈니스 아래에 연결
- CRM 전체에서 구조화되고 일관되며 연결된 데이터 유지

## 적합하지 않은 경우

- 하나의 폼에서 여러 객체 타입을 섞어 사용
- 대량 업데이트나 고급 리포팅(아직 지원되지 않음)
- 회사 또는 커스텀 객체 필드가 포함된 폼/설문/퀴즈를 템플릿 라이브러리를 통해 생성하거나 공유
- 하위 계정 간에 커스텀 필드나 커스텀 객체 공유
- 커스텀 객체 필드에 커스텀 값 추가(지원되지 않음)

## 문제 해결 방법

- 연관 관계가 로드되지 않나요? 인터넷 연결이나 객체 설정을 확인하세요.
- 객체 혼합 에러가 발생하나요? 새 객체 필드를 추가하기 전에 기존 객체 필드를 제거하세요.
- 기본 필드가 없나요? 설정에서 하나를 정의하세요. 없으면 시스템이 첫 번째 필드를 사용합니다.
- 기본 필드를 제거했나요? 빌더에서 모든 필드가 지워집니다. 수동으로 다시 추가하세요.
- 연락처 아래에 회사가 보이지 않나요? Settings(설정) > Companies(회사)에 해당 회사가 존재하는지 확인하세요.

## 활용 예시

**부동산**: "매물"을 위한 커스텀 객체를 생성하여 위치, 크기, 가격을 저장합니다. 각 매물 레코드는 여러 연락처(임차인, 구매자, 소유자)에 연결될 수 있습니다.

**동물병원**: "반려동물"을 위한 커스텀 객체를 사용합니다. 각 반려동물 레코드는 소유자의 연락처에 연결되고 품종, 의료 기록, 백신 데이터를 포함합니다.

**에이전시나 B2B 설정**: 회사 객체를 사용하여 회사명, 전화번호, 청구 주소 같은 비즈니스 레벨 정보를 저장합니다. 그다음 여러 고객 연락처를 해당 회사에 연결합니다.

## 요약

커스텀 객체를 사용하면 비즈니스 필요에 맞게 CRM 데이터를 구조화할 수 있습니다. 유연하고 확장 가능하며 커스터마이징에 최적화되어 있죠.

회사 객체는 더 간단합니다. 조직과 관련 연락처를 관리하는 고정된 구조를 제공합니다.

두 가지를 함께 사용하여 관계에 대한 완전한 뷰를 얻으세요:

- 전문 데이터를 위한 커스텀 객체
- 비즈니스 레벨 연결을 위한 회사 객체

깔끔하고 연결되며 목적에 맞는 CRM 데이터를 한 곳에서 관리할 수 있습니다.
