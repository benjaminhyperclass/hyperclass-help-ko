# 블로그 변경사항

블로그는 "웹(web)"과 "로그(log)"의 결합으로 탄생한 디지털 무대입니다. 작가들이 매력적인 이야기를 선별하여 독자들에게 다양한 지식과 창의성의 세계로 가는 관문을 제공합니다. 열정과 산문이 만나는 동적인 플랫폼으로, 클릭 한 번으로 평범함을 특별함으로 변화시킵니다.

피드백과 지속적인 개선을 바탕으로, 블로그는 다음과 같은 기능들을 더 간단하게 만들어 관점을 바꿨습니다!

### **1. URL 단순화**

URL에서 b와 c를 제거했습니다. 이제 블로그는 다음과 같은 방식으로 작동합니다.

a) 도메인이 **abc.com**인 경우:

- 사용자는 **abc.com/blog**를 /blog 슬러그로 지정할 수 있습니다. (도메인 페이지가 이미 존재하는 경우, 퍼널/웹사이트의 모든 URL과 도메인을 스캔한 후 고유한 매개변수를 추가합니다)
- 블로그 포스트명은 **abc.com/post/post-name**으로 정의됩니다
- 카테고리 URL은 **abc.com/blog/category/category-name**입니다
- 태그 URL은 **abc.com/blog/tag/tag-name**입니다
- 저자 URL은 **abc.com/blog/author/author-name**입니다

b) 서브도메인이 **blog.abc.com**인 경우:

- 사용자가 **blog.abc.com**을 지정할 수 있으며, 슬러그를 추가해야 하지만 고려되지 않습니다
- 도메인에서 편집으로 이동하여 블로그 홈의 기본 페이지를 추가하세요
- 블로그 포스트명은 **blog.abc.com/post/post-name**으로 정의됩니다
- 카테고리 URL은 **blog.abc.com/category/category-name**입니다
- 태그 URL은 **blog.abc.com/tag/tag-name**입니다
- 저자 URL은 **blog.abc.com/author/author-name**입니다

### **2. 자동 사이트맵**

블로그 포스트가 발행될 때마다 사용자가 사이트맵을 업데이트하거나 robots.txt를 추가할 필요가 없습니다. 도메인의 사이트맵은 블로그 포스트가 발행될 때마다 자동으로 업데이트됩니다.

### 3. 각 블로그 포스트를 새 페이지에서 열기

블로그 포스트 페이지는 새 페이지에서 열리며, 같은 페이지에서 열리지 않습니다. 폼(구독 폼) 또는 커스텀 댓글 영역이나 텍스트/부제/캘린더 같은 기타 커스터마이징으로 모양과 느낌을 맞춤화할 수 있습니다.

### 4. 캐노니컬 태그로 강력한 SEO

SEO에서 캐노니컬 태그는 구글에게 어떤 버전의 페이지를 색인하고, 링크 지분(순위 강도)을 통합하며, 검색 결과에 표시할지 제안합니다.

### 5. 다양한 레이아웃과 디자인으로 커스터마이징

블로그의 카드 뷰에서 표시하고 싶은 내용을 선택할 수 있는 커스텀 레이아웃으로 블로그를 만들 수 있습니다.

### 기존 블로그와 포스트는 어떻게 되나요?

퍼널과 웹사이트에 추가된 기존 블로그는 기존 방식대로 작동하며 사용자가 새 블로그 포스트를 만들 수 있습니다. 기존 블로그는 URL 단순화, SEO 색인 문제를 해결하는 캐노니컬 URL, XML 사이트맵 자동 업데이트와 관련된 문제가 있습니다. 모든 블로그 포스트는 Default 블로그의 일부가 됩니다.

이러한 문제를 해결하려면 새 블로그와 새 블로그 포스트로 이전하는 것이 더 좋고 권장됩니다. 출시 이후 생성되는 새 하위 계정은 새 블로그 플로우로 작동합니다.

### 기존 블로그에서 새 블로그로 이전하는 방법

1. Sites(사이트) > Blogs(블로그)로 이동합니다

2. 블로그에서 퍼널/웹사이트에 호스팅된 Default Blog(기본 블로그) 옵션이 있습니다

3. Default Blog(기본 블로그)에 블로그 포스트가 있습니다

4. 블로그 포스트의 체크박스를 선택합니다

5. 선택한 블로그 포스트에서 "Move Blog(블로그 이동)"을 클릭합니다

참고: 블로그 포스트의 기존 URL이 중단되고 새 URL로 업데이트됩니다!
