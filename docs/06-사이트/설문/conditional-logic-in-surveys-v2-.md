# 설문 조건부 로직 (v2)

조건부 로직 v2는 Hyperclass에서 더 똑똑하고 동적인 설문을 만들 수 있도록 도와줍니다. 응답자가 답변하는 내용에 따라 슬라이드를 보이거나 숨기고, 메시지를 표시하고, 사용자를 다른 페이지로 리다이렉트하고, 응답을 자격 박탈하거나, 슬라이드 간을 점프할 수 있습니다.

이 가이드에서는 설문에서 조건부 로직을 사용하는 방법, v2의 새로운 기능, 지원되는 필드에 대해 설명합니다.

### ⚠️ 중요

조건부 로직 v2는 새로 생성되거나 복제된 설문에서만 사용할 수 있습니다. 계정 스냅샷을 생성할 때 로직 규칙은 이전되지 않습니다.

## 설문 조건부 로직이란

조건부 로직은 사용자 입력에 따라 설문 내부의 액션을 자동화합니다. 다음과 같은 조건을 정의할 수 있습니다:

- 답변이 "아니오"라면 맞춤 메시지를 표시
- 점수가 5점 미만이면 리드를 자격 박탈
- 날짜가 특정 범위 내에 있으면 다른 페이지로 리다이렉트
- 모든 슬라이드를 보여주는 대신 특정 슬라이드로 사용자를 점프

이렇게 하면 설문을 더욱 인터랙티브하고 관련성 있게 만들어 효율성을 높일 수 있습니다.

## v2의 새로운 기능

- 단순화된 3단계 빌더: 필드 선택 → 상태 선택 → 값 제공
- "사이" 연산자: 숫자, 점수, 통화, 날짜 필드용
- 점프 투 분기: 설문 내에서 슬라이드 간 점프 생성
- 확장된 필드 지원: 체크박스, 다중 드롭다운, 약관1/약관2, 파일 업로드, 서명
- 새로운 표시/숨김 대상: 텍스트, HTML, 이미지, 약관 요소
- 규칙 목록 필터링: 필드나 슬라이드별로 규칙 필터링
- 순환 보호: 루프 방지 (슬라이드 A → 슬라이드 B → 슬라이드 A)
- 현지화 및 접근성: 키보드 내비게이션과 번역된 라벨

## 사용 가능한 액션

조건부 로직을 열면 사용 가능한 액션 유형을 볼 수 있습니다:

| 액션 | 설명 |
|------|------|
| 필드 표시/숨김 | 질문이나 슬라이드를 동적으로 표시하거나 숨김 |
| 맞춤 메시지 표시 | 개인화된 안내나 메시지를 표시 |
| URL로 리다이렉트 | 제출 후 응답자를 맞춤 웹페이지로 이동 |
| 리드 자격 박탈 | 부적격 응답을 자동으로 표시 |
| 점프 투 (슬라이드) | 답변에 따라 다른 슬라이드로 건너뛰거나 분기 (설문 전용) |

**규칙은 위에서 아래로 실행됩니다.**
**해당되는 경우 나중 일치가 이전 일치를 덮어씁니다.**

## 설문에서 조건부 로직 설정하기

설문 빌더 내에서 직접 규칙을 추가하고 관리할 수 있습니다.

### 1. 조건부 로직 열기

- 설문 빌더에서 상단 바의 Conditional Logic(조건부 로직)을 클릭합니다.
- 또는 우측 사이드바의 Conditional Logic(조건부 로직) 링크에서 엽니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055505701/original/_r54-LIqjTPUsHKiGGV61zG1YvSEQPIbLA.png?1759919572)

### 2. 새 조건 추가

- Add New Condition(새 조건 추가)을 클릭합니다.
- 액션을 선택합니다 (표시/숨김, 메시지, 리다이렉트, 자격 박탈, 또는 점프 투).
- 3단계 로직 빌더가 열립니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055505810/original/2AklOvUBEagY-NTHR7Q9L1En-vfvVMpX_A.png?1759919604)

### 3. 규칙 만들기

안내된 흐름을 따라하세요:

- Select Field(필드 선택) – 규칙의 기준이 될 설문 필드나 슬라이드를 선택합니다.
- Select State(상태 선택) – 연산자를 선택합니다 (예: 같음, 비어있음).
- Provide Value(값 제공) – 값을 입력합니다 (텍스트, 숫자, 날짜, 옵션).

AND 또는 OR 연결자를 사용하여 같은 규칙에 여러 조건을 추가할 수 있습니다.

- AND = 모든 조건이 참이어야 함
- OR = 어떤 조건이든 참이면 됨

AND와 OR 간 전환 시 블록의 모든 조건이 업데이트됩니다.

일부 연산자는 값이 필요하지 않습니다 (예: 비어있음, 체크됨).

### 4. 액션 결과 선택

선택 사항에 따라:

- Redirect to URL(URL로 리다이렉트): 유효한 링크를 입력합니다 (예: https://example.com)
- Display Message(메시지 표시): 맞춤 메시지 텍스트를 추가합니다
- Show/Hide Fields(필드 표시/숨김): 표시하거나 숨길 필드를 선택합니다
- Disqualify Lead(리드 자격 박탈): 제출을 자동으로 중단합니다
- Jump To(점프 투) (설문 전용): 대상 슬라이드나 질문을 선택합니다

### 5. 규칙 저장

- Save(저장) 버튼은 모든 필수 입력이 완료된 후에만 활성화됩니다.
- 시스템이 누락된 값을 확인하고 루프 로직을 방지합니다.
- 규칙은 목록에서 위에서 아래로 실행됩니다.

## 지원되는 필드

조건부 로직 v2는 다양한 설문 필드를 지원합니다.

### 설문 필드

- 한 줄 입력
- 여러 줄 입력  
- 숫자
- 통화
- 점수
- 날짜 선택기
- 드롭다운 (단일)
- 드롭다운 (다중)
- 라디오 선택
- 체크박스
- 이용약관 (약관1 / 약관2)
- 파일 업로드
- 서명

### 정적 요소

- 텍스트
- HTML
- 이미지

참고: 캘린더 필드는 조건부 로직에서 지원되지 않습니다.

## 필드 유형별 지원 연산자

| 필드 유형 | 연산자 |
|-----------|--------|
| 텍스트 (한 줄/여러 줄) | 같음, 다름, 포함, 포함하지 않음, 시작함, 끝남, 비어있음, 채워짐 |
| 숫자 / 점수 / 통화 | 같음, 다름, 보다 큼, 보다 작음, 사이, 비어있음, 채워짐 |
| 날짜 | 같음, 이전, 이후, 사이, 비어있음, 채워짐 |
| 드롭다운 (단일) / 라디오 | 같음, 다름, 비어있음, 채워짐 |
| 다중 드롭다운 / 체크박스 | {옵션}과 같음, {옵션}과 다름, 비어있음, 채워짐 |
| 약관1 / 약관2 | 체크됨, 체크되지 않음 |
| 파일 업로드 / 서명 | 비어있음, 채워짐 |

## 설문 전용 기능

### 점프 투 로직

- IF 기준: 필드 또는 슬라이드
- THEN 액션: 슬라이드나 질문으로 점프
- 실행 시점: Next(다음) 버튼 클릭 시
- 충돌 처리: 여러 점프 투 규칙이 일치할 경우 첫 번째 규칙이 실행
- 보호: 루프 방지 (슬라이드 A ↔ 슬라이드 B)

### 규칙 필터링

설문에서 더 쉬운 편집을 위해 규칙을 필터링할 수 있습니다:

- 필드별: 특정 필드와 연결된 규칙 표시
- 슬라이드별: 선택된 슬라이드에 영향을 주거나 대상으로 하는 규칙 표시

## 동작 및 평가

- 규칙은 순서대로 위에서 아래로 실행됩니다.
- 리다이렉트 / 메시지 / 자격 박탈: 첫 번째 일치하는 규칙만 실행됩니다.
- 표시/숨김: 나중 규칙이 이전 규칙을 덮어씁니다.
- 점프 투: 슬라이드 전환 시 평가됩니다 (Next 버튼).
- 텍스트 매칭은 대소문자를 구분하지 않으며 여분의 공백을 무시합니다.
- 날짜 로직은 계정 시간대를 사용합니다.

## 팁

- 숫자나 날짜 범위에는 "사이" 연산자를 사용하세요 (포함).
- 오늘, 이번 주 같은 빠른 날짜 프리셋을 활용하세요.
- "라디오 기타"는 맞춤 입력이 활성화된 경우에만 나타납니다.
- 로직과 점프 투 플로우를 검증하기 위해 항상 설문을 미리보기하세요.

## FAQ

1. **같은 규칙에서 AND와 OR을 혼합할 수 있나요?**
   아니요. 각 규칙은 한 가지 유형의 연결자만 사용해야 합니다.

2. **같은 설문에 여러 규칙을 가질 수 있나요?**
   네. 각 규칙은 독립적으로 실행됩니다.

3. **둘 이상의 조건이 일치하면 어떻게 되나요?**
   리다이렉트, 메시지, 자격 박탈 액션의 경우 첫 번째 일치가 우선됩니다.
   표시/숨김의 경우 나중 규칙이 이전 규칙을 덮어씁니다.

4. **점프 투 로직에서 루프를 만들 수 있나요?**
   아니요. 빌더가 루프를 차단하고 감지되면 경고를 표시합니다.

5. **IF 드롭다운에서 필드를 선택할 수 없는 이유는 무엇인가요?**
   각 설문 필드가 고유한 쿼리 키를 가지는지 확인한 후 패널을 새로고침하세요.
