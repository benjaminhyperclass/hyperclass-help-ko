# 블로그 포스트 요소란?

블로그 포스트 요소(Blog Post Element)를 사용하면 발행된 블로그 콘텐츠를 Hyperclass 내의 퍼널 단계나 웹사이트 페이지로 가져올 수 있습니다. 방문자를 메인 블로그 사이트로만 보내는 대신, 특정 페이지 내에 블로그 포스트를 직접 표시하여 SEO를 지원하고 콘텐츠 가시성을 높이며 더 타겟화된 브라우징 경험을 만들 수 있습니다. 최근 게시글을 소개하거나, 특정 카테고리의 포스트를 강조하거나, 특정 작성자가 쓴 콘텐츠를 노출하고 싶을 때 유용합니다.

블로그 포스트 요소는 이미 Hyperclass에서 블로그를 설정하고 최소 하나 이상의 포스트를 발행한 사용자를 위해 설계되었습니다. 페이지에 추가하면, 선택한 정렬이나 필터링 옵션에 따라 블로그 콘텐츠를 표시할 수 있습니다.

## 블로그 포스트 요소 설정하는 방법

- 사이트(Sites) 메뉴로 이동해서 퍼널(Funnels) 또는 웹사이트(Websites)를 열어주세요. 편집하려는 특정 **퍼널 단계** 또는 **웹사이트 페이지**를 선택하고 편집(Edit) 버튼을 클릭하세요.

![퍼널 또는 웹사이트 편집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066833207/original/BfTDm0-B4YPaU1YUrrnxjgpf1yj_B2uTIQ.png?1773350814)

- 요소 추가(Add Elements) 버튼을 클릭하세요.

![요소 추가 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066833286/original/XsN-gV6uYpusoXcVjrwmh_wFa2G3cMabjA.png?1773350934)

- 블로그 포스트(Blog Posts) 요소를 찾아서 페이지의 원하는 위치로 드래그하세요.

![블로그 포스트 요소 드래그](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066833306/original/Io3NUVH61JwjCymDAL8r7vEOtYQWbjN0QA.png?1773350972)

- 요소를 클릭해서 요소 설정을 열어주세요. 여기서 블로그 콘텐츠가 어떻게 표시될지 선택할 수 있습니다:

**표시 스타일(Display Style)**: 페이지에서 요소가 어떻게 보일지 선택합니다.

**블로그 사이트 선택(Choose Blog Site)**: 콘텐츠를 가져올 블로그를 선택합니다.

**정렬 기준(Sort By)**: 어떤 블로그가 먼저 표시될지 선택합니다. *최근 포스트*, *카테고리별 최근 포스트*, *작성자별 최근 포스트* 중에서 선택하세요.

**카테고리 선택(Choose Categories)**: 포함할 카테고리를 선택합니다.

**페이지네이션(Pagination)**: 표시할 블로그 개수를 선택합니다.

**버튼 옵션(Button Options)**: 버튼 텍스트와 색상을 커스터마이징합니다.

![블로그 포스트 요소 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066833446/original/ATLQYBI6-GZXcmhk_4XPqJkXxYJrRHw0Mg.png?1773351375)

- 변경 사항을 **저장(Save)**하고 사이트/퍼널을 **발행(Publish)**하세요.

![저장 및 발행](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066833488/original/sM2MJLAVhL1vlTXuXU9elef5u8opLEBX8w.png?1773351441)

## 자주 묻는 질문

**Q: 요소를 추가한 후 포스트가 나타나지 않으면 어떻게 하나요?**
블로그가 제대로 생성되었는지, 포스트가 발행되었는지, 선택한 카테고리나 작성자 필터가 사용 가능한 콘텐츠와 일치하는지 확인해보세요.

**Q: 이 요소를 사용해서 콘텐츠 기반 랜딩 페이지를 만들 수 있나요?**
네, 가능합니다. 이 요소는 전환 전에 방문자를 교육하기 위해 블로그 콘텐츠를 사용하는 랜딩 페이지나 리소스 페이지를 지원하는 데 도움이 될 수 있습니다.
