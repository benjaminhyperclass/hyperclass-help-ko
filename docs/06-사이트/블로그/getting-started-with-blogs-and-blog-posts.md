# **Hyperclass의 블로그 기능이란?**

Hyperclass 블로그는 웹사이트나 퍼널에 글을 작성하고 정리할 수 있는 내장 퍼블리싱 시스템입니다. 작성자, 카테고리, SEO, 디자인을 관리하여 꾸준히 포스팅하고 적절한 페이지로 트래픽을 유도할 수 있습니다.

## Hyperclass 블로그의 주요 장점

- 통합 퍼블리싱: 모든 포스트, 작성자, 카테고리, SEO를 한 곳에서 관리
- 유연한 배치: 블로그 포스트 목록 요소를 사용하여 웹사이트 페이지나 퍼널 단계 어디든 포스트 표시 가능
- 내장 SEO 필드: 슬러그, 메타 데이터, 표준 URL을 설정하여 검색 노출도 향상
- 디자인 옵션: 목록과 포스트 뷰에 표준형 또는 컴팩트 레이아웃 선택 가능
- 확장 가능한 워크플로우: 라이브러리가 확장되면서 초안 작성, 예약, 발행 또는 대량 가져오기 가능

## Hyperclass에서 블로그 만드는 방법

블로그를 처음 시작하거나 나만의 블로그를 만들고 싶지만 어디서부터 시작해야 할지 모른다면, 다음 단계를 따라해보세요:

### 1단계: 블로그 사이트(컨테이너) 만들기

- Sites(사이트) 메뉴로 이동
- Blogs(블로그) 클릭
- Create Blog(블로그 생성) 클릭하여 처음부터 만들거나 템플릿에서 선택

![블로그 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064779424/original/OiYF3mABKqw7rZqW5H23dRkDcSvWob6htg.jpeg?1770900245)

### 2단계: 블로그 사이트 세부사항 추가

- 블로그 **사이트 제목** 추가 (브라우저 탭 제목과 검색 결과에 표시됨)
- **도메인** 선택 또는 추가/편집
- **슬러그** 추가
- **블로그 메타 설명** 추가 (선택 사항이지만 권장)

![블로그 사이트 세부사항](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064779485/original/M-l1QTgtlbe4xtiEOCEf57_wpanMIKzu1A.jpeg?1770900284)

### 3단계: 블로그 포스트 만들기

- Sites(사이트) → Blogs(블로그)로 이동
- 목록에서 방금 만든 블로그 사이트를 클릭하거나 블로그 포스트를 추가하려는 기존 블로그 사이트 클릭

![블로그 포스트 만들기](https://jumpshare.com/share/W1ZseR5HZIB5gqCKFqVA+/GIF+Recording+2026-02-12+at+6.22.19+PM.gif)

- New Post(새 포스트) 클릭
- + New Post(새 포스트)를 클릭하여 처음부터 블로그 포스트 생성. [URL에서 블로그 포스트 가져오기](blogs-import-blog-posts-using-url.md)나 CSV에서 블로그 포스트 가져오기도 가능
- 처음부터 블로그 포스트를 만들거나 편집하려면 [**블로그 포스트 만들기**](../create-blog-posts.md) 문서를 확인하세요

![새 포스트](https://jumpshare.com/share/lauN1KBkpiMjO4w8pQju+/Screen+Shot+2026-02-12+at+6.23.44+PM.png)

### 4단계: 블로그 포스트 세부사항 추가

- 편집기에서 블로그 포스트의 콘텐츠 추가 및 서식 설정을 완료한 후 Continue(계속) 클릭

![포스트 내용 완료](https://jumpshare.com/share/ODj6plhHTcGJP049Rix9+/Screen+Shot+2026-02-12+at+6.33.01+PM.png)

- 블로그 포스트 URL 슬러그 추가

![URL 슬러그](https://jumpshare.com/share/2qSl2zO7wamgKp1EYRyN+/Screen+Shot+2026-02-12+at+6.38.18+PM.png)

- 카테고리와 **작성자** 선택 또는 생성

![카테고리와 작성자](https://jumpshare.com/share/TRJGa96erSfwYce2526S+/GIF+Recording+2026-02-12+at+6.40.37+PM.gif)

- [표준 링크](../canonical-urls-explained-boost-your-highlevel-blog-seo.md) 편집 또는 그대로 유지
- 키워드 추가

![키워드 추가](https://jumpshare.com/share/yIPdQ9TzuhrBXv0OwOUn+/Screen+Shot+2026-02-12+at+6.43.37+PM.png)

### 5단계: SEO 세부사항 추가

- SEO 제목 추가
- 포스트 설명 추가
- 커버 **이미지**와 **대체 텍스트** 추가

![SEO 세부사항](https://jumpshare.com/share/JAMVKS0xLVx9C9PIGEAX+/Screen+Shot+2026-02-12+at+6.47.35+PM.png)

### 6단계: 블로그 포스트 발행

- 추후에 추가 편집을 하려면 포스트를 **Draft(초안)** 상태로 유지
- 포스트를 즉시 **Publish(발행)**하거나 향후 시간으로 **Schedule(예약)** 가능

참고: **퍼널이나 웹사이트에 블로그를 추가**하려면 자세한 [**가이드**](add-blog-post-element-in-funnel-step-or-website-page.md)를 확인하세요.

![포스트 발행](https://jumpshare.com/share/NUDN2nvuLcneDX3T7hnt+/GIF+Recording+2026-02-12+at+6.51.58+PM.gif)

## **발행된 블로그 포스트나 초안의 세부사항 업데이트하기**

- 블로그 포스트로 이동하여 포스트 옆의 **점 세 개**를 클릭한 후 **Update Post(포스트 업데이트)** 클릭
- 편집기에서 블로그 포스트 내용 편집
- **작성자**, **카테고리**, **슬러그**, **SEO** 등 **블로그 포스트 세부사항을 변경**하려면 **Update Settings(설정 업데이트)** 버튼 클릭
- 필요한 업데이트를 하고 **Publish(발행)** 클릭

## **자주 묻는 질문**

**Q: 블로그에서 도메인은 어떻게 작동하나요?**

도메인은 블로그 목록 요소가 추가된 퍼널이나 웹사이트에 추가됩니다.

블로그 URL 형식:
| 도메인 | 블로그 포스트 슬러그 | 최종 URL |
|--------|---------------------|-----------|
| abc.com | b/sample-now | abc.com/b/sample-now |
| abc.com/xyz | b/sample-now | abc.com/xyz/b/sample-now |
| blog.abc.com | b/sample-now | blog.abc.com/b/sample-now |

**Q: 각 카테고리별로 별도의 블로그 페이지를 만들 수 있나요?**

현재는 지원되지 않습니다. 향후 업데이트에서 카테고리 필터를 기반으로 슬러그와 SEO 세부사항이 포함된 퍼널/웹사이트 페이지를 만들 수 있도록 할 예정입니다.

**Q: 작성자, 카테고리, 키워드 추가가 선택 사항인가요?**

URL 슬러그, 카테고리, 작성자, 제목, 포스트 설명은 필수 항목입니다.

**Q: 블로그 포스트에서 커스텀 값을 사용할 수 있나요?**

아니요, 현재는 블로그 포스트에서 커스텀 값을 지원하지 않습니다. {{contact.name}}이나 {{custom_values.xyz}} 같은 값은 사용할 수 없습니다.

**Q: 블로그 포스트에 폼, 설문, 커스텀 코드를 어떻게 추가하나요?**

현재는 블로그 포스트에 폼이나 설문을 추가할 수 없습니다. 커스텀 코드 추가 옵션은 곧 출시될 예정입니다.

**Q: 블로그 포스트에 권장되는 이미지 크기가 있나요?**

권장 크기는 따로 없습니다. 다만 블로그 포스트 편집기 내부의 페이지 너비는 736px입니다.
