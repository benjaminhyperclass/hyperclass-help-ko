# 블로그 및 블로그 포스트 상태란?

Hyperclass의 블로그와 블로그 포스트 상태는 두 가지 레벨에서 가시성과 발행 상태를 나타냅니다: 블로그(사이트 레벨 컨테이너)와 개별 블로그 포스트입니다. 이러한 상태를 알면 실수로 발행하는 것을 방지하고 초안 작성, 발행, 예약, 보관, 삭제를 통해 콘텐츠를 관리할 수 있습니다.

## 블로그 및 블로그 포스트 상태의 주요 장점

- **명확성:** 블로그나 포스트의 현재 상태를 명확한 라벨로 표시
- **제어:** 콘텐츠를 초안, 발행됨, 예약됨, 보관됨, 삭제됨 상태 간에 쉽게 이동
- **타이밍:** 내장된 예약 기능으로 콘텐츠 공개를 캘린더와 일치시킴
- **정리:** 보관 또는 초안으로 설정하여 작업 중인 콘텐츠와 실제 게시된 콘텐츠를 구분

## 블로그 상태 (사이트 레벨)

블로그 상태는 전체 블로그가 방문자에게 보이는지를 결정합니다. 이는 개별 포스트 상태와는 별개입니다.

- **초안(Draft):** 블로그가 존재하지만 공개적으로 보이지 않습니다. 구조, 디자인, 설정을 구성하는 동안 사용합니다.
- **라이브(Live):** 블로그가 공개적으로 보입니다. 블로그 내의 발행된 포스트는 각자의 상태에 따라 접근할 수 있습니다.

- **삭제(Delete):** 블로그가 영구적으로 제거됩니다. 콘텐츠가 더 이상 필요하지 않을 때 신중하게 사용하세요.

## 블로그 포스트 상태 (포스트 레벨)

포스트 상태는 개별 글의 가시성과 타이밍을 제어합니다. 상위 블로그가 발행됨 상태일 때도 적용됩니다.

- **초안(Draft):** 포스트가 저장되었지만 공개되지 않습니다. 실제 사이트에 노출하지 않고 계속 작성하고 편집할 때 사용합니다.
- **예약됨(Scheduled/Queued):** 포스트가 지정된 날짜와 시간에 자동으로 발행되도록 설정됩니다. 계획된 캘린더에 따라 콘텐츠 공개를 관리합니다.
- **발행됨(Published):** 포스트가 라이브 상태이며 방문자에게 보입니다. 이미 발행된 포스트를 변경할 때는 **Update Post**를 사용합니다.

![포스트 상태 옵션](https://jumpshare.com/share/vc8u1AcSmCXbwzkh6alg+/Screen+Shot+2025-12-24+at+4.46.37+PM.png)

- **보관됨(Archive):** 포스트가 목록과 네비게이션에서 제거되지만 삭제되지는 않습니다. 콘텐츠는 여전히 라이브 URL에서 접근할 수 있습니다.
- **삭제(Delete):** 포스트가 영구적으로 제거됩니다. 콘텐츠가 더 이상 필요하지 않을 때 신중하게 사용하세요.

## 자주 묻는 질문

**Q: 포스트를 라이브로 만들지 않고 계속 작업할 수 있나요?**
네. 준비될 때까지 포스트를 **초안**으로 저장하세요.

**Q: 포스트를 보관하면 어떻게 되나요?**
포스트가 목록과 네비게이션에서 숨겨지지만 라이브 URL에서는 여전히 접근할 수 있습니다.

**Q: 포스트가 나중에 라이브되도록 시간을 설정하려면 어떻게 하나요?**
블로그를 미래 날짜와 시간으로 예약하세요.

**Q: 라이브 포스트를 편집해야 한다면?**
포스트를 편집하고 **Update Post**를 선택하여 **발행됨** 포스트에 변경사항을 적용하세요.

**Q: 언제 포스트를 삭제해야 하나요?**
콘텐츠가 더 이상 필요하지 않을 때만 삭제하세요. 삭제된 포스트는 영구적으로 제거됩니다.
