# **블로그 포스트 가져오기 도구란?**

블로그 포스트 가져오기 도구는 외부 웹사이트의 홈페이지 URL을 스캔하여 사용 가능한 포스트를 감지하고, Hyperclass로 블로그 포스트를 마이그레이션하는 기능입니다. 블로그 마이그레이션을 간소화하고, 수동 복사-붙여넣기 작업을 줄이며, 전체 과정에서 더 나은 가시성을 제공하여 Hyperclass로 콘텐츠를 쉽게 이동할 수 있도록 설계되었습니다.

## **블로그 포스트 가져오기 도구의 주요 장점**

- **안내식 가져오기 흐름**: 스캔 → 검토 → 매핑 → 가져오기의 명확한 과정을 통해 진행할 수 있습니다.

- **향상된 마이그레이션 제어**: 감지된 포스트를 블로그로 가져오기 전에 검토할 수 있습니다.

- **수동 메타데이터 매핑**: 가져오기 과정에서 누락된 작성자, 카테고리, 태그를 추가할 수 있습니다.

- **유연한 가져오기 결과**: 매핑된 포스트와 매핑되지 않은 포스트 모두 차단 없이 가져올 수 있습니다.

- **빠른 게시 준비**: 매핑된 포스트는 발행된 상태로 가져올 수 있고, 매핑되지 않은 포스트는 후속 작업을 위해 초안으로 저장됩니다.

## **URL을 사용하여 블로그 포스트를 가져오는 방법**

### **1단계: 블로그 목록 열기**

- `Sites(사이트) → Blogs(블로그)`로 이동하여 블로그 목록을 확인하세요.

- **기존 블로그**를 클릭하거나 [블로그를 새로 생성](create-blog-posts.md)하세요.

![](https://jumpshare.com/share/43G7UZ1hz9B3j3DKsOga+/Screen+Shot+2026-03-11+at+19.37.44.png)

### **2단계: 블로그 포스트 가져오기 도구 열기**

- 기존 블로그 또는 새 블로그의 **블로그 포스트 목록**으로 이동하세요.

- `New Post(새 포스트) → Import Blog Posts from URL(URL에서 블로그 포스트 가져오기)`를 클릭하세요.

![](https://jumpshare.com/share/bUz6DJ4prqCWzRlYTwRo+/GIF+Recording+2026-03-11+at+19.44.15.gif)

### **3단계: 가져오기 시작**

- **블로그 홈페이지** URL을 입력하세요.

- **최근 포스트 URL**을 추가하세요.

- **플랫폼**을 선택하세요.

- **Continue(계속)**를 클릭하면 매핑 완료를 기다려야 한다는 메시지가 표시됩니다.

![](https://jumpshare.com/share/cePF5mvLLheYwuZikCCU+/Screen+Shot+2026-03-11+at+19.49.10.png)

### **4단계: 콘텐츠 매핑**

**가져오기 시간** 1,000개 포스트 기준 약 10-20분이 소요됩니다.

**스캔/결과 오류**
블로그 사이트맵에 URL이 없으면 블로그 포스트를 가져올 수 없습니다. 현재 실패 상태의 경우 '스캔 진행 중'이 표시되며, 클릭하면 빈 페이지가 나타납니다.

- 콘텐츠 매핑: 지시사항에 따라 페이지의 어떤 요소가 블로그 포스트의 어떤 정보에 해당하는지 식별하세요.

- 모든 **체크 표시**가 **녹색**이 되면 모든 매핑이 완료된 것을 확인할 수 있습니다.

![](https://jumpshare.com/share/oqpbM800rIPkOA8QzkIV+/GIF+Recording+2026-03-11+at+20.00.17.gif)

- 가져오기가 완료되면 "날짜에 X개의 포스트를 가져왔습니다"라는 메시지가 표시됩니다.

![](https://jumpshare.com/share/1rpHqWbvTeKPgB8DURYF+/Screen+Shot+2026-03-11+at+20.18.06.png)

### **5단계: 포스트 게시**

- 블로그에서 가져온 포스트를 검토하세요:

완전히 매핑된 포스트는 발행됨 상태로 가져옵니다.

- 매핑되지 않은 포스트는 **초안** 상태로 가져옵니다.

- **가져오려는 블로그 포스트**를 선택하세요.

- **Import Selected(선택 항목 가져오기)**를 클릭하세요.

![](https://jumpshare.com/share/Q7ZT6xifO8jUyrGY0FER+/Screen+Shot+2026-03-11+at+20.22.36.png)

## **자주 묻는 질문**

**Q: 가져온 모든 블로그 포스트가 초안 상태로 남나요?**
아니요. 매핑된 포스트는 **발행됨** 상태로 가져오고, 매핑되지 않은 포스트는 **초안** 상태로 가져옵니다.

**Q: 일부 메타데이터가 누락된 경우에도 포스트를 가져올 수 있나요?**
네. 가져오기 과정에서 누락된 작성자, 카테고리, 태그를 수동으로 추가하거나 매핑할 수 있습니다.

**Q: 일부 포스트는 매핑되고 다른 포스트는 매핑되지 않은 경우 어떻게 되나요?**
두 유형 모두 가져올 수 있습니다. 매핑이 완료된 포스트는 발행된 상태로 가져오고, 매핑이 불완전한 포스트는 초안으로 가져옵니다.

**Q: 블로그를 가져올 때 SEO에 어떤 영향이 있나요? SEO 권위나 순위가 자동으로 이전되나요?**
가져오기 도구는 제목, 설명 등 사용 가능한 SEO 메타데이터와 함께 포스트 콘텐츠를 마이그레이션합니다. 그러나 단순히 콘텐츠를 가져오는 것만으로는 SEO 권위가 자동으로 이전되지 않습니다. 순위와 링크 권한을 보존하려면 적절한 **301 리다이렉트**를 설정하고 가능한 한 동일한 **도메인과 URL 구조**를 유지해야 합니다. 리다이렉트 없이는 검색 엔진이 가져온 포스트를 새로운 페이지로 간주할 수 있습니다.

**Q: 가져오기 후 카테고리, 작성자, 태그를 수동으로 다시 생성해야 하나요?**
항상 그런 것은 아닙니다. 가져오기 중에 메타데이터가 누락된 경우, 매핑 단계에서 추가하거나 매핑할 수 있습니다.
