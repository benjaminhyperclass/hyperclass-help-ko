# 스냅샷을 통한 하위 계정 간 블로그 콘텐츠 공유

여러 하위 계정(Sub-account)에서 블로그(Blog)와 블로그 게시물 콘텐츠를 재사용하는 것이 훨씬 쉬워졌습니다! 스냅샷(Snapshot)의 강력한 기능을 통해 에이전시 설정에서 블로그, 블로그 게시물, 작성자, 카테고리를 모든 하위 계정에 공유할 수 있습니다.

## 원본 하위 계정에서는 어떻게 작동하나요?

원본 하위 계정은 스냅샷의 원점입니다. 스냅샷이 생성되고 파생되는 하위 계정입니다.

- `Sites(사이트)` 탭 > `Blog(블로그)`로 이동하세요
- New Blog(새 블로그) 생성 > 블로그 이름, 이미지, 간단한 설명 등 SEO 메타 정보를 추가하고 도메인을 연결하세요
- 디자인과 스타일을 원하는 대로 블로그 사이트를 편집하세요
- 카테고리와 작성자와 함께 New Blog Post(새 블로그 게시물) 생성 - 다른 위치로 전송하려면 블로그 게시물을 발행하세요

## 에이전시 설정 > 스냅샷에서는 어떻게 작동하나요?

- Agency view(에이전시 보기) > Snapshots(스냅샷)로 이동하세요

- 'Refresh the snapshot(스냅샷 새로고침)' ([가이드 보기](refresh-or-update-snapshots.md)) 또는 'upload the snapshot(스냅샷 업로드)' ([가이드 보기](pushing-loading-snapshot-updates-to-client-accounts.md))

- 확장하면 모든 블로그 이름이 선택 가능한 옵션으로 표시됩니다.

## 대상 하위 계정에서는 어떻게 작동하나요?

대상 하위 계정은 스냅샷을 업로드하려는 계정입니다.

- Sites(사이트) > Blogs(블로그)로 이동하세요

- 블로그를 확인하면 Draft(초안) 상태로 표시됩니다.

참고 - 초안 상태는 사용자가 도메인 정보를 추가해야 한다는 의미입니다.

- 사용자가 모든 블로그 도메인 정보를 추가하면 모든 블로그를 발행(Published) 상태로 만들 수 있습니다.
