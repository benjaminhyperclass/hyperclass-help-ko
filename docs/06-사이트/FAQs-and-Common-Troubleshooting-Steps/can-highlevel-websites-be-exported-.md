# Hyperclass 웹사이트를 내보낼 수 있나요?

Hyperclass 웹사이트는 전용 페이지 빌더(Page Builder)로 제작되며, 다른 플랫폼에서 사용하기 위해 웹사이트를 내보내는 공식적인 방법을 지원하지 않습니다.

이 문서에서는 고객이 Hyperclass 웹사이트를 내보내려고 할 때 지원되는 것과 지원되지 않는 것을 설명합니다.

## **Hyperclass 웹사이트를 내보낼 수 있나요?**

아니요. Hyperclass는 웹사이트를 Hyperclass 플랫폼 외부에서 관리하거나 편집할 수 있는 방식으로 내보내는 기능을 지원하지 않습니다.

지원되는 1:1 웹사이트 내보내기 기능은 없습니다.

## **웹사이트 HTML 복사에 대해**

기술적 관점에서는 웹사이트의 렌더링된 HTML 출력을 복사해서 다른 곳에 호스팅할 수 있습니다.

Hyperclass 웹사이트에서 이를 수행할 경우:

- 복사된 페이지는 더 이상 Hyperclass에서 관리되지 않음
- 페이지를 Hyperclass 페이지 빌더를 통해 편집할 수 없음
- Hyperclass는 외부 호스팅된 복사본에 대해 지원을 제공하지 않음
- 지속적인 기능은 원본 Hyperclass 로케이션(Location)과 에셋이 활성 상태로 유지되는 것에 의존함

이는 지원되는 웹사이트 내보내기로 간주되지 않습니다.

## **페이지 HTML 복사 방법 (미지원)**

Hyperclass 페이지의 렌더링된 HTML을 복사하기로 선택한다면, 웹 브라우저에서 직접 수행할 수 있습니다.

예시 방법:

- 발행된 페이지를 브라우저에서 열기
- 페이지 아무 곳이나 마우스 오른쪽 버튼을 클릭하고 '페이지 소스 보기'(또는 유사한 메뉴) 선택
- HTML 콘텐츠를 복사하여 .html 파일로 저장

이 방법은 페이지의 렌더링된 출력만 복사합니다.

Hyperclass는 외부 호스팅된 HTML 파일의 복사, 호스팅 또는 유지 관리를 위한 도구, 가이드 또는 지원을 제공하지 않습니다.

---

## 복사된 HTML의 활용 용도

복사된 HTML은 다음 용도로 사용할 수 있습니다:

- 콘텐츠 참조나 복구
- 다른 플랫폼에서 웹사이트를 수동으로 재구축
- 기록 보관이나 아카이브 목적
