# Google Analytics(GA4) 디버깅 체크리스트

## 목적

이 체크리스트는 지원팀에 문의하기 전에 퍼널 페이지에서 Google Analytics(GA4) 태그와 이벤트가 올바르게 설치되어 데이터를 전송하고 있는지 확인하는 데 도움을 줍니다.

## 1. Google 태그 설치 여부 확인

**1단계:** Google Chrome에서 퍼널 페이지를 엽니다.

**2단계:** 페이지에서 마우스 우클릭 후 '검사'를 클릭합니다.

**3단계:** Network 탭으로 이동합니다.

**4단계:** 페이지를 새로고침합니다.

**5단계:** 검색/필터 바에 다음을 입력합니다:

- gtag/js
- collect

다음과 같은 요청을 찾아보세요:

- google-analytics.com/collect
- google.com/ccm/collect

**이 요청들이 보인다면:** Google 태그가 설치되어 데이터를 전송하고 있습니다.

**이 요청들이 보이지 않는다면:** 태그가 올바르게 추가되지 않았거나 차단되었을 수 있습니다.

## 2. Google Tag Assistant 사용하기 (권장)

**1단계:** Google Tag Assistant로 이동합니다.

**2단계:** 퍼널 도메인을 입력합니다.

**3단계:** 디버그 세션을 시작합니다.

**4단계:** 퍼널 페이지를 새로고침합니다.

다음을 확인하세요:

- page_view 이벤트
- 예상되는 커스텀 이벤트(예: 폼 제출, 버튼 클릭)

**이벤트가 나타난다면:** 태그가 페이지에서 올바르게 실행되고 있습니다.

**이벤트가 나타나지 않는다면:** 태그가 올바르게 구성되지 않았을 수 있습니다.

## 3. GA4 실시간 보고서에서 데이터 확인

**1단계:** Google Analytics 계정에 로그인합니다.

**2단계:** 올바른 GA4 속성을 선택합니다.

**3단계:** 보고서 → 실시간으로 이동합니다.

**4단계:** 퍼널 페이지를 방문하거나 이벤트를 트리거합니다.

**실시간에서 방문이나 이벤트가 보인다면:** 데이터가 GA4에 성공적으로 도달하고 있습니다.

**보이지 않는다면:** 다음과 같은 이유가 가능합니다:

- 잘못된 측정 ID
- 다른 도메인에 태그 설치
- 속성 불일치
- 트래픽을 차단하는 데이터 필터

## 4. 브라우저 오류 확인

검사 → Console을 열고 다음과 같은 오류를 찾아보세요:

- gtag is not defined
- Script blocked errors
- Content Security Policy(CSP) errors

오류가 보인다면 추적이 올바르게 작동하지 않을 수 있습니다.

## 지원팀에 문의해야 하는 경우

다음 상황에서 지원팀에 문의하세요:

- 태그가 설치된 것으로 보이지만 이벤트가 실행되지 않는 경우
- Tag Assistant에서는 이벤트가 실행되지만 GA4에서는 나타나지 않는 경우
- 해결할 수 없는 스크립트 또는 차단 오류가 보이는 경우
