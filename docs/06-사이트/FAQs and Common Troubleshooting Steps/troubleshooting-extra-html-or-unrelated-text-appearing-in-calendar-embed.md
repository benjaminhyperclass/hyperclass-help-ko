# 문제 해결: 캘린더 임베드에 불필요한 HTML이나 관련 없는 텍스트가 나타나는 경우

## 개요

일부 사용자는 캘린더를 웹사이트나 퍼널에 임베드할 때 관련 없는 텍스트(예: "Marketing Agency", "Demo Reviews", "Free Trial Offers" 같은 서드파티 참조)가 나타나는 것을 볼 수 있습니다. 이는 SEO, 사이트 신뢰도, 스크립트 보안에 대한 우려를 불러일으킬 수 있습니다.

대부분의 경우 이런 추가 텍스트는 캘린더 임베드 코드에서 나오는 것이 아닙니다. 대신 임베드가 배치된 페이지의 일부입니다.

## 실제로 일어나는 일

- 캘린더 임베드 스크립트는 깨끗하며 서드파티 텍스트나 링크를 삽입하지 않습니다.

- 퍼널이나 웹사이트에서 추가 텍스트(리뷰나 관련 없는 비즈니스 이름 등)는 보통 페이지 빌더 콘텐츠 내의 숨겨진 블록입니다.

- 외부 사이트(예: 워드프레스)에서는 이런 텍스트가 페이지에 이미 있던 관련 없는 요소에서 나올 수 있습니다.

- 이런 요소들은 실수로 추가되었거나, 템플릿에서 남겨진 것이거나, 스타일링(예: display: none)으로 숨겨진 것일 수 있습니다.

## 확인 방법

- `캘린더 설정(Calendar Settings) → 임베드 코드(Embed Code)`에서 임베드 코드를 직접 복사하세요.

- 메모장 같은 일반 텍스트 에디터에 붙여넣기하세요.

예상되는 `<iframe>` 또는 `<script>` 캘린더 임베드만 보여야 합니다.

- 여기서 관련 없는 텍스트가 나타나면 지원팀에 문의하세요(드문 경우입니다).

- 임베드 코드에서 여는 괄호나 닫는 괄호가 누락되지 않았는지 다시 확인하세요:

흔한 실수의 예:

```
<script>
  ... 코드 줄들 ...
</script
```

여기서 닫는 `>`가 누락되었습니다. 마찬가지로 때때로 시작하는 `<`가 빠질 수도 있습니다.

- 이를 발견하면 괄호를 수정하고 코드를 다시 임베드하세요.

- 캘린더가 추가된 웹페이지를 검사하세요:

퍼널/웹사이트의 경우: 빌더를 열고 숨겨진 섹션이나 사용하지 않는 섹션이 있는지 확인하세요.

- 외부 사이트의 경우: 사이트 편집기나 브라우저 요소 검사 도구(우클릭 → 검사)를 사용해서 페이지 콘텐츠를 검토하세요.

## 해결 방법

- **퍼널/웹사이트를 사용하는 경우:**

캘린더가 임베드된 페이지로 이동하세요.

- 숨겨진 블록이나 사용하지 않는 블록을 찾아보세요.

- 관련 없는 텍스트를 삭제하거나 편집하세요.

- **요소를 찾을 수 없는 경우:**

편집기 내에서 검색(Ctrl+F 또는 Cmd+F)을 사용해서 "trial", "review", "demo" 같은 키워드를 찾아보세요.

- 여전히 불분명하면 다음과 함께 지원팀에 문의하세요:

문제가 나타나는 페이지 URL

- 수행한 단계를 보여주는 Loom 비디오

## 핵심 정리

캘린더 임베드 코드 자체는 외부 텍스트를 추가하거나 SEO에 해를 끼치지 않습니다. 예상치 못한 콘텐츠는 임베드된 페이지에서 오는 것이며 빌더나 사이트 편집기에서 직접 제거할 수 있습니다. 임베드 코드가 모든 괄호를 올바르게 포함하여 완전히 복사되었는지 항상 확인하세요.
