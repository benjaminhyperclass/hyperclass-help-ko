# **워드프레스 사이트 복제란?**

워드프레스 사이트 복제는 기존 사이트를 복사해서 새롭고 완전히 독립적인 사이트를 만드는 과정입니다. 이 기능은 원본 사이트의 설정, 테마, 플러그인, 콘텐츠를 모두 그대로 보존합니다. 복제는 비슷한 레이아웃, 디자인, 설정을 가진 여러 사이트를 만들 때 특히 유용합니다.

---

# **워드프레스 사이트를 복제하는 이유**

- **효율성**: 처음부터 새로 만드는 대신 기존 사이트 설정을 재사용해서 시간을 절약할 수 있습니다.

- **일관성**: 여러 사이트에서 테마, 플러그인, 설정의 통일성을 보장할 수 있습니다.

- **유연성**: 서로 독립적인 사이트를 만들어서 한 사이트의 변경이 다른 사이트에 영향을 주지 않습니다.

- **편의성**: 업데이트 테스트나 개발 환경 구축 같은 복잡한 작업을 간소화합니다.

---

# **복제 기능의 역할별 접근 권한**

보안을 위해 복제 기능에 대한 접근 권한은 사용자 역할에 따라 제한됩니다. 다음 역할을 가진 사용자만 워드프레스 사이트를 복제할 수 있습니다:

- 에이전시 소유자
- 에이전시 사용자  
- 계정 사용자 (동일한 하위 계정 내에서만)

필요한 권한이 없는 사용자는 처음부터 새 사이트만 만들 수 있습니다. 이를 통해 권한이 있는 사용자만 사이트를 복제할 수 있도록 보장합니다.

---

# **Hyperclass에서 워드프레스 사이트 복제하는 방법**

1. **워드프레스 대시보드로 이동**:
   Hyperclass 계정에 로그인한 후 **워드프레스 대시보드**로 이동합니다.

2. **Create New(새로 만들기)** 또는 **Select Blank WordPress(빈 워드프레스 선택)**을 클릭합니다.

3. **복제 옵션 접근**:
   **Incomplete Setup Card(미완료 설정 카드)**를 클릭합니다.

4. **Choose From Existing Site(기존 사이트에서 선택)**을 선택합니다.

5. **원본 사이트와 대상 선택**:
   - 새 사이트가 위치할 **Sub-Account(하위 계정)**를 선택합니다.
   - 복제하고 싶은 **Source Site(원본 사이트)**를 선택합니다.
   - **New Website Name(새 웹사이트 이름)**을 입력합니다.

6. **복제 시작**:
   **Clone(복제)** 버튼을 클릭해서 복제 과정을 시작합니다.

7. **복제 진행 상황 모니터링**:
   진행 표시기가 복제 작업의 실시간 상태를 보여줍니다.

8. **새 사이트 확인**:
   - 복제가 완료되면 새 사이트가 워드프레스 대시보드에 독립적인 개체로 나타납니다.
   - 복제된 사이트에 대한 모든 변경 사항은 원본에 영향을 주지 않습니다.

# **자주 묻는 질문**

**Q: 누가 복제 기능을 사용할 수 있나요?**
특정 역할(에이전시 소유자, 에이전시 사용자, 동일한 하위 계정 내의 계정 사용자)을 가진 사용자만 복제 기능에 접근할 수 있습니다.

**Q: 필요한 권한이 없으면 어떻게 되나요?**
복제 권한이 없는 역할인 경우, 시스템이 자동으로 새 사이트 생성 과정으로 전환됩니다.

**Q: 한 사이트의 변경사항이 다른 사이트에도 영향을 주나요?**
아니요, 복제된 사이트는 원본과 완전히 독립적입니다. 한 사이트의 수정이 다른 사이트에 영향을 주지 않습니다.

**Q: 복제 과정을 추적할 수 있나요?**
네, 복제 과정 중에 진행 표시기가 실시간 업데이트를 보여줍니다.

**Q: 복제 시 무엇이 보존되나요?**
원본 사이트의 모든 설정, 테마, 플러그인, 세팅이 복제된 사이트에 그대로 보존됩니다.
