# 사이트 감사: 웹사이트의 테크니컬 SEO 모니터링, 분석 및 최적화

Hyperclass SEO의 사이트 프로젝트는 웹사이트(도메인)의 상태와 성능을 추적하는 중심 허브입니다. 이 기능은 웹사이트의 권위도, 검색 트래픽, 백링크, 그리고 기술적 상태에 대한 데이터를 한 곳에서 통합적으로 제공합니다. 이 가이드에서는 사이트 프로젝트를 생성하고 사용하는 방법을 다루고, 표시되는 지표들을 설명하며, Hyperclass을 사용하여 웹사이트의 SEO를 분석하는 단계별 가이드를 제공하겠습니다. 또한 Google Search Console(구글 서치 콘솔) 데이터를 활용하는 방법과 사이트 모니터링 및 감사에서 크레딧이 어떻게 사용되는지도 설명하겠습니다.

## 사이트 프로젝트란?

사이트 프로젝트는 모니터링하고 최적화하려는 웹사이트(또는 특정 도메인/서브도메인)를 나타냅니다. 사이트 프로젝트를 추가하면 Hyperclass의 SEO 도구가 다음 작업을 수행합니다:

- 사이트의 도메인 파워를 가져오거나 계산합니다
- 사이트가 보유한 백링크 수(및 참조 도메인)를 표시합니다
- 오가닉 트래픽 동향(검색을 통한 월간 예상 방문자 수)을 표시합니다
- 사이트 상태 백분율 또는 점수를 제공합니다 – 사이트 감사를 통한 기술적 SEO 상태 요약
- 발견된 SEO 이슈의 수를 나열합니다(감사가 실행된 경우)
- (Google Search Console이 연결된 경우) 총 클릭 수나 키워드와 같은 실제 검색 성능 데이터를 표시합니다

Hyperclass 대시보드의 사이트(Sites) > SEO > 사이트 프로젝트에서 각 프로젝트는 일반적으로 위의 열들과 함께 행으로 나열됩니다. 이를 통해 모니터링되는 각 사이트의 성과를 한눈에 파악할 수 있습니다.

**예시:** "example.com"을 사이트 프로젝트로 추가한 후, 다음과 같은 정보를 볼 수 있습니다:
도메인 파워 45, 백링크 1.2K, 오가닉 트래픽 월 ~5,000회, 사이트 상태 82%, 이슈 10개. 이는 example.com이 적당한 권위도, 어느 정도의 트래픽, 그리고 수정할 몇 가지 이슈가 있는 괜찮은 기술적 상태 점수를 가지고 있음을 의미합니다.

## 새 사이트 프로젝트 추가하기 (단계별 가이드)

1. **프로젝트 생성:**
   사이트 프로젝트 탭에서 "New Site Project(새 사이트 프로젝트)"를 클릭합니다(버튼 레이블은 "Add Site" 또는 유사할 수 있습니다). 양식에서 사이트의 도메인 또는 URL을 입력하라고 합니다. 웹사이트의 홈페이지 URL(예: https://www.example.com)을 입력합니다.

2. **GSC 연결 (선택사항이지만 권장):**
   Google Search Console 데이터를 가져오려면 Google 계정을 연결해야 합니다. "Connect Google Search Console(Google Search Console 연결)"을 클릭합니다(표시되는 경우). Google 계정에 로그인하고 액세스를 허용합니다. 그런 다음 도메인에 대한 올바른 속성을 선택합니다(GSC에서 해당 사이트가 확인되어 있는지 확인하세요).
   이 단계를 건너뛸 수 있으며, 여전히 프로젝트를 추가할 수 있습니다. 단지 GSC 성능 데이터를 볼 수 없을 뿐입니다. (나중에 설정 탭이나 고급 SEO 인터페이스의 GSC 모듈을 통해 언제든지 GSC를 연결할 수 있습니다.)

3. **프로젝트 초기화:**
   추가 후, 시스템이 초기 데이터를 가져오는 데 시간이 걸릴 수 있습니다. 그런 다음 사이트 프로젝트 표에 새 사이트가 나열되는 것을 볼 수 있습니다. 처음에는 데이터가 여전히 로딩 중이거나 감사가 아직 실행되지 않은 경우 일부 지표가 "–" 또는 0으로 표시될 수 있습니다.

4. **사이트 세부 정보 보기:**
   사이트의 행을 클릭합니다(또는 제공되는 경우 "View"/"Details" 버튼을 클릭합니다). Hyperclass 대시보드에서는 최근 지표가 있는 기본 드릴다운을 보여줄 수 있습니다. 완전한 제어와 세부 분석을 위해 "Launch Advanced SEO(고급 SEO 시작)"를 클릭합니다.

5. **사이트의 고급 도구 사용:**
   고급 SEO 인터페이스에서 사이트 프로젝트가 세부 대시보드로 열립니다. 여기서 해당 사이트의 여러 탭을 탐색할 수 있습니다:
   - **개요(Overview)**: 고급 지표 (도메인 파워, 총 키워드, 트래픽 동향 등)
   - **오가닉 키워드(Organic Keywords)**: 사이트가 랭킹하는 키워드, 그들의 위치와 볼륨 (GSC 및 SearchAtlas 데이터를 통해 채워짐)
   - **백링크(Backlinks)**: 사이트의 백링크 프로필 분석 (참조 도메인 수, 앵커 텍스트, 상위 백링크)
   - **상위 페이지(Top Pages)**: 가장 많은 트래픽을 받거나 가장 많은 키워드로 랭킹하는 사이트 페이지들
   - **사이트 감사 / 이슈(Site Audit / Issues)**: 크롤 감사가 수행된 경우 기술적 이슈의 요약, 사이트 상태 점수 및 이슈 수 포함

6. **지표 해석:**
   데이터를 사용하여 사이트를 이해합니다. 높은 도메인 파워는 랭킹이 좋을 가능성이 높은 강한 사이트를 나타냅니다. 오가닉 트래픽 동향은 SEO가 개선되고 있는지 판단하는 데 도움이 됩니다. 백링크 수와 품질은 오프페이지 SEO를 암시합니다. 사이트 상태와 이슈는 기술적 개선이 필요한 곳을 알려줍니다. GSC 데이터(연결된 경우)는 키워드에 대한 실제 클릭, 노출, 평균 랭킹을 보여주며 – 어떤 쿼리가 트래픽을 유도하거나 주의가 필요한지 파악하는 데 매우 중요합니다.

## Google Search Console 연동 사용하기

Hyperclass의 SEO 연동의 큰 장점 중 하나는 Google Search Console(GSC) 성능 모듈입니다.

GSC를 연결하면 사이트가 랭킹하는 모든 키워드를 클릭, 노출, 클릭률(CTR), 평균 위치와 함께 자동으로 가져옵니다. 이는 자신의 사이트에 대한 수동 키워드 입력이 필요 없다는 의미입니다 – 데이터의 보물창고를 얻게 됩니다:

- **모든 랭킹 키워드**: Hyperclass은 GSC에서 사이트당 상위 10,000개 키워드까지 가져옵니다(수동으로 추적하지 않더라도)
- **성능 지표**: 각 키워드에 대해 Google 검색에서 얼마나 자주 표시되는지(노출), 얼마나 많은 클릭을 받았는지, 평균 랭킹 등을 확인할 수 있습니다
- **페이지 데이터**: 연동은 페이지 수준 데이터도 가져옵니다 – 사이트당 최대 2,000개 페이지(가장 많은 노출을 받은 페이지부터 시작)가 분석을 위해 저장됩니다

**사용 방법:**
고급 SEO 인터페이스에서 사이트 지표(Site Metrics) > GSC 성능(GSC Performance)(또는 유사한 섹션)으로 이동합니다. 쿼리, 페이지, 국가, 기기 등에 대한 표와 차트를 찾을 수 있습니다. 필터를 사용하여 날짜 범위를 조정하거나 특정 용어를 검색할 수 있습니다.

**예시:**
지난 3개월로 필터링하고 클릭순으로 정렬하여 최고 성과 키워드를 보거나, 위치로 필터링하여 5~15위에 랭킹된 쿼리를 찾을 수 있습니다(상위 3위로 향상시킬 좋은 기회).

**할당량 참고:**
GSC 연동 자체는 다른 도구처럼 크레딧을 "소비"하지 않습니다 – 사이트 연결은 GSC 프로젝트 슬롯 중 하나를 사용합니다(제한: 4개 사이트). 연결되면 데이터 업데이트는 자동으로 이루어집니다. 할당량 사용량 이해하기 - [SEO 빠른 둘러보기: 검색 순위 향상 가이드](seo-a-guide-to-boosting-your-search-rankings.md)

## 온페이지 SEO 감사 실행하기 (단일 페이지 분석)

사이트 프로젝트 내에서 웹사이트의 특정 페이지에 대해 온페이지 감사를 수행할 수 있습니다. 이 도구는 단일 페이지의 콘텐츠와 SEO 요소를 목표 키워드에 대해 분석하여 콘텐츠 점수와 기술 점수를 제공하고 권장사항을 제시합니다. 랜딩 페이지나 블로그 게시물과 같은 중요한 페이지를 최적화하는 데 훌륭합니다.

**온페이지 감사 사용 방법:**

1. 고급 SEO 인터페이스에서 콘텐츠(Content) > 온페이지 감사(On-Page Audit)로 이동합니다(또는 Hyperclass 대시보드의 사이트 프로젝트 보기에서 페이지 감사로 가는 빠른 링크가 있을 수 있습니다).
2. 감사하려는 페이지의 URL을 입력합니다.
3. 해당 페이지가 랭킹을 목표로 하는 하나 이상의 목표 키워드를 입력합니다.
4. 감사 실행을 클릭합니다. 도구가 페이지를 크롤링하고 해당 키워드에 대한 최상위 랭킹 페이지들과 내용 및 HTML을 비교합니다.
5. 짧은 처리 시간 후, 다음을 포함한 결과를 볼 수 있습니다:
   - 콘텐츠 점수(콘텐츠가 주제/키워드를 얼마나 잘 다루는지)와 기술 점수(온페이지 SEO 기술적 요소가 얼마나 잘 구현되었는지)
   - 콘텐츠, 기술, 구조와 같은 카테고리별 권장사항
   - 페이지와 경쟁업체 평균의 비교
6. 필요에 따라 페이지에 변경사항을 적용합니다(이는 도구 외부에서 웹사이트/CMS에서 수행됩니다). 그런 다음 감사를 다시 실행하여 점수가 개선되는지 확인할 수 있습니다.

**할당량 사용량:**
각 온페이지 감사는 입력하는 키워드 수에 관계없이 분석된 페이지당 1개의 감사 크레딧을 소모합니다. 월간 최대 40개의 온페이지 감사 크레딧이 있습니다. 같은 페이지를 다시 감사하면 또 다른 크레딧을 사용합니다. 할당량 사용량 이해하기 - [SEO 빠른 둘러보기: 검색 순위 향상 가이드](seo-a-guide-to-boosting-your-search-rankings.md)

## 사이트 감사로 사이트 상태 모니터링하기

단일 페이지 감사 외에도, 사이트 프로젝트 섹션에서는 전체 사이트 감사(기술적 SEO 이슈를 찾기 위해 전체 웹사이트를 크롤링)도 가능합니다.

**사이트 감사 실행 방법:**

1. 사이트 프로젝트를 방금 추가했다면, 초기 크롤링이 자동으로 수행되었을 수 있습니다.
2. 고급 SEO 시작을 클릭하고 사이트 감사(Site Audit)로 이동합니다.
3. 새 감사 프로젝트를 생성합니다. 도메인을 제공하고 기본 설정을 지정합니다:
   - 크롤링할 최대 페이지 수(예: 5,000페이지)
   - 크롤링 빈도: 수동 또는 예약
   - 제외 사항: /admin 또는 추적 URL과 같은 폴더 건너뛰기
4. 감사를 시작합니다. 크롤러가 수십 개의 요소를 확인합니다.
5. 사이트 상태 점수와 분류된 이슈 목록을 받게 됩니다:
   - 오류(Errors): 심각한 문제
   - 경고(Warnings): 보통 수준의 이슈
   - 알림(Notices): 사소한 팁

**할당량 사용량:**
- 크롤링된 1페이지 = 1페이지 크레딧
- 모든 사이트에 걸쳐 월간 총 50,000페이지
- 최대 4개 사이트를 활발히 감사할 수 있습니다
할당량 사용량 이해하기 - [SEO 빠른 둘러보기: 검색 순위 향상 가이드](seo-a-guide-to-boosting-your-search-rankings.md)

## AI 권장사항 또는 원클릭 수정

사이트 감사가 실행된 후, AI가 일부 온페이지 이슈를 자동으로 수정할 수 있습니다.

**AI 권장사항 사용 방법:**

1. 사이트 프로젝트를 추가한 후 AI 권장사항(AI Recommendations)으로 이동합니다.
2. 각 수정 제안은 1개의 크레딧을 소모합니다(월간 500개 사용 가능).
3. 수정에는 제목 업데이트, alt 태그, 내부 링크 개선이 포함됩니다.
4. AI는 메타 설명이나 콘텐츠 단락 제안까지 생성할 수 있습니다.

**중요:** 수정을 적용하기 전에 항상 검토하세요. 콘텐츠가 브랜드의 톤과 일치하는지 확인하세요.

## 사이트 프로젝트 및 감사 모범 사례

- **정기적 모니터링**: 대시보드를 주간 단위로 확인하세요
- **GSC 데이터 활용**: 키워드 기회를 찾는 데 사용하세요
- **크롤 크레딧 낭비하지 마세요**: 스마트 스케줄과 제외 설정을 사용하세요
- **AI 수정 – 현명하게 사용**: 승인된 사이트에서만 적용하세요
- **과거 진행 상황 추적**: Hyperclass은 과거 SEO 데이터를 저장합니다

## 문제 해결 및 일반적인 이슈

- **사이트가 크롤링되지 않음**: robots.txt, 사이트맵, 최대 페이지 수를 확인하세요
- **데이터가 업데이트되지 않음**: 고급 SEO 인터페이스에서 새로고침을 시도하세요
- **GSC가 연결되지 않음**: 확인된 속성에서 정확한 도메인을 일치시키세요
- **AI를 사용할 수 없음**: 사이트 호환성과 슬롯 제한을 확인하세요
- **지표 차이**: SEO 플랫폼 간에 변동이 있을 것으로 예상하세요
