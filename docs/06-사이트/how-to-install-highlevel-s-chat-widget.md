# 하이퍼클래스 채팅 위젯 설치하는 방법

하이퍼클래스 채팅 위젯(Chat Widget)을 고객 웹사이트에 설치하는 것은 광고비 없이 리드를 생성하는 가장 쉬운 방법 중 하나입니다.

채팅 위젯은 라이브 채팅(영업팀과 지원팀이 즉시 응답할 수 있는 회사에 유용), SMS 메시징, 그리고 왓츠앱 메시징(계정에서 활성화된 경우)을 지원합니다. 어떤 경우든 채팅 위젯 설치는 간단하며 개발 지식이나 기술적 스킬이 필요하지 않습니다.

## 채팅 위젯 설정하고 테스트하는 방법

**1단계**: 채팅 위젯 설정이 완료되면, 채팅 위젯을 통해 대화가 시작될 때 알림을 받을 수 있도록 워크플로우 트리거(Workflow Trigger)를 구성해야 합니다.

이를 위해 트리거 이벤트는 "Customer Replied(고객 응답)"로 설정하고, "Reply Channel(응답 채널)" 필터를 "Chat Widget(채팅 위젯)"으로 설정해야 합니다:

![채팅 위젯 워크플로우 트리거 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155037846628/original/DOyTmYpSaauoLhd0zo445wymedQpbgigPg.gif?1733429258)

각 "응답 채널"에는 각각의 트리거가 필요합니다. 위 예시에서는 "채팅 위젯"만 사용하고 있으며, Google Messages를 사용하려면 해당 특정 이벤트에 대한 새로운 트리거나 워크플로우를 설정해야 합니다.

**2단계**: 다음으로 + 버튼 클릭 > "Internal notifications(내부 알림)" 검색 > 3가지 옵션 중 선택:

1. 이메일(Email) - 특정 사용자나 커스텀 이메일로 이메일 전송
2. 인앱 알림(In-app notification) - 웹 앱이나 모바일 앱을 통해 특정 사용자나 모든 사용자에게 알림 전송
3. SMS - 특정 사용자나 커스텀 번호로 SMS 전송

**팁**: 
알림 이메일/문자에 "{{message.body}}" 커스텀 값을 사용하면 사용자가 채팅 위젯에 입력한 메시지를 표시할 수 있습니다.

## 자주 묻는 질문

**Q: 페이지별로 채팅 위젯 메시지를 커스터마이징할 수 있나요?**

현재 웹채팅 위젯 구성에서는 동적 메시징을 허용하지 않습니다. 위젯을 여러 페이지에서 사용할 수 있지만, 메시지는 초기 설정과 일관되게 유지됩니다.

**Q: 하위 계정에서 여러 개의 채팅 위젯을 만들 수 있나요?**

네, 하위 계정당 하나 이상의 채팅 위젯을 만들 수 있습니다!
