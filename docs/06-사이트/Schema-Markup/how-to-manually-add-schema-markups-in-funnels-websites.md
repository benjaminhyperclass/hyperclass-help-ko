# 퍼널과 웹사이트에서 스키마 마크업을 수동으로 추가하는 방법

스키마 마크업 생성기는 하이레벨의 웹사이트와 퍼널 빌더에 내장된 도구로, 페이지의 스키마 마크업(JSON-LD)을 생성, 검증, 게시할 수 있습니다.
이 가이드에서는 이 기능이 무엇인지, 왜 중요한지, 그리고 웹사이트와 퍼널에서 스키마를 추가하고 관리하는 방법을 설명합니다.

## 스키마 마크업 생성기란?

스키마 마크업 생성기는 하이레벨 웹사이트 및 퍼널 빌더에 내장된 도구로, 개별 페이지의 스키마 마크업(구조화된 데이터)을 생성, 검증, 게시할 수 있습니다. 구조화된 데이터는 검색 엔진과 AI 시스템이 비즈니스 유형, 상품, FAQ, 기사, 이벤트 등 주요 정보를 이해하는 데 도움을 줍니다.

## 스키마 마크업 생성기의 주요 장점

- **140가지 이상의 스키마 유형**: LocalBusiness, Product, Article/BlogPosting, FAQPage, Event, Organization 등 일반적인 스키마 유형을 지원합니다.

- **폼 뷰(코드 없이)**: 가이드 필드를 통해 기술적 지식 없이도 쉽게 스키마 세부정보를 완성할 수 있습니다.

- **JSON 뷰(고급)**: 완전한 제어가 필요할 때 JSON-LD를 직접 붙여넣거나 작성할 수 있습니다.

- **내장 검증 기능**: 저장 시 문제점을 강조하여 게시 전에 스키마 문제를 해결할 수 있습니다.

- **원클릭 저장 및 게시**: 스키마가 페이지 설정에 저장되고 페이지 게시 시 즉시 적용됩니다.

- **페이지당 여러 스키마**: 동일한 페이지에서 둘 이상의 스키마 유형을 추가하고 관리할 수 있습니다(예: LocalBusiness + FAQPage).

## 하이레벨 퍼널/웹사이트에서 스키마 마크업 생성하는 방법

올바른 설정은 최적화하고자 하는 정확한 웹사이트 페이지나 퍼널 단계를 열고, 방문자가 볼 수 있는 콘텐츠와 일치하는 스키마를 추가하는 것부터 시작됩니다. 스키마는 페이지별로 적용되므로, 구조화된 데이터를 원하는 각 페이지나 단계에서 이 과정을 반복해야 합니다.

**참고:** AI를 사용하면 스키마 마크업을 쉽게 생성할 수 있습니다. [퍼널과 웹사이트에서 AI를 사용한 스키마 마크업 생성하기](generate-schema-markups-using-ai-in-funnels-websites.md) 가이드를 확인해 보세요.

**참고:** 스키마는 **페이지별**로 적용되며 **다른 페이지나 퍼널 단계로 이어지지 않습니다**.

## 스키마 뷰: 폼 뷰와 JSON 뷰 비교

올바른 편집 뷰를 선택하면 더 빠르게 작업하고 오류를 줄일 수 있습니다. 폼 뷰는 대부분의 사용자에게 이상적이며, JSON 뷰는 다른 소스에서 스키마를 붙여넣거나 고급 필드를 맞춤설정해야 할 때 가장 좋습니다.

**폼 뷰:**
- 코드 없는 설정에 최적
- 가이드 필드를 사용하여 필수 및 권장 속성 완성
- 형식 오류 방지에 도움

**JSON 뷰:**
- 고급 사용자와 개발자에게 최적
- JSON-LD 직접 편집 또는 기존 스키마 붙여넣기 가능
- 폼 뷰에서 보여지지 않는 특수 속성 추가에 유용

## 지원되는 스키마 유형

스키마 마크업 생성기는 다양한 스키마 유형을 지원하여 페이지 목표에 맞는 구조화된 데이터를 사용할 수 있습니다. 올바른 스키마 유형을 사용하면 검색 엔진과 AI 시스템의 이해도가 향상되고 해당되는 경우 향상된 결과에 페이지가 표시될 자격을 얻을 수 있습니다.

일반적으로 지원되는 유형:

- Organization
- Website
- BreadcrumbList
- Article / BlogPosting
- FAQPage
- QAPage
- HowTo
- VideoObject
- Person
- Product
- LocalBusiness
- Event

참고: 각 카테고리 하위 유형도 포함됩니다(예: LocalBusiness 하위 유형).

## 성공을 위한 모범 사례

- **정확성이 중요합니다**: 페이지에서 볼 수 있는 정보만 마크업하세요. 가짜 리뷰나 숨겨진 콘텐츠는 절대 추가하지 마세요.

- **최신 상태 유지**: 영업시간, 가격, 연락처 정보가 변경되면 스키마를 업데이트하세요.

- **구체적인 유형 사용**: 가능하면 일반적인 "Local Business" 대신 "Dentist"를 선택하세요. 더 구체적일수록 좋습니다.

- **여러 스키마 결합**: 홈페이지에는 Organization과 Local Business 스키마를 모두 포함하여 포괄적인 커버리지를 제공할 수 있습니다.

- **모든 주요 페이지에 스키마 추가**: 홈페이지, 연락처 페이지, 상품/서비스 페이지, 블로그 게시물 모두 적절한 스키마 마크업의 혜택을 받습니다.

## 로컬 비즈니스에 가장 유용한 스키마 유형

- **Local Business**: 물리적 위치가 있는 모든 비즈니스에 필수입니다. 이름, 주소, 전화번호, 영업시간, 서비스 지역이 포함됩니다.

- **Product**: 판매하는 항목에 사용합니다. 가격, 재고, 평점, 이미지를 검색 결과에 직접 표시합니다.

- **Service**: 제공하는 서비스에 사용합니다. 서비스 설명, 가격, 서비스 지역을 표시합니다.

- **Review**: 검색 결과에 별점 평가로 나타나는 고객 후기(실제 리뷰여야 함).

- **FAQ**: Google 검색에서 직접 확장되는 질문과 답변으로, 더 많은 화면 공간을 차지합니다.

- **Event**: 워크샵, 세일, 수업, 또는 예정된 모든 행사. 날짜와 등록 링크를 표시합니다.

## 알아두면 좋은 점

편집기에서 스키마가 어떻게 작동하는지 아는 것은 빠른 문제 해결과 게시 후 예상치 못한 상황을 피하는 데 도움이 됩니다. 이런 세부사항은 페이지를 자주 업데이트하거나 여러 퍼널에서 작업할 때 특히 중요합니다.

- 스키마는 **페이지별**로 적용되며 다른 페이지나 퍼널 단계로 자동으로 이어지지 않습니다.

- 저장 후 편집기/미리보기에서 스키마가 나타날 수 있지만, 페이지가 **게시**된 후에만 실제 적용됩니다.

- AI로 새 페이지를 만들 때 스키마가 **자동 생성되지 않습니다**. 콘텐츠가 완성된 후 수동으로 추가하세요.

- 보이는 세부정보가 변경될 때(예: 영업시간, 가격, 연락처 정보) 스키마를 업데이트하세요.

## 자주 묻는 질문

**Q: 스키마 마크업 생성기를 사용하려면 기술적 지식이 필요한가요?**

아니요. 폼 뷰는 코드 없는 설정을 위해 설계되었습니다. 고급 사용자는 JSON 뷰를 사용하여 JSON-LD를 붙여넣거나 맞춤설정할 수 있습니다.

**Q: 수동 설정과 AI 생성 스키마에서 스키마 선택이 어떻게 다른가요?**

수동 설정에서는 사용 가능한 스키마 유형이 미리 정의된 목록으로 제한됩니다. 보통 SEO, AEO(Answer Engine Optimization), GEO(Generative Engine Optimization)에 가장 많이 사용되는 것들입니다.

AI를 사용하면 이런 제한이 없습니다. AI는 페이지 콘텐츠를 기반으로 유효한 모든 스키마 유형을 생성할 수 있습니다.

**Q: AI가 수동 목록에 없는 스키마 유형을 생성할 수 있나요?**

예. AI는 페이지 콘텐츠를 분석하여 수동 선택 옵션에 포함되지 않은 유효한 스키마 유형이라도 스키마 마크업을 만들 수 있습니다.

**Q: 스키마가 내 페이지의 어디에 게시되나요?**

페이지가 게시될 때 스키마가 페이지의 `<head>`에 JSON-LD로 게시됩니다.

**Q: 미리보기에서는 스키마가 보이는데 실제 페이지에서는 보이지 않는 이유는 무엇인가요?**

저장은 해당 페이지에 스키마를 저장하지만, 페이지나 퍼널 단계를 게시한 후에만 실제로 적용됩니다.

**Q: 하나의 페이지에 둘 이상의 스키마 유형을 추가할 수 있나요?**

예. 페이지당 여러 스키마를 추가하고 관리할 수 있습니다(예: LocalBusiness + FAQPage).

**Q: 다른 도구나 웹사이트에서 스키마를 붙여넣을 수 있나요?**

예. JSON 뷰를 사용하여 JSON-LD를 붙여넣고, Validate & Save(검증 및 저장)를 통해 게시 전에 문제를 확인할 수 있습니다.

**Q: "Validate & Save(검증 및 저장)"는 무엇을 확인하나요?**

검증은 누락되거나 잘못된 필드와 형식 문제를 식별하여 게시 전에 문제를 수정할 수 있도록 도움을 줍니다.

**Q: 스키마가 페이지에서 보이는 내용과 일치해야 하나요?**

예. 오해를 일으키는 마크업을 피하기 위해 사용자가 실제로 볼 수 있는 콘텐츠를 반영하는 구조화된 데이터만 추가하세요.
