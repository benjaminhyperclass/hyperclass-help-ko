# 폼과 설문의 이메일 알림

폼과 설문의 이메일 알림(Email Notifications) 기능은 폼 제출을 수동으로 확인할 필요 없이 자동으로 업데이트를 받을 수 있는 편리한 방법입니다. 이 기능으로 신속하게 응답하고, 팀원들과 제출 내용을 공유하며, 원활한 폼 관리가 가능해집니다. 또한 자동 응답(Auto Responder) 기능을 통해 폼을 작성한 사람에게 확인 이메일을 자동으로 발송할 수 있습니다.

---

# **폼과 설문의 이메일 알림이란?**

이메일 알림은 폼이나 설문이 제출될 때마다 발송되는 알림입니다. 이 알림은 지정된 메일함으로 직접 전송되어 팀이 실시간으로 정보를 받을 수 있게 해줍니다. 자동 응답은 이 기능과 함께 작동하여 폼을 제출한 사람에게 자동 회신을 보내며, 제출한 내용의 사본도 포함할 수 있습니다.

---

# **폼과 설문 이메일 알림의 주요 장점**

이메일 알림과 자동 응답 기능은 폼과 설문 데이터를 처리하는 팀의 효율성, 속도, 협업을 향상시킵니다.

- **즉시 알림**: 누군가 폼이나 설문을 제출하면 즉시 이메일을 받을 수 있습니다.

- **응답 시간 개선**: 리드와 연락처에게 더 빠른 후속 조치가 가능합니다.

- **협업**: 여러 팀원에게 동시에 알림을 보낼 수 있습니다.

- **커스터마이징**: 더 나은 정리를 위해 제목, 발신자 이름, 회신 이메일을 개인화할 수 있습니다.

- **자동 응답**: 폼을 제출한 사람에게 자동으로 확인 메시지를 보내어 전문적인 느낌을 줍니다.

## 알림 아이콘 찾기

알림 옵션은 폼 편집기의 보조 바에서 찾을 수 있습니다. 여기에서 이메일 알림과 자동 응답을 활성화할 수 있습니다.

### 폼(Forms) 또는 설문(Surveys)으로 이동

상단 네비게이션 바에서 Sites(사이트) → Forms(폼) (또는 Sites(사이트) → Surveys(설문))으로 이동합니다. 여기에서 모든 폼이나 설문을 저장하고 관리할 수 있습니다.

대시보드에서 목록에서 기존 폼이나 설문을 선택하거나 우측 상단의 Add Form(폼 추가) 버튼을 클릭하여 새로운 것을 만들 수 있습니다.

# **이메일 알림 설정**

이메일 알림 사이드바에서 누가 알림을 받을지, 이메일이 어떻게 나타날지 설정할 수 있습니다.

이메일 알림을 제대로 설정하면 적절한 팀원들이 정보를 받게 되고, 알림이 조직의 기준에 맞게 일관되게 나타나게 됩니다.

폼 편집기에서 **Bell(벨) 아이콘**을 클릭합니다. **Email Notification(이메일 알림)**을 토글로 켭니다.

설정 필드를 채웁니다:

- Subject(제목): 제목 줄을 입력합니다. 비워두면 폼이나 설문 이름이 사용됩니다. 커스텀 필드도 포함할 수 있습니다.

- **Email (To)(수신 이메일)**: 하나 또는 여러 이메일 주소를 입력합니다. 각 주소는 태그로 변환됩니다.

- **Reply-to Email(회신 이메일)**: 회신을 받을 이메일 주소를 추가합니다.

- **Sender Name(발신자 이름)**: 비워두면 기본적으로 로케이션이나 에이전시 이름이 사용됩니다.

- **저장**합니다.

**참고:** 이메일 알림은 이메일 크레딧을 사용합니다. 계정에 사용 가능한 크레딧이 있는지 확인하세요.

# **자동 응답 설정**

자동 응답(Auto Responder)은 폼을 제출한 연락처에게 자동으로 이메일을 보냅니다.

자동 응답은 제출을 즉시 확인하고 선택적으로 답변 사본을 보내줌으로써 연락처와의 신뢰를 구축하는 데 도움이 됩니다.

**Form Notification(폼 알림)** 사이드바를 엽니다. **Auto Responder(자동 응답)**을 토글로 켭니다.

설정 필드를 채웁니다:

- **Subject(제목)**: 제목 줄을 입력합니다. 기본값은 폼 이름입니다.

- **Reply-to Email(회신 이메일)**: 연락처 회신을 위한 이메일을 추가합니다.

- **Sender Name(발신자 이름)**: 비워두면 기본적으로 하위 계정 이름이 사용됩니다.

- **저장**합니다.

**참고:** 여기서 설정하는 Reply-To는 제출자에게 가는 자동 응답 이메일에만 적용됩니다. 팀 알림 이메일에는 영향을 주지 않습니다.

이 알림에 대한 회신은 "Reply-to-email" 필드에 설정된 주소로 전달됩니다.

# **이메일 알림을 받은 후 설문 응답을 보는 방법**

이메일 알림은 알림 역할만 하며 전체 설문 응답은 포함하지 않습니다. 응답을 보려면:

**단계:**

- 왼쪽 메뉴에서 **Sites(사이트)**로 이동합니다.

- **Surveys(설문)** 탭을 선택한 다음 **Submissions(제출)**을 클릭합니다.

- 타임스탬프와 답변이 포함된 제출 목록을 검토합니다.

- 개별 항목을 클릭하여 모든 응답을 확인합니다.

# **자주 묻는 질문**

**Q: 폼이나 설문 질문에 대한 답변에 따라 조건부로 이메일 알림을 보낼 수 있나요?**

아니오, Form Notification(폼 알림) 사이드바의 기본 이메일 알림은 모든 제출에 동일하게 적용됩니다. 조건부 라우팅을 원한다면 (예: 답변이 "옵션 A"이면 한 팀원에게, "옵션 B"이면 다른 팀원에게 보내기) **워크플로우(Workflows)**를 사용해야 합니다. "Form Submitted(폼 제출)" 워크플로우 트리거를 통해 조건을 설정하고 그에 따라 알림을 라우팅할 수 있습니다.

**Q: 자동 응답 이메일도 알림처럼 이메일 크레딧 사용량에 포함되나요?**

예. **이메일 알림** (팀에게 전송)과 **자동 응답 이메일** (제출자에게 전송) 모두 계정의 이메일 크레딧을 소모합니다. 크레딧이 소진되면 크레딧을 추가할 때까지 두 종류의 이메일 모두 발송되지 않습니다. 계정의 이메일 리포팅이나 청구 영역에서 크레딧 사용량을 모니터링할 수 있습니다.

**Q: "Email (To)(수신 이메일)" 필드에 여러 이메일을 입력했는데 그중 하나가 유효하지 않다면 나머지는 여전히 알림을 받을 수 있나요?**

예. 유효한 주소에는 계속 알림이 전달됩니다. 유효하지 않은 주소는 조용히 실패하며 다른 수신자에게의 전달을 차단하지 않습니다. 하지만 반복적인 전달 실패를 피하기 위해 "Email (To)(수신 이메일)" 필드에 입력된 주소를 정기적으로 확인하는 것이 좋습니다.

**Q: 알림의 "Email (To)(수신 이메일)" 필드에서 동적 머지 필드({{user.email}} 또는 {{contact.email}} 같은)를 사용할 수 있나요?**

현재 "Email (To)(수신 이메일)" 필드에서는 머지 필드가 지원되지 않습니다. 정적 이메일 주소를 입력해야 합니다. 동적 수신자에게 알림을 보내야 한다면 폼 제출로 트리거되는 **워크플로우(Workflow)**를 설정하고 워크플로우 액션을 사용하여 적절한 연락처나 사용자에게 이메일을 보내세요.
