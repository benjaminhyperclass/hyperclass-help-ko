# 웹사이트와 퍼널용 클릭하면 전화/문자/이메일하기 링크 및 버튼

이 글에서는 Hyperclass 퍼널(Funnels)과 웹사이트(Websites)에 클릭하면 전화, 문자, 이메일을 보낼 수 있는 링크와 버튼을 추가하는 방법을 알려드립니다. 이런 클릭 가능한 링크는 전환율을 높이고, 특히 모바일 기기에서 리드가 한 번의 탭으로 쉽게 연락할 수 있게 해줍니다.

# 클릭하면 전화/문자/이메일하기 링크와 버튼이란?

전화 걸기, 문자 보내기, 이메일 보내기를 시작하는 클릭 가능한 링크는 사용자가 즉시 행동을 취할 수 있게 해줍니다. 올바르게 설정하면 이런 링크를 클릭했을 때 사용자의 전화 다이얼러, SMS 앱, 기본 이메일 클라이언트가 미리 입력된 정보와 함께 열립니다. 이런 링크는 퍼널과 웹사이트에서 사용자 경험을 개선하고 전환율을 높이는 데 자주 사용됩니다.

---

## 클릭 액션 사용의 주요 장점

클릭 링크는 리드가 한 번의 탭으로 쉽게 연락할 수 있게 해서 소통을 간단하게 만듭니다.

- **원탭 커뮤니케이션**: 사용자가 복사나 입력 없이 즉시 전화, 문자, 이메일을 보낼 수 있습니다
- **높은 모바일 참여율**: 모바일 사용자가 클릭 가능한 액션과 상호작용할 가능성이 더 높습니다
- **리드의 시간 절약**: 전화번호, 메시지 내용, 이메일 내용 등을 자동으로 입력해 줍니다
- **트리거 준비 자동화**: 트리거 링크(Trigger Links)와 결합하여 후속 워크플로우를 실행할 수 있습니다
- **유연한 CTA 디자인**: 텍스트와 버튼 요소(Elements) 모두에 클릭 링크를 추가할 수 있습니다

## 클릭하면 전화하기 링크 추가하는 방법

사람들이 클릭/탭해서 전화를 걸 수 있는 링크를 만들려면:

- 퍼널이나 웹사이트 빌더에서 텍스트를 선택하세요
- 링크 아이콘을 클릭하세요
- 다음 형식으로 링크를 입력하세요: tel:+1234567890
- 저장을 클릭하고 모바일 기기에서 미리보기/테스트해보세요

## 자주 묻는 질문

**Q: 이런 링크의 클릭을 어떻게 추적할 수 있나요?**
트리거 링크(Trigger Links)를 사용해서 클릭을 기반으로 액션을 추적하고 자동화하세요.

**Q: 사용자의 기기가 클릭 링크를 지원하지 않으면 어떻게 되나요?**
기기가 해당 프로토콜(예: 데스크톱에서 sms:)을 지원하지 않으면 클릭해도 아무 일이 일어나지 않습니다. 다양한 기기에서 링크를 테스트하고 대체 연락처 정보를 포함하는 것을 권장합니다.

**Q: 하나의 링크에 여러 액션(전화와 문자 같은)을 추가할 수 있나요?**
아니요. 각 링크는 전화, 문자, 이메일 중 한 번에 하나의 액션만 수행할 수 있습니다. 여러 옵션을 제공하려면 각 액션에 대해 별도의 링크나 버튼을 사용하세요.

**Q: 이미지나 아이콘 안에 클릭 링크를 사용할 수 있나요?**
네. 퍼널이나 웹사이트 빌더에서 동일한 형식을 사용해서 이미지나 아이콘을 하이퍼링크로 만들 수 있습니다. 전화나 메시지 아이콘 같은 시각적 CTA에 유용합니다.
