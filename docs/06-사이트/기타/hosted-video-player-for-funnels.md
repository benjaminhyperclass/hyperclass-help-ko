# 퍼널용 호스팅 동영상 플레이어

완전히 새로워진 동영상 플레이어로 동영상을 업로드하고 스트리밍하세요:

- 동영상 업로드, 변경, 삭제를 위한 개선된 인터페이스
- 다양한 화질로 동영상 재생 (480P, 720P, 1080P, 자동)
- 재생 속도 변경 (0.5x, 0.75x, 1x, 2x)
- [PiP(Picture in Picture)](https://en.wikipedia.org/wiki/Picture-in-picture) 모드로 동영상 재생
- 재생 컨트롤, 전체화면 옵션, 남은 시간 표시/숨기기
- 진행률 표시줄 표시/숨기기 및 동영상 탐색 비활성화
- 자동 재생 활성화/비활성화
- 모든 주요 포맷에서 최대 4GB까지 동영상 지원
- 모든 주요 브라우저에서 지원

## 호스팅 동영상 기능 사용 방법

**1단계:**
- 호스팅 동영상 기능을 사용하려면 퍼널에 동영상 요소(Video Element)를 추가하세요.

![퍼널에 동영상 요소 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48263987310/original/VnPpKixTTumIYcjEkWOvE2Oqm0Aa5tuCAg.png?1668693872)

**2단계:**
- 동영상 요소를 클릭하고, 왼쪽의 동영상 설정(Video Settings)으로 이동한 다음 아래로 스크롤하여 동영상 타입(Video Type) > 호스팅 동영상(Hosted Video)을 선택하세요.

![동영상 타입에서 호스팅 동영상 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48263987868/original/2avejriMf-Jluly1dckklYBs4B0nkH8bZw.png?1668693968)

**3단계:**
- 동영상 선택(Select Video)을 클릭하여 미디어 라이브러리로 이동하세요.

![동영상 선택 버튼 클릭](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48263988423/original/klhhSEr3nXFnE4dsWmo3K1g0443jM6LphA.png?1668694098)

**4단계:**
- 호스팅할 동영상이 없다면 업로드하세요.

![동영상 업로드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48263988778/original/NQzSWrbhOKDkGOib8IdshXYAT62i9wVpQg.png?1668694174)

**참고:** 동영상은 4GB 이하라면 모든 주요 동영상 포맷이 가능합니다.

**5단계:**
- 동영상 업로드가 완료되면, 동영상 파일을 더블클릭하여 인코딩을 시작하세요.

![동영상 파일 더블클릭으로 인코딩 시작](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48263989442/original/odjpw6Z-XA46FTXMCTlRnh4yc420RG7DmA.png?1668694300)

![인코딩 시작 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48263990013/original/HKInsAb5rkwCIY6cloFFxweJWmaD7Jw1GQ.png?1668694369)

**참고:** 동영상 추적을 위해 인코딩이 필요한 이유에 대한 자세한 내용은 [여기](https://www.loom.com/share/a8bf99ab64e64a7ca11814dd81e56e5f?t=58)를 클릭하세요. 동영상 인코딩이 실패 상태인 경우 동영상을 다시 선택하여 인코딩을 재시도할 수 있습니다. 하루 이상 처리 중인 모든 동영상은 자동으로 실패로 표시됩니다. 또한 시청자가 여러 동영상 화질 중에서 선택할 수 있도록 하기 위해서도 필요합니다. 원본 동영상은 한 가지 화질로만 재생할 수 있습니다.

## 호스팅 동영상 요소 설정 > 재생 컨트롤(Playback Controls)의 각 토글은 무엇을 하나요?

### 자동 재생(Autoplay):
![자동 재생 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48263993735/original/k0ZU3XqKsi24uyp5x8s87A4gUTturicrrA.png?1668695049)

**참고:**
자동 재생 규정에 따라 IOS, Safari, Chrome, Android, Firefox, Edge에서 자동 재생되는 동영상은 음소거 상태로 시작되어야 합니다. 이 정책은 브라우저 제조업체에서 관리하며, 모든 동영상 호스팅 서비스가 이를 준수해야 합니다.

각 브라우저의 정책 변경 링크:
- [IOS 자동 재생 동영상 정책](https://webkit.org/blog/6784/new-video-policies-for-ios/)
- [Safari 자동 재생 동영상 정책](https://webkit.org/blog/7734/auto-play-policy-changes-for-macos/)
- [Chrome 자동 재생 동영상 정책](https://developer.chrome.com/blog/autoplay/)
- [Android 자동 재생 동영상 정책](https://developer.chrome.com/blog/autoplay-2/)
- [Firefox 자동 재생 동영상 정책](https://developer.mozilla.org/en-US/docs/Web/Media/Autoplay_guide)
- [Edge 자동 재생 동영상 정책](https://winaero.com/enable-or-disable-media-autoplay-in-microsoft-edge/#:~:text=Download%20Microsoft%20Edge-,Media%20Autoplay%20Options,-Options%20are%20as)

Chrome은 시청자의 [미디어 참여 지수](https://developer.chrome.com/blog/autoplay/#mei)에 따라 동영상을 소리와 함께 자동 재생할지 결정합니다. Chrome은 미디어가 정기적으로 재생되는 사이트에서 가장 높은 미디어 참여 점수를 계산합니다. 충분히 높으면 데스크톱에서만 소리와 함께 미디어 자동 재생이 허용됩니다. 자동 재생되지 않는 동영상은 음소거 해제 상태로 시작됩니다.

### 미등록 방문자 리디렉션(Redirect Unregistered Visitors):
![미등록 방문자 리디렉션 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48267952758/original/w76KHRgcauRmx3kNv1mVyCiYC_4nQsLfLg.png?1670414859)

사람들이 URL을 통해 동영상 페이지에 직접 도착했지만 시스템에 연락처로 등록되지 않았고 옵트인 양식에 이름과 정보를 입력하지 않은 경우, 방문자의 동영상 시청 진행률을 추적하려 하지만 동영상에 직접 도착한 경우, 동영상 진행률 추적 워크플로우는 이름있는 연락처에 대한 유용한 통계를 제공하기 위한 이름과 세부 정보를 제공하지 않습니다. 워크플로우는 빈 리드를 자체에 추가하게 됩니다.

이러한 상황을 피하기 위해 미등록 방문자 리디렉션(Redirect Unregistered Visitors) 토글을 사용하여 그러한 방문자들을 옵트인 폼이 있는 다른 페이지로 보내서 이름과 기타 세부 정보를 입력하도록 할 수 있습니다. 입력을 완료하면 동영상으로 다시 돌아옵니다. 이 리디렉션은 방문자에 대한 정보가 포함되어야 할 쿠키가 첨부되지 않은 상태로 지정된 페이지로만 방문자를 보냅니다.

방문자가 브라우저에 쿠키를 추가하도록 허용하지 않으면 이 리디렉션이 작동하지 않을 수 있습니다. 리디렉션할 단계/페이지를 반드시 지정해야 합니다:

![리디렉션 페이지 지정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48267954795/original/Uo4FdStB5duO62t0RjJAGSbu6X4aWKXCSQ.png?1670415238)

## 문제해결: 인코딩된 동영상을 선택할 수 없어요

미디어 라이브러리에서 동영상이 "인코딩됨"으로 표시되지만 퍼널 페이지에 삽입할 때 선택할 수 없다면 다음 단계를 시도해보세요:

- **몇 분 기다리기**: 완료된 것처럼 보여도 백그라운드에서 인코딩 프로세스가 아직 완료되고 있을 수 있습니다.
- **페이지 새로고침**: 퍼널 편집기나 미디어 라이브러리를 새로고침하여 동영상이 사용 가능해지는지 확인해보세요.
- **인코딩 재시도**: 문제가 지속되면 미디어 라이브러리에서 파일을 다시 선택하여 새로운 인코딩을 시도해보세요.
- **파일 포맷과 크기 확인**: 동영상이 포맷(.mp4 등)과 크기(4GB 미만)를 포함한 지원되는 미디어 요구사항을 충족하는지 확인하세요.
