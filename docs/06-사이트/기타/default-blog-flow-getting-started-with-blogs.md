# **Hyperclass의 블로그란?**

Hyperclass 블로그는 웹사이트나 퍼널에 글을 작성하고 정리할 수 있는 내장형 발행 시스템입니다. 작성자, 카테고리, SEO, 디자인을 한 곳에서 관리하여 일관성 있게 발행하고 트래픽을 적절한 페이지로 유도할 수 있습니다.

## Hyperclass 블로그의 주요 장점

- 
중앙화된 발행: 모든 포스트, 작성자, 카테고리, SEO를 한 곳에서 관리


- 
유연한 배치: 블로그 포스트 목록 요소를 사용해 웹사이트나 퍼널의 어느 페이지에든 포스트 표시 가능


- 
내장 SEO 필드: 슬러그, 메타 데이터, 정규 URL 설정으로 검색 노출 개선


- 
디자인 옵션: 목록과 포스트 뷰에서 표준형 또는 컴팩트 레이아웃 선택


- 
확장 가능한 워크플로우: 라이브러리가 성장함에 따라 초안 작성, 예약, 발행 또는 일괄 가져오기 지원

## Hyperclass에서 블로그 만들기

블로깅이 처음이거나 블로그를 시작하고 싶지만 어디서부터 시작해야 할지 모르겠다면, 다음 단계를 따라해보세요:

### 1단계: 블로그 사이트(컨테이너) 만들기

- 
사이트(Sites) 메뉴로 이동


- 
블로그(Blogs) 클릭


- 
블로그 만들기(Create Blog) 클릭. 처음부터 만들거나 템플릿 선택


![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064779424/original/OiYF3mABKqw7rZqW5H23dRkDcSvWob6htg.jpeg?1770900245)

### 2단계: 블로그 사이트 세부 정보 추가

- 블로그 **사이트 제목** 추가 (브라우저 탭 제목과 검색 결과에 나타나는 내용)

- **도메인** 선택 또는 추가/편집

- **슬러그** 추가

- **블로그 메타 설명** 추가 (선택사항이지만 권장)

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064779485/original/M-l1QTgtlbe4xtiEOCEf57_wpanMIKzu1A.jpeg?1770900284)

### 3단계: 블로그 포스트 작성

- 
사이트(Sites) → 블로그(Blogs)로 이동


- 
**목록**에서 생성한 블로그 사이트를 클릭하거나, 블로그 포스트를 추가하고 싶은 기존 블로그 사이트를 클릭


![](https://jumpshare.com/share/W1ZseR5HZIB5gqCKFqVA+/GIF+Recording+2026-02-12+at+6.22.19+PM.gif)

- 
새 포스트(New Post) 클릭


- 
+ 새 포스트(+ New Post)를 클릭해 처음부터 블로그 포스트 작성. [URL에서 블로그 포스트 가져오기](../블로그/blogs-import-blog-posts-using-url.md)나 CSV에서 블로그 포스트 가져오기도 가능


- 
처음부터 블로그 포스트를 만들거나 편집하려면 [**블로그 포스트 작성**](../create-blog-posts.md) 문서를 확인하세요


![](https://jumpshare.com/share/lauN1KBkpiMjO4w8pQju+/Screen+Shot+2026-02-12+at+6.23.44+PM.png)

### 4단계: 블로그 포스트 세부 정보 추가

- 편집기에서 블로그 포스트 콘텐츠를 추가하고 서식을 설정한 후, 계속(Continue) 클릭

![](https://jumpshare.com/share/ODj6plhHTcGJP049Rix9+/Screen+Shot+2026-02-12+at+6.33.01+PM.png)

- 블로그 포스트 URL 슬러그 추가

![](https://jumpshare.com/share/2qSl2zO7wamgKp1EYRyN+/Screen+Shot+2026-02-12+at+6.38.18+PM.png)

- 카테고리와 **작성자** **선택** 또는 생성

![](https://jumpshare.com/share/TRJGa96erSfwYce2526S+/GIF+Recording+2026-02-12+at+6.40.37+PM.gif)

- [정규 링크](../canonical-urls-explained-boost-your-highlevel-blog-seo.md) 편집하거나 그대로 유지

- 키워드 추가

![](https://jumpshare.com/share/yIPdQ9TzuhrBXv0OwOUn+/Screen+Shot+2026-02-12+at+6.43.37+PM.png)

### 5단계: SEO 세부 정보 추가

- SEO 제목 추가

- 포스트 설명 추가

- 커버 **이미지**와 **대체 텍스트** 추가

![](https://jumpshare.com/share/JAMVKS0xLVx9C9PIGEAX+/Screen+Shot+2026-02-12+at+6.47.35+PM.png)

### 6단계: 블로그 포스트 발행

- 나중에 추가 편집을 위해 포스트를 **초안**으로 저장

- 포스트를 즉시 **발행**하거나 미래 시간으로 **예약**

참고: **퍼널이나 웹사이트에 블로그를 추가**하려면 상세한 [**가이드**](../블로그/add-blog-post-element-in-funnel-step-or-website-page.md)를 확인하세요.

![](https://jumpshare.com/share/NUDN2nvuLcneDX3T7hnt+/GIF+Recording+2026-02-12+at+6.51.58+PM.gif)

## **발행된 블로그 포스트 또는 초안 수정하기**

- 블로그 포스트로 이동하여 포스트 옆의 **점 세 개**를 클릭한 후 **포스트 업데이트**를 클릭

- 편집기에서 블로그 포스트 콘텐츠를 편집

- **작성자**, **카테고리**, **슬러그**, **SEO** 등과 같은 **블로그 포스트 세부 정보를 변경**하려면 **설정 업데이트** 버튼을 클릭

- 필요한 업데이트를 한 후 **발행** 클릭

![](https://jumpshare.com/share/WwE17o2QD3KECvtgabk0+/GIF+Recording+2026-02-12+at+6.58.41+PM.gif)

## **자주 묻는 질문**

**Q: 블로그에서 도메인은 어떻게 작동하나요?**

도메인은 블로그 목록 요소가 추가된 퍼널이나 웹사이트에 추가됩니다.

블로그 URL 형식:
| 도메인 | 블로그 포스트 슬러그 | 최종 URL |
|--------|-------------------|-----------|
| abc.com | b/sample-now | abc.com/b/sample-now |
| abc.com/xyz | b/sample-now | abc.com/xyz/b/sample-now |
| blog.abc.com | b/sample-now | blog.abc.com/b/sample-now |

**Q: 각 카테고리별로 별도의 블로그 페이지를 가질 수 있나요?**

현재는 지원되지 않습니다. 향후 업데이트에서 카테고리 필터를 기반으로 슬러그 및 SEO 세부 정보와 함께 퍼널/웹사이트 페이지를 만들 수 있도록 할 예정입니다.

**Q: 작성자, 카테고리, 키워드 추가는 선택사항인가요?**

URL 슬러그, 카테고리, 작성자, 제목, 포스트 설명은 필수입니다.

**Q: 블로그 포스트에서 커스텀 값을 사용할 수 있나요?**

현재 블로그 포스트에서는 커스텀 값(Custom Values)이 지원되지 않습니다. {{contact.name}}이나 {{custom_values.xyz}}와 같은 값을 사용할 수 없습니다.

**Q: 블로그 포스트에 폼, 설문 또는 커스텀 코드를 어떻게 추가하나요?**

현재 블로그 포스트에는 폼이나 설문을 추가할 수 없습니다. 커스텀 코드 추가 옵션은 곧 출시될 예정입니다.

**Q: 블로그 포스트에 권장되는 이미지 크기는?**

권장 크기는 없습니다. 하지만 블로그 포스트 편집기 내부의 페이지 너비는 736px입니다.
