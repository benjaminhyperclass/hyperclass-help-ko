# 사이트 - 트래픽 소스 분석

Hyperclass의 트래픽 소스 분석 기능을 사용하면 웹사이트와 퍼널 트래픽이 어디서 오는지 한눈에 파악할 수 있어요. 유료 vs 오가닉 성과를 비교하고, 일별 트렌드를 확인하며, 특정 날짜나 채널을 자세히 분석할 수 있습니다. 이 가이드에서는 트래픽 소스 차트 읽는 법과 어트리뷰션 작동 원리를 설명해드려요.

## **트래픽 소스 분석이란?**

트래픽 소스 분석은 사이트(Sites) → 분석(Analytics) 안에 있는 시각적 대시보드로, 일별 트래픽을 마케팅 채널과 소스별로 분류해서 보여줍니다. 누적 막대 차트와 관련 탭(트래픽 채널, 소스/매체)을 통해 적절한 캠페인으로 결과를 어트리뷰션하고, 시간별 채널을 비교하며, 필요한 경우 특정 날짜로 전체 대시보드를 필터링할 수 있어요.

## 트래픽 소스 분석의 주요 장점

- 
**채널 명확성**: 유료 검색(Paid Search), 유료 소셜(Paid Social), 오가닉 검색(Organic Search), 직접 유입(Direct), 추천(Referral), 소셜(유기적, 가능한 경우), 기타(알 수 없음/매핑되지 않음)에서 오는 트래픽 양을 빠르게 확인할 수 있어요.

- 
**빠른 의사결정**: 일별 누적 막대와 즉석 툴팁으로 성장, 하락, 계절적 변화를 한눈에 파악할 수 있어요.

- 
**향상된 어트리뷰션**: UTM과 클릭 ID를 검증하여 유료 클릭이 직접 유입이나 기타로 잘못 분류되는 것을 방지해요.

- 
**팀 협업**: 일관된 색상과 라벨을 사용하여 이해관계자들이 리뷰에서 차트를 동일하게 해석할 수 있어요.

- 
**집중 분석**: 특정 날짜를 클릭하여 대시보드를 필터링하고 맥락을 가지고 이상 현상을 조사할 수 있어요.

## Hyperclass에서 채널을 결정하는 방법

- 
**UTM 매개변수**: Hyperclass는 utm_source, utm_medium, utm_campaign이 있을 때 이를 읽어요.

- 
**클릭 ID 및 자동 태깅**: gclid/msclkid/fbclid와 같은 식별자가 유료 트래픽 분류 확인에 도움을 줄 수 있어요.

- 
**리퍼러 감지**: 알려진 검색 엔진과 소셜 네트워크를 매칭하며, 직접 유입은 리퍼러/UTM이 없는 경우를 의미해요.

- 
**기타(알 수 없음/매핑되지 않음)**: 사용 가능한 UTM/리퍼러가 없거나 잘못된 형식의 값을 가진 세션이에요.

## **사이트에서 트래픽 소스 분석 사용하기**

적절한 설정이 정확한 어트리뷰션과 깔끔한 시각화를 보장해요. 다음 단계에 따라 대시보드를 찾고, UTM을 검증하며, 드릴다운을 효과적으로 사용하세요.

- 
사이트(Sites) → 분석(Analytics)으로 이동하세요.

퍼널/웹사이트를 선택한 다음 **날짜 범위**를 선택하세요.

![날짜 범위 선택](https://jumpshare.com/share/2tc6aL62tmso6HJHOuz7+/GIF+Recording+2025-10-23+at+10.26.40+PM.gif)

**획득 데이터(Acquisition Data)**로 스크롤하고 **트래픽 채널(Traffic Channel)** 탭을 선택하여 채널 레벨 뷰(예: 직접 트래픽, 유료 소셜, 유료 검색, 소셜 미디어, 추천)를 확인하세요.

**소스/매체(Source/Medium)** 탭으로 전환하여 정확한 출처(예: google, facebook, direct, TikTok, other)를 확인하세요.

## **자주 묻는 질문**

**Q: 누적 막대 차트에는 어떤 지표가 표시되나요?**

선택한 날짜 범위 내에서 사이트 자산(웹사이트/퍼널)에 대한 일별 방문 수가 표시됩니다.

**Q: 일부 트래픽이 기타(알 수 없음/매핑되지 않음)로 분류되는 이유는 무엇인가요?**

세션에 UTM이 없거나, 잘못된 형식의 매개변수가 포함되어 있거나, 리퍼러 없이 도달한 경우(예: 앱 브라우저, 리다이렉트)일 수 있어요. 일관된 UTM을 추가하고 리다이렉트를 확인하여 이 항목을 줄일 수 있습니다.

**Q: 대시보드는 어떤 시간대를 사용하나요?**

데이터는 계정/로케이션 시간대로 집계됩니다. 팀이 분산되어 있다면 혼동을 피하기 위해 시간대를 공유하세요.
