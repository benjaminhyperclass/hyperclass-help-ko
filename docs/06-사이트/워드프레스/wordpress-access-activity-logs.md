# 워드프레스 접근 및 활동 로그

이 가이드는 Hyperclass 플랫폼의 워드프레스 관리자 패널에서 새롭게 제공되는 **접근 및 활동 로그(Access & Activity Logs)** 기능을 이해하는데 도움을 드리기 위해 작성되었습니다. 이 기능은 사용자 활동을 추적하고, 접근 요청을 모니터링하며, 사이트 관리의 투명성을 유지할 수 있는 포괄적인 방법을 제공합니다. 로그에는 보안 강화, 문제 해결, 책임 추적을 위한 상세한 인사이트가 포함됩니다.

# 주요 기능 개요

## **활동 로그**

- **목적**: Hyperclass 워드프레스 계정이나 특정 사이트에서 수행된 모든 작업을 검토합니다.
- **수집되는 정보**:
  - **사용자 식별**: 각 작업을 수행한 사람을 확인합니다.
  - **타임스탬프**: 작업이 수행된 정확한 시간을 알 수 있습니다.
  - **작업 상태**: 작업이 성공했는지 또는 오류가 발생했는지 확인합니다.
- **활용 사례**: 사이트 구성 변경 사항을 추적하거나 사용자 활동을 모니터링하는 데 적합합니다.

### 활동 로그 사용법

- **워드프레스 대시보드** → **사이트 관리(Manage Site)**로 이동합니다
- 하위 메뉴에서 활동 로그(Activity Logs)를 선택 → **활성화(Enable)**
- **시간 범위(time range)**를 선택 → 로그가 활성화되면 선택한 시간이 지난 후부터 로그가 표시되기 시작합니다

![활동 로그 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155036265511/original/GUpi6xqLctEvqluSbov8Uq8Z0t33y3ZLNQ.png?1731088535)

## **접근 로그**

- **목적**: 트래픽 패턴과 사용자 행동을 이해하기 위해 모든 서버 요청을 모니터링합니다.
- **수집되는 정보**:
  - **요청된 URL**: 접근한 특정 URL을 추적합니다.
  - **클라이언트 IP 주소**: 요청이 시작된 위치를 식별합니다.
  - **타임스탬프**: 각 요청의 정확한 시간을 알 수 있습니다.
  - **HTTP 상태 코드**: 응답 상태를 검토합니다(예: 200은 성공, 404는 찾을 수 없음).
  - **응답 크기**: 클라이언트에게 전송된 데이터 크기를 파악합니다.
- **활용 사례**: 사이트 트래픽 분석, 비정상적인 접근 패턴 감지, 사이트 성능 최적화에 유용합니다.

### 접근 로그 사용법

- 워드프레스 대시보드 → **사이트 관리(Manage Site)**로 이동합니다
- 분석(Analytics) → **방문자 보고서(Visitors Report)** 선택 → 하단으로 스크롤하여 접근 로그(Access Logs)를 확인합니다

![접근 로그 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155036265527/original/0woR1jXOxQN6k1pinuFlN6wA9VAITwoeyQ.png?1731088560)

## **감사 로그**

- **목적**: 포괄적인 사용자 활동 추적을 위해 워드프레스 로그를 기존 Hyperclass 플랫폼의 감사 로그(Audit Logs)와 통합합니다.
- **수집되는 정보**:
  - **사용자 작업**: 워드프레스 구매, 구독 해지, 사이트 생성, 사이트 삭제 등 중요한 작업을 수행한 사용자를 추적합니다.
- **활용 사례**: 워드프레스 계정의 중요한 이벤트에 대한 명확한 감사 추적을 제공하여 책임 추적을 강화합니다.

### 감사 로그 사용법

- 좌측 사이드메뉴에서 하위 계정 설정(Subaccount Settings)으로 이동합니다
- 맨 아래로 스크롤하여 **감사 로그(Audit Logs)**를 선택합니다

![감사 로그 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155036265627/original/M6HDffWGIcRJr9IuCFi3gGKfZ1INAHflug.png?1731088689)

# 문제 해결 및 모범 사례

- **로그가 표시되지 않음**: 로그를 볼 수 있는 필요한 권한이 있는지 확인하세요.
- **의심스러운 활동 감지**: 즉시 비밀번호를 변경하고 서버 접근 제어를 검토하세요.
- **로그에 높은 오류율**: 로그를 사용하여 실패하는 플러그인이나 잘못 구성된 설정을 식별하세요.
