# 워드프레스 호스팅: 단계별 마이그레이션 가이드

안녕하세요! 에이전시로서 고객의 워드프레스 웹사이트를 Hyperclass 호스팅으로 이전할 수 있는 모든 마이그레이션 옵션을 사용하실 수 있습니다.

워드프레스 대시보드로 이동하여 마이그레이션할 사이트를 선택하세요. 기술적 숙련도와 고객 요구사항에 맞는 옵션을 선택하시면 됩니다. 세 가지 옵션이 있습니다:

- **LC Migrator 플러그인 다운로드 (권장)**: 몇 번의 클릭만으로 전체 사이트를 쉽게 마이그레이션할 수 있습니다.

- **수동 마이그레이션 (All-in-One Migration 플러그인 사용)**: .wpress 파일을 생성하여 수동으로 사이트를 마이그레이션합니다.

- **에이전시 지원 (에이전시 사용자 전용)**: 간단한 양식을 작성하면 워드프레스 전문가가 마이그레이션을 도와드립니다.

![워드프레스 마이그레이션 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045794827/original/hXUmjzErlT5a0MYfxxu5MNJ0P1IAK7rFgw.png?1745865132)

## 옵션 1: 플러그인으로 손쉬운 마이그레이션

**적합한 경우**: 빠르고 기술적 지식 없이 최소한의 노력으로 마이그레이션  
**권장도**: ✅ 예

### 사용 방법:

### 1단계: LC Migrator 플러그인 다운로드

Download Plugin(플러그인 다운로드)을 클릭하여 ZIP 파일을 다운로드합니다.

- 이 ZIP 파일을 사이트 마이그레이션에 사용합니다.

![LC Migrator 플러그인 다운로드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795144/original/tC3m1wJALQ6nJkQpgD9-gV4oAmB1hXKwpA.png?1745865716)

### 2단계: 마이그레이션할 워드프레스 사이트에 플러그인 업로드

마이그레이션할 사이트의 워드프레스 대시보드에 로그인합니다.

- Plugins(플러그인) → Add New(새로 추가)로 이동합니다.

- 상단의 Upload Plugin(플러그인 업로드)을 클릭합니다.

- 앞서 다운로드한 ZIP 파일을 업로드하고 Install Now(지금 설치)를 클릭합니다.

![플러그인 업로드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795157/original/mXxpWKS-n1QmEoXg0HEG4LFJJP-qNJmpsQ.png?1745865759)

![플러그인 설치](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795171/original/KmrYwrRXkAa61zwKgKmRHEEV8PGss3KhvQ.png?1745865784)

### 3단계: 플러그인 활성화

설치 완료 후 Activate Plugin(플러그인 활성화)을 클릭합니다.

- LC Migrator 플러그인이 활성화되어 다음 단계를 준비합니다.

![플러그인 활성화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795187/original/S8P5LuVoRbcqzZQvLHJe2QOPP5GzgSVeow.png?1745865825)

### 5단계: LC Migrator 플러그인 열기

사이드바에서 LC Migrator를 찾아 클릭합니다 (바로 보이지 않으면 아래로 스크롤하세요).

- 로그인을 요청하면 Hyperclass(Hyperclass) 계정 정보를 사용합니다.

- 대상 워드프레스 인스턴스가 있는 하위 계정을 선택합니다.

![LC Migrator 로그인](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795198/original/mK6HvyHsI8xLw593UnKRyK2tw3_bf7btpQ.png?1745865844)

![계정 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795212/original/tvSuGjGTITBc-V0PsOevwPAbq8PJ_DSPEA.png?1745865857)

![하위 계정 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795240/original/AMJ2qImQMNYWLEa4WJ4_fbGroXP2HRZthg.png?1745865898)

![계정 확인](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795244/original/LNid27E078m2xNSh_ZlxGGoNvH_fy59_jw.png?1745865911)

### 6단계: 마이그레이션 시작

하위 계정을 선택하면 플러그인으로 다시 돌아갑니다.

- Continue(계속)를 클릭합니다.

- 대상 웹사이트를 선택합니다(예: Migration Demo).

- 대상 워드프레스 사이트의 모든 콘텐츠가 제거된다는 확인 박스를 체크합니다.

- Start Migration(마이그레이션 시작)을 클릭합니다.

![마이그레이션 시작](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795254/original/r2fAdmAxQ9G1tmGOx6Zjm8b1c_15zeLO8A.png?1745865926)

### 7단계: 마이그레이션 모니터링

마이그레이션 프로세스가 시작되며 대규모 사이트의 경우 일반적으로 10-15분이 소요됩니다.

- 최종 단계 전에는 언제든지 Cancel Migration(마이그레이션 취소)을 클릭하여 취소할 수 있습니다 (주의: 마지막 단계에서는 취소가 불가능합니다).

### 8단계: 마이그레이션 완료

완료되면 Migration Completed(마이그레이션 완료) 메시지가 표시됩니다.

- 이제 마이그레이션된 웹사이트를 방문할 수 있습니다.

- (선택사항이지만 환영합니다) 플러그인 개선에 도움이 되도록 리뷰를 남겨주세요!

![마이그레이션 완료](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795255/original/6UmUmChzDxE7gL79mFSlpkRY-BQPuCTNnQ.png?1745865944)

## 옵션 2: 간단한 수동 마이그레이션 (AIO 파일 업로드)

**적합한 경우**: All-in-One Migration 플러그인을 통한 .wpress 내보내기에 익숙한 사용자  
**사용 상황**: 고객이 .wpress 파일에 접근할 수 있거나 백업에서 마이그레이션하는 경우

### 사용 방법:

**원본 워드프레스 사이트에서:**

All-in-One WP Migration 플러그인을 설치하고 활성화합니다.

- 웹사이트를 .wpress 파일로 내보냅니다.

![All-in-One WP Migration](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795282/original/z79lV8uAIHfi7tafthpDYwPXaz291fRX1A.png?1745866002)

**Hyperclass에서:**

하위 계정의 WordPress(워드프레스) 탭 > Import Your Existing Website(기존 웹사이트 가져오기)로 이동합니다.

- Choose File(파일 선택)을 클릭하고 .wpress 파일을 업로드합니다 (최대: 2GB).

- 업로드 완료를 기다립니다.

- 완료되면 내보낸 파일의 모든 콘텐츠로 사이트가 자동 배포됩니다.

⚠️ 이 옵션은 증분 업데이트를 지원하지 않습니다. 대상 사이트를 완전히 덮어씁니다.

## 옵션 3: 화이트 글러브 마이그레이션 지원 (에이전시 전용)

**적합한 경우**: 복잡한 마이그레이션, 대규모 사이트, 또는 손을 대지 않고 처리하고 싶은 경우  
**참고**: 에이전시 소유자에게만 표시됩니다

### 사용 방법:

Import Your Existing Website(기존 웹사이트 가져오기) 섹션에서 Request Help(도움 요청)를 클릭합니다.

다음 정보를 포함한 요청 양식을 작성합니다:

- 원본 웹사이트 URL
- 호스팅 제공업체
- 접근 자격 증명
- 대상 Hyperclass 하위 계정
- 희망 마이그레이션 마감일

양식을 제출합니다.

**워드프레스 전문가가 다음을 수행합니다:**

- 1영업일 이내에 요청을 검토합니다.
- 추가 정보가 필요한 경우 연락드립니다.
- 안전하고 종합적으로 마이그레이션을 처리합니다.
- 마이그레이션 완료 시 알림 이메일을 보내드립니다.

이는 화이트라벨 경험입니다. 고객은 이 옵션을 보지 못하며 저희가 관여한다는 것을 알 수 없습니다.

![화이트 글러브 마이그레이션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045795299/original/X8wifSFSZje30hoQ3UVot_y02E6lLxrqbg.png?1745866031)

## 어떤 옵션을 선택해야 하나요?

| 상황 | 최적 옵션 |
|------|----------|
| 가장 빠른 셀프 서비스 방법 | 손쉬운 플러그인 마이그레이션 |
| .wpress 파일이 준비된 경우 | 수동 (AIO) 마이그레이션 |
| 기술적인 부분을 건드리고 싶지 않은 경우 | 화이트 글러브 마이그레이션 지원 |
| 여러 사이트를 마이그레이션해야 하는 경우 | 플러그인 또는 화이트 글러브 (선호) |
