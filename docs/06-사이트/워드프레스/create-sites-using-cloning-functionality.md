# 사이트 복제 기능으로 사이트 만들기

사이트 복제 기능을 사용하면 기존 워드프레스 사이트를 콘텐츠, 설정, 테마, 플러그인까지 모두 포함해서 빠르게 복사할 수 있습니다. 이 가이드는 단계별로 이 기능을 활용하는 방법을 안내하여 원본 설정을 그대로 유지하면서 효율적으로 복제할 수 있도록 도와드립니다. 에이전시 소유자, 사용자, 같은 하위 계정(Sub-account) 내 계정 사용자에게 이상적입니다. 이 복제 기능으로 만들어진 각 사이트는 독립적으로 작동하며, 원본 사이트에 영향을 주지 않고 유연하게 관리할 수 있습니다.

# **1단계: 워드프레스 대시보드로 이동**

워드프레스 대시보드에 접근합니다.

- GoHighLevel 계정에 로그인한 후 워드프레스 섹션으로 이동하세요.
- **Create New** 또는 **Select Blank WordPress**를 클릭하세요.

[**참고**: 복제 옵션을 보려면 적절한 역할 기반 권한(에이전시 소유자, 에이전시 사용자, 또는 계정 사용자)이 있어야 합니다.]

![워드프레스 대시보드 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155036263447/original/bJxAs8rRN1QxT6ddV72jCCJhmHeP9csUrQ.png?1731085495)

# **2단계: 사이트 복제 프로세스 시작**

기존 사이트를 이용한 복제 옵션 또는 미완료 설정 카드를 찾습니다.

- **Incomplete Setup Card**를 클릭하세요.
- **"Choose From Existing Site"** 옵션을 선택하세요.

[**팁**: 복제 옵션이 보이지 않는다면 사용자 권한이 부족한 것일 수 있습니다.]

![사이트 복제 옵션 선택 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155036263483/original/aOMQkm3b6832CBkgq41Q6izG8IdndEWKiA.png?1731085522)

# **3단계: 소스 사이트 선택 및 복제 설정**

하위 계정 및 소스 사이트를 선택합니다.

- 드롭다운 메뉴에서 **하위 계정(Sub-Account)**을 선택하세요.
- 복제하고 싶은 **소스 사이트(Source Site)**를 선택하세요.

새 사이트 세부 정보를 입력합니다.

- **New Website Name** 필드에 복제할 새 사이트의 이름을 입력하세요.
- **Clone** 버튼을 클릭하여 프로세스를 시작하세요.

[**참고:**
- 실시간 진행률 표시기를 통해 복제 상태를 모니터링할 수 있습니다.
- 권한이 복제 요구사항에 맞지 않으면 시스템이 표준 새 사이트 생성 플로우로 자동 전환됩니다.]

# 4단계: 검토 및 완료

**새로 복제된 사이트에 접근하기**

- 복제된 사이트는 원본 사이트와 완전히 독립적으로 작동합니다.
- 어느 한 사이트에 변경사항을 적용해도 다른 사이트에는 영향을 주지 않습니다.

[**참고:**
- 복제된 사이트는 Hyperclass 대시보드의 다른 워드프레스 사이트와 마찬가지로 관리할 수 있습니다.]

# 유용한 팁

## 주제 1: 사용자 권한 및 접근 제어

- 특정 역할을 가진 사용자(에이전시 소유자, 에이전시 사용자, 같은 하위 계정 내 계정 사용자)만 복제 기능에 접근할 수 있습니다.
- 사이트를 복제하기 전에 사용자 역할에 필요한 권한이 있는지 확인하세요.

## 주제 2: 일반적인 문제 해결

- **문제**: 복제 옵션이 보이지 않나요?
  **해결책**: 사용자 역할 권한을 확인하세요. 에이전시 또는 계정 수준의 접근 권한으로 로그인했는지 확인하세요.
- **문제**: 복제 프로세스가 멈췄나요?
  **해결책**: 대시보드를 새로고침하고 진행률 표시기를 확인하세요. 문제가 지속되면 고객지원팀에 문의하세요.

[자세히 알아보기](https://wordpress.com/support/invite-people/user-roles/) - 사용자 역할 및 권한 관리에 대한 상세 문서
