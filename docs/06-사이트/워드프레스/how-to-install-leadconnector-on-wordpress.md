# 워드프레스에 LeadConnector 설치하기

이 가이드에서는 Hyperclass 호스팅이 새로 생성된 워드프레스 사이트에 LeadConnector(LC) 플러그인을 자동으로 설치하는 방법을 설명합니다. 또한 Hyperclass의 플러그인 상태 표시기와 플러그인 연결이 끊어졌을 때 OAuth를 통해 다시 연결하는 방법도 다룹니다.

**중요**: **Hyperclass 호스팅**을 통해 새 워드프레스 사이트를 생성하면, Hyperclass가 설정 중에 **LeadConnector(LC) 플러그인**을 자동으로 설치합니다.

---

## **LC 워드프레스 플러그인이란?**

LC 워드프레스 플러그인을 사용하면 워드프레스 사이트 내에서 Hyperclass의 주요 기능을 직접 삽입하고 관리할 수 있습니다. 이 연동을 통해 라이브 채팅 경험, 리드 퍼널, 이벤트 추적을 더 쉽게 제어할 수 있으며, 플랫폼을 전환하거나 임베드 코드를 수동으로 관리할 필요가 없습니다.

**Hyperclass 호스팅**을 통해 새 워드프레스 사이트를 생성하면, Hyperclass가 설정 중에 **LeadConnector(LC) 플러그인**을 자동으로 설치합니다.

또한 Hyperclass는 대시보드에 **플러그인 상태 표시기**를 제공하므로, 워드프레스에 로그인하지 않고도 플러그인이 연결되었는지 확인할 수 있습니다.

---

## **워드프레스 LC 플러그인의 주요 장점**

이번 업데이트에서는 채팅 위젯 활성화를 위한 새로운 컨트롤, 여러 위젯 지원, 더 나은 성능과 분석을 위한 기반 개선사항을 도입했습니다.

- **워드프레스 관리자 내에서** 채팅 위젯을 직접 **활성화/비활성화** 가능
- **드롭다운 선택기**를 통해 여러 채팅 위젯 중에서 선택 가능 (한 번에 하나만 활성화)
- 선택된 위젯이 Hyperclass 대시보드와 **자동 동기화**되어 일관성 보장
- 검색 엔진 가시성 향상을 위해 **퍼널 SEO 메타 태그 재활성화**
- 플러그인 사용 행동 모니터링을 위한 **Pendo 이벤트 추적 추가**
- 예약된 작업(cron) 안정성을 포함한 **백엔드 안정성 개선**

## **워드프레스에서 채팅 위젯 활성화 및 관리하기**

이제 LC 플러그인 설정에서 채팅 위젯의 표시 여부와 동작을 쉽게 제어할 수 있습니다. 임베드 코드를 복사/붙여넣기하거나 매번 Hyperclass 대시보드에 접속할 필요가 없습니다.

플러그인 상태 표시기를 사용하여 LC 플러그인이 연결되었는지 확인하세요.

1. Hyperclass 계정에서 **사이트(Sites) → 워드프레스(WordPress)**로 이동하세요.

2. 확인하려는 호스팅된 워드프레스 사이트를 여세요.

3. **LeadConnector(LC)**의 **플러그인 상태(Plugin Status)** 표시기를 찾으세요.

![플러그인 상태 표시기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063939335/original/ciM02SNNPhCJXCJnC5f8HZYvZScb4HSQ1A.png?1769976933)

4. 표시된 상태를 확인하세요:

-    * **연결됨(Connected)**

![연결됨 상태](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063939287/original/Bt_abGXCcYfa9zttPAEFy5izDILN0SrO7w.png?1769976711)

-    * **연결 안됨(Not Connected)**

![연결 안됨 상태](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063939273/original/JPbFFs6QtWuc1OdfMmAbrY8AMw95Uq6lNQ.png?1769976649)

-    * **설치됨(Installed)**
-    * **연결됨 + 활성(Connected + Active)**
-    * **연결됨 + 비활성(Connected + Not Active)**

![연결됨 + 비활성 상태](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063939308/original/qYypqVVO22U3DAE7-7DFDY1NyIeXMSYyMw.png?1769976772)

해결된 문제: 이전에는 Hyperclass 대시보드에서 선택한 위젯이 사이트에서 항상 업데이트되지 않는 문제가 있었습니다. 이 문제가 이제 해결되었습니다. 플러그인은 이제 항상 대시보드의 현재 선택과 변경사항을 반영합니다.

## **퍼널에서 SEO 메타 태그 재활성화**

메타 태그는 검색 엔진 결과에서 퍼널 페이지의 가시성을 향상시키는 데 도움이 됩니다. 이번 업데이트에서 Hyperclass는 퍼널 페이지의 SEO 태그 지원을 재활성화했습니다.

### 장점:

- 퍼널 페이지에 이제 `<title>`, `<meta name="description">` 및 기타 주요 태그가 포함됩니다.
- 검색 엔진에서의 검색 가능성과 클릭률이 향상됩니다.
- 수동 재구성이 필요하지 않습니다. 퍼널 설정에서 SEO 설정이 활성화되어 있는지만 확인하세요.

![SEO 메타 태그 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048560647/original/5p95qE16Gj1jUb3FpHzA3Nsw9ZTsPw0rdw.png?1750344254)

## **Pendo를 통한 플러그인 사용 추적**

사용자가 플러그인과 어떻게 상호작용하는지 더 잘 이해하기 위해 Pendo 이벤트 추적이 추가되었습니다.

### **추적 항목:**

- LC 플러그인 인터페이스 내에서의 사용자 상호작용
- 토글, 드롭다운, 플러그인 참여에 대한 사용 데이터

**참고**: 이 데이터는 Hyperclass 팀이 제품을 개선하는 데 사용되며, 현재 사용자가 직접 접근할 수는 없습니다.

---

## **백엔드 개선사항 및 버그 수정**

다양한 백그라운드 업그레이드를 통해 플러그인이 더욱 견고하고 확장 가능해졌습니다.

- **인증 크론 수정**: 예약된 작업(위젯 동기화 등)이 이제 안정적으로 실행됩니다.
- **플러그인 아키텍처 리팩토링**: 향후 개선사항과 유지보수 편의성을 지원하기 위해 내부 구조를 단순화하고 최적화했습니다.

이러한 업데이트는 성능을 향상시키고 버그나 업데이트 실패 가능성을 줄입니다.

## **자주 묻는 질문**

**Q: 한 번에 여러 채팅 위젯을 활성화할 수 있나요?**
아니오. 한 번에 하나의 위젯만 선택하고 삽입할 수 있습니다.

**Q: Hyperclass 대시보드에서 선택한 위젯에 대한 업데이트가 내 사이트에 반영되나요?**
네. 이제 대시보드에서 변경될 때마다 위젯 구성이 안정적으로 동기화됩니다.

**Q: 퍼널에서 SEO 태그를 수동으로 다시 활성화해야 하나요?**
아니오. 기능이 자동으로 복원됩니다. 퍼널 SEO 설정이 올바르게 구성되어 있는지 확인하세요.

**Q: Pendo 추적은 선택사항인가요?**
아니오. 백그라운드에서 자동으로 실행되며 사이트 성능에 영향을 주지 않고 사용자 구성이 필요하지 않습니다.

**Q: 이러한 업데이트가 이전 버전의 플러그인에 영향을 주나요?**
이러한 개선사항을 활용하려면 최신 버전으로 업데이트하는 것을 권장합니다.

**Q: LC 플러그인을 어떻게 설치하나요? / Q: 어떻게 다시 연결하나요?**

답변은 자동 설치, OAuth 연결 CTA, 범위(새 사이트만 해당)를 반영해야 합니다.
