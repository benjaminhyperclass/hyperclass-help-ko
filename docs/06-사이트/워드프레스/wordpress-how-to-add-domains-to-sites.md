# 워드프레스 - 사이트에 도메인 추가하는 방법

# **Hyperclass에서 워드프레스 사이트에 도메인 추가란?**

워드프레스 도메인 관리는 구매한 도메인(예: example.com)을 Hyperclass 워드프레스 사이트에 연결합니다. 시스템이 DNS 레코드를 검증하고 SSL을 발급하여 방문자가 안전하고 브랜드화된 웹사이트에 접근할 수 있게 합니다. 기본 도메인, 서브도메인, DNS 검증이 어떻게 작동하는지 이해하면 사이트의 안정적이고 안전한 연결을 보장할 수 있습니다.

**참고:** 도메인 관리는 **라이브 환경에서만 사용 가능**합니다.

스테이징 환경에서는 도메인 영역이 비활성화되어 "**이 기능은 라이브 환경에서만 사용할 수 있습니다. 접근하려면 전환하세요**"라고 표시됩니다.

우측 상단 헤더 액션 바의 환경 선택기를 사용하여 라이브로 전환하세요.

![환경 선택기 위치](https://jumpshare.com/share/1I4xuG6ynV5ETXtCcls4+/Screenshot+2026-02-11+at+5.38.08%E2%80%AFPM.png)

## **워드프레스 사이트에 도메인 추가의 주요 장점**

- **기본 도메인 및 SSL 발급:** DNS 검증 후 기본 도메인에 대해 자동으로 SSL이 발급되어 안전한 HTTPS 접근이 가능합니다.

- **다중 도메인 지원:** 필요에 따라 추가 도메인이나 서브도메인을 연결하여 트래픽을 라우팅할 수 있습니다(제품 제한 내에서).

- **서브도메인 유연성:** 기본 도메인에 사용되는 SSL TXT 검증 단계 없이 서브도메인(예: blog.example.com)을 매핑할 수 있습니다.

- **도메인 연결 옵션:** 지원되는 등록업체와 인증하여 수동 레코드 편집 없이 DNS를 자동 구성할 수 있습니다.

- **명확한 상태 표시:** DNS/SSL 상태를 한눈에 확인하여 문제를 빠르게 해결하고 확신을 가지고 서비스를 시작할 수 있습니다.

## **사전 요구사항**

- 이러한 항목을 미리 확인하면 설정 오류를 방지하고 SSL 발급 속도를 높이며 서비스 시작 중 다운타임을 줄일 수 있습니다.

- 라이브 환경에서 작업 중입니다(우측 상단 헤더의 환경 선택기를 사용하여 스테이징에서 전환).

- 워드프레스 사이트가 올바른 하위 계정에 이미 프로비저닝되어 있습니다.

- 등록업체나 DNS 호스트(예: Cloudflare, GoDaddy, IONOS)에서 도메인의 DNS 설정에 접근할 수 있습니다.

- DNS 제공업체가 Cloudflare인 경우 프록시 설정을 끄세요.

## **Hyperclass에서 워드프레스 사이트 도메인 추가 방법**

- 하위 계정에 로그인합니다.

- **Sites(사이트)** > **WordPress(워드프레스)**로 이동합니다.

![사이트 메뉴](https://jumpshare.com/share/NGK21FCbtFdfuiqL3xej+/Screen+Shot+2026-02-11+at+5.15.43+PM.png)

- 도메인을 추가할 사이트의 **Manage Website(웹사이트 관리)** 버튼을 클릭합니다.

![웹사이트 관리 버튼](https://jumpshare.com/share/ZUR1k8q62GCmfoC2wPcp+/Screen+Shot+2026-02-11+at+5.17.23+PM.png)

- 사이트 관리 대시보드에서 Manage Your Domain(도메인 관리) 섹션으로 이동하여 + Add Domain(+ 도메인 추가)을 클릭합니다.

![도메인 추가 버튼](https://jumpshare.com/share/Ov0iQQe0ATfMbI8GJv1C+/Screen+Shot+2026-02-11+at+6.49.15+PM.png)

- 도메인 이름 설정: 원하는 도메인 이름을 입력합니다.

- **Auto Configure(자동 구성)** 또는 **Add Manually(수동 추가)**를 선택합니다. (시스템이 도메인이 이미 다른 로케이션이나 에이전시와 연결되어 있는지 자동으로 확인합니다.)

- 도메인을 사용할 수 있으면 다음 단계로 진행합니다.

![도메인 구성 옵션](https://jumpshare.com/share/ctti3dvwRVyj0RXnD6GT+/Screen+Shot+2026-02-11+at+6.52.48+PM.png)

- 도메인이 검증된 후 **SSL 인증서**를 구성해야 합니다.

- 제공된 **TXT 레코드**를 GoDaddy, Hostinger, Namecheap 등의 **DNS 제공업체**에 추가합니다.

- 대부분의 SSL 레코드는 한 시간 내에 전파되지만, **전 세계 업데이트는 최대 48시간**이 걸립니다. 최적의 SSL 레코드 전파를 위해 **TTL 값을 600ms 또는 가능한 가장 낮은 값으로 구성**하세요.

- TXT 레코드가 **성공적으로 추가되고 검증되면** 마지막 단계로 넘어갈 수 있습니다.

![SSL TXT 레코드 추가](https://jumpshare.com/share/4IIbR8naF6wr7la8ErGY+/jvDFyZPA87UqHBxsEtgj1B8l9Ct5TZujug.png)

- 제공된 **CNAME** 및 **A 레코드**를 **DNS 제공업체**에 추가합니다.

- 레코드가 성공적으로 추가되면 '**Verify DNS Records(DNS 레코드 확인)**' 버튼을 클릭합니다.

![DNS 레코드 확인](https://jumpshare.com/share/vr5hZ7tZJiLnsY3pJBtl+/cDKrS-1v1iSnRRwt6-b5ZuBDGu75RefZ9w.png)

## **워드프레스 사이트에 서브도메인 추가 방법**

서브도메인 추가는 도메인 추가와 비슷합니다. 추가 도메인의 SSL 인증서는 DNS 업데이트 단계에서 검증되므로 TXT 레코드 추가는 필요하지 않습니다.

- 사이트 관리 대시보드에서 **Manage Your Domain(도메인 관리)** 섹션으로 이동하여 **+ Add Domain(+ 도메인 추가)**를 클릭합니다.

![서브도메인 추가](https://jumpshare.com/share/Ov0iQQe0ATfMbI8GJv1C+/Screen+Shot+2026-02-11+at+6.49.15+PM.png)

- 도메인 이름 설정: 원하는 도메인 이름을 입력합니다.

- **Auto Configure(자동 구성)** 또는 **Add Manually(수동 추가)**를 선택합니다. (시스템이 도메인이 이미 다른 로케이션이나 에이전시와 연결되어 있는지 자동으로 확인합니다.)

- 시스템이 도메인이 이미 다른 로케이션이나 에이전시와 연결되어 있는지 자동으로 확인합니다. 도메인을 사용할 수 있으면 DNS 레코드 업데이트의 마지막 단계로 진행합니다.

![서브도메인 구성](https://jumpshare.com/share/ctti3dvwRVyj0RXnD6GT+/Screen+Shot+2026-02-11+at+6.52.48+PM.png)

- 제공된 **CNAME** 및 **A 레코드**를 **DNS 제공업체**에 추가합니다.

- 레코드가 성공적으로 추가되면 '**Verify DNS Records(DNS 레코드 확인)**' 버튼을 클릭합니다.

![서브도메인 DNS 확인](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064692102/original/2um_cn8g9ewwAHEArYffzvuAf3_I1G8MlA.png?1770817791)

## **종합 도메인/서브도메인 대시보드**

새로운 대시보드는 도메인 관리의 종합적인 개요를 제공합니다.

- **SSL 발급 상태:** 'SSL issued/SSL Not Issued(SSL 발급됨/SSL 미발급)' 태그를 사용하여 SSL 인증서 발급 상태를 모니터링하고 올바르게 구성되었는지 확인합니다.

- **DNS 레코드 검증:** 'Verified/Not Verified(검증됨/미검증)' 태그를 사용하여 DNS 레코드의 검증 상태를 확인하여 올바르게 설정되었는지 확인합니다.

- **도메인 관리:** 최대 5개의 도메인 또는 서브도메인을 추가할 수 있습니다. 더 나은 조직을 위해 하나를 기본 도메인으로 지정합니다.

- **쉬운 접두사 조정:** 'Change prefix to WWW/Remove WWW Prefix(WWW 접두사로 변경/WWW 접두사 제거)' 옵션을 사용하여 대시보드에서 간단한 조정으로 www와 non-www 접두사를 쉽게 전환할 수 있습니다.

![도메인 대시보드](https://jumpshare.com/share/UyN2sWamlXhG9AaKeTzP+/S-oyWWLKWjNB7CPKvMB6GC8XSmGRHD_6KQ.png)

## **자주 묻는 질문**

**Q: 기본 도메인은 SSL TXT 레코드가 필요한데 서브도메인은 왜 필요하지 않나요?**

SSL TXT 검증 단계는 인증서 발급을 위한 최상위 도메인의 제어권을 확인합니다. 서브도메인은 기본 도메인의 소유권이 검증된 후에 따라갑니다.

**Q: DNS 전파는 얼마나 걸리나요?**

제공업체에 따라 다릅니다. 많은 변경 사항이 몇 분 내에 나타나지만 일부는 더 오래 걸릴 수 있습니다. TTL을 약 600초로 설정했다면 10-30분 내에 다시 확인해보세요.

**Q: 내 도메인이 이미 다른 Hyperclass 로케이션에 연결되어 있다면 어떻게 해야 하나요?**

현재 소유자에게 해당 로케이션에서 도메인을 제거해달라고 요청하거나, 소유권이 불분명하다면 지원팀에 도움을 요청하세요.

**Q: A 레코드와 CNAME 레코드가 모두 필요한가요?**

사이트를 위해 UI에 표시된 레코드를 정확히 추가하세요. 일부 설정은 둘 다 필요하고, 다른 설정은 한 가지 유형만 사용할 수 있습니다. 값은 설치마다 고유합니다.
