# SEO - 사이트 프로젝트 삭제하는 방법

## SEO 사이트 프로젝트 삭제란?

사이트 프로젝트를 삭제하면 해당 프로젝트의 감사 결과가 영구적으로 제거되고, 그 프로젝트를 통해 적용된 SEO 수정사항이 자동으로 되돌려지며, 사용 가능한 할당량이 즉시 복원되어 새로운 사이트를 추가할 수 있습니다.

**참고:** 사이트 프로젝트 삭제는 **영구적**이며 **되돌릴 수 없습니다**.

## 사이트 프로젝트 삭제의 주요 장점

- **할당량 확보:** 4개 사이트 한도에 도달했을 때 즉시 한 자리를 확보합니다.
- **운영 유연성:** 새로운 고객 사이트나 높은 우선순위의 속성을 빠르게 교체할 수 있습니다.
- **깔끔한 롤백:** 프로젝트에서 자동 적용된 SEO 수정사항이 되돌려져 남은 변경사항을 방지합니다.
- **명확성:** SEO 작업 공간을 활성 상태이고 관련성 있는 도메인에 집중하도록 유지합니다.

## SEO 사이트 프로젝트 삭제 방법

다음 단계를 따르면 할당량 즉시 복원과 적용된 수정사항의 자동 롤백과 함께 깔끔하고 예측 가능한 제거가 보장됩니다.

- **Sites(사이트)** > **SEO**로 이동합니다.

![SEO 메뉴 이동](https://jumpshare.com/share/v5ZeGdULNUEzEWWayMRx+/Screen+Shot+2026-01-28+at+7.32.45+PM.png)

- **Site Projects(사이트 프로젝트)** 목록을 열고 제거할 프로젝트를 찾습니다.

- 프로젝트 이름 옆의 **점 세 개**(⋯)를 클릭하고 Delete Project를 클릭합니다.

![프로젝트 삭제 메뉴](https://jumpshare.com/share/3gI7uM0eZc47hYsJg40N+/Screen+Shot+2026-01-28+at+7.35.23+PM.png)

- **확인 모달**을 **검토**하고 **영향 요약**(할당량 복원, 수정사항 되돌림, 영구 작업)을 읽어보세요.

- **Delete Project(프로젝트 삭제)** 버튼을 클릭합니다.

![삭제 확인 모달](https://jumpshare.com/share/vRyz7Te7RVzLMHPhtumg+/Screen+Shot+2026-01-28+at+7.40.46+PM.png)

## SEO 수정사항 롤백

무엇이 되돌려지는지 이해하면 라이브 자산에서 예상치 못한 변경을 방지하고 수동으로 다시 적용해야 할 사항을 명확히 할 수 있습니다.

사이트 프로젝트를 삭제하면 해당 프로젝트를 통해 적용된 SEO 수정사항이 되돌려집니다:

- 퍼널 경로 수정사항
- 블로그 경로 수정사항
- 외부 수정사항 (프로젝트에 의해 트리거된 자동 조정)
- 프로젝트의 자동 적용 수정사항과 관련 없는 수동 편집은 일반적으로 유지됩니다.

삭제 후, 여전히 필요하다면 주요 항목들(리다이렉트, 사이트맵, Search Console 커버리지)을 재검증하는 것을 고려해보세요.

## 자주 묻는 질문

**Q: 고객지원에서 삭제된 사이트 프로젝트를 복원할 수 있나요?**

아니요. 삭제는 영구적이며 되돌릴 수 없습니다.

**Q: 수동으로 변경한 메타 제목/설명이 되돌려지나요?**

사이트 프로젝트를 통해 적용된 수정사항만 되돌려집니다. 해당 수정사항과 관련 없는 수동 편집은 일반적으로 유지됩니다.

**Q: 누가 사이트 프로젝트를 삭제할 수 있나요?**

충분한 권한(일반적으로 관리자 또는 사이트/SEO 접근 권한이 있는 사용자)을 가진 사용자가 삭제할 수 있습니다.

**Q: 삭제 후 같은 도메인을 다시 추가할 수 있나요?**

네. 할당량이 복원되면 해당 도메인을 새로운 사이트 프로젝트로 추가하고 새로운 감사를 시작할 수 있습니다.

**Q: 삭제가 프로젝트의 수정사항 외에 라이브 퍼널/블로그에 영향을 주나요?**

프로젝트의 자동 적용 수정사항을 제거합니다. 관련 없는 다른 콘텐츠와 수동 편집은 변경되지 않은 채로 유지됩니다.

**Q: 할당량 설정을 어떻게 확인할 수 있나요?**

**SEO** > **Settings(설정)** > Quota Usage에서 할당량을 확인할 수 있습니다.
