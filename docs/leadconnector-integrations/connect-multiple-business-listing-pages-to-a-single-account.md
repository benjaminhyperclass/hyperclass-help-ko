# 하나의 계정에 여러 비즈니스 리스팅 페이지 연결하기

여러 비즈니스 리스팅 페이지를 연결하면 여러 위치의 고객 메시지, 신규 리드, 리뷰를 한 곳에서 관리할 수 있어요. 리스팅이나 계정을 번갈아 확인할 필요 없이, 모든 고객 참여 데이터를 하나의 작업 공간에서 보고, 모니터링하고, 응답할 수 있습니다. 이 설정은 여러 위치나 서비스 지역을 운영하면서 일관된 가시성과 중앙화된 커뮤니케이션을 원하는 비즈니스에 특히 유용해요.

여러 페이지가 연결되면 어떤 페이지에서 메시지를 받을지 제어하고, 선택적으로 통화 추적을 활성화하며, 각 페이지의 설정을 명확성과 제어력을 잃지 않고 관리할 수 있어요.

# 연결 프로세스 시작하기

![연동 관리 영역](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063312353/original/5TzsUEshIkqApPoRWTfgh-Njqh3Z89cSEA.png?1769157476)

이 이미지는 다양한 외부 연결을 관리하는 메인 연동(Integration) 영역을 보여줍니다. 여기서 비즈니스 리스팅 페이지 연결을 시작하고 기존 연결을 관리할 수 있어요.

시작하려면 계정 설정(Settings) 안의 연동(Integration) 섹션으로 이동하세요. 이곳에서 모든 외부 서비스와 페이지 연결을 시작하고 관리할 수 있어요.

---

## **새로운 비즈니스 리스팅 연결 시작**

![연결 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063312648/original/QP_1MwuA6uCw1aKhwFTbJaMD69azL0Mi5Q.png?1769157585)

이 이미지는 비즈니스 리스팅 페이지의 인증 프로세스를 시작하는 연결 버튼을 강조 표시합니다.

연결 옵션을 선택하면 관리 권한이 있는 모든 비즈니스 리스팅 페이지를 가져오는 안전한 인증 흐름이 시작돼요. 관리 권한이 있는 페이지만 선택할 수 있어요.

---

## **연결할 페이지 선택**

![페이지 선택 목록](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063312841/original/8r8YDiMCKrQdOWVrqhl8Ov885YjjRq35gg.png?1769157643)

이 이미지는 연결할 수 있는 비즈니스 리스팅 페이지 목록을 표시합니다.

이 단계에서 하나의 페이지 또는 여러 페이지를 선택할 수 있어요. 선택된 각 페이지는 계정에 연결되어 메시지 처리, 리드 가시성, 리뷰 모니터링에 사용할 수 있게 돼요.

---

## **메시징 및 통화 설정 구성**

![메시징과 통화 설정 토글](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063313074/original/vpGxxCHQq1HB0kccDwDepm2axFWRzoYEFA.png?1769157742)

이 이미지는 연결된 각 페이지에 대해 메시징과 통화 추적을 활성화하거나 비활성화하는 토글을 보여줍니다.

각 페이지에 대해 메시징을 독립적으로 활성화할 수 있어서, 리스팅에서 보낸 고객 메시지가 작업 공간에 나타나게 할 수 있어요. 통화 추적도 활성화할 수 있지만, 한 번에 하나의 페이지에서만 통화 추적을 활성화할 수 있어요. 다른 페이지에서 이미 통화 추적이 활성화되어 있다면, 새 페이지에서 활성화하기 전에 그곳에서 먼저 비활성화해야 해요.

---

## **연결 완료 및 확인**

선택된 페이지가 연결되는 확인 단계입니다.

선택 사항과 기본 설정을 지정했다면, 연결을 완료하여 페이지를 계정에 연결하세요. 페이지가 활성화되기 전에 시스템이 잠시 요청을 처리할 수 있어요.

---

## **연결된 페이지 및 상태 확인**

![연결된 페이지와 상태](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063313498/original/cPzXKn7ssV2l7Sntozes1yhl5wiogNiJKA.png?1769157877)

이 이미지는 연결된 페이지와 현재 상태를 표시합니다.

연결이 성공하면 모든 연결된 페이지가 한 곳에 나열돼요. 여기서 어떤 페이지가 활성화되어 있는지, 어떤 기능이 활성화되어 있는지 확인하고, 필요에 따라 페이지를 관리하거나 연결 해제하는 옵션에 액세스할 수 있어요.

---

## **페이지 업데이트 또는 연결 해제**

![설정 업데이트 및 연결 해제 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063313550/original/0nivqqSlnYqgoJvP8KnjpEyQqr_7JkmwAg.png?1769157917)

이 이미지는 설정을 업데이트하거나 페이지 연결을 완전히 해제하는 옵션을 보여줍니다.

언제든지 페이지 설정을 수정하고, 메시징이나 통화 기본 설정을 변경하거나, 더 이상 필요하지 않은 페이지의 연결을 해제할 수 있어요. 연결 해제는 계정에서 페이지를 제거하지만 리스팅 자체에는 영향을 주지 않아요.

---

## **중요 고려사항**

관리 권한이 있는 페이지만 연결할 수 있어요. 메시징은 여러 페이지에서 동시에 활성화할 수 있지만, 통화 추적은 계정당 하나의 페이지로 제한돼요. 설정 변경 사항이 공개적으로 반영되는 데 약간의 시간이 걸릴 수 있어요.

---

## **자주 묻는 질문**

**Q: 동시에 둘 이상의 비즈니스 리스팅 페이지를 연결할 수 있나요?**

네. 관리 권한이 있는 한 설정 과정에서 여러 페이지를 선택하고 연결할 수 있어요.

**Q: 연결된 모든 페이지에 대해 메시징을 활성화할 수 있나요?**

네. 기본 설정에 따라 각 페이지에 대해 메시징을 개별적으로 켜거나 끌 수 있어요.

**Q: 통화 추적이 한 페이지에서만 활성화되는 이유가 뭔가요?**

통화 추적은 정확한 라우팅과 리포팅을 보장하기 위해 한 번에 하나의 페이지에서만 작동하도록 설계되어 있어요.

**Q: 페이지 연결을 해제하면 어떻게 되나요?**

페이지 연결을 해제하면 계정 뷰에서 제거돼요. 해당 페이지의 메시지, 통화, 리뷰는 더 이상 나타나지 않아요.

**Q: 필요한 경우 나중에 페이지를 다시 연결할 수 있나요?**

네. 연결 해제된 페이지는 연결 프로세스를 반복하여 언제든지 다시 연결할 수 있어요.

---

## **더 많은 도움이 필요한가요?**

페이지 연결, 메시징이나 통화 설정 조정, 연결 문제 해결에 도움이 필요하시면, 계정 내 도움말 섹션을 열고 관련 문서를 검색하거나, 안내형 지원을 받기 위해 고객 지원팀에 문의하세요.
