# 리포트 스케줄러로 하위 계정 자동 이메일 리포트 설정하기

리포트 스케줄러(Report Scheduler)를 사용하면 에이전시 사용자가 하위 계정 관리자와 사용자에게 일간, 주간 또는 월간 성과 리포트를 자동으로 전송할 수 있습니다. 이 기능은 필수 계정 지표를 이메일로 직접 전달하여 팀이 지속적으로 정보를 얻고 협력하며 능동적으로 대응할 수 있도록 도와줍니다. 이 글에서는 리포트 스케줄러가 무엇인지, 어떤 리포트를 보내는지, 그리고 자동 리포팅을 예약하고 관리하는 방법을 안내해드립니다.

에이전시 뷰 예약 기능은 월 $497 이상 요금제에서 사용할 수 있습니다.

에이전시 뷰가 없는 하위 계정 사용자도 해당 하위 계정 내에서 로컬 일정은 관리할 수 있습니다.

# 리포트 스케줄러란?

리포트 스케줄러는 하위 계정에 성과 리포트를 자동으로 전송하는 하이퍼클래스의 기능입니다. 주요 대시보드 지표를 하나의 이메일로 통합하여 에이전시가 설정한 빈도와 시간에 맞춰 전송합니다. 이를 통해 사용자는 여러 대시보드에 일일이 로그인하지 않고도 계정 상태를 일관되게 모니터링할 수 있습니다.

리포트 스케줄러를 이해하면 에이전시가 하위 계정 관리자 및 사용자와의 소통을 간소화할 수 있습니다. 대시보드를 수동으로 내보내거나 확인하는 대신, 자동 리포트를 통해 시기적절한 인사이트를 제공하고 정보에 기반한 의사결정을 장려하며 수신자가 계정 성과와 연결된 상태를 유지할 수 있습니다.

리포트에는 다음 대시보드의 데이터가 포함됩니다:

- 사용자(Users)
- 연락처(Contacts)  
- 아웃리치(Outreach)
- 제출(Submissions)
- SMS
- 예약(Appointments)
- 거래 개요(Deal Overview)
- 이메일(Email)
- 지연된 할 일(Tasks Overdue)
- 대화(Conversations) (읽지 않음 + 놓친 대화)

## 리포트 스케줄러의 주요 장점

자동 리포팅의 장점을 인식하면 에이전시와 하위 계정 팀이 예약된 리포트 활성화의 가치를 이해할 수 있습니다. 이러한 장점은 기능이 어떻게 책임감을 강화하고 가시성을 향상시키며 전반적인 운영 효율성을 높이는지 보여줍니다.

- **자동화**: 수동 리포팅을 없애고 에이전시와 클라이언트의 시간을 절약합니다
- **맞춤 설정**: 사용자가 빈도, 시작 날짜, 시간, 수신자를 선택할 수 있습니다
- **일관된 업데이트**: 이해관계자가 정기적인 성과 인사이트를 받을 수 있도록 보장합니다
- **중앙 집중식 정보**: 여러 대시보드를 하나의 이해하기 쉬운 리포트로 결합합니다
- **향상된 소통**: 관리자가 정보를 얻고 계정 활동과 일치하도록 유지합니다

## 리포트 예약 방법

올바른 설정은 리포트가 적절한 시간에 의도된 수신자에게 도달하도록 보장하여, 불필요한 지연이나 설정 오류 없이 팀이 정보를 얻을 수 있도록 도와줍니다.

### **에이전시 뷰로 전환**

이 단계는 사용자가 계정 전환기에서 **Switch to Agency View(에이전시 뷰로 전환)**을 선택하여 에이전시 레벨 대시보드로 돌아가는 방법을 보여줍니다. 이 옵션을 클릭하면 개별 하위 계정에서 나와 리포트 예약 기능을 관리하는 메인 에이전시 인터페이스로 돌아갑니다.

![에이전시 뷰로 전환](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059899800/original/Z_f176FlGVEEg5mLMvn4srUcOhItbCHHjQ.png?1764853299)

### **하위 계정 열기**

에이전시 뷰에서 왼쪽 사이드바의 **Sub-Accounts(하위 계정)**으로 이동하여 에이전시에서 관리하는 모든 클라이언트 로케이션에 접근합니다. 하위 계정 목록이 나타나면, 예약된 리포트를 생성하거나 관리하려는 특정 로케이션의 **Reports(리포트)** 버튼을 클릭합니다.

![하위 계정 열기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942593/original/BAq_zXwwo6SPySSkqZhqY5vvhpiJtHf8NQ.png?1764909434)

### **기존 일정 편집**

**Edit Schedule(일정 편집)**을 클릭하여 선택된 하위 계정의 현재 자동 리포트 설정을 검토하거나 업데이트합니다. 이 버튼을 사용하면 일정을 처음부터 다시 만들지 않고도 리포트 빈도, 타이밍, 수신자, 포함된 대시보드를 조정할 수 있습니다.

![기존 일정 편집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942642/original/j2MY0J6g0oOi02AMnmcC5qsQ4kDkq3TyNA.png?1764909522)

### **리포트 대시보드 선택**

이 화면에서 각 카드의 체크박스를 선택하여 자동 이메일 리포트에 포함하려는 특정 대시보드를 선택합니다. 대화(Conversations)나 연락처(Contacts)와 같은 개별 지표를 활성화하거나, **Select all(모두 선택)** 옵션을 사용하여 하위 계정에 전체 성과 요약을 보낼 수 있습니다.

![리포트 대시보드 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942664/original/0jpJHBNtErjYhXpgdOf_Dfy1ARmkh4u53w.png?1764909642)

### **다음 단계로 진행**

리포트에 포함하려는 대시보드를 선택한 후, 우측 상단의 **Next(다음)** 버튼을 클릭하여 예약 과정을 진행합니다. 이렇게 하면 자동 리포트의 이메일 제목, 빈도, 타이밍, 수신자를 설정하는 구성 화면으로 이동합니다.

![다음 단계로 진행](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942666/original/4wNTikDHEGV_7HhG9QgtYhYCAC2P2QBXvg.png?1764909663)

### **이메일 제목 입력**

이 단계에서 하위 계정으로 전송되는 자동 리포트 이메일에 표시될 제목을 입력합니다. 명확한 제목은 수신자가 받은 편지함에서 리포트가 도착했을 때 빠르게 식별하고 목적을 이해하는 데 도움이 됩니다.

![이메일 제목 입력](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942704/original/5yyu7cd6PIHf7i5yghYOhecRLnLMYZVXTA.png?1764909815)

## 리포트 전송 시점

리포트 타이밍은 수신자가 예상하는 시점에 인사이트를 받을 수 있도록 보장하는 데 중요합니다. 빈도, 날짜, 시간대 설정이 전송에 미치는 영향을 이해하면 예측 가능한 리포팅 리듬을 만들 수 있습니다.

리포트를 예약할 때 사용자는 다음을 선택합니다:

- **빈도(Frequency)**
- **시작 날짜(Start Date)**  
- **시간(Time)**

### **빈도 로직:**

- **일간**: 설정된 시간에 매일 전송됩니다.
- **주간**: **선택한 시작 날짜와 같은 요일**에 매주 전송됩니다. 예를 들어, 시작 날짜가 화요일이면 주간 리포트는 항상 화요일에 전송됩니다.
- **월간**: **시작 날짜로 선택한 같은 달력 날짜**에 매월 전송됩니다. 예를 들어, 시작 날짜가 5일이면 매월 5일에 리포트가 전송됩니다.

**시간대:**

리포트는 **하위 계정의 시간대**를 기준으로 전송됩니다.

### 리포트 빈도 선택

Frequency(빈도) 드롭다운 메뉴에서 **Daily(일간)**, **Weekly(주간)** 또는 **Monthly(월간)** 중 선택하여 리포트 전송 빈도를 설정합니다. 선택한 빈도에 따라 하위 계정이 리포트에 포함된 대시보드를 기반으로 자동 성과 업데이트를 받는 빈도가 결정됩니다.

![리포트 빈도 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942725/original/UyckpdVNfBbNtYstkUnZ_r1YxXV8R0ZsBg.png?1764909931)

### **다음 예약 날짜 설정**

**Next schedule date(다음 예약 날짜)** 필드에서 값을 선택하여 첫 번째 리포트가 전송될 날짜를 선택합니다. 이 날짜는 또한 주간 또는 월간 일정의 반복 요일을 결정하여, 선택에 따라 일관된 전송 주기를 따르도록 보장합니다.

![다음 예약 날짜 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942737/original/s3GomGn_i3Anzh7Oes5WAkdvbVGShNJutg.png?1764909974)

### **전송 시간 선택**

드롭다운 메뉴에서 옵션을 선택하여 수신자에게 리포트가 전송될 **Time of day(시간대)**를 선택합니다. 리포트는 하위 계정의 시간대에 따라 전송되어, 수신자의 현지 시간을 기준으로 일관되고 정확한 전송을 보장합니다.

![전송 시간 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942775/original/CnzvKCReQIlVZ12n-8oa-bf4KrHnXho3Og.png?1764910081)

### **리포트 수신자 추가**

**Recipients(수신자)** 필드를 사용하여 하나 이상의 이메일 주소를 추가해 자동 리포트를 받을 사람을 지정합니다. 관리자, 팀원 또는 외부 이해관계자를 포함할 수 있어, 성과 업데이트가 필요한 모든 사람이 자동으로 받을 수 있도록 보장합니다.

![리포트 수신자 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942817/original/XHC9E9V5huC8gvlNHSArpZQOK26haEBb2A.png?1764910245)

### **테스트 이메일 보내기**

**Test email(테스트 이메일)** 필드를 사용하여 공식적으로 예약되기 전에 리포트의 미리보기 버전을 받을 하나 이상의 주소를 추가합니다. **Send test(테스트 전송)**을 클릭하면 샘플 이메일이 전송되어 일정을 확정하기 전에 리포트 내용, 형식, 수신자 가시성이 올바른지 확인할 수 있습니다.

![테스트 이메일 보내기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059942890/original/1wA60c4HbouR50f3MpJFVJlhJXAwEJLgXA.png?1764910533)

### 일정 일시정지 또는 재개

**Pause/Resume(일시정지/재개)** 버튼을 사용하여 하위 계정의 예약된 리포트 전송을 일시적으로 중지하거나 다시 시작합니다. 아이콘이 **일시정지 기호**를 보여주면 리포트가 현재 활성화되어 있고 클릭하면 일정이 일시정지됩니다. 아이콘이 **재생 기호**를 보여주면 일정이 일시정지된 상태이고 클릭하면 자동 리포트 전송이 재개됩니다. 이 컨트롤을 통해 에이전시는 기존 일정을 삭제하거나 재구성하지 않고도 리포팅 활동을 쉽게 관리할 수 있습니다.

![일정 일시정지 또는 재개](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059975952/original/he5bYAT8kh9zv0WG6bsdvnjWPEGuAiMb_Q.png?1764934700)

# 리포트 요약 확인하기

전송 요약을 보면 사용자가 리포트 수신자를 확인하고, 전송 활동을 모니터링하며, 이메일이 성공적으로 전송되었는지 확인할 수 있습니다. 이는 투명성을 높이고 에이전시가 참여도를 추적할 수 있게 해줍니다.

리포트 화면 하단으로 스크롤하여 리포트를 받은 이메일 주소가 포함된 전송 로그를 확인합니다.

# 자주 묻는 질문

**Q: 서로 다른 하위 계정에 대해 다른 리포트를 예약할 수 있나요?**

A: 네. 각 하위 계정은 고유한 일정과 구성을 가질 수 있습니다.

**Q: 리포트에 여러 수신자를 추가할 수 있나요?**

A: 네, 쉼표와 공백으로 구분하여 여러 이메일을 입력할 수 있습니다.

**Q: 하위 계정 사용자가 리포트를 받으려면 특정 권한이 필요한가요?**

A: 관리자는 자동으로 리포트를 받습니다. 추가 수신자는 특별한 권한이 필요하지 않습니다.

**Q: 리포트를 일시정지하면 어떻게 되나요?**

A: 일정이 재개될 때까지 리포트가 전송되지 않습니다.

**Q: 리포트에 포함된 대시보드 데이터를 변경할 수 있나요?**

A: 아니요. 리포트는 자동으로 미리 정의된 대시보드를 포함하며 맞춤 설정할 수 없습니다.

**Q: 비즈니스 이메일을 편집하면 전송에 영향을 주나요?**

A: 네. 비즈니스 이메일을 변경하면 수신자를 수동으로 조정하지 않는 한 리포트 전송 위치에 영향을 줍니다.
