# 에이전시 통합 리포팅

모든 하위 계정의 핵심 데이터가 에이전시 계정으로 통합되어 리포팅됩니다. 한 곳에서 에이전시 산하 모든 하위 계정의 성과를 명확하게 볼 수 있어요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48258801812/original/_7pV4LsbZzmunPZEuDah4_mzQbkbKWZndg.gif?1666621167)

참고 사항:

이 기능은 Pro 플랜($497/월, $4970/년)에서만 사용할 수 있습니다.

## 하위 계정 통합 보고서

**에이전시 계정(Agency Account) > 하위 계정(Sub-accounts)**으로 이동해서 **컬럼(Columns)** 버튼을 클릭하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048428356/original/xJb4MWFinihNceExFvq0ogtqWRjLDM4YEQ.png?1750186878)

모든 하위 계정 요약 카드에 포함할 컬럼을 최대 8개까지 선택할 수 있어요.

- **활성 사용자(Active users)**: 
선택한 기간 동안 웹 또는 모바일 앱(iOS 및 Android)에서 한 번 이상 로그인한 사용자입니다.

- 79/100 형태로 표시됩니다. 79는 활성 사용자 수, 100은 선택한 기간 동안 생성된 해당 로케이션의 전체 사용자 수입니다.

- 사용자는 로케이션 관리자 또는 로케이션 에이전트일 수 있습니다.

- **예약(Appointment)**: 선택한 기간 동안 생성된 모든 예약 건수입니다. 예약 상태에 관계없이 계산됩니다.

- **연락처(Contact)**: 선택한 기간 동안 해당 로케이션에서 생성된 연락처 수입니다.

- **이메일(Email)**:
수신 이메일 - 선택한 기간 동안 연락처가 로케이션 관리자/사용자에게 보낸 모든 이메일 수

발신 이메일 - 하위 계정 관리자/사용자가 연락처에게 보낸 모든 이메일 수 (자동 이메일은 제외)

- **문자 메시지(Text Messages)**:
수신 SMS - 선택한 기간 동안 연락처가 로케이션 관리자/사용자에게 보낸 모든 SMS 수

발신 SMS - 하위 계정 관리자/사용자가 연락처에게 보낸 모든 SMS 수 (자동 SMS는 제외)

- **전화(Calls)**:
수신 전화 - 선택한 기간 동안 연락처가 로케이션 관리자/사용자에게 건 모든 전화 수

발신 전화 - 하위 계정 관리자/사용자가 연락처에게 건 모든 전화 수 (자동 전화는 제외)

통화 시간 - 사용자와 연락처 간 발생한 모든 통화의 총 지속 시간 합계

- **제출(Submissions)**:
폼 - 제출된 전체 폼 수

설문 - 제출된 전체 설문 수

페이스북 폼 - 제출된 전체 페이스북 폼 수

- **왓츠앱(WhatsApp)**:
수신 - 선택한 기간 동안 로케이션/사용자에게 전송된 모든 왓츠앱 메시지 수

발신 - 전송된 모든 왓츠앱 메시지 수

- **GBP (구글 비즈니스 프로필)**:
수신 메시지 - 선택한 기간 동안 GMB를 통해 연락처가 로케이션 관리자/사용자에게 보낸 모든 메시지 수

발신 메시지 - 하위 계정 관리자/사용자가 연락처에게 보낸 모든 메시지 수 (자동 메시지는 제외)

- **인스타그램(Instagram)**:
수신 메시지 - 선택한 기간 동안 인스타그램을 통해 연락처가 로케이션 관리자/사용자에게 보낸 모든 메시지 수

발신 메시지 - 하위 계정 관리자/사용자가 연락처에게 보낸 모든 메시지 수 (자동 메시지는 제외)

- **페이스북(Facebook)**:
수신 메시지 - 선택한 기간 동안 페이스북을 통해 연락처가 로케이션 관리자/사용자에게 보낸 모든 메시지 수

발신 메시지 - 하위 계정 관리자/사용자가 연락처에게 보낸 모든 메시지 수 (자동 메시지는 제외)

- **리뷰(Review)**:
전체 - 해당 로케이션에서 받은 리뷰 수

긍정적 - 받은 긍정적 리뷰 수

부정적 - 받은 부정적 리뷰 수

- **생성된 사이트(Sites Created)**:
퍼널 - 생성된 퍼널 수

웹사이트 - 생성된 웹사이트 수

폼 - 생성된 폼 수

설문 - 생성된 설문 수

## 하위 계정 예약 보고서 관리

에이전시 뷰에서 중앙 집중식 인터페이스를 통해 모든 하위 계정의 예약 보고서를 관리할 수 있어요. 일시정지, 재개, 필터링, 새로 추가된 계정에 대한 동작 구성이 포함됩니다.

### 예약 보고서 테이블

이동 경로: 에이전시 뷰(Agency View) → 하위 계정(Sub-Accounts) → 예약 보고서 보기(View Scheduled Reports)

- 하위 계정의 모든 예약 보고서 목록을 확인하세요

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049079548/original/mUtZIGCYBWMbLYXYiSNLxh1opUv-CXLCuQ.png?1751283692)

- 체크박스를 사용해서 하나 또는 여러 보고서를 선택하세요

- 선택한 보고서에 대해 일시정지 또는 재개 일괄 작업을 적용하세요

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049079568/original/8lC4urkJJXVvpEA4-OlByVM0OZfz8FH6Jw.png?1751283703)

## **상태별 필터**

목록을 활성 또는 일시정지된 일정만으로 좁혀서 전송 상황을 빠르게 점검하세요.

- **상태(Status)** 필터를 선택하고 **활성(Active)** 또는 **일시정지(Paused)**를 선택하세요.

- **일시정지**를 사용해서 재개할 때까지 전송되지 않을 일정을 찾으세요.

## **설정 패널**

온보딩, 휴일 또는 템플릿 변경 중에 예상치 못한 전송을 방지하기 위해 글로벌 동작을 제어하세요.

- 예약 보고서 뷰에서 **설정(Settings)** 아이콘(톱니바퀴)을 클릭하세요.

- 모든 예약 보고서 일시정지: 에이전시 레벨에 나열된 모든 예약 보고서를 즉시 일시정지합니다. 광범위한 템플릿이나 수신자 변경 전에 사용하세요.

- 새 하위 계정에 대한 예약 보고서 자동 생성(토글):

켜짐: 새 하위 계정이 추가되면 기본 예약 보고서가 자동으로 생성됩니다.

꺼짐: 새 하위 계정은 자동 생성된 보고서를 **받지 않아** 온보딩 중 실수로 전송되는 것을 방지합니다.

**모범 사례**

- 수동 설정 없이 모든 새 고객이 기본 보고서를 받기를 원한다면 **자동 생성**을 활성화하세요.

- 에이전시에서 보고서 전송 전에 템플릿, 수신자, 일정을 검토하는 것을 선호한다면 **자동 생성**을 비활성화하세요.

### 사용 방법

- 에이전시 계정(Agency Account) → 하위 계정(Sub-Accounts) → 예약 보고서 보기(View Scheduled Reports)로 이동하세요

- 체크박스를 사용해서 관리하고자 하는 보고서를 선택하세요

- 일괄 작업 바에서 일시정지 또는 재개를 선택하세요

- 설정 아이콘을 클릭해서 모든 보고서를 일시정지하거나 자동 생성 동작을 구성하세요

## 자주 묻는 질문

**Q: 누가 에이전시 통합 리포팅 제어에 접근할 수 있나요?**
A: $497/월 이상 플랜에서 에이전시 뷰 접근 권한이 있는 사용자입니다.

**Q: 여러 보고서를 한 번에 일시정지 해제할 수 있나요?**
A: 네. 체크박스로 여러 행을 선택한 다음 일괄 작업 바에서 재개를 선택하세요.

**Q: 전체 일시정지가 나중에 자동 생성되는 보고서도 중단시키나요?**
A: 전체 일시정지는 현재 일정을 일시정지합니다. 자동 생성 토글은 새 하위 계정이 자동으로 일정을 받을지 제어합니다. 필요에 따라 함께 사용하세요.

**Q: 전송되지 않는 보고서를 빠르게 찾는 방법은?**
A: 상태 필터를 사용해서 일시정지를 선택하세요.
