# 기회 관리, 회사, 커스텀 객체 간의 연관 관계

이 기능을 사용하면 실제 비즈니스 관계를 시스템에 반영할 수 있습니다. 예를 들어 기회 관리(영업 건)를 회사나 부동산, 기타 커스텀 객체와 연결하거나, 회사를 커스텀 객체와 직접 연결할 수 있습니다. 이를 통해 비즈니스 레코드 전반에 걸쳐 통합된 맥락적 뷰를 제공하여 팀이 더 빠르게 일하고 정보에 기반한 의사결정을 내릴 수 있도록 돕습니다.

# **왜 중요한가요?**

연관 관계는 CRM을 더 강력하고 연결된 시스템으로 만듭니다. 중요한 이유는 다음과 같습니다:

- 기회 관리, 회사, 커스텀 객체에 걸친 360° 뷰 제공
- 실제 비즈니스 관계 반영 (예: 영업 건 ↔ 회사 ↔ 부동산이나 보험 같은 자산)
- 관련 정보를 한 곳에서 관리하여 레코드 간 이동 시간 단축
- 워크플로우, 보고서, 내보내기에서 연관 관계 데이터를 활용해 더 나은 추적과 개인화 제공

---

## 주요 특징

- 다음 간의 연관 관계 지원:
  - 기회 관리 ↔ 커스텀 객체
  - 회사 ↔ 커스텀 객체
  - 회사 ↔ 기회 관리
- 1:1, 1:다, 다:1, 다:다 관계 지원
- 레코드 패널에서 직접 연관 관계 생성 및 관리
- 작업 흐름을 벗어나지 않고 기존 레코드를 즉시 검색하고 연결
- 레코드 상세, 칸반 카드, 목록 보기에서 연관된 레코드 확인
- 필드 관리 → 연관 관계 라벨(객체 유형)을 통한 연관 관계 필드 추가

---

## **연관 관계 생성하는 방법**

#### **1단계: 연관 관계 탭 열기**

- **Settings(설정) → Custom Objects(커스텀 객체) → [대상 객체]**로 이동
- **Associations(연관 관계)** 탭 클릭
- **Create Association(연관 관계 생성)** 클릭

![연관 관계 탭 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055756163/original/GSVW_zRGJRjoL0vseXdaRvn0VUla61-5AQ.png?1760135540)

#### **2단계: 연관 관계 설정**

- 연결할 객체 선택 (기회 관리 또는 회사)
- 연관 관계 이름 설정:
  - **단일 라벨** (예: "영업 건")
  - **쌍 라벨** (예: "구매자 ↔ 판매자")
- 관계 유형 설정:
  - 1:1 (One-to-One)
  - 1:다 / 다:1 (One-to-Many / Many-to-One)
  - 다:다 (Many-to-Many)
- (선택사항) 레코드 제한 설정 (최대 1,000개)
- 미리 보기 확인 → **Save(저장)** 클릭

![연관 관계 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055756453/original/SB9gwCmMaEzSmrtNsUZe79XRd8RxyFIhfQ.png?1760135929)

## 레코드에서 연관 관계 관리하기

**지원되는 모든 레코드 유형**에서 연관 관계를 관리할 수 있습니다:

#### **기회 관리에서**

- 기회 관리 열기
- **Associated Objects(연관된 객체)** 섹션으로 이동
- **+ Associate(+ 연결)** 클릭
- 회사 또는 커스텀 객체 선택

#### **회사에서**

- 회사 열기
- **Associations / Related Objects(연관 관계 / 관련 객체)** 섹션으로 이동
- 기회 관리 또는 커스텀 객체 연결

#### **커스텀 객체에서**

- 커스텀 객체 레코드 열기
- 연관된 기회 관리 또는 회사 추가

모든 연관 관계는 즉시 표시되며 언제든지 편집할 수 있습니다.

## **CRM에서 연관 관계 보기**

#### **레코드 상세**

- 연결된 모든 레코드(기회 관리, 회사, 커스텀 객체)가 **Associated Objects(연관된 객체) 패널**에 표시됩니다
- 현재 화면을 벗어나지 않고 사이드 패널에서 연결된 레코드를 열 수 있습니다

#### **칸반 보기 (기회 관리)**

- **Opportunities(기회 관리) → Manage Fields(필드 관리)**로 이동
- 카드에 **Association Label (Object Type)(연관 관계 라벨 (객체 유형))** 추가
- 카드에서 직접 연결된 회사나 커스텀 객체 확인

![칸반 보기에서 연관 관계](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055812605/original/p2-PEiWc-OGfiznohmbrNGhLJrTAsFX2VQ.png?1760331692)

#### **목록 보기**

- **연관 관계 필드를 열로 추가**
- 연결된 모든 레코드를 한 곳에서 확인
- 필터링, 일괄 작업, 내보내기에 유용

![목록 보기에서 연관 관계](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055812981/original/1i5aZaVtwmrqD25nTBYWvpdUvJulEUw4hg.png?1760332266)

## **팁**

- 객체 쌍당 최대 **10개의 라벨** 생성 가능
- 각 라벨은 한 쪽에 최대 **1,000개의 연결된 레코드** 지원
- 레코드가 삭제되면 연관 관계도 자동으로 제거됨
- **Manage Fields(필드 관리)**를 통해 보고서와 내보내기에 연관 관계 필드 추가 가능
- API와 자동화 지원은 곧 제공 예정

---

## **활용 사례**

**부동산**

부동산(커스텀 객체) ↔ 기회 관리 ↔ 회사를 연결하여 구매자, 매물, 중개사를 한 곳에서 추적

**이벤트**

이벤트(커스텀 객체) ↔ 회사 ↔ 후원 기회 관리 연결

**보험**

보험 상품/견적(커스텀 객체) ↔ 기회 관리 ↔ 회사를 연결하여 전체 라이프사이클 추적

## **자주 묻는 질문**

**Q: 관계를 제한할 수 있나요?**

네, 1:1 관계를 사용하거나 설정에서 제한을 설정할 수 있습니다.

**Q: 라벨 이름을 바꾸면 연관 관계가 깨지나요?**

아니요, 표시 라벨만 변경됩니다.

**Q: 연관 관계를 일괄 가져오기할 수 있나요?**

아직 지원하지 않습니다.

**Q: 보고서에서 이 기능을 사용할 수 있나요?**

네, 연관 관계 필드를 통해 가능합니다.

**Q: 삭제 시 어떻게 되나요?**

연관 관계가 자동으로 제거됩니다.
