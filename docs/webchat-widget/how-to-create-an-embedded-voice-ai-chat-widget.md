# 임베드형 음성 AI 채팅 위젯 만들기

음성 AI 채팅 위젯(Voice AI Chat Widget)을 이제 웹사이트 콘텐츠 내부에 직접 임베드할 수 있습니다(플로팅 채팅 버블이 아닌). 이를 통해 더욱 자연스러운 페이지 내 음성 경험을 만들고, 레이아웃에 가장 적합한 위치에 정확히 위젯을 배치할 수 있습니다.

---

## 임베드형 음성 AI 위젯이란?

임베드형 음성 AI 채팅 위젯은 화면 모서리에 떠있는 대신 웹사이트나 퍼널 콘텐츠 내에 인라인으로 표시되는 음성 AI 위젯입니다. 랜딩 페이지, 리드 수집 페이지, 서비스 페이지, 오퍼 섹션 등 핵심 페이지에서 음성 AI를 표시하는 방식에 더 많은 유연성을 제공합니다.

기본 고정형(Sticky) 배치와 달리, 임베드/인라인 배치는 위젯이 페이지 레이아웃의 일부가 되도록 합니다. 이를 통해 더욱 원활한 방문자 경험을 만들면서 주변 콘텐츠와 대화 흐름을 일치시킬 수 있습니다.

---

## 임베드형 음성 AI 채팅 위젯의 주요 장점

임베드형 음성 AI 채팅 위젯은 방문자들이 가장 참여할 가능성이 높은 곳에 정확히 대화를 배치하여 더욱 의도적인 페이지 경험을 만들어줍니다.

- **인라인 배치 유연성**: 플로팅 채팅 버블에 의존하는 대신 페이지 섹션 내부에 직접 음성 AI 위젯을 추가할 수 있습니다.

- **더욱 자연스러운 사용자 경험**: 별도의 오버레이보다는 웹사이트 콘텐츠의 일부처럼 느껴지도록 대화를 만들 수 있습니다.

- **더 깔끔한 페이지 디자인 제어**: 더욱 브랜드화된 경험을 위해 퍼널이나 웹사이트 레이아웃과 위젯을 일치시킬 수 있습니다.

- **모바일 최적화 경험**: 임베드형 위젯은 기본적으로 모바일 방문자를 위한 반응형입니다.

- **유연한 배포 옵션**: 여러 인라인 배치를 사용하고 같은 웹사이트나 퍼널에서 임베드형과 고정형 위젯 경험을 결합할 수 있습니다.

---

## 임베드형 음성 AI 채팅이 가장 효과적인 곳

임베드형 음성 AI 채팅은 다음에서 사용할 수 있습니다:

- 퍼널
- 웹사이트
- 랜딩 페이지
- 커스텀 HTML 사이트
- 채팅 위젯을 설치할 수 있는 기타 지원 페이지

임베드형 위젯은 기본적으로 모바일 반응형이므로, 페이지 콘텐츠의 일부로 남아있으면서도 작은 화면에 맞게 자동으로 조정됩니다.

---

## 임베드형 음성 AI 채팅 위젯 만들기

적절한 설정을 통해 위젯이 원하는 위치에 정확히 나타나고 웹사이트나 퍼널 전체에서 일관되게 작동하도록 보장할 수 있습니다.

#### **1단계: 새 채팅 위젯 만들기**

Sites(사이트) → Chat Widget(채팅 위젯)으로 이동한 후 + New(새로 만들기)를 클릭하거나 기존 위젯을 여세요.

![새 채팅 위젯 만들기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740337/original/ICfKFoK48ZH01PvKtzeFmNYcV9hpIq4U8g.png?1776882857)

#### **2단계: Voice AI Chat 선택**

Voice AI chat 옵션을 선택하세요.

![Voice AI Chat 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740359/original/apy8w9cyOj2WLlbXhM-ZPsOlFVWlzIAnRw.png?1776882870)

#### **3단계: 위젯 배치를 Embedded로 설정**

- Style(스타일) 탭으로 이동하세요.

- Widget Placement(위젯 배치) 하에서 Embedded / Inline (renders inside content)(임베드 / 인라인 (콘텐츠 내부 렌더링))을 선택하세요.

이렇게 하면 위젯이 화면에 떠있는 대신 페이지 레이아웃 내에 나타나게 됩니다.

![위젯 배치 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740377/original/cI-J9e8wQb-Ax6kXLfeQJ-XSBn-FlcbZLg.png?1776882882)

#### **4단계: 위젯 커스터마이징**

Chat Window(채팅 창) 탭에서 Button Text(버튼 텍스트)를 편집하세요 (인라인 레이아웃에서 아바타 아래에 나타남).

- 예시: "Call us here"를 "Talk to our AI" 또는 "Ask a Question"으로 변경하세요.

선택적으로, Additional Options(추가 옵션)을 열어 사업체 이름이나 규정 준수 푸터와 같은 브랜딩 요소를 조정할 수 있습니다. 미리보기가 실시간으로 업데이트되어 임베드형 위젯 디자인을 보여줍니다.

![위젯 커스터마이징](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740380/original/Fuz82ZwIjqd6wZnL6MXQh3E8TOye8trJNQ.png?1776882893)

#### **5단계: 임베드 코드 생성**

위젯이 만족스러우면, Get Code(코드 가져오기)(우상단)를 클릭하세요. 고유한 임베드 코드 스니펫이 있는 팝업 창이 나타납니다. Copy(복사)를 클릭하여 클립보드에 저장하세요.

![임베드 코드 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740395/original/RYfqtyFpw6os8gS4_wiPnzWUzWfFX0q-1w.png?1776882904)

#### **6단계: 웹사이트나 퍼널에 위젯 임베드**

Code Block 요소를 사용하여 사이트나 퍼널 어디에든 위젯을 배치할 수 있습니다.

퍼널에 임베드하려면:

- Sites(사이트) > Funnels(퍼널)로 이동하여 위젯을 추가하려는 퍼널 페이지를 여세요.

- Edit Page(페이지 편집)를 클릭하세요.

- 좌측 사이드바에서 Quick Add(빠른 추가) > Custom(커스텀) 하의 Code(코드)를 검색하여 Code Element(코드 요소)를 페이지로 드래그하세요.

![코드 요소 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740399/original/1NugCoSeU16a25BCQwpbnPynkgkujgGVuQ.png?1776882917)

- Open Code Editor(코드 에디터 열기)를 클릭하세요.

![코드 에디터 열기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740401/original/iPlZ9ePfeNTyGtrG6FQz8qhlYWlqpUZPwQ.png?1776882931)

- 채팅 위젯 빌더에서 복사한 임베드 코드를 붙여넣으세요.

![코드 붙여넣기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740452/original/hMW5y55zzXNDGmO6RUci5_6s1WrkPytQfQ.png?1776882959)

- Save(저장)을 클릭한 후 페이지를 Publish(발행)하세요.

![저장 및 발행](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069740473/original/L8HwAggfot3O1WBbzTOui-LkcuH8SK2Naw.png?1776882974)

발행되면, 음성 AI 위젯이 다른 콘텐츠 블록과 마찬가지로 인라인으로 임베드되어 나타납니다.

## 중요한 동작 참고사항

- 임베드형 위젯은 완전히 모바일 반응형이며 작은 화면에서 자동으로 크기를 조정합니다.

- 같은 사이트나 퍼널에 여러 임베드형 위젯을 가질 수 있으며, 모두 같은 음성 AI 에이전트에 연결됩니다.

- 최대 참여를 위해 다양한 섹션에서 임베드형과 고정형 위젯을 혼합하여 사용할 수 있습니다.

## 모범 사례

잘 배치된 위젯은 페이지 경험을 방해하지 않고 전환을 지원할 수 있습니다. 목표는 채팅이 방해가 되지 않고 도움이 되고 시기적절하게 느껴지도록 하는 것입니다.

- 폼, 오퍼, 예약 영역, 지원 프롬프트와 같은 높은 의도의 콘텐츠 근처에 위젯을 배치하세요.

- 더욱 원활한 외관을 위해 위젯 브랜딩을 페이지와 일치시키세요.

- 좁은 섹션 내에서 위젯이 답답해 보이지 않도록 간격과 정렬을 테스트하세요.

- 발행 후 모바일 레이아웃을 검토하여 가독성과 간격을 확인하세요.

- 인트로와 확인 메시지를 명확하고 행동 지향적으로 유지하세요.

## 자주 묻는 질문

**Q: 페이지에서 임베드형 위젯을 어디에 배치해야 하나요?**
좋은 배치 옵션으로는 히어로 섹션, 오퍼 섹션, 리드 수집 영역, 서비스 페이지, 그리고 방문자들이 참여할 가능성이 높은 기타 높은 의도 영역이 있습니다.

**Q: Embedded/Inline을 선택한 후 일부 커스터마이징 설정이 사라진 이유는 무엇인가요?**
일부 컨트롤은 플로팅 위젯에만 적용되기 때문에 자동으로 숨겨집니다. Embedded/Inline 모드에서는 인라인 배치와 관련된 설정만 표시됩니다.

**Q: Live Chat 위젯에도 Embedded/Inline 배치를 사용할 수 있나요?**
네, Embedded/Inline 지원은 Live Chat에서도 사용 가능합니다.
