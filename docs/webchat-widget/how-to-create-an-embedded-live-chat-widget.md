# 임베디드 실시간 채팅 위젯 만들기

떠다니는 채팅 버블 대신 웹사이트나 퍼널 콘텐츠 안에 실시간 채팅을 직접 삽입하세요. 임베디드 실시간 채팅 위젯(Embedded Live Chat Widget)을 사용하면 방문자가 행동을 취할 가능성이 높은 곳에 대화 기능을 배치할 수 있습니다.

# 임베디드 실시간 채팅 위젯이란?

임베디드 실시간 채팅 위젯은 화면 모서리에 떠다니는 대신 페이지 콘텐츠 안에 인라인으로 채팅 위젯을 배치하는 옵션입니다. 이를 통해 채팅이 페이지 디자인의 일부처럼 느껴지며, 방문자가 결정을 내리는 정확한 위치에 지원 및 전환 기회를 배치할 수 있습니다.

위젯은 여전히 동일한 실시간 채팅 경험을 제공하며, 배치만 바뀔 뿐 대화 라우팅이나 관리 방식은 변경되지 않습니다.

## 임베디드 실시간 채팅 위젯의 주요 장점

- **높은 가시성**: 방문자가 이미 참여하고 있는 히어로 섹션, 폼, 기능 블록, 행동 유도 버튼 근처에 채팅을 배치하세요.

- **자연스러운 페이지 경험**: 콘텐츠 위에 떠다니는 오버레이가 아닌 웹사이트 레이아웃의 일부로 채팅을 제공하세요.

- **워크플로우 변경 없음**: 라우팅, 리드 포착, 실시간 채팅 관리는 기존 채팅 시스템을 통해 계속 작동합니다.

- **유연한 배포**: 퍼널, 웹사이트, 랜딩 페이지 및 기타 지원되는 영역에서 위젯을 사용하세요.

---

## 임베디드 실시간 채팅이 가장 효과적인 곳

임베디드 실시간 채팅은 다음에서 사용할 수 있습니다:

- 퍼널
- 웹사이트
- 랜딩 페이지
- 커스텀 HTML 사이트
- 채팅 위젯이 설치될 수 있는 기타 지원 페이지

임베디드 위젯은 기본적으로 모바일 반응형이며, 페이지 콘텐츠와 인라인을 유지하면서 작은 화면에 자동으로 적응합니다.

---

## 임베디드 실시간 채팅 위젯 설정 방법

올바른 설정을 통해 위젯이 원하는 정확한 위치에 나타나고 사이트나 퍼널 전체에서 일관되게 작동하도록 할 수 있습니다.

#### **1단계:** 새 채팅 위젯 만들기

Sites(사이트) → Chat Widget(채팅 위젯)으로 이동한 후 + New(새로 만들기)를 클릭하거나 기존 위젯을 여세요.

![새 채팅 위젯 만들기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069630470/original/eTmCceucah4ZXJ-MeZboWWtM3I9IWzAq8g.png?1776785411)

#### **2단계:** Live Chat(실시간 채팅) 선택

Live Chat(실시간 채팅) 옵션을 선택하세요.

![실시간 채팅 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069630544/original/ZMf7-FDv2kCxfSMlCmuWXOx5ICtTJk8HSQ.png?1776785439)

#### **3단계:** 위젯 배치를 Embedded로 설정

- Style(스타일) 탭을 여세요.
- Widget Placement(위젯 배치)에서 "Embedded / Inline (Renders inside content)"를 선택하세요.

이렇게 하면 위젯이 화면에 떠다니는 대신 페이지 레이아웃 내에 나타나게 됩니다.

![위젯 배치 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069630609/original/RlZ_TknFtaQkiNLPeo9MSTgznYZUR0M_Mw.png?1776785471)

#### **4단계:** 위젯 커스터마이즈

필요에 맞게 위젯을 구성하세요:

- **테마 및 외관**
  - 색상 테마 선택
  - 브랜딩 요소 조정 (아바타, 헤더, 색상)
  - 웹사이트와 스타일 매칭

- **채팅창 설정**
  - 인트로 메시지 편집
  - 연락처 폼 필드 구성 (이름, 전화번호 등)
  - 필수 필드 설정
  - 버튼 텍스트 커스터마이즈

- **메시징 및 동작**
  - 자동 응답 설정
  - 실시간 채팅 배정 구성
  - 영업 시간 정의
  - "채팅 종료" 메시지 커스터마이즈

![위젯 커스터마이즈](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069630647/original/QnGjWO-DvZow1dlTd_gXHG0HBvzZ1AI2lw.png?1776785496)

#### **5단계:** 임베드 코드 가져오기

- Get Code(코드 가져오기)를 클릭하세요 (우상단).
- 임베드 스크립트를 복사하세요.

![임베드 코드 가져오기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069630663/original/Kc7LBtipjBc9TE_QTlr8U3oiZCjL5ig6dw.png?1776785516)

#### **6단계:** 퍼널, 웹사이트 또는 기타 페이지에 위젯 추가

**옵션 1: 퍼널**

- 퍼널 편집기를 여세요.
- 채팅을 표시하고 싶은 곳에 Custom Code(커스텀 코드) 요소를 추가하세요.
- 임베드 코드를 붙여넣으세요.
- 저장하고 발행하세요.

**옵션 2: 웹사이트 / 외부 페이지**

- 웹사이트 편집기나 HTML 파일을 여세요.
- 원하는 섹션에 임베드 코드를 붙여넣으세요.
- 변경사항을 발행하세요.

위젯은 코드가 배치된 정확한 위치에 렌더링됩니다.

![위젯 배치 결과](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069630686/original/_5cua8OlUBoJ3fiDFEr_hn_uAh0RKT6tvA.png?1776785528)

## 작동 방식

위젯은 페이지 콘텐츠 내에서 인라인으로 로드됩니다. 채팅 동작은 표준 실시간 채팅과 동일하게 유지됩니다(라우팅, 자동화 등).

방문자는 다음을 할 수 있습니다:

- 대화 시작
- 연락처 정보 제출
- 실시간으로 채팅 계속

## 임베디드 실시간 채팅의 일반적인 사용 사례

임베디드 배치는 채팅이 특정 행동을 지원하거나 방문자가 계속 진행하기 전의 망설임을 줄일 때 가장 효과적입니다.

일반적인 사용 사례는 다음과 같습니다:

- 폼 제출 전 질문 포착
- 예약 또는 상담 페이지에서 도움 제공
- 퍼널 단계에서 반대 의견 해결
- 서비스 페이지에서 더욱 안내된 지원 경험 제공
- 전용 연락처 섹션 내에 보이는 대화 옵션 추가

## 모범 사례

- 폼, 오퍼, 예약 영역, 지원 프롬프트 등 높은 의도를 가진 콘텐츠 근처에 위젯을 배치하세요.
- 매끄러운 외관을 위해 위젯 스타일을 페이지와 매치시키세요.
- 위젯이 좁은 섹션에서 답답해 보이지 않도록 간격과 정렬을 테스트하세요.
- 가독성과 간격을 확인하기 위해 발행 후 모바일 레이아웃을 검토하세요.
- 인트로 및 확인 메시지를 명확하고 행동 지향적으로 유지하세요.

## 중요한 동작 참고사항

- 임베디드 위젯은 페이지에서 떠다니지 않습니다.
- 다른 페이지에서는 여전히 떠다니는(sticky) 위젯을 사용할 수 있습니다.
- 라우팅, 자동화 또는 연락처 데이터 포착에는 변경사항이 없습니다.

## 임베디드 실시간 채팅 문제 해결

- **위젯이 보이지 않는 경우**: 임베드 코드가 올바르게 배치되었고 페이지가 발행되었는지 확인하세요.
- **레이아웃 문제**: 페이지 빌더에서 컨테이너 너비를 확인하고 CSS 충돌이 있는지 검토하세요.
- **채팅이 응답하지 않는 경우**: 실시간 채팅 배정 설정을 검토하고 영업 시간 구성을 확인하세요.

## 자주 묻는 질문

**Q: 임베디드 위젯과 스티키 위젯을 모두 사용할 수 있나요?**
A: 네, 일부 페이지에는 임베디드 위젯을, 다른 페이지에는 스티키 위젯을 사용할 수 있습니다.

**Q: 이것이 채팅 성능에 영향을 주나요?**
A: 아니요, 채팅 기능은 동일하게 유지됩니다.

**Q: 한 페이지에 여러 위젯을 임베드할 수 있나요?**
A: 권장하지 않습니다. 사용자 경험과 추적에 충돌이 발생할 수 있습니다.
