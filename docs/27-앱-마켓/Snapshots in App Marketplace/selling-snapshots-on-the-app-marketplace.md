# 스냅샷을 위한 앱 마켓플레이스란?

앱 마켓플레이스(App Marketplace)를 통해 에이전시 관리자는 CRM 전문 지식을 수익화하고 자신들의 설정을 확장 가능한 상품으로 패키징할 수 있습니다!

- 자신들의 스냅샷을 다른 Hyperclass 에이전시에 판매
- 일회성, 월간, 연간 구독 등 다양한 가격 모델 활용
- IP 보호 기능으로 무단 재판매로부터 스냅샷 자산 보호

- **전문 지식 수익화**: CRM 설정을 다른 에이전시가 구매할 수 있는 상품으로 변환하세요.
- **도달 범위 확대**: Hyperclass의 앱 마켓플레이스가 광범위한 고객층에 스냅샷을 선보일 플랫폼을 제공합니다.

# 앱 마켓플레이스에서 스냅샷 판매 방법

## 간편한 스냅샷 등록 플로우 (마켓플레이스에서 판매)

Agency View → Account Snapshots에서 "Sell on Marketplace"를 선택하면, Hyperclass가 즉시 스냅샷 등록 과정을 시작하고 개발자 포털의 스냅샷 등록 모듈로 직접 안내합니다.

이 모듈에서는 계속 진행하기 전에 플로우 초기에 스냅샷에 포함된 자산을 검토할 수 있습니다. Hyperclass는 또한 불필요한 단계와 화면을 줄이기 위해 스냅샷 전용 등록 경로를 사용합니다.

### 1단계: 개발자 포털 가입

- [https://marketplace.gohighlevel.com](https://marketplace.gohighlevel.com) 방문
- 가입하고 등록 양식 완성
- 이메일 인증을 통해 개발자 포털 액세스 권한 획득

개발자 포털을 처음 이용하는 경우, 등록 중 안내가 나오면 위의 설정 단계를 완료하세요. 기존 판매자는 "Sell on Marketplace" 선택 후 스냅샷 등록 모듈로 바로 이동될 수 있습니다.

### 2단계: 앱 생성

- 개발자 포털에 로그인하여 My Apps로 이동
- Create an App 클릭
- 앱 설정:
  - **Name(이름)**: 앱에 대한 명확하고 설명적인 이름 제공
  - **App Type(앱 유형)**: Public으로 설정
  - **Distribution Type(배포 유형)**: 다음 중 선택:
    - **Agency**: 에이전시에 직접 청구
    - **Sub-Account(하위 계정)**: 스냅샷이 푸시될 때마다 하위 계정에 청구
  - **Listing Type(등록 유형)**: Standard 또는 White-Label 선택 (더 넓은 도달을 위해 권장)

### 3단계: 앱 세부사항 제공

- 앱의 로고, 태그라인, 설명 추가
- 잠재 구매자와의 신뢰 구축을 위해 회사 이름과 웹사이트 명시
- 앱의 가치 제안을 강화하기 위해 미리보기 이미지나 데모 영상 업로드
- 구매자가 필요 시 연락할 수 있도록 고객 지원 세부사항 포함

### 4단계: 가격 설정

- 앱이 무료인지 유료인지 결정:
  - **Free(무료)**: 구매자에게 무료
  - **Freemium(프리미엄)**: 무료이지만 프리미엄 기능은 유료
  - **Paid(유료)**: 일회성 요금, 월간 또는 연간 구독 청구
- 가격 계획, 이름, 포함 내용 정의

### 5단계: 스냅샷 추가

- Hyperclass 에이전시 계정에서 판매하고자 하는 스냅샷 선택
- 가이드 모달에서 App Name 입력
- 미리 채워진 스냅샷 등록 설정 검토 (스냅샷용 자동 설정)
- 모달에 표시된 마켓플레이스 링크를 복사하여 등록을 즉시 공유
- "Sell on Marketplace"를 선택하면 개발자 포털의 스냅샷 등록 모듈로 이동
- 구매자가 받게 될 내용을 확인할 수 있도록 플로우 초기에 스냅샷에 패키징된 자산 검토
- 더 적은 화면과 네비게이션으로 게시하기 위해 스냅샷 전용 등록 단계 계속 진행

![스냅샷 등록 과정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064695562/original/9z0vqjGO0IlvA3K7wQutrK1TE4ljU_ty_w.gif?1770819611)

### 6단계: 검토 제출

- 개발자 포털의 Basic Info 페이지로 돌아가기
- 모든 필요한 설정이 완료되었는지 확인
- Submit for Review 클릭. Hyperclass 팀이 7-10일 내에 화이트라벨 호환성을 위해 앱을 검토합니다.

# 지적재산권(IP) 보호

Get Marketplace Share Link를 활성화하면 IP 보호가 자동으로 적용됩니다. 이는 다음을 보장합니다:

- 구매자가 스냅샷 자산을 재포장하거나 재판매할 수 **없음**
- 스냅샷이 원래 제작자에게만 독점적으로 유지됨

# 자주 묻는 질문

**Q: 한 앱에 여러 스냅샷을 등록할 수 있나요?**

현재는 한 앱에 하나의 스냅샷만 포함할 수 있습니다. 스냅샷의 자산이 앱의 가치 제안과 일치하는지 확인하세요.

**Q: 화이트라벨과 표준 등록 유형의 차이점은 무엇인가요?**

- **White-Label(화이트라벨)**: 브랜딩을 제거하여 다양한 Hyperclass 고객들에게 앱의 어필을 증가시킵니다.
- **Standard(표준)**: 보이는 브랜딩을 유지하여 도달 범위가 제한될 수 있습니다.

**Q: 앱 제출이 거부되면 어떻게 되나요?**

앱이 승인되지 않으면 Hyperclass 팀으로부터 필요한 변경사항을 자세히 알려주는 피드백을 받게 됩니다. 앱을 적절히 업데이트하고 다시 제출하세요.

**Q: 앱이 게시된 후에도 업데이트할 수 있나요?**

아니요, 현재는 앱이 게시된 후 스냅샷 링크를 수정할 수 없습니다. 앱 업데이트 기능은 2025년 1분기에 곧 출시 예정입니다.

**Q: 만들 수 있는 앱 수에 제한이 있나요?**

아니요, 앱 마켓플레이스에서 만들고 판매할 수 있는 앱 수에 제한은 없습니다.

**Q: 구매자는 구매 후 내 스냅샷을 어떻게 사용하나요?**

구매 후 구매자는 Hyperclass 대시보드를 통해 스냅샷을 자신들의 로케이션에 직접 푸시할 수 있습니다.

**Q: 다른 스냅샷에 대해 다른 가격 모델을 설정할 수 있나요?**

현재 가격 모델은 앱별로 설정됩니다. 개별 스냅샷에 대해 다른 가격을 제공하려면 별도의 앱을 만들어야 합니다.
