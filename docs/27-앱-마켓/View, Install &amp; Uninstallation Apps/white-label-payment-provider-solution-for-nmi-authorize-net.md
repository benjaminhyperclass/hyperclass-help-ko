# NMI & Authorize.net용 화이트라벨 결제 제공업체 솔루션

NMI와 AuthorizeNet 모두를 위한 화이트라벨 결제(WLP, White Label Payment) 제공업체 솔루션 지원을 도입하는 최신 업데이트를 발표하게 되어 기쁩니다. 이번 추가로 플랫폼의 기능이 크게 향상되었으며, NMI와 AuthorizeNet과 같은 기본 솔루션을 기반으로 구축된 새로운 결제 제공업체와의 원활한 통합이 가능해졌습니다.

## 통합된 마켓플레이스 목록으로 향상된 사용자 경험

사용자 경험을 한층 더 높이기 위해 결제 연동 페이지에 마켓플레이스 앱 목록을 직접 통합했습니다. 이는 사용자가 마켓플레이스를 통해 제공되는 모든 연동 제공업체를 손쉽게 찾고 설치할 수 있음을 의미합니다. 이 페이지에는 편리한 검색 기능이 있어 사용자가 원하는 제공업체를 빠르게 찾을 수 있습니다. 또한, 이제 사용자는 목록 페이지에서 직접 앱을 설치할 수 있어 프로세스가 더욱 간소화되었습니다.

## 작동 원리

개발자는 화이트라벨 결제 제공업체 카테고리로 분류된 마켓플레이스 앱을 만들기만 하면 됩니다. 설치하면 앱이 자동으로 설치 위치에 특화된 제공업체 구성을 생성합니다. 설치가 완료되면 사용자는 다른 기본 제공업체와 마찬가지로 연결 속성을 관리할 수 있습니다. 구성을 마친 후, 사용자는 WLP 제공업체를 기본 결제 옵션으로 지정하고 NMI나 AuthorizeNet의 기능을 그대로 반영하여 원활하게 활용할 수 있습니다.

## 확장된 지원

새로운 WLP 제공업체가 NMI와 AuthorizeNet이 처음부터 지원되었던 모든 플랫폼에서 지원된다는 소식을 기쁘게 알려드립니다. 여기에는 주문 양식(원스텝 및 투스텝), 전자상거래 스토어, 인보이스, 결제 링크가 포함됩니다. 🎉🎉

## 공개 API 지원

마켓플레이스 통합 외에도 결제 앱 목록에 두 개의 공개 API 지원을 추가했습니다. 이를 통해 NMI와 AuthorizeNet용 WLP 솔루션과의 원활한 통합이 가능해져 개발자가 더욱 강력한 결제 경험을 만들 수 있습니다.

POST - payments/integrations/provider/whitelabel [여기서 확인](https://highlevel.stoplight.io/docs/integrations/38fc7b49d107d-create-white-label-integration-provider)

GET - payments/integrations/provider/whitelabel [여기서 확인](https://highlevel.stoplight.io/docs/integrations/cbdced5c59dfd-list-white-label-integration-providers)
