# 워크플로우 액션: 예약 생성

예약 생성(Book Appointment) 워크플로우 액션을 사용하면 트리거가 실행될 때 선택한 캘린더에 자동으로 예약을 생성할 수 있습니다. 폼 제출, 인바운드 웹훅, CRM 업데이트 등의 이벤트로부터 후속 예약을 수동 작업 없이 자동으로 잡고 싶은 팀에게 최적화된 기능입니다.

## 예약 생성 워크플로우 액션이란?

예약 생성 액션을 사용하면 지정된 시간과 캘린더에 새로운 예약을 생성할 수 있습니다. 이제 사용자는 워크플로우를 통해 예약 생성 액션을 설정할 수 있으며, 이를 통해 트리거 이벤트에 따라 예약이 자동으로 생성됩니다. 특정 트리거를 통해 예약 예약을 자동화하고자 하는 비즈니스에게 이상적입니다.

**주요 사항:**

- 예약을 받을 캘린더를 반드시 선택해야 합니다.
- 팀원은 캘린더 기본값(예: 라운드 로빈) 또는 특정 팀원으로 설정할 수 있습니다.
- 동적 시간은 트리거 데이터(예: 웹훅 페이로드 또는 폼 필드)에서 매핑할 수 있습니다. 들어오는 형식이 지원되는 패턴과 일치하지 않으면 날짜 시간 포맷터(Date Time Formatter)를 사용하세요.
- 아무도 예약 가능 시간이 없고 강제 설정이 꺼져 있으면 예약이 생성되지 않습니다.

**제한사항:** '반복(Recurring)'이 활성화된 캘린더는 '예약 생성' 액션에서 선택할 수 없습니다.

---

## 예약 생성 워크플로우 액션 사용의 주요 이점

실제 예약 가능 시간을 존중하는 트리거 기반 스케줄링으로 수동 작업을 줄이고, 성공/실패에 따라 다음 단계를 깔끔하게 분기할 수 있습니다.

- **효율적인 후속 조치** — 폼, 채팅, 웹훅에서 자동 예약
- **높은 전환율** — 의도가 신선할 때 시간을 확정
- **유연한 배정** — 캘린더 기본값(예: 라운드 로빈) 사용 또는 특정 팀원 지정
- **신뢰할 수 있는 로직** — **예약 성공** 조건으로 확인, 알림 또는 대체 액션 분기

## 예약 생성 워크플로우 액션 설정하기

예약 생성 단계를 만들면 워크플로우를 대상 캘린더에 연결하고 언제/누구와 예약을 생성할지 정의할 수 있습니다.

### 1단계: 자동화(Automations)로 이동

Automations(자동화) → Create New Workflow(새 워크플로우 생성) → Start From Scratch(처음부터 시작)으로 이동합니다. 그런 다음 트리거를 추가하세요. 예시: **Form Submitted(폼 제출됨)**, **Inbound Webhook(인바운드 웹훅)** 또는 기타 관련 트리거.

![워크플로우 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052586842/original/0riEYOvbFN-W1cUn2lcfd4jm7A_8xWP-NA.gif?1756384342)

### 2단계: 액션 추가

**Add Action(액션 추가) → Book Appointment(예약 생성)**을 클릭하고 명확한 **Action name(액션 이름)**(예: "영업팀과 데모 예약")을 입력하세요.

![액션 추가 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052587208/original/PlSK-h6vSCDjLAUPpKBZr9mgvrSW9O0V5Q.png?1756384481)

### 3단계: 캘린더, 팀원 선택 및 예약 날짜/시간 설정

**캘린더:** 예약을 받을 대상 캘린더를 선택합니다.

**팀원:**
- **Calendar Default(캘린더 기본값)** — 캘린더 배정 규칙(예: 라운드 로빈)을 따릅니다.
- **Specific member(특정 팀원)** — 특정 한 명에게 명시적으로 라우팅합니다.

**예약 날짜 및 시간 설정:**

동적 시작 날짜와 시간의 경우, 형식이 호환되는지 확인하세요. 필요한 경우 날짜 시간 포맷터(Date Time Formatter) 액션을 사용하여 날짜 및 시간 형식을 변환하세요.

- **Standard(표준)** — 이 워크플로우로 생성되는 모든 예약에 고정된 날짜/시간
- **Dynamic(동적)** — 트리거 데이터(예: 웹훅 페이로드, 폼 필드 또는 커스텀 값)에서 날짜/시간 매핑

**지원되는 형식 (예시):**
- MM-DD-YYYY HH:MM → 12-21-2021 08:30 AM
- DD-MMM-YYYY HH:MM → 21-OCT-2021 08:30 AM

입력이 지원되는 형식과 일치하지 않으면 예약 생성 전에 **Date Time Formatter(날짜 시간 포맷터)** 단계를 삽입하여 다시 포맷하세요.

![예약 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052589247/original/kB3ftIpttgUl3WnvXJc-OGTR13VAvEksbQ.png?1756385194)

**중요**: 예약 가능 시간 무시(Override availability) —
1. Off(꺼짐): 예약 전 예약 가능 시간을 확인합니다.
2. On(켜짐): 예약 가능 시간 확인 없이 예약합니다 (신중하게 사용).

(선택사항) 템플릿에 포함되어 있다면 액션 설명 또는 내부 메모를 추가하세요.

## 자주 묻는 질문

**Q. 이 액션이 캘린더 예약 가능 시간을 존중하나요?**
네, **예약 가능 시간 무시**를 활성화하지 않는 한 존중합니다.

**Q. 성공/실패에 따라 로직을 분기할 수 있나요?**
네—**If/Else(조건 분기)**를 추가하고 **예약 성공**(True/False)을 사용하여 결과를 라우팅할 수 있습니다.

**Q. 동적 설정에서 어떤 날짜/시간 형식이 지원되나요?**
**3단계**의 형식을 참조하세요. 필요시 **날짜 시간 포맷터** 단계를 추가하세요.

**Q. 라운드 로빈을 통해 예약을 배정할 수 있나요?**
네—**캘린더 기본값**을 선택하면 캘린더의 배정 규칙을 따릅니다.

**Q. 예약이 실패하면 어떻게 되나요?**
False 분기를 사용하여 팀에 알리거나, 할 일을 생성하거나, 대안 시간을 시도해 보세요.

**Q. 예약 생성 워크플로우 액션에 제한사항이 있나요?**
**반복**이 활성화된 캘린더는 예약 생성 액션에서 선택할 수 없습니다.
