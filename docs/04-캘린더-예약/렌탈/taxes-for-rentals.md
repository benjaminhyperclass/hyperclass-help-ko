# 렌탈 세금이란?

렌탈의 세금 기능을 사용하면 깔끔하고 항목별로 나뉜 체크아웃 경험을 제공하면서 리포팅 정확도를 유지할 수 있습니다. 각 **리스팅**은 자동으로 연결된 상품을 생성하며, 해당 상품에 수동 세율을 적용할 수 있습니다.

세율은 Payments(결제) 섹션에서 설정한 후 렌탈 리스팅에 직접 적용됩니다. 렌탈 예약이 생성되면 선택한 세율이 자동으로 계산되어 다음 위치에 표시됩니다:

- 공개 예약 페이지
- 예약 상세 보기
- 렌탈 인보이스
- 영수증 및 내보내기

하나의 예약에 여러 리스팅이 포함된 경우, 리스팅별로 세금이 계산되어 최종 총액에 합산됩니다.

# 렌탈 세금의 주요 장점

이러한 장점들은 렌탈 거래 전반에서 투명성, 규정 준수, 리포팅 정확도에 중점을 둡니다.

- **규정 준수**: 리스팅별로 정확한 수동 판매세/부가세/소비세율 적용
- **일관성**: 예약 페이지, 예약 기록, 인보이스, 내보내기에서 총액 일치
- **투명성**: 결제 전 고객에게 항목별 내역 표시
- **제어**: 필요시 다른 렌탈 리스팅에 다른 세율 지정

# 렌탈 수동 세금 설정

수동 세금은 리스팅에 직접 적용되는 것이 아니라 상품(Product)에 적용됩니다. 각 리스팅에는 연결된 상품이 있기 때문에, 해당 상품에 세율을 지정하면 리스팅이 예약될 때마다 올바른 세금이 예약 총액에 자동으로 추가됩니다.

**작동 방식**

- 각 리스팅이 자동으로 연결된 상품을 생성합니다
- Payments(결제) → Settings(설정) → Taxes(세금)에서 해당 상품에 하나 이상의 세율을 추가합니다
- 리스팅이 예약되면 해당 세율이 자동으로 적용됩니다

결제 설정에서 세금 설정이 포함(inclusive)으로 되어 있으면 시스템이 리스팅 가격에서 세금 부분을 추출합니다. 별도(exclusive)로 설정되어 있으면 렌탈 소계에 세금이 추가됩니다.

## 예약 및 인보이스에서의 표시

세금이 어디에 표시되는지 알면 총액을 확인하고 고객 질문에 답변하는 데 도움이 됩니다.

세금이 표시되는 위치:

- 공개 예약 페이지 (총액 위에 항목별로 표시)
- 렌탈의 예약 편집 보기 내부
- 예약의 결제(payments) 탭 내부
- 생성된 인보이스

내부 리포팅 및 정산을 위해 렌탈 모듈 전체에서 동일한 총액이 표시됩니다.

# 렌탈 리스팅에 세금 적용하는 방법

올바른 설정을 통해 모든 예약이 정확한 총액을 반영하고, 고객 혼란을 줄이며, 정산을 빠르게 할 수 있습니다. 세금과 수수료를 모두 설정하려면 다음 단계를 따르세요.

1. 하위 계정에서 Settings(설정) → Calendars(캘린더) → Rentals(렌탈)을 클릭하고 편집할 리스팅을 선택합니다.
2. **Inventory & Pricing(재고 및 가격)** 탭으로 이동합니다.
3. **Configure Taxes for This Listing(이 리스팅의 세금 설정)**을 클릭합니다.
4. 연결된 상품 창에서 **Add Tax Rate(세율 추가)**를 선택하거나 기존 세율을 선택한 후 저장합니다.
5. 리스팅을 저장합니다. 향후 예약에서 예약 페이지, 예약 기록, 인보이스에 수수료와 세금 항목이 자동으로 표시됩니다.

적용 후에는 해당 리스팅의 향후 예약에서 체크아웃과 인보이스 내역에 자동으로 세금이 포함됩니다.

# 자주 묻는 질문

**Q: 리스팅마다 다른 세율을 사용할 수 있나요?**

네. 세금은 렌탈 리스팅별로 설정되므로 각 리스팅마다 고유한 세율을 적용할 수 있습니다.

**Q: 세금 변경이 과거 예약에 영향을 주나요?**

아니요. 변경사항은 앞으로의 새 예약에만 적용됩니다. 기존 예약은 수정하지 않는 한 변경되지 않습니다.

**Q: 다중 리스팅 예약에서 세금은 어떻게 처리되나요?**

각 리스팅이 독립적으로 세금을 계산합니다. 그런 다음 시스템이 모든 세금 금액을 최종 예약 총액에 합산합니다.

**Q: 관련 보증금에도 세금이 적용되나요?**

아니요. 세금은 리스팅에만 적용됩니다.
