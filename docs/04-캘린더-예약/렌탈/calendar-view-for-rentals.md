## 렌탈용 캘린더 뷰란?

**캘린더 뷰**는 모든 예약과 리스팅 가용성을 시각적 타임라인으로 보여줍니다. 리스팅이나 변형별로 점유된 시간대, 빈 기간, 겹치는 예약을 쉽게 파악할 수 있습니다. 새롭게 디자인된 레이아웃은 더 넓은 공간, 간소화된 필터, 대량 예약 관리를 위한 향상된 가독성을 제공합니다.

캘린더 뷰로 할 수 있는 일:

- 리스팅별, 일별, 주별로 모든 예약을 한눈에 확인
- 카테고리, 상태, 결제 상태별로 리스팅과 예약 필터링
- 캘린더에서 직접 새 예약 생성
- 기존 예약을 빠르게 열고 편집

---

## 캘린더 뷰의 주요 장점

- **한눈에 보는 계획**: 일/주/월별 예약을 보면서 빈틈과 겹침을 빠르게 발견
- **집중된 컨텍스트**: 리스팅, 변형, 렌탈 상태를 미팅/서비스와 분리하여 관리
- **스마트 필터링**: 리스팅, 카테고리, 예약 상태, 결제 상태별로 좁혀서 운영 우선순위 관리
- **빠른 예약 접수**: 캘린더에서 직접 새 예약을 만들고 기존 예약을 클릭 한 번으로 열기
- **도구 간 일관성**: 여기서 만든 항목이 예약(Appointments) 목록 뷰에 즉시 나타나 운영을 동기화

## 캘린더 뷰의 예약 유형

렌탈의 캘린더 뷰는 플랫폼의 여러 예약 유형을 지원하여 서로 다른 캘린더 경험이 겹치지 않고 공존할 수 있습니다. 하지만 렌탈은 미팅이나 서비스와 분리된 **전용 예약 유형**으로 설계되었습니다.

#### 예약 유형 개요

**미팅(Meetings)**
- 영업 캘린더, 수업, 개인 또는 공동 예약에 적합
- 개인, 라운드 로빈, 공동, 수업 캘린더 같은 기존 캘린더 유형 포함
- 클라이언트 미팅이나 내부 세션을 관리하는 팀이나 개인용

**서비스(Services)**
- 서비스 기반 비즈니스를 위해 특별히 제작
- 변형, 애드온, 다양한 서비스, 직원별 가격 같은 고급 옵션 지원
- 직원, 다양한 서비스 유형, 여러 위치가 관련된 예약 관리

**렌탈(Rentals)**
- **렌탈 비즈니스**를 위해 특별히 제작된 독립적인 예약 유형
- 다일 예약, 가용성 추적, 자원 기반 예약에 집중
- 리스팅 변형, 재고, 시간별 가격 조정 같은 고급 기능 지원
- 서비스와 분리되어 자체적인 예약 플로우, 인보이스 로직, 글로벌 설정 도입

#### 예약 유형 활성화 또는 비활성화

- 예약 유형은 **에이전시 설정(Agency Settings)**에서 관리할 수 있습니다
- 한 가지 유형(예: 렌탈)만 활성화되면 캘린더 뷰가 자동으로 해당 유형으로 기본 설정되고 예약 유형 선택기가 숨겨집니다
- 여러 예약 유형(예: 미팅과 렌탈)이 활성화되면 캘린더 뷰 상단에 드롭다운이 나타나 사용자가 전환할 수 있습니다

#### 뷰 변화 방식

드롭다운을 사용해 예약 유형을 전환할 때:

- 레이아웃과 필터 옵션이 선택된 예약 유형에 맞게 조정됩니다
- 각 유형은 자체 관련 예약, 차단된 시간대, 필터만 표시하여 유형 간 혼동을 방지합니다
- 예를 들어 **렌탈**은 리스팅, 변형, 렌탈 예약을 표시하고, **미팅**은 예약(appointments)을 보여줍니다

참고: **캘린더 뷰 개선 사항(Labs Flag)**은 **렌탈**이 활성화될 때 자동으로 활성화됩니다.

## 캘린더 뷰 이해하기

캘린더 뷰는 세 가지 주요 섹션으로 구성됩니다:

#### 1. 리스팅 (왼쪽)

- 모든 리스팅과 해당 변형을 접을 수 있는 목록으로 표시
- 각 리스팅 이름을 확장하면 변형(해당되는 경우)을 보여줍니다
- 패널을 통해 리스팅과 활성 예약을 시각적으로 매칭할 수 있습니다

![리스팅 패널](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155056624276/original/PdaI80JH3u9WWYSQsRnlfOnjeN6BKAanqQ.png?1761155999)

#### 2. 캘린더 그리드 (중앙)

- 선택한 타임라인에 걸쳐 예약을 표시
- 각 예약 블록은 하나의 예약된 구간을 나타냅니다
- 예약 위에 마우스를 올리면 예약 세부사항(고객명, 리스팅, 기간, 예약 상태)이 툴팁으로 표시됩니다
- 색상 코드로 예약 상태를 표시:
  - 연한 파란색: 예약됨(Booked)
  - 파란색: 활성(Active)
  - 초록색: 완료됨(Completed)
  - **빨간색**: 취소됨(Canceled)
  - 회색 비어있음: 미확인(Unconfirmed)
- 예약 블록을 클릭하면 **예약 세부사항**이 열립니다

![캘린더 그리드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155056624295/original/SD40G9wvQKT_NDhnlc5efD5TFZSisDqWzA.png?1761156028)

#### 3. 관리 뷰 패널 (접을 수 있음)

- 필터와 표시 기본 설정을 제어합니다
- 여러 필터 카테고리를 사용해 캘린더에 나타나는 내용을 세부 조정할 수 있습니다

![관리 뷰 패널](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155056624314/original/2s7Il0c3vUI7cJ5v92I97E90b1mE3QNxrg.png?1761156066)

## 네비게이션 컨트롤

상단 네비게이션 바는 예약을 보고 관리하는 방식을 빠르게 제어할 수 있습니다.

| 컨트롤 | 설명 |
|--------|------|
| **오늘(Today)** | 현재 날짜로 즉시 이동 |
| **날짜 선택기/화살표** | 시간상 앞뒤로 이동 |
| **일/주/월 뷰** | 세부적인 뷰와 광범위한 타임라인 뷰 전환 |
| **렌탈 드롭다운** | 서비스와 렌탈 같은 모듈 간 전환 |
| **관리 뷰** | 캘린더의 필터와 커스터마이제이션 옵션 열기 |
| **+ 신규** | 캘린더에서 직접 새 예약 생성 |

💡 **팁**: 일간 및 주간 뷰는 운영 계획에 이상적이고, 월간 뷰는 가용성과 점유 트렌드의 고수준 개요를 제공합니다.

## 관리 뷰 사용하기

**관리 뷰** 패널을 통해 예약과 리스팅을 필터링하여 가장 중요한 데이터에 집중할 수 있습니다.

#### 유형별 보기

- **전체**: 예약과 차단된 시간대 모두 표시
- **예약**: 렌탈 예약만 표시
- **차단된 시간대**: 수동으로 차단된 가용성 슬롯과 제3자 플랫폼에서 가져온 차단 표시

#### 사용 가능한 필터

- **리스팅** – 캘린더에 표시할 특정 리스팅이나 변형 선택
- **카테고리** – 카테고리별로 예약 필터링(예: 아파트, 차량, 장비)
- **상태** – 예약 상태별 필터링: 미확인, 예약됨, 활성, 완료됨, 취소됨
- **결제 상태** – 결제 진행 상황별로 예약 필터링: 대기 중, 부분 결제, 결제 완료

**검색**을 사용해 리스팅, 카테고리, 상태를 빠르게 찾고, **모두 지우기**로 필터를 재설정할 수 있습니다.

## 예약 표시 및 상호작용

- 각 예약은 그리드에서 색상 블록으로 표시됩니다
- 블록 위에 마우스를 올리면 요약 툴팁(고객명, 예약 ID, 날짜, 상태)을 볼 수 있습니다
- 예약을 클릭하면 세부 뷰가 열립니다
- 같은 리스팅이나 변형에 대해 예약이 두 개 이상 있으면 캘린더 뷰가 동적으로 크기를 확장합니다

**참고**: 여기에 표시되는 시간대는 내 프로필(My Profile)의 예약 보기 시간대(View Appointments In) 설정과 일치합니다.

## 캘린더 뷰에서 예약 생성하기

관리자는 + 신규 버튼을 사용하거나 캘린더 그리드에서 사용 가능한 날짜를 드래그하여 캘린더에서 직접 새 예약을 만들 수 있습니다.

**방법 1: + 신규 버튼 사용**

- 우상단의 **+ 신규** 버튼을 클릭합니다
- 예약 세부사항(고객, 리스팅, 기간 등)을 입력합니다
- 예약을 저장합니다

새 예약이 캘린더에 즉시 나타나고 **예약 뷰(Appointments View)**와 동기화됩니다.

💡 **팁**: 캘린더 뷰에서 직접 예약을 생성하면 리스팅 가용성과의 일치를 보장할 수 있습니다.

![새 예약 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155056624383/original/Don0KNnrWSttqXpzdVMoX8gCjxaAwwpbUQ.png?1761156199)

**방법 2: 드래그로 생성 (신규)**

사용자는 캘린더 그리드 내에서 날짜나 시간 범위를 선택하여 직접 예약을 생성할 수 있습니다.

- **리스팅 행** 내의 **미래 날짜**나 시간 슬롯을 **클릭하고 드래그**합니다
- 마우스를 놓으면 예약 생성 패널이 열립니다
- 예약 세부사항을 완성하고 **저장**합니다

선택한 시작 및 종료 날짜/시간이 자동으로 미리 채워집니다.

### ⚠️ 중요 사항

- 드래그로 생성은 미래 날짜에만 사용 가능합니다
- 과거 날짜나 현재 날짜는 지원되지 않습니다
- 오늘 날짜 예약을 생성하려면 + 신규 버튼을 사용하세요

## 차단된 시간대

차단된 시간대는 리스팅이 예약에 사용할 수 없는 기간을 나타냅니다.

- 수동으로 추가되거나 제3자 플랫폼에서 자동으로 가져올 수 있습니다
- 쉽게 구별할 수 있도록 활성 예약과 다르게 표시됩니다

**예시**: 차량이 정비를 받고 있다면, 차단된 시간대를 설정해서 해당 기간에 예약이 겹치지 않도록 할 수 있습니다.

## 모범 사례

- 여러 리스팅을 관리할 때 운영 명확성을 위해 **주간 뷰**를 사용하세요
- 캘린더를 떠나지 않고 세부사항을 빠르게 미리 보려면 예약 위에 마우스를 올리세요
- 성수기에는 **관리 뷰 → 카테고리 또는 상태** 필터를 사용해 뷰를 단순화하세요

## 자주 묻는 질문

**Q: 캘린더에서 직접 예약을 편집하거나 드래그할 수 있나요?**
현재 예약은 예약 블록을 클릭하고 **예약 편집** 페이지를 열어서만 편집할 수 있습니다.

**Q: 각 색상이 무엇을 의미하나요?**
색상은 예약 상태를 반영합니다 — 연한 파란색은 예약됨, 파란색은 활성, 초록색은 완료됨, 빨간색은 취소됨, 회색(비어있음)은 미확인을 의미합니다.

**Q: 차단된 시간대를 숨길 수 있나요?**
네. **관리 뷰 → 유형별 보기 → 예약**을 사용하면 확인된 예약만 표시됩니다.

**Q: 캘린더 업데이트가 실시간으로 나타나나요?**
네, 캘린더는 활성 세션을 가진 모든 사용자에게 자동으로 새로고침됩니다.

**Q: 내 리스팅이 보이지 않아요. 어떻게 해야 하나요?**
**관리 뷰 필터**를 확인하세요 — 특정 리스팅, 상태, 결제 필터가 적용되어 있을 수 있습니다.

**Q: 제3자 플랫폼에서 새 예약이 들어왔는데 캘린더 뷰에 표시되지 않아요.**
외부 플랫폼에서 예약이 몇 시간마다 자동으로 동기화됩니다. 특정 리스팅 > 편집 > 캘린더 동기화 탭 > 가져온 캘린더 옆의 새로고침 아이콘으로 이동하여 수동으로 재동기화하고 예약을 가져올 수 있습니다.
