# 서비스의 리소스

서비스의 리소스 기능을 사용하면 예약 제공에 필요한 실제 자산(방, 의자, 스테이션 등)을 예약하여 정확한 예약 관리를 할 수 있습니다. 이 가이드에서는 리소스가 무엇인지, 언제 사용하는지, 설정 방법을 알아보겠습니다.

# **서비스 리소스 개요**

리소스는 서비스 제공에 필요한 방, 의자, 스테이션 같은 물리적 자산을 나타냅니다. 서비스에 리소스를 할당하면 해당 서비스가 예약될 때 관련 리소스도 함께 예약되어 중복 예약을 방지하고 예약 가능 여부를 정확하게 유지할 수 있습니다.

리소스는 가용성이 중요할 때만 생성해야 합니다. 예를 들어:

- 헤어컷용 가위는 충분히 많기 때문에 리소스로 만들 필요가 없습니다.
- 헤어컷 의자는 2개만 있다면 리소스로 만들어야 합니다. 직원이 3명 있어도 의자 제한 때문에 동시에 2건의 예약만 가능하기 때문입니다.

참고: 현재 각 서비스는 한 번에 하나의 리소스만 예약할 수 있지만, 여러 리소스(예: 의자 1, 의자 2)를 할당할 수 있습니다. 시스템이 자동으로 사용 가능한 리소스 중 하나를 예약합니다.

## 리소스의 주요 장점

장점을 이해하면 일정 관리 프로세스에서 리소스를 가장 효과적으로 활용할 수 있는 부분을 결정할 수 있습니다.

- 일정 관리 실수 감소: 중복 예약을 방지하여 막판 취소, 일정 변경, 고객 불만을 피할 수 있습니다.
- 수용 인원 제어: 리소스의 정의된 수용 인원을 기반으로 동시 예약 가능 건수를 제한할 수 있습니다.
- 수동 감시 작업 감소: 서비스 예약 시 필요한 자산을 자동으로 예약하여 팀이 지속적으로 확인할 필요가 없습니다.
- 예약 흐름 유지: 동등한 여러 리소스가 할당된 경우 시스템이 사용 가능한 리소스 중 하나를 예약하므로, 한 방이나 의자가 바빠도 전체 일정이 멈추지 않습니다.

## **수용 인원 vs. 수량**

리소스 수량과 리소스 수용 인원을 구분하는 것이 중요합니다:

- 수량: 각 물리적 단위마다 별도의 리소스를 생성합니다(예: 방 1, 방 2, 방 3).
- 수용 인원: 해당 단위가 동시에 처리할 수 있는 사람/예약 수를 설정합니다(예: 방 1의 수용 인원은 4명).

예시:
- 스파에 방이 3개 있음 → 3개 리소스 생성: 스파룸 1, 스파룸 2, 스파룸 3
- 스파룸 1이 동시에 4명을 수용할 수 있음 → 수용 인원을 4로 설정

## **리소스 생성 방법**

아래 단계에 따라 리소스를 설정하세요. 세부 정보, 시간, 가격, 직원, 미리보기를 설정하여 예약 페이지가 정확하고 공유할 준비가 되도록 합니다.

#### **1단계:** + 새 리소스로 이동

- 리소스 페이지로 이동합니다 (**캘린더(Calendar)** > **캘린더 설정(Calendar Settings)** > **서비스(Services)** > 리소스(Resources))
- 우측 상단의 + 새 리소스(New Resource) 버튼을 클릭합니다

![새 리소스 생성 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053607943/original/u-8iQ_g3rGmJtk0hWsj7jmichttUeiSzrQ.png?1757621639)

#### **2단계:** 리소스 세부 정보 추가

- 리소스 이름을 입력하고 선택적으로 설명을 추가합니다.
- 이 리소스를 사용할 서비스를 선택합니다.
- 리소스를 사용할 수 있는 위치(로케이션)를 선택합니다.
- 각 위치별로 수용 인원을 정의합니다(여러 위치인 경우).
- 파란색 생성(Create) 버튼을 클릭하여 변경 사항을 저장합니다

생성되면 관련 서비스가 예약될 때마다 리소스가 자동으로 예약됩니다.

![리소스 세부 정보 입력 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053607978/original/qt_R0Dk-HrlUUGo7t_eVmZYOEaeRaf83ng.png?1757621736)

## **서비스에 리소스 할당**

리소스를 생성하거나 편집할 때 다음을 수행해야 합니다:

- 서비스 할당 – 이 리소스가 필요한 서비스를 선택합니다.
- 위치 할당 – 리소스를 사용할 수 있는 위치를 선택합니다(여러 위치를 운영하는 경우 유용).
- 수용 인원 설정 – 유연한 일정 관리를 위해 각 위치별 최대 수용 인원을 정의합니다.

서비스(Services) 탭에서 또는 리소스(Resources) 탭에서 직접 리소스를 할당할 수 있습니다. 연결되면 할당된 서비스가 예약될 때마다 시스템이 자동으로 해당 리소스를 차단합니다. 요청된 시간에 리소스를 사용할 수 없으면 예약할 수 없습니다.

## **모범 사례**

- 실제 제한 요소(예: 치료실, 의자, 기계)에 대해서만 리소스를 생성하세요.
- 명확하고 설명적인 이름을 사용하세요(예: 스파룸 1, 헤어컷 의자 2).
- 과다 예약을 피하기 위해 항상 올바른 수용 인원을 설정하세요.
- 정확한 일정 관리를 위해 모든 위치에서 일관되게 리소스를 할당하세요.

리소스는 서비스가 직원에게만 의존하지 않고 서비스 제공에 필요한 실제 자산과도 연결되도록 합니다. 이를 통해 더 원활한 일정 관리, 정확한 예약 가능 여부, 더 나은 고객 경험을 보장할 수 있습니다.

## **자주 묻는 질문**

**Q: 리소스 탭이 보이지 않는데 어떻게 켜나요?**  
하위 계정에서 서비스가 활성화되어 있는지 확인하세요. [서비스를 켜는 방법은 여기를 클릭하세요](../services-overview-how-to-get-started.md)

**Q: 서비스가 한 번에 둘 이상의 리소스를 예약할 수 있나요?**  
아니요. 서비스는 한 번에 하나의 리소스만 예약할 수 있습니다. 서비스에 여러 리소스를 할당할 수 있으며, 시스템이 사용 가능한 리소스 중 하나를 예약합니다.

**Q: 필요한 리소스를 사용할 수 없으면 어떻게 되나요?**  
해당 시간대가 제공되지 않으며, 그 시간에는 예약할 수 없습니다.

**Q: 방/장비와 서비스의 리소스의 차이점은 무엇인가요?**  
방 & 장비는 캘린더 방 & 장비에서 관리되어 캘린더 수준에서 중복 예약을 방지하는 반면, 리소스는 서비스 안에 있으며 서비스 예약과 연결됩니다. 별도의 설정이므로 워크플로우에 가장 적합한 것을 선택하세요.

**Q: 내 서비스가 공개 예약 페이지에 표시되지 않는 이유는 무엇인가요?**  
서비스가 비공개(Private)로 설정되었는지 확인하세요. 비공개 서비스는 공개 예약 페이지에 나타나지 않으니 직접 링크를 공유하세요.
