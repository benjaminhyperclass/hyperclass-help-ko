## 캘린더에 커스텀 폼 추가란

Hyperclass의 커스텀 폼을 사용하면 연락처가 예약을 잡기 전에 개인화된 정보를 수집할 수 있습니다. 캘린더에 폼을 연결하면 사용자가 시간대를 선택한 후 반드시 폼을 작성해야 합니다. 이를 통해 중요한 데이터가 연락처의 CRM 레코드에 직접 저장되어 개인화되고 효율적인 후속 조치가 가능해집니다.

캘린더에 커스텀 폼을 추가하려면 먼저 하위 계정에서 폼을 생성해야 합니다. [커스텀 폼 생성 방법은 여기를 클릭하세요.](../../06-사이트/폼/how-to-quickly-add-and-edit-custom-fields-in-forms-and-surveys.md)

## 캘린더에 커스텀 폼을 사용하는 주요 장점

캘린더 예약 흐름에 커스텀 폼을 연결하면 예약 설정 전에 가장 관련성 높은 정보를 수집할 수 있습니다. 이를 통해 리드 처리가 개선되고 데이터 수집이 자동화되며 개인화된 경험 제공 능력이 향상됩니다.

- **리드 자격 심사**: 예약 전 중요한 질문을 통해 자격 미달 리드를 걸러내세요.
- **자동 데이터 수집**: 폼 응답을 연락처 레코드에 직접 매핑하여 CRM 효율성을 높이세요.
- **개인화 개선**: 수집된 데이터를 사용해 커뮤니케이션, 서비스, 제안을 맞춤화하세요.
- **불필요한 소통 감소**: 필요한 모든 정보를 미리 받아 지연이나 후속 조치를 피하세요.
- **규정 준수 및 동의**: 법적 요구사항을 충족하는 약관, 계약서, 동의 체크박스를 추가하세요.
- **워크플로우 트리거**: 폼에 제출된 커스텀 필드 응답을 기반으로 자동화를 시작하세요.
- **더 나은 고객 경험**: 고객이 예약 전 정보를 공유할 수 있는 원활하고 체계적인 방법을 제공하세요.

## 캘린더에 커스텀 폼 설정하는 방법

캘린더에 커스텀 폼을 연결하면 사용자가 예약하기 전에 핵심 정보를 수집할 수 있습니다. 몇 가지 간단한 단계만으로 설정할 수 있습니다.

### 1단계: 원하는 캘린더로 이동

- 메인 메뉴에서 캘린더(Calendars)로 이동한 후 보조 네비게이션 리본에서 캘린더 설정(Calendar Settings)을 클릭합니다.
- 편집하려는 캘린더를 찾아 캘린더 이름 오른쪽에 있는 편집(Edit) 버튼을 클릭합니다.

![캘린더 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044905277/original/S3Sy_oBahEFwxfShSp0ddxzFCOreTosp9Q.png?1744309552)

### 2단계: 캘린더에 커스텀 폼 연결

- 왼쪽 네비게이션 리본에서 폼 & 결제(Forms & Payment) 탭으로 이동합니다.
- 메인 패널의 폼 선택(Select Form) 드롭다운에서 원하는 폼을 선택합니다.
- 오른쪽 상단의 저장(Save) 버튼을 클릭합니다.

![폼 연결 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044905349/original/_Y-mZAbGDWIfsPaRWOIDaFFDtyVDBDvX4A.png?1744309637)

## 여러 캘린더에 같은 폼을 추가하는 방법

비즈니스에서 여러 예약 유형이나 팀원 캘린더를 제공하는 경우, 하나의 일관된 폼을 재사용하면 시간을 절약하고 균일한 데이터 수집을 유지할 수 있습니다. 서비스나 부서 전반에 걸쳐 동일한 접수 또는 자격 심사 데이터가 필요할 때 유용합니다.

- 사이트(Sites) > 폼(Forms)에서 폼을 생성한 후 여러 캘린더에 할당할 수 있습니다.
- 폼을 할당하려는 각 캘린더에 대해 동일한 과정을 반복하기만 하면 됩니다:
  - 캘린더 설정(Calendar Settings)으로 이동
  - 편집(Edit) 버튼 클릭
  - 폼 선택(Select Form) 드롭다운에서 폼 선택

**커스텀 폼은 캘린더별로 개별 추가해야 합니다**
폼을 캘린더에 추가하는 것은 수동 과정이며, 폼이 모든 캘린더에 자동 적용되지 않습니다. 여러 캘린더에 같은 폼을 사용하더라도 각 캘린더 설정에 개별적으로 추가해야 합니다. 이를 통해 어떤 예약 흐름에 어떤 폼이 나타나는지 제어할 수 있습니다.

## 자주 묻는 질문

**Q: 하나의 캘린더에 여러 개의 커스텀 폼을 사용할 수 있나요?**
아니오. 캘린더에는 한 번에 하나의 커스텀 폼만 연결할 수 있습니다.

**Q: 폼에 제출된 데이터는 어떻게 되나요?**
폼 응답은 폼 필드가 커스텀 필드에 매핑되어 있는 경우 CRM의 연락처 레코드에 자동으로 저장됩니다.

**Q: 여러 캘린더에 같은 폼을 사용할 수 있나요?**
네. 하나의 폼을 각 캘린더 설정에서 선택하여 여러 캘린더에 할당할 수 있습니다.

**Q: 캘린더 드롭다운에 내 폼이 표시되지 않는 이유는 무엇인가요?**
폼이 사이트(Sites) > 폼(Forms)에 저장되어 있고 최소 한 개의 필드를 포함하고 있는지 확인하세요. 저장 후 캘린더 설정 페이지를 새로고침해야 할 수도 있습니다.

**Q: 폼은 시간대 선택 전에 나타나나요, 후에 나타나나요?**
폼은 시간대 선택 후에 나타납니다. 사용자가 먼저 시간을 선택한 다음 커스텀 폼을 작성하여 예약을 확정합니다.

**Q: 캘린더에 연결한 후 폼을 편집할 수 있나요?**
네. 폼에 대한 모든 변경 사항은 연결된 캘린더에 자동으로 반영됩니다. 하지만 자동화나 레코드와 연결된 중요한 필드는 삭제하지 않도록 주의하세요.

**Q: 예약이 완료되지 않으면 커스텀 폼 데이터가 저장되나요?**
커스텀 폼 응답은 방문자가 최종 '예약하기(Book Appointment)' 버튼을 클릭한 후에만 연락처 레코드에 저장됩니다. 방문자가 그 단계 전에 과정을 중단하면 CRM에 데이터가 기록되지 않습니다.
