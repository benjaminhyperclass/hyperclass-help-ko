## **구글 캘린더 연동이 끊어지는 원인은 무엇인가요?**

구글 캘린더 연동이 중단되는 경우, 일반적으로 다음 중 하나가 원인입니다:

- 사용자가 구글 계정 비밀번호를 변경한 경우
- 구글 설정에서 Hyperclass의 권한이 취소된 경우
- 구글이 의심스러운 활동을 감지하여 자동으로 접근을 차단한 경우
- OAuth 토큰(보안 연결용)이 만료되거나 무효화된 경우

이런 상황이 발생하면 캘린더 동기화가 중단되고, 새로운 예약(Appointment)이 구글 캘린더에 반영되지 않을 수 있습니다.

## **끊어진 구글 캘린더 연동을 복구하는 방법**

구글 캘린더가 연결 해제되거나 동기화에 실패하는 경우, 일반적으로 구글이 Hyperclass의 접근 권한을 취소했기 때문입니다. 이는 비밀번호 변경, 권한 취소, 또는 구글 계정의 보안 트리거로 인해 발생할 수 있습니다.

다음 단계에 따라 Hyperclass에서 구글 캘린더 연동을 다시 인증하세요:

### 1단계: 구글 캘린더 삭제

- 하위 계정(Sub-account)에서 Settings(설정)으로 이동합니다.
- My Profile(내 프로필) 옵션을 선택하고 Calendar Settings(캘린더 설정)으로 이동합니다.
- 연결된 구글 계정의 휴지통/삭제 아이콘을 클릭합니다.
- 팝업에서 확인 체크박스를 선택하고 Confirm(확인)을 클릭합니다.

### 2단계: 구글 캘린더 재연동

- 하위 계정 설정(Settings)을 클릭하고 내 프로필(My Profile) 옵션을 선택합니다.
- 캘린더 설정(Calendar Settings) 하에서 + Add New(+ 새로 추가) 버튼을 클릭합니다.

### 3단계: 테스트 및 확인

재연동 과정을 완료한 후, 구글 캘린더를 테스트하여 동기화 과정이 제대로 작동하는지 확인하세요. 테스트 후에도 여전히 문제가 발생한다면, 추가 지원을 위해 고객 지원팀에 문의해 주세요.

이 재연동 과정에 대한 스크린샷과 더 자세한 내용이 포함된 시각적 가이드를 원하신다면, [이 단계별 가이드 문서](../기타/how-to-re-integrate-google-calendar-with-highlevel-for-a-user.md)를 참조하세요.

## **자주 묻는 질문**

**Q: 구글 캘린더 연동이 끊어졌는지 어떻게 알 수 있나요?**
새로운 예약이 구글 캘린더에 나타나지 않거나, 연동 설정에서 "연결 해제됨" 메시지를 보게 됩니다.

**Q: 구글 비밀번호를 변경한 후 재연동해야 하나요?**
네. 비밀번호 변경은 Hyperclass를 포함한 타사 접근 권한을 취소합니다. 수동으로 접근 권한을 다시 승인해야 합니다.

**Q: 캘린더 연결이 끊어지면 기존 예약을 잃게 되나요?**
아니요, Hyperclass의 기존 예약은 그대로 유지됩니다. 재연동 후 동기화가 재개됩니다.

**Q: Hyperclass에서 여러 구글 캘린더를 사용할 수 있나요?**
사용자당 하나의 주 구글 캘린더를 동기화용으로 선택할 수 있습니다. 필요시 캘린더 설정 화면에서 캘린더를 변경할 수 있습니다.
