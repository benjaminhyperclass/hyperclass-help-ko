# 서비스 만들기 가이드

서비스(Services) 기능을 사용하면 고객이 온라인에서 직접 예약할 수 있는 예약 가능한 서비스를 만들 수 있습니다. 하나의 서비스 아래에 여러 옵션을 제공하는 변형(Variations), 주문 금액을 높이는 애드온(Add-ons), 예약 충돌을 방지하는 리소스(Resources)를 활용해보세요. 이 가이드에서는 서비스(Services) 패널 접근, 카테고리로 서비스 정리, 새 서비스 만드는 방법을 안내합니다.

## 서비스란?

서비스(Services)는 코칭 세션, 스파 트리트먼트, 피트니스 프로그램 등 비즈니스가 고객에게 제공하는 서비스를 정의하는 곳입니다. 고객이 온라인으로 예약할 수 있는 서비스 목록을 만들면서 뒤에서는 체계적으로 관리할 수 있습니다.

언제, 어떻게 서비스를 제공할지, 누가 담당할지 관리하고, 선택적 애드온(Add-ons), 리소스 관리, 카테고리로 서비스를 강화할 수 있습니다. 서비스는 단순한 예약뿐만 아니라 다음과 같은 고급 비즈니스 플로우를 지원합니다:

- 다중 지점 예약
- 서비스 변형 (예: 다양한 시간이나 가격대)
- 애드온을 통한 업셀
- 예약금 결제

## 서비스 생성의 주요 장점

각 기능이 무엇인지 알면 설정이 더 빨라지고 예약 페이지를 정확하게 유지할 수 있습니다. 아래 장점들은 서비스를 만들거나 편집할 때 직접 설정하는 옵션들입니다:

- **변형(Variations)**: 하나의 서비스 내에서 여러 옵션(다른 길이나 등급)을 제공하며, 각각 고유한 시간과 가격 설정
- **애드온(Add-ons)**: 결제 시 선택할 수 있는 추가 옵션으로 예약 가격을 높일 수 있음
- **리소스(Resources)**: 룸이나 스테이션 같은 제한된 항목을 예약하여 중복 예약 방지
- **처리 시간 및 버퍼**: 원활한 예약을 위한 현실적인 간격과 서비스 내 시간 설정
- **직원 및 가용성**: 서비스에 팀원을 배정하고 근무 시간 관리
- **카테고리**: 서비스를 그룹화하고 예약 페이지에 표시되는 순서 제어
- **비공개 서비스**: 특정 서비스를 공개 목록에서 숨기고 직접 공유할 때만 사용
- **커스텀 URL 및 미리보기**: 깔끔한 링크 공유와 홍보 전 예약 경험 확인
- **예약금 및 가격**: 예약 시 예약금 수취와 가격 설정 관리

## 서비스 패널 접근하기

서비스 패널(Services Panel)은 예약 경험을 구축, 정리, 미리보기하는 시작점입니다. 서비스 패널에서는 다음을 확인할 수 있습니다:

- 검색 및 정렬 가능한 전체 서비스 목록
- 새 서비스 만들기 옵션
- 각 서비스의 이름, 시간, 가격, 배정된 직원
- 쉬운 정리를 위해 좌측에 표시되는 카테고리
- 편집, 복사, 삭제, 공유 액션을 위한 점 3개 메뉴
- 고객용 접근을 위한 예약 페이지 보기 링크

서비스 패널에서 서비스 목록에 접근하려면 다음 단계를 따르세요:

- 좌측 내비게이션 바에서 `Calendars`(캘린더) 탭을 클릭
- 상단 내비게이션 리본에서 `Calendar Settings`(캘린더 설정) 선택
- 캘린더 설정 페이지 상단 내비게이션 리본에서 `Services`(서비스) 탭 선택

## 카테고리 만들기 및 관리

서비스 카테고리는 유사하거나 관련된 서비스를 그룹화하는 방법으로, 고객이 서비스를 더 쉽게 탐색할 수 있게 합니다. 이 카테고리들은 예약 페이지에 표시되며, 그 아래에 구체적인 서비스들이 나열됩니다.

- `+ New Category`(+ 새 카테고리) 버튼을 클릭하여 새 카테고리 생성
- 점 3개(⋮) 아이콘을 사용하여 카테고리 편집
- 카테고리를 드래그하여 순서 재배치; 순서는 예약 페이지 레이아웃에 반영됨

![카테고리 관리 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053193881/original/6Dly2UQCoo1DUa-WUMzCBkaJ-2L1Nfx7mQ.png?1757089527)

## 서비스 관리 기능

서비스 에디터를 사용하여 기존 기능을 편집하고 업데이트하세요. 에디터를 열려면 서비스 패널에서 서비스 옆의 점 3개(⋮) 아이콘을 클릭하고 `Edit`(편집)을 선택하세요. 서비스 에디터에는 다음이 포함됩니다:

- **서비스 세부사항**: 직원, 시간, 버퍼 시간, 처리, 변형, 가격, 결제, 예약금
- **애드온**: 추가 옵션으로 서비스를 업셀하거나 강화
- **리소스**: 서비스에 필요한 룸, 의자 또는 기타 자산 배정

![서비스 에디터 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053195733/original/IXCx7zTww4I0Vpf_veiiL1h7A98S2-wMvQ.png?1757092032)

## 애드온이란?

애드온(Add-ons)은 고객이 메인 서비스와 함께 예약할 수 있는 선택적 추가 옵션을 제공하여 서비스를 강화하는 기능입니다. 마사지 오일, 트리트먼트, 액세서리 등의 추가 기능을 업셀하거나 고객이 예약을 개인화할 수 있게 하는 데 완벽합니다. 애드온은 예약 페이지에서 고객용으로 표시되거나 직원이 예약 생성 시 내부적으로 사용할 수 있습니다.

### 예약 페이지에서 애드온 표시 방법

- 고객용 애드온은 서비스 예약 시 선택 가능한 체크박스로 표시
- 애드온 가격과/또는 시간이 총 예약 비용에 추가됨
- 숨겨진 애드온은 고객에게 표시되지 않지만 직원이 예약 모달에서 선택 가능

### 애드온 접근하기

- 서비스 패널로 이동 (`Calendar`(캘린더) > `Calendar Settings`(캘린더 설정) > `Services`(서비스))
- 선택한 서비스의 서비스 에디터 열기 (`점 3개(⋮)` > `Edit`(편집)) 후 `Add-ons`(애드온) 탭 클릭
- 여기서 선택한 서비스에 연결된 애드온 목록과 가격, 시간, 최대 수량을 확인할 수 있습니다
- 이 탭에서 새 애드온을 만들거나 기존 애드온을 편집할 수 있습니다

[애드온 생성 및 편집 방법을 포함한 자세한 정보는 여기를 클릭하세요](서비스/add-ons-in-services.md)

## 리소스란?

리소스(Resources)는 룸, 의자, 스테이션, 도구 등 서비스가 진행되기 위해 반드시 사용 가능해야 하는 공유 자산입니다. 서비스에 리소스를 배정하면 중복 예약을 방지하고 정확한 용량 관리를 보장할 수 있습니다.

각 리소스는 하나 이상의 서비스에 연결될 수 있으며 다음으로 설정됩니다:

- **지점**: 각 리소스가 사용 가능한 위치 배정
- **용량**: 리소스가 수용할 수 있는 예약 또는 인원 수 설정

예를 들어, "Room 1" 리소스는 여러 서비스에서 사용될 수 있지만 한 번에 4명만 지원할 수 있습니다. 세밀한 제어를 위해 지점별로 이 용량을 지정할 수 있습니다.

### 리소스 접근하기

- 서비스 패널로 이동 (`Calendar`(캘린더) > `Calendar Settings`(캘린더 설정) > `Services`(서비스))
- 선택한 서비스의 서비스 에디터 열기 (`점 3개(⋮)` > `Edit`(편집)) 후 `Resources`(리소스) 탭 클릭
- 여기서 리소스 목록과 총 용량, 연결된 서비스, 연결된 지점, 상태를 확인할 수 있습니다
- 이 탭에서 새 리소스를 만들거나 기존 리소스를 편집할 수 있습니다

[리소스 생성 및 편집 방법을 포함한 자세한 정보는 여기를 클릭하세요](서비스/resources-in-services.md)

## 새 서비스 만들기

아래 단계를 따라 서비스를 설정하세요—세부사항, 시간, 가격, 직원, 미리보기—예약 페이지가 정확하고 공유할 준비가 되도록 합니다.

### 1단계: + New Service 클릭

- 서비스 패널로 이동 (`Calendar`(캘린더) > `Calendar Settings`(캘린더 설정) > `Services`(서비스))
- 우측 상단의 `+ New Service`(+ 새 서비스) 버튼 클릭

![새 서비스 만들기 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053200512/original/JllFIQ5or4T6MV8J87DSZJtJpDmKe_tX2Q.png?1757099763)

### 2단계: 기본 정보 추가

서비스의 기본 정보를 입력하세요. 이 정보에는 다음이 포함됩니다:

- **Service Image**(서비스 이미지): 서비스를 시각적으로 나타내는 이미지 업로드. 기기에서 업로드하거나 미디어 라이브러리 사용 가능
- **Service Name**(서비스 이름): 고객에게 표시되는 서비스 제목
- **Service Color**(서비스 색상): 캘린더에서 이 서비스를 식별하는 데 도움이 되는 색상 배정
- **Category**(카테고리): 관련 카테고리 아래에 서비스 배치 (선택하지 않으면 새 서비스가 기본 카테고리에 자동 추가)
- **Custom URL**(커스텀 URL): 서비스의 고유한 예약 링크 설정
- **Description**(설명): 서비스가 제공하는 내용에 대한 간단한 요약 추가. 리치 텍스트 에디터를 사용하여 이 콘텐츠를 포맷할 수 있습니다 (볼드/이탤릭/밑줄/취소선, 불릿 또는 번호 목록, 하이퍼링크, 텍스트/배경색, 제목, 텍스트 정렬)
- **Private Service**(비공개 서비스): 공개 예약 페이지에서 이 서비스를 숨김. 새 예약은 플랫폼을 통해 내부적으로만 예약 가능
- **Staff Members**(직원): 이 서비스를 제공할 수 있는 직원 배정. 서비스가 예약 가능하려면 최소 한 명의 직원이 배정되어야 함
- **Duration**(시간): 서비스가 지속될 시간 추가
- 모든 기본 정보를 추가한 후 `Advanced Settings`(고급 설정)을 클릭하여 추가 서비스 세부사항 구성

![서비스 기본 정보 입력 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064240832/original/mtgpoUtmjJTfbLEIars6IidCO77vcShExQ.png?1770282550)

### 3단계: 서비스 세부사항 구성

- **Duration**(시간): 서비스가 지속되는 시간 설정
- **Processing Time**(처리 시간): 시간을 3단계(초기, 처리, 최종)로 나눔. 직원은 초기/최종 단계에서만 예약되며, 처리 단계 중에는 다른 예약에 사용 가능
- **Buffer Time**(버퍼 시간): 준비나 정리를 위해 예약 전후에 시간 추가

![서비스 세부사항 구성 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053200659/original/3SudWPP0pe6TcJ3z9FBQS9RJcEcPyyolmQ.png?1757100317)

### 4단계: 결제 설정

- **Price**(가격): 고객이 지불할 가격 입력
- **Partial Payments**(부분 결제): 고객이 전액 대신 예약금을 지불할 수 있도록 허용. 고정 금액 또는 백분율 중 선택

![결제 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053200704/original/Q1UOse8gPASRMJNB-W_Hn8cvzePFg1KUqw.png?1757100437)

### 5단계: 변형 구성 (선택사항)

변형(Variations)을 활성화하여 하나의 서비스 아래에 여러 옵션을 제공하세요 (예: 다른 시간이나 가격).

각 변형의 이름, 시간, 가격, 처리 시간(있는 경우) 추가

예시:
- 30분 – $120
- 45분 – $100 
- 60분 – $350

![변형 구성 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053200766/original/28G7T1XuQ7SmiRxpjXjH8u0sSH1-UBj-Jw.png?1757100623)

### 6단계: 저장 및 미리보기

- `Save Changes`(변경사항 저장)을 클릭하여 업데이트 적용
- `View Booking Page`(예약 페이지 보기)를 사용하여 고객이 서비스를 어떻게 볼지 미리보기

![저장 및 미리보기 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053200841/original/sKKRX3ykjoipospSSPJNtwBNv7P_7zxc9w.png?1757100758)

- 예약 가능하게 하려면 모든 서비스에 직원을 배정하고 가용성을 구성하세요
- 서비스를 의미 있게 카테고리로 정리하세요
- 업셀과 커스텀 옵션을 위해 애드온을 활용하세요
- 리소스를 사용하여 과도한 활용을 방지하고 연결된 리소스의 가용성을 최적화하세요

## 자주 묻는 질문

**Q: 공개 예약 페이지에서 서비스를 숨길 수 있나요?**
네. 서비스에 "This is a private service"(비공개 서비스)를 활성화하고 대신 직접 링크를 공유하세요. 이 기능을 켜면 서비스가 공개 예약 페이지에 표시되지 않습니다.

**Q: 카테고리가 예약 페이지에 어떤 영향을 주나요?**
카테고리 배정과 순서는 공개 예약 페이지에서 서비스가 그룹화되고 표시되는 방식을 결정합니다. 에디터에서 카테고리 순서를 변경하면 예약 페이지의 순서도 변경됩니다.

**Q: 처리 시간과 버퍼의 차이는 무엇인가요?**
처리 시간은 서비스가 여전히 진행 중인 예약의 일부입니다. 버퍼는 예약을 간격 두기 위해 예약 전후에 추가되는 여분의 시간입니다(예약 전 준비나 예약 후 정리에 유용).

**Q: 서비스에 연결된 상품의 재고 추적이 서비스 예약과 동기화되나요?**
아니요. 서비스에 연결된 상품의 재고 추적은 서비스 예약 생명주기와 관리되거나 동기화되지 않습니다.
