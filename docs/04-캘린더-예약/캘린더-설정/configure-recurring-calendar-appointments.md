# 반복 예약 캘린더 설정하기

반복 예약이란 매일, 매주, 매월과 같이 정기적인 간격으로 발생하는 예약이나 미팅을 말합니다. 반복 예약을 캘린더에서 사용하면 정기적인 미팅이나 예약을 일정 잡을 때 시간과 노력을 절약할 수 있습니다. 매번 새로운 예약을 만드는 대신, 하나의 반복 예약을 설정해서 여러 번의 발생을 한 번에 처리할 수 있죠. 이는 주간 회의나 월간 점검처럼 정기적으로 반복되는 예약에 특히 유용합니다.

**Availability(예약 가능 시간)** 탭에서 **Recurring Appointments(반복 예약)**을 활성화하세요:

반복 예약을 활성화하면 반복 예약 설정이 나타나며, 사용 사례에 따라 설정을 선택할 수 있습니다.

![반복 예약 설정 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840128/original/88X8wjNcPWLFIiAF-8gZUpdcYlTmN-28Yg.png?1677421681)

**참고사항:**

"반복 횟수" 필드에는 제한이 있습니다. 이벤트는 최대 24회까지 반복할 수 있습니다.

"시간대가 사용할 수 없는 경우" 드롭다운에는 세 가지 옵션이 있습니다:

![시간대 사용 불가 시 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840460/original/AOfeyq6Z6N0P8PpxZtbKhK4zpZbZl_fm_Q.png?1677422054)

이는 기본적으로 의도한 시간대에 이미 다른 예약이나 이벤트가 예약되어 있는 경우, 사용자가 다음 옵션 중에서 선택할 수 있다는 의미입니다:

**Continue Booking(예약 계속)**

"시간대가 사용할 수 없는 경우" 드롭다운에서 Continue Booking을 선택하면, 이중 예약이 발생하는 경우 어떻게 할지를 묻는 추가 드롭다운이 나타납니다:

![이중 예약 처리 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840273/original/DHilYjLU6R1SX388AQrbu1ACjhRMO5u3PQ.png?1677421858)

Continue Booking을 선택하면 캘린더가 해당 시간대에 이중 예약을 허용한다는 의미입니다.

**Skip booking Unavailable slot(사용 불가 시간대 건너뛰기)**

이는 단순히 사용 가능한 시간대만 예약하고 나머지는 무시한다는 뜻입니다.

**Book Next Available Slot(다음 사용 가능 시간대 예약)**

이는 캘린더가 사용 가능한 시간대를 찾기 위해 미래의 얼마나 멀리까지 검색해야 하는지에 관계없이 반복 예약을 진행한다는 의미입니다.

**참고사항**

사용 가능성을 확인할 때 다음 3개 시간대까지만 확인하며, 사용할 수 없으면 예약을 취소합니다. 사용 가능하면 계속 진행하여 모든 반복 예약을 완료합니다. 반복 예약 옵션을 선택할 때는 신중하게 선택해야 합니다(다음 시간대 예약 vs 건너뛰기 vs 계속 예약).

반복 예약은 표준 시간대에서만 예약할 수 있습니다(커스텀 시간대의 경우 반복 예약 옵션이 표시되지 않음). 사용자는 기회 관리 페이지에서 반복 예약을 할 수 없습니다. 반복 캘린더는 드롭다운에서 예약할 수 있지만 단일 시간대에 대해서만 가능합니다.

## **FAQ**

### 기존 워크플로우 트리거가 반복 예약에도 실행되나요?

아니요, 실행되지 않습니다. 예약 트리거 단계에 새로운 필터를 추가했습니다. 따라서 기존에 설정된 모든 트리거에 워크플로우 트리거 단계를 위한 새 필터를 추가해야 합니다.

![예약 트리거 필터 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283840827/original/5LJVNP4M4V1PcJdw4bhHXsd1U5LSVroS8Q.png?1677422413)

예약 트리거 단계에 새로운 이벤트 유형 필터를 추가하여, 이제 예약 유형에 따라 실행될 수 있는 워크플로우 트리거를 제어할 수 있습니다. 워크플로우 트리거가 반복 예약에만 실행되도록 하려면 이벤트 유형으로 Recurring을 선택하면 됩니다.

계정에 있는 모든 기존 예약 상태 트리거에 대해 기본적으로 새 필터를 추가했으며, 이 필터는 이벤트 유형: Normal입니다. 반복 예약에 대해서는 실행되지 않습니다.

**Event Type: Normal(이벤트 유형: 일반)**

이 필터를 선택하면, 일반 예약(반복되지 않는)과 모든 기존 비반복 예약이 일반으로 간주됩니다. 이를 선택하면 현재와 동일하게 작동합니다.

**Event Type: Recurring(이벤트 유형: 반복)**

이 필터를 선택하면, 반복 예약 설정이 켜져 있는 캘린더에서 예약된 예약에 대해서만 트리거가 실행됩니다.

**Event Type: Any(이벤트 유형: 모든 유형)**

이 필터를 선택하면, 모든 유형의 예약에 대해 트리거가 실행됩니다.

**참고사항:**

Customer Booked Appointment Trigger(고객 예약 트리거)는 더 이상 사용하지 않습니다. 따라서 이제 Modified By Filter(수정자 필터)를 선택하면:

![수정자 필터 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48283841067/original/oc4eOeQBCx4TbF0am_aA-sClJBhUgf1_3g.png?1677422783)

**Customer(고객):** 고객 예약과 동일하게 작동하므로, 이 필터가 추가되면 고객이 예약한 예약, 즉 위젯에서 예약한 경우에만 실행됩니다.

**User(사용자):** 시스템 내부나 위젯이 아닌 다른 소스에서 예약된 모든 예약이 실행됩니다.

**API:** API를 통해 생성된 예약에만 작동합니다.

### 예약을 일괄 수정할 수 있나요?

사용자는 반복 예약을 일괄 수정(취소 또는 일정 변경)할 수 없습니다.

### 모든 반복 예약에 동일한 미팅 장소를 사용할 수 있나요?

미팅 장소가 Zoom이나 Google Meet인 경우, 시스템이 기본적으로 각 미팅마다 동적 미팅 URL을 생성하므로 모든 반복 예약은 서로 다른 미팅 URL을 갖게 됩니다.

### 반복 이벤트는 연동된 캘린더에 어떻게 동기화되나요?

동기화가 활성화되어 있으면 - 이벤트는 제3자 캘린더와 개별 이벤트로 동기화됩니다.

### 반복 예약에서 결제는 어떻게 처리하나요?

현재 반복 예약이 활성화된 캘린더에서는 결제를 사용할 수 없습니다.
