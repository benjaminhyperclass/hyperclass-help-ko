## 하이퍼클래스의 그룹 캘린더 ID란?

그룹 캘린더 ID는 하이퍼클래스에서 생성한 각 그룹 캘린더에 부여되는 고유 식별자예요. 연동, 워크플로우, API 호출에서 해당 캘린더를 참조할 때 사용됩니다. 여러 팀원이나 서비스를 운영할 때 이벤트, 자동화, 예약이 올바른 그룹 캘린더에 연결되도록 보장해주는 역할을 해요.

## 캘린더 ID가 필요한 경우

다음과 같은 상황에서 캘린더 ID가 필요할 수 있어요:

- 캘린더 예약을 기반으로 워크플로우를 실행할 때
- 웹훅이나 API 호출을 통해 데이터를 전송할 때
- 외부 도구(예: Zapier, Pabbly, Make)와 연동할 때
- 고급 문제 해결 및 지원이 필요한 경우

## 특정 그룹 캘린더 ID 찾는 단계

### 1단계: 캘린더 그룹으로 이동

- 하이퍼클래스 하위 계정에 접속한 후 왼쪽 메뉴에서 **Settings(설정)**을 클릭하세요.
![설정 메뉴](https://jumpshare.com/v/5l54qyjUarKvLAtkbEHr+/Screen+Shot+2025-04-14+at+10.15.49+PM.png)

- **Calendars(캘린더)**를 클릭하고 캘린더 설정 하위에서 **Groups(그룹)**을 선택하세요.
![캘린더 그룹 메뉴](https://jumpshare.com/v/nk1KrZRbZ27eaeLblxAg+/Screen+Shot+2025-04-14+at+10.18.06+PM.png)

### 2단계: 그룹 캘린더 ID 찾기 및 복사

- 그룹 캘린더 목록에서 원하는 그룹의 **Share Group(그룹 공유)** 아이콘을 클릭하세요.
![그룹 공유 아이콘](https://jumpshare.com/v/SVF0JPHv5INPlDAdWkAQ+/Screen+Shot+2025-04-14+at+10.04.05+PM.png)

- **Scheduling Link(예약 링크)**를 클릭한 다음 영구 링크의 마지막 부분(slug)을 선택해서 복사하세요.
![예약 링크에서 ID 복사](https://jumpshare.com/v/hqLSq896QOKSjIiiNkJL+/Screen+Shot+2025-04-14+at+9.55.32+PM.png)

- 또는 **Embed Code(임베드 코드)**를 클릭한 다음 밑줄(_) 앞 부분의 ID를 선택해서 복사할 수도 있어요.
![임베드 코드에서 ID 복사](https://jumpshare.com/v/oZ1axMWQ5U30fH2C8QAw+/Screen+Shot+2025-04-14+at+9.59.43+PM.png)

## 자주 묻는 질문

**Q: 같은 캘린더 ID를 여러 워크플로우에서 사용할 수 있나요?**
네! 워크플로우가 동일한 그룹 캘린더에 연결되어 있다면, 캘린더 ID는 서로 다른 워크플로우와 연동에서 재사용할 수 있어요.

**Q: 하위 계정의 모든 사용자에게 캘린더 ID가 동일한가요?**
네. 캘린더 ID는 캘린더 자체에 연결되어 있어서, 권한이 있는 사용자라면 누가 접근하든 동일한 ID를 갖게 돼요.

**Q: 캘린더 이름을 바꾸면 캘린더 ID도 바뀌나요?**
아니요, 캘린더 이름을 바꿔도 캘린더 ID에는 영향을 주지 않아요. ID는 캘린더의 표시 이름과 관계없이 일정하게 유지됩니다.

**Q: 캘린더 ID 형식이 변경되면 어떻게 되나요?**
하이퍼클래스에서 캘린더 구조나 URL 형식에 변경사항이 있으면 사용자에게 알려드려요. 이 문서를 북마크해두시면 업데이트를 확인할 수 있어요.

**Q: 캘린더를 삭제하면 ID가 여전히 작동하나요?**
아니요, 캘린더가 삭제되면 해당 ID는 무효가 됩니다. 워크플로우나 연동에서 해당 ID를 참조하는 부분은 작동을 멈추게 돼요.

**Q: API에서 예약을 필터링할 때 캘린더 ID를 사용할 수 있나요?**
네, API에서 데이터를 가져올 때 캘린더 ID를 사용해서 특정 캘린더의 예약이나 약속 데이터만 필터링할 수 있어요.
