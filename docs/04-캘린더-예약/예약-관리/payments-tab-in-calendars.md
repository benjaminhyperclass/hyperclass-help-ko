# 캘린더의 결제 탭

Hyperclass 예약 모달의 결제(Payments) 탭은 모든 예약에 대한 결제 추적과 수금을 중앙화합니다. 잔액, 참석자별 상태, 거래 내역을 확인하고 미수금을 수금할 수 있어요. 이 가이드에서는 작동 방식, 활성화 방법, 일반적인 시나리오 처리 방법을 설명해드릴게요.

## 캘린더의 결제 탭이란?

예약 모달의 결제 탭을 통해 예약과 관련된 모든 결제 세부사항을 한 곳에서 관리할 수 있어요. 결제된 금액, 잔여 잔액, 참석자별 결제 내역, 거래 내역을 추적하고 필요시 미결제 금액을 수금할 수 있습니다.

### 결제 탭의 주요 이점

이점을 이해하면 팀에서 언제 이 기능을 도입할지 결정하고 더 빠른 수금과 청구 문의 감소를 위해 직원을 교육하는 데 도움이 됩니다.

- **단일 정보원:** 모든 예약별 금액, 참석자, 거래 내역을 한 곳에서 관리
- **빠른 수금:** 모달에서 바로 **결제 수금**을 실행하여 카드 결제 처리나 오프라인 결제 기록
- **참석자 명확성:** 누가 결제했고, 부분 결제했고, 아직 미납인지 확인 가능. **게스트 추가** 예약에 이상적
- **할인 가시성:** 쿠폰 사용과 조정된 총액이 요약과 거래 내역에 직접 반영
- **깔끔한 회계 관리:** 캘린더 결제가 활성화되면 관련 **주문**이 자동으로 생성되어 후속 보고가 더 쉬워짐

## 주요 기능

#### 1. 결제 요약

- **결제 완료 금액:** 예약에 대해 이미 수금된 총 금액
- **잔여 잔액:** 아직 수금해야 하는 미결제 금액
- **총 금액:** 할인/쿠폰 적용 후 예약의 전체 비용

#### 2. 참석자별 결제 세부사항

- 개별 참석자 단위(예약자 + 게스트)로 결제 확인
- 각 참석자의 상태가 명확히 표시됨:
  - **결제 완료:** 전액 결제 수신
  - **부분 결제:** 일부 금액이 아직 미결제
  - **잔여:** 수금해야 할 미결제 잔액

#### 3. 거래 내역

- 예약과 관련된 모든 거래 목록 확인

#### 4. 잔여 결제 수금

- 미결제 잔액이 있으면 **결제 수금** 버튼이 표시됨
- 옵션은 다음과 같음:
  - **카드 결제:** 신용/직불 카드 결제를 직접 처리
  - **수동 기록:** 시스템 외부에서 수금한 결제(현금, 수표 등)를 기록

## 결제 작동 원리

- 캘린더에 결제가 활성화되면 관련 상품이 자동으로 생성됨(상품 탭에서 Calendar 태그로 표시)
- 이러한 캘린더 연결 상품은 상품 탭에서 직접 삭제할 수 없음
- 예약이 생성되면 주문이 자동으로 생성됨(주문 탭에서 확인 가능). 별도로 주문을 관리할 필요 없이 예약 모달에서 모든 것을 처리할 수 있음

## 일반적인 결제 시나리오 (빠른 참조)

일반적인 흐름에 따라 총액, 상태, 거래 내역이 어떻게 보일지 예상하는 데 이 미니 가이드를 활용하세요.

| 시나리오 | 설정/작업 | 결제 탭에서 보이는 내용 |
|---------|----------|---------------------|
| **예약 시 전액 결제** | 캘린더에서 **결제 수락** 활성화 후 일반적으로 예약 | **결제 완료 금액**이 **총 금액**과 동일; 참석자 상태 = **결제 완료**; 거래 내역에 청구 표시 |
| **보증금(부분 결제)** | **부분 결제/보증금** 활성화 후 예약 시 보증금 수금 | **잔여 잔액**에 미결제 금액 표시; 나중에 **결제 수금**으로 수금 후 상태가 **부분 결제**에서 **결제 완료**로 업데이트 |
| **쿠폰 적용** | 캘린더에서 **쿠폰 코드** 활성화 후 고객이 위젯에서 코드 입력 | 요약에 **할인(쿠폰)** 라인 표시; **총 금액**에 할인 반영; 거래 내역에 할인된 총액에 대한 청구 포함 |
| **예약 후 게스트 추가** | 예약 편집 → 게스트 추가 → 참석자별 수금 | 초기 예약에서 하나의 주문 존재; **게스트 결제 수금**이 활성화된 경우 저장할 때마다 추가 주문이 생성될 수 있음 |
| **반복 예약 - 첫 번째만** | 위젯에서 첫 번째 발생에 대해서만 수금 | 초기 예약에서 결제 표시; 이후 발생은 수동 수금이 필요하며 처음에는 미결제 잔액 표시 |
| **반복 예약 - 모든 발생** | 위젯에서 모든 발생에 대해 수금 | 시리즈 총액이 선불로 수금됨; 예약 기록에서 전체 금액에 대한 결제 완료 총액과 거래 내역 표시 |

## 결제 탭 사용 방법

올바른 설정을 통해 탭이 표시되고 총액, 할인, 참석자 상태가 예상대로 작동하도록 보장합니다.

**캘린더에서 결제 수락 활성화**

- `Calendars(캘린더) → Settings(설정) → 원하는 캘린더 편집`으로 이동

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062985339/original/XQziwznjElOpgOikWRHAxoZQm1sFDife_g.png?1768834422)

- **Accept Payments(결제 수락)**을 토글하세요. Hyperclass에서 **Calendar** 태그가 붙은 **상품**을 자동 생성합니다.
- (선택사항) **Enable Coupon Code(쿠폰 코드 활성화)**를 토글하여 위젯에 쿠폰 필드를 추가하세요.
- (선택사항) 예약 시 보증금을 위한 **Partial Payment/Deposit(부분 결제/보증금)** 규칙을 설정하세요.
- (선택사항) 반복 예약의 경우 예약 위젯에서 **First Appointment Only(첫 번째 예약만)** 또는 **All Appointments(모든 예약)**에 대해 청구할지 선택하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062985462/original/_e7fqvSKj9iQK0l2C_PVZdHeTaTC6KZafA.png?1768834503)

**결제 탭 찾기**

- `Calendars(캘린더) → Appointments(예약)`으로 이동
- 임의의 예약을 열어주세요
- **Payments(결제)**를 클릭하세요

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062985754/original/koIViYbj2J2pg_upxtb1-UmYwl3lE16GeA.png?1768834663)

**예약에서 결제 수금 또는 기록**

- 예약을 열고 **Payments(결제)**로 이동하세요
- **Collect Payment(결제 수금)** → **Charge a Card(카드 결제)** 또는 **Record Manually(수동 기록)** 선택
- 필수 필드를 작성하고 저장하세요
- **결제 완료 금액**, **잔여 잔액**, 참석자 상태가 그에 따라 업데이트되는지 확인하세요

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062986121/original/ToVJc1vmsLBii11oz1hHlt1Zla3cymDu5Q.png?1768834816)

## 중요한 주의사항 및 제한사항

이러한 제약사항은 혼란이나 중복 주문을 야기하는 예외 상황을 피하는 데 도움이 됩니다.

- 기존 **Service Menu (v1)**을 통한 예약은 지원되지 않음
- 앱 내 모달에서 직접 생성된 **커스텀 반복 예약**은 지원되지 않음
- 그룹 예약: 초기 예약 시 전체 그룹에 대해 하나의 주문이 생성됨
- **Collect Payment for Guest(게스트 결제 수금)**이 활성화된 상태에서 예약을 편집하고 게스트를 추가하면 **저장할 때마다 새 주문이 생성될 수 있음**. 의도하지 않은 여러 주문을 피하기 위해 편집과 수금을 신중하게 계획하세요

## 자주 묻는 질문

**Q: 예약 후 쿠폰 할인이 어디에 표시되나요?**
예약의 **결제** 탭에서 **결제 요약**에 **할인(쿠폰)**과 할인 후 **총 금액**이 반영됩니다. **거래 내역**에서는 할인된 총액에 대한 청구가 기록됩니다.

**Q: 결제 탭에서 청구를 환불할 수 있나요?**
환불은 `Payments(결제) → Transactions(거래)`에서 관리합니다(또는 PayPal 같은 게이트웨이를 통해). 환불이 완료되면 예약의 **거래 내역**에 환불 항목이 반영됩니다.

**Q: PayPal이 결제 탭과 함께 작동하나요?**
네, 그렇습니다. 예약 위젯을 통해 수금된 PayPal 거래는 **거래 내역**에 나타나며 예약의 주문과 연결됩니다.

**Q: 반복 시리즈 청구는 어떻게 표시되나요?**
**첫 번째 예약만**에 대해 수금하는 경우, 나중 인스턴스는 수금될 때까지 미결제 잔액을 보여줍니다. **모든 예약**에 대해 수금하는 경우, 총액이 초기 예약 기록에서 선불로 결제된 것으로 표시됩니다.
