# 동기화 이해하기와 중요성

하이퍼클래스 캘린더와 Calendly를 동기화하면 선택한 Calendly 캘린더를 통해 예약된 새 이벤트가 자동으로 하이퍼클래스의 예약으로 나타납니다. 이를 통해 실시간 예약을 기반으로 한 원활한 연락처 생성과 자동화 트리거가 가능해지며, 수동 데이터 입력이 불필요합니다. 연동은 단방향 동기화로 작동하여 이벤트가 Calendly에서 하이퍼클래스로만 가져오며, 그 반대는 불가능합니다. 동기화는 캘린더 설정(Calendar Settings) > 연결(Connections)을 통해 연동이 연결된 후에만 시작되며, 설정 중에 선택한 캘린더에만 적용됩니다. 나중에 캘린더 선택을 해제하면 해당 캘린더의 동기화가 중단되고, 다시 선택해도 누락된 이벤트는 가져오지 않으며 앞으로의 새 이벤트만 동기화됩니다. 동기화는 각 Calendly 예약 후 실시간 또는 거의 실시간으로 발생합니다.

원활하고 안정적인 경험을 보장하려면 지원되는 Calendly 요금제를 사용하고, 관련 캘린더만 선택하며, 이벤트 유형이 이름과 이메일 같은 필수 게스트 정보를 수집하도록 설정해야 합니다. 동기화 동작이 캘린더 선택과 어떻게 연결되는지 이해하면 데이터 충돌을 방지하고 워크플로우가 의도한 대로 작동하도록 보장할 수 있습니다.

## Calendly에서 동기화되는 항목

연동은 CRM 워크플로우를 실행하고 예약을 효과적으로 관리하는 데 필요한 핵심 데이터에 중점을 둡니다.

| 동기화됨 | 동기화 안됨 |
|---------|-----------|
| 이벤트 날짜 & 시간 | 취소되거나 일정 변경된 이벤트는 하이퍼클래스에서 업데이트되지 않음 |
| 게스트 이름 & 이메일 | 과거 이벤트 (설정 이전에 생성된) |
| 예약 제목 | 첨부파일이나 메모 |
| 출처 ("Calendly"로 표시) | |
| 이벤트가 속한 캘린더 | |
| 위치나 Q&A 필드의 전화번호 | |

## **Calendly 연동 작동 방식**

Calendly 연동은 Google, Outlook, iCloud와 같은 다른 캘린더 플랫폼과 약간 다르게 작동합니다. Calendly에서 하이퍼클래스로 이벤트를 가져오도록 설계되었으며, 그 반대는 아닙니다. 이 동기화가 어떻게 작동하는지 이해하면 최대한 활용할 수 있습니다.

### Calendly 이벤트 동기화

Calendly 계정이 연결되고 최소한 하나의 캘린더가 선택되면, 하이퍼클래스는 새 이벤트를 자동으로 가져오기 시작합니다. 연동이 설정된 후에 생성된 이벤트만 캘린더에 동기화됩니다.

나중에 연결에서 캘린더를 제거하면 해당 캘린더의 새 이벤트는 더 이상 하이퍼클래스에 동기화되지 않습니다. 그리고 나중에 그 캘린더를 다시 연결하면, Calendly 계정을 연결할 때를 기준으로 지난 180일 내에 생성된 향후 이벤트와 과거 이벤트가 모두 동기화됩니다. 캘린더가 연결 해제된 동안 만들어진 예약은 소급하여 가져오지 않습니다.

### 단방향 동기화

Calendly 연동은 단방향입니다. 하이퍼클래스가 Calendly에서 이벤트를 가져오지만 그 반대는 불가능합니다. 즉:

- 하이퍼클래스에서 생성된 이벤트는 Calendly에 나타나지 않습니다.

- Calendly에서 가져온 이벤트는 하이퍼클래스 내에서 편집할 수 없으며, 모든 것이 정확하게 유지되도록 하려면 Calendly에서 직접 변경해야 합니다.

### 예약에서 연락처 생성

하이퍼클래스에 동기화되는 모든 새 Calendly 이벤트는 (단순한 자리 표시자나 차단된 시간이 아닌) 예약으로 처리됩니다. 이벤트에 게스트가 포함되어 있으면 그들의 이름과 이메일을 사용하여 시스템에 연락처가 자동으로 생성됩니다. 그런 다음 자동화를 트리거하고, 워크플로우에 추가하며, CRM의 다른 연락처처럼 관리할 수 있습니다.

### 예약 소유자는 누구인가요?

소유권은 캘린더 유형에 따라 달라집니다:

- 개인 Calendly 링크의 경우, Calendly 계정을 연결한 사용자가 예약 소유자로 지정됩니다.

- 공유 캘린더(라운드 로빈이나 팀 링크 등)의 경우, 시스템이 연결된 Calendly 사용자 중에서 예약 소유자를 무작위로 지정합니다.

### 연락처 배정

연락처의 배정된 사용자 필드는 가장 최근 예약의 소유자와 일치하도록 자동으로 업데이트됩니다. 이를 통해 올바른 팀원이 연락처와 연결되고 후속 자동화가 올바른 사용자에게 전달됩니다.

## 연락처 생성 및 관리

### Calendly 예약이 동기화되면:

- 게스트의 이름과 이메일을 사용하여 하이퍼클래스에 연락처가 자동으로 생성됩니다.

- 연락처가 이미 존재하면 예약이 해당 레코드에 추가됩니다.

- 이러한 연락처는 예약 상태에 의해 트리거되는 자동화의 대상이 됩니다.

### 예약 소유권 및 사용자 배정

하이퍼클래스의 예약에는 작업, 알림, 자동화를 적절히 라우팅하기 위해 "소유자"가 필요합니다.

**예약 소유자**

- 개인 캘린더: Calendly를 연결한 사용자가 소유자입니다.
- 공유 캘린더 (예: 라운드 로빈): 연결된 팀 중 한 사용자가 무작위로 지정됩니다.

**연락처 배정된 사용자**

연락처의 배정된 사용자는 가장 최근 예약의 소유자와 일치하도록 업데이트됩니다. 이를 통해 연락처가 후속 조치를 해야 하는 사용자에게 적절히 라우팅됩니다.

## 설정 후 캘린더 관리

언제든지 캘린더 선택을 변경할 수 있습니다:

하이퍼클래스에서 Calendly 캘린더 선택을 업데이트하려면 **캘린더 설정(Calendar Settings)**으로 이동한 다음 **연결(Connections)** 탭을 클릭하세요. 연결된 캘린더 목록에서 **Calendly 연동**을 찾으세요. 옆에 있는 **편집 아이콘**을 클릭하여 캘린더 선택 패널을 열어주세요. 여기서 동기화하려는 새 캘린더를 추가하거나 더 이상 필요하지 않은 캘린더를 제거할 수 있습니다. 변경사항을 완료했으면 반드시 **저장**을 클릭하여 업데이트를 적용하세요. 현재 선택된 캘린더의 이벤트만 앞으로 동기화됩니다.

참고: 새로 추가된 캘린더의 이벤트는 선택된 시점부터만 동기화됩니다. 과거 데이터는 가져오지 않습니다.

**동기화된 Calendly 이벤트 삭제**

캘린더 연결을 끊거나 Calendly를 제거할 때 동기화된 이벤트를 삭제할지 선택하라는 메시지가 표시됩니다.

**하나의 캘린더에서 이벤트 삭제**

특정 캘린더에서 동기화된 모든 이벤트를 삭제하도록 선택할 수 있습니다.

삭제된 이벤트는 나중에 복원할 수 없습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045368934/original/n9wsbf8nBBDNQQVrVGudhhC-fl4wqAOO6A.png?1745157796)

**공유 캘린더**

여러 사용자가 같은 공유 캘린더에 Calendly를 연결한 경우, 모든 사용자가 이벤트 삭제를 선택해야 제거됩니다.

한 사용자가 캘린더를 유지하면 공유 이벤트가 계속 표시됩니다.

**Calendly 완전 연결 해제**

모든 캘린더에서 동기화된 모든 이벤트를 삭제하는 옵션이 제공됩니다. 삭제는 영구적입니다. Calendly가 다시 연결되면, Calendly 계정을 연결할 때를 기준으로 지난 180일 내에 생성된 향후 이벤트와 과거 이벤트를 자동으로 가져옵니다.

## Calendly 연동의 알려진 제한 사항

Calendly 연동 사용 시 알아야 할 주요 제한 사항은 다음과 같습니다:

- 하위 계정당 사용자당 하나의 Calendly 계정만 가능

- 소속된 여러 하위 계정에서 같은 Calendly 계정을 연결할 수 있음

- Calendly의 Standard, Teams, Enterprise 요금제만 지원

- 무료 또는 Essentials Calendly 요금제는 지원되지 않음

- 하이퍼클래스에서 이벤트를 편집할 수 없음

- 게스트의 이름과 이메일만 동기화됨 (Q&A나 커스텀 필드는 제외)

## 전화번호 수집 규칙 (Calendly → 연락처)

하이퍼클래스는 수동 정리를 줄이기 위해 명확한 소스 규칙을 사용하여 Calendly에서 전화번호를 수집합니다.

- 게스트 전화번호가 미팅 위치에 제공되고 Calendly 폼에 전화 필드가 없는 경우, 하이퍼클래스는 위치 전화번호를 연락처에 자동으로 동기화합니다.

- Calendly 폼에 전화 필드가 포함된 경우, 하이퍼클래스는 소스를 선택할 수 있는 매핑 화면을 보여줍니다 (폼 전화, 위치 전화(있는 경우), 또는 동기화 안 함).

- 전화 소스가 없는 경우 (위치 전화도 폼 전화도 없음), 전화 동기화를 건너뜁니다.

**공유 캘린더: 전화 동기화 옵션 업데이트는 해당 캘린더에 연결된 모든 사용자에게 영향을 미칩니다.**

## 자주 묻는 질문

**Q: 이전 Calendly 이벤트가 하이퍼클래스에 나타나지 않는 이유는 무엇인가요?**

연동이 설정되고 캘린더가 선택된 후에 생성된 이벤트만 가져옵니다.

**Q: 하이퍼클래스에서 Calendly 이벤트를 편집하거나 일정을 변경할 수 있나요?**

아니요. 이벤트는 Calendly에서 직접 편집해야 합니다.

**Q: 캘린더를 다시 연결하거나 Calendly를 다시 추가하면 어떻게 되나요?**

앞으로의 새 이벤트만 동기화됩니다. 이전에 누락되거나 연결이 끊어진 이벤트는 나타나지 않습니다.

**Q: 공유 캘린더에서 동기화된 예약의 소유자는 누구인가요?**

공유 캘린더의 경우 Calendly에 연결된 사용자 중 한 명이 무작위로 소유자로 지정됩니다.

**Q: 내 Calendly 계정을 여러 하이퍼클래스 사용자나 하위 계정에 연결할 수 있나요?**

예. 하나의 Calendly 계정을 다른 하위 계정의 여러 사용자에 연결할 수 있습니다.
