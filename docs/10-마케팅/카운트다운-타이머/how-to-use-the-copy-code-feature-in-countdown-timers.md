# 카운트다운 타이머에서 "코드 복사" 기능 사용하는 방법

카운트다운 타이머는 웹사이트나 이메일 캠페인에 긴박감을 더하는 강력한 도구입니다. 특별 할인, 이벤트, 신제품 출시까지 남은 시간을 보여줌으로써 고객의 전환율을 높일 수 있습니다. 여기서 유용한 기능 중 하나가 "코드 복사(Copy Code)" 기능인데, 이를 통해 카운트다운 타이머를 콘텐츠에 빠르게 통합할 수 있습니다.

이 가이드에서는 카운트다운 타이머에서 "코드 복사" 기능을 사용하는 방법을 단계별로 설명합니다. 타이머 생성부터 웹사이트나 이메일용 코드 복사, 정렬 및 다이나믹 타이머 기능 커스터마이징까지 모든 과정을 다룹니다.

## 카운트다운 타이머 코드 복사란?

**"코드 복사(Copy Code)"** 기능은 카운트다운 타이머를 **다른** 플랫폼에 **임베드**하고 싶을 때 사용합니다. 예를 들어 웹사이트, 이메일, 기타 빌더에 삽입할 때 사용하는 기능입니다. 다음과 같은 경우에 활용하세요:

- **GHL이 아닌 웹사이트에서:**
웹페이지에 카운트다운 타이머를 표시하고 싶을 때, "코드 복사" 옵션을 통해 HTML 코드 스니펫을 받아서 웹사이트 HTML에 직접 붙여넣을 수 있습니다.

- 시간 한정 할인, 이벤트 카운트다운, 신제품 출시 등을 보여주는 데 유용합니다.

- **GHL이 아닌 이메일에서:**
이메일 캠페인을 보낼 때 "코드 복사" 기능을 사용해서 이메일의 HTML에 타이머 코드를 삽입할 수 있습니다.

- 이메일 콘텐츠 안에 긴박감을 만들어서 수신자가 이메일 안에서 바로 카운트다운을 볼 수 있게 됩니다.

## 카운트다운 타이머 유형

카운트다운 타이머는 세 가지 유형이 있습니다:

- **고정(Fixed)**: 항상 같은 시간에 끝나는 타이머입니다. endDate(종료 날짜)가 필요합니다.

- **반복(Recurring)**: 카운트다운이 끝나면 새로운 시간으로 다시 시작해서 반복되는 타이머입니다. startDate(시작 날짜), 지속 시간, 반복 횟수가 필요합니다.

- **다이나믹(Dynamic)**: 보는 사람마다 다른 마감 시간을 가질 수 있는 타이머입니다. uniqueID(고유 ID)와 지속 시간이 필요합니다 (startDate는 개별 사용자가 활성화하는 시점에 따라 결정됩니다).

## 코드를 수정해야 하나요?

| 타이머 사용 위치 | startDate=… 수정 필요? | uniqueID=… 수정 필요? | 이유 |
|---|---|---|---|
| 웹사이트나 랜딩 페이지 (모든 빌더) | 아니요. 그대로 붙여넣으세요. | 선택사항. 페이지별 분석이 필요하지 않다면 기본값 사용. | 모든 방문자가 같은 "벽시계" 마감시간을 봅니다. |
| 정적 이메일 (모든 사람이 같은 마감시간) | 예. 플레이스홀더를 이메일 발송 시점으로 교체 (예: 2025-01-01T15:30:00Z). | 삭제하거나 무시. 모든 사람이 같은 마감시간을 공유하므로 고유성은 불필요. | 모든 수신자가 같은 마감시간을 봅니다. |
| GHL 내 다이나믹 이메일 (각 연락처별 상대적 마감시간) | 예. 머지 필드 {{right_now.date_time_iso}}로 교체. | 기본 {{contact.id}} 유지 또는 다른 연락처 필드로 교체. | GHL이 발송 시점에 머지 태그를 채워서 각 연락처마다 개인별 시계 제공. |
| 다른 플랫폼의 다이나믹 이메일 (예: 메일침프, 허브스팟) | 예. 해당 플랫폼의 "현재 날짜시간" 머지 태그 사용 (예: `*NOW*`). | 해당 플랫폼의 고유 ID 머지 태그 사용. | 각 플랫폼별 머지 시스템 활용. |

**핵심 개념:**
- **정적** 타이머 = 모든 사람이 공유하는 하나의 마감시간
- **다이나믹** 타이머 = 각 수신자별로 개별 계산되는 마감시간. 따라서 개인별 **startDate**와 **uniqueID**가 모두 필요

**startDate**는 스톱워치 트리거입니다. 빌더가 "현재 시간"을 자동으로 계산할 수 없다면, 타이머에게 "현재 시간"이 언제인지 알려줘야 합니다.

**uniqueID**는 스톱워치 일련번호입니다. 캐싱을 방지해서 모든 수신자가 같은 이미지를 보는 것을 막아줍니다.

## 카운트다운 타이머 코드 복사 사용 방법

**1. 카운트다운 타이머 생성**

**종료 날짜와 시간**, **외관**, **형식**(일, 시간, 분, 초)을 선택해서 카운트다운을 설정합니다. 완료되면 다음 단계로 이동하세요.

**2. 웹사이트나 이메일용 코드 복사**

**"코드 복사(Copy Code)"** 버튼을 클릭해서 코드를 가져옵니다.

![코드 복사 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040310426/original/jQZNKBE9RpsBO8KGrAzA4SllK8zaigfI_g.png?1737618751)

**3. 사용 위치와 타이머 정렬 선택**
코드를 사용할 위치에 따라 이메일 또는 웹사이트를 선택하고, 타이머의 정렬을 선택합니다. 페이지나 이메일 디자인에 맞는 정렬을 선택하세요.

![사용 위치 및 정렬 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040310385/original/9B7_0J5oMU3GIokzZzMvdWQv8qmRVGlq6g.png?1737618709)

**4. 다이나믹 타이머용 고유 ID 입력 (선택사항)**
**다이나믹 타이머**의 경우, 중괄호 안의 contact.id를 관련 고유 ID로 교체해서 각 사용자별로 개인화된 카운트다운을 제공하세요 (예: contactID, userID 등). 이렇게 하면 각자에게 맞춤형 타이머가 표시됩니다.

**예시**: 강조된 텍스트를 변경해야 합니다. 고유 ID가 contact.no라면 다음과 같이 사용하세요:

...............uniqueID={{contact.no}}&.......

![고유 ID 설정 예시](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040310604/original/jdn8K4NCeUFaHi2oCYxo5mKTIMp8rEHN3A.png?1737618875)

**5. 올바른 발송 시간으로 교체**
타이머를 빌더의 코드 스니펫 블록에서 사용할 때는 시작 날짜 필드가 필요합니다.

GHL 빌더 외부의 이메일에서 코드 복사를 사용할 때는 트리거가 오픈이든 클릭이든 관계없이 이메일 발송 날짜를 입력해야 합니다. 예를 들어 2025년 1월 1일 오후 3시 30분에 이메일을 발송했다면, startDate=YYYY-MM-DDTHH:mm:ss.sssZ를 startDate=2025-01-01T15:30:00:00으로 교체합니다.

GHL 이메일 빌더 내에서 다이나믹 타이머 코드 복사를 사용할 때는 startDate=YYYY-MM-DDTHH:mm:ss.sssZ를 startDate={{right_now.date_time_iso}}로 교체할 수 있습니다.

## 문제 해결 체크리스트

**타이머가 시작되지 않는 경우** → **startDate**에 실제 ISO 날짜나 유효한 머지 태그 {{merge.tag}}가 입력되었는지 확인하세요.

**모든 사람이 같은 시간을 보는 경우** → **uniqueID**가 각 수신자별로 다른지 확인하세요 (하드코딩이 아닌 머지 태그 사용).

**타임존이 잘못된 경우** → 입력한 날짜가 Z(UTC)로 끝나거나 타임존 오프셋 (예: -05:00)을 포함하고 있는지 확인하세요.

## 자주 묻는 질문

**GHL 웹사이트에 임베드할 때 코드를 바꿔야 하나요?**

아니요. 기본 스니펫이 이미 작동합니다. 모든 방문자가 마감시간을 공유하기 때문입니다.

**고유 ID가 정말 "선택사항"인가요?**

정적 타이머에만 선택사항입니다. 다이나믹 타이머에는 필요합니다. 그렇지 않으면 모든 수신자가 같은 캐시된 이미지를 보게 됩니다.

**다이나믹 이메일 타이머를 발송 전에 테스트할 수 있나요?**

네. 자신에게 테스트를 보내보세요. 머지 태그가 본인의 연락처 데이터로 채워져서 실제 수신자가 볼 내용을 정확히 확인할 수 있습니다.

**GHL의 이메일/사이트 빌더 안에서도 코드 복사가 작동하나요?**

GHL 빌더 안에서는 카운트다운 타이머를 직접 임베드할 수 있어서 코드 복사가 필요하지 않지만, 코드 스니펫 요소(Element)에서는 사용할 수 있습니다.
