# 스냅샷 워크플로우 이메일 액션 – 향상된 플로우

템플릿 관리가 훨씬 간단해졌어요! 이제 대상 로케이션에는 원본 템플릿과 워크플로우만 표시되고, 이메일 액션은 백그라운드에서 자동으로 처리됩니다.

## 개요

이제 대상 로케이션의 템플릿 목록에는 원본 템플릿과 워크플로우만 표시되어, 이메일 액션 템플릿으로 인한 혼란이 사라졌습니다. 이메일 액션 템플릿은 여전히 백그라운드에서 가져와지고 매핑되지만, 사용자에게는 숨겨집니다. 이 향상된 기능으로 템플릿 관리 프로세스가 간소화되고 워크플로우와 템플릿의 동기화가 개선되었습니다.

- **혼란 감소**: 핵심 템플릿과 워크플로우만 보여서 탐색이 쉬워집니다.
- **안정적인 동기화**: 이메일 액션 템플릿이 자동으로 매핑되고 워크플로우와 동기화됩니다.

## ✅ 새로운 기능

- **깔끔한 대상 목록 보기**: 대상 로케이션의 템플릿 목록에는 원본 템플릿만 표시되고, 이메일 액션 템플릿은 숨겨지지만 백그라운드에서 여전히 가져와집니다.
- **간소화된 스냅샷 목록**: 스냅샷 모달에는 원본 템플릿과 워크플로우만 표시됩니다. 이메일 액션 템플릿은 숨겨지지만 백그라운드에서 가져와지고 매핑됩니다.
- **향상된 동기화 로직**: 워크플로우와 템플릿을 함께 푸시하거나, 워크플로우보다 템플릿을 먼저 푸시할 때 동기화가 예상대로 작동합니다.

## ⚒️ 사용 방법

1. **스냅샷 생성**: 소스 로케이션에서 스냅샷할 워크플로우와 템플릿을 선택합니다.
2. **대상 로케이션에서 확인**: 대상 로케이션의 템플릿 목록에는 원본 템플릿과 워크플로우만 표시됩니다.
3. **자동 백그라운드 관리**: 이메일 액션 템플릿이 자동으로 가져와지고 선택한 워크플로우에 매핑되며, 추가 사용자 작업이 필요하지 않습니다.

## 참고 사항

예상된 동작을 보장하려면, 기존 워크플로우와 이메일 액션을 삭제하고 스냅샷 새로고침 후 다시 푸시해야 최신 변경사항이 적용됩니다.

## ⏳ 제한 사항

- **이메일 액션 이용 불가**: 연결된 템플릿 없이 워크플로우만 푸시하면 이메일 액션을 사용할 수 없습니다. 워크플로우 이후에 템플릿을 푸시하면 매핑과 동기화가 실패합니다. (스냅샷의 알려진 제한사항)
- **동기화 실패**: 해당 워크플로우 이후에 템플릿을 푸시하면 매핑과 동기화가 실패합니다.
- **일괄 삭제 미지원**: 이메일 액션은 일괄 삭제를 지원하지 않습니다. 대량 삭제에는 개발자 도움이 필요합니다.

## ❓ 자주 묻는 질문

**Q: 이메일 액션 템플릿이 여전히 가져와지나요?**
네, 이메일 액션 템플릿은 백그라운드에서 여전히 가져와지고 관련 워크플로우에 매핑되지만, 템플릿 목록에는 표시되지 않습니다.

**Q: 템플릿 없이 워크플로우만 푸시하면 어떻게 되나요?**
연결된 템플릿 없이 워크플로우만 푸시하면 이메일 액션을 사용할 수 없고 동기화가 실패합니다.

**Q: 이메일 액션 템플릿을 일괄 삭제할 수 있나요?**
현재 이메일 액션 템플릿의 일괄 삭제는 지원되지 않습니다. 대량 삭제가 필요한 경우 개발 지원이 필요합니다.

**Q: 최신 변경사항을 적용하려면 어떻게 해야 하나요?**
최신 변경사항을 적용하려면 기존 워크플로우와 이메일 액션을 삭제하고, 업데이트된 스냅샷을 사용하여 다시 푸시하세요.

**Q: 스냅샷을 통해 이메일 템플릿을 푸시할 때 조건부 발송 규칙이 보존되나요?**
네, 이메일 템플릿의 조건부 발송 규칙은 스냅샷을 통해 한 로케이션에서 다른 로케이션으로 템플릿을 푸시할 때 유지되므로, 대상 로케이션에서도 표시 로직이 그대로 유지됩니다.

**Q: 워크플로우 이후에 템플릿을 푸시하면 동기화 프로세스에 영향이 있나요?**
네, 워크플로우 이후에 템플릿을 푸시하면 동기화가 실패합니다. 적절한 동기화를 위해 워크플로우보다 먼저 템플릿을 푸시하거나 둘을 함께 푸시하세요.
