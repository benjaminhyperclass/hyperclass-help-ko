# 연락처에서 일괄 작업으로 이메일 작성하는 새 UI

Hyperclass의 연락처 화면에 있는 새롭게 디자인된 대량 이메일 빌더는 대량 이메일을 만들고 발송하는 현대적이고 통합된 경험을 제공합니다. 내장된 발송 전 연락처 필터링, 간소화된 디자인 워크플로우, 강력한 추적 도구로 기술적 복잡함 없이 개인화된 메시지를 대규모로 자신 있게 발송할 수 있습니다.

## 대량 이메일 빌더란 무엇인가요?

대량 이메일 빌더는 **연락처(Contacts) → 스마트 리스트(Smart Lists)** 섹션에서 **이메일 발송(Send Email)**을 선택할 때 나타나는 향상된 이메일 제작 인터페이스입니다. Hyperclass의 세 가지 이메일 편집 옵션인 **빠른 작성(Quick Compose)**, **스마트 빌더(Smart Builder)**, **템플릿 선택(Template Selection)**을 하나의 매끄러운 흐름으로 통합합니다.

이 경험을 통해 이메일 캠페인을 디자인, 개인화, 전달하는 방법을 간소화하면서 동시에 유효하지 않거나 구독을 취소한 연락처를 자동으로 필터링하여 컴플라이언스와 전달성을 보장합니다.

## 동적 커스텀 객체 라벨 (새 기능)

대량 이메일 빌더는 이제 하드코딩된 "Contact" 또는 "Contacts" 텍스트 대신 **동적 커스텀 객체 라벨**을 사용합니다. 이를 통해 대량 이메일 모달이 계정에 구성된 객체 라벨(예: **리드(Leads)**, **클라이언트(Clients)** 또는 다른 커스텀 객체)을 자동으로 반영합니다.

### 이것이 중요한 이유

- CRM 구성과 일관된 대량 이메일 경험을 유지합니다
- 기본 연락처 객체를 이름을 바꾸거나 교체한 계정을 지원합니다
- 커스텀 객체로 작업할 때 명확성을 향상시킵니다
- 설정이나 작업이 필요 없으며 라벨이 자동으로 업데이트됩니다

### 어디서 볼 수 있나요

**연락처(Contacts) → 스마트 리스트(Smart Lists) → 일괄 작업(Bulk Actions) → 이메일 발송(Send Email)**을 통해 이메일을 발송할 때 **대량 이메일 모달 헤더와 도움말 텍스트**에서 볼 수 있습니다.

이 업데이트는 표시된 라벨에만 영향을 줍니다. 대량 이메일 기능, 수신자 선택, 필터링, 전달 동작은 변경되지 않습니다.

## 새로운 대량 이메일 빌더의 주요 장점

새로운 대량 이메일 빌더는 사용성, 정확성, 성능 모니터링을 개선하도록 설계된 여러 가지 향상된 기능을 제공합니다.

- **통합 워크플로우:** 모든 이메일 제작 옵션(빠른 작성, 스마트 빌더, 템플릿)을 한 곳에서 접근할 수 있습니다
- **발송 전 필터링:** 발송 전에 유효하지 않거나 구독 취소한 연락처를 자동으로 제외합니다
- **일관된 경험:** 모든 빌더가 유사한 레이아웃을 공유하여 팀 온보딩이 더 쉬워집니다
- **고급 추적:** 편집기에서 직접 링크 추적, UTM 태깅, 자동화 태그를 활성화할 수 있습니다
- **신뢰할 수 있는 전달 지표:** 모든 이메일 소스(연락처, 캠페인, 워크플로우)에서 표준화된 보고서와 통계에 접근할 수 있습니다
- **통합 편집기:** 캠페인과 워크플로우와 일치하여 일관된 이메일 작성 경험을 제공합니다

## 발송 전 연락처 필터링

발송 전 연락처 필터링은 유효한 수신자만 메시지를 받도록 보장합니다.

Hyperclass는 선택한 연락처를 자동으로 검토하고 다음을 보여주는 요약을 표시합니다:

- 선택된 총 연락처 수
- 필터링된 연락처(유효하지 않거나 구독 취소한 연락처)
- 최종 수신자 목록

이 기능은 발신자 평판을 보호하고 이메일 전달성을 향상시키는 데 도움이 됩니다.

## 이메일 제작 모드

대량 이메일 빌더는 워크플로우와 디자인 선호도에 맞는 세 가지 유연한 제작 모드를 제공합니다.

### 빠른 작성(Quick Compose)

빠른 공지나 거래 업데이트에 이상적입니다. 빠른 커뮤니케이션을 위한 간단한 텍스트 중심 편집기입니다.

### 스마트 빌더(Smart Builder)

편집 경험을 선택할 수 있는 강력한 빌더입니다:

- **디자인 편집기(Design Editor)** (시각적, 드래그 앤 드롭)
- **코드 편집기(Code Editor)**
- **일반 텍스트 편집기(Plain Text Editor)**

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064218844/original/ngcorEPkoMlrOw8YkfHwoSmJ9e36UsVfeQ.png?1770246905)

### 기존 템플릿 선택(Select Existing Template)

일관성과 속도를 위해 저장된 템플릿에서 선택하거나 템플릿 라이브러리를 탐색하여 미리 디자인된 레이아웃을 사용하세요.

## 추가 이메일 설정

고급 설정은 마케터가 성능을 측정하고, 작업을 자동화하며, 개인화를 향상시키는 데 도움이 됩니다. **추가 설정(Additional Settings)** 서랍을 확장하여 다음에 접근하세요:

- **클릭 추적(Track Clicks):** 어떤 링크가 클릭되었고 누가 클릭했는지 모니터링합니다
- **UTM 추적(UTM Tracking):** 분석을 위해 URL에 캠페인 매개변수를 자동으로 추가합니다
- **태그 추가(Add Tags):** 참여 행동에 따라 연락처 태그를 자동으로 적용합니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064218877/original/gsQFZ5ksg69zXIkJJCWezrILiqouj-sR-A.png?1770247017)

## 테스트 이메일 발송 및 캠페인 검토

발송하기 전에 레이아웃과 내용을 확인하기 위해 테스트 버전을 발송할 수 있습니다.

- **테스트 이메일 ID** 입력
- **테스트 메일 발송(Send Test Mail)** 클릭
- **캠페인 검토(Review Campaign)**를 클릭하여 발송 전 세부 사항을 미리보고 확인

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064218879/original/vwMhPG4ikAo8MsK4H83s0juPlIwZDO-xzg.png?1770247040)

## 발송 옵션

대량 이메일 빌더는 전달 요구사항에 맞는 여러 발송 모드를 지원합니다:

- **모두 한번에 발송(Send All at Once):** 모든 수신자에게 즉시 발송합니다
- **예약된 시간에 발송(Send at Scheduled Time):** 특정 발송 날짜와 시간을 예약합니다
- **드립 모드로 발송(Send in Drip Mode):** 받은 편지함 배치를 최적화하기 위해 배치로 점진적으로 이메일을 발송합니다

## 대량 이메일 설정 및 발송 방법

1. **연락처(Contacts) → 스마트 리스트(Smart Lists)**로 이동
2. 이메일을 보낼 연락처나 스마트 리스트 선택
3. 일괄 작업 도구 모음에서 **이메일 발송(Send Email)** 클릭
4. 제작 모드 선택:
   - 빠른 작성(Quick Compose)
   - 스마트 빌더(Smart Builder)
   - 기존 템플릿 선택(Select Existing Template)
5. 이메일 내용 작성
6. 확인 모달에서 수신자 검토
7. 필요시 테스트 이메일 발송
8. **캠페인 검토(Review Campaign)** 클릭
9. 발송 모드를 선택하고 이메일 발송 클릭

## 심층 일괄 작업 성능 보기 (통계 더 보기)

통계 더 보기(Load More Stats)는 빠른 결과 지표를 제공합니다. 더 깊은 성능 보고(열람, 클릭, 반송, 발송 소스 간 비교)를 위해서는 통계 더 보기를 사용하여 전체 이메일 통계 대시보드를 여세요.

**마케팅(Marketing) → 이메일(Emails) → 통계(Statistics) (통합 대시보드)**로 이동하세요.

이 대시보드는 캠페인, 워크플로우, 일괄 작업을 통합합니다.

## 자주 묻는 질문

**Q: 구독 취소했거나 유효하지 않은 연락처도 이메일을 받나요?**
A: 아닙니다. 시스템이 구독 취소, 유효하지 않음, DND 표시된 연락처를 자동으로 제외합니다.

**Q: Hyperclass의 다른 영역에서 기존 템플릿을 사용할 수 있나요?**
A: 네. 내 템플릿(My Templates)이나 템플릿 라이브러리(Template Library)의 템플릿을 사용할 수 있습니다.

**Q: 대량 이메일의 성능을 추적할 수 있나요?**
A: 네. 열람, 클릭, 반송은 일괄 작업(Bulk Actions) → 통계 보기(Show Stats)에서 추적됩니다.

**Q: 참여도에 따라 태깅을 자동화할 수 있나요?**
A: 네. 추가 설정에서 **태그 추가(Add Tags)** 옵션을 사용하세요.

**Q: 예약 발송에서 스마트 리스트가 동적으로 업데이트되나요?**
A: 아닙니다. 연락처는 예약 시점에 캡처됩니다.
