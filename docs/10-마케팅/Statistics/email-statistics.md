## **이메일 통계란?**

Hyperclass의 이메일 통계는 캠페인, 워크플로우, 일괄 작업 전반에 걸쳐 수신자가 이메일과 어떻게 상호작용하는지 추적할 수 있게 해줍니다. 이러한 인사이트를 통해 전달성을 최적화하고, 콘텐츠 전략을 개선하며, 좋은 발신자 평판을 유지할 수 있습니다.

## **이메일 통계의 주요 이점**

이메일 전달성과 성과를 개선하기 위한 실행 가능한 인사이트를 얻으세요.

- 열람, 클릭, 반송, 스팸 지표를 실시간으로 확인
- 캠페인, 워크플로우, 일괄 이메일 수준에서 통계 추적
- 가장 성과가 좋은 제목과 템플릿 식별
- 세부적인 문제 해결을 위한 연락처 수준 인사이트 접근
- 통계를 활용해 목록을 분할하고 자동화 실행
- 구독 취소나 전달 실패 같은 문제 정확히 파악

## **이메일 마케팅 통계**

일회성 및 예약 이메일 성과에 대한 인사이트를 얻으세요. 이메일 마케팅 통계를 보려면 **Marketing(마케팅) > Emails(이메일)**로 이동하세요.

### **참여도 요약**

이 그래프를 사용해 이메일의 전달, 참여도, 전환 측면에서 성과를 빠르게 파악하세요.

전달률, 반송률, 구독 취소, 스팸 신고 같은 핵심 지표를 통해 시간에 따른 이메일 성과 추세를 추적하세요. 드롭다운을 사용해 참여도 지표를 전환하고 선택한 날짜 범위에서 다양한 캠페인 유형의 성과를 시각화할 수 있습니다.

## **워크플로우 이메일 통계**

워크플로우 내에서 각 자동화된 이메일 단계의 성과를 직접 모니터링하세요. 이 보기는 자동화 메시지가 고객에게 얼마나 잘 도달하고 참여를 유도하는지 이해하는 데 도움이 됩니다. 워크플로우 이메일 통계를 보려면:

- **Automation(자동화) > Workflows(워크플로우)**로 이동
- 워크플로우를 클릭하고 Email(이메일) 액션을 선택
- 우측 패널에서 **Statistics(통계)** 클릭

다음 지표를 확인할 수 있습니다:

- 전달됨 (Delivered)
- 열람됨 (Opened)
- 클릭됨 (Clicked)
- 답장됨 (Replied)
- 반송됨 (Bounced)
- 구독 취소됨 (Unsubscribed)
- 신고됨 (Complained)
- 수락됨 (Accepted)
- 거부됨 (Rejected)

워크플로우 이메일 통계는 요약 보기와 일관성을 유지하기 위해 기본적으로 지난 30일로 설정됩니다.

![워크플로우 이메일 통계 패널](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047444702/original/Z2UIfHmUNTtgGi2GxUcuPxgzx8BKRHC-yQ.png?1748533775)

## **일괄 이메일 통계**

Contacts(연락처) > Bulk Actions(일괄 작업) 탭에서 직접 일회성 일괄 이메일 발송의 성과를 확인하세요. 이는 자동화나 캠페인 외부의 대량 발송을 검토하는 데 이상적입니다. 이 보기에서는 개별 수신자별 성과도 볼 수 있어 전달 문제를 해결하거나 후속 작업을 분할하기가 더 쉽습니다. 통계에 접근하려면:

- Contacts(연락처) > Bulk Actions(일괄 작업)으로 이동
- 활동 목록에서 해당 이메일 발송을 찾기

다음 내용의 분석을 확인할 수 있습니다:

- 총 시도 (Total Attempts)
- 전달됨 (Delivered)
- 열람됨 (Opened)
- 클릭됨 (Clicked)
- 답장됨 (Replied)
- 반송됨 (Bounced)
- 구독 취소됨 (Unsubscribed)

## **이메일 지표 정의**

| 지표 | 의미 |
|------|------|
| **Recipients(수신자)** | 대상 연락처 수 |
| **Delivered(전달됨)** | 수신자 서버가 성공적으로 수락한 것 |
| **Opened(열람됨)** | 이메일을 한 번 이상 열어본 것 |
| **Clicked(클릭됨)** | 링크를 클릭한 것 |
| **Soft Bounce(소프트 반송)** | 일시적 문제 (예: 받은편지함 가득참) |
| **Hard Bounce(하드 반송)** | 영구적 실패 (예: 이메일 주소가 존재하지 않음) |
| **Unsubscribed(구독 취소됨)** | 푸터를 통해 수신거부를 선택한 것 |
| **Failed(실패)** | 유효하지 않은 이메일, 잘못된 API 키 |
| **Spam Complaints(스팸 신고)** | 수신자가 스팸으로 표시한 것 |
| **Replied(답장됨)** | 연락처가 답장한 것 |
| **Accepted / Rejected (수락됨 / 거부됨)** | SMTP 수준의 수락 결과 |

## **자주 묻는 질문**

**Q: 소프트 반송과 하드 반송의 차이점은 무엇인가요?**
소프트 반송은 일시적인 것입니다(예: 받은편지함 가득참). 하드 반송은 영구적인 것이며(예: 유효하지 않은 이메일), 발신자 평판에 해를 끼칠 수 있습니다.

**Q: 이메일 통계가 업데이트되는 데 얼마나 걸리나요?**
대부분의 이메일 통계는 거의 실시간으로 업데이트되지만, 일부 이벤트(클릭, 구독 취소, 스팸 신고 등)는 수신자 활동과 이메일 제공업체 지연에 따라 나타나는 데 몇 분이 걸릴 수 있습니다.

**Q: 전체 이메일 통계를 보려면 Mailgun을 사용해야 하나요?**
네. Mailgun만이 전달됨, 반송됨, 구독 취소됨, 스팸 신고 같은 핵심 지표에 대한 완전한 가시성을 지원합니다. 다른 SMTP 제공업체는 열람과 클릭 같은 기본 추적만 지원할 수 있습니다.
