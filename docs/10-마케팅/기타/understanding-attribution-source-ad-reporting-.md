# 어트리뷰션 소스 이해하기

# 어트리뷰션이란?

어트리뷰션(Attribution)은 연락처가 어떤 채널이나 소스로부터 유입되었는지를 분석해주는 기능입니다. 이 정보를 통해 성장 기회를 식별하고 어떤 채널이 가장 큰 잠재력을 제공하는지 파악할 수 있습니다. 예를 들어, 페이스북 광고와 구글 광고를 동시에 운영하고 있다면, 어느 쪽이 더 효과적으로 하이퍼클래스에 리드를 생성하는지 알고 싶을 것입니다. 어트리뷰션이 바로 이를 가능하게 해줍니다.

아래 아티클에서는 하이퍼클래스의 어트리뷰션 기능과 광고 리포팅 설정 방법까지 자세히 안내해드리겠습니다. 시작해보세요!

# "첫 번째" 또는 "최신" 어트리뷰션이란?

연락처를 특정 소스에 어트리뷰션할 때 일반적으로 첫 번째와 최신 어트리뷰션을 고려합니다. 두 가지 모두 모든 연락처에 저장됩니다.

첫 번째 어트리뷰션 - 연락처가 우리 시스템과 처음 상호작용한 시점으로, 이때 해당 세션의 어트리뷰션 데이터를 기록합니다. 예: 누군가가 문의 폼(Contact Us Form)을 작성하는 경우

최신 어트리뷰션 - 우리와의 가장 최근 상호작용으로, 이때 가장 최근 세션의 어트리뷰션 데이터도 기록합니다. 예: 문의 폼 작성 후 하이퍼클래스 투스텝 주문 폼(Two-Step Order Form)을 통해 제품을 구매하는 경우, 최신 어트리뷰션 데이터가 기록됩니다.

**최신 어트리뷰션은 마지막으로 기록된 상호작용에 따라 항상 변경됩니다. 하지만 첫 번째 어트리뷰션은 변경되지 않습니다.*

이제 첫 번째와 최신 어트리뷰션이 무엇인지 알았으니, 이 정보가 어디에 저장되는지 확인해보겠습니다.

# 연락처 어트리뷰션 데이터는 어디에서 확인하나요?

첫 번째든 최신 어트리뷰션 데이터든, 연락처 상세(Contact Record) 우측 하단에서 이 정보를 확인할 수 있습니다.

하위 계정(Sub-Account)에서 연락처(Contacts) > 연락처 선택 > 활동 탭(Activity Tab) 하단 우측 컬럼에서 어트리뷰션 데이터를 확인하세요.

![어트리뷰션 데이터 위치](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48247964170/original/VSgXZHb-pBWW3cGC7emsU4xKKLFyaKQJ3g.gif?1661800754)

연락처 생성에 대한 첫 번째와 최신 어트리뷰션은 다음과 같이 정의됩니다:

- 첫 번째 어트리뷰션: 연락처의 첫 번째 상호작용이 시스템에 기록된 것입니다. 보통 웹사이트를 처음 방문하거나 위에 나열된 소스 중 하나와의 상호작용입니다.
- 최신 어트리뷰션: 시스템에서 연락처로 전환되기 직전의 마지막 상호작용입니다.

## 유료 검색 (구글 광고)

유료 검색으로 분류된 트래픽은 유료 검색 캠페인(예: Google AdWords)에서 발생합니다. 이는 구글 광고 리포팅 분석에 도움이 됩니다.

다음과 같은 UTM 매개변수가 일치해야 합니다 (대소문자 구분):

{YourLandingPageUrl.com}?utm_source=adwords&utm_medium={adname}&utm_campaign={campaignname}&utm_content={adgroupname}&utm_keyword={keyword}&utm_matchtype={matchtype}&campaign_id={campaignid}&ad_group_id={adgroupid}&ad_id={creative}

구글 광고 UTM 매개변수

| 이름 | 키 | 값 | 타입 | 예시 |
|------|-----|-----|------|------|
| UTM Source | utm_source | adwords | 고정 | adwords |
| UTM Medium | utm_medium | {adname} | 동적 | {adname} |
| UTM Campaign | utm_campaign | {campaignname} | 수동, 변경 가능 | summer_sale |
| UTM Content | utm_content | {adgroupname} | 동적 | {adgroupname} |
| Match Type | utm_matchtype | {matchtype} | 동적 | {matchtype} |
| Campaign ID | campaign_id | {campaignid} | Google Ads 설정, 고정 | 123456789 |
| Ad Group ID | ad_group_id | {adgroupid} | 동적 | {adgroupid} |
| Ad ID | ad_id | {creative} | 동적 | {creative} |

자세한 정보는 [구글 광고 리포팅 설정하기](how-to-set-up-google-ad-reporting.md)를 참조하세요.

## 유료 소셜

유료 소셜로 분류된 트래픽은 유료 소셜 캠페인에서 발생합니다. 페이스북에서 리드가 유입되는 경우 페이스북 광고 리포팅 분석에 도움이 됩니다.

다음과 같은 UTM 매개변수가 일치해야 합니다 (대소문자 구분):

{YourLandingPageUrl.com}?utm_source=fb_ad&utm_medium={{adset.name}}&utm_campaign={{campaign.name}}&utm_content={{ad.name}}&campaign_id={{campaign.id}}

페이스북 및 인스타그램 광고 UTM 매개변수

| 이름 | 키 | 값 | 타입 | 예시 |
|------|-----|-----|------|------|
| UTM Source | utm_source | fb_ad | 고정 | fb_ad |
| UTM Medium | utm_medium | {{adset.name}} | 동적 | {{adset.name}} |
| UTM Campaign | utm_campaign | {{campaign.name}} | 수동, 변경 가능 | summer_sale |
| UTM Content | utm_content | {{ad.name}} | 동적 | {{ad.name}} |
| Campaign ID | campaign_id | {{campaign.id}} | Facebook Ads 설정, 고정 | 123456789 |

자세한 정보는 [페이스북 광고 리포팅 설정하기](how-to-set-up-facebook-ad-reporting.md)를 참조하세요.

## 직접 트래픽

직접 트래픽으로 분류된 트래픽은 소스 정보가 없습니다 (위의 8번째 행 참조). 일반적으로 이런 사용자는 브라우저에 URL을 직접 입력하거나 사이트에 진입하기 전에 모든 쿼리 매개변수를 제거한 경우입니다.

이런 방문자의 소스 URL을 확인하려면 직접 트래픽 소스를 클릭하세요.

![직접 트래픽 소스 확인](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/48247964541/original/eIy9MeT34UCvxRzk02GYCS7qwWMvBizWbQ.gif?1661800923)

## 자연 검색

자연 검색으로 분류된 트래픽은 Google, Bing, Yahoo, DuckDuckGo와 같은 알려진 검색 엔진의 무료 검색 결과에서 발생합니다.

검색 엔진에서 사용된 키워드를 확인하려면 소스 표의 자연 검색 소스를 클릭하세요.

알 수 없는 키워드(SSL)가 있는 경우, 검색 엔진에서 사용자 데이터를 암호화했기 때문일 가능성이 높습니다. 예를 들어, Google은 사용자가 입력한 모든 검색어를 암호화합니다.

## 소셜 미디어

소셜 미디어로 분류된 트래픽은 소셜 미디어 웹사이트나 앱에서 발생합니다. 예를 들어, 방문자가 자신의 소셜 미디어 계정에 콘텐츠나 웹사이트를 공유하고, 팔로워가 해당 콘텐츠를 방문하거나 소셜 미디어 메시지 안의 링크를 방문하는 경우입니다.

## 레퍼럴

레퍼럴로 분류된 트래픽은 웹사이트로 링크하는 외부 사이트에서 발생합니다. 검색 엔진이나 소셜 미디어 사이트는 제외됩니다. 하나의 레퍼링 도메인에서 여러 페이지가 사이트로 링크할 수 있습니다.

## 기타

수신 전화, SMS, 이메일, 왓츠앱 메시지, 페이스북 메시지 등 다양한 채널을 통해 리드가 생성되는 경우, 모든 정보를 한 곳에 표시하는 것이 중요합니다. 이를 통해 모든 리드를 추적하고 누락되는 것이 없도록 보장할 수 있습니다. 중앙화된 시스템으로 모든 리드를 관리함으로써 기업은 잠재 고객을 놓치지 않을 수 있습니다.

## CRM UI

하이퍼클래스 앱 CRM을 통해 리드가 수동으로 생성된 경우, 소스가 이렇게 정의됩니다. 이 분류는 하이퍼클래스 앱 CRM 내에서만 수동으로 생성된 리드를 식별하는 데 도움이 됩니다.

## 서드파티

Zap과 같은 서드파티 도구로 리드가 생성된 경우, 소스가 이렇게 정의됩니다. 이 분류는 서드파티 연동을 통해 생성된 리드를 식별하는 데 도움이 됩니다.

# 어떤 이벤트에서 어트리뷰션이 기록되나요?

같은 세션 내에서 연락처는 다음 중 하나 이상의 행동을 해야 합니다:

- 폼(Form)/설문(Survey) 제출
- 캘린더 예약 제출
- 채팅 위젯(Chat Widget) (연락처 정보 제출 후)
- 주문 폼(Order Form) 제출 (원스텝 또는 투스텝)

이 행동은 반드시 하이퍼클래스 폼, 설문, 캘린더, 채팅 위젯, 주문 폼이어야 모든 어트리뷰션 데이터가 캡처됩니다. 하이퍼클래스 이벤트가 아닌 경우에는 UTM 매개변수 데이터를 포함한 어트리뷰션 데이터가 캡처되지 않습니다.

# 어트리뷰션 소스는 어떻게 결정되나요?

연락처 어트리뷰션 소스가 하이퍼클래스에 의해 어떻게 추가되었는지 이해하려면, 어트리뷰션 데이터를 결정할 때 우리가 무엇을 찾는지 알아야 합니다.

모든 어트리뷰션 이벤트는 트래픽을 특정 소스로 분류하기 위한 규칙 세트를 따르며, 전체 페이지 URL과 레퍼링 도메인(사용 가능한 경우)을 이러한 규칙에 대조하여 확인합니다.

다음 순서로 적용됩니다:

| 순서 | 규칙 | 소스 |
|------|------|------|
| 1 | "utm_source" 매개변수에 구글 광고용 "adwords"가 포함됨 | 유료 검색 |
| 2 | "gclid", "wbraid" 또는 "gbraid" 매개변수가 있음 (구글 클릭 ID) / "msclkid" 매개변수가 있음 (Bing/Yahoo용 Microsoft 클릭 ID) | 유료 검색 |
| 3 | "utm_source", "utm_medium" 또는 "utm_campaign" 매개변수가 있고 레퍼링 도메인이 google.com임 | 유료 검색 |
| 4 | "utm_source" 매개변수에 페이스북 광고용 "fb_ad" 포함 / LinkedIn 광고용 "linkedin_ad", Twitter 광고용 "twitter_ad", Reddit용 "reddit_ad" 포함 / 왓츠앱 대화용 "ctwa_clid" 존재 | 유료 소셜 |
| 5 | 레퍼링 도메인이 소셜 미디어 사이트임 | 소셜 미디어 |
| 6 | 레퍼링 도메인이 검색 엔진임 (Google, Yahoo, Bing, DuckDuckGo) | 자연 검색 |
| 7 | 레퍼링 도메인이 소셜 미디어나 검색 엔진이 아님 | 레퍼럴 |
| 8 | 레퍼링 도메인이나 추적 URL이 없음 | 직접 트래픽 |
| 9 | 수신 전화/SMS/이메일/왓츠앱/페이스북 메시지 또는 위에 다루지 않은 기타 소스에서 리드가 유입됨 | 기타 |
| 10 | CRM 시스템 내에서 수동으로 리드 생성 | CRM UI |
| 11 | Zap과 같은 서드파티 연동 도구에서 리드 생성 | 서드파티 |

이제 하이퍼클래스가 특정 어트리뷰션 소스를 적용하는 방법을 알았으니, 각 어트리뷰션 소스를 함께 살펴보겠습니다.

# 광고 리포팅 설정

유료 광고 어트리뷰션 추적을 설정해야 하는 경우, 페이스북, 인스타그램, 구글 광고 설정에 대한 다음 문서를 참조하세요.

## 페이스북 및 인스타그램 광고 리포팅 설정

# 광고 리포팅 문제 해결

[페이스북 또는 인스타그램 광고 리포팅](how-to-set-up-facebook-ad-reporting.md) 또는 [구글 광고 리포팅](how-to-set-up-google-ad-reporting.md)을 설정한 후에도 어트리뷰션 데이터가 하이퍼클래스에 기록되지 않는 경우, 다음 문제 해결 단계를 시도해볼 수 있습니다...

- **최종 URL의 공백이나 오타 확인**
UTM 매개변수를 생성할 때 오타, 복사 시 생긴 불필요한 공백, 대소문자 구분을 확인하세요. 제공된 예시를 사용하되, 복사해서 붙여넣기를 권장합니다. 올바르게 작성되었는지 확인하는 시간을 갖으세요. 또한 예시는 대소문자를 구분하므로, 잘못된 위치에 대문자나 소문자가 있으면 어트리뷰션 오류가 발생합니다.

최종 URL = 연락처가 광고에서 클릭하는 URL로, 사이트나 폼에서 다음 행동을 취하는 곳으로 이동합니다.

- **제출이 최종 URL에서 발생하는지 확인**
UTM 매개변수가 기록되려면, 사용자가 광고에서 도착한 후 이 페이지를 벗어나면 안 됩니다. 예를 들어, 누군가가 페이지에 도착한 후 다른 페이지로 이동하는 버튼을 클릭해서 폼을 제출하면 UTM의 어트리뷰션 데이터가 추가되지 않습니다. 따라서 폼을 작성할 때 이 페이지에 머물러야 합니다.

해결 방법으로는 팝업이나 제출 폼이 있는 섹션을 추가하여 데이터가 캡처되도록 하는 것입니다.

- **페이스북, 인스타그램, 구글 광고에 커스텀 UTM 매개변수를 추가하지 마세요. 설정 안내에 따라 하이퍼클래스 UTM 템플릿을 사용하세요.**
설정 가이드에서 제공하는 템플릿 예시에 커스텀 UTM 매개변수를 추가하지 마세요. 이는 문제를 일으킬 수 있으며 데이터 누락이나 데이터 기록 실패를 초래할 수 있습니다.
