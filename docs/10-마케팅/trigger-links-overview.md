# 트리거 링크(Trigger Links) - 개요

트리거 링크는 SMS, 이메일, GMB, FB/IG DM, 왓츠앱에서 사용자가 클릭했을 때 특정 페이지로 리디렉션하는 유용한 방법입니다. 사용자가 트리거 링크를 클릭하면 시스템이 해당 연락처의 활동 타임라인에 기록하고, 워크플로우 내에서 미리 정의된 액션을 실행합니다. 워크플로우 구성에서 트리거 링크 클릭으로 실행할 수 있는 다양한 액션과 트리거가 있습니다.

참고사항:

트리거 링크는 SMS, 이메일, GMB, FB/IG/왓츠앱 DM에서만 지원됩니다.

## 이 문서에서 다루는 내용

**트리거 링크 활용 사례**
- 워크플로우에서의 트리거 링크
- 캠페인에서의 트리거 링크

**트리거 링크 사용법**
- 커스텀 값과 트리거 링크를 사용하여 URL을 동적으로 변경하는 방법
- 트리거 링크 클릭에 대한 워크플로우 대기(Wait) 단계 사용법
- 커뮤니케이션 채널별(SMS, 이메일, GMB, FB/IG DM) 구독 취소 트리거 링크 설정법

**FAQ**
- 트리거 링크에 브랜딩을 적용하는 방법
- 연락처의 활동 타임라인을 찾는 방법

## 트리거 링크 활용 사례

### 이메일 캠페인의 동적 URL 업데이트

트리거 링크를 사용하여 이메일 캠페인의 URL을 동적으로 업데이트할 수 있습니다. 예를 들어, 월별 또는 분기별 프로모션을 진행하면서 동일한 이메일 템플릿을 사용하고 싶다면, 현재 프로모션 URL을 가리키는 커스텀 값으로 트리거 링크를 설정할 수 있습니다. 연락처가 이 링크를 클릭하면 최신 프로모션 페이지로 이동하게 됩니다. 이렇게 하면 매번 링크를 수동으로 업데이트할 필요 없이 이메일 템플릿을 재사용할 수 있습니다.

### 트리거 링크 클릭 기반 워크플로우 자동화

트리거 링크 클릭에 반응하는 워크플로우를 설정할 수 있습니다. 예를 들어, 예약 확인 및 리마인더를 보내는 워크플로우를 만들 수 있습니다. 연락처가 예약 리마인더 이메일의 트리거 링크를 클릭하면, 시스템이 클릭을 기록하고 설문조사나 리뷰 요청 발송과 같은 워크플로우의 다음 단계를 실행합니다.

### 다양한 커뮤니케이션 채널의 구독 취소 링크

SMS, 이메일, 구글 마이 비즈니스(GMB), 페이스북, 인스타그램 등 다양한 커뮤니케이션 채널에 대한 구독 취소 트리거 링크를 구현할 수 있습니다. 연락처가 이러한 링크를 클릭하면 해당 채널에서 구독이 취소됩니다. 이는 옵트아웃 규정 준수를 보장하고 커뮤니케이션 선호도를 효과적으로 관리하는 데 도움이 됩니다.

### 트리거 링크 브랜딩

에이전시의 브랜딩에 맞게 트리거 링크의 외관을 커스터마이징할 수 있습니다. API 도메인을 설정하면([아래 FAQ 참조](trigger-links-overview.md) 트리거 링크가 에이전시의 도메인을 표시하도록 하여 사용자에게 더욱 전문적이고 브랜드화된 경험을 제공할 수 있습니다.

### 연락처 활동 추적

트리거 링크를 사용하여 SMS 및 이메일 캠페인 내에서 연락처의 참여도와 행동을 추적할 수 있습니다. 연락처가 트리거 링크를 클릭하면 해당 활동이 연락처의 활동 타임라인에 기록됩니다. 이 정보는 고객 행동과 캠페인 효과를 평가하는 데 유용할 수 있습니다.

### 캠페인 내 교육 리소스

이메일 캠페인에 트리거 링크를 포함시켜 교육 리소스를 제공할 수 있습니다. 예를 들어, 제품이나 서비스와 관련된 튜토리얼, 사용법 비디오, 또는 기사에 대한 링크를 포함할 수 있습니다. 이러한 트리거 링크를 클릭하면 자동화된 후속 조치를 실행하거나 특정 주제에 대한 연락처의 관심도를 추적할 수 있습니다.

### 이벤트 등록 및 업데이트

이벤트를 개최하는 경우, 이메일 초대장에 트리거 링크를 사용할 수 있습니다. 수신자가 RSVP하거나 이벤트 업데이트를 받기 위해 트리거 링크를 클릭하면, 참석 확인, 이벤트 세부사항 전송, 또는 추가 정보 수집 과정을 자동화할 수 있습니다.

### 상품 추천 및 개인화

이메일 마케팅에서 연락처의 선호도와 과거 상호작용을 바탕으로 개인화된 상품 추천으로 이어지는 트리거 링크를 포함시킬 수 있습니다. 이 링크를 실행하면 향후 커뮤니케이션과 상품 제안을 맞춤화하는 자동화된 프로세스를 시작할 수 있습니다.

# 트리거 링크 사용법

## 커스텀 값과 트리거 링크를 사용하여 URL을 동적으로 변경하는 방법

월별 또는 분기별로 프로모션을 진행하면서 동일한 워크플로우를 사용하고 싶을 때 유용합니다. 커스텀 값 URL을 새로운 "오퍼 URL"로 업데이트하면, 시스템이 트리거 링크를 동적으로 업데이트합니다.

참고사항:

- 이는 과거 캠페인의 기존 링크 URL에 소급적으로 영향을 주지 않습니다.
- 워크플로우를 설정할 때 SMS 또는 이메일 메시지 본문에 "트리거 링크"를 추가해야 합니다. 메시지 본문에 "커스텀 값"을 직접 배치하면 클릭 시 트리거 링크가 "실행"되지 않습니다.

커스텀 값과 트리거 링크를 올바르게 사용하려면, 먼저 플랫폼에서 트리거 링크를 만든 다음, 해당 트리거 링크에 동적 URL(커스텀 값)을 할당하고, 마지막으로 메시지 본문에 원시 커스텀 값이 아닌 실제 트리거 링크를 삽입해야 합니다.

## 트리거 링크 클릭에 대한 워크플로우 대기(Wait) 단계 사용법

이 비디오는 트리거 링크 클릭을 기반으로 다음 단계를 실행하기 위해 워크플로우 내에서 "대기" 단계 액션을 사용하는 방법을 다룹니다. 이 워크플로우는 워크플로우 라이브러리에서 "Recipe - 예약 확인 + 리마인더 + 설문조사 + 리뷰 요청"으로 찾을 수 있습니다.

## 커뮤니케이션 채널별(SMS, 이메일, GMB, FB/IG DM) 구독 취소 트리거 링크 설정법

참고사항:

기본적으로 SMS 통신사들은 ['STOP', 'STOPALL', 'UNSUBSCRIBE', 'CANCEL', 'END', 'QUIT']와 같은 미리 구축된 옵트아웃 키워드를 가지고 있습니다. 이러한 키워드가 사용되면 리드/연락처가 자동으로 구독 취소됩니다. 다시 옵트인하려면 리드/연락처가 "Start"로 응답해야 합니다.

# FAQ

### 트리거 링크에 브랜딩을 적용하려면 어떻게 하나요?

에이전시 도메인을 표시하도록 API 도메인을 설정하려면 [여기를 클릭하세요: 내 에이전시 API 도메인 설정하기](../11-설정/branding-system-generated-links-api-domain-.md).

현재는 각 로케이션/하위 계정이 자체 API 도메인을 가지도록 설정할 수 없습니다. 모든 로케이션/하위 계정에서 공유될 하나의 도메인을 추가할 수 있습니다.

### 연락처의 활동 타임라인을 어떻게 찾나요?

Contacts(연락처) > Smart List(스마트 리스트)를 클릭 > 연락처를 검색 > 'Activity(활동)'를 클릭하세요.

### Wait Action(대기 액션)의 'Timeout(타임아웃)' 토글은 무엇을 하나요?

활성화하면 타임아웃 기능이 토글 아래의 입력 상자에서 정의한 시간 후에 대기 액션(단계)을 건너뜁니다(활성화했을 때만 표시됨).
