# 반복 타이머를 만드는 방법

반복 타이머는 정기적으로 발생하는 이벤트, 세일, 활동을 관리하고 홍보하는 데 유용한 도구입니다. 단일 이벤트까지 카운트다운하는 고정 타이머와 달리, 반복 타이머는 각 사이클이 끝난 후 자동으로 리셋되어 반복되는 이벤트나 마감일에 대한 지속적인 알림을 제공합니다.

# 반복 타이머란 무엇인가요?

반복 카운트다운 타이머는 정해진 기간(예: 7일) 동안 카운트다운하여 0에 도달한 후, 설정된 루프 횟수만큼 자동으로 재시작하는 타이머입니다. 마감일이 모든 사람에게 동일하므로, 모든 방문자는 각 사이클마다 동일한 긴급감을 느끼게 됩니다.

영구적으로 만료되는 일회성 타이머와 달리, 반복 타이머는 세 가지 상태를 순환합니다:

- **대기(Wait):** 활성 카운트다운이 시작되기 전
- **활성(Active):** 카운트다운이 보이고 작동 중
- **만료(Expired):** 카운트다운이 끝나고 리다이렉트되거나 콘텐츠 숨김

---

## **반복 타이머의 주요 장점**

반복 타이머는 지속적인 관리 없이도 긴급감을 유지시켜, 모든 프로모션이 신선하고 신뢰할 수 있게 느껴지도록 합니다.

- **자동화:** 일정에 따라 자동으로 리셋되므로 타이머를 다시 만들거나 복제할 필요 없음
- **일관된 긴급감:** 방문자가 항상 활성 카운트다운을 보게 되어 행동을 유도
- **다채널 재사용:** 동일한 타이머를 퍼널, 이메일, 서드파티 사이트에서 사용 가능
- **유연한 루프:** 타이머가 실행될 특정 사이클 횟수 선택 가능
- **브랜드 스타일링:** 폰트, 색상 등 디자인을 브랜드에 맞게 커스터마이징

---

## **반복 타이머를 어디에 사용할 수 있나요?**

**주간 세일이나 프로모션**

정기적으로 발생하는 주간 세일이나 특별 할인을 강조하기 위해 반복 타이머를 사용하세요. 웹사이트, 이메일, 소셜 미디어에 타이머를 표시하여 고객에게 반복되는 절약 기회를 상기시켜 줍니다.

**월간 구독이나 갱신**

구독 기반 서비스나 멤버십 갱신의 경우, 반복 타이머로 매월 갱신일을 사용자에게 알릴 수 있습니다. 구독 관리 이메일이나 사용자 대시보드에 포함하세요.

**정기 웨비나나 수업**

정기적인 일정(예: 매주 화요일)으로 웨비나나 수업을 진행한다면, 반복 타이머로 다음 세션까지 남은 시간을 보여줄 수 있습니다. 이벤트 등록 페이지나 홍보 이메일에 타이머를 삽입하세요.

**일일 딜이나 플래시 오퍼**

매일 또는 매주 반복되는 일일 딜이나 한정 시간 플래시 오퍼의 경우, 반복 타이머가 긴급감을 조성하고 고객에게 다음 딜 기회를 상기시켜 줍니다.

---

## **루프 이벤트란 무엇인가요?**

루프 이벤트(Loop Events)를 통해 반복 타이머가 카운트다운을 완료한 후 몇 번 리셋할지 제어할 수 있습니다. 예를 들어, 루프 이벤트 값을 3으로 설정하면 타이머가 카운트다운하고, 자동으로 리셋되어 총 3번(원래 카운트다운 + 추가 2번 루프) 실행됩니다.

타이머가 매주 재시작되게 하려면:

- 타이머 시작 날짜를 원하는 다음 시작일(예: 월요일 오전 8시)로 설정
- 카운트다운 기간을 7일로 설정  
- 루프 이벤트를 주간 사이클 반복 횟수로 설정(예: 4주 연속이면 4)

각 루프는 시간대 설정에 따라 이전 카운트다운이 끝난 후 자동으로 새로 고쳐집니다. 카운트다운 기간이 1일이 아닌 이상 루프는 매일 리셋되지 않습니다. 루프는 전체 기간이 완료된 후에만 새로 고쳐집니다.

## 반복 타이머 만드는 방법

다음 단계에 따라 반복 타이머를 빠르게 만드세요:

#### 1단계: 마케팅으로 이동

- 왼쪽 네비게이션 바에서 **Marketing(마케팅)**을 선택합니다
- 상단 네비게이션 리본에서 Countdown Timers를 선택합니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050461464/original/e6lM1ZgRyR_J8zbdo3tDQOQDmIzmBy_aXA.png?1753473301)

#### **2단계:** 타이머 선택

- 새 타이머를 만들려면 파란색 New 버튼을 클릭하세요. 템플릿을 사용하면 새 타이머 디자인 시간을 절약할 수 있습니다
- 기존 타이머를 편집하려면 타이머 옆의 Edit(편집) 아이콘을 클릭하세요

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050461487/original/e_5Nvvh1oucYRWCK_HwPkwuMJRB5_4yIwg.png?1753473386)

#### 3단계: 타이머 유형 선택

- 설정 패널의 General 탭에서 Timer Type을 Recurring으로 선택합니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050461593/original/RpdouKk_sA_jXGDt7g_XM2OGQXz7_pOr2A.png?1753473680)

#### 4단계: 타이머 세부사항 설정

타이머 설정을 정의하세요:

- **Start Date/Time:** 첫 번째 사이클이 시작되는 시점
- **Loop Events:** 리셋/사이클 횟수 정의
- **Countdown Duration:** 각 사이클의 길이(예: 7일)
- **Countdown Timer Waits For:** 대기 상태가 활성 상태 다음에 오면 사이트가 자동으로 만료 페이지로 리다이렉트하지 않습니다. [이 설정에 대한 자세한 정보](#반복-타이머의-대기-시간)
- **Hide Timer on Site in Wait State:** 비활성화하면 사이트에서 대기 시간 동안 타이머가 0 상태로 표시됩니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050461875/original/ckHLglv14mBoy7tg_iNBd11QZ-2blN_qtw.png?1753474653)

#### **5단계:** 링크 추가

- **Active Link:** 타이머가 활성 상태일 때 사용자가 이동할 URL을 지정합니다
- **Expired Link:** 타이머가 만료된 후 사용자가 이동할 URL을 지정합니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050461907/original/YUguWA8gg2BKmMGbA-OzYruZmBwbtekK-w.png?1753474734)

#### **6단계:** 커스터마이징 및 저장

- 색상, 폰트, 만료 이미지, 시간 표시 형식 등 타이머 모양을 조정합니다
- 타이머에 설명적인 이름을 지정합니다
- 파란색 **Save(저장)** 버튼을 클릭하여 카운트다운 타이머를 완성하고 활성화합니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050461916/original/jCeLWqFBkCPKgWAmVx-OIiUnpht7thm_ZQ.png?1753474772)

## 반복 타이머의 대기 시간

Countdown Timer Waits For 설정을 통해 캠페인 전략에 맞는 타이머 사이클 간 정의된 일시 정지 기간을 설정할 수 있습니다. 대기 시간을 커스터마이징하면 반복 타이머에 유연성을 도입하고 맞춤형 자동화 워크플로우를 지원합니다. 다음은 이메일과 퍼널에서의 대기 시간 동작입니다:

**이메일**

- 타이머가 0 상태로 표시됩니다
- 이메일 링크 클릭 시 만료 페이지로 리다이렉트됩니다

**퍼널**

- 대기 중에 타이머를 숨길 수 있습니다(활성화된 경우)
- 같은 세션에서 타이머가 활성 → 대기로 전환되면 사용자는 페이지에 머물러 있습니다(자동 리다이렉트 없음)
- 페이지 새로고침 시 대기 상태이면 만료 페이지로 리다이렉트됩니다

## **자주 묻는 질문**

**Q: 대기 시간이란 무엇이고 왜 사용하나요?**
대기 시간은 사이클 간 선택적 간격입니다(예: 주간 세일 간 24시간). 타이머가 즉시 재시작되는 것을 방지하여 팀이 오퍼나 재고를 업데이트할 시간을 제공합니다.

**Q: 타이머가 라이브된 후 대기 시간이나 루프 이벤트를 편집할 수 있나요?**
네. **Marketing(마케팅) → Countdown Timers**에서 타이머를 편집하면 타이머가 임베드된 모든 곳에 변경사항이 적용됩니다.

**Q: 타이머가 방문자의 현지 시간대에 맞춰 조정되나요?**
아니요. 반복 타이머는 설정 시 선택한 시간대로 실행되며, 모든 방문자가 동일한 기준 시계를 봅니다.

**Q: 활성화할 수 있는 반복 타이머 개수에 제한이 있나요?**
하이레벨(HighLevel)에서 개수를 제한하지는 않지만, 각 타이머는 요청 시 이미지 자산을 생성합니다. 최적의 성능을 위해 목록을 깔끔하게 유지하고 사용하지 않는 타이머는 삭제하세요.
