# 반복 타이머의 커스텀 대기 시간이란?

커스텀 대기 시간은 반복 타이머가 00 : 00 : 00에 도달한 후 설정된 시간 동안 일시정지한 다음, 자동으로 다음 주기를 시작하는 기능입니다. 일시정지 기간(대기 상태) 동안 퍼널에서 타이머를 0으로 표시하거나 완전히 숨길 수 있으며, 이메일에서는 만료 페이지로 연결되는 0 상태 배너가 표시됩니다.

![커스텀 대기 시간 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155050757246/original/Zvnf4TGlWMr5O3hF3Tu93wuF8-EY-GqO5A.jpeg?1753986890)

## 커스텀 대기 시간의 주요 장점

주기 사이의 일시정지를 추가하면 새로운 캠페인 아이디어를 실현하고 반복적인 설정 작업을 대폭 줄일 수 있습니다.

- "매주 금요일 세일"이나 주간 챌린지 같은 반복 프로모션을 자동화할 수 있습니다.

- 매주 다음 세션까지 카운트다운하여 에버그린 웨비나를 운영할 수 있습니다.

- 노출 제어—휴무 시간 동안 퍼널에서 타이머를 숨겨 페이지를 깔끔하게 유지할 수 있습니다.

- 타이머를 복제하는 대신 하나의 에셋을 이메일과 퍼널에서 재사용할 수 있습니다.

- 일관된 0 상태 및 리디렉션 동작으로 이메일과 페이지 로직을 맞출 수 있습니다.

## 대기 시간이 있는 반복 타이머 설정 방법

이 과정을 한 번만 설정하면 타이머가 모든 주기를 자동으로 관리합니다.

- Marketing(마케팅) ▸ Countdown Timer(카운트다운 타이머) ▸ New Timer(새 타이머)로 이동합니다.

- Recurring(반복)을 선택하고 시작 및 종료 오프셋을 설정합니다.

- Custom Wait Time(커스텀 대기 시간)을 토글하고 기간을 입력합니다(예: 5일 12시간).

- (선택사항) Hide Timer in Wait State(대기 상태에서 타이머 숨기기)를 활성화하여 일시정지 동안 퍼널에서 타이머를 제거합니다.

- 대기 중에 방문자가 타이머를 클릭하거나 퍼널을 새로고침할 때 보여줄 Expiry Page(만료 페이지) / URL을 선택합니다.

- 저장하고 퍼널 및/또는 이메일에 타이머 블록을 삽입합니다.

## 타이머 라이프사이클 설명

각 단계에서 연락처가 보는 내용을 이해하면 매끄러운 고객 여정을 설계할 수 있습니다.

```
활성 주기 ─┐
           │ 00:00 도달
           ▼
      대기 상태 (커스텀 기간)
           │ 자동 재시작
           ▼
    다음 활성 주기 → … (반복)
```

| 상태 | 이메일 동작 | 퍼널 동작 |
|------|-------------|----------|
| 활성 | 실시간 카운트다운 | 실시간 카운트다운 |
| 대기 | 타이머가 00 : 00 : 00 표시; 클릭 시 만료 페이지로 리디렉션 | • 타이머 숨김 또는 00 : 00 : 00 표시 (토글 설정에 따라)<br>• 대기 중 페이지 새로고침 시 만료 페이지로 리디렉션 |
| 재설정 | 새 주기 자동 시작 | 새 주기 자동 시작 |

## 활용 사례 템플릿

바로 사용할 수 있는 설정값으로 빠르게 시작해보세요.

| 시나리오 | 시작 → 종료 | 대기 시간 | 결과 |
|----------|-------------|----------|------|
| 매주 금요일 세일 | 금 00:00 → 금 23:59 | 6일 1분 | 자정에 다음 금요일을 위해 타이머 재시작 |
| 주간 웨비나 | 월 09:00 → 목 17:00 | 3일 16시간 | 매주 월요일 오전에 카운트다운 재개 |
| 월간 출시 | 1일 00:00 → 3일 23:59 | 27일 | 매월 첫 3일 동안 실행 |

## 알려진 동작 방식 및 팁

예외 상황 로직을 이해하여 예상치 못한 문제를 방지하세요.

- 이메일 0 상태는 스타일이 적용되지 않은 "00 : 00 : 00"입니다. 대기 중에는 커스텀 CSS/스타일이 적용되지 않습니다.

- 타이머 숨기기 토글은 퍼널에만 영향을 줍니다. 이메일에서는 항상 0 상태가 표시됩니다.

- 방문자의 세션이 페이지 새로고침 없이 활성 → 대기 상태를 거치면 퍼널 페이지에 그대로 머물러 있습니다(자동 리디렉션 없음).

- 최대 대기 기간: 365일; 최소: 1분.

- 분석에서는 대기가 시작되는 순간 "Timer Expired(타이머 만료)" 이벤트가 기록됩니다. 이를 활용해 프로모션 후 자동화를 트리거할 수 있습니다.

## 자주 묻는 질문

**Q1. 대기 기간이 일광절약시간 변경을 걸칠 수 있나요?**
A: 네, 가능합니다. 타이머는 고정된 날짜가 아닌 기간을 저장하므로 일광절약시간이 변경될 때 엔진이 오프셋을 재계산합니다.

**Q2. 나중에 대기 시간을 편집하면 어떻게 되나요?**
A: 새로운 기간은 타이머가 다음번에 대기 상태에 들어갈 때 적용되며, 현재 주기에는 영향을 주지 않습니다.

**Q3. 이메일의 0 상태를 스타일링할 수 있나요?**
A: 아직 불가능합니다. 0 상태는 전달성 일관성을 위해 기본 타이머 폰트와 색상을 사용합니다.

**Q4. 타이머 숨기기를 비활성화해도 퍼널에서 리디렉션이 실행되나요?**
A: 네, 가시성과 관계없이 대기 중 페이지를 새로고침하면 항상 리디렉션됩니다.

**Q5. 하나의 타이머로 몇 번의 주기를 실행할 수 있나요?**
A: 제한이 없습니다. 타이머를 보관하거나 삭제할 때까지 주기가 계속됩니다.

**Q6. 캐시된 이메일에서 연락처가 다른 시간을 보게 되나요?**
A: 이메일 타이머는 열릴 때 실시간 데이터로 새로고침되며, 캐시된 이미지는 서버 측에서 교체됩니다.

**Q7. 반복되지 않는 타이머에도 대기 기간을 추가할 수 있나요?**
A: 아니요. 대기 시간은 반복(Recurring) 타이머 타입에서만 사용할 수 있습니다.
