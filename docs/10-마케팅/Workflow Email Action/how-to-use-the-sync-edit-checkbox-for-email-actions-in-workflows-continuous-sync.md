# 워크플로우 이메일 액션의 동기화 편집 체크박스 사용법(지속 동기화)

새로운 동기화 변경 기능을 통해 워크플로우의 이메일 액션 변경사항이 템플릿을 업데이트할지 여부를 제어할 수 있습니다. 이메일 액션용 템플릿을 선택할 때 체크박스로 간편하게 동기화를 활성화하거나 비활성화하세요.

## 새로운 기능

### 템플릿 선택 시 동기화 편집 체크박스란?

이메일 액션을 생성하고 기존 템플릿을 선택하면 이제 "템플릿과 편집 동기화(Sync Edits to Template)" 체크박스가 나타납니다.

"템플릿과 편집 동기화" 체크박스를 통해 워크플로우 이메일과 연결된 템플릿의 변경사항이 항상 자동으로 서로 업데이트될지 제어할 수 있습니다. 켜면 동기화 상태를 유지하고, 끄면 독립적으로 개별 편집이 가능해집니다.

![템플릿과 편집 동기화 체크박스](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052494303/original/uSBciiXrkLhMU66n1pNYcaeEFFfqgD-dRw.png?1756300264)

### 동기화 가능성: 동기화가 켜져 있거나 꺼져 있을 때 어떤 일이 일어나는지

**동기화가 켜져 있을 때:**
- 워크플로우 이메일에서 변경한 내용이 자동으로 원본 템플릿을 업데이트하고, 템플릿의 변경사항도 워크플로우 이메일을 업데이트합니다.
- 이것은 양방향 동기화입니다—워크플로우 액션과 템플릿이 항상 일치합니다.

**동기화가 꺼져 있을 때:**
- 워크플로우 이메일에서 변경한 내용이 원본 템플릿에 영향을 주지 않으며, 향후 템플릿 변경사항도 워크플로우에 영향을 주지 않습니다.
- 워크플로우 이메일과 템플릿이 이제 독립적인 콘텐츠와 업데이트를 갖게 됩니다.

## 장점

**동기화가 켜져 있을 때:**
- 워크플로우 이메일이나 원본 템플릿의 변경사항이 항상 다른 쪽을 업데이트합니다.
- 템플릿과 워크플로우 전반에서 콘텐츠를 완벽하게 일치시킵니다.
- 한 곳에서 업데이트를 관리하여 시간을 절약합니다.

**동기화가 꺼져 있을 때:**
- 워크플로우 이메일 액션과 원본 템플릿이 완전히 독립적입니다.
- 다른 템플릿이나 액션에 영향을 주지 않고 특정 워크플로우에 맞게 이메일을 커스터마이징할 수 있습니다.
- 특정 사용 사례를 위해 템플릿에서 '독립'하고 싶을 때 이상적입니다.

## 사용 방법

1. 워크플로우에서 이메일 액션을 추가하거나 편집할 때 템플릿을 선택합니다.

2. "템플릿과 편집 동기화" 체크박스가 나타납니다.

**체크하는 경우(켜기):**
- 모달이 확인 메시지를 표시합니다: 워크플로우나 템플릿 중 어느 쪽의 변경사항이든 다른 쪽을 업데이트합니다.
- "동기화 켜기(Turn On Sync)"를 클릭하여 확인하면, 이제 모든 편집사항이 동기화 상태를 유지합니다.
- 예: 워크플로우 이메일을 편집하면 템플릿이 업데이트되고, 템플릿을 편집하면 워크플로우 이메일도 업데이트됩니다.

![동기화 켜기 모달](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052494386/original/7oTiQYlZJ3DSAYu1VZI4EMpZsvtTTOZKBA.png?1756300305)

**체크하지 않거나 끄는 경우:**
- 모달이 워크플로우 이메일과 템플릿이 서로 영향을 주지 않을 것임을 설명합니다.
- "동기화 끄기(Turn Off Sync)"를 클릭하여 확인하면, 이제 편집사항이 독립적입니다.
- 예: 워크플로우 이메일의 변경사항이 템플릿을 업데이트하지 않고, 템플릿 변경사항도 이 워크플로우 이메일을 업데이트하지 않습니다.

![동기화 끄기 모달](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052494668/original/gwi9Dl-AyUTgWS0foDd-eGa1LubTcAlU0g.png?1756300437)

### 동기화 변경 모달:

이 모달은 템플릿 섹션에 있어서 어떤 워크플로우의 이메일 액션이 템플릿에 연결되어 있고 변경사항을 동기화하고 있는지 확인할 수 있습니다.

- 템플릿의 이메일 빌더에서 케밥 메뉴 → "동기화 변경(Sync Changes)"을 통해 접근할 수 있습니다.
- 모달 내부에서 다음을 볼 수 있습니다:
  - 템플릿에 연결된 모든 이메일 액션 목록
  - 각 액션의 지속 동기화(Continuous Sync) 상태(활성화/비활성화)
- 활성화된 이메일 액션은 체크박스가 선택되고, 비활성화된 이메일 액션은 체크박스가 해제됩니다.
- 지속 동기화가 비활성화된 이메일 액션에 대해서는 이메일 액션을 선택하여 템플릿 콘텐츠를 이메일 액션에 일회성으로 동기화할 수 있습니다.
- **한 번 동기화(Sync Once)** CTA를 클릭하면 화면에 성공 알림이 표시됩니다.
- 지속 동기화가 활성화된 이메일 액션의 경우 이 모달에서 체크박스를 해제할 수 없습니다.

![동기화 변경 모달](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053097326/original/IuD3P_DhlFj6yq1OVFdjwyM-VFYo-dVrIg.png?1756987254)

## 자주 묻는 질문

**Q1: "템플릿과 편집 동기화" 기능은 무엇인가요?**
**A:** 워크플로우 이메일 액션에서 변경한 내용이 연결된 템플릿을 자동으로 업데이트하고, 그 반대도 가능하게 해줍니다. 활성화하면 양쪽이 항상 동기화 상태를 유지합니다.

**Q2: 언제 동기화 변경 옵션이 나타나나요?**
**A:** 이메일 액션을 추가하거나 편집할 때 기존 템플릿을 선택하면 "템플릿과 편집 동기화" 체크박스를 볼 수 있습니다.

**Q3: 동기화를 켜면 어떻게 되나요?**
**A:** 워크플로우 이메일이나 원본 템플릿의 모든 편집사항이 양쪽 모두에 반영됩니다—모든 것이 완벽하게 일치합니다.

**Q4: 동기화를 끄면 어떻게 되나요?**
**A:** 워크플로우 이메일과 템플릿이 독립적이 됩니다. 한쪽의 변경사항이 다른 쪽에 영향을 주지 않습니다.

**Q5: 어떤 워크플로우가 템플릿과 동기화되는지 어떻게 확인하나요?**
**A:** 이메일 빌더의 케밥 메뉴에서 "동기화 변경"을 선택하세요.

**Q6: 설정 후 동기화 설정을 변경할 수 있나요?**
**A:** 네, 나중에 어떤 이메일 액션에 대해서든 동기화를 켜거나 끌 수 있습니다. 변경사항은 모달에서 확인됩니다.

**Q7: "템플릿과 편집 동기화" 체크박스의 기본 동작은 무엇인가요?**
**A:** 동기화 변경 체크박스의 기본 동작:
- 템플릿에 변경사항이 있는 기존 이메일 액션: 기본값은 꺼짐
- 템플릿에 변경사항이 없는 기존 이메일 액션: 기본값은 켜짐
- 템플릿을 사용하는 새 이메일 액션: 기본값은 꺼짐

**Q8: 스냅샷을 찍거나 워크플로우를 복제할 때 동기화 체크박스는 어떻게 되나요?**
**A:** 동기화 체크박스 상태가 보존됩니다. 스냅샷 후 대상 계정으로 이전되며, 복제된 워크플로우에서도 동일하게 유지됩니다.
