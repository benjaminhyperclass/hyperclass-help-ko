# 하위 계정을 위한 템플릿 라이브러리 표시 제어

이 가이드는 특정 하위 계정에서 템플릿 라이브러리를 숨기거나 모듈별로 표시를 제한하는 방법을 안내합니다. 이를 통해 로케이션 사용자의 사용 경험을 간소화하고 혼란이나 오용을 방지할 수 있습니다.

## 템플릿 라이브러리 표시 제어란?

템플릿 라이브러리 표시 제어를 통해 에이전시 관리자는 하위 계정이 템플릿 라이브러리에 접근할 수 있는지 여부를 결정하고, 접근이 허용된 경우 어떤 제품(모듈)을 탐색할 수 있는지 설정할 수 있습니다. 이 설정은 로케이션 사용자의 주의를 분산시키지 않고, 에이전시가 계정 전반에서 템플릿 사용을 표준화하는 데 도움이 됩니다.

- 에이전시 뷰에서 하위 계정별로 설정
- **숨기기**, **표시**, 또는 **특정 제품에서 숨기기** 선택 가능
- 변경 사항은 즉시 적용되며 **로케이션(하위 계정) 사용자**에게 영향을 미침

---

## 템플릿 라이브러리 표시 제어의 주요 장점

- **세부적인 제어**: 전체 라이브러리를 숨기거나, 모두 표시하거나, 선택된 제품만 하위 계정에 노출
- **더 깔끔한 경험**: 불필요한 템플릿을 제거하여 로케이션 사용자가 필요한 것만 볼 수 있도록 함
- **정책 시행**: 특정 모듈에서 오용이나 정책 위반 템플릿 사용 방지
- **빠른 온보딩**: 승인된 템플릿으로 사용자를 안내하고 사용법 문의 감소
- **즉시 변경**: 저장 후 업데이트가 즉시 적용됨

---

## 접근 권한

표시를 설정하려면 적절한 역할 접근 권한이 필요하며, 에이전시와 하위 계정 사용자의 범위가 다릅니다. 이 맥락은 누가 무엇을 볼 수 있는지에 대한 혼란을 방지합니다.

- **에이전시 관리자**만 하위 계정의 템플릿 라이브러리 표시를 설정할 수 있음
- **에이전시 사용자/관리자**는 모든 로케이션에서 항상 템플릿 라이브러리를 볼 수 있음. 제한은 **로케이션(하위 계정) 사용자에게만** 적용됨
- **시스템 이메일 템플릿**은 표시 설정에 관계없이 사용자에게 항상 표시됨

## 템플릿 라이브러리 표시 설정 방법 (하위 계정별)

특정 하위 계정의 템플릿 라이브러리 표시를 빠르고 쉽게 설정하려면 다음 단계를 따르세요.

#### **1단계:** 템플릿 라이브러리로 이동

에이전시 관리자로 로그인하세요. 에이전시 대시보드에서 메인 메뉴로 이동하여 템플릿 라이브러리를 선택하세요.

![템플릿 라이브러리 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058795466/original/Dnbj9BmT0DsttQbtyEdfak38ELI5x9Y5_Q.png?1763586922)

#### **2단계:** 템플릿 설정 열기

템플릿 라이브러리 모듈 우측 상단의 **설정**(톱니바퀴 아이콘) 버튼을 클릭하세요.

![템플릿 설정 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058795510/original/HbAaszvZb3_T2hlGNTj84MyYXqtAv40Ohw.png?1763587016)

#### 3단계: 하위 계정 커스터마이즈 설정 열기

왼쪽 네비게이션 바를 사용하여 **Customize for Sub-Account Settings** 탭을 선택하세요.

![하위 계정 설정 탭](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058795563/original/vf3wvRWTrcg3LaEIIp17wzFwjDQgl8izUA.png?1763587168)

#### 4단계: 하위 계정 검색 및 선택

검색 상자를 사용하여 업데이트하고자 하는 하위 계정을 찾으세요. 선택된 하위 계정 옆의 편집 버튼을 클릭하세요.

![하위 계정 검색 및 편집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058881203/original/sPP_1x6iIIAPK6rRXIhx_EuxIEKcotlngg.png?1763657676)

#### **5단계:** 표시 설정 선택

General Settings 탭에서 다음 중 하나를 선택하세요:

- **Hide**: 해당 하위 계정에서 전체 템플릿 라이브러리를 숨김
- **Show**: 전체 템플릿 라이브러리를 표시
- **Hide for specific products**: 모듈별로 표시를 제한하는 선택기 열기 (퍼널, 웹사이트, 스토어, 폼 등)

![표시 설정 옵션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058881555/original/PmMdOhHtn1gLIUvNyEXZp1Dg8K4vyWSD3w.png?1763657902)

#### **6단계:** 카테고리별 편집

**Categories** 탭을 클릭하세요. 제품 드롭다운(예: **Websites**, **Funnels**, **Emails**)을 사용하여 표시를 편집하고자 하는 제품을 선택하세요. 그 다음:

- **Hide all Categories** 토글을 사용하여 선택된 제품의 모든 카테고리를 숨기거나
- 개별 **Hide** 슬라이더를 사용하여 해당 제품의 특정 카테고리를 숨김

![카테고리별 표시 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155058885559/original/f4FbJQ_x2nRg-1z2hFn8FqsmHwXZyLImmA.png?1763661950)

## 자주 묻는 질문

**Q: 라이브러리를 숨기면 에이전시 레벨 사용자도 접근할 수 없나요?**
아니오. 에이전시 사용자/관리자는 모든 로케이션에서 항상 템플릿 라이브러리를 볼 수 있습니다.

**Q: 시스템 이메일 템플릿도 이 설정의 영향을 받나요?**
아니오. 시스템 이메일 템플릿은 항상 표시됩니다.

**Q: 여러 하위 계정에 동일한 표시 설정을 한 번에 적용할 수 있나요?**
현재는 불가능합니다. 하위 계정별로 표시를 설정해야 합니다.

**Q: 변경 사항이 템플릿으로 이미 생성된 기존 자산에 영향을 주나요?**
아니오. 라이브러리를 숨기는 것은 새 템플릿의 탐색/가져오기를 제한합니다. 하위 계정에서 템플릿으로 이미 생성된 자산은 편집 가능한 상태로 유지됩니다.
