# 색상 선택기의 전역 커스텀 색상

전역 커스텀 색상은 브랜드에 맞는 안전한 색상을 한 번 저장하여 Hyperclass 전체에서 재사용할 수 있게 해줍니다. 브랜드 보드에 색상 팔레트를 추가하면 이메일, 퍼널, 폼의 색상 선택기에 표시됩니다. 색상은 양방향으로 동기화되어 저장된 색상을 업데이트하면 해당 전역 색상을 사용하는 디자인이 자동으로 업데이트됩니다.

## 전역 커스텀 색상이란?

전역 커스텀 색상은 **브랜드 보드 → 글로벌 설정**에서 정의하는 재사용 가능한 색상 견본입니다. 빌더 전체의 색상 선택기에 나타나 팀이 정확히 동일한 브랜드 색상을 적용하고, 에셋의 일관성을 유지하며, 색상이 변경될 때 중앙에서 업데이트할 수 있습니다.

**디자인 키트 색상(브랜드 보드)과의 차이점:** 디자인 키트 색상은 브랜드 보드 내에 있으며 선택기에도 나타납니다. 전역 커스텀 색상은 빠른 재사용과 적용된 인스턴스의 양방향 동기화를 위해 설계된 추가적인 중앙 관리 팔레트입니다. 두 가지 모두 선택기에 나타날 수 있으며, 전역 커스텀 색상은 최소 하나의 색상을 추가한 후에만 표시됩니다.

---

## 전역 커스텀 색상의 주요 장점

- **에셋 간 일관성:** 모든 디자인에서 동일한 모습과 느낌을 유지합니다.

- **시간 절약:** 색상을 한 번 업데이트하면 모든 곳에 반영됩니다.

- **쉬운 리브랜딩:** 에이전시는 퍼널, 이메일을 복제/공유하고 전역 색상만 업데이트하여 새로운 로케이션을 빠르게 리브랜딩할 수 있습니다.

- **항상 접근 가능:** 전역 색상은 모든 색상 선택기에 나타납니다.

- **빠른 가져오기:** 기존 커스텀 값을 불러와서 더 빠른 설정이 가능합니다.

---

## 동기화 작동 방식 및 모범 사례

동기화 작동 방식을 이해하면 실시간 디자인을 손상시키지 않고 자신 있게 색상을 변경할 수 있습니다.

- **적용된 인스턴스의 양방향 동기화:** 저장된 전역 색상을 업데이트하면 해당 특정 전역 견본을 사용한 디자인이 업데이트됩니다.

- **최소 하나의 색상 추가:** 전역 커스텀 색상 섹션은 최소 하나의 색상을 생성한 후에만 선택기에 나타나 디자인 키트 색상과 구별하는 데 도움이 됩니다.

- **명확한 이름 지정:** 기능적 이름(Primary, Secondary, Accent)을 사용하여 팀이 어떤 견본을 사용해야 하는지 알 수 있게 합니다.

- **리브랜딩 계획:** 대규모 변경의 경우 필요하다면 디자인을 복제한 후 전역 색상을 조정하여 일관되게 업데이트를 전파합니다.

## 글로벌 설정을 사용한 전역 색상 추가, 편집 및 삭제

글로벌 설정을 사용하여 팀이 빌더 전체에서 재사용할 색상을 추가, 명명, 정리 및 관리합니다. 최소 하나의 색상을 생성하면 색상 선택기 내에 전역 커스텀 색상 섹션이 나타납니다.

### 새 전역 색상 추가

- Marketing(마케팅) → Brand Boards(브랜드 보드) → **Global Settings(글로벌 설정)**로 이동합니다.

![글로벌 설정으로 이동](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055025570/original/ZqO6b_vTELt36zUV3riYrSNJyxDzEDsn7w.png?1759335619)

- **+ Add(추가)**를 클릭한 후 **Add New(새로 추가)**를 클릭합니다.

![새 색상 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055025622/original/ljAkjeA_CznVSIP8CCRWd0Eih8UqpNQCsA.png?1759335661)

- **Name(이름)**(예: Primary, Accent, Background)과 HEX/RGB 값(예: #1A73E8)을 입력하거나 색상 선택기에서 색상을 선택합니다.

![색상 정보 입력](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055025696/original/Qwq4yLXAk7k9YXbgD-mVfihglAHWQy-8rQ.png?1759335736)

- **Save(저장)**을 클릭합니다.

![색상 저장](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055025720/original/BvuBURcJZypMtOWvbFQx7FLmuOkajIEbNw.png?1759335765)

### 전역 색상 편집, 이름 변경 또는 삭제

브랜드 보드의 글로벌 설정에서 기존 전역 색상을 편집, 이름 변경 및 삭제할 수 있습니다.

**주의:** 삭제하면 선택기에서 색상이 제거되며, 빌더가 견본을 저장하는 방식에 따라 해당 색상을 사용한 디자인이 제거를 반영하여 업데이트될 수 있습니다.

- 전역 색상을 삭제하려면 **휴지통** 아이콘을 클릭합니다.

![색상 삭제](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055025839/original/i6jhTZ6JpItE7uYcazq_eSQ4ygDiLW4CqQ.png?1759335938)

- 전역 색상을 편집하거나 이름을 변경하려면 **Edit(편집)(연필)** 아이콘을 클릭합니다. 필요한 편집을 할 수 있는 색상 선택기가 열립니다.

![색상 편집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055025927/original/__NFp1Xhgh5mNBbfNk5piCMMN-scZ7PdsQ.png?1759336024)

## 커스텀 값에서 전역 커스텀 색상 가져오기

이미 브랜드 색상을 커스텀 값으로 저장하고 있다면 이를 가져와서 전역 커스텀 색상 팔레트를 빠르게 채울 수 있습니다. 가져오기는 적격한 값을 글로벌 설정에 복사하여 중앙 관리할 수 있게 합니다.

### 시작하기 전에

- 커스텀 값에 유효한 색상 항목(일반적으로 #000000과 같은 HEX 코드)이 포함되어 있는지 확인합니다.

- 가져올 때 매핑을 쉽게 하기 위해 명확한 이름(예: Primary_Color, Button_BG)을 사용합니다.

### 가져오기 단계

- **Marketing(마케팅) → Brand Boards(브랜드 보드) → Global Settings(글로벌 설정)**으로 이동합니다.

![글로벌 설정 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055026094/original/GlDDTugnVQsfCsd2OUR-J3isPueA_G8TEw.jpeg?1759336218)

- **+ Add(추가)**를 클릭한 후 **Import(가져오기)**를 클릭합니다.

![가져오기 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055026139/original/Hfe6C7a42US4Dt-a9vQclDI3CdaV7w4VdQ.png?1759336291)

- 색상을 나타내는 커스텀 값을 선택하고 **Import(가져오기)**를 클릭합니다. 선택한 값 중 유효한 색상이 아닌 것(예: HEX 코드가 아닌 것)이 있다면 가져오기 전에 건너뛰거나 수정합니다.

![색상 값 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055026205/original/oBHz1P0lH9irGCccU6Qn_3cJk7Bz3kQP3w.png?1759336350)

## 빌더에서 전역 커스텀 색상 사용하기(색상 선택기)

최소 하나의 전역 커스텀 색상이 저장되면 지원되는 빌더 내 색상 선택기에 "Global Custom Colors(전역 커스텀 색상)" 그룹이 나타납니다. 이를 통해 정확하고 승인된 견본을 모든 곳에 쉽게 적용할 수 있습니다.

### 표시되는 위치

- 이메일(드래그 앤 드롭 빌더)

- 퍼널 및 웹사이트(페이지 빌더)

- 폼 및 설문

### 편집기에서 전역 색상 적용하기

- 빌더(예: 이메일, 퍼널 페이지 또는 폼)를 열고 색상 설정이 있는 요소(텍스트, 버튼, 배경, 테두리)를 선택합니다.

- **Color Picker(색상 선택기)**를 엽니다.

- **Global Custom Colors(전역 커스텀 색상)** 아래의 견본을 클릭하여 적용합니다.

![전역 색상 적용](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055027781/original/vP4OO7Vv4okDp9mvz9Wsl3GiiJuocBAy1g.png?1759338087)

### 색상 선택기에서 관리하기

색상 선택기에서 **Manage(관리)**를 클릭하여 글로벌 설정으로 바로 이동할 수 있습니다(사용 가능한 경우). 여기서 색상을 추가하거나 조정할 수 있습니다.

![색상 관리](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055027806/original/ll37T6ttin2ZOxIq-AUbNMEfMnkev8rTuw.png?1759338117)

## 자주 묻는 질문

**Q: 전역 색상을 변경하면 기존 에셋이 업데이트되나요?**
네. 전역 커스텀 색상은 양방향으로 동기화됩니다. 변경 사항은 해당 색상이 적용된 모든 디자인에 즉시 반영됩니다.

**Q: 색상 선택기에서 전역 색상이 보이지 않는 이유는 무엇인가요?**
글로벌 설정에서 최소 하나의 색상을 추가한 경우에만 섹션이 나타납니다.

**Q: 기존 커스텀 값 기반 색상을 가져올 수 있나요?**
네. 커스텀 값 섹션에서 글로벌 설정으로 가져올 수 있습니다. 이는 복사본만 생성하며 커스텀 값 기반 색상을 이동시키지는 않습니다.

**Q: 디자인 키트 색상과 어떻게 다른가요?**
디자인 키트 색상은 빠른 액세스만을 위한 것이며 동기화되지 않습니다. 전역 커스텀 색상은 모든 에셋에서 동기화됩니다.

**Q: 에이전시가 클라이언트 온보딩에 사용할 수 있나요?**
물론입니다! 에이전시는 퍼널/이메일을 복제하고 전역 색상만 업데이트하여 새 클라이언트를 위해 즉시 리브랜딩할 수 있습니다.
