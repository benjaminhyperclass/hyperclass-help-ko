# 이메일 캠페인/템플릿 미리보기 및 테스트란?

미리보기 및 테스트는 하이퍼클래스 이메일 빌더의 도구로, 실제 발송 전에 콘텐츠를 시각적으로 확인하고 안전한 테스트 사본을 보낼 수 있게 해줍니다. 이 옵션들은 실제 연락처나 캠페인 지표에 영향을 주지 않으면서 개인화, 레이아웃, 링크를 검증할 수 있도록 도와줍니다. **Preview(미리보기)**를 사용해 빌더에서 이메일이 어떻게 렌더링되는지 확인하고, **Send Test(테스트 발송)**를 사용해 최종 인박스 확인을 위해 샘플을 내 인박스로 보내보세요.

## **이메일 미리보기 및 테스트의 주요 이점**

- 정확성: 병합 필드, 커스텀 값, 조건부 콘텐츠가 의도한 대로 표시되는지 확인합니다.
- 컴플라이언스: 런치 전에 필수 요소(예: 수신거부 섹션)가 있는지 검증합니다.
- 품질 보증: 특히 HTML을 가져온 경우 레이아웃, 이미지, 링크 문제를 조기에 발견합니다.
- 속도: 실제 목록에 발송하거나 분석 데이터를 왜곡하지 않으면서 빠르게 반복 작업할 수 있습니다.

## 이메일 캠페인 미리보기 및 테스트 발송하는 방법

**참고:** 이메일 템플릿과 캠페인을 생성, 편집, 발송하는 방법을 배우려면 [시작하기 - 이메일 캠페인 런치](getting-started-launch-an-email-campaign.md)와 [이메일 빌더 및 인라인 에디터 사용법](how-to-use-the-email-builder-and-its-in-line-editor.md) 상세 가이드를 확인해주세요.

- `Marketing(마케팅) → Emails(이메일) → Campaigns(캠페인)`로 이동합니다.
- 캠페인을 클릭해서 엽니다.

![이메일 캠페인 열기](https://jumpshare.com/share/Y8i4lTrITYfFmnOYk1ut+/GIF+Recording+2025-12-03+at+6.20.08+PM.gif)

- 우측 상단의 **Preview(미리보기)** 버튼을 클릭하고 **Desktop/Tablet/Mobile** 뷰를 확인합니다.

![미리보기 확인](https://jumpshare.com/share/tfbIpnG6wPI5x9lzBby4+/GIF+Recording+2025-12-03+at+6.21.58+PM.gif)

- 테스트 이메일을 발송하려면 Preview(미리보기) 버튼 옆의 **Three Dots(세 점)** 메뉴를 클릭하고 Test Email 버튼을 클릭합니다.

![테스트 이메일 버튼](https://jumpshare.com/share/y9qA4pgUtprtzoJNyFi2+/Screen+Shot+2025-12-03+at+6.24.13+PM.png)

- **Sender Email(발신자 이메일)**을 선택하고, **Sender Name(발신자 이름)**을 추가하고, Receiver Email(수신자 이메일)을 선택하고, **Preview Text(미리보기 텍스트)**와 **Email Subject(이메일 제목)**을 추가합니다.
- **Send Test Email(테스트 이메일 발송)** 버튼을 클릭합니다.

![테스트 이메일 설정](https://jumpshare.com/share/nVETX2d4xu24bpHxvVqa+/Screen+Shot+2025-12-03+at+6.25.21+PM.png)

## 이메일 템플릿 미리보기 및 테스트하는 방법

**참고:** 수신거부 링크는 **테스트 이메일에서 작동하지 않습니다**.

- `Marketing(마케팅) → Emails(이메일) → Templates(템플릿)`으로 이동합니다.
- 템플릿을 클릭해서 엽니다.

- 테스트 이메일을 발송하려면 Preview(미리보기) 버튼 옆의 **Three Dots(세 점)** 메뉴를 클릭하고 Test Email 버튼을 클릭합니다.

- **Sender Email(발신자 이메일)**을 선택하고, **Sender Name(발신자 이름)**을 추가하고, Receiver Email(수신자 이메일)을 선택하고, **Preview Text(미리보기 텍스트)**와 **Email Subject(이메일 제목)**을 추가합니다.
- **Send Test Email(테스트 이메일 발송)** 버튼을 클릭합니다.

## 자주 묻는 질문

**Q: 테스트 이메일에서 수신거부 링크가 작동하지 않는 이유는 무엇인가요?**

수신거부는 목록을 보호하기 위해 테스트에서 의도적으로 비활성화됩니다. 설정된 플로우에 따라 실제 발송에서는 정상적으로 작동합니다.

**Q: 테스트 오픈과 클릭이 내 분석에 영향을 주나요?**

아니요. 테스트는 검증용이며, 성과 지표는 실제 발송에서 수집됩니다.

**Q: 테스트 이메일이 도착하지 않았는데 먼저 무엇을 확인해야 하나요?**

스팸/정크 메일함을 확인하고, From/서비스 설정을 검증하고, 수신자가 차단되지 않았는지 확인한 후, 다른 이메일 주소로 시도해보세요.

**Q: 커스텀 수신거부 링크가 테스트에서 작동하나요?**

표시는 되지만 테스트에서는 수신거부 작업이 완료되지 않습니다. 실제 발송에서는 설정대로 작동합니다.
