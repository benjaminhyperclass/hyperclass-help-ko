# 하위 계정 런치패드

런치패드(LaunchPad)는 에이전시와 하위 계정(로케이션) 두 레벨에서 안내형 인터랙티브 온보딩 경험을 제공합니다. 초기 설정 작업, 동영상 가이드, 진행 상황 추적을 한 곳에 모아 팀이 더 빠르게 생산성을 높일 수 있도록 도와줍니다. 이 가이드에서는 런치패드의 작동 방식, 역할과 권한에 따른 적응 방식, 그리고 지속적인 설정을 위해 언제든 접근하는 방법을 설명합니다.

## 하위 계정 런치패드란?

런치패드는 필수 작업, 동영상, 바로가기를 한 곳에 모은 온보딩 작업 공간입니다. 사용자가 자신의 역할에 가장 효과적인 설정 작업을 수행할 수 있도록 안내하여 Hyperclass를 자신 있고 효율적으로 구성할 수 있게 해줍니다.

런치패드는 신규 사용자가 처음 로그인할 때 나타나며, 나중에 앱 내에서 다시 열 수 있습니다. 콘텐츠는 사용자 역할과 하위 계정에서 사용 가능한 기능에 따라 조정되어, 각 사용자가 가장 관련성 높은 단계를 볼 수 있도록 합니다.

## 하위 계정 런치패드의 주요 장점

런치패드가 제공하는 결과를 이해하면 팀이 온보딩 노력을 비즈니스 목표와 맞출 수 있습니다. 아래 장점들은 더 빠른 가치 실현 시간, 적은 인수인계, 명확한 책임에 초점을 맞춥니다.

- **간편한 온보딩**: 안내형 체크리스트와 동영상으로 어디서 시작해야 할지 고민하는 시간을 줄입니다.

- **역할별 맞춤 경험**: 에이전시 소유자, 관리자, 일반 사용자가 자신의 권한에 맞는 작업과 팁을 볼 수 있습니다.

- **명확한 진행 상황 추적**: 시각적 표시로 완료된 작업과 다음 할 일을 추적할 수 있습니다.

- **일관된 교육**: 표준화된 경로로 모든 신규 팀원이 동일한 핵심 워크플로우를 학습할 수 있습니다.

- **지원 부담 감소**: 앱 내 도움말과 링크로 일반적인 실수와 반복 질문을 방지합니다.

---

## 결과 중심의 설정 작업

런치패드는 가장 중요한 비즈니스 결과를 중심으로 초기 설정을 그룹화하여, 팀이 각 단계의 "이유"를 이해하고 더 빠르게 운영을 시작할 수 있도록 도와줍니다.

- **기초 설정**: 연락처 생성 또는 가져오기, 비즈니스 세부사항/브랜딩/운영시간 설정, 캘린더 구성으로 작업 공간을 구축합니다.

- **마케팅 및 리드 생성**: 퍼널/폼과 첫 번째 캠페인을 시작하여 리드를 확보하고 참여를 시작합니다.

- **영업 및 대화**: 이메일, SMS, 전화를 활성화하고 파이프라인을 설정하여 대화를 관리하고 거래를 진행합니다.

- **웹사이트 및 수익화**: 도메인 연결, 사이트/퍼널 발행, 결제 및 멤버십 구성을 합니다.

- **전자상거래**: 상품, 결제, 세금, 배송을 구성하여 온라인 판매를 시작합니다.

## 안내형 온보딩 동영상

모든 작업에는 사용자가 실습을 통해 학습할 수 있도록 내장된 가이드가 포함되어 있습니다. 안내형 온보딩 경험은 **모든 하위 계정에서 사용 가능**하며 각 작업 내에서 일관된 도움을 제공합니다.

- **동영상 튜토리얼**: 짧고 집중된 설명; 작업하는 동안 동영상이 떠있도록 **PiP**(Picture-in-Picture)를 활성화하세요.

- **제품 투어 가이드**: 상황별 포인터가 작업을 완료하는 데 필요한 정확한 클릭을 안내합니다.

- **일관된 경험**: 모든 하위 계정에서 안내형 도움말이 포함된 새로운 런치패드에 액세스할 수 있습니다. 실험이 필요하지 않습니다.

---

## 런치패드 작업의 제품 투어 가이드

제품 투어 가이드는 런치패드 작업에서 열리는 앱 내 클릭별 가이드로, 사용자가 설정을 완료하는 정확한 단계를 안내합니다. 화면에 직접 가이드를 오버레이하고, 각 단계를 검증하며, 완료 시 작업을 완료로 표시합니다. 이를 통해 신규 사용자가 페이지를 벗어나지 않고도 성공할 수 있습니다.

- **화면 강조 표시 및 프롬프트**: 다음 클릭, 입력할 필드, 제출할 버튼을 안내합니다.

- **작업 완료**: 투어가 성공적으로 끝나면 런치패드 작업이 자동으로 **완료**로 표시됩니다.

- **재생 및 초기화**: 사용자는 언제든지 투어를 다시 시작하여 구성을 검토하거나 수정할 수 있습니다.

- **권한과 연동**: 투어는 사용자의 현재 역할과 액세스 권한 내에서 실행됩니다. 권한으로 인해 사용자가 단계를 수행할 수 없으면 투어에서 오류가 나타나거나 액세스 권한을 부여받을 때까지 진행이 차단됩니다.

- **안정된 가이드**: 안내형 온보딩 경험(제품 투어 가이드 포함)은 **모든 하위 계정**에서 사용할 수 있습니다.

- **동영상 + 투어의 조화**: 일부 작업은 Picture-in-Picture가 포함된 짧은 동영상을 사용하고, 다른 작업은 제품 투어 가이드를 사용합니다. 두 방식 모두 동일한 안내 경험의 일부입니다.

**에이전시를 위한 모범 사례**

- 사용자가 **전체 투어를 완료**한 후 다른 곳으로 이동하도록 권장하여 작업이 완료로 표시되도록 하세요.

- 투어에서 전제 조건(예: 도메인 또는 전화번호)을 언급하는 경우, 먼저 해당 항목이 하위 계정에서 사용 가능한지 확인하세요.

- 교육 통화에서 런치패드를 사용하세요. **화면 공유 + 투어**는 반복 가능한 온보딩 플레이북을 만듭니다.

## 진행 상황 추적 및 자동 완료

런치패드는 완료된 작업과 다음 할 일을 보여주어 사용자가 집중할 수 있도록 도와줍니다. 작업의 안내 플로우가 완료되면 체크리스트가 그에 따라 업데이트됩니다.

- 기본 설정이 완료되면(예: 결제 연결) 작업이 완료로 표시됩니다.

- 언제든지 런치패드로 돌아가서 업데이트된 상태를 확인하고 다음 단계를 선택하세요.

## 자주 묻는 질문

**Q: 런치패드가 보이지 않아요. 어떻게 액세스하나요?**
먼저 앱 네비게이션이나 전역 검색을 사용해서 런치패드를 찾아보세요. 여전히 보이지 않으면 관리자에게 다음을 확인하도록 요청하세요:

- 하위 계정 설정(Sub-account Settings) → 기능 및 제한(Features and Limits) (런치패드 토글).
- 런치패드에 대한 사용자 역할 권한.
- SaaS 플랜 설정(계정이 SaaS 플랜을 사용하는 경우).

**Q: 팀을 위해 런치패드 콘텐츠를 커스터마이징할 수 있나요?**
브랜딩은 에이전시 설정을 반영할 수 있습니다. 콘텐츠 업데이트와 더 깊은 커스터마이징 옵션은 다양하니, 지원되는 내용은 계정 설정과 교육 리소스를 참조하세요.

**Q: 어떤 역할이 관리자 설정 작업을 완료할 수 있나요?**
관리자 레벨 작업은 관리자 권한이 필요합니다. 필요한 권한이 없는 사용자는 해당 항목을 보거나 완료할 수 없습니다.

**Q: 런치패드를 닫았다가 나중에 다시 불러올 수 있나요?**
네. 세션 중에 닫았다가 나중에 앱 네비게이션에서 다시 열 수 있습니다.

**Q: 진행 상황은 어떻게 추적되나요?**
런치패드는 사용자별로 완료된 항목을 시각적으로 추적합니다. 일부 구성은 기본 설정이 저장되면 완료를 반영하기도 합니다.

**Q: 런치패드가 기존 워크플로우나 자동화에 영향을 주나요?**
아니요. 런치패드는 가이드와 바로가기를 제공하며, 작업을 완료하면서 변경사항을 만들기로 선택하지 않는 한 자동화를 수정하지 않습니다.

**Q: 에이전시와 하위 계정 런치패드에 차이가 있나요?**
네. 에이전시 뷰는 에이전시 레벨 구성과 관리에 초점을 맞추고, 하위 계정 뷰는 특정 로케이션 내의 운영 설정에 초점을 맞춥니다.

**Q: 런치패드가 모바일에서도 사용할 수 있나요?**
최상의 경험을 위해서는 데스크톱에서 핵심 가이드에 액세스할 수 있습니다. 모바일에서의 가용성은 작업에 따라 다를 수 있습니다.
