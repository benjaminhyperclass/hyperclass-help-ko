# Labs 기능 - 완전 가이드

Hyperclass Labs는 사용자에게 새로운 기능과 개발 중인 기능을 미리 체험할 수 있는 기회를 제공합니다. 정식 출시 전에 변경사항을 테스트하고, 피드백을 공유하며, 팀을 미리 준비시킬 수 있어요. 이 가이드는 Labs를 어디서 찾을 수 있는지, 토글이 어떻게 작동하는지, 누가 활성화할 수 있는지, 하위 계정에 어떻게 할당하는지, 그리고 피드백을 어떻게 제출하는지 설명합니다.

# Labs란 무엇인가요?

Labs는 Hyperclass의 얼리 액세스 프로그램으로, 선택된 기능들을 모든 사용자에게 출시되기 전에 미리 켤 수 있는 기능입니다. Labs의 기능들은 개발이 진행되면서 변경될 수 있으며, 여러분의 피드백이 최종 동작 방식과 출시 일정을 결정하는 데 도움이 됩니다. Labs에는 에이전시 계정에 적용되는 항목과 하위 계정에 적용되는 항목이 포함되어 있어요.

Labs를 이해하면 다가오는 변경사항을 통제된 환경에서 평가할 수 있습니다. 테스트 환경에서 기능을 활성화하고, 영향을 검증하며, 피드백을 보내면 출시 과정에서 위험을 줄이고 제품 방향에 영향을 줄 수 있어요.

## Labs의 주요 장점

이러한 장점들은 Labs가 일상 운영을 어떻게 개선하는지, 프로덕션 하위 계정을 보호하면서 개선사항 도입 속도를 어떻게 높이는지에 중점을 둡니다.

- **얼리 액세스**: 정식 출시 전에 다가오는 기능을 사용해보고 프로세스와 교육을 계획할 수 있어요.
- **타겟 배포**: 선택된 하위 계정에만 활성화하여 통제된 파일럿 테스트를 실행할 수 있어요.
- **빠른 피드백 루프**: 직접 피드백(스크린샷/동영상 포함)을 보내 제품 개선에 기여할 수 있어요.
- **변화 준비**: 정식 출시 전에 SOP와 자료를 미리 업데이트하여 중단을 최소화할 수 있어요.
- **위험 감소**: 모든 클라이언트에게 활성화하기 전에 동작, 성능, 호환성을 검증할 수 있어요.

# 에이전시 Labs vs 하위 계정 Labs

### 에이전시 Labs

- **영향**: 에이전시와 하위 계정(할당을 통해) 또는 선택한 특정 하위 계정에 영향을 줍니다.
- **활성화 권한**: 에이전시 소유자/관리자.
- **위치**: 에이전시 뷰 → 설정(Settings) → Labs (에이전시/하위 계정 탭).
- **배포 방식**: 에이전시 레벨에서 토글 ON → 관리(Manage) → 하나 이상의 하위 계정 할당.
- **적합한 용도**: 중앙 관리, 여러 하위 계정에 걸친 통제된 파일럿, 글로벌 정책/도구.

### 하위 계정 Labs

- **영향**: 현재 하위 계정에만 영향을 주며, 다른 계정에는 영향 없음.
- **활성화 권한**: 하위 계정 관리자(그리고 하위 계정 내에서 작업할 때 에이전시 역할).
- **위치**: 하위 계정 뷰 → 설정(Settings) → Labs.
- **배포 방식**: 해당 하위 계정에서 직접 토글 ON(기능에서 요구하지 않는 한 할당 불필요).
- **적합한 용도**: 빠르고 위험이 낮은 시험, 팀/클라이언트별 검증.

## Labs 기능을 이해하는 방법

모든 Labs 기능은 "카드" 형태로 나타납니다. 각 요소를 알면 기능을 자신있게 활성화하고 이해관계자에게 일정을 전달할 수 있어요.

각 카드에는 일반적으로 다음이 포함됩니다:

- **제목**: 기능 이름.
- **설명**: 기능이 제공하는 변경사항이나 능력.
- **X일 후 출시**: 정식 출시까지 예상 일수(변경 가능).
- **토글 ON/OFF**: 현재 환경에서 기능을 활성화/비활성화.
- **피드백 제출**: 코멘트, 이슈, 요청을 제품 팀에 직접 전송.

## 표시 vs 활성화 제어 (하위 계정 Labs 기능)

Labs 제어는 두 가지 다른 설정을 사용합니다:

- **표시**: Labs 기능이 하위 계정 뷰 → 설정(Settings) → Labs에 나타나는지 여부.
- **활성화**: 해당 하위 계정에서 기능이 켜져 있는지 여부.

**중요**: Unlimited와 Pro 플랜은 표시 및 활성화 기본값과 하위 계정별 표시 및 활성화 관리를 지원합니다. 
Starter 플랜은 Labs 기능을 기본적으로 표시하며 표시 또는 활성화 제어를 보여주지 않습니다.

![표시 vs 활성화 제어](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064178284/original/_lyXouZNCWkLXYkZks0wq-x1eYYU5ZnzTg.png?1770209216)

기능이 표시되지만 활성화되지 않을 수 있습니다. 이는 기능을 활성화하지 않고도 발견할 수 있게 소개하는 데 도움이 됩니다.

### **기본 표시 및 재정의 (에이전시 뷰)**

해당 플랜에서 에이전시 관리자는 다음을 할 수 있습니다:

- 새 Labs 기능의 기본 표시 설정(표시 또는 숨김).
- 필요할 때 하위 계정별로 표시 재정의.

## Labs 사용 방법

올바른 워크스페이스에서 Labs를 사용하는 것이 중요합니다. 에이전시 뷰는 에이전시 레벨 기능과 하위 계정 기능 할당 도구를 제공하며, 하위 계정 뷰는 로컬에서 활성화할 수 있는 기능을 보여줍니다.

### 1: 에이전시 전용 기능 활성화 (에이전시 뷰)

- 에이전시 계정에 로그인하세요.
- 설정(Settings) → Labs → 에이전시 탭으로 이동하세요.
- 기능 카드를 찾고 설명과 출시 일정을 검토하세요.
- 토글을 ON으로 설정하세요.

![에이전시 기능 활성화](https://jumpshare.com/share/txIBztOpI4GLQo1U1lUA+/GIF+Recording+2025-08-12+at+12.03.52+AM.gif)

### 2: 에이전시 할당을 통한 하위 계정 기능 활성화

참고: 일부 하위 계정 기능은 사용하기 전에 에이전시가 권한을 부여해야 합니다. 관리(Manage) 제어를 통해 에이전시 관리자/소유자가 어떤 하위 계정이 얼리 액세스를 받을지 선택할 수 있습니다.

- 에이전시 계정에 로그인하세요.
- 설정(Settings) → Labs → 하위 계정 탭으로 이동하세요.
- 기능 카드를 찾고 설명과 출시 일정을 검토하세요.
- 관리(Manage)를 클릭하고 표시 및/또는 활성화 액세스를 제공할 하위 계정을 선택하세요.
- 저장(Save)을 클릭하세요.
- 할당된 하위 계정으로 전환하고 관련 영역을 새로고침하여 기능이 나타나거나 활성화되었는지 확인하세요.

![하위 계정 기능 할당](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064179682/original/ypYnjb8wnXAoH9HBaTUlEcCaU15WlA0ACw.gif?1770209920)

### 3: 하위 계정 기능 직접 활성화

일부 하위 계정 기능은 에이전시 할당 없이 하위 계정 내에서 직접 활성화할 수 있습니다.

- 대상 하위 계정에서 설정(Settings) → Labs를 여세요.
- 기능 카드를 찾고 토글을 ON으로 설정하세요.
- 변경사항을 보려면 관련 모듈(빌더, CRM 등)을 새로고침하거나 다시 여세요.

![하위 계정 직접 활성화](https://jumpshare.com/share/5KUEORHRlnYiDNfFANZw+/GIF+Recording+2025-08-12+at+12.16.19+AM.gif)

## 피드백 제출

Labs에서 제공하는 피드백은 제품 팀에게 직접 전달되어 수정과 개선의 우선순위를 정하는 데 도움이 됩니다. 고품질 피드백은 모든 사람을 위한 결과를 가속화합니다.

피드백을 제출하려면:

- 설정(Settings)(에이전시 또는 하위 계정)을 클릭하세요.
![설정 메뉴](https://jumpshare.com/share/MiaKBmexb4ZQb4Rlwcqz+/Screen+Shot+2025-08-12+at+12.28.49+AM.png)

- Labs를 클릭하세요.
- 관련 기능 카드에서 피드백 제출(Submit Feedback)을 클릭하세요.
![피드백 제출 버튼](https://jumpshare.com/share/DXKBF80DUqjfywcUkzFE+/Screen+Shot+2025-08-12+at+12.30.48+AM.png)

- 제목과 명확한 메시지를 추가하세요.
- 선택적으로 사용 사례나 이슈를 설명하는 스크린샷이나 동영상을 업로드하세요.
- 제출(Submit)을 클릭하세요.

![피드백 양식](https://jumpshare.com/share/nK7g4skJbDR3xcsDLQco+/Screen+Shot+2025-08-12+at+12.36.20+AM.png)

## 자주 묻는 질문

**Q: 누가 Labs에 액세스할 수 있나요?**
A: 모든 에이전시 관리자와 하위 계정 관리자, 그리고 설정 보기 및 관리(View & Manage Settings) 권한을 가진 사용자들이 액세스할 수 있습니다.

**Q: Labs가 보이지 않아요. 무엇을 확인해야 하나요?**
A: 올바른 환경(에이전시 vs. 하위 계정)에 있는지, 그리고 설정에 액세스할 권한이 있는 역할인지 확인하세요. 여전히 Labs가 보이지 않으면 계정 관리자에게 문의하세요.

**Q: Labs 기능을 나중에 비활성화할 수 있나요?**
A: 네, 활성화했던 곳에서 기능의 토글을 OFF로 설정하면 됩니다.

**Q: "X일 후 출시"는 정확히 무엇을 의미하나요?**
A: 정식 출시까지의 예상 일정이며, 피드백과 개발 상황에 따라 변경될 수 있습니다.

**Q: 버그 신고와 기능 요청은 어디에 해야 하나요?**
A: 제품 개선 의견과 작은 이슈는 카드의 피드백 제출(Submit Feedback)을 사용하세요. 팀을 방해하는 긴급한 지원이 필요한 경우에는 표준 지원/티켓 워크플로우를 따르세요.

**Q: 하위 계정에서 특정 Labs 기능이 보이지 않는 이유는 무엇인가요?**
A: 해당 플랜에서 에이전시가 기능을 **숨김**(표시)으로 설정했을 수 있습니다. 에이전시 소유자/관리자에게 기능의 표시 설정을 변경해달라고 요청한 다음 하위 계정을 새로고침하세요.

**Q: 모든 플랜이 표시 제어 기능을 가지고 있나요?**
A: 아니요. **Unlimited와 Pro**는 표시 기본값과 하위 계정별 표시 관리를 지원합니다. **Starter**는 Labs 기능을 기본적으로 표시하며 표시 제어를 보여주지 않습니다.
