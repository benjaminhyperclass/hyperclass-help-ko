# 스냅샷: 새로운 디자인 시스템(UI) 소개

Hyperclass의 스냅샷(Snapshots) 영역이 새로운 디자인 시스템(UI)으로 업그레이드되어 더 빠른 네비게이션, 통합된 스타일링, 그리고 강력한 데이터 필터링 도구를 제공합니다. 이제 에이전시의 모든 스냅샷을 더 쉽게 생성, 새로고침, 배포, 감사할 수 있습니다.

## **스냅샷 새 디자인 시스템(UI)이란?**

새 디자인 시스템은 Hyperclass의 차세대 인터페이스 프레임워크로, 이제 모든 스냅샷 화면에 적용됩니다. 이 UI로 마이그레이션함으로써 스냅샷은 일관된 룩앤필, 향상된 성능, 그리고 새로운 필터링, 정렬, 검색 유틸리티를 갖추게 되어 대량의 스냅샷 데이터를 더 효율적으로 찾고 관리할 수 있게 됩니다.

---

## **스냅샷 새 디자인 시스템의 주요 장점**

이번 업그레이드는 실질적인 워크플로우 개선사항을 제공합니다:

- **일관된 사용자 경험** — 모든 스냅샷 탭에서 동일한 스타일링, 아이콘, 네비게이션 제공

- **더 빠른 탐색** — 즉시 검색, 컬럼 정렬, 다중 선택 필터로 올바른 스냅샷을 찾는 시간 단축

- **더 높은 데이터 신뢰도** — 캘린더 필터로 정확한 날짜 범위 내에서 생성, 가져오기, 공유된 스냅샷을 빠르게 감사 가능

- **더 매끄러운 작업 실행** — 명확하게 그룹화된 액션(생성, 새로고침, 배포, 히스토리, 설정)으로 클릭 수와 혼란 최소화

- **미래 대비 기반** — 새 디자인 시스템이 지속적으로 롤아웃되므로 여기서 익혀두면 다른 곳의 향후 UI 업데이트에도 대비 가능

---

## **새 UI의 핵심 스냅샷 기능**

주요 워크플로우 버튼들이 이제 하나의 고정 위치 액션 바에 모여 스크롤을 줄여줍니다:

- **스냅샷 생성(Create Snapshot)** — 모든 하위 계정에서 새 템플릿 생성

![스냅샷 생성 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055722087/original/6FBdCwZIEup75UVZd8P3Rlqt36cZtNVD0A.png?1760102770)

- **새로고침(Refresh)** — 기존 스냅샷을 소스의 최신 자산과 설정으로 업데이트

![새로고침 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055721889/original/28XOuFX6326L52WPJzSRI2dELhUfFKVQEw.png?1760102694)

- **배포(Push)** — 스냅샷(또는 선택한 자산만)을 하나 이상의 하위 계정에 배포

![배포 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055719859/original/4PxBH4Yp9OEvrKGfsiLadz6MJacqXsjeBg.png?1760102128)

- **히스토리(History)** — 각 스냅샷을 누가 생성, 새로고침, 배포했는지 포함한 전체 감사 기록 열기

![히스토리 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055719904/original/ETVI_dpRg525t-UHEI153dk2MScL_hin8w.png?1760102155)

- **설정(Settings)** — 스냅샷 이름 변경, 포함된 자산 편집, 배포 대상 조정

![설정 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055720985/original/zLnjpVkHinOeDUvJrBeTIgt_lV1Ez8NKwQ.png?1760102606)

## **고급 필터링, 정렬 및 검색**

향상된 데이터 도구들이 모든 스냅샷 목록 위에 나타납니다:

- **검색 바** — 스냅샷 이름이나 ID의 일부분을 입력하면 즉시 목록이 좁혀집니다

![검색 바](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055720743/original/2liRXXg4gdEEuJkYQLi6Lk8zYUUJ2-fyhg.png?1760102493)

- **컬럼 정렬** — 헤더(이름, 크기, 마지막 업데이트 등)를 클릭하면 오름차순/내림차순으로 재정렬

![컬럼 정렬](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055720339/original/o8yrpOFK5FhrsstRyaQnbsNXFSEocyDrbg.png?1760102350)

- **캘린더 선택기** — 절대 또는 상대 날짜 범위(오늘, 최근 30일, 사용자 지정)를 선택하여 가져오기, 공유, 배포를 찾기

![캘린더 선택기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055720051/original/mGUpfaNLsfJOGZ2pnD77SRhMLHfNLjDV2A.png?1760102210)

이러한 컨트롤들은 함께 사용할 수 있으며, Hyperclass는 돌아올 때 마지막 필터 설정을 기억합니다.

## **일관된 디자인 시스템 요소**

레이블, 아이콘, 버튼, 간격이 이제 다른 개선된 Hyperclass 영역들(예: 인보이스, 활동 섹션)과 동일하게 맞춰져서 네비게이션이 자연스러워지고 새 직원의 교육 시간이 단축됩니다.

## **자주 묻는 질문**

**Q: 새 UI가 스냅샷에 포함되는 내용을 바꾸나요?**

아니요, 자산 범위는 동일하게 유지됩니다. 인터페이스와 데이터 도구만 변경되었습니다.

**Q: 기존 UI로 되돌릴 수 있나요?**

기존 UI는 단계적으로 폐지될 예정입니다. 전환 기간 동안 "레거시 뷰(Legacy View)" 토글을 찾을 수 있지만, 지금 워크플로우를 마이그레이션할 것을 권장합니다.

**Q: 기존의 세분화된 권한이 여전히 적용되나요?**

네, Settings(설정) → Team(팀)에서 설정한 사용자 권한이 생성, 배포, 새로고침 등을 계속 관리합니다.

**Q: 필터링이 다른 사용자에게 영향을 주나요?**

필터는 사용자별 및 세션별로 적용되며 팀원의 목록 보기를 변경하지 않습니다.

**Q: 키보드 단축키가 있나요?**

⌘/Ctrl + F로 검색 바에 포커스를 맞추고, 포커스된 상태에서 ↑/↓ 키로 목록 행을 탐색할 수 있습니다.

**Q: 한 번에 얼마나 많은 스냅샷을 표시할 수 있나요?**

목록이 가상화되어 있어서 수천 개의 항목이 있어도 성능이 부드럽게 유지됩니다. 하드 캡은 없습니다.

**Q: 날짜 필터가 내 로컬 시간대를 사용하나요?**

네, 캘린더 필터링은 사용자 프로필에 설정된 시간대를 준수합니다.
