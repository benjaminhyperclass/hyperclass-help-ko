# 스냅샷(Snapshots) - 개요

커스텀 리포트를 스냅샷에 추가하여 하위 계정 간에 리포팅 템플릿을 패키징하고 전송할 수 있습니다. 이를 통해 다시 작성할 필요 없이 모든 클라이언트 계정에서 일관된 분석 환경을 제공할 수 있어요. 이 기능으로 온보딩이 간소화되고, 복제 속도가 빨라지며, 모든 클라이언트가 정확하고 브랜드가 적용된 인사이트를 받을 수 있습니다.

## 스냅샷의 커스텀 리포트란?

스냅샷의 커스텀 리포트를 사용하면 에이전시가 **커스텀 리포트 빌더(Custom Report Builder)**에서 만든 리포팅 템플릿을 스냅샷의 일부로 포함할 수 있어요. 해당 스냅샷을 다른 하위 계정에 로드하면, 해당 리포트들이 **리포팅(Reporting) → 커스텀 리포트(Custom Reports)**에 표시되어 검토, 예약, 배포가 가능합니다.

이 기능을 통해 가장 성과가 좋은 리포팅 레이아웃을 계정 간에 손쉽게 공유할 수 있어 시간을 절약하고 브랜드 일관성을 유지할 수 있어요.

---

## 주요 이점

- **빠른 온보딩:** 미리 구성된 분석 대시보드를 몇 분 내에 제공
- **일관된 리포팅:** 하위 계정 전반에 걸쳐 동일한 지표, 레이아웃, KPI 보장
- **오류 감소:** 수동 리포트 재생성으로 인한 부정확성 방지
- **중앙 집중식 관리:** 마스터 스냅샷을 한 번 업데이트하여 클라이언트 전체에 다시 로드
- **클라이언트 유연성:** 각 계정에서 독립적으로 예약 및 수신자 조정 가능

## 스냅샷 생성 시 커스텀 리포트 추가하기

스냅샷을 생성하거나 로드할 때 **커스텀 리포트**를 스냅샷 자산 목록의 일부로 선택할 수 있어요. 로드되면 대상 하위 계정에 편집 가능한 템플릿으로 나타나며, 예약하거나 커스터마이즈할 준비가 됩니다.

- **Agency View(에이전시 뷰) → Account Snapshots(계정 스냅샷)**로 이동
- **Create New Snapshot(새 스냅샷 생성)**을 클릭하고 **소스 하위 계정**을 선택
- **Reporting(리포팅) → Custom Reports(커스텀 리포트)** 섹션을 확장
- 포함할 리포트를 선택한 후 **Save Snapshot(스냅샷 저장)** 클릭

## 다른 하위 계정에 커스텀 리포트 로드하기

스냅샷에서 커스텀 리포트를 로드하면 리포팅 템플릿을 다른 하위 계정에 즉시 복사하여 모든 클라이언트에서 일관된 분석 환경을 보장할 수 있어요.

- **Agency View(에이전시 뷰) → Sub-Accounts(하위 계정) → Actions(작업) → Load Snapshot(스냅샷 로드)**로 이동
- 커스텀 리포트가 포함된 스냅샷을 선택
- 자산 체크리스트에서 **Custom Reports(커스텀 리포트)**가 선택되었는지 확인
- 확인하여 로딩 시작
- 완료 확인하는 **벨 아이콘 알림**을 기다림
- 대상 계정에서 **Reporting(리포팅) → Custom Reports(커스텀 리포트)**를 열어 가져온 리포트 확인

## 중요 사항 및 제한사항

- **권한:** 스냅샷을 생성하거나 로드하려면 에이전시 레벨 권한이 있어야 합니다
- **템플릿 전송만:** 예약 및 수신자 목록은 포함되지 않습니다
- **데이터 소스:** 대상 하위 계정에 연동(예: Google Ads)이 없으면 해당 위젯은 다시 연결할 때까지 "데이터 없음"으로 표시됩니다
- **플랜 이용 가능성:** 커스텀 리포트는 **$297(제한된)** 및 **$497+(무제한)** 플랜에서 이용 가능합니다

## 자주 묻는 질문

**Q: 리포트 예약과 수신자가 자동으로 전송되나요?**

아니요. 각 하위 계정에서 예약과 수신자를 수동으로 설정해야 합니다.

**Q: 개인 대시보드도 스냅샷의 일부로 복사되나요?**

아니요. 커스텀 리포트만 전송됩니다. 개인 대시보드는 소스 계정에 남아있어요.

**Q: 가져온 후 일부 위젯에 "데이터 없음"이 표시되는 이유는?**

대상 하위 계정에 동일한 연결된 데이터 소스가 없을 수 있어요. 이를 해결하려면 다시 연결하세요.
