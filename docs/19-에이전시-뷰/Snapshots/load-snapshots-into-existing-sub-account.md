# **기존 계정에 스냅샷 로드란?**

기존 계정에 스냅샷을 로드한다는 것은 이미 활성 상태인 하위 계정에 워크플로우, 퍼널, 이메일 등과 같은 미리 구축된 Hyperclass 자산을 가져오는 것을 의미합니다. 이 방법을 통해 에이전시는 수동으로 다시 만들지 않고도 표준화된 시스템을 고객에게 빠르게 배포할 수 있습니다. 스냅샷으로 새 계정을 만드는 것과 달리, 이 방법은 기존 고객 데이터를 보존하면서 선택한 자산을 추가하거나 업데이트합니다.

## 기존 계정에 스냅샷을 로드하는 주요 이점

스냅샷을 로드하는 것은 여러 고객에게 자산을 복제하는 전략적인 방법입니다.

- **효율성**: 워크플로우, 폼 및 기타 자산을 재사용하여 설정 속도를 높입니다
- **일관성**: 하위 계정이 동일한 구조와 프로세스를 따르도록 보장합니다
- **유연성**: 어떤 자산을 가져올지, 어떤 기존 자산을 덮어쓸지 제어할 수 있습니다
- **충돌 경고**: 작업을 수행하기 전에 무엇이 덮어써질지 미리 확인할 수 있습니다

## **기존 하위 계정에 스냅샷을 로드하는 방법**

기존 하위 계정에 스냅샷을 가져오려면 다음 과정을 따르세요.

### 1단계: "클라이언트 관리"로 이동

- 에이전시 대시보드에서 왼쪽 메뉴를 사용하여 Sub-Accounts(하위 계정)으로 이동합니다
- 아래로 스크롤하거나 검색창을 사용하여 스냅샷을 추가할 하위 계정을 찾습니다
- 하위 계정 오른쪽에 있는 **3점 아이콘**을 클릭합니다. 팝업 메뉴에서 Manage Client(클라이언트 관리)를 선택합니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047309772/original/yA2ym7DzLDFboqlwe9N7FLe4fsS03Yi0-A.png?1748362317)

### **2단계:** 스냅샷 선택 및 로드

- 오른쪽 상단의 파란색 Actions(작업) 버튼을 클릭합니다
- 드롭다운에서 Load Snapshot(스냅샷 로드)을 선택합니다

참고: "Load Snapshot"이 보이지 않는다면, 에이전시 대시보드에서 **User(사용자) > Roles and Permissions(역할 및 권한)** 아래에서 Snapshots(스냅샷) 접근 권한이 켜져 있는지 확인하세요

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047310271/original/dEt-CxBAyDMY49OqA-VBjOTsSFahomz7Ig.png?1748362867)

### **3단계:** 스냅샷 선택

- "Load Snapshot" 팝업에서 Pick a Snapshot(스냅샷 선택) 드롭다운을 클릭합니다
- 하위 계정에 로드할 스냅샷을 선택합니다
- 파란색 Proceed(진행) 버튼을 클릭합니다

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047310396/original/_QPoTUXoOthgaiTP_RQYOdMZC1636crwAA.png?1748363022)

### **4단계:** 자산 선택

- 체크박스를 사용하여 가져올 자산을 선택합니다

사용 가능한 모든 자산을 가져오려면 **Select All(모두 선택)**을 클릭합니다

- 카테고리별로 가져오려면 필요한 섹션만 체크합니다
- **+ 아이콘**을 클릭하여 카테고리를 확장하고 개별 자산을 선택합니다
- 완료되면 파란색 **Proceed(진행)** 버튼을 클릭합니다

참고: 스냅샷에 웨비나가 포함된 경우, 가져오기 시 웨비나 페이지가 복제되고 해당 웨비나 세션이 자동으로 생성됩니다. 관련 웨비나 데이터는 가져오기 중에 복원됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047310630/original/oNBseK_rnYitl1-Cs9Z2M2UdKGWyBesoHA.png?1748363231)

#### **LinkedIn 광고 캠페인 (마케팅 자산)**

스냅샷에 LinkedIn 캠페인이 포함된 경우, 대상 하위 계정으로 가져올 수 있습니다:

자산 목록에서 Ad Campaigns – LinkedIn을 선택합니다.

가져오기 전에 표준 충돌을 검토하는 충돌 확인 단계로 계속 진행합니다.

가져온 LinkedIn 캠페인은 항상 Draft(초안) 상태로 생성됩니다.

대상 하위 계정에 LinkedIn이 연결되지 않은 경우에도 가져오기가 완료되고 캠페인이 초안으로 생성됩니다.

Lead Gen Forms(리드 생성 폼)은 가져오지 않습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067812094/original/9XBFQOV39r2KLgmZan4wsUlkpBmV6py5RQ.png?1774530447)

### **5단계:** 충돌 해결

- 플랫폼이 기존 자산과 스냅샷의 자산 간 충돌을 확인합니다. 각 충돌에 대해 두 가지 옵션이 있습니다:

**Override(덮어쓰기)**: 기존 하위 계정 항목을 스냅샷 버전으로 교체합니다

- **Skip(건너뛰기)**: 현재 버전을 유지하고 충돌하는 스냅샷 항목을 제외합니다
- 검토 후 파란색 **Proceed(진행)** 버튼을 클릭합니다

참고: 충돌 해결은 웨비나에도 다른 자산 유형과 동일하게 적용됩니다. 기존 웨비나 자산을 교체하려면 Override를, 현재 버전을 유지하려면 Skip을 선택하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047310944/original/tB_gDphi0N9_RBN0qHqvTOTvtbEKn36U9g.png?1748363508)

### **6단계**: 가져오기 확인

- 변경 사항을 승인하기 위해 "confirm" 단어를 입력하라는 요청을 받습니다
- "confirm"을 입력한 후 파란색 Proceed(진행) 버튼을 클릭합니다
- 성공 알림이 나타나며 스냅샷이 하위 계정으로 푸시되고 있음을 확인합니다. 모든 정보가 전송되면 이메일 알림을 받게 됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047311036/original/pJu4UOgflq4e4RbBobEBNPFi7nn1St_fiQ.png?1748363608)

## **자주 묻는 질문**

**Q: 스냅샷을 로드하면 현재 데이터가 삭제되나요?**  
아니요. 충돌 확인 중에 덮어쓰기를 선택한 경우에만 데이터가 교체됩니다.

**Q: 스냅샷에 무엇이 들어있는지 미리 볼 수 있나요?**  
직접적으로는 불가능합니다. 하지만 선택적으로 로드하고 과정 중에 자산을 건너뛸 수 있습니다.

**Q: 실수로 자산을 덮어썼다면 어떻게 되나요?**  
이 작업은 되돌릴 수 없습니다. 항상 충돌 목록을 신중하게 검토하세요.

**Q: 같은 하위 계정에 여러 스냅샷을 로드할 수 있나요?**  
네, 같은 하위 계정에 하나 이상의 스냅샷을 로드할 수 있습니다. 스냅샷을 로드할 때마다 동일한 과정을 거치게 됩니다. 겹치는 자산이 충돌을 일으킬 수 있으므로 의도하지 않은 덮어쓰기를 피하기 위해 선택 사항을 신중하게 검토하세요.
