# 스냅샷의 연결된 자산 관리

스냅샷(Snapshot) 배포에 포함될 항목을 더욱 세밀하게 제어할 수 있습니다! 이제 각 스냅샷 내부에서 자산을 확인하고, 연결하고, 연결을 해제할 수 있습니다. 이를 통해 스냅샷 관리가 더 쉽고, 빠르고, 유연해집니다.

## **스냅샷 자산 연결이란?**

스냅샷 자산 연결 기능을 사용하면 스냅샷을 배포할 때 포함될 자산(워크플로우, 퍼널, 폼 등)을 수동으로 제어할 수 있습니다. 모든 항목을 자동으로 포함하는 대신, 이제 연결된 자산과 연결되지 않은 자산을 완전히 파악할 수 있어 각 스냅샷 배포에 정확히 무엇이 포함될지 직접 결정할 수 있습니다.

## **스냅샷 자산 연결의 주요 장점**

자산 포함 여부를 관리하여 스냅샷을 쉽게 간소화하고 맞춤화할 수 있습니다. 이 기능은 정확성을 높이고 하위 계정(Sub-account)의 불필요한 요소를 방지합니다.

- 배포하려는 자산만 선택 가능
- 오래되거나 사용하지 않는 무관한 자산이 복사되는 것을 방지
- 활성화된(연결된) 자산과 비활성화된(연결 해제된) 자산을 한눈에 확인
- 더 깔끔한 배포 보장 및 수동 정리 작업 감소
- 스냅샷 관리를 더욱 확장 가능하고 반복 가능하게 만듦
- 클라이언트 또는 내부 계정으로 배포되는 내용에 대한 신뢰도 증가

## **스냅샷에서 연결된 자산 관리 방법**

업데이트된 스냅샷 인터페이스를 사용하여 포함될 자산을 제어하는 방법을 단계별로 알아보세요.

- `Account Snapshots(계정 스냅샷)`로 이동
- `My Snapshots(내 스냅샷)` 클릭
- 관리하려는 스냅샷을 찾아 **점 세 개 메뉴**(추가 작업) 클릭
- `Edit Snapshot(스냅샷 편집)` 선택
- 새로운 UI를 사용하여 자산을 검토하고 토글:
  - **활성화된 자산**은 배포됩니다
  - **비활성화된 자산**은 포함되지 않습니다
- 선택이 완료되면 **저장** 클릭

## **기능 위치**

새로운 자산 제어 인터페이스에 빠르게 접근하려면 다음 경로를 사용하세요.

**경로**: `Account Snapshots(계정 스냅샷)` → `My Snapshots(내 스냅샷)` → `More Actions(추가 작업)` (점 세 개) → `Edit Snapshot(스냅샷 편집)`

## **이 기능이 중요한 이유**

이번 업데이트 이전에는 스냅샷 배포가 종종 "전부 아니면 전무" 방식이었습니다. 이로 인해 실수가 발생할 여지가 있었고 정리 단계가 추가로 필요했습니다. 이제 새로운 제어 기능으로 정확히 필요한 것만 배포할 수 있습니다. 시간을 절약하고, 혼란을 줄이며, 여러 계정이나 템플릿을 관리하는 에이전시의 정확성을 높입니다.

새 클라이언트를 런칭하거나 템플릿 라이브러리를 관리할 때, 이 기능은 스냅샷을 필요한 만큼 간결하고 깔끔하게 유지할 수 있도록 보장합니다.

## **자주 묻는 질문**

**Q: 스냅샷이 배포된 후에 자산을 편집할 수 있나요?**

배포된 스냅샷을 직접 편집할 수는 없지만, 새 버전을 만들거나 변경 사항으로 업데이트할 수 *있습니다*.

**Q: 자산 연결을 해제하면 기존 하위 계정에서 삭제되나요?**

아니요. 연결 해제는 향후 스냅샷 배포에만 영향을 미칩니다. 이미 배포된 자산에는 영향을 주지 않습니다.

**Q: 시스템에서 제공하는 스냅샷에도 이 기능을 사용할 수 있나요?**

이 기능은 현재 사용자가 만든 스냅샷에서만 사용할 수 있습니다. 시스템 스냅샷은 편집할 수 없습니다.

**Q: 연결된 자산을 편집한 후 변경사항을 저장해야 하나요?**

네. 자산 선택을 확정하려면 반드시 **저장**을 클릭해야 합니다.

**Q: 편집할 때 특정 자산을 필터링하거나 검색할 수 있나요?**

네. UI에는 필요한 항목을 빠르게 찾을 수 있도록 필터와 검색 도구가 포함되어 있습니다.
