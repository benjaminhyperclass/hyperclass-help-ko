# 스냅샷 공유 방법

# **스냅샷 공유란 무엇인가요?**

스냅샷 공유란 다른 에이전시나 하위 계정이 여러분의 Hyperclass 계정에서 퍼널, 워크플로우, 폼, 템플릿 등의 자산 패키지를 가져올 수 있도록 안전한 링크(또는 초대)를 생성하는 과정입니다. 공유 방식을 이해하면 올바른 링크 유형을 선택하고, 지적 재산을 보호하며, 온보딩을 간소화할 수 있습니다.

- 공유를 통해 수신자는 스냅샷을 자신의 에이전시/하위 계정으로 **가져올** 수 있습니다.
- 일부 링크 유형은 **일회용**이고, 다른 것들은 **영구적**이거나 특정 수신자로 **제한됩니다**.
- 자산 보호 스냅샷은 수신자가 보호된 콘텐츠를 복제하거나 재공유하는 것을 방지합니다.

## 스냅샷 공유의 주요 장점

- **빠른 배포**: 검증된 퍼널, 자동화, 설정을 여러 계정에 몇 분 만에 배포할 수 있습니다.
- **일관성**: 표준화된 빌드를 배포하여 설정 편차를 제거합니다.
- **제어**: 각 시나리오에 맞는 링크 유형(일회용, 영구, 제한)을 선택할 수 있습니다.
- **보호**: 자산 보호 스냅샷으로 고가치 자산을 잠글 수 있습니다.
- **SaaS 확장**: 미리 선택한 기본 스냅샷으로 새로운 SaaS 가입자를 자동 프로비저닝합니다.

## **스냅샷 공유 방법**

스냅샷 목록에서 다음 단계를 따라 올바른 유형의 공유 링크를 생성하고 수신자에게 배포하세요.

참고: 연결된 하위 계정에 업데이트를 푸시하기 전에 [스냅샷을 새로 고침](refresh-or-update-snapshots.md)하는 것이 중요합니다. 업데이트를 푸시한 후, 어떤 업데이트를 가져올지 선택할 수 있습니다.

- Agency View(에이전시 뷰) → Account Snapshots(계정 스냅샷) > My Snapshots(내 스냅샷)으로 이동합니다.

![스냅샷 메뉴 접근](https://jumpshare.com/share/QmzbOAYgy2d10wuvVx9A+/Screen+Shot+2025-12-10+at+8.01.02+PM.png)

- 스냅샷을 찾아 ⋯ 세 점을 클릭 → Share Snapshot(스냅샷 공유)을 선택합니다.

![스냅샷 공유 메뉴](https://jumpshare.com/share/ogUl0SQhW4w2Fy8vph9O+/Screen+Shot+2025-12-10+at+8.06.36+PM.png)

- **공유 옵션**을 선택합니다:

Permanent Link(영구 링크): 반복적인 가져오기를 위한 재사용 가능한 링크를 생성합니다.

- One-Time Link(일회용 링크): 직접 전송하는 일회용 링크를 생성합니다.
- Email Link(이메일 링크): 이메일을 통해 전송되는 일회용 링크입니다.
- Agency-Restricted(에이전시 제한): 파트너 에이전시의 관계 번호를 입력합니다.
- Sub-Account Restricted(하위 계정 제한): 정확한 대상 하위 계정을 선택합니다.

*팁: 이 스냅샷을 마켓플레이스에서 판매하는 것이 목표라면, Account Snapshots(계정 스냅샷)에서 ⋯ → Sell on Marketplace(마켓플레이스에서 판매)를 사용하여 안내된 목록 생성 흐름을 시작하고 마켓플레이스 링크를 즉시 받을 수 있습니다. Marketplace Share Link(마켓플레이스 공유 링크)는 마켓플레이스 목록에 사용되는 백엔드 URL 개념입니다.*

![스냅샷 공유 과정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064696055/original/FFdPVRdO1QCb0YJu2yo0CaeVfGr8Ss_JUw.gif?1770819813)

- Assets Protected(자산 보호)를 토글하여 보호된 콘텐츠의 복사/재공유를 방지합니다. (선택사항)
- Get Link(링크 생성)를 클릭합니다.
- 링크를 복사하여 의도된 수신자에게 공유합니다.

![링크 복사](https://jumpshare.com/share/KAsxVvxJwfGqQHYm728A+/Screen+Shot+2025-12-10+at+9.35.56+PM.png)

## 링크 유형 및 사용 시점

올바른 링크 유형을 선택하면 우발적인 재사용을 방지하고, 노출을 제한하며, 실제 공유 요구사항에 맞출 수 있습니다.

| 링크 유형 | 가져올 수 있는 대상 | 재사용/동작 |
|---------|---------------|-----------|
| Permanent Link(영구 링크) | URL을 가진 모든 사람 | 무제한 가져오기; 수동으로 취소하거나 스냅샷을 삭제할 때까지 활성 상태 유지 |
| One-Time Share Link(일회용 공유 링크) | URL을 가진 모든 사람 (첫 번째 가져오기까지) | 한 번의 성공적인 가져오기 후 만료; 새로운 구매자/고객마다 새 링크를 생성해야 함 |
| Email Link(이메일 링크) | 이메일 수신자 | 일회용 링크와 기능적으로 동일하지만 이메일로 전달; Hyperclass에 로그인한 모든 에이전시 소유자가 이메일에서 사용 가능 |
| Agency-Restricted Link(에이전시 제한 링크) | 관계 ID를 나열한 에이전시만 | 무제한 가져오기이지만 지정된 에이전시 계정으로만 제한; 여러 관계 번호를 나열할 수 있음 |
| Sub-Account Restricted Link(하위 계정 제한 링크) | 지정된 정확한 하위 계정만 | 단일 대상; 해당 특정 하위 계정만 링크에서 가져오기 가능 (에이전시 간 재사용 불가) |
| Marketplace Share Link(마켓플레이스 공유 링크) | Hyperclass App 마켓플레이스 목록을 통한 모든 구매자 | 마켓플레이스 "앱"의 백엔드 URL로 사용; 기본적으로 IP 보호가 켜져 있어 자산 재판매가 어려움; 가져오기는 마켓플레이스 라이선싱으로 제어됨 |

## **자주 묻는 질문**

**Q: 모든 Hyperclass 요금제($97/$297/$497)에서 모든 링크 유형을 사용할 수 있나요?**

네, 모든 요금제 등급에서 모든 링크 유형을 사용할 수 있습니다.

**Q: 내 에이전시 외부의 에이전시와 스냅샷을 공유한 후 나중에 업데이트를 하면, 이전에 다른 에이전시와 공유한 스냅샷에 이러한 변경사항이 반영되거나 자동으로 업데이트되나요?**

아니요, 공유 옵션(1-4)에 관계없이 스냅샷은 항상 특정 시점의 특정 하위 계정 복제본으로 간주됩니다. 따라서 스냅샷에 대한 모든 업데이트는 이전에 공유된 스냅샷에 자동으로 반영되지 않습니다.

**Q: 내 에이전시 내의 하위 계정을 위해 스냅샷을 어떻게 업데이트할 수 있나요?**

에이전시 내 하위 계정의 스냅샷을 업데이트하려면 다음 단계를 따라야 합니다:

- 먼저 원하는 변경사항으로 스냅샷을 업데이트합니다.
- 다음으로, 업데이트된 스냅샷을 에이전시 내의 연결된 하위 계정에 푸시합니다.

**Q: 위의 공유 옵션 1, 2, 3을 사용하여 내 에이전시 외부의 에이전시와 공유한 스냅샷을 어떻게 업데이트해야 하나요?**

공유 옵션 1, 2, 또는 3을 사용하여 스냅샷을 공유했고 업데이트하려면 다음 단계를 따르세요:

- 원하는 변경사항으로 기존 스냅샷을 새로 고침하거나 업데이트합니다.
- 새로운 스냅샷 URL을 관련 에이전시와 다시 공유합니다.

새로운 스냅샷 URL을 생성하면 이전 URL이 무효화된다는 점이 중요합니다. 따라서 업데이트된 스냅샷의 새 URL을 공유해야 합니다.

**Q: '하위 계정' 또는 '에이전시' 제한 공유 링크(위의 공유 옵션 3 및 4)를 사용하여 에이전시와 업데이트된 스냅샷을 어떻게 공유할 수 있나요?**

'하위 계정' 또는 '에이전시' 제한 공유 링크를 사용하여 에이전시와 새로운 스냅샷을 공유하려면 다음 단계를 따르세요:

- 원하는 변경사항으로 현재 스냅샷을 [새로 고침하거나 업데이트](refresh-or-update-snapshots.md)합니다.
- 스냅샷 공유 모달로 이동하여 '하위 계정' 또는 '에이전시' 제한 공유 링크를 선택합니다.
- **기존 및 새로운** '하위 계정' 또는 '에이전시' 관계 번호를 모두 입력합니다.
- '공유'를 클릭하여 새로운 스냅샷 URL을 생성합니다.
- 이전 URL이 더 이상 작동하지 않으므로 이 새 URL을 관련 에이전시와 공유합니다.

**Q: 스냅샷을 공유했는데 '새로운' 항목이 보이지 않습니다. 어떻게 된 건가요?**

스냅샷을 공유했는데 '새로운' 항목이 보이지 않는다면, 공유하기 전에 기존 스냅샷을 업데이트하지 않았을 수 있습니다. 스냅샷은 특정 시점의 특정 계정 복사본이라는 점을 기억하세요.

공유된 스냅샷에 '새로운' 항목을 포함하려면 다음 단계를 따라야 합니다:

- '새로운' 항목을 포함하도록 기존 스냅샷을 업데이트합니다.
- 업데이트된 스냅샷을 다시 공유하거나 관련 하위 계정에 [업데이트를 푸시](refresh-or-update-snapshots.md)합니다.
