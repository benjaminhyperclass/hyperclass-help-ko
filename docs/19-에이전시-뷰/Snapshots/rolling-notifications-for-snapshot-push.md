# 스냅샷 배포 시 단계별 알림

이 문서는 Hyperclass의 "단계별 알림" 기능이 여러 하위 계정에 스냅샷을 배포할 때 어떻게 사용자 경험을 개선하는지 설명합니다. 각 로케이션마다 별도의 알림을 받는 대신, 그룹화된 업데이트로 알림이 더욱 깔끔하고 따라가기 쉬워집니다.

**참고사항:** 이 문서는 스냅샷 배포 시 앱 내 단계별 알림이 어떻게 표시되는지만 설명합니다. 이메일 알림에 대한 정보를 찾고 계시다면 [스냅샷 알림 설정]([snapshot-notification-preferences](snapshot-notification-preferences.md))을 참조해 주세요.

---

## **스냅샷 배포(Snapshot Push)란 무엇인가요?**

스냅샷 배포는 미리 정의된 설정, 퍼널, 워크플로우 또는 전체 계정 설정을 한 Hyperclass 계정(주로 에이전시 템플릿)에서 다른 하위 계정으로 복사하는 과정입니다. 이를 통해 에이전시는 각각을 처음부터 다시 만들지 않고도 여러 클라이언트 계정에 표준화된 설정을 빠르게 배포할 수 있습니다.

---

## **단계별 알림 기능이란 무엇인가요?**

단계별 알림은 여러 하위 계정에 스냅샷을 배포할 때 알림이 전달되는 방식을 개선합니다. 기존에는 각 로케이션마다 별도의 알림을 받았지만, 이제 10개 이상의 하위 계정에 배포할 때는 알림이 그룹화되어 진행률 구간별로(10%, 30%, 70%, 100%) 전송됩니다.

---

## **단계별 알림의 주요 이점**

그룹화된 알림으로 복잡함을 줄이고 스냅샷 진행 상황에 대한 가시성을 개선합니다.

- 10개 이상의 하위 계정에 배포할 때 알림 과부하를 줄입니다
- 4개의 주요 단계(10%, 30%, 70%, 100%)에서 그룹화된 진행 상황 업데이트를 제공합니다
- 10개 미만의 하위 계정에 대해서는 개별 알림을 유지합니다
- 관리자가 대규모 스냅샷 배포를 더 쉽게 모니터링할 수 있게 도와줍니다
- 별도 설정이 필요 없습니다 — 이 업데이트는 자동으로 활성화됩니다
- 실패한 스냅샷 배포에 대한 개별 알림으로 가시성을 유지합니다

---

## 10개 이하 하위 계정으로 스냅샷 배포

10개 미만의 로케이션으로 스냅샷을 배포하면 기존과 마찬가지로 각 로케이션마다 개별 알림을 받게 됩니다.

## **스마트 알림 그룹화 (10개 이상 하위 계정으로 스냅샷 배포)**

스마트 알림 그룹화는 이 기능의 핵심으로, 명확성과 효율성을 모두 우선시합니다. 스냅샷 배포 시 대상 로케이션 수에 따라 알림 전략을 분류합니다.

- 10개 미만의 로케이션으로 배포하는 경우, 기존처럼 개별 알림이 전달되어 각 로케이션에 대한 자세한 정보를 제공합니다
- 10개 이상의 로케이션에 배포하는 경우, 주요 단계인 10%, 30%, 70%, 그리고 최종 100%에서 트리거되는 단계별 알림으로 통합됩니다
- 이러한 그룹화 전략은 복잡함을 줄이고 중요한 단계는 계속 표시하면서 전체 진행 상황에 집중할 수 있게 도와줍니다

![스마트 알림 그룹화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155051043116/original/BUqjVZeIkOkBG6K5mnZMQmq_WNfEdpjxAQ.png?1754489193)

## **개별 실패 알림**

개별 실패 알림은 통합된 단계별 시스템을 사용할 때도 어떤 오류도 놓치지 않도록 보장합니다.

- 실패한 각 스냅샷 배포는 즉각적인 주의를 끌기 위해 자체 알림을 생성합니다
- 이러한 타겟 알림 시스템은 빠른 문제 해결을 가능하게 하고 시스템적 문제가 해결되지 않은 채 남아있을 위험을 최소화합니다
- 이 접근 방식은 스냅샷 프로세스 전반에 걸쳐 완전한 투명성을 유지하여 더 나은 유지보수와 더 빠른 해결을 위한 기반을 마련합니다

![개별 실패 알림](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155051042781/original/SiU5Nf1-sjOCe3bcDX2K11CQNrDwSFzAbA.png?1754489052)

**중요**: 모든 단계별 또는 실패 알림에는 해당 배포로 미리 필터링된 스냅샷 → 배포 기록(Push History)을 열어주는 파란색 "자세히 보기(View Details)" 링크가 포함되어 있어, 여러 번 클릭할 필요가 없습니다.

## **자주 묻는 질문**

**Q: 개별 알림을 받을지 단계별 알림을 받을지는 무엇에 의해 결정되나요?**

10개 미만의 로케이션으로 스냅샷을 배포하면 개별 알림을, 10개 이상의 로케이션으로 배포하면 주요 단계에서 단계별 알림으로 그룹화됩니다.

**Q: 대규모 배포 중에 단계별 알림을 몇 개나 받게 되나요?**

10%, 30%, 70%, 100% 진행률 단계에서 총 4개의 통합 알림을 받게 됩니다.

**Q: 스냅샷 배포 실패 알림도 통합되나요?**

아니요, 실패한 각 배포는 즉각적인 주의를 위해 개별 알림을 생성합니다.

**Q: 스냅샷 배포에 대한 자세한 정보는 어떻게 볼 수 있나요?**

모든 알림에는 전체 스냅샷 배포 기록으로 바로 이동하는 "자세히 보기" 링크가 포함되어 있습니다.

**Q: 그룹화된 알림을 끌 수 있나요?**

아니요, 이것은 시스템 수준의 개선사항이므로 전환할 수 없습니다.
