# 스냅샷 관리: 선택적 자산 스냅샷과 강제 스냅샷 생성

이 문서는 Hyperclass의 향상된 스냅샷 관리 도구를 사용해 선택적으로 스냅샷을 만들고 새로 고치거나, 일부 자산이 로드되지 않아도 강제로 생성하는 방법을 보여드립니다. 이러한 업데이트는 시간을 절약하고 오류를 피하며 하위 계정 데이터를 더 효율적으로 관리할 수 있게 도와줍니다.

## **스냅샷 관리란 무엇인가요?**

스냅샷 관리는 하위 계정의 설정(워크플로우, 퍼널, 이메일 등)을 캡처해서 다른 계정에서 재사용할 수 있게 해주는 기능입니다. 향상된 기능을 통해 이제 스냅샷에 포함할 자산을 정확히 선택하고, 특정 부분만 새로 고치며, 필요할 때 자산 로딩 오류를 우회할 수 있습니다.

## **향상된 스냅샷 관리의 주요 장점**

Hyperclass의 새로운 스냅샷 개선 기능은 제어, 안정성, 효율성에 중점을 둡니다. 여러 하위 계정을 관리하는 팀에게 필수적인 기능들의 장점은 다음과 같습니다:

- **선택적 자산 포함**으로 용도에 맞게 집중된 효율적인 스냅샷을 만들 수 있습니다.
- **선택적 스냅샷 새로 고침**으로 원하는 자산만 업데이트해 처리 시간을 단축합니다.
- **강제 스냅샷 생성**으로 일시적인 자산 로드 오류로 인해 작업이 중단되지 않습니다.
- **개선된 자산 선택 UI**로 포함할 항목을 더 빠르고 직관적으로 선택할 수 있습니다.
- **스마트 오류 처리**로 명확한 시스템 프롬프트와 함께 작업을 원활하게 진행합니다.
- **생산성 최적화**로 수동 정리 작업을 줄이고 실패한 스냅샷 시도를 줄입니다.
- **명확한 자산 로딩 진행률**로 스냅샷 생성 또는 새로 고침 중 얼마나 처리되었는지 볼 수 있습니다.
- **실패한 로드의 자산별 재시도**로 개별 자산의 일시적인 문제가 나머지 스냅샷을 차단하지 않습니다.

## 선택적 자산으로 새 스냅샷 만들기

선택적 스냅샷 도구를 사용해 하위 계정에서 필요한 것만 정확히 캡처하세요.

- **Account Snapshots(계정 스냅샷)로 이동**: 메인 메뉴에서 **Account Snapshots(계정 스냅샷)**을 선택합니다.
- **Create New Snapshot(새 스냅샷 생성)**을 클릭합니다.
- **스냅샷 이름 지정**: 스냅샷의 설명적인 제목을 입력합니다.
- **하위 계정 선택**: 드롭다운을 사용해 스냅샷이 가져올 하위 계정을 선택합니다.
- **포함할 자산 선택**:
  - 선택한 하위 계정의 모든 자산 목록이 표시됩니다.
  - 포함하고 싶은 항목 옆의 체크박스를 선택합니다.
  - 모든 것을 선택하거나 특정 워크플로우, 퍼널, 이메일 등만 선택할 수 있습니다.
- **Create(생성)**을 클릭해 스냅샷을 저장합니다.

## **기존 스냅샷 선택적으로 새로 고치기**

전체 재설정 없이 필요한 변경사항만으로 기존 스냅샷을 업데이트하세요.

- **스냅샷 찾기**: **Account Snapshots(계정 스냅샷)**으로 가서 업데이트하려는 스냅샷을 찾습니다.
- **새로 고침 아이콘 클릭**: 각 스냅샷 옆에 새로 고침 기호가 나타납니다.
- **자산 선택 수정**:
  - 현재 포함된 자산 목록이 표시됩니다(미리 선택됨).
  - 업데이트가 필요한 내용에 맞춰 자산을 추가하거나 제거합니다.
- **Refresh(새로 고침)**을 클릭해 변경사항을 적용합니다.

## **강제 스냅샷 생성 작동 방식**

일시적인 오류로 스냅샷이 차단되지 않도록 하세요. 강제 생성으로 작업을 계속 진행하세요.

- 특정 자산 로딩 중 시스템이 실패를 감지하면 경고가 나타납니다.
- **Force Create Snapshot(강제 스냅샷 생성)** 옵션이 제공됩니다.
- 성공적으로 로드된 모든 자산이 스냅샷에 저장됩니다.
- 요약 보고서에서 건너뛴 자산 목록을 확인할 수 있어 나중에 필요하면 후속 조치를 취할 수 있습니다.

## **자주 묻는 질문**

**Q: 스냅샷 생성 중 자산을 선택하지 않으면 어떻게 되나요?**

스냅샷은 생성되지만 자산이 포함되지 않습니다. 나중에 새로 고침을 통해 콘텐츠를 포함해야 합니다.

**Q: 나중에 스냅샷의 내용을 편집할 수 있나요?**

기존 스냅샷을 새로 고쳐서 다른 자산을 포함하거나 제외할 수 있지만, 스냅샷의 콘텐츠를 직접 수정할 수는 없습니다.

**Q: 강제 스냅샷 생성을 사용하면 데이터가 손실될 위험이 있나요?**

아니요. 로드할 수 없는 자산은 삭제되지 않고 건너뛸 뿐이며, 향후 새로 고침이나 스냅샷에서 다시 시도할 수 있습니다.

**Q: 강제 생성 중 건너뛴 자산을 확인할 수 있나요?**

네, 스냅샷이 생성된 후 건너뛰거나 로드에 실패한 자산 목록이 제공됩니다.

**Q: 스냅샷을 강제로 새로 고칠 수도 있나요?**

아니요, 강제 스냅샷 생성은 **최초** 생성 시에만 사용할 수 있고, 새로 고침 중에는 사용할 수 없습니다.
