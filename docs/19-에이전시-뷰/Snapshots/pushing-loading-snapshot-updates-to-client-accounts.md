# 클라이언트 계정에 스냅샷 업데이트 푸시 및 로딩하기

## 차이점은 무엇인가요?

- **업데이트 푸시** = 업데이트된 스냅샷의 변경사항을 이전에 해당 스냅샷을 로딩했던 하위 계정들에 전송합니다.

- **스냅샷 로딩** = 기존 콘텐츠를 삭제하거나 수정하지 않고 스냅샷 항목을 기존 하위 계정에 추가합니다.

## 1부: 하위 계정에 업데이트 푸시하기

### 언제 사용하나요:
원본 하위 계정을 변경하고 스냅샷을 새로고침했을 때 - 해당 스냅샷을 이전에 로딩했던 하위 계정들에 **변경사항을 동기화**하고 싶을 때 사용합니다.

**중요**: 업데이트를 푸시하기 전에 항상 스냅샷을 새로고침하여 최신 변경사항이 포함되도록 해야 합니다.

### 스냅샷 업데이트를 푸시하는 방법

- `Agency View(에이전시 뷰) > Account Snapshots(계정 스냅샷)`으로 이동합니다

- 스냅샷을 찾아 ? 새로고침 아이콘을 클릭하여 최신 변경사항을 가져옵니다

- 그다음 스냅샷 옆의 Push Updates(푸시 업데이트) 아이콘을 클릭합니다

- OK를 클릭하여 확인 메시지를 승인합니다

- 업데이트하려는 하위 계정을 선택합니다

- 푸시할 에셋을 선택합니다(예: 워크플로우, 퍼널, 폼)

### 업데이트를 푸시하면 어떤 일이 일어나나요?

- 스냅샷 새로고침을 통해 추가된 새로운 에셋들이 하위 계정에 추가됩니다

- 기존 에셋(예: 워크플로우, 캠페인)은 스냅샷의 최신 버전으로 덮어씌워집니다

- 클라이언트가 스냅샷 연결 에셋에 대해 만든 커스텀 변경사항은 먼저 복제하지 않는 한 손실됩니다

### 알아야 할 중요한 규칙:

| 동작 | 결과 |
|------|------|
| 원래 스냅샷으로 생성된 항목 | 푸시 중 덮어씌워짐 |
| 하위 계정에서 수동으로 편집된 항목 | 푸시에 포함된 경우 되돌려짐 |
| 사용자가 복제한 항목(예: 캠페인 사본) | 안전함 - 건드리지 않음 |
| 커스텀 값(Custom Values) | 절대 편집하거나 삭제하지 않음 |
| 사용자(예: 캠페인에 배정된) | 수정되지 않음 |
| 외부 에이전시 계정 | 푸시 업데이트를 받을 수 없음 - 공유 링크를 통해 다시 가져와야 함 |

### 💡 전문가 팁: 덮어쓰기를 방지하는 방법

클라이언트가 덮어씌워지지 않고 캠페인을 커스터마이징할 수 있게 하려면:

- 원본 항목의 사본을 만듭니다(예: "캠페인 X (사본)")

- 클라이언트에게 복사된 버전을 편집하도록 안내합니다

- 연결된 자동화가 사본을 사용하도록 업데이트합니다

💡 복사된 버전은 스냅샷이 추적하지 않으므로 업데이트 중에도 그대로 유지됩니다.

## 2부: 기존 계정에 스냅샷 로딩하기

이는 라이브 하위 계정에 스냅샷 콘텐츠를 수동으로 추가하는 기능입니다.

### 어떤 일이 일어나나요:

- 아무것도 삭제되지 않습니다

- 스냅샷 항목이 기존 콘텐츠 위에 추가됩니다

- 동일한 스냅샷을 여러 번 로딩하면 항목이 중복됩니다

- 100개 이상의 계정에 스냅샷을 로딩하는 에이전시의 경우:
  - 처음 100개는 즉시 처리됩니다
  - 다음 100개는 10분 후 처리됩니다
  - 그 이후부터는 10분마다 100개씩 배치로 처리됩니다

💡 **템플릿으로 클라이언트를 온보딩할 때는 스냅샷 로딩**을 사용하세요
💡 **이미 온보딩되었고 동기화된 변경사항이 필요할 때는 업데이트 푸시**를 사용하세요
