# 새로운 SaaS 하위 계정의 기본 이름 규칙이란?

이 설정은 체크아웃 과정에서 HighLevel이 새로운 SaaS 하위 계정을 생성할 때 사용되는 기본 이름 규칙을 정의합니다. 에이전시는 회사 이름(체크아웃 시 제공되는 비즈니스명) 또는 기존의 고객 이름 + '님의 계정' 형식 중 선택할 수 있습니다. 이 선택은 향후 신규 가입자에게 적용되며 기존 계정의 이름은 변경하지 않습니다.

참고: 이 옵션은 에이전시 수준의 SaaS Configurator에 있으며, Agency Pro($497) 플랜에서 SaaS 모드를 사용하는 에이전시에서만 이용할 수 있습니다. Starter 또는 Freelancer 플랜을 사용 중이시라면 Agency Pro로 업그레이드하여 SaaS Configurator 기능에 접근하세요.

---

## 새로운 SaaS 구독자를 위한 기본 이름 규칙의 주요 장점

- **명확성 개선**: 하위 계정 이름이 구매자의 개인 이름이 아닌 실제 운영 중인 비즈니스를 반영합니다.

- **클라이언트 경험 향상**: 선호하는 이름 규칙을 자동으로 적용하여 온보딩 후 수동 업데이트를 줄입니다.

- **운영 효율성**: 지원팀과 성공팀이 계정을 검색, 필터링, 인식하기 쉽게 만듭니다.

- **유연한 옵션**: 회사 이름과 고객 이름 + '님의 계정' 간 전환하여 내부 워크플로우에 맞춥니다.

---

## 새로운 SaaS 구독자를 위한 기본 이름 규칙 설정 방법

- SaaS Configurator → Advanced Settings(고급 설정) → **Sub-Account(하위 계정)** Onboarding(온보딩)으로 이동합니다.

- Sub-Account Onboarding(하위 계정 온보딩) 하에서 다음 중 하나를 선택합니다:
  - Company Name(회사 이름) (비즈니스 중심의 명확성을 위해 권장), 또는
  - Customer Name(고객 이름) + 's Account(님의 계정) (팀이 주로 소유자 기준으로 검색하는 경우 유용)

- Save(저장)를 클릭합니다. HighLevel 퍼널(및 지원되는 외부 결제)을 통해 생성되는 새로운 SaaS 하위 계정은 이제 선택한 규칙을 따릅니다.

## 이름 규칙 옵션

각 옵션이 어떻게 작동하는지 확인하여 팀이 새로운 고객을 식별하고 지원하는 방식에 맞는 기본값을 설정할 수 있습니다.

- **Company Name(회사 이름)** — 체크아웃 시 제공된 비즈니스 이름을 사용하여 하위 계정이 클라이언트의 브랜드를 명확히 나타냅니다.
  예시: Acme Dental

- **Customer Name(고객 이름) + 's Account(님의 계정)** — 구매자의 개인 이름에 '님의 계정'을 추가합니다.
  예시: Jane Smith님의 계정

참고: 체크아웃 과정에서 Company Name(회사 이름) 필드가 없거나 비어있는 경우, HighLevel은 Customer Name(고객 이름) + '님의 계정'으로 기본 설정됩니다.

---

## 외부 결제 연동

클라이언트의 비즈니스 이름을 외부 결제에서 HighLevel로 메타데이터로 전달하여 HighLevel 퍼널 체크아웃 **외부에서도** 동일한 이름 규칙을 적용할 수 있습니다.

- 외부 결제에서 비어있지 않은 companyName 구독 메타데이터 값을 전송하도록 확인합니다.

- 메타데이터가 있을 때 비즈니스 이름을 적용하려면 SaaS Configurator를 Use Company Name(회사 이름 사용)으로 설정해 두세요.

- companyName이 누락되거나 비어있는 경우, HighLevel은 Customer Name(고객 이름) + '님의 계정'으로 대체됩니다.

예시 메타데이터 페이로드(참고용):

```json
{
  "subscription": {
    "metadata": {
      "companyName": "Acme Dental"
    }
  }
}
```

## 동작 & 예시

다음 간단한 예시들로 가입 후 본인과 클라이언트가 보게 될 내용을 명확히 하여 기대치를 맞춰보세요.

- "Company Name(회사 이름)"이 선택되고 체크아웃 시 제공된 경우:
  결과 하위 계정: Acme Dental

- "Company Name(회사 이름)"이 선택되었지만 제공되지 않은 경우:
  결과 하위 계정: Jane Smith님의 계정 (고객 이름으로 대체)

- "Customer Name(고객 이름) + 's Account(님의 계정)"이 선택된 경우:
  결과 하위 계정: Jane Smith님의 계정

**참고:** 계정 생성 후 관리자는 필요한 경우 나중에 **Location의 Friendly Business Name(로케이션의 친근한 비즈니스 이름)**을 업데이트할 수 있습니다. 나중에 변경해도 이미 발송된 과거 확인서나 이메일에는 소급 적용되지 않습니다.

## **회사 이름 캡처**

HighLevel은 고객이 체크아웃 시 입력한 회사 이름을 캡처하여 `companyName`으로 구독의 메타데이터에 포함합니다.

다음에 적용됩니다:
- Payments 생태계를 사용하는 V2 퍼널(API를 통해 메타데이터 전달)
- Stripe를 사용하는 SaaS V1 설정(Stripe를 통해 메타데이터 전달)

## 자주 묻는 질문

**Q: 이 설정이 기존 SaaS 계정에 영향을 주나요?**
아니요. 설정을 저장한 후 새로운 SaaS 가입자에게만 적용됩니다.

**Q: 체크아웃 중에 회사 이름이 제공되지 않으면 어떻게 되나요?**
HighLevel이 고객 이름 + '님의 계정'으로 기본 설정됩니다.

**Q: 나중에 기본값을 변경할 수 있나요?**
네. 언제든지 SaaS Configurator에서 옵션을 업데이트할 수 있습니다. 향후 가입자에게 적용됩니다.

**Q: 외부 결제에서 이 기능을 활성화하려면 어떻게 하나요?**
구독 메타데이터에서 비어있지 않은 companyName을 전송하고 설정을 Use Company Name(회사 이름 사용)으로 유지하세요.

**Q: 설정을 변경하면 이전에 생성된 계정의 이름이 바뀌나요?**
아니요. 소급 적용되는 이름 변경은 없습니다.

**Q: 이것이 커뮤니케이션에서 {{location.name}}과 같은 머지 필드에 영향을 주나요?**
초기 로케이션 이름은 생성 시 선택된 규칙을 반영합니다. {{location.name}}을 사용하는 메시지는 나중에 로케이션 설정에서 편집하지 않는 한 해당 이름을 표시합니다.

**Q: 회사 이름의 특수 문자가 문제를 일으킬 수 있나요?**
최상의 결과를 위해 가능한 한 표준 영숫자를 사용하세요. 외부 결제에서 이름이 받아들여지지 않거나 잘못 보이는 경우, 생성 후 로케이션 이름을 편집하세요.
