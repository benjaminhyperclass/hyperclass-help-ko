# 문서 & 계약 사용 가이드

Hyperclass의 문서 & 계약(Documents & Contracts) 기능을 사용하면 CRM 내에서 직접 전문적인 계약서, 제안서, 견적서를 생성하고 전송하며 추적할 수 있습니다. 템플릿, PDF 업로드, 전자 서명, 결제, 워크플로우 자동화 기능을 통해 문서 관리를 중앙화하면서 전문성과 효율성을 개선할 수 있습니다.

# 문서 & 계약이란?

문서 & 계약은 Hyperclass의 **결제(Payments)** 섹션에 내장된 도구로, 법적 구속력이 있는 계약을 관리할 수 있게 해줍니다. 빈 문서부터 시작하거나, 기존 PDF를 업로드하거나, 미리 만들어진 템플릿을 사용할 수 있습니다. 전자 서명, 결제 수집, 자동화 트리거, 완전한 감사 추적도 지원합니다.

이 기능은 에이전시, 소상공인, 서비스 제공업체가 고객 계약을 간소화하고 서류 작업을 없애며 더 빠르게 계약을 성사시키도록 도와줍니다.

---

## 문서 & 계약의 핵심 장점

- **중앙화된 대시보드**: 결제 섹션 아래에서 모든 계약서와 제안서를 하나의 체계적인 곳에서 관리합니다. 여러 도구를 검색하는 시간을 줄이고 팀이 모든 문서의 상태를 항상 파악할 수 있도록 합니다.

- **커스텀 템플릿**: 서비스 계약이나 제안서 같은 반복되는 계약에 미리 만들어진 템플릿을 재사용합니다. 문서 생성 속도를 높이면서 고객 간 일관성과 전문성을 보장합니다.

- **감사 추적이 포함된 전자 서명**: 서명자 세부정보, IP 주소, 타임스탬프를 기록하는 인증서와 함께 법적 구속력이 있는 서명을 온라인으로 수집합니다. 고객에게는 간편한 서명 과정을 제공하면서 법적 보호를 받을 수 있습니다.

- **통합 결제**: 계약서 내에 직접 일회성 또는 정기 결제를 추가합니다. 별도의 인보이스 과정 없이 즉시 계약을 성사시키고 수익을 수집할 수 있습니다.

- **워크플로우 자동화**: 문서가 생성, 전송, 조회, 서명될 때 자동으로 액션을 트리거합니다. 계약이 자동으로 진행되도록 하여 수동 후속 조치를 줄이고 전환율을 개선합니다.

# 문서 & 계약 대시보드 개요

대시보드는 모든 계약서, 견적서, 제안서를 한눈에 볼 수 있는 화면을 제공합니다. 이곳이 계약 관리의 제어 센터입니다.

**대시보드 접근하기**

모든 계약을 관리하려면 좌측 메뉴에서 **결제(Payments) → 문서 & 계약(Documents & Contracts)**으로 이동하세요. 드롭다운에서 **모든 문서 & 계약(All Documents & Contracts)**을 선택하면 초안, 전송됨, 완료됨, 보관된 모든 문서를 한 곳에서 볼 수 있습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054013600/original/b6K01QKYbW9ZyWluiwnw2pPg7nyymR8Mtw.png?1758180898)

**날짜 필터**

캘린더에서 시작일과 종료일을 선택하여 결과를 좁힐 수 있습니다. 특정 청구 기간 동안 생성된 계약을 추적하거나 특정 기간 내에 발송된 제안서를 검토할 때 특히 유용합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054013970/original/VhfehSG2aLfc1RN3pqZrAYt60Kk6HNDxpA.png?1758181098)

**문서 상태 및 대시보드 컬럼**

- 대시보드는 계약서와 제안서를 **초안(Draft), 타인 대기중(Waiting for others), 완료(Completed), 결제(Payments), 보관됨(Archived)** 탭으로 정리합니다. 이 탭들로 진행 상황을 쉽게 추적할 수 있습니다. 예를 들어, *타인 대기중*에서 아직 서명이 필요한 고객을 빠르게 확인할 수 있습니다.

- 각 문서 행에는 **제목(Title), 상태(Status), 고객(Customer), 수정일(Date Modified), 금액(Value)** 등 중요한 세부정보가 표시됩니다. 어떤 계약이 활성 상태인지, 어떤 계약이 후속 조치가 필요한지, 어떤 계약이 완료되었거나 기록 보관용으로 보관되었는지 빠르게 평가할 수 있습니다.

- 각 레코드 옆의 점 세 개 메뉴는 **보기(View), 복제(Clone), 완료로 표시(Mark as Completed), PDF 다운로드(Download PDF), 템플릿으로 변환(Convert to Template), 링크로 공유(Share via Link), 초안으로 이동(Move to Draft), 거절로 표시(Mark as Declined)** 같은 빠른 작업을 제공합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054054890/original/JMpofkXF1p3BE1KdyXAxo16q9p1_n3FwTg.png?1758201218)

# 새 문서 생성하기

필요에 따라 세 가지 방법으로 문서를 생성할 수 있습니다.

대시보드 오른쪽 상단의 **+ New** 버튼을 클릭하세요. 다음 중에서 선택할 수 있습니다:

- **새 문서(New Document)**: 편집기를 사용해 처음부터 시작합니다. 예: 텍스트 블록, 가격표, 동영상 삽입이 포함된 새로운 제안서 작성

- **기존 PDF 업로드(Upload Existing PDF)**: PDF를 가져와서 서명 또는 텍스트 필드를 오버레이합니다. 예: 고객 전자 서명용으로 업로드한 코칭 계약서

- **템플릿 라이브러리에서 가져오기(Import from Template Library)**: 기존 템플릿에서 선택합니다. 예: 청소 사업체가 표준 서비스 계약을 재사용

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054055258/original/0ilqxRWMqzRTETE5RqLlmfvJADEATM8qxQ.png?1758201378)

# 문서 편집기 도구 모음

편집기는 문서를 설계, 구성, 완성할 수 있는 유연한 드래그 앤 드롭 환경을 제공합니다.

### **요소 추가**

- 텍스트, 이미지, 동영상, 표, 상품 목록, 페이지 나누기를 추가합니다.

- **서명(Signature), 텍스트 필드(Text Field), 날짜(Date), 이니셜(Initials), 체크박스(Checkbox)** 같은 입력 가능한 필드를 삽입합니다. 입력 가능한 필드(예: 서명)를 추가한 후, 필드를 선택하고 오른쪽 속성(Properties) 패널을 사용하세요. **서명자(To be signed by)**를 사용해 하위 계정의 특정 **직원(staff)** 멤버를 포함하여 올바른 서명자에게 필드를 할당합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054055235/original/zynodi0J5lc9ZQzp-VGvHTjzlFNzgpoz0A.png?1758201357)

### 페이지 관리

- 여러 페이지 문서를 관리합니다.

- 필요에 따라 페이지를 추가, 복제, 제거합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054055305/original/kM7D4kyPqUj2urexPNVLNcY5_55x1qqgHw.png?1758201399)

### 문서 변수

- **생성일(Created Date), 참조 번호(Reference Number), 하위 계정명(Sub-Account Name)**, 커스텀 필드 같은 동적 자리표시자를 삽입합니다.

- 변수는 문서가 발송될 때 자동으로 올바른 데이터로 채워집니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054055358/original/iWOyklLcxDFO39MtLrNpZthkcFYU9FVlWg.png?1758201427)

### 콘텐츠 라이브러리

- 문서 & 계약용 재사용 가능한 블록과 전체 페이지의 중앙화된 라이브러리

- 콘텐츠 라이브러리에 추가(Add to Content Library)를 통해 재사용할 섹션/페이지를 저장하고 쉽게 찾을 수 있도록 이름을 지정

- 콘텐츠 라이브러리(Content Library) → 커스텀(Custom)에서 드래그 앤 드롭으로 저장된 항목을 모든 문서/템플릿에 삽입

- 문서에 추가한 후 필요에 따라 삽입된 콘텐츠를 업데이트/커스텀

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066874970/original/Z6fpt1B1iTEXeti67Zi4fIySEYB5Y-vC5Q.gif?1773400839)

### 수신자

- 문서에 서명하거나 받을 사람을 정의합니다.

- 여러 수신자를 추가하고 서명 순서(순차적 또는 동시)를 설정합니다.

- 각 수신자에게는 특정 필드(예: 서명, 이니셜)가 할당됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054055462/original/pXzOH22B60ZyQubq0yMF9oAB69wAJyDNuQ.png?1758201453)

### 결제 설정

- 계약서 내에 직접 결제 옵션을 추가합니다.

- **일회성(One-Time)** 또는 **정기(Recurring)** 청구 중에서 선택합니다.

- 서명 시 직접 결제를 활성화하거나 자동으로 인보이스를 발송합니다.

- 실시간 결제 모드와 자동 결제 구성을 지원합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054055521/original/u_xboLsFqGzSlELkJJKtx2Uwrk4rtW8fkA.png?1758201489)

### **이메일 편집 및 템플릿 선택**

발송 직전에 템플릿을 선택하고 이메일을 편집하면 수신자가 가장 정확한 메시지를 볼 수 있습니다. 이렇게 하면 오류가 줄어들고 고객 승인이 빨라집니다.

**할 수 있는 일**

- 템플릿 선택: **이메일 템플릿(Email Template)** 드롭다운을 사용해 상황에 맞는 버전을 선택합니다.
- 발송 전 편집: 편집 아이콘이 항상 보이므로 막바지 변경이 쉽습니다.

**단계**

- 문서에서 발송(Send)을 클릭해 발송 모달을 엽니다.

- 사용하고 싶은 **이메일 템플릿(Email Template)**을 선택합니다.

- 필요하면 편집(Edit)을 클릭해 제목이나 이메일 본문을 조정합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063305986/original/kO1BqIyTad268QP0eqmXP7PaQrdUSkifZg.png?1769153298)

- 미리보기를 확인하고 발송합니다.

### **문서 설정**

- 문서가 전달되는 방식을 제어합니다.

- 기본 이메일 구성(발신자명, 발신 이메일, 제목, 템플릿)을 재정의합니다.

- 서명 후 고객을 커스텀 URL로 리디렉션합니다.

- 문서 이메일에 포함될 첨부 파일을 업로드합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054055706/original/FeEhlZioHSSaCRlS-k7zLOm5NbJ_SBtE0w.png?1758201592)

### **편집기 내 액션 메뉴**

- 점 세 개 메뉴(⋮)에 접근해 추가 옵션을 확인할 수 있습니다: **완료로 표시(Mark as Completed), PDF 다운로드(Download PDF), 템플릿으로 변환(Convert to Template), 만료일 추가(Add Expiry), 삭제(Delete)**.

- 완료되면 **발송(Send)**을 클릭해 전달합니다.

- **이메일로 발송(Send via Email)** 또는 **링크로 공유(Share via Link)** 중에서 선택합니다.

# 자주 묻는 질문

**Q: 고객이 부분적으로 서명했지만 문서를 완료하지 않으면 어떻게 되나요?**

여러 서명자가 할당되었는데 일부만 필드를 완료한 경우, 문서는 **타인 대기중(Waiting for others)**에 남아 있습니다. 대시보드에서 재발송하여 미완료 서명자에게 알림을 보낼 수 있습니다. 모든 필수 서명자가 완료할 때까지 문서는 완료됨으로 이동하지 않습니다.

**Q: 문서 내 전자 서명과 결제 세부정보는 얼마나 안전한가요?**

Hyperclass는 모든 문서와 결제 데이터에 SSL 암호화를 적용합니다. 각 서명은 IP 주소, 이메일, 타임스탬프가 포함된 인증서를 생성합니다. 결제 정보는 통합된 결제 제공업체를 통해 안전하게 처리되며 Hyperclass에는 저장되지 않습니다.

**Q: 문서에서 조건부 콘텐츠를 만들 수 있나요? (예: 고객에 따라 다른 가격표 표시)**

현재 문서 내 조건부 로직은 지원되지 않습니다. 각 문서는 생성되면 정적입니다. 변형이 필요하다면 여러 템플릿을 만들거나 워크플로우를 사용해 조건에 따라 다른 템플릿을 발송하세요.

**Q: 문서를 발송한 후 템플릿을 편집하면 어떻게 되나요?**

템플릿을 편집해도 이미 발송된 문서에는 영향을 주지 않습니다. 변경사항은 해당 템플릿으로 생성하는 향후 문서에만 적용됩니다. 이렇게 해서 이미 서명되었거나 대기 중인 계약의 법적 무결성을 보장합니다.

**Q: 한 문서에서 여러 결제 방법을 허용할 수 있나요?**

네, 해당 게이트웨이가 계정에 연결되어 있다면 여러 결제 처리업체(예: Stripe, PayPal)를 허용하도록 문서를 구성할 수 있습니다. 수신자는 결제 시 선호하는 방법을 선택할 수 있습니다.

**Q: 문서의 만료일은 어떻게 작동하나요?**

문서 액션 메뉴에서 만료일을 추가할 수 있습니다. 날짜가 지나면 문서는 자동으로 **만료됨(Expired)**으로 이동하고 더 이상 서명할 수 없습니다. 기간 한정 오퍼나 제안서에 유용합니다.

**Q: 워크플로우가 특정 템플릿 사용을 기반으로 트리거될 수 있나요?**

네, 워크플로우는 **문서 상태(Document Status)**와 **템플릿명(Template Name)** 모두로 필터링할 수 있습니다. 예를 들어, 특정 "서비스 계약" 템플릿이 3일 이상 서명되지 않은 상태로 남아 있을 때만 알림을 보낼 수 있습니다.
