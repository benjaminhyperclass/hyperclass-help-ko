# 계약서 서명 후 특정 페이지로 리다이렉트하는 방법

### 개요

이제 고객이 문서(Document)에 서명하거나 문서를 통해 생성된 인보이스에서 결제를 완료한 후, 특정 웹 페이지로 자동 이동시킬 수 있습니다. 이를 통해 감사 페이지 표시, 예약 링크 안내, 추가 서비스 추천 등 매끄럽고 일관된 브랜드 경험을 제공할 수 있어요.

### 1. 전역 커스텀 URL 리다이렉트 활성화

- **문서 및 계약서(Documents & Contracts)로 이동**  
  좌측 메뉴에서 결제(Payments) → 문서 및 계약서(Documents & Contracts)를 클릭하세요.
  
- **설정(Settings) 열기**  
  우측 상단의 설정 버튼(톱니바퀴 아이콘)을 클릭합니다.
  
- **문서 설정(Document Settings) 선택**  
  사이드바에서 문서 설정을 선택하세요.
  
- **"커스텀 URL로 리다이렉트" 활성화**  
  스위치를 켜서 모든 문서에 대해 전역적으로 활성화합니다.
  
- **URL 입력** (예: https://example.com)
  
- **기존 탭에서 열기** 또는 **새 탭에서 열기** 선택
  
- **변경사항 저장**  
  저장(Save)을 클릭해서 확인하세요.

**참고:** 전역 설정은 특정 문서나 템플릿에 직접 추가된 커스텀 URL을 덮어쓰지 않습니다.

### 2. 문서별 커스텀 URL 설정

- **문서 생성 또는 편집**  
  문서 및 계약서에서 기존 문서를 열거나 + 새로 만들기를 클릭해서 문서를 만드세요.
  
- **문서 설정(Document Settings)**  
  상단에서 설정(Settings) → 문서 설정을 클릭합니다.
  
- **리다이렉트 활성화 및 URL 입력**  
  "커스텀 URL로 리다이렉트 활성화(Enable redirection to custom URL)" 토글을 켜세요.
  
- 원하는 URL을 입력합니다.
  
- 같은 탭에서 열지 새 탭에서 열지 선택하세요.
  
- **문서 저장**  
  필요에 따라 저장(Save) 또는 보내기(Send)를 클릭합니다.

### 3. 결제 후 리다이렉트

문서 내에서 결제 옵션을 활성화했다면, 결제 완료 후에도 동일한 리다이렉트 설정이 적용됩니다:

- **전역 설정:** 활성화되어 있으면 고객 결제 처리 후 자동으로 리다이렉트됩니다.
- **문서 수준 설정:** 해당 문서에서 URL을 활성화하고 지정했다면, 이것이 우선순위를 가집니다.

### 4. 문제 해결 및 팁

- **전송 전 테스트:** 문서의 미리보기(Preview) 옵션을 사용해서 리다이렉트가 예상대로 작동하는지 확인하세요.
- **우선순위 로직:** 문서별 리다이렉트가 전역 리다이렉트를 덮어씁니다. 전역 리다이렉트만 사용하고 싶다면 문서 수준 설정을 끄거나 제거하세요.

### 5. 일반적인 활용 사례

- **감사 페이지:** 새로운 서명자/결제자를 개인화된 감사 또는 확인 페이지로 안내
- **온보딩:** 서명 직후 고객이 온보딩 단계를 바로 완료하도록 유도
- **업셀링:** 보완적인 오퍼나 예약 페이지를 보여주어 추가 참여 유도
- **리뷰 수집:** 완료 계약서 서명 후 고객으로부터 리뷰 수집
