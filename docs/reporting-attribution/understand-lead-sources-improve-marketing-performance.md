# 어트리뷰션이란?

어트리뷰션은 각 연락처가 처음 그리고 최근에 여러분의 비즈니스와 어떻게 상호작용했는지를 기반으로 소스를 추적합니다. 이를 통해 다음이 가능합니다:

- 최고 성과 마케팅 채널 식별
- 캠페인 효과 측정
- 성장 기회 발견
- 전환 전략 개선

---

## **첫 번째 vs 최근 어트리뷰션**

각 연락처는 고객 여정을 완전히 파악할 수 있도록 두 가지 유형의 어트리뷰션 데이터를 저장합니다.

- **첫 번째 어트리뷰션:** 연락처가 여러분의 비즈니스와 가진 첫 번째 상호작용 (예: 폼 제출)
- **최근 어트리뷰션:** 전환 전 가장 최근의 상호작용 (예: 구매)

### **주요 차이점**

- 첫 번째 어트리뷰션은 변경되지 않음
- 최근 어트리뷰션은 새로운 상호작용이 있을 때마다 업데이트됨

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068955754/original/R9fPnBkAvXPO39Matcupvvi28Dq-fj-Khg.png?1775956198)

이 예시는 소스 위로 마우스를 가져갔을 때 어트리뷰션 세부사항이 어떻게 표시되는지 보여주며, 식별자와 채널 분류 같은 세션 레벨 추적 데이터를 포함합니다.

---

## **어트리뷰션 데이터를 찾는 방법**

어트리뷰션 데이터는 각 연락처 레코드 안에 직접 저장되어 있어, 리드 소스를 쉽게 검토하고 분석할 수 있습니다.

찾는 방법:

- 연락처 레코드를 열기
- 활동 섹션으로 이동
- 우측 하단 패널에서 어트리뷰션 세부사항 확인

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068955757/original/1W8q4g8zrIQdLQhfQmhGfg9LfyJYrAc16w.png?1775956223)

이 화면은 연락처 프로필 내에서 첫 번째와 최근 어트리뷰션이 모두 나타나는 위치를 보여주며, 활동 타임라인과 세션 소스 세부사항을 포함합니다.

이 화면에서는 연락처 프로필 내에서 첫 번째와 최근 어트리뷰션 소스를 모두 확인할 수 있습니다.

어트리뷰션 소스 위로 마우스를 가져가면 세션 데이터 같은 추가 추적 세부사항을 볼 수 있습니다.

---

## **어트리뷰션 소스 유형**

연락처는 시스템에 어떻게 유입되었는지에 따라 특정 소스로 분류됩니다.

### **유료 채널**

- Paid Search: 검색 광고를 통한 트래픽
- Paid Social: 유료 소셜 캠페인을 통한 트래픽

### **자연 유입 채널**

- Organic Search: 검색엔진을 통한 트래픽 (유료 광고 제외)
- Social Media: 소셜 플랫폼을 통한 트래픽

### **기타 트래픽 소스**

- Direct Traffic: 식별 가능한 소스가 없음 (예: URL 직접 입력)
- Referrals: 외부 웹사이트를 통한 트래픽
- Others: 전화, SMS, 이메일, 메시징 플랫폼

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155068955759/original/kDOTGTwKTLfp_MkkXg58alLy0NjgGhezmw.png?1775956273)

이 예시는 직접 트래픽으로 어트리뷰션된 연락처가 어떻게 표시되는지 보여주며, 참조 소스나 추적 파라미터가 감지되지 않았음을 나타냅니다.

### **시스템 기반 소스**

- CRM UI: 수동으로 생성된 연락처
- Third-Party: 연동이나 외부 도구를 통한 연락처

---

## **어떤 액션이 어트리뷰션을 기록하는가**

어트리뷰션은 연락처가 세션 중 특정 액션을 수행할 때 캡처됩니다:

- 폼 또는 설문 제출
- 캘린더 예약
- 채팅 상호작용 (세부사항 제출 후)
- 주문 폼 제출

### **중요사항**

어트리뷰션은 이러한 액션이 추적된 페이지나 폼에서 발생할 때만 기록됩니다.

---

## **어트리뷰션 결정 방식**

어트리뷰션은 다음을 기반으로 한 구조화된 규칙 시스템을 사용하여 배정됩니다:

- URL 파라미터 (UTM)
- 클릭 식별자
- 참조 도메인

### **간단한 어트리뷰션 로직**

- 유료 검색 식별자 → Paid Search
- 유료 소셜 식별자 → Paid Social
- 소셜 플랫폼 참조 → Social Media
- 검색엔진 참조 → Organic Search
- 외부 웹사이트 → Referral
- 소스 데이터 없음 → Direct Traffic
- 메시징이나 커뮤니케이션 → Others
- 수동 생성 → CRM UI
- 연동 도구 → Third-Party

---

## **광고 추적 설정 (개요)**

유료 캠페인을 정확히 추적하려면 적절한 URL 태깅이 필요합니다.

### **주요 요구사항**

- 일관된 추적 파라미터 사용
- 값이 정확히 일치하는지 확인 (대소문자 구분)
- 랜딩 페이지 URL에 추적 적용

이렇게 하면 어트리뷰션 데이터가 올바르게 기록되고 적절한 소스에 연결됩니다.

---

## **어트리뷰션 문제 해결**

어트리뷰션 데이터가 누락되거나 잘못된 경우, 다음을 검토해보세요:

### **1. URL 형식 확인**

- 여분의 공백이나 오타 제거
- 파라미터가 올바르게 형식화되었는지 확인

### **2. 사용자를 같은 페이지에 유지**

- 어트리뷰션은 제출이 같은 페이지에서 발생할 때만 캡처됨
- 폼 제출 전에 사용자를 리다이렉트하지 말 것

### **3. 커스텀 수정 피하기**

- 표준 추적 형식을 변경하지 말 것
- 변경하면 어트리뷰션 추적이 중단될 수 있음

---

## **자주 묻는 질문**

**Q: 일부 연락처에 어트리뷰션이 누락된 이유는?**

어트리뷰션은 추적된 페이지에서 필요한 액션이 완료되지 않으면 기록되지 않을 수 있습니다.

**Q: 어트리뷰션이 시간이 지나면서 변할 수 있나요?**

최근 어트리뷰션만 업데이트됩니다. 첫 번째 어트리뷰션은 고정됩니다.

**Q: "Direct Traffic"은 무엇을 의미하나요?**

해당 방문에 대해 식별 가능한 소스가 감지되지 않았다는 의미입니다.

**Q: 어트리뷰션이 "Others"로 표시되는 이유는?**

일반적으로 전화, SMS, 메시지 같은 커뮤니케이션 채널을 포함합니다.

**Q: 외부 도구에서 어트리뷰션을 추적할 수 있나요?**

네, 하지만 해당 연락처들은 서드파티 소스로 분류됩니다.
