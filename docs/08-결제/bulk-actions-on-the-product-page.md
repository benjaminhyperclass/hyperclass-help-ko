# 상품 페이지 일괄 작업

상품 페이지 일괄 작업을 통해 여러 상품이나 변형을 한 번에 업데이트할 수 있어, 반복적인 편집을 줄이고 재고 정확도를 높일 수 있습니다. 이 가이드에서는 가시성, 가격, 컬렉션, 삭제, 재고량 및 재고 상태를 포함한 상품 페이지의 일괄 편집을 다룹니다.

# **상품 페이지 일괄 작업이란?**

일괄 작업 기능을 통해 쇼핑몰 운영자는 하나의 인터페이스에서 대량의 상품을 효율적으로 관리할 수 있습니다. 이제 여러 상품을 선택하여 스토어 가시성, 가격, 컬렉션 등 주요 업데이트를 단 몇 번의 클릭으로 적용할 수 있습니다! 대규모 카탈로그나 빈번한 가격/재고 업데이트에 이상적입니다.

## 일괄 작업의 주요 이점

- **시간 절약:** 한 번의 변경으로 수백 개 아이템을 몇 번의 클릭으로 업데이트
- **일관성:** 변형 및 상품 전반에 걸쳐 가격이나 재고 정책의 일관성 보장
- **오류 감소:** 미리보기와 검증을 통해 수동 편집 대비 실수 감소
- **확장성:** 필터와 페이지네이션으로 대규모 카탈로그를 안심하고 관리
- **운영 정확도:** 재고 수준과 판매 규칙을 실제 재고와 일치시켜 유지

## **사용 가능한 일괄 작업**

| 작업 | 설명 |
|------|------|
| 온라인 스토어에 포함 | 선택한 상품을 온라인 스토어에 표시되도록 설정 |
| **온라인 스토어에서 제외** | 선택한 상품을 온라인 스토어에서 숨김 처리 |
| 가격 변경 | 고정 값이나 백분율을 사용하여 상품 가격을 인상, 인하 또는 새 가격 설정 |
| 정가 변경 | 위와 동일한 가격 옵션을 '정가' 필드에 적용 |
| 컬렉션에 추가 | 상품을 하나 이상의 컬렉션에 추가하거나 새 컬렉션을 즉석에서 생성 |
| 삭제 | 안전 확인 모달과 함께 선택한 상품을 일괄 삭제 |

⚠️ **주의:** 가격 업데이트는 선택한 상품의 모든 변형에 영향을 미칩니다.

## **일괄 작업 접근 방법**

#### **1단계:** 상품으로 이동

이커머스 스토어 내에서 결제(Payments) > 상품(Products)으로 이동하세요.

![상품 페이지 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055466197/original/7NAAYMAc3rYuLeYkOWxrWsPfvamlpcMuzA.png?1759872698)

#### **2단계:** 상품 선택

- 각 행의 체크박스를 사용하여 개별 상품을 선택하거나, 상단 체크박스를 클릭하여 현재 페이지의 모든 상품을 선택하세요.
- 하위 계정의 모든 상품을 선택하려면, 한 페이지의 모든 항목을 선택한 후 나타나는 "모두 선택" 안내를 사용하세요.

![상품 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055466250/original/Y2EO20iN9hCW1XbweBE41dVqrMNAb1tnag.png?1759872811)

#### **3단계:** 일괄 작업 선택

일괄 작업(Bulk Actions) 드롭다운을 열어 원하는 변경 사항을 적용하세요.

![일괄 작업 드롭다운](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055466285/original/UXVSspDqorr4JLDr0CfZdocpX6GXBImwRQ.png?1759872908)

## **일괄 작업을 위한 상품 필터링**

일괄 작업을 적용하기 전에 필터링 패널을 사용하여 상품 선택을 좁혀보세요. 필터를 모두 선택과 결합하여 더 정확한 일괄 작업을 수행할 수 있습니다. 작업 수행 전에 필터를 사용하여 그룹을 분리하면 잘못된 상품을 편집할 위험을 줄일 수 있습니다.

사용 가능한 필터:

- **스토어 가시성:** 포함됨 / 숨김
- **컬렉션:** 이름별
- **소스:** 모든 상품, Woocommerce

필터를 적용하려면 우측 상단의 필터(Filters) 버튼을 클릭한 후 드롭다운에서 필터를 선택하세요.

![필터 적용](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055466377/original/FdMysgt6m5pE_GfWYSUxcDnYtNroW-CVoA.png?1759873120)

## 특수 상황 및 주요사항

### 동작 및 제한사항:

- **마이너스 값 보호:** 마이너스 값으로 이어지는 가격 변경은 해당 상품에서 건너뛰어집니다. 다른 상품들은 여전히 업데이트됩니다.
- **한 번에 하나의 작업:** 현재 일괄 작업이 완료될 때까지 새로운 작업을 시작할 수 없습니다.
- **글로벌 스토어 가시성:** 가시성 변경은 연결된 모든 스토어에 적용됩니다. 특정 스토어를 제외하려면 사이트(Sites) 섹션을 사용하세요 (사이트 > 스토어 > 스토어 선택 > 상품).
- **가격 업데이트에 값 필요:** '인상' 또는 '인하' 작업에는 가격 또는 정가에 기존 값이 필요합니다. 기본 값이 없는 상품은 영향받지 않습니다.
- **대량 세트 삭제 확인:** 50개 이상의 상품을 삭제할 때는 확인 창에 "Delete"를 입력해야 합니다. 삭제된 상품은 복구할 수 없습니다.
- **일괄 삭제와 제한된 상품:** 제한된 상품이 선택되어도 일괄 삭제는 실패하지 않습니다. Hyperclass는 자동으로 제한된 상품을 건너뛰고, 삭제 가능한 상품만 삭제합니다. 제한된 상품은 다음과 같습니다:
  - 캘린더에서 생성된 상품
  - 멤버십/커뮤니티에서 생성된 상품
  - 활성 또는 연체 상태의 구독이 있는 SaaS 상품

자세한 내용은 [상품 삭제 작동 방식](how-product-deletion-works.md)을 참조하세요.

- **일괄 성능 경고:** 300개 이상의 상품에 대한 변경은 반영되는 데 시간이 걸릴 수 있습니다.

### **선택 동작:**

| 작업 | 결과 |
|------|------|
| 상단 체크박스 클릭 | 현재 페이지의 모든 상품 선택 |
| 다른 페이지로 이동 후 복귀 | 선택 상태 유지 (현재 페이지 선택만) |
| 전체 목록에 "모두 선택" 사용 | 다른 페이지로 이동하면 선택 상태가 유지되지 않음 |

## **자주 묻는 질문**

**Q: 디지털 상품과 실물 상품에 일괄 작업을 함께 적용할 수 있나요?**
네, 배송 유형에 관계없이 작업이 적용됩니다.

**Q: 일괄 업데이트가 변형에도 영향을 미치나요?**
네, 모든 가격 관련 변경사항은 변형 수준에서도 반영됩니다.

**Q: 일괄 삭제를 취소할 수 있나요?**
아니요. 삭제된 상품은 영구적으로 제거됩니다. 신중하게 사용하세요.
