# 주문 폼에서 상품 판매하기 - 결제 업체별 가이드

퍼널 버전 2에서는 1단계 및 2단계 주문 폼을 통해 일회성 상품과 구독 상품을 모두 판매할 수 있습니다. (참고: 새로 만드는 모든 퍼널과 웹사이트는 버전 2로 생성됩니다) 이 가이드에서는 주문 폼에서 상품을 판매하는 다양한 사용 사례와 결제 흐름을 설계할 때 고려해야 할 제한사항들을 알아봅니다.

### **이용 가능한 결제 업체**

주문 폼에서 상품 판매 시 여러 결제 업체를 이용할 수 있습니다:

- Stripe 
- Authorize.net
- NMI
- PayPal 
- Square
- 커스텀 결제 업체 (마켓플레이스에서 설치)

## 주문 폼 판매 옵션

- **메인/기본 상품** - 주문 폼에서 판매하는 핵심 상품입니다.
- **범프 상품** - 고객이 기본 상품을 구매할 때 추가로 선택할 수 있는 옵션 상품입니다.
- **업셀** - 고객이 기본 상품을 구매한 직후, 버튼 클릭으로 즉시 제안하는 추가 서비스/상품입니다.

## 주의사항

- Apple Pay와 Google Pay는 Stripe에서만 지원됩니다. Apple Pay를 결제 수단으로 표시하려면 결제(Payments) → 연동(Integrations)에서 Stripe 연결 카드의 토글을 활성화해야 합니다
- Google Pay는 Stripe 연결 시 기본적으로 표시됩니다(구매자 국가에서 지원하는 경우)
- 어떤 결제 업체를 사용하더라도 하나의 결제 과정에서 서로 다른 구독 상품을 동시에 판매할 수 없습니다. 하나의 플로우에서 두 개의 다른 구독 상품을 판매하려면 업셀 구매를 활용하세요
- PayPal은 고객이 구독 상품을 구매할 때 범프 상품 구매를 지원하지 않습니다
- 결제(Payments) → 상품(Products)에서 구독 상품을 만들 때 설정 수수료, 무료 체험 기간, 결제 횟수를 설정할 수 있으며, 고객이 해당 상품을 구매할 때 자동으로 적용됩니다
- 모든 주문은 결제(Payments) → 주문(Orders)에서, 구독은 결제(Payments) → 구독(Subscriptions)에서 추적할 수 있으며, 거래(Transactions) 테이블에서 환불을 관리할 수 있습니다
- 주문할 때 상품에 무료 체험 기간이 있고 체험 시작 전에 명시적으로 부과되는 설정 수수료가 없다면, 주문 완료 시 차감이나 청구가 발생하지 않으며 금액이 '0'으로 표시됩니다. 체험 기간이 끝날 때 요금이 청구됩니다.
