# 기프트카드 발송하는 방법

기프트카드 발송은 비즈니스가 고객에게 구매나 결제 없이 직접 기프트카드를 발행할 수 있는 강력한 기능입니다. 이 가이드는 Hyperclass에서 **기프트카드를 발송**하여 수신자에게 직접 전달하는 방법을 설명합니다.

## 기프트카드 발송이란?

기프트카드 발송은 **일회성 선불 크레딧**을 생성하여 연락처가 구매할 필요 없이 이메일, SMS, PDF 또는 QR 코드로 선택한 수신자에게 전달하는 기능입니다. 이는 서비스 보상, 로열티 혜택, VIP 특전, 내부 테스트에 이상적입니다. 기프트카드가 지원되는 Hyperclass 기반 결제 페이지에서 사용할 수 있으며, 발송 기능은 현재 라이브 모드에서만 작동하여 실제로 완전히 기능하는 기프트카드만 전달됩니다.

### 기프트카드 발송의 핵심 장점

- **1:1 발행**: 개별 수신자에게 즉시 또는 예약하여 기프트카드를 발송
- **결제 불필요**: 수신자에게 요금을 청구하지 않고 선불 가치 발행
- **유연한 전달**: 연락처 데이터와 사용 사례에 따라 이메일, SMS, PDF 또는 QR로 전달
- **간편한 사용**: 수신자가 코드나 링크로 지원되는 결제 페이지에서 쉽게 사용
- **운영 가시성**: 기프트카드 주문(Gift Card Orders)에서 주문, 코드, 금액, 상태 추적
- **서비스 복구에 효과적**: 고객 지원 문제를 빠르고 전문적으로 보상

## 기프트카드 발송 활용 사례

수동 기프트카드 발송은 다음과 같은 경우에 이상적입니다:

- **로열티 및 보상**: 마일스톤 달성, 추천, 프리미엄 플랜 갱신에 대한 고객 보상
- **고객 지원 보상**: 수동 크레딧이나 환불 대신 기프트카드 발행
- **프로모션**: 경품 이벤트, 휴일 선물, 인플루언서 캠페인, 타겟 혜택 실행
- **내부 테스트**: QA 팀, 관리자, 개발자가 사용 흐름 테스트
- **대량 수동 배포**: VIP 고객이나 이벤트 참석자에게 선불 잔액 제공

## 기프트카드 발송 사전 요구사항

기프트카드를 발송하기 전에 다음을 확인하세요:

- 기프트카드 상품(Gift Card Product)이 존재할 것
- 기프트카드 상품이 활성(Active) 상태일 것 (비활성화되거나 보관되지 않음)
- 기프트카드를 받을 고객이 다음 중 하나를 보유할 것:
  - 이메일 주소, 또는
  - 전화번호 (SMS 전달용)

## 기프트카드 발송하는 방법

### 1단계: 기프트카드 메뉴로 이동

- 하위 계정에 로그인합니다.
- Payments(결제) → Gift Cards(기프트카드)로 이동합니다.

![](https://jumpshare.com/share/WO8nSGZWS8RLjB2nvM4m+/Screen+Shot+2025-11-26+at+9.02.39+PM.png)

- 기프트카드를 생성하거나 **All Gift Cards(전체 기프트카드)** 목록에서 **기존 기프트카드를 선택**합니다.

![](https://jumpshare.com/share/lwAqTaYBJlHZUyLQOnHW+/Screen+Shot+2025-11-26+at+9.07.49+PM.png)

- 발송하려는 기프트카드에서 Send(발송) 버튼을 클릭합니다. 이제 발송 절차가 시작됩니다.

![](https://jumpshare.com/share/BfuXE2nejTPExaY10OSk+/Screen+Shot+2025-11-26+at+9.10.09+PM.png)

### 2단계: 고객/연락처 선택

- 수신자 선택: 목록에서 기존 고객/연락처를 선택합니다.
- 또는 발송 전에 **연락처 추가(Add a Contact)**를 합니다.
- **Continue(계속)**를 클릭합니다.

![](https://jumpshare.com/share/loy8HURAsCLH25bOIfTb+/Screen+Shot+2025-11-26+at+9.15.10+PM.png)

### 3단계: 개인 메시지 추가

- 맞춤 인사말, 특별 지시사항 또는 감사 메시지를 작성합니다.
- 또는 미리 작성된 메시지 중에서 선택합니다.
- 또는 Write with AI(AI로 작성)를 사용하여 메시지를 생성합니다.
- Continue(계속)를 클릭합니다.

![](https://jumpshare.com/share/wUpCkc5IeaOSTyVzG2rN+/Screen+Shot+2025-11-26+at+9.21.05+PM.png)

### 4단계: 금액 선택

- 상품에 설정된 **미리 정의된 금액** 중 하나를 선택합니다.

![](https://jumpshare.com/share/0Jfwr6zimU6IGHcdZIMo+/Screenshot+2025-11-26+at+9.29.28%E2%80%AFPM.png)

### 5단계: 전달 방법 선택

참고: **고객의 이용 가능한 정보**(이메일/전화)가 지원하는 방법만 활성화됩니다.
다음 중 하나의 방법으로 기프트카드를 전달할 수 있습니다:

- 이메일
- SMS
- QR 코드 (이메일로)
- PDF (이메일로)

![](https://jumpshare.com/share/2s6Cy4C4QE85akAC0ifg+/Screen+Shot+2025-11-26+at+9.39.03+PM.png)

### 6단계: 전달 날짜 선택

참고: 기프트카드의 **이용 가능한 전달 날짜 범위**는 카드의 만료 설정에 따라 결정됩니다. 기프트카드에 **만료일이 있는** 경우, 전달 날짜는 오늘부터 만료일 사이여야 합니다. 기프트카드에 만료일이 없으면 **미래의 어떤 날짜든** 예약할 수 있습니다.

다음 중 선택할 수 있습니다:

- 미리 설정된 옵션: 즉시 전달, 내일, 또는 1주일 후
- 또는 미래 날짜로 전달 예약
- Continue(계속)를 클릭합니다.

![](https://jumpshare.com/share/fXBLYnUAf7ab0c2nbQEB+/GIF+Recording+2025-11-26+at+9.54.26+PM.gif)

### 7단계: 검토 및 발송

- 확인하기 전에 **자동 생성된 이메일 또는 SMS 미리보기**를 **검토**합니다. 여기에는 다음이 포함됩니다:
  - 기프트카드 금액
  - 커스텀 메시지
  - 만료일
  - 사용 안내
  - 아트워크

- Send Gift Card(기프트카드 발송)를 클릭합니다.

![](https://jumpshare.com/share/Dx07l8CphQzWnfDpO63v+/GIF+Recording+2025-11-26+at+10.20.40+PM.gif)

## 전달 방법 설명

### 이메일

- 아트워크가 포함된 완전한 시각적 기프트카드
- 만료일, 잔액, 사용 링크 포함
- 대부분의 경우에 권장

![](https://jumpshare.com/share/nE4p8Ho3cp0j09xXcDCu+/Screen+Shot+2025-11-26+at+10.40.12+PM.png)

### SMS

- 기프트카드 링크가 포함된 깔끔한 메시지 전송
- SMS 형식 제한으로 아트워크 미포함

![](https://jumpshare.com/share/zcbVS0C3sqcyIU6lnTJd+/Screen+Shot+2025-11-26+at+10.42.47+PM.png)

### PDF (이메일로)

- 다운로드 가능한 인쇄용 PDF 전송
- 매장 선물, 실물 포장, 이벤트에 적합

![](https://jumpshare.com/share/4J3mtirC8dkp0JPOZJNp+/Screen+Shot+2025-11-26+at+10.44.51+PM.png)

### QR 코드 (이메일로)

- 기프트카드를 여는 스캔 가능한 QR 코드 전송
- 대면 선물이나 이벤트에 유용

![](https://jumpshare.com/share/R9SYiFnfOc0WsAnmTeVP+/GIF+Recording+2025-11-26+at+10.47.14+PM.gif)

## 기프트카드 발송 후 처리 과정

발송된 기프트카드는 다음 위치에 나타납니다:

Payments(결제) → Gift Cards(기프트카드) → Gift Card Orders(기프트카드 주문)

![](https://jumpshare.com/share/sZYp5xQTee7yLigsfBll+/Screen+Shot+2025-11-26+at+10.50.18+PM.png)

기프트카드 주문 상세 정보에는 다음이 포함됩니다:

- 수신자 세부정보
- 전달 방법
- 전달 타임스탬프
- 금액
- 만료일
- 남은 잔액
- 상태: Scheduled(예약됨), Active(활성), Inactive(비활성)
- 숨겨진 기프트카드 코드

![](https://jumpshare.com/share/OWNNCXZESfhm7DusXHwM+/Screen+Shot+2025-11-26+at+10.53.18+PM.png)

## 주요 특징 및 제한사항

- 발송 기능은 라이브 모드에서만 작동
- 한 번에 1개의 기프트카드를 1명의 고객에게 발송 가능
- 발송 과정에서 결제가 이루어지지 않음
- 직접 발송된 기프트카드는 Payments(결제) → Orders(주문)에 나타나지 않음 (주문 없음, 거래 없음)
- 발송을 통해 발행된 기프트카드는 모든 곳에서 완전히 사용 가능 - 전액 및 부분 사용 모두 가능
- 전달을 위해 고객의 이메일 또는 전화번호가 필요

## 자주 묻는 질문

**Q: 테스트 모드에서 기프트카드를 발송할 수 있나요?**
아직 지원되지 않습니다. 발송 기능은 라이브 모드만 지원합니다. 테스트 모드 지원은 향후 버전에서 제공될 예정입니다.

**Q: 일부 고객에게 기프트카드를 발송할 수 없는 이유는 무엇인가요?**
고객이 다음을 보유해야 합니다:
- 유효한 이메일, 또는
- 유효한 전화번호
그렇지 않으면 전달 방법을 사용할 수 없습니다.

**Q: 여러 기프트카드를 한 번에 발송할 수 있나요?**
현재는 지원되지 않습니다. 발송 과정에서 한 번에 한 명의 고객에게 한 개의 기프트카드만 발송할 수 있습니다. 대량 발송은 향후 버전에서 계획되어 있습니다.

**Q: 발송된 기프트카드는 모든 곳에서 사용할 수 있나요?**
네, 기프트카드 사용을 지원하는 모든 결제 페이지에서 사용할 수 있습니다:
- 퍼널
- 폼
- 인보이스
- 스토어
- 결제 링크
- 캘린더 (상품이 지원되는 경우)

**Q: 예약된 기프트카드가 전달 시간 전에 어떻게 처리되나요?**
예약된 전달 시간까지 기프트카드는 Scheduled(예약됨) 상태로 유지되지만 여전히 활성 상태이며 사용 가능합니다.

**Q: 예약된 기프트카드를 전달 시간 전에 사용할 수 있나요?**
네. 누군가 기프트카드 코드를 가지고 있어서 예약된 전달 시간 전에 결제 페이지에서 사용하려고 하면 정상적으로 작동합니다.

**Q: 발송 전에 이메일/SMS 템플릿을 편집할 수 있나요?**
템플릿은 시스템에서 자동 생성되며 현재 편집할 수 없습니다. 이메일 및 SMS 템플릿 사용자 정의 기능은 나중에 제공될 예정입니다.

**Q: 발송된 기프트카드가 결제의 주문 목록에 나타나나요?**
아니요, 구매된 기프트카드만 Payments(결제) → Orders(주문)에 나타납니다. 발송된 카드는 기프트카드 주문(Gift Card Orders)에만 나타납니다.

**Q: 기프트카드를 익명으로 발송할 수 있나요?**
아직 지원되지 않습니다. 수신자는 이메일/SMS 템플릿에 설정된 발신자의 비즈니스 정보를 항상 볼 수 있습니다.
