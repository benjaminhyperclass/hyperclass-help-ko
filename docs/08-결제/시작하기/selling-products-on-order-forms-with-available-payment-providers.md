# 주문 폼에서 결제 서비스를 통해 상품 판매하기

퍼널 버전 2에서는 1단계 및 2단계 주문 폼(Order Form)을 통해 일회성 상품과 구독 상품을 모두 판매할 수 있습니다. (참고: 새로 생성되는 모든 퍼널과 웹사이트는 버전 2로 만들어집니다) 이 가이드에서는 주문 폼에서 상품을 판매하는 다양한 사용 사례와 결제 흐름을 설계할 때 고려해야 할 제한사항들을 다룹니다.

### **사용 가능한 결제 서비스**

주문 폼에서 상품 판매에 사용할 수 있는 다양한 결제 서비스가 있습니다:

- Stripe 
- Authorize.net
- NMI
- PayPal 
- Square
- 커스텀 결제 서비스 (마켓플레이스에서 설치)

### 주문 폼 판매 옵션

- **메인/기본 상품** - 주문 폼에서 판매하는 주요 상품입니다.
- **범프 상품** - 연락처/리드가 기본 상품을 구매할 때 체크할 수 있는 추가 옵션으로 제공됩니다.
- **업셀** - 리드가 기본 상품을 구매한 후 결제 완료 직후 버튼 클릭 한 번으로 추가 서비스/상품을 즉시 제안하는 방식입니다.

### 주의사항

- Apple Pay와 Google Pay는 Stripe에서만 지원됩니다. Apple Pay를 결제 방법으로 표시하려면 `Payments(결제) → Integrations(연동)`의 Stripe 연결 카드에서 토글을 켜야 합니다.
- 구글페이는 업체가 Stripe에 연결되어 있으면 기본적으로 표시됩니다(구매자의 국가에서 지원하는 경우).
- 서로 다른 구독 상품은 어떤 결제 서비스를 사용하더라도 단일 결제에서 함께 판매할 수 없습니다. 하나의 흐름에서 두 개의 서로 다른 구독 상품을 판매하려면 업셀 구매를 활용해야 합니다.
- PayPal은 리드가 구독 상품을 구매할 때 범프 상품을 함께 구매하는 것을 지원하지 않습니다.
- `Payments(결제) → Products(상품)`에서 구독 상품을 만들 때 설정 수수료, 체험 기간, 결제 횟수를 설정할 수 있으며, 리드가 해당 상품을 구매할 때 이 설정이 자동 적용됩니다.
- 제출된 모든 주문은 `Payments(결제) → Orders(주문)`에서, 생성된 구독은 `Payments(결제) → Subscriptions(구독)`에서 추적할 수 있고, 환불은 거래 테이블에서 관리할 수 있습니다.
- 주문 시 상품에 체험 기간이 있고 체험 기간 시작 전에 명시적으로 청구되는 설정 수수료가 없다면, 주문 완료 시 차감이나 청구가 적용되지 않으며 금액은 '0'으로 표시됩니다. 요금은 체험 기간이 끝날 때 적용됩니다.

## PayPortal 기능

**결제 방법 선택:** 
- Stripe (카드)
- Stripe > Apple Pay / Google Pay
- Stripe > Affirm/Klarna/Afterpay
- Stripe > ACH Direct Debit
- PayPal
- Square (카드)
- Razor Pay
- 서드파티 연동
