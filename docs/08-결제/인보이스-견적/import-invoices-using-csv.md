# CSV를 사용한 인보이스 가져오기

이 가이드에서는 CSV 파일을 사용하여 외부 플랫폼에서 인보이스(Invoice)를 빠르게 가져오는 방법을 알려드립니다. 다른 시스템에서 이전하거나 과거 청구 데이터를 정리할 때, 이 기능으로 몇 시간의 수작업을 절약할 수 있어요.

## CSV 인보이스 가져오기란?

CSV 인보이스 가져오기는 스프레드시트나 다른 플랫폼에서 내보낸 외부 인보이스 데이터를 Hyperclass 계정으로 직접 불러오는 기능입니다. 가져오기 과정에서 발행일, 만료일, 세금 등의 필수 필드를 매핑할 수 있으며, 가져오기 결과에 대한 자동 요약을 받을 수 있어요. 이를 통해 기존 시스템에서 원활하게 이전하고 모든 청구 정보를 한 곳에서 통합 관리할 수 있습니다.

## CSV 인보이스 가져오기의 주요 장점

CSV 파일을 사용한 인보이스 가져오기는 운영을 간소화하고 중복 데이터 입력을 없애는 다양한 효율성과 정확성 이점을 제공합니다.

- 다른 청구 시스템이나 스프레드시트에서 인보이스 데이터를 빠르게 이전할 수 있어요.

- 발행일, 만료일, 세금 값 등 필수 필드를 매핑할 수 있어요.

- 가져오기 후 투명성을 위한 이메일 요약을 자동으로 받을 수 있어요.

- 수동 입력 오류 위험을 줄일 수 있어요.

- 인보이스 기록을 한 곳에 집중화하여 보고서 정확성을 향상시킬 수 있어요.

## CSV를 사용한 인보이스 가져오기 방법

인보이스 가져오기는 데이터를 업로드하고 올바르게 매핑한 다음 결과를 검토하는 간단한 과정입니다. 다음 단계를 따라 가져오기를 완료하세요.

1. `Payments(결제) → Invoices & Estimates(인보이스 및 견적)`로 이동하세요.

2. 우측 상단의 `+ New` 버튼을 클릭하세요.

3. 드롭다운 메뉴에서 `Import Invoices using CSV`를 선택하세요.

4. 필요한 컬럼 구조를 이해하기 위해 샘플 CSV 파일을 다운로드하세요.

5. 샘플 형식에 맞춰 CSV 파일을 준비하세요.

6. `Settings(설정) → Taxes(세금)`로 이동해서 해당되는 모든 세율이 이미 플랫폼에 생성되어 있는지 확인하세요.

7. CSV 파일을 업로드하세요.

8. CSV의 각 컬럼을 적절한 인보이스 필드에 매핑하세요. 날짜 필드(발행일, 만료일)는 올바른 날짜 형식을 사용해야 합니다.

9. 시스템이 세금 관련 컬럼을 감지하면 기존 세율에 매핑하라는 메시지가 표시됩니다.

10. `Start Import`를 클릭하여 프로세스를 시작하세요.

11. 가져온 인보이스와 실패한 인보이스의 세부 정보가 포함된 인보이스 가져오기 요약 이메일을 받게 됩니다.

## 세금 매핑

적절한 세금 매핑은 가져온 인보이스가 기존 세금 구성과 일치하는 정확한 세금 값을 반영하도록 보장합니다.

가져오기 중에 시스템이 세금 관련 컬럼을 감지하면:

- 감지된 각 컬럼은 `Settings(설정) → Taxes(세금)`에서 이전에 생성된 세율과 일치시켜야 합니다.

- 일치하는 세율이 존재하지 않으면 매핑 프로세스를 완료하기 전에 추가해야 합니다.

- 세금 필드가 매핑되지 않은 상태로 남아있으면 가져오기 프로세스가 일시정지되어 부정확한 데이터가 가져오지 않도록 합니다.

## 자동 가져오기 요약 이메일

가져오기가 완료된 후, 등록된 이메일 주소로 자동 요약 이메일이 전송됩니다. 이 메시지에는 다음이 포함됩니다:

- 성공적으로 가져온 인보이스 수

- 실패한 항목의 세부 목록과 각 실패의 구체적인 이유

- 각 인보이스 행의 가져오기 상태를 보여주는 CSV 파일 첨부

## 중요 참고사항

다음 주요 특징들은 가져온 인보이스와 플랫폼에서 직접 생성된 인보이스를 구분합니다.

**자동 알림 없음**
가져온 인보이스는 자동 알림 이메일이나 문자 메시지를 트리거하지 않습니다. 알림 워크플로우는 플랫폼 내에서 수동으로 생성된 새 인보이스에만 적용됩니다.

**연체료와 팁**
가져온 인보이스는 설정된 기본값 또는 전역 설정에 따라 연체료와 팁이 적용됩니다. 특정 인보이스에 대해 이러한 기능을 비활성화하려면 가져오기 후 수동으로 편집해야 합니다.

## 자주 묻는 질문

**Q: 여러 세금 유형이 있는 인보이스를 가져올 수 있나요?**
네. CSV 파일에서 각 세금 유형이 적절히 라벨링되고 `Settings(설정) → Taxes(세금)` 하위 계정에 이미 존재한다면, 시스템에서 매핑하고 가져올 수 있습니다.

**Q: 가져온 인보이스가 고객에게 이메일이나 알림을 보내나요?**
아니요. 가져온 인보이스는 과거 기록으로 처리되며 알림이나 안내 이메일을 트리거하지 않습니다.

**Q: 가져오기 도구를 사용해서 기존 인보이스를 업데이트하거나 덮어쓸 수 있나요?**
아니요. CSV 가져오기 도구는 새 인보이스를 생성하는 용도로만 사용됩니다. 이전에 생성된 인보이스를 업데이트하거나 교체할 수 없습니다.

**Q: 가져오기 중에 일부 행이 실패하면 어떻게 되나요?**
가져오기에 실패한 행은 잘못된 날짜 형식이나 일치하지 않는 세금 값 등의 구체적인 실패 이유와 함께 요약 이메일에 나열됩니다.

**Q: 한 번에 가져올 수 있는 인보이스 수에 제한이 있나요?**
엄격한 상한선은 없지만, 최적의 성능과 더 쉬운 오류 관리를 위해 한 번에 500개 이하의 배치로 가져오기를 권장합니다.
