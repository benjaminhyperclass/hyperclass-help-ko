# **인보이스의 ACH 직불 결제 방식이란?**

ACH 직불 결제 방식은 기업이 ACH(자동 결제 시스템) 이체를 통해 고객의 은행 계좌에서 직접 결제를 받을 수 있는 기능입니다. 이 기능은 기존 신용카드 결제에 대한 원활하고 비용 효율적인 대안을 제공하며, 특히 큰 금액의 거래에 유용합니다. Stripe와 연동되어 플랫폼에서 생성된 인보이스에 사용할 수 있으며, 여러분과 고객 모두에게 간소화된 결제 경험을 제공합니다.

ACH 직접 직불을 통해 고객은 자신의 은행 계좌에서 직접 일회성 또는 반복 결제를 승인할 수 있어, 안정적인 현금 흐름을 보장하고 수동 결제 알림에 대한 의존도를 줄일 수 있습니다.

---

## 인보이스 ACH 직불 결제 방식의 주요 장점

- **비용 효율적인 결제 솔루션**: ACH 거래는 일반적으로 신용카드 결제보다 낮은 처리 수수료가 발생하여 기업의 거래 비용을 절약할 수 있습니다.

- **고객 편의성 향상**: 고객이 신용카드나 직불카드를 사용하지 않고도 은행 계좌에서 직접 결제할 수 있어 결제 경험이 개선됩니다.

- **Stripe와의 원활한 연동**: 연동을 통해 안전하고 규정을 준수하는 결제 처리가 가능하며, 기업이 Stripe 대시보드에서 직접 거래와 결제 방식을 관리할 수 있습니다.

- **대규모 거래 지원**: ACH 결제는 신용카드 한도를 우회하고 큰 금액 결제를 위한 안전한 메커니즘을 제공하므로 고액 거래에 이상적입니다.

- **반복 인보이스 자동 결제**: 기업은 반복 인보이스에 대해 자동 결제를 활성화하여 수동 개입 없이 적시에 결제를 보장할 수 있습니다.

---

## 인보이스 ACH 직불 결제 방식 설정 방법

- **Stripe에서 ACH 직접 직불 활성화**:
  - Stripe 계정에 로그인합니다.
  - **Settings(설정) → Payment Methods(결제 방식)**으로 이동하여 계정에서 ACH 직접 직불이 활성화되어 있는지 확인합니다.

- **플랫폼과 Stripe 연동**:
  - 플랫폼에서 **Settings(설정) → Integrations(연동)**으로 이동합니다.
  - **Connect Stripe** 버튼을 클릭하고 안내에 따라 로그인하여 Stripe 계정을 연결합니다.

- **인보이스 결제 옵션 구성**:
  - 인보이스 생성 시 ACH 직접 직불을 결제 방식 옵션으로 선택합니다.
  - 해당되는 경우 반복 인보이스에 대해 **자동 결제** 기능을 활성화합니다.

- **고객에게 인보이스 발송**:
  - 설정이 완료되면 고객이 인보이스를 확인할 때 ACH 직불 옵션을 볼 수 있습니다.
  - 고객은 자신의 은행 계좌 정보를 입력하여 안전하게 결제를 승인할 수 있습니다.

- **Stripe에서 결제 모니터링**:
  - Stripe 대시보드를 사용하여 ACH 결제를 추적하고, 실패한 거래를 해결하며, 필요에 따라 고객 결제 정보를 업데이트합니다.

## 자주 묻는 질문

**Q:** Stripe를 통한 ACH 직접 직불 결제 수락과 관련된 수수료가 있나요?

네, Stripe는 ACH 직접 직불 결제 처리에 대해 일반적인 카드 거래 수수료와 다른 특정 수수료를 적용합니다. 최신 요금 정보는 Stripe 공식 요금 페이지를 참조하세요.

**Q: 자동 결제가 활성화된 반복 인보이스에 ACH 직접 직불을 사용할 수 있나요?**

네, 자동 결제가 활성화된 반복 인보이스에 ACH 직접 직불을 사용할 수 있습니다. 반복 인보이스를 설정할 때 인보이스 발송 과정에서 자동 결제 옵션을 토글하여 자동 결제를 활성화할 수 있습니다. 이렇게 하면 이후 결제가 예정된 결제 날짜에 자동으로 청구됩니다.

**Q: 인보이스에서 고객에게 표시되는 결제 방식을 어떻게 관리할 수 있나요?**

Stripe 대시보드에서 설정을 구성하여 고객에게 표시되는 결제 방식을 관리할 수 있습니다. Stripe에서 Settings(설정) → Connect(연결) → Payment Methods(결제 방식)로 이동하여 적절한 구성(예: Invoice, InvoiceWithAutopayment)을 선택하고 원하는 대로 특정 결제 방식을 활성화하거나 비활성화합니다.

**Q: 잔액 부족이나 기타 이유로 ACH 직접 직불 결제가 실패하면 어떻게 해야 하나요?**

ACH 직접 직불 결제가 실패하면 Stripe에서 잔액 부족이나 유효하지 않은 계좌 번호와 같은 실패 이유를 알려드립니다. 문제를 해결하기 위해 고객에게 연락하는 것이 좋으며, 은행 계좌 정보 업데이트나 대체 결제 방식 선택이 필요할 수 있습니다.

**Q: 향후 결제를 위해 고객의 ACH 은행 계좌를 어떻게 저장할 수 있나요?**

고객이 ACH 직접 직불을 사용하여 인보이스를 결제하고 자동 결제를 선택하면(사용 가능한 경우), Stripe가 향후 사용을 위해 은행 계좌를 결제 방식으로 안전하게 저장합니다.

자동 결제를 선택하지 않은 경우, Stripe의 고객 관리 기능을 사용하여 재사용을 위해 은행 계좌를 수동으로 저장해야 합니다. 또한 고객에게 은행 계좌를 기본적으로 저장하는 결제 링크를 사용하여 결제하도록 요청할 수도 있습니다(예: 반복 결제 설정).
