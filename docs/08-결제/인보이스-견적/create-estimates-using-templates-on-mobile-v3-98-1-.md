# 모바일에서 템플릿으로 견적서 생성하기(v3.98.1)

팀이 웹 앱에서 만들어둔 템플릿을 재사용하여 이동 중에도 전문적인 견적서를 빠르게 생성할 수 있습니다. 모바일 앱에서 템플릿을 사용하면 영업 프로세스가 간소화되고, 가격과 브랜딩의 일관성이 유지되며, 현장 팀이 몇 분 만에 견적서를 작성하고 발송할 수 있습니다. 이 가이드에서는 기능 소개, 장점, 설정 방법, 다음 단계를 안내해드립니다.

## 모바일 결제: 템플릿으로 견적서 생성이란?

모바일 견적서 템플릿은 웹 앱에 미리 만들어둔 템플릿을 사용하여 새 견적서를 시작할 수 있는 기능입니다. 모바일 앱이 웹 템플릿을 읽어와서 담당자가 레이아웃, 항목, 조건을 처음부터 다시 만들지 않고도 일관성 있고 오류 없는 견적서를 생성할 수 있습니다.

---

## 모바일 결제: 템플릿으로 견적서 생성의 주요 장점

장점을 이해하면 템플릿 사용과 직접 작성 중 언제 어떤 방식을 선택할지 결정하는 데 도움이 됩니다. 템플릿은 속도, 브랜드 일관성, 예측 가능한 가격 책정이 필요할 때 특히 유용합니다.

- **시간 절약:** 미리 만들어둔 템플릿으로 몇 번의 터치만으로 새 견적서를 작성할 수 있습니다.

- **일관성:** 담당자 간 브랜딩, 섹션, 가격 형식을 통일할 수 있습니다.

- **이동성:** 고객과 함께 있는 어디서든 견적서를 생성하고 발송할 수 있습니다.

- **원활한 접근:** 웹에서 만든 템플릿을 모바일에서 자동으로 사용할 수 있습니다.

- **전문성:** 수동 오류를 줄이고 매번 완성도 높은 견적서를 제시할 수 있습니다.

## 사용 가능 조건 및 필수사항

미리 준비사항을 확인하면 나중에 설정 문제를 예방할 수 있습니다. 앱 버전과 템플릿 관리 위치를 확인하여 팀이 즉시 모바일에서 사용할 수 있도록 하세요.

- 모바일 앱 버전: **Hyperclass v3.98.1** 또는 **LeadConnector v3.98.1**

- 화이트라벨 앱: 계정 담당자/지원팀에 **v3.98.1 업데이트 요청**이 필요합니다.

- 템플릿은 **웹에서 생성하고 관리**합니다. 모바일에서는 템플릿을 **사용**만 할 수 있고 생성/편집/삭제는 불가능합니다.

- 사용자가 하위 계정과 결제(Payments) > 견적서(Estimates)에 접근 권한이 있는지 확인하세요.

## 자주 묻는 질문

**Q: 모바일 앱에서 견적서 템플릿을 생성하거나 편집할 수 있나요?**

아니요. 템플릿은 **웹**에서만 생성하고 관리할 수 있습니다. 모바일 앱에서는 해당 템플릿을 사용하여 새 견적서를 작성만 할 수 있습니다.

**Q: 웹에서 만든 템플릿이 모바일 앱에 자동으로 표시되나요?**

네. 웹에서 생성한 템플릿은 모바일 앱의 **템플릿(Templates)** 탭에 자동으로 나타납니다.

**Q: 이 기능을 지원하는 버전은 무엇인가요?**

**Hyperclass v3.98.1**, **LeadConnector v3.98.1**, 그리고 **v3.98.1 업데이트를 요청**한 화이트라벨 앱에서 사용할 수 있습니다.

**Q: 템플릿을 선택한 후 항목을 변경할 수 있나요?**

네. 견적서를 발송하기 전에 새 견적서에서 지원되는 필드(수량, 가격 등)를 편집할 수 있습니다.

**Q: 템플릿 탭이 보이지 않는데 무엇을 확인해야 하나요?**

앱 버전이 **v3.98.1**인지, 올바른 **하위 계정**에 있는지, 사용자가 결제(Payments) > 견적서(Estimates)에 접근 권한이 있는지 확인하세요. 화이트라벨인 경우 v3.98.1 빌드가 배포되었는지 확인하세요.

**Q: 템플릿 첨부파일이 모바일에서도 그대로 유지되나요?**

첨부파일은 일반적인 견적서/템플릿 규칙을 따릅니다. 자세한 내용과 제한사항은 아래 연결된 **첨부파일** 가이드를 참고하세요.

**Q: 고객이 승인할 때 즉시 결제를 활성화하려면 어떻게 하나요?**

모바일 견적서에서 **직접 결제(Direct Payments)**를 사용하면 고객이 승인과 결제를 한 번에 처리할 수 있습니다(아래 링크 참고).
