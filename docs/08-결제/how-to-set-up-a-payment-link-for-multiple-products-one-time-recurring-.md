# 여러 상품을 위한 결제 링크 설정하기 (일회성 & 구독 상품)

이 문서에서는 일회성 상품과 구독 상품을 모두 포함하는 하이퍼클래스 결제 링크(Payment Link)를 설정하는 방법을 안내합니다. 결제 과정을 간소화하고, 여러 상품을 묶어서 판매하며, 결제를 효율적으로 관리하려는 사용자를 위해 설계되었습니다. 디지털 상품, 구독 서비스, 서비스 패키지를 판매하든 - 이 가이드를 통해 모든 것을 한 곳에서 설정할 수 있습니다.

# **여러 상품 결제 링크란 무엇인가요?**

하이퍼클래스의 결제 링크는 여러 상품(일회성 상품과 구독 상품 모두)을 하나의 결제 경험으로 결합할 수 있게 해줍니다. 이는 패키지 상품, 디지털 상품, 구독 서비스를 제공하는 비즈니스에 특히 유용합니다. 각 항목별로 별도의 링크를 만드는 대신, 모든 것을 하나의 통합된 결제 흐름으로 관리하여 고객 경험과 백엔드 추적을 개선할 수 있습니다.

---

## 여러 상품 결제 링크 사용의 주요 장점

- **통합된 결제**: 고객이 모든 일회성 상품과 구독 상품 옵션을 하나의 정리된 인터페이스에서 볼 수 있습니다.

- **향상된 영업 전략**: 업셀과 크로스셀 옵션을 함께 제시하여 총 주문 금액을 증대시킵니다.

- **깔끔한 고객 경험**: 구독 상품은 한 번에 하나씩 선택 가능하며, 일회성 상품은 다중 선택이 가능합니다.

- **관리해야 할 링크 수 감소**: 모든 상품 제안을 하나의 공유 가능하고 추적 가능한 링크로 묶습니다.

- **커스텀 브랜딩 및 동작**: 페이지 디자인, URL, 구매 후 동작을 브랜드에 맞게 제어할 수 있습니다.

## **여러 상품 결제 링크 설정 방법**

### **결제 링크 섹션 접근하기**

좌측 메뉴에서 Payments(결제) 탭으로 이동한 후, 화면 상단의 Payment Links(결제 링크)를 클릭합니다. 여기서 기존 결제 링크와 새 결제 링크를 생성, 관리, 모니터링할 수 있습니다.

![결제 링크 섹션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600532/original/-4K_Wq7MBjQbV2qbIdygu9XOjxGBFrIVNg.png?1750410002)

### 새 결제 링크 생성하기

결제 링크 대시보드 우측 상단의 + Create New Payment Link(+ 새 결제 링크 생성) 버튼을 클릭합니다. 이제 상품을 선택하고, 결제 화면을 커스터마이징하며, 추가 옵션을 설정하는 설정 과정이 시작됩니다.

![새 결제 링크 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600598/original/cjUFrlzCpJYslgapPWF5y1O9W6M6oTCkqQ.png?1750410072)

### 상품 선택하기

결제 링크에 포함할 첫 번째 상품을 선택합니다. 상품 라이브러리에서 일회성 상품과 구독 상품 모두 선택할 수 있습니다. 추가 상품을 더하거나 결제 설정을 추가로 구성하기 전에 이 초기 선택이 필요합니다.

![상품 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600644/original/wQxAwsPoZ_D3amZMFYn01DBl-SZBIHnv4w.png?1750410107)

### 다른 상품 추가하기

+ Add Another Product(+ 다른 상품 추가) 버튼을 클릭하여 같은 결제 링크에 여러 상품을 포함시킵니다. 여러 일회성 상품과 하나의 구독 상품을 결합하여 고객이 하나의 결제 세션에서 다중 선택을 할 수 있게 합니다. 구독 상품은 결제 페이지에서 별도로 그룹화됩니다.

![다른 상품 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600727/original/_CiLXvT6B-RwQFRQD7b6Zn8td3jCrM1Mag.png?1750410157)

### **상품 수량 선택 활성화**

**Let customers adjust quantities(고객이 수량을 조정하도록 허용)** 체크박스를 사용하여 구매자가 일회성 상품의 구매 수량을 선택할 수 있도록 합니다. 최소 및 최대 한계를 설정하여 대량 주문을 제어할 수 있습니다. 이러한 유연성은 여러 항목이 필요한 고객의 결제 경험을 개선합니다.

![수량 선택 활성화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600756/original/BfjtTmvOSX5WiuqZWT08CWZ3KNmsu1NVCQ.png?1750410195)

### **고객 전화번호 수집**

Require customers to add a phone number(고객이 전화번호를 추가하도록 요구) 옵션을 활성화하여 결제 중 전화번호 수집을 필수로 만듭니다. 이는 후속 조치, 서비스 조정 또는 확인 목적에 도움이 됩니다. 전화번호 필드는 양식의 다른 연락처 세부 정보와 함께 표시됩니다.

![전화번호 수집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600801/original/5vp0oMLXrfzSg1GoqGqnXS1U8V7i5ijLeg.png?1750410235)

### 고객 주소 수집

세부적인 배송 또는 청구 정보가 필요한 경우 Collect customer addresses(고객 주소 수집) 옵션을 활성화합니다. 선택하면 거리, 도시, 주, 국가, 우편번호 등의 주소 필드가 결제 페이지에 표시됩니다. 실제 상품 배송이나 규정 준수 요구사항에 이상적입니다.

![고객 주소 수집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600876/original/LoixILuUX1P3pohukwL8jX1YycZG73oapA.png?1750410269)

### **결제 시 쿠폰 코드 활성화**

Allow coupon codes(쿠폰 코드 허용) 옵션을 선택하여 결제 페이지에 쿠폰 코드 필드를 표시합니다. 이를 통해 고객이 구매 시 할인 코드를 적용할 수 있어 프로모션 캠페인을 진행하고 더 높은 전환율을 유도할 수 있습니다.

![쿠폰 코드 활성화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600962/original/xyC73GpDSPrUvi4LbXP_qSKfZNHNjuwLHw.png?1750410324)

### **구매 후 리다이렉트 URL 추가**

Enable redirection to custom URL(커스텀 URL로 리다이렉트 활성화) 옵션을 활성화하여 결제 완료 후 고객을 특정 페이지로 자동 안내합니다. 리다이렉트 대상을 정의하고 새 탭에서 열지 또는 현재 탭에서 열지를 선택할 수 있습니다. 감사 페이지나 다음 단계 안내에 이상적입니다.

![리다이렉트 URL 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048600983/original/v2m03c5md8G2K14FeQZNzCDVlvis1X2Gzw.png?1750410364)

### **CTA 버튼 텍스트 커스터마이징**

Call to action for payment link button(결제 링크 버튼의 CTA) 아래 드롭다운을 사용하여 상품에 따라 버튼 라벨을 개인화합니다. "Pay(결제)", "Book(예약)", "Donate(기부)" 등의 옵션에서 선택하여 CTA를 상품 유형이나 사용자 여정에 맞춥니다. 이는 올바른 기대를 설정하고 전환 명확성을 향상시킵니다.

![CTA 버튼 텍스트](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601002/original/03PzJFoAfUkcIf36kBteTwAiMx_UsfqH3w.png?1750410394)

### **브랜딩 메시지 추가**

Branding(브랜딩) 옵션을 활성화하여 결제 페이지에 커스텀 "Powered by" 라벨을 포함시킵니다. 텍스트를 편집하여 에이전시나 비즈니스 이름을 표시하고, 결제 경험 중 브랜드 가시성과 신뢰를 강화할 수 있습니다.

![브랜딩 메시지](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601034/original/eJWkFLuqr5KLotXLkAIVcIIdlk2mKxh-VA.png?1750410427)

### **이용약관 추가**

Add Terms and Conditions(이용약관 추가) 옵션을 사용하여 거래에 대한 중요한 법적 또는 정책 관련 정보를 포함시킵니다. 메시지는 결제 페이지의 Pay(결제) 버튼 바로 아래에 표시되어 고객이 구매를 완료하기 전에 주요 약관을 검토할 수 있도록 합니다. 명확성과 강조를 위해 텍스트를 포맷할 수 있습니다.

![이용약관 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601053/original/9t7fgIL0owliEUJ56VyapoVfKby-bGMfIg.png?1750410448)

### **자동 비활성화 날짜 설정**

Automatic Deactivation(자동 비활성화) 설정을 사용하여 결제 링크가 만료될 날짜를 지정합니다. 이는 기간 한정 오퍼, 제한된 캠페인 또는 구독 창에 유용합니다. 선택된 날짜에 도달하면 링크는 더 이상 고객이 접근할 수 없게 됩니다.

![자동 비활성화 날짜](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601113/original/Nk2MWmjph7o4kRkJ1QlQB08lXGSIHXcGhA.png?1750410487)

### **테스트와 실제 결제 모드 선택**

Payment Mode(결제 모드) 토글을 사용하여 Test(테스트)와 **Live(실제)** 환경 사이를 전환합니다. 테스트 모드에서는 실제 거래를 처리하지 않고 테스트 카드 번호를 사용하여 결제를 시뮬레이션할 수 있습니다. 고객으로부터 실제 결제를 받을 준비가 되면 실제 모드로 전환합니다.

![결제 모드 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601161/original/jwui0qEAgW9MWF_4o-uafCRNZ3JGWJ_icQ.png?1750410517)

### **결제 링크 이름 지정**

화면 상단의 필드를 사용하여 결제 링크의 커스텀 이름을 입력합니다. 이 내부 이름은 대시보드에서 링크를 정리하고 식별하는 데 도움이 됩니다. 고객이 보는 결제 페이지에는 영향을 주지 않습니다.

![결제 링크 이름](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601199/original/E1evz36kl3gPxes3NuHXlfUs4Yp_HP2VkA.png?1750410561)

### **결제 링크 공유하기**

**Copy / Send(복사/전송)** 버튼을 클릭하여 결제 링크를 배포하는 옵션에 접근합니다. 어떤 고객과도 공유할 수 있는 일반 링크를 복사하거나 하이퍼클래스 내에서 이메일이나 SMS를 통해 개인화된 버전을 직접 보낼 수 있습니다.

![결제 링크 공유](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601261/original/Yt1Vv9QawWQkYRxAo_fuql2M_V0q4EpAHA.png?1750410598)

### **결제 링크 복사 또는 개인화**

일반적인 사용을 위한 표준 링크를 복사하거나 미리 채워진 고객 데이터로 개인화된 결제 링크를 생성하도록 선택합니다. 드롭다운 메뉴를 사용하여 연락처를 선택하고 선택된 연락처에 대한 커스터마이즈된 결제 링크를 받습니다.

![링크 복사 또는 개인화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601413/original/qsKn9wSPse8-AQ35QHVj1yTbitkygLbTiA.png?1750410664)

### 연락처에게 링크 보내기

Send to Contacts(연락처에게 보내기) 옵션을 사용하여 이메일, SMS 또는 둘 다를 통해 결제 링크를 전달합니다. 연락처를 선택하고 커뮤니케이션 방법을 선택한 다음 캠페인이나 오퍼에 맞게 제목 줄과 템플릿을 커스터마이즈합니다.

![연락처에게 링크 보내기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601640/original/LajFAuBQcQO8NBhvq4kxoUhyzBI-JlF2pw.png?1750410851)

### **결제 링크 결제 페이지 미리보기**

Preview(미리보기) 버튼을 클릭하여 고객에게 결제 링크가 어떻게 보일지 확인합니다. 이를 통해 링크를 공개적으로 공유하기 전에 상품 그룹화, 브랜딩 요소, 폼 필드, 전체 레이아웃을 확인할 수 있습니다.

![결제 페이지 미리보기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048601696/original/8uQYT_C-9jLmcX6bjWJ1lcRvHkegiU8tYQ.png?1750410918)

### **결제 링크 저장하기**

상품과 설정이 완료되면 **Save(저장)** 버튼을 클릭하여 결제 링크를 게시합니다. 이 작업으로 링크가 활성화되어 선택한 채널을 통해 고객과 공유를 시작할 수 있습니다.

## 자주 묻는 질문

**Q: 추가할 수 있는 일회성 상품 수에 제한이 있나요?**

제한이 없습니다. 비즈니스 요구사항만큼 많은 일회성 상품을 포함시킬 수 있습니다.

**Q: 쿠폰 코드를 모든 상품에 적용할 수 있나요?**

네, 쿠폰 코드는 결제 시 허용되며 장바구니의 해당 상품에 적용됩니다.

**Q: 결제 화면에서 상품이 어떻게 정리되나요?**

명확한 구분을 위해 "One-time Products(일회성 상품)"과 "Recurring Products(구독 상품)" 아래에 그룹화됩니다.

**Q: 결제 링크를 임베드하거나 공유할 수 있나요?**

네. 링크는 이메일, 소셜 미디어를 통해 공유하거나 웹사이트에 임베드할 수 있습니다.
