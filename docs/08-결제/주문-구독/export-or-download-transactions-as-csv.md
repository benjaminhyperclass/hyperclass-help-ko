# 거래 내역을 CSV로 내보내기 또는 다운로드하기

이 문서는 거래 기록을 내보내는 방법, 내보내기 파일이 전달되는 방식, 그리고 거래당 여러 라인 항목을 포함한 CSV 형식을 해석하는 방법을 설명합니다.

## 개요

거래 CSV 내보내기(Transactions CSV Export)를 통해 보고, 대사, 회계, 컴플라이언스 목적으로 자세한 결제 기록을 다운로드할 수 있습니다.

내보내기에 포함되는 항목:

- 모든 거래 또는
- 적용된 필터나 검색을 기반으로 한 거래만

내보내기는 이메일 다운로드 링크를 통해 전달되며, 모든 스프레드시트 도구에서 열 수 있습니다.

## 거래 내보내기 방법

- Payments(결제) → Transactions(거래)로 이동
- 필요하면 필터/검색 적용 (예: 날짜 범위, 결제 상태, 소스 유형)
- Download Icon(다운로드 아이콘) 클릭
- 로그인한 사용자에게 보안 다운로드 링크가 이메일로 전송됨

보안 참고사항: 이메일 링크는 7일간 유효하며 요청한 사용자에게만 고유합니다.

## 이메일 전달 동작

- 내보내기 링크가 몇 분 내에 도착
- 파일은 앱 내에서 직접 다운로드되지 않고 이메일 링크를 통해 다운로드
- 링크가 만료되면 → 새로운 내보내기를 생성하기만 하면 됨

## CSV 구조 및 형식 규칙

각 행이 항상 하나의 거래와 같지는 않습니다.

단일 거래에는 다음이 여러 개 포함될 수 있습니다:

- 상품 → 여러 라인 행을 생성
- 세금 → 상품당 고유한 세금 이름당 하나의 행을 생성
- 기타 요금 (처리 수수료, 연체료, 팁)

이렇게 하면 각 금융 구성 요소를 개별적으로 식별할 수 있습니다. 명확성을 위해 동일한 거래에 여러 행이 있는 경우, 모든 컬럼이 각 행에 채워지지는 않으며, 대부분 고유한 상품이나 세금 관련 데이터가 각 행에 채워지고 전체 데이터는 거래당 1개 행에서만 사용할 수 있습니다.

## 데이터 일관성 보장 (내보내기 업그레이드)

이러한 내보내기는 안정적인 형식과 예측 가능한 값으로 대사, 회계, BI 워크플로우를 위해 설계되었습니다.

**0 vs 빈 값**

- 0은 해당 필드가 적용되고 값이 실제로 0일 때만 나타남
- 빈 값은 해당 필드가 해당 행/레코드에 적용되지 않을 때만 나타남

**보고를 위한 안정적인 구조**

- 컬럼 순서는 피벗, 가져오기, BI 파이프라인을 지원하기 위해 일관됨
- 다중 행 형식은 상품 수준 및 세금 수준 할당을 위해 의도적임 ("행 1개 = 거래 1개"라고 가정하지 마세요)

## 특별 형식 주의사항

- 여러 상품 → 동일한 거래 ID 아래에 여러 행을 생성
- 여러 세금 → 각 세금은 별도 행에 나타나며 상품 라인 항목에 속함 즉, 각 세금은 상품에 대해 고유한 행을 가짐.

따라서 2개 상품이 같은 세금 요율(2개)을 가지는 경우, 거래의 총 라인 항목은 4개가 됩니다. 즉, 세금 1과 세금 2가 있는 라인 항목 1이 2행을 만들고, 세금 1과 세금 2가 있는 라인 항목 2가 2행을 더 만듭니다.

- 결합 결제 (예: 카드 + 기프트 카드) → 값 컬럼에 명확히 표시
총액으로 필터링할 때는 스프레드시트에서 항상 신중하게 집계하세요.

- 세금율 정밀도
세금율(백분율) 값은 소수점 **4자리**로 내보내집니다(예: **7.1250%**). 이는 소수점 세금율과 일관된 보고를 지원하기 위함입니다. 세금 금액 값은 통화 표준에 따라 **소수점 2자리로 반올림**된 상태로 유지됩니다.

## FAQ

### 1. CSV에서 단일 거래가 여러 행으로 표시되는 이유는 무엇인가요?

상품은 라인 항목별로 추적되고 세금 세부사항은 고유한 세금 이름별로 추적되기 때문입니다. 이는 회계 등급 보고의 표준입니다.

### 2. 결제 예정 금액(Total Amount Due)이 실제 결제 금액(Total Amount Paid)과 다른 이유는 무엇인가요?

- 처리 수수료
- 팁
- 부분 결제... 등이 원래 결제 예정 금액이 아닌 최종 결제에 영향을 줍니다.

### 3. 내 내보내기에서 일부 거래가 보이지 않는 이유는 무엇인가요?

다음을 확인하세요:

- 필터가 해제되었는지
- 올바른 날짜 범위가 적용되었는지

### 4. 내보내기에 실패한 거래가 포함되나요?

예 — 성공 또는 환불된 거래만 포함하도록 필터링하지 않는 한 포함됩니다.

### 5. 내보내기에 부분 환불이 포함되나요?

예 — 환불 금액(Amount Refunded)은 전체 누적 환불 금액을 반영합니다.

### 6. 이전에 내보낸 파일을 다시 생성할 수 있나요?

예 — 내보내기는 언제든지 반복할 수 있습니다.

### 7. 누가 거래를 내보낼 수 있나요?

거래를 볼 수 있는 권한과 기록을 내보낼 수 있는 권한이 있는 사용자만 가능합니다.
