# 거래 내역 & 주문 가져오기란?

GHL에서는 이제 CSV 파일을 사용하여 과거 거래 내역과 주문을 일괄 가져올 수 있습니다. 이 기능을 통해 과거 재무 데이터를 하위 계정으로 통합하여 대시보드와 연락처 타임라인에 정확한 매출 기록이 반영되도록 할 수 있습니다.

## 거래 내역 & 주문 가져오기의 주요 장점

가져오기 기능을 통해 깔끔하고 중앙 집중화된 재무 기록을 유지할 수 있습니다. 이 기능의 주요 장점은 다음과 같습니다:

- 이전 플랫폼의 과거 데이터를 쉽게 가져올 수 있습니다
- 매출 대시보드와 지표에 자동으로 반영됩니다
- 가져오기 시 고객 기록을 생성하거나 업데이트할 수 있습니다
- 데이터 유입 시간과 실제 거래 시간을 별도로 설정할 수 있습니다
- 업로드 전 간편한 미리보기 및 검증 기능을 제공합니다
- 내장된 형식 가이드와 다운로드 가능한 샘플 CSV를 제공합니다

## 거래 내역 가져오기 방법

거래 내역을 가져오면 주문 항목을 생성하지 않고도 실제 결제 이벤트를 반영하는 데이터를 가져올 수 있습니다.

- **Payments(결제) > Transactions(거래 내역)**로 이동합니다
- **Import(가져오기)** 버튼을 클릭합니다
- **Sample CSV(샘플 CSV)**를 다운로드하여 데이터가 필수 형식과 일치하는지 확인합니다
- 파일을 업로드하고 검증을 위해 **preview modal(미리보기 모달)**을 검토합니다
- 가져오기를 확인합니다

![거래 내역 가져오기 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048479428/original/8zRY5lm6EdGJmGSyl0fa1mna_mBBuzAu0Q.png?1750253844)

**참고:** 거래 내역은 추가되는 즉시 매출 지표에 반영됩니다.

## 주문 가져오기 방법

주문은 항목별 판매를 나타내며 연결된 거래 내역 항목을 자동으로 생성합니다.

- **Payments(결제) > Orders(주문)**로 이동합니다
- **Import(가져오기)**를 선택한 다음, 올바른 형식의 CSV를 업로드합니다
- 미리보기에서 데이터를 검증합니다
- 각 주문 기록은 자동으로 거래 내역을 생성하고 매출에 반영됩니다

![주문 가져오기 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048479458/original/xJI-izG6J90dtTt715A6igSaT5OPjE3xaA.png?1750253871)

**참고:** 주문과 거래 내역 모두 대시보드와 보고서에 반영됩니다.

## CSV 파일 필드 형식

가져오기를 위한 필수 구조를 이해하면 오류 없이 데이터를 업로드할 수 있습니다.

- 필드 이름은 정의된 대로 **정확히** 일치해야 합니다
- 연락처를 매칭하거나 생성하기 위해 고객 세부 정보가 포함되어야 합니다
- 실행 날짜와 GHL 유입 날짜를 올바르게 설정해야 합니다

## 자주 묻는 질문

**Q: 이미 GHL에 있는 연락처가 중복으로 생성되나요?**
아니요. GHL은 새 기록을 생성하기 전에 이메일이나 전화번호를 통해 연락처를 매칭하려고 시도합니다.

**Q: 거래를 소급 적용할 수 있나요?**
예. GHL에 추가되는 시점과 별도로 거래 실행 날짜를 지정할 수 있습니다.

**Q: CSV에 오류가 있으면 어떻게 되나요?**
가져오기를 진행하기 전에 검증 문제를 강조 표시하는 미리보기를 볼 수 있습니다.

**Q: 환불이나 취소된 거래를 가져올 수 있나요?**
현재는 지원되지 않습니다. 성공적으로 완료된 거래/주문만 지원됩니다.

**Q: 가져오기가 매출 추적에 영향을 주나요?**
예. 매출 및 기타 재무 대시보드가 가져온 데이터를 기반으로 업데이트됩니다.
