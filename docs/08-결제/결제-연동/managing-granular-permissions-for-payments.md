# 결제 세부 권한 관리

Hyperclass의 세부 결제 권한 시스템을 사용하면 결제 관련 업무를 안전하게 위임할 수 있습니다. 결제(Payments)의 각 영역에서 누가 조회, 생성, 편집, 환불, 내보내기, 설정할 수 있는지 정확히 제어할 수 있죠. 이 가이드는 모든 권한 카테고리를 정의하고, 일반적인 역할 프리셋을 설명하며, 올바른 접근 권한 설정 방법을 보여드립니다. 매출 운영과 감사 추적을 보호하면서 팀원 온보딩 속도를 높이는 데 활용하세요.

## 결제 세부 권한이란?

결제 세부 권한은 기존의 단일 결제 접근 권한을 주문, 구독, 거래, 세금, 상품, 결제 설정 등 집중된 카테고리로 나누고, 각각에 표준화된 액션을 제공합니다. 이 구조를 통해 관리자는 정밀한 제어가 가능하며, 팀원들은 민감한 작업에 노출되지 않으면서도 업무에 필요한 접근 권한만 받을 수 있습니다.

결제 세부 권한은 **로케이션**(하위 계정) 수준의 역할 기반 접근 제어로, 어떤 사용자가 결제(Payments) 모듈 전반에서 **조회, 생성, 수정, 삭제, 내보내기, 설정**할 수 있는지 결정합니다. 관리자는 카테고리를 조합하여 지원팀, 회계팀, 주문 처리팀, 관리팀에 맞춤형 접근 권한을 부여할 수 있습니다.

---

## **결제 세부 권한의 주요 이점**

이점을 이해하면 팀별로 어떤 액션을 허용할지 결정하는 데 도움이 됩니다. 목표는 일상 업무 속도를 늦추지 않으면서 리스크를 줄이는 것입니다.

- **리스크 최소화**: 환불, 상품 삭제, 세금 설정 등 큰 영향을 미치는 작업을 신뢰할 수 있는 사용자에게만 제한합니다.

- **빠른 온보딩**: 각 역할에 필요한 카테고리와 액션만 할당하여 신입 직원이 안전하게 업무를 시작할 수 있습니다.

- **깔끔한 감사**: 좁은 권한 범위로 **감사 로그**를 사용해 누가 언제 무엇을 변경했는지 추적하기가 더 쉬워집니다.

- **운영 명확성**: 명확한 카테고리로 작업이 어디에 속하는지 헷갈리는 일이 줄어듭니다(예: 환불 vs 주문 편집).

- **워크플로우 제어**: 사용자의 권한에 따라 일괄 작업, 가져오기, 내보내기 기능이 제한되어 데이터 품질을 보호합니다.

---

## **결제 권한의 새로운 변경사항**

기존의 단일 결제 토글에서 무엇이 바뀌었는지 이해하면 관리자가 자신 있게 마이그레이션하고 직원에게 정확한 기대치를 설정할 수 있습니다. 이러한 업데이트는 할당을 간소화하고 감사 가능성을 향상시키며, 지나치게 광범위한 접근 권한을 부여할 리스크를 줄입니다.

- **권한 재배치**: **거래 환불**과 **구독 관리** 등 기존 기능이 이제 전용 카테고리(거래 및 구독) 안에 자리잡아, 환불 권한 없이 조사 접근 권한을 부여하거나 전체 결제 접근 없이 구독 감독 권한을 부여하기가 더 쉬워졌습니다.

- **표준화된 액션**: 각 카테고리가 이제 일관된 액션 세트(**조회, 생성, 수정, 삭제, 내보내기, 설정**)를 사용하여 결제 전반에 걸쳐 역할 비교, 교육, 감사가 더 간단해졌습니다.

- **UI 강제 종속성**: 특정 고영향 도구는 기본 권한에 의존합니다. 예를 들어, **상품 일괄 작업과 CSV 가져오기는 생성/수정 접근이 필요**하여 읽기 전용 사용자의 실수로 인한 다중 레코드 변경을 방지합니다.

- **민감한 작업의 강화된 제어**: 관리자가 누가 환불을 실행하거나, 구독을 편집하거나, 세금 설정을 조정할 수 있는지 명시적으로 결정할 수 있어 일상적 효율성을 보존하면서 재정 통제력을 높입니다.

## 주문 권한

주문 권한은 결제 링크(Payment Links), 체크아웃 페이지, 스토어, POS를 통해 생성된 일회성 주문과 결제를 관리합니다. 주문을 처리하고 조회/내보내기 접근이 필요한 팀에게 사용하며, 편집은 책임 있는 담당자에게만 제한하세요.

![주문 권한 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054979827/original/TWdwmo9quu883vOwegLmZV7yoXklVWekAQ.png?1759302691)

**추천 역할**: 주문을 조회하고 목록을 내보내며 제한적 조정이 필요한 주문 처리팀이나 지원팀

---

## **구독 권한**

구독 권한은 정기 결제 생애주기를 다룹니다. 플랜 생성, 연락처에 구독 할당, 일시정지/재개, 취소, 결제 방법 변경 등의 업데이트를 포함합니다. 구독 상태와 이탈 방지에 책임이 있는 역할에게 부여하세요.

![구독 권한 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054980224/original/MTdvdd83enOr1SWGs093SwXj7FwhTPpzgQ.png?1759303088)

**추천 역할**: 구독 상태와 유지를 담당하는 재무/청구팀 및 계정 관리자. 지원팀에게는 조회 권한을, 지정된 담당자에게만 생성/수정/취소 권한을 부여하세요.

---

## **거래 권한**

거래 권한은 완료된 결제와 환불에 적용됩니다. 환불과 공식 영수증 다운로드가 가능한 민감한 영역이므로 엄격히 통제하고, 가능하면 "조회"와 "환불"을 분리하세요.

![거래 권한 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054980262/original/CEv-bTfSxhXMNMpepwR9iGHDg3HDDXtWcg.png?1759303119)

**추천 역할**: 조사용(조회/내보내기/영수증)으로는 청구 지원팀, 환불용으로는 관리자/회계사. 환불 권한은 재무 책임자에게만 제한하세요.

---

## **세금 권한**

세금 권한은 수동 세율과 자동 세금 계산 기능을 포함하여 세금 동작을 설정하거나 조정할 수 있는 사용자를 제어합니다. 세금 설정은 컴플라이언스에 영향을 미치므로 이 권한은 재무 담당자에게만 부여하세요.

![세금 권한 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054980290/original/a3uGaX2G3hQpnn4Sk4ykQIySDI12G-DDnA.png?1759303154)

**추천 역할**: 컴플라이언스를 담당하는 회계사/관리자 또는 재무 관리자. 비재무 사용자에게는 설정/삭제 권한을 부여하지 마세요.

---

## **상품 권한**

상품 권한은 상품을 생성, 편집, 복제, 가져오기/내보내기, 삭제할 수 있는 사용자를 결정합니다. 이러한 변경사항은 결제 링크, 스토어, 체크아웃 페이지에 연쇄 영향을 미칩니다. 파괴적인 작업을 제한하고 카탈로그 변경에는 검토를 요구하세요.

![상품 권한 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054980302/original/m0yCgNs6TIOEizAmaLnY5pRWhxz0LZEeDA.png?1759303170)

**추천 역할**: 일상적 업데이트를 위한 스토어 관리자나 상품/카탈로그 담당자. 가져오기/내보내기/삭제는 관리자에게만 부여하세요.

---

## 결제 설정 권한

결제 설정 권한은 영수증, 구독 기본값, 결제 관련 알림 등 전역 결제 기본 설정을 다룹니다. 이는 고객 경험을 형성하므로 관리자와 브랜드 담당자에게만 제한해야 합니다.

![결제 설정 권한 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054980311/original/b4YbAGTiagpNZstE6SPHYj9p1m7bt0xFPw.png?1759303193)

**추천 역할**: 영수증, 기본값, 알림을 관리하는 관리자나 브랜드/재무 담당자. 설정을 중앙집중식으로 유지하세요.

---

## **결제 세부 권한 설정 방법**

적절한 설정을 통해 사용자가 불필요한 위험 없이 업무를 완료할 수 있습니다. 다음 단계를 따라 필요한 접근 권한만 할당하고 결과를 확인하세요.

1. **하위 계정 설정(Sub-Account Settings) → 내 직원(My Staff)**으로 이동하여 사용자를 찾고 연필(편집) 아이콘을 클릭합니다.

![직원 편집 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054983320/original/pzOCaMK00vyyHdOiYXEe4eS7DqNlSqKbJg.png?1759305479)

2. 역할 및 권한(Roles & Permissions) 섹션을 확장하고 필요한 카테고리를 선택합니다.

3. 각 카테고리(주문, 구독, 거래, 세금, 상품, 결제 설정)에 대해 해당 역할이 필요로 하는 액션(**조회, 생성, 수정, 삭제, 내보내기, 설정**)을 토글합니다. 일부 일괄/가져오기/내보내기 작업은 생성/수정 권한이 없는 사용자에게는 사용할 수 없을 수 있습니다.

![권한 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054983292/original/5gFD7-EtbDhYb0b116gpfk6T53-hd9Ls4A.gif?1759305452)

4. **저장**을 클릭합니다.

5. (선택사항) **권한 복사**를 클릭하여 같은 설정을 다른 사용자에게 적용합니다.

![권한 복사 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155054983497/original/mQbtdjcgooi4l3d8pUvrgNs21eFVj_wEgw.png?1759305624)

6. 사용자가 로그아웃 후 다시 로그인하도록 하고(필요시) 테스트 레코드(예: 상품 열기, 거래 조회)를 사용하여 접근 권한을 검증합니다.

## 권장 역할 프리셋

다음 프리셋은 일반적인 업무 기능을 더 안전한 권한 세트에 매핑하는 권장사항입니다. 항상 조직의 정책에 맞게 조정하세요.

**예시 프리셋**

| 역할 | 설정 카테고리 | 하위 설정(액션) | 비고 |
|------|---------------|----------------|------|
| **청구 지원** | 거래 | **조회, 내보내기, 영수증 다운로드** *(환불 꺼짐)* | 환불 권한 없이 조사와 영수증 허용. 감독하에 환불이 필요하면 일시적으로 권한 상승 |
| | 구독 | 조회, 일시정지/재개/취소 | 생애주기 변경 관리 교육을 받지 않았다면 생성/수정/취소 금지 |
| | 주문 | **조회, 내보내기** | 주문 상태 질문 답변에 유용. 시니어 에이전트만 수정 고려 |
| | 상품 | **조회** | 카탈로그 가시성만. 편집 없음 |
| | 결제 설정 | — | 설정은 관리자/재무와 함께 유지 |
| **주문 처리/스토어 관리자** | 주문 | **조회, 내보내기, 수정** *(생성 선택사항, 삭제 꺼짐)* | 배송/조정 지원. 삭제는 관리자에게만 |
| | 상품 | **조회, 수정** *(카탈로그 담당자용 생성; 가져오기/내보내기 꺼짐; 삭제 꺼짐)* | 일괄/가져오기는 생성/수정 필요할 수 있음; 교육받은 담당자에게만 제한 |
| | 거래 | **조회** | 환불 기능 없음. 환불 요청은 재무로 에스컬레이션 |
| | 구독 | **조회(선택사항)** | 맥락만. 생애주기 변경은 계정 관리자/재무가 처리 |
| **계정 관리자** | 구독 | **조회, 생성, 수정, 일시정지/재개/취소, 내보내기** | 구독 상태, 갱신, 이탈 방지 담당; 필요시 카드 업데이트 관리 |
| | 거래 | **조회** | 청구 기록 조사; 환불 없음 |
| | 주문 | **조회** | 고객 대화를 위한 맥락 |
| | 상품 | **조회(선택사항)** | 가격 논의를 위한 읽기 전용 상품 맥락 |
| **회계사/관리자** | 거래 | **조회, 내보내기, 환불** | 환불은 재무 책임자에게만 제한; 감사 로그로 모니터링 |
| | 세금 | **조회, 설정, 수정, 삭제** | 민감함. 자격 있는 재무 직원에게만 제한; 세율 삭제/교체 시 주의 |
| | 결제 설정 | **설정, 수정** | 영수증, 기본값, 결제 알림 제어 |
| | 주문 | **조회** | 읽기 전용 운영 맥락 |
| | 상품 | **조회, 내보내기(선택사항)** | 대조를 위한 내보내기; 필요하지 않으면 편집 금지 |
| **관리자** | 전체(주문, 구독, 거래, 세금, 상품, 결제 설정) | **조회, 생성, 수정, 삭제, 내보내기, 설정** *(해당하는 경우)* | 최소한의 교육받은 관리자 그룹을 위한 전체 제어. 삭제는 신중히 사용; 변경 관리와 검토 시행 |

**팁**: 사용자 레코드의 **권한 복사** 기능을 사용하여 테스트된 설정을 유사한 역할에 복제하세요.

## 자주 묻는 질문

**Q: 권한이 결제의 CSV 가져오기/내보내기에 영향을 미치나요?**
네. 가져오기/내보내기 기능은 사용자에게 할당된 액션과 역할에 따라 제한될 수 있습니다. 사용자가 일괄 도구를 볼 수 없다면 관리자가 해당 카테고리의 생성/수정/내보내기 권한을 검토해주세요.

**Q: 환불 권한 없이 거래를 조사할 수 있나요?**
네. **거래 — 조회**(필요시 내보내기)를 부여하면서 **환불** 접근은 재무 담당자에게만 제한할 수 있습니다.

**Q: 상품 권한을 비활성화하면 기존 체크아웃이나 결제 링크가 중단되나요?**
아니요. 실제 체크아웃 경험은 설정된 대로 계속 판매됩니다. 다만 사용자는 기본 상품이나 링크를 편집할 수 없습니다.

**Q: 상품, 세금 설정, 결제 기본 설정을 누가 변경했는지 어디서 볼 수 있나요?**
**감사 로그**를 사용하여 사용자 액션과 변경사항을 검토하세요. 에이전시 관리자는 에이전시 수준에서 로그에 접근할 수 있으며, 퍼널, 웹사이트, 스토어용 자산 수준 감사 로그도 이용할 수 있습니다.

**Q: 결제 세부 권한을 에이전시 수준에서 사용할 수 있나요?**
결제 권한은 **로케이션** 수준에서 할당됩니다. 에이전시 사용자는 에이전시 범위에서 사용자와 역할을 관리하며, 이러한 할당은 설정에 따라 로케이션 역할과 스택되거나 이를 대체할 수 있습니다.

**Q: 사용자가 결제에서 버튼이 회색으로 표시된다고 보고하면 어떻게 해야 하나요?**
보통 해당 카테고리에서 생성/수정이나 내보내기 권한이 부족함을 나타냅니다. 역할 설정을 검토하거나 관리자 역할로 임시 테스트하여 확인해보세요.

**Q: 세금 권한에 자동 계산 기능이 포함되나요?**
네, 사용 가능한 곳에서는 포함됩니다. 재무 담당자가 자동 세금 계산을 활성화하고 관리할 수 있습니다. 정확한 결과를 위해 결제 경험에서 주소를 수집하도록 하세요.
