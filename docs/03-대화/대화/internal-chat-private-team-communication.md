# 내부 채팅: 팀 전용 비공개 소통

## 개요

내부 채팅(Internal Chat)은 대화(Conversations) 모듈 내에서 하위 계정 사용자들과 비공개로 소통할 수 있는 기능입니다. 내부 논의를 간소화하고 팀 협업을 향상시킵니다. 고객 대화와 달리, 내부 채팅은 참가자에게만 보이므로 비공개성이 보장됩니다.

## 인박스 개요

Labs에서 내부 채팅을 활성화하면, 대화 인터페이스 왼쪽에 새로운 인박스 패널이 나타납니다. 이 패널에는 소통을 체계적으로 관리하는 두 개의 핵심 인박스가 있습니다:

### 1. 팀 인박스(Team Inbox)

팀 인박스는 사용자의 접근 권한에 따라 하위 계정의 대화를 표시합니다.

- "본인 데이터만 보기(Only Assigned Data)" 권한이 있는 사용자(예: 하위 계정 사용자)는 자신이 연락처 담당자이거나 팔로워인 모든 대화를 볼 수 있습니다.

- "전체 접근(All Access)" 권한이 있는 사용자(예: 관리자)는 하위 계정의 모든 대화를 볼 수 있습니다.

이렇게 하여 사용자는 자신과 관련된 대화만 보게 되고, 관리자는 전체를 감독할 수 있습니다.

### 2. 내 인박스(My Inbox)

내 인박스는 사용자가 직접 참여하는 대화에 중점을 둔 개인화된 공간입니다. 다음 폴더들이 포함되어 있습니다:

- 나에게 배정됨(Assigned to Me): 사용자가 연락처 담당자인 모든 대화를 나열하여, 담당하는 상호작용을 명확하게 볼 수 있습니다.

- 내부 채팅(Internal Chat): 사용자가 참가자인 내부 전용 대화 스레드를 보여줍니다. 이러한 채팅은 비공개이며 팀 협업을 위해 설계되어, 고객 대면 대화와는 별개입니다.

## 내부 채팅 시작하는 방법

- 대화를 열고 새로 만들기(New)를 클릭합니다. 내부 채팅(Internal Chat) 옵션을 선택하세요.

![내부 채팅 시작](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044723847/original/2K203g2mWgiS_OjXq832-SrBxfV_ad1AuA.png?1744112331)

- 채팅에 추가할 사용자를 선택하고 채팅 시작(Start Chat)을 클릭하여 대화를 시작합니다.

![사용자 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044725105/original/y4sNWaPO5IORAhijfbNp-418Be90WonHMQ.png?1744113282)

- 내부 채팅에서는 텍스트 메시지, 이모지, 첨부파일을 보낼 수 있습니다.

![채팅 메시지 전송](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047695620/original/1dO66w89f9ur9E8JPXA9251fM-VgF1NvTQ.jpeg?1748988092)

- 대화를 더 잘 정리하려면, 상단 중앙 패널의 편집 아이콘을 클릭하여 제목을 추가할 수 있습니다. 제목을 추가하면 제목, 참가자 이름 또는 마지막으로 전송된 메시지로 내부 채팅을 검색할 수 있습니다.

![제목 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047695625/original/HHVF4Lr0RPvZSAR9g5xbGiTW03D9uJ4NxA.jpeg?1748988105)

## 기존 내부 채팅에 사용자 추가하기

내부 채팅을 열고 오른쪽 패널에서 사용자 추가(Add Users)를 클릭합니다.

![사용자 추가 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047695317/original/ebqN4tF-X-jxfHcmscRKXVWLKxjdADRvdw.jpeg?1748987401)

추가할 팀원을 선택하고 다음(Next)을 클릭합니다. 이전 채팅 기록에 접근할 수 있게 할지 선택합니다. 확인(Confirm)을 클릭하세요. 한 번 추가된 사용자는 채팅에서 제거할 수 없습니다.

![사용자 추가 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044725977/original/4GvJRYQOoxn6-o7j81aVq6-ywdeCepscQQ.png?1744113893)

## 내부 채팅 알림 받기

설정(Settings) → 내 프로필(My Profile) → 알림(Notifications)으로 이동하여 알림을 관리합니다. 사용자는 다음을 선택할 수 있습니다:

- 참여 중인 내부 채팅의 모든 메시지에 대한 알림 받기

- 새 내부 채팅에 추가될 때만 알림 받기

## 접근 권한

### 생성 권한:

- 대화에 접근할 수 있는 모든 사용자가 내부 채팅을 생성할 수 있습니다.

- 에이전시 관리자/사용자는 하위 계정에 속하지 않더라도 내부 채팅을 생성할 수 있습니다.

### 참가자 권한:

- 하위 계정 사용자만 내부 채팅에 추가될 수 있습니다.

### 보기 권한:

- 내부 채팅에 추가된 사용자만 대화를 볼 수 있습니다.

- 다른 팀원들은 명시적으로 추가되지 않는 한 내부 채팅을 볼 수 없습니다.

---

## 이 기능이 도움되는 이유

- **모든 소통을 한 곳에**: Slack이나 이메일 같은 외부 앱을 사용할 필요 없이 내부 논의가 CRM 내에서 이루어져 대화가 체계적이고 접근하기 쉽게 유지됩니다.

- **원활한 협업**: 팀이 대화 모듈을 벗어나지 않고 즉시 소통할 수 있어 빠른 의사결정과 더 나은 조정이 가능합니다.

- **향상된 맥락과 가시성**: 내부 채팅이 CRM에 연결되어 있어 팀원들이 리드, 거래, 지원 사례를 논의할 때 전체적인 맥락을 파악할 수 있습니다.

- **효율적인 팀워크**: 고객 상호작용이 일어나는 같은 플랫폼에서 업데이트를 공유하고, 질문하고, 전략을 세울 수 있습니다.

- **알림으로 신속한 대응**: 새 내부 메시지에 대한 알림을 받아 외부 대화를 관리하면서도 중요한 업데이트를 놓치지 않습니다.

**참고:**

1. 내부 채팅은 플랫폼에 기본 제공되는 기능으로 추가 비용이 발생하지 않습니다.

2. 다른 사용자로 "로그인(Login As)"하는 경우 내부 채팅을 볼 수 없어 비공개성이 보장됩니다.
