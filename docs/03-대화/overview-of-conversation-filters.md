# **대화 필터란?**

대화 필터는 대화(Conversations) 인박스에서 특정 조건에 따라 표시할 대화를 결정하는 도구입니다. 사용자가 체계적으로 관리하고, 우선순위를 정하며, 인박스를 수동으로 검색하지 않고도 가장 관련성 높은 대화에 집중할 수 있도록 설계되었습니다.

대화 필터는 인박스 탭, 정렬 옵션, 일괄 작업과 함께 작동하여 팀이 대화를 보고 관리하는 방식을 완전히 제어할 수 있게 해줍니다.

---

## **대화 필터의 주요 장점**

필터를 올바르게 사용하면 대화 가시성과 제어력이 크게 향상됩니다.

- **집중된 인박스 보기**: 배정된 사용자, 언급, 태그와 같은 특정 기준에 맞는 대화만 표시합니다.

- **팀 효율성 향상**: 노이즈를 줄이고 팀원들이 조치가 필요한 대화에 집중할 수 있도록 도와줍니다.

- **고급 논리 지원**: AND / OR 논리를 사용하여 여러 규칙을 결합해 정교한 필터링을 제공합니다.

- **확장 가능한 인박스 관리**: 수동 정렬 없이 대량의 대화를 관리할 수 있습니다.

## **필터 패널**

필터 패널은 업데이트된 필터링 환경의 핵심으로, 여러 조건을 사용한 규칙 기반 필터링을 가능하게 합니다.

### **필터 패널 열기**

필터 아이콘(세 개의 수평선)을 클릭하면 화면 오른쪽에 필터 패널이 열립니다.

## 필터 빌더

필터 빌더를 사용하면 인박스에 표시될 대화를 제어하는 세부 규칙을 정의할 수 있습니다.

### **사용 가능한 필터 유형은 다음과 같습니다:**

### 담당자별 필터

대화 소유자로 배정된 사용자를 기준으로 대화를 필터링합니다. 팀원들이 자신이 담당하는 대화를 빠르게 보거나 소유권이 필요한 미배정 대화를 식별하는 데 도움이 됩니다.

사용자 드롭다운에서 '로그인한 사용자'를 선택하면 현재 로그인한 팀원에게 필터가 적용되도록 할 수 있습니다. 이는 공유 대화 보기에 유용합니다.

![담당자별 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064856353/original/xIJzsYqUTQSP8rUFzxkaRjPfgvqzeYHFDA.png?1770979331)

### 팔로워별 필터

특정 사용자가 팔로워로 추가된 대화를 필터링합니다. 팔로워는 주 소유자가 아니면서도 대화에 대한 가시성을 받으므로, 협업과 감독에 유용합니다.

사용자 드롭다운에서 '로그인한 사용자'를 선택하면 현재 로그인한 사용자가 팔로워인 대화를 동적으로 필터링할 수 있습니다.

### 언급별 필터

@언급을 사용하여 사용자가 언급된 대화를 필터링합니다. 주의가 필요하거나 특정 팀원에게 명시적으로 플래그된 대화를 찾는 데 도움이 됩니다.

### 마지막 메시지 방향별 필터

가장 최근 메시지가 **인바운드**(연락처에서 온 것) 또는 **아웃바운드**(사용자나 자동화로 보낸 것)인지에 따라 대화를 필터링합니다. 응답을 기다리는 대화와 이미 답변한 대화를 구분하는 데 도움이 됩니다.

### 마지막 발신 메시지 유형별 필터

가장 최근 발신 메시지가 사용자가 **수동으로** 보낸 것인지 워크플로우나 캠페인을 통해 **자동으로** 보낸 것인지에 따라 대화를 필터링합니다. 인간의 참여와 자동화를 추적하는 데 유용합니다.

### 마지막 메시지 채널별 필터

SMS, 이메일, 전화, 음성메일, 라이브 채팅, 왓츠앱, 페이스북, 인스타그램, GBP 등 가장 최근 메시지에 사용된 커뮤니케이션 채널로 대화를 필터링합니다. 팀이 특정 채널에 집중할 수 있게 해줍니다.

### 태그별 필터

특정 연락처 태그와 연관된 대화를 필터링합니다. 태그는 연락처와 대화를 분류하는 데 도움이 되어, 커스텀 라벨을 기반으로 대화를 세분화하고 관리하기 쉽게 만듭니다.

![태그별 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064171943/original/8WeIAwnrYwk-GMRn7d8BVV8Dl4CX5k3RvA.png?1770206025)

### **필터 조건**

조건은 선택한 필터 유형이 어떻게 평가될지를 결정합니다.

**일반적인 조건은 다음과 같습니다:**

- **일치**: 선택한 값과 일치하는 대화를 포함합니다.
- **일치하지 않음**: 선택한 값과 일치하는 대화를 제외합니다.

### **필터 값**

값 필드는 필터에서 사용되는 정확한 사용자, 태그, 채널 또는 속성을 정의합니다.

**작동 방식:**

- 검색 가능한 드롭다운을 지원합니다.
- 필터 유형에 따라 하나 또는 여러 값을 선택할 수 있습니다.

## **AND / OR 논리**

AND / OR 논리는 여러 필터 규칙이 서로 어떻게 상호작용하는지를 제어합니다.

### **필터 그룹 내 논리**

같은 그룹 내에 여러 규칙이 있을 때, 논리 토글이 평가 방식을 결정합니다.

- **AND**: 모든 규칙이 참이어야 합니다.
- **OR**: 어떤 규칙이라도 참이면 됩니다.

### **필터 그룹 간 논리**

여러 필터 그룹이 생성되면, 추가 AND / OR 토글이 전체 그룹이 어떻게 상호작용하는지 정의합니다.

이를 통해 배정, 언급, 태그를 단일 필터 설정에서 결합하는 것과 같은 고급 논리가 가능합니다.

## 여러 대화 관리하기

대량의 대화를 처리하는 것은 어려울 수 있지만, 여러 "대화 작업"을 통해 여러 대화를 동시에 효율적으로 관리하여 바쁜 환경에서도 제어력과 체계성을 유지할 수 있습니다.

### **대화 선택 체크박스**

인박스의 각 대화 맨 왼쪽에는 체크박스가 있습니다. 하나 이상의 체크박스를 선택하면 일괄 관리 기능이 활성화됩니다.

**작동 방식:**

- 단일 대화를 선택하면 해당 대화에만 작업을 적용합니다.
- 여러 대화를 선택하면 일괄 작업이 활성화됩니다.
- 여러 대화가 있을 때 전체 선택 옵션이 나타납니다.

### **일괄 작업 도구모음**

하나 이상의 대화가 선택되면 일괄 작업(Bulk Actions) 도구모음이 자동으로 나타납니다.

**사용 가능한 작업은 다음과 같습니다:**

- **읽음으로 표시**: 선택한 대화를 읽음으로 표시합니다.
- **읽지 않음으로 표시**: 대화를 읽지 않음 상태로 되돌립니다.
- **즐겨찾기 추가**: 대화를 중요한 것으로 플래그합니다.
- **즐겨찾기 제거**: 즐겨찾기 상태를 제거합니다.
- **삭제**: 선택한 대화를 영구적으로 삭제합니다.

![일괄 작업 도구모음](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064171207/original/7fSv7hrUaWcOLS6oPdByX5sY1081Sf8OKA.jpeg?1770205708)

## 즐겨찾기 대화

즐겨찾기 대화는 중요한 메시지를 강조하고 재방문하는 간단하면서도 효과적인 방법으로, 우선순위가 높은 상호작용을 쉽게 추적할 수 있게 해줍니다.

### **별 아이콘**

별 아이콘은 대화 목록과 개별 대화 보기 내에 모두 나타납니다.

**작동 방식:**

- 별을 클릭하면 대화가 즐겨찾기로 표시됩니다.
- 즐겨찾기 대화는 **즐겨찾기(Starred)** 인박스 탭을 사용하여 접근할 수 있습니다.

![즐겨찾기 대화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064171557/original/tfM-9vvEfKEtKyOhFIGdZcKQi4e7sIlZUA.jpeg?1770205870)

## 대화 정렬하기

정렬 옵션을 사용하면 가장 편리한 방식으로 대화를 배열할 수 있습니다. 여러 정렬 방법을 제공하여 항상 원하는 것을 빠르게 찾을 수 있습니다.

### **정렬 아이콘**

정렬 아이콘(위/아래 화살표)을 클릭하면 사용 가능한 정렬 옵션이 포함된 드롭다운이 열립니다.

사용 가능한 정렬 옵션은 다음과 같습니다:

- **최신 전체 메시지**: 수동이든 자동이든 관계없이 가장 최근에 보내거나 받은 메시지가 있는 대화를 맨 위에 표시합니다.

- **오래된 전체 메시지**: 마지막 메시지가 가장 오래된 대화를 맨 위에 표시하여 가장 오랫동안 비활성 상태였던 대화를 찾는 데 도움이 됩니다.

- **최신 수동 메시지**: 사용자가 **수동으로 보낸** 가장 최근 메시지를 기준으로 대화를 정렬하며, 워크플로우나 캠페인의 자동 메시지는 제외합니다.

- **오래된 수동 메시지**: 가장 오래된 수동 발송 메시지로 대화를 정렬하여 최근 인간의 상호작용이 없었던 대화를 쉽게 찾을 수 있게 합니다.

- **가장 긴 SLA 초과**: SLA를 가장 오랫동안 초과한 대화를 우선시하여 팀이 가장 심각한 응답 지연을 먼저 해결할 수 있도록 도와줍니다.

- **다음 SLA 목표**: 다가오는 SLA 마감일이 가장 가까운 순으로 대화를 정렬하여 팀이 SLA 위반 전에 사전에 응답할 수 있게 합니다.

## **자주 묻는 질문**

**Q: 대화 필터는 실시간으로 업데이트되나요?**

네, 필터가 적용, 제거 또는 수정되면 인박스가 즉시 업데이트됩니다.

**Q: 필터 조합을 저장할 수 있나요?**

현재 필터는 세션 기반이므로 인박스에 다시 돌아올 때 다시 적용해야 합니다.

**Q: 담당자와 팔로워 필터의 차이점은 무엇인가요?**

담당자는 대화의 주 소유자를 의미하며, 팔로워는 가시성이나 협업을 위해 추가된 사용자를 포함합니다.

**Q: 단일 필터에서 여러 채널을 사용할 수 있나요?**

네, 마지막 메시지 채널 필터는 여러 채널 선택을 지원합니다.

**Q: 일괄 작업을 사용하여 대화를 삭제하면 영구적으로 제거되나요?**

네, 삭제된 대화는 복구할 수 없습니다.
