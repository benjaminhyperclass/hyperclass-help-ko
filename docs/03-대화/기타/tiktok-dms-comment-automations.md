# 틱톡 DM & 댓글 자동화

틱톡 DM과 댓글을 하이퍼클래스로 가져와서 워크플로우가 자동으로 응답하게 하세요. 틱톡 DM & 댓글 자동화 연동은 모든 개인 메시지와 공개 댓글을 대화(Conversations) 인박스로 직접 라우팅하여, 자동화로 리드를 육성하고 자격을 평가하며 전환까지 완전 자동으로 처리합니다.

---

## 틱톡 DM & 댓글 자동화란?

틱톡 DM & 댓글 자동화는 틱톡 계정을 하이퍼클래스에 연결하여 다음을 가능하게 합니다:

- 다이렉트 메시지(DM)와 게시물 댓글이 대화 인박스에 실시간으로 표시됩니다.

- 들어오는 메시지로 워크플로우가 트리거되어 인터랙티브 답변 전송, 리드 배정, 할 일 생성, 육성 시퀀스 시작 등을 할 수 있습니다.

- 모든 터치포인트가 연락처 레코드에 저장되어 모든 채널에 걸친 완전한 대화 기록을 제공합니다.

---

## 틱톡 DM & 댓글 자동화의 주요 이점

- **통합 인박스**: 틱톡 대화를 SMS, 이메일, 왓츠앱 등과 함께 하이퍼클래스 대화에서 관리하세요. 탭 전환 불필요합니다.

- **빠른 응답 시간**: 워크플로우를 사용해 즉시 24/7 답변을 보내서 잠재 고객의 참여를 유지하세요.

- **자동화 기반 효율성**: 트리거와 액션을 사용해 자동 답변, 리드 라우팅, 상담원 인계 등을 수동 작업 없이 처리하세요.

- **전환 중심**: 틱톡 참여를 CRM 연락처와 추적 가능한 육성 시퀀스로 전환하세요.

- **팀 규모 확장**: 많은 메시지 볼륨을 공유 인박스와 규칙으로 처리하는 에이전시와 다중 로케이션 브랜드에 이상적입니다.

## 틱톡 계정 연동하기

진행하기 전에 틱톡 계정이 하위 계정과 연동되어 있는지 확인하세요.

하위 계정 대시보드에서 `Settings(설정) > Integrations(연동) > TikTok` 카드로 이동하세요.

## 틱톡용 대화 인박스

들어오는 DM과 댓글이 다른 채널과 마찬가지로 인박스에 동기화되어 다음을 쉽게 할 수 있습니다:

- 프로필 사진, 사용자명, 메시지 기록을 하나의 스레드에서 확인하세요.

- 텍스트, 이모지, 미디어 또는 저장된 스니펫으로 수동 응답하세요.

- 빠른 작업 버튼을 사용해 노트, 태그 또는 할 일을 추가하세요.

## 틱톡 자동화 구성 요소

**트리거**

- **틱톡 릴 댓글**: 틱톡 게시물에 새 댓글이 달리면 워크플로우가 시작됩니다. 이 트리거에 대한 자세한 정보는 여기를 클릭하세요.

- **고객 답변**: 틱톡 메신저와 같은 답변 채널을 선택하세요. 이 트리거에 대한 자세한 정보는 여기를 클릭하세요.

**액션**

- **댓글로 답변**: 댓글 아래에 댓글로 답변합니다. 이 액션에 대한 자세한 정보는 여기를 클릭하세요.

- **틱톡 인터랙티브 메신저**: 자동 답변이나 육성 시퀀스를 자동화합니다.

## 틱톡 DM 자동화

누군가 개인 메시지를 보내면 대화형 후속 조치를 자동화하세요.

- "고객 답변" 트리거와 "답변 채널 = 틱톡" 필터를 결합하면 틱톡 DM에 대해서만 워크플로우가 실행됩니다.

- 틱톡 인터랙티브 메시지 액션을 사용해 왓츠앱 인터랙티브 메시지와 유사한 리치 버튼 답변을 보내세요.

---

## 틱톡 댓글 자동화

댓글 섹션을 리드 마그넷으로 전환하세요:

- 모든 게시물 또는 특정 게시물에 대해 "틱톡 릴 댓글" 트리거를 설정하세요.

- "댓글로 답변" 액션을 사용해 공개적으로 답변하거나 댓글에 좋아요를 누르거나 사용자를 태그하세요.

- DM으로 후속 조치하거나 연락처를 담당자에게 배정하세요.

## 트리거 설정 방법: 틱톡 릴 댓글

#### 1단계: 트리거 추가

워크플로우 빌더에서 + Add New Trigger(새 트리거 추가)를 클릭하세요. TikTok - Comment(s) on a Post 트리거를 선택하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055994218/original/B4fMb4rCv5MZPkVg3USOjmM-hRHfQEu52g.png?1760464547)

#### 2단계: 계정 선택

트리거를 클릭하면 사이드바가 열립니다. 여러 필터가 있습니다. 첫 번째 단계는 Account(계정)를 선택하는 것입니다.

필터에서 특정 계정이 선택되지 않으면, 시스템은 로케이션 레벨에서 연동된 모든 계정을 자동으로 고려합니다. 자동화는 이렇게 연결된 모든 계정에서 실행됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055994237/original/2cBuRls08NB9ZGkCKICAmfCsiNBAD0I9tQ.png?1760464588)

#### 3단계: 게시물 유형 선택

페이지를 선택한 후 Post Type(게시물 유형)을 선택해야 합니다. 게시물 유형은 "Published(발행됨)" 또는 "Custom(커스텀)"이 될 수 있습니다.

- **발행된 게시물**: '발행된 게시물' 탭은 비즈니스 페이지의 모든 게시물을 가져옵니다. 이 선택에 따라 계정의 모든 게시물이 드롭다운으로 나타납니다.

- **커스텀 게시물**: 커스텀 게시물 탭을 사용하면 URL을 제공하여 고려할 게시물을 정의할 수 있습니다.

**게시물 선택 동작**: 특정 게시물이 선택되지 않으면, 시스템은 연결된 틱톡 계정과 관련된 모든 게시물을 자동으로 포함합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055994265/original/W8gtEHtViF_zVt7VKCR7b6p6CAlp5mkaJQ.png?1760464645)

#### 4단계: 게시물 또는 URL 선택

자동화를 실행할 게시물을 선택하거나 게시물의 URL을 제공하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055994406/original/-OekPY-yvKmo4wGSl8A1e8U1V0aKBB8uqg.png?1760464696)

#### 5단계: 일치 문구 지정

게시물을 선택한 후 댓글에서 찾고 있는 내용을 입력해야 합니다. "Contains Phrase(문구 포함)"와 "Exact Match(정확히 일치)" 두 가지 옵션 중 선택할 수 있습니다. 다음은 이 두 드롭다운을 더 잘 이해하기 위한 예시입니다.

- 정확히 일치 - 가격
입력 메시지 - 가격
결과 - 통과

- 정확히 일치 - 가격 알려주세요
입력 메시지 - 가격 알려주세요
결과 - 통과

- 정확히 일치 - 가격 알려주세요
입력 메시지 - 가격을 알려주세요
결과 - 실패

- 정확한 문구 포함 - 구매했어요
입력 메시지 - 구매했어요
결과 - 통과

- 정확한 문구 포함 - 구매했어요
입력 메시지 - 어제 구매했어요
결과 - 통과

- 정확한 문구 포함 - 구매했어요
입력 메시지 - 하나 구입했어요
결과 - 실패

트리거를 통해 연락처가 들어오면 연락처로 저장되고 연락처의 이름과 성이 저장됩니다.

## 트리거 설정 방법: 고객 답변

워크플로우를 생성할 때 "Add Trigger(트리거 추가)"를 클릭하세요. DM 자동화와 관련된 트리거인 "Customer replied(고객 답변)"는 Events(이벤트) 카테고리에 있습니다. 트리거를 직접 검색하거나 카테고리로 스크롤하여 찾을 수 있습니다.

#### 1단계: 트리거 추가

트리거로 이동하여 사용 사례에 따라 사용 가능한 트리거 중에서 선택하세요. 여기서는 Customer replied(고객 답변) 트리거를 선택해야 합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055994597/original/dUmsjjmFbbINboUXWGhsjQT1ksmdE0hl2g.png?1760464847)

#### 2단계: 답변 채널 추가

트리거를 클릭하면 사이드바가 열립니다. 여러 필터가 있습니다. 첫 번째 단계는 "Reply channel(답변 채널)" 필터를 선택하는 것입니다. 여기서 필터 값은 TikTok입니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055994631/original/GWT4pYt4pXO62pOHnrQm7lDKb_EvE2-bGA.png?1760464872)

#### 3단계: 자동화 규칙 구성

사용자는 '문구 포함' 또는 '정확한 문구 일치'와 같은 특정 일치 조건을 설정하여 다이렉트 메시지에 대한 자동화 규칙을 구성할 수 있습니다. 이러한 구성을 기반으로 자동화의 트리거를 정의할 수 있습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055994665/original/9dbR6hCxeJF6_BRNXJZXCEyW8NSkyaHinA.png?1760464911)

## 액션 설정 방법: 틱톡 인터랙티브 메신저

#### 1단계: 액션 추가

워크플로우 빌더에서 + Add New Action(새 액션 추가)를 클릭하세요. Tiktok Interactive Messenger 액션을 선택하세요.

#### 2단계: 메시지 제공

틱톡 인터랙티브 메신저는 상당히 간단합니다. 여기서는 정의된 인바운드 트리거로 보낼 메시지를 제공하고 액션을 저장해야 합니다.

주의할 중요한 점들:

- 틱톡 인터랙티브 메신저 액션은 FB & IG와 달리 사용자가 "Reply as DM(DM으로 답변)", "Reply to Comment via DM(DM을 통해 댓글에 답변)" 값을 정의할 수 있는 "Reply Type(답변 유형)" 필드를 지원하지 않습니다. 틱톡 API가 이를 지원하기 시작하면 활성화됩니다.

- 틱톡 인터랙티브 메신저 액션은 FB & IG와 달리 템플릿을 지원하지 않습니다. 점진적으로 활성화될 예정입니다.

- 틱톡 인터랙티브 메신저 액션은 첨부파일, 버튼, 빠른 답변을 지원하지 않습니다. 틱톡 API가 이를 지원하기 시작하면 활성화됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055994713/original/0df-St2_VtcFVMu8-xzN_8PiJvr2gpCE3Q.png?1760464998)

## 자주 묻는 질문

**Q: 댓글로 답변 액션에 몇 개의 답변 변형을 추가할 수 있나요?**

최대 10개의 답변을 무작위 순서로 사용할 수 있습니다.

**Q: 특정 게시물에 자동화를 범위 지정할 수 있나요?**
네. 틱톡 – 릴 댓글 트리거를 사용하고 모니터링할 특정 비디오/게시물을 선택하세요.

**Q: 여러 틱톡 계정을 연결할 수 있나요?**
하위 계정 레벨에서 사용하려는 계정을 인증할 수 있습니다. 워크플로우를 구축할 때 트리거와 필터를 올바른 계정으로 범위 지정하세요.
