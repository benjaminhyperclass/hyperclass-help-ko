# Ask AI에서 MCP 서버 사용하기

Ask AI가 단순히 질문에 답하는 것을 넘어서 이제 실제 작업을 수행합니다. 연락처를 생성하거나, 예약을 변경하거나, 기회를 일괄 업데이트해야 할 때, 이제는 채팅으로 지시만 하면 Hyperclass의 MCP(Master Control Panel)가 즉시 작업을 처리합니다. 메뉴를 찾아 다니거나 탭을 전환할 필요 없이, 대화형 명령어로 시간을 크게 절약할 수 있습니다.

## MCP 기반 Ask AI의 핵심 이점

교육용 개요: 채팅을 액션 엔진으로 바꾸면 팀 운영 방식이 완전히 달라집니다. 복잡한 클릭 경로를 한 문장으로 압축할 수 있습니다.

- 
**제로 클릭 워크플로우**: "Jane Smith를 'NY 리드' 파이프라인에 추가해줘"라고 입력하면 완료됩니다.

- 
**맥락 ↔ 액션 루프**: 데이터를 요청한 후 같은 대화에서 바로 작업을 수행할 수 있습니다("이분들을 Hot 태그로 표시해줘").

- 
**전방위 커버리지**: MCP API가 지원하는 모든 기능(연락처, 결제, 자동화 등)을 채팅에서 실행할 수 있습니다.

- 
**역할 기반 보안**: 권한은 사용자 역할과 일치하므로 누구도 자신의 접근 수준을 초과할 수 없습니다.

- 
**감사 및 취소**: 모든 액션은 활동 피드에 기록되며, 위험한 명령은 확인을 요청합니다.

## 작동 원리

- 
**의도 분석**: AI가 동사(생성, 업데이트, 삭제)와 객체(연락처, 인보이스)를 식별합니다.

- 
**페이로드 생성**: 사용자의 말을 MCP 필드(예: firstName, pipelineId)로 매핑합니다.

- 
**권한 확인**: 시스템이 해당 액션을 실행할 권한이 있는지 확인합니다.

- 
**확인 (위험한 액션만)**: 삭제나 일괄 편집의 경우, Ask AI가 요약을 보여주고 "진행할까요?"라고 묻습니다.

- 
**실행 및 응답**: MCP가 호출을 실행하고, 성공 또는 오류 세부 정보가 인라인으로 반환되며 감사 피드에 기록됩니다.

## 명령어 구문 및 예시

| 액션 유형 | 자연어 예시 | 백그라운드 MCP 호출 |
|----------|------------|-------------------|
| 생성 | "로케이션 A에 전화번호 555-1234인 John Doe라는 연락처를 만들어줘." | createContact() |
| 업데이트 | "기회 #456의 파이프라인 단계를 '성사'로 변경해줘." | updateOpportunity() |
| 조회 | "내일 예약된 약속들을 보여줘." | listAppointments() |
| 삭제 | "쿠폰 코드 SUMMER25를 삭제해줘." | deleteCoupon() (확인 필요) |
| 일괄 | "지난주 웨비나 참석자 모두에게 '웨비나-2025' 태그를 추가해줘." | updateContact() 반복 |

팁: 마지막에 "그리고 결과를 보여줘"라고 추가하면 빠른 검증을 위한 JSON 스니펫을 받을 수 있습니다.

## 보안 및 권한

- 
**역할 기반 제한**: 채팅 권한은 UI 권한과 동일하며, 비밀 백도어는 없습니다.

- 
**확인 프롬프트**: 되돌릴 수 없는 변경사항은 사람의 "예/아니오" 단계를 거칩니다.

- 
**감사 로그**: Settings(설정) ▸ Audit Logs(감사 로그) ▸ AI Chat(AI 채팅)에서 AI가 시작한 모든 액션을 확인할 수 있습니다.

- 
**크레딧 사용량**: 성공한 MCP 명령어당 1 AI 크레딧이 소모되며, 미리보기/거부된 명령은 무료입니다.

## 권장 프롬프트 예시

- 
**단일 액션 명령**
"john@demo.com 연락처에게 30일 후 결제 예정인 $299 인보이스를 생성해줘."

- 
**2단계 데이터 → 액션**
"이번 주에 실패한 기회들을 나열하고, '재참여' 단계로 이동시켜줘."

- 
**조건부 로직**
"'평가판-종료' 태그가 있는 연락처 중에서 마지막 로그인이 14일 전인 경우, '이탈 위험' 파이프라인으로 이동시켜줘."

## 자주 묻는 질문

**Q: AI가 실제로 서버에 보낸 내용을 어떻게 확인하나요?**
A: 채팅 응답의 타임스탬프를 클릭하면 감사 로그 사이드바에서 원시 JSON 페이로드를 볼 수 있습니다.

**Q: 실수한 액션을 되돌릴 수 있나요?**
A: 5분 이내에 "마지막 액션 취소"를 사용하거나 백업(연락처 CSV 등)에서 복원할 수 있습니다.

**Q: 일괄 액션에 제한이 있나요?**
A: 네, 시스템 성능 보호를 위해 분당 1,000건으로 제한됩니다.

**Q: 모바일에서도 작동하나요?**
A: 현재는 읽기 전용이며, MCP 액션은 2025년 4분기에 iOS 및 Android에서 출시 예정입니다.

**Q: 특정 동사(예: 삭제)를 제한할 수 있나요?**
A: 관리자는 Settings(설정) ▸ Security(보안) ▸ AI Controls(AI 제어)에서 역할별로 'AI 삭제 허용'을 토글할 수 있습니다.

**Q: AI가 내 프롬프트를 잘못 이해하면 어떻게 되나요?**
A: 명확화 질문으로 응답하며, 확인할 때까지 어떤 액션도 실행되지 않습니다.
