# Agent Studio에서 AI 대화를 테스트하고 디버깅하는 방법

Agent Studio의 메시지 실행 타임라인과 채팅 에뮬레이터를 통해 AI 대화의 모든 단계를 완전히 파악할 수 있어, 더 빠른 디버깅과 병목지점 식별이 가능하며 자신 있게 AI 에이전트를 배포할 수 있습니다.

## 메시지 실행 타임라인과 채팅 에뮬레이터란?

메시지 실행 타임라인(Message Execution Timeline)과 채팅 에뮬레이터(Chat Emulator)는 Agent Studio에 내장된 도구로, 완전한 가시성을 제공하여 AI 대화를 테스트하고 디버깅할 수 있게 해줍니다. 타임라인은 AI 에이전트가 수행하는 모든 이벤트, 도구 호출, 결정 사항을 모델 지표와 함께 기록하며, 채팅 에뮬레이터를 통해서는 대화를 시뮬레이션하고 세션을 재설정하며 특정 응답을 담당하는 노드로 직접 이동할 수 있습니다. 이 두 도구가 함께 작동하여 에이전트 동작을 명확하게 파악할 수 있어, 문제를 식별하고 로직을 검증하며 성능을 자신 있게 개선할 수 있습니다.

이러한 도구들은 각 실행 단계에서 정확히 무슨 일이 일어나는지 보여줌으로써 AI 워크플로우에서 흔히 발생하는 "블랙박스" 효과를 제거하여, 로직을 추적하고 입력과 출력을 검사하며 실제 데이터를 바탕으로 타겟 개선을 수행할 수 있게 합니다.

## AI 대화 테스트 및 디버깅의 주요 이점

Agent Studio에서 AI 대화를 테스트하고 디버깅하면 에이전트가 어떻게 생각하고, 응답하며, 성능을 발휘하는지 완전히 파악할 수 있습니다. 실시간 실행 로그와 운영 환경과 유사한 테스트 환경을 결합하여 로직을 검증하고 병목지점을 식별하며 라이브 환경 배포 전에 성능을 개선할 수 있습니다.

- **완전한 실행 투명성**: 모든 노드 전환과 도구 호출을 정확한 순서대로 추적하여 추측의 여지를 없앱니다.

- **단계별 성능 인사이트**: 느린 프롬프트, API 호출, 도구 단계를 즉시 감지하고 더 빠른 응답을 위해 최적화할 수 있습니다.

- **모델 및 비용 인식**: 각 메시지와 함께 모델 세부사항, 토큰 사용량, 온도 및 기타 실행 설정을 직접 검토할 수 있습니다.

- **즉시 문제 해결**: 로그 항목에서 담당 노드로 바로 이동하여 빠르게 타겟 수정을 수행할 수 있습니다.

- **현실적인 테스트 환경**: 최종 사용자 경험을 반영하는 채팅 인터페이스에서 대화를 검증하여 정확한 QA를 보장합니다.

## 메시지 실행 타임라인

타임라인은 채팅 에뮬레이터 옆에 나타나며 에이전트가 실행될 때 실시간으로 업데이트됩니다.

- 이벤트 카드는 노드 이름, 모델, 입출력 토큰, 실행 시간을 나열합니다
- 색상별 지연 시간 배지(빨간색 = 느림; 녹색 = 빠름)로 병목지점을 강조표시합니다
- 마우스를 올리면 상세 검사를 위한 원시 JSON 입출력을 볼 수 있습니다

![타임라인에 단계별 지연 시간 배지가 표시된 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066089570/original/wVBB5PA0V4Q_WNDmHHcxop9xlJHMxjeQEQ.png?1772540757)

## 전환 노드 버튼

**실행 타임라인**에서 각 단계는 보라색 라벨을 표시합니다. 이 클릭 가능한 **전환 라벨**은 실행 중 대화가 이동한 정확한 노드를 나타냅니다. 라벨을 클릭하면 해당 노드가 캔버스에서 자동으로 포커스되고 강조표시되어, 플로우를 수동으로 검색할 필요 없이 즉시 로직을 검토하거나 수정할 수 있습니다.

이를 통해 실행 로그를 비주얼 빌더와 직접 연결하여 디버깅을 더 빠르게 수행할 수 있습니다.

## 성능 지표 패널

각 메시지 아래 지표 푸터에는 다음이 표시됩니다:

- 모델 및 버전 (예: gpt-4-turbo)
- 입력/출력 토큰 수
- 온도, 최대 토큰, top-p, 빈도/존재 패널티
- 총 비용 추정치 (청구가 활성화된 경우)

이 정보는 테스트 중 성능과 효율성을 모두 평가하는 데 도움이 됩니다.

## 채팅 에뮬레이터

채팅 에뮬레이터는 AI 대화를 안전하게 테스트하는 데 사용되는 시뮬레이션 채팅 인터페이스입니다. 실제 배포 시나리오를 반영하는 간소화된 테스트 환경을 제공합니다.

- 아바타를 지원하는 깔끔한 모바일 반응형 채팅 창
- 실행 간 컨텍스트를 지우는 재설정 버튼
- 동시 검토를 위한 타임라인과의 나란히 배치

## Agent Studio에서 AI 대화를 테스트하고 디버깅하는 방법

이 섹션에서는 타임라인과 채팅 에뮬레이터를 함께 사용하여 실행 단계를 검사하고, 문제를 식별하며, 에이전트를 게시하기 전에 성능을 최적화하는 방법을 안내합니다.

### **테스트 환경 열기**

AI Agents → Agent Studio로 이동하여 테스트할 에이전트를 엽니다. 오른쪽 상단의 Test 탭을 클릭하여 채팅 에뮬레이터와 실행 뷰에 접근합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066089570/original/wVBB5PA0V4Q_WNDmHHcxop9xlJHMxjeQEQ.png?1772540757)

### **테스트 세션 시작**

Start를 클릭하여 채팅 에뮬레이터를 실행합니다. 메시지 실행 타임라인이 채팅 창과 함께 자동으로 나타납니다.

에뮬레이터에서 테스트 메시지를 보내거나 API를 통해 에이전트를 트리거하세요. 에이전트가 요청을 처리하면서 각 노드 전환, 도구 호출, 모델 실행 단계가 타임라인 패널에 실시간으로 로그됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066089707/original/DqrJ6Bfd2KGhUPxFr2HYxbEtmjDpzx3W6g.png?1772540838)

### **실행 로그 검토**

타임라인의 이벤트 카드를 검사하여 에이전트가 대화를 어떻게 처리했는지 이해하세요. 노드 이름, 도구 활동, 토큰 사용량, 실행 시간, 그리고 느린 단계를 나타내는 지연 시간 배지를 확인하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066090103/original/dC8gPiZ0kekZj0pDJ0kjtOK98YsMsBuKhA.png?1772541043)

### **성능 지표 검사**

개별 로그 항목을 클릭하여 모델 버전, 입출력 토큰, 온도, 비용 추정치(활성화된 경우) 등 상세 지표를 검토합니다. 이 정보를 사용하여 효율성을 평가하고 프롬프트나 도구 구성을 미세 조정하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066090232/original/52PnOW_D9wLDTPr6gWQWpMpHp8Rkb34YTw.png?1772541147)

### **소스 노드로 이동**

로그 카드를 클릭하면 빌더에서 해당 노드가 자동으로 강조표시됩니다. 이를 통해 캔버스를 수동으로 검색하지 않고도 로직, 프롬프트, 변수를 빠르게 조정할 수 있습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066094090/original/oITv_TT5kxyPX5biTAVa1yBKiPp9Qbm8nw.png?1772543340)

### **재설정 및 재테스트**

채팅 에뮬레이터의 Reset 버튼을 사용하여 대화와 실행 로그를 지우세요. 업데이트가 문제를 해결했는지 확인하기 위해 대화를 다시 실행하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066091467/original/pDXZVnJOZ0U3mfLRl8Nluz8evYXGGT8EQA.png?1772541918)

### **자신 있게 게시**

테스트를 통해 대화가 예상대로 작동하고 성능이 최적화되었음을 확인하면, 평소처럼 에이전트를 스테이징 또는 운영 환경으로 이동시키세요.

## 자주 묻는 질문

**Q: Voice AI 에이전트에서도 타임라인을 사용할 수 있나요?**

네. Voice AI 에이전트의 Test 탭을 열면 테스트 통화가 완료된 후 동일한 로그 뷰가 나타납니다.

**Q: 타임라인이 토큰 사용량이나 비용을 증가시키나요?**

아니요. 타임라인은 모델과 도구 노드에서 이미 반환된 메타데이터를 기록합니다.

**Q: 타임라인 로그를 내보낼 수 있나요?**

타임라인 우측 상단의 다운로드 아이콘을 클릭하여 전체 실행의 JSON 파일을 저장할 수 있습니다.

**Q: 노드로 이동 버튼이 보이지 않는 이유는 무엇인가요?**

이 버튼은 에이전트가 최소 한 번 저장된 후에만 나타납니다. 저장되지 않은 초안을 보고 있지 않은지 확인하세요.

**Q: 채팅 에뮬레이터를 재설정하면 타임라인도 지워지나요?**

네. 재설정하면 채팅 컨텍스트와 관련 실행 로그가 모두 지워져 새로운 세션이 시작됩니다.

**Q: 이 기능이 특정 요금제로 제한되나요?**

타임라인과 채팅 에뮬레이터는 Agent Studio에 필요한 것과 동일한 요금제인 Unlimited AI Employee에 포함됩니다.
