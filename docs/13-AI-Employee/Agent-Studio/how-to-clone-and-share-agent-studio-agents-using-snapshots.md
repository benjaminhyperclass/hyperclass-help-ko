# 스냅샷을 사용하여 에이전트 스튜디오 에이전트를 복제하고 공유하는 방법

계정 스냅샷(Account Snapshots)을 사용해서 최고 성과를 내는 AI 에이전트를 하위 계정 간에 복제, 배포, 배치할 수 있습니다. 이 가이드에서는 에이전트 스튜디오(Agent Studio) 에이전트를 패키징하고, 종속 요소를 포함시키고, 다른 계정에 로드하고, 스냅샷 링크나 마켓플레이스를 통해 공유하거나 판매하는 방법을 설명합니다.

# **에이전트 스튜디오 스냅샷이란?**

에이전트 스튜디오 스냅샷을 사용하면 하나 이상의 AI 에이전트를 계정 스냅샷 안에 패키징하여 다른 하위 계정에 로드할 수 있습니다.

스냅샷에 포함될 때 다음 항목이 보존됩니다:

- 에이전트 플로우 구조
- 노드 구성
- 연결된 지식 베이스
- 버전 상태 (초안, 스테이징, 프로덕션)
- 관련 지원 자산 (선택한 경우)

이를 통해 새 계정마다 에이전트를 수동으로 다시 만들 필요가 없어집니다.

---

## **에이전트 스튜디오에서 스냅샷 사용의 주요 이점**

스냅샷을 사용해서 에이전트를 배포하면 에이전시가 더 빠르게 확장하고 일관성을 유지할 수 있습니다.

- **에이전트 즉시 복제**: 복잡한 AI 에이전트를 몇 분 안에 다른 계정으로 복사할 수 있습니다.

- **구성 보존**: 에이전트가 버전 상태와 로직을 유지합니다.

- **종속성 포함**: 지식 베이스와 선택된 자산이 에이전트와 함께 이동합니다.

- **온보딩 표준화**: 새 고객이 완전히 구성된 AI 자동화로 시작할 수 있습니다.

- **배포 및 수익화 가능**: 스냅샷을 외부로 공유하거나 판매할 수 있습니다.

---

## 에이전트 스냅샷 만들기

에이전트 스튜디오 에이전트를 패키징하려면 다음 단계를 따르세요.

### **1단계: 계정 스냅샷으로 이동**

- **에이전시 보기(Agency View)**로 전환
- **계정 스냅샷(Account Snapshots)**으로 이동
- **새 스냅샷 만들기(Create New Snapshot)** 클릭

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066819121/original/HleAevJiPdYqrKr6yERei_HgxdWf0ItOwA.png?1773334189)

### **2단계: 이름 지정 및 소스 계정 선택**

- 스냅샷 이름 입력
- 에이전트가 포함된 하위 계정 선택
- **다음(Next)** 클릭

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066819528/original/-3Nvfr6v8TfkHQUk8mvZX-TdVC2J8AiqVQ.png?1773334423)

### **3단계: 에이전트 스튜디오 자산 선택**

- **에이전트 스튜디오(Agent Studio)** 카테고리 펼치기
- 포함할 에이전트 선택
- 지원 자산(예: 지식 베이스, 워크플로우, 커스텀 값) 선택
- 오른쪽 아래의 **만들기(Create)** 탭 클릭

시스템이 선택된 에이전트와 자산을 스냅샷으로 패키징합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066819942/original/SlEEvHp3IJoXKKHX2SHt6bdpcEMP-LjuDw.png?1773334679)

**중요**: 에이전트가 지식 베이스를 사용하는 경우, 스냅샷을 만들 때 해당 지식 베이스를 반드시 선택해야 합니다.

스냅샷은 선택한 자산만 패키징합니다.

연결된 지식 베이스가 선택되지 않으면, 에이전트가 참조 자료 없이 로드될 수 있습니다.

진행하기 전에 항상 자산 선택 화면을 검토하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066838299/original/awGX4bvcPjZjTuR9X__U2w4lnVrFoZQHVg.png?1773368022)

### **4단계: 스냅샷 세부 정보 검토**

만든 후에는 스냅샷 세부 정보 화면을 볼 수 있습니다.

이 화면에서 다음을 확인합니다:

- 포함된 에이전트
- 포함된 지식 베이스
- 총 자산 수

## **하위 계정에 에이전트 스냅샷 로드하기**

스냅샷을 하나 또는 여러 하위 계정에 로드할 수 있습니다.

### **옵션 1: 단일 하위 계정에 로드**

- **에이전시 보기(Agency View) → 하위 계정(Sub-Accounts)**으로 이동

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066838348/original/b3ysmTJbUNGOoaztWYjLZTm_vmWP-0x0dw.png?1773368390)

- 계정 옆의 점 3개 메뉴 클릭. **클라이언트 관리(Manage Client)** 옵션 선택

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066838382/original/TY9XBjHiWjLqh5z1i1EhWozAjazwGEWjTA.png?1773368520)

- **작업(Actions) → 스냅샷 로드(Load Snapshot)** 선택

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066838434/original/1nJUggswqmXRs19WqKdwhOfQBO62tkAL0g.png?1773368685)

- 스냅샷 선택

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066838458/original/TDnGVjFheDThnS-zlInvlNm74t6egphrHA.png?1773368798)

- 에이전트 스튜디오 카테고리 선택 (또는 모두 선택). 가져오기 확인

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066838579/original/0RPunN1c3JMGsagchAbL1iJPIyxUvG1xFg.png?1773369214)

로드 후 에이전트가 대상 하위 계정의 에이전트 스튜디오에 나타납니다.

### 옵션 2: 여러 하위 계정에 스냅샷 배포

- **계정 스냅샷(Account Snapshots)**으로 이동
- 스냅샷 열기
- **스냅샷 배포(Push Snapshot)** 클릭
- 하위 계정 선택
- 자산 선택
- 충돌 검사를 통해 진행

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066838680/original/5HmXCmjbP5dgBLNWv7Loo3FNE4t4psGl_Q.png?1773369590)

이 방법을 통해 대량 배포가 가능합니다.

## **가져오기 중 충돌 해결**

같은 이름의 자산이 대상 계정에 이미 존재하는 경우, Hyperclass가 충돌을 해결하라는 메시지를 표시합니다.

다음 중 하나를 선택할 수 있습니다:

- 두 복사본 모두 유지
- 기존 항목 덮어쓰기
- 충돌 검사와 함께 진행

완료하기 전에 항상 충돌을 검토하세요.

---

## **스냅샷 버전 관리 및 새로고침**

각 스냅샷은 버전을 유지합니다.

소스 계정에서 에이전트를 업데이트한 경우:

- **계정 스냅샷(Account Snapshots)**으로 돌아가기
- 스냅샷 선택
- **새로고침(Refresh)** 클릭

이렇게 하면 최신 에이전트 구성을 반영하도록 스냅샷이 업데이트되고 새 버전이 생성됩니다.

그런 다음 업데이트된 버전을 하위 계정에 배포할 수 있습니다.

---

## **에이전트 스냅샷 공유 또는 판매**

스냅샷을 에이전시 외부로 공유할 수 있습니다.

공유하려면:

- **계정 스냅샷(Account Snapshots)**으로 이동
- 점 3개 메뉴 클릭
- **스냅샷 공유(Share Snapshot)** 선택
- 공유 링크 또는 마켓플레이스 등록 생성

수신자는 자신의 Hyperclass 계정에 스냅샷을 로드할 수 있습니다.

참고: 마켓플레이스 수익화에는 SaaS 플랜 요구 사항이 적용됩니다.

---

## **스냅샷에 포함되지 않는 항목**

스냅샷에는 다음이 포함되지 **않습니다**:

- 연락처(Contacts)
- 대화 기록
- 메시지
- 실시간 데이터

구성과 선택된 자산만 전송됩니다.

---

## **필요한 권한**

사용자는 다음이 있어야 합니다:

- 만들기 위한 "스냅샷 만들기" 권한
- 로드하거나 공유하기 위한 "스냅샷 공유/가져오기" 권한

권한은 관리자가 관리합니다.

---

## **자주 묻는 질문**

**Q: 스냅샷을 로드하면 기존 에이전트를 덮어쓰나요?**
같은 ID를 가진 에이전트가 존재하는 경우에만 그렇습니다. 충돌을 해결하라는 메시지가 표시됩니다.

**Q: 지식 베이스가 자동으로 포함되나요?**
스냅샷 만들기 중에 선택한 경우에만 포함됩니다.

**Q: 내 에이전시 외부에서도 스냅샷을 로드할 수 있나요?**
예, 공유 링크나 마켓플레이스 배포를 사용하여 가능합니다.

**Q: 버전 상태가 보존되나요?**
예. 초안, 스테이징, 프로덕션 상태가 그대로 유지됩니다.

**Q: 가져오기가 API 엔드포인트에 영향을 주나요?**
API 엔드포인트는 사용 가능한 상태로 유지되지만, 계정별 토큰은 업데이트가 필요할 수 있습니다.
