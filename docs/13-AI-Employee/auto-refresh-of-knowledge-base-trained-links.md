# 지식 베이스 훈련 링크 자동 새로고침

번거로운 작업 없이 대화 AI(Conversation AI) 봇의 정확성을 유지하세요. 지식 베이스(Knowledge Base) 훈련 링크 자동 새로고침은 하이퍼클래스가 웹 URL 소스를 매일, 매주 또는 매월 자동으로 다시 크롤링하고 재훈련하게 해주므로, 팀이 더 가치 있는 업무에 집중하는 동안 고객 응답이 최신 상태를 유지합니다. 한 번 설정하면 단일 테이블 뷰에서 상태와 로그를 모니터링할 수 있습니다.

---

## 지식 베이스 훈련 링크 자동 새로고침이란?

자동 새로고침은 지식 베이스에 추가한 웹 URL의 자동 재훈련을 예약하는 하이퍼클래스의 기능입니다. 각 URL마다 수동으로 **새로고침** 버튼을 클릭하는 대신, 하이퍼클래스가 주기적으로 페이지를 다시 크롤링하고 테이블 데이터를 업데이트하며, 봇이 의존하는 최신 콘텐츠를 다시 색인화합니다. 이를 통해 수동 유지보수 없이도 봇이 최신 제품 페이지, 도움말 문서, 블로그 글을 참조할 수 있습니다.

자동 새로고침은 반복적인 수동 업데이트 필요성을 제거하고 오래된 답변의 위험을 줄입니다. 지식 베이스 URL을 일정에 따라 지속적으로 동기화함으로써, 이 기능은 봇이 가장 최신 소스 자료에서 가져온 일관되고 신뢰할 수 있는 응답을 제공하도록 도와줍니다.

---

## **자동 새로고침의 주요 장점**

장점을 이해하면 적절한 주기와 범위를 결정하는 데 도움이 됩니다. 이러한 장점은 특히 여러 하위 계정(Sub-account)에서 많은 링크를 관리하는 팀에게 정확성, 시간 절약, 운영 가시성에 초점을 맞춥니다.

- **항상 최신 답변:** 봇이 더 높은 정확성과 신뢰성을 위해 최신 콘텐츠를 사용하도록 보장합니다.

- **대규모 시간 절약:** 특히 하위 계정 전반에서 많은 URL을 관리할 때 수동 새로고침 작업을 줄입니다.

- **오래된 응답 감소:** 오래된 안내, 가격 또는 정책이 고객에게 전달되는 것을 방지합니다.

- **인적 오류 위험 감소:** 반복적인 새로고침 작업을 자동화하므로 한 번의 클릭 실수가 소스를 오래된 상태로 남겨두지 않습니다.

- **예측 가능한 스케줄링:** 일일, 주간 또는 월간 주기를 선택하여 콘텐츠 업데이트 리듬에 맞춥니다.

- **부하 관리:** 새로고침을 분산시켜 피크 시간 크롤링을 피하고 대형 사이트의 성능을 관리합니다.

---

## **새로고침 빈도 설정**

스케줄링은 하이퍼클래스가 URL을 자동으로 다시 크롤링하는 빈도를 제어합니다. 소스 콘텐츠가 업데이트되는 빈도와 URL 라이브러리 크기에 맞는 빈도를 선택하세요.

자동 새로고침 아이콘(순환 화살표)을 클릭하고 원하는 일정을 선택하세요:

- **일일:** 약 24시간마다 새로고침합니다.

- **주간:** 선택한 요일에 새로고침합니다.

- **월간:** 매월 같은 날짜에 새로고침합니다.

---

## **새로고침 상태 및 로그 모니터링**

각 크롤링 주기에 대한 가시성은 새로고침이 작동하는지 확인하고 그렇지 않으면 오래된 콘텐츠를 남겨둘 오류를 신속히 해결하는 데 도움이 됩니다.

**상태(Status)** 컬럼은 각 URL의 현재 상태를 반영합니다:

- **데이터 가져오는 중(Getting Data):** 하이퍼클래스가 활발히 페이지를 다시 크롤링하고 있습니다.

- **훈련 완료(Trained):** 최신 데이터가 저장되었습니다. 타임스탬프는 **마지막 데이터 새로고침 시간**을 확인하세요.

- **실패(Failed):** 크롤링에서 오류가 발생했습니다. 새로고침을 다시 시도하거나 페이지 접근성을 검토하세요.

## **지식 베이스 훈련 링크 자동 새로고침 설정 방법**

신뢰할 수 있는 설정은 새로고침이 정시에 실행되고 봇이 최신 콘텐츠의 이익을 얻도록 보장합니다. 놓친 일정을 피하기 위해 이 단계들을 순서대로 완료하세요.

- 하위 계정에서 AI 에이전트(AI Agents) → 지식 베이스(Knowledge Base)로 이동합니다.

## **건강한 지식 베이스를 위한 모범 사례**

이러한 모범 사례를 따르면 크롤링 안정성이 향상되고 실패가 줄어들며, 봇이 대규모로 더 정확한 답변을 제공하는 데 도움이 됩니다.

- **안정적인 URL 사용:** 자주 리다이렉트되거나 콘텐츠를 보기 위해 로그인이 필요한 주소는 피하세요.

- **페이지를 가볍게 유지:** 매우 길거나 스크립트가 많은 페이지는 크롤링이 느려질 수 있고 타임아웃에 걸릴 수 있습니다.

- **주간 실패 감사:** **실패** 상태에 멈춰 있는 행을 조사하고, 깨진 링크를 수정한 후 다시 시도하세요.

- **무거운 사이트 분산:** 대형 라이브러리의 경우, 시간에 걸쳐 부하를 분산시키기 위해 **주간** 또는 **월간**을 선호하세요.

- **중요한 페이지 우선순위:** 자주 변경되고 고객에게 직접 영향을 주는 페이지는 **일일**로 설정하세요.

## 자주 묻는 질문

**Q: 자동 새로고침이 추가 크레딧 비용을 발생시키나요?**
A: 아니요. URL 재크롤링은 기존 대화 AI 사용량에 포함됩니다.

**Q: 새로고침이 실패하면 어떻게 되나요?**
A: **상태**가 **실패**로 변경되고 이전에 훈련된 콘텐츠가 활성 상태로 유지됩니다. URL을 수정하거나 수동으로 다시 시도하세요.

**Q: 자동 새로고침이 커스텀 FAQ 쌍을 업데이트하나요?**
A: 아니요. 커스텀 Q&A 항목은 정적입니다. 필요에 따라 수동으로 업데이트하세요.

**Q: 자동 새로고침이 단일 URL뿐만 아니라 도메인 크롤링에서도 작동하나요?**
A: 예. 자동 새로고침은 웹 URL 테이블에 해당 URL이 나타나는 곳 어디든 적용할 수 있습니다. 주기와 성능은 페이지 깊이와 크기에 따라 달라집니다.

**Q: 자동 새로고침을 일시적으로 일시정지할 수 있나요?**
A: 예. **자동화 관리(Manage Automation)**를 클릭하고 자동 새로고침을 **비활성화**하세요. 데이터 손실 없이 나중에 다시 활성화할 수 있습니다.

**Q: Google 문서나 PDF가 지원되나요?**
A: 자동 새로고침은 현재 공개적으로 접근 가능한 웹 페이지에 초점을 맞춥니다. 문서의 경우 웹에서 접근 가능한 페이지로 변환하거나 게시하세요.
