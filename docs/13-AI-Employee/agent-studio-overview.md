# 에이전트 스튜디오 개요

에이전트 스튜디오(Agent Studio)는 하이레벨의 비주얼 환경으로, 지능형 AI 에이전트를 구축할 수 있습니다. 유연한 노드 기반 캔버스에서 에이전트가 언제 트리거되고, 정보를 어떻게 처리하며, 어떤 결정을 내리고, 어떤 작업을 수행할지 설계할 수 있습니다.

이 글은 에이전트 스튜디오를 처음 탐색하는 사용자를 위한 것으로, 고급 자동화를 구축하기 전에 구조, 목적, 핵심 구성 요소를 이해하고자 하는 분들을 대상으로 합니다.

# **에이전트 스튜디오란?**

에이전트 스튜디오는 이벤트에 응답하고, 사용자와 상호작용하며, 정보를 검색하고, 콘텐츠를 생성하며, 외부 시스템과 연결할 수 있는 AI 기반 에이전트를 만드는 비주얼 빌더입니다.

단순한 규칙 기반 자동화와 달리, 에이전트 스튜디오에서 구축된 에이전트는 입력을 해석하고, 맥락적 결정을 내리며, 대화를 동적으로 라우팅하고, 지식 검색이나 API 같은 도구를 사용할 수 있습니다.

각 에이전트는 캔버스에서 연결된 노드들로 구축되어, 어떻게 시작하고, 정보를 어떻게 처리하며, 어떻게 작업을 완료할지 정의할 수 있습니다.

---

## **에이전트 스튜디오의 주요 장점**

- 비주얼 AI 빌더: 드래그 앤 드롭 캔버스로 에이전트 동작을 설계합니다.

- 이벤트 기반 실행: 채팅 메시지, 폼 제출, 리드 태그 등 CRM 이벤트로 에이전트를 트리거합니다.

- 유연한 결정 로직: AI 의도 감지나 규칙 기반 조건으로 대화를 라우팅합니다.

- 통합 지식 & 도구: 지식 베이스(Knowledge Base)를 연결하거나 웹을 검색하거나 API를 호출할 수 있습니다.

- 멀티모달 출력: 하나의 플로우에서 텍스트, 이미지, 오디오, 비디오를 생성합니다.

- 구조화된 데이터 수집: 상호작용 중에 이메일, 전화번호, 사용자 선택을 캡처합니다.

## **에이전트 스튜디오 인터페이스 이해하기**

### 에이전트 스튜디오 대시보드

**AI Agents → Agent Studio**로 이동하면 계정의 모든 에이전트 목록을 볼 수 있습니다.

대시보드에서 할 수 있는 작업:

- 기존 에이전트 보기
- 상태 확인 (초안(Draft) 또는 발행됨(Published))
- 생성 또는 마지막 업데이트 날짜 확인
- 에이전트를 폴더로 정리
- 새 에이전트 생성
- 사용 가능한 템플릿 둘러보기

이곳은 AI 에이전트의 중앙 관리 영역입니다.

### 새 에이전트 만들기

대시보드에서 **Create Agent(에이전트 생성)**를 클릭합니다.

다음 중 선택할 수 있습니다:

- 처음부터 시작
- 템플릿 라이브러리에서 템플릿 사용

처음부터 시작하면 빈 캔버스가 제공됩니다. 템플릿을 사용하면 커스터마이즈할 수 있는 미리 구축된 구조가 제공됩니다.

### 에이전트 템플릿 라이브러리

템플릿 라이브러리는 일반적인 사용 사례를 위해 설계된 미리 만들어진 에이전트 청사진을 제공합니다.

빈 캔버스에서 구축하는 대신 다음을 할 수 있습니다:

- 카테고리별 템플릿 둘러보기
- 에이전트 구조 미리보기
- 계정에 템플릿 설치
- 설치 후 플로우의 모든 부분 수정

템플릿에는 일반적으로 미리 정의된 트리거, AI 노드, 라우팅 로직, 액션이 포함됩니다. 처음 사용하는 사용자에게는 템플릿이 에이전트의 구조와 연결 방식을 이해하는 가장 빠른 방법입니다.

![에이전트 템플릿 라이브러리](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155065467554/original/Hx0kM4KOVSCbclD-pkhVXECAkt6Y34_8ww.png?1771836848)

### 빌더 캔버스

에이전트를 열거나 **생성**하면 비주얼 빌더 캔버스로 들어갑니다.

여기서 할 수 있는 작업:

- 노드를 드래그하고 연결
- 실행 로직 정의
- 에이전트의 각 단계 구성

에이전트는 트리거로 시작해서 액션이나 완료 지점으로 끝나는 노드를 왼쪽에서 오른쪽으로 연결하여 구축됩니다. 노드 간의 연결은 실제 시나리오에서 에이전트가 어떻게 동작할지 정의합니다.

### 시작 트리거

모든 에이전트는 **시작 트리거(Start Trigger)**로 시작합니다.

시작 트리거는 에이전트가 언제 실행되어야 하는지 정의합니다. 일반적인 예시:

- 채팅 메시지(Chat Message)
- 폼 제출(Form Submission)
- 리드 태그 추가(Lead Tag Added)

트리거가 없으면 에이전트는 실행되지 않습니다. 트리거는 특정 조건이 충족될 때만 에이전트가 응답하도록 보장합니다.

![시작 트리거](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155065467704/original/jm98QJcg8ElCvZmjypz51nJYGPYV7OSJSQ.png?1771836978)

### 노드 카테고리

에이전트 스튜디오에서 캔버스의 모든 것은 **노드**를 사용하여 구축됩니다. 노드는 에이전트 시작, 응답 생성, 정보 수집, 다른 시스템 연결 등 하나의 특정 기능을 수행하는 구성 요소입니다. 노드를 연결하여 에이전트가 처음부터 끝까지 어떻게 동작할지 정의합니다.

![노드 카테고리](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155065466013/original/j4oMLxMOzUDEiNY0V7eX1F1IkCKNcPyfeQ.png?1771835869)

#### AI Agent: AI Agent 노드는 플로우의 "두뇌"입니다. 입력을 읽고, 맥락을 이해하며, 제공한 지시사항을 바탕으로 지능적인 응답을 생성합니다. 예를 들어, 사용자가 가격에 대해 질문하면 이 노드가 질문을 해석하고 관련된 답변을 생성할 수 있습니다.

#### Sequential: Sequential 노드는 여러 단계를 그룹화하여 고정된 순서로 실행할 수 있게 합니다. 먼저 이메일을 수집한 후 확인 메시지를 보내는 것처럼 특정 작업이 차례로 일어나야 할 때 유용합니다.

#### End Node: End Node는 에이전트 실행이 중단되는 지점을 표시합니다. 플로우가 이 노드에 도달하면 상호작용이 완료됩니다.

#### Router: Router 노드는 조건이나 AI가 감지한 의도를 바탕으로 에이전트가 다른 경로 중에서 선택하도록 돕습니다. 결정 지점으로 생각하면 됩니다. 사용자가 지원에 대해 묻으면 한 방향으로, 영업에 대해 묻으면 다른 방향으로 갑니다.

#### Search Knowledge Base: 이 노드는 에이전트가 연결된 지식 베이스에 저장된 정보를 찾을 수 있게 합니다. 예를 들어, 회사 문서에서 답변을 직접 가져올 수 있습니다.

#### Search Web: Search Web 노드는 필요할 때 에이전트가 온라인 소스에서 정보를 수집할 수 있게 합니다. 공개적으로 이용 가능하거나 최신 데이터가 필요한 질문에 답변할 때 유용합니다.

#### MCP Server: MCP Server 노드는 에이전트를 지원되는 외부 도구 서버에 연결합니다. 이를 통해 에이전트가 외부 시스템을 사용하여 전문적인 작업을 수행할 수 있는 고급 연동이 가능합니다.

---

![노드 카테고리 2](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155065466189/original/oilx-pQ2JI0o4w5Q8deexvh1uFZ02VS7Nw.png?1771835998)

#### API Call: API Call 노드는 에이전트가 다른 애플리케이션과 정보를 주고받을 수 있게 합니다. 예를 들어, 리드 데이터를 다른 시스템에 보내거나 고객 세부 정보를 검색할 수 있습니다.

#### Audio Generation: 이 노드는 에이전트가 플로우 내 지시사항을 바탕으로 음성 응답을 생성할 수 있게 합니다.

#### Image Generation: Image Generation 노드는 설명적 프롬프트를 바탕으로 이미지를 생성하여, 동적 비주얼 콘텐츠에 유용합니다.

#### Text Generation: Text Generation 노드는 메시지, 요약, 구조화된 응답 등 작성된 콘텐츠를 생성합니다.

#### Email Address: 이 노드는 상호작용 중에 사용자의 이메일 주소를 구조화된 형식으로 수집합니다.

#### Phone Number: Phone Number 노드는 정확한 저장을 위해 사용자의 전화번호를 표준화된 형식으로 수집합니다.

#### Single Choice: Single Choice 노드는 미리 정의된 옵션을 사용자가 선택할 수 있게 제시하여, 구조화된 의사결정을 안내합니다.

#### Text Input: Text Input 노드는 사용자로부터 자유로운 응답을 수집하여, 에이전트가 자세한 정보를 수집할 수 있게 합니다.

### 변수와 글로벌 프롬프트

빌더 상단에서 전체 플로우에서 에이전트의 동작을 제어하는 추가 구성 도구들을 찾을 수 있습니다.

#### 변수

변수는 정보를 저장하는 재사용 가능한 자리표시자 역할을 합니다. 여러 곳에서 같은 값을 반복하는 대신, 한 번 정의하고 에이전트 전체에서 참조할 수 있습니다. 예를 들어, 비즈니스 이름을 저장하고 응답 전체에서 일관되게 사용할 수 있습니다.

#### **글로벌 프롬프트**

글로벌 프롬프트는 에이전트의 동작을 안내하는 전체적인 지시사항을 정의합니다. 모든 상호작용에서 에이전트가 따라야 할 톤, 성격, 규칙을 설정합니다.

### 테스트 및 발행

에이전트를 활성화하기 전에 다음을 할 수 있습니다:

- **Test(테스트)** 기능으로 실행 시뮬레이션
- 변경사항을 초안으로 저장
- 준비되면 에이전트 발행

에이전트는 실제 트리거에 응답하려면 반드시 발행되어야 합니다.

---

## **에이전트 스튜디오 시작하기**

첫 번째 에이전트를 만들려면:

- **AI Agents → Agent Studio**로 이동
- **Create Agent(에이전트 생성)** 클릭
- 처음부터 시작하거나 템플릿 설치 선택
- **시작 트리거(Start Trigger)** 추가
- **AI Agent 노드**를 캔버스에 드래그하고 연결
- **Test(테스트)**로 동작 시뮬레이션
- 에이전트 저장 및 **발행(Publish)**

## **자주 묻는 질문**

**Q: 에이전트가 자동으로 실행되나요?**
에이전트는 설정된 트리거 조건이 충족될 때만 실행됩니다.

**Q: 에이전트 스튜디오가 워크플로우와 어떻게 다른가요?**
워크플로우는 미리 정의된 자동화 규칙을 따릅니다. 에이전트는 입력을 해석하고 AI를 사용하여 동적 결정을 내릴 수 있습니다.

**Q: 에이전트가 작동하려면 발행해야 하나요?**
네. 에이전트가 실제 이벤트에 응답하려면 반드시 발행되어야 합니다.

**Q: 에이전트가 외부 시스템과 연동할 수 있나요?**
네. API Call과 MCP Server 노드를 통해 외부 연동이 가능합니다.

**Q: 에이전트가 텍스트 외에 다른 콘텐츠도 생성할 수 있나요?**
네. 에이전트 스튜디오는 이미지, 오디오, 비디오 생성을 지원합니다.
