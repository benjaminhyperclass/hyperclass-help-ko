# 에이전트 스튜디오 트리거로 실시간 시작 설정하는 방법

에이전트 스튜디오 트리거는 하이퍼클래스 AI 에이전트를 위한 실시간, 이벤트 기반 자동화를 제공합니다. 누군가 폼을 제출하거나, 태그가 붙거나, 채팅을 보내면 하이퍼클래스가 이벤트를 기록하고 백그라운드에서 에이전트를 시작하여 페이지와 API는 빠르게 유지하면서 에이전트는 즉시 응답할 수 있도록 합니다. 이 아키텍처는 대용량 처리와 안전한 테스트를 위해 설계되어, 실제 트래픽을 방해하지 않으면서 트리거를 점진적으로 배포할 수 있습니다.

이 가이드에서는 트리거가 무엇인지, 왜 중요한지, 그리고 모범 사례 팁과 함께 빠르게 설정하는 방법을 설명합니다.

# **에이전트 스튜디오 트리거란?**

에이전트 스튜디오 트리거는 에이전트의 시작(Start) 노드에 연결된 이벤트 리스너로, 지원되는 플랫폼 이벤트가 발생하면 자동으로 에이전트를 실행합니다. 폼 제출이나 채팅 전송 같은 실제 사용자 행동에 반응하여, 에이전트가 외부 API 호출이나 수동 단계 없이 즉시 연락처와 소통할 수 있게 합니다.

에이전트 스튜디오 트리거는 현재 세 가지 진입 이벤트를 제공합니다: **폼 제출**, **리드 태그** (추가 또는 제거), **채팅 메시지**. 이러한 이벤트 중 하나가 발생하면 에이전트가 **시작 트리거**부터 시작하여 설계한 흐름을 따라 진행됩니다.

---

## **에이전트 스튜디오 트리거의 주요 장점**

구체적인 장점을 이해하면 자동화 전략에서 트리거가 어디에 적합한지, 수동이나 시간 지연 프로세스를 어떻게 대체할지 결정하는 데 도움이 됩니다.

- **실시간 참여**: 리드가 행동하는 순간—폼 제출, 비즈니스 메시지, 태그 부여—즉시 응답합니다.

- **완전 자동 확장**: 폴링 스크립트와 커스텀 웹훅을 제거하고 제품 내 트리거를 설정하고 발행하기만 하면 됩니다.

- **일관된 진입점**: 에이전트 시작 방식을 표준화하여 테스트와 문제 해결이 더 쉬워집니다.

- **기존 시스템 영향 없음**: 트리거는 선택사항이며 추가 기능이므로 기존 API 기반 에이전트는 계속 작동합니다.

- **성장의 기반**: 이벤트 기반 설계로 시간이 지나면서 추가 이벤트 유형의 토대를 마련합니다.

---

## **에이전트 스튜디오 트리거 설정 방법**

다음 단계에 따라 처음부터 끝까지 신뢰할 수 있고, 테스트 가능하며, 프로덕션 준비가 완료된 트리거 설정을 만들어보세요.

**사전 요구사항 및 액세스:**

트리거 설정이 원활하게 진행되고 배포 프로세스와 일치하도록 액세스 및 환경을 확인하세요.

올바른 하위 계정/로케이션 내에서 **AI Agents → Agent Studio**에 액세스하세요.

캔버스 탐색, 테스트, 발행 생명주기는 하이퍼클래스에서 AI 에이전트 스튜디오 사용 방법을 참조하세요.

**에이전트 스튜디오 열기**

왼쪽 사이드바에서 **AI Agents**를 클릭한 다음 상단의 **Agent Studio** 탭을 선택합니다. 현재 로케이션의 AI 에이전트를 생성하고 관리하는 작업공간이 열립니다.

## 트리거 유형

**Select trigger** 패널에서 **Form submitted**, **Lead tag**, 또는 **Chat message**를 선택하여 에이전트를 시작할 조건을 정의합니다. 선택에 따라 다음에 나타나는 설정 옵션이 결정됩니다.

---

#### 폼 제출 트리거

연락처가 하이퍼클래스에서 추적하는 폼을 완료한 직후 에이전트가 즉시 참여하기를 원할 때 사용합니다. 즉시 육성, 자격 확인 또는 라우팅에 이상적입니다:

- 선택된 폼이 제출될 때 에이전트가 시작됩니다.
- 제출된 폼 필드를 변수로 노출하여 에이전트가 나중 단계에서 참조할 수 있습니다.

---

#### 리드 태그 트리거 (추가 / 제거)

연락처에 태그가 적용되거나 제거될 때 타겟 자동화를 실행합니다. VIP 온보딩, 재참여 또는 이탈 방지 등의 일반적인 용도가 있습니다.

- 선택된 태그가 연락처에서 **추가**되거나 **제거**될 때 에이전트가 시작됩니다.
- 연락처 컨텍스트(기존 태그와 커스텀 필드 포함)를 흐름에 제공합니다.

---

#### 채팅 메시지 트리거

지원되는 채팅 위젯/제공업체를 통해 새로운 인바운드 메시지가 도착하는 즉시 에이전트를 시작합니다. 즉시 답변, 스마트 라우팅 또는 FAQ 처리에 이상적입니다.

- 추가 설정이 필요 없으며, 트리거를 추가하고 발행하기만 하면 됩니다.
- 메시지 텍스트, 발신자 세부정보, 대화 컨텍스트를 흐름 로직에 제공합니다.

**폼 선택**

폼 제출 트리거의 경우 드롭다운을 사용하여 하나 이상의 폼을 선택하거나, 폼이 아직 생성되지 않았다면 **Add new**를 클릭하세요. 선택된 폼이 제출될 때마다 에이전트가 시작됩니다.

![폼 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063614943/original/0xVfvAozJ9TcExA5vbc-dYomPdcQElYwvg.png?1769533754)

**태그 액션 선택**

**Lead tag**의 경우 **Tag action**을 **Tag added** 또는 **Tag removed**로 설정합니다. 이는 태그가 연락처에 적용될 때 또는 제거될 때 에이전트가 시작되는지 결정합니다.

![태그 액션 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063615612/original/3O2X7hl-d6AagMxgDdzfVVto7vj-FP2Tng.png?1769534170)

**리드 태그 선택**

목록에서 특정 태그를 선택하거나 **Add new**로 생성하세요. 에이전트는 여기서 선택한 태그의 변경 사항만 수신합니다.

테스트를 위해 태그를 대량으로 추가할 때는 많은 에이전트 실행을 한 번에 트리거하지 않도록 작은 테스트 세그먼트를 먼저 사용하는 것을 고려하세요.

![리드 태그 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063615457/original/AAb19E7_R1j56U-AHrk3DiIturL-cbVIrg.png?1769534099)

**채팅 트리거 활성화**

**Chat message**의 경우 추가 설정이 필요하지 않습니다. 연결된 채팅 채널을 통해 새로운 인바운드 메시지가 도착할 때마다 에이전트가 자동으로 시작됩니다.

Ask AI에서 에이전트로 대화를 라우팅하는 경우 채팅이 **Chat message**를 통해 에이전트를 시작할 수 있습니다. 매핑 세부사항은 Ask AI + 에이전트 스튜디오 연동을 참조하세요.

![채팅 트리거 활성화](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155063615109/original/yEovCmZ6InRZrsAP_cuXLaaSw1G40sz-8Q.png?1769533977)

**검증 및 라이브 배포**

- **Test**: 선택된 트리거의 안전한 시뮬레이션을 실행하여 에이전트가 받는 변수와 페이로드를 정확히 확인합니다. 실제 연락처에는 영향을 주지 않습니다. 이를 통해 프롬프트/노드를 빠르게 반복하고 발행 전에 흐름을 검증할 수 있습니다.

- **Save**: 현재 변경사항을 스테이징 버전으로 저장하여 작업을 잃지 않고 나중에 돌아올 수 있습니다. 아직 라이브되지는 않습니다. 각 설정 단계(예: 폼이나 태그 선택) 후에 저장하여 깨끗한 작업 버전을 유지하는 것이 이상적입니다.

- **Publish**: 현재 에이전트와 시작 트리거를 이 하위 계정에서 라이브로 푸시하여 실제 이벤트가 즉시 에이전트를 시작하도록 합니다. 성공적인 테스트 후에 발행하며, 시작 트리거가 첫 번째 노드에 연결되어 있는지 확인하세요. 에이전트가 다른 기능(예: Ask AI)에 매핑된 경우 일반적으로 선택 가능하려면 프로덕션 상태여야 합니다.

## **자주 묻는 질문**

**Q: 트리거가 기존 API 호출 에이전트의 실행 방식을 변경하나요?**

아니요. 트리거는 추가 기능입니다. 기존 API 흐름은 계속 작동하며, 트리거는 시작 트리거를 추가한 에이전트에서만 실행됩니다.

**Q: 트리거는 어디서 켜나요?**

**AI Agents → Agent Studio**에서 각 에이전트의 캔버스 내부에서 켭니다. **Start trigger**를 추가하고 설정한 다음 **Test**하고 **Publish**하면 됩니다.

**Q: 라이브 배포 전에 테스트할 수 있나요?**

네. 캔버스의 **Test** 옵션을 사용하여 이벤트를 시뮬레이션하고 발행 전에 변수와 응답을 검증할 수 있습니다.

**Q: 어떤 채팅 소스가 지원되나요?**

트리거는 하위 계정에 연결된 하이퍼클래스의 기존 채팅 위젯/제공업체와 작동합니다. 시작하기 위한 추가 설정은 필요하지 않습니다.

**Q: 트리거가 발생했을 때 에이전트에 어떤 데이터를 사용할 수 있나요?**

폼 제출에는 매핑된 폼 필드가 포함되고, 리드 태그 이벤트에는 연락처 컨텍스트가 포함되며, 채팅 메시지 이벤트에는 메시지 텍스트와 대화 메타데이터가 포함됩니다.

**Q: 드롭다운에서 내 폼이나 태그가 보이지 않으면 어떻게 하나요?**

올바른 하위 계정/로케이션에 있는지 확인하고 해당 자산이 그곳에 존재하는지 확인하세요. 새 폼/태그를 만든 후 빌더를 새로고침하세요.
