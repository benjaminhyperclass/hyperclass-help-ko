# 음성 AI 에이전트 테스트하기

에이전트를 배포하기 전에 기능을 테스트하는 것이 중요해요. AI 에이전트가 잠재 고객이나 기존 고객과의 대화에서 발생할 수 있는 특정 문의를 어떻게 처리하는지 알아볼 수 있고, 이 정보를 활용해 음성 AI 에이전트를 세밀하게 조정하여 궁극적으로 더 높은 전환율을 얻을 수 있어요!

Hyperclass 음성 AI 에이전트를 **두 가지 테스트 방법**—**전화 통화** 또는 웹 통화—으로 빠르게 검증하여 프롬프트, 플로우, 설정을 자신 있게 반복 개선할 수 있어요. 실제 전화 경로를 원할 때는 전화 옵션을 사용하고, 번호 프로비저닝을 건너뛰고 즉시 테스트하려면 브라우저 기반 웹 통화를 실행하세요.

# **음성 AI 에이전트의 테스트 통화란?**

테스트 통화를 통해 음성 AI 에이전트와의 실제 대화를 라이브 배포 전에 시뮬레이션할 수 있어요. 에이전트로부터 내 전화번호로 **전화 통화**를 받거나 브라우저에서 직접 **웹 통화**를 시작할 수 있어요. **두 방법 모두** 동작을 검증하고, 프롬프트를 조정하고, 액션을 확인하는 데 도움이 돼요. **웹 통화** 테스트 중에는 **통화 전환이 지원되지 않으며**, 세션은 **로그인한 연락처**를 발신자 신원으로 사용해요.

---

## 테스트 통화의 주요 장점

다음 장점들을 활용해 설정 속도를 높이고, 마찰을 줄이고, 런치 전 품질을 개선하세요.

- **더 빠른 테스트:** **웹 통화**를 즉시 시작—전화번호 구매나 다이얼링이 필요 없어요.

- **현실적인 검증:** **전화 통화** 방법을 사용해 엔드투엔드 전화 통신 동작을 테스트하세요.

- **빠른 피드백 루프:** 응답을 즉시 듣고 프롬프트, 지식, 설정을 개선하세요.

- **높은 기능 패리티:** 테스트 중 거의 모든 에이전트 액션을 실행할 수 있어요 (**웹 통화는 통화 전환 제외**).

- **명확한 발신자 신원:** 웹 통화는 일관된 컨텍스트를 위해 **로그인한 사용자의 연락처**로 세션을 매핑해요.

- **테스트 데이터 위생:** 테스트 활동을 분석 대시보드에서 제외하고 프로덕션 인사이트에 집중하세요.

---

## **테스트 통화 설정 방법 (웹 통화 & 전화 통화)**

브라우저와 전화 통신 설정이 준비되었는지 확인하면서 신뢰할 수 있는 테스트를 시작하려면 다음 단계를 따르세요.

### **유형 1 : 웹 통화 시작하기**

- 좌측 사이드바에서 AI 에이전트를 열고 상단 네비게이션 리본에서 Voice AI를 클릭하세요

- 보조 네비게이션 리본에서 **에이전트 목록**을 선택한 다음 테스트하려는 에이전트를 편집하세요

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053575402/original/U6-ugDbqmwvl2bIkDDUcV5-BkKsrga68Lg.png?1757598166)

- 우측의 에이전트 테스트 패널에서 웹 통화를 클릭하고 시나리오 **(인바운드 또는 아웃바운드)**를 선택하세요. 그런 다음 웹 통화 시작을 클릭하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061006529/original/9AuyNOWBRwzJea60qNd5ZUfmqCom15CH_g.png?1766147986)

- 프롬프트가 나타나면 브라우저에서 **마이크 접근을 허용**하세요.

- 에이전트와 대화하고 응답과 액션을 관찰하세요. 웹 통화 테스트 중에는 통화 화면에 **실시간 전사** 패널이 표시돼요. 당신과 에이전트가 말하는 대로 실시간으로 업데이트돼요.
(참고: 웹 통화 테스트에서는 **통화 전환이 지원되지 않아요**.)

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066262303/original/jPXJaaZ7-adGmuZXmZ858Xj8jpbl-tiGPQ.gif?1772706800)

- 필요에 따라 프롬프트, 지식 베이스 또는 설정을 조정한 다음 테스트를 반복하세요.

### 참고사항:

- 더 빠른 테스트 – **다이얼**하거나 전화번호를 **구매**할 필요가 없어요. 단순히 클릭해서 바로 테스트를 시작하세요.

- 기능: 브라우저 기반 통화를 열어서 에이전트와 즉시 대화할 수 있어요.

- 발신자 신원: **로그인한 사용자의 연락처**를 자동으로 사용해요.

- 제한사항: 웹 통화 테스트에서는 **통화 전환이 지원되지 않아요**.

- 디바이스 팁: 마이크 접근을 허용하고, 입력/출력 디바이스를 확인하고, 최신 브라우저를 사용하세요.

### **유형 2: 전화 통화 시작하기**

- 좌측 사이드바에서 AI 에이전트를 열고 상단 네비게이션 리본에서 Voice AI를 클릭하세요

- 보조 네비게이션 리본에서 **에이전트 목록**을 선택한 다음 테스트하려는 에이전트를 편집하세요

- 우측의 **에이전트 테스트** 섹션에서 전화 통화를 클릭하고 시나리오 **(인바운드 또는 아웃바운드)**를 선택하세요. 계속해서 **발신자 ID 선택** (에이전트가 당신에게 전화할 때 사용할 번호)으로 진행하세요.

- **당신의 전화번호** (테스트 통화를 받을 번호)를 입력하세요.

- **전화 걸기**를 클릭해서 테스트를 시작하고 디바이스에서 **수신 전화에 응답**하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061007218/original/Y2bn_8j72TB39HBJIorQlQRKkiuSnvPj1g.png?1766148493)

- **통화 기록** 또는 **대시보드 로그**를 검토해서 전사, 녹음, 요약을 분석하세요.

### 참고사항:

- **발신자 ID 선택:** 에이전트가 당신에게 전화할 때 사용할 전화번호를 선택하세요.

- **당신의 전화번호 입력:** 테스트 통화를 받을 목적지 번호를 제공하세요.

- **통화 수신:** 발신자처럼 에이전트와 답변하고 상호작용하세요.

- **요금 안내:** 전화 기반 테스트 통화는 일반 전화 통신 설정을 사용하므로 **표준 요금이 적용될 수 있어요**.

## **에이전트 생성 중 테스트 통화**

음성 AI 에이전트를 생성하는 동안에도 테스트할 수 있어요. 에이전트 생성 플로우에서 전화 및 예약 가능 시간 단계의 에이전트 테스트 패널을 사용해서 위에서 설명한 것과 동일한 웹 통화 및 전화 통화 테스트를 실행하세요. 이러한 테스트 통화는 통화 기록, 전사, 녹음, 요약을 포함해서 다른 테스트와 동일하게 동작해요.

## 테스트 통화 기록 검토하기

음성 AI 에이전트를 테스트할 때, 이전 테스트 통화를 돌아보며 대화와 **AI 에이전트**가 **발신자**와 상호작용한 방식에 대해 더 자세히 알아볼 수 있어요. 각 필드가 무엇을 의미하는지 이해하기 위해 이 빠른 참조를 사용해서 문제를 더 빠르게 진단하고 에이전트가 의도한 결과를 달성했는지 확인하세요.

이전 테스트 통화를 열려면 통화 기록이라는 드롭다운을 클릭한 다음 검토하려는 테스트 통화를 선택하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053588377/original/M_OflcxBgocoaDSET2PnZbI6o9O5aClpiA.png?1757603758)

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155053588564/original/TkIzDY7P4lr2F6vh_kgtN-QHb22Gb92Slw.png?1757603834)

| 필드 | 설명 |
|------|------|
| 시나리오 | 통화가 **인바운드** 또는 **아웃바운드**로 실행되었는지 표시하여 다양한 통화 방향에서의 동작을 비교할 수 있어요. |
| 통화 시간 | 총 연결 시간. 매우 짧은 시간은 조기 종료나 권한 문제를 나타낼 수 있어요. |
| 통화 상태 (예: 완료, 부재중) | 완료, 부재중, 실패 또는 취소와 같은 최종 결과. |
| 대화 전사 | 검토 및 프롬프트 조정을 위한 전체 대화 텍스트. |
| 통화 녹음 재생 | 음성 품질, 속도, 톤을 평가하기 위한 오디오 재생 (사용 가능한 경우). |
| 발신자 신원 | 웹 통화의 경우 로그인한 사용자의 연락처로 세션이 매핑되고, 전화 통화의 경우 다이얼한 목적지 번호가 표시돼요. |
| 통화 요약 | 빠른 스캔을 위한 대화 및 결과의 자동 생성된 요약. |

## 자주 묻는 질문

**Q: 테스트를 위해 전화번호를 구매해야 하나요?**
**웹 통화:** 번호가 필요하지 않아요. **전화 통화:** 기존 전화 통신 설정을 사용하며 **표준 요금이 적용될 수 있어요**.

**Q: 테스트 통화가 분석에 포함되나요?**
아니요. **테스트 통화는** 대시보드 분석 및 성능 지표에서 **제외**돼요.

**Q: 테스트에서 전사와 녹음을 볼 수 있나요?**
네. **통화 기록** 또는 **대시보드 & 로그**를 열어서 테스트 통화의 **전사, 녹음, 통화 요약**에 액세스하세요.

**Q: 통화 전환을 테스트할 수 있나요?**
**웹 통화:** 지원되지 않아요. **전화 통화:** 에이전트 구성 및 전화 통신 설정에 따라 동작해요.

**Q: 웹 통화에서 누가 발신자로 표시되나요?**
**로그인한 사용자의 연락처**가 자동으로 테스트의 발신자 신원으로 사용돼요.

**Q: 어떤 방법을 선택해야 하나요?**
번호 설정 없이 빠른 반복을 위해서는 **웹 통화**를 사용하세요. 전화 통신 관련 경험을 엔드투엔드로 검증하려면 **전화 통화**를 사용하세요.

**Q: 테스트 통화가 분석에 영향을 미치나요?**
테스트 통화는 품질 평가를 위한 것이며 분석에서 **제외**돼요.
