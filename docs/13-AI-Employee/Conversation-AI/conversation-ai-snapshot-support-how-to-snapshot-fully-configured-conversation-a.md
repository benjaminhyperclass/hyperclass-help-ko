# 대화 AI: 스냅샷 지원 - 완전 설정된 대화 AI 봇을 스냅샷하는 방법

### 대화 AI 봇 스냅샷 기능이란?

대화 AI 봇 스냅샷 기능을 사용하면 프롬프트, 액션, 설정을 포함한 전체 봇을 복사하여 재사용 가능한 스냅샷으로 만들 수 있습니다. 이를 통해 하위 계정마다 봇을 수동으로 재생성할 필요가 없어지고, 고객 상호작용, 학습 데이터, 봇 로직의 일관성을 보장할 수 있습니다.

### 대화 AI 봇 스냅샷의 주요 장점

봇 스냅샷을 생성하면 계정 간 설정을 간소화하고 인적 오류를 줄일 수 있습니다.

- 설정, 프롬프트, 액션을 포함한 전체 봇 설정 캡처
- 모든 하위 계정에서 반복적인 봇 설정 작업 제거
- 모든 클라이언트 계정에서 톤과 기능의 일관성 유지
- 가져온 봇을 자동으로 기본 봇으로 설정 (기존에 없는 경우)
- 모든 봇 학습 데이터(웹 크롤러 URL 및 FAQ) 포함
- 대상에 기존 봇이 있을 때 처리 방법 선택 (덮어쓰기 또는 건너뛰기)

### 봇을 포함한 스냅샷 생성 방법

하위 계정 간에 봇을 복제하려면 다음 단계에 따라 대화 AI 설정을 포함하는 스냅샷을 생성하세요.

- 좌측 네비게이션 메뉴에서 **Account Snapshots(계정 스냅샷)** 탭으로 이동합니다.
- **Create New Snapshot(새 스냅샷 생성)**을 클릭합니다.
- 스냅샷할 봇이 있는 **소스 하위 계정**을 선택합니다.
- 스냅샷의 **이름**을 설명적으로 입력합니다.
- **Conversation AI(대화 AI)** 섹션으로 스크롤합니다.
- 나열된 봇을 검토합니다. 이 섹션에는 다음이 포함됩니다:
  - 봇 이름
  - 봇 학습 웹 크롤러 URL
  - 커스텀 봇 응답(FAQ)
- 검토를 완료한 후 **Create Snapshot(스냅샷 생성)**을 클릭합니다.

### 대화 AI 스냅샷 옵션 및 동작

무엇이 복사되고 가져오기 규칙이 어떻게 작동하는지 알아두면 더 원활한 봇 마이그레이션 과정을 진행할 수 있습니다.

**기본 봇 자동 지정**

대상 하위 계정에 기본 봇이 없는 경우, 스냅샷에서 가져온 봇이 자동으로 기본 봇으로 설정됩니다.

**충돌 처리**

대상 하위 계정에 같은 이름의 봇이 이미 있는 경우, 두 가지 옵션을 선택할 수 있습니다:

- **Overwrite(덮어쓰기)** – 기존 봇을 스냅샷의 봇으로 교체합니다.
- **Skip(건너뛰기)** – 기존 봇을 유지하고 스냅샷의 봇을 무시합니다.

**스냅샷에 포함되는 항목**

- 완전한 봇 구조: 모든 액션 및 프롬프트-응답 매핑
- 봇 설정 및 구성
- 봇 학습 웹 크롤러 URL
- Bot Training(봇 학습) 탭을 통해 생성된 커스텀 FAQ 데이터

### 자주 묻는 질문

**Q: 대상 하위 계정에 같은 이름의 봇이 이미 있으면 어떻게 되나요?**

가져오기 중에 기존 봇을 덮어쓸지 아니면 새 봇을 건너뛸지 선택하라는 메시지가 나타납니다.

**Q: 프롬프트와 액션 로직이 보존되나요?**

네, 스냅샷은 모든 플로우, 액션, 응답을 설정된 그대로 정확히 보존합니다.

**Q: 스냅샷을 가져온 후 기본 봇을 변경할 수 있나요?**

네, 스냅샷 봇이 기본 봇으로 지정되지 않은 경우 대화 AI 설정에서 수동으로 기본 봇으로 지정할 수 있습니다.

**Q: FAQ와 웹 크롤러 링크 같은 학습 데이터도 포함되나요?**

네, URL과 FAQ를 포함한 모든 학습 데이터가 스냅샷에 포함됩니다.

**Q: 스냅샷을 나중에 편집할 수 있나요?**

아니요, 스냅샷은 수정할 수 없습니다. 원본 봇을 변경한 경우 새로운 스냅샷을 생성해야 합니다.
