# 대화 AI(Conversation AI) 가이드 폼 기반 설정

이 글에서는 Hyperclass에서 가이드 폼 방식을 사용하여 대화 AI V3를 설정하는 단계별 가이드를 제공합니다. 가이드 폼 방식을 사용하면 프롬프트를 직접 작성할 필요 없이 간단한 양식만 작성하고, 목표를 정의하면 시스템이 논리를 처리해줍니다. 효과적이고 브랜드에 맞는 AI 대화를 빠르게 구축하기에 완벽한 방법입니다.

## **가이드 폼 봇이란?**

가이드 폼 봇은 대화 AI 내에서 제공되는 사용자 친화적인 설정 옵션으로, 수동으로 프롬프트를 작성할 필요를 없애줍니다. 대신 구조화된 폼을 사용하여 봇을 만드는 과정을 안내해줍니다. 연락처 정보 수집이나 FAQ 응답 같은 주요 목표를 정의하면, 시스템이 논리 기반의 목표 중심 대화 흐름을 자동으로 구축해줍니다.

## **가이드 폼 봇의 주요 장점**

가이드 폼 봇은 깊은 기술적 전문 지식 없이도 강력한 봇 기능을 원하는 사용자에게 이상적입니다. 단순함과 유연성의 균형을 맞춰 더 짧은 시간에 고품질 대화를 가능하게 합니다.

- 프롬프트 작성 불필요
- 빠른 설정—3분 이내에 봇 생성
- 입력 내용에 맞춘 신뢰할 수 있는 논리 기반 답변
- 리드 수집 및 연락처 검증을 위한 쉬운 방법
- 여러 캠페인에 재사용 가능한 봇 템플릿
- 대화 AI 워크플로우 및 태그와 완벽 연동
- 반복 줄이기 및 참여도 증가
- 사용자가 이미 공유한 내용에 맞춰 후속 조치 조정
- 주제에서 벗어난 답변을 부드럽게 폼으로 다시 유도
- 폼을 계속 진행하면서도 공감적인 프롬프트 제공
- 이탈률 감소 및 완료율 개선 도움

## 가이드 폼 봇 설정 방법

가이드 폼 봇을 설정하고, 동작을 정의하고, CRM 내 주요 액션과 연결하는 단계별 가이드를 따라해보세요.

### **1단계: 봇 생성 페이지 접근**

시작하려면:

- 좌측 메뉴에서 **Conversation AI(대화 AI) → Bots(봇)**으로 이동하세요.
- **Create Bot(봇 생성)**을 클릭하세요.
- 가이드 폼 인터페이스가 기본으로 열립니다.

![가이드 폼 봇 생성 인터페이스](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047436709/original/VVaKMBmEkSeIyQbe47TKCGVO9gU-Wk7g5w.png?1748526984)

### **2단계: 브랜드 정보 입력 (선택사항)**

선택사항이지만 브랜드 정보를 추가하면 봇의 톤과 성격을 커스터마이징하는 데 도움이 됩니다.

- 비즈니스 이름
- 비즈니스 유형
- 목소리 톤 (예: 전문적, 친근한, 캐주얼)

![브랜드 정보 입력 섹션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047436798/original/k2Lq9-uAiddzXh49qFnzurEYGUE7GeNGzw.png?1748527051)

이렇게 하면 AI가 브랜드 아이덴티티와 일관성 있는 톤으로 더 자연스럽게 응답할 수 있습니다.

### 3단계: 기본 봇 설정 구성

이러한 핵심 설정들이 대화에서 봇의 동작을 결정합니다:

- **초기 메시지**: 봇이 사용자에게 처음 할 말
- **초기 메시지가 설정된 경우**: 봇이 먼저 초기 메시지를 보내고, 다음 사용자 메시지에서 가이드 폼 플로우(목표/질문)가 시작됩니다. 이전에는 첫 번째 플로우 단계가 초기 메시지 직후에 바로 붙었습니다.
- **초기 메시지가 설정되지 않은 경우**: 봇이 첫 번째 메시지에서 바로 가이드 폼 플로우를 시작합니다 (변경 없음).
- **언어**: 봇이 사용할 언어 선택
- **봇 이름**: 봇에 친근한 이름 부여
- **동작 유형**:
  - **자동 파일럿(Auto-Pilot)**: 완전 자동화
  - **제안 모드(Suggestive)**: 사용자에게 답변 제안 전송

### 4단계: 목표 및 질문 정의

봇이 완수했으면 하는 모든 목표를 추가하세요.

- **Add Question / Objective(질문/목표 추가)** 버튼을 사용하여 각각 생성하세요.
- **Skip if Already Filled(이미 입력된 경우 건너뛰기)**를 활성화하여 기존 연락처 데이터에 대한 질문을 방지하세요.
- 질문을 드래그 앤 드롭으로 순서를 변경하세요.

일반적인 사용 사례:
- 이름, 이메일, 전화번호 요청
- 예/아니오 질문으로 리드 검증
- 예약 링크 제공

### **추가 지시사항 (폼 기반)**

추가 지시사항을 사용하여 봇에게 추가 컨텍스트와 가드레일을 제공하면 프로세스를 일관되게 따르고 예외 상황을 자신 있게 처리할 수 있습니다. **문자 제한**: 최대 2,000자 (이전 1,200자에서 증가).

**모범 사례**
- 봇의 역할과 경계 정의
- 까다로운 시나리오에 대한 명확한 해야 할 일/하지 말아야 할 일 규칙 사용
- 글머리 기호를 간결하게 유지하고 중요한 단계를 먼저 배치

**찾는 방법**
- Conversation AI(대화 AI) → Bots(봇) → Create Bot(봇 생성) (가이드 폼)으로 이동
- Objectives & Questions(목표 및 질문)를 열고 Additional Instructions(추가 지시사항) 찾기
- 가이드를 입력하고 Save(저장)

### **5단계: 각 목표에 대한 액션 설정**

질문을 정의했다면, 목표가 완료될 때 봇이 무엇을 해야 할지 결정하세요.

옵션에는 다음이 포함됩니다:
- **Workflow(워크플로우) 트리거** (예: 이메일 시퀀스에 등록)
- **Custom Field(커스텀 필드) 업데이트** (예: "선호 연락 시간" 저장)
- **Tag(태그) 할당** (예: "데모에 관심 있음")

이러한 액션은 사용자 응답에 따라 후속 조치를 자동화하고 개인화하는 데 도움이 됩니다.

### **6단계: 최종 검토 및 추가 설정**

봇을 저장하기 전에 모든 섹션을 검토하여 모든 것이 올바른지 확인하세요. 그다음 고급 설정을 구성하세요:

- **Bot Availability(봇 가용성)**: 시간/요일 가용성 설정
- **Channels(채널)**: 봇이 활성화될 위치 선택 (SMS, 웹채팅, 인스타그램 등)
- **Fallback Message(대체 메시지)**: 봇이 질문을 처리할 수 없을 때의 커스텀 메시지

Save(저장)를 클릭하여 봇을 실행하세요!

## **자주 묻는 질문**

**Q: 가이드 폼과 프롬프트 기반 봇 사이를 전환할 수 있나요?**

네! 봇 생성 화면에서 "Create Bot(봇 생성)" 옆의 드롭다운 화살표를 클릭하여 설정 방법을 선택하세요.

**Q: 봇을 생성한 후 편집할 수 있나요?**

물론입니다. Bot Manager(봇 관리자)에서 봇을 열고 "Edit(편집)"을 클릭하세요.

**Q: 사용자가 질문에 답하지 않으면 어떻게 되나요?**

봇은 각 목표를 어떻게 구성했는지에 따라 재시도하거나, 넘어가거나, 대화를 종료합니다.

**Q: 봇을 복제할 수 있나요?**

네. 봇 메뉴의 **Duplicate(복제)** 옵션을 사용하여 구성을 재사용하세요.

**Q: 인스타그램이나 왓츠앱 같은 채널에서 가이드 폼 봇을 사용할 수 있나요?**

네, 해당 채널들이 Hyperclass 설정에 통합되어 있는 한 가능합니다.

**Q: 초기 메시지를 설정하면 가이드 폼 플로우는 언제 시작되나요?**

초기 메시지를 구성하면 봇이 먼저 해당 환영 메시지를 보내고 연락처의 다음 답변을 기다린 후 목표/질문 플로우를 시작합니다. 초기 메시지를 비워두면 첫 번째 메시지에서 즉시 플로우가 시작됩니다.

### **다음 단계**

- **봇 테스트**: 테스트 채팅 창을 사용하여 설정을 검증하고 목표가 예상대로 트리거되는지 확인하세요.
- **대화 모니터링**: Conversation(대화) 보기에서 참여도를 추적하여 사용자가 어떻게 상호작용하는지 확인하세요.
- **워크플로우와 함께 사용**: 워크플로우 트리거와 액션을 활용하여 리드 육성 프로세스를 자동화하세요.
- **봇 변형 탐색**: 고객 지원이나 이벤트 등록 같은 다른 사용 사례를 위한 봇 변형을 생성하세요.
