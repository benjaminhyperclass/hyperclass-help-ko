# **대화에서 이메일 전달이란?**

이메일 전달 기능을 통해 상담원은 대화(Conversation) 내의 특정 수신 이메일을 Hyperclass 안에서 바로 외부 수신자에게 보낼 수 있습니다. 전달된 메시지는 원본 이메일의 내용과 서식을 그대로 보존하므로, 계정 외부의 관련자들도 다른 도구로 전환하지 않고 맥락을 파악할 수 있습니다.

- 스레드에서 바로 수신 이메일 메시지를 외부 수신자에게 전달
- 맥락 연속성을 위해 원본 내용과 레이아웃 보존
- 감사 추적을 위해 동일한 대화(Conversation) 내에서 전체 기록 유지

## **이메일 전달의 주요 장점**

장점을 이해하면 팀이 언제 복사-붙여넣기나 별도 인박스 전환 대신 전달 기능을 사용할지 결정하는 데 도움이 됩니다. 이러한 장점들은 인수인계 마찰을 줄이고 기록을 중앙화합니다.

- **단일 작업 공간:** 대화(Conversations)를 이메일 스레드와 작업의 단일 정보 소스로 유지
- **빠른 에스컬레이션:** 몇 번의 클릭으로 검토자나 제3자에게 메시지 라우팅 - 앱 전환 불필요
- **일관성:** 원본 서식을 보존하여 관련자가 고객이 보낸 내용을 정확히 볼 수 있음
- **추적 가능성:** 팀 가시성을 위해 대화(Conversation) 타임라인에 전달 이벤트와 메시지 사본 보관
- **집중:** 수동 복사/붙여넣기 워크플로우 대비 맥락 손실과 인적 오류 감소

## **권한 및 이용 가능성**

전달하기 전에 올바른 사용자가 액세스 권한을 가지고 있고 이메일 채널이 올바르게 연결되어 있는지 확인하세요. 적절한 전제 조건을 갖추면 나중에 버튼이 보이지 않거나 전송 실패를 방지할 수 있습니다.

- 활성 이메일 채널이 하위 계정에 연결되어 있어야 함(예: LC 이메일, Gmail 양방향 동기화, Outlook 양방향 동기화 또는 SMTP)
- 사용자는 대화(Conversations) 액세스 권한과 이메일 발송 권한이 있어야 함
- 모바일 이용 가능성은 다를 수 있음; 전달 기능이 보이지 않으면 웹 앱을 사용하세요
- 전달은 계정의 구성된 발송 메일박스와 라우팅 규칙을 사용

## **발신자 및 From 이름 동작**

어떤 발신자 주소와 표시 이름이 사용되는지 알면 브랜드 가이드라인을 준수하고 수신자 기대치를 설정하는 데 도움이 됩니다. 이 섹션을 통해 현재 동작과 향후 개선 사항을 이해하세요.

- 전달은 대화(Conversation)의 이메일 채널과 연결된 발송 메일박스와 발송 우선순위 규칙을 사용하여 전송됩니다
- From 이름은 현재 계정 규칙을 따릅니다. 전달용 From 이름 편집 기능은 개선 사항으로 계획되어 있습니다
- 수신자는 기존 발송 우선순위(Sending Priority) 구성에 따라 발신자를 봅니다

## **첨부파일 및 서식**

전달에는 종종 스크린샷, PDF, 또는 인라인 이미지가 포함됩니다. 시스템이 서식과 파일을 어떻게 처리하는지 이해하면 재전송 시도와 수신자 혼란을 줄일 수 있습니다.

- 전달된 이메일은 명확성을 위해 원본 내용과 서식을 보존합니다
- 인라인 이미지와 서명은 소스와 제공업체에서 지원할 때 포함됩니다
- 원본 이메일의 첨부파일은 지원될 때 포함됩니다; 매우 큰 파일이나 차단된 유형은 제공업체나 보안 설정에 의해 제외될 수 있습니다
- 제공업체 및 도메인 정책(예: 크기 제한, 파일 유형 제한)이 여전히 적용됩니다

## **스레드 및 기록 보관**

대화(Conversation) 내에서 완전한 기록을 유지하면 팀이 협업하고 감사를 준수하는 데 도움이 됩니다. 이 섹션에서는 전달이 타임라인에 어떻게 표시되고 원본 메시지와 어떤 관련이 있는지 명확히 합니다.

- 전달은 동일한 대화(Conversation)에 발신 이메일 이벤트로 나타나므로 팀원들이 맥락을 볼 수 있습니다
- 타임라인에는 누가, 언제 전달했는지, 그리고 수신자(적절한 경우 편집됨)가 표시됩니다
- 전달된 메시지에 대한 답변은 연결된 메일박스로 다시 라우팅되고 동기화가 활성화된 경우 대화(Conversation)에 나타납니다

## **문제 해결**

전달이 실패하거나 예상대로 작동하지 않는 경우, 이러한 체크리스트를 사용하여 구성 문제, 제공업체 제한 또는 권한 제약을 빠르게 격리하세요.

- **전달 옵션이 보이지 않음:** 이메일 메시지를 보고 있는지(SMS나 소셜이 아닌), 발송 권한이 있는지 확인하세요. 계정에 이메일 채널이 연결되어 있는지 확인하세요.
- **반송 또는 전송 오류:** 대화(Conversation)의 오류 배너나 상태를 검토하세요. 도메인 DNS, 발신자 평판, 제공업체 제한을 확인하세요.
- **첨부파일 누락 또는 서식 깨짐:** 매우 큰 파일, 차단된 유형 또는 특정 인라인 형식은 제공업체에 의해 제거될 수 있습니다. 파일을 수동으로 다시 첨부하거나 보안 링크를 통해 공유하세요.
- **답변이 스레드로 돌아오지 않음:** 발송 메일박스에 대해 양방향 동기화가 활성화되고 정상 작동하는지 확인하세요.
- **예상치 못한 발신자:** 발송 우선순위(Sending Priority)와 메일박스 연결을 검토하여 Hyperclass가 전달에 선택한 주소를 확인하세요.

## **자주 묻는 질문**

**Q: 수신자에게 어떤 이메일 주소가 표시되나요?**
주소는 계정의 발송 우선순위와 대화(Conversation)의 이메일 채널에 연결된 메일박스를 따릅니다.

**Q: 전달하면 새로운 대화(Conversation)가 생성되나요?**
아니오, 전달은 동일한 대화(Conversation)에서 발신 메시지로 게시되므로 팀이 전체 맥락을 유지합니다.

**Q: 내부 메모나 비공개 댓글이 전달에 포함되나요?**
아니오, 전달에 포함한 이메일 내용만 외부로 전송됩니다. 내부 메모는 팀을 위해 대화(Conversation) 내에 남아있습니다.

**Q: 수신자가 전달된 메시지에 답장하면 어떻게 되나요?**
양방향 동기화가 활성화되어 있으면 답장이 연결된 메일박스로 다시 라우팅되고 동일한 대화(Conversation) 스레드에 나타납니다.

**Q: 모바일에서도 사용할 수 있나요?**
모바일 이용 가능성은 다를 수 있습니다. 전달 옵션이 보이지 않으면 웹 앱을 사용하세요.
