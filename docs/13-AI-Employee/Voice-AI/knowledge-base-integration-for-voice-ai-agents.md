# 음성 AI 에이전트의 지식 베이스 연동

음성 AI 에이전트를 더 스마트하게 만드는 방법을 소개합니다! Hyperclass의 새로운 음성 AI 에이전트 지식 베이스(Knowledge Base) 연동 기능을 사용하면 각 에이전트에 전용 지식 베이스를 연결하여 맞춤 프롬프트와 사용자 질문에 따라 정확하고 상황에 맞는 답변을 제공할 수 있습니다.

## 음성 AI 지식 베이스 연동이란?

음성 AI 지식 베이스 연동은 Hyperclass의 각 음성 AI 에이전트에 특정 지식 베이스를 할당할 수 있는 기능입니다. 사용자의 음성 입력이 설정된 트리거 프롬프트와 일치하면, 에이전트가 자동으로 연결된 지식 베이스를 검색하여 가장 관련성 높은 답변을 제공합니다. 이를 통해 음성 에이전트가 비즈니스 요구에 맞는 지능적이고 최신 정보를 담은 응답을 할 수 있습니다.

## 음성 AI 지식 베이스 연동의 주요 장점

이 강력한 기능으로 음성 AI 에이전트의 모든 잠재력을 활용하세요:

- **개인화된 에이전트 응답**: 각 에이전트에 고유한 지식 베이스를 할당하여 해당 영역이나 목적에 맞는 답변을 제공합니다.
- **트리거 기반 활성화**: 맞춤 프롬프트를 사용하여 지식 베이스 검색 시점을 정확히 제어하고, 사용자 의도와 일치하는 응답을 보장합니다.
- **실시간 동적 답변**: 음성 에이전트가 사용자 질문을 분석하고 지식 베이스에서 정확한 답변을 즉시 가져옵니다.
- **간편한 업데이트**: 지식 베이스의 변경 사항이 추가 설정 없이 에이전트 응답에 즉시 반영됩니다.
- **확장 가능하고 유지보수 용이**: 복잡한 로직이나 지속적인 개발 없이 여러 에이전트와 지식 베이스를 쉽게 관리할 수 있습니다.

## 음성 AI 지식 베이스 연동 설정 방법

올바른 설정을 통해 음성 AI 에이전트가 매번 정확하고 상황에 맞는 응답을 제공하도록 하세요. 다음 단계에 따라 에이전트에 지식 베이스를 연동하세요.

### 1단계: 음성 AI 에이전트 패널 열기

- Hyperclass 계정에서 AI 직원(AI Agents)을 클릭한 다음 음성 AI(Voice AI) 옵션을 클릭하세요.

![음성 AI 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046776161/original/ymUGO2sv0S4ftb8vi5i6kQ0jzETd0ani0w.png?1747410217)

### 2단계: 기존 에이전트 편집 또는 새 에이전트 생성

- 새 에이전트를 만들 수 있습니다. ([음성 AI 에이전트 생성 가이드](../how-to-create-voice-ai-agents.md) 참조)
- 또는 기존 에이전트를 편집할 수 있습니다.
- 편집하려면 에이전트 이름 옆 점 세 개를 클릭하고 편집(Edit)을 선택하세요.

![에이전트 편집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046776495/original/-YIkGk_vCPz9NKt3aGw7W7yUKwEXWpqg9A.png?1747410561)

### 3단계: 에이전트 목표 탭으로 이동

- 에이전트 편집기에서 에이전트 목표(Agent Goals) 탭을 클릭하여 에이전트의 성격과 로직을 설정하세요.
- 지식 베이스 선택(Select knowledge base) 항목에서 기존 지식 베이스 중 선택하거나 새로 만들기(Create New)를 클릭하여 새 지식 베이스를 추가하세요.
- 팁: 에이전트의 기능이나 대상에 맞는 지식 베이스를 할당하세요(예: 영업, 제품 지원).

![지식 베이스 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046776949/original/Uvc_ds_usGtNRCjJFD733aRKVA-bevndJQ.png?1747411107)

### 4단계: 지식 베이스 사용 시점 설정 (트리거 프롬프트)

- '이 지식 베이스를 언제 사용할지(When to use this knowledge base)' 필드에 에이전트가 언제 지식 베이스를 활성화할지 안내하는 상세한 프롬프트를 작성하세요.
- 예시: "사용자가 비즈니스, 서비스, 제품, 연락처 정보, 또는 정확하고 최신 정보를 제공하기 위해 비즈니스 위키에 접근이 필요한 기타 관련 정보에 대해 질문할 경우 이 지식 베이스를 사용하세요."

![트리거 프롬프트 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046792061/original/DH5XyCYWRq5ZW-RNQTc7YKLSnUTKcGGx-A.png?1747467737)

### 5단계: 명확하고 상세한 프롬프트 작성

- 잘 작성된 프롬프트는 에이전트가 질문을 더 정확하게 매칭하는 데 도움이 됩니다.
- 지시사항이 정확할수록 에이전트의 실시간 응답이 더 좋아집니다.
- 저장(Save)을 클릭하여 설정을 적용하고 지식 베이스를 에이전트에 연결하세요.

![프롬프트 작성 및 저장](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155046791767/original/RvE59PrM-r43E-7340t2BaeAhcBtPtc9qA.png?1747466764)

## 음성 AI 지식 베이스 관리 모범 사례

효과적인 지식 베이스 활용은 단순한 설정을 넘어 신중한 계획, 프롬프트 설계, 콘텐츠 관리, 성능 모니터링이 필요합니다.

### 여러 지식 베이스 관리
다양한 에이전트에 서로 다른 지식 베이스를 할당하면 특정 부서, 제품 또는 서비스에 맞는 응답을 제공할 수 있습니다.
- 영업, 지원, 제품 정보를 위한 전문 지식 베이스를 만드세요.
- 각 에이전트에 관련 지식 베이스를 연결하여 전문성을 높이세요.

### 트리거 프롬프트 최적화
잘 만들어진 트리거 프롬프트는 에이전트가 적절한 시점에 지식 베이스를 검색하도록 보장합니다.
- 일반적인 사용자 의도와 일치하는 명확하고 구체적인 언어를 사용하세요.
- 다양한 프롬프트를 테스트하여 대상 고객에게 가장 효과적인 것을 찾으세요.

### 지식 베이스 콘텐츠 업데이트
에이전트 응답의 정확성을 유지하려면 지식 베이스 콘텐츠를 최신 상태로 유지하세요.
- 정기적으로 아티클이나 FAQ를 검토하고 업데이트하세요.
- 변경사항은 모든 에이전트 상호작용에 즉시 반영됩니다.

### 에이전트 성능 모니터링
지식 베이스가 얼마나 자주 트리거되는지와 응답 품질을 추적하세요.
- 분석을 사용하여 지식 베이스의 부족한 부분을 파악하세요.
- 사용자 피드백과 에이전트 로그를 바탕으로 프롬프트나 콘텐츠를 개선하세요.

## 자주 묻는 질문

**Q: 하나의 음성 AI 에이전트에 둘 이상의 지식 베이스를 할당할 수 있나요?**
현재는 각 에이전트가 한 번에 하나의 지식 베이스만 연결할 수 있습니다.

**Q: 사용자 입력이 트리거 프롬프트와 일치하지 않으면 어떻게 되나요?**
지식 베이스가 검색되지 않으며, 에이전트는 기본 동작을 따릅니다.

**Q: 기본 모드와 고급 에이전트 모드 둘 다에서 이 기능을 사용할 수 있나요?**
네, 지식 베이스 연동은 에이전트 목표(Agent Goals)에서 기본 모드와 고급 모드 모두에서 사용할 수 있습니다.

**Q: 지식 베이스 업데이트가 에이전트 응답에 얼마나 빨리 반영되나요?**
업데이트는 즉시 반영됩니다. 지식 베이스의 모든 변경사항이 에이전트에서 즉시 사용 가능합니다.

**Q: 연결할 수 있는 지식 베이스 크기에 제한이 있나요?**
공식적인 제한은 없지만, 매우 큰 지식 베이스의 경우 성능이 달라질 수 있습니다.

**Q: 각 에이전트마다 트리거 프롬프트를 맞춤 설정할 수 있나요?**
네, 모든 에이전트에 고유한 트리거 프롬프트를 설정할 수 있습니다.

**Q: 이 연동이 서드파티 지식 베이스와 작동하나요?**
현재 이 기능은 Hyperclass의 내장 지식 베이스용으로 설계되었습니다.

**Q: 지식 베이스 연동이 제대로 작동하는지 어떻게 테스트할 수 있나요?**
에이전트 설정의 테스트 기능을 사용하거나 트리거 프롬프트를 사용하여 에이전트와 상호작용해 응답을 확인하세요.
