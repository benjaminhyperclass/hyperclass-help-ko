# 자동 정기 인보이스 설정 단계별 가이드

정기 인보이스(Recurring Invoices) 자동화는 청구 업무를 간소화하고 수동 작업을 줄이며, 고객으로부터 일관된 결제를 보장하는 데 도움이 됩니다. 워크플로우(Workflow) 자동화를 사용하면 일정에 따라 인보이스를 자동으로 발송하고, 청구 빈도를 제어하며, 자동 결제를 활성화하여 원활한 경험을 제공할 수 있습니다.

# **워크플로우의 정기 인보이스란?**

정기 인보이스는 사전에 정의된 간격으로 고객에게 자동으로 인보이스를 발송하는 기능입니다. 한 번 설정하면 수동 작업 없이도 인보이스가 자동으로 생성되고 전달되어, 안정적인 현금 흐름을 유지하고 결제 누락을 줄이는 데 도움이 됩니다.

구독, 보유계약, 멤버십, 할부 청구에 이상적입니다.

---

## **주요 장점**

- **시간 절약:** 수동 인보이스 발송과 후속 작업을 없앱니다
- **유연한 청구:** 일일, 주간, 월간, 연간 일정을 지원합니다
- **자동 결제:** 첫 결제 후 고객에게 자동으로 요금을 청구합니다
- **일관성:** 누락되거나 중복된 인보이스를 방지합니다
- **확장성:** 여러 고객의 청구를 쉽게 관리할 수 있습니다

---

## **사전 준비사항**

정기 인보이스를 설정하기 전에 다음을 확인해 주세요:

- 최소 하나의 인보이스 템플릿(Invoice Template)이 생성되어 있어야 합니다
- 결제 수단이 구성되어 있고 활성화되어 있어야 합니다
- 고객 레코드에 유효한 청구 정보가 포함되어 있어야 합니다

## **워크플로우에서 정기 인보이스 설정하는 방법**

### **1단계: 새 워크플로우 생성**

**워크플로우 생성(Create Workflow)** 버튼을 클릭하여 새로운 자동화를 만들기 시작하세요.

### **2단계: 트리거 선택**
![트리거 선택 패널](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069415036/original/yfZWRUtqwQCb1i639zj_Oib7GAk5sZnacw.jpeg?1776469311)

트리거 선택 패널 화면입니다.

**새 트리거 추가(Add New Trigger)**를 클릭한 다음, 워크플로우를 시작할 조건을 선택하세요 (예: 연락처가 생성되거나 업데이트될 때).

### **3단계: 정기 인보이스 액션 추가**
![액션 선택 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069415038/original/wNNDpPdrZhFva7vPb1f81DgbaCb5QSUJ0Q.jpeg?1776469320)

액션 선택 메뉴 화면입니다.

**정기 인보이스 발송(Send Recurring Invoice)**을 검색한 후 선택하여 워크플로우에 추가하세요.

### **4단계: 설정 구성**
![전체 구성 패널](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155069415040/original/JWkx8fAlwZ4A4G_4A1PzfFBT9m4DSA-Qgg.jpeg?1776469328)

전체 구성 패널 화면입니다.

다음 항목을 설정하세요:

- 인보이스 **발송자** 선택
- **인보이스 템플릿** 선택
- **테스트 또는 라이브 모드** 선택
- **시작 날짜** 설정 (트리거 기반 또는 고정 날짜)
- 청구 **종료 시점** 정의
- 청구 **빈도** 선택 (일일, 주간, 월간, 연간)
- 간격 규칙 설정 (예: X일마다)
- 필요한 경우 **자동 결제** 활성화

완료되면 **액션 저장(Save Action)**을 클릭하세요.

---

## **구성 설정 상세 설명**

### **시작 날짜**

청구가 시작되는 시점을 결정합니다. 워크플로우가 실행될 때 즉시 시작하거나 고정 날짜를 선택할 수 있습니다.

### **종료 규칙**

청구가 언제 중단되는지 제어합니다. 무기한으로 실행하도록 하거나 정해진 청구 주기 수 후에 중단하도록 설정할 수 있습니다.

### **청구 빈도**

인보이스가 발송되는 주기를 선택하세요:

- 일일
- 주간
- 월간
- 연간

각 옵션은 청구 요구사항에 따라 추가 맞춤 설정이 가능합니다.

### **자동 결제**

활성화하면 첫 번째 성공적인 거래 후 결제가 자동으로 수집됩니다. 이는 수동 후속 조치를 줄이고 결제 일관성을 향상시킵니다.

---

## **모범 사례**

- 워크플로우를 활성화하기 전에 테스트하세요
- 액션과 템플릿에 명확한 이름을 사용하세요
- 자동 결제를 활성화하기 전에 결제 설정을 확인하세요
- 첫 번째 청구 주기를 모니터링하여 모든 것이 올바르게 작동하는지 확인하세요

---

## **자주 묻는 질문**

**Q: 자동 결제 없이 인보이스만 자동화할 수 있나요?**

네, 고객이 수동으로 결제하도록 하면서 인보이스만 자동으로 발송할 수 있습니다.

**Q: 자동 결제가 실패하면 어떻게 되나요?**

인보이스는 미결제 상태로 남으며, 고객에게 알리는 후속 조치가 트리거될 수 있습니다.

**Q: 나중에 청구 빈도를 변경할 수 있나요?**

네, 언제든지 워크플로우를 편집하여 빈도 설정을 업데이트할 수 있습니다.

**Q: 인보이스가 자동으로 중단되나요?**

종료 조건을 정의한 경우에만 중단됩니다. 그렇지 않으면 수동으로 중단할 때까지 청구가 계속됩니다.

**Q: 다른 고객에게 다른 인보이스 템플릿을 보낼 수 있나요?**

네, 워크플로우 조건을 사용하여 고객 데이터를 기반으로 다른 템플릿을 보낼 수 있습니다.

---

## **도움이 필요하세요?**

정기 인보이스 설정이나 자동화 문제 해결에 도움이 필요하시면, 지원팀에 문의하거나 시스템 관리자에게 도움을 요청해 주세요.
