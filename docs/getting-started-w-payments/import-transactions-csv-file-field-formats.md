# 거래 내역 CSV 파일 가져오기 - 필드 형식 가이드

CSV 파일을 통한 거래 내역 업로드는 대량의 결제 기록을 시스템에 빠르게 가져올 수 있는 방법입니다. 올바른 형식을 사용하면 데이터가 정확하게 처리되고 일반적인 가져오기 오류를 방지할 수 있습니다.

---

## CSV 형식 요구사항

성공적인 업로드를 위해서는 올바른 파일 구조가 필수입니다. 시스템이 데이터를 올바르게 매핑하고 처리할 수 있도록 각 열이 예상 형식과 일치해야 합니다.

- 파일은 반드시 **.csv 형식**으로 저장해야 합니다
- 열 헤더는 필수 필드 이름과 **정확히 일치**해야 합니다  
- 각 행은 **하나의 거래**를 나타내야 합니다
- 모든 필수 필드에는 유효한 데이터가 포함되어야 합니다
- 불필요한 공백, 기호 또는 지원되지 않는 형식은 피해주세요

---

## 필수 거래 필드 설명

CSV의 각 열은 거래의 특정 부분을 나타냅니다. 정확한 데이터를 제공하면 적절한 기록 생성과 보고가 보장됩니다.

- **고객 이름/성:** 거래와 연결된 고객을 식별합니다
- **고객 이메일:** 연락처 식별 및 커뮤니케이션에 사용됩니다
- **고객 전화번호:** 정확성을 위해 국가 코드를 포함하세요
- **통화:** 표준 3글자 코드를 사용해야 합니다 (예: USD, EUR)
- **결제 금액:** 고객으로부터 최종 수금된 금액입니다
- **거래 날짜 및 시간:** 결제가 발생한 시점을 나타냅니다
- **결제 수단:** 결제 방법을 명시합니다 (예: 카드, 현금)
- **결제 제공업체:** 사용된 결제 처리업체를 식별합니다
- **주소 필드:** 청구지 또는 고객 위치 세부정보를 포함합니다

세금, 할인, 팁, 상품 세부정보와 같은 선택적 필드를 포함하여 더 자세한 거래 통찰력을 제공할 수 있습니다.

## 피해야 할 일반적인 업로드 오류

올바르지 않은 형식은 가져오기 실패의 가장 흔한 원인입니다. 업로드 전에 파일을 검토하면 시간을 절약하고 데이터 문제를 예방할 수 있습니다.

- **결제 금액, 거래 날짜 또는 고객 이메일**과 같은 필수 필드 누락
- 잘못된 날짜 또는 시간 형식
- ISO 통화 코드 대신 기호 사용 (예: USD 대신 $ 사용)
- 유효하지 않은 이메일 또는 전화번호 형식
- 일치하지 않거나 철자가 틀린 열 헤더

---

## 성공적인 가져오기를 위한 팁

몇 가지 간단한 확인으로 CSV가 문제 없이 업로드되고 데이터가 정확하게 유지되는지 확인할 수 있습니다.

- 업로드 전에 열 헤더를 다시 확인하세요
- 모든 행에서 형식을 일관되게 유지하세요
- 내보내기 전에 스프레드시트 도구로 데이터를 검증하세요
- 큰 데이터셋을 가져오기 전에 작은 파일로 테스트하세요

---

## 자주 묻는 질문

**Q: 여러 거래를 한 번에 업로드할 수 있나요?**
네, 각 행은 하나의 거래를 나타내므로 하나의 파일로 여러 기록을 업로드할 수 있습니다.

**Q: 필수 필드가 누락되면 어떻게 되나요?**
문제에 따라 파일 업로드가 실패하거나 유효하지 않은 행을 건너뛸 수 있습니다.

**Q: 목록에 없는 추가 필드를 포함할 수 있나요?**
지원되는 필드만 처리됩니다. 추가 열은 무시될 수 있습니다.

**Q: 날짜는 어떤 형식으로 해야 하나요?**
시스템 설정에 따라 DD/MM/YYYY 또는 MM/DD/YYYY와 같은 일관된 형식을 사용하세요.

**Q: CSV를 사용하여 기존 거래를 업데이트할 수 있나요?**
이는 시스템 동작에 따라 다릅니다. 일반적으로 별도로 지정하지 않는 한 CSV 가져오기는 새 기록을 생성합니다.

---

CSV 파일이 업로드되지 않거나 형식에 대해 확실하지 않은 경우, 지원팀에 문의하거나 샘플 구조와 비교하여 파일을 검토해 문제를 식별하세요.
