## 1. 레코드 충돌: 다중 A 레코드
![레코드 충돌 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045995788/original/XUdHwVShoR99l69a6uzoeSz1LC8azlgMHg.png?1746169033)

### A 레코드란 무엇인가요?

A 레코드는 도메인 이름을 IP 주소에 연결합니다. 방문자를 웹사이트가 호스팅되는 올바른 서버로 안내하는 역할을 합니다.

### 언제 발생하나요?

도메인에 여러 개의 A 레코드가 있으면 시스템이 어떤 IP 주소를 사용해야 할지 혼란스러워합니다. 이로 인해 웹사이트가 제대로 로드되지 않을 수 있습니다.

### 해결 방법

- 도메인 등록업체의 DNS 설정(Settings)에 로그인하세요(예: GoDaddy, Namecheap, Cloudflare).
- 도메인의 DNS 설정을 찾으세요.
- @나 yourdomain.com처럼 IP 주소를 가리키는 A 레코드를 찾으세요.
  - @는 루트 도메인을 의미합니다(예: yourdomain.com).
- 도메인이나 서브도메인(예: www.yourdomain.com)을 IP 주소에 매핑하는 A 레코드를 모두 삭제하세요.
- 변경사항을 저장하고 1-2분 정도 기다린 후 적용되도록 하세요.
- 플랫폼에 도메인을 다시 연결하세요.

## 2. AAAA 레코드 충돌
![AAAA 레코드 충돌 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045995807/original/FAULhnOFiIfW-vPg3FQzZAezCEEwoMSBZw.png?1746169070)

### AAAA 레코드란 무엇인가요?

AAAA 레코드는 A 레코드와 비슷하지만 IPv6 주소에 사용되는 새로운 버전의 IP 주소를 위한 것입니다.

### 언제 발생하나요?

- 플랫폼이 도메인에서 AAAA 레코드를 감지했을 때
- 시스템은 IPv4 레코드만 허용하므로 AAAA 레코드(IPv6)가 있으면 시스템 요구사항과 충돌하여 도메인 연결이 제대로 되지 않습니다.

### 해결 방법

- 도메인 등록업체의 DNS 관리 패널에 로그인하세요.
- AAAA 레코드(보통 AAAA 또는 IPv6로 표시됨)를 찾으세요.
- AAAA 레코드를 삭제하세요.
- 변경사항을 저장하고 1-2분 정도 기다리세요.
- 플랫폼에 도메인을 다시 연결하세요.

### Cloudflare 사용자를 위한 중요 참고사항

Cloudflare에서 A 레코드에 프록시가 켜져 있어도(주황색 구름) DNS 해석 도구에서 여전히 AAAA 레코드를 표시할 수 있습니다. 이는 다음과 같은 경우일 수 있습니다:

- Cloudflare가 프록시된 트래픽을 처리하는 방식으로 인한 시각적 표시 오류(실제로 추가한 레코드가 아님)
- 수동으로 생성했거나 DNS 영역에 존재하는 실제 AAAA 레코드 충돌

Cloudflare의 DNS Records(DNS 레코드) 섹션에서 AAAA 레코드가 실제로 존재하는지 확인해야 합니다. 존재한다면 삭제 절차를 진행하세요.

## 3. CAA 레코드 충돌
![CAA 레코드 충돌 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045995838/original/89frjlT0kXQx02SOWVvButGEeJWf8BH6gQ.png?1746169109)

### CAA 레코드란 무엇인가요?

CAA(Certificate Authority Authorization) 레코드는 어떤 인증기관(CA)이 도메인의 SSL 인증서를 발급할 수 있는지 지정합니다. 인증된 제공업체로만 인증서 발급을 제한하여 추가 보안 계층을 제공합니다.

### 언제 발생하나요?

CAA 레코드가 존재하지만 필요한 값(pki.goog)이 포함되지 않은 경우 플랫폼이 도메인의 SSL 인증서를 발급할 수 없습니다.

### 해결 방법

#### 권장 방법 (대부분의 사용자)

- 도메인 등록업체의 DNS 관리 패널에 로그인하세요.
- CAA 레코드(보통 CAA로 표시됨)를 찾으세요.
- CAA 레코드를 삭제하세요.
- 변경사항을 저장하고 1-2분 정도 기다리세요.
- 플랫폼에 도메인을 다시 연결하세요.

#### 고급 사용자 (삭제 대신 수정)

CAA 레코드를 유지하면서 플랫폼을 인증하려면 기존 CAA 레코드에 다음 값을 추가하세요:

- pki.goog

이렇게 하면 다른 보안 설정(Settings)을 제거하지 않고도 플랫폼이 도메인의 SSL 인증서를 발급할 수 있습니다.

## 4. DNS 레코드 불일치

![DNS 레코드 불일치 오류 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045995875/original/zs7rfSjmgWnZZ1G2JmpwAJXEDLjonqTr7g.png?1746169166)

### 이게 무슨 뜻인가요?

이 오류는 도메인의 DNS 설정이 플랫폼 요구사항과 일치하지 않을 때 발생합니다. 다음과 같은 원인이 있을 수 있습니다:

- 충돌하는 레코드
- 잘못된 설정
- 도메인이 다른 서비스에 연결되어 있음

### 언제 발생하나요?

- DNS 설정이 올바르게 구성되지 않았을 때
- DNS 설정이 올바르더라도 전파 지연이 발생할 수 있습니다. 변경사항이 반영될 때까지 몇 분 정도 기다리세요.

### 해결 방법

#### 1. DNS 레코드 확인

- DNS 조회 도구(예: DNS Checker)를 사용하여 DNS 전파를 확인하세요.
- 플랫폼에서 요구하는 DNS 구성과 비교해보세요. 대부분의 플랫폼은 특정 값(예: A 레코드용 IP 주소 또는 CNAME 레코드)을 제공합니다.

#### 2. 충돌 확인

- 레코드가 일치하는 경우 → 전파까지 몇 분 기다리세요(최대 48시간 소요 가능).
- 레코드가 충돌하는 경우 → 잘못된 항목을 제거하거나 업데이트하세요:
  - A 레코드가 필요한 경우 추가 A 레코드가 없는지 확인하세요.
  - CNAME 레코드가 필요한 경우 충돌하는 A나 AAAA 레코드를 제거하세요.

#### 3. 도메인 다시 연결

- 변경사항을 저장하고 1-2분 정도 기다리세요.
- 도메인을 다시 연결하고 확인하세요.

## 중요 참고사항 (GoDaddy 사용자용)

도메인 제공업체가 GoDaddy인 경우 도메인이 파킹되어 있지 않은지 확인하세요.

- GoDaddy 계정에 로그인하세요.
- My Products(내 상품) > Domains(도메인)으로 이동하여 도메인 옆의 Manage DNS(DNS 관리)를 클릭하세요.
- 도메인이 파킹되지 않았는지 확인하세요:
  - 도메인이 파킹되어 있으면 GoDaddy가 도메인을 보유하고 있어 어떤 서비스에도 연결할 수 없습니다.
  - GoDaddy 지침에 따라 도메인 파킹을 해제하세요.

## 5. 도메인이 다른 곳에 연결됨

### 언제 발생하나요?

도메인이 이미 다른 에이전시나 하위 계정(Sub-account)에 연결되어 있을 때입니다.

### 해결 방법

- 필요한 경우 도메인 해제를 위해 지원팀에 문의하세요.

---

## 추가 참고사항

✅ 전파 시간: DNS 변경사항은 전 세계적으로 전파되는 데 24-48시간이 걸릴 수 있지만, 보통 몇 분 내에 적용됩니다. 문제가 지속되면 참을성 있게 기다리고 DNS 설정을 다시 확인하세요.

✅ 레코드 재확인: 도메인을 다시 연결하기 전에 DNS 조회 도구를 사용하여 변경사항이 올바르게 전파되었는지 확인하세요.

✅ 캐시 삭제: 오래된 DNS 정보를 보지 않도록 브라우저 캐시를 삭제하거나 시크릿 모드를 사용하세요.
