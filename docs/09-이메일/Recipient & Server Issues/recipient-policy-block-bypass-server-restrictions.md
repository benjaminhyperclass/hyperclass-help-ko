# 수신자 정책 차단 해결 가이드

## 무엇이 일어나고 있나요?

당신의 이메일이 관리 정책, 보안 제한 또는 액세스 제어 규칙으로 인해 수신자 메일 서버에서 거부되고 있습니다. 기술적 전달 실패와 달리, 이러한 차단은 수신 조직이 원치 않거나 잠재적으로 해로운 메시지로부터 이메일 환경을 보호하기 위해 내린 의도적인 정책 결정입니다.

## 빠른 진단: 정책 차단 식별하기

### 일반적인 정책 차단 메시지

- "수신자의 메일 서버가 관리 또는 정책 제한으로 인해 메시지를 차단했습니다"
- "수신자의 메일 서버가 사용자 없음, 릴레이 거부 또는 스팸 감지와 같은 정책으로 인해 메시지를 거부했습니다"
- "수신자의 메일함은 승인된 발신자의 메일만 수신합니다. 귀하의 주소는 승인되지 않았습니다"
- "수신자의 메일 서버가 정책 또는 블랙리스트 규칙으로 인해 발신자를 거부했습니다"
- "수신자의 도메인이 자체 정책으로 인해 메일을 적극적으로 거부하고 있습니다"
- "수신자의 조직이 보안 또는 정책상의 이유로 외부 이메일 전달을 차단합니다"
- "수신자가 수신 거부했거나 옵트아웃했으므로 정책에 의해 이메일 전달이 차단됩니다"
- "그룹 멤버십 또는 외부 발신자 제한으로 인해 발신자가 이 그룹에 이메일을 보낼 수 없습니다"
- "수신자의 메일 서버가 허용된 IP 또는 인증서를 제한하는 엄격한 커넥터 정책으로 인해 메시지를 거부했습니다"

## 수신자 정책 차단 이해하기

### 핵심 개념

**중요 정보:**

- 정책 차단은 의도적인 제한이며, 기술적 실패가 아닙니다
- 이러한 차단은 조직을 스팸, 피싱 및 원치 않는 커뮤니케이션으로부터 보호합니다
- 해결을 위해서는 발신자와 수신자 양쪽의 조치가 필요한 경우가 많습니다
- 일부 정책 차단은 조직의 영구적 결정입니다

### 정책 차단 카테고리

**발신자 기반 차단:**
- IP 주소 또는 도메인 블랙리스트
- 발신자 평판 제한
- 인증되지 않은 발신자 정책

**콘텐츠 기반 차단:**
- 스팸 필터 트리거
- 콘텐츠 정책 위반
- 첨부파일 제한

**접근 제어 차단:**
- 화이트리스트 전용 정책
- 외부 발신자 제한
- 그룹 멤버십 요구사항

## 단계별 정책 차단 해결 방법

### 1단계: 이메일 인증 확인

도메인 인증 설정을 확인하세요:

- **인증 설정으로 이동:** `Settings(설정) → Email Services(이메일 서비스) → Sending Domain & IP(발신 도메인 및 IP)`로 이동
- SPF, DKIM, DMARC 레코드가 올바르게 구성되었는지 확인
- 모든 인증 검사가 "Verified(확인됨)" 상태인지 확인
- **외부에서 인증 테스트:** MXToolbox.com을 사용해 SPF 및 DKIM 레코드 확인
- dmarcian.com을 사용해 DMARC 정책 일치 확인

#### 인증 성공 지표

- 플랫폼의 모든 도메인 확인 검사가 통과
- 외부 도구가 올바른 SPF, DKIM, DMARC 설정을 확인
- 인증 관련 반송 메시지가 없음

### 2단계: 발신자 평판 모니터링

**발신 평판을 평가하세요:**

- **평판 점수 확인:** senderscore.org에서 IP 평판 확인 (80점 이상 목표)
- Talos Intelligence를 사용해 도메인 평판 확인
- Outlook/Hotmail 사용자에게 발송하는 경우 Microsoft SNDS 확인
- **지표 검토:** `Marketing(마케팅) → Email Marketing(이메일 마케팅) → Analytics(분석)`로 이동
- 반송률, 스팸 신고, 수신 거부율 모니터링
- 정책 차단 메시지의 패턴 식별

### 3단계: 수신자 조직 연락

**화이트리스트 고려를 요청하세요:**

- **적절한 연락처 식별:** IT 관리자나 이메일 보안 팀 연락처 찾기
- 이메일 전달 문의를 위한 조직 웹사이트 확인
- 대체 커뮤니케이션 채널 사용 (전화, SMS)
- **화이트리스트 요청 준비:** 발신 도메인과 IP 주소 포함
- 비즈니스 관계와 이메일 목적 설명
- 적절한 인증과 규정 준수의 증거 제공

### 4단계: 이메일 콘텐츠 및 관행 최적화

**정책 트리거 가능성을 줄이세요:**

- **이메일 콘텐츠 검토:** 스팸 트리거 단어와 과도한 프로모션 언어 피하기
- 적절한 텍스트 대 이미지 비율 보장
- 명확한 수신 거부 메커니즘 포함
- **리스트 위생 관리:** 반송되거나 수신 거부한 연락처를 정기적으로 정리
- 참여도에 따른 리스트 세분화

#### 콘텐츠 최적화 성공

- mail-tester.com 테스트 시 스팸 점수 감소
- 참여율 향상 및 신고 감소
- 이전에 차단했던 도메인에 대한 일관된 전달

## 복구 일정 및 예상 결과

### 1단계: 즉시 조치 (1-3일)

- **인증 수정:** 인증 기반 차단에 대한 즉각적 개선
- **예상 결과:** 기술적 인증 문제 해결

### 2단계: 평판 구축 (2-4주)

- **일관된 좋은 관행:** 발신자 평판의 점진적 개선
- **예상 결과:** 평판 기반 필터의 정책 차단 감소

### 3단계: 관계 구축 (지속적)

- **화이트리스트 요청:** 조직의 응답성에 따른 가변적 일정
- **예상 결과:** 특정 수신자 도메인에 대한 영구적 해결

## 예방 전략

### 사전 예방적 모니터링

- **주간 평판 점검:** Sender Score 및 도메인 평판 모니터링
- **반송 분석:** 분석에서 주간 반송 메시지 검토
- **참여도 추적:** 오픈율, 클릭률, 신고 모니터링

### 이메일 모범 사례

- **허가 기반 리스트:** 명시적으로 옵트인한 연락처에게만 이메일 발송
- **정기적 리스트 정리:** 비활성 및 반송 연락처를 매월 제거
- **점진적 볼륨 증가:** 새로운 도메인으로의 발송량을 천천히 늘리기
- **일관된 발송 패턴:** 정기적인 발송 일정과 볼륨 유지

## 여전히 문제가 있나요?

정책 차단 문제가 계속된다면:

- **패턴 문서화:** 어떤 도메인이 차단하고 있는지와 구체적인 오류 메시지를 상세히 기록
- **대체 채널 고려:** 중요한 커뮤니케이션을 위해 전화, 우편 또는 소셜 미디어 사용
- **이메일 전략 평가:** 현재 접근법이 수신자의 기대와 일치하는지 검토
