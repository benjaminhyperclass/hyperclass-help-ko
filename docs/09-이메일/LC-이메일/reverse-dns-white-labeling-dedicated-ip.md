# 역방향 DNS 화이트라벨링 - 전용 IP

고객이 이메일을 열어볼 때, "Sent by Mailgun"이나 다른 서드파티 서비스 이름 대신 회사명이 표시되는 것이 훨씬 유리합니다. 역방향 DNS(rDNS)를 설정하면 메일 서비스 제공업체가 "보낸 사람" 또는 "발신자" 필드에 회사명을 표시하도록 안내할 수 있습니다. 이런 방식은 이메일 커뮤니케이션의 전문성을 높일 뿐만 아니라 브랜드 정체성도 강화해줍니다.

또한 "보낸 사람" 필드에 서드파티 서비스 이름 대신 회사명이 표시되면 전달률과 추적 성능이 향상됩니다. 수신자의 신뢰를 높이고, 이메일이 스팸으로 분류될 가능성을 줄이며, 모든 이메일 캠페인에서 일관된 브랜딩을 보장합니다.

요약하자면, 역방향 DNS 설정을 통해 회사명을 적용하는 것은 더욱 긍정적이고 브랜드 중심적인 이메일 커뮤니케이션 방법을 제공하며, 전달률과 추적 성능을 모두 향상시킵니다.

## 역방향 DNS(rDNS)란 무엇인가요?

역방향 DNS(rDNS)는 기본적으로 일반적인 DNS 쿼리와 반대입니다. 도메인 이름을 IP 주소로 매핑하는 대신, rDNS는 IP 주소를 연결된 호스트네임으로 변환합니다. 이 프로세스는 IPv4와 IPv6 주소 모두에 적용되며, 이메일을 보내는 서버의 진위를 검증합니다.

실제 적용에서 rDNS는 이메일 발송 서버의 IP 주소를 해당하는 호스트네임(예: example.com)으로 변환합니다. 이 검증을 통해 IP 주소가 합법적인 도메인과 관련이 있음을 확인할 수 있습니다. 이메일 마케팅에서는 Mailgun 같은 서비스를 사용하면서도 "Sent by Mailgun.com" 대신 회사명이 발신자로 표시되도록 할 수 있습니다.

명확히 하자면, 이는 DNS 설정에서 mailgun.org의 CNAME을 마스킹하는 것과는 다릅니다. 여기서는 이메일 발송에 사용되는 전용 IP에 대한 rDNS 설정에 초점을 맞춥니다. 적절한 rDNS 설정은 수신자의 인박스에서 회사명이 발신자로 표시되도록 하여 이메일 전달률과 브랜딩을 개선합니다.

## rDNS 설정 시 자주하는 실수

rDNS 레코드를 정확하게 설정하는 것은 이메일 전달률과 신뢰할 수 있는 발신자 상태 유지에 필수적입니다. 자주 발생하는 rDNS 설정 실수는 다음과 같습니다:

**잘못된 PTR 레코드**: 가장 흔한 실수 중 하나는 IP 주소와 연결된 유효한 PTR(Pointer) 레코드가 없는 것입니다. PTR 레코드는 발송 메일 서버가 이메일 메시지에서 주장하는 IP 주소와 일치하는지 확인하는 데 필수적입니다. PTR 레코드가 올바르게 설정되어 있고 쿼리된 IP 주소에 대한 호스트네임을 반환하는지 확인하세요.

**호스트네임 불일치**: PTR 레코드가 반환하는 호스트네임은 동일한 IP 주소로 다시 해석되어야 합니다. 즉, IP 주소는 호스트네임을 가리켜야 하고, 호스트네임은 다시 동일한 IP 주소를 가리켜야 합니다. 불일치가 발생하면 rDNS 조회 실패로 이어질 수 있습니다. 예를 들어, PTR 레코드가 IP 주소 8.8.8.8에 대해 "mail.yourwebsite.com"을 반환하도록 설정되어 있다면, "mail.yourwebsite.com"에 대해 rDNS 조회를 수행할 때 8.8.8.8로 해석되어야 합니다.

**호스트네임 해석 실패**: 경우에 따라 PTR 레코드에 지정된 호스트네임이 전혀 해석되지 않을 수 있습니다. 이는 전달 문제와 이메일 제공업체와의 신뢰 부족으로 이어질 수 있는 심각한 문제입니다. 이 문제를 해결하려면 지정된 호스트네임이 올바른 IP 주소로 해석되는지 확인하세요.

요약하자면, rDNS는 이메일 인증과 전달률의 핵심 구성 요소입니다. 일반적인 rDNS 실수를 피하려면 PTR 레코드가 올바르게 설정되어 있고 PTR 레코드가 반환하는 호스트네임이 연결된 IP 주소와 정확히 일치하는지 확인하세요. 이러한 일치는 이메일 제공업체와의 신뢰를 구축하고 이메일이 수신자의 인박스에 도달할 가능성을 높이는 데 도움이 됩니다.

## rDNS 설정 방법

여러 호스트네임으로 역방향 DNS(rDNS)를 설정하는 것은 전용 IP를 사용할 때만 가능하다는 점을 알아두세요. 공유 IP는 이 기능을 지원하지 않습니다. rDNS는 IP 주소당 하나의 호스트네임으로만 해석될 수 있기 때문입니다. 이 목적으로 전용 IP를 사용하면 이메일 커뮤니케이션에 전문적인 느낌을 더하고 인박스 제공업체와의 평판에 긍정적인 영향을 줄 수 있습니다.

전용 IP에서 여러 호스트네임으로 rDNS를 설정하려면 아래 단계를 따라하세요:

- 호스팅 제공업체의 존 파일에 "A 레코드"를 입력하세요. "A 레코드"는 다음과 같습니다: A, mail.customer.com, 123.45.67.89

- `Settings(설정) → Email Service(이메일 서비스)`로 이동하세요.
- SMTP Service(SMTP 서비스) 탭에서 LeadConnector 섹션의 Dedicated Domain And IP(전용 도메인 및 IP)를 클릭하세요.
![전용 도메인 및 IP 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155009612052/original/IUIljk9XIJS27U3BT_o4jUzNTpWR7LasGg.png?1696855911)

- 전용 IP 섹션으로 이동하여 전용 IP의 점 3개 메뉴를 클릭한 후 Reverse DNS (PTR)를 클릭하세요.
![역방향 DNS 설정 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155009612160/original/cwkp94jqGl-jc1vsViWXa2bvRMaphEwmkw.png?1696855977)

- 입력란에 추가한 것과 동일한 A 레코드를 제공하시면, 사용하시는 호스트네임과 일치하도록 PTR 레코드를 업데이트해드립니다.
