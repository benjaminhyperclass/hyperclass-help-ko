# LC 이메일 캠페인에서 발신 도메인 작동 방식

발신 도메인(Sender Domain)은 이메일 캠페인 발송 시 어떤 도메인을 사용할지와 발신 이메일 주소가 유효하고 규정을 준수하는지를 결정합니다. 올바른 발신 도메인 선택은 캠페인 실패를 방지하고 도메인 평판을 보호하며 받은편지함 도달률을 향상시킵니다.

발신 도메인 기능은 **LC Email** 하위 계정에만 적용됩니다. SMTP 기반 이메일 발송은 기존 워크플로우를 따릅니다.

# **발신 도메인 설정이란?**

발신 도메인 설정은 마케팅(Marketing) → 이메일(Emails) → 캠페인(Campaigns)에서 이메일 캠페인을 발송하거나 예약할 때 사용할 수 있습니다. 이 설정은 캠페인 발송에 사용할 도메인을 결정하고 발신 이메일 주소가 해당 도메인과 일치하는지 확인합니다.

이메일 제공업체들은 메시지를 올바르게 인증하기 위해 이러한 일치를 기대합니다. 발송 도메인과 발신 이메일이 일치하지 않으면 캠페인이 발송에 실패하거나 스팸으로 분류될 수 있습니다. 발신 도메인 설정을 통해 각 캠페인에 적절한 발송 도메인을 선택하고, 발신 이메일을 실시간으로 검증하며, 도메인 준수 요건이 충족되지 않으면 캠페인 시작을 방지할 수 있습니다.

이러한 캠페인별 제어는 발송 문제를 줄이고 전반적인 발송 신뢰성을 향상시킵니다. 캠페인이 발송되거나 예약되면 선택한 발신 도메인이 해당 캠페인에 고정됩니다.

---

## **발신 도메인 설정의 주요 장점**

- **발송 실패 감소:** 일치하지 않거나 규정을 준수하지 않는 발신 도메인으로 캠페인이 발송되는 것을 방지하여 발송 시 하드 오류를 줄입니다.

- **향상된 전달률:** 적절한 도메인 정렬을 보장하여 이메일 제공업체가 메시지를 인증하도록 돕고 받은편지함 도달률을 개선합니다.

- **캠페인별 제어:** 단일 기본값에 의존하는 대신 각 캠페인에 가장 적절한 발송 도메인을 선택할 수 있습니다.

- **자동 보안장치:** 지원되는 발신 이메일을 자동으로 조정하고 수동 조치가 필요할 때 명확한 경고를 표시합니다.

- **명확한 가시성:** 실시간 검증과 도메인 구성에 대한 빠른 접근을 제공하여 문제를 쉽게 식별하고 해결할 수 있습니다.

## **다양한 시나리오에서의 발신 도메인 동작**

**공유 도메인(에이전시 또는 HighLevel)**

캠페인이 공유 발송 도메인을 사용할 때 Gmail이나 Yahoo 같은 공개 이메일 주소가 지원됩니다. HighLevel이 규정 요건을 충족하기 위해 백그라운드에서 발신 이메일을 자동으로 조정하여 사용자 개입 없이 캠페인을 성공적으로 발송할 수 있습니다.

**단일 전용 도메인**

하위 계정에 하나의 검증된 전용 도메인이 있는 경우, 발송 또는 예약 단계에서 자동으로 선택됩니다. 발신 이메일은 이 도메인과 일치해야 하며, 실시간 검증을 통해 발송 전 캠페인 준수를 확인합니다.

**여러 전용 도메인**

여러 검증된 도메인을 사용할 수 있는 경우, 캠페인에 특정 도메인을 선택하거나 전체 도메인(All Domains)을 선택할 수 있습니다. 전체 도메인을 선택하면 HighLevel이 내부 규칙에 따라 적합한 도메인 간 발송을 순환하여 대규모 전달성을 관리합니다.

아래 표는 일반적인 캠페인 시나리오에서 발신 도메인이 어떻게 작동하는지 요약합니다.

| **시나리오** | **사용자 입력** | **결과** |
|---|---|---|
| 공유 도메인 + 공개 이메일 | name@gmail.com | 규정 준수를 위해 이메일이 자동 조정됨 |
| 단일 전용 도메인 | 일치하는 도메인 이메일 | 도메인이 자동 선택되고 검증됨 |
| 여러 도메인(단일 선택) | 하나의 도메인 선택 | 선택된 도메인만 사용 |
| 여러 도메인(전체 도메인) | marketing@... 입력 | 도메인이 자동으로 순환됨 |
| 전용 도메인 불일치 | 도메인 ≠ 발신 이메일 | 발송 차단됨 |
| 도메인 미검증 | DNS 설정 미완료 | 수정 링크와 함께 경고 표시 |

## **캠페인의 발신 도메인 설정 및 구성 방법**

발신 도메인 선택은 이메일 캠페인 발송 또는 예약 과정의 자연스러운 일부로 이루어집니다. 미리 켜거나 구성할 별도의 설정은 없습니다.

**마케팅(Marketing) → 이메일(Emails) → 캠페인(Campaigns)**으로 이동하세요

![이메일 캠페인 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062541657/original/ze0sRG7VX2OdNL3Lnn58GbebG9a5f9CBlA.png?1768304730)

새 캠페인을 열거나 기존 초안을 선택하세요.

![캠페인 편집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062541708/original/uvGDXxJ6OmcIoJVp0Fm0hEi5H3TaIKdGhg.png?1768304758)

이메일 캠페인 빌더에서 **발송(Send)** 또는 **예약(Schedule)** 버튼을 클릭하여 캠페인 세부사항을 검토하세요.

![캠페인 발송 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062541742/original/cVjtJTns0eY4IIlmYzVj8Nl-4ScM1SJPFg.png?1768304799)

**발신자 세부사항(Sender Details)** 섹션에서 발신자 이름(Sender Name)과 발신 이메일(Sender Email)을 확인하세요.

![발신자 세부사항](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062542511/original/yUhh4H19Wi1FRvcYcSN93HpXGeQJx8F8Iw.png?1768305076)

**발신 도메인(Sender Domain)** 드롭다운을 열어 하위 계정에서 사용 가능한 발송 옵션을 확인하세요.

**발신 도메인** 드롭다운은 하위 계정에서 사용 가능한 발송 옵션을 보여주며 이메일 도메인 구성에 따라 달라집니다.

**공유 도메인**을 사용하는 경우 공개 이메일 주소가 지원되고 규정 준수를 위해 자동으로 조정됩니다. **하나의 전용 도메인**을 사용할 수 있는 경우 미리 선택되며 발신 이메일과 일치해야 합니다. **여러 전용 도메인**이 구성된 경우 특정 도메인을 선택하거나 **전체 도메인(All Domains)**을 선택하여 HighLevel이 전달성 관리를 위해 적합한 도메인 간 발송을 순환하도록 할 수 있습니다.

![발신 도메인 드롭다운](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062544375/original/xpDMY7KgIttDxgOURKmFGCDmFjfgTAx-uQ.png?1768305964)

경고나 오류가 나타나면 하이퍼링크를 사용하여 도메인 구성 검토(Review Domain Configuration)를 통해 도메인 검증, 워밍업 상태, IP 할당 등을 확인하세요.

![도메인 구성 경고](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062543611/original/BSUw9vRW7fKWi1HL_x50G8SVAKLSoIprtA.png?1768305582)

발신 도메인을 구성할 때 다양한 유형의 메시지가 표시될 수 있습니다. **경고**는 발신 이메일과 발신 도메인 불일치와 같은 잠재적 문제를 나타내며, 캠페인은 여전히 발송할 수 있지만 스팸 필터링 위험을 줄이기 위해 일치하는 도메인을 권장합니다. **오류**는 문제가 해결될 때까지 캠페인을 발송할 수 없는 차단 문제를 나타냅니다.

![오류 메시지](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062565977/original/7dUXgMQxCCQeFiv1rW-g-rt7g95y4W-0ZQ.png?1768316471)

모든 것이 정상적으로 보이면 캠페인 발송을 완료하거나 예약하세요. 선택한 발신 도메인은 캠페인이 발송되거나 예약될 때 해당 캠페인에 고정됩니다.

## **자주 묻는 질문**

**Q: 캠페인을 여전히 발송할 수 있는데 왜 경고가 표시되나요?**

발신 이메일과 도메인 불일치와 같은 잠재적 전달성 문제가 있을 때 경고가 나타납니다. 캠페인은 여전히 발송할 수 있지만 스팸 필터링을 방지하기 위해 일치하는 도메인을 권장합니다.

**Q: 발신 도메인이 검증되지 않았거나 워밍업되지 않았다면 어떻게 되나요?**

도메인이 검증되고 준비 요건을 충족할 때까지 캠페인을 발송할 수 없습니다. 경고나 오류 메시지가 필요한 설정을 완료하도록 안내합니다.

**Q: 발신 도메인과 다른 회신 주소(Reply-To)를 사용할 수 있나요?**

네. 회신 주소는 발신 도메인과 독립적이며 유효한 이메일 주소라면 어떤 것이든 지정할 수 있습니다.

**Q: "전체 도메인(All Domains)" 선택이 이메일 전달성에 영향을 주나요?**

아니요. 선택하면 HighLevel이 검증되고 워밍업된 도메인에서만 이메일을 발송하며, 평판 관리와 전달성 유지를 위해 내부 규칙에 따라 도메인을 순환합니다.
