# LC Email - 전용 IP 완전 가이드

전용 IP는 고유하고 독점적인 IP 주소에서 이메일 메시지를 발송하여 뚜렷한 이점을 제공합니다. 이메일 서비스 제공업체(ESP)는 IP 주소의 평판과 동작을 면밀히 모니터링하여 해당 IP와 연결된 특정 도메인과 관련된 이메일의 전달 가능성을 결정합니다.

전용 IP 주소를 확보하면 조직이 독점적으로 소유권을 가지게 되어, 해당 IP와 연결된 이메일 발송자 평판 및 전달 가능성을 완전히 제어할 수 있습니다.

이 가이드에서는 LC 이메일의 전용 IP가 무엇인지, 가격, 기능이 개발된 이유, 그리고 자주 묻는 질문들을 다룹니다.

# 언제 전용 IP 주소를 확보해야 하나요?

상당한 양의 이메일을 발송하는 경우, 월 수십만에서 수백만 통 정도(특히 20만 통을 초과하는 경우)라면 전용 IP에서 평판을 구축하는 것이 현명합니다. 전용 IP 확보에는 별도의 투자가 필요할 수 있다는 점을 고려해야 합니다.

또한 이메일 스푸핑 방지를 강화하기 위해 SPF, DMARC, DKIM, SSL 인증서를 포함한 이메일 인증 프로토콜을 설정할 때 전용 IP가 필수적입니다. 공유 IP는 같은 수준의 보안을 제공하지 않으며, 동일한 IP를 사용하는 다른 발송자의 행동으로 인해 취약점이 노출될 수 있습니다.

추가로, 전용 IP를 보유하면 잠재적인 도메인 하이재킹 위험으로부터 보호받을 수 있어 이메일 커뮤니케이션의 무결성과 보안이 보장됩니다.

## 전용 IP의 이점은 무엇인가요?

전용 IP는 공유 IP에 비해 더 높은 비용이 들 수 있지만, 여러 가지 뚜렷한 이점을 제공합니다:

- **향상된 이메일 전달**: 전용 IP는 종종 더 빠른 이메일 전달을 제공합니다. Gmail과 Yahoo 같은 유명한 이메일 서비스 제공업체들은 전용 IP에서 오는 이메일을 선호하는 경향이 있어, 메시지가 더 빨리 받은편지함에 도달합니다. 또한 대역폭을 공유하지 않으므로 동일한 IP를 사용하는 다른 사용자의 영향을 받지 않습니다.

- **발송자 및 IP 평판에 대한 완전한 제어**: 전용 IP의 유일한 사용자로서 발송 습관, IP 평판, 웹사이트의 경우 SEO 노력에 대한 완전한 제어권을 가집니다. 이를 통해 외부 간섭 없이 전략을 맞춤화할 수 있습니다.

- **효율적인 문제 해결**: 전용 IP에서 유일한 발송자가 되면 IP 관련 문제를 식별하고 해결하는 과정이 단순해집니다. 문제를 정확히 찾아내고 해결책을 구현하는 것이 더 간단해져 전반적인 서비스 안정성이 향상됩니다.

- **강화된 사이버 보안**: 전용 IP는 도메인 하이재킹, 멀웨어, 해킹 시도와 같은 위협에 대한 추가 보호층을 제공하여 온라인 리소스의 보안을 개선합니다.

결론적으로, 전용 IP는 더 비용이 많이 들 수 있지만, 성능, 제어, 문제 해결, 보안 면에서 제공하는 실질적인 이점들은 상당한 온라인 및 이메일 요구사항이 있는 조직에게 가치 있는 투자가 됩니다.

## 전용 IP로 이메일을 발송할 때 팁이 있나요?

이메일 서비스 제공업체(ESP) 세계에서는 평판이 매우 중요합니다. 전용 IP에서는 스팸 필터를 활성화하지 않도록 이메일 볼륨과 타이밍을 능숙하게 관리하는 것이 핵심입니다. 이상적으로는 이메일 발송 속도를 조절하여 24시간마다 점진적으로 볼륨을 증가시켜야 합니다.

전용 IP는 지속적으로 대량의 이메일을 발송할 때 가장 유익합니다. 그러나 산발적이고 대량의 발송은 역효과를 낳을 수 있으며, 이메일 필터에 스패머로 표시되거나 심지어 블랙리스트에 등록될 수 있습니다. ESP는 이메일 발송량의 급격한 증가를 특히 경계합니다.

이메일 트래픽이 일관되지 않게 많다면, 공유 IP가 더 유리할 수 있습니다. 이는 간헐적인 대량 발송을 동일한 IP의 다른 발송자들의 안정적인 트래픽 패턴과 일치시킵니다.

철저한 테스트와 조정을 통해 전용 IP 웜업 과정을 완벽하게 자동화하여 강력한 발송자 평판을 보장합니다. 수동으로 IP를 웜업하는 것이 부담스럽다면, 저희 솔루션이 원활한 경험을 제공합니다. 사용자는 전용 IP가 완전히 웜업될 때까지 이메일이 전용 IP와 기존 공유 IP 사이에서 50/50으로 번갈아 발송되는 전환 기간을 경험할 수 있습니다. 이 웜업 과정은 전용 IP의 최적의 전달 가능성과 평판을 보장하는 데 중요합니다. IP가 할당된 후 상당한 시간이 지났음에도 사용자가 여전히 웜업 단계에 있을 수 있다는 점을 유의하세요.

계속 변화하는 이메일 전달 영역에서는 ESP와의 뛰어난 평판을 유지하고 메시지가 의도한 수신자에게 도달하도록 보장하기 위해 세심함과 전략이 필수적입니다.

## 누가 전용 IP를 구매할 수 있나요?

**전용 IP**는 **$297/$2970 및 $497/$4970 요금제**의 **에이전시**에게만 **독점적으로 제공**됩니다. 강력한 평판을 유지하고 메일박스 제공업체의 스로틀링 문제를 줄이기 위해 **주당 최소 50,000개의 메시지** 발송을 강력히 권장합니다. 또한 월간 발송되는 메시지 100만 개당 하나의 전용 IP 주소를 할당할 것을 권장합니다. 이러한 적극적인 접근 방식은 최적의 전달 가능성을 보장하고 전반적인 이메일 성능을 향상시킵니다.

**에이전시**: 지난 14일 이내에 설립된 에이전시는 IP를 구매할 자격이 있습니다.

**하위 계정**: 
총 연락처 수가 **10,000명**을 초과하거나 **지난 10일 동안 50,000개 이상의 이메일**을 발송했으며, **지난 14일 이내에 생성된** **에이전시 유형**에 **속하는** 계정들이 IP를 구매할 자격이 있습니다.

**하위 계정에 결제 설정이 존재**: 결제 설정이 없으면 구매 프로세스가 진행되지 않습니다.

**에이전시에서 전용 IP 구매가 활성화됨**: 에이전시가 하위 계정의 전용 IP 구매를 활성화하지 않았다면 구매 프로세스가 진행되지 않습니다.

에이전시와 하위 계정 모두 LC - Email을 이메일 제공업체로 사용하는 경우 전용 IP를 구매할 수 있습니다.

## 전용 IP는 어떻게 구매하나요?

LC Email 사용자는 이메일 서비스를 통해 전용 IP를 구매할 수 있습니다.

- 에이전시 설정 페이지로 이동하여 Email Services(이메일 서비스)를 클릭합니다.
- LC 이메일 제공업체 섹션에서 Dedicated Domain And IP(전용 도메인 및 IP)를 클릭합니다.

![이메일 서비스 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155031452640/original/BvQDYf4cDhJ52G4FTdvZ60VPhNuh8A6phA.jpg?1724268225)

- 다음 페이지에서 "Buy IP"를 클릭합니다.

![IP 구매 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155009567725/original/MGcEGrG94lo1h91nalxOHYlcQ2jNRhkEBA.png?1696837840)

- 새 IP를 구매하는 옵션이 있는 모달이 나타납니다. "Buy Now" 버튼을 클릭하여 구매를 진행합니다.
- 월 $59의 IP 1개가 에이전시에 청구됩니다. 에이전시는 이 비용을 충당하기 위해 재청구를 설정할 수 있습니다.

![구매 확인 모달](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155009567791/original/JQF8N0VlbKo9vpFxWz3yw8SaFMVyf1TAMg.png?1696837899)

## 전용 IP를 관리하는 방법

전용 발송 도메인을 소유하고 있다면, 도메인이 전용 또는 공유 IP와 연결되어 있는지 나타내는 태그를 확인할 수 있습니다.

![도메인 IP 유형 표시](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155009569127/original/jKoAvC0wqCE_gqBohUVKQvjc0_r6EzdScQ.png?1696838511)

참고: 전용 IP는 전용 발송 도메인에만 연결할 수 있습니다.

또한 하나 이상의 전용 IP를 도메인에 할당할 수 있습니다. 이 구성에서 단일 도메인을 여러 IP에 연결할 수 있으며, 그 반대도 가능합니다. 도메인이 여러 IP와 연결된 경우, 이메일은 라운드 로빈 방식으로 모든 IP를 통해 발송됩니다. 이 방법은 도메인에 연결된 IP 간에 이메일 트래픽의 균형 잡힌 분산을 보장합니다.

![다중 IP 할당](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155009569325/original/IJPw9dOn_0_MuKiwiLg41ljMp-jn2Hmkvg.png?1696838572)
