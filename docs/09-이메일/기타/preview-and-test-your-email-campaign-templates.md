# 이메일 캠페인/템플릿 미리보기 및 테스트하는 방법

Hyperclass에서 이메일을 미리보기하고 테스트하면 발송 전에 레이아웃, 링크, 개인화가 올바르게 렌더링되는지 확인할 수 있습니다. 이 가이드는 미리보기 모드와 테스트 발송을 다루어 캠페인과 템플릿이 처음부터 완벽하게 보이도록 도와드립니다.

## 이메일 캠페인/템플릿 미리보기 및 테스트란?

미리보기 및 테스트는 Hyperclass의 이메일 빌더에서 콘텐츠를 시각적으로 확인하고 실제 발송 전에 안전한 테스트 복사본을 보낼 수 있는 도구를 의미합니다. 이러한 옵션은 실제 연락처나 캠페인 지표에 영향을 주지 않고 개인화, 레이아웃, 링크를 검증하는 데 도움이 됩니다. **미리보기(Preview)**를 사용하여 빌더에서 이메일이 어떻게 렌더링되는지 확인하고, **테스트 발송(Send Test)**을 사용하여 최종 인박스 확인을 위해 샘플을 내 인박스로 전달하세요.

## 이메일 미리보기 및 테스트의 주요 장점

- **정확성**: 병합 필드, 커스텀 값, 조건부 콘텐츠가 의도한 대로 표시되는지 확인합니다.
- **규정 준수**: 런칭 전에 필수 요소(수신거부 섹션 등)가 있는지 검증합니다.
- **품질 보증**: 특히 HTML을 가져온 경우 레이아웃, 이미지, 링크 문제를 조기에 발견합니다.
- **속도**: 실제 목록으로 보내거나 분석을 왜곡하지 않고 빠르게 반복합니다.

## 이메일 캠페인 미리보기 및 테스트 발송하는 방법

**참고:** 이메일 템플릿과 캠페인을 생성, 편집, 발송하는 방법을 알아보려면 [이메일 캠페인 시작하기](getting-started-launch-an-email-campaign.md) 및 [이메일 빌더와 인라인 편집기 사용법](how-to-use-the-email-builder-and-its-in-line-editor.md) 상세 가이드를 확인하세요.

- Marketing(마케팅) > Emails(이메일) > Campaigns(캠페인)로 이동합니다.
- **Campaign(캠페인)**을 클릭하여 엽니다.

![이메일 캠페인 열기](https://jumpshare.com/share/Y8i4lTrITYfFmnOYk1ut+/GIF+Recording+2025-12-03+at+6.20.08+PM.gif)

- 우측 상단의 **Preview(미리보기)** 버튼을 클릭하고 **Desktop(데스크톱)/Tablet(태블릿)/Mobile(모바일)** 뷰를 확인합니다.

![미리보기 확인](https://jumpshare.com/share/tfbIpnG6wPI5x9lzBby4+/GIF+Recording+2025-12-03+at+6.21.58+PM.gif)

- 테스트 메일을 발송하려면 Preview 버튼 옆의 **점 세 개(Three Dots)**를 클릭하고 Test Email 버튼을 클릭합니다.

![테스트 이메일 버튼](https://jumpshare.com/share/y9qA4pgUtprtzoJNyFi2+/Screen+Shot+2025-12-03+at+6.24.13+PM.png)

- **Sender Email(발신자 이메일)**을 선택하고, **Sender Name(발신자 이름)**을 추가하고, Receiver Email(수신자 이메일)을 선택하고, **Preview Text(미리보기 텍스트)**와 **Email Subject(이메일 제목)**를 추가합니다.
- **Send Test Email(테스트 이메일 발송)** 버튼을 클릭합니다.

![테스트 이메일 설정](https://jumpshare.com/share/nVETX2d4xu24bpHxvVqa+/Screen+Shot+2025-12-03+at+6.25.21+PM.png)

## 이메일 템플릿 미리보기 및 테스트하는 방법

참고: 수신거부 링크는 **테스트 이메일에서 작동하지 않습니다**.

- Marketing(마케팅) > Emails(이메일) > Templates(템플릿)로 이동합니다.
- **Template(템플릿)**을 클릭하여 엽니다.

## 자주 묻는 질문

**Q: 테스트 이메일에서 수신거부 링크가 작동하지 않는 이유는 무엇인가요?**

수신거부는 목록을 보호하기 위해 테스트에서는 의도적으로 비활성화됩니다. 설정된 플로우를 기반으로 실제 발송에서는 정상적으로 작동합니다.

**Q: 테스트 열기와 클릭이 내 분석에 영향을 주나요?**

아니요. 테스트는 검증용이며 성과 지표는 실제 발송에서 수집됩니다.

**Q: 테스트가 도착하지 않았는데 먼저 무엇을 확인해야 하나요?**

스팸/정크 폴더를 확인하고, From/서비스 설정을 검증하고, 수신자가 차단되지 않았는지 확인한 후, 다른 이메일 주소로 시도해보세요.

**Q: 커스텀 수신거부 링크가 테스트에서 작동하나요?**

표시되지만 테스트에서는 수신거부 작업을 완료하지 않습니다. 설정된 대로 실제 발송에서 작동합니다.
