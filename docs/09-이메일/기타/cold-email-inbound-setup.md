# 인바운드 이메일이란?

인바운드 이메일을 사용하면 도메인으로 전송된 이메일을 수신하고 처리할 수 있습니다. Mailgun을 사용하면 인바운드 이메일을 다음과 같이 처리할 수 있습니다:

- 다른 이메일 주소로 전달
- 웹훅으로 전송
- 워크플로우 내에서 로그 기록 및 처리

## 루트 도메인 vs 서브도메인

인바운드 이메일을 설정하기 전에 도메인 구성이 어떻게 작동하는지 이해하는 것이 중요합니다.

### 핵심 제한사항

도메인은 하나의 MX(메일 교환) 레코드 세트만 가질 수 있습니다.

이것의 의미:

- 동일한 도메인에서 Mailgun과 다른 이메일 제공업체(Gmail 또는 Outlook 등)를 동시에 사용하여 이메일을 수신할 수 없습니다.

### 실제로는 어떤 의미인가요?

| MX 레코드가 가리키는 곳 | 결과 |
|---|---|
| Mailgun | 이메일이 Mailgun에서 처리됨 (받은편지함 없음) |
| Google Workspace / Outlook | 이메일이 받은편지함으로 전송됨 (Mailgun이 수신할 수 없음) |

Mailgun은 받은편지함을 제공하지 않으며, 이메일을 라우팅하거나 전달만 합니다.

## 루트 도메인에 Mailgun MX 레코드 추가가 실패하는 이유

루트 도메인(예: yourdomain.com)이 이미 Google Workspace와 같은 이메일 제공업체에 연결되어 있는 경우:

- 기존 MX 레코드와 함께 Mailgun MX 레코드를 추가하는 것은 안정적으로 작동하지 않습니다
- 이메일 전송이 실패하거나 예측할 수 없게 작동할 수 있습니다

이는 이메일 시스템이 여러 제공업체 간에 인바운드 처리를 분할하도록 설계되지 않았기 때문입니다.

### 권장 설정 (모범 사례)

충돌을 방지하고 안정적인 이메일 전송을 보장하려면:

**Mailgun용 서브도메인 사용**

**예시:**
- Mailgun 도메인: mg.yourdomain.com
- 루트 도메인(yourdomain.com)은 Gmail 또는 Outlook에 연결된 상태 유지

**장점**
- Mailgun을 통한 이메일 발송
- 일반 받은편지함에서 이메일 수신
- MX 충돌 방지

## 루트 도메인으로 Mailgun 사용 (권장하지 않음)

루트 도메인에서 Mailgun을 구성하기로 선택하는 경우:

**반드시 해야 할 일:**
- 루트 도메인 MX 레코드를 Mailgun으로 지정

**결과:**
- 기존 받은편지함 손실 (Gmail/Outlook이 이메일 수신을 중단함)
- Mailgun 라우트를 통해서만 이메일 처리

전통적인 받은편지함이 없고, 라우팅/전달만 가능합니다.

## 인바운드 이메일 설정 단계 (Mailgun)

### 1. Mailgun 설정

[Mailgun 설정 방법](../step-by-step-guide-to-set-up-mailgun.md)을 확인하세요.

agency.com / mg.agency.com을 Mailgun으로 설정하겠습니다.

모든 하위 계정에 동일한 Mailgun 계정을 사용할 수 있습니다.

예를 들어, agency.com과 같은 도메인이 있다면 각 하위 계정에 대해 subaccountname.agency.com과 같은 고유한 서브도메인을 설정할 수 있으므로 각 하위 계정(Sub-Account)이 모든 이메일 답장을 캡처하기 위한 자체 Mailgun 서브도메인 설정을 갖게 됩니다.

고객이 자체 도메인을 보유하고 있다면 그들을 위한 고유한 도메인/서브도메인도 설정할 수 있습니다. 하지만 메인 도메인이 이미 다른 이메일 서비스에 사용되고 있다면 이 경우 서브도메인을 사용해야 합니다.

그렇게 하면 이메일을 어떤 계정으로 라우팅할지 알 수 있습니다.

### 2. Mailgun 도메인이 하나의 하위 계정에만 구성되어 있는지 확인

방금 설정한 Mailgun 도메인에 하나의 하위 계정만 매핑되어 있다면 모든 인바운드 이메일을 해당 하위 계정으로 라우팅합니다.

Agency View(에이전시 보기) > Settings(설정) > Email Services(이메일 서비스) > Location Settings(로케이션 설정)을 확인하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067377057/original/GY8CBptqw6GZALWo8rDOGdD3-MQGRkpeCw.png?1774014342)

여러 Hyperclass 하위 계정에 동일한 서브도메인/도메인을 사용하는 경우, 리드가 Hyperclass에서 발송된 이메일에 답장하는 대신 답장 이메일 주소로 직접 이메일을 보낼 때 어떤 하위 계정으로 이메일 답장을 라우팅해야 하는지 알 수 없습니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067377058/original/2QfjK-wOOm3Kl6ArJkEaGbSgqLGA_TxK9Q.png?1774014343)

하나의 하위 계정에만 도메인/서브도메인을 구성했는데도 작동하지 않는다면, Domain Services(도메인 서비스) 탭에서도 동일한 도메인이 구성되어 있는지 확인하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155067377056/original/oRExe121v81xK0RNZaBO1q1YOwmbGWN_LQ.png?1774014342)

동일한 Mailgun 도메인이 선택된 삭제된 하위 계정이 있을 수 있다고 생각되면 지원팀에 문의하세요.

### 3. 테스트

domain.com을 설정한 경우, testing@domain.com을 사용하여 수신 이메일을 캡처할 수 있습니다. 따라서 연락처가 anything@domain.com으로 이메일을 보내 대화(Conversations)를 시작하면 대화 탭에 표시됩니다.

mg.domain.com과 같은 서브도메인을 설정한 경우, anything@mg.domain.com을 사용하여 수신 이메일을 캡처할 수 있습니다.

## 설정 예시 시나리오

### 시나리오 1: 표준 설정 (권장)

- 발송 도메인: mg.yourdomain.com
- 받은편지함: yourdomain.com의 Google Workspace

발송과 수신 모두 완벽하게 작동합니다.

### 시나리오 2: Mailgun만 사용 (고급 사용 사례)

- MX → 루트 도메인의 Mailgun
- 라우트를 통한 이메일 처리
- 받은편지함 접근 불가

### 빠른 결정 가이드

| 목표 | 권장 설정 |
|---|---|
| 이메일 발송만 | Mailgun 서브도메인 사용 |
| 발송 + 받은편지함에서 수신 | 루트 → Gmail, 서브도메인 → Mailgun |
| Mailgun 라우트를 통한 수신 | MX를 Mailgun으로 지정 (받은편지함 없음) |

### 피해야 할 일반적인 실수

- 기존 MX 레코드를 제거하지 않고 Mailgun MX 레코드 추가
- 동일한 도메인에서 Gmail과 Mailgun을 인바운드용으로 동시 사용 시도
- Mailgun이 메일박스를 제공할 것이라는 기대

### 최종 권장사항

콜드 이메일 및 인바운드 라우팅을 설정할 때는 항상 Mailgun용 서브도메인을 사용하세요.

이렇게 하면:
- 책임의 명확한 분리
- 기본 받은편지함에 대한 방해 없음
- 안정적인 이메일 전송 및 라우팅

여전히 도움이 필요하시면 지원팀에 문의하거나 DNS 제공업체에 문의하여 구성을 확인하세요.

## 자주 묻는 질문

**Q. 동일한 도메인에서 Mailgun과 Gmail을 사용할 수 있나요?**

아니요. 도메인은 이메일 수신을 위한 하나의 MX 레코드 세트만 가질 수 있습니다. MX 레코드가 Gmail을 가리키면 Mailgun이 해당 도메인의 이메일을 수신할 수 없고, 그 반대도 마찬가지입니다.

**Q. 루트 도메인에 Mailgun MX 레코드를 추가해도 작동하지 않는 이유는 무엇인가요?**

도메인이 이미 MX 레코드(예: Google Workspace 또는 Outlook)를 가지고 있을 가능성이 높기 때문입니다. 이메일 시스템은 여러 제공업체 간에 인바운드 이메일을 안정적으로 분할할 수 없어 실패나 일관성 없는 전송을 야기합니다.

**Q. Mailgun을 사용해서 루트 도메인에서 이메일을 발송할 수 있나요?**

네. 이메일 발송은 MX 레코드의 영향을 받지 않습니다. 루트 도메인이 다른 이메일 제공업체에 연결되어 있어도 Mailgun을 사용해서 발송할 수 있습니다.

**Q. Mailgun이 받은편지함을 제공하나요?**

아니요. Mailgun은 메일박스 인터페이스를 제공하지 않습니다. 라우트나 웹훅을 통해 수신 이메일을 라우팅, 전달 또는 처리만 합니다.

**Q. 콜드 이메일과 답장에 권장되는 설정은 무엇인가요?**

Mailgun용 서브도메인(예: mg.yourdomain.com)을 사용하고 이메일 수신을 위해서는 루트 도메인을 Gmail 또는 Outlook에 연결된 상태로 유지하세요.

**Q. 인바운드 이메일용으로 Mailgun과 루트 도메인을 사용할 수 있나요?**

네, 하지만 MX 레코드를 Mailgun으로 지정해야 합니다. 이렇게 하면 Mailgun이 모든 수신 이메일을 처리하므로 기존 받은편지함(Gmail/Outlook)이 비활성화됩니다.

**Q. Mailgun에서 서브도메인 사용을 권장하는 이유는 무엇인가요?**

서브도메인을 사용하면 메인 받은편지함과의 충돌을 방지하고, 도메인 평판을 보호하며, 안정적인 이메일 전송을 보장합니다.
