# 이메일 빌더: 다른 플랫폼 템플릿을 코드 에디터로 가져오기

이제 어떤 플랫폼에서든 사용하던 이메일 템플릿을 Hyperclass **코드 에디터(Code Editor)**로 쉽게 가져올 수 있습니다. 다른 플랫폼에서 이전하거나 기존 작업을 재사용하고 싶을 때, 시스템에서 생성한 이메일 주소로 테스트 이메일을 보내기만 하면 템플릿을 가져와서 편집할 수 있어요.

## **이메일 템플릿 가져오기 기능이란?**

이메일 템플릿 가져오기 기능을 사용하면 어떤 플랫폼에서 만든 기존 이메일 템플릿이든 상관없이 Hyperclass 계정으로 손쉽게 이전할 수 있습니다. Hyperclass에서 제공하는 동적 이메일 주소로 테스트 이메일이나 캠페인 이메일을 보내면, 템플릿이 자동으로 **코드 에디터**에 나타나며 필요에 따라 수정하고 재사용할 수 있어요.

이 도구를 사용하면 시간을 절약하고 원본 디자인을 보존할 수 있어서, 처음부터 다시 만들거나 이메일 형식을 수동으로 재현할 필요가 없습니다.

## **다른 플랫폼에서 템플릿 가져오기의 주요 장점**

이 기능은 다른 이메일 플랫폼에서 넘어오는 사용자들의 이전 과정을 단순화하고 템플릿 관리를 개선하도록 설계되었습니다.

- ✅ **간단한 온보딩** – Hyperclass로 템플릿을 빠르게 가져올 수 있어요
- 🎨 **템플릿 보존** – 원본 디자인, 레이아웃, 코드를 그대로 유지합니다
- 🔄 **쉬운 재사용** – 다시 만들 필요 없이 바로 편집을 시작하세요
- 🛠️ **유연한 편집** – **코드 에디터** 안에서 완전히 커스터마이징 가능합니다
- 🌐 **플랫폼 제약 없음** – 테스트 또는 캠페인 이메일을 지원하는 모든 플랫폼과 호환됩니다
- ⚡️ **빠르고 자동** – 이메일 전송 후 몇 초 안에 템플릿이 나타납니다

## **코드 에디터로 이메일 템플릿 가져오는 방법**

다음 단계를 따라 기존 템플릿을 Hyperclass의 코드 에디터로 직접 가져오세요:

- `Marketing(마케팅) → Email Marketing(이메일 마케팅) → Templates(템플릿)`으로 이동합니다
- **New(새로 만들기)**를 클릭한 다음 **Import(가져오기)**를 선택합니다
- 가져오기 메뉴에서 목록에서 **Other Platform(다른 플랫폼)**을 선택합니다
- 화면에 표시되는 **시스템에서 생성한 이메일 주소**를 복사합니다
- 원래 플랫폼(예: Mailchimp, Klaviyo)에서 해당 주소로 템플릿을 **테스트 이메일** 또는 **캠페인 이메일**로 보냅니다
- Hyperclass로 돌아오면 이메일이 가져와져서 **Templates(템플릿)** 영역에 목록으로 표시됩니다
- 가져온 템플릿을 클릭하면 **코드 에디터**에서 열립니다

이제 가져온 템플릿을 편집, 복사하거나 이메일 캠페인에 할당할 수 있습니다.

## **코드 에디터**

코드 에디터를 사용하면 이메일의 구조, 스타일링, 동적 콘텐츠를 완전히 제어할 수 있습니다. 가져온 템플릿은 이 에디터에 직접 나타나므로 다음과 같은 유연한 작업이 가능해요:

- HTML 및 인라인 CSS 커스터마이징
- 개인화 변수 추가
- 실시간으로 변경 사항 미리보기
- 재사용을 위한 템플릿 저장
- 캠페인과 자동화에 디자인 적용

코드 작업에 익숙하거나 개발자와 함께 작업한다면, 코드 에디터는 정밀한 커스터마이징을 위한 최고의 도구입니다.

## **자주 묻는 질문**

**Q: 어떤 플랫폼에서 템플릿을 가져올 수 있나요?**

테스트 이메일이나 캠페인을 보낼 수 있는 **모든 이메일 플랫폼**에서 가져올 수 있습니다. Mailchimp, Klaviyo, ConvertKit, ActiveCampaign, Constant Contact 등이 포함됩니다.

**Q: 원본 이메일의 형식과 이미지가 그대로 유지되나요?**

네. HTML 코드와 디자인 요소가 원본 전송에서 나타나는 그대로 정확히 보존됩니다.

**Q: 가져온 템플릿을 바로 캠페인에서 사용할 수 있나요?**

물론입니다. 가져온 후에는 모든 이메일 캠페인이나 자동화에서 바로 사용할 수 있어요.

**Q: 다른 플랫폼에서 원본 템플릿을 업데이트하면 어떻게 되나요?**

같은 시스템 생성 이메일 주소로 이메일을 다시 보내서 업데이트된 버전을 다시 가져올 수 있습니다.

**Q: 일반 텍스트 이메일도 작동하나요?**

네, 하지만 **코드 에디터**는 구조화된 콘텐츠가 있는 HTML 기반 이메일에 가장 적합합니다.

**Q: 가져온 템플릿은 Hyperclass 계정의 어디에 나타나나요?**

수동으로 만든 템플릿과 마찬가지로 `Marketing(마케팅) → Email Marketing(이메일 마케팅) → Templates(템플릿)` 아래에 표시됩니다.
