## 현재 상황

여러 이메일 전송 문제가 동시에 발생하여, 다양한 요소가 복합적으로 작용해 이메일이 차단되는 복잡한 상황을 겪고 계십니다. 이는 일반적으로 기본적인 이메일 인프라 문제가 시간이 지나면서 누적되어, 발신자 평판 저하, 인증 실패, 콘텐츠 관련 차단이 모두 동시에 발생할 때 나타납니다.

## 빠른 진단: 다중 문제 식별

### 일반적인 반송 메시지 조합

여러 가지 서로 다른 문제들이 동시에 나타납니다: 수신자 요금 제한, 낮은 도메인 평판, 스팸성 콘텐츠, PTR 레코드 누락, DMARC/인증 실패 등이 복합적으로 발생합니다.

---

## 다중 이메일 전송 문제 이해

### 핵심 개념

**중요한 이해사항:**

- 다중 문제는 복합적 효과를 만들어냄 - 각 문제가 다른 문제들을 더욱 악화시킴
- 복구를 위해서는 올바른 우선순위로 문제들을 해결해야 함
- 일부 수정 작업은 다른 작업이 효과를 발휘하기 전에 완료되어야 함
- 다중 문제가 있을 때는 해결 기간이 상당히 길어짐
- 완전한 해결 없이 부분적 수정만 하면 전체적인 평판이 더욱 악화될 수 있음

### 문제 카테고리 및 영향도

**인프라 문제 (최우선 순위):**
- PTR 레코드 누락 또는 잘못된 설정
- DMARC 인증 실패
- SPF 및 DKIM 설정 오류

**평판 문제 (높은 우선순위):**
- 낮은 도메인 평판 점수
- IP 평판 문제
- 신고로 인한 수신자 요금 제한

**콘텐츠 문제 (중간 우선순위):**
- 스팸성 콘텐츠 패턴
- 트리거 단어 및 구문
- 텍스트 대 이미지 비율 불량

## 단계별 다중 문제 해결

### 1단계: 인프라 기반 구축 (1주차)

**우선순위 - 2단계로 넘어가기 전에 모든 작업 완료:**

**인증 수정:**
- `Settings(설정) → Email Services(이메일 서비스) → Sending Domain & IP(발송 도메인 및 IP)`로 이동
- 모든 DNS 레코드가 "Verified(확인됨)" 상태인지 확인
- "Pending(대기 중)" 또는 "Failed(실패)" 상태가 있다면, DNS 제공업체에서 DNS 레코드 업데이트
- DNS 전파를 위해 24-48시간 대기

**PTR 레코드 설정:**
- 호스팅 제공업체 또는 IT 관리자에게 연락
- 발송 IP에 대한 PTR 레코드 설정 요청
- MXToolbox PTR 조회 도구를 사용하여 확인

**DMARC 정책 설정:**
- DNS에 DMARC 레코드 추가: "v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.com"
- 인증 실패에 대한 DMARC 보고서 모니터링

#### 인프라 성공 지표

- 모든 DNS 레코드가 "Verified(확인됨)" 상태 표시
- PTR 레코드가 MXToolbox에서 올바르게 해결됨
- DMARC 보고서에서 인증 통과 표시
- "authentication failure(인증 실패)" 반송 메시지 더 이상 없음

### 2단계: 콘텐츠 최적화 (2주차)

**콘텐츠 문제 정리:**

**이메일 템플릿 검토:**
- `Marketing(마케팅) → Email Templates(이메일 템플릿)`으로 이동
- 모든 활성 템플릿에서 스팸 트리거 단어 검토
- 과도한 대문자 사용, 여러 개의 느낌표 제거
- 적절한 텍스트 대 이미지 비율 보장 (텍스트 80%, 이미지 20%)

**콘텐츠 품질 테스트:**
- Mail-Tester.com을 사용하여 이메일 점수 측정
- 8/10 이상의 점수 목표
- 플래그된 모든 콘텐츠 문제 해결

### 3단계: 평판 복구 (3-8주차)

**체계적인 평판 재구축:**

**점진적 발송량 증가 구현:**
- 가장 활발히 참여하는 연락처에게 하루 50-100통의 이메일로 시작
- 전송률이 개선되는 경우에만 매주 25%씩 증가
- 반송률과 스팸 신고를 면밀히 모니터링

**평판 지표 모니터링:**
- senderscore.org에서 주간 발신자 점수 확인
- Talos Intelligence 평판 모니터링
- MXToolbox 블랙리스트 모니터링 사용

### 4단계: 목록 정리 및 재참여 유도 (지속적)

**연락처 목록 정리 및 최적화:**

**목록 세분화:**
- `Contacts(연락처) → Smart Lists(스마트 리스트)`로 이동
- 참여도 수준에 따라 세그먼트 생성
- 초기 발송은 높은 참여도의 연락처에게만 집중

**문제 연락처 제거:**
- 다중 반송이나 신고가 있는 연락처 식별
- 이러한 연락처를 캠페인에서 제거하거나 억제
- 신규 구독자에 대해 이중 옵트인 구현

---

## 복구 일정 및 예상 결과

### 1단계: 인프라 (1-2주차)

- **작업:** 모든 인증 및 DNS 수정 완료
- **예상 결과:** 인증 관련 반송 제거
- **성공 지표:** 모든 DNS 레코드 확인됨

### 2단계: 콘텐츠 및 초기 복구 (3-4주차)

- **작업:** 콘텐츠 정리 및 제한적 발송량 테스트
- **예상 결과:** 콘텐츠 관련 차단 감소
- **성공 지표:** Mail-Tester 점수 8/10 이상

### 3단계: 평판 구축 (5-12주차)

- **작업:** 참여도 높은 연락처에게 점진적 발송량 증가
- **예상 결과:** 전송률의 꾸준한 개선
- **성공 지표:** 발신자 점수 80 이상, 전송률 95% 이상

### 4단계: 완전 복구 (13-16주차)

- **작업:** 정상 발송량으로 복귀
- **예상 결과:** 지속적인 높은 전송률
- **성공 지표:** 98% 이상의 전송률 지속

---

## 복구 중 필수 모니터링

### 일일 모니터링 (첫 4주)

- **분석:** 반송률과 전송 통계 확인
- **반송 메시지 분석:** 반송 사유 변화를 분류하고 추적
- **신고 모니터링:** 스팸 신고 증가 관찰

### 주간 모니터링 (지속적)

- **발신자 점수:** 평판 점수 개선 추적
- **블랙리스트 상태:** 주요 블랙리스트 데이터베이스 모니터링
- **DMARC 보고서:** 인증 성공률 검토

## 주의해야 할 경고 신호

#### 다음 사항이 보이면 즉시 발송 중단:

- 2일 연속 반송률 5% 이상
- 새로운 블랙리스트 등재 나타남
- 스팸 신고율 0.1% 이상
- 발신자 점수가 현재 수준 이하로 하락
- 주요 ISP에서 임시 지연 구현

## 문제가 계속 발생하나요?

이 체계적인 접근 방식을 따른 후에도 다중 전송 문제가 계속 발생한다면:

- **모든 것을 문서화:** 모든 반송 메시지, 평판 점수, 수행된 작업에 대한 자세한 기록 보관
- **전문가 평가 고려:** 다중 문제는 종종 숨겨진 문제를 식별하기 위한 전문가 분석이 필요
- **인프라 평가:** 일부 상황에서는 전용 IP나 고급 설정이 필요할 수 있음
