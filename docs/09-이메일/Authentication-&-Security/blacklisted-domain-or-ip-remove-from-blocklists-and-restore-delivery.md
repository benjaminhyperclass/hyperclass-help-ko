# 블랙리스트 URL/IP/도메인 이메일 차단 해결하기

## 어떤 상황인가요?

이메일 캠페인의 한 개 이상 구성 요소가 공개 블랙리스트(RBL/DBL)에 신고되어 이메일이 차단되고 있습니다. 발송 IP 주소, 도메인, 또는 이메일 콘텐츠에 포함된 URL이 문제일 수 있습니다. 이메일 제공업체들은 스팸 방어의 첫 번째 수단으로 이러한 블랙리스트를 사용하며, 목록에 등재되면 여러 플랫폼에서 이메일 전달률이 심각하게 영향을 받을 수 있습니다.

## 빠른 진단: 블랙리스트 문제 식별하기

### 반송 메시지 예시

- 공개 블랙리스트(RBL/DBL)에 등재된 URL 또는 IP/도메인으로 인해 이메일이 차단됨

---

## 블랙리스트 차단 이해하기

### 핵심 개념

**중요 정보:**

- 블랙리스트는 의심스러운 IP, 도메인, URL을 추적하는 제3자 데이터베이스입니다
- 주요 이메일 제공업체(Gmail, Outlook, Yahoo)는 이메일을 수락하기 전에 여러 블랙리스트를 확인합니다
- 블랙리스트 등재는 특정 캠페인이 아닌 계정의 모든 이메일에 영향을 미칩니다
- 근본적인 문제를 해결한 후에도 블랙리스트 제거에는 24-72시간이 걸릴 수 있습니다
- 일부 블랙리스트는 일정 기간 건전한 행동을 유지하면 자동으로 항목을 제거합니다

### 일반적인 블랙리스트 유형

**IP 기반 블랙리스트(RBL):**

- Spamhaus SBL/CSS/XBL/PBL - 가장 널리 사용됨
- Barracuda Reputation Block List
- SURBL - 스팸 URL에 중점
- SpamCop - 커뮤니티 주도 신고

**도메인/URL 블랙리스트(DBL):**

- Spamhaus DBL - 도메인 평판
- URIBL - URL 평판 추적
- SURBL - 스팸 URL 실시간 차단 목록
- Google Safe Browsing - 악성 코드/피싱 보호

## 단계별 블랙리스트 해결 방법

### 1단계: 블랙리스트 등재 대상 식별

**발송 인프라 확인:**

- 발송 IP 찾기:
  - `Settings(설정) → Email Services(이메일 서비스) → Sending Domain and IP(발송 도메인 및 IP)`로 이동
  - 구성된 발송 도메인과 표시된 IP 정보를 기록
  - 공유 인프라를 사용하는 경우 IP 세부 정보는 지원팀에 문의
- 외부 블랙리스트 검사기 사용:
  - MXToolbox.com → Blacklist Check 방문
  - 발송 IP 주소 입력
  - 발송 도메인을 별도로 확인
  - 캠페인에서 자주 사용하는 URL 테스트

### 2단계: 이메일 콘텐츠 분석

**최근 캠페인 검토:**

- 캠페인 기록 접근:
  - `Marketing(마케팅) → Emails(이메일) → Campaigns(캠페인)`으로 이동
  - 지난 7-14일 동안 발송된 캠페인 검토
  - 새로운 URL, 도메인, 단축 링크 찾기
- 문제가 있는 콘텐츠 확인:
  - 단축 URL(bit.ly, tinyurl 등)
  - 새로 등록된 도메인
  - 의심스러운 확장자(.tk, .ml, .ga)를 가진 URL
  - 파일 공유 서비스로의 링크

### 3단계: 목록 삭제 요청 제출

**블랙리스트 운영자에게 연락:**

- Spamhaus 목록의 경우:
  - spamhaus.org → SBL/CSS/PBL Removal 방문
  - IP/도메인을 입력하고 삭제 절차 따르기
  - 개선 조치에 대한 자세한 설명 제공
- 기타 주요 블랙리스트의 경우:
  - Barracuda: barracudacentral.org/rbl/removal-request
  - SpamCop: spamcop.net → Reporting → Remove
  - SURBL: surbl.org → Whitelist Request

#### 성공적인 목록 삭제 지표

- 블랙리스트 검사기에서 "등재되지 않음" 상태 표시
- 테스트 이메일이 반송되지 않고 받은 편지함에 도달
- 반송률이 정상 수준(2% 미만)으로 돌아옴
- 이메일 전달 보고서에서 개선된 수락률 표시

### 4단계: 예방 조치 구현

**이메일 관행 강화:**

- 이메일 템플릿 업데이트:
  - `Marketing(마케팅) → Emails(이메일) → Templates(템플릿)`으로 이동
  - 신고된 URL이나 도메인 제거
  - 단축 URL을 완전하고 신뢰할 수 있는 링크로 교체
  - 명확한 수신 거부 링크와 실제 주소 추가
- 목록 위생 개선:
  - `Contacts(연락처) → Smart Lists(스마트 리스트)`로 이동
  - 참여도가 높은 구독자만으로 세그먼트 생성
  - 반송 및 비참여 연락처 정기적으로 제거

## 복구 일정과 예상 결과

### 1단계: 즉시 조치(0-24시간)

- **조치:** 블랙리스트 등재 구성 요소 식별 및 삭제 요청 제출
- **예상 결과:** 블랙리스트 상태 확인 및 삭제 요청 제출 확인

### 2단계: 목록 삭제 과정(24-72시간)

- **조치:** 블랙리스트 상태 모니터링 및 삭제 요청 후속 조치
- **예상 결과:** 블랙리스트에서 점진적으로 제거, 전달률 개선

### 3단계: 평판 복구(1-4주)

- **조치:** 깨끗한 발송 관행 유지 및 전달 가능성 모니터링
- **예상 결과:** 이메일 전달 가능성 및 발신자 평판 완전 복구

## 지속적인 모니터링과 예방

### 주간 모니터링 루틴

- **블랙리스트 상태 확인:**
  - MXToolbox.com 주간 블랙리스트 모니터링 사용
  - 새로운 등재에 대한 알림 설정
  - senderscore.org에서 Sender Score 모니터링
- **지표 검토:**
  - `Reporting(리포팅) → Email Analytics(이메일 분석)`으로 이동
  - 반송률 모니터링(2% 미만 유지)
  - 갑작스러운 전달률 하락 감시

### 예방을 위한 모범 사례

**콘텐츠 가이드라인:**

- 가능한 한 모든 링크에 자체 도메인 사용
- 절대 필요한 경우가 아니면 URL 단축기 피하기
- 캠페인에 포함하기 전에 모든 URL 테스트
- 일관된 발송 패턴과 볼륨 유지
- 명확한 발신자 식별 및 수신 거부 옵션 포함

## 여전히 문제가 있나요?

블랙리스트 관련 전달 문제가 계속되는 경우:

- **모든 것을 문서화:** 모든 목록 삭제 요청과 응답을 기록으로 보관
- **전용 IP 고려:** 가능한 경우 전용 IP 옵션 탐색
- **발송 관행 검토:** 규정 준수 문제에 대한 전체 이메일 전략 감사
- **전문가 도움 받기:** 복잡한 블랙리스트 문제에는 종종 전문가 개입이 필요
