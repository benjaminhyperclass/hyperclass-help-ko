## 무슨 일이 일어나고 있나요?

수신자 서버가 귀하의 발송 도메인이나 IP 주소에서 메시지를 받아들이는 속도를 제한하고 있어서 이메일이 일시적으로 지연되거나 거부되고 있습니다. 이는 이메일 제공업체가 스팸을 방지하고 서버 부하를 관리하기 위해 사용하는 보호 조치입니다. 하지만 발송량이 급증하거나 평판이 떨어질 때는 정당한 발송자에게도 영향을 줄 수 있습니다.

## 빠른 진단: 속도 제한 문제 식별

### 일반적인 속도 제한 반송 메시지

- "수신자 서버가 높은 볼륨이나 정책으로 인해 일시적으로 수신 메시지를 제한하거나 지연시키고 있습니다"
- "발송 서버가 높은 볼륨이나 낮은 평판으로 인해 일시적으로 제한되었습니다"
- "수신자가 너무 빠르게 너무 많은 이메일을 받고 있어서 추가 메시지가 일시적으로 차단되었습니다"
- "야후가 높은 볼륨이나 사용자 신고로 인해 메시지를 일시적으로 지연시키고 있습니다"
- "수신자 메일함이 시간당 메시지 수신 한도에 도달했습니다"
- "수신 서버가 너무 많은 배달 시도로 인해 메시지를 지연시키거나 거부했습니다"
- "수신자 서버가 높은 발송량으로 인해 메시지를 일시적으로 제한하거나 거부하고 있습니다"
- "발송 서버가 높은 볼륨이나 사용자 신고로 인해 메시지를 일시적으로 지연시키고 있습니다"

## 속도 제한 및 스로틀링 이해

### 핵심 개념

**속도 제한 기본 사항:**

- **일시적 보호:** 속도 제한은 보통 일시적 제한이며 영구 차단이 아닙니다
- **볼륨 기반:** 같은 제공업체에 너무 빨리 너무 많은 이메일을 보낼 때 발생합니다
- **평판 영향:** 발송자 평판이 낮을수록 더 엄격한 속도 제한이 적용됩니다
- **제공업체별 차이:** 각 이메일 제공업체(Gmail, Yahoo, Outlook)마다 임계값이 다릅니다
- **복구 가능:** 대부분의 속도 제한은 적절한 조정으로 몇 시간에서 며칠 내에 해결됩니다
- **자동 반송 처리:** 저희 플랫폼이 자동으로 유효하지 않은 주소를 표시하여 리스트 품질을 유지합니다

### 속도 제한 카테고리

**볼륨 기반 스로틀링:**
- 짧은 시간 내에 너무 많은 이메일 발송
- 시간당 또는 일일 발송 한도 초과
- 발송량의 급작스러운 급증

**평판 기반 제한:**
- 낮은 발송자 점수로 인한 보수적 한도 적용
- 최근 신고나 반송률 증가
- 제한된 기록을 가진 새로운 발송 도메인

**수신자 수준 한도:**
- 개별 메일함 수신 한도 초과
- 도메인 전체의 수신 메시지 제한

## 단계별 속도 제한 해결 방법

### 1단계: 배치 발송 전략 구현

배치 발송 기능을 사용하여 이메일 볼륨을 조절하세요:

- **캠페인 생성 접근:**
  `Marketing(마케팅) → Campaigns(캠페인) → Create Campaign(캠페인 만들기)`로 이동
- Email Campaign(이메일 캠페인) 유형 선택
- 이메일 템플릿과 대상자 선택
- **배치 발송 설정:**
  - 캠페인 설정에서 Scheduling Options(예약 옵션) 찾기
  - Batch Sending(배치 발송) 또는 Schedule in Batches(배치로 예약) 선택
  - 보수적인 배치 크기 설정: 배치당 50-100개 이메일로 시작
  - 처음에는 배치 간 2-4시간 간격 설정
- **기존 대용량 캠페인 일시정지:**
  - 활성 캠페인으로 이동하여 대량 발송 중인 캠페인 일시정지
  - 배치 발송 형식으로 변환
  - 필수 커뮤니케이션만 우선순위로 설정

#### 배치 발송 성공 지표

- "대용량" 또는 "너무 빠르게"를 언급하는 반송 메시지 감소
- 각 배치 내에서 전달률 개선
- 즉각적인 거부 없이 원활하게 처리되는 이메일
- 배치 이메일 가이드 참조: [링크는 원문 유지]

### 2단계: 외부 평판 지표 모니터링

저희 플랫폼에는 내장 평판 모니터링이 없으므로 다음 외부 도구를 사용하세요:

- **발송자 점수 확인:**
  - senderscore.org 방문
  - 발송 도메인 또는 IP 주소 입력
  - 70 미만 점수는 속도 제한에 기여하는 평판 문제를 나타냅니다
- **IP 평판 확인:**
  - mxtoolbox.com/blacklists.aspx 사용
  - 발송 IP가 블랙리스트에 나타나는지 확인
  - "정책" 또는 "평판" 관련 리스팅 모니터링
- **제공업체별 도구:**
  - **Gmail:** Google Postmaster Tools에서 평판 데이터 확인
  - **Microsoft:** Smart Network Data Services (SNDS) 검토
  - **Yahoo:** Yahoo Sender Hub 피드백 모니터링

### 3단계: 자동 반송 처리 활용

저희 플랫폼이 자동으로 반송된 주소를 관리하여 발송자 평판을 개선합니다:

- **반송 리포트 검토:**
  - `Marketing(마케팅) → Email(이메일) → Reports(보고서)`로 이동
  - 최근 캠페인의 반송률과 유형 확인
  - 하드 반송이 자동으로 무효로 표시되는 것을 확인
- **무효 연락처 관리 확인:**
  - `Contacts(연락처) → All Contacts(전체 연락처)`로 이동
  - Invalid(무효) 또는 Bounced(반송) 상태로 필터링
  - 이러한 연락처가 향후 캠페인에서 제외되는지 확인
- **참여도 높은 세그먼트에 집중:**
  - `Contacts(연락처) → Smart Lists(스마트 리스트)`로 이동
  - 최근 이메일 참여도 기반 세그먼트 생성
  - 가장 참여도가 높은 연락처에 우선적으로 배치 발송

### 4단계: 발송 도메인 인증 확인

평판 개선을 위해 적절한 도메인 인증을 확인하세요:

- **도메인 설정 확인:**
  - `Settings(설정) → Email Services(이메일 서비스) → Sending Domain(발송 도메인)`으로 이동
  - SPF, DKIM, DMARC 레코드가 확인됨으로 표시되는지 확인
  - 녹색 체크 마크 또는 "Verified(확인됨)" 상태 확인
- **필요시 DNS 레코드 업데이트:**
  - 확인되지 않은 DNS 레코드 복사
  - 도메인의 DNS 설정에 추가
  - 전파 및 재확인을 위해 24-48시간 대기

## 복구 타임라인 및 기대사항

### 1단계: 즉시 완화 (2-6시간)

- **조치:** 보수적인 볼륨으로 배치 발송 전환
- **예상 결과:** 반송률 감소, "너무 빠르게" 메시지 감소
- **성공 지표:** 속도 제한 반송 50% 감소

### 2단계: 단기 안정화 (1-3일)

- **조치:** 참여도 높은 연락처와 함께 점진적으로 배치 크기 증가
- **예상 결과:** 각 배치 내에서 일관된 전달
- **성공 지표:** 높은 참여 세그먼트에서 80% 이상 전달률

### 3단계: 완전 복구 (1-2주)

- **조치:** 평판 모니터링과 함께 더 큰 배치 크기로 복귀
- **예상 결과:** 모든 세그먼트에서 정상 전달률
- **성공 지표:** 속도 제한 반송 2% 미만

## 예방 모범 사례

### 스마트 배치 관리

- **보수적 시작:** 작은 배치(50-100개 이메일)로 시작하여 점진적으로 증가
- **일관된 타이밍:** 업무 시간 내에 배치를 균등하게 배치
- **제공업체 인식:** Yahoo 등 엄격한 제공업체에는 더 작은 배치 고려
- **참여도 우선순위:** 각 배치에서 가장 참여도 높은 연락처에게 먼저 발송

### 지속적인 평판 유지

- **주간 모니터링:** 주간 발송자 점수 및 블랙리스트 상태 확인
- **반송률 추적:** 캠페인 보고서에서 반송률 모니터링
- **시스템 신뢰:** 자동 반송 처리가 리스트 품질을 유지하도록 하기
- **콘텐츠 품질:** 스팸 트리거 단어를 피하고 전문적인 형식 유지

## 여전히 문제가 있나요?

배치 발송 구현 후에도 계속 속도 제한 문제가 발생하면:

- **배치 크기 추가 감소:** 더 긴 간격으로 배치당 25-50개 이메일 시도
- **제공업체별 세그먼트:** Gmail, Yahoo, Outlook 수신자를 위한 별도 배치 생성
- **콘텐츠 검토:** 공격적인 필터링을 유발할 수 있는 이메일 콘텐츠 요소 분석
- **타이밍 고려:** 제공업체가 가장 제한적인 피크 발송 시간 피하기

### 전문적인 속도 제한 해결이 필요하신가요?

속도 제한 문제는 기본적인 배치 발송을 넘어서는 정교한 평판 관리와 제공업체별 전략이 필요한 경우가 많습니다.

다음과 같은 전문적인 지원을 받으세요:

- 고급 IP 워밍 및 평판 복구
- 제공업체별 속도 제한 협상
- 정교한 배치 발송 최적화
- 엔터프라이즈급 전달 가능성 전략
- 맞춤형 모니터링 및 알림 시스템

속도 제한이 이메일 마케팅 ROI를 해치지 않도록 하세요 - 지금 전문가의 도움을 받으세요!
