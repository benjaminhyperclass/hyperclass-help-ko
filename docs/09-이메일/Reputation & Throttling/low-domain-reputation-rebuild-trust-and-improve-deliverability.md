## 무엇이 일어나고 있나요?

귀하의 도메인이 이메일 제공업체와 보안 서비스에서 나쁜 평판을 갖게 되어 이메일이 차단되거나 지연되거나 스팸 폴더로 전송되고 있습니다. 도메인 평판은 이메일 발송에 대한 신용점수와 같습니다. 발송 관행, 수신자 참여도, 신고율을 기반으로 시간이 지나면서 형성됩니다.

도메인 평판이 낮으면 Gmail, Outlook, Yahoo와 같은 주요 이메일 제공업체가 사용자를 잠재적인 스팸이나 악성 콘텐츠로부터 보호하기 위해 자동으로 메시지를 거부하거나 격리시킵니다.

## 빠른 진단: 평판 문제 식별하기

도메인 평판 문제를 나타내는 다음 반송 메시지를 확인하세요:

### Gmail 평판 차단

- "The sending domain has a very low reputation, causing Gmail to block the message as potentially suspicious or spam"
- "The sending domain has a poor reputation, causing Gmail to block or defer messages as potential spam"  
- "The sending domain has a very low reputation, causing Gmail to block the message as suspicious or spam-like"

### 일반적인 평판 문제

- "The sending domain has a poor reputation, causing the recipient's mail system to reject the message"

### 블랙리스트 관련 문제

- "The sending domain is blacklisted by Spamhaus, causing email rejection"
- "The sending domain is listed on industry blocklists, causing emails to be rejected for potential spam or abuse"
- "The sending domain is blacklisted by reputation services like Spamhaus or Abusix"

## 도메인 평판 이해하기

### 도메인 평판에 영향을 주는 요소는?

**도메인 나이:** 신규 도메인(3개월 미만)은 ISP에서 의심스럽게 보므로 적절한 워밍업이 필요합니다.

**인증 상태:** SPF, DKIM, DMARC 레코드가 없거나 잘못 구성되면 평판에 심각한 손상을 줍니다.

**참여 지표:** 낮은 열람률, 높은 반송률, 스팸 신고는 평판 점수에 직접적인 영향을 미칩니다.

**발송 패턴:** 갑작스런 볼륨 증가, 불규칙한 발송 일정, 부실한 목록 관리는 평판을 해칩니다.

**콘텐츠 품질:** 스팸 같은 콘텐츠, 오해의 소지가 있는 제목, 부실한 포맷팅은 평판 페널티를 발생시킵니다.

### 도메인 평판 작동 방식

**도메인 평판을 0-100 범위의 신용점수로 생각하세요:**

- **85-100:** 우수한 평판 - 높은 인박스 배치율
- **70-84:** 좋은 평판 - 대부분 인박스, 일부 프로모션 탭
- **50-69:** 보통 평판 - 인박스/스팸 혼재 배치
- **50 미만:** 나쁜 평판 - 대부분 스팸 폴더 또는 차단

## 단계별 평판 복구

### 단계 1: 도메인 인증 상태 확인

- 계정에 로그인
- Settings(설정) → Email Services(이메일 서비스) → Dedicated Domain and IP(전용 도메인 및 IP)로 이동
- 도메인에 대한 Verify Domain(도메인 인증) 클릭
- 6개 항목이 모두 제대로 구성되었는지 확인
- 실패한 항목이 있으면 제공된 DNS 설정 지침을 따르세요
- DNS 전파를 위해 2분 대기

### 단계 2: 외부 도구를 사용하여 평판 모니터링

**필수 평판 확인 도구:**

- **Google Postmaster Tools** - postmaster.google.com에서 설정
  - Gmail에 대한 도메인 평판, 스팸 비율, 전달 오류 표시
  - IP 평판 및 인증 데이터 제공
  - 대부분의 이메일 목록에서 Gmail이 60% 이상을 차지하므로 필수

- **Sender Score** - senderscore.org 방문
  - 무료 평판 점수(0-100)
  - 도메인을 입력하여 평판 점수 확인
  - 70점 미만은 평판 문제를 나타냄

- **MXToolbox Blacklist Check** - 포괄적인 블랙리스트 모니터링

### 단계 3: 현재 성과 분석

- Marketing(마케팅) → Email(이메일) → Statistics(통계)로 이동
- 최근 캠페인 성과 검토:
  - **전달률:** 95% 이상이어야 함 (낮으면 평판 문제 가능성)
  - **열람률:** 20% 이상이어야 함 (낮으면 스팸 폴더 배치 가능성)
  - **반송률:** 2% 미만이어야 함 (높으면 목록 품질 문제)
  - **수신 거부율:** 2% 미만이어야 함 (높으면 콘텐츠/타겟팅 문제)

- 핵심 지표: 전달률은 떨어지는데 반송률이 증가하지 않는다면, 평판 문제로 인해 이메일이 스팸 폴더로 가고 있을 가능성이 높습니다.
- Settings(설정) → Email Service(이메일 서비스) → Bounce Classification(반송 분류)로 이동
- 반송 분류 패턴 확인 - 평판 문제는 특정 오류 메시지를 표시합니다

### 단계 4: 인박스 배치 테스트 수행

직접적인 평판 점수를 얻기 어려우므로 실제 인박스 배치를 테스트하세요:

- **수동 시드 테스트:**
  - Gmail, Outlook, Yahoo, Apple Mail에서 테스트 계정 생성
  - 이 계정들에 테스트 캠페인 발송
  - 인박스 대 스팸 폴더 배치 확인
  - 제공업체별 배치율 문서화

- **서드파티 테스트 도구:**
  - Mail-Tester.com - 무료 스팸 점수 테스트
  - GlockApps - 인박스 배치 테스트
  - EmailOnAcid - 전달성 테스트
  - 250ok (현재 Validity) - 전문 전달성 모니터링

### 단계 5: 블랙리스트 등재 확인 및 제거

- 위에서 언급한 블랙리스트 검사기를 사용하여 등재 식별
- 등재된 각 블랙리스트에 대해 제거 페이지 방문
- 다음 정보로 제거 요청 제출:
  - 도메인 이름
  - 문제 해결 설명
  - 재발 방지를 위한 조치
- 제거 상태 매일 모니터링 (24시간에서 7일 소요)

### 단계 6: 이메일 목록 적극적으로 정리

**문제가 있는 연락처 제거:**

- Contacts(연락처)로 이동
- 다음 항목을 식별하고 제거하기 위한 필터 생성:
  - **하드 반송** - Valid Email is Invalid 필터
  - **비참여 연락처** - 90일 이상 열람/클릭 없음
    - Last Email Opened Date - More than - 90 days ago
    - Last Email Clicked Date - More than - 90 days ago
  - **역할 기반 이메일** - info@, admin@, noreply@ 주소
- 향후 캠페인에서 이러한 연락처를 차단하거나 삭제
- 참여하고 옵트인한 구독자에게만 집중

### 단계 7: 도메인 워밍업 전략 구현

**최소 발송량으로 시작:**

- 발송량 통제
- 다음 점진적 증가 일정을 따르세요:

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155057622696/original/Ourk8oTy8NxHcbyPpgecGt1kVvAj1jRoiw.png?1762342316)

- 가장 참여도가 높은 구독자에게 먼저 발송
- 외부 평판 도구와 인박스 배치를 매일 모니터링
- 참여도가 높게 유지될 때만(20% 이상 열람률) 볼륨 증가

### 단계 8: 이메일 콘텐츠 및 관행 최적화

**최근 캠페인 검토 및 개선:**

- Marketing(마케팅) → Email(이메일) → Statistics(통계)로 이동
- 성과가 나쁜 캠페인을 식별하고 개선:
  - 스팸 트리거 단어 제거 ("무료", "보장", "지금 행동", "한정 시간")
  - 명확한 발신자 식별 보장
  - 실제 주소 및 쉬운 수신 거부 포함
  - 개인화 및 관련성 있는 콘텐츠 사용
  - 일관된 발신자 이름 및 "From" 주소 유지
  - HTML 및 일반 텍스트 버전 모두 포함

## 대체 평판 모니터링 방법

### 성과 기반 평판 평가

직접적인 평판 점수가 제한적이므로 다음 핵심 지표를 모니터링하세요:

**주간 추적 지표:**

- **전달률 추이:** 일관된 95% 이상은 좋은 평판을 나타냄
- **열람률 안정성:** 급격한 하락은 스팸 폴더 배치를 나타낼 수 있음
- **참여 패턴:** 높은 참여도는 좋은 인박스 배치를 시사
- **반송 메시지 분석:** 평판 관련 오류 메시지 확인

**인박스 배치 지표:**

- **일관된 열람률:** 인박스 배치의 좋은 신호
- **클릭률:** 높은 CTR은 이메일이 보여지고 있음을 나타냄
- **답장률:** 답장은 강력한 긍정적 신호
- **전달률:** 전달은 높은 참여도를 나타냄

### 정기 모니터링 일정

**매일:**
- 캠페인 성과 확인
- 평판 지표를 위한 반송 분류 검토
- Google Postmaster Tools 스팸율 모니터링

**매주:**
- 블랙리스트 확인 실행
- Sender Score 확인
- 인박스 배치 테스트 수행
- 참여 추이 분석

**매월:**
- 인증 상태 확인
- 이메일 목록 정리
- 전반적인 성과 추이 검토
- 모니터링 도구 및 프로세스 업데이트

## 장기 평판 건전성을 위한 모범 사례

### 목록 관리 우수성

- **모든 새 구독자에 대해 이중 옵트인 사용**
- **정기적인 목록 정리** - 매월 비활성 구독자 제거
- **참여도별 목록 세그먼트** - 참여하는 사용자에게 더 자주 발송
- **이메일 목록 절대 구매 금지** - 유기적으로 성장한 목록만 사용
- **비활성 구독자 제거 전 재참여 캠페인 구현**

### 콘텐츠 및 발송 모범 사례

- **일관된 발송 일정 유지** - 급작스런 볼륨 증가 방지
- **명확하고 관련성 있는 제목 사용** - 오해의 소지나 클릭베이트 제목 피함
- **즉각적인 가치 제공** - 모든 이메일을 열어볼 가치 있게 만들기
- **명확한 발신자 식별 포함** - 인식 가능한 "From" 이름 사용

### 기술적 유지관리

- **인증 레코드 업데이트 유지** - 매월 SPF, DKIM, DMARC 모니터링
- **일관된 "From" 도메인 사용** - 도메인을 자주 바꾸지 마세요
- **서브도메인 평판 전이 모니터링** - 나쁜 서브도메인이 메인 도메인에 영향을 줄 수 있음

## 복구 일정 및 기대사항

### 즉시 조치 (1-7일차)

- 사용 가능한 모니터링 도구 설정 (Sender Score)
- 활성 블랙리스트에서 제거
- 인증 문제 수정
- 이메일 목록 적극적으로 정리
- 발송량을 최소로 줄이기
- 인박스 배치 테스트 시작

### 단기 복구 (2-8주차)

- 사용 가능한 평판 도구 매일 모니터링
- 주간 인박스 배치 테스트 수행
- 점진적인 발송량 증가
- 전달률 개선 관찰 (95% 이상 도달해야 함)
- 긍정적인 참여 지표 구축

### 장기 평판 구축 (2-6개월차)

- 일관되게 높은 참여율 달성
- 낮은 신고 및 반송률 유지
- 안정적인 발송 패턴 확립
- 완전한 평판 복구 및 인박스 배치
- 정상 발송량으로 복귀

## 모니터링해야 할 경고 신호

- **95% 미만의 전달률** - 평판 문제를 나타냄

- **반송률 증가** - 목록 품질 저하
- **수신 거부율 증가** - 콘텐츠 또는 타겟팅 문제

### 외부 도구에서

- **Sender Score:** 점수가 70 미만으로 하락
- **Talos Intelligence:** 상태가 "Good"에서 "Neutral" 또는 "Poor"로 변경
- **Blacklist Checkers:** 새로운 블랙리스트 출현
- **Google Postmaster:** 스팸 비율이 0.1% 이상으로 증가
- **Inbox Placement Tests:** 인박스 배치율 감소

## 여전히 문제가 있나요?

이러한 단계를 따른 후에도 평판 문제가 지속되면:

- 평판 개선이 완전히 적용되기까지 30-90일 기다리세요
- **사용 가능한 도구 지속 모니터링** - 성과 지표와 인박스 배치에 집중
- **잠재적 ISP 이의제기를 위해 모든 개선 노력 문서화**
- **메인 도메인이 복구되는 동안 새로운 시작을 위해 서브도메인 사용 고려**
