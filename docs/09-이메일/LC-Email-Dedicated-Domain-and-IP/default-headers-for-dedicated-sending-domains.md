# 전용 발송 도메인의 기본 헤더

"기본 헤더"는 전용 발송 도메인을 통해 발송되는 모든 이메일의 발신자명과 발신자 이메일에 대한 대체 설정을 제공합니다. 이를 통해 전달성과 브랜드 일관성을 보장할 수 있어요.

## 기본 헤더를 사용해야 하는 경우

- **DMARC 실패에 대한 대체**: 캠페인의 "발신자" 주소가 전용 발송 도메인의 DKIM/SPF 레코드와 일치하지 않는 경우, Hyperclass는 기본 헤더를 사용해서 메시지가 여전히 인증되도록 합니다.

- **일관된 브랜딩 유지**: 오타나 잘못된 설정으로 "발신자"가 일치하지 않더라도, 수신자는 거부된 주소 대신 브랜드 도메인을 발신자 필드에서 볼 수 있어요.

- **규제/규정 준수 요구사항**: 엄격한 이메일 인증이 필요한 업종(금융, 의료 등)에서는 DMARC 거부나 격리를 방지하기 위해 기본 헤더를 설정해야 합니다.

## 사전 요구사항

기본 헤더를 설정하기 전에 다음 사항을 확인하세요:

- **하위 계정(Sub-Account)에 있어야 함**

하위 계정만 전용 발송 도메인을 추가하고 관리하며 기본 헤더를 설정할 수 있어요. 에이전시 레벨 도메인은 하위 계정별 기본 헤더를 지원하지 않습니다.

- **전용 발송 도메인이 추가되고 검증되어야 함**

사용자 정의 발송 도메인(예: email.yourbrand.com)에 대한 DNS 설정(CNAME, DKIM, SPF)을 완료해야 해요.

`Settings(설정) → Email Service(이메일 서비스) → SMTP Service → Dedicated Domain and IP(전용 도메인 및 IP)`에서 도메인이 검증되었음을 나타내는 녹색 체크 표시가 Hyperclass에 표시되어야 합니다.

## 기본 헤더 설정 (발신자명 & 발신자 이메일)

전용 도메인에 대한 기본 "발신자 주소 및 이메일"은 하위 계정에서 생성한 도메인에서만 설정할 수 있어요.

하위 계정 `Settings(설정) - Email Service(이메일 서비스) → SMTP Service → Dedicated Domain and IP(전용 도메인 및 IP) → 전용 도메인 하위의 Set headers(헤더 설정)`로 이동하세요.

지정된 헤더는 DMARC 정렬이 실패할 때 전용 도메인을 통해 발송되는 이메일에 적용되어요.

## 기본 헤더가 실제로 작동하는 방식

- **캠페인 설정**

브로드캐스트나 워크플로우 이메일을 만들 때, 캠페인 레벨의 "발신자명"과 "발신자 이메일"을 설정할 수 있어요.

전용 도메인(email.yourbrand.com)과 일치하지 않는 다른 "발신자 이메일"(예: sales@differentdomain.com)을 의도적으로 사용하면 DMARC가 실패해요.

- **DMARC 확인 및 대체**

발송 시, Hyperclass는 캠페인의 "발신자" 주소와 발송 도메인의 DKIM/SPF 레코드 간의 DMARC 정렬을 확인합니다.

정렬이 통과하면: 이메일은 캠페인에서 선택한 "발신자"로 발송돼요.

정렬이 실패하면: Hyperclass는 해당 전용 도메인 하위에서 설정한 기본 헤더를 사용해요.

- **수신자 경험**

수신자는 인증된 발송 도메인과 일치하는 유효한 "발신자명"과 "발신자 이메일"을 볼 수 있어요.

이메일은 DMARC를 준수하며 거부되거나 스팸으로 표시될 가능성이 줄어들어요.

## 예시: 완료된 기본 헤더 설정

### 예시 시나리오

- 전용 발송 도메인: email.yourbrand.com (검증됨)
- 기본 헤더 발신자명: YourBrand Support
- 기본 헤더 발신자 이메일: support@email.yourbrand.com

### 사용자가 설정한 브로드캐스트/워크플로우 이메일

- 캠페인 발신자명: Jane from YourBrand  
- 캠페인 발신자 이메일: jane@anotherdomain.com (email.yourbrand.com과 일치하지 않음)

### 발송 시 발생하는 상황

- 캠페인 발신자 이메일이 전용 발송 도메인과 일치하는 경우, 수신자는 다음을 볼 수 있어요:

From: Jane from YourBrand <jane@email.yourbrand.com> (또는 선택한 일치하는 주소)

- 캠페인 발신자 이메일이 일치하지 않고 DMARC 정렬이 실패하는 경우, Hyperclass는 기본 헤더를 적용하고 수신자는 다음을 볼 수 있어요:

From: YourBrand Support <support@email.yourbrand.com>

### 작동 확인 방법

- 일치하지 않는 발신자 이메일(도메인 불일치)을 사용해서 테스트 이메일을 보내세요.
- 받은 이메일의 "발신자" 주소를 확인하고 기본 헤더와 일치하는지 확인하세요.

## 모범 사례 및 고려사항

- **발송 전 기본 헤더 설정**: 주요 캠페인을 시작하기 전에 항상 대체 헤더를 설정하세요. DNS나 DMARC 잘못 설정이 발생하면 기본 헤더가 자동으로 작동해요.

- **답장 모니터링**: 기본 "발신자 이메일"은 발송 도메인의 어떤 주소든 될 수 있지만, 메일함이나 별칭이 모니터링되는지 확인하세요(예: support@yourbrand.com이 올바른 팀으로 전달되도록).

- **확장 전 테스트**: 일치하지 않는 "발신자" 주소를 사용해서 소규모 내부 테스트를 보내어 Hyperclass가 올바르게 기본값을 적용하는지 확인하세요. 받은편지함 배치와 DMARC 상태를 모두 확인하세요.

- **전파 지연 이해**: DNS 변경(특히 DKIM/SPF/TXT 레코드)은 완전히 전파되는 데 최대 48시간이 걸릴 수 있어요. 기본 헤더가 없으면 전파 중에 보낸 메시지가 거부될 수 있어요.

- **하위 계정별 분리**: 여러 브랜드나 부서를 관리하는 경우, 각각 자체 전용 도메인과 기본 헤더를 가진 별도의 하위 계정을 만들어서 격리된 제어를 하세요.

## 일반적인 문제 해결 시나리오

| 문제 | 가능한 원인 | 해결책 |
|------|-------------|--------|
| "Set headers(헤더 설정)" 옵션이 보이지 않음 | 에이전시 레벨 보기에 있거나 도메인이 하위 계정에서 검증되지 않음 | 올바른 하위 계정으로 전환하세요. Settings(설정) → Email Service(이메일 서비스) → SMTP Service → Dedicated Domain and IP(전용 도메인 및 IP)에서 전용 도메인을 추가/검증하세요. |
| 대체 적용 후에도 이메일 발송 실패 | DNS 레코드(DKIM/SPF)가 불완전하거나 잘못된 형식임 | 발송 도메인에 대한 DNS 항목을 다시 확인하세요. 온라인 DKIM/SPF 검증기를 사용하세요. 전파를 기다리세요. |
| 예상대로 대체 헤더가 적용되지 않음 | 캠페인 "발신자" 주소가 발송 도메인과 일치함(의도하지 않았더라도) | 캠페인의 "발신자 이메일"이 진짜로 일치하지 않는지 확인하세요(예: 도메인 불일치). DMARC 보고서를 확인해서 실패를 확인하세요. |
| 수신자가 잘못된 "발신자명" 또는 "발신자 이메일"을 봄 | 캠페인이나 워크플로우에서 예상을 무시하는 표시명을 하드코딩했을 수 있음 | 캠페인/워크플로우의 "발신자명"과 "발신자 이메일"을 편집하세요. 대체가 올바르게 철자되고 여분의 공백이 없는지 확인하세요. |
| 발송 도메인에 대한 SSL 인증서가 발급되지 않음 | 도메인 검증을 위한 CNAME/DNS 레코드가 누락되거나 잘못됨 | Settings(설정) → Dedicated Domain(전용 도메인)에서 "Edit DNS Info(DNS 정보 편집)"을 클릭하고 지시사항을 정확히 따르세요. 재시도하기 전에 전파를 기다리세요. |
