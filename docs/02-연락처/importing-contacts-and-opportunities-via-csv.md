# CSV로 연락처와 기회 일괄 가져오기

CSV 파일을 사용해서 연락처와 기회를 대량으로 가져오는 기능은 Hyperclass에서 데이터 입력과 정리를 효율적으로 처리할 수 있는 핵심 기능입니다. 필드를 매핑하고, 레코드를 생성하거나 업데이트하며, 가져오기 과정을 원활하게 모니터링할 수 있어 정확한 데이터 입력과 대량 작업의 시간 절약이 가능합니다. 이 가이드는 CSV 파일 준비부터 가져오기 과정 모니터링 및 오류 해결까지 단계별로 설명합니다.

## 연락처와 기회 가져오기 사전 요구사항

가져오기 과정을 시작하기 전에 다음 요구사항을 충족하는지 확인하세요:

- **사용자 권한:** **Admin(관리자)** 권한이 있는 사용자만 연락처를 가져올 수 있습니다.

- **파일 형식:** 파일은 반드시 **CSV (.csv) 형식**이어야 합니다.

- **파일 크기 제한:** CSV 파일은 30MB 미만이어야 합니다. 더 큰 경우 작은 파일로 분할하세요.

- **단일 시트 요구사항:** CSV 파일에는 반드시 **하나의 시트/탭만** 포함되어야 합니다.

- **CSV 요구사항**: 첫 번째 행은 비어있으면 안 됩니다! 시스템의 필드에 해당하는 최소 하나의 열 헤더가 포함된 헤더 행을 포함해야 합니다.

## 피해야 할 일반적인 실수

❌ **헤더 불일치**: CSV 헤더가 시스템 필드와 일치하는지 확인하세요.

❌ **필수 필드 공백**: 모든 행에는 최소 하나의 필수 필드(이름, 이메일 또는 전화번호)가 있어야 합니다.

❌ **전화번호에 특수 문자 사용**: 공백, 대시 또는 문자를 제거하세요.

❌ **CSV 내 중복 레코드**: 가져오기 전에 목록을 정리하세요.

## 연락처와 기회 가져오기

Hyperclass에서 연락처와 기회 가져오기 기능에 접근하는 두 가지 방법이 있습니다.

- **연락처(Contacts) 탭에서**
왼쪽 메뉴에서 연락처로 이동하고, 스마트 리스트(Smart Lists) 탭을 클릭한 후 연락처 가져오기(Import Contacts) 버튼을 클릭하세요.

![연락처 탭에서 가져오기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040930961/original/yW_9vq1gWBiCtPOzzi2VJmvf6boUSaN6zg.jpeg?1738599031)

- **기회 관리(Opportunities) 탭에서**
왼쪽 메뉴에서 기회 관리로 이동한 후 우측 상단의 가져오기(Import) 버튼을 클릭하세요.

![기회 관리 탭에서 가져오기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040930986/original/8MqKQhBikQMrYyLxrl1XnGtTPblojpxx-A.jpeg?1738599048)

두 방법 모두 동일한 가져오기 과정으로 이어지며, 다음 중 선택해서 가져올 수 있습니다:
1. 연락처
2. 기회
3. 연락처 & 기회 동시에

### CSV 파일 업로드

**1단계**: 연락처, 기회 또는 두 가지 모두 가져오기를 선택한 후 다음(Next)을 클릭하고, 파일 업로드(Upload File)를 클릭해서 컴퓨터에서 CSV 파일을 선택하세요.

![CSV 파일 업로드](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040673232/original/U6Ezequ3JHtmkRbRUYlCdowbOAOFPlw1RA.gif?1738168901)

**2단계**: 가져오기용 파일을 업로드한 후 "연락처/기회를 가져올 방법 선택(Choose how to import contacts/opportunities)"을 클릭해서 생성, 업데이트 또는 둘 다 수행할지 결정하세요.

![가져오기 방법 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040673389/original/1wbH4t9zwC4jss4xrDWDyMQBGScc9oeYnw.png?1738169062)

**참고**: 기존 연락처 찾기 기준(Find existing contacts based on Filter)은 기본적으로 이메일, 그다음 전화번호로 설정되어 있습니다. 즉, 연락처/기회는 먼저 이메일로 필터링되고, 이메일이 없으면 전화번호로 필터링됩니다.

### 열을 필드에 매핑하기

파일의 각 열이 시스템의 해당 필드에 올바르게 매핑되었는지 확인하세요.

몇 가지 주의사항:

- "빈 값 업데이트 안 함(Don't update Empty Values)" 체크박스를 선택하면, 가져오는 파일에 빈 값이 있고 해당 기회의 필드에 어떤 값이 있는 경우 어떤 필드도 업데이트하지 않습니다.

- 특정 필드를 가져오지 않기로 결정한 경우, 매핑하지 않은 상태로 두고 하단의 박스를 체크해서 진행할 수 있습니다. - "모든 필수 필드가 매핑되었는지 확인하세요. 진행하려면 모든 필드를 매핑하거나 매핑되지 않은 열에 대해 데이터 가져오지 않음을 선택하세요"

- 이 기능은 또한 여러 이메일 주소와 전화번호를 가진 연락처 생성을 허용합니다. 이러한 추가 세부 정보는 가져오기 과정에서 자동으로 추가됩니다.

- CSV에서 연락처와 기회가 같은 줄에 있으면 자동으로 서로 매핑됩니다.

### **가져오기 검토 및 완료**

매핑을 검토한 후 확인하고 가져오기를 진행하세요. 가져온 연락처에 대해 **스마트 리스트** 생성, **워크플로우**에 연락처 추가 또는 새 **태그** 추가를 선택할 수도 있습니다.

## 가져오기 상태 모니터링

- 연락처와 기회 **일괄 작업(Bulk Actions)** 페이지를 통해 가져오기 진행 상황과 상태를 모니터링할 수 있습니다.
- 연락처와 기회 가져오기는 모두 **일괄 작업** 아래 같은 곳에 표시됩니다.
- "**통계 보기(Show Stats)**" 버튼을 클릭해서 가져오기 통계를 확인하세요 - 오류 로그를 다운로드하고 "**오류(Error)**" 탭에서 오류에 대한 자세한 내용과 해결 방법을 볼 수도 있습니다.

![가져오기 상태 모니터링](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155040675205/original/9137ojlvH-YH7-gdZ0MwQa2zapRtYwe4kg.gif?1738170977)

Keap의 CSV 내보내기에서 나온 전화번호 필드는 Hyperclass과 호환되지 않습니다. 괄호 '()'와 대시 '-'로 인해 호환되지 않습니다. CSV 파일에서 전체 열을 강조 표시하고 숫자 형식으로 되어 있는지 확인하며, 괄호와 대시를 모두 제거한 후에 CRM의 데이터를 손상시키지 않고 가져올 수 있습니다.

## **자주 묻는 질문**

**Q. CSV 파일이 50MB를 초과하면 어떻게 되나요?**

파일을 **더 작은 섹션(50MB 미만)으로 분할**해서 별도로 가져오세요.

**Q. 여러 이메일이나 전화번호가 있는 연락처를 가져올 수 있나요?**

네! 추가 이메일과 전화번호는 기본 데이터를 대체하는 대신 **보조 필드로 저장**됩니다.

**Q. Hyperclass는 중복 연락처를 어떻게 처리하나요?**

- Hyperclass는 전화번호나 이메일을 기준으로 연락처를 자동으로 병합합니다.
- 일치하는 항목이 발견되면 시스템은 중복을 생성하는 대신 **기존 레코드를 업데이트**합니다.

**Q. CSV 가져오기를 통해 기존 연락처를 업데이트할 수 있나요?**

네! 가져오기 중에 **업데이트만** 또는 **생성 및 업데이트**를 선택하세요.

**Q. 가져오기가 실패하면 어떻게 해야 하나요?**

- **가져오기 상태 페이지**에서 **오류 메시지**를 확인하세요.
- **오류 로그를 다운로드**하고, 문제를 수정한 후 **수정된 CSV를 다시 가져오세요**.

**Q: 가져온 모든 연락처가 모든 채널에 대해 DND로 설정되는 이유는 무엇인가요? 특정 채널에만 DND를 적용하려면 어떻게 해야 하나요?**

CSV에 **DND(방해 금지) 열**이 있는 연락처를 가져올 때, Hyperclass는 **모든 커뮤니케이션 채널**(SMS, 이메일, 전화, 음성메일 등)에 대해 **자동으로 DND를 적용**합니다.

**특정 채널에만 DND를 설정**하려면 DND 열 대신 **워크플로우와 태그**를 사용해야 합니다.

**Q. 노트가 포함된 CSV를 가져오려고 하는데, 연락처 레코드당 하나 이상의 노트를 생성할 수 없습니다.**

연락처 노트를 가져올 때는 연락처 레코드당 최대 5000자 제한으로 하나의 노트만 가질 수 있습니다.
