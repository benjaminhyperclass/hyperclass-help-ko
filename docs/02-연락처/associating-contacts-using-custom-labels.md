# 커스텀 라벨을 사용한 연락처 연결

연락처 간의 관계를 관리하는 것은 고객 데이터를 효율적으로 정리하는 중요한 부분입니다. Hyperclass CRM의 연락처 연결(Contact Associations) 기능을 사용하면 비즈니스 파트너, 가족 구성원, 직원 등과 같은 연락처와 기타 객체 간의 구조화된 연결을 만들 수 있습니다. 이 가이드에서는 CRM 정리를 개선하기 위해 이러한 연결을 설정하고 관리하는 방법을 종합적으로 설명합니다.

# 연락처 연결이란?

연락처 연결(Contact Associations)은 Hyperclass CRM에서 연락처와 객체 간의 구조화된 관계를 만들 수 있게 해줍니다. 개별적으로 관리되는 프로필 대신, 연결을 통해 배우자, 매니저, 직원 또는 연결된 객체와 같은 관련 엔티티 간의 명확한 연결을 제공합니다.

#### 연결 라벨 유형

- 단일 라벨: 연결된 두 연락처 모두에 적용되는 라벨 (예: "동료")
- 쌍 라벨: 관계에서 각 연락처에 대한 별개의 라벨 (예: "매니저"와 "직원", "구매자"와 "판매자")

## 연락처 연결의 주요 장점

- **개선된 연락처 정리:** 연결을 통해 연락처 간의 관계를 구조화된 방식으로 관리할 수 있어, 연결을 추적하고 관련 정보에 한 곳에서 접근하기 쉬워집니다.

- **간소화된 커뮤니케이션:** 관련 연락처를 연결함으로써 비즈니스는 연결된 개인을 빠르게 참조하고 연락할 수 있어, 더 원활한 상호작용과 개선된 협업을 보장합니다.

- **향상된 관계 관리:** 가족 관계, 비즈니스 파트너십, 내부 계층 구조 등 연결을 통해 비즈니스는 관계를 효과적으로 이해하고 관리할 수 있습니다.

- **효율적인 CRM 워크플로우:** 명확한 연결이 설정되면 사용자는 중복 데이터 입력을 줄이고, 연결된 연락처를 빠르게 업데이트하며, 잘 정리된 CRM 시스템을 유지할 수 있습니다.

---

## 연락처 연결 확인하기

연락처 연결은 두 가지 주요 영역에서 접근할 수 있습니다:

- 연락처 프로필 보기: 연락처 프로필 내에서 오른쪽 패널에 있는 연결 아이콘을 통해 관련 객체와 연결된 연락처를 표시합니다.

- 객체 관리 섹션: Settings(설정) > Objects(객체) > Contacts(연락처) 섹션에서 모든 연결의 개요를 제공하며, 라벨과 관계를 생성하고 관리할 수 있습니다.

## 연락처 연결 만들기

연결 생성은 몇 가지 단계를 포함합니다:

### 객체 섹션 열기

Settings(설정) > Objects(객체)로 이동한 다음 Contacts(연락처)를 선택합니다.

![객체 섹션 열기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042764435/original/icaJSGWvD4zLfPCra5H2p_QaMAz38Igvyw.png?1741255187)

### "연결 만들기" 클릭

새로운 관계를 정의하려면 "+ Create Association(연결 만들기)" 버튼을 선택합니다.

![연결 만들기 버튼](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042764456/original/VYaJ46n1_FIxelROYwCJ47GWdG88V8Aw-A.png?1741255210)

### 연결된 객체 선택

Contacts(연락처), Surfboards 또는 기타 사용 가능한 엔티티와 같은 객체를 선택합니다.

![연결된 객체 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042764485/original/bQ3d4Zgz0nEMGT602BQCiMmVxiYu2pxRtQ.png?1741255230)

### 라벨 유형 선택

관계에 단일 라벨(예: "파트너")이 필요한지 쌍 라벨(예: "구매자 - 판매자")이 필요한지 결정합니다.

![라벨 유형 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042764538/original/wUckypt_EZjBO-f_0OvZFnyaXftFTFhGZQ.png?1741255249)

### 관계 유형 정의

다대다(Many-to-Many) 또는 다대일(Many-to-One) 연결 중에서 선택합니다.

![관계 유형 정의](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042764594/original/zujX-ekNhLF60GVrIVWfzePlZsJSgAAvyQ.png?1741255274)

## 연결을 사용하여 연락처 보기/링크하기

연결이 생성되면 연락처를 연결하는 단계는 다음과 같습니다:

### 연락처 프로필 열기

Contacts(연락처) 섹션으로 이동하여 연락처 프로필을 선택합니다.

![연락처 프로필 열기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042764620/original/6pnOVoglVfYRHUauQNzwiVCgePi9W4n5Qw.png?1741255300)

### 연결 아이콘 클릭

오른쪽 패널에 위치한 이 아이콘을 통해 연결 옵션에 접근할 수 있습니다.

![연결 아이콘 클릭](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042764672/original/JuQM4tlAWGX9Qg0L-DdsNiWVuJR1aIIVJg.png?1741255325)

### 관계 추가

"Add(추가)"를 클릭하고 관련 연락처를 선택한 후 적절한 라벨을 지정합니다.

![관계 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042766212/original/tYwGsSppQbUaxBLCknQ2qWxpUYzxxnWbNA.gif?1741256064)

### 연결 확인

연결된 연락처가 Related Objects(관련 객체) 섹션에 나타나며, 연결이 두 프로필을 자동으로 업데이트합니다.

## 연결 관리 및 편집

### 기존 연결 편집

Settings(설정) > Objects(객체) > Contacts(연락처)로 이동하여 해당 연결을 찾고, 점 3개 메뉴(⋮)를 사용하여 수정합니다.

![기존 연결 편집](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042764834/original/0vI3b0qJz7Bkt6GLczej1qMpJE1gjbcXrA.png?1741255410)

### 연결 제거

연락처 프로필 내에서 Related Objects(관련 객체) 섹션으로 이동하여 연결된 연락처를 찾고 링크를 제거합니다.

![연결 제거](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155042765305/original/2faMy4o40dghTlB4DnCg0DtRQgSPU_BUkg.gif?1741255625)

## 자주 묻는 질문

**Q. 단일 라벨과 쌍 라벨의 차이점은 무엇인가요?**

단일 라벨: 두 연락처 모두 같은 라벨을 받습니다 (예: "동료").

쌍 라벨: 각 연락처가 관계에서 별개의 역할을 합니다 (예: "매니저"와 "직원").

**Q. 연결을 자동화나 워크플로우에서 사용할 수 있나요?**

현재 연결은 조직적 도구로 사용됩니다. 향후 업데이트에서 자동화 기능이 도입될 수 있습니다.

**Q. 연락처를 커스텀 객체에 연결할 수 있나요?**

예, 연락처는 Surfboards와 같은 사전 정의된 객체와 연결될 수 있으며, 향후 업데이트에서 추가적인 커스텀 객체 지원이 예상됩니다.

**Q. 연결 라벨을 삭제할 수 있나요?**

예, 하지만 라벨 삭제는 영구적이며 먼저 모든 연결된 연결을 제거해야 합니다.

**Q. 하나의 연락처가 가질 수 있는 연결의 개수는?**

연락처는 최대 10개의 연결을 가질 수 있으며, 하나의 라벨은 최대 1,000개의 연락처를 연결할 수 있습니다.

이 기능을 사용하면 사용자가 커스텀 라벨을 사용하여 여러 연락처를 서로 연결할 수 있습니다. 이는 가족 구성원이나 비즈니스 동료와 같은 연락처 간의 관계를 관리해야 하는 비즈니스에 특히 유용합니다. 연결된 연락처에 대한 커뮤니케이션과 데이터 정리를 간소화하려는 경우, 아래 정보가 주요 기능, 장점 및 이 기능을 효과적으로 사용하는 방법을 안내해 드릴 것입니다.
