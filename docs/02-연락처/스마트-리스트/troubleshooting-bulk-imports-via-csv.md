# CSV 대량 가져오기 문제 해결

이 가이드는 CSV 파일을 통해 연락처를 가져올 때 발생할 수 있는 오류를 해결하는 데 도움을 드립니다. 각 오류 코드에는 구체적인 원인과 해결 방법이 있습니다. 이를 이해하면 깔끔하고 원활하며 성공적인 대량 가져오기를 수행할 수 있습니다.

## 대량 가져오기란 무엇인가요?

대량 가져오기를 사용하면 CSV 파일을 통해 대량의 연락처 데이터를 빠르게 업로드할 수 있습니다. 온보딩, 리드 마이그레이션, 목록 동기화에 유용한 강력한 도구이지만, 형식 오류로 인해 진행이 차단될 수 있습니다. 이 가이드는 그러한 오류를 해결하는 데 도움이 됩니다.

다음 동영상은 특정 연락처 가져오기 일괄 작업과 연결된 오류 로그로 이동하는 방법과 오류를 해결하기 위해 작업하는 방법을 다룹니다:

---

## **성공적인 CSV 가져오기의 주요 장점**

대량 가져오기 문제 해결을 마스터하면 시간을 절약하고 좌절감을 줄일 수 있습니다. 이것이 중요한 이유는 다음과 같습니다:

- 재작업을 줄여 시간 절약
- 중복되거나 충돌하는 연락처 레코드 생성 방지
- 깔끔하고 검증된 연락처 데이터 유지
- 워크플로우 자동화를 차단하는 가져오기 실패 방지
- 정확한 소유자/사용자 배정 보장
- 캠페인 개인화 및 태그 보존

---

## **CSV 가져오기 문제 해결 체크리스트**

일반적인 CSV 오류를 방지하기 위해 가져오기 전에 이 체크리스트를 사용하세요:

- 파일에 하나의 시트만 포함
- 모든 필수 필드가 채워져 있고 올바르게 형식화되었는지 확인
- 이메일, 전화번호, 날짜를 올바르게 형식화
- 각 행에 고유한 연락처 ID 사용
- 열거형/드롭다운 값이 정확히 일치하는지 확인
- 소유자 열에 유효한 하이퍼클래스 사용자 이메일 사용
- 신뢰할 수 있는 기본 열 선택(이메일 또는 연락처 ID 등)

## 자주 묻는 질문

**Q: 어떤 파일 형식을 업로드해야 하나요?**

.csv 파일만 허용됩니다. Excel이나 Google Sheets에서 작업하는 경우 먼저 데이터를 CSV 파일로 내보내세요.

**Q: 대량 가져오기를 취소할 수 있나요?**

직접적인 "취소" 옵션은 없지만 업로드 중에 가져온 연락처에 태그를 지정할 수 있습니다. 필요한 경우 해당 태그를 사용하여 필터링하고 삭제하세요.

**Q: 가져오기 파일의 최대 크기는 얼마인가요?**

한도는 20MB 또는 10,000행 중 먼저 도달하는 것입니다.

**Q: 업로드하기 전에 전화번호를 검증하려면 어떻게 해야 하나요?**

스프레드시트 수식이나 온라인 도구를 사용하여 E.164 형식을 따르는지 확인하세요.

**Q: 기본 열로 무엇을 선택해야 하나요?**

여러 연락처에서 반복되지 않는 이메일이나 연락처 ID와 같은 고유 식별자를 사용하세요.

**Q: 내 CSV 목록이 혼합되어 있습니다. 일부 연락처는 이메일이나 전화번호만 있고, 일부는 이메일과 전화번호를 모두 가지고 있습니다. 어떻게 진행해야 하나요?**

혼합된 CSV에 대해서는 '보조 기본설정' 옵션을 사용하세요. 시스템은 첫 번째 기본설정을 검색한 다음 첫 번째를 찾을 수 없는 경우 두 번째를 검색합니다. 위 예에서 첫 번째 기본설정은 '이메일'이고 두 번째는 '전화번호'입니다.

이제 보조 필드를 기반으로 기존 연락처를 찾을 수 있으므로 연락처 레코드에 전화번호만 있는 경우에도 업로드할 수 있습니다!

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045601330/original/4zlCOqVjoq6LY8PnLxg6pZjqPyAfvDWsNg.png?1745499014)

**Q: 내 연락처 중 일부는 여러 개의 이메일이나 전화번호를 가지고 있습니다. 어떻게 진행해야 하나요?**

연락처에 여러 전화번호나 이메일이 있는 경우 업로드할 수 있도록 연락처를 여러 개로 분할하세요. 예:

**두 개의 전화번호 열이 있는 원본:**

Josephine
810-292-9388
810-374-9840
josephine_darakjy@darakjy.org

**업로드용으로 정리된 버전:**

Josephine
810-292-9388
josephine_darakjy@darakjy.org

Josephine
810-374-9840

전화번호를 고유 필드로 사용하여 연락처를 업로드할 때 중복이 생성되므로 다른 전화번호가 있는 두 번째 리드의 이메일은 공백으로 두세요.

**Q: Keap를 사용하는 사람들을 위한 중요 참고사항:**

Keap의 CSV 내보내기에서 나오는 전화번호 필드는 GHL과 호환되지 않습니다. 괄호 '()'와 대시 '-'로 인해 호환되지 않습니다. CSV 파일에서 전체 열을 강조 표시하고 숫자로 되어 있는지 확인하며, 괄호와 대시를 모두 제거한 다음 CRM의 데이터를 손상시키지 않고 가져올 수 있습니다.

**Q: 전화번호를 가져올 때 오류 코드 1007 '잘못된 필수 속성'이 발생합니다:**

하위 계정 국가 외부의 번호를 가져올 때 국가 코드를 추가하세요. 하위 계정 국가가 미국으로 설정된 경우 가져온 전화번호에 +1(미국 국가 코드)을 추가할 필요가 없습니다. 하지만 미국 외부의 번호를 가져오는 경우 [국가 코드](https://countrycode.org/)를 추가해야 합니다. 예를 들어, 비즈니스 위치가 미국으로 설정된 하위 계정은 영국으로 전화하려면 '+44'를 추가해야 합니다. 하위 계정의 국가가 미국이 아닌 다른 국가(영국, 호주 등)로 설정된 경우 미국 번호로 전화/SMS를 보내려면 CSV 파일의 국가 지역 코드에 '+1'을 추가해야 합니다.

**Q: DND 열이 있는 CSV를 가져오려고 하는데 모든 연락처가 모든 채널에 대해 DND로 설정됩니다.**

CSV에 DND 열이 있는 연락처를 가져올 때 연락처는 모든 채널에 대해 DND가 됩니다. 특정 채널에 대해 DND하려면 먼저 워크플로우를 설정하고 각 채널에 대해 태그를 사용하세요.

예: John을 SMS에 대해서만 DND하고 싶습니다. 'DND SMS' 태그가 추가될 때 연락처를 DND하는 워크플로우를 만들 것입니다. 이제 CSV에서 John의 이름 옆에 'DND SMS' 태그를 추가할 것입니다. 가져올 때 이것은 John을 SMS에 대해서만 DND하는 자동화를 실행할 것입니다.

**Q: 메모가 있는 CSV를 가져오려고 하는데 연락처 레코드당 하나 이상의 메모를 생성할 수 없습니다.**

연락처 메모를 가져올 때 연락처 레코드당 하나의 메모만 가질 수 있으며 5000자 제한이 있습니다. 하나 이상의 메모가 있거나 허용된 양보다 많은 문자가 있는 경우 [대형 텍스트 커스텀 필드](how-to-use-custom-fields.md)를 사용하세요.

**Q: 메모가 있는 CSV를 내보내려고 합니다.**

현재 메모에 대한 CSV 가져오기 제한은 없으며, 현재 내보내기 동작에 따르면 마지막 메모에 대해 최대 255자를 지원합니다.
