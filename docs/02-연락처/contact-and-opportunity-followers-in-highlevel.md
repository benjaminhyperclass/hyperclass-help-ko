# 연락처와 기회 관리의 팔로워 기능

연락처(Contacts)와 기회 관리(Opportunities)의 '팔로워(Followers)' 기능을 사용하면 여러 사용자가 동일한 레코드를 팔로우하고 접근할 수 있습니다. "본인 데이터만 보기(Only Assigned Data)" 사용자 권한이 활성화된 상황에서도 최대 10명의 사용자를 팔로워로 추가하여 CRM 레코드를 보고 편집할 수 있게 해줍니다.

## **팔로워란 무엇인가요?**

팔로워는 연락처나 기회에 추가된 사용자로, "본인 데이터만 보기" 권한 설정으로 인해 일반적으로는 데이터 접근이 제한되어 있더라도 해당 레코드를 보고 관리할 수 있는 권한을 얻습니다.

### **팔로워 권한 및 제한사항**

- 팔로워는 레코드 소유자와 동일한 권한을 상속받습니다.

- 이를 통해 연락처나 기회 정보를 보고 편집할 수 있습니다.

- 하지만 소유권 자체는 수정할 수 없습니다. 즉, 레코드의 소유자를 변경할 수는 없습니다.

## **주요 포인트**

- 팔로워 기능은 팀 간 협업을 개선하는 데 도움이 됩니다.

- 워크플로우(Workflows)와 결합하여 자동으로 팔로워를 배정할 수 있어 특히 유용합니다.

- 소유권 계층을 손상시키지 않으면서도 가시성과 접근성을 유지하기 위해 이 기능을 전략적으로 활용하세요.
