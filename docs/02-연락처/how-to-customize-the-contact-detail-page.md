# 연락처 상세 페이지 커스터마이징하는 방법

Hyperclass의 연락처 상세 페이지(Contact Detail Page) 커스터마이징 기능을 사용하면 관리자가 각 워크플로우에 맞게 연락처 레코드를 맞춤 설정할 수 있습니다. 이 가이드에서는 뷰를 생성하고, 패널과 모듈을 정리하며, 표시 여부를 관리하고, 사용자에게 레이아웃을 배정하는 방법을 설명합니다.

## 연락처 상세 페이지 커스터마이징이란?

연락처 상세 페이지 커스터마이징을 통해 관리자는 Hyperclass 내에서 연락처 레코드가 표시되는 방식을 맞춤 설정할 수 있습니다. 커스터마이즈 가능한 뷰를 사용해서 패널을 정리하고, 표시할 모듈을 선택하며, 필드 표시 여부를 제어하고, 특정 뷰를 사용자에게 배정할 수 있습니다. 이를 통해 영업, 지원, 관리자 등의 역할에 맞는 맞춤형 경험을 만들 수 있습니다.

**참고:** **관리자만** 뷰를 **생성**하고 **관리**할 수 있습니다.

## 연락처 상세 페이지 커스터마이징의 주요 장점

- 역할별 뷰: 사용자별로 다른 연락처 레이아웃을 생성할 수 있습니다.
- 더 나은 정리: 워크플로우 우선순위에 따라 모듈, 폴더, 필드를 배치할 수 있습니다.
- 불필요한 요소 제거: 필요 없는 필드와 모듈을 숨겨서 페이지를 간소화할 수 있습니다.
- 효율성 향상: 자주 사용하는 도구를 중앙 패널이나 다이나믹 탭에 배치할 수 있습니다.
- 제어 강화: 삭제 권한을 제한하고 적합한 사용자에게 적합한 뷰를 배정할 수 있습니다.

## 뷰와 접근 권한 관리

뷰를 사용하면 관리를 중앙화하면서도 서로 다른 사용자를 위한 맞춤형 레이아웃을 생성할 수 있습니다. 관리자는 어떤 사용자가 어떤 레이아웃을 볼지 결정할 수 있고, 계정의 기본 경험을 설정할 수 있습니다.

- 계정당 **최대 5개의 뷰**를 생성할 수 있습니다.
- **관리자만** 뷰를 **생성**하고 **관리**할 수 있습니다.
- 기본 뷰는 **미리 설정**되어 있으며 **변경할 수 없습니다**.
- 뷰는 **사용자 관리(Manage Users)**를 통해 사용자에게 배정됩니다.
- 각 뷰에 매핑된 사용자를 확인하고 필요에 따라 재배정할 수 있습니다.

![](https://jumpshare.com/share/et5ahKjAXLLjGUcN1NTr+/Screen+Shot+2026-03-27+at+15.49.07.png)

## 연락처 상세 페이지 레이아웃 구조

연락처 상세 페이지는 명확한 섹션으로 나뉘어 있어 관리자가 사용자의 작업 방식에 따라 정보를 정리할 수 있습니다. 각 영역을 이해하면 탐색과 유지보수가 더 쉬운 레이아웃을 구축할 수 있습니다.

각 뷰는 4개의 주요 섹션으로 구성됩니다:

**중요:** 대화(Conversations) 모듈은 고정되어 있으며 항상 중앙 패널에 나타납니다.

- 연락처 카드(Contact Card): 주요 연락처 정보를 표시하는 상단 섹션
- 왼쪽 패널(Left Panel): 커스텀 필드, 표준 필드, 폴더에 사용
- 중앙 패널(Center Panel): 주요 모듈을 위한 기본 작업 공간
- 오른쪽 패널(Right Panel): 추가 모듈에 사용

![](https://jumpshare.com/share/f8e4SgmKsxc5a9OQFglB+/Screen+Shot+2026-03-27+at+16.14.50.png)

## 연락처 상세 페이지 커스터마이징 방법

신중한 설정을 통해 연락처 상세 페이지를 더 사용하기 쉽게 만들고, 각 사용자가 적절한 위치에서 올바른 정보를 볼 수 있도록 도와줍니다. 다음 단계에 따라 커스터마이즈된 뷰를 생성하고 배정하세요.

#### 1단계: 연락처 상세 뷰 커스터마이즈 탭으로 이동하기

- **연락처(Contacts)** 탭을 클릭합니다.
- **스마트 리스트(Smart Lists)**로 이동합니다.
- 점 세 개를 클릭합니다.
- **설정(Settings)**을 클릭합니다.

![](https://jumpshare.com/share/vQSG6BokMXzMoCgnCXAH+/Screen+Shot+2026-03-27+at+16.44.34.png)

- **연락처 상세 뷰 커스터마이즈(Customize Contact detail view)** 탭을 클릭합니다.

![](https://jumpshare.com/share/TZbd82upGM8qEuli7qN8+/Screen+Shot+2026-03-27+at+16.49.58.png)

#### 2단계: 뷰 생성 및 관리하기

필요한 경우 새 뷰를 생성하고 지원할 사용자나 워크플로우에 따라 이름을 지정합니다.

참고: **계정당 최대 5개의 뷰**를 생성할 수 있으며, **관리자만** 뷰를 **생성**하고 **관리**할 수 있습니다.

- **+ 뷰 추가(+ Add View)** 버튼을 클릭합니다.

![](https://jumpshare.com/share/BWmH13Y26MiiLWHSoDBg+/Screen+Shot+2026-03-27+at+17.28.06.png)

- **뷰 이름(View Name)**을 추가합니다.
- 뷰에 사용자를 배정합니다. (**사용자**는 하나의 **뷰**에만 속할 수 있습니다. 제거되면 기본 **뷰**로 **돌아갑니다**.)
- **생성(Create)**을 클릭합니다.

![](https://jumpshare.com/share/mWx2CiKeHGOc6vfsRgdX+/Screen+Shot+2026-03-27+at+17.31.03.png)

- 뷰가 생성되면 뷰 우측 상단의 **점 세 개**를 클릭하여 다음을 수행할 수 있습니다:
  - 뷰 이름 변경(Rename View)
  - 사용자 관리(Manage Users)
  - 뷰 복제(Duplicate View)
  - 뷰 삭제(Delete View)

![](https://jumpshare.com/share/li5pKONNqpfiFBOv6lS2+/Screen+Shot+2026-03-27+at+17.33.34.png)

- **사용자 관리(Manage Users)**에서 해당 사용자에게 뷰를 배정합니다.
- **모든 사용자 관리(Manage all users)**를 클릭하여 사용자 배정의 전체 현황을 확인합니다.
- 각 뷰에 매핑된 사용자를 검토하고 필요에 따라 재배정합니다.

![](https://jumpshare.com/share/1vac4a3cvk8RsKvsHdA7+/Screen+Shot+2026-03-27+at+17.38.34.png)

#### 3단계: 상세 페이지 커스터마이징하기

레이아웃 편집기를 사용하여 연락처 상세 페이지를 정리합니다.

1. **연락처 카드 커스터마이징**

연락처 카드는 중요한 연락처 정보를 레코드 상단에 계속 표시합니다.

다음 항목을 추가하고 배치할 수 있습니다:

- 전화번호
- 이메일 주소
- 연락처 주소
- 담당자 및 팔로워(Owner & Followers)
- 태그

중요한 세부사항:

- **기본 전화번호**만 표시할 수 있습니다.
- **기본 이메일 주소**만 표시할 수 있습니다.
- 연락처 카드 내에서 사용 가능한 항목의 순서를 재정렬할 수 있습니다.
- 필요 없는 필드를 숨길 수 있습니다.
- 실수로 연락처를 삭제하는 것을 방지하기 위해 **삭제(Delete)** 버튼을 비활성화할 수 있습니다.

![](https://jumpshare.com/share/9aPWkofhiQKLljhgafsk+/GIF+Recording+2026-03-27+at+18.23.30.gif)

2. **왼쪽 패널 커스터마이징**

왼쪽 패널은 필드 데이터를 정리하여 사용자가 연락처 정보를 빠르게 검토하고 업데이트할 수 있도록 도와줍니다.

왼쪽 패널을 사용하여 다음을 할 수 있습니다:

- 커스텀 폴더와 필드 재배열
- 명확성을 위해 폴더 이름 변경
- 관련 없는 필드 숨기기
- **추가(Add)** 버튼을 사용하여 제거된 필드 다시 추가
- 현재 표시되는 필드 수 검토

![](https://jumpshare.com/share/Ih5yKoVC5ZwmSY5TLrmj+/GIF+Recording+2026-03-27+at+18.46.09.gif)

3. **다이나믹 탭 기능 사용**

다이나믹 탭은 우선순위가 높은 하나의 모듈을 전용 탭에 표시하여 더 빠른 접근을 가능하게 합니다.

- 다이나믹 탭은 **모든 필드(All Fields)**와 **액션(Actions)** 사이에 나타날 수 있습니다.
- 지원되는 모듈을 이 탭에 배정할 수 있습니다.
- 모듈이 다이나믹 탭에 배치되면 이전 탭 위치에서 제거됩니다.
- 빠른 접근이 필요한 기회 관리(Opportunities)나 클라이언트 포털(Client Portal) 같은 항목에 유용합니다.

![](https://jumpshare.com/share/vA0oRyMKNbh3Y5STTzYY+/GIF+Recording+2026-03-27+at+19.09.37.gif)

4. **중앙 패널 커스터마이징**

액션 탭이나 오른쪽 패널에서 모듈을 드래그 앤 드롭하여 레이아웃을 커스터마이징합니다. 대화 모듈은 항상 중앙에 고정되어 있으며 제거할 수 없습니다.

참고:
- 액션 모듈은 중앙 패널로 이동하거나 왼쪽 패널에 유지할 수 있습니다.
- 액션 모듈은 오른쪽 패널로 이동할 수 없습니다.
- 오른쪽 패널 모듈은 **오른쪽 패널에 유지하거나 중앙 패널로 이동할 수 있습니다**.

표시 옵션은 다음과 같습니다:

- 탭 뷰(Tabs View): 모듈이 스마트 리스트와 유사하게 탭으로 나타납니다.
- 드롭다운 뷰(Dropdown View): 모듈이 드롭다운 메뉴로 그룹화되어 시각적 혼잡함을 줄입니다.

필요에 가장 적합한 표시 스타일을 선택하세요.

![](https://jumpshare.com/share/dGoP40fVMImankvdVXem+/GIF+Recording+2026-03-27+at+18.53.00.gif)

5. **오른쪽 패널 커스터마이징**

빠른 접근을 위해 모듈 순서를 재정렬합니다.

![](https://jumpshare.com/share/djBdfqcELGTGXwVGH9oh+/GIF+Recording+2026-03-27+at+19.02.57.gif)

6. **패널 레이아웃 선택**

한 번에 확인해야 하는 정보량에 가장 적합한 레이아웃 스타일을 선택합니다.

사용 가능한 레이아웃 옵션:

- 3패널 뷰(Three-Panel View) (기본값): 왼쪽, 중앙, 오른쪽 패널을 모두 표시합니다.
- 2패널 뷰(Two-Panel View): 오른쪽 패널 모듈을 중앙 패널로 병합하여 더 컴팩트한 경험을 제공합니다.

중요한 세부사항:

- 2패널 뷰에서는 오른쪽 패널 항목이 중앙 패널로 병합됩니다.
- 다시 전환할 때 레이아웃이 이전 배치를 복원합니다.

![](https://jumpshare.com/share/FPbwQouJ61dg8uJLm7nQ+/GIF+Recording+2026-03-27+at+19.07.05.gif)

#### 4단계: 레이아웃 검토 및 저장

라이브 미리보기를 검토하여 레이아웃을 확인한 다음 뷰를 **저장(Save)**합니다.

![](https://jumpshare.com/share/qZJzJyyKXRWnYMj0tXgm+/Screen+Shot+2026-03-27+at+19.14.06.png)

## 연락처 상세 페이지 커스터마이징 모범 사례

몇 가지 간단한 모범 사례를 따르면 각 커스텀 뷰를 더 유용하고 유지보수하기 쉽게 만들 수 있습니다.

- 마케팅, 영업, 지원, 관리자 등을 위한 별도 뷰를 생성하세요.
- 우선순위가 높은 모듈을 중앙 패널에 유지하세요.
- 사용하지 않는 필드를 숨겨서 혼잡함을 줄이세요.
- 가장 자주 사용하는 액션이나 모듈에 다이나믹 탭을 사용하세요.
- 연락처 데이터 보호가 우선순위라면 삭제 버튼을 비활성화하세요.

## 자주 묻는 질문

**연락처 상세 페이지 뷰를 생성하거나 관리할 수 있는 사람은 누구인가요?**
관리자만 뷰를 생성, 편집, 관리할 수 있습니다.

**계정당 몇 개의 뷰를 생성할 수 있나요?**
계정당 최대 5개의 뷰를 생성할 수 있습니다.

**기본 뷰를 여러 개 설정할 수 있나요?**
아니요. 한 번에 하나의 기본 뷰만 활성화할 수 있습니다.

**사용자에게 다른 뷰를 배정할 수 있나요?**
네. 관리자는 사용자 관리를 통해 뷰를 배정하거나 재배정할 수 있습니다.

**대화 모듈을 다른 패널로 이동할 수 있나요?**
아니요. 대화 모듈은 중앙 패널에 고정되어 있습니다.

**액션을 오른쪽 패널로 이동할 수 있나요?**
아니요. 액션은 왼쪽 패널에 남아있거나 중앙 패널로 이동할 수 있지만, 오른쪽 패널로는 이동할 수 없습니다.

**2패널과 3패널 레이아웃 사이를 전환할 수 있나요?**
네. 관리자는 필요에 따라 두 레이아웃 옵션 사이를 전환할 수 있습니다.

**숨겨진 모듈이나 필드를 나중에 복원할 수 있나요?**
네. 제거된 모듈과 숨겨진 필드는 추가 버튼을 사용하여 다시 추가할 수 있습니다.
