# 삭제된 연락처 복원하기 또는 일괄 삭제 취소하기

하이퍼클래스의 업데이트된 연락처(Contacts) → 복원 페이지로 실수로 삭제한 리드를 빠르게 복구하세요. 새로워진 인터페이스, 다중 선택 체크박스, 명확한 연락처 컬럼, 확인 모달이 하나 또는 수백 개의 연락처를 더욱 빠르고 안전하게 복원할 수 있도록 도와줍니다. 이 가이드에서는 무엇이 바뀌었는지, 왜 중요한지, 그리고 정확히 어떻게 사용하는지 설명합니다.

## 연락처 복원 페이지란 무엇인가요?

연락처 복원 페이지는 CRM에서 제거된 연락처를 찾아오는 안전망입니다. 최근 삭제된 기록들(데이터 보관 기간 내)을 한 곳에 모아서 재가져오기나 복잡한 우회 방법 없이 되돌릴 수 있습니다. 이 페이지는 실수로 삭제한 후나 작은 정리 작업을 되돌릴 때 특히 유용합니다.

하이퍼클래스의 2025년 10월 업데이트는 더욱 빠르고 명확한 복원 경험을 제공합니다. 깔끔하고 현대적인 레이아웃, 여러 연락처를 한 번에 선택하고 복구할 수 있는 빠른 체크박스, 이름, 이메일, 전화번호, 삭제일을 보여주는 눈에 띄는 목록 컬럼, 그리고 복원할 레코드 수를 명확히 보여주는 간소화된 확인 대화상자를 볼 수 있습니다. 이러한 개선사항들이 함께 작용하여 클릭 수를 줄이고, 실수를 방지하며, 개별 및 일괄 복원을 모두 안전하고 예측 가능하게 만듭니다.

---

## 연락처 복원 페이지의 주요 장점

실용적인 장점을 이해하면 하나의 기록을 복원하든 여러 개를 복원하든 가장 빠르고 안전한 복구 경로를 선택하는 데 도움이 됩니다.

- **일괄 처리 효율성:** 새로운 체크박스를 사용해 여러 레코드를 한 번에 선택하여 더 적은 단계로 배치 복원이 가능합니다.

- **한눈에 보는 정보:** 복원을 확정하기 전에 주요 컬럼(이름, 이메일, 전화번호, 삭제일)을 검토할 수 있습니다.

- **안전망:** 확인 모달이 복구하려는 연락처 수를 정확히 요약해서 보여줍니다.

- **시간 절약:** CSV를 다시 가져오는 대신 하이퍼클래스에서 직접 복원할 수 있습니다.

- **일관성:** 복원된 연락처가 CRM 전체에 다시 나타나므로 팀이 중단 없이 계속 작업할 수 있습니다.

---

## 연락처 복원 시 중요한 정보

- **HIPAA 계정의 제한사항**: HIPAA 준수 계정에서는 엄격한 데이터 보안 규정으로 인해 삭제된 연락처를 복원할 수 없습니다.

- **시간 제한**: 연락처는 60일 이내에만 복원할 수 있습니다. 이 기간을 넘으면 데이터가 영구적으로 삭제됩니다.

- **복원 포함 항목**: 복원 시 연락처는 다음과 같은 관련 데이터와 함께 복구됩니다:
  - 기회 관리
  - 노트 및 할 일
  - 대화
  - 예약
  - 워크플로우 및 캠페인 참여 내역

## 확인 모달

명확한 확인 단계를 통해 작업 범위를 요약하여 오류를 줄입니다.

- 모달에 복원할 선택된 연락처의 총 개수가 표시됩니다.

- 한 번 클릭으로 확인하면 복원이 즉시 시작됩니다.

- 보관 기간을 초과했거나 복원할 수 없는 연락처가 있으면 완료 전에 알림을 받습니다.

## 개별/여러 연락처 복원하기

수동으로 하나 이상의 연락처를 삭제한 경우, 하이퍼클래스를 통해 복원 탭에서 쉽게 되돌릴 수 있습니다. 이 방법은 전체 일괄 작업을 되돌리지 않고 개별 연락처나 작은 배치를 복구하는 데 이상적입니다.

### 1단계: 연락처 섹션의 복원 탭에 접근하기

하이퍼클래스 대시보드의 연락처(Contacts) 섹션으로 이동하여 복원 탭을 선택하세요.

![연락처 복원 탭 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055894287/original/N0lOCzASqwWP3W_KVYRo9mmDAOVdPaYhWA.png?1760377431)

### 2단계: 복원할 연락처 찾기 및 선택하기

복원하려는 연락처를 찾아 이름 옆의 체크박스를 클릭하여 선택하세요. 개별 연락처를 선택하거나 복원 시 목록의 모든 연락처를 선택할 수 있습니다.

![연락처 선택하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055894297/original/nt8PeYyV0qXoBNddj9_OlyHCOLfZ9L_SNA.png?1760377453)

### 3단계: 연락처 복원하기

연락처 이름 옆의 **복원(Restore)** 버튼을 클릭하고 작업을 확인하세요.

![연락처 복원 확인](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055894323/original/KtxJV-goIoWfJhiGkzJ-9eMdOs48V6crBQ.png?1760377471)

## 일괄 삭제 취소하기

일괄 삭제(Bulk Delete) 기능을 사용하여 실수로 여러 연락처를 삭제한 경우, 쉽게 작업을 되돌리고 삭제된 연락처와 관련 데이터를 복원할 수 있습니다.

### 1단계: 일괄 작업 페이지로 이동하기

연락처(Contacts) 섹션으로 이동하여 네비게이션 메뉴에서 일괄 작업(Bulk Actions)을 클릭하세요. 일괄 작업 페이지에는 일괄 삭제를 포함하여 계정에서 수행된 모든 일괄 작업의 기록이 표시됩니다.

![일괄 작업 페이지 이동](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055894368/original/KpiEY7ySp9yvf1RdXUA7g91o6hiVDEDkXw.png?1760377570)

### 2단계: 일괄 삭제 작업 찾기

일괄 작업 목록을 스크롤하여 되돌리려는 특정 일괄 삭제 작업을 찾고 점 세 개 > 복원을 클릭하세요.

![일괄 삭제 작업 찾기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055894475/original/ipf-dKEEzpP9NpqrWyS574uVmgzdLrJkBw.png?1760377702)

### 3단계: 복원 옵션을 클릭하여 일괄 삭제 복원하기

팝업에서 **복원(Restore)**을 다시 선택하세요.

![일괄 삭제 복원 확인](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055894506/original/wEyh1yYeBSrF2UgJ34OsyvjdcYyCbqoZag.png?1760377726)

**참고**: 복원 프로세스가 즉시 시작됩니다. 일괄 작업 페이지에서 진행 상황을 모니터링할 수 있으며, 상태가 복원 진행 상황을 나타내도록 변경됩니다(예: "진행 중" 또는 "완료"). 프로세스가 완료된 후 연락처 섹션으로 돌아가서 복원된 연락처를 확인하세요.

## 자주 묻는 질문

**Q: 삭제 후 언제까지 연락처를 복원할 수 있나요?**
최대 60일까지입니다. 그 이후에는 연락처가 영구적으로 제거되어 복구할 수 없습니다.

**Q: 다른 사람이 삭제한 연락처도 복원할 수 있나요?**
네, 60일 기간 내이고 관리자 권한이 있다면 가능합니다.

**Q: 관련 데이터는 어떻게 되나요?**
복원된 연락처는 별도로 삭제되지 않은 한 대화나 기회 관리 등의 관련 데이터를 되찾습니다.

**Q: 정확히 무엇이 복원되나요?**
독립적으로 삭제되지 않은 한 모든 관련 데이터(대화, 할 일 등)가 복원됩니다.

**Q: "모두 선택"은 결과의 모든 페이지에 있는 연락처를 포함하나요?**
현재 목록 보기의 모든 연락처가 포함됩니다. 여러 페이지가 있는 경우, 페이지당 행 수를 늘리거나 복원하기 전에 후속 페이지에서 선택을 반복하세요.

**Q: 일괄 작업의 복원 프로세스는 얼마나 걸리나요?**
복원되는 연락처 수에 따라 시간이 달라집니다. 대용량 일괄 작업의 경우 완료까지 몇 분이 걸릴 수 있습니다.

**Q: HIPAA 준수 계정에서 연락처를 복원할 수 있나요?**
아니요, HIPAA 준수 계정은 데이터 보안상의 이유로 삭제된 연락처 복원을 허용하지 않습니다.

**Q: 연락처를 복원할 수 없는 이유가 무엇인가요?**
다음 중 하나가 가능합니다: 연락처가 2개월 전에 삭제되어 더 이상 복구할 수 없거나, HIPAA 준수 계정에서는 연락처 복원을 허용하지 않습니다.
