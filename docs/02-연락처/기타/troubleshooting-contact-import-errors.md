# CSV 연락처 가져오기 오류 해결 방법

이 가이드는 Hyperclass에서 CSV 파일로 연락처를 가져올 때 발생할 수 있는 오류를 해결하는 방법을 설명합니다. 각 오류 코드마다 구체적인 원인과 해결책을 제공하여, 대량 가져오기가 깔끔하고 원활하게 성공할 수 있도록 도와드립니다.

## 대량 가져오기란?

대량 가져오기는 CSV 파일을 사용하여 많은 양의 연락처 데이터를 빠르게 업로드할 수 있는 기능입니다. 온보딩, 리드 마이그레이션, 목록 동기화에 매우 유용한 도구지만, 형식 오류가 있으면 진행이 차단될 수 있습니다. 이 가이드는 그런 오류를 해결하는 데 도움을 드립니다.

아래 동영상은 특정 연락처 가져오기 일괄 작업과 연결된 오류 로그로 이동하는 방법과 오류를 해결하기 위해 작업하는 방법을 다룹니다:

---

## **성공적인 CSV 가져오기의 핵심 이점**

대량 가져오기 문제 해결을 마스터하면 시간을 절약하고 답답한 장애물을 피할 수 있습니다. 중요한 이유는 다음과 같습니다:

- 재작업을 줄여 시간 절약
- 중복되거나 충돌하는 연락처 레코드 생성 방지
- 깨끗하고 검증된 연락처 데이터 유지
- 워크플로우 자동화를 차단하는 가져오기 실패 방지
- 정확한 소유자/사용자 할당 보장
- 캠페인 개인화 및 태그 보존

---

## **CSV 가져오기 문제 해결 체크리스트**

일반적인 CSV 오류를 방지하기 위해 가져오기 전에 이 체크리스트를 사용하세요:

- 파일에 시트가 하나만 있는지 확인
- 모든 필수 필드가 채워져 있고 올바르게 형식화되었는지 확인
- 이메일, 전화번호, 날짜를 올바르게 형식화
- 각 행마다 고유한 연락처 ID 사용
- 열거형/드롭다운 값이 정확히 일치하는지 확인
- 소유자 열에 유효한 Hyperclass 사용자 이메일 사용
- 신뢰할 수 있는 기본 열(이메일 또는 연락처 ID 등) 선택

| | 1003 | 잘못된 시트 수 | 가져오기 파일에 시트가 두 개 이상 포함되어 있습니다. 데이터가 가져오기되지 않았습니다. | 가져오기 파일에서 추가 시트를 모두 제거한 다음 다시 가져오세요. Hyperclass로 가져오기 위한 추가 파일 요구 사항을 검토하세요. |
| | 1004 | 잘못된 필수 속성 | 가져오기 파일의 필수 속성에 잘못된 값이 포함되어 있습니다. 해당 행의 데이터가 가져오기되지 않았습니다. | 특정 속성 또는 속성 유형의 형식 요구 사항을 확인하고, 올바른 형식에 맞게 파일을 업데이트한 다음 다시 가져오세요. |
| | 1005 | 잘못된 열거형 옵션 | 가져오기 파일에 열거형 속성의 기존 옵션과 일치하지 않는 값이 포함되어 있습니다. 이는 다중 선택, 드롭다운 선택, 라디오 선택 및 체크박스 유형 속성에 적용됩니다. 영향을 받은 레코드는 여전히 가져오기되었지만 열거형 속성에는 값이 포함되지 않습니다. | 가져오기 파일을 Hyperclass의 속성과 비교하여 잘못된 값을 식별하세요. 기존 옵션과 일치하도록 가져오기 파일의 값을 업데이트하거나 해당 값을 속성의 커스텀 옵션으로 추가할 수 있습니다. 완료되면 파일을 다시 가져오거나 영향을 받은 레코드의 속성 값을 수동으로 업데이트하세요. |
| | 1006 | 잘못된 연락처 ID | 가져오기 중에 연락처 ID를 사용하여 기존 레코드를 업데이트하는 체크박스가 선택되었습니다. 그러나 파일에 나열된 연락처 ID가 Hyperclass의 기존 ID와 일치하지 않습니다. 레코드가 가져오기되지 않았습니다. | [객체] ID가 올바른지 확인하세요. 올바른 객체 ID를 얻기 위해 기존 레코드를 내보내기해야 할 수 있습니다. Hyperclass에서 기존 객체를 업데이트하거나 연결하지 않는 경우 가져오기 프로세스 중에 체크박스를 선택하지 마세요. |
| | 1007 | 파일을 찾을 수 없음 | 선택한 파일을 찾을 수 없습니다. 데이터가 가져오기되지 않았습니다. | 파일을 컴퓨터에 저장했는지, 올바른 파일 형식인지 확인하세요. 확인 후 파일을 다시 업로드하세요. |
| HL/파일의 중복 항목 | 1008 | 이메일 검증 실패 | 가져오기 파일에 일반적인 이메일 주소 형식(예: name@domain.com)에 맞지 않는 이메일 주소가 포함되어 있습니다. 가져오기에서 레코드를 생성하거나 업데이트하지 않았습니다. | 연락처의 경우, '@' '.com'이 포함된 올바른 형식인지 확인한 후 가져오기 파일의 올바른 이메일로 업데이트한 다음 파일을 다시 가져오세요. |
| | 1009 | 파일에서 고유 속성 값이 비어 있음 | 가져오기 파일에 고유 식별자(기본 열)의 레코드에 값이 없어 처리할 수 없는 데이터가 있는 행이 포함되어 있습니다. | 데이터 값이 가져오기하는 속성의 올바른 형식과 일치하는지 확인한 다음 파일을 다시 가져오세요. 영향을 받은 레코드의 속성 값을 수동으로 업데이트할 수도 있습니다. |
| | 1010 | HL에서 고유 속성 값 중복 | 기본 열의 동일한 값이 여러 레코드에 대해 존재합니다. 데이터는 모든 HL 레코드에 대해 가져오기되었지만 Hyperclass에서 동일한 레코드의 중복이 있을 수 있습니다. | 다른 열 헤더를 기본 열로 선택하거나 실수를 수정한 다음 파일을 다시 가져오세요. |

| | 1012 | 파일에서 연락처 ID 중복 | 파일 내에서 동일한 연락처 ID가 여러 번 사용되었습니다. 영향을 받은 레코드는 Hyperclass에서 업데이트되거나 연결되지 않았습니다. | 해당 연락처 ID가 있는 행이 하나만 있도록 가져오기 파일을 업데이트하세요. Hyperclass에 포함하려는 가장 최신 정보가 있는 행을 유지하세요. 중복을 제거한 후 파일을 다시 가져오세요. |

## 자주 묻는 질문

**Q: 어떤 파일 형식을 업로드해야 하나요?**

.csv 파일만 허용됩니다. Excel이나 Google 시트에서 작업하는 경우, 먼저 데이터를 CSV 파일로 내보내기하세요.

**Q: 대량 가져오기를 취소할 수 있나요?**

직접적인 "취소" 옵션은 없지만, 업로드 중에 가져온 연락처에 태그를 지정할 수 있습니다. 필요한 경우 해당 태그를 사용하여 필터링하고 삭제하세요.

**Q: 가져오기할 수 있는 최대 파일 크기는 얼마인가요?**

제한은 20MB 또는 10,000행 중 먼저 도달하는 것입니다.

**Q: 업로드하기 전에 전화번호의 유효성을 검사하려면 어떻게 해야 하나요?**

스프레드시트 공식이나 온라인 도구를 사용하여 E.164 형식을 따르는지 확인하세요.

**Q: 기본 열로 무엇을 선택해야 하나요?**

여러 연락처에서 반복되지 않는 고유 식별자(이메일 또는 연락처 ID 등)를 사용하세요.

**Q: 내 CSV 목록이 혼합되어 있습니다. 일부 연락처는 이메일이나 전화번호만 있고, 일부는 이메일과 전화번호가 모두 있습니다. 어떻게 진행해야 하나요?**

혼합된 CSV의 경우 '보조 선호도' 옵션을 사용하세요. 시스템은 첫 번째 선호도를 검색한 다음 첫 번째를 찾을 수 없으면 두 번째를 검색합니다. 위 예시에서 첫 번째 선호도는 '이메일'이고 두 번째는 '전화번호'입니다.

이제 보조 필드를 기반으로 기존 연락처를 찾을 수 있으므로, 연락처 레코드에 전화번호만 있는 경우에도 업로드할 수 있습니다!

**Q: 일부 연락처에 여러 개의 이메일이나 전화번호가 있습니다. 어떻게 진행해야 하나요?**

연락처에 여러 전화번호나 이메일이 있는 경우, 연락처를 여러 개로 분할하여 업로드하세요. 예시:

**두 개의 전화번호 열이 있는 원본:**
Josephine
810-292 9388
810-374-9840
josephine_darakjy@darakjy.org

**업로드를 위한 정리된 버전:**
Josephine
810-292-9388
josephine_darakjy@darakjy.org

Josephine
810-374-9840

전화번호를 고유 필드로 사용하여 연락처를 업로드할 때 중복이 생성되므로 다른 전화번호가 있는 두 번째 리드의 이메일은 비워두세요.

**Q: Keap을 사용하는 사람들을 위한 중요한 참고사항:**

Keap의 CSV 내보내기에서 전화 필드는 Hyperclass과 호환되지 않습니다. 괄호 '()'와 대시 '-'로 인해 호환되지 않습니다. CSV 파일의 전체 열을 강조 표시하고 숫자로 형식화되었는지 확인하고, 괄호와 대시를 모두 제거한 다음 CRM의 데이터를 손상시키지 않고 가져올 수 있습니다.

**Q: 전화번호를 가져올 때 오류 코드 1007 '잘못된 필수 속성' 오류가 발생합니다:**

번호를 가져올 때 하위 계정 국가 외부의 번호에 국가 코드를 추가하세요. 하위 계정 국가가 미국으로 설정된 경우, 가져온 전화번호에 +1(미국 국가 코드)을 추가할 필요가 없습니다. 그러나 미국 외부 번호를 가져오는 경우, [국가 코드](https://countrycode.org/)를 추가해야 합니다. 즉, 비즈니스 위치를 미국으로 설정한 하위 계정의 경우 영국에 전화하려면 '+44'를 추가해야 합니다. 하위 계정의 국가가 미국이 아닌 다른 국가(영국, 호주 등)로 설정된 경우, 미국 번호로 전화/SMS를 하려면 CSV 파일 내에서 해당 국가의 지역 코드에 '+1'을 추가해야 합니다.

**Q: DND 열이 있는 CSV를 가져오려고 하는데 모든 연락처가 모든 채널에 대해 DND로 설정됩니다.**

CSV 내에 DND 열이 있는 연락처를 가져올 때, 연락처가 모든 채널에 대해 DND가 된다는 점에 주의하세요. 특정 채널에 대해 DND를 설정하려면 먼저 워크플로우를 설정하고 각 채널에 대해 태그를 사용하세요.

예: John을 SMS만 DND로 설정하고 싶다면 'DND SMS' 태그가 추가될 때 연락처를 DND하는 워크플로우를 만들겠습니다. 그래서 이제 CSV에서 John의 이름 옆에 'DND SMS' 태그를 추가하겠습니다. 가져올 때 이것은 John을 SMS에 대해서만 DND하는 자동화를 실행합니다.

**Q: 노트가 있는 CSV를 가져오려고 하는데 연락처 레코드당 하나 이상의 노트를 생성할 수 없습니다.**

연락처 노트를 가져올 때 연락처 레코드당 5000자 제한으로 하나의 노트만 가질 수 있습니다. 하나 이상의 노트가 있거나 허용된 양보다 많은 문자가 있는 경우 [대형 텍스트 커스텀 필드](../../23-레거시-자동화/Logic-&amp;-Fulfillment/how-to-use-custom-fields.md)를 사용하세요.

**Q: 노트가 있는 CSV를 내보내려고 합니다.**

현재 노트에 대한 CSV 가져오기 제한은 없으며, 현재 내보내기 동작에 따라 마지막 노트에 대해 최대 255자를 지원합니다.
