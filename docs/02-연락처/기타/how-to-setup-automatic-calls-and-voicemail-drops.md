# 링리스 음성메일과 음성메일 드롭 설정 가이드

음성메일은 고객에게 다가가서 사업을 알리는 훌륭한 방법입니다. 음성메일 드롭(Voicemail Drop)은 메시지가 제대로 전달되는지 확인하는 빠르고 저렴한 방법이에요. 음성메일 드롭이 매출 증가와 고객 만족도 향상의 첫걸음이 될 수 있습니다.

# 음성메일 드롭이란?

음성메일 드롭(링리스 음성메일이라고도 함)은 실제 전화를 걸거나 벨을 울리지 않고 미리 녹음된 음성메일 메시지를 연락처(Contacts)에게 전달하는 방법입니다. 예약 업데이트, 리드(Lead) 육성, 재접촉 등에 빠르고 일관된 후속 조치를 위해 주로 사용되며, 개인적이면서도 효율적인 아웃리치를 유지할 수 있어요.

---

## 링리스 음성메일(음성메일 드롭)의 주요 이점

- **전화를 울리지 않고 메시지 전달:** 연락처가 최소한의 방해로 음성메일 메시지를 받을 수 있습니다.

- **개인적인 느낌의 아웃리치 확장:** 미리 녹음된 음성메일을 사용해서 인간적인 톤을 유지하면서 대량 처리가 가능해요.

- **후속 조치 속도와 반응성 개선:** 즉시 전달되어 주요 액션(Action) 후 더 빠른 응답이 가능합니다(리드 업데이트, 예약, 노쇼 등).

- **다양한 실제 워크플로우 지원:** 예약 확인, 노쇼 복구, 리드 육성, 재접촉 시퀀스에 유용합니다.

## 사전 준비사항

- 하위 계정(Sub-account)에서 전화번호를 구매해야 합니다([LC Phone 가이드](what-is-lc-phone-system-.md) 또는 Twilio 가이드 참조)
- mp3/wav 파일(64kbps)로 미리 녹음된 오디오 파일이 있어야 합니다([자세한 내용은 여기](voicemail-drop-file-conversion.md))
- 음성메일을 남기기 위해 자동화된 통화에 대한 사용자 동의를 받아야 합니다.

## **워크플로우를 사용한 링리스 음성메일 설정**

워크플로우(Workflow)를 사용하면 연락처가 특정 조건(상태 업데이트나 예약 변경 등)을 충족할 때 자동으로 링리스 음성메일을 보낼 수 있어서 후속 조치가 즉시 일관되게 이뤄집니다. 워크플로우에서 설정하는 방법을 알아보세요.

- 
### 워크플로우 열기

하위 계정에서 좌측 네비게이션 메뉴의 "Automation(자동화)"을 클릭하고 기존 워크플로우를 열거나 "+ Create Workflow(워크플로우 만들기)" → "+ Start from scratch(처음부터 시작)"를 클릭하세요.

![워크플로우 생성](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064864511/original/B_TKey_IwnM_Wci7egbX8TtW5jkGeNxfzQ.png?1770984750)

- 
### **트리거 선택**

음성메일 드롭(링리스 음성메일) 액션을 위한 워크플로우 트리거(Trigger)를 선택하세요. 이 예시에서는 Contact Changed(연락처 변경됨) 트리거를 사용했습니다.
링리스 음성메일/음성메일 드롭을 언제 전달할지에 따라 다른 관련 트리거를 사용할 수도 있습니다(예: 예약 확인 후, 노쇼 후, 상태 업데이트 후 등).

![트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064778674/original/PgjOYymo0ls79mvb1S-2KCdzdFZSIJ4jdA.gif?1770899845)

- 
### **(선택사항) 연락처를 사용자에게 배정**

워크플로우 액션을 추가하려면 + 버튼을 클릭하세요. Assign to User(사용자에게 배정)를 검색해서 선택합니다.
이 연락처들을 전담 팀원에게 라우팅하고 싶다면 **드롭다운에서 사용자를 선택**하세요. 아무도 배정되지 않으면 비즈니스 프로필(Business Profile)에 저장된 회사 번호로 통화가 라우팅됩니다.

Only Apply To Unassigned Contacts(미배정 연락처에만 적용): 아직 담당자가 없는 연락처에만 배정하고 싶다면 이 옵션을 활성화하세요. 워크플로우 빌더로 돌아가기 전에 Save Action(액션 저장)을 클릭하세요.

![사용자 배정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064778717/original/m8ZBXZCdN7g8gnX12wzl-SAV6ZHS1E834g.gif?1770899875)

- 
### **음성메일 액션 추가(링리스 음성메일/음성메일 드롭)**

+ 버튼을 다시 클릭해서 다른 액션을 추가하세요. Voicemail(음성메일)을 스크롤하거나 검색해서 선택합니다. Upload the File(파일 업로드)를 클릭하세요.
미리 녹음된 mp3/wav(64kbps) 오디오 파일을 업로드합니다. 파일이 업로드되면 반드시 Save Action(액션 저장)을 클릭하세요.

![음성메일 액션 추가](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064778880/original/NggqqtMYKYxJC82hJtPTFDc8baIOTJi26g.png?1770899965)

## **활용 사례**

예약 확인, 노쇼 복구, 신규 리드 육성 등 적절한 시점에 링리스 음성메일을 보내기 위해 사용할 수 있는 몇 가지 일반적인 워크플로우 + 트리거 조합을 소개합니다.

| 활용 사례 / 워크플로우 | 권장 트리거 | 제안 액션 (예시) | 목표 |
|---|---|---|---|
| 예약 확인 후속 조치 | **Appointment Status Changed → Confirmed(예약 상태 변경됨 → 확인됨)** | (선택사항) **Assign to User(사용자에게 배정)** → **Voicemail(음성메일)**로 "예상 사항 + 일정 변경 방법" 안내 | 혼란 줄이고 출석률 개선 |
| 노쇼 복구 | **Appointment Status Changed → No Show(예약 상태 변경됨 → 노쇼)** | **Voicemail(음성메일)**로 친근한 재예약 안내 + 다음 단계 | 놓친 예약 복구 |
| 신규 리드 육성 (필드/상태 변경) | **Contact Changed(연락처 변경됨)** (예: Lead Status가 *New Lead* / *Qualified*가 됨) | **Assign to User(사용자에게 배정)** → **Voicemail(음성메일)**로 배정된 담당자 소개 메시지 | 빠른 "인간적 터치" 추가 |
| 폼 제출 후 감사 인사 | **Form Submitted(폼 제출됨)** | **Voicemail(음성메일)**로 감사 인사 + 응답 시간 안내 | 신뢰와 참여도 증가 |
| 콜드 리드 재접촉 | **Contact Tag Added(연락처 태그 추가됨)** (예: *Cold Lead*, *Re-engage*) | **Voicemail(음성메일)**로 간단한 "아직 관심 있으신가요?" 메시지 + CTA | 잠자는 리드 재활성화 |

## 자주 묻는 질문

### 음성메일 파일이 작동하지 않아요.

1. [https://online-audio-converter.com/](https://online-audio-converter.com/)에 가서 음성메일 파일을 업로드하세요. 아니면 여기서 새 파일을 만들 수도 있습니다: [https://www.rev.com/onlinevoicerecorder](https://www.rev.com/onlinevoicerecorder)

2. 64kbps MP3를 선택하세요. 낮을수록 좋습니다.

3. Hyperclass 음성메일 캠페인/워크플로우에 다시 업로드하세요.

### 음성메일 드롭이 합법인가요?

음성메일을 남기기 전에 사용자의 동의가 있다면 음성메일 드롭은 합법입니다. 그렇지 않으면 불법이에요. TCPA는 자동 다이얼링 시스템과 사전 녹음 메시지를 사용하는 커뮤니케이션 형태로 간주되므로 링리스 음성메일 드롭에도 적용됩니다. 따라서 링리스 음성메일 드롭을 보내기 전에 수신자의 사전 명시적 동의를 받아야 합니다. 그렇지 않으면 연방통신위원회(FCC)나 개인으로부터 소송과 벌금에 직면할 수 있어요.

### 더 많은 음성메일을 남길 수 없는 이유는 무엇인가요?

음성메일 드롭 제한은 사용 시간이 늘어날수록 증가합니다.

1일차: 하루 100회 드롭
2일차: 하루 200회 드롭
3일차: 하루 300회 드롭 ...
7일차: 하루 700회 드롭
7일차 이후: 하루 2000회 드롭

이 제한을 늘리고 싶다면 고객지원팀에 문의해 주세요.

**링리스 음성메일 드롭을 보내기 전에 추가 등록이나 인증이 필요한가요?**

하위 계정에 LC 전화번호(또는 Twilio 번호)가 이미 설정되어 있고 연락처로부터 사전 명시적 동의를 받았다면 바로 링리스 음성메일 드롭을 보낼 수 있습니다. **TCPA** 및 해당 지역의 기타 관련 법률 준수 외에는 추가 법적 등록이 필요하지 않아요. 하지만 요구사항은 지역마다 다를 수 있으므로 항상 해당 지역의 텔레마케팅 규정을 확인하세요.

요약: 음성메일 드롭은 통신사의 기술적 특성을 이용하는 것으로, 모든 음성메일 드롭이 작동하는 방식이며, 그 결과 약 70%의 성공률을 보입니다.
