# CSV 블로그 가져오기란?

CSV 블로그 가져오기는 Hyperclass 블로그(Blogs) 섹션의 대량 업로드 도구로, 올바른 형식의 CSV나 Excel 파일을 발행 준비가 완료된 블로그 글로 변환합니다. 헤더를 검증하고, 작성자와 카테고리를 자동으로 매칭하며, 3단계 검토 과정을 통해 수동으로 복사-붙여넣기 작업을 거의 완전히 제거합니다.

## CSV 블로그 가져오기의 주요 장점

- **원활한 마이그레이션**: 외부 플랫폼에서 콘텐츠를 수동으로 복사하는 번거로움을 제거합니다.

- **대량 가져오기**: 한 번에 최대 60개의 글을 업로드하여 온보딩 속도를 높입니다.

- **스마트 정리**: 작성자와 카테고리를 자동으로 생성하거나 매칭하여 수동 정리 작업을 최소화합니다.

- **유연한 일정 관리**: Hyperclass가 업로드된 파일에서 발행 날짜를 자동으로 해석합니다.

- **내장 SEO 미리보기**: 발행 전에 제목, 메타 설명, 소셜 이미지를 확인할 수 있습니다.

- **오류 없는 설정**: 다운로드 가능한 샘플 CSV를 사용하여 헤더나 형식 실수를 방지합니다.

- **즉시 검증**: 인라인 오류 메시지가 누락되거나 잘못된 데이터를 즉시 강조 표시합니다.

## 중요 사항

- **자동 검증**: Hyperclass가 업로드 시 CSV 헤더를 스캔하고 누락되거나 맞춤법이 틀린 필드를 표시하여 빠른 수정을 도와줍니다.

- **필수 필드**: 모든 필수 헤더가 있어야 하지만, 개별 셀 값은 비워둘 수 있습니다.

- **오류 감지**: 인라인 오류 표시가 누락된 이미지, 잘못된 날짜, 긴 메타 설명과 같은 문제를 강조합니다.

- **스마트 일정 관리**: 과거 날짜 = 발행됨 / 미래 날짜 = 예약됨 / 빈 날짜 = **초안**.

- **파일 크기**: 브라우저 시간 초과를 피하기 위해 파일을 **25MB** 이하로 유지하고, 더 큰 파일은 작은 배치로 나누세요.

- **CSV 형식**: 잘못 형식화된 파일은 수정될 때까지 인라인 오류 메시지를 표시합니다.

- **시간대 처리**: 가져오기 도구는 각 하위 계정의 시간대를 고려하여 글이 올바른 현지 시간에 발행되도록 합니다.

## CSV 블로그 가져오기 사용법

### 1단계: 블로그 접속 및 대상 블로그 선택

- 하위 계정에 로그인합니다.

- 좌측 메뉴에서 **Sites(사이트)** → **Blogs(블로그)**를 클릭합니다.

![블로그 메뉴 접속](https://jumpshare.com/share/ZinFPo1RVnGZCTMRniu4+/Screen+Shot+2025-10-06+at+5.31.49+PM.png)

- 새 블로그 사이트를 만들려면: **Create Blog(블로그 만들기)** 드롭다운 클릭 → **Create a blog(블로그 만들기)** 또는 **Create a Blog from Templates(템플릿으로 블로그 만들기)**를 선택합니다.

- 기존 블로그를 사용하려면: **블로그 사이트 이름**을 클릭하여 엽니다.

![블로그 선택 과정](https://jumpshare.com/share/9REccb1888mfPVeLhq1g+/GIF+Recording+2025-10-06+at+5.51.05+PM.gif)

### 2단계: CSV 가져오기 시작

- **New Post(새 글)** 드롭다운을 클릭합니다.

- **Import Blog Post from CSV(CSV에서 블로그 글 가져오기)** 옵션을 선택합니다.

![CSV 가져오기 선택](https://jumpshare.com/share/rFQDvrSv5q52S7owU4L3+/Screen+Shot+2025-10-06+at+6.17.22+PM.png)

### 3단계: 파일 준비 및 업로드

- **Download sample(샘플 다운로드)**를 클릭하고 **CSV** 또는 **XLSX 템플릿**을 작성하여 헤더 불일치를 방지합니다.

![샘플 다운로드](https://jumpshare.com/share/JBpiBZHh2eesb3OXfpWx+/Screen+Shot+2025-10-06+at+6.22.04+PM.png)

- **Click to Upload or Drag & Drop(업로드하려면 클릭하거나 드래그 앤 드롭)** → 준비한 파일을 선택합니다.

![파일 업로드 과정](https://jumpshare.com/share/o4bLxrXRbgwCKkYPrKCA+/GIF+Recording+2025-10-06+at+6.33.56+PM.gif)

- **File summary(파일 요약)**(파일 이름, 총 글 수)를 확인합니다.

- **Formatting Tips(형식 팁)**을 살펴봅니다.

- **Continue(계속)**를 클릭합니다.

![파일 요약 및 계속](https://jumpshare.com/share/duunvK3U2jGc4718cuLQ+/Screen+Shot+2025-10-06+at+6.40.11+PM.png)

### 4단계: 글 미리보기, 편집 및 일정 설정

- 블로그 카드를 클릭하여 **Preview(미리보기)**를 열고 (데스크톱 및 모바일 보기를 확인합니다).

- **연필 아이콘**을 클릭하여 필요에 따라 콘텐츠를 편집합니다.

- 날짜 선택기로 **Publish/Schedule date(발행/예약 날짜)**를 설정합니다:
  - **과거 날짜** → 발행됨으로 가져오기
  - **미래 날짜** → 예약됨으로 가져오기
  - **빈 날짜** → 초안으로 가져오기

참고: 잘못된 날짜는 실수로 발행되는 것을 방지하기 위해 **초안**으로 되돌아갑니다.

![미리보기 및 편집 과정](https://jumpshare.com/share/D1YqRuPC3Xvd1EenQ23Q+/GIF+Recording+2025-10-06+at+6.59.47+PM.gif)

### 5단계: 가져오기 및 확인

- **Import(가져오기)**를 클릭합니다.

![가져오기 버튼](https://jumpshare.com/share/JyaHfuT5HGJimwAS2kvW+/Screen+Shot+2025-10-06+at+7.08.03+PM.png)

- 확인 모달에서 **요약**(예: 1개 발행됨, 1개 초안)을 검토합니다.

- **Import(가져오기)**를 클릭하여 확인합니다.

![가져오기 확인](https://jumpshare.com/share/foSNSJ3iTWv9CsEiwXeP+/Screen+Shot+2025-10-06+at+7.10.23+PM.png)

**오류가 있는 글(행별 보고서)**

검토 중에 행이 검증에 실패하면 **Posts with Errors(오류가 있는 글)**를 열어 실패한 내용과 이유에 대한 행별 보고서를 확인할 수 있습니다.

각 실패한 행에는 다음이 표시됩니다:
- 글 제목(또는 행 식별자)
- 검증에 실패한 필드
- 수정해야 할 내용을 설명하는 검증 메시지

## 자주 묻는 질문

**Q: 한 번에 가져올 수 있는 최대 글 수는?**
CSV/XLSX 파일당 최대 60개 글입니다.

**Q: 가져오기 도구가 내 이미지를 다운로드하고 호스팅하나요?**
네. 전체 이미지 URL이나 공개 인터넷에서 접근 가능한 상대 경로를 제공하면, Hyperclass가 이미지를 가져와서 저장합니다.

**Q: 가져온 후 글을 편집할 수 있나요?**
물론입니다. 가져온 글은 다른 블로그 글처럼 작동합니다. 편집기를 열어 콘텐츠를 조정하고 다시 발행하세요.

**Q: 같은 제목으로 글을 다시 가져오면 어떻게 되나요?**
새 초안이 생성됩니다. 중복은 자동으로 합쳐지지 않습니다.

**Q: 파일 크기 제한이 있나요?**
브라우저 시간 초과를 피하기 위해 파일을 25MB 이하로 유지하세요. 최적의 성능을 위해 큰 파일은 작은 배치로 나누세요.

**Q: 발행되지 않은 초안도 60개 글 제한에 포함되나요?**
네. CSV의 모든 행은 최종 상태와 관계없이 60행 제한에 포함됩니다.

**Q: 가져오기를 취소할 수 있나요?**
원클릭 롤백은 없습니다. 가져온 글을 수동으로 삭제하거나 블로그 글 목록에서 일괄 삭제를 사용하세요.

**Q: CSV에 기존 작성자나 카테고리를 포함하지 않으면 어떻게 되나요?**
시스템이 가져오기 과정에서 자동으로 생성합니다.

**Q: 발행 날짜를 비워두면 어떻게 되나요?**
글이 초안으로 가져와집니다.

**Q: 오류가 있으면 CSV를 다시 업로드할 수 있나요?**
네. 파일의 오류를 수정하고 언제든지 다시 업로드할 수 있습니다.

**Q: 어떤 플랫폼에서 마이그레이션할 수 있나요?**
구조화된 콘텐츠를 CSV나 Excel로 내보낼 수 있는 모든 플랫폼(예: Blogger, Wix, WordPress 등)입니다.

**Q: 실패한 오류를 해결하고 다시 시도하려면?**
**Posts with Errors(오류가 있는 글)**를 엽니다. 실패한 행을 선택하여 메시지를 확인합니다. (인라인 편집이 지원되는 경우) 행을 업데이트하거나, 원본 CSV/XLSX를 수정하고 다시 업로드합니다. 오류를 해결한 후 가져오기를 다시 시도합니다.
