# 음성 AI 로그: 통화 검토 및 디버깅

AI가 대화를 어떻게 처리하는지 이해하는 것은 성능을 향상시키고 문제를 빠르게 해결하는 데 필수적입니다. 에이전트 로그는 모든 상호작용을 단계별로 명확하게 보여주어 대화를 검토하고, 실행을 추적하며, 자신있게 문제를 해결할 수 있도록 도와줍니다. 에이전트 로그는 AI 상호작용에 대한 완전한 가시성을 제공하여 통화 중 무슨 일이 일어났는지, 지연이 어디서 발생했는지, 그리고 결정이 어떻게 내려졌는지를 이해할 수 있게 해줍니다. 이를 통해 문제 디버깅, 성능 최적화, 전반적인 경험 개선이 더 쉬워집니다.

# **음성 AI 에이전트 로그란 무엇인가요?**

에이전트 로그는 대화 기록, 실행 단계, 타임스탬프, 시스템 레벨 세부 정보를 보여주는 AI 상호작용의 중앙집중식 뷰입니다. 추측 없이 상호작용을 처음부터 끝까지 분석할 수 있게 해줍니다.

---

## 음성 AI 에이전트 로그의 주요 이점

에이전트 로그는 다음을 제공하여 AI 동작을 더 잘 이해하고 최적화할 수 있도록 도와줍니다:

- **대화 가시성:** 사용자와 AI 간의 전체 상호작용을 확인
- **실행 추적:** 정확한 타임스탬프와 함께 각 단계를 추적
- **전달 인사이트:** 상담원 인수인계가 어떻게 그리고 언제 발생하는지 식별
- **지연 시간 분석:** 대화에서 느린 단계 감지
- **데이터 검사:** 원시 데이터와 형식화된 입력/출력 데이터 보기
- **실패 감지:** 문제가 발생한 지점을 빠르게 식별

## **자주 묻는 질문**

**Q: 통화 녹음이 보이지 않는 이유는 무엇인가요?**
통화 녹음은 상호작용에 사용된 전화번호에 대해 녹음이 활성화되어 있는 경우에만 나타납니다.

**Q: 에이전트 로그를 무엇에 사용할 수 있나요?**
대화 검토, 실행 단계 추적, 문제 디버깅, 성능 분석, 실패 식별에 사용할 수 있습니다.

**Q: 대화 뷰와 실행 타임라인의 차이점은 무엇인가요?**
대화 뷰는 실제 발생한 상호작용을 보여주고, 실행 타임라인은 그 뒤의 기술적 단계들을 보여줍니다.

**Q: Raw(원시)와 Parsed(파싱됨)의 차이점은 무엇인가요?**
Raw는 원본 시스템 데이터를 보여주고, Parsed는 더 깔끔하고 읽기 쉬운 버전을 표시합니다.

**Q: 상담원 전달을 추적할 수 있나요?**
예, 어떤 상담원이 전달을 시작했는지, 어떤 상담원이 인수했는지, 그리고 성공했는지를 확인할 수 있습니다.

**Q: 실패한 상호작용을 어떻게 문제 해결하나요?**
로그 세부사항을 열고, 실행 타임라인을 검토하며, 실패가 발생한 단계를 검사하세요.
