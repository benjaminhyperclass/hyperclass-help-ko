# 외부 폼 페이지뷰 추적 완전 가이드

외부 추적 스크립트(External Tracking Script)를 사용하면 계정 내에서 호스팅되지 않는 웹사이트에서 페이지뷰를 기록하고 호환되는 폼 제출을 추적할 수 있습니다. WordPress, Shopify, Wix, Webflow, 커스텀 HTML 웹사이트 등의 플랫폼이 포함됩니다.

추적 스크립트를 설치하면 방문자 활동을 캡처하고, 전환을 올바르게 어트리뷰션하며, 제출이 발생했을 때 자동화를 트리거할 수 있습니다.

다음과 같은 경우에 이 설정을 사용하세요:

- 외부 웹사이트에 플랫폼 폼을 임베드하는 경우
- 표준 HTML `<form>`을 사용하는 경우 (iframe 기반 아님)
- 어트리뷰션을 위해 페이지뷰를 추적하고 싶은 경우
- 폼 제출 시 워크플로우가 트리거되기를 원하는 경우

iframe 기반 또는 위젯 기반의 제3자 폼은 지원되지 않습니다.

폼은 페이지의 DOM에 직접 존재하고 유효한 HTML `<form>` 요소를 사용해야 합니다.

---

## 외부 추적 스크립트란?

외부 추적 스크립트는 계정에 고유하게 생성되는 JavaScript 스니펫입니다.

외부 웹사이트에 설치하면 다음과 같은 기능을 수행합니다:

- 페이지뷰 기록
- 호환되는 폼 제출 추적
- 매핑된 커스텀 필드 값 캡처
- 폼 제출 전 익명 활동 저장
- 추적된 이벤트를 CRM으로 전송하여 보고서 작성 및 자동화에 활용

외부 웹사이트와 CRM 사이의 연결 역할을 합니다.

---

## **중요한 동작 특징**

폼 제출이 발생하기 전까지는 연락처나 활동 기록이 나타나지 않습니다.

동작 방식:

- 페이지뷰는 익명으로 추적됩니다
- 익명 활동은 브라우저에 저장됩니다
- 폼이 제출되면:
  - 연락처가 생성되거나 업데이트됩니다
  - 이전 활동이 해당 연락처에 연결됩니다

이 스크립트는 다음과 같은 기능은 지원하지 않습니다:

- iframe 기반 또는 위젯 기반 폼에서 연락처 생성
- 스크롤 깊이, 클릭, 세션 분석 추적
- 명시적으로 구성되지 않은 숨겨진 필드나 비활성화된 필드 캡처

---

## **필요한 이유**

추적 스크립트가 설치되지 않으면 다음과 같은 것들을 할 수 없습니다:

- 외부 웹사이트에서 페이지뷰 기록
- 리드를 캠페인에 올바르게 어트리뷰션
- 외부 활동 기반으로 워크플로우 트리거

스크립트가 설치되면 다음이 가능합니다:

- 외부 웹사이트에 호스팅된 폼 추적
- 어트리뷰션 보고서에서 페이지 성과 측정
- 방문자가 폼을 제출할 때 자동화 트리거

---

## **추적하는 내용**

추적 스크립트는 다음을 기록합니다:

- 페이지뷰
- 호환되는 폼 제출
- 해당 이벤트 기반 워크플로우 트리거

다음은 추적하지 않습니다:

- 스크롤 동작
- 클릭 이벤트
- 페이지 체류 시간
- 도메인 간 세션 분석

---

## **시작하기 전에**

스크립트를 설치하기 전에:

- 웹사이트 도메인이 계정에 연결되어 있는지 확인
- 폼이 다음 중 하나인지 확인:
  - 플랫폼 폼 임베드, 또는
  - 표준 HTML `<form>`

iframe, 팝업, JavaScript 위젯 안의 폼은 추적되지 않습니다.

### **1단계: 추적 스크립트 가져오기**

- **Settings(설정) → External Tracking(외부 추적)**으로 이동
- **Copy Script(스크립트 복사)** 클릭

스니펫은 다음과 유사합니다:

```html
<script 
  src="https://yourdomain.com/js/external-tracking.js" 
  data-tracking-id="tk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx">
</script>
```

`data-tracking-id`는 수정하지 마세요. 계정에 고유한 값입니다.

### **2단계: 스크립트 설치**

웹사이트의 `</body>` 태그 바로 앞에 스크립트를 붙여넣으세요.

### **권장 설치 방법**

```html
<script 
  src="https://yourdomain.com/js/external-tracking.js" 
  data-tracking-id="tk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx">
</script>
</body>
</html>
```

다음 플랫폼에서 사용 가능:

- 커스텀 HTML 웹사이트
- WordPress
- Wix
- Squarespace
- Shopify
- Webflow

### **태그 매니저를 통한 설치**

태그 매니저를 통해 설치하는 경우:

- Custom HTML 태그 사용
- 수정하지 말고 전체 스크립트를 붙여넣기
- 모든 페이지에서 트리거
- `data-tracking-id` 속성이 그대로 유지되는지 확인

속성이 제거되면 사이트 푸터에 수동으로 스크립트를 설치하세요.

### **3단계: 폼 호환성 확인**

지원되는 폼:

- 임베드된 플랫폼 폼
- DOM에 직접 렌더링되는 표준 HTML 폼

### **폼 요구사항**

추적이 작동하려면:

- 실제 `<form>` 태그가 존재해야 함
- 필드가 보여야 함
- 필드에 유효한 name 속성이 포함되어야 함
- 이메일 필드가 있어야 함

### **지원되지 않는 경우**

- iframe 안의 폼
- 팝업 기반 폼
- 실제 HTML 입력을 렌더링하지 않는 제3자 위젯

### **커스텀 필드 캡처**

추적 스크립트는 이름과 이메일 외에도 더 많은 정보를 캡처합니다.

### **지원되는 필드 유형**

- 텍스트 필드
- 텍스트 영역
- 드롭다운 (단일 및 다중 선택)
- 체크박스
- 라디오 버튼
- 숫자 필드
- 날짜 선택기
- 평점
- 파일 업로드 (메타데이터만)

### **필드 캡처 규칙**

다음 조건을 만족하는 필드만 캡처됩니다:

- 보이는 상태여야 함
- 유효한 name 속성이 있어야 함

숨겨진 필드나 비활성화된 필드는 별도로 구성되지 않는 한 무시됩니다.

### **필드 매핑 로직**

필드는 다음 우선순위로 매핑됩니다:

- 필드 이름이 CRM 속성 또는 커스텀 필드 키와 일치
- 필드 라벨이 CRM 표시 이름과 일치
- 필드 이름이 라벨 텍스트와 일치 (대소문자 구분 안함)

일치하는 항목이 없으면 값이 unmapped_field로 저장됩니다.

### **4단계: 제출 확인**

추적된 제출을 보려면:

- **Sites(사이트) → Forms(폼) → Submissions(제출)**로 이동
- External Form Name으로 필터링

제출은 다음과 같이 태그됩니다:

external form: {form_name}

### **5단계: 워크플로우 트리거**

다음을 기반으로 자동화를 트리거할 수 있습니다:

### **페이지뷰 트리거**

추적 스크립트가 포함된 페이지를 방문자가 볼 때 실행됩니다. 익명 활동이 저장되고 방문자가 폼을 제출하면 연결됩니다.

### **폼 제출 트리거**

호환되는 폼이 제출될 때 실행됩니다.

사용 가능한 필터:

- 도메인
- 페이지 경로
- 외부 폼 이름
- UTM source
- UTM medium
- UTM campaign

## **고급 문제 해결**

### **디버그 모드 활성화**

스크립트에 `data-debug="true"`를 추가하세요:

```html
<script 
  src="https://yourdomain.com/js/external-tracking.js" 
  data-tracking-id="tk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" 
  data-debug="true">
</script>
```

브라우저 개발자 도구 → 콘솔을 열어 추적 로그를 확인하세요.

### **네트워크 확인**

- 성공적인 2xx 응답 확인
- CORS 오류가 나타나지 않는지 확인

### **쿠키 확인**

브라우저 Application → Cookies에서 세션 쿠키가 존재하는지 확인하세요.

누락된 경우 브라우저 개인정보 설정이 추적을 차단하고 있을 수 있습니다.

### **일반적인 문제**

**콘솔 로그가 없음**
– 스크립트 배치를 확인하고 디버그 모드를 활성화하세요

**폼이 추적되지 않음**
– 실제 `<form>` 태그와 이메일 입력이 있는지 확인하세요

**추적 ID 누락**
– 태그 매니저에서 제거되지 않았는지 확인하세요

## **폼 구조 예시**

```html
<form name="signup">
  <label>이메일</label>
  <input type="email" name="email" required />
  <label>이름</label>
  <input type="text" name="first_name" />
  <button type="submit">제출</button>
</form>
```

## **자주 묻는 질문**

**Q: 외부 폼에서 커스텀 필드를 캡처할 수 있나요?**
A: 네. 지원되는 커스텀 필드 유형은 올바르게 구성되면 자동으로 캡처되고 매핑됩니다.

**Q: 제3자 폼이 지원되나요?**
A: 페이지 DOM에 직접 실제 HTML 입력을 렌더링하는 경우에만 지원됩니다. iframe 기반 폼은 지원되지 않습니다.

**Q: 업로드된 파일이 저장되나요?**
A: 아니요. 파일 메타데이터만 캡처됩니다.

**Q: 어트리뷰션 보고서에 영향을 주나요?**
A: 네. 페이지뷰와 제출이 어트리뷰션 보고서와 워크플로우 트리거에 기여합니다.

**Q: 페이지뷰가 아직 보이지 않는 이유는 무엇인가요?**
A: 페이지뷰는 익명으로 저장되고 폼 제출이 발생한 후에만 연결됩니다.

---

## **도움이 필요하신가요?**

추적이 올바르게 작동하지 않는 경우:

- 스크립트가 `</body>` 태그 앞에 설치되었는지 확인
- 폼이 실제 `<form>` 요소를 사용하는지 확인
- 이메일 필드가 존재하는지 확인
- 추적 ID가 수정되지 않았는지 확인
- 디버그 모드를 사용하여 콘솔 로그 검토

문제가 지속되면 다음 정보를 수집하세요:

- 웹사이트 URL
- 폼 HTML 스크린샷
- External Tracking 설정 스크린샷
- 콘솔 로그 출력 (가능한 경우)

이러한 세부 정보와 함께 고객지원팀에 문의하시면 더 빠른 문제 해결이 가능합니다.
