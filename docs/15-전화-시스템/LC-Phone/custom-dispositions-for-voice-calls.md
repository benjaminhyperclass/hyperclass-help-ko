# 음성 통화용 커스텀 처리 결과

모든 음성 통화 결과를 몇 초 만에 기록하고 하이퍼클래스가 자동으로 후속 조치를 진행하도록 하세요. 커스텀 처리 결과는 "그 통화에서 무슨 일이 있었나요?"를 실행 가능한 데이터로 바꿔 워크플로우를 실행하고 보고서에 반영합니다.

## 음성 통화용 커스텀 처리 결과란?

커스텀 처리 결과를 통해 상담원(또는 파워 다이얼러)이 각 통화 직후 단일 결과를 선택할 수 있습니다. 예를 들어 후속 조치, 예약 요청, 관심 없음 등을 선택할 수 있죠. 선택한 내용은 통화 기록에 저장되고, 자동화를 실행하며, 분석을 위해 통화 보고서에 표시됩니다.

---

## 음성 통화용 커스텀 처리 결과의 주요 장점

커스텀 처리 결과는 다음과 같은 이점으로 모든 통화를 의미 있게 만듭니다:

- **빠른 마무리**: 상담원이 한 번의 클릭으로 결과를 선택하여 다음 통화로 더 빨리 넘어갈 수 있습니다.

- **깔끔한 자동화**: 특정 처리 결과가 선택되는 순간 워크플로우가 시작될 수 있습니다(예: SMS 발송, 파워 다이얼러에 재등록).

- **향상된 가시성**: 표준화된 결과가 통화 보고서에서 필터링 가능한 열로 표시됩니다.

- **매끄러운 마이그레이션**: 데이터 손실 없이 기존 통화 상태를 교체합니다(통화 세부사항 트리거를 업데이트하세요).

---

## 처리 결과 생성 및 관리

**관리자**는 팀이 다이얼러에서 볼 목록을 제어합니다. 파이프라인 단계와 일치하는 간결하고 행동 지향적인 결과를 만드세요. 처리 결과는 상담원의 편의를 위해 순서를 바꿀 수 있으며 언제든지 편집할 수 있습니다. 처리 결과 이름이 바뀌어도 기존 워크플로우는 계속 작동합니다.

---

## 워크플로우에서 커스텀 통화 처리 결과 사용하기

처리 결과는 새로운 커스텀 처리 결과 필터를 통해 통화 세부사항 워크플로우 트리거에 직접 연결됩니다. 처리 결과를 다른 필터(방향, 지속 시간, 태그)와 결합하여 다음을 수행할 수 있습니다:

- 개인화된 SMS/이메일 시퀀스 발송
- 파워 다이얼러에 리드 재등록
- 태그 적용 또는 제거
- 할 일이나 기회 생성

- 링크: Workflow Trigger – Call Details

### **설정 방법:**

### **1단계:** 자동화(Automations) 하위에서 트리거 추가: 통화 세부사항(Call Details).

### **2단계:** 필터 커스텀 처리 결과(Custom Disposition) 선택

원하는 후속 조치를 자동화할 하나 이상의 처리 결과를 선택하세요.

### **3단계:** 선택한 처리 결과가 다음과 같은 워크플로우를 자동으로 실행할 수 있습니다:

후속 SMS 또는 이메일 발송

- 예약 일정 잡기
- 연락처를 특정 파이프라인이나 캠페인에 추가

## 음성 통화용 커스텀 처리 결과 설정 방법

**기본 처리 결과**: 기본적으로 모든 계정은 가시성과 사용 편의성을 위해 미리 설정된 처리 결과로 시작합니다. 관리자가 이를 추가하거나 수정할 수 있습니다.

처리 결과 생성 및 편집: 하위 계정 관리자만 처리 결과를 생성, 편집, 삭제할 수 있습니다. 하위 계정당 최대 10개의 처리 결과를 가질 수 있습니다.

설정 방법: **하위 계정 설정(Sub-account Settings) → 전화 시스템(Phone System) → 음성(Voice) → 통화 처리 결과(Call Dispositions)**로 이동하세요. 기존 것을 편집하거나 새로 추가하세요.

## 통화 처리 결과 작동 방법

웹 다이얼러에서 통화가 종료되면 선택할 수 있는 최대 10개의 통화 처리 결과가 표시됩니다. 예를 들어 후속 조치나 예약 요청 등이죠. 결과를 기록하려면 적절한 것을 선택하기만 하면 됩니다.

## 보고서에서 처리 결과 사용하기

보고서: 선택된 모든 처리 결과가 저장되어 다음에서 확인할 수 있습니다: 리포팅(Reporting) → 통화 보고서(Call Reporting)

이를 통해 사용자별 통화 결과나 팀 성과 같은 패턴을 추적할 수 있습니다. 통화 보고서 표에서 통화 처리 결과를 위한 새로운 열을 볼 수 있습니다. 또한 이를 필터로 사용하여 처리 결과별 성과를 분석할 수 있습니다. 예를 들어 "콜백 요청"으로 이어진 통화가 몇 건인지 확인할 수 있죠.

**중요**: 이 기능은 곧 출시될 예정입니다!

## 모바일에서 커스텀 처리 결과

커스텀 처리 결과가 이제 모바일에서 완전히 지원되어, 명확한 통화 결과를 기록하고 다음 단계를 즉시 실행하는 것이 쉬워졌습니다. 휴대폰에서 바로 가능하죠. 이 기능은 iOS와 Android의 HighLevel, LeadConnector, 화이트라벨 모바일 앱 전반에서 실제 운영됩니다.

**새로운 기능**

- **통화 종료 화면 지원:** 통화가 끝나거나 통화 후 보기에서 즉시 처리 결과를 선택할 수 있으며, 데스크톱이 필요 없습니다.
- **통화당 단일 처리 결과:** 하나의 명확한 결과(예: 후속 조치, 자격 있음)를 기록하여 깔끔한 데이터와 일관된 보고서를 보장합니다.
- **워크플로우 준비 완료:** 워크플로우에서 커스텀 처리 결과 필터를 사용하여 후속 조치, 할 일, 파이프라인 이동 등을 자동으로 실행할 수 있습니다.

### 모바일 관련 참고사항

- 하위 계정 일치 필요: 선택된 하위 계정이 통화의 로케이션과 일치할 때만 처리 결과가 표시됩니다. 모바일 앱은 여러 하위 계정에서 통화를 받을 수 있기 때문에 교차 하위 계정 처리 결과는 지원되지 않습니다. 필요하면 전환(Switch) 옵션을 사용하세요.

## 지원되는 통화 유형

1. 통화 처리 결과가 작동하는 경우:

- 웹 또는 모바일 앱을 통해 발신하거나 수신한 모든 통화
- LC Phone과 Twilio 하위 계정 모두

2. 지원되지 않는 경우: 개인 번호나 데스크 전화로 응답한 통화

## 제한사항

- 통화당 하나의 처리 결과만 가능
- 계정당 최대 10개 처리 결과
- 하위 처리 결과(중첩 결과)는 사용할 수 없음
- 다중 선택은 지원되지 않음

## 마이그레이션 참고: 통화 상태 → 통화 처리 결과

이전에 파워 다이얼러(수동 작업)에서 통화 상태를 사용하고 있었다면:

- 수동 통화 상태 선택 옵션이 통화 처리 결과로 교체되었습니다.
- "통화 상태" 트리거를 사용하는 기존 워크플로우는 대신 통화 세부사항 트리거의 커스텀 처리 결과 필터를 사용하도록 업데이트해야 합니다.
- 기존 데이터는 계속 표시되지만, 새로운 자동화는 새 시스템을 사용해야 합니다.

## 자주 묻는 질문

**Q: 상담원이 모든 통화 후에 처리 결과를 선택해야 하나요?**

아니요. 처리 결과 선택은 선택사항이지만, 정확한 보고서와 자동화를 위해 강력히 권장합니다.

**Q: 워크플로우에서 사용하는 처리 결과의 이름을 바꾸면 어떻게 되나요?**

워크플로우는 계속 실행됩니다. 시스템이 처리 결과의 내부 ID를 추적하기 때문입니다.

**Q: 처리 결과를 삭제하면 어떻게 되나요?**

해당 필터를 제거하거나 교체할 때까지 연관된 워크플로우의 실행이 중지됩니다.

**Q: 통화당 여러 처리 결과가 지원되나요?**

현재로서는 아닙니다. 통화당 하나의 처리 결과만 선택할 수 있습니다.

**Q: 어떤 통화 유형이 처리 결과를 지원하나요?**

LC Phone 또는 Twilio 번호의 모든 웹 또는 모바일 앱 통화입니다. 개인 또는 데스크 전화로 응답한 통화는 제외됩니다.

**Q: 기존 통화 상태 데이터는 계속 표시되나요?**

네, 하지만 새로운 자동화는 커스텀 처리 결과 필터로 전환해야 합니다.
