# SMS 전송 오류 이해하기

**개요**

이 가이드는 Hyperclass에서 자주 발생하는 SMS 전송 오류를 이해하고 해결하는 방법을 자세히 설명합니다. 일반적인 실패 원인을 파악하고 실용적인 해결책을 제시하여, 메시지 전송 성공률을 높이고 고객과의 효과적인 소통을 보장할 수 있습니다.

## SMS 전송 오류란 무엇인가요?

SMS 전송 오류는 잘못된 번호, 통신사 차단, 사용자 수신 거부, 기술적 제한 등의 이유로 메시지가 수신자에게 전달되지 못할 때 발생합니다. 이러한 오류를 이해하면 기업이 높은 메시지 전달률을 유지하고 업계 규정을 준수하는 데 도움이 됩니다.

대부분의 오류는 통신사, Twilio 또는 LC Messaging에서 발생하며 특정 오류 코드로 표시됩니다. 이러한 코드를 식별하고 해석하면 사용자가 신속하고 확실하게 수정 조치를 취할 수 있습니다.

## SMS 오류 이해의 주요 이점

SMS 오류를 신속하게 인식하고 해결하면 커뮤니케이션 전략을 크게 개선할 수 있습니다.

- **전달률 향상**: 일반적인 오류를 파악하여 메시지 실패를 예방하는 사전 조치를 취할 수 있습니다
- **효율적인 문제 해결**: 오류 코드를 이해하면 문제를 더 빠르게 해결할 수 있습니다
- **규정 준수 개선**: 통신사와 플랫폼 제한 사항을 인식하여 규정을 준수하고 벌금을 피할 수 있습니다
- **메시징 전략 최적화**: 오류에 대한 인사이트를 통해 더 나은 메시징 방식과 콘텐츠를 구성할 수 있습니다

## SMS 오류 확인 및 해석 방법

SMS 오류를 정확히 진단하려면 전송 로그를 확인하고, 오류 코드를 식별하며, 이 정보를 활용하여 메시징 또는 수신자 데이터를 조정해야 합니다.

### 1단계: 대화 로그 확인하기

- Hyperclass 계정에서 Conversations(대화) 탭을 클릭합니다
- 전송 실패한 SMS 로그를 검색합니다

![SMS 로그 확인](https://jumpshare.com/v/3y1TTEGBoYVOYYkLuXR0+/Screen+Shot+2025-05-29+at+6.59.38+PM.png)

- 실패한 대화를 클릭합니다
- 빨간색 삼각형 오류 상자에 마우스를 올립니다

![오류 상자 확인](https://jumpshare.com/v/GQuCGpZiPfibXmvZGEOw+/Screen+Shot+2025-05-29+at+7.06.05+PM.png)

### 2단계: SMS 오류 코드 찾기 및 식별하기

- 대화 상자에 해당 대화에 대한 숫자 오류 코드와 오류에 대한 간단한 설명이 표시됩니다 (예: Error 30034, Error 21610)

## 자주 발생하는 오류 코드와 의미

Hyperclass에서 가장 자주 발생하는 SMS 오류는 다음과 같습니다:

### 1. Error 11751 – 미디어 메시지 크기 제한 초과

**설명**: 미디어 메시지의 총 크기가 메시징 제공업체에서 허용하는 최대 제한을 초과했습니다.

**해결 방법**:
- 미디어 파일을 압축하거나 크기를 조정하세요
- Twilio의 지원 MMS 크기 제한 내에서 파일을 유지하세요
- MMS (미국/캐나다): 메시지당 최대 크기는 5MB입니다

### 2. Error 21610 – 사용자 수신 거부

**설명**: 수신자가 STOP으로 응답하여 메시지 수신을 거부했습니다.

**해결 방법**:
- 사용자가 START를 보내서 다시 수신을 동의하지 않는 한 이 번호로 메시지를 보내지 마세요
- 적절한 수신 동의 메커니즘을 확보하세요

### 3. Error 21612 – 현재 발신자로부터 '수신' 번호에 도달 불가

**설명**: 현재 '발신' 번호를 사용하여 수신자 번호에 도달할 수 없습니다 (예: 통신사 제한 또는 지원되지 않는 국제 경로로 인해).

**해결 방법**:
- 현지 또는 국제 지원 번호를 사용하세요
- '수신' 번호와 '발신' 번호 간의 라우팅 호환성을 확인하세요

### 4. Error 30002 – 전송 전 계정 정지

**설명**: 메시지 전송 시점과 전달 사이에 Twilio 계정이 정지되었습니다.

**해결 방법**:
- 계정 상태를 해결하기 위해 Twilio 지원팀에 문의하세요
- 메시지 대기열을 위해 계정을 활성 상태로 유지하세요

### 5. Error 30003 – 사용할 수 없거나 비활성 번호/도달 불가능한 수신 단말기

**설명**: 수신자의 전화기가 꺼져 있거나, 범위를 벗어나거나, 일시적으로 사용할 수 없습니다.

**해결 방법**:
- 나중에 메시지를 다시 시도하세요
- 번호가 활성 상태인지 확인하세요

### 6. Error 30004 – 통신사에 의한 메시지 차단

**설명**: 수신 번호에서 이 메시지 수신이 차단되었습니다.

**해결 방법**:
- 번호가 유효하고 통신사에 의해 차단되지 않았는지 확인하세요
- 이전에 전송 실패 기록이 있는 번호로는 메시지 전송을 피하세요

### 7. Error 30005 – 알 수 없는 수신 단말기

**설명**: 수신 장치에 도달할 수 없습니다 (예: 장기간 꺼져 있거나 범위를 벗어남).

**해결 방법**:
- 일정 시간 후 다시 시도하세요
- 목록을 정리하여 도달할 수 없거나 비활성 번호를 제거하세요

### 8. Error 30006 – 유선 전화 또는 지원되지 않는 번호

**설명**: 번호가 SMS를 받을 수 없습니다. 일반적으로 유선 전화이거나 SMS와 호환되지 않는 경우입니다.

**해결 방법**:
- 번호 검증 도구를 사용하여 목록에서 유선 전화를 제거하세요
- 메시지 전송 전에 번호가 SMS를 지원하는지 확인하세요

### 9. Error 30007 – 통신사 필터링 (스팸 또는 정책 위반)

**설명**: 메시지가 콘텐츠 정책을 위반하거나 스팸으로 보여 통신사에 의해 차단되었습니다.

**해결 방법**:
- 스팸 같은 콘텐츠와 단축 URL을 피하세요
- 개인화되고 규정을 준수하는 메시징을 사용하세요
- A2P 10DLC 또는 Toll-Free 인증으로 번호를 등록하세요

자세한 내용은 [통신사 SMS 필터링 방지 방법: Error 30007](how-to-prevent-sms-filtering-by-carriers-error-30007.md)을 참조하세요.

### 10. Error 30034 – US A2P 10DLC: 미등록 번호에서 메시지

**설명**: 등록된 A2P 캠페인과 연결되지 않은 10DLC 번호에서 미국 번호로 메시지를 보내려고 시도했습니다.

**해결 방법**:
- Twilio의 Trust Hub를 통해 번호를 등록하세요
- 승인된 메시징 서비스에 번호를 연결하세요
- 적절한 캠페인 연결을 확인하세요

### 11. Error 30008 – 알 수 없는 오류

**설명**: 메시지 전송 중 지정되지 않은 오류가 발생했습니다.

**해결 방법**:
- 메시지 본문의 형식이나 내용 문제를 확인하세요
- 다른 번호나 내용 구조로 테스트하세요

### 12. Error 11200 – HTTP 검색 실패

**설명**: 이 오류는 대화에서 이미지 로드 또는 전송이 실패할 때 자주 발생합니다.

**해결 방법**:
- MXToolbox를 사용하여 도메인의 DNS 구성을 확인하세요. DNS 레코드를 찾을 수 없거나 brand.ludicrous.cloud를 가리키지 않으면 이미지 검색이 실패할 수 있습니다

![DNS 구성 오류](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155048038110/original/DSMUXHzu7zkN9Fk7fHyrtnbxHsMZ_Py14g.png?1749592564)

해결하려면:
브랜드 도메인을 다음으로 가리키는 CNAME 레코드가 올바르게 설정되어 있는지 확인하세요:

brand.ludicrous.cloud

이것이 누락되거나 잘못 구성되면 대화에서 이미지가 깨져 보입니다. [브랜딩 시스템 생성 링크 (API 도메인)](../../11-설정/branding-system-generated-links-api-domain-.md)을 확인하세요.

### 13. Error 30024 - 통신사에서 숫자 발신자 ID가 프로비저닝되지 않음

**설명**: 숫자 발신자 ID의 사전 등록 및 프로비저닝이 필요한 국가의 모바일 번호로 메시지를 보내려고 시도했지만, 숫자 발신자 ID가 아직 통신사에서 프로비저닝되지 않았습니다.

**해결 방법**: 숫자 발신자 ID가 이미 등록되어 있다고 생각되면, 오류 코드 30024와 함께 "전송되지 않음" 상태를 표시하는 최소 3개의 메시지 ID를 수집하세요. 수집 후 지원팀에 문의하면 등록 상태를 검토하여 문제 해결을 도와드립니다.

**참고**: Hyperclass에서 가장 일반적으로 발생하는 오류 외에 추가 오류 코드를 확인하려면 [전체 오류 코드 카탈로그](https://www.twilio.com/docs/api/errors)를 참조하세요.

## 추가 문제 해결 팁

기술적인 측면을 이해하는 것도 도움이 되지만, 다음 모범 사례를 통해 오류를 사전에 줄일 수 있습니다:

**인증된 발신 번호 사용**
- A2P 10DLC와 Toll-Free 번호는 검증 과정을 거쳐 전달률을 개선하고 필터링을 줄입니다

**SMS 콘텐츠 적절히 형식화**
- 모든 대문자, 스팸성 문구, 또는 적절한 링크 단축기 없이 긴 URL 포함을 피하세요
- 콘텐츠 차단을 피하기 위해 대화체 톤을 사용하세요

**깔끔한 목록 유지**
- 번호가 유효하고 올바르게 형식화되어 있으며 사용자가 수신에 동의했는지 확인하세요

**오류 트렌드 모니터링**
- Hyperclass의 보고 도구를 사용하여 전송 문제의 패턴을 발견하고 사전 조치를 취하세요

## 자주 묻는 질문

**Q: 수신 거부 후 사용자가 다시 구독하게 하는 방법은?**
A: 사용자가 구독을 취소한 동일한 번호로 START라는 단어가 포함된 메시지를 보내야 합니다.

**Q: 30007 오류로 실패한 메시지를 다시 보낼 수 있나요?**
A: 네, 하지만 다시 시도하기 전에 먼저 통신사 필터링 가이드라인을 준수하도록 콘텐츠를 수정하세요.

**Q: 번호가 유효해도 일부 메시지가 실패하는 이유는?**
A: 통신사가 콘텐츠 필터링이나 수신자 설정으로 인해 여전히 메시지를 차단할 수 있습니다.

**Q: 번호가 유선 전화인지 어떻게 알 수 있나요?**
A: 번호 검증 도구를 사용하거나 Hyperclass에 업로드하기 전에 목록을 정리하세요.

**Q: SMS 오류가 자주 발생할 때 Hyperclass에서 알려주나요?**
A: 전송 보고서를 검토할 수 있지만, 실시간 알림을 위해 워크플로우를 통한 알림 설정을 권장합니다.
