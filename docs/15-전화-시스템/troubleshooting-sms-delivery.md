## **메시지 전송 실패의 주요 원인과 해결 방법**

### 1. Hyperclass 시스템 단계

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|
| 연락처가 수신거부(DND) 상태 | 해당 연락처가 특정 메시지 채널 또는 전체적으로 수신거부 설정을 해둔 경우, Hyperclass는 메시지를 보내지 않고 대화 화면에 오류를 표시합니다. | 연락처의 수신거부 상태를 해제하여 메시지 전송을 재개하세요.<br/>➡️ [수신거부(DND) 설정 관리](../02-연락처/스마트-리스트/how-to-use-do-not-disturb-dnd-.md) |
| 일반전화 번호로 전송 시도 | 번호 검증이 활성화된 경우, 시스템이 해당 번호가 문자 수신 가능한지(즉, 일반전화가 아닌지) 확인합니다. 불가능하면 메시지 전송을 차단합니다. | 휴대폰 번호를 사용하거나 필요시 번호 검증을 비활성화하세요.<br/>➡️ [번호 검증 기능](number-intelligence-spam-detection-caller-id-sms-validation.md) |
| 신규 하위 계정의 단계적 전송 제한 | Hyperclass는 스팸 방지를 위해 신규 하위 계정의 메시지 발송량을 제한하며, 시간이 지나면서 점진적으로 한도를 늘립니다. | 한도가 자동으로 증가할 때까지 기다리거나, 에이전시에서 제한 해제를 요청하세요.<br/>➡️ [LC Phone 메시징 정책 (단계적 증가 모델 참고)](메시징/lc-phone-messaging-policy.md) |
| 일일 메시지 한도 초과 | 각 하위 계정은 에이전시에서 설정한 일일 메시지 한도가 있습니다. 한도 초과 시 대량 메시지와 자동 메시지는 중단되지만, 개별 답장은 여전히 보낼 수 있습니다. | 에이전시 설정에서 일일 한도를 조정하거나 다음날까지 기다리세요.<br/>➡️ [LC Phone 메시징 정책](메시징/lc-phone-messaging-policy.md) |
| 높은 오류율, 수신거부, 신고로 인한 전송 중단 | 스팸, 수신거부, 통신사 신고 임계값을 초과한 하위 계정은 메시지 전송이 중단됩니다. 중단 중 메시지 전송 시도 시 건너뛰기되며 오류가 표시됩니다. | 전달률 모범 사례를 따르고 필요시 고객지원팀에 문의하세요.<br/>➡️ [전달률 향상 모범 사례](메시징/best-practices-for-sms-deliverability-and-avoiding-sms-restrictions.md) |

### 2. Twilio/LC Phone 단계

| 오류 | 원인 | 해결 방법 |
|------|------|-----------|
| Twilio 수신거부 목록에 등록된 번호 | 이전 수신거부나 통신사 신고로 인해 해당 번호가 Twilio 수신거부 목록에 있습니다. 재가입하지 않는 한 메시지가 실패합니다. | 해당 연락처가 같은 Twilio 번호로 "START"를 문자로 보내서 메시징을 재활성화해야 합니다. |
| 30001: 대기열 오버플로우 | 발신자나 계정의 전송률에 따라 메시지가 대기열에 들어갑니다. 4시간 이상 대기열에 머물면 실패합니다. | 메시징 서비스 설정이나 API 요청에서 유효 기간을 줄여 대기 시간을 단축하세요. |
| 30002: 하위 계정 중단 | 대기열 진입과 전송 사이에 Twilio 하위 계정이 중단된 경우 메시지가 실패합니다. | Twilio나 Hyperclass 고객지원팀에 문의하여 중단을 해결하세요. |

### 3. 발신 통신사 네트워크 단계

| 오류 | 원인 | 해결 방법 |
|------|------|-----------|
| 30003: 도달 불가능한 수신 단말기 | 수신자의 휴대폰이 꺼져 있거나, 신호가 없거나, 일반전화이거나, 이동통신사 문제가 있을 수 있습니다. | 나중에 다시 시도하거나 번호가 휴대폰이고 연결 가능한지 확인하세요. |
| 30004: 메시지 차단 | 메시지가 전송되지 않거나 전달되지 않는 가능한 이유: 수신 번호가 차단됨, 기기 신호 부족 또는 메시지 수신 불가(예: 일반전화), 인도 수신거부 등록부에 등록된 번호, 이동통신사 문제, 또는 이전에 수신거부한 단말기로 미국/캐나다 수신자부담 번호 전송. | 번호가 유효하고 수신 동의되었는지 확인하세요. 수신거부나 차단된 번호로 전송하지 마세요. |
| 첨부 파일 크기 초과 | 파일 크기가 MMS 통신사 제한을 초과합니다. | ➡️ [SMS 첨부파일 크기 제한](메시징/file-size-limits-for-attachments-in-sms-and-email.md) |
| 국제 전송 권한 | 계정에 국제 메시지 전송 권한이 없을 수 있습니다. | 고객지원팀에 문의하세요. |
| 메시지 본문 길이 초과 | • 트리거로 메시지를 보내는 경우 메시지 본문을 확인하세요. 또한 {{message.body}} 커스텀 값이 있는지 확인하세요.<br/>• 예를 들어, 누군가 이메일로 답장했는데 이메일이 1,600자 제한을 초과하면 {{message.body}} 커스텀 값이 있을 때 메시지가 전송되지 않습니다.<br/>![메시지 본문 길이 예시](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155047990631/original/PvnlWbqLsSDvP46SqUbwxdC7wgjSuYDhkQ.png?1749549640) | 메시지를 수동으로 줄이거나 트리거 로직을 수정하여 큰 커스텀 값을 제외하세요. |

**참고**: 오류 코드와 문제 해결에 대한 자세한 내용은 [오류 및 경고 사전](https://www.twilio.com/docs/api/errors)을 참조하세요.

## **전송 전 필수 체크리스트**

- **대화 확인**: 빨간색 오류 배지가 있는지 확인하세요.
- **오류 코드 매칭**: 위 표를 사용하여 원인을 찾으세요.
- **해결 방법 적용**: 해당 문제의 "해결 방법" 항목을 따르세요.
- **재시도**: 조정 후 메시지를 다시 보내세요.
- **모니터링**: 전달 확인하거나 새로운 오류 코드를 기록하세요.

## **자주 묻는 질문**

**Q: 실패한 메시지에도 요금이 부과되나요?**
메시지 전송을 시도하면 나중에 실패하더라도 요금이 부과됩니다.

**Q: 메시지가 실패한 정확한 이유를 어떻게 알 수 있나요?**
Twilio의 메시지 로그에서 확인할 수 있습니다.
➡️ [Twilio 로그 확인 방법](https://help.gohlighlevel.com/en/support/solutions/articles/48001222601)
