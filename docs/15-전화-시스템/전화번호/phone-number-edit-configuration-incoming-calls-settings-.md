# 전화번호 구성 편집 (수신 전화 설정)

**수신 전화 설정(Incoming Call Settings)** 인터페이스는 인바운드 전화의 모든 라우팅, 벨 울림, 음성사서함, 백업 동작을 하나의 구성 화면에서 통합 관리할 수 있게 해줍니다. 최신 개선 사항으로 Hyperclass는 이제 **건전한 수신 전화 타임아웃 설정에 대한 가이드**를 제공하여, 통신사 음성사서함 대신 CRM 음성사서함으로 전화가 연결되도록 하고 통화 상태 정확성을 향상시킵니다.

## 개요

수신 전화 설정 인터페이스는 인바운드 전화를 라우팅, 차단 또는 리디렉트할 수 있는 모든 요소를 하나의 통합된 화면에 표시합니다. 여기에는 착신 전환, 팀원 벨 울림, IVR, 음성 AI, 음성사서함, 타임아웃 동작이 포함됩니다.

통화 흐름 우선순위를 시각화하고 모범 사례 가이드를 제공함으로써, 이 화면은 통화 흐름 문제를 진단하고, 잘못된 구성을 방지하며, 통화가 올바르게 처리되도록 하는데 도움을 줍니다.

---

## 수신 전화 설정이란?

이 인터페이스는 모든 인바운드 전화 라우팅 로직을 하나의 읽기 쉬운 창에 통합합니다. 각 섹션은 다음을 보여줍니다:

- 라우팅 옵션의 활성화 여부
- 어떤 사용자, 번호 또는 시스템이 담당하는지
- 시스템이 통화 목적지를 평가하는 우선순위

이를 통해 전화가 예상대로 울리지 않거나 라우팅되지 않을 때의 추측을 없앨 수 있습니다.

---

## 수신 전화 설정의 주요 장점

- **완전한 투명성** – 어떤 규칙이 첫 번째, 두 번째, 세 번째로 응답하는지 정확히 확인
- **인라인 편집** – 페이지를 떠나지 않고 대부분의 설정 업데이트
- **명확한 우선순위** – 시스템이 경로를 평가하는 방식 이해
- **더 빠른 문제 해결** – 잘못 구성된 타임아웃이나 사용자를 신속하게 식별
- **향상된 통화 처리** – 놓친 전화와 라우팅 오류 감소

---

## 전화번호 구성 편집 접근하기

수신 전화 설정을 열려면:

- **Settings(설정) → Phone Numbers(전화번호)**로 이동
- **Phone Numbers(전화번호)** 탭 열기
- 전화번호 옆의 **점 3개 메뉴** 클릭
- **Edit Configuration(구성 편집)** 선택

## 기본 정보

**Basic Details(기본 정보)** 탭에서는 다음을 구성할 수 있습니다:

- **Friendly Name(표시 이름)** – 전화번호 이름 변경
- **Calls Go To(전화 받을 사람)** – 기본 팀원 선택
- **Ring Options(벨 울림 옵션)** – 동시에 울릴 위치 선택:
  - Web App(웹 앱)
  - Mobile App(모바일 앱)
  - Phone Number(전화번호)
  - VoIP Deskphone(VoIP 데스크폰)
- **Call Recording(통화 녹음)**
  - 녹음 활성화 또는 비활성화
  - 녹음 고지 메시지 재생

## 착신 전환 탭

**Call Forwarding(착신 전환)** 탭은 전화를 즉시 받지 못할 때 통화가 어떻게 라우팅되는지 제어합니다.

### 전화 라우팅 (우선순위)

- **IVR** – 워크플로우 또는 메뉴 입력에 따라 통화 라우팅
- **Team Member(팀원) (1순위)**
- **External Phone Number(외부 전화번호) (2순위)**
- **Voice AI(음성 AI) (3순위)**

우선순위에 따라 한 번에 하나의 목적지만 울립니다.

### 더 많은 팀원에게 벨 울리기

- 활성화하면 **최대 6명의 추가 사용자**에게 벨 울림
- 디바이스 아이콘은 각 사용자가 활성화한 디바이스를 표시

## 수신 전화 타임아웃 (새로운 가이드)

수신 전화 타임아웃은 **백업 목적지(음성 AI 또는 음성사서함)로 이동하기 전에 전화가 얼마나 오래 울리는지**를 제어합니다.

### 권장 타임아웃 범위

Hyperclass는 이제 이 값이 건전한 범위를 벗어날 때 가이드를 제공합니다:

- **너무 높음 (약 20초 초과)**
  - 전화가 CRM 음성사서함 대신 **통신사 음성사서함**으로 갈 수 있음
  - 통화 상태가 **"Voicemail(음성사서함)"** 대신 **"Completed(완료)"**로 표시될 수 있음

- **너무 낮음 (약 15초 미만)**
  - 전화가 너무 짧게 울리거나 연결되지 않을 수 있음
  - 발신자가 응답할 충분한 시간이 없을 수 있음

- **권장: 약 20초**

권장 범위를 벗어나는 값을 입력하면 더 건전한 시간을 선택하도록 도와주는 **경고 메시지**가 나타납니다.

![타임아웃 가이드 경고](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064214884/original/45JX6qVuEXUbp-EZxDcpm-1w3sSX8rWzgw.png?1770236921)

**참고:** 이는 가이드일 뿐입니다. 범위를 벗어나는 값도 여전히 저장할 수 있지만, 그렇게 하면 음성사서함 라우팅과 통화 상태 정확성에 영향을 줄 수 있습니다.

---

## 백업 동작

수신 전화 타임아웃에 도달하면 통화는 백업 목적지로 라우팅됩니다:

- **Voice AI(음성 AI)** – AI 에이전트에게 통화 전송
- **Voicemail(음성사서함)** – CRM 음성사서함으로 통화 전송

건전한 타임아웃을 선택하면 통신사 음성사서함 시스템 대신 CRM 음성사서함으로 통화가 연결됩니다.

## 통화 흐름 우선순위 이해하기

시스템은 인바운드 통화 라우팅을 고정된 순서로 평가합니다:

1. 번호에 연결된 워크플로우, IVR 또는 자동화
2. 전화번호에 직접 할당된 사용자
3. 연락처 레코드에 할당된 사용자
4. 여러 사용자에게 수신 전화 울리기 (Ring-All)
5. 전환 번호

**가장 높은 우선순위**만 울립니다. 통화가 응답되거나 라우팅되면 낮은 우선순위는 건너뜁니다.

## 자주 묻는 질문

**Q: 누가 수신 전화 설정에 접근할 수 있나요?**
A: 전화번호 관리 권한이 있는 모든 사용자입니다.

**Q: 새로운 타임아웃 가이드가 내 통화 로직을 자동으로 변경하나요?**
A: 아니요. 권장사항만 표시합니다. 귀하의 구성이 강제되거나 재정의되지 않습니다.

**Q: 왜 일부 통화가 이전에 "Voicemail(음성사서함)" 대신 "Completed(완료)"로 표시되었나요?**
A: 타임아웃이 너무 높으면 CRM 음성사서함 대신 통신사 음성사서함에 도달하여 잘못된 상태가 표시될 수 있습니다.

**Q: 이것이 발신 전화에 영향을 주나요?**
A: 아니요. 발신 전화 타임아웃은 별도로 구성되며 이 가이드의 영향을 받지 않습니다.

**Q: 여전히 매우 높거나 낮은 타임아웃 값을 사용할 수 있나요?**
A: 네, 하지만 라우팅과 상태 정확성 문제로 인해 권장하지 않습니다.
