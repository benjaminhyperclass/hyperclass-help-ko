# Call Connect로 정확한 통화 상태 추적하기

이 가이드에서는 Call Connect 기능을 활용해 음성 메일, 연결됨, 부재중 통화 등의 통화 결과를 정확하게 추적하는 방법을 알아보겠습니다. 단순히 시도된 통화가 아닌 실제로 연결된 통화를 명확히 파악할 수 있습니다.

## Call Connect란 무엇인가요?

Call Connect는 통화가 연결됨으로 표시되기 전에 실제 사람이 응답했는지 확인하는 통화 라우팅 기능입니다. 고객이 전화를 받으면 시스템이 담당 상담원에게 연결하고, 상담원이 키패드(예: "1")를 누를 때까지 기다린 후 두 당사자를 연결합니다. 이 키패드 확인 과정을 통해 통화가 실제로 상담원에 의해 응답됐는지 확인할 수 있어, 음성 메일로 넘어가거나 놓친 통화를 "완료됨"으로 잘못 표시하여 보고서와 자동화에 문제를 일으키는 일을 방지합니다.

## Call Connect로 통화 상태 추적하기의 주요 장점

- **향상된 추적 정확도**: 상담원의 입력을 통해 응답을 확인하여 통화 상태 오분류를 줄입니다
- **신뢰할 수 있는 통화 보고서**: 놓친 통화나 음성 메일이 아닌 실제 대화만 집계됩니다
- **정확한 워크플로우 트리거**: 실제 상담원이 연결된 통화에만 자동화가 반응합니다
- **음성 메일 필터링**: 진짜 통화 연결과 음성 메일 라우팅을 구분해줍니다
- **책임감 향상**: 상담원이 언제 응답하고 놓쳤는지 쉽게 파악할 수 있습니다
- **더 나은 리드 후속 조치**: 단순한 통화 시도가 아닌 실제 사람과의 상호작용을 바탕으로 후속 조치를 진행합니다

## "완료됨" 통화 상태가 오해의 소지가 있는 이유

Twilio는 아무도 응답하지 않아도 라우팅이 끝나면 모든 통화를 "완료됨"으로 표시합니다. 즉, 음성 메일로 넘어가거나 상담원이 놓치거나 시간 초과된 통화도 성공적으로 연결된 것처럼 표시될 수 있습니다.

Call Connect가 이 문제를 해결합니다. Call Connect가 활성화되면:

- 상담원은 통화를 수락하려면 아무 버튼이나 누르라는 안내 메시지를 받습니다
- 상담원이 응답하지 않거나 키를 누르지 않으면 통화가 연결되지 않아 응답한 것으로 잘못 표시되지 않습니다

이 간단한 확인 단계를 통해 실제 대화만을 성공적인 연결로 집계하는 데 큰 차이를 만들어냅니다.

## Call Connect 활성화 방법

전화번호(Phone Number) 설정에서 직접 Call Connect를 활성화할 수 있습니다. 특히 배정된 사용자나 팀을 통해 라우팅되는 수신 전화에 유용합니다.

1. `Settings(설정) → Phone Numbers(전화번호)`로 이동하세요
2. 설정하려는 전화번호 옆의 점 3개를 클릭한 후 **Edit Configuration(설정 편집)**을 선택하세요
3. **Call Connect**를 활성화하세요
4. **Play Whisper Message(안내 메시지 재생)**을 활성화하세요
5. 사용자에게 연결 지시사항을 알려주는 안내 메시지를 커스터마이징하세요 (예: "새로운 통화입니다. 연결하려면 아무 키나 누르세요")
6. **Save(저장)**을 클릭하여 변경사항을 적용하세요

## 자주 묻는 질문

**Q: 아무도 응답하지 않았는데 통화가 "완료됨"으로 표시되는 이유가 무엇인가요?**
A: Twilio의 기본 동작은 통화가 연결되지 않았거나 음성 메일로 넘어가도 모든 통화 시도를 "완료됨"으로 표시합니다.

**Q: Call Connect가 이를 어떻게 개선하나요?**
A: 사람의 확인(키 입력)을 요구하므로 시스템이 통화가 실제로 연결됐는지 알 수 있습니다.

**Q: 발신 및 수신 통화 모두에 적용되나요?**
A: 네, 양방향으로 작동합니다. 특히 수신 라우팅과 빠른 리드 응답에 유용합니다.

**Q: 상담원이 키를 누르지 않으면 어떻게 되나요?**
A: 통화가 연결된 것으로 간주되지 않아 더 나은 추적과 깔끔한 보고가 가능합니다.

**Q: 이것이 자동화에 영향을 주나요?**
A: 네, 좋은 방향으로요! 실제로 연결된 통화에만 자동화를 트리거할 수 있게 됩니다.
