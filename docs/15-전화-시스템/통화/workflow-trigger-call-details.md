# **통화 상태 변경 워크플로우 트리거란?**

**통화 정보(Call Details)** 워크플로우 트리거는 전화 통화와 관련된 정보가 업데이트될 때 워크플로우를 시작합니다. 여기에는 통화 상태, 통화 방향, 연결된 전화번호, 번호 풀, 커스텀 처리 상태의 변경이 포함됩니다.

이 트리거는 통화 방향, 전화번호, 번호 풀, 커스텀 처리 상태 같은 추가 필터를 지원하여 특정 통화 시나리오에서만 워크플로우가 반응하도록 할 수 있습니다. 통화가 어떻게 종료되거나 분류되는지에 따라 후속 조치를 자동화하거나, 태그를 적용하거나, 내부 액션을 트리거하는 데 주로 사용됩니다.

---

## **통화 상태 변경 워크플로우 트리거의 주요 장점**

- **통화 결과에 따른 후속 조치 자동화**: 통화 상태가 변경될 때 워크플로우를 자동으로 트리거하여 수동 개입 없이 적시에 후속 액션을 보장합니다.

- **통화 기반 필터를 사용한 정교한 제어**: 통화 방향, 통화 상태, 전화번호, 번호 풀, 커스텀 처리 상태 같은 필터를 사용하여 특정 통화 시나리오에서만 워크플로우가 실행되도록 할 수 있습니다.

- **팀 간 대응 일관성 향상**: 부재중 통화, 음성 메시지, 응답 없음으로 표시된 통화가 할 일(Task) 생성, 태그 지정, 알림(Notification) 같은 미리 정의된 액션으로 일관되게 처리되도록 보장합니다.

- **수동 추적 및 오류 감소**: 통화 로그를 수동으로 모니터링할 필요를 없애고 통화 상태 업데이트가 발생하는 즉시 반응합니다.

## **통화 정보 워크플로우 트리거 설정 방법**

**통화 정보(Call Details)** 워크플로우 트리거 설정에 포함되는 주요 요소들은 다음과 같습니다. 각 설정은 **언제** 그리고 **어떤 조건에서** 연락처가 워크플로우에 추가될지 결정합니다.

### 워크플로우 빌더 열기

먼저 `Automation(자동화) → Workflows(워크플로우)`로 이동하여 새 워크플로우를 만들거나 기존 워크플로우를 여세요. 워크플로우 빌더는 모든 트리거와 액션이 정의되는 곳입니다. 이 단계에서 통화 기반 트리거가 작동할 자동화 컨텍스트를 설정합니다.

![워크플로우 빌더](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061322826/original/877GivrHZahADR6mM0zROjLyVQyyQhs4yA.png?1766588556)

### 워크플로우 트리거 추가

워크플로우 캔버스에서 `Add Trigger(트리거 추가)`를 선택하여 워크플로우를 시작할 이벤트를 선택하세요. 트리거는 연락처가 워크플로우에 진입하는 조건을 정의합니다. 올바른 트리거 유형을 선택하는 것은 통화 관련 옵션에 접근하는 데 필수적입니다.

### 통화 정보 트리거 유형 선택

사용 가능한 트리거 카테고리에서 `Call Details(통화 정보)`를 선택하세요. 이렇게 하면 통화 상태, 방향, 전화번호, 처리 상태 기반 조건을 포함한 전화 통화 활동 관련 트리거가 활성화됩니다. 선택하면 설정을 위한 추가 통화 관련 필터를 사용할 수 있습니다.

![통화 정보 트리거 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061323483/original/as6KeZpSKctZitA5odrFg8jaxIU_2t6I3A.png?1766589178)

### 워크플로우 트리거 이름

트리거 이름은 워크플로우 내에서 트리거의 목적을 식별하는 데 도움이 됩니다. 실행에는 영향을 주지 않지만, 명확한 이름은 가독성과 장기적인 유지보수를 향상시킵니다. 여러 트리거가 있는 워크플로우에서는 설명적인 이름이 특히 유용합니다.

![트리거 이름 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061323562/original/i7UZfe5j9eEOqPTJdX8fx3qkI1n--9gKKQ.jpeg?1766589235)

### 필터 추가

`Add Filters(필터 추가)` 옵션을 사용하면 하나의 트리거 내에서 여러 조건을 결합할 수 있습니다. 워크플로우가 실행되려면 선택된 모든 필터 조건이 충족되어야 합니다. 이를 통해 정교하고 매우 타겟팅된 자동화가 가능합니다.

### **통화 방향**

통화 방향(Call Direction)은 트리거가 수신 통화, 발신 통화, 또는 둘 다에 반응해야 하는지 지정합니다. 이를 통해 통화가 어떻게 시작되었는지에 따라 워크플로우가 다르게 동작하도록 할 수 있습니다. 인바운드 리드 처리와 아웃바운드 통화 프로세스를 분리하는 데 유용합니다.

![통화 방향 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061322639/original/Rks01lfEtsKumb9c9lFjMDS0yku7tfBm2g.png?1766588441)

### 통화 상태

통화 상태(Call Status)는 응답 없음, 음성 메시지, 통화 중 등과 같은 어떤 통화 결과가 트리거를 활성화해야 하는지 정의합니다. 워크플로우는 통화가 선택된 상태 중 하나로 업데이트될 때만 실행됩니다. 이를 통해 자동화가 의미 있는 통화 결과와 연결되도록 보장합니다.

![통화 상태 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061322664/original/NUCgoi_uKg5tMJECgXEa2FtB4fN3Si8NOA.png?1766588459)

### 번호 풀

이 필터는 특정 번호 풀과 연관된 통화로만 트리거를 제한합니다. 서로 다른 캠페인이나 팀에 서로 다른 번호 풀을 사용할 때 유용합니다. 이 필터를 적용하면 워크플로우가 의도된 소스의 통화에만 반응하도록 보장할 수 있습니다.

![번호 풀 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061322700/original/cBQQkoKgdB-FGl6WmpNDLkXWoMTpI89_NA.png?1766588476)

### **커스텀 처리 상태**

커스텀 처리 상태(Custom Disposition)는 통화 종료 후 적용되는 통화 분류에 트리거가 반응하도록 합니다. 표준 통화 상태를 넘어선 추가적인 제어 레이어를 제공합니다. 사용자나 시스템이 표시한 내부 통화 결과에 워크플로우가 반응해야 할 때 유용합니다.

![커스텀 처리 상태 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061322781/original/I_YpN2Dc9FHQqw8ylGEXM8DgeDueeic6Cw.png?1766588521)

### **전화번호**

이 필터는 특정 전화번호와 연관된 통화로만 트리거를 제한합니다. 개별 전화번호가 서로 다른 비즈니스 기능을 담당할 때 유용합니다. 이 필터를 사용하면 워크플로우를 타겟팅하고 관련성을 유지하는 데 도움이 됩니다.

![전화번호 설정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155061322762/original/3si76Y7tgpyrbZ0mbYCpGNJPpKSOx66G8w.png?1766588498)

### 트리거 저장

트리거를 저장하면 설정이 적용되고 워크플로우 내에서 활성화됩니다. 저장된 후에는 정의된 통화 정보 조건이 충족될 때마다 연락처가 워크플로우에 추가됩니다. 이 단계에서 트리거가 실행 준비를 완료합니다.

## 자주 묻는 질문

**Q: 통화 정보 트리거는 통화가 기록될 때마다 실행되나요, 아니면 특정 정보가 변경될 때만 실행되나요?**

트리거는 통화 상태, 방향, 처리 상태 같은 선택된 통화 정보가 설정된 조건과 일치할 때만 실행됩니다. 기본적으로 모든 통화에 대해 활성화되지는 않습니다. 이를 통해 워크플로우가 관련 통화 이벤트에만 반응하도록 보장합니다.

**Q: 통화 정보 트리거가 같은 연락처에 대해 여러 번 실행될 수 있나요?**

네, 별도의 통화가 트리거 조건을 만족하면 같은 연락처에 대해 여러 번 실행될 수 있습니다. 각 자격을 갖춘 통화 이벤트는 독립적으로 평가됩니다. 반복 등록을 제어하려면 재진입 규칙이나 추가 필터 같은 워크플로우 설정을 사용하세요.

**Q: 통화 정보 트리거와 시간 기반 또는 활동 기반 워크플로우 트리거는 어떻게 다른가요?**

통화 정보는 예정된 시간이나 일반적인 연락처 활동이 아닌 통화 관련 정보의 변경에 특별히 반응합니다. 이는 통화 결과를 기반으로 한 실시간 또는 거의 실시간 자동화에 적합합니다. 특정 통화 결과 후 부재중 통화, 음성 메시지, 후속 조치를 처리하는 데 특히 유용합니다.

**Q: 여러 워크플로우에 걸쳐 여러 통화 정보 트리거가 존재하면 어떻게 되나요?**

각 워크플로우는 자체 트리거 설정을 기반으로 통화 이벤트를 독립적으로 평가합니다. 트리거 조건이 충족되면 하나의 통화가 여러 워크플로우에 연락처를 추가할 수 있습니다. 신중한 필터 설계는 겹치거나 충돌하는 자동화를 방지하는 데 도움이 됩니다.
