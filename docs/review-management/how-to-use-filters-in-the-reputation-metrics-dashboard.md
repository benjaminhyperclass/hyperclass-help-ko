# 평판 지표 대시보드에서 필터 사용하는 방법

평판 지표 대시보드(Reputation Metrics Dashboard)는 리뷰 요청, 리뷰, 응답, AI 도입, 위젯, QR 코드 등에서 무엇이 잘되고 있고, 무엇이 안되고 있으며, 어디를 개선해야 하는지 파악할 수 있도록 설계되었습니다.

대시보드 상단에는 보고 있는 데이터를 제어하는 필터가 있습니다. 이 가이드에서는 이러한 필터의 작동 방식, 영향을 미치는 범위, 그리고 실제 비즈니스 질문에 답하기 위해 어떻게 사용하는지 설명합니다.

## 필터가 표시되는 위치

대시보드 상단에는 다음이 있습니다:

- 날짜 범위(Date Range) 필터 (필수)
- 페이지/플랫폼(Page/Platform) 필터 (선택사항; 예: Google, Facebook, Yelp 등)

또한 "선택한 날짜 범위를 기준으로 데이터가 표시됩니다"와 같은 메시지도 볼 수 있습니다. 이는 페이지의 모든 지표가 적용한 필터를 기반으로 업데이트됨을 의미합니다.

## 1) 날짜 범위 필터

### 기능

날짜 범위 필터는 초대 발송, 리뷰 수신, 응답 시간, 감정 추세, 전환율 등 모든 대시보드 지표를 계산하는 데 사용되는 시간 범위를 제어합니다.

### 사용 가능한 옵션

- 오늘(Today)
- 이번 주(This Week)  
- 이번 달(This Month)
- 지난 3개월(Last 3 Months)
- 지난 6개월(Last 6 Months)
- 전체 기간(All Time)
- 사용자 지정 범위(Custom Range)

### 사용 방법

- 날짜 범위를 클릭합니다
- 미리 설정된 옵션(예: 이번 달)을 선택하거나 사용자 지정 범위를 선택합니다
- 대시보드가 업데이트된 시간 범위로 자동으로 새로고침됩니다

### 날짜 범위 변경 시 변화하는 항목

선택한 시간 범위에 맞춰 다음이 업데이트됩니다:

- 추세 차트 (리뷰, 평점, 감정, 동영상 제출 추세)
- 퍼널 지표 (발송 → 전달 → 클릭 → 리뷰 제출 등)
- 비율 (전환율, AI 응답률 등)
- 총합 (위젯 노출수, QR 스캔 등)
- "이전 기간 대비" 비교 (표시되는 경우)

### 사용 사례 예시

- 주간 성과 확인: "이번 주"로 설정하여 리뷰 요청과 응답이 현재 어떻게 진행되고 있는지 확인합니다.
- 출시 후 영향 측정: "지난 3개월"로 설정하여 전환율이나 AI 응답 도입률의 개선을 평가합니다.
- 기준선 성과: "전체 기간"을 사용하여 장기적인 평판 성장을 이해합니다.

## 2) 페이지/플랫폼 필터 (Google, Facebook, Yelp 등)

### 기능

페이지/플랫폼 필터는 대시보드 지표를 특정 리뷰 플랫폼(예: Google만)으로 좁혀서 플랫폼별 성과를 비교할 수 있게 합니다.

### 사용 시기

다음과 같은 질문에 답하고 싶을 때 페이지/플랫폼 필터를 사용하세요:

- "Google과 Facebook 중 어디서 더 많은 리뷰를 받고 있나요?"
- "Google과 Yelp 중 어디에서 응답이 가장 느린가요?"
- "최근 어느 플랫폼에서 부정적 감정이 더 많나요?"

### 영향을 미치는 범위

지표에 따라 페이지/플랫폼 필터는 다음을 업데이트할 수 있습니다:

- 수신된 리뷰 + 평점 추세
- 응답 성과 (평균 응답 시간, 미응답 리뷰 등)
- 감정 추세 (긍정/중립/부정)
- 플랫폼별 보고서 (예: Google 리뷰 응답 성과)

참고: 일부 초대/요청 지표는 플랫폼 기반이 아닌 채널 기반(이메일/SMS/왓츠앱)입니다. 이런 경우 플랫폼 필터링이 "발송" 부분에는 적용되지 않을 수 있지만, 해당 리뷰가 플랫폼과 연결된 경우 리뷰 수신 부분에는 적용될 수 있습니다.

## 주요 대시보드 섹션에서 필터가 작동하는 방식

### 리뷰 요청 지표 (채널 분할 + 퍼널)

이러한 지표는 전달 채널별 성과를 확인하는 데 도움이 됩니다:

- 이메일
- SMS
- 왓츠앱

날짜 범위를 사용하여 기간을 정의한 다음, 채널 분할/토글(사용 가능한 경우)을 사용하여 퍼널 단계를 비교하세요:

- 발송 → 전달 → 클릭 → 리뷰 수신

최적화할 채널: 어떤 채널이 결과를 가져다주는지 최적화하는 데 적합합니다.

### 리뷰 요청 전환율

전환율은 "내 리뷰 요청이 얼마나 효과적인가?"라는 질문에 답합니다.

대시보드가 다음 로직을 따르는 경우:

- 초대 발송 > 0인 경우: 전환율 = 요청을 통해 받은 리뷰 / 초대 발송
- 초대 발송 = 0인 경우: "이 기간 동안 리뷰 요청을 보내지 않았습니다. 전환율을 추적하려면 요청 보내기를 시작하세요!"와 같은 빈 상태 메시지를 볼 수 있습니다.

날짜 범위를 사용하여 단기 대 장기 전환을 비교하세요.

### 리뷰 응답 + AI 도입 (AI 응답률, AI 에이전트 분석)

이러한 지표는 몇 개의 리뷰에 응답했는지 보여줍니다:

- AI 대 수동
- AI 에이전트별 분석 (여러 에이전트가 있는 경우)

사용법:

- 날짜 범위를 사용하여 시간에 따른 도입 추세 확인
- 플랫폼 필터를 사용하여 AI 사용량이 플랫폼별로 다른지 확인 (지원되는 경우)

### 평균 응답 시간 + 미응답 리뷰 (특히 Google 리뷰)

이러한 지표는 응답성을 개선하는 데 도움이 됩니다:

- 평균 응답 시간 (AI + 수동)
- 미응답 리뷰 수

사용법:

- 플랫폼 필터를 사용하여 특정 플랫폼(예: Google)에 집중
- 날짜 범위를 사용하여 악화 상황 발견 (예: 이번 주 응답 시간 증가)

### 위젯, QR 코드, 동영상 추천서

이러한 기능은 일반적으로 날짜 범위 비교에서 가장 많은 이점을 얻습니다:

- 위젯 노출수 (선택적 제출)
- QR 스캔
- 동영상 추천서 퍼널 (전달 → 열람 → 녹화 시작 → 제출)
- 시간에 따른 제출 추세

캠페인이나 런칭과 성과를 맞출 때는 사용자 지정 범위를 사용하세요.

## 정확한 인사이트를 얻기 위한 팁

- 넓게 시작한 다음 좁혀가기: 먼저 "지난 3개월"을 사용한 다음 "이번 달"로 전환하여 최근 진전을 검증하세요.
- 플랫폼 필터를 사용하여 문제 진단: 예시—전체적으로 응답 시간이 높아 보이나요? Google 대 Yelp로 필터링하여 원인을 찾아보세요.
- 캠페인에는 사용자 지정 범위 사용: 3월 1-15일에 프로모션을 진행했다면, 사용자 지정 범위를 설정하여 영향을 명확하게 측정하세요.

## 문제 해결 및 FAQ

### "날짜 범위를 변경했는데 왜 숫자가 바뀌었나요?"

선택한 시간 범위를 기반으로 모든 지표가 다시 계산되기 때문입니다. 추세, 총합, "이전 기간" 비교가 모두 업데이트됩니다.

### "왜 지표에 데이터가 표시되지 않나요?"

대부분의 경우:

- 선택한 시간 범위 내에서 활동이 발생하지 않았습니다 (지난 3개월 또는 전체 기간으로 확장해 보세요)
- 플랫폼 필터가 데이터를 제외했습니다 (모든 플랫폼으로 다시 전환하거나 필터를 제거하세요)

### "왜 전환율이 누락되었나요?"

선택한 범위에서 초대 발송 = 0이면 전환율을 계산할 수 없습니다. 범위를 확장하거나 리뷰 요청 보내기를 시작하세요.

### "플랫폼 필터링이 모든 것에 적용되나요?"

항상 그런 것은 아닙니다. 일부 지표는 본질적으로 플랫폼별(리뷰, 응답)이지만, 다른 지표는 채널별(이메일/SMS/왓츠앱 발송)입니다. 필터가 지표에 적용되지 않는 경우 변화가 없을 수 있습니다.
