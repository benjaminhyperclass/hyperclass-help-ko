# 2단계 인증(2FA)을 위한 인증 앱 지원

하이퍼클래스의 새로운 2단계 인증(2FA) 인증 앱 지원으로 계정 보안을 강화하세요. 이 기능을 사용하면 안전하고 유연하며 신뢰할 수 있는 로그인 검증을 위해 인기 있는 인증 앱을 사용할 수 있습니다. 피싱과 SIM 스왑 공격에 대한 추가적인 저항력을 제공하면서 보안 프로세스를 간소화하세요.

## **2단계 인증(2FA)을 위한 인증 앱 지원이란?**

2단계 인증(2FA)을 위한 인증 앱 지원은 시간 기반 일회용 패스워드(TOTP) 기능을 하이퍼클래스 계정에 통합하는 보안 강화 기능입니다. 이 업데이트를 통해 사용자는 Google Authenticator, Microsoft Authenticator, Authy와 같은 인증 앱으로 계정을 연결하여 추가 보안 계층을 제공받을 수 있습니다. 앱 기반 검증으로 전환함으로써 기존 SMS나 이메일 검증 방법보다 강력한 보안 조치의 혜택을 누릴 수 있습니다.

**참고**: Google Authenticator, Microsoft Authenticator, Authy 등 TOTP 기반의 모든 인증 앱을 지원합니다.

---

## **2단계 인증(2FA) 인증 앱 지원의 주요 장점**

인증 앱 연동: 인기 있는 인증 앱(Google Authenticator, Microsoft Authenticator, Authy)을 사용하여 쉽게 계정을 보호하세요. 안전한 로그인을 위한 시간 제한 검증 코드(TOTP)를 생성합니다.

- QR 코드 또는 수동 입력을 통한 빠른 설정
- 보안 강화를 위한 빠른 만료 코드 생성
- 계정 접근을 유지하기 위한 백업 코드 제공

이 기능은 현대적인 규정 준수 표준에 부합하면서 보안과 사용성을 모두 개선하는 중요한 장점을 제공합니다. 앱 기반 인증을 구현함으로써 다음과 같은 이점을 얻을 수 있습니다:

- **향상된 보안**: 시간 기반 검증 코드를 사용하여 피싱과 SIM 스왑 공격으로부터 보호합니다.
- **사용자 유연성**: 선호하는 인증 앱을 선택할 수 있어 개인 워크플로우와 보안 요구사항에 맞춰 사용할 수 있습니다.
- **규정 준수**: 현재 보안 모범 사례를 충족하여 전체 플랫폼 거버넌스를 강화합니다.
- **지원 티켓 감소**: 2FA 전송과 관련된 문제를 최소화하여 월 약 150개의 지원 문의를 줄일 것으로 예상됩니다.

## 2단계 인증(2FA)을 위한 인증 앱 설정 방법

2단계 인증(2FA)은 계정에 추가적인 보안 계층을 제공합니다. 인증 앱을 설정하면 로그인 시 본인 확인을 위해 전화번호와 이메일과 함께 인증 앱을 사용할 수 있는 옵션이 생깁니다.

**1단계:** 2단계 인증(2FA) 앱으로 이동

에이전시 설정(Agency settings) → 내 프로필(My Profile)로 이동하세요. 시작하려면 설정(Setup) 버튼을 클릭하세요. 선호하는 인증 앱으로 표시된 QR 코드를 스캔하거나 제공된 설정 코드를 수동으로 입력하세요.

![인증 앱 설정 과정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049296774/original/9yQjidc3PBU2r2MBJF6CBaeM2_EPk_m0Jw.gif?1751548145)

**2단계:** 인증 앱에서 OTP를 입력하여 설정을 완료하세요.

제공된 코드를 인증 앱에 스캔하거나 수동으로 입력한 후, 일회용 패스워드(OTP)를 받게 됩니다. 하이퍼클래스 CRM에서 다음(Next)을 클릭하고, 이 OTP를 입력하여 설정을 완료하세요.

![OTP 입력 과정](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049298353/original/BM53WlBK0c0PgfHCEx3lDRpO9X4z6tFnQw.gif?1751549225)

**3단계:** 백업 코드 저장

앱 접근이 불가능한 경우 시스템에 접근하기 위해 백업 코드를 안전하게 저장하세요. 백업 코드를 분실하거나 다른 사람이 접근한 경우, 사용자는 내 프로필(My Profile)에서 백업 코드를 재설정할 수 있습니다.

![백업 코드 저장](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049298466/original/iBvr_Dx4MI286FOApbOWHWNljB2Ev653aw.png?1751549338)

**참고:** 다음 로그인 시, 2FA를 위해 인증 앱 옵션을 선택하고 생성된 OTP를 사용하여 인증하세요.

![로그인 시 인증 앱 선택](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155049298555/original/fnSz7eHA4YZ-h3jJY1DyLgnmqrv9e3b5YA.png?1751549408)

**중요사항:**
1. 기본 로그인 사용자만 자신의 계정에 인증 앱을 설정할 수 있습니다. 관리자는 "다른 사용자로 로그인" 기능을 사용하여 다른 사용자를 위해 인증 앱을 설정할 수 없습니다.
2. 한 번에 10개의 백업 코드가 생성되며, 각 코드는 한 번만 사용할 수 있습니다.
3. 인증 앱에는 에이전시 이름이 표시됩니다. 인증 앱은 사용자-에이전시 조합으로 설정됩니다.

## **자주 묻는 질문**

**Q: 이 기능을 위해 어떤 인증 앱이든 사용할 수 있나요?**

네, Google Authenticator, Microsoft Authenticator, Authy 등 TOTP 기반의 모든 인증 앱을 사용할 수 있습니다.

**Q: 인증 기기나 백업 코드를 분실하면 어떻게 해야 하나요?**

내 프로필(My Profile)에서 백업 코드를 재생성할 수 있습니다. 하지만 무단 접근을 방지하기 위해 안전하게 저장하는 것이 중요합니다.

**Q: 인증 앱을 설정할 수 있는 사람에게 제한이 있나요?**

기본 사용자만 자신의 계정에 대해 인증 앱을 설정할 수 있습니다. 관리자는 "다른 사용자로 로그인" 기능을 사용하여 구성할 수 없습니다.

**Q: 몇 개의 백업 코드가 생성되며, 어떻게 사용되나요?**

한 번에 10개의 백업 코드가 생성되며, 각 코드는 계정 복구를 위해 한 번만 사용할 수 있습니다.

**Q: 이 기능이 SMS나 이메일 인증보다 어떤 보안 개선을 제공하나요?**

인증 앱은 SMS와 이메일 방식에 비해 피싱과 SIM 스왑 공격에 더 안전한 시간 기반 코드를 생성합니다.
