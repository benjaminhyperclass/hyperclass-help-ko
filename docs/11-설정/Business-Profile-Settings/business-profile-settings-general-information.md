# 비즈니스 프로필 설정 - 일반 정보

일반 정보는 브랜드 아이덴티티와 주요 계정 속성을 다룹니다. 로고, 사업체명, 연락처 이메일/전화번호, 브랜드가 적용된 시스템 생성 링크(API 도메인), 사업체 웹사이트, 업종, API 키를 설정하면 다른 기능들이 정확하고 브랜드에 일치하는 데이터를 사용할 수 있습니다.

비즈니스 프로필 설정(Business Profile Settings)에 대해 더 자세히 알아보시려면 설정 패널로 이동하는 방법을 포함하여 다음을 참고하세요: [비즈니스 프로필 설정 개요](business-profile-settings.md)

# **일반 정보 설정**

이 필드들은 Hyperclass 전반에서 사용되는 브랜드 자산과 회사 식별자의 단일 정보원 역할을 합니다. 각 항목을 신중하게 작성하되, 명시된 경우 공식 기록과 일치시키세요. 업데이트 후 변경사항을 저장하고 몇 가지 자산(예: 캘린더 링크, 이메일 템플릿)을 확인하여 새 값이 예상대로 표시되는지 검토해보세요.

![일반 정보 설정 화면](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155052721902/original/rwfEn8KcBuBqCMCJByYkhT6vckUNzMiHcQ.png?1756501586)

#### 로케이션 ID

고객 지원팀과 소통할 때, 문제가 발생한 로케이션 ID를 요청받을 수 있습니다. 로케이션 ID는 에이전시 계정 내 모든 하위 계정에 부여되는 고유 ID입니다. 이를 통해 고객의 계정을 식별하고 문제 해결 범위를 좁힐 수 있습니다.

#### 사업체 로고

이 하위 계정과 연결할 사업체 로고를 업로드하세요. 로고는 에이전시 대시보드 뷰의 "계정" 목록에서 이 특정 하위 계정을 나타내며, 인보이스와 이메일을 포함한 Hyperclass의 많은 기능에서 사용할 수 있고, 메시징에서 커스텀 필드로도 활용할 수 있습니다.

**예시:** *{{location.logo_url}}*

#### 친근한 사업체명

메시징, 워크플로우 및 앱의 다른 영역에서 커스텀 필드 "계정 → 이름"을 사용할 때 하위 계정(로케이션)에 사용될 이름입니다.

**예시:** *{{location.name}}*

#### 법적 사업체명

법적 사업체명은 향후 컴플라이언스 등록에서 사용될 이름입니다. 여기에 추가하는 내용은 사업자 등록증이나 등록 서류와 같은 법적 사업 문서에 표시된 사업체명과 정확히 일치해야 합니다.

#### 사업체 이메일

메시지, 이메일, 워크플로우 또는 커스텀 필드를 허용하는 앱의 다른 영역에 커스텀 필드 "계정 → 이메일"을 추가할 때 이 계정에 사용할 이메일을 입력하세요.

**예시:** *{{location.email}}*

#### 사업체 전화번호

메시지, 이메일, 워크플로우 또는 커스텀 필드를 허용하는 앱의 다른 영역에 커스텀 필드 "계정 → 전화번호"를 추가할 때 이 계정에 사용할 전화번호를 입력하세요.

**예시**: {{location.phone}}

이 번호가 통화 및 메시징에 사용되는 번호와 일치하도록 하려면, 먼저 LC 전화번호를 구성한 다음 구매한 번호를 "사업체 전화번호" 필드에 복사/붙여넣기 해야 합니다.

#### 브랜드 도메인

각 하위 계정(로케이션)은 호스팅 제공업체의 서브도메인에 연결하여 "브랜드 도메인"을 만들 수 있습니다. 이 브랜드 도메인은 "트리거 링크", "캘린더", "폼", "설문" 등의 URL 주소로 사용됩니다.

다음은 브랜드 도메인을 더 자세히 설명하고 하위 계정에 구성하는 방법을 알려주는 도움말 문서들입니다:

- Hyperclass의 도메인

- [데스크톱 웹 앱용 화이트라벨 도메인 설정 방법](../../25-애드온-세일즈/Whitelabel-Desktop-App/how-to-set-up-a-whitelabel-domain-for-the-desktop-web-app.md)

#### 사업체 웹사이트

메시지, 이메일, 워크플로우 또는 커스텀 필드를 허용하는 앱의 다른 영역에 커스텀 필드 "계정 → 웹사이트"를 추가할 때 이 계정에 사용할 웹사이트 URL을 입력하세요.

**예시:** *{{location.website}}*

#### 사업 분야

드롭다운 목록에서 선택하여 하위 계정에 사업 분야를 지정하세요. 여러 분야를 대상으로 하는 경우, 타겟 시장을 가장 잘 설명하는 분야를 선택하세요.

#### 사업체 통화

하위 계정의 기본 통화를 선택하세요. 이 통화 선택은 모든 결제 관련 페이지와 플로우에 자동으로 적용됩니다. 통화를 선택하지 않으면 사업체의 실제 주소가 있는 국가의 기본값으로 설정됩니다. 이 설정은 실제 위치와 다른 국가에서 사업을 하는 경우에 유용합니다.

#### API 키

API 키는 정보 공유 및/또는 관련 소프트웨어 애플리케이션의 승인과 검증이 필요한 작업 수행을 목적으로 2개 이상의 소프트웨어 애플리케이션을 연결하는 데 사용됩니다.

예를 들어, Zapier에서 Zap을 만들 때, 서로 다른 소프트웨어 애플리케이션의 API에 연결하는 것입니다. Zapier는 최종 사용자에게 깔끔하고 "멋진" 경험을 제공하지만, Zapier가 제공하는 모든 멋진 소프트웨어 아래에는 Zap 구성 방식에 따라 서로 연결되는 수많은 API 키들이 있습니다.

하위 계정(또는 로케이션) API 키는 특정 하위 계정(로케이션)으로 정보를 보내거나 받기 위해 API 키가 필요한 Zapier 및 기타 애플리케이션에 연결할 때 사용할 수 있습니다.
