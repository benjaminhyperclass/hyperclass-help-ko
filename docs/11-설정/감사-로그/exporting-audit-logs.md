# 감사 로그 내보내기

감사 로그 내보내기는 에이전시와 로케이션이 규정 준수 검토, 조사, 내부 보고를 위해 감사 기록을 CSV로 내보낼 수 있게 도와줍니다. 이 기능은 감사 로그(Audit Logs)에서 제공하는 동일한 필터를 사용하고, 작업을 추적하기 위한 전용 내보내기(Exports) 탭을 추가하며, 완료된 파일을 30일간 사용할 수 있도록 보관합니다. 이 가이드에서는 감사 로그 내보내기가 무엇인지, 왜 중요한지, 권한은 어떻게 작동하는지, 그리고 하이퍼클래스에서 어떻게 활성화하고 사용하는지 설명합니다.

## **감사 로그 내보내기란?**

감사 로그 내보내기(CSV 다운로드)는 하이퍼클래스의 감사 로그 경험을 확장하여 권한이 있는 사용자가 필터링된 감사 기록을 CSV 파일로 내보낼 수 있게 해줍니다. 감사 로그는 중요한 계정 활동에 대한 시간 기록을 제공하므로 관리자와 파워 사용자가 변경 사항을 검토하고, 책임감을 향상시키며, 문제 해결을 지원할 수 있습니다. 하이퍼클래스는 앱 내에서 감사 로그를 60일간 보관하며, 이 내보내기 기능은 팀이 플랫폼 외부에서 필요한 기록을 다운로드하고 저장할 수 있는 방법을 제공합니다.

이 기능은 전체적인 기본 활성화 전에 Labs를 통해 출시되므로, 일부 계정은 먼저 Agency Settings에서 활성화해야 할 수 있습니다.

## 감사 로그 내보내기(CSV 다운로드)의 주요 장점

- **다운로드 가능한 기록**: 필터링된 감사 기록을 CSV로 내보내어 팀이 하이퍼클래스 외부에서 활동을 저장, 검토, 참조할 수 있습니다.

- **규정 준수 및 조사 지원 개선**: 내부 검토, 감사 준비 또는 보안 분석을 위한 계정 활동의 이동 가능한 기록을 보관합니다.

- **백그라운드 처리**: 내보내기가 비동기적으로 실행되므로 사용자는 내보내기가 준비되는 동안 계속 작업할 수 있습니다.

- **제어된 접근**: 전용 '감사 로그 내보내기' 권한으로 민감한 감사 기록을 내보낼 수 있는 사용자를 제한합니다.

- **명확한 제한 및 보관**: 각 내보내기는 최대 500,000개의 레코드를 지원하며, 완료된 파일은 30일간 다운로드할 수 있습니다.

- **추적 가능한 내보내기 활동**: 내보내기 요청 및 파일 다운로드도 감사 로그에 기록되어 책임감을 강화하는 데 도움이 됩니다.

## **감사 로그 내보내기 권한**

다운로드 가능한 감사 기록에 대한 접근은 내보낸 파일에 민감한 운영 데이터가 포함될 수 있기 때문에 신뢰할 수 있는 사용자로 제한되어야 합니다. 권한 제어는 에이전시와 로케이션이 최소 권한 접근 관행을 따르는 동시에 관리자와 규정 준수 중심 사용자에게 필요한 도구를 제공하는 데 도움이 됩니다. 하이퍼클래스는 또한 에이전시와 하위 계정별로 사용자 관리 워크플로우를 분리하므로, 각 상황에 맞는 올바른 탐색 경로를 사용하는 것이 중요합니다.

**중요한 권한 세부사항:**

- 기본적으로 에이전시 소유자, 에이전시 관리자, 계정 관리자가 기능 출시 노트에 따라 이 접근을 관리할 수 있습니다.

- 사용자가 감사 로그를 볼 수는 있지만 내보낼 수는 없을 수 있습니다.

- 내보내기 접근은 보고, 규정 준수 또는 조사를 위해 다운로드 가능한 감사 데이터가 필요한 사용자에게만 부여되어야 합니다.

**접근 관리 방법:**

- 에이전시 뷰에서 Settings(설정) → Team으로 가서 사용자 프로필을 열어 역할과 권한을 검토합니다.

- 하위 계정 뷰에서 Settings(설정) → My Staff로 가서 사용자 프로필을 열어 역할과 권한을 검토합니다.

## 내보내기 탭

내보내기(Exports) 탭은 요청이 제출된 후 사용자에게 내보내기 활동을 추적할 수 있는 중앙 장소를 제공합니다. 이를 통해 진행 상황을 모니터링하고, 중복된 내보내기 요청을 피하며, 준비가 완료되면 완성된 파일을 다운로드하는 것이 더 쉬워집니다. 큰 날짜 범위나 반복적인 검토를 진행하는 팀의 경우, 전용 내보내기 추적 뷰가 가시성을 향상시키고 프로세스를 체계적으로 유지합니다.

내보내기가 완료되면 자격을 갖춘 사용자는 내보내기 탭에서 CSV를 다운로드할 수 있습니다. 내보내기 요청과 다운로드 활동은 추가적인 추적성을 위해 감사 로그(Audit Logs) 모듈에도 기록됩니다.

감사 로그 내보내기가 활성화되면 감사 로그 페이지에는 다음이 포함됩니다:

- 활동을 보고 필터링하는 기존 감사 로그(Audit Logs) 탭
- 내보내기 작업을 추적하는 새로운 내보내기(Exports) 탭

## **내보내기 제한, 성능 및 파일 가용성**

명확한 제한은 전체 플랫폼 성능을 보호하면서 내보내기를 안정적으로 유지하는 데 도움이 됩니다. 파일이 얼마나 오래 사용 가능하고 내보내기가 얼마나 클 수 있는지 아는 것은 팀이 보고 프로세스를 더 효과적으로 계획하는 데 도움이 됩니다. 이는 감사 로그 데이터를 중심으로 반복적인 규정 준수 또는 검토 워크플로우를 구축할 때 특히 유용합니다.

| 항목 | 세부사항 |
|------|----------|
| 내보내기당 최대 레코드 수 | 각 내보내기는 최대 **500,000개의 레코드**를 지원합니다. |
| 내보내기 처리 | 내보내기는 백그라운드에서 **비동기적으로** 실행되므로 파일이 준비되는 동안 사용자는 계속 작업할 수 있습니다. |
| 파일 가용성 | 완료된 CSV 파일은 **30일간** 사용할 수 있습니다. |
| 만료된 파일 | 30일 후에는 파일을 더 이상 다운로드할 수 없으며, 새로운 내보내기를 요청해야 합니다. |
| 앱 내 감사 로그 보관 | 감사 로그는 하이퍼클래스에서 **60일간** 보관되므로, 정기적으로 내보내기를 하면 중요한 기록 데이터를 앱 외부에 보존하는 데 도움이 됩니다. |

## 감사 로그 내보내기 설정 방법

계정에서 기능을 사용할 수 있게 되면 감사 로그 내보내기를 사용하는 것은 간단합니다. 아래 단계를 따라 감사 로그를 내보내고, 내보내기 작업을 추적하며, 내보내기 관련 활동을 검토하세요.

- 내보내려는 감사 로그에 따라 에이전시 뷰 또는 하위 계정 뷰에서 Settings(설정) → Audit Logs로 이동합니다.
감사 로그(Audit Logs) 탭에 머물면서 사용하려는 필터를 적용하고 Export(내보내기)를 클릭합니다.

- 내보내기(Exports) 탭을 열고, 내보내기 상태를 모니터링하며, 완료되면 파일을 다운로드합니다.

## **자주 묻는 질문**

**Q: 누가 감사 로그를 CSV로 내보낼 수 있나요?**
A: '감사 로그 내보내기' 권한이 있는 사용자만 내보내기를 생성하고 다운로드할 수 있습니다.

**Q: 내보내기에 모든 감사 레코드가 자동으로 포함되나요?**
A: 아니요. 내보내기는 내보내기가 요청될 때 감사 로그에 적용된 필터를 사용합니다.

**Q: 한 번의 내보내기에 몇 개의 레코드가 포함될 수 있나요?**
A: 각 내보내기는 최대 500,000개의 레코드를 지원합니다. 결과가 너무 크면 필터를 좁혀서 다시 시도하세요.

**Q: CSV 파일을 얼마나 오래 다운로드할 수 있나요?**
A: 완료된 파일은 생성 후 30일간 다운로드할 수 있습니다.

**Q: 하이퍼클래스 내부에서 감사 로그는 얼마나 오래 저장되나요?**
A: 감사 로그는 하이퍼클래스에서 60일간 보관됩니다.

**Q: 내보내기가 요청되거나 다운로드되었는지 확인할 수 있나요?**
A: 네. 내보내기 요청과 다운로드 작업도 감사 로그에 기록됩니다.
