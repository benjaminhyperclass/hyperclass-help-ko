# 감사 로그

Hyperclass의 새롭게 디자인된 **감사 로그(Audit Logs)**를 활용해 보세요. 새로운 디자인 시스템 UI에 맞춰 개편되었습니다. 우측 드로어(drawer)를 통해 맥락을 유지하면서, 키보드 내비게이션으로 빠른 검토가 가능합니다. 영역별 필터의 차이점, 데이터 보관 기간(60일), 그리고 변경사항을 효율적으로 조사하는 실용적인 팁을 알아보세요.

# **감사 로그란?**

감사 로그는 Hyperclass 계정에서 누가, 무엇을, 언제 했는지에 대한 신뢰할 수 있는 기록입니다. 시간 스탬프가 포함된 항목(사용자, 모듈, 액션 맥락 포함)을 제공하여 관리자가 변경사항을 문제해결하고, 책임을 유지하며, 컴플라이언스 요구사항을 충족할 수 있도록 돕습니다. 새로운 디자인 시스템 UI는 메인 테이블을 벗어나지 않고도 세부사항을 검토할 수 있도록 경험을 현대화했습니다.

감사 로그는 Hyperclass 계정에서 수행된 주요 작업의 시간 스탬프 기록을 제공하여 관리자와 고급 사용자에게 투명성과 추적 가능성을 제공합니다. 업데이트된 인터페이스는 다른 새로운 디자인 시스템 화면과 일관된 더 깔끔하고 빠른 워크플로우로 동일한 인사이트를 제공합니다.

## **새로운 감사 로그 UI의 주요 장점**

재디자인의 가치를 이해하면 팀이 빠르게 적응하고 조사 중에 감사 로그를 자신 있게 활용할 수 있습니다.

- **드로어 뷰:** 세부사항이 우측 패널에서 열리므로 테이블 위치를 잃지 않습니다.

- **키보드 내비게이션:** ←/→ 키를 사용하여 드로어를 열어둔 채로 빠르게 항목을 이동할 수 있습니다.

- **일관된 컴포넌트:** 새로운 디자인 시스템 간격과 타이포그래피로 가독성이 향상되고 스캔 피로도가 줄어듭니다.

- **빠른 우선순위 결정:** 맥락을 유지하면서 변경사항을 확인하고, 증거를 수집하며, 추가 페이지 로드 없이 진행할 수 있습니다.

- **향상된 가시성 및 책임:** 사용자, 계정, 설정(Settings)에 대한 모든 변경사항을 추적하여 수정 사항의 명확한 기록을 유지합니다.

- **빠른 문제 해결:** 예상치 못한 변경사항을 신속하게 식별하고 문제를 해결하여 시간과 노력을 절약합니다.

- **향상된 보안 및 컴플라이언스:** 사용자 작업을 모니터링하여 무단 수정을 방지하고 정책 준수를 보장합니다.

- **효율적인 사용자 관리:** 계정 생성, 편집, 삭제를 포함한 사용자 활동을 추적하여 더 나은 팀 감독을 제공합니다.

## **드로어 & 키보드 내비게이션**

새롭게 디자인된 우측 드로어는 변경사항을 조사하는 동안 맥락을 유지해 줍니다. 드로어가 열리면 패널을 닫지 않고도 키보드를 사용하여 기록을 이동할 수 있어, 긴 감사 추적을 빠르고 정확하게 스캔하는 데 완벽합니다.

- **제자리에서 세부사항 열기:** 아무 행이나 클릭하면 필드 수준의 변경 전/후 값, 사용자, 타임스탬프가 포함된 세련된 우측 드로어가 열립니다. 페이지 새로고침이나 스크롤 위치 손실이 없습니다.

- **화살표 키로 내비게이션:** 드로어가 열린 상태에서 **좌우 화살표 키**를 눌러 이전 또는 다음 감사 기록으로 이동할 수 있습니다. 드로어가 제자리에서 업데이트되므로 집중력을 유지할 수 있습니다.

- **더 빠르게 작업, 더 적게 클릭:** 키보드에 손을 두어 잘못된 클릭을 최소화하고 긴 조사 세션 중 검토 속도를 높입니다.

- **통제력 유지:** **X** 아이콘이나 **Esc** 키로 드로어를 닫으면 테이블의 정확한 위치로 돌아갑니다.

## **삭제된 페이지 빌더 에셋이 로깅됩니다**

감사 로그는 이제 사용자가 저장된 페이지 빌더(Page Builder) 에셋을 삭제할 때 **삭제됨** 항목을 생성합니다. 포함되는 에셋:

- 섹션 템플릿
- 글로벌 섹션
- 유니버설 섹션
- 요소(Element) 템플릿
- 유니버설 요소 템플릿

각 삭제 항목에는 **에셋 이름**과 **에셋 ID**가 포함됩니다. 이는 퍼널(Funnel), 웹사이트(Website), 스토어, 웨비나에서 사용되는 에셋에 적용됩니다.

## 감사 로그 필터 이해하기

각 필터는 결과를 집중시켜 가장 관련성 높은 이벤트를 빠르게 분리할 수 있도록 도와줍니다. 각 옵션이 무엇을 하는지 이해하면 더 적은 클릭으로 더 빠르게 감사할 수 있습니다.

- **사용자:** 특정 팀원을 선택하여 그들이 수행한 액션만 봅니다. 개별 기여도 모니터링과 의심되는 편집 조사에 이상적입니다.

![사용자 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055831976/original/8XutF7qifjQ5ZCxF1n-1MWDDCmahm8XHDg.png?1760346219)

- **모듈:** 플랫폼의 영향받은 영역(예: **사용자, 계정, 연락처(Contacts), 기회 관리(Opportunities), 웹사이트/퍼널/스토어, 커스텀 오브젝트, 리스팅**)을 선택하여 해당 시스템의 이벤트에 집중합니다.

![모듈 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055832012/original/0gndVsLkGlGh1R6_bx-3uo6CwSS5jGSxhg.png?1760346234)

- **액션 타입:** **생성, 업데이트, 삭제**와 같은 변경 유형으로 결과를 제한합니다. 중요한 변경사항을 빠르게 찾는 데 유용합니다.

![액션 타입 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055832024/original/emKTcDtli8VRTbJjO79_c-xCEZDlVSZA0Q.png?1760346246)

- **시간 범위:** 정의된 창(예: 오늘, 최근 7일, 또는 최대 60일)으로 결과를 제한하여 변경이 발생한 시점과 일치시킵니다.

![시간 범위 필터](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055832034/original/rKVwzwK1WHGuhVvCoGqa7rhI2bcpcTVxyw.png?1760346257)

## 감사 로그 접근 방법

재디자인은 **에이전시(Agency)**와 **하위 계정(Sub-Account, Location)** 맥락 모두에서 실시간으로 적용됩니다. 검토가 필요한 맥락에서 감사 로그를 열어주세요. 각 뷰는 자체 범위만 표시합니다(에이전시 전체 대 로케이션별).

- **맥락 선택:**
  에이전시 전체 활동: **에이전시 → 설정(Settings) → 감사 로그(Audit Logs)**로 이동
  하위 계정 활동: **하위 계정 → 설정(Settings) → 감사 로그(Audit Logs)**로 이동

![감사 로그 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055835291/original/Pnzxpj5NKxcRNUrlIRqd1WHHRGBOkjTxuQ.gif?1760347591)

- 감사 로그 목록에서 미리보기(눈 모양) 버튼을 클릭하거나 행의 아무 곳이나 클릭하여 해당 로그의 세부사항 **드로어**를 **엽니다**

![로그 세부사항 열기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055834650/original/9Oft3dRvCq76JHfDu5NXJqJ3he3bWvYu-w.png?1760347370)

- 키보드의 ←/→ 키를 사용하여 드로어를 열어둔 채로 항목을 순환합니다.

![키보드 내비게이션](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055835409/original/6jLq2dLOG-Y7jIhz1YI2mt45liW0QYgLOA.gif?1760347629)

- **필터를 적용**하여 결과를 좁힙니다. **사용자** 필터는 **에이전시와 하위 계정** 감사 로그 모두에서 사용할 수 있습니다. (다른 필터 옵션인 **모듈**과 **액션**은 모듈에 따라 다를 수 있습니다.)

![필터 적용](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055834175/original/6gWoK2woPaa6YUu0lT01f52tQSHzqWG1Dw.png?1760347099)

- 모듈별 검토(예: 퍼널/웹사이트/웨비나/스토어)를 위해서는 **관련 하위 계정 내에서 설정(Settings) → 감사 로그(Audit Logs)**를 열어 해당 항목을 확인하세요.

## **자주 묻는 질문**

**Q: 누가 감사 로그에 접근할 수 있으며, 어디서 찾을 수 있나요?**
에이전시 수준 로그는 **에이전시 → 설정(Settings) → 감사 로그(Audit Logs)**에서 **에이전시 관리자**가 사용할 수 있습니다. 하위 계정 내의 모듈 수준 로그는 일반적으로 **하위 계정 → 설정(Settings) → 감사 로그(Audit Logs)**에서 **계정 관리자**가 사용할 수 있습니다.

**Q: 로그는 얼마나 오래 보관되나요?**
감사 로그 항목은 **60일** 동안 보관된 후 삭제됩니다. 해당 기간 내에 필요한 증거를 수집하도록 계획하세요.

**Q: 감사 로그를 내보낼 수 있나요?**
대부분의 영역에서 기본 UI 내보내기는 현재 사용할 수 없습니다. 일부 모듈(예: 리스팅)은 API를 통한 내보내기를 지원할 수 있습니다.

**Q: 많은 항목을 검토하는 가장 빠른 방법은 무엇인가요?**
먼저 **날짜**, 그 다음 **액션/모듈**을 적용하여 목록을 좁힙니다. 첫 번째 행을 열고 **←/→** 키를 사용하여 드로어를 열어둔 채로 결과를 단계적으로 확인합니다.

**Q: 삭제된 항목(예: 기회 관리)을 빠르게 찾아서 복원하는 방법은?**
**액션 = 삭제됨**과 관련 **모듈**(예: 기회 관리)로 필터링합니다. 항목을 열어서 지원되는 경우 복원 옵션에 접근합니다. 참조: [기회 관리 및 파이프라인 삭제 & 복원](../05-기회-파이프라인/시작하기/deleting-and-restoring-opportunities-pipelines.md).

**Q: 팀이나 기기마다 타임스탬프가 다르게 보이는 이유는?**
A: 시간대 차이로 인해 불일치가 발생할 수 있습니다. 기기의 시간대와 하위 계정 로케일을 확인하세요.

**Q: WordPress 접근/활동 로그는 감사 로그와 어떤 관련이 있나요?**
A: WordPress 로그는 주요 이벤트(예: 사이트 생성/삭제)에 대한 플랫폼 감사 로그를 보완합니다. 두 가지 모두 사용하여 종단 간 감사 추적을 구성합니다. 참조: [WordPress 접근 및 활동 로그](../06-사이트/워드프레스/wordpress-access-activity-logs.md).
