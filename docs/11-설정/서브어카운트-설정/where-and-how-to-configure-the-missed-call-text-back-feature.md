# 부재중 전화 문자 답장 기능 설정 방법과 위치

중요한 전화를 놓쳐서 비즈니스 기회를 잃고 싶지 않으신가요? Hyperclass는 편리한 부재중 전화 문자 답장 기능을 제공하여 항상 고객과의 연결을 유지할 수 있도록 도와드립니다. 이 기능을 빠르게 활성화하면 부재중 전화에 자동으로 문자 메시지를 보낼 수 있습니다. 고객이 가장 필요로 하는 순간에 연결할 수 있는 모든 기회를 놓치지 않도록 보장합니다.

이 가이드에서는 Hyperclass의 부재중 전화 문자 답장 기능을 설정하고 사용하는 데 필요한 모든 정보를 안내해드립니다. 이를 통해 커뮤니케이션을 간소화하고 비즈니스 성장에 집중할 수 있습니다. 시작해볼까요!

**중요사항:** 부재중 전화 문자 답장 기능은 짧은 시간 내에 발신자가 여러 번 시도하더라도 모든 부재중 전화에 대해 SMS 알림을 발송합니다. 부재중 전화에 대한 여러 SMS 알림을 받는 것을 방지하려면 [워크플로우에서 기존 레시피를 생성하고 커스터마이징](https://www.youtube.com/watch?v=ZwvCPUxgMaM)하여 선호도에 따라 관리하는 것을 권장합니다 (20분 정도의 대기 단계를 추가하거나, 태그를 사용하여 필터링 - 1차 통화, 2차 통화 등).

# **부재중 전화 문자 답장이란?**

부재중 전화 문자 답장은 수신 전화를 놓쳤을 때 발신자에게 자동으로 문자 메시지를 보내는 기능입니다. 이 기능은 비즈니스가 부재중 전화를 즉시 확인했다는 것을 알리고, 기대치를 설정하며, 팀이 부재중일 때 발신자가 문자로 대화를 계속할 수 있도록 격려합니다.

이 기능은 특히 업무시간 외 전화를 받는 비즈니스나 대량의 수신 전화를 처리하는 경우, 또는 수동 콜백 과정에 의존하지 않고 더 빠른 첫 응답을 원하는 비즈니스에 특히 유용합니다.

---

## **부재중 전화 문자 답장의 주요 장점**

- **더 빠른 후속 조치**: 부재중 전화 발신자에게 즉시 자동 문자를 보내 전화가 확인되었음을 알립니다.

- **향상된 고객 경험**: 빠른 응답은 발신자를 안심시키고 대화를 계속할 수 있는 다른 방법을 제공합니다.

- **더 많은 개인화**: 머지 필드를 사용하여 연락처 이름이나 비즈니스 이름 같은 세부 정보를 메시지에 포함할 수 있습니다.

- **간단한 설정**: 기능을 활성화하고, 메시지를 커스터마이징하고, 테스트를 보내고, 변경사항을 저장하면 됩니다.

- **향상된 리드 응답**: 팀이 전화를 받을 수 없을 때 잠재 리드와의 연결을 유지합니다.

## 부재중 전화 문자 답장 기능 설정 방법

#### 1단계: 설정 열기

하위 계정에서 왼쪽 하단의 **Settings(설정)**을 클릭합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066734906/original/OjFHNIoeULj0h8e-0ZPWraTyv3Akzr7Zhw.png?1773264995)

#### 2단계: 부재중 전화 문자 답장으로 이동

- 왼쪽 사이드 메뉴에서 **Phone System(전화 시스템)**을 클릭합니다.

- 상단 네비게이션에서 **Voice(음성)**를 클릭합니다.

- Voice 아래에서 **Voicemail & Missed Call Text Back(음성사서함 및 부재중 전화 문자 답장)**을 엽니다.

- 사이드 패널에서 **Missed Call Text Back(부재중 전화 문자 답장)**을 클릭합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066734967/original/hCHydUVEIyYEVu_BVFFDM246xJXztOXPSw.png?1773265066)

#### 3단계: 기능 활성화

Enable Missed Call Text Back(부재중 전화 문자 답장 활성화) 체크박스를 선택하여 기능을 켭니다. 그다음 **Customize(커스터마이징)**를 클릭하여 누군가 전화를 놓쳤을 때 보낼 메시지를 맞춤 설정합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066735467/original/4EWpJaWoQ9luC5sgv6_WANHjVeaO3_e4jA.png?1773266017)

#### 4단계: 메시지 커스터마이징

**Customize(커스터마이징)** 버튼을 클릭하면 자동 응답 메시지를 작성할 수 있는 팝업이 열립니다. **Enter message(메시지 입력)** 필드에 전화를 놓쳤을 때 보낼 내용을 입력합니다.

커스텀 값 태그 아이콘을 사용하여 연락처 이름과 같은 동적 플레이스홀더를 삽입할 수도 있습니다. 단, 이러한 기능은 시스템이 이전 상호작용을 통해 연락처를 이미 인식하고 있을 때만 작동한다는 점을 기억하세요.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066735521/original/pMhp0ohGahjc31N7CeMboNaTo8iHB28GJg.jpeg?1773266117)

#### 5단계: 테스트 메시지 보내기

메시지 편집기 아래에서 테스트 메시지를 보낼 수 있는 필드를 찾을 수 있습니다. 고객에게 전송될 때 메시지가 어떻게 보일지 미리 보기 위해 사용할 번호를 선택합니다.

검토 후 **Save(저장)** 버튼을 클릭하여 설정을 완료하고 업데이트된 부재중 전화 문자 답장 구성을 활성화합니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066735539/original/r7OeTZfb0lEhUEmClUY-X3M8JnQH4dPtvA.png?1773266177)

## **음성사서함 설정**

음성사서함 설정은 수신 전화가 응답되지 않을 때 어떤 일이 발생하는지를 결정합니다. 이 옵션을 통해 음성사서함이 트리거되기 전에 전화벨이 울리는 시간을 제어하고, 선택적으로 기본 음성사서함 인사말을 커스텀 녹음으로 대체할 수 있습니다.

### 수신 전화 타임아웃 조정

이 옵션을 통해 부재중 전화 문자 답장이 트리거되기 전에 전화벨이 울려야 하는 시간을 설정할 수 있습니다. 슬라이더를 사용하여 타임아웃 시간을 선택합니다(일반적으로 10~20초가 권장됨).

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066036199/original/NTcwyGYzj0J8NJgwbl1AhEdPpRwY4k4xBA.png?1772475723)

### 음성사서함 오디오 파일 업로드 (선택사항)

원한다면 여기에서 커스텀 음성사서함 메시지를 추가할 수도 있습니다. mp3, wav 또는 유사한 형식의 사전 녹음된 파일을 업로드합니다. 이는 통신사의 일반적인 음성사서함 대신 재생됩니다.

![](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155066036213/original/375vNtucIblJVv2XLsfRPNTC-B-cJAWcJw.png?1772475746)

## 자주 묻는 질문

**Q: 부재중 전화 문자 답장 메시지가 메시징 한도에 포함되나요?**
네. 이러한 SMS 메시지는 전화 메시징 사용 한도에 포함됩니다.

**Q: 부재중 전화 문자 답장 메시지를 보내는 데 어떤 번호가 사용되나요?**
부재중 전화 문자 답장 메시지는 기본 번호를 사용합니다.

**Q: 부재중 전화 후속 조치에 SMS 대신 왓츠앱을 사용할 수 있나요?**
SMS 기반 부재중 전화 문자 답장 기능으로는 불가능합니다. 왓츠앱을 통해 부재중 전화 발신자에게 메시지를 보내려면 Hyperclass의 별도 부재중 전화 왓츠앱 답장 기능을 사용하세요.

**Q: 부재중 전화 문자 답장이 업무시간 외 전화에도 작동하나요?**
네. 이 기능은 업무시간 외 부재중 전화에 유용합니다. 자동으로 발신자를 확인하고 팀이 부재중일 때에도 문자로 대화를 계속할 수 있는 방법을 제공하기 때문입니다.
