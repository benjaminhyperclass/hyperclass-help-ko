# 미디어 저장소 완벽 가이드!

이 가이드는 Hyperclass의 업그레이드된 미디어 저장소(Media Storage) 기능을 탐색하고 활용하는 방법에 대한 포괄적인 지침을 제공합니다. 파일 업로드와 정리부터 향상된 미리보기와 보안 통합까지, 모든 업데이트는 워크플로우를 더욱 원활하고 직관적으로 만들어줍니다.

## 미디어 저장소 접근하기

미디어 저장소에 접근하려면 대시보드의 좌측 사이드바에서 미디어 저장소(Media Storage)를 클릭하세요.

![미디어 저장소 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062681093/original/013C4N9WatdiKmmQLiBb-ckIURezCEDYyQ.png?1768421404)

## 파일 업로드하기

드래그 앤 드롭으로 파일을 쉽게 업로드하세요. 컴퓨터에서 미디어 저장소 인터페이스로 미디어를 드래그하기만 하면 됩니다. 버튼을 클릭할 필요가 없습니다. 개선된 기능으로 더 빠르고 직관적인 업로드 경험을 제공합니다.

또한 폰트 파일(.ttf, .otf, .woff, .woff2)을 최대 100MB까지 업로드할 수 있습니다. 이 파일들은 Custom CSS를 사용하여 웹사이트나 퍼널의 브랜딩을 커스터마이징하는 데 활용할 수 있습니다.

## **미디어 사용량 모달로 저장소 사용량 확인**

계정이 얼마나 많은 저장 공간을 사용하고 있는지, 그 사용량이 어디에서 발생하는지 세부 내용을 확인할 수 있습니다. 모달은 색상으로 구분된 막대로 총 저장 용량을 시각화하고, 모듈별 세부 내역(미디어 라이브러리, 대화, 멤버십, 연락처, 캘린더 등)을 표시하며, 각 모듈을 확장하여 파일 타입별(이미지, 비디오, 문서, 오디오) 사용량을 볼 수 있습니다.

미디어 사용량 모달로 저장소 사용량을 확인하려면 다음 단계를 따르세요:

- **점 세 개 버튼(⋮)**을 클릭하고 저장소(Storage)를 선택하세요.

![저장소 메뉴](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062681430/original/XI8ljBWL91zXbe_ZyMzRJOdu8ruXKBmIyQ.png?1768422309)

- 파일 타입별 사용량을 보려면 원하는 모듈을 클릭하세요.

![모듈별 사용량](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062681354/original/DE-M111JGPTTIIrXoiEw0z3VKURzojMwvA.png?1768422129)

## 미디어 정렬

강력한 정렬 옵션을 사용하여 미디어 라이브러리를 쉽게 정리하세요:

- 이름순 정렬: A부터 Z / Z부터 A
- 크기순 정렬: 작은 것부터 / 큰 것부터
- 수정일순 정렬: 최신순 / 오래된 순

## 그리드와 목록 뷰

선호도에 따라 그리드 뷰와 목록 뷰 사이를 토글할 수 있습니다. 두 뷰 모두 향상된 메타데이터 가시성을 제공합니다.

- 방향 배지(그리드 뷰): 각 이미지는 이제 가로형, 세로형, 정사각형 형식을 표시하는 배지를 보여줍니다.
- 픽셀 치수(목록 뷰): 각 이미지 이름 옆에 너비 × 높이 값이 표시되므로 디자인에 적합한지 빠르게 평가할 수 있습니다.

**참고:** 이전에 업로드한 이미지는 백그라운드 처리 지연으로 인해 즉시 치수가 표시되지 않을 수 있습니다.

## 우클릭과 호버 메뉴를 통한 빠른 작업

우클릭 드롭다운 메뉴나 마우스 오버 시 나타나는 빠른 작업 버튼을 사용하여 파일 옵션에 더 빠르게 접근하세요.

이를 통해 상단 네비게이션 바에만 의존할 필요가 없습니다. 사용 가능한 작업은 다음과 같습니다:

- 이름 바꾸기
- 삭제
- 폴더로 이동
- 미리보기
- 삽입

## 새로운 삽입 및 미리보기 기능

제품 내에서 사용되는 미디어 저장소(예: 웹사이트/퍼널 빌더)에서 다음 기능을 볼 수 있습니다:

- 단일 파일 선택용 "미디어 삽입" 버튼 – 페이지를 이동하지 않고 즉시 삽입
- 미리보기 옵션 – 빠른 확인을 위해 점 세 개 메뉴에서 접근 가능
- 미리보기 화면에서 직접 삽입 – 뷰 간 왔다 갔다 하는 번거로움 제거

## 폴더 관리 업그레이드

#### 폴더 섹션 접기 또는 펼치기

큰 폴더 구조를 가지고 있다면, 폴더 섹션을 접어서 정리하고 파일에 집중할 수 있습니다. 미디어 저장소에서 폴더 옆의 화살표를 클릭하여 폴더 목록을 접으세요. 다시 클릭하면 폴더 목록이 펼쳐집니다.

![폴더 접기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155064213379/original/G4GeHcJHvWE6FQh7TbvL_lxFwkvmKIpD3g.png?1770233460)

#### 전체 폴더 이동

내용(중첩된 파일과 하위 폴더 포함)과 함께 전체 폴더를 이동할 수 있습니다. 방법은 다음과 같습니다:

- 원하는 폴더 옆의 **점 세 개(⋮)** 메뉴를 클릭하고 폴더로 이동(Move to Folder)을 선택하세요.

![폴더 이동](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062681158/original/brQ-banAB_56UTODCRgje7xoH4QnU7-xzw.jpeg?1768421594)

- 기존 폴더를 선택하거나 새 폴더를 만드세요.
- 이동을 확인하면 모든 것이 함께 재배치됩니다.

참고: 이 과정에서 하위 폴더는 그대로 유지됩니다.

## 안전한 삭제를 위한 휴지통 기능

삭제된 미디어 파일은 휴지통으로 이동하여 영구 삭제되기 전까지 30일 동안 보관됩니다.

- 우측 상단의 점 세 개 메뉴를 클릭하고 휴지통(Trash)을 선택하세요.

![휴지통 접근](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062681193/original/XNTvb1LKzxjgQsuL9BM67N6fYXpyt8KMRA.jpeg?1768421700)

- 파일을 복원(Restore)하거나 영구 삭제(Delete Permanently)하도록 선택하세요.

![휴지통 관리](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155062681195/original/vrmNhmNMZex7hYAbEnDI5yLl6dSCU1qZwA.jpeg?1768421711)

중요: 30일 후에는 파일을 복구할 수 없습니다.

휴지통 접근은 로케이션 전체에 적용됩니다. 같은 로케이션에 로그인한 모든 사용자가 누가 파일을 휴지통으로 이동했는지에 관계없이 휴지통의 항목을 보고 복원할 수 있습니다. 이는 원래 사용자가 삭제되거나 더 이상 접근할 수 없는 경우 파일 손실을 방지하는 데 도움이 됩니다.

## Google Drive 연동 개선사항

Drive 연결(Connect Drive)을 통해 Google Drive에서 파일을 가져올 수 있습니다(소유한 파일만).

**중요:** 공유 파일이나 공유 파일의 바로가기는 접근 및 권한 설정으로 인해 제한되어 표시할 수 없습니다. 현재로서는 사용자가 소유한 파일만 표시할 수 있습니다. 이 제한은 Google Drive가 가시성과 소유권을 처리하는 방식 때문입니다.

**해결 방법:** 바로가기 대신 공유 파일의 복사본을 만들 것을 권장합니다. 이렇게 하면 예상대로 나타나고 작동합니다.

개인정보 보호 안내: "사용자로 로그인(Login As)" 기능 사용 시(예: 관리자 접근):

- Drive 연결 버튼이 비활성화되고 회색으로 표시됩니다.
- 원래 사용자의 Google Drive 데이터는 표시되지 않아 엄격한 개인정보 보호를 보장합니다.

![구글 드라이브 연동](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155045264698/original/HP5NHp025nopa0g-0c_PzcnKLh-1RnUeGA.png?1744887963)

## 제품 전반의 원활한 연동

미디어 저장소는 다른 모듈과 깊이 연동됩니다. 퍼널을 편집하거나 웹사이트를 디자인할 때, 미디어는 전체 화면이 아닌 모달 창을 통해 열려 더 쾌적하고 비방해적인 사용자 경험을 제공합니다.

**참고:** 미디어 라이브러리가 퍼널 모달 내에서 열릴 때도 드래그 앤 드롭 업로드가 작동합니다. 삽입하기 전에 업로드하려면 모달의 드롭 영역 어디에나 파일을 놓으세요.

### 크로스 브라우저 호환성(Chrome, Firefox, Safari)

이제 지원되는 모든 형식에 대해 **Chrome**, **Firefox**, **Safari**에서 표준화된 검증, 업로드, 목록 표시, 미리보기 동작이 적용됩니다. 이는 브라우저별 문제를 줄이고 미리보기를 더 예측 가능하게 만듭니다.

**JSON, CSS, OGG/OGA, M4V, HEIC**를 포함한 정식 목록과 일관된 **ZIP** 처리에 대해서는 미디어 라이브러리 사양의 **지원 형식 및 브라우저 호환성** 매트릭스를 참조하세요.

### 미리보기 참고사항

인라인 썸네일/미리보기는 브라우저/OS에서 지원할 때 렌더링됩니다. 그렇지 않으면 파일을 다운로드로 이용할 수 있습니다. 더 빠른 미리보기를 위해 에셋 크기를 적절하게 유지하세요.

## 자주 묻는 질문

**Q: 그리드 뷰와 목록 뷰를 모두 사용할 수 있나요?**  
네! 추가 이미지 방향과 픽셀 치수 표시기가 포함되어 있습니다.

**Q: 어떤 정렬 옵션을 사용할 수 있나요?**  
이름, 크기, 수정일 모두 오름차순과 내림차순으로 정렬할 수 있습니다.

**Q: 어떤 파일 타입을 업로드할 수 있나요?**  
이미지, 문서, 그리고 이제 폰트 파일(.ttf, .otf, .woff, .woff2)도 업로드할 수 있습니다.

**Q: 삭제된 파일은 어떻게 되나요?**  
삭제된 파일은 휴지통으로 이동하여 30일 동안 보관됩니다. 같은 로케이션에 로그인한 모든 사용자가 휴지통 항목을 보고 복원하거나 영구 삭제할 수 있습니다. 30일 후에는 파일이 자동으로 제거되어 복구할 수 없습니다.

**Q: 미디어 저장소를 Google Drive와 연동할 수 있나요?**  
네, 하지만 소유한 파일만 가능합니다. 공유 파일 바로가기는 표시되지 않습니다.

**Q: "사용자로 로그인" 기능이 미디어 접근에 어떤 영향을 주나요?**
