# 대시보드 권한 관리 방법

대시보드는 만든 사람만 사용하는 비공개로 설정할 수도 있고, 다른 역할의 사용자들과 다양한 접근 수준으로 공유할 수도 있습니다.

## **대시보드 권한이란?**

대시보드는 다음과 같은 권한 수준을 제공합니다:

| 수준 | 설명 |
|------|------|
| 전체 권한 (FULL) | 대시보드를 생성, 편집, 공유, 삭제할 수 있는 권한을 부여합니다 |
| 편집 (EDIT) | 대시보드와 위젯을 편집할 수 있는 권한을 부여합니다. 대시보드 삭제는 불가능합니다 |
| 보기 (VIEW) | 대시보드에 읽기 전용으로 접근할 수 있는 권한을 부여합니다 |
| 접근 불가 (NO ACCESS) | 대시보드 접근을 완전히 제한합니다 |

다음은 권한 수준별로 대시보드에서 수행할 수 있는 작업입니다:

| 작업 | 전체 권한 | 편집 | 보기 | 접근 불가 | 비공개 대시보드 소유자 |
|------|----------|------|------|----------|-------------------|
| 권한 관리 | ✅ | ❌ | ❌ | ❌ | ✅ |
| 대시보드 삭제 | ✅ | ❌ | ❌ | ❌ | ✅ |
| 기본 대시보드로 설정 | ✅ | ❌ | ❌ | ❌ | ✅ |
| 대시보드 편집 | ✅ | ✅ | ❌ | ❌ | ✅ |
| 대시보드 복제 | ✅ | ✅ | ❌ | ❌ | ✅ |
| 대시보드 보기 | ✅ | ✅ | ✅ | ❌ | ✅ |
| 다른 하위 계정으로 복사 (에이전시 레벨만) | ✅ | ✅ | ✅ | ❌ | ✅ |

대시보드 소유자나 전체 권한을 가진 역할만이 대시보드 권한을 편집할 수 있습니다.

에이전시 관리자는 항상 모든 비공개가 아닌 대시보드에 전체 접근 권한을 갖습니다.

하위 계정 관리자나 대시보드에 전체 접근 권한을 가진 에이전시 역할만이 해당 대시보드를 기본 대시보드로 설정할 수 있습니다.

에이전시 역할만이 대시보드를 다른 하위 계정으로 복제할 수 있습니다.

## **대시보드 권한 관리하기**

### 새 대시보드에 권한 설정하기

**1단계: 대시보드 추가하기**

대시보드(Dashboard)로 이동해서 대시보드 선택 메뉴(파란색)를 클릭한 다음, 대시보드 추가(Add Dashboard)를 클릭합니다.

![대시보드 추가하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433881/original/rQJh1RSrr244FcvqH8VsYlSNaL1_5L9_6A.png?1743641657)

**2단계: 빈 대시보드 선택하기**

빈 대시보드 만들기(Create a Blank Dashboard) 아래에서 선택(Select)을 클릭합니다.

![빈 대시보드 선택하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433890/original/lpbrI6m-ixLatlPsrrpGNDCwvkylimB_Fw.png?1743641724)

**3단계: 권한 설정하고 저장하기**

새 대시보드(New Dashboard) 모달에서 대시보드를 비공개로 두거나, 이 옵션을 끄고 공유 권한에 접근하세요. 설정이 만족스러우면 확인(Confirm)을 클릭하여 해당 권한으로 새 대시보드를 생성합니다.

![권한 설정하고 저장하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433908/original/9V9Sr9PLG29YqUu0VXW1O2t2yRRyqmHvdQ.png?1743641801)

## 기존 대시보드에 권한 설정하기

**1단계: 권한 관리 액션 실행하기**

기존 대시보드에서 점 3개 "액션 메뉴"를 클릭한 다음, 권한 관리(Manage Permissions)를 클릭합니다.

![권한 관리 액션 실행하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433750/original/XBQjE4b9QWgmSCGpAV3ShVjvdPCuPizE5g.png?1743641029)

**2단계: 권한 설정하고 저장하기**

비공개 토글을 사용하여 비공개(본인만) 또는 공유 권한 간에 전환합니다.

권한 변경이 만족스러우면 저장(Save)을 클릭하여 모달을 닫고 대시보드로 돌아갑니다.

![권한 설정하고 저장하기](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044433989/original/whua1cN1ou81Jrr3_UI7SGEnO-PPebUdng.png?1743642044)

## **에이전시 vs 하위 계정 뷰**

로그인한 사용자의 역할에 따라 권한 화면과 옵션이 달라집니다.

### **에이전시 대시보드 권한 뷰**

에이전시 관리자는 만든 사람이 비공개로 표시한 대시보드를 제외하고는 항상 모든 대시보드에 전체 접근 권한을 갖습니다. 에이전시 뷰에서는 접근 설정이 가능한 세 가지 역할이 있습니다: 에이전시 사용자(Agency User), 하위 계정 관리자(Account Admin), 하위 계정 사용자(Account User).

![에이전시 대시보드 권한 뷰](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155044434191/original/ag_owqTRkCZiAF5mHoJ3MUHYMr0kLRkY-g.png?1743643164)

### **하위 계정 대시보드 권한 뷰**

하위 계정 뷰에서는 접근 설정이 가능한 두 가지 역할이 있습니다: 하위 계정 관리자(Account Admin)와 하위 계정 사용자(Account User).

![하위 계정 대시보드 권한 뷰](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155023290478/original/LNohQGZJfVvUZVBvP4IWBFbbhDDct6fjHA.png?1711017852)

## **역할과 권한 계층구조**

대시보드 권한은 접근 수준의 원활한 관리를 위해 명확한 계층구조를 따릅니다:

에이전시 관리자 > 에이전시 사용자 > 하위 계정 관리자 > 하위 계정 사용자

상위 역할의 권한이 하위 역할의 권한보다 우선합니다. 하위 역할은 상위 역할보다 높은 권한을 가질 수 없습니다.

예를 들어, 대시보드가 하위 계정 관리자:편집으로 설정되어 있고 하위 계정 사용자:편집이 하위 계정 관리자:보기로 변경되면, 하위 계정 사용자도 자동으로 보기로 변경됩니다.

## **데이터 내보내기**

데이터 내보내기 권한은 사용자가 대시보드 위젯에서 데이터를 내보낼 수 있는지를 제어합니다. 이를 통해 팀이 보고서를 공유하면서도 관리자가 대시보드 데이터를 언제 다운로드할 수 있는지 제어할 수 있습니다.

**자주 묻는 질문**

**Q: 권한이 설정되지 않으면 어떻게 되나요?**  
기본적으로 대시보드는 제한되지 않는 한 모든 사용자에게 표시됩니다.

**Q: "본인 데이터만 보기"는 정확히 무엇을 제한하나요?**  
사용자에게 배정된 레코드(연락처, 기회 등)와 관련된 데이터만 보여주도록 대시보드 위젯을 필터링합니다.

**Q: 대시보드에서 사용자나 역할을 제거하려면 어떻게 하나요?**  
해당 대시보드의 권한 모달을 다시 열고 사용자나 역할의 체크를 해제하세요.
