# 커스텀 오브젝트 시작하기

커스텀 오브젝트는 **연락처, 회사, 기회 관리에 딱 맞지 않는 비즈니스 추적 대상을 모델링**할 수 있게 해줍니다. CRM이 비즈니스 고유의 실제 엔티티, 관계, 프로세스를 반영해야 할 때 이상적입니다.

관리자는 **Object Settings(오브젝트 설정)**에서 커스텀 오브젝트를 정의하고, 커스텀 필드를 추가하며, 연관 관계를 설정하고, **워크플로우, 폼, 리포팅** 등의 도구에서 활용할 수 있습니다.

이 가이드는 커스텀 오브젝트가 무엇인지 이해하고, 사용 사례에 적합한지 판단하는 데 도움을 드립니다.

# 왜 커스텀 오브젝트를 사용하나요?

커스텀 오브젝트는 사람 중심의 CRM을 넘어서 비즈니스 실제 운영 방식을 반영하는 데이터 모델을 구축하는 데 도움을 줍니다.

커스텀 오브젝트를 사용하면 다음이 가능합니다:

- **대규모 CRM 체계적 관리**
  관련 없는 데이터를 연락처나 기회 관리에 억지로 넣어서 발생하는 복잡한 레코드와 사용성 저하를 방지할 수 있습니다.

- **실제 엔티티를 일급 레코드로 모델링**
  부동산, 반려동물, 프로젝트, 보험 등을 각각의 라이프사이클, 필드, 자동화를 가진 독립적인 레코드로 처리할 수 있습니다.

- **더 깔끔한 자동화와 리포팅 구현**
  데이터가 올바르게 구조화되면 워크플로우, 필터, 대시보드가 더 간단하고 정확하며 유지하기 쉬워집니다.

- **복잡한 관계를 자연스럽게 지원**
  일대다, 다대다 관계(예: 한 연락처가 여러 반려동물이나 부동산을 소유)를 더 쉽게 관리하고 이해할 수 있습니다.

요약하면: 커스텀 오브젝트는 CRM을 깔끔하고 확장 가능하며 비즈니스 실제 운영과 일치하도록 유지하는 데 도움을 줍니다.

# 커스텀 오브젝트란 무엇인가요?

커스텀 오브젝트를 다음과 같은 항목들의 일급 CRM 레코드라고 생각해보세요:

- 부동산
- 반려동물
- 프로젝트
- 구독
- 보험

각 커스텀 오브젝트는 다음을 가집니다:

- 고유한 레코드와 필드
- 다른 CRM 오브젝트와의 연관 관계
- 워크플로우와 자동화 지원

# 언제 커스텀 오브젝트를 사용해야 하나요?

다음의 경우 커스텀 오브젝트가 적합합니다:

**데이터가 사람이나 회사에 속하지 않는 경우**

"사물"(자산, 목록, 보험, 반려동물)을 추적하는 것이라면, 사람이 아닌 경우 커스텀 오브젝트가 일반적으로 적절한 추상화입니다.

**구조화되고 반복 가능한 레코드가 필요한 경우**

동일한 유형의 엔티티가 여러 번 존재하고 일관된 필드, 보기, 자동화가 필요한 경우입니다.

**CRM에서 더 깊은 관계가 필요한 경우**

커스텀 오브젝트는 연락처, 회사, 기회 관리, 심지어 다른 커스텀 오브젝트와도 연결할 수 있어 고급 워크플로우를 구현할 수 있습니다.

**예시**

- 구매자와 에이전트에 연결된 부동산 목록
- 소유자에 연결된 여러 반려동물
- 고객과 기회에 연결된 프로젝트

# 언제 표준 CRM 오브젝트를 유지해야 하나요

커스텀 오브젝트는 강력하지만 항상 올바른 답은 아닙니다.

다음의 경우 커스텀 오브젝트를 피하세요:

**표준 오브젝트로 이미 작동하는 경우**

연락처, 회사, 기회 관리는 커스텀 오브젝트가 항상 지원하지 않는 내장 기능들을 제공합니다.

**연락처 전용 기능이 필요한 경우**

일괄 이메일, 마케팅 캠페인, 특정 커뮤니케이션 도구는 연락처에서만 작동합니다.

**같은 데이터를 중복으로 만드는 경우**

기존 오브젝트를 미러링하는 커스텀 오브젝트를 만들면 혼란과 일관성 없는 데이터가 발생할 수 있습니다.

대신 스마트 리스트를 사용해서 표준 오브젝트 내에서 다양한 세분화를 만들어보세요.

# 일반적인 예시

## 좋은 사용 사례

**부동산 – 주택**
"주택" 오브젝트를 만들어서 부동산 상세 정보(가격, 상태, 위치)를 추적하고 구매자, 판매자, 에이전트와 연관시킵니다.

**수의 클리닉 – 반려동물**
"반려동물" 오브젝트를 사용해서 의료 기록, 예방 접종, 예약을 저장하고 반려동물 소유자와 연결합니다.

## 권장하지 않는 경우

**이벤트 참석자**
주된 목적이 이메일 발송, 알림, 후속 조치라면 연락처가 이미 더 잘 처리합니다.

**내부 노트나 활동**
커스텀 오브젝트는 노트, 할 일, 활동을 대체해서는 안 됩니다. 이는 불필요한 복잡성을 추가합니다.

# 다음 단계는?

커스텀 오브젝트를 만든 후에는 다음이 가능합니다:

- 커스텀 필드 추가
- 연관 관계와 라벨 정의
- 워크플로우와 자동화 구축
- 커스텀 오브젝트에서 데이터를 수집하는 폼 구축
- 대시보드에서 데이터 시각화
