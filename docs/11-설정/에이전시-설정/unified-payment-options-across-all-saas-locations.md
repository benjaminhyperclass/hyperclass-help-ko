# 모든 SaaS 로케이션의 통합 결제 옵션

통합 결제 옵션은 Hyperclass의 모든 SaaS 로케이션에 원활하고 유연한 청구를 제공합니다. 여러 하위 계정을 관리하든 하나의 로케이션을 관리하든, 이제 저장된 카드, 월렛 크레딧 또는 (에이전시의 경우) 에이전시 카드로 결제할 수 있습니다. 더 이상 버전별 불일치는 없습니다!

# **통합 결제 옵션 개요**

통합 결제 옵션은 SaaS V1과 V2 전반에 걸쳐 결제 선택을 표준화하고 향상시켜, 에이전시와 로케이션 사용자 모두가 모든 SaaS 로케이션에서 월렛과 카드 지원이 포함된 일관성 있는 역할 기반 체크아웃을 볼 수 있도록 합니다. 통합 결제 옵션은 버전 간 결제 흐름을 일치시킵니다. 이제 모든 SaaS 로케이션이 월렛 및 카드 기반 결제를 지원하여 일관성과 유연성을 보장합니다.

에이전시 사용자는 **카드로 결제**, **월렛으로 결제**, 에이전시 카드로 결제에 액세스할 수 있습니다.

로케이션 사용자는 **카드로 결제**와 **월렛으로 결제**에 액세스할 수 있습니다.

---

## 통합 결제 옵션의 주요 장점

- **일치된 경험:** 체크아웃 흐름이 SaaS V1과 V2에서 일관됩니다.

- **명확한 역할 가시성:** 에이전시와 로케이션 사용자는 자신에게 해당하는 옵션만 볼 수 있습니다.

- **월렛 가용성:** 모든 SaaS 로케이션에서 월렛 결제가 활성화됩니다.

- **불일치 해결:** V1과 V2 간 결제 방법의 기존 차이점이 해결되어 더욱 원활한 경험을 제공합니다.

---

## 결제 방법 개요

각 결제 옵션의 간단한 개요:

- **카드로 결제:** **설정(Settings)** → **결제/청구(Billing)** 하에 저장된 기본 카드로 청구됩니다.

![카드로 결제](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055112920/original/MMz10K1ZOmP3-A8K0ZdNWibqhqY2ysaMIg.png?1759419023)

- **월렛으로 결제:** 미리 충전된 크레딧에서 차감되며, 잔액이 임계값 아래로 떨어지면 자동 충전이 트리거됩니다.

![월렛으로 결제](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055112752/original/Qpx6xGfBnKO6oJO6fz_dS17jflc5Lv5rKA.png?1759418952)

- **에이전시 카드로 결제:** 에이전시 레벨 사용자에게만 표시되며, Hyperclass가 등록된 에이전시 카드로 청구합니다.

## **자주 묻는 질문**

**Q: 내 고객이 실수로 에이전시 카드를 사용할 수 있나요?**

아니요. 로케이션 사용자는 에이전시 카드 버튼을 볼 수 없으며, 자신의 월렛과 카드 옵션만 볼 수 있습니다.

**Q: 월렛의 자동 충전 임계값을 어떻게 조정하나요?**

설정(Settings) → 결제/청구(Billing) → 월렛 및 거래(Wallets & Transactions) → 자동 충전(Auto-Recharge)으로 이동하세요.

**Q: 에이전시 카드 청구가 내 고객에게 표시되나요?**

아니요. 에이전시 카드로 라우팅된 청구는 로케이션 인보이스에 표시되지 않습니다.

**Q: 새로운 체크아웃을 보려면 마이그레이션이 필요한가요?**

아니요. SaaS V1과 V2에서 결제 흐름이 자동으로 일치됩니다.
