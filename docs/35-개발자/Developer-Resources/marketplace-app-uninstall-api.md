# 마켓플레이스: 앱 제거 API

새로운 Hyperclass 마켓플레이스 기능인 앱 제거 API를 살펴보세요. 이 강력한 도구를 통해 개발자는 비활성 사용자 계정에서 휴면 상태인 앱을 제거할 수 있어 불필요한 웹훅 활동을 줄이고 성능을 최적화할 수 있습니다. 상세한 설정 가이드를 확인하고 유연한 웹훅 관리를 위한 향후 개선 사항에 대해 알아보세요.

## **앱 제거 API란 무엇인가요?**

앱 제거 API는 Hyperclass 마켓플레이스 내의 전용 기능으로, 앱 개발자가 비활성 또는 휴면 사용자 계정에서 자신의 애플리케이션을 프로그래밍 방식으로 제거할 수 있게 해줍니다. 사용하지 않는 계정에서 앱을 대상으로 제거함으로써 이 API는 앱 관리를 간소화할 뿐만 아니라 중복 웹훅 이벤트 처리를 최소화하여 리소스가 활성 사용자 상호작용에만 집중할 수 있도록 보장합니다.

---

## **앱 제거 API의 주요 장점**

이 기능은 여러 실질적인 이점을 제공하여 앱 생태계 내에서 향상된 효율성과 성능에 직접적으로 기여합니다:

- **오버헤드 비용 감소** – 휴면 계정에서 앱을 제거하면 중복 웹훅 트리거가 줄어들고 처리 비용이 절약됩니다.

- **시스템 지연 시간 개선** – 불필요한 이벤트가 줄어들면 활성 사용자에게 더 빠른 응답과 원활한 작업을 제공할 수 있습니다.

- **간소화된 앱 관리** – 비활성 계정의 정리를 자동화하여 더 깔끔하고 관리하기 쉬운 앱 배포가 가능합니다.

- **비용 효율성** – 활성화되고 수익을 창출하는 계정에 처리 능력을 집중하여 리소스 활용을 최소화합니다.

---

## **웹훅 구독 관리**

앱 제거 API가 휴면 계정의 웹훅 노이즈를 제어하는 즉각적인 필요를 해결하는 동안, 향후 개선사항이 준비되고 있습니다. 개발자는 계정별 웹훅 구독에 대한 제어 기능 향상을 기대할 수 있습니다. 이 예정된 기능은 이벤트 알림에 대한 더욱 세밀한 제어를 부여하여 특정 운영 요구사항에 맞게 성능 개선을 추가로 맞춤화할 수 있도록 설계되었습니다:

- **향상된 유연성** – 각 계정에 대해 웹훅 이벤트 구독 방식을 사용자 정의할 수 있습니다.

- **개선된 맞춤화** – 활성 계정과 비활성 계정 활동을 더 잘 구분할 수 있도록 이벤트 트리거를 조정할 수 있습니다.

- **최적화된 리소스 관리** – 웹훅 구성을 세밀하게 조정하여 처리 부하를 더욱 줄일 수 있습니다.

## **앱 제거 API 설정 방법**

앱 제거 API를 워크플로우에 통합하는 것은 간단하며 불필요한 웹훅 트래픽을 즉시 줄이기 시작할 수 있습니다. 원활한 설정 과정을 위해 다음 단계를 따라하세요:

1. **인증**: Hyperclass에서 요청을 승인할 수 있는 유효한 API 자격 증명이 있는지 확인하세요.

2. **휴면 계정 식별**: 앱의 분석 도구나 데이터베이스를 활용하여 제거가 필요한 비활성 사용자 계정을 찾아내세요.

3. **API 엔드포인트 호출**: [Public API 문서](https://marketplace.gohighlevel.com/docs/ghl/marketplace/uninstall-application)에 설명된 전용 앱 제거 API 엔드포인트를 사용하여 제거 프로세스를 시작하세요.

![앱 제거 API 문서 스크린샷](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155055533358/original/YHVXP1etZ8BJdZMFB_6OxgDQR1xSstvPMg.png?1759933243)

4. **웹훅 활동 모니터링**: 웹훅 로그를 확인하여 제거된 휴면 계정의 이벤트가 중단되었는지 확인하고 활성 계정이 향상된 효율성으로 작동하는지 보장하세요.

5. **프로세스 자동화**: 휴면 계정의 지속적이고 자동화된 관리를 위해 정기 유지보수나 모니터링 스크립트에 API 호출을 통합하세요.

이러한 체계적인 접근 방식은 앱이 가볍고 반응성 있게 유지되어 가장 중요한 곳에 리소스를 제공하도록 보장합니다.

## **자주 묻는 질문**

**Q: 앱 제거 API란 무엇인가요?**

휴면 사용자 계정에서 개발자가 프로그래밍 방식으로 자신의 앱을 제거할 수 있게 해주는 Hyperclass 마켓플레이스 기능입니다.

**Q: 휴면 계정에서 앱을 제거하는 것이 제 애플리케이션에 어떤 도움이 되나요?**

불필요한 웹훅 이벤트를 최소화하여 오버헤드 비용을 줄이고 활성 계정의 지연 시간을 개선합니다.

**Q: 앱 제거 API에 대한 문서는 어디서 찾을 수 있나요?**

상세한 지침과 엔드포인트 정보는 [애플리케이션 제거](https://marketplace.gohighlevel.com/docs/ghl/marketplace/uninstall-application/index.html)의 Public API 문서에서 확인할 수 있습니다.

**Q: API를 자동화된 워크플로우에 통합할 수 있나요?**

네, 휴면 계정을 정기적으로 관리하기 위해 자동화된 스크립트에 API 호출을 포함시킬 수 있습니다.

**Q: 웹훅 이벤트와 관련된 향후 개선사항이 있나요?**

물론입니다. 4분기에는 개발자가 계정별로 웹훅 구독을 관리할 수 있게 해주는 추가 API가 제공되어 더 많은 유연성을 제공할 예정입니다.

**Q: 이 API를 사용하기 전에 필요한 전제조건은 무엇인가요?**

유효한 API 자격 증명이 있는지 확인하고 앱의 분석을 통해 휴면 계정을 식별할 수 있는 신뢰할 만한 방법을 갖추고 있어야 합니다.
