# 하이퍼클래스 웹훅(Webhook) 로그 대시보드 개발자 가이드

하이퍼클래스의 웹훅 로그 대시보드는 개발자에게 통합 연동에서 발송된 모든 아웃바운드 웹훅에 대한 완전한 가시성을 제공합니다. 페이로드, 타임스탬프, 시도 횟수, 응답 코드에 대한 실시간 접근을 통해 웹훅 문제를 디버깅할 때 지원 티켓을 생성할 필요가 없습니다. 개발자는 강력한 검색 및 필터링 도구를 사용하여 전송 상태를 빠르게 확인하고, 상세한 이벤트 기록을 검토하며, 실패 사례를 해결할 수 있습니다.

이 문서에서는 대시보드가 어떻게 작동하는지, 무엇을 확인할 수 있는지, 그리고 효과적으로 사용하는 방법을 설명합니다.

---

# 웹훅 로그 대시보드란?

웹훅 로그 대시보드는 **Developer Portal(개발자 포털) → Insights(인사이트) → Logs(로그)** 하위에 위치한 실시간 모니터링 도구입니다. 이 도구는 모든 아웃바운드 웹훅 이벤트와 해당 페이로드, 응답, 재시도 기록을 저장하여 개발자가 하이퍼클래스에서 전송된 내용을 독립적으로 검증할 수 있도록 합니다. 이벤트 활동을 확인하고, 통합 문제를 진단하며, 다운스트림 시스템이 예상된 데이터를 수신했는지 확인하는 데 필수적인 도구입니다.

---

## 웹훅 로그 대시보드의 주요 장점

대시보드의 장점을 이해하면 가시성 향상, 마찰 감소, 통합 속도 향상에 어떻게 도움이 되는지 알 수 있습니다. 각 기능은 특히 복잡한 이벤트 플로우를 진단할 때 개발자가 더 효율적으로 작업할 수 있도록 설계되었습니다.

- **즉시 전송 확인:** 하이퍼클래스가 웹훅을 전송했는지 또는 재시도했는지 즉시 확인할 수 있습니다.

- **완전한 페이로드 가시성:** 정확한 디버깅을 위해 완전한 JSON 페이로드와 헤더를 검토할 수 있습니다.

- **30일 로그 보관:** 확장된 조사를 위해 과거 로그에 대한 접근을 유지합니다.

- **페이로드 전체 부분 문자열 검색:** 이메일, 연락처 ID, 이벤트 이름 등 모든 값을 즉시 찾을 수 있습니다.

- **시간대 인식 타임스탬프:** 일관된 감사를 위해 원하는 시간대로 로그를 볼 수 있습니다.

- **지원 의존성 감소:** 지원 티켓을 열지 않고 내부적으로 문제 해결을 처리할 수 있습니다.

---

## 검색 및 필터링

효율적인 검색 도구는 개발자가 조사해야 하는 특정 웹훅 이벤트를 빠르게 좁혀나가는 데 도움을 줍니다. 단일 연락처 ID, 특정 이메일, 또는 특정 이벤트 유형과 연결된 웹훅을 찾든 상관없이, 검색 기능을 통해 문제를 추적하는 것이 간단해집니다.

전역 검색창은 페이로드 전체에서 부분 문자열 일치를 지원하여 부분 값을 사용해서 이벤트를 찾을 수 있습니다. 추가 필터는 다음 기준으로 결과를 정제합니다:

- 웹훅 ID
- 이벤트 유형
- 시간 범위
- 상태 코드

---

## 상세 패널

상세 패널은 특정 웹훅 이벤트에 대한 더 깊은 뷰를 제공하여 개발자가 처리를 검증하고 문제를 해결하는 데 필요한 정보를 제공합니다. 이 확장된 뷰를 통해 데이터를 한 줄씩 쉽게 검사할 수 있습니다.

로그 항목을 선택하면 다음을 포함하는 패널이 열립니다:

- 웹훅 ID
- 시도 번호
- 이벤트 이름
- 타임스탬프 기록 (생성, 전송, 재시도)
- HTTP 응답 코드 및 전송 상태
- 원클릭 복사 옵션이 있는 전체 JSON 페이로드

개발자는 페이로드를 Postman이나 Webhook.site 같은 도구에 붙여넣어 재실행하거나 시뮬레이션할 수 있습니다.

## 시간대 제어

친숙한 시간대로 타임스탬프를 보는 것은 이벤트를 진단하거나 팀 간 협업할 때 혼란을 줄이는 데 도움이 됩니다. 대시보드는 모든 사용자가 자신만의 시간대 기본 설정을 선택할 수 있도록 합니다.

웹훅 로그 대시보드는 기본적으로 중부 시간(Central Time)으로 설정되어 있지만, 페이지 상단의 선택기를 사용하여 원하는 시간대로 조정할 수 있습니다. 하이퍼클래스는 향후 모든 세션에서 여러분의 선택을 기억하여 로그 검토를 일관되고 직관적으로 유지합니다.

---

## 보관 기간

웹훅 로그는 30일 동안 자동으로 저장되어 개발자에게 전송 상태를 검토하고 문제를 해결할 충분한 시간을 제공합니다. 이 보관 기간은 최근 활동과 확장된 디버깅 프로세스 모두에 대한 가시성을 보장합니다.

30일 후에는 로그가 영구적으로 삭제됩니다. 개발자는 장기간 감사나 컴플라이언스를 위해 보관하고 싶은 페이로드를 저장하거나 복사해야 합니다.

## 웹훅 로그 대시보드 설정 방법

웹훅 로그 대시보드에 접근하고 사용하면 웹훅 활동을 즉시 확인하고 문제 해결을 시작할 수 있습니다. 애플리케이션에서 생성된 이벤트를 검토하기 시작하려면 아래 단계를 따르세요.

**marketplace.Hyperclass.com**에 로그인하고 **Developer Portal(개발자 포털) → Insights(인사이트) → Logs(로그)**로 이동합니다.

**Webhook Logs(웹훅 로그)** 탭을 선택합니다.

![웹훅 로그 탭](https://s3.amazonaws.com/cdn.freshdesk.com/data/helpdesk/attachments/production/155059717274/original/3s7evYSgDhAUXuTBODI_LUdMv1T8O7rE7g.png?1764686192)

(선택사항) 우측 상단에서 현지 시간에 맞게 시간대를 조정합니다.

앱에서 이벤트를 트리거하여 테스트 웹훅을 생성합니다.

검색창에 고유 식별자(예: 이메일)를 입력하여 이벤트를 찾습니다.

이벤트 행을 클릭하여 상세 패널을 열고 페이로드와 응답을 검토합니다.

추가 분석이나 팀과 공유를 위해 **Copy(복사)** 버튼을 사용합니다.

디버깅 협업 시 필요에 따라 웹훅 ID나 페이로드를 제공합니다.

## 자주 묻는 질문

**Q: 대시보드에 실패한 전송 시도가 표시되나요?**

네. 실패와 재시도를 포함한 각 시도가 응답 코드와 타임스탬프와 함께 나타납니다.

**Q: 로그를 내보낼 수 있나요?**

직접 내보내기는 불가능하지만, 개별 페이로드를 복사하거나 API를 통해 웹훅 데이터를 검색할 수 있습니다.

**Q: 30일 보관 기간이 지나면 어떻게 되나요?**

30일이 지난 로그는 영구적으로 삭제됩니다.

**Q: 시간대를 변경하면 다른 사용자에게 영향을 주나요?**

아니요. 시간대 선택은 개별 사용자별로 저장됩니다.

**Q: 하위 계정(Sub-account)이 별도의 대시보드를 가지나요?**

대시보드는 여러분의 개발자 계정 및 애플리케이션과 연결된 웹훅 이벤트를 표시합니다.

**Q: 페이로드 데이터가 마스킹되거나 잘리나요?**

아니요. 페이로드는 모든 필드를 포함하여 전송된 그대로 정확히 나타납니다.

**Q: 개발자 액세스 권한이 없는 에이전시 관리자가 이 로그를 볼 수 있나요?**

Developer Portal(개발자 포털) 권한이 있는 사용자만 웹훅 로그 대시보드에 접근할 수 있습니다.

**Q: 로그 보기가 API 속도 제한에 포함되나요?**

아니요. 로그 접근은 API 호출을 소모하지 않습니다.
