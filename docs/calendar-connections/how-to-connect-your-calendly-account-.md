# Calendly 계정 연결하는 방법

Calendly를 연결하면 예약된 이벤트가 캘린더 시스템에 자동으로 예약으로 표시됩니다. 연결 후에는 새로운 예약이 실시간으로 동기화되어 일정을 정확하게 유지하고, 수동 작업을 줄이며, 고객 정보가 일관되게 캡처되도록 도와줍니다.

이 연동은 간편성과 안정성을 고려해 설계되어, 추가 단계 없이도 예약 프로세스가 체계적으로 유지됩니다.

## Calendly 계정 연결에 필요한 사항

Calendly 계정에 대한 액세스 권한과 본인이 속한 캘린더가 최소 하나 이상 필요합니다. 선택된 캘린더에서 생성된 이벤트만 동기화됩니다.

## 이벤트 동기화 작동 방식

Calendly 이벤트는 일방향 흐름으로 가져옵니다.

- Calendly에서 시스템으로 이벤트가 가져옵니다
- 변경사항은 Calendly에서 직접 수정해야 합니다
- 동기화된 후에는 가져온 이벤트를 편집할 수 없습니다

동기화된 각 이벤트는 예약으로 처리됩니다. 참가자 정보가 포함되어 있으면 연락처 레코드가 자동으로 생성되어 후속 조치가 더 쉬워집니다.

캘린더가 제거되었다가 나중에 다시 추가되면 새로 생성된 이벤트만 동기화됩니다.

## 제한사항

- 사용자당 하나의 Calendly 계정만 연결할 수 있습니다
- 동일한 Calendly 계정을 여러 사용자가 동시에 공유할 수 없습니다
- 이벤트 동기화는 일방향만 지원됩니다
- 과거 이벤트는 소급 적용되지 않습니다

## 자주 묻는 질문

**Q: 계정 연결 후 어떤 Calendly 이벤트가 동기화되나요?**

연결이 완료된 후에 생성된 이벤트만 동기화됩니다. 연결 이전에 생성된 이벤트는 가져오지 않습니다.

**Q: 동기화할 Calendly 캘린더나 이벤트 유형을 선택할 수 있나요?**

네. 동기화할 캘린더나 이벤트 유형을 선택할 수 있으며, 언제든지 선택사항을 업데이트할 수 있습니다.

**Q: 캘린더를 제거하거나 Calendly 연동을 해제하면 어떻게 되나요?**

새 이벤트 동기화가 중단됩니다. 이전에 동기화된 이벤트를 삭제하라는 메시지가 나타날 수도 있으며, 삭제된 이벤트는 복원할 수 없습니다.

**Q: 동기화된 이벤트가 표시된 후에 편집할 수 있나요?**

아니오. 동기화된 이벤트는 표시된 후에 편집할 수 없습니다. 정확성을 유지하려면 Calendly에서 직접 변경사항을 수정해야 합니다.

## 더 많은 도움이 필요하신가요?

앱 내 도움말 검색을 사용하여 캘린더 연결, 예약 동기화, 스케줄링 동작에 관련된 문서를 탐색하거나, 추가 지원이 필요하시면 지원팀에 문의하세요.
