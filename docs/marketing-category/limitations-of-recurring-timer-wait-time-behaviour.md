# 반복 타이머 대기 시간 동작의 제한사항

이 가이드는 이메일 캠페인과 퍼널에서 반복 카운트다운 타이머가 어떻게 작동하는지, 특히 대기(Wait), 활성(Active), 만료(Expired) 상태와 관련된 동작을 설명합니다.

## 반복 타이머란 무엇인가요?

반복 타이머는 고정된 일정(예: 매일, 매주)에 따라 재설정되는 카운트다운 메커니즘입니다. 영구적으로 만료되는 일회성 타이머와 달리, 반복 타이머는 세 가지 상태를 순환합니다:

- **대기(Wait)** – 활성 카운트다운이 시작되기 전
- **활성(Active)** – 카운트다운이 표시되고 작동 중
- **만료(Expired)** – 카운트다운이 끝나고 리다이렉트되거나 콘텐츠가 숨겨짐

### 반복 타이머는 언제 사용해야 하나요?

다음과 같은 경우에 반복 타이머를 사용하세요:

- 정기적으로 긴급감을 조성하고 싶을 때 (예: "매일 2시간 후 혜택 종료")
- 매일, 매주, 매월 리셋되는 시간 제한 프로모션이 있을 때
- 일관되고 예측 가능한 동작이 필요한 자동화 캠페인을 운영할 때 (예: 플래시 세일, 일일 웨비나 카운트다운, 강의 수강 가능 시간)

💡 **이상적인 용도**: 에버그린 퍼널, 자동화된 판매 시퀀스, 일일 한정 혜택, 반복 이벤트 프로모션

## 이메일 동작

### 타이머 상태:

- **대기**: 카운트다운이 아직 시작되지 않음
- **활성**: 타이머가 카운트다운 중
- **만료**: 카운트다운이 완료됨

### 타이머 상태에 따른 동작:

| 타이머 상태 | 사용자가 이메일 링크 클릭 → | 리다이렉트 페이지 |
|-----------|------------------------|-----------------|
| 대기 | 동적 링크 클릭됨 | 만료 페이지 |
| 활성 | 동적 링크 클릭됨 | 활성 페이지 |
| 만료 | 동적 링크 클릭됨 | 만료 페이지 |

⚠️ **주의사항**: 이메일에서는 타이머가 대기 또는 만료 상태일 때 사용자가 활성 페이지에 접근할 수 없습니다. 활성 기간 중에만 사용자가 올바르게 리다이렉트됩니다.

## 퍼널 동작

현재 퍼널은 다소 다른 동작을 보입니다.

### 케이스 1: 활성 페이지에 있는 동안 타이머가 만료됨

- **동작**: 타이머가 만료되면 사용자가 자동으로 만료 페이지로 리다이렉트됩니다.

### 케이스 2: 활성 페이지에 있는 동안 타이머가 대기 상태로 전환됨

- **동작**: 사용자는 활성 페이지에 그대로 남아 있습니다.
- **선택사항**: 타이머 설정 시 "대기 상태에서 타이머 숨기기" 체크박스를 사용하여 대기 상태 동안 타이머 표시를 비활성화할 수 있습니다.

✅ **모범 사례**: "대기 상태에서 타이머 숨기기" 옵션을 활성화하여 활성화 전 작동하지 않는 타이머로 사용자가 혼란스러워하는 것을 방지하세요.

## 요약: 주요 제한사항

- **이메일**: 클릭 시점의 타이머 상태에 따라 리다이렉션이 엄격하게 결정됩니다.
- **퍼널**: 이미 페이지에 있는 사용자는 실시간으로 타이머 전환의 영향을 받으며, 대기 기간 동안 제어된 동작을 보입니다.

## 추가 팁

- 캠페인 타이밍과 일치하도록 타이머 시작 및 만료 시간을 신중하게 계획하세요.
- 퍼널의 현재 제한사항으로 인한 예상 동작을 보장하기 위해 세 가지 타이머 상태 모두에서 퍼널 동작을 항상 테스트하세요.
