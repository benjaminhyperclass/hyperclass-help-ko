# 스키마 마크업 생성기(Schema Markup Generator)

스키마 마크업 생성기는 웹사이트 및 퍼널 빌더에 내장된 도구로, 페이지에 대한 스키마 마크업(JSON-LD)을 생성하고 검증하며 게시할 수 있게 해줍니다. 이 글에서는 이 기능이 무엇인지, 왜 중요한지, 그리고 웹사이트와 퍼널에 스키마를 추가하고 관리하는 방법을 설명합니다.

## 스키마 마크업 생성기란 무엇인가요?

스키마 마크업 생성기는 웹사이트 및 퍼널 빌더에 내장된 도구로, 개별 페이지에 대한 스키마 마크업(구조화된 데이터)을 생성하고 검증하며 게시할 수 있게 해줍니다. 구조화된 데이터는 검색 엔진과 AI 시스템이 비즈니스 유형, 상품, FAQ, 기사, 이벤트 등의 핵심 정보를 해석하는 데 도움을 줍니다.

참고: 스키마는 페이지별로 적용되며 다른 페이지나 퍼널 단계로 이어지지 않습니다.

## 스키마 마크업 생성기의 주요 장점

- **140가지 이상의 스키마 유형:** LocalBusiness, Product, Article/BlogPosting, FAQPage, Event, Organization 등 일반적인 스키마 유형을 포괄합니다.

- **폼 보기(노코드):** 안내 필드로 기술적 지식 없이도 쉽게 스키마 세부사항을 완성할 수 있습니다.

- **JSON 보기(고급):** 완전한 제어가 필요할 때 JSON-LD를 직접 붙여넣거나 작성할 수 있습니다.

- **내장 검증 기능:** 저장 시 문제를 강조해서 게시 전에 스키마 문제를 수정할 수 있습니다.

- **원클릭 저장 및 게시 흐름:** 스키마가 페이지 설정에 저장되고 페이지 게시 시 실시간으로 적용됩니다.

- **페이지당 여러 스키마:** 같은 페이지에 하나 이상의 스키마 유형을 추가하고 관리할 수 있습니다(예: LocalBusiness + FAQPage).

## AI로 스키마 마크업을 생성하는 방법

AI 기반 스키마 생성을 통해 페이지 콘텐츠를 기반으로 구조화된 데이터를 자동으로 생성할 수 있습니다. 이는 검색 가시성을 향상시키고 수동 설정 없이 정확한 스키마를 보장하는 데 도움이 됩니다.

스키마 마크업을 생성하기 전에 페이지를 올바르게 준비하는 것이 중요합니다. AI는 기존 콘텐츠에 의존해서 적절한 스키마 유형과 값을 결정하므로, 불완전한 페이지는 부정확하거나 제한적인 스키마 출력을 가져올 수 있습니다.

### 시작하기 전에

스키마 마크업을 생성하기 전에:

- 페이지 콘텐츠가 완전히 완성되었는지 확인하세요
- AI는 기존 콘텐츠를 분석해서 정확한 스키마를 생성합니다
- 스키마는 페이지별로 적용되며 다른 페이지로 이어지지 않습니다

1. **Sites(사이트)**로 이동합니다.

2. **Funnels(퍼널)** 또는 **Websites(웹사이트)** 중 스키마를 추가하려는 항목을 클릭합니다.

## 수동 방식에서 지원되는 스키마 유형

스키마 마크업 생성기는 다양한 스키마 유형을 지원하므로 페이지 목표에 맞춰 구조화된 데이터를 매칭할 수 있습니다. 올바른 스키마 유형을 사용하면 검색 엔진과 AI 시스템의 명확성을 개선하고 해당되는 경우 향상된 결과에 대한 페이지 자격을 얻는 데 도움이 됩니다.

일반적으로 지원되는 유형은 다음과 같습니다:

- Organization
- Website
- Breadcrumb List
- Article / Blog Posting
- FAQ Page
- QA Page
- How To
- Video Object
- Person
- Product
- Local Business
- Event

참고: 여기에는 각 카테고리 하위의 하위 유형이 포함됩니다(예: LocalBusiness 하위 유형).

## 중요한 참고사항

- 빈 페이지에 대해서는 스키마를 생성할 수 없습니다
- 저장하기 전에 항상 AI가 생성한 스키마를 검토하세요
- 각 페이지에는 자체 스키마 설정이 필요합니다
- 페이지 콘텐츠를 업데이트하면 스키마를 재생성해야 할 수 있습니다
- 스키마는 게시 후에만 적용됩니다

## 스키마 보기: 폼 보기 vs JSON 보기

올바른 편집 보기를 선택하면 더 빠르게 작업하고 오류를 줄이는 데 도움이 됩니다. 폼 보기는 대부분의 사용자에게 이상적이며, JSON 보기는 다른 소스에서 스키마를 붙여넣거나 고급 필드를 사용자 정의해야 할 때 최적입니다.

#### **폼 보기**

- 노코드 설정에 최적
- 안내 필드를 사용해서 필수 및 권장 속성을 완성
- 형식 실수를 방지하는 데 도움

#### **JSON 보기**

- 고급 사용자와 개발자에게 최적
- 직접적인 JSON-LD 편집 또는 기존 스키마 붙여넣기 허용
- 폼 보기에서 표시되지 않는 전문 속성 추가에 유용

## 성공을 위한 모범 사례

- **정확성이 중요합니다:** 페이지에서 볼 수 있는 정보만 마크업하세요. 가짜 리뷰나 숨겨진 콘텐츠를 절대 추가하지 마세요.

- **최신 상태로 유지:** 영업 시간, 가격, 연락처 정보가 변경되면 스키마를 업데이트하세요.

- **구체적인 유형 사용:** 가능하면 일반적인 "LocalBusiness" 대신 "Dentist"를 선택하세요. 더 구체적일수록 좋습니다.

- **여러 스키마 결합:** 홈페이지에는 포괄적인 커버리지를 위해 Organization과 LocalBusiness 스키마를 모두 포함할 수 있습니다.

- **모든 핵심 페이지에 스키마 추가:** 홈페이지, 연락처 페이지, 상품/서비스 페이지, 블로그 게시물 모두 적절한 스키마 마크업의 혜택을 받습니다.

## 지역 비즈니스에 가장 유용한 스키마 유형

- **Local Business:** 물리적 위치가 있는 모든 비즈니스에 필수. 이름, 주소, 전화번호, 영업 시간, 서비스 지역을 포함합니다.

- **Product:** 판매하는 상품용. 검색 결과에 가격, 재고 상태, 평점, 이미지를 직접 표시합니다.

- **Service:** 제공하는 서비스용. 서비스 설명, 가격, 서비스 지역을 표시합니다.

- **Review:** 검색 결과에 별점으로 나타나는 고객 증언(실제 리뷰여야 함).

- **FAQ:** 검색 결과에서 직접 확장되는 질문과 답변.

- **Event:** 워크샵, 세일, 클래스 또는 모든 예정된 행사. 날짜와 등록 링크를 표시합니다.

## 참고사항

- 스키마는 페이지별로 적용되며 자동으로 다른 페이지나 퍼널 단계로 이어지지 않습니다.

- 스키마는 저장 후 편집기나 미리보기에서 나타날 수 있지만, 페이지 게시 후에만 실시간으로 적용됩니다.

- AI로 새 페이지를 생성할 때 스키마가 자동 생성되지 않으므로, 콘텐츠 완성 후 수동으로 추가하세요.

- 보이는 세부사항이 변경되면 스키마를 업데이트하세요(예: 영업 시간, 가격, 연락처 정보).

## 자주 묻는 질문

**Q: 수동 설정과 AI 생성 스키마 간에 스키마 선택이 어떻게 다른가요?**

수동 설정에서는 사용 가능한 스키마 유형이 미리 정의된 목록으로 제한됩니다. 보통 SEO, AEO(답변 엔진 최적화), GEO(생성 엔진 최적화)에 가장 일반적으로 사용되는 것들입니다.

AI의 경우에는 그런 제한이 없습니다. AI는 페이지 콘텐츠를 기반으로 모든 유효한 스키마 유형을 생성할 수 있습니다.

**Q: AI가 수동 목록에서 사용할 수 없는 스키마 유형을 생성할 수 있나요?**

네. AI는 페이지 콘텐츠를 분석하고 수동 선택 옵션에 포함되지 않은 유효한 스키마 유형이라도 사용해서 스키마 마크업을 생성할 수 있습니다.

**Q: 스키마 마크업 생성기를 사용하는 데 기술적 지식이 필요한가요?**

아니요. 폼 보기는 노코드 설정을 위해 설계되었습니다. 고급 사용자는 JSON 보기를 사용해서 JSON-LD를 붙여넣거나 사용자 정의할 수 있습니다.

**Q: 스키마가 내 페이지의 어디에 게시되나요?**

스키마는 페이지가 게시될 때 페이지의 <head>에 JSON-LD로 게시됩니다.

**Q: 미리보기에서는 스키마가 보이는데 실시간 페이지에서는 보이지 않는 이유는 무엇인가요?**

저장은 페이지에 스키마를 저장하지만, 페이지나 퍼널 단계를 게시한 후에만 실시간으로 적용됩니다.

**Q: 한 페이지에 하나 이상의 스키마 유형을 추가할 수 있나요?**

네. 페이지당 여러 스키마를 추가하고 관리할 수 있습니다(예: LocalBusiness + FAQPage).

**Q: 다른 도구나 웹사이트에서 스키마를 붙여넣을 수 있나요?**

네. JSON 보기를 사용해서 JSON-LD를 붙여넣은 다음, Validate & Save를 클릭해서 게시 전에 문제가 있는지 확인하세요.

**Q: "Validate & Save"는 무엇을 확인하나요?**

검증은 누락되거나 잘못된 필드와 형식 문제를 식별해서 게시 전에 문제를 수정할 수 있도록 도움을 줍니다.

**Q: 내 스키마가 페이지에서 보이는 것과 일치해야 하나요?**

네. 오해를 불러일으키는 마크업을 피하기 위해 사용자가 볼 수 있는 실제 콘텐츠를 반영하는 구조화된 데이터만 추가하세요.

**Q: 페이지 콘텐츠 없이 스키마를 생성할 수 있나요?**

아니요. AI가 정확한 스키마를 생성하려면 페이지에 콘텐츠가 있어야 합니다.

**Q: 생성된 스키마를 편집할 수 있나요?**

네. 저장하기 전에 폼 보기와 JSON 보기 모두에서 스키마를 수정할 수 있습니다.
